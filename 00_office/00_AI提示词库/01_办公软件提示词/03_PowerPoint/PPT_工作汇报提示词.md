# PPT 工作汇报提示词

## 适用场景

用于制作项目总结、周报月报、年度总结、业务汇报、领导汇报等可编辑 PPT。

## 推荐工具

Codex、PowerPoint、`$presentations skill`

## 使用前准备

- 准备汇报周期、汇报对象和汇报目的。
- 准备关键数据、项目进展、问题、下一步计划。
- 说明公司或团队常用风格。

## 完整提示词

```text
请使用 $presentations skill 帮我制作一份工作汇报类可编辑 PowerPoint PPTX。

汇报目标：
把我的工作材料整理成一份逻辑清晰、重点明确、适合向领导或团队展示的 PPT。

硬性要求：
1. 输出必须是可编辑 PPTX。
2. 不要整页图片。
3. 所有标题、正文、数据说明、结论文字都必须是 PowerPoint 可编辑文本框。
4. 流程图、进度图、问题归因图、计划路线图尽量使用 PowerPoint 原生形状和连接线。
5. 数据表必须使用 PowerPoint 原生表格。
6. 可以使用图标和少量图片，但不能把页面内容做成不可编辑图片。
7. 最后必须做排版 QA 检查并修复问题。

推荐结构：
1. 封面：汇报主题、汇报人、日期
2. 目录：本次汇报结构
3. 核心结论：先给领导最关心的结论
4. 工作完成情况：完成事项、关键成果、数据表现
5. 重点项目进展：里程碑、当前状态、下一节点
6. 问题与风险：问题、影响、原因、解决方案
7. 下一步计划：时间表、负责人、交付物
8. 需要支持：资源、决策、协作事项

制作流程：
第一步：先根据材料输出 PPT 大纲。
第二步：先做 1 页样张，建议优先做“核心结论页”或“项目进展页”。
第三步：等我确认样张风格后，再生成完整 PPTX。
第四步：生成完整 PPT 后，做排版 QA 检查：
- 文字是否溢出
- 是否有元素遮挡
- 数据表是否可读
- 图形和文本是否对齐
- 页面标题、页码、配色是否一致
- 是否有过密页面需要拆分

汇报类型：
【周报 / 月报 / 年度总结 / 项目复盘 / 业务汇报 / 其他】

汇报对象：
【领导 / 客户 / 团队 / 跨部门】

页数要求：
【填写页数】

视觉风格：
【简洁商务 / 稳重专业 / 数据化 / 公司模板风格】

已有材料：
【粘贴工作内容、数据、问题、计划】

请先输出 PPT 大纲，不要直接生成完整 PPT。
```

## 使用方法

把原始工作材料粘进去，让 Codex 先整理出汇报逻辑。样张确认后再生成完整 PPTX。
