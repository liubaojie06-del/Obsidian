# 工具网站整理提示词

## 适用场景

用于把 PDF、图片、图表、开发、电气、素材、接码等工具网站整理进 Obsidian 网站管理目录。

## 对应笔记类型

工具网站二级分类页、具体网站说明页、网站资料卡片。

## 使用前准备

- 提供网站名称、链接、用途、备注。
- 提供当前 `网站管理/工具网站` 的目录结构。
- 说明是否需要新建分类。

## 完整提示词

```text
请帮我把下面网站整理进 Obsidian 的“网站管理 / 工具网站”体系。

必须遵守：
- 先扫描或参考当前目录结构和已有页面风格。
- 不删除已有内容。
- 不重命名已有文件。
- 不覆盖同名文件。
- 添加前先输出准备修改的文件清单和方案，等我确认后再执行。
- 总览页只保留小卡片分类入口，不要堆全部网站。
- 二级分类页列出该分类下的网站。
- 具体网站说明可以放在二级页“详细说明”中；如果内容很多，再建议新建三级网站说明页。
- 网站条目必须包含：网站名称、官网链接、主要用途、适合场景、是否免费 / 是否需要登录、推荐程度、备注。
- 风格要沿用我现有蓝色卡片 / callout 页面，不要改成普通散乱列表。

请按下面分类规则归类：
- PDF、Word、Excel、文件转换、OCR -> [[PDF_文件处理]]
- 图片压缩、抠图、设计、海报、配色 -> [[图片_设计工具]]
- 流程图、思维导图、架构图、白板 -> [[图表_流程图]]
- 代码、API、正则、JSON、开发文档 -> [[编程开发工具]]
- 电路、电子元件、PCB、电气计算 -> [[电气_硬件工具]]
- 图标、插画、素材、字体、SVG -> [[图标_素材工具]]
- 接码、账号辅助、临时手机号 -> 建议新增 [[账号_接码工具]]

二级分类页面请使用下面格式：

# 分类名称

> 一句话说明这个分类整理哪些工具。

## 网站列表

| 网站 | 用途 | 免费情况 | 推荐程度 | 链接 |
|---|---|---|---|---|
| 网站名称 | 主要用途 | 免费 / 部分免费 / 付费 / 需登录 | ⭐⭐⭐⭐ | 官网链接 |

## 详细说明

### 网站名称

- 官网：官网链接
- 主要用途：
- 适合场景：
- 免费情况：
- 是否需要登录：
- 推荐程度：
- 备注：

请先输出：
1. 识别到的网站分类。
2. 准备修改或创建的文件。
3. 每个网站放入哪个分类。
4. 是否需要新增分类。
5. 需要我确认的事项。

等待我确认后再给出可写入的 Markdown 内容。
```

## 使用方法

复制提示词后，把网站列表粘贴到后面。适合每次新增一批工具网站前使用。

## 输出格式要求

- 先方案，后执行。
- 使用 Markdown 表格。
- 二级页保留“网站列表 + 详细说明”。
- 总览页只放卡片入口。
