# 计算机基础概念提示词

## 适用场景

用于整理 CPU、内存、总线、缓存、操作系统、编译流程等底层概念。

## 对应笔记类型

计算机组成原理、操作系统、编译原理、底层硬件互联。

## 使用前准备

- 提供概念名称。
- 提供参考资料。
- 说明希望偏理论、偏工程还是偏考试。

```text
3. 使用 `> [!abstract] 核心结论`。
4. 解释底层原理，不只写定义。
5. 用类比帮助理解。
6. 用文本结构图或表格展示组成关系。
7. 说明典型使用场景或工程影响。
8. 写常见误区。

- 不要写成百科词条。
- 每一节都要服务于理解和复习。
- 如果涉及硬件性能瓶颈，要说明瓶颈来自哪里。
```

## 使用方法

适合整理计算机组成原理、Linux 底层、C 语言内存模型等概念。

## 输出格式要求

- 必须有结构图或表格。
- 必须有类比和误区。
- 保持知识卡片风格。
