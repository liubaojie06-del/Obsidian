# PPT 排版检查提示词

## 适用场景

用于检查已有 PPTX 或生成后的 PPTX，重点发现文字溢出、遮挡、重叠、对齐、页码、表格和连接线问题，并在确认后修复。

## 推荐工具

- Codex
- `$presentations skill`
- PowerPoint

## 使用前准备

- 准备需要检查的 PPTX 文件路径。
- 明确是否允许修改内容，或只允许调整排版。
- 如有品牌模板、字体、颜色或页脚规则，提前说明。
- 指定修复后输出路径，避免覆盖原文件。

## 完整提示词

```text
请使用 $presentations skill 对我提供的 PowerPoint PPTX 做排版 QA 检查，并在我确认后输出修复版 PPTX。

待检查 PPTX：
检查范围：
是否允许改写文字：
是否允许调整页数：
品牌或模板规则：
修复后输出路径：

硬性要求：
1. 不要覆盖原文件，修复版另存为新的可编辑 PPTX。
2. 不要把页面导出成整页图片再放回 PPT。
3. 所有文字必须保持 PowerPoint 可编辑文本框。
4. 表格必须保持 PowerPoint 原生表格。
5. 流程图、连接关系、箭头和步骤图尽量保持或修复为 PowerPoint 原生形状和连接线。
6. 如发现文字已经是图片，请标记出来，并建议重建为可编辑文本框。

请按以下流程执行：
第一步：先输出排版 QA 大纲和检查计划。
- 说明会检查哪些项目：文字溢出、遮挡、重叠、标题换行、对齐、层级、表格截断、图表标签、连接线、页码页脚、占位符、可编辑性。
- 暂时不要直接生成最终修复版。

第二步：渲染并检查全部页面，输出问题清单。
- 按页码列出问题。
- 每个问题说明严重程度、影响和建议修复方式。
- 区分“必须修复”和“建议优化”。

第三步：我确认修复方向后，先修复 1 页样张 PPTX。
- 选择问题最典型的一页。
- 修复时保持文字可编辑、表格为原生表格、流程图为原生形状和连接线。
- 输出样张文件路径，并说明修复规则。

第四步：我确认样张后，再修复完整 PPTX。
- 按样张规则批量修复全稿。
- 不改变核心内容，除非我明确允许改写。
- 对每一页重新渲染检查。

第五步：最终 QA。
- 确认没有文字溢出、遮挡、异常重叠、表格截断、连接线穿字、标题意外换行、页码错误和未替换占位符。
- 确认可编辑性：文字不是图片，主要表格不是图片，流程图不是截图。
- 最后交付修复版 PPTX 文件路径，并附简短 QA 结果。
```

## 使用方法

1. 先复制 PPTX 文件路径，并明确是否允许改写内容。
2. 让 Codex 先输出问题清单，不要直接全量修复。
3. 确认典型页样张后，再批量修复全稿。
4. 最后用 QA 结果确认没有覆盖原文件。
