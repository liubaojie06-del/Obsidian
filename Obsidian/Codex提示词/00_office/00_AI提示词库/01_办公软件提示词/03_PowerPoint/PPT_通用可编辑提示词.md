# PPT 通用可编辑提示词

## 适用场景

用于从主题、资料或简短需求出发，生成一份结构完整、内容可编辑、适合继续修改的 PowerPoint PPTX。

## 推荐工具

- Codex
- `$presentations skill`
- PowerPoint

## 使用前准备

- 明确主题、受众、页数、语言和交付场景。
- 准备可引用的资料、数据、图片或参考风格。
- 如需套用模板，准备 PPTX 模板文件路径。
- 明确最终输出路径和文件名。

## 完整提示词

```text
请使用 $presentations skill 帮我制作一份可编辑的 PowerPoint PPTX。

主题：
受众：
使用场景：
页数：
语言：
风格偏好：
参考资料：
输出路径：

硬性要求：
1. 输出必须是可编辑 PPTX，不要输出整页图片或截图式幻灯片。
2. 所有正文、标题、页码、标签和注释都必须是 PowerPoint 可编辑文本框。
3. 流程图、时间线、组织结构图、步骤图尽量使用 PowerPoint 原生形状和连接线制作。
4. 表格必须使用 PowerPoint 原生表格，不要把表格做成图片。
5. 如需图片，只能作为插图、背景或局部视觉素材使用，不能承载主要文字内容。
6. 不要为了视觉效果牺牲可编辑性。

请按以下流程执行：
第一步：先给我完整 PPT 大纲。
- 列出每页页码、标题、核心信息、建议版式、需要的数据或素材。
- 标记哪些信息需要我补充或确认。
- 暂时不要生成完整 PPTX。

第二步：在我确认大纲后，只生成 1 页样张 PPTX。
- 请选择最能代表整体风格的一页。
- 样张也必须是可编辑 PPTX。
- 样张里的文字必须是可编辑文本框，流程图用原生形状和连接线，表格用原生表格。
- 生成后请告诉我样张文件路径，并说明这一页的设计规则。

第三步：在我确认样张后，再生成完整 PPTX。
- 保持样张的视觉风格、字号层级、间距和配色规则。
- 内容要覆盖已确认大纲中的所有页面。
- 若资料不足，请使用占位文本并清楚标注“待补充”，不要编造事实。

第四步：生成后做排版 QA 检查。
- 渲染并检查每一页。
- 修复文字溢出、遮挡、重叠、标题换行异常、对齐不齐、连接线穿过文字或形状、表格截断、页码不一致等问题。
- 确认没有未替换的占位符。
- 最后交付 PPTX 文件路径，并简要说明完成的 QA 项目。
```

## 使用方法

1. 先补齐主题、受众、页数、资料和输出路径。
2. 第一次发送后只让 Codex 输出大纲，不急着生成完整 PPT。
3. 确认大纲后，让 Codex 生成 1 页样张。
4. 确认样张后，再生成完整 PPTX 并做排版 QA。
