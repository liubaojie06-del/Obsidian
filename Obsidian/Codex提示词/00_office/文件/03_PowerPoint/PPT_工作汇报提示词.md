# PPT 工作汇报提示词

## 适用场景

用于项目总结、阶段复盘、领导汇报、业务经营分析、部门周报月报等工作汇报 PPT。

## 推荐工具

- Codex
- `$presentations skill`
- PowerPoint
- 可选：Excel 数据表、项目计划、会议纪要、已有汇报模板

## 使用前准备

- 准备汇报周期、汇报对象、项目背景、关键指标和结论。
- 准备真实数据来源，无法确认的数据先标记为“待核对”。
- 明确汇报目的：同步进展、争取资源、复盘问题、推动决策。
- 如有公司模板或品牌规范，准备对应 PPTX 文件路径。

## 完整提示词

```text
请使用 $presentations skill 帮我制作一份工作汇报 PowerPoint PPTX。

汇报主题：
汇报对象：
汇报周期：
汇报目的：
页数：
已有资料：
关键数据：
必须表达的结论：
公司模板或参考风格：
输出路径：

硬性要求：
1. 输出必须是可编辑 PPTX，不要整页图片。
2. 所有标题、正文、数据标签、结论、页码都必须是 PowerPoint 可编辑文本框。
3. 表格必须使用 PowerPoint 原生表格，不要截图 Excel 表。
4. 流程图、项目路径、问题闭环、里程碑尽量使用 PowerPoint 原生形状和连接线。
5. 图表优先使用可编辑图表或可继续维护的图形结构，不要用截图代替。
6. 不确定的数据、事实和归因必须标记“需核对”，不要编造。

请按以下流程执行：
第一步：先给出汇报大纲。
- 建议结构包括：封面、核心结论、进展概览、关键数据、问题与风险、解决动作、下一步计划、资源需求。
- 每页列出页码、标题、核心信息、推荐版式和需要补充的数据。
- 明确哪些页面适合表格、图表、流程图或时间线。
- 暂时不要生成完整 PPTX。

第二步：我确认大纲后，只生成 1 页样张 PPTX。
- 优先选择“核心结论页”或“关键数据页”作为样张。
- 样张必须可编辑：文字为文本框，表格为原生表格，流程或里程碑为原生形状和连接线。
- 样张要体现汇报风格：清楚、克制、便于领导快速抓重点。
- 生成后提供样张文件路径，并说明后续全稿会如何保持一致。

第三步：我确认样张后，再生成完整工作汇报 PPTX。
- 每页只保留对汇报目标有用的信息。
- 先给结论，再给证据，再给行动建议。
- 对项目风险要写清影响、原因、负责人和下一步。
- 对数据页要给出一句可直接汇报的结论，而不是只堆图表。

第四步：最终排版 QA 检查。
- 渲染并检查每一页。
- 修复文字溢出、遮挡、元素重叠、标题换行异常、表格截断、图表标签拥挤、连接线穿过文字或形状、页码不一致等问题。
- 确认没有整页图片，没有不可编辑的文字截图。
- 最后交付完整 PPTX 文件路径，并列出已完成的 QA 检查项。
```

## 使用方法

1. 把汇报对象和汇报目的写清楚，决定汇报是“同步信息”还是“推动决策”。
2. 先让 Codex 输出大纲，确认结构和逻辑。
3. 用核心结论页或关键数据页做样张。
4. 完整生成后检查数据准确性和排版 QA。
