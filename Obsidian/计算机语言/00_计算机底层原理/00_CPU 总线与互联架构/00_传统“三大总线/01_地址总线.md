# 01 | 地址总线：CPU 指定访问位置的电气通道

> [!abstract] 核心结论
> 地址总线不是抽象概念，而是一组真实的电路连线。每一根地址线表示地址中的 1 个二进制位，多根地址线组合起来，就形成一个地址编号，用来指定 CPU 当前要访问哪一个内存单元或外设寄存器。

### ① 底层原理：地址总线本质是一组并行地址线

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>地址总线的硬件本质：</strong>
<ul>
  <li><strong>一根地址线表示 1 bit：</strong> 高电平通常表示 <code>1</code>，低电平通常表示 <code>0</code>。</li>
  <li><strong>多根地址线组成地址：</strong> 16 根地址线可以表示 16 位地址，32 根地址线可以表示 32 位地址。</li>
  <li><strong>地址总线通常由 CPU 输出：</strong> CPU 把要访问的位置编号放到地址线上。</li>
  <li><strong>地址不是数据内容：</strong> 地址只是“位置编号”，表示要访问哪里。</li>
  <li><strong>接收端根据地址译码：</strong> 内存芯片或外设电路根据地址线组合判断自己是否被访问。</li>
</ul>
</div>

### ② 8 位地址总线示意

假设有 8 根地址线：

```text
A7 A6 A5 A4 A3 A2 A1 A0
```

某一时刻电平如下：

```text
A7 A6 A5 A4 A3 A2 A1 A0
 0  0  0  1  0  0  1  1
```

组合起来就是：

```text
00010011
```

对应十六进制：

```text
0x13
```

也就是说：

```text
CPU 不是发送“0x13”这个字符串。
而是让 8 根地址线同时呈现 00010011 这一组电平状态。
接收端把这组电平解释成地址 0x13。
```

> [!tip] 记忆口诀
> **一根地址线一位，多根地址线组成地址；电平组合不是数据，而是位置编号。**

# 02 | 地址总线宽度：决定最多能表示多少个地址

> [!abstract] 核心结论
> 地址总线宽度决定 CPU 最多能产生多少个不同地址。地址线越多，可表示的地址编号越多，理论可寻址空间越大。

### ① 地址线数量和地址数量

```text
n 根地址线可以表示 2^n 个不同地址。
```

例如：

```text
8 根地址线  →  2^8  = 256 个地址
16 根地址线 →  2^16 = 65536 个地址
20 根地址线 →  2^20 = 1048576 个地址
32 根地址线 →  2^32 = 4294967296 个地址
```

### ② 如果一个地址对应 1 字节

现代大多数系统按字节寻址，也就是：

```text
1 个地址编号对应 1 个字节空间。
```

那么：

```text
16 位地址总线：
2^16 个地址 = 65536 字节 = 64 KB

20 位地址总线：
2^20 个地址 = 1048576 字节 = 1 MB

32 位地址总线：
2^32 个地址 = 4 GB

64 位地址总线：
理论空间极大，但实际 CPU 和系统通常不会全部实现。
```

### ③ 地址总线宽度和寻址空间

```text
寻址空间 = 2^地址线数量 × 每个地址单元大小
```

如果按字节寻址：

```text
寻址空间 = 2^地址线数量 字节
```

示例：

```text
地址线数量 = 24
寻址空间 = 2^24 字节 = 16 MB
```

> [!tip] 记忆口诀
> **地址线越多，门牌号越多；n 根地址线，最多 2 的 n 次方个地址。**

# 03 | 地址不是数据：地址只是访问位置的编号

> [!abstract] 核心结论
> 地址总线上的二进制值不是要写入内存的数据，而是 CPU 要访问的位置编号。真正的数据内容会出现在对应的数据通道中，而地址只负责“选中位置”。

### ① 地址和值的区别

示例代码：

```c
int a = 10;
```

可以理解为：

```text
变量 a 存在某个内存地址中。
这个地址可能是 0x1000。
这个地址里面保存的值是 10。
```

区别：

```text
0x1000 是地址。
10 是数据。
```

### ② CPU 访问变量时的理解

当 CPU 要读取变量 `a`：

```text
1. CPU 得到变量 a 的地址。
2. CPU 把这个地址放到地址线上。
3. 内存系统根据地址找到对应存储单元。
4. 该存储单元中的内容被读取出来。
```

也就是说：

```text
地址总线上的内容：0x1000
内存单元中的内容：10
```

### ③ 一个地址对应一块存储单元

如果按字节寻址：

```text
地址 0x1000 → 1 个字节
地址 0x1001 → 下一个字节
地址 0x1002 → 再下一个字节
地址 0x1003 → 再下一个字节
```

如果读取一个 32 位整数：

```text
int 常见占 4 字节。
可能需要访问 0x1000、0x1001、0x1002、0x1003 这 4 个连续地址。
```

> [!tip] 记忆口诀
> **地址是门牌号，数据是屋里的东西；CPU 先给门牌，再取屋里的内容。**

# 04 | 地址译码：地址线如何选中具体芯片或存储单元

> [!abstract] 核心结论
> 地址总线上的二进制地址不会自动“找到”设备，硬件中需要地址译码电路。译码电路根据地址线组合，判断应该选中哪一片内存、哪一个外设、哪一个寄存器。

### ① 什么是地址译码

```text
地址译码 = 把地址线上的二进制组合，翻译成具体的选通信号。
```

例如：

```text
CPU 输出地址：0x20000010
译码电路判断：
这个地址属于某个外设寄存器范围。
于是选中对应外设。
```

### ② 简单译码示意

假设用高 2 位地址线选择 4 个设备：

```text
A15 A14 = 00 → 选中 ROM
A15 A14 = 01 → 选中 RAM
A15 A14 = 10 → 选中 外设 1
A15 A14 = 11 → 选中 外设 2
```

图示：

```text
A15 A14
 0   0   → ROM 片选有效
 0   1   → RAM 片选有效
 1   0   → IO1 片选有效
 1   1   → IO2 片选有效
```

### ③ 片选信号

```text
片选信号 Chip Select，常写作 CS。
```

作用：

```text
只有被片选的芯片才响应当前访问。
没有被片选的芯片不参与本次传输。
```

例如：

```text
CPU 发出地址。
译码电路判断地址属于 RAM。
RAM 的 CS 有效。
ROM 和其他外设保持不响应。
```

### ④ 为什么需要地址译码

如果没有译码：

```text
多个设备可能同时以为 CPU 在访问自己。
这样会造成响应混乱。
```

有了译码：

```text
同一个地址范围只对应一个目标设备。
访问路径清晰。
系统才能稳定工作。
```

> [!tip] 记忆口诀
> **地址线给编号，译码器来翻译；片选谁有效，谁就被访问。**

# 05 | 地址映射：某一段地址对应某个硬件区域

> [!abstract] 核心结论
> CPU 看到的是一整片地址空间。不同地址范围会被硬件映射到不同区域，例如 Flash、RAM、外设寄存器、显存、启动 ROM 等。

### ① 什么是地址空间

```text
地址空间 = CPU 能产生的所有地址编号集合。
```

例如 32 位地址空间：

```text
0x00000000 到 0xFFFFFFFF
```

理论上一共有：

```text
4 GB 地址范围
```

### ② 地址映射示例

一个嵌入式系统可能这样划分：

```text
0x00000000 ~ 0x0007FFFF  → Flash
0x20000000 ~ 0x2000FFFF  → SRAM
0x40000000 ~ 0x4000FFFF  → 外设寄存器
0xE0000000 ~ 0xE00FFFFF  → 内核控制寄存器
```

也就是说：

```text
访问 0x20000000，可能是在访问 SRAM。
访问 0x40000000，可能是在访问某个外设寄存器。
```

### ③ 内存映射 I/O

```text
Memory-Mapped I/O，内存映射输入输出。
```

含义：

```text
外设寄存器被映射到普通地址空间中。
CPU 像访问内存一样访问外设。
```

示例理解：

```c
#define GPIO_REG (*(volatile unsigned int*)0x40000000)

GPIO_REG = 1;
```

底层含义：

```text
CPU 访问地址 0x40000000。
这个地址不是普通 RAM。
它对应 GPIO 外设的某个寄存器。
写入 1 可能会让某个引脚输出高电平。
```

### ④ 为什么同样是地址，结果不同

因为：

```text
地址只是编号。
这个编号最终连到哪里，由硬件地址映射决定。
```

所以：

```text
0x20000000 可能是 RAM。
0x40000000 可能是外设。
0x08000000 可能是 Flash。
```

> [!tip] 记忆口诀
> **CPU 只会发地址，地址背后接什么，由硬件映射决定。**

# 06 | 地址对齐：为什么某些数据要放在特定地址

> [!abstract] 核心结论
> 地址不仅要正确，还经常要满足对齐要求。多字节数据如果放在合适的边界上，CPU 访问更快；如果不对齐，可能变慢，甚至在某些架构上直接出错。

### ① 什么是地址对齐

```text
地址对齐 = 数据存放地址是某个字节数的整数倍。
```

例如：

```text
4 字节 int 最好放在 4 的倍数地址上。
```

常见：

```text
地址 0x1000：能被 4 整除，对齐。
地址 0x1004：能被 4 整除，对齐。
地址 0x1002：不能被 4 整除，对 int 来说不对齐。
```

### ② 为什么需要对齐

假设 CPU 一次更适合按 4 字节读取。

如果 int 放在：

```text
0x1000 ~ 0x1003
```

CPU 可能一次就能读完。

如果 int 放在：

```text
0x1002 ~ 0x1005
```

它跨过了边界，可能需要两次读取再拼接。

### ③ 对齐访问示意

对齐：

```text
地址：
0x1000 0x1001 0x1002 0x1003
┌──────┬──────┬──────┬──────┐
│      一个 int，4 字节       │
└──────┴──────┴──────┴──────┘
```

不对齐：

```text
地址：
0x1000 0x1001 0x1002 0x1003 0x1004 0x1005
              ┌──────┬──────┬──────┬──────┐
              │      一个 int，4 字节       │
              └──────┴──────┴──────┴──────┘
```

问题：

```text
数据跨边界。
硬件可能需要额外周期。
某些 CPU 甚至不允许这种访问。
```

### ④ 结构体对齐

示例：

```c
struct A {
    char c;
    int i;
};
```

虽然：

```text
char 占 1 字节。
int 常见占 4 字节。
理论相加是 5 字节。
```

但实际可能是：

```text
sizeof(struct A) = 8
```

原因：

```text
编译器会插入填充字节，让 int 成员放在更适合访问的地址边界上。
```

> [!tip] 记忆口诀
> **地址不是随便放，对齐访问更顺畅；不对齐可能变慢，严重还会异常。**

# 07 | 地址总线与指针：C 语言地址最终会变成硬件地址访问

> [!abstract] 核心结论
> C 语言中的指针保存地址。程序访问 `*p` 时，底层会生成内存访问指令，CPU 根据这个地址去访问对应内存位置。

### ① 指针保存地址

```c
int a = 10;
int *p = &a;
```

理解：

```text
a 是变量。
&a 是 a 的地址。
p 保存这个地址。
```

### ② 解引用时发生什么

```c
int b = *p;
```

底层可以理解为：

```text
1. CPU 读取 p 里面保存的地址。
2. CPU 根据这个地址发起一次内存读取。
3. 读到该地址中的内容。
4. 把内容放入 b。
```

### ③ 写入指针指向的地址

```c
*p = 20;
```

底层可以理解为：

```text
1. CPU 读取 p 中保存的地址。
2. CPU 对这个地址发起写入。
3. 把 20 写入该地址对应的内存单元。
```

### ④ 指针运算和地址变化

```c
int arr[3] = {10, 20, 30};
int *p = arr;

p++;
```

如果：

```text
int 占 4 字节。
p 原来指向 arr[0]。
```

那么：

```text
p++ 后，地址增加 4。
p 指向 arr[1]。
```

原因：

```text
指针加 1，不是地址数值简单加 1。
而是加上它所指向类型的大小。
```

### ⑤ 指针类型决定地址步长

```text
char*   加 1，地址增加 1 字节。
int*    加 1，地址常见增加 4 字节。
double* 加 1，地址常见增加 8 字节。
```

> [!tip] 记忆口诀
> **指针里装地址，解引用就访问；类型定步长，加一不是加一字节。**

# 08 | 虚拟地址与物理地址：程序看到的地址不一定是真实地址

> [!abstract] 核心结论
> 在现代操作系统中，程序中的地址通常是虚拟地址，不一定直接等于物理内存地址。CPU 访问内存前，可能要经过 MMU 把虚拟地址转换成物理地址。

### ① 程序看到的地址

示例：

```c
int a = 10;
printf("%p\n", (void*)&a);
```

输出可能是：

```text
0x7ffd12345678
```

这个地址通常是：

```text
虚拟地址。
```

### ② 什么是虚拟地址

```text
虚拟地址 = 程序以为自己拥有的地址空间。
```

每个进程都像是拥有独立的地址空间：

```text
进程 A 可以有地址 0x1000。
进程 B 也可以有地址 0x1000。
```

但它们最终可能映射到不同的物理内存位置。

### ③ MMU 地址转换

```text
MMU = Memory Management Unit
内存管理单元
```

作用：

```text
把虚拟地址转换成物理地址。
检查访问权限。
支持分页、隔离、保护。
```

访问过程：

```text
程序发起访问虚拟地址
        ↓
CPU / MMU 查询页表
        ↓
虚拟地址转换成物理地址
        ↓
再访问实际内存
```

### ④ 为什么需要虚拟地址

```text
1. 让每个进程拥有独立地址空间。
2. 防止进程随意访问别人的内存。
3. 支持内存保护。
4. 支持分页和换页。
5. 简化程序链接和加载。
```

### ⑤ 嵌入式裸机情况

在很多简单单片机或裸机程序中：

```text
没有复杂操作系统。
没有完整 MMU。
程序访问的地址可能直接就是物理地址或总线地址。
```

例如：

```c
#define REG (*(volatile unsigned int*)0x40000000)
```

这个地址可能直接对应某个硬件寄存器。

> [!tip] 记忆口诀
> **应用程序看到多是虚拟地址，硬件最终访问物理地址；MMU 负责翻译和保护。**

# 09 | 地址缓存与 TLB：地址转换也需要加速

> [!abstract] 核心结论
> 虚拟地址转换如果每次都查页表，会很慢。TLB 是地址转换缓存，用来保存最近使用过的虚拟地址到物理地址的映射，加速内存访问。

### ① 为什么需要 TLB

如果每次访问内存都要：

```text
1. 查页表。
2. 得到物理地址。
3. 再访问内存。
```

那么：

```text
一次普通内存访问可能变成多次内存访问。
性能会明显下降。
```

### ② TLB 的作用

```text
TLB = Translation Lookaside Buffer
地址转换旁路缓存
```

它保存：

```text
虚拟页号 → 物理页框号
```

也就是：

```text
最近用过的地址翻译结果。
```

### ③ TLB 命中

```text
CPU 要访问虚拟地址。
先查 TLB。
如果 TLB 中已有对应映射：
直接得到物理地址。
这叫 TLB 命中。
```

### ④ TLB 未命中

```text
如果 TLB 没有对应映射：
需要查页表。
查到后，把结果填入 TLB。
以后再访问类似地址就更快。
```

### ⑤ 和程序性能的关系

连续访问数组：

```c
for (int i = 0; i < n; i++) {
    sum += arr[i];
}
```

通常更友好，因为：

```text
地址连续。
页表映射和 TLB 命中率更好。
缓存命中率也更好。
```

随机访问大数组：

```text
可能频繁跨页。
TLB 命中率下降。
访问变慢。
```

> [!tip] 记忆口诀
> **虚拟地址要翻译，TLB 缓存翻译结果；命中就快，未命中就查页表。**

# 10 | 地址总线总结

> [!abstract] 核心结论
> 地址总线的底层核心是“多根地址线 + 二进制电平组合 + 地址译码 + 片选 + 地址映射”。它不搬运数据内容，而是负责告诉硬件当前要访问哪个位置。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>地址总线底层总结：</strong>
<ul>
  <li><strong>本质：</strong> 一组并行地址线。</li>
  <li><strong>一根线：</strong> 表示地址中的 1 个二进制位。</li>
  <li><strong>地址组合：</strong> 多根地址线的高低电平组成一个地址编号。</li>
  <li><strong>位宽：</strong> 决定最多能表示多少个地址。</li>
  <li><strong>寻址空间：</strong> n 根地址线最多表示 <code>2^n</code> 个地址。</li>
  <li><strong>地址译码：</strong> 把地址范围翻译成具体芯片或外设选择。</li>
  <li><strong>片选：</strong> 只有被选中的硬件响应访问。</li>
  <li><strong>地址映射：</strong> 不同地址范围连接不同硬件区域。</li>
  <li><strong>地址对齐：</strong> 合理地址边界可以提高访问效率。</li>
  <li><strong>虚拟地址：</strong> 现代系统中程序地址通常要经过 MMU 转换。</li>
</ul>
</div>

### ② 最底层一句话

```text
地址总线 = 多根电路线在某个时刻的电平组合，用来表示 CPU 要访问的位置编号。
```
