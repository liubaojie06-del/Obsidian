# 01 | 浮点运算单元 FPU：CPU 内部专门处理小数的硬件

> [!abstract] 核心结论
> 浮点运算单元 FPU 是 CPU 内部专门执行小数运算的硬件部件。它负责处理 `float`、`double` 这类浮点数的加减乘除、平方根、比较、类型转换、舍入和异常状态。

### ① 底层原理：FPU 为什么不能只用 ALU 替代

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>FPU 的硬件本质：</strong>
<ul>
  <li><strong>ALU 主要处理整数：</strong> 适合整数加减、逻辑运算、移位和比较。</li>
  <li><strong>FPU 专门处理浮点数：</strong> 浮点数包含符号位、指数、尾数，运算流程比整数复杂。</li>
  <li><strong>浮点运算需要对阶：</strong> 加减法前必须先让两个数的指数对齐。</li>
  <li><strong>浮点运算需要规格化：</strong> 运算后要调整尾数和指数，让结果符合浮点格式。</li>
  <li><strong>浮点运算需要舍入：</strong> 因为二进制位数有限，很多小数不能精确表示。</li>
</ul>
</div>

### ② 一句话理解

```text
ALU 像整数计算器。
FPU 像小数科学计算器。
```

例如 C 语言代码：

```c
double c = a + b;
```

底层可以理解为：

```text
1. CPU 把 a 和 b 的浮点编码送入 FPU。
2. FPU 拆出符号位、指数、尾数。
3. FPU 对齐指数。
4. FPU 对尾数做加法。
5. FPU 规格化结果。
6. FPU 按规则舍入。
7. 结果写回浮点寄存器。
```

> [!tip] 记忆口诀
> **整数交给 ALU，小数交给 FPU；浮点不是直接加，要先拆开再处理。**

# 02 | IEEE 754：浮点数的二进制存储规则

> [!abstract] 核心结论
> 现代 CPU 中的浮点数通常遵循 IEEE 754 标准。一个浮点数不是直接存十进制小数，而是拆成符号位、指数位和尾数位。

### ① 浮点数三部分

浮点数通常由三部分组成：

```text
符号位 Sign
指数位 Exponent
尾数位 Fraction / Mantissa
```

可以理解为：

```text
数值 = (-1)^符号位 × 有效数字 × 2^指数
```

### ② float 单精度结构

C 语言中的 `float` 通常是 32 位：

```text
┌──────┬──────────┬───────────────────────┐
│ 符号位 │ 指数 8 位 │ 尾数 23 位             │
└──────┴──────────┴───────────────────────┘
  1 bit   8 bits       23 bits
```

也就是：

```text
float = 1 + 8 + 23 = 32 bit
```

### ③ double 双精度结构

C 语言中的 `double` 通常是 64 位：

```text
┌──────┬───────────┬──────────────────────────────┐
│ 符号位 │ 指数 11 位 │ 尾数 52 位                    │
└──────┴───────────┴──────────────────────────────┘
  1 bit   11 bits       52 bits
```

也就是：

```text
double = 1 + 11 + 52 = 64 bit
```

### ④ 为什么要这样设计

这种格式可以表示：

```text
非常大的数。
非常小的数。
小数。
正数和负数。
正无穷和负无穷。
NaN 非数字。
接近 0 的非规格化数。
```

### ⑤ 浮点数不是精确小数

例如：

```c
double x = 0.1;
```

在二进制中：

```text
0.1 通常无法被有限位二进制精确表示。
```

所以它只能存成：

```text
最接近 0.1 的二进制近似值。
```

> [!tip] 记忆口诀
> **浮点数不是直接存小数，而是存符号、指数和尾数。**

# 03 | 符号位、指数、尾数：浮点数如何表示大小

> [!abstract] 核心结论
> 符号位决定正负，指数决定数量级，尾数决定有效精度。浮点数能表示很大范围，是因为指数可以放大或缩小尾数。

### ① 符号位 Sign

```text
符号位 = 0 表示正数。
符号位 = 1 表示负数。
```

例如：

```text
+3.5 的符号位是 0。
-3.5 的符号位是 1。
```

### ② 指数 Exponent

指数决定这个数乘以多少个 2。

例如：

```text
1.101 × 2^3
```

表示：

```text
1.101 这个二进制小数向左移动 3 位。
```

指数越大：

```text
数值越大。
```

指数越小：

```text
数值越接近 0。
```

### ③ 尾数 Mantissa / Fraction

尾数保存有效数字。

例如：

```text
1.101
```

其中：

```text
101 是尾数部分的重要信息。
```

尾数位越多：

```text
精度越高。
```

### ④ 阶码偏移 Bias

IEEE 754 中指数不是直接存真实指数，而是存偏移后的值。

对于 float：

```text
Bias = 127
```

真实指数：

```text
真实指数 = 存储指数 - 127
```

对于 double：

```text
Bias = 1023
```

真实指数：

```text
真实指数 = 存储指数 - 1023
```

### ⑤ 例子理解

假设有一个规格化浮点数：

```text
符号位 = 0
尾数 ≈ 1.5
指数 = 3
```

那么：

```text
数值 = (+1) × 1.5 × 2^3
     = 12
```

> [!tip] 记忆口诀
> **符号管正负，指数管大小，尾数管精度。**

# 04 | 浮点加减：为什么小数相加比整数复杂

> [!abstract] 核心结论
> 浮点加减法不能直接把所有 bit 相加。FPU 必须先比较指数、移动尾数对阶，再执行尾数加减，最后规格化和舍入。

### ① 浮点加法基本流程

```text
1. 拆分两个浮点数：
   符号、指数、尾数。

2. 比较指数大小。

3. 对阶：
   把指数较小的数的尾数右移。

4. 尾数相加或相减。

5. 规格化结果。

6. 舍入。

7. 重新打包成 IEEE 754 格式。
```

### ② 什么是对阶

例如：

```text
1.25 × 2^5
+
1.50 × 2^2
```

指数不同，不能直接加尾数。

要把第二个数改成同样指数：

```text
1.50 × 2^2
= 0.1875 × 2^5
```

然后：

```text
1.25 × 2^5
+
0.1875 × 2^5
=
1.4375 × 2^5
```

### ③ 为什么小数加法会丢精度

如果两个数差距很大：

```text
100000000.0 + 0.000001
```

小数部分在对阶右移时可能被移出尾数范围。

结果可能变成：

```text
100000000.0
```

很小的数被“吃掉”。

### ④ 符号不同怎么办

如果两个浮点数符号不同：

```text
A + (-B)
```

本质上变成：

```text
A - B
```

FPU 要比较绝对值大小，再决定结果符号。

### ⑤ 加减法硬件模块

浮点加减单元通常包括：

```text
指数比较器。
尾数移位器。
尾数加减器。
规格化移位器。
舍入单元。
异常检测逻辑。
```

> [!tip] 记忆口诀
> **浮点加减先对阶，尾数再加减；算完要规格化，最后还要舍入。**

# 05 | 浮点乘除：指数和尾数分开处理

> [!abstract] 核心结论
> 浮点乘法通常把符号、指数、尾数分开处理：符号异或，指数相加，尾数相乘。浮点除法则是符号异或，指数相减，尾数相除。

### ① 浮点乘法流程

对于：

```text
A = (-1)^Sa × Ma × 2^Ea
B = (-1)^Sb × Mb × 2^Eb
```

乘法：

```text
A × B = (-1)^(Sa xor Sb) × (Ma × Mb) × 2^(Ea + Eb)
```

也就是：

```text
1. 符号位异或。
2. 指数相加。
3. 尾数相乘。
4. 规格化。
5. 舍入。
6. 检查溢出 / 下溢。
```

### ② 乘法示例

```text
1.5 × 2^3
×
1.25 × 2^2
```

先乘尾数：

```text
1.5 × 1.25 = 1.875
```

再加指数：

```text
2^(3 + 2) = 2^5
```

结果：

```text
1.875 × 2^5
```

### ③ 浮点除法流程

除法：

```text
A / B = (-1)^(Sa xor Sb) × (Ma / Mb) × 2^(Ea - Eb)
```

也就是：

```text
1. 符号位异或。
2. 指数相减。
3. 尾数相除。
4. 规格化。
5. 舍入。
6. 检查除零、溢出、下溢等异常。
```

### ④ 乘法通常比加法更规则

浮点加法需要对阶：

```text
指数不同要移动尾数。
```

浮点乘法直接：

```text
指数相加。
尾数相乘。
```

所以乘法流程在结构上更规则，但尾数乘法器面积较大。

### ⑤ 除法通常更慢

除法硬件更复杂，常见实现包括：

```text
迭代除法。
牛顿迭代。
查表加近似。
流水线除法器。
```

通常：

```text
浮点除法比浮点加法、乘法更慢。
```

> [!tip] 记忆口诀
> **乘法指数相加，除法指数相减；符号都靠异或，尾数各自乘除。**

# 06 | 规格化、舍入与精度损失

> [!abstract] 核心结论
> 浮点运算后，结果必须调整成 IEEE 754 要求的格式，这叫规格化。因为尾数位数有限，超出的位必须舍入，所以浮点运算天然可能产生精度损失。

### ① 什么是规格化

规格化要求有效数字保持类似：

```text
1.xxxxx × 2^E
```

也就是：

```text
最高有效位通常是 1。
```

例如：

```text
0.1101 × 2^5
```

可以规格化为：

```text
1.101 × 2^4
```

### ② 为什么要规格化

规格化可以：

```text
让同一个数表示方式唯一。
最大化尾数有效位利用率。
提高表示精度。
方便硬件比较和运算。
```

### ③ 什么是舍入

运算结果可能有很多位：

```text
1.101010111001010101...
```

但 float 只能保存有限尾数位。

所以必须：

```text
保留能存下的位。
根据舍入规则处理多余的位。
```

### ④ 常见舍入模式

IEEE 754 常见舍入模式：

```text
向最近值舍入，遇到正中间取偶数。
向 0 舍入。
向 +∞ 舍入。
向 -∞ 舍入。
```

默认常见是：

```text
Round to nearest, ties to even
就近舍入，正好一半时取偶。
```

### ⑤ 精度损失示例

```c
double x = 0.1 + 0.2;
printf("%.17f\n", x);
```

可能输出：

```text
0.30000000000000004
```

原因：

```text
0.1 和 0.2 在二进制中本来就是近似值。
运算后再经过舍入。
最终结果不一定刚好等于十进制 0.3。
```

> [!tip] 记忆口诀
> **规格化让格式统一，舍入让结果装得下；尾数有限，精度必有限。**

# 07 | 特殊值：0、无穷大、NaN 与非规格化数

> [!abstract] 核心结论
> IEEE 754 不只表示普通小数，还定义了特殊值：正负零、正负无穷、NaN 和非规格化数。这些特殊值让浮点硬件能处理极端情况。

### ① 正零和负零

浮点数中有：

```text
+0
-0
```

它们数值比较时通常相等：

```text
+0 == -0
```

但符号位不同，在某些运算中可能产生不同结果。

例如：

```text
1 / +0 = +∞
1 / -0 = -∞
```

### ② 正无穷和负无穷

当结果超出可表示范围时，可能得到：

```text
+∞
-∞
```

例如：

```text
1.0 / 0.0
```

可能产生无穷大。

### ③ NaN

```text
NaN = Not a Number
不是一个数字
```

常见来源：

```text
0.0 / 0.0
sqrt(-1.0)
∞ - ∞
非法浮点操作
```

NaN 的特点：

```text
NaN 和任何数比较通常都为假。
甚至 NaN == NaN 也为假。
```

### ④ 非规格化数

非规格化数也叫：

```text
Denormal / Subnormal
```

它用于表示非常接近 0 的极小数。

作用：

```text
让浮点数从最小规格化数到 0 之间不是突然断崖式消失。
```

### ⑤ 非规格化数可能更慢

某些硬件处理非规格化数比较慢。

因此高性能计算中有时会启用：

```text
Flush-to-zero
把非规格化数直接当 0 处理。
```

这样可以提高速度，但会牺牲一部分极小数精度。

> [!tip] 记忆口诀
> **浮点不只有普通数，还有正负零、无穷大、NaN 和极小非规格化数。**

# 08 | 浮点异常与状态标志

> [!abstract] 核心结论
> FPU 在运算时会检测各种异常情况，例如无效操作、除零、溢出、下溢和不精确结果，并把这些状态记录到浮点状态寄存器中。

### ① 常见浮点异常

```text
Invalid Operation：
无效操作，例如 0/0、sqrt(-1)。

Divide by Zero：
非零数除以 0。

Overflow：
结果太大，超出可表示范围。

Underflow：
结果太小，接近 0，无法正常规格化表示。

Inexact：
结果需要舍入，不是精确结果。
```

### ② 异常不一定让程序崩溃

很多浮点异常默认不会直接中断程序，而是：

```text
产生特殊结果。
设置状态标志。
程序继续执行。
```

例如：

```text
1.0 / 0.0 → +∞
0.0 / 0.0 → NaN
```

### ③ 浮点状态寄存器

FPU 通常有状态寄存器，用来记录：

```text
异常标志。
舍入模式。
精度控制。
比较结果。
控制位。
```

不同架构名字不同，例如：

```text
x87 Status Word
MXCSR
FPSCR
FP Status Register
```

### ④ 为什么要有异常标志

程序可以检查：

```text
本次计算是否溢出。
是否出现 NaN。
是否发生不精确舍入。
是否出现除零。
```

这对下面场景很重要：

```text
科学计算。
金融计算。
图形计算。
数值算法调试。
安全关键系统。
```

### ⑤ 不精确异常很常见

因为很多十进制小数不能精确表示：

```text
0.1
0.2
0.3
```

所以浮点运算经常需要舍入。

这意味着：

```text
Inexact 标志可能非常常见。
```

> [!tip] 记忆口诀
> **FPU 算完不只给结果，还会记异常；除零、溢出、NaN，都有状态可查。**

# 09 | 浮点流水线、SIMD 与 FMA

> [!abstract] 核心结论
> 现代 CPU 的 FPU 不只是一个简单小数计算器，而是高度流水线化、向量化的执行单元。它可以通过 SIMD 同时处理多个浮点数，也可以通过 FMA 一次完成乘加运算。

### ① 浮点流水线

浮点运算较复杂，常被拆成多个阶段：

```text
取操作数。
拆分符号 / 指数 / 尾数。
对阶或指数处理。
尾数运算。
规格化。
舍入。
写回。
```

这些阶段可以流水线化。

好处：

```text
单条浮点指令延迟可能较长。
但流水线填满后，可以每周期持续产出结果。
```

### ② SIMD 是什么

```text
SIMD = Single Instruction, Multiple Data
单指令多数据
```

含义：

```text
一条指令同时处理多个数据。
```

例如 4 个 float 同时相加：

```text
[a0, a1, a2, a3]
+
[b0, b1, b2, b3]
=
[c0, c1, c2, c3]
```

### ③ SIMD 常见场景

```text
图像处理。
音视频编码。
矩阵运算。
机器学习。
游戏物理。
科学计算。
信号处理。
```

### ④ FMA 是什么

```text
FMA = Fused Multiply-Add
融合乘加
```

它执行：

```text
a * b + c
```

但不是先乘法舍入一次，再加法舍入一次。

而是：

```text
乘法和加法融合，中间不舍入，最后只舍入一次。
```

### ⑤ FMA 的好处

```text
速度更快。
精度更好。
适合矩阵乘法。
适合点积运算。
适合神经网络和科学计算。
```

例如：

```c
sum += a[i] * b[i];
```

底层就很适合用 FMA 加速。

> [!tip] 记忆口诀
> **现代 FPU 有流水线，SIMD 一次算一排，FMA 乘加合一步。**

# 10 | FPU 总结

> [!abstract] 核心结论
> FPU 的底层核心是“浮点格式解析 + 指数处理 + 尾数运算 + 规格化 + 舍入 + 异常检测”。它让 CPU 能高效处理小数、科学计算、图形计算和向量运算。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>FPU 底层总结：</strong>
<ul>
  <li><strong>全称：</strong> Floating Point Unit，浮点运算单元。</li>
  <li><strong>作用：</strong> 处理 <code>float</code>、<code>double</code> 等浮点数运算。</li>
  <li><strong>标准：</strong> 现代浮点数通常遵循 IEEE 754。</li>
  <li><strong>组成：</strong> 符号位、指数位、尾数位。</li>
  <li><strong>加减法：</strong> 需要对阶、尾数加减、规格化、舍入。</li>
  <li><strong>乘除法：</strong> 符号异或，指数相加或相减，尾数乘除。</li>
  <li><strong>舍入：</strong> 因为尾数位数有限，结果经常只能近似保存。</li>
  <li><strong>特殊值：</strong> 支持正负零、无穷大、NaN、非规格化数。</li>
  <li><strong>异常：</strong> 检测无效操作、除零、溢出、下溢和不精确结果。</li>
  <li><strong>现代优化：</strong> 浮点流水线、SIMD 向量计算、FMA 融合乘加。</li>
</ul>
</div>

### ② 最底层一句话

```text
FPU = CPU 内部专门把 IEEE 754 浮点编码拆开、运算、规格化、舍入并重新打包的硬件单元。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **浮点不是普通小数，而是符号、指数、尾数；加减先对阶，乘除分开算；规格化后要舍入，NaN 无穷也要管。**
