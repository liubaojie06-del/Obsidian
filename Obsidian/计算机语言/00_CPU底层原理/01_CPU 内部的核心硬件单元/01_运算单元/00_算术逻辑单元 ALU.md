# 01 | 算术逻辑单元 ALU：CPU 内部真正做运算的核心部件

> [!abstract] 核心结论
> 算术逻辑单元 ALU 是 CPU 运算器的核心，负责执行整数加减、逻辑与或非、比较、移位、标志位更新等操作。程序中的 `+`、`-`、`&`、`|`、`==` 等运算，最终都会在类似 ALU 的硬件电路中完成。

### ① 底层原理：ALU 本质是组合逻辑电路

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>ALU 的硬件本质：</strong>
<ul>
  <li><strong>ALU 全称：</strong> Arithmetic Logic Unit，算术逻辑单元。</li>
  <li><strong>算术运算：</strong> 执行加法、减法、加一、减一、比较等整数运算。</li>
  <li><strong>逻辑运算：</strong> 执行与、或、非、异或等按位逻辑操作。</li>
  <li><strong>移位操作：</strong> 执行左移、右移、算术右移、循环移位等操作。</li>
  <li><strong>标志位更新：</strong> 根据运算结果更新零标志、进位标志、溢出标志、符号标志等。</li>
</ul>
</div>

### ② 一句话理解

```text
ALU = CPU 里面负责“算数”和“判断逻辑”的硬件计算器。
```

例如 C 语言代码：

```c
int c = a + b;
```

底层可以理解为：

```text
1. CPU 从寄存器或内存中取出 a。
2. CPU 从寄存器或内存中取出 b。
3. a 和 b 被送入 ALU。
4. ALU 执行加法。
5. 结果写回寄存器或内存。
```

> [!tip] 记忆口诀
> **寄存器负责存，ALU 负责算；控制器发命令，总线搬数据。**

# 02 | ALU 的输入与输出

> [!abstract] 核心结论
> ALU 通常有两个操作数输入、一个控制输入、一个结果输出，以及一组状态标志输出。控制信号决定 ALU 当前做加法、减法、与、或、异或还是移位。

### ① ALU 基本结构

```text
        操作数 A
           │
           ▼
      ┌─────────┐
控制 →│   ALU   │→ 运算结果 Result
      └─────────┘
           ▲
           │
        操作数 B

同时输出：
Zero / Carry / Overflow / Sign 等标志位
```

### ② 主要输入

```text
A 输入：
第一个操作数。

B 输入：
第二个操作数。

ALU Control：
控制 ALU 执行哪种操作。
```

例如：

```text
ALU Control = ADD  → 执行 A + B
ALU Control = SUB  → 执行 A - B
ALU Control = AND  → 执行 A & B
ALU Control = OR   → 执行 A | B
ALU Control = XOR  → 执行 A ^ B
```

### ③ 主要输出

```text
Result：
运算结果。

Zero：
结果是否为 0。

Carry：
是否产生进位或借位。

Overflow：
有符号运算是否溢出。

Sign：
结果最高位是否为 1。
```

### ④ 示例

假设：

```text
A = 00000101  // 5
B = 00000011  // 3
操作 = ADD
```

ALU 输出：

```text
Result = 00001000  // 8
Zero   = 0
Carry  = 0
Sign   = 0
```

> [!tip] 记忆口诀
> **A、B 进 ALU，控制信号选操作；结果出来后，标志位顺手更新。**

# 03 | 加法器：ALU 算术运算的基础

> [!abstract] 核心结论
> ALU 中最核心的算术电路是加法器。减法、比较、地址计算等操作，很多都可以建立在加法器基础上完成。

### ① 1 位全加器

1 位全加器接收：

```text
A
B
Cin 低位进位
```

输出：

```text
Sum 当前位结果
Cout 向高位进位
```

逻辑关系：

```text
Sum  = A ⊕ B ⊕ Cin
Cout = A·B + A·Cin + B·Cin
```

### ② 多位加法器

多个 1 位全加器串起来，就能形成多位加法器。

```text
第 0 位加法器 → 进位给第 1 位
第 1 位加法器 → 进位给第 2 位
第 2 位加法器 → 进位给第 3 位
...
```

示意：

```text
A0 B0 → FA0 → S0, C1
A1 B1 → FA1 → S1, C2
A2 B2 → FA2 → S2, C3
A3 B3 → FA3 → S3, C4
```

### ③ 4 位加法示例

```text
  0101   // 5
+ 0011   // 3
------
  1000   // 8
```

每一位都由加法器完成：

```text
低位相加产生进位。
进位继续参与高位相加。
最终得到完整结果。
```

### ④ 加法器速度问题

普通串行进位加法器的问题：

```text
高位必须等待低位进位传上来。
位数越多，延迟越大。
```

为了加速，硬件会使用：

```text
超前进位加法器。
选择进位加法器。
并行前缀加法器。
流水线加法器。
```

> [!tip] 记忆口诀
> **加法器是一位一位算，进位一层一层传；进位越快，加法越快。**

# 04 | 减法与补码：ALU 为什么用加法器也能做减法

> [!abstract] 核心结论
> 在二进制补码系统中，减法可以转换成加法：`A - B = A + (~B + 1)`。因此 ALU 不一定需要单独设计完整减法器，可以利用加法器完成减法。

### ① 补码减法公式

```text
A - B = A + 补码(B)
```

其中：

```text
补码(B) = 按位取反 B + 1
```

也就是：

```text
A - B = A + (~B + 1)
```

### ② 示例：5 - 3

用 4 位二进制表示：

```text
5 = 0101
3 = 0011
```

先求 3 的补码：

```text
0011 按位取反 = 1100
1100 + 1      = 1101
```

然后：

```text
  0101
+ 1101
------
 10010
```

只取低 4 位：

```text
0010 = 2
```

所以：

```text
5 - 3 = 2
```

### ③ ALU 中如何实现减法

ALU 做减法时可以：

```text
1. 把 B 输入按位取反。
2. 把加法器的 Cin 设为 1。
3. 执行 A + ~B + 1。
```

也就是：

```text
A - B
```

被硬件转换成：

```text
A + B 的补码
```

### ④ 比较也常用减法

例如判断：

```c
if (a == b)
```

底层可以做：

```text
a - b
```

如果结果为 0：

```text
说明 a == b。
```

判断大小也常借助：

```text
减法结果的符号位、进位位、溢出位。
```

> [!tip] 记忆口诀
> **减法不用另起炉灶，取反加一送加法器；比较也常先做减法。**

# 05 | 逻辑运算：按位处理 0 和 1

> [!abstract] 核心结论
> ALU 的逻辑运算是按位进行的。每一位独立执行与、或、非、异或，不需要进位传播，所以逻辑运算通常比加减法结构更直接。

### ① 按位与 AND

```text
规则：
1 & 1 = 1
1 & 0 = 0
0 & 1 = 0
0 & 0 = 0
```

示例：

```text
  1100
& 1010
------
  1000
```

用途：

```text
清零某些位。
检测某一位是否为 1。
提取掩码范围。
```

### ② 按位或 OR

```text
规则：
只要有 1，结果就是 1。
```

示例：

```text
  1100
| 1010
------
  1110
```

用途：

```text
把某些位置 1。
合并标志位。
设置控制位。
```

### ③ 按位异或 XOR

```text
规则：
相同为 0，不同为 1。
```

示例：

```text
  1100
^ 1010
------
  0110
```

用途：

```text
翻转某些位。
比较位是否不同。
简单校验。
```

### ④ 按位非 NOT

```text
规则：
0 变 1。
1 变 0。
```

示例：

```text
~00001111 = 11110000
```

### ⑤ 逻辑运算的硬件本质

```text
AND 由与门完成。
OR 由或门完成。
XOR 由异或门完成。
NOT 由反相器完成。
```

多位逻辑运算就是：

```text
每一位各放一套逻辑门，并行完成。
```

> [!tip] 记忆口诀
> **逻辑运算按位算，各位互不打扰；与清位，或置位，异或翻转最常见。**

# 06 | 移位器：ALU 中负责移动 bit 的部件

> [!abstract] 核心结论
> 移位器负责把二进制位整体左移或右移。左移常用于乘 2，右移常用于除 2，算术右移会保留符号位，逻辑右移会补 0。

### ① 逻辑左移

```text
0011 << 1 = 0110
```

理解：

```text
所有位向左移动。
低位补 0。
高位移出丢弃。
```

在不溢出的情况下：

```text
左移 1 位 ≈ 乘以 2
```

### ② 逻辑右移

```text
1000 >> 1 = 0100
```

理解：

```text
所有位向右移动。
高位补 0。
低位移出丢弃。
```

常用于：

```text
无符号数除以 2。
```

### ③ 算术右移

算术右移用于有符号数，通常保留最高符号位。

例如：

```text
11111000 算术右移 1 位
结果：
11111100
```

这里最高位继续补 1，用来保持负数符号。

### ④ 循环移位

循环移位会把移出去的位重新放回另一端。

例如 8 位循环左移：

```text
10000001 rol 1 = 00000011
```

用途：

```text
加密算法。
校验算法。
位操作优化。
处理器特殊指令。
```

### ⑤ 桶形移位器

为了快速完成多位移位，CPU 可能使用：

```text
Barrel Shifter
桶形移位器
```

它可以在一个周期内完成：

```text
左移 1 位。
左移 8 位。
右移 13 位。
```

而不是一位一位慢慢移。

> [!tip] 记忆口诀
> **左移像乘二，右移像除二；逻辑右移补零，算术右移保符号。**

# 07 | 标志位：ALU 运算后留下的状态结果

> [!abstract] 核心结论
> ALU 不只输出运算结果，还会更新状态标志位。条件跳转、比较大小、判断是否为零、判断溢出等操作，都依赖这些标志位。

### ① 常见标志位

```text
ZF：Zero Flag，零标志
CF：Carry Flag，进位标志
OF：Overflow Flag，溢出标志
SF：Sign Flag，符号标志
PF：Parity Flag，奇偶标志
```

### ② ZF 零标志

```text
如果运算结果为 0，ZF = 1。
否则 ZF = 0。
```

例如：

```text
5 - 5 = 0
```

则：

```text
ZF = 1
```

常用于：

```c
if (a == b)
```

底层可能通过：

```text
a - b 后检查 ZF。
```

### ③ CF 进位标志

无符号加法时：

```text
如果最高位产生进位，CF = 1。
```

无符号减法时：

```text
如果需要借位，CF 可能表示借位情况。
```

常用于：

```text
无符号数比较。
多精度整数加减。
```

### ④ OF 溢出标志

有符号运算结果超出可表示范围时：

```text
OF = 1
```

例如 8 位有符号数：

```text
127 + 1
```

结果超出：

```text
-128 ~ 127
```

因此会产生有符号溢出。

### ⑤ SF 符号标志

```text
SF 通常等于结果最高位。
```

如果结果最高位为 1：

```text
在补码有符号解释中，结果通常为负。
```

> [!tip] 记忆口诀
> **ALU 算完不只给结果，还会留标志；跳转判断看标志位。**

# 08 | ALU 与寄存器：数据从哪里来，又到哪里去

> [!abstract] 核心结论
> ALU 本身通常不长期保存数据。操作数来自寄存器、立即数、缓存或内部总线；运算结果再写回寄存器、缓存或流水线寄存器。

### ① ALU 数据来源

ALU 输入可能来自：

```text
通用寄存器。
立即数。
程序计数器 PC。
地址生成单元。
内存读取结果。
旁路转发结果。
```

示例：

```asm
ADD R3, R1, R2
```

底层：

```text
R1 → ALU 输入 A
R2 → ALU 输入 B
ALU 执行加法
结果 → R3
```

### ② ALU 结果去向

ALU 输出可能写入：

```text
通用寄存器。
标志寄存器。
地址寄存器。
流水线寄存器。
Store 数据路径。
分支判断单元。
```

### ③ ALU 不等于寄存器

```text
寄存器：
负责保存数据。

ALU：
负责处理数据。

总线 / 数据通路：
负责把数据送来送去。
```

### ④ 执行一条加法指令

```text
1. 从寄存器堆读出 R1 和 R2。
2. 把两个操作数送入 ALU。
3. ALU 做加法。
4. 结果送回寄存器堆。
5. 标志位根据结果更新。
```

### ⑤ 为什么寄存器很重要

如果操作数在寄存器中：

```text
ALU 很快就能拿到数据。
```

如果操作数在内存中：

```text
要先经过缓存或内存读取。
ALU 可能等待。
```

> [!tip] 记忆口诀
> **ALU 只负责算，寄存器负责供料和收结果；数据离 ALU 越近，执行越快。**

# 09 | ALU 与 CPU 流水线

> [!abstract] 核心结论
> 在流水线 CPU 中，ALU 通常位于执行阶段。取指、译码、读寄存器完成后，指令进入执行阶段，由 ALU 完成运算、地址计算或条件判断。

### ① 典型五级流水线

```text
IF   取指
ID   译码 / 读寄存器
EX   执行 / ALU 运算
MEM  访存
WB   写回
```

ALU 通常在：

```text
EX 执行阶段
```

### ② 不同指令中 ALU 的作用

算术指令：

```text
ADD / SUB：
ALU 执行加减。
```

逻辑指令：

```text
AND / OR / XOR：
ALU 执行逻辑运算。
```

访存指令：

```text
LOAD / STORE：
ALU 计算有效地址。
```

分支指令：

```text
BEQ / BNE：
ALU 比较两个值是否相等。
```

### ③ 地址计算也是 ALU 常见任务

例如：

```asm
LOAD R1, [R2 + 8]
```

ALU 会计算：

```text
有效地址 = R2 + 8
```

然后访存单元再按这个地址访问缓存或内存。

### ④ 分支判断

例如：

```c
if (a == b)
```

底层可能：

```text
ALU 执行 a - b。
如果结果为 0，ZF = 1。
分支单元根据 ZF 决定跳不跳。
```

### ⑤ 流水线中的旁路转发

如果上一条指令的 ALU 结果马上被下一条使用：

```text
CPU 可能不等它写回寄存器。
直接把 ALU 输出转发给下一条指令。
```

这叫：

```text
Forwarding / Bypassing
旁路转发。
```

> [!tip] 记忆口诀
> **流水线里 ALU 在执行段；加减逻辑它来算，地址分支也常靠它。**

# 10 | ALU 总结

> [!abstract] 核心结论
> ALU 的底层核心是“加法器 + 逻辑门 + 移位器 + 标志位生成电路 + 选择器”。控制器通过控制信号选择 ALU 做哪种操作，寄存器和执行总线负责提供输入与接收输出。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>ALU 底层总结：</strong>
<ul>
  <li><strong>全称：</strong> Arithmetic Logic Unit，算术逻辑单元。</li>
  <li><strong>本质：</strong> CPU 内部执行整数运算和逻辑运算的组合逻辑电路。</li>
  <li><strong>算术部分：</strong> 加法、减法、比较、加一、减一。</li>
  <li><strong>逻辑部分：</strong> 与、或、非、异或。</li>
  <li><strong>移位部分：</strong> 左移、右移、算术右移、循环移位。</li>
  <li><strong>输入：</strong> 操作数 A、操作数 B、ALU 控制信号。</li>
  <li><strong>输出：</strong> 运算结果和状态标志位。</li>
  <li><strong>标志位：</strong> Zero、Carry、Overflow、Sign 等。</li>
  <li><strong>减法实现：</strong> 通常通过补码，把减法转换成加法。</li>
  <li><strong>流水线作用：</strong> 通常位于执行阶段，负责运算、地址计算和分支比较。</li>
</ul>
</div>

### ② 最底层一句话

```text
ALU = 用加法器、逻辑门、移位器和选择器组成的 CPU 内部计算电路。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **ALU 是 CPU 的计算核心；加减靠加法器，逻辑靠门电路，移位靠移位器，结果出来看标志位。**
