# 01 | 数据总线：CPU 搬运二进制数据的电气通道

> [!abstract] 核心结论
> 数据总线不是抽象概念，而是一组真实存在的电路连线。每一根线在某一时刻表示一个二进制位，线上的高低电平组合起来，就是 CPU、内存、外设之间传输的数据。

### ① 底层原理：数据总线本质是一组并行电路线

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>数据总线的硬件本质：</strong>
<ul>
  <li><strong>一根数据线传 1 bit：</strong> 高电平通常表示 <code>1</code>，低电平通常表示 <code>0</code>。</li>
  <li><strong>多根数据线并行传输：</strong> 8 根线一次传 8 bit，32 根线一次传 32 bit。</li>
  <li><strong>总线宽度决定单次搬运量：</strong> 例如 64 位数据总线一次可以搬运 64 bit，也就是 8 字节。</li>
  <li><strong>数据不是“流过去”的文字：</strong> 它本质上是每根线在某个时刻的电压状态。</li>
  <li><strong>接收方按时钟采样：</strong> 在规定时刻读取每根线的电平，组合成一个二进制数。</li>
</ul>
</div>

### ② 8 位数据总线示意

假设数据总线有 8 根线：

```text
D7 D6 D5 D4 D3 D2 D1 D0
```

某一时刻线上电平如下：

```text
D7 D6 D5 D4 D3 D2 D1 D0
 0  1  0  1  0  1  0  1
```

组合起来就是：

```text
01010101
```

对应十六进制：

```text
0x55
```

也就是说：

```text
数据总线不是传“0x55”这个字符。
而是 8 根线同时呈现 01010101 这一组电平状态。
```

> [!tip] 记忆口诀
> **一根线一位，多根线并行；高低电平组合起来，就是二进制数据。**

# 02 | 数据总线宽度：一次能并行传多少 bit

> [!abstract] 核心结论
> 数据总线宽度就是数据线数量。总线越宽，同一个时钟周期内能并行传输的数据位数越多。

### ① 总线宽度含义

```text
8 位数据总线  = 8 根数据线  = 一次传 8 bit  = 1 字节
16 位数据总线 = 16 根数据线 = 一次传 16 bit = 2 字节
32 位数据总线 = 32 根数据线 = 一次传 32 bit = 4 字节
64 位数据总线 = 64 根数据线 = 一次传 64 bit = 8 字节
```

### ② 读取 32 位整数时的差异

假设要读取一个 32 位整数：

```text
32 bit = 4 字节
```

不同数据总线需要的传输次数：

```text
8 位数据总线：
一次 8 bit，需要 4 次。

16 位数据总线：
一次 16 bit，需要 2 次。

32 位数据总线：
一次 32 bit，需要 1 次。

64 位数据总线：
一次 64 bit，可以 1 次传完，但实际只用其中 32 bit。
```

### ③ 总线宽度和带宽

理论带宽可以粗略理解为：

```text
带宽 = 数据总线宽度 × 传输频率
```

例如：

```text
64 位数据总线 = 8 字节
总线频率 = 100 MHz

理论带宽 = 8 字节 × 100,000,000 次/秒
        = 800 MB/s
```

注意：

```text
真实带宽还会受到等待周期、协议开销、缓存命中率、突发传输长度等影响。
```

> [!tip] 记忆口诀
> **总线宽度像车道数，频率像发车速度；车道越多、发车越快，理论带宽越高。**

# 03 | 数据总线上的一个传输周期

> [!abstract] 核心结论
> 数据总线传输不是瞬间完成的，而是按照时钟节拍分阶段完成：驱动数据、稳定电平、采样数据、释放总线。

### ① 一个读周期可以抽象为

```text
1. 发起方准备读操作。
2. 被读取设备把数据驱动到数据总线上。
3. 数据线电平经过短暂变化后稳定。
4. CPU 在规定时钟边沿采样数据。
5. 数据被锁存进 CPU 内部寄存器。
6. 被读取设备释放数据总线。
```

### ② 数据线上不是立刻稳定

当设备把数据放到总线上时，电平需要一点时间才能稳定。

```text
理想理解：
0 立刻变 1

真实电路：
0 → 上升过程 → 稳定为 1
```

所以需要：

```text
建立时间 setup time：
数据必须在采样时钟到来前提前稳定。

保持时间 hold time：
采样时钟到来后，数据还要继续保持一小段时间。
```

### ③ 时序理解

```text
时钟：  ┌─┐ ┌─┐ ┌─┐
       ┘ └─┘ └─┘ └─

数据：  xxxx 01010101 xxxx
             ↑
          采样时刻
```

解释：

```text
xxxx 表示数据无效或正在变化。
01010101 表示数据已经稳定。
CPU 只会在规定时刻读取稳定区间的数据。
```

> [!tip] 记忆口诀
> **总线传输看时序，数据稳定再采样；太早太晚都可能读错。**

# 04 | 数据总线方向：谁驱动，谁接收

> [!abstract] 核心结论
> 数据总线通常是共享的，同一时刻只能有一个设备负责“驱动”总线，其他设备只能接收或进入高阻态。否则就会发生总线竞争。

### ① 什么叫驱动总线

```text
驱动总线 = 某个设备主动把每根数据线拉成 0 或 1。
```

例如要输出：

```text
0xA5 = 10100101
```

设备会让数据线变成：

```text
D7 D6 D5 D4 D3 D2 D1 D0
 1  0  1  0  0  1  0  1
```

### ② 什么叫接收总线

```text
接收总线 = 设备不改变数据线电平，只读取当前电平组合。
```

接收方会在指定时刻采样：

```text
D7 到 D0 的电平
      ↓
组合成二进制数据
      ↓
写入内部寄存器
```

### ③ 为什么不能多个设备同时驱动

如果设备 A 输出：

```text
11110000
```

设备 B 同时输出：

```text
00001111
```

那么某些数据线上会出现：

```text
一个设备想拉高，另一个设备想拉低
```

后果：

```text
1. 数据错误。
2. 电流异常。
3. 总线电平不确定。
4. 严重时可能损坏硬件。
```

> [!tip] 记忆口诀
> **总线同一时刻只能一个人说话，其他人要么听，要么闭嘴。**

# 05 | 高阻态：设备临时退出总线

> [!abstract] 核心结论
> 高阻态是总线共享的关键。设备不使用数据总线时，会让自己的输出端进入高阻态，相当于从电气上“断开”，不影响其他设备传输数据。

### ① 三态输出

普通数字输出只有两种状态：

```text
0：低电平
1：高电平
```

总线设备常用三态输出：

```text
0：输出低电平
1：输出高电平
Z：高阻态
```

### ② 高阻态是什么

```text
高阻态不是 0，也不是 1。
它可以理解为设备暂时不连接这根线。
```

也就是：

```text
设备不拉高。
设备不拉低。
让其他设备决定这根线的电平。
```

### ③ 多设备共享数据线

```text
设备 A：输出数据
设备 B：高阻态
设备 C：高阻态
设备 D：高阻态
```

此时总线上的数据由设备 A 决定。

如果轮到设备 B 输出：

```text
设备 A：高阻态
设备 B：输出数据
设备 C：高阻态
设备 D：高阻态
```

### ④ 三态门在数据总线中的作用

```text
三态门负责控制设备是否连接到总线。
使能信号有效：设备输出 0 或 1。
使能信号无效：设备进入高阻态。
```

> [!tip] 记忆口诀
> **高阻不是零不是一，而是暂时不参与；谁不用总线，谁就高阻让路。**

# 06 | 数据锁存：为什么需要寄存器和锁存器

> [!abstract] 核心结论
> 数据总线上的电平只是暂时存在，不能长期保存。CPU 或外设需要在正确时刻把数据“锁存”到寄存器中，才能稳定使用。

### ① 数据总线不保存数据

```text
数据总线只是通道。
它不负责长期存储。
```

就像：

```text
公路负责运输车辆。
但公路本身不保存货物。
```

数据传输结束后：

```text
设备释放总线。
数据线可能变成高阻态。
下一次传输又会出现新的数据。
```

### ② 寄存器负责保存数据

当 CPU 从数据总线上读取数据时：

```text
数据线稳定
      ↓
CPU 在时钟边沿采样
      ↓
数据被锁存到内部寄存器
      ↓
后续运算使用寄存器中的数据
```

### ③ 锁存过程

```text
数据总线：
D7~D0 = 01010101

采样时钟到来：
      ↓

CPU 内部寄存器：
REG = 01010101
```

### ④ 为什么不能直接一直用总线数据

因为：

```text
1. 总线马上可能被其他设备使用。
2. 数据线状态会变化。
3. 总线是共享资源，不能长期占用。
4. CPU 内部运算需要稳定数据源。
```

> [!tip] 记忆口诀
> **总线只负责路上传，寄存器才负责存下来。**

# 07 | 数据总线和缓存行：现代 CPU 不是一个字节一个字节慢慢读

> [!abstract] 核心结论
> 现代 CPU 访问内存时，通常不会只读一个变量那么小的一点数据，而是按缓存行成块搬运。数据总线常常一次或多次突发传输，把一整块数据送入缓存。

### ① 为什么需要成块搬运

CPU 速度远快于内存。

如果每次只读一个字节：

```text
CPU 等内存的次数会非常多。
性能很差。
```

所以现代系统常用：

```text
缓存 Cache
缓存行 Cache Line
突发传输 Burst Transfer
```

### ② 什么是缓存行

```text
缓存行是 CPU Cache 和内存之间搬运数据的基本单位。
常见缓存行大小是 64 字节。
```

如果程序读取：

```c
int a = arr[0];
```

CPU 可能不是只读 `arr[0]` 这 4 字节，而是把附近一整块数据搬入缓存。

```text
arr[0]
arr[1]
arr[2]
arr[3]
...
可能一起进入缓存
```

### ③ 数据总线在缓存行传输中的作用

假设：

```text
数据总线宽度 = 64 bit = 8 字节
缓存行大小 = 64 字节
```

那么搬运一条缓存行大约需要：

```text
64 字节 / 8 字节 = 8 次数据传输
```

也就是说：

```text
数据总线每次搬 8 字节。
连续搬 8 次。
组成一整条 64 字节缓存行。
```

### ④ 为什么数组连续访问更快

例如：

```c
for (int i = 0; i < 1000; i++) {
    sum += arr[i];
}
```

数组在内存中连续存放。

CPU 读取 `arr[0]` 时，可能顺便把后面的元素也装入缓存。

所以访问：

```text
arr[1]
arr[2]
arr[3]
```

时可能已经在缓存里，速度更快。

> [!tip] 记忆口诀
> **现代 CPU 读内存，常常不是拿一个，而是搬一块；数组连续，所以缓存友好。**

# 08 | 数据总线和字节序：多字节数据怎么排列

> [!abstract] 核心结论
> 数据总线一次可以传多个字节，但多字节数据在内存中的排列顺序由字节序决定。常见字节序有小端序和大端序。

### ① 一个 32 位整数

```c
int x = 0x12345678;
```

这个数有 4 个字节：

```text
0x12
0x34
0x56
0x78
```

### ② 小端序

小端序中，低位字节放在低地址：

```text
低地址 → 高地址

0x78  0x56  0x34  0x12
```

### ③ 大端序

大端序中，高位字节放在低地址：

```text
低地址 → 高地址

0x12  0x34  0x56  0x78
```

### ④ 数据总线传输时的理解

如果数据总线一次传 32 bit：

```text
CPU 可以一次读到 0x12345678。
```

但如果按字节观察内存：

```text
看到的排列可能是 78 56 34 12。
```

这不是数据错了，而是字节序规则不同。

### ⑤ 为什么字节序重要

字节序常见于：

```text
1. 网络通信。
2. 文件二进制格式。
3. 嵌入式寄存器。
4. 不同 CPU 架构之间传数据。
5. 指针强制转换查看内存。
```

示例：

```c
#include <stdio.h>

int main() {
    int x = 0x12345678;
    unsigned char *p = (unsigned char*)&x;

    printf("%x\n", p[0]);
    printf("%x\n", p[1]);
    printf("%x\n", p[2]);
    printf("%x\n", p[3]);

    return 0;
}
```

在小端机器上常见输出：

```text
78
56
34
12
```

> [!tip] 记忆口诀
> **小端低位放低址，大端高位放低址；数值没变，只是内存排列不同。**

# 09 | 数据总线、DMA 与 CPU 不亲自搬数据

> [!abstract] 核心结论
> 不是所有数据搬运都必须由 CPU 亲自完成。DMA 可以让外设和内存之间直接搬运数据，减少 CPU 负担，但底层仍然依赖数据通道完成传输。

### ① CPU 搬数据的传统方式

例如从外设读取大量数据：

```text
1. CPU 从外设读一个数据。
2. CPU 写入内存。
3. CPU 再读下一个数据。
4. CPU 再写入内存。
5. 重复很多次。
```

问题：

```text
CPU 被大量简单搬运工作占用。
真正计算任务被拖慢。
```

### ② DMA 的思想

DMA 全称：

```text
Direct Memory Access
直接内存访问
```

它的作用：

```text
让外设可以在 DMA 控制器帮助下，
直接把数据搬到内存，
或者从内存搬到外设，
不需要 CPU 每个字节都亲自参与。
```

### ③ DMA 传输过程

```text
1. CPU 配置 DMA：
   源地址、目标地址、数据长度、传输方向。

2. DMA 控制器开始搬运数据。

3. 数据在外设和内存之间传输。

4. 搬运完成后，DMA 通知 CPU。

5. CPU 继续处理结果。
```

### ④ DMA 和数据总线的关系

```text
DMA 不是不用总线。
DMA 是替 CPU 去使用总线搬数据。
```

也就是说：

```text
普通方式：
CPU 占用总线搬运数据。

DMA 方式：
DMA 控制器占用总线搬运数据。
CPU 可以去做别的事情。
```

### ⑤ 常见场景

```text
1. 网卡接收数据包。
2. 磁盘读取文件。
3. 声卡播放音频。
4. 摄像头采集图像。
5. 串口 / SPI / ADC 大量数据搬运。
```

> [!tip] 记忆口诀
> **DMA 不是不用路，而是让搬运工替 CPU 走这条路。**

# 10 | 数据总线总结

> [!abstract] 核心结论
> 数据总线的底层核心是“多根电路线 + 电平状态 + 时钟采样 + 三态控制 + 寄存器锁存”。它不是保存数据的地方，而是设备之间临时搬运二进制数据的通道。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>数据总线底层总结：</strong>
<ul>
  <li><strong>本质：</strong> 一组并行电路线。</li>
  <li><strong>一根线：</strong> 表示 1 个二进制位。</li>
  <li><strong>电平：</strong> 高低电平组合成二进制数据。</li>
  <li><strong>位宽：</strong> 决定一次能传多少 bit。</li>
  <li><strong>时序：</strong> 数据必须稳定后才能被采样。</li>
  <li><strong>高阻态：</strong> 不使用总线的设备必须退出总线。</li>
  <li><strong>锁存：</strong> 总线不保存数据，寄存器负责保存数据。</li>
  <li><strong>缓存行：</strong> 现代 CPU 常常按块搬运数据。</li>
  <li><strong>DMA：</strong> 可以替 CPU 完成大量数据搬运。</li>
</ul>
</div>

### ② 最底层一句话

```text
数据总线 = 多根电路线在某个时钟时刻的电平组合。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **数据总线不存数，只在瞬间传电平；一线一位，多线并行；稳定后采样，采完就锁存。谁不用总线，谁就高阻让路。**
