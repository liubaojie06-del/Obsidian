# 01 | 常量：程序运行中不应该被修改的数据

> [!abstract] 核心结论
> 常量是在程序中表示固定值的数据，例如整数 `10`、小数 `3.14`、字符 `'A'`、字符串 `"hello"`。它们通常用于表达固定配置、固定数值、固定文本，让代码更清晰、更安全。

### ① 底层原理：常量不是一种单一东西

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>常量的底层本质：</strong>
<ul>
  <li><strong>字面常量：</strong> 直接写在代码里的固定值，例如 <code>10</code>、<code>3.14</code>、<code>'A'</code>、<code>"hello"</code>。</li>
  <li><strong>const 常变量：</strong> 用 <code>const</code> 修饰的变量，表示不允许通过普通方式修改。</li>
  <li><strong>宏常量：</strong> 用 <code>#define</code> 定义的符号，在预处理阶段直接文本替换。</li>
  <li><strong>枚举常量：</strong> 用 <code>enum</code> 定义的命名整数常量。</li>
  <li><strong>字符串常量：</strong> 通常存放在只读数据区，不能被修改。</li>
</ul>
</div>

### ② 一句话理解

```text
常量 = 程序中固定不变、用于表达明确含义的数据。
```

例如：

```c
printf("%d\n", 100);
```

这里：

```text
100 就是整型常量。
```

再如：

```c
printf("%s\n", "hello");
```

这里：

```text
"hello" 是字符串常量。
```

> [!tip] 记忆口诀
> **直接写死是字面量，const 修饰不能改，define 替换靠预处理，enum 命名整数更清楚。**

# 02 | 字面常量：直接写在代码里的固定值

> [!abstract] 核心结论
> 字面常量就是直接出现在代码中的值。编译器会根据写法判断它是什么类型，例如 `10` 是整型常量，`3.14` 是浮点常量，`'A'` 是字符常量，`"hello"` 是字符串常量。

### ① 整型常量

```c
10
100
-20
0
```

示例：

```c
#include <stdio.h>

int main() {
    printf("%d\n", 100);
    return 0;
}
```

这里：

```text
100 是整型常量。
```

### ② 浮点常量

```c
3.14
0.5
-2.718
1.0
```

示例：

```c
#include <stdio.h>

int main() {
    printf("%f\n", 3.14);
    return 0;
}
```

这里：

```text
3.14 是浮点常量。
默认通常按 double 类型处理。
```

### ③ 字符常量

```c
'A'
'0'
'\n'
'\t'
```

示例：

```c
#include <stdio.h>

int main() {
    printf("%c\n", 'A');
    return 0;
}
```

这里：

```text
'A' 是字符常量。
```

### ④ 字符串常量

```c
"hello"
"C language"
"123"
""
```

示例：

```c
#include <stdio.h>

int main() {
    printf("%s\n", "hello");
    return 0;
}
```

这里：

```text
"hello" 是字符串常量。
```

### ⑤ 字面常量的问题

直接写大量字面常量会让代码不清晰。

不推荐：

```c
if (score >= 60) {
    printf("pass\n");
}
```

更清晰：

```c
#define PASS_SCORE 60

if (score >= PASS_SCORE) {
    printf("pass\n");
}
```

> [!tip] 记忆口诀
> **写在代码里的固定值，就是字面常量；多了要命名，别让魔法数字满天飞。**

# 03 | 整型常量：十进制、八进制、十六进制

> [!abstract] 核心结论
> 整型常量可以用不同进制表示。普通数字是十进制，以 `0` 开头是八进制，以 `0x` 或 `0X` 开头是十六进制。

### ① 十进制常量

```c
int a = 10;
```

含义：

```text
10 就是十进制整数十。
```

### ② 八进制常量

```c
int a = 010;
```

注意：

```text
010 不是十进制 10。
它是八进制 10。
换算成十进制是 8。
```

示例：

```c
#include <stdio.h>

int main() {
    int a = 010;
    printf("%d\n", a);
    return 0;
}
```

输出：

```text
8
```

### ③ 十六进制常量

```c
int a = 0x10;
```

含义：

```text
0x10 是十六进制。
换算成十进制是 16。
```

示例：

```c
#include <stdio.h>

int main() {
    int a = 0x10;
    printf("%d\n", a);
    return 0;
}
```

输出：

```text
16
```

### ④ 常见后缀

```c
10L    // long
10LL   // long long
10U    // unsigned int
10UL   // unsigned long
```

示例：

```c
unsigned int a = 100U;
long long b = 100LL;
```

### ⑤ 常见错误

```c
int month = 09;
```

错误原因：

```text
09 以 0 开头，被当成八进制。
但八进制没有数字 9。
```

正确：

```c
int month = 9;
```

> [!tip] 记忆口诀
> **普通数字十进制，0 开头八进制，0x 开头十六进制；09 是错误。**

# 04 | 字符常量：单引号里的一个字符

> [!abstract] 核心结论
> 字符常量用单引号表示，例如 `'A'`、`'0'`、`'\n'`。字符在底层通常以整数编码保存，所以字符也可以参与整数运算。

### ① 基本写法

```c
char ch = 'A';
```

含义：

```text
'A' 是字符常量。
ch 保存字符 A 的编码值。
```

### ② 示例代码

```c
#include <stdio.h>

int main() {
    char ch = 'A';

    printf("%c\n", ch);
    printf("%d\n", ch);

    return 0;
}
```

可能输出：

```text
A
65
```

说明：

```text
字符 A 的 ASCII 编码是 65。
```

### ③ 转义字符

常见转义字符：

```c
'\n'  // 换行
'\t'  // 制表符
'\\'  // 反斜杠
'\''  // 单引号
'\"'  // 双引号
'\0'  // 字符串结束符
```

示例：

```c
printf("hello\nworld\n");
```

输出：

```text
hello
world
```

### ④ 单引号和双引号不同

```c
'A'
```

是：

```text
字符常量。
```

```c
"A"
```

是：

```text
字符串常量，实际包含 'A' 和 '\0'。
```

### ⑤ 常见错误

错误：

```c
char ch = "A";
```

原因：

```text
"A" 是字符串地址，不是单个 char。
```

正确：

```c
char ch = 'A';
```

> [!tip] 记忆口诀
> **单引号是字符，双引号是字符串；字符是一个，字符串带结尾零。**

# 05 | 字符串常量：只读区域中的字符序列

> [!abstract] 核心结论
> 字符串常量用双引号表示，例如 `"hello"`。它本质上是一段以 `'\0'` 结尾的字符序列，通常存放在只读数据区，不能修改。

### ① 基本写法

```c
"hello"
```

底层实际类似：

```text
'h' 'e' 'l' 'l' 'o' '\0'
```

所以：

```text
字符串长度是 5。
占用空间是 6 个 char。
```

### ② 示例代码

```c
#include <stdio.h>

int main() {
    const char *str = "hello";

    printf("%s\n", str);

    return 0;
}
```

输出：

```text
hello
```

### ③ 为什么建议用 const char *

推荐：

```c
const char *str = "hello";
```

不推荐：

```c
char *str = "hello";
```

原因：

```text
"hello" 是字符串常量。
通常在只读数据区。
不应该通过 str 修改它。
```

### ④ 错误示例

```c
char *str = "hello";
str[0] = 'H';
```

问题：

```text
试图修改字符串常量。
这是未定义行为。
可能崩溃。
```

### ⑤ 如果想修改字符串

应该使用字符数组：

```c
char str[] = "hello";
str[0] = 'H';

printf("%s\n", str);
```

输出：

```text
Hello
```

区别：

```text
char *str = "hello";
str 指向只读字符串常量。

char str[] = "hello";
数组在栈区创建了一份可修改副本。
```

> [!tip] 记忆口诀
> **字符串常量在只读区，想改内容用字符数组。**

# 06 | const 常变量：有类型的只读变量

> [!abstract] 核心结论
> `const` 修饰变量后，表示该变量不允许通过普通方式修改。它有明确类型，会参与编译器类型检查，比宏常量更安全。

### ① 基本写法

```c
const int max_size = 100;
```

含义：

```text
max_size 初始化后不允许修改。
```

错误：

```c
max_size = 200;
```

原因：

```text
const 变量不可被普通赋值修改。
```

### ② 示例代码

```c
#include <stdio.h>

int main() {
    const int max_size = 100;

    printf("%d\n", max_size);

    return 0;
}
```

输出：

```text
100
```

### ③ const 不是宏替换

```c
const int size = 10;
```

这是：

```text
一个有类型的变量，只是不能改。
```

而：

```c
#define SIZE 10
```

是：

```text
预处理阶段的文本替换。
```

### ④ const 局部变量一般在哪里

```c
void test() {
    const int x = 10;
}
```

通常：

```text
x 仍然可能是栈上的局部变量。
```

也就是说：

```text
const 不等于一定放在只读区。
```

### ⑤ const 和 static const

```c
static const int max_size = 100;
```

如果在全局位置：

```text
static 限制当前文件可见。
const 限制不可修改。
```

适合做：

```text
文件内部只读配置。
```

> [!tip] 记忆口诀
> **const 管能不能改，static 管生命周期或文件可见性；二者不是一回事。**

# 07 | 宏常量：预处理阶段的文本替换

> [!abstract] 核心结论
> 宏常量用 `#define` 定义，本质是在预处理阶段把名字替换成指定文本。它没有变量类型，不占普通变量存储空间，但也缺少类型检查。

### ① 基本写法

```c
#define PI 3.14159
#define MAX_SIZE 100
```

使用：

```c
double area = PI * r * r;
```

预处理后类似：

```c
double area = 3.14159 * r * r;
```

### ② 示例代码

```c
#include <stdio.h>

#define PASS_SCORE 60

int main() {
    int score = 80;

    if (score >= PASS_SCORE) {
        printf("pass\n");
    }

    return 0;
}
```

输出：

```text
pass
```

### ③ 宏常量的优点

```text
简单直接。
不需要变量存储。
适合定义固定配置。
可用于条件编译。
```

### ④ 宏常量的问题

```text
没有类型检查。
只是文本替换。
调试时不如变量直观。
容易受优先级影响。
```

### ⑤ 宏表达式要加括号

不推荐：

```c
#define SQUARE(x) x * x
```

如果：

```c
SQUARE(1 + 2)
```

会展开成：

```c
1 + 2 * 1 + 2
```

结果错误。

正确：

```c
#define SQUARE(x) ((x) * (x))
```

> [!tip] 记忆口诀
> **define 是替换，不是变量；宏表达式多加括号，避免优先级坑。**

# 08 | 枚举常量：命名整数常量

> [!abstract] 核心结论
> `enum` 可以定义一组命名整数常量，让代码比直接写数字更清楚，常用于状态、选项、类型编号等场景。

### ① 基本写法

```c
enum Color {
    RED,
    GREEN,
    BLUE
};
```

默认值：

```text
RED   = 0
GREEN = 1
BLUE  = 2
```

### ② 示例代码

```c
#include <stdio.h>

enum Color {
    RED,
    GREEN,
    BLUE
};

int main() {
    enum Color c = GREEN;

    printf("%d\n", c);

    return 0;
}
```

输出：

```text
1
```

### ③ 可以手动指定值

```c
enum Status {
    SUCCESS = 0,
    ERROR = -1,
    TIMEOUT = 100
};
```

### ④ 适合场景

```text
状态码。
菜单选项。
颜色编号。
错误类型。
开关模式。
协议命令类型。
```

### ⑤ 比魔法数字更清楚

不推荐：

```c
if (status == 1) {
    printf("running\n");
}
```

更推荐：

```c
enum State {
    STOPPED,
    RUNNING,
    PAUSED
};

if (status == RUNNING) {
    printf("running\n");
}
```

> [!tip] 记忆口诀
> **enum 给整数起名字，状态选项更清楚。**

# 09 | 常量在内存中的位置

> [!abstract] 核心结论
> 不同类型的常量在内存中的位置不同。普通字面整数可能直接编码进指令，字符串常量通常在只读数据区，`const` 局部变量可能在栈区，`static const` 全局常量通常在只读数据区或静态区。

### ① 整数字面常量

```c
int a = 10;
```

这里的：

```text
10 可能直接作为立即数编码进机器指令。
```

也可以理解为：

```text
它不一定单独占一块内存。
```

### ② 字符串常量

```c
const char *s = "hello";
```

其中：

```text
"hello" 通常存放在只读数据区。
s 变量本身保存它的地址。
```

内存理解：

```text
只读数据区：
"hello\0"

栈区：
s = "hello" 的地址
```

### ③ const 局部变量

```c
void test() {
    const int x = 10;
}
```

通常：

```text
x 仍然可能作为局部变量存放在栈区。
```

注意：

```text
const 不一定代表放只读区。
```

### ④ static const 全局常量

```c
static const int MAX_SIZE = 100;
```

通常适合放在：

```text
只读数据区或静态存储区。
```

具体由编译器和平台决定。

### ⑤ 字符数组和字符串常量区别

```c
char *p = "hello";
```

理解：

```text
p 在栈区。
"hello" 在只读数据区。
```

```c
char arr[] = "hello";
```

理解：

```text
arr 是数组，通常在栈区。
数组内部保存一份可修改的 'h' 'e' 'l' 'l' 'o' '\0'。
```

> [!tip] 记忆口诀
> **整数常量可能进指令，字符串常量多在只读区；const 不等于一定只读区。**

# 10 | 常量总结

> [!abstract] 核心结论
> 常量的底层核心是“固定值 + 命名 + 保护 + 可读性”。字面常量直接表示值，`const` 提供类型和只读约束，`#define` 做文本替换，`enum` 给整数状态命名。理解它们的区别，可以写出更安全、更清晰的 C 代码。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>常量核心总结：</strong>
<ul>
  <li><strong>字面常量：</strong> 直接写在代码里的固定值，例如 <code>10</code>、<code>3.14</code>、<code>'A'</code>。</li>
  <li><strong>字符串常量：</strong> 双引号包裹，通常存放在只读数据区，不能修改。</li>
  <li><strong>整型常量：</strong> 可用十进制、八进制、十六进制表示。</li>
  <li><strong>字符常量：</strong> 单引号包裹，本质是字符编码值。</li>
  <li><strong>const：</strong> 有类型的只读约束，防止普通方式修改。</li>
  <li><strong>define：</strong> 预处理阶段文本替换，没有类型检查。</li>
  <li><strong>enum：</strong> 给整数常量起名字，适合状态和选项。</li>
  <li><strong>魔法数字：</strong> 不推荐大量直接写固定数字，应使用命名常量。</li>
  <li><strong>内存位置：</strong> 不同常量位置不同，字符串常量通常在只读数据区。</li>
  <li><strong>核心价值：</strong> 提高代码可读性、安全性和可维护性。</li>
</ul>
</div>

### ② 最底层一句话

```text
常量 = 程序中固定含义的数据；用名字管理常量，可以让代码更清楚、更安全。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **字面常量直接写，const 有类型不能改；define 预处理替换，enum 给整数起名；字符串常量多只读，想改就用字符数组。**
