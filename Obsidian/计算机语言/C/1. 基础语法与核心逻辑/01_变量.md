# 01 | C 语言变量：程序运行时可以改变的数据

> [!abstract] 核心结论
> 变量就是程序运行过程中可以被修改的数据容器。C 语言中的变量本质上是“内存空间 + 数据类型 + 变量名”的组合：变量名方便程序员使用，数据类型决定空间大小和解释方式，内存地址决定变量真正存放的位置。

### ① 底层原理：变量在内存中放在哪里

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>C 程序运行时的变量存放区域：</strong>
<ul>
  <li><strong>栈区 Stack：</strong> 存放局部变量、函数参数、返回地址。函数调用时创建，函数结束时自动销毁。</li>
  <li><strong>堆区 Heap：</strong> 存放动态申请的变量空间，例如 <code>malloc</code> 申请的空间。需要程序员手动释放。</li>
  <li><strong>全局区 / 静态区 Data / BSS：</strong> 存放全局变量、静态变量。程序启动时创建，程序结束时销毁。</li>
  <li><strong>常量区 / 只读数据区 Rodata：</strong> 存放字符串常量、部分只读数据，例如 <code>"Hello"</code>。</li>
  <li><strong>代码区 Text：</strong> 存放程序编译后的机器指令，不存普通变量。</li>
</ul>
</div>

### ② 变量的三要素

```text
变量 = 变量名 + 数据类型 + 内存空间
```

例如：

```c
int age = 18;
```

可以理解为：

```text
变量名：age
数据类型：int
变量值：18
内存空间：系统在内存中分配的一块空间
```

底层变化：

```text
程序运行到 int age = 18;
        ↓
根据 int 类型申请一块空间
        ↓
给这块空间起名叫 age
        ↓
把 18 存进去
```

### ③ 变量基本语法

```c
数据类型 变量名;
```

示例：

```c
int age;
double score;
char grade;
```

声明并初始化：

```c
int age = 18;
double score = 95.5;
char grade = 'A';
```

说明：

```text
声明：告诉编译器有这个变量。
初始化：第一次给变量赋值。
赋值：后续修改变量的值。
```

# 02 | 局部变量：函数内部的临时数据

> [!abstract] 核心结论
> 局部变量定义在函数内部，通常存放在栈区。函数调用时创建，函数结束时销毁，所以它的生命周期很短。

### ① 基本用法

```c
#include <stdio.h>

int main() {
    int age = 18;

    printf("%d\n", age);

    return 0;
}
```

输出：

```text
18
```

### ② 内存理解

```text
main 函数开始执行
        ↓
栈区创建 age
        ↓
age = 18
        ↓
printf 读取 age 的值
        ↓
main 函数结束
        ↓
age 被自动销毁
```

图示：

```text
栈区 Stack
┌────────────────────┐
│ age = 18            │
└────────────────────┘
```

### ③ 局部变量的特点

```text
1. 定义在函数内部。
2. 只能在当前函数内部使用。
3. 通常存放在栈区。
4. 函数执行时创建。
5. 函数结束时自动销毁。
6. 未初始化时，值是不确定的。
```

错误示例：

```c
#include <stdio.h>

int main() {
    int num;

    printf("%d\n", num);

    return 0;
}
```

说明：

```text
局部变量 num 没有初始化。
它里面可能是随机值。
所以使用局部变量前一定要先赋值。
```

# 03 | 全局变量：整个程序都能访问的数据

> [!abstract] 核心结论
> 全局变量定义在所有函数外部，通常存放在全局区 / 静态区。程序启动时创建，程序结束时销毁。

### ① 基本用法

```c
#include <stdio.h>

int global_num = 100;

int main() {
    printf("%d\n", global_num);

    return 0;
}
```

输出：

```text
100
```

### ② 多个函数共享全局变量

```c
#include <stdio.h>

int count = 0;

void add() {
    count++;
}

int main() {
    add();
    add();

    printf("%d\n", count);

    return 0;
}
```

输出：

```text
2
```

### ③ 内存理解

```text
程序加载到内存
        ↓
全局区创建 count
        ↓
count 初始化为 0
        ↓
main 调用 add
        ↓
add 修改全局变量 count
        ↓
程序结束
        ↓
count 被操作系统回收
```

图示：

```text
全局区 / 静态区
┌────────────────────┐
│ count = 2           │
└────────────────────┘
```

### ④ 全局变量的特点

```text
1. 定义在函数外部。
2. 可以被多个函数访问。
3. 生命周期贯穿整个程序。
4. 未显式初始化时，默认初始化为 0。
5. 滥用会降低代码可维护性。
```

# 04 | 静态变量 static：生命周期变长的变量

> [!abstract] 核心结论
> `static` 可以改变变量的生命周期或作用范围。局部静态变量定义在函数内部，但存放在全局区 / 静态区，不会随着函数结束而销毁。

### ① 局部静态变量

```c
#include <stdio.h>

void test() {
    static int count = 0;
    count++;

    printf("%d\n", count);
}

int main() {
    test();
    test();
    test();

    return 0;
}
```

输出：

```text
1
2
3
```

说明：

```text
count 虽然写在 test 函数内部，
但它不是每次调用都重新创建。
它只初始化一次，之后一直保留上次的值。
```

### ② 内存理解

```text
程序启动
        ↓
全局区 / 静态区准备 static count
        ↓
第一次调用 test：count = 1
        ↓
第二次调用 test：count = 2
        ↓
第三次调用 test：count = 3
        ↓
程序结束后 count 被回收
```

### ③ static 局部变量特点

```text
1. 作用域仍然在函数内部。
2. 生命周期贯穿整个程序。
3. 存放在全局区 / 静态区。
4. 只初始化一次。
5. 适合记录函数调用次数、保存状态。
```

# 05 | 动态变量：堆区中的手动申请空间

> [!abstract] 核心结论
> 动态变量不是通过普通变量声明创建，而是通过 `malloc`、`calloc`、`realloc` 在堆区申请空间。它的生命周期由程序员控制，必须手动 `free`。

### ① malloc 基本用法

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *p = malloc(sizeof(int));

    *p = 100;

    printf("%d\n", *p);

    free(p);

    return 0;
}
```

输出：

```text
100
```

### ② 内存理解

```text
int *p = malloc(sizeof(int));
```

可以理解为：

```text
p 是局部指针变量，放在栈区。
malloc 申请出来的 int 空间，放在堆区。
p 保存堆区空间的地址。
```

图示：

```text
栈区 Stack
┌────────────────────┐
│ p = 堆区地址         │
└─────────┬──────────┘
          │
          ▼
堆区 Heap
┌────────────────────┐
│ 100                 │
└────────────────────┘
```

### ③ 动态变量特点

```text
1. 通过 malloc 等函数申请。
2. 存放在堆区。
3. 不会随着函数结束自动释放。
4. 必须手动 free。
5. 如果忘记 free，会造成内存泄漏。
```

错误示例：

```c
int *p = malloc(sizeof(int));
*p = 100;
```

问题：

```text
没有 free(p)，堆区空间一直没有释放。
程序复杂时可能造成内存泄漏。
```

正确写法：

```c
int *p = malloc(sizeof(int));
*p = 100;
free(p);
p = NULL;
```

# 06 | 变量的作用域：变量在哪里能被访问

> [!abstract] 核心结论
> 作用域决定变量名在哪一段代码中有效。局部变量只在当前代码块中有效，全局变量从定义位置开始到文件结束有效。

### ① 代码块作用域

```c
#include <stdio.h>

int main() {
    int a = 10;

    {
        int b = 20;
        printf("%d\n", b);
    }

    printf("%d\n", a);

    return 0;
}
```

说明：

```text
a 在 main 函数内部有效。
b 只在内部大括号中有效。
```

错误写法：

```c
#include <stdio.h>

int main() {
    {
        int b = 20;
    }

    printf("%d\n", b);

    return 0;
}
```

原因：

```text
b 已经离开自己的作用域，不能再访问。
```

### ② 局部变量遮蔽全局变量

```c
#include <stdio.h>

int num = 100;

int main() {
    int num = 20;

    printf("%d\n", num);

    return 0;
}
```

输出：

```text
20
```

说明：

```text
局部变量和全局变量同名时，优先使用离当前代码最近的局部变量。
```

# 07 | 变量的生命周期：变量什么时候创建和销毁

> [!abstract] 核心结论
> 生命周期决定变量在内存中存在多久。局部变量跟随函数调用生灭，全局变量和静态变量贯穿整个程序，堆区变量由 malloc 和 free 控制。

### ① 生命周期对比

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #8E44AD; color: #333; margin-bottom: 10px;">
<strong>变量生命周期总览：</strong>
<ul>
  <li><strong>局部变量：</strong> 函数调用时创建，函数结束时销毁。</li>
  <li><strong>全局变量：</strong> 程序启动时创建，程序结束时销毁。</li>
  <li><strong>静态变量：</strong> 程序启动时创建，程序结束时销毁。</li>
  <li><strong>堆区变量：</strong> <code>malloc</code> 时创建，<code>free</code> 时释放。</li>
</ul>
</div>

### ② 示例代码

```c
#include <stdio.h>
#include <stdlib.h>

int global_num = 10;

void test() {
    static int static_num = 20;
    int local_num = 30;
    int *heap_num = malloc(sizeof(int));

    *heap_num = 40;

    printf("%d\n", global_num);
    printf("%d\n", static_num);
    printf("%d\n", local_num);
    printf("%d\n", *heap_num);

    free(heap_num);
}

int main() {
    test();

    return 0;
}
```

运行时变化：

```text
1. 程序启动，全局区创建 global_num。
2. 程序启动，全局区 / 静态区准备 static_num。
3. main 函数开始执行。
4. 调用 test 函数。
5. 栈区创建 local_num 和 heap_num。
6. malloc 在堆区申请 int 空间。
7. heap_num 保存堆区地址。
8. printf 输出各个变量。
9. free 释放堆区空间。
10. test 结束，local_num 和 heap_num 从栈区销毁。
11. main 结束，程序退出。
12. global_num 和 static_num 由操作系统回收。
```

# 08 | 变量初始化：第一次赋值很重要

> [!abstract] 核心结论
> 变量声明后不一定有可靠的值。局部变量如果没有初始化，值是随机的；全局变量和静态变量如果没有初始化，默认值为 0。

### ① 局部变量未初始化

```c
#include <stdio.h>

int main() {
    int a;

    printf("%d\n", a);

    return 0;
}
```

问题：

```text
a 是局部变量，未初始化时值不确定。
可能输出任意数字。
```

### ② 全局变量未初始化

```c
#include <stdio.h>

int global_num;

int main() {
    printf("%d\n", global_num);

    return 0;
}
```

输出：

```text
0
```

说明：

```text
全局变量未初始化时，默认是 0。
```

### ③ 静态变量未初始化

```c
#include <stdio.h>

void test() {
    static int count;

    printf("%d\n", count);
}

int main() {
    test();

    return 0;
}
```

输出：

```text
0
```

说明：

```text
静态变量未初始化时，默认是 0。
```

### ④ 推荐习惯

```c
int age = 0;
double score = 0.0;
char grade = '\0';
int *p = NULL;
```

说明：

```text
变量定义时尽量初始化。
指针变量不知道指向哪里时，先初始化为 NULL。
```

# 09 | 变量命名规则与命名规范

> [!abstract] 核心结论
> 变量名是内存空间的名字，命名要符合 C 语言语法规则，也要符合可读性规范。好的变量名能减少理解成本。

### ① 语法规则

```text
1. 只能由字母、数字、下划线组成。
2. 不能以数字开头。
3. 不能使用 C 语言关键字。
4. 区分大小写。
```

正确示例：

```c
int age;
int user_age;
int score1;
int _count;
```

错误示例：

```c
int 1age;
int user-age;
int int;
```

### ② 命名建议

```text
1. 见名知意。
2. 不要使用无意义的 a、b、c，除非是临时变量。
3. 多个单词建议用下划线连接。
4. 常量宏通常使用大写。
5. 指针变量可以用 p 开头，但不是必须。
```

推荐写法：

```c
int user_age = 18;
double total_price = 99.5;
char user_grade = 'A';
int *p_count = NULL;
```

# 10 | 变量常见错误与总结

> [!abstract] 核心结论
> C 语言变量最常见的问题集中在：未初始化、作用域越界、指针未分配空间、堆区忘记释放、局部变量返回地址。

### ① 常见错误

**错误 1：使用未初始化的局部变量**

```c
int a;
printf("%d\n", a);
```

问题：

```text
a 的值不确定，输出结果不可预测。
```

正确：

```c
int a = 0;
printf("%d\n", a);
```

---

**错误 2：访问已经离开作用域的变量**

```c
{
    int x = 10;
}

printf("%d\n", x);
```

问题：

```text
x 只在大括号内部有效，离开后不能访问。
```

---

**错误 3：指针没有指向有效空间就使用**

```c
int *p;
*p = 10;
```

问题：

```text
p 没有初始化，不知道指向哪里。
直接 *p = 10 很危险。
```

正确：

```c
int num = 0;
int *p = &num;
*p = 10;
```

或者：

```c
int *p = malloc(sizeof(int));
*p = 10;
free(p);
p = NULL;
```

---

**错误 4：malloc 后忘记 free**

```c
int *p = malloc(sizeof(int));
*p = 100;
```

问题：

```text
堆区空间没有释放，可能造成内存泄漏。
```

正确：

```c
int *p = malloc(sizeof(int));
*p = 100;
free(p);
p = NULL;
```

---

**错误 5：返回局部变量地址**

错误写法：

```c
int* test() {
    int a = 10;
    return &a;
}
```

问题：

```text
a 是局部变量，函数结束后就销毁。
返回它的地址会产生悬空指针。
```

### ② 变量总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>C 语言变量核心总结：</strong>
<ul>
  <li><strong>变量本质：</strong> 一块可以被读写的内存空间。</li>
  <li><strong>变量名：</strong> 内存空间的别名，方便程序员访问。</li>
  <li><strong>数据类型：</strong> 决定变量占多大空间，以及如何解释二进制数据。</li>
  <li><strong>局部变量：</strong> 通常在栈区，函数结束自动销毁。</li>
  <li><strong>全局变量 / 静态变量：</strong> 在全局区 / 静态区，程序结束才销毁。</li>
  <li><strong>动态变量：</strong> 在堆区，需要手动申请和释放。</li>
</ul>
</div>

> [!tip] 记忆口诀
> **变量就是内存格，类型决定怎么搁；局部上栈自动亡，全局静态程序长；malloc 进堆要 free，未初始化别乱用。**
