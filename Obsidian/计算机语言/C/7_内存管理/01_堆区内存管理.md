# 01 | 堆区内存管理：程序员手动申请和释放内存

> [!abstract] 核心结论
> 堆区内存管理就是程序运行时通过 `malloc`、`calloc`、`realloc` 动态申请内存，并通过 `free` 手动释放内存。堆区适合保存大小运行时才确定、生命周期需要自己控制的数据。

### ① 底层原理

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>堆区内存管理的本质：</strong>
<ul>
  <li><strong>堆区：</strong>程序运行时动态申请内存的区域。</li>
  <li><strong>申请：</strong>使用 <code>malloc</code>、<code>calloc</code>、<code>realloc</code>。</li>
  <li><strong>释放：</strong>使用 <code>free</code>。</li>
  <li><strong>特点：</strong>不会像栈变量一样函数结束自动释放。</li>
  <li><strong>核心风险：</strong>内存泄漏、野指针、重复释放、越界访问、释放后继续使用。</li>
</ul>
</div>

### ② 代码用法

```c
#include <stdlib.h>

int *p = malloc(sizeof(int));

if (p == NULL) {
    // 申请失败
}

*p = 100;

free(p);
p = NULL;
```

### ③ 使用场景

| 场景 | 为什么用堆区 |
| :--- | :--- |
| 数组大小运行时才知道 | 栈数组大小不方便固定 |
| 数据需要跨函数存在 | 栈变量函数结束就销毁 |
| 数据量较大 | 栈空间有限 |
| 链表、树、图 | 节点动态创建和删除 |
| 文件数据读取 | 文件大小运行时才知道 |

> [!tip] 记忆口诀
> **堆区自己申请，自己释放；malloc 要配 free。**

# 02 | malloc：申请指定字节数的堆空间

> [!abstract] 核心结论
> `malloc` 用来从堆区申请指定字节数的内存。申请成功返回这块内存的首地址，申请失败返回 `NULL`。`malloc` 申请到的内存内容不会自动清零。

### ① 基本语法

```c
void *malloc(size_t size);
```

含义：

```text
size：
要申请的字节数。

返回值：
成功返回堆空间首地址。
失败返回 NULL。
```

### ② 申请一个 int

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *p = malloc(sizeof(int));

    if (p == NULL) {
        printf("malloc failed\n");
        return 1;
    }

    *p = 100;

    printf("%d\n", *p);

    free(p);
    p = NULL;

    return 0;
}
```

输出：

```text
100
```

### ③ 底层理解

```c
int *p = malloc(sizeof(int));
```

分成两部分理解：

```text
p：
局部指针变量，通常在栈区。

malloc(sizeof(int))：
申请到的 int 空间在堆区。

p 保存堆区空间的首地址。
```

### ④ 申请数组

```c
int n = 10;

int *arr = malloc(sizeof(int) * n);
```

含义：

```text
从堆区申请能存放 10 个 int 的连续空间。
```

### ⑤ 重点

```text
malloc 只负责申请空间。
不会自动初始化为 0。
使用前最好主动赋值。
```

> [!tip] 记忆口诀
> **malloc 要字节数，成功给地址，失败给 NULL。**

# 03 | calloc：申请并清零

> [!abstract] 核心结论
> `calloc` 也用于申请堆区内存，但它会把申请到的内存初始化为 0。它常用于创建数组，并希望所有元素初始为 0 的场景。

### ① 基本语法

```c
void *calloc(size_t count, size_t size);
```

含义：

```text
count：
元素个数。

size：
每个元素大小。

总申请字节数：
count * size。
```

### ② 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int n = 5;

    int *arr = calloc(n, sizeof(int));

    if (arr == NULL) {
        printf("calloc failed\n");
        return 1;
    }

    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }

    free(arr);
    arr = NULL;

    return 0;
}
```

输出：

```text
0 0 0 0 0
```

### ③ 和 malloc 对比

```c
int *a = malloc(sizeof(int) * 5);
int *b = calloc(5, sizeof(int));
```

区别：

```text
malloc：
只申请空间，内容不确定。

calloc：
申请空间，并把每个字节初始化为 0。
```

### ④ 适合场景

```text
动态数组初始值要为 0。
结构体数组需要清空。
计数数组。
状态表。
缓冲区初始化。
```

### ⑤ 重点

```text
calloc 清零的是字节。
对于整数数组通常表现为 0。
```

> [!tip] 记忆口诀
> **malloc 只拿空间，calloc 拿空间还清零。**

# 04 | realloc：调整堆空间大小

> [!abstract] 核心结论
> `realloc` 用来改变已经申请的堆内存大小。它可能在原位置扩容，也可能重新申请一块新空间并复制旧数据，所以使用时要特别注意返回值。

### ① 基本语法

```c
void *realloc(void *ptr, size_t new_size);
```

含义：

```text
ptr：
原来的堆空间地址。

new_size：
新的字节数。

返回值：
调整后的空间地址。
```

### ② 基本用法

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *arr = malloc(sizeof(int) * 3);

    if (arr == NULL) {
        return 1;
    }

    for (int i = 0; i < 3; i++) {
        arr[i] = i + 1;
    }

    int *tmp = realloc(arr, sizeof(int) * 6);

    if (tmp == NULL) {
        free(arr);
        return 1;
    }

    arr = tmp;

    for (int i = 3; i < 6; i++) {
        arr[i] = i + 1;
    }

    for (int i = 0; i < 6; i++) {
        printf("%d ", arr[i]);
    }

    free(arr);
    arr = NULL;

    return 0;
}
```

输出：

```text
1 2 3 4 5 6
```

### ③ 为什么用临时指针

不推荐：

```c
arr = realloc(arr, new_size);
```

风险：

```text
如果 realloc 失败，返回 NULL。
原来的 arr 地址丢失。
原堆空间无法 free，造成内存泄漏。
```

推荐：

```c
int *tmp = realloc(arr, new_size);

if (tmp != NULL) {
    arr = tmp;
}
```

### ④ 扩容理解

```text
如果原空间后面还有足够空间：
可能原地扩容。

如果没有：
可能申请新空间，复制旧数据，释放旧空间。
```

### ⑤ 重点

```text
realloc 后，原指针可能失效。
一定使用返回的新地址。
```

> [!tip] 记忆口诀
> **realloc 会搬家，返回地址要接住；失败别丢老地址。**

# 05 | free：释放堆区内存

> [!abstract] 核心结论
> `free` 用来释放之前由 `malloc`、`calloc`、`realloc` 申请的堆内存。释放后，这块空间不再属于当前程序使用，不能继续访问。

### ① 基本语法

```c
void free(void *ptr);
```

含义：

```text
释放 ptr 指向的堆空间。
```

### ② 代码用法

```c
int *p = malloc(sizeof(int));

if (p == NULL) {
    return 1;
}

*p = 100;

free(p);
p = NULL;
```

### ③ free 后发生什么

```text
free(p)：
把 p 指向的堆空间归还给堆管理器。

p = NULL：
让 p 不再保存旧地址，避免野指针。
```

注意：

```text
free 不会自动把 p 变成 NULL。
```

### ④ 可以 free NULL

```c
int *p = NULL;

free(p);
```

这是安全的。

### ⑤ 重点

只能释放堆空间：

```c
int a = 10;
free(&a);  // 错误
```

原因：

```text
a 是栈变量。
不是 malloc 申请的堆空间。
```

> [!tip] 记忆口诀
> **谁 malloc，谁 free；free 后置 NULL。**

# 06 | 堆区和栈区的区别

> [!abstract] 核心结论
> 栈区由系统自动管理，堆区由程序员手动管理。局部变量通常在栈区，`malloc` 申请的空间在堆区。

### ① 栈区变量

```c
void test(void) {
    int a = 10;
}
```

特点：

```text
进入函数创建。
函数结束销毁。
自动管理。
速度快。
空间有限。
```

### ② 堆区变量

```c
void test(void) {
    int *p = malloc(sizeof(int));
}
```

特点：

```text
malloc 时创建。
free 时释放。
手动管理。
适合动态大小数据。
如果忘记 free，会泄漏。
```

### ③ 关键区别

```c
int *p = malloc(sizeof(int));
```

这句里：

```text
p 本身：
局部指针变量，通常在栈区。

*p 指向的空间：
malloc 申请的空间，在堆区。
```

### ④ 对比表

| 对比项 | 栈区 | 堆区 |
| :--- | :--- | :--- |
| 管理方式 | 自动管理 | 手动管理 |
| 创建方式 | 定义局部变量 | malloc / calloc / realloc |
| 释放方式 | 函数结束自动释放 | free 手动释放 |
| 生命周期 | 通常较短 | 可由程序员控制 |
| 常见风险 | 返回局部变量地址 | 内存泄漏、野指针、重复释放 |

### ⑤ 重点

```text
指针变量在哪里，和它指向的空间在哪里，是两回事。
```

> [!tip] 记忆口诀
> **指针可能在栈，空间可能在堆；栈自动，堆手动。**

# 07 | 堆区内存泄漏

> [!abstract] 核心结论
> 内存泄漏是指堆空间申请后没有释放，且程序已经失去访问它的指针。程序短时间运行可能不明显，长期运行会导致内存越来越少。

### ① 错误示例

```c
void test(void) {
    int *p = malloc(sizeof(int));

    *p = 100;

    // 忘记 free(p)
}
```

问题：

```text
test 函数结束后，p 这个局部指针消失。
但 malloc 申请的堆空间还在。
从此没有指针能找到它。
```

这就是：

```text
内存泄漏。
```

### ② 更隐蔽的泄漏

```c
int *p = malloc(sizeof(int));
p = malloc(sizeof(int));
```

问题：

```text
第一次 malloc 的地址被覆盖。
第一次申请的堆空间无法释放。
```

正确：

```c
int *p = malloc(sizeof(int));

free(p);

p = malloc(sizeof(int));
```

### ③ 循环中的泄漏

```c
while (1) {
    int *p = malloc(sizeof(int));
}
```

问题：

```text
循环不断申请堆空间。
从不释放。
内存越来越少。
```

### ④ 避免方法

```text
每次 malloc 后，想清楚在哪里 free。
指针重新赋值前，先释放旧空间。
函数异常返回前，也要释放已经申请的空间。
```

### ⑤ 重点

```text
堆空间不会因为函数结束自动释放。
只要没 free，它就一直占着。
```

> [!tip] 记忆口诀
> **堆区忘 free，就像借房不退；指针丢了，房也找不到。**

# 08 | 野指针、悬空指针和重复释放

> [!abstract] 核心结论
> 堆区内存管理中最危险的问题之一是指针指向无效空间。释放后继续使用叫悬空指针使用，未初始化指针叫野指针，对同一块空间多次 `free` 叫重复释放。

### ① 野指针

```c
int *p;

*p = 10;  // 错误
```

问题：

```text
p 没有初始化。
它可能指向随机地址。
```

正确：

```c
int *p = NULL;
```

### ② 悬空指针

```c
int *p = malloc(sizeof(int));

*p = 10;

free(p);

printf("%d\n", *p);  // 错误
```

问题：

```text
p 指向的堆空间已经释放。
继续访问危险。
```

### ③ 避免悬空指针

```c
free(p);
p = NULL;
```

之后：

```c
if (p != NULL) {
    printf("%d\n", *p);
}
```

### ④ 重复释放

```c
int *p = malloc(sizeof(int));

free(p);
free(p);  // 错误
```

正确：

```c
free(p);
p = NULL;

free(p);  // 安全，因为 free(NULL) 安全
```

### ⑤ 重点

```text
未初始化指针不要用。
free 后不要再用。
同一块堆空间只能释放一次。
```

> [!tip] 记忆口诀
> **指针先初始化，free 后置 NULL；不用野指针，不碰悬空针。**

# 09 | 堆区内存越界访问

> [!abstract] 核心结论
> 堆区申请多少空间，就只能访问多少空间。访问超过申请范围的内存叫越界访问，可能破坏其他数据，导致程序崩溃或产生难查的错误。

### ① 错误示例

```c
int *arr = malloc(sizeof(int) * 5);

for (int i = 0; i <= 5; i++) {
    arr[i] = i;
}

free(arr);
```

问题：

```text
arr 只有 5 个元素，下标范围是 0 ~ 4。
i == 5 时访问 arr[5]，越界。
```

### ② 正确写法

```c
int *arr = malloc(sizeof(int) * 5);

if (arr == NULL) {
    return 1;
}

for (int i = 0; i < 5; i++) {
    arr[i] = i;
}

free(arr);
arr = NULL;
```

### ③ 字符串空间不足

错误：

```c
char *s = malloc(5);

strcpy(s, "hello");
```

问题：

```text
"hello" 需要 6 个字节：
h e l l o \0
```

正确：

```c
char *s = malloc(6);

strcpy(s, "hello");
```

### ④ 常见原因

```text
数组下标写错。
忘记字符串结尾 \0。
sizeof 用错。
申请元素个数和访问元素个数不一致。
```

### ⑤ 重点

```text
malloc 的参数是字节数。
访问时要严格控制范围。
```

> [!tip] 记忆口诀
> **申请几个用几个，字符串别忘反斜零。**

# 10 | 堆区内存管理总结

> [!abstract] 核心结论
> 堆区内存管理的底层核心是“程序员手动控制内存生命周期”。用 `malloc/calloc/realloc` 申请，用 `free` 释放；申请后检查 `NULL`，释放后置 `NULL`，避免泄漏、越界、重复释放和悬空指针。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>堆区内存管理核心总结：</strong>
<ul>
  <li><strong>堆区：</strong>运行时动态申请内存的区域。</li>
  <li><strong><code>malloc</code>：</strong>申请指定字节数，内容不自动清零。</li>
  <li><strong><code>calloc</code>：</strong>申请多个元素空间，并初始化为 0。</li>
  <li><strong><code>realloc</code>：</strong>调整已申请堆空间大小。</li>
  <li><strong><code>free</code>：</strong>释放堆空间。</li>
  <li><strong>检查返回值：</strong>申请失败会返回 <code>NULL</code>。</li>
  <li><strong>释放后：</strong>建议把指针置为 <code>NULL</code>。</li>
  <li><strong>内存泄漏：</strong>申请后没释放，或指针丢失导致无法释放。</li>
  <li><strong>悬空指针：</strong>释放后继续使用原地址。</li>
  <li><strong>越界访问：</strong>访问超过申请范围的内存。</li>
</ul>
</div>

### ② 最底层一句话

```text
堆区内存 = 程序运行时向系统借来的空间；malloc 借，free 还，不还会泄漏，乱用会崩溃。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **malloc 申请，calloc 清零，realloc 扩缩，free 释放；检查 NULL，释放置空，别泄漏别越界。**
