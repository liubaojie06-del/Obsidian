# 01 | 顺序表是什么：用一段连续内存存放线性数据

> [!abstract] 核心结论  
> 顺序表就是“数组 + 当前长度”的组合。它用连续内存保存元素，适合按位置快速访问。

### ① 底层原理

顺序表的本质是线性表的一种存储方式。

在 C 语言里，最常见的顺序表就是用数组实现：

```text
下标：  0    1    2    3    4
数据： 10   20   30   40   50
内存：连续  连续  连续  连续  连续
```

数组元素在内存中是连续存放的，所以只要知道第一个元素地址，就可以通过下标快速找到任意元素。

比如 `a[3]` 的地址本质上是：

```text
首地址 + 3 * 每个元素占用的字节数
```

---

### ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int main(void) {
    SeqList list = {{10, 20, 30}, 3};

    printf("第一个元素: %d\n", list.data[0]);
    printf("当前长度: %d\n", list.length);

    return 0;
}
```

---

### ③ 含义 / 说明

```text
#define MAX_SIZE 100
表示顺序表最多能存 100 个 int 元素。

typedef struct { ... } SeqList;
定义一个顺序表类型。

int data[MAX_SIZE];
真正存数据的数组。

int length;
记录当前顺序表里已经存了几个元素。

SeqList list = {{10, 20, 30}, 3};
初始化顺序表，前三个位置有数据，当前长度是 3。
```

---

### ④ 输出

```text
第一个元素: 10
当前长度: 3
```

---

### ⑤ 重点 / 注意 / 常见错误

顺序表不是只有数组，还必须有 `length`。

错误理解：

```c
int data[100];
```

这只是数组，不知道当前有多少有效数据。

正确做法：

```c
typedef struct {
    int data[100];
    int length;
} SeqList;
```

---

> [!tip] 记忆口诀  
> **顺序表，数组装，length 记住有多长。**

# 02 | 顺序表的结构定义：数组负责存储，length 负责管理

> [!abstract] 核心结论  
> 顺序表通常用结构体封装数组和长度。数组管“空间”，length 管“有效元素个数”。

### ① 底层原理

数组本身不会告诉你哪些位置是有效数据。

例如：

```text
data[0] data[1] data[2] data[3] ...
  10      20      30      ?    ...
```

如果 `length = 3`，就表示只有下标 `0 ~ 2` 是有效数据。

`data[3]` 后面的内容不应该随便访问。

---

### ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int main(void) {
    SeqList list;

    list.data[0] = 11;
    list.data[1] = 22;
    list.length = 2;

    for (int i = 0; i < list.length; i++) {
        printf("%d ", list.data[i]);
    }

    return 0;
}
```

---

### ③ 含义 / 说明

```text
SeqList list;
创建一个顺序表变量。

list.data[0] = 11;
把第一个有效元素设置为 11。

list.data[1] = 22;
把第二个有效元素设置为 22。

list.length = 2;
说明当前只有 2 个有效元素。

for 循环只遍历到 length，不遍历整个数组。
```

---

### ④ 输出

```text
11 22
```

---

### ⑤ 重点 / 注意 / 常见错误

不要把 `MAX_SIZE` 当成当前长度。

|名称|含义|
|---|---|
|`MAX_SIZE`|顺序表最大容量|
|`length`|当前有效元素个数|
|`data[i]`|第 `i` 个数组位置|

错误写法：

```c
for (int i = 0; i < MAX_SIZE; i++) {
    printf("%d ", list.data[i]);
}
```

正确写法：

```c
for (int i = 0; i < list.length; i++) {
    printf("%d ", list.data[i]);
}
```

---

> [!tip] 记忆口诀  
> **容量看 MAX，个数看 length。**

# 03 | 初始化顺序表：先把 length 设为 0

> [!abstract] 核心结论  
> 初始化顺序表最重要的动作是把 `length` 设为 0。否则程序不知道顺序表当前有多少有效元素。

### ① 底层原理

在 C 语言中，局部变量不会自动清零。

如果你写：

```c
SeqList list;
```

里面的 `length` 可能是随机值。

这会导致遍历、插入、删除都出错。

所以初始化时必须明确：

```c
list.length = 0;
```

---

### ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

void initList(SeqList *list) {
    list->length = 0;
}

int main(void) {
    SeqList list;

    initList(&list);

    printf("初始化后的长度: %d\n", list.length);

    return 0;
}
```

---

### ③ 含义 / 说明

```text
void initList(SeqList *list)
使用指针接收顺序表地址。

list->length = 0;
通过指针修改原来的顺序表。

initList(&list);
把 list 的地址传给函数。

如果不用指针，函数里改的是副本，外面的 list 不会变。
```

---

### ④ 输出

```text
初始化后的长度: 0
```

---

### ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
void initList(SeqList list) {
    list.length = 0;
}
```

这个函数改的是拷贝，不是原来的顺序表。

正确写法：

```c
void initList(SeqList *list) {
    list->length = 0;
}
```

---

> [!tip] 记忆口诀  
> **要改原表传地址，初始化先清零。**

# 04 | 顺序表插入：先搬元素，再放新值

> [!abstract] 核心结论  
> 顺序表插入元素时，需要把插入位置后面的元素整体后移。因为数组内存连续，不能直接“挤进去”。

### ① 底层原理

假设顺序表是：

```text
下标：0   1   2   3
数据：10  20  30  40
```

如果要在下标 `1` 插入 `99`，必须先把后面的元素后移：

```text
10  20  30  40
10  20  30  40  40
10  20  30  30  40
10  20  20  30  40
10  99  20  30  40
```

移动时必须从后往前移动，避免覆盖数据。

---

### ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int insert(SeqList *list, int index, int value) {
    if (list->length >= MAX_SIZE) {
        return 0;
    }

    if (index < 0 || index > list->length) {
        return 0;
    }

    for (int i = list->length; i > index; i--) {
        list->data[i] = list->data[i - 1];
    }

    list->data[index] = value;
    list->length++;

    return 1;
}

int main(void) {
    SeqList list = {{10, 20, 30}, 3};

    insert(&list, 1, 99);

    for (int i = 0; i < list.length; i++) {
        printf("%d ", list.data[i]);
    }

    return 0;
}
```

---

### ③ 含义 / 说明

```text
insert(&list, 1, 99);
在下标 1 的位置插入 99。

if (list->length >= MAX_SIZE)
判断顺序表是否已经满了。

if (index < 0 || index > list->length)
判断插入位置是否合法。

for 循环从后往前移动元素。
这样不会把原来的数据覆盖掉。

list->data[index] = value;
把新值放到目标位置。

list->length++;
有效元素个数加 1。
```

---

### ④ 输出

```text
10 99 20 30
```

---

### ⑤ 重点 / 注意 / 常见错误

插入时移动方向不能错。

错误写法：

```c
for (int i = index; i < list->length; i++) {
    list->data[i + 1] = list->data[i];
}
```

这样会把后面的数据覆盖掉。

正确写法：

```c
for (int i = list->length; i > index; i--) {
    list->data[i] = list->data[i - 1];
}
```

---

> [!tip] 记忆口诀  
> **插入先后移，移动从尾起。**

---

# 05 | 顺序表删除：先覆盖元素，再减少长度

> [!abstract] 核心结论  
> 删除顺序表中的元素时，不是真的清空内存，而是用后面的元素向前覆盖。最后让 `length--` 即可。

### ① 底层原理

顺序表删除下标 `1` 的元素：

```text
删除前：10  20  30  40
删除后：10  30  40
```

底层其实是：

```text
data[1] = data[2]
data[2] = data[3]
length--
```

原来最后一个位置可能还残留旧值，但因为 `length` 变小了，它已经不是有效数据。

---

### ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int deleteAt(SeqList *list, int index) {
    if (index < 0 || index >= list->length) {
        return 0;
    }

    for (int i = index; i < list->length - 1; i++) {
        list->data[i] = list->data[i + 1];
    }

    list->length--;

    return 1;
}

int main(void) {
    SeqList list = {{10, 20, 30, 40}, 4};

    deleteAt(&list, 1);

    for (int i = 0; i < list.length; i++) {
        printf("%d ", list.data[i]);
    }

    return 0;
}
```

---

### ③ 含义 / 说明

```text
deleteAt(&list, 1);
删除下标 1 的元素，也就是 20。

if (index < 0 || index >= list->length)
判断删除位置是否合法。

for 循环从前往后移动元素。
把后一个元素覆盖到前一个位置。

list->length--;
有效元素个数减少 1。
```

---

### ④ 输出

```text
10 30 40
```

---

### ⑤ 重点 / 注意 / 常见错误

删除时不需要手动把最后一个元素清零。

错误理解：

```c
list->data[list->length - 1] = 0;
```

这不是必须的。

因为顺序表判断有效数据只看 `length`。

---

> [!tip] 记忆口诀  
> **删除往前盖，长度减下来。**

---

# 06 | 顺序表查找：按下标快，按值要遍历

> [!abstract] 核心结论  
> 顺序表按下标访问非常快，因为数组支持随机访问。按值查找通常要从头到尾一个个比较。

### ① 底层原理

顺序表的内存是连续的，所以访问 `data[index]` 很快。

但是如果只知道一个值，比如要找 `30`，程序不知道它在哪个下标，只能遍历：

```text
10 -> 20 -> 30 -> 40
```

找到后返回下标。

---

### ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int find(SeqList *list, int value) {
    for (int i = 0; i < list->length; i++) {
        if (list->data[i] == value) {
            return i;
        }
    }

    return -1;
}

int main(void) {
    SeqList list = {{10, 20, 30, 40}, 4};

    int index = find(&list, 30);

    printf("30 的下标是: %d\n", index);

    return 0;
}
```

---

### ③ 含义 / 说明

```text
find(&list, 30);
查找值 30。

for 循环从下标 0 开始遍历。

if (list->data[i] == value)
如果当前元素等于目标值，就返回下标。

return -1;
如果找不到，返回 -1。
```

---

### ④ 输出

```text
30 的下标是: 2
```

---

### ⑤ 重点 / 注意 / 常见错误

下标从 `0` 开始，不是从 `1` 开始。

|逻辑位置|C 数组下标|
|---|---|
|第 1 个元素|`0`|
|第 2 个元素|`1`|
|第 3 个元素|`2`|

---

> [!tip] 记忆口诀  
> **知道下标直接拿，只知数值慢慢查。**

---

# 07 | 顺序表遍历：只遍历有效长度，不遍历最大容量

> [!abstract] 核心结论  
> 遍历顺序表时，循环条件应该是 `i < length`。不要遍历整个数组容量，否则可能读到无效数据。

### ① 底层原理

顺序表数组可能很大，但有效数据可能很少。

```text
MAX_SIZE = 100
length = 3

有效数据：data[0] data[1] data[2]
无效区域：data[3] data[4] ...
```

无效区域里的值可能是随机垃圾值，也可能是旧数据。

---

### ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 10

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

void printList(SeqList *list) {
    for (int i = 0; i < list->length; i++) {
        printf("%d ", list->data[i]);
    }
    printf("\n");
}

int main(void) {
    SeqList list = {{5, 10, 15}, 3};

    printList(&list);

    return 0;
}
```

---

### ③ 含义 / 说明

```text
void printList(SeqList *list)
传入顺序表地址，避免复制整个结构体。

i < list->length
只访问有效元素。

printf("%d ", list->data[i]);
打印当前元素。
```

---

### ④ 输出

```text
5 10 15
```

---

### ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
for (int i = 0; i < MAX_SIZE; i++) {
    printf("%d ", list->data[i]);
}
```

这样会打印很多无效数据。

正确写法：

```c
for (int i = 0; i < list->length; i++) {
    printf("%d ", list->data[i]);
}
```

---

> [!tip] 记忆口诀  
> **遍历看 length，不看大容量。**

---

# 08 | 顺序表的优缺点：访问快，插删慢

> [!abstract] 核心结论  
> 顺序表最大的优点是按下标访问快。最大的缺点是插入和删除可能要移动大量元素。

### ① 底层原理

因为顺序表底层是数组，所以它有数组的特点。

按下标访问时：

```text
首地址 + 下标 * 元素大小
```

可以一步算到目标位置。

但是插入和删除时，为了保持元素连续，就必须移动元素。

---

### ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int main(void) {
    SeqList list = {{1, 2, 3, 4, 5}, 5};

    printf("按下标访问 list.data[3]: %d\n", list.data[3]);

    return 0;
}
```

---

### ③ 含义 / 说明

```text
list.data[3]
直接访问下标 3 的元素。

因为数组内存连续，所以访问速度很快。

但是如果在开头插入元素，后面的元素都要后移。
```

---

### ④ 输出

```text
按下标访问 list.data[3]: 4
```

---

### ⑤ 重点 / 注意 / 常见错误

|操作|特点|
|---|---|
|按下标访问|快|
|按值查找|慢，需要遍历|
|尾部插入|较快|
|中间插入|慢，需要后移|
|中间删除|慢，需要前移|

顺序表适合：

```text
经常访问元素
较少插入删除
数据规模固定或变化不大
```

不适合：

```text
频繁在中间插入删除
数据规模经常变化
```

---

> [!tip] 记忆口诀  
> **查得快，插删累，连续内存是关键。**

---

# 09 | 顺序表和链表对比：一个靠连续内存，一个靠指针连接

> [!abstract] 核心结论  
> 顺序表适合随机访问，链表适合频繁插入删除。两者没有绝对好坏，关键看使用场景。

### ① 底层原理

顺序表：

```text
元素在内存中连续存放。
通过下标计算地址。
```

链表：

```text
元素不一定连续。
每个节点用指针连接下一个节点。
```

顺序表像一排连续座位，链表像每个人手里拿着下一个人的地址。

---

### ② 基本语法 / 代码用法

```c
#include <stdio.h>

int main(void) {
    int arr[3] = {10, 20, 30};

    printf("arr[0] 地址: %p\n", (void *)&arr[0]);
    printf("arr[1] 地址: %p\n", (void *)&arr[1]);
    printf("arr[2] 地址: %p\n", (void *)&arr[2]);

    return 0;
}
```

---

### ③ 含义 / 说明

```text
&arr[0]
第一个元素的地址。

&arr[1]
第二个元素的地址。

int 类型通常占 4 字节。
所以连续元素的地址通常相差 4。
```

---

### ④ 输出

```text
arr[0] 地址: 0x...
arr[1] 地址: 0x...
arr[2] 地址: 0x...
```

---

### ⑤ 重点 / 注意 / 常见错误

|对比项|顺序表|链表|
|---|---|---|
|内存|连续|不一定连续|
|访问元素|快，可下标访问|慢，需要从头找|
|插入删除|可能要移动元素|改指针即可|
|空间占用|只存数据|还要存指针|
|实现难度|较简单|指针更容易出错|

初学阶段先学顺序表，可以更好理解数组、下标、长度和内存连续性。

---

> [!tip] 记忆口诀  
> **顺序靠数组，链表靠指针。**

---

# 10 | 顺序表常见错误：越界、忘改长度、传参传错

> [!abstract] 核心结论  
> 顺序表最常见的问题都和 `length` 有关。写代码时要时刻检查容量、下标和长度变化。

### ① 底层原理

C 语言不会自动帮你检查数组越界。

如果你访问：

```c
data[100]
```

但数组最大下标只有 `99`，程序可能不会立刻报错，却可能破坏别的内存。

这就是 C 语言危险但高效的地方：它相信程序员自己负责。

---

### ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 3

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int append(SeqList *list, int value) {
    if (list->length >= MAX_SIZE) {
        return 0;
    }

    list->data[list->length] = value;
    list->length++;

    return 1;
}

int main(void) {
    SeqList list = {{0}, 0};

    append(&list, 10);
    append(&list, 20);
    append(&list, 30);

    if (!append(&list, 40)) {
        printf("插入失败：顺序表已满\n");
    }

    return 0;
}
```

---

### ③ 含义 / 说明

```text
MAX_SIZE 是 3，最多只能存 3 个元素。

append(&list, 10);
在尾部插入 10。

list->data[list->length] = value;
把新元素放到当前 length 指向的位置。

list->length++;
插入成功后，长度加 1。

第 4 次插入时，length 已经等于 MAX_SIZE。
所以插入失败。
```

---

### ④ 输出

```text
插入失败：顺序表已满
```

---

### ⑤ 重点 / 注意 / 常见错误

常见错误 1：数组越界

```c
list.data[MAX_SIZE] = 10;
```

正确理解：

```text
数组最大下标是 MAX_SIZE - 1。
```

常见错误 2：插入后忘记修改长度

```c
list->data[list->length] = value;
```

正确写法：

```c
list->data[list->length] = value;
list->length++;
```

常见错误 3：函数传值导致修改无效

错误写法：

```c
void append(SeqList list, int value) {
    list.data[list.length] = value;
    list.length++;
}
```

正确写法：

```c
void append(SeqList *list, int value) {
    list->data[list->length] = value;
    list->length++;
}
```

> **查边界，改 length，传地址。**

# 11 | 顺序表主题总结：连续内存带来快访问，也带来搬元素

> [!abstract] 核心结论  
> 顺序表的核心就是用数组保存数据，用 `length` 管理有效长度。它访问快，但插入删除可能慢。

### ① 底层原理

顺序表的三个关键词：

```text
连续内存
随机访问
长度管理
```

它快，是因为数组能通过下标直接算地址。

它慢，是因为插入删除时，为了保持连续性，要移动元素。

---

### ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

void initList(SeqList *list) {
    list->length = 0;
}

int append(SeqList *list, int value) {
    if (list->length >= MAX_SIZE) {
        return 0;
    }

    list->data[list->length] = value;
    list->length++;

    return 1;
}

void printList(SeqList *list) {
    for (int i = 0; i < list->length; i++) {
        printf("%d ", list->data[i]);
    }
    printf("\n");
}

int main(void) {
    SeqList list;

    initList(&list);

    append(&list, 10);
    append(&list, 20);
    append(&list, 30);

    printList(&list);

    return 0;
}
```

---

### ③ 含义 / 说明

```text
initList
负责初始化顺序表。

append
负责尾部插入元素。

printList
负责遍历并打印有效元素。

main
把初始化、插入、遍历串起来。
```

### ④ 输出

```text
10 20 30
```

### ⑤ 重点 / 注意 / 常见错误

学习顺序表时要抓住这几个问题：

|问题|关键点|
|---|---|
|数据放哪里|放在数组里|
|当前有几个|看 length|
|能不能继续插|看 length 是否小于 MAX_SIZE|
|为什么访问快|内存连续，可直接算地址|
|为什么插删慢|要移动元素|
|函数如何修改表|传结构体指针|
