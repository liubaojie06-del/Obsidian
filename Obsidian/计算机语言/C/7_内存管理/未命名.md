# 01 | 顺序表的基本概念：用数组实现的线性表

> [!abstract] 核心结论  
> 顺序表本质上是“数组 + 长度”。数组负责存数据，长度负责说明当前有多少个有效元素。

## ① 底层原理

> 顺序表的本质：
> 
> - **连续内存**：所有元素挨着存放。
>     
> - **随机访问**：知道下标就能直接访问元素。
>     
> - **长度管理**：用 `length` 记录当前有效元素个数。
>     
> - **容量限制**：数组大小固定，不能无限插入。
>     

顺序表底层通常长这样：

```text
下标：  0    1    2    3    4
数据： 10   20   30   40   50
地址：连续 连续 连续 连续 连续
```

在 C 语言中，数组名代表数组首元素地址。

所以访问 `data[i]` 时，底层会计算：

```text
data[i] 的地址 = 首地址 + i * sizeof(元素类型)
```

这也是顺序表按下标访问很快的原因。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int main(void) {
    SeqList list = {{10, 20, 30}, 3};

    printf("第 1 个元素: %d\n", list.data[0]);
    printf("当前长度: %d\n", list.length);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
#define MAX_SIZE 100
定义顺序表的最大容量。

int data[MAX_SIZE];
真正存放数据的数组。

int length;
记录当前有效元素个数。

SeqList list = {{10, 20, 30}, 3};
数组前三个位置存了 10、20、30，length 表示当前有 3 个元素。

list.data[0]
访问第 1 个元素。注意 C 语言下标从 0 开始。
```

---

## ④ 输出

```text
第 1 个元素: 10
当前长度: 3
```

---

## ⑤ 重点 / 注意 / 常见错误

> 顺序表不是单纯的数组，而是数组和长度一起管理。

错误理解：

```c
int data[100];
```

这只是数组，不知道当前有多少个有效数据。

正确写法：

```c
typedef struct {
    int data[100];
    int length;
} SeqList;
```

|名称|作用|
|---|---|
|`data`|存放元素|
|`length`|记录有效元素个数|
|`MAX_SIZE`|限制最大容量|

> [!tip] 记忆口诀  
> **数组负责装，length 负责管。**

# 02 | 顺序表的结构定义：把数组和长度封装到结构体里

> [!abstract] 核心结论  
> 顺序表通常用 `struct` 定义。这样可以把数据数组和当前长度绑定在一起，避免管理混乱。

## ① 底层原理

> 结构体的作用：
> 
> - 把多个相关变量打包成一个整体。
>     
> - 让顺序表既有存储空间，又有长度信息。
>     
> - 函数传参时可以整体操作这张表。
>     

如果只用数组：

```c
int data[100];
```

程序只知道有 100 个位置，但不知道哪些位置已经存了有效数据。

所以需要：

```c
typedef struct {
    int data[100];
    int length;
} SeqList;
```

这样 `SeqList` 就变成了一个完整的顺序表类型。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int main(void) {
    SeqList list;

    list.data[0] = 11;
    list.data[1] = 22;
    list.length = 2;

    for (int i = 0; i < list.length; i++) {
        printf("%d ", list.data[i]);
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
SeqList list;
创建一个顺序表变量。

list.data[0] = 11;
把 11 放到数组下标 0 的位置。

list.data[1] = 22;
把 22 放到数组下标 1 的位置。

list.length = 2;
说明当前只有 2 个有效元素。

for 循环使用 list.length。
这样只访问有效数据，不访问无效空间。
```

---

## ④ 输出

```text
11 22
```

---

## ⑤ 重点 / 注意 / 常见错误

> `MAX_SIZE` 是最大容量，`length` 是当前长度，不能混用。

|对比项|含义|
|---|---|
|`MAX_SIZE`|最多能放多少个元素|
|`length`|现在已经放了多少个元素|
|`data[i]`|第 `i` 个数组位置|

错误写法：

```c
for (int i = 0; i < MAX_SIZE; i++) {
    printf("%d ", list.data[i]);
}
```

正确写法：

```c
for (int i = 0; i < list.length; i++) {
    printf("%d ", list.data[i]);
}
```

> [!tip] 记忆口诀  
> **容量看 MAX，个数看 length。**

# 03 | 顺序表初始化：先把 length 设置为 0

> [!abstract] 核心结论  
> 初始化顺序表时，最关键的是把 `length` 设为 `0`。否则后续遍历、插入、删除都会乱掉。

## ① 底层原理

> C 语言的局部变量不会自动清零。

如果直接写：

```c
SeqList list;
```

此时 `list.length` 里面可能是随机值。

如果随机值很大，程序可能误以为顺序表里已经有很多元素，然后访问非法内存。

所以初始化必须明确写：

```c
list.length = 0;
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

void initList(SeqList *list) {
    list->length = 0;
}

int main(void) {
    SeqList list;

    initList(&list);

    printf("初始化后的长度: %d\n", list.length);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
void initList(SeqList *list)
函数接收顺序表的地址。

SeqList *list
list 是一个指针，指向外面的顺序表变量。

list->length = 0;
通过指针修改原来的顺序表。

initList(&list);
把 list 的地址传给函数。
```

---

## ④ 输出

```text
初始化后的长度: 0
```

---

## ⑤ 重点 / 注意 / 常见错误

> 要在函数里修改原来的顺序表，必须传地址。

错误写法：

```c
void initList(SeqList list) {
    list.length = 0;
}
```

这个函数改的是副本，外面的 `list` 不会变。

正确写法：

```c
void initList(SeqList *list) {
    list->length = 0;
}
```

|写法|是否能修改原表|
|---|---|
|`SeqList list`|不能，传的是副本|
|`SeqList *list`|能，传的是地址|

> [!tip] 记忆口诀  
> **初始化清长度，改原表传地址。**

# 04 | 顺序表尾插：把新元素放到 length 指向的位置

> [!abstract] 核心结论  
> 尾插是顺序表最简单的插入方式。新元素直接放到 `data[length]`，然后 `length++`。

## ① 底层原理

假设当前顺序表是：

```text
下标：  0    1    2    3
数据： 10   20   30   ?
length = 3
```

此时 `length` 的值是 `3`，正好指向下一个可插入位置：

```text
data[3] = 新元素
length++
```

尾插不需要移动其他元素，所以效率比较高。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int append(SeqList *list, int value) {
    if (list->length >= MAX_SIZE) {
        return 0;
    }

    list->data[list->length] = value;
    list->length++;

    return 1;
}

int main(void) {
    SeqList list = {{10, 20}, 2};

    append(&list, 30);

    for (int i = 0; i < list.length; i++) {
        printf("%d ", list.data[i]);
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
append(&list, 30);
向顺序表尾部插入 30。

if (list->length >= MAX_SIZE)
插入前先判断是否已满。

list->data[list->length] = value;
把新元素放到当前长度对应的位置。

list->length++;
插入成功后，长度加 1。
```

---

## ④ 输出

```text
10 20 30
```

---

## ⑤ 重点 / 注意 / 常见错误

> 插入前一定要判断容量，插入后一定要修改长度。

错误写法：

```c
list->data[list->length] = value;
```

问题是忘了：

```c
list->length++;
```

正确写法：

```c
list->data[list->length] = value;
list->length++;
```

> [!tip] 记忆口诀  
> **尾插看 length，放完 length 加一。**

# 05 | 顺序表按位置插入：先后移，再插入

> [!abstract] 核心结论  
> 在顺序表中间插入元素时，必须先把插入位置后面的元素整体后移。移动顺序要从后往前。

## ① 底层原理

假设顺序表是：

```text
下标：0   1   2   3
数据：10  20  30  40
```

想在下标 `1` 插入 `99`：

```text
目标结果：10  99  20  30  40
```

因为数组空间是连续的，不能直接把 `99` 塞进去。

必须先移动：

```text
40 后移
30 后移
20 后移
空出 data[1]
放入 99
```

移动必须从后往前，否则原来的数据会被覆盖。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int insertAt(SeqList *list, int index, int value) {
    if (list->length >= MAX_SIZE) {
        return 0;
    }

    if (index < 0 || index > list->length) {
        return 0;
    }

    for (int i = list->length; i > index; i--) {
        list->data[i] = list->data[i - 1];
    }

    list->data[index] = value;
    list->length++;

    return 1;
}

int main(void) {
    SeqList list = {{10, 20, 30, 40}, 4};

    insertAt(&list, 1, 99);

    for (int i = 0; i < list.length; i++) {
        printf("%d ", list.data[i]);
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
insertAt(&list, 1, 99);
在下标 1 的位置插入 99。

index < 0 || index > list->length
插入位置不能小于 0，也不能大于当前长度。

for (int i = list->length; i > index; i--)
从最后一个有效元素的后一个位置开始，向后移动。

list->data[i] = list->data[i - 1];
把前一个元素复制到后一个位置。

list->data[index] = value;
空位出来后，把新值放进去。
```

---

## ④ 输出

```text
10 99 20 30 40
```

---

## ⑤ 重点 / 注意 / 常见错误

> 插入移动方向必须是从后往前。

错误写法：

```c
for (int i = index; i < list->length; i++) {
    list->data[i + 1] = list->data[i];
}
```

这样会让 `20` 覆盖后面的数据，导致数据丢失。

正确写法：

```c
for (int i = list->length; i > index; i--) {
    list->data[i] = list->data[i - 1];
}
```

> [!tip] 记忆口诀  
> **中间插，先让路；从后搬，不丢数。**

# 06 | 顺序表删除元素：后面的元素向前覆盖

> [!abstract] 核心结论  
> 删除顺序表中的元素，不是真的擦掉内存，而是让后面的元素向前覆盖，最后 `length--`。

## ① 底层原理

删除下标 `1` 的元素：

```text
删除前：10  20  30  40
删除后：10  30  40
```

底层动作是：

```text
data[1] = data[2]
data[2] = data[3]
length--
```

原来最后一个位置可能还残留旧值。

但只要 `length` 减少了，它就不再属于有效数据。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int deleteAt(SeqList *list, int index) {
    if (index < 0 || index >= list->length) {
        return 0;
    }

    for (int i = index; i < list->length - 1; i++) {
        list->data[i] = list->data[i + 1];
    }

    list->length--;

    return 1;
}

int main(void) {
    SeqList list = {{10, 20, 30, 40}, 4};

    deleteAt(&list, 1);

    for (int i = 0; i < list.length; i++) {
        printf("%d ", list.data[i]);
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
deleteAt(&list, 1);
删除下标 1 的元素，也就是 20。

index < 0 || index >= list->length
删除位置必须在有效元素范围内。

for 循环从 index 开始向前覆盖。
后一个元素覆盖前一个位置。

list->length--;
有效元素数量减少 1。
```

---

## ④ 输出

```text
10 30 40
```

---

## ⑤ 重点 / 注意 / 常见错误

> 删除后通常不需要手动清空最后一个位置。

错误理解：

```c
list->data[list->length - 1] = 0;
```

这不是必须的。

因为顺序表判断数据是否有效，只看 `length`。

|操作|是否需要|
|---|---|
|元素前移|需要|
|`length--`|需要|
|清空最后一个旧值|通常不需要|

> [!tip] 记忆口诀  
> **删除向前盖，长度减下来。**

# 07 | 顺序表查找元素：按下标快，按值要遍历

> [!abstract] 核心结论  
> 顺序表按下标访问很快，因为可以直接计算地址。按值查找通常要从头到尾比较。

## ① 底层原理

按下标访问：

```text
data[3]
```

程序可以直接根据地址公式找到元素。

按值查找：

```text
查找 30
```

程序不知道 `30` 在哪里，只能从头开始比较：

```text
10 -> 20 -> 30
```

所以按值查找比按下标访问慢。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int findValue(SeqList *list, int value) {
    for (int i = 0; i < list->length; i++) {
        if (list->data[i] == value) {
            return i;
        }
    }

    return -1;
}

int main(void) {
    SeqList list = {{10, 20, 30, 40}, 4};

    int index = findValue(&list, 30);

    printf("30 的下标是: %d\n", index);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
findValue(&list, 30);
查找值为 30 的元素。

for 循环从 0 遍历到 length - 1。

if (list->data[i] == value)
如果当前元素等于目标值，就返回当前下标。

return -1;
如果整个表都没找到，就返回 -1。
```

---

## ④ 输出

```text
30 的下标是: 2
```

---

## ⑤ 重点 / 注意 / 常见错误

> C 语言数组下标从 `0` 开始。

|逻辑位置|C 数组下标|
|---|---|
|第 1 个元素|`0`|
|第 2 个元素|`1`|
|第 3 个元素|`2`|

错误理解：

```text
第 3 个元素的下标是 3
```

正确理解：

```text
第 3 个元素的下标是 2
```

> [!tip] 记忆口诀  
> **下标直接拿，数值慢慢查。**

# 08 | 顺序表遍历：只访问 length 范围内的数据

> [!abstract] 核心结论  
> 遍历顺序表时，只能遍历 `0 ~ length - 1`。不要遍历整个数组容量。

## ① 底层原理

数组容量可能很大，但有效元素可能很少。

```text
MAX_SIZE = 10
length = 3

有效区：data[0] data[1] data[2]
无效区：data[3] data[4] ... data[9]
```

无效区可能是旧数据，也可能是随机垃圾值。

C 语言不会提醒你“这里不是有效数据”。

所以遍历时必须靠 `length` 控制范围。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 10

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

void printList(SeqList *list) {
    for (int i = 0; i < list->length; i++) {
        printf("%d ", list->data[i]);
    }

    printf("\n");
}

int main(void) {
    SeqList list = {{5, 10, 15}, 3};

    printList(&list);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
printList(&list);
打印顺序表中的有效元素。

i < list->length
循环只访问有效范围。

printf("%d ", list->data[i]);
打印当前下标对应的元素。
```

---

## ④ 输出

```text
5 10 15
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
for (int i = 0; i < MAX_SIZE; i++) {
    printf("%d ", list.data[i]);
}
```

正确写法：

```c
for (int i = 0; i < list.length; i++) {
    printf("%d ", list.data[i]);
}
```

> 记住：数组容量不等于顺序表长度。

> [!tip] 记忆口诀  
> **遍历看 length，不看 MAX_SIZE。**

---

# 09 | 顺序表的优缺点：访问快，插删慢

> [!abstract] 核心结论  
> 顺序表的优势是按下标访问快。缺点是中间插入和删除时，需要移动大量元素。

## ① 底层原理

顺序表底层是连续数组。

所以它的特点来自数组：

```text
优点：地址连续，下标访问快。
缺点：中间不能直接插入，必须移动元素。
```

比如在开头插入一个元素：

```text
原来：10 20 30 40 50
插入：99 10 20 30 40 50
```

为了插入 `99`，原来的所有元素都要后移。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int main(void) {
    SeqList list = {{1, 2, 3, 4, 5}, 5};

    printf("list.data[3] = %d\n", list.data[3]);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
list.data[3]
直接访问下标为 3 的元素。

因为数组连续存储，所以访问速度很快。

但如果在 data[0] 插入元素，后面的元素都要移动。
```

---

## ④ 输出

```text
list.data[3] = 4
```

---

## ⑤ 重点 / 注意 / 常见错误

|操作|特点|
|---|---|
|按下标访问|快|
|按值查找|慢，需要遍历|
|尾部插入|快|
|中间插入|慢，需要后移|
|中间删除|慢，需要前移|

顺序表适合：

```text
经常按下标访问
数据规模比较固定
插入删除不频繁
```

顺序表不适合：

```text
频繁在中间插入删除
数据规模经常变化
```

> [!tip] 记忆口诀  
> **访问快是连续，插删慢因搬移。**

# 10 | 顺序表常见错误：越界、漏改长度、传参错误

> [!abstract] 核心结论  
> 顺序表最容易错在三个地方：数组越界、忘记修改 `length`、函数没有传地址。

## ① 底层原理

> C 语言不会自动检查数组越界。

如果数组最大容量是 `MAX_SIZE`，合法下标是：

```text
0 ~ MAX_SIZE - 1
```

访问 `data[MAX_SIZE]` 就越界了。

越界不一定马上报错，但可能破坏别的变量。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 3

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int append(SeqList *list, int value) {
    if (list->length >= MAX_SIZE) {
        return 0;
    }

    list->data[list->length] = value;
    list->length++;

    return 1;
}

int main(void) {
    SeqList list = {{0}, 0};

    append(&list, 10);
    append(&list, 20);
    append(&list, 30);

    if (!append(&list, 40)) {
        printf("插入失败：顺序表已满\n");
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
MAX_SIZE = 3
说明最多只能存 3 个元素。

append(&list, 10)
向顺序表尾部插入 10。

if (list->length >= MAX_SIZE)
插入前检查是否还有空间。

第 4 次插入 40 时，length 已经等于 3。
所以插入失败。
```

---

## ④ 输出

```text
插入失败：顺序表已满
```

---

## ⑤ 重点 / 注意 / 常见错误

错误 1：数组越界

```c
list.data[MAX_SIZE] = 100;
```

正确理解：

```text
最大合法下标是 MAX_SIZE - 1。
```

错误 2：插入后忘记改长度

```c
list->data[list->length] = value;
```

正确写法：

```c
list->data[list->length] = value;
list->length++;
```

错误 3：函数传值导致修改无效

```c
void append(SeqList list, int value) {
    list.data[list.length] = value;
    list.length++;
}
```

正确写法：

```c
void append(SeqList *list, int value) {
    list->data[list->length] = value;
    list->length++;
}
```

> [!tip] 记忆口诀  
> **防越界，改长度，传地址。**

# 11 | 顺序表主题总结：连续数组带来快访问，也带来搬元素

> [!abstract] 核心结论  
> 顺序表就是用连续数组存数据，用 `length` 管有效范围。它适合查找和访问，不适合频繁中间插入删除。

## ① 底层原理

顺序表要抓住三个关键词：

> - **连续内存**：元素挨着放。
>     
> - **下标访问**：能直接算地址。
>     
> - **长度控制**：只访问 `length` 范围内的数据。
>     

顺序表的核心模型：

```text
SeqList = data 数组 + length 当前长度
```

这句话基本就把顺序表讲透了。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

void initList(SeqList *list) {
    list->length = 0;
}

int append(SeqList *list, int value) {
    if (list->length >= MAX_SIZE) {
        return 0;
    }

    list->data[list->length] = value;
    list->length++;

    return 1;
}

void printList(SeqList *list) {
    for (int i = 0; i < list->length; i++) {
        printf("%d ", list->data[i]);
    }

    printf("\n");
}

int main(void) {
    SeqList list;

    initList(&list);

    append(&list, 10);
    append(&list, 20);
    append(&list, 30);

    printList(&list);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
initList
初始化顺序表，把 length 设为 0。

append
向顺序表尾部插入元素。

printList
遍历并打印顺序表中的有效元素。

main
把初始化、插入、遍历流程串起来。
```

---

## ④ 输出

```text
10 20 30
```

---

## ⑤ 重点 / 注意 / 常见错误

|问题|关键点|
|---|---|
|数据放在哪里|放在 `data` 数组里|
|当前有几个元素|看 `length`|
|能不能插入|看 `length < MAX_SIZE`|
|为什么访问快|内存连续，可以直接算地址|
|为什么插删慢|要移动元素|
|函数怎么修改顺序表|传结构体指针|

> [!tip] 记忆口诀  
> **顺序表：数组存，长度管；访问快，插删搬。**
