# 01 | 顺序表的基本概念：用连续数组保存线性数据

> [!abstract] 核心结论  
> 顺序表本质上是“数组 + 长度”。数组负责存数据，`length` 负责记录当前有多少个有效元素。

## ① 底层原理

> 顺序表的本质：
> 
> - **连续内存**：元素在内存中挨着存放。
>     
> - **随机访问**：知道下标就能直接访问元素。
>     
> - **长度管理**：用 `length` 记录有效元素个数。
>     
> - **容量限制**：数组大小固定，不能无限插入。
>     

顺序表底层通常长这样：

```text
下标：  0    1    2    3    4
数据： 10   20   30   40   50
地址：连续 连续 连续 连续 连续
```

数组元素连续存放，所以访问 `data[i]` 时，底层会计算：

```text
data[i] 的地址 = 首地址 + i * sizeof(元素类型)
```

这就是顺序表按下标访问很快的原因。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int main(void) {
    SeqList list = {{10, 20, 30}, 3};

    printf("第一个元素: %d\n", list.data[0]);
    printf("当前长度: %d\n", list.length);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
#define MAX_SIZE 100
表示顺序表最多能存 100 个 int 元素。

int data[MAX_SIZE];
真正存放数据的数组。

int length;
记录当前有效元素个数。

SeqList list = {{10, 20, 30}, 3};
前三个位置有数据，当前长度是 3。
```

---

## ④ 输出

```text
第一个元素: 10
当前长度: 3
```

---

## ⑤ 重点 / 注意 / 常见错误

> 顺序表不是单纯的数组，而是数组和长度一起管理。

错误理解：

```c
int data[100];
```

正确写法：

```c
typedef struct {
    int data[100];
    int length;
} SeqList;
```

|名称|含义|
|---|---|
|`data`|存放元素|
|`length`|当前有效元素个数|
|`MAX_SIZE`|最大容量|

> [!tip] 记忆口诀  
> **数组负责装，length 负责管。**

# 02 | 顺序表初始化：先把 length 设置为 0

> [!abstract] 核心结论  
> 初始化顺序表时，最重要的是把 `length` 设为 `0`。否则程序不知道当前有多少个有效元素。

## ① 底层原理

> C 语言中的局部变量不会自动清零。

如果直接写：

```c
SeqList list;
```

此时 `list.length` 可能是随机值。

如果随机值很大，程序可能误以为顺序表里已经有很多元素，然后访问非法内存。

所以初始化时必须明确写：

```c
list.length = 0;
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

void initList(SeqList *list) {
    list->length = 0;
}

int main(void) {
    SeqList list;

    initList(&list);

    printf("初始化后的长度: %d\n", list.length);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
void initList(SeqList *list)
函数接收顺序表的地址。

list->length = 0;
通过指针修改原来的顺序表。

initList(&list);
把 list 的地址传给函数。
```

---

## ④ 输出

```text
初始化后的长度: 0
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
void initList(SeqList list) {
    list.length = 0;
}
```

这个函数改的是副本，外面的 `list` 不会变。

正确写法：

```c
void initList(SeqList *list) {
    list->length = 0;
}
```

> [!tip] 记忆口诀  
> **初始化清长度，改原表传地址。**

# 03 | 顺序表尾插：把新元素放到 length 指向的位置

> [!abstract] 核心结论  
> 尾插是顺序表最简单的插入方式。把新元素放到 `data[length]`，然后让 `length++`。

## ① 底层原理

当前顺序表：

```text
下标：  0    1    2    3
数据： 10   20   30   ?
length = 3
```

`length` 的值是 `3`，正好指向下一个可插入位置。

所以尾插就是：

```text
data[length] = 新元素
length++
```

尾插不需要移动元素，所以效率比较高。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int append(SeqList *list, int value) {
    if (list->length >= MAX_SIZE) {
        return 0;
    }

    list->data[list->length] = value;
    list->length++;

    return 1;
}

int main(void) {
    SeqList list = {{10, 20}, 2};

    append(&list, 30);

    for (int i = 0; i < list.length; i++) {
        printf("%d ", list.data[i]);
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
append(&list, 30);
向顺序表尾部插入 30。

if (list->length >= MAX_SIZE)
插入前判断顺序表是否已满。

list->data[list->length] = value;
把新元素放到当前 length 对应的位置。

list->length++;
插入成功后，长度加 1。
```

---

## ④ 输出

```text
10 20 30
```

---

## ⑤ 重点 / 注意 / 常见错误

> 插入前检查容量，插入后修改长度。

错误写法：

```c
list->data[list->length] = value;
```

问题是忘了：

```c
list->length++;
```

正确写法：

```c
list->data[list->length] = value;
list->length++;
```

> [!tip] 记忆口诀  
> **尾插看 length，放完 length 加一。**

# 04 | 顺序表按位置插入：先后移，再插入

> [!abstract] 核心结论  
> 在顺序表中间插入元素时，要先把插入位置后面的元素整体后移。移动方向必须从后往前。

## ① 底层原理

原顺序表：

```text
下标：0   1   2   3
数据：10  20  30  40
```

要在下标 `1` 插入 `99`：

```text
目标：10  99  20  30  40
```

因为数组内存连续，不能直接“挤进去”。

必须先移动：

```text
40 后移
30 后移
20 后移
空出 data[1]
放入 99
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int insertAt(SeqList *list, int index, int value) {
    if (list->length >= MAX_SIZE) {
        return 0;
    }

    if (index < 0 || index > list->length) {
        return 0;
    }

    for (int i = list->length; i > index; i--) {
        list->data[i] = list->data[i - 1];
    }

    list->data[index] = value;
    list->length++;

    return 1;
}

int main(void) {
    SeqList list = {{10, 20, 30, 40}, 4};

    insertAt(&list, 1, 99);

    for (int i = 0; i < list.length; i++) {
        printf("%d ", list.data[i]);
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
insertAt(&list, 1, 99);
在下标 1 的位置插入 99。

index < 0 || index > list->length
判断插入位置是否合法。

for 循环从后往前移动元素。
这样不会覆盖原来的数据。

list->data[index] = value;
把新值放到空出来的位置。

list->length++;
有效元素个数加 1。
```

---

## ④ 输出

```text
10 99 20 30 40
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
for (int i = index; i < list->length; i++) {
    list->data[i + 1] = list->data[i];
}
```

这样会把后面的元素覆盖掉。

正确写法：

```c
for (int i = list->length; i > index; i--) {
    list->data[i] = list->data[i - 1];
}
```

> [!tip] 记忆口诀  
> **中间插，先让路；从后搬，不丢数。**

# 05 | 顺序表删除元素：后面的元素向前覆盖

> [!abstract] 核心结论  
> 删除顺序表元素时，不是真的清空内存，而是让后面的元素向前覆盖，最后 `length--`。

## ① 底层原理

删除下标 `1` 的元素：

```text
删除前：10  20  30  40
删除后：10  30  40
```

底层动作是：

```text
data[1] = data[2]
data[2] = data[3]
length--
```

最后一个位置可能还残留旧值，但它已经不属于有效数据。

顺序表判断元素是否有效，只看 `length`。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int deleteAt(SeqList *list, int index) {
    if (index < 0 || index >= list->length) {
        return 0;
    }

    for (int i = index; i < list->length - 1; i++) {
        list->data[i] = list->data[i + 1];
    }

    list->length--;

    return 1;
}

int main(void) {
    SeqList list = {{10, 20, 30, 40}, 4};

    deleteAt(&list, 1);

    for (int i = 0; i < list.length; i++) {
        printf("%d ", list.data[i]);
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
deleteAt(&list, 1);
删除下标 1 的元素，也就是 20。

index < 0 || index >= list->length
判断删除位置是否合法。

for 循环从 index 开始向前覆盖。

list->length--;
有效元素个数减少 1。
```

---

## ④ 输出

```text
10 30 40
```

---

## ⑤ 重点 / 注意 / 常见错误

> 删除后一般不需要清空最后一个旧值。

不是必须写：

```c
list->data[list->length - 1] = 0;
```

真正关键的是：

```c
list->length--;
```

|操作|是否需要|
|---|---|
|元素前移|需要|
|`length--`|需要|
|清空旧位置|通常不需要|

> [!tip] 记忆口诀  
> **删除向前盖，长度减下来。**

---

# 06 | 顺序表查找元素：按下标快，按值要遍历

> [!abstract] 核心结论  
> 顺序表按下标访问很快，因为能直接计算地址。按值查找通常要从头到尾一个个比较。

## ① 底层原理

按下标访问：

```text
data[3]
```

程序可以直接算出地址。

按值查找：

```text
查找 30
```

程序不知道 `30` 在哪里，只能顺着数组遍历：

```text
10 -> 20 -> 30
```

所以按值查找比按下标访问慢。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int findValue(SeqList *list, int value) {
    for (int i = 0; i < list->length; i++) {
        if (list->data[i] == value) {
            return i;
        }
    }

    return -1;
}

int main(void) {
    SeqList list = {{10, 20, 30, 40}, 4};

    int index = findValue(&list, 30);

    printf("30 的下标是: %d\n", index);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
findValue(&list, 30);
查找值为 30 的元素。

for 循环从 0 遍历到 length - 1。

如果找到目标值，就返回下标。

如果没找到，就返回 -1。
```

---

## ④ 输出

```text
30 的下标是: 2
```

---

## ⑤ 重点 / 注意 / 常见错误

> C 语言数组下标从 `0` 开始。

|逻辑位置|C 数组下标|
|---|---|
|第 1 个元素|`0`|
|第 2 个元素|`1`|
|第 3 个元素|`2`|

错误理解：

```text
第 3 个元素的下标是 3
```

正确理解：

```text
第 3 个元素的下标是 2
```

> [!tip] 记忆口诀  
> **下标直接拿，数值慢慢查。**

---

# 07 | 顺序表遍历：只访问 length 范围内的数据

> [!abstract] 核心结论  
> 遍历顺序表时，只能访问 `0 ~ length - 1`。不要遍历整个数组容量。

## ① 底层原理

数组容量可能很大，但有效元素可能很少：

```text
MAX_SIZE = 10
length = 3

有效区：data[0] data[1] data[2]
无效区：data[3] data[4] ... data[9]
```

无效区可能是旧数据，也可能是随机垃圾值。

C 语言不会自动告诉你哪些数据有效，所以必须用 `length` 控制范围。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 10

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

void printList(SeqList *list) {
    for (int i = 0; i < list->length; i++) {
        printf("%d ", list->data[i]);
    }

    printf("\n");
}

int main(void) {
    SeqList list = {{5, 10, 15}, 3};

    printList(&list);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
printList(&list);
打印顺序表中的有效元素。

i < list->length
只访问有效范围。

printf("%d ", list->data[i]);
打印当前元素。
```

---

## ④ 输出

```text
5 10 15
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
for (int i = 0; i < MAX_SIZE; i++) {
    printf("%d ", list.data[i]);
}
```

正确写法：

```c
for (int i = 0; i < list.length; i++) {
    printf("%d ", list.data[i]);
}
```

> 数组容量不等于顺序表长度。

> [!tip] 记忆口诀  
> **遍历看 length，不看 MAX_SIZE。**

---

# 08 | 顺序表的优缺点：访问快，插删慢

> [!abstract] 核心结论  
> 顺序表的优势是按下标访问快。缺点是中间插入和删除时，需要移动大量元素。

## ① 底层原理

顺序表底层是连续数组。

所以它的特点来自数组：

```text
优点：地址连续，下标访问快。
缺点：中间不能直接插入，必须移动元素。
```

比如在开头插入一个元素：

```text
原来：10 20 30 40 50
插入：99 10 20 30 40 50
```

为了插入 `99`，原来的元素都要后移。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int main(void) {
    SeqList list = {{1, 2, 3, 4, 5}, 5};

    printf("list.data[3] = %d\n", list.data[3]);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
list.data[3]
直接访问下标为 3 的元素。

因为数组连续存储，所以访问速度很快。

但是如果在 data[0] 插入元素，后面的元素都要移动。
```

---

## ④ 输出

```text
list.data[3] = 4
```

---

## ⑤ 重点 / 注意 / 常见错误

|操作|特点|
|---|---|
|按下标访问|快|
|按值查找|慢，需要遍历|
|尾部插入|快|
|中间插入|慢，需要后移|
|中间删除|慢，需要前移|

适合顺序表的场景：

```text
经常按下标访问
数据规模比较固定
插入删除不频繁
```

不适合顺序表的场景：

```text
频繁在中间插入删除
数据规模经常变化
```

> [!tip] 记忆口诀  
> **访问快是连续，插删慢因搬移。**

# 09 | 顺序表常见错误：越界、漏改长度、传参错误

> [!abstract] 核心结论  
> 顺序表最容易错在三个地方：数组越界、忘记修改 `length`、函数没有传地址。

## ① 底层原理

> C 语言不会自动检查数组越界。

如果数组最大容量是 `MAX_SIZE`，合法下标是：

```text
0 ~ MAX_SIZE - 1
```

访问 `data[MAX_SIZE]` 就越界了。

越界不一定马上报错，但可能破坏别的变量。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 3

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

int append(SeqList *list, int value) {
    if (list->length >= MAX_SIZE) {
        return 0;
    }

    list->data[list->length] = value;
    list->length++;

    return 1;
}

int main(void) {
    SeqList list = {{0}, 0};

    append(&list, 10);
    append(&list, 20);
    append(&list, 30);

    if (!append(&list, 40)) {
        printf("插入失败：顺序表已满\n");
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
MAX_SIZE = 3
说明最多只能存 3 个元素。

append(&list, 10)
向顺序表尾部插入 10。

if (list->length >= MAX_SIZE)
插入前检查是否还有空间。

第 4 次插入 40 时，length 已经等于 3。
所以插入失败。
```

---

## ④ 输出

```text
插入失败：顺序表已满
```

---

## ⑤ 重点 / 注意 / 常见错误

错误 1：数组越界

```c
list.data[MAX_SIZE] = 100;
```

正确理解：

```text
最大合法下标是 MAX_SIZE - 1。
```

错误 2：插入后忘记改长度

```c
list->data[list->length] = value;
```

正确写法：

```c
list->data[list->length] = value;
list->length++;
```

错误 3：函数传值导致修改无效

```c
void append(SeqList list, int value) {
    list.data[list.length] = value;
    list.length++;
}
```

正确写法：

```c
void append(SeqList *list, int value) {
    list->data[list->length] = value;
    list->length++;
}
```

> [!tip] 记忆口诀  
> **防越界，改长度，传地址。**

# 10 | 顺序表主题总结：连续数组带来快访问，也带来搬元素

> [!abstract] 核心结论  
> 顺序表就是用连续数组存数据，用 `length` 管有效范围。它适合访问和遍历，不适合频繁中间插入删除。

## ① 底层原理

顺序表要抓住三个关键词：

> - **连续内存**：元素挨着放。
>     
> - **下标访问**：能直接算地址。
>     
> - **长度控制**：只访问 `length` 范围内的数据。
>     

顺序表的核心模型：

```text
SeqList = data 数组 + length 当前长度
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SeqList;

void initList(SeqList *list) {
    list->length = 0;
}

int append(SeqList *list, int value) {
    if (list->length >= MAX_SIZE) {
        return 0;
    }

    list->data[list->length] = value;
    list->length++;

    return 1;
}

void printList(SeqList *list) {
    for (int i = 0; i < list->length; i++) {
        printf("%d ", list->data[i]);
    }

    printf("\n");
}

int main(void) {
    SeqList list;

    initList(&list);

    append(&list, 10);
    append(&list, 20);
    append(&list, 30);

    printList(&list);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
initList
初始化顺序表，把 length 设为 0。

append
```

## ⑤ 重点 / 注意 / 常见错误

|问题|关键点|
|---|---|
|数据放在哪里|放在 `data` 数组里|
|当前有几个元素|看 `length`|
|能不能插入|看 `length < MAX_SIZE`|
|为什么访问快|内存连续，可以直接算地址|
|为什么插删慢|要移动元素|
|函数怎么修改顺序表|传结构体指针|

> [!tip] 记忆口诀  
> **顺序表：数组存，长度管；访问快，插删搬。**
