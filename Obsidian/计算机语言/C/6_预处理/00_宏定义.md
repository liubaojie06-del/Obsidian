# 01 | 宏定义：预处理阶段的文本替换

> [!abstract] 核心结论
> 宏定义是 C 语言预处理器提供的功能，使用 `#define` 定义。它不是变量，也不是函数，本质是在编译前的预处理阶段，把宏名替换成指定内容。

### ① 底层原理：宏在真正编译前就被替换了

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>宏定义的本质：</strong>
<ul>
  <li><strong>宏属于预处理：</strong> 在编译器正式分析 C 语法之前处理。</li>
  <li><strong>本质是文本替换：</strong> 把宏名替换成宏内容。</li>
  <li><strong>没有类型：</strong> 宏不像变量那样有明确数据类型。</li>
  <li><strong>不分配普通变量内存：</strong> 宏名本身不会像变量一样占用运行时存储空间。</li>
  <li><strong>容易出坑：</strong> 因为是文本替换，所以优先级、重复求值、作用范围都要特别注意。</li>
</ul>
</div>

### ② 基本语法

```c
#define 宏名 替换内容
```

示例：

```c
#define PI 3.14159
#define MAX_SIZE 100
```

使用：

```c
double area = PI * r * r;
int arr[MAX_SIZE];
```

预处理后可以理解为：

```c
double area = 3.14159 * r * r;
int arr[100];
```

### ③ 一句话理解

```text
宏定义 = 编译前的批量文本替换。
```

> [!tip] 记忆口诀
> **define 不是变量，不是函数；预处理先替换，编译器后分析。**

# 02 | 常量宏：给固定值起名字

> [!abstract] 核心结论
> 常量宏用于给固定数值、字符串、配置项起名字。它可以减少魔法数字，让代码更清晰。

### ① 基本示例

```c
#include <stdio.h>

#define PASS_SCORE 60

int main() {
    int score = 80;

    if (score >= PASS_SCORE) {
        printf("pass\n");
    }

    return 0;
}
```

输出：

```text
pass
```

### ② 预处理理解

代码：

```c
if (score >= PASS_SCORE)
```

预处理后类似：

```c
if (score >= 60)
```

也就是说：

```text
PASS_SCORE 在编译前被替换成 60。
```

### ③ 字符串宏

```c
#define APP_NAME "StudentSystem"
```

使用：

```c
printf("%s\n", APP_NAME);
```

预处理后：

```c
printf("%s\n", "StudentSystem");
```

### ④ 常量宏的好处

```text
避免魔法数字。
统一管理配置。
修改方便。
提升代码可读性。
```

### ⑤ 常量宏的问题

```text
没有类型检查。
调试时看不到普通变量那样的内存对象。
容易和其他名字冲突。
```

> [!tip] 记忆口诀
> **固定值起宏名，代码清楚好维护；但宏无类型，别当变量。**

# 03 | 函数宏：像函数一样使用的宏

> [!abstract] 核心结论
> 函数宏可以带参数，看起来像函数调用，但本质仍然是文本替换。它没有真正的函数调用开销，但也容易出现优先级和重复求值问题。

### ① 基本写法

```c
#define SQUARE(x) ((x) * (x))
```

使用：

```c
int a = SQUARE(5);
```

预处理后类似：

```c
int a = ((5) * (5));
```

结果：

```text
25
```

### ② 完整示例

```c
#include <stdio.h>

#define SQUARE(x) ((x) * (x))

int main() {
    int a = 5;

    printf("%d\n", SQUARE(a));

    return 0;
}
```

输出：

```text
25
```

### ③ 为什么函数宏没有类型

```c
#define MAX(a, b) ((a) > (b) ? (a) : (b))
```

它可以用于：

```c
MAX(10, 20)
MAX(3.14, 2.71)
MAX('a', 'b')
```

因为：

```text
宏只是替换文本。
真正类型由替换后的表达式决定。
```

### ④ 函数宏和普通函数区别

| 对比项 | 函数宏 | 普通函数 |
| :--- | :--- | :--- |
| 处理阶段 | 预处理阶段 | 编译和运行阶段 |
| 本质 | 文本替换 | 函数调用 |
| 类型检查 | 弱 | 强 |
| 调用开销 | 无普通函数调用开销 | 有调用过程 |
| 风险 | 优先级、重复求值 | 相对安全 |

### ⑤ 重点

```text
函数宏看起来像函数。
但它不是函数。
它只是带参数的文本替换。
```

> [!tip] 记忆口诀
> **函数宏像函数，实质还是替换；速度可能快，坑也更多。**

# 04 | 宏参数为什么必须加括号

> [!abstract] 核心结论
> 宏定义中的参数和整体表达式都应该尽量加括号。因为宏是文本替换，如果不加括号，很容易受到运算符优先级影响，产生错误结果。

### ① 错误写法

```c
#define SQUARE(x) x * x
```

使用：

```c
int result = SQUARE(1 + 2);
```

预处理后：

```c
int result = 1 + 2 * 1 + 2;
```

实际结果：

```text
1 + 2 * 1 + 2 = 5
```

不是：

```text
9
```

### ② 正确写法

```c
#define SQUARE(x) ((x) * (x))
```

使用：

```c
int result = SQUARE(1 + 2);
```

预处理后：

```c
int result = ((1 + 2) * (1 + 2));
```

结果：

```text
9
```

### ③ 整体也要加括号

错误：

```c
#define DOUBLE(x) (x) + (x)
```

使用：

```c
int result = 10 * DOUBLE(2);
```

预处理后：

```c
int result = 10 * (2) + (2);
```

结果：

```text
22
```

### ④ 更安全写法

```c
#define DOUBLE(x) ((x) + (x))
```

预处理后：

```c
int result = 10 * ((2) + (2));
```

结果：

```text
40
```

### ⑤ 重点规则

```text
宏参数加括号。
宏整体也加括号。
```

> [!tip] 记忆口诀
> **宏参数要套括号，宏整体也套括号；否则优先级会坑你。**

# 05 | 宏的重复求值问题

> [!abstract] 核心结论
> 函数宏会把参数原样替换到多个位置。如果参数中带有 `++`、函数调用等有副作用的表达式，可能会被执行多次，导致结果出错。

### ① 问题示例

```c
#include <stdio.h>

#define SQUARE(x) ((x) * (x))

int main() {
    int a = 3;

    int result = SQUARE(a++);

    printf("result = %d\n", result);
    printf("a = %d\n", a);

    return 0;
}
```

预处理后类似：

```c
int result = ((a++) * (a++));
```

问题：

```text
a++ 被执行了两次。
```

结果可能不是你想的：

```text
result = 12
a = 5
```

具体表达式中多次修改同一变量还可能涉及未定义行为风险，应避免。

### ② MAX 宏也有问题

```c
#define MAX(a, b) ((a) > (b) ? (a) : (b))
```

使用：

```c
int m = MAX(x++, y++);
```

预处理后：

```c
int m = ((x++) > (y++) ? (x++) : (y++));
```

问题：

```text
x++ 或 y++ 可能执行不止一次。
```

### ③ 为什么普通函数不会这样

普通函数：

```c
int square(int x) {
    return x * x;
}
```

调用：

```c
square(a++);
```

过程：

```text
a++ 只在传参时执行一次。
函数内部用的是形参副本 x。
```

### ④ 安全建议

不要给函数宏传入：

```text
i++
++i
函数调用
赋值表达式
有副作用的表达式
```

更推荐传：

```text
普通变量。
常量。
简单表达式。
```

### ⑤ 重点

```text
宏不是函数。
宏参数出现几次，实参表达式就可能被复制几次。
```

> [!tip] 记忆口诀
> **宏会复制表达式，副作用可能执行多次；带 ++ 的参数别乱传宏。**

# 06 | # 和 ##：字符串化与连接符

> [!abstract] 核心结论
> 宏定义中 `#` 可以把宏参数转换成字符串，`##` 可以把两个记号拼接成一个新的记号。它们常用于调试输出、自动生成变量名或函数名。

### ① # 字符串化

```c
#define TO_STR(x) #x
```

使用：

```c
printf("%s\n", TO_STR(hello));
```

预处理后：

```c
printf("%s\n", "hello");
```

完整示例：

```c
#include <stdio.h>

#define TO_STR(x) #x

int main() {
    printf("%s\n", TO_STR(C Language));

    return 0;
}
```

输出：

```text
C Language
```

### ② ## 连接符

```c
#define MAKE_VAR(name) var_##name
```

使用：

```c
int MAKE_VAR(age) = 18;
```

预处理后：

```c
int var_age = 18;
```

### ③ 完整示例

```c
#include <stdio.h>

#define MAKE_VAR(name) var_##name

int main() {
    int MAKE_VAR(score) = 100;

    printf("%d\n", var_score);

    return 0;
}
```

输出：

```text
100
```

### ④ 常见用途

```text
调试日志。
生成变量名。
生成函数名。
简化重复代码。
条件编译辅助。
```

### ⑤ 注意

```text
# 和 ## 只在宏展开中使用。
不是普通 C 运算符。
```

> [!tip] 记忆口诀
> **# 把参数变字符串，## 把记号粘一起。**

# 07 | 条件编译：根据条件决定编译哪段代码

> [!abstract] 核心结论
> 条件编译可以让部分代码根据宏是否定义、宏值大小等条件决定是否参与编译。常用于调试开关、平台适配、功能裁剪和头文件保护。

### ① #ifdef

```c
#ifdef DEBUG
    printf("debug mode\n");
#endif
```

含义：

```text
如果定义了 DEBUG 宏，就编译里面的代码。
```

### ② #ifndef

```c
#ifndef DEBUG
    printf("release mode\n");
#endif
```

含义：

```text
如果没有定义 DEBUG 宏，就编译里面的代码。
```

### ③ #if / #else / #endif

```c
#define VERSION 2

#if VERSION == 1
    printf("version 1\n");
#else
    printf("other version\n");
#endif
```

### ④ 调试开关示例

```c
#include <stdio.h>

#define DEBUG

int main() {
#ifdef DEBUG
    printf("调试信息\n");
#endif

    printf("程序运行\n");

    return 0;
}
```

输出：

```text
调试信息
程序运行
```

### ⑤ 取消宏定义

```c
#undef DEBUG
```

含义：

```text
取消 DEBUG 宏。
```

> [!tip] 记忆口诀
> **ifdef 看有没有定义，if 看宏值条件；条件编译决定代码进不进编译器。**

# 08 | 头文件保护：防止头文件重复包含

> [!abstract] 核心结论
> 头文件保护是宏定义的重要应用。它可以防止同一个头文件被多次包含，避免重复定义、重复声明等问题。

### ① 常见写法

```c
#ifndef MY_HEADER_H
#define MY_HEADER_H

// 头文件内容

#endif
```

### ② 示例：student.h

```c
#ifndef STUDENT_H
#define STUDENT_H

void print_student(void);

#endif
```

### ③ 工作过程

第一次包含：

```text
STUDENT_H 没有定义。
进入头文件内容。
定义 STUDENT_H。
```

第二次包含：

```text
STUDENT_H 已经定义。
跳过头文件内容。
```

### ④ 为什么需要保护

如果没有头文件保护，多个源文件或头文件互相包含时，可能导致：

```text
重复定义结构体。
重复声明冲突。
编译报错。
编译速度变慢。
```

### ⑤ 另一种写法

很多编译器支持：

```c
#pragma once
```

作用：

```text
告诉编译器这个头文件只包含一次。
```

但传统 C 代码中，宏保护更通用。

> [!tip] 记忆口诀
> **ifndef 开门，define 上锁，endif 关门；头文件只进一次。**

# 09 | 宏定义和 const、enum、inline 的区别

> [!abstract] 核心结论
> 宏定义是预处理文本替换；`const` 是有类型的只读变量；`enum` 是命名整数常量；`inline` 是函数优化建议。能用类型安全方式时，不要滥用宏。

### ① 宏定义

```c
#define SIZE 100
```

特点：

```text
预处理替换。
没有类型。
不占普通变量空间。
可能有优先级坑。
```

### ② const

```c
const int size = 100;
```

特点：

```text
有类型。
有作用域。
编译器可以类型检查。
不能通过普通方式修改。
```

### ③ enum

```c
enum {
    SIZE = 100
};
```

特点：

```text
适合整数常量。
有名字。
常用于状态、选项、固定编号。
```

### ④ inline 函数

```c
static inline int square(int x) {
    return x * x;
}
```

特点：

```text
有类型检查。
像函数一样安全。
编译器可能内联优化。
不会重复求值参数。
```

### ⑤ 对比表

| 方式 | 本质 | 类型检查 | 常见用途 |
| :--- | :--- | :--- | :--- |
| `#define` | 文本替换 | 弱 | 条件编译、头文件保护、简单宏 |
| `const` | 只读变量 | 强 | 有类型常量 |
| `enum` | 整数常量 | 较强 | 状态码、选项 |
| `inline` | 函数 | 强 | 替代复杂函数宏 |

> [!tip] 记忆口诀
> **宏是替换，const 有类型，enum 管整数，inline 像函数更安全。**

# 10 | 宏定义总结

> [!abstract] 核心结论
> 宏定义的底层核心是“预处理阶段的文本替换”。它适合定义编译期开关、头文件保护、简单常量和少量函数宏，但复杂逻辑应优先考虑 `const`、`enum`、`inline` 或普通函数。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>宏定义核心总结：</strong>
<ul>
  <li><strong>关键字：</strong> 使用 <code>#define</code> 定义宏。</li>
  <li><strong>本质：</strong> 预处理阶段进行文本替换。</li>
  <li><strong>常量宏：</strong> 给固定值起名字，例如 <code>#define SIZE 100</code>。</li>
  <li><strong>函数宏：</strong> 带参数的宏，例如 <code>#define SQUARE(x) ((x) * (x))</code>。</li>
  <li><strong>无类型：</strong> 宏本身没有变量类型，不做普通类型检查。</li>
  <li><strong>括号规则：</strong> 宏参数和整体表达式都要尽量加括号。</li>
  <li><strong>副作用风险：</strong> 宏参数可能被重复求值。</li>
  <li><strong>#：</strong> 把宏参数字符串化。</li>
  <li><strong>##：</strong> 把两个记号拼接成一个记号。</li>
  <li><strong>条件编译：</strong> 用于调试开关、平台适配、头文件保护。</li>
</ul>
</div>

### ② 最底层一句话

```text
宏定义 = 编译前由预处理器完成的文本替换机制；它很强大，但也很容易因为“只是替换”而出错。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **宏定义先替换，编译器后判断；参数整体加括号，副作用表达式别乱传。**
