# 01 | 字符串化运算符：把宏参数变成字符串

> [!abstract] 核心结论
> 字符串化运算符 `#` 是 C 语言宏定义中的预处理运算符，只能用在宏函数里。它的作用是把传入的宏参数转换成字符串字面量。

### ① 底层原理

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>字符串化运算符的本质：</strong>
<ul>
  <li><strong>符号：</strong><code>#</code>。</li>
  <li><strong>使用位置：</strong>只能出现在带参数宏的宏体中。</li>
  <li><strong>处理阶段：</strong>发生在预处理阶段，不是程序运行时执行。</li>
  <li><strong>核心作用：</strong>把宏参数原始文本转换成字符串字面量。</li>
  <li><strong>典型用途：</strong>调试输出、日志宏、表达式打印、行号字符串化。</li>
</ul>
</div>

### ② 代码用法

```c
#define TO_STRING(x) #x
```

示例：

```c
#include <stdio.h>

#define TO_STRING(x) #x

int main() {
    printf("%s\n", TO_STRING(hello));
    printf("%s\n", TO_STRING(1 + 2));

    return 0;
}
```

输出：

```text
hello
1 + 2
```

### ③ 使用场景

| 写法 | 展开结果 | 含义 |
| :--- | :--- | :--- |
| `TO_STRING(hello)` | `"hello"` | 把 hello 变成字符串 |
| `TO_STRING(1 + 2)` | `"1 + 2"` | 只转文本，不计算 |
| `TO_STRING(a * b)` | `"a * b"` | 表达式原样转字符串 |

> [!tip] 记忆口诀
> **井号放宏里，参数变字符串；只转文本，不算结果。**

# 02 | 字符串化不是计算表达式

> [!abstract] 核心结论
> `#` 只会把宏参数转换成字符串，不会计算表达式的值。`#x` 得到的是参数文本本身，而不是运行结果。

### ① 底层原理

```text
TO_STRING(1 + 2)
```

预处理后是：

```c
"1 + 2"
```

不是：

```c
"3"
```

因为：

```text
字符串化发生在预处理阶段。
预处理器只处理文本替换。
它不会执行 C 表达式计算。
```

### ② 代码用法

```c
#include <stdio.h>

#define STR(x) #x

int main() {
    printf("%s\n", STR(10 + 20));
    printf("%s\n", STR(a * b));

    return 0;
}
```

输出：

```text
10 + 20
a * b
```

### ③ 使用场景

| 宏调用 | 得到的字符串 | 是否计算 |
| :--- | :--- | :--- |
| `STR(10 + 20)` | `"10 + 20"` | 不计算 |
| `STR(a * b)` | `"a * b"` | 不计算 |
| `STR(printf("hi"))` | `"printf(\"hi\")"` | 不执行 |

> [!tip] 记忆口诀
> **字符串化只包双引号，不负责算答案。**

# 03 | 字符串化常用于打印表达式和值

> [!abstract] 核心结论
> 字符串化最常见的用法是调试宏：一边打印表达式本身，一边打印表达式计算结果。

### ① 底层原理

```c
#define PRINT_INT(expr) printf(#expr " = %d\n", (expr))
```

这里：

```text
#expr：
把表达式本身变成字符串。

(expr)：
真正计算表达式的值。
```

### ② 代码用法

```c
#include <stdio.h>

#define PRINT_INT(expr) printf(#expr " = %d\n", (expr))

int main() {
    int a = 10;
    int b = 20;

    PRINT_INT(a);
    PRINT_INT(b);
    PRINT_INT(a + b);
    PRINT_INT(a * b);

    return 0;
}
```

输出：

```text
a = 10
b = 20
a + b = 30
a * b = 200
```

### ③ 使用场景

```c
PRINT_INT(a + b);
```

预处理后类似：

```c
printf("a + b" " = %d\n", (a + b));
```

相邻字符串会自动拼接，所以等价于：

```c
printf("a + b = %d\n", (a + b));
```

> [!tip] 记忆口诀
> **井号打印表达式，不加井号算表达式。**

# 04 | 字符串化和宏展开顺序

> [!abstract] 核心结论
> 宏参数被 `#` 字符串化时，通常不会先展开成宏值，而是直接把参数原文转换成字符串。如果想先展开宏值再字符串化，需要使用二级宏。

### ① 底层原理

```c
#define NUM 100
#define STR(x) #x
```

调用：

```c
STR(NUM)
```

得到：

```c
"NUM"
```

不是：

```c
"100"
```

因为：

```text
NUM 被 # 字符串化时，不会先展开。
```

### ② 代码用法

```c
#include <stdio.h>

#define NUM 100

#define STR(x) #x
#define XSTR(x) STR(x)

int main() {
    printf("%s\n", STR(NUM));
    printf("%s\n", XSTR(NUM));

    return 0;
}
```

输出：

```text
NUM
100
```

### ③ 使用场景

| 写法 | 结果 | 含义 |
| :--- | :--- | :--- |
| `STR(NUM)` | `"NUM"` | 直接把宏名字符串化 |
| `XSTR(NUM)` | `"100"` | 先展开宏值，再字符串化 |
| `STR(100)` | `"100"` | 直接字符串化数字文本 |

> [!tip] 记忆口诀
> **一层转宏名，两层转宏值。**

# 05 | 字符串化和预定义宏

> [!abstract] 核心结论
> `__LINE__`、`__FILE__` 等预定义宏常和字符串化运算符配合使用。尤其是 `__LINE__` 想变成字符串时，通常需要二级宏。

### ① 底层原理

```c
#define STR(x) #x
#define XSTR(x) STR(x)
```

调用：

```c
XSTR(__LINE__)
```

会先把：

```text
__LINE__
```

展开成当前行号，再字符串化。

### ② 代码用法

```c
#include <stdio.h>

#define STR(x) #x
#define XSTR(x) STR(x)

int main() {
    printf("文件名：%s\n", __FILE__);
    printf("当前行号：" XSTR(__LINE__) "\n");

    return 0;
}
```

输出示例：

```text
文件名：main.c
当前行号：8
```

### ③ 使用场景

```text
日志定位。
错误提示。
调试输出。
断言宏。
编译期信息生成。
```

> [!tip] 记忆口诀
> **行号要成字符串，二级宏来帮忙。**

# 06 | 字符串化和日志宏

> [!abstract] 核心结论
> 字符串化运算符可以让日志自动显示表达式文本、表达式结果、文件名和行号，非常适合调试。

### ① 底层原理

```c
#define LOG_INT(expr) \
    printf("[LOG] %s:%d: " #expr " = %d\n", __FILE__, __LINE__, (expr))
```

这里包含：

```text
__FILE__：
当前文件名。

__LINE__：
当前行号。

#expr：
表达式文本。

(expr)：
表达式结果。
```

### ② 代码用法

```c
#include <stdio.h>

#define LOG_INT(expr) \
    printf("[LOG] %s:%d: " #expr " = %d\n", __FILE__, __LINE__, (expr))

int main() {
    int a = 10;
    int b = 20;

    LOG_INT(a + b);
    LOG_INT(a * b);

    return 0;
}
```

输出示例：

```text
[LOG] main.c:9: a + b = 30
[LOG] main.c:10: a * b = 200
```

### ③ 使用场景

| 场景 | 作用 |
| :--- | :--- |
| 调试表达式 | 自动显示表达式和值 |
| 错误定位 | 自动带文件名和行号 |
| 日志输出 | 减少重复 printf |
| 断言检查 | 打印失败条件 |

> [!tip] 记忆口诀
> **日志想带表达式，井号一加自动变文字。**

# 07 | 字符串化中的空格处理

> [!abstract] 核心结论
> 字符串化会把宏参数转换成字符串，但参数中的多余空白通常会被规范化处理。它保留表达式含义，不保证完整保留原始排版。

### ① 底层原理

```c
STR(a     +      b)
```

通常会得到：

```c
"a + b"
```

不是：

```c
"a     +      b"
```

因为：

```text
预处理器处理的是记号序列，不是源文件排版截图。
```

### ② 代码用法

```c
#include <stdio.h>

#define STR(x) #x

int main() {
    printf("%s\n", STR(a     +      b));

    return 0;
}
```

输出通常是：

```text
a + b
```

### ③ 使用场景

```text
调试表达式时，通常只关心表达式内容。
不关心中间有多少空格。
```

> [!tip] 记忆口诀
> **字符串化保留表达式，不保留所有排版空格。**

# 08 | 字符串化和字符串自动拼接

> [!abstract] 核心结论
> 字符串化经常和 C 语言的字符串字面量自动拼接配合使用。相邻的字符串字面量会在编译时自动合并。

### ① 底层原理

```c
#define SHOW(x) printf(#x " = %d\n", (x))
```

调用：

```c
SHOW(num);
```

展开后类似：

```c
printf("num" " = %d\n", (num));
```

C 语言会自动拼接：

```c
"num" " = %d\n"
```

等价于：

```c
"num = %d\n"
```

### ② 代码用法

```c
#include <stdio.h>

#define SHOW(x) printf(#x " = %d\n", (x))

int main() {
    int num = 100;

    SHOW(num);
    SHOW(num + 1);

    return 0;
}
```

输出：

```text
num = 100
num + 1 = 101
```

### ③ 使用场景

```text
#x 生成表达式字符串。
后面的固定字符串自动拼接。
最终形成完整 printf 格式字符串。
```

> [!tip] 记忆口诀
> **井号生成字符串，相邻字符串自动粘。**

# 09 | 字符串化和记号连接符的区别

> [!abstract] 核心结论
> `#` 是字符串化运算符，用来把宏参数变成字符串；`##` 是记号连接符，用来把两个记号拼接成一个新记号。二者作用完全不同。

### ① 字符串化 `#`

```c
#define STR(x) #x
```

调用：

```c
STR(hello)
```

展开：

```c
"hello"
```

作用：

```text
把参数变成字符串。
```

### ② 记号连接 `##`

```c
#define MAKE_VAR(name) var_##name
```

调用：

```c
int MAKE_VAR(age) = 18;
```

展开：

```c
int var_age = 18;
```

作用：

```text
把 var_ 和 age 拼成 var_age。
```

### ③ 代码用法

```c
#include <stdio.h>

#define STR(x) #x
#define MAKE_VAR(name) var_##name

int main() {
    int MAKE_VAR(score) = 100;

    printf("%s = %d\n", STR(var_score), var_score);

    return 0;
}
```

输出：

```text
var_score = 100
```

### ④ 对比表

| 运算符 | 名称 | 作用 | 示例 |
| :--- | :--- | :--- | :--- |
| `#` | 字符串化 | 参数变字符串 | `STR(a)` → `"a"` |
| `##` | 记号连接 | 两个记号拼接 | `var_##age` → `var_age` |

> [!tip] 记忆口诀
> **一个井号变字符串，两个井号拼名字。**

# 10 | 字符串化运算符总结

> [!abstract] 核心结论
> 字符串化运算符 `#` 的底层核心是“在预处理阶段把宏参数转换成字符串字面量”。它常用于调试宏、日志宏、表达式打印和预定义宏字符串化。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>字符串化运算符核心总结：</strong>
<ul>
  <li><strong>符号：</strong><code>#</code>。</li>
  <li><strong>本质：</strong>预处理阶段把宏参数转换成字符串。</li>
  <li><strong>使用范围：</strong>只能用于宏函数的宏体中。</li>
  <li><strong>基本写法：</strong><code>#define STR(x) #x</code>。</li>
  <li><strong>典型展开：</strong><code>STR(hello)</code> 展开为 <code>"hello"</code>。</li>
  <li><strong>不计算：</strong><code>STR(1 + 2)</code> 得到 <code>"1 + 2"</code>。</li>
  <li><strong>宏值展开：</strong>想先展开再字符串化，要用二级宏。</li>
  <li><strong>常见用途：</strong>表达式打印、调试日志、行号字符串化。</li>
  <li><strong>字符串拼接：</strong><code>#x " = %d\n"</code> 会自动拼接成完整字符串。</li>
  <li><strong>区别：</strong><code>#</code> 是字符串化，<code>##</code> 是记号连接。</li>
</ul>
</div>

### ② 最底层一句话

```text
字符串化运算符 # = 在宏展开时，把宏参数原文包成一个字符串字面量。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **井号字符串化，参数变文字；一层转宏名，两层转宏值；它只替换，不计算。**
