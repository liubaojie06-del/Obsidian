# 01 | 头文件：给其他源文件看的接口说明书

> [!abstract] 核心结论
> 头文件通常以 `.h` 结尾，用来存放函数声明、结构体声明、宏定义、类型别名、`extern` 全局变量声明等内容。它的作用是让多个 `.c` 文件共享同一套接口声明。

### ① 底层原理

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>头文件的本质：</strong>
<ul>
  <li><strong>头文件不是单独运行的程序：</strong><code>.h</code> 文件不能直接运行。</li>
  <li><strong>头文件通常不单独编译：</strong>它会被 <code>#include</code> 展开到对应的 <code>.c</code> 文件里。</li>
  <li><strong>核心作用：</strong>告诉其他源文件有哪些函数、类型、宏、外部变量可以使用。</li>
  <li><strong>接口和实现分离：</strong><code>.h</code> 放声明，<code>.c</code> 放真正实现。</li>
  <li><strong>多文件桥梁：</strong>多个 <code>.c</code> 文件通过包含同一个头文件，获得统一接口。</li>
</ul>
</div>

### ② 代码用法

头文件示例：

```c
#ifndef CALC_H
#define CALC_H

int add(int a, int b);
int sub(int a, int b);

#endif
```

源文件实现：

```c
#include "calc.h"

int add(int a, int b) {
    return a + b;
}

int sub(int a, int b) {
    return a - b;
}
```

主函数调用：

```c
#include <stdio.h>
#include "calc.h"

int main() {
    printf("%d\n", add(10, 20));
    return 0;
}
```

### ③ 使用场景

| 文件 | 作用 | 常放内容 |
| :--- | :--- | :--- |
| `.h` | 接口声明 | 函数声明、结构体声明、宏、typedef、extern 声明 |
| `.c` | 功能实现 | 函数定义、变量定义、static 内部函数 |
| `main.c` | 程序入口 | 调用各个模块功能 |
| `Makefile` | 自动编译 | 管理多个 `.c` 文件编译链接 |

> [!tip] 记忆口诀
> **头文件是说明书，源文件是施工队；别人 include 说明书，就知道怎么调用。**

# 02 | 头文件里应该放什么

> [!abstract] 核心结论
> 头文件应该放“别人需要知道的声明”，不要随便放真正的变量定义和函数实现。头文件是对外接口，不是主要实现区。

### ① 底层原理

```text
头文件的目标：
让其他 .c 文件知道如何使用当前模块。
```

所以头文件应该放：

```text
函数声明。
结构体类型声明。
宏定义。
枚举定义。
typedef 类型别名。
extern 全局变量声明。
```

### ② 函数声明

```c
int add(int a, int b);
void print_menu(void);
```

含义：

```text
告诉编译器：
这些函数存在。
参数是什么。
返回值是什么。
```

### ③ 类型声明

```c
typedef struct Student {
    int id;
    char name[32];
    int age;
} Student;
```

作用：

```text
让其他 .c 文件也能使用 Student 类型。
```

### ④ 宏定义

```c
#define MAX_NAME_LEN 32
#define VERSION 1
```

作用：

```text
多个源文件共享同一份常量配置。
```

### ⑤ extern 声明

```c
extern int g_debug_mode;
```

含义：

```text
g_debug_mode 在某个 .c 文件中真正定义。
当前这里只是声明它可以被外部使用。
```

> [!tip] 记忆口诀
> **头文件放声明，声明给别人看；真正干活的代码，放到源文件里。**

# 03 | 头文件里不应该放什么

> [!abstract] 核心结论
> 头文件不要随便放普通全局变量定义和普通函数定义。因为头文件可能被多个 `.c` 文件包含，容易导致重复定义错误。

### ① 不要放普通全局变量定义

错误写法：

```c
// config.h
int g_num = 100;
```

问题：

```text
如果 main.c 和 test.c 都 include config.h，
那么两个 .c 文件都会各自生成一个 g_num 定义。
链接时可能重复定义。
```

正确写法：

```c
// config.h
extern int g_num;
```

```c
// config.c
int g_num = 100;
```

### ② 不要放普通函数定义

不推荐：

```c
// calc.h
int add(int a, int b) {
    return a + b;
}
```

问题：

```text
多个 .c 文件 include 后，add 会被定义多次。
```

正确：

```c
// calc.h
int add(int a, int b);
```

```c
// calc.c
int add(int a, int b) {
    return a + b;
}
```

### ③ 可以放 static inline 小函数

有时可以写：

```c
static inline int square(int x) {
    return x * x;
}
```

原因：

```text
static 限制每个 .c 文件内部可见。
inline 适合短小函数。
```

但初学阶段：

```text
普通函数实现优先放 .c 文件。
```

### ④ 不要放大量业务逻辑

```text
头文件越复杂，依赖越混乱。
编译越慢。
维护越难。
```

### ⑤ 重点

```text
头文件被 include 几次，内容就可能被复制展开几次。
所以不要在头文件里随便写真正定义。
```

> [!tip] 记忆口诀
> **头文件别造实体，只放声明和接口；变量定义放 c，函数实现也放 c。**

# 04 | 头文件保护：防止重复包含

> [!abstract] 核心结论
> 每个头文件都应该写头文件保护，防止同一个头文件被重复包含，避免结构体、类型、函数声明重复引发问题。

### ① 标准写法

```c
#ifndef CALC_H
#define CALC_H

int add(int a, int b);
int sub(int a, int b);

#endif
```

### ② 底层原理

第一次包含：

```text
CALC_H 还没有定义。
进入头文件内容。
执行 #define CALC_H。
```

第二次包含：

```text
CALC_H 已经定义。
#ifndef CALC_H 不成立。
跳过头文件内容。
```

### ③ 命名规则

头文件保护宏通常用：

```text
文件名大写 + 下划线 + H
```

例如：

```c
CALC_H
STUDENT_H
CONFIG_H
MY_PROJECT_UTILS_H
```

### ④ 另一种写法

```c
#pragma once
```

作用：

```text
告诉编译器这个头文件只包含一次。
```

优点：

```text
写法简单。
```

但更传统通用的是：

```c
#ifndef
#define
#endif
```

### ⑤ 推荐模板

```c
#ifndef 文件名大写_H
#define 文件名大写_H

// 头文件内容

#endif
```

> [!tip] 记忆口诀
> **ifndef 开门，define 上锁，endif 关门；头文件只进一次。**

# 05 | `#include < >` 和 `#include " "` 的区别

> [!abstract] 核心结论
> `#include <stdio.h>` 通常用于包含系统头文件；`#include "calc.h"` 通常用于包含自己写的头文件。二者主要区别是头文件搜索路径不同。

### ① 系统头文件

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
```

含义：

```text
从系统标准头文件目录中查找。
```

常用于：

```text
C 标准库头文件。
系统库头文件。
第三方库安装目录中的头文件。
```

### ② 自定义头文件

```c
#include "calc.h"
#include "student.h"
#include "config.h"
```

含义：

```text
通常先从当前源文件所在目录查找。
找不到再去其他 include 搜索路径查找。
```

### ③ 示例

```c
#include <stdio.h>
#include "calc.h"
```

含义：

```text
stdio.h 是系统库头文件。
calc.h 是自己项目里的模块头文件。
```

### ④ 配合 GCC 的 `-I`

如果头文件在 `include/` 目录：

```bash
gcc src/main.c src/calc.c -Iinclude -o app
```

这里：

```text
-Iinclude
```

表示：

```text
把 include 目录加入头文件搜索路径。
```

### ⑤ 对比表

| 写法 | 常见用途 | 搜索位置 |
| :--- | :--- | :--- |
| `#include <stdio.h>` | 系统头文件 | 系统 include 路径 |
| `#include "calc.h"` | 自定义头文件 | 当前目录优先，再查 include 路径 |

> [!tip] 记忆口诀
> **尖括号找系统，双引号找自己；找不到就加 -I 路径。**

# 06 | 头文件和源文件如何配合

> [!abstract] 核心结论
> 一个模块通常由一个 `.h` 和一个 `.c` 组成。`.h` 对外声明能用什么，`.c` 内部实现具体怎么做。

### ① 项目结构

```text
project/
├── main.c
├── student.c
└── student.h
```

### ② student.h

```c
#ifndef STUDENT_H
#define STUDENT_H

typedef struct Student {
    int id;
    char name[32];
    int age;
} Student;

void print_student(Student s);

#endif
```

### ③ student.c

```c
#include <stdio.h>
#include "student.h"

void print_student(Student s) {
    printf("id = %d\n", s.id);
    printf("name = %s\n", s.name);
    printf("age = %d\n", s.age);
}
```

### ④ main.c

```c
#include "student.h"

int main() {
    Student s = {1, "Tom", 18};

    print_student(s);

    return 0;
}
```

### ⑤ 编译运行

```bash
gcc main.c student.c -o app
./app
```

输出：

```text
id = 1
name = Tom
age = 18
```

> [!tip] 记忆口诀
> **h 暴露接口，c 实现功能；main include h，就能调用模块。**

# 07 | 头文件中的 extern 声明

> [!abstract] 核心结论
> 如果全局变量需要跨文件使用，头文件中应该写 `extern` 声明，真正变量定义只放在一个 `.c` 文件中。

### ① config.h

```c
#ifndef CONFIG_H
#define CONFIG_H

extern int g_debug_mode;

void print_config(void);

#endif
```

这里：

```text
extern int g_debug_mode;
只是声明，不分配空间。
```

### ② config.c

```c
#include <stdio.h>
#include "config.h"

int g_debug_mode = 1;

void print_config(void) {
    printf("g_debug_mode = %d\n", g_debug_mode);
}
```

这里：

```text
int g_debug_mode = 1;
才是真正定义。
```

### ③ main.c

```c
#include "config.h"

int main() {
    print_config();

    g_debug_mode = 0;

    print_config();

    return 0;
}
```

### ④ 编译运行

```bash
gcc main.c config.c -o app
./app
```

输出：

```text
g_debug_mode = 1
g_debug_mode = 0
```

### ⑤ 重点

```text
头文件里写 extern。
源文件里写真正定义。
```

如果头文件中写：

```c
int g_debug_mode = 1;
```

多个源文件包含后容易：

```text
重复定义。
```

> [!tip] 记忆口诀
> **共享变量头文件 extern，一处源文件真正定义。**

# 08 | 头文件中的结构体声明

> [!abstract] 核心结论
> 如果多个 `.c` 文件都需要使用同一个结构体类型，就应该把结构体类型声明放到头文件中。

### ① 完整结构体定义放头文件

student.h：

```c
#ifndef STUDENT_H
#define STUDENT_H

typedef struct Student {
    int id;
    char name[32];
    int age;
} Student;

void print_student(Student s);

#endif
```

这样其他文件可以：

```c
Student s;
s.id = 1;
```

因为：

```text
其他文件知道 Student 的完整内部结构。
```

### ② 不透明结构体声明

如果不想让外部知道结构体内部字段，可以只在头文件写：

```c
typedef struct Student Student;
```

然后在 `.c` 文件中写完整定义：

```c
struct Student {
    int id;
    char name[32];
    int age;
};
```

这种方式叫：

```text
不透明结构体。
```

外部只能通过函数操作它。

### ③ 普通场景

学习阶段常用：

```c
typedef struct Student {
    int id;
    char name[32];
    int age;
} Student;
```

放头文件。

### ④ 封装场景

工程中如果想隐藏内部实现：

```c
typedef struct Student Student;
```

放头文件。

完整字段放：

```text
student.c
```

### ⑤ 重点

```text
外部需要访问字段：
头文件放完整结构体。

外部只需要拿指针操作：
头文件可以只放不透明声明。
```

> [!tip] 记忆口诀
> **要让别人看字段，结构体放头文件；想隐藏内部，就只暴露名字。**

# 09 | 头文件常见错误

> [!abstract] 核心结论
> 头文件常见错误包括没有头文件保护、在头文件中定义全局变量、在头文件中写普通函数实现、循环包含、声明和定义不一致。

### ① 错误 1：没有头文件保护

错误：

```c
// calc.h
int add(int a, int b);
```

更推荐：

```c
#ifndef CALC_H
#define CALC_H

int add(int a, int b);

#endif
```

### ② 错误 2：头文件定义全局变量

错误：

```c
// config.h
int g_debug_mode = 1;
```

正确：

```c
// config.h
extern int g_debug_mode;
```

```c
// config.c
int g_debug_mode = 1;
```

### ③ 错误 3：头文件写普通函数实现

错误：

```c
// calc.h
int add(int a, int b) {
    return a + b;
}
```

正确：

```c
// calc.h
int add(int a, int b);
```

```c
// calc.c
int add(int a, int b) {
    return a + b;
}
```

### ④ 错误 4：声明和定义不一致

calc.h：

```c
int add(int a, int b);
```

calc.c：

```c
double add(double a, double b) {
    return a + b;
}
```

问题：

```text
声明和定义类型不一致。
会导致编译或链接问题。
```

### ⑤ 错误 5：循环包含

a.h：

```c
#include "b.h"
```

b.h：

```c
#include "a.h"
```

问题：

```text
两个头文件互相包含，容易导致依赖混乱。
```

解决：

```text
使用头文件保护。
减少不必要 include。
能前向声明就前向声明。
```

> [!tip] 记忆口诀
> **头文件三不要：不要定义变量，不要写普通实现，不要忘保护。**

# 10 | 头文件总结

> [!abstract] 核心结论
> 头文件的底层核心是“被 `#include` 展开到源文件中的接口声明集合”。它让多个 `.c` 文件共享函数、类型、宏和外部变量声明，是 C 语言多文件编程的基础。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>头文件核心总结：</strong>
<ul>
  <li><strong>后缀：</strong>通常是 <code>.h</code>。</li>
  <li><strong>本质：</strong>接口声明文件，被 <code>#include</code> 展开到 <code>.c</code> 中。</li>
  <li><strong>主要内容：</strong>函数声明、结构体声明、宏定义、typedef、extern 声明。</li>
  <li><strong>不建议放：</strong>普通全局变量定义、普通函数定义、大量业务逻辑。</li>
  <li><strong>头文件保护：</strong>使用 <code>#ifndef</code>、<code>#define</code>、<code>#endif</code> 防止重复包含。</li>
  <li><strong>系统头文件：</strong>通常用 <code>#include &lt;stdio.h&gt;</code>。</li>
  <li><strong>自定义头文件：</strong>通常用 <code>#include "calc.h"</code>。</li>
  <li><strong>extern：</strong>头文件中声明外部变量，源文件中定义变量。</li>
  <li><strong>结构体：</strong>多个文件共用结构体时，类型声明放头文件。</li>
  <li><strong>核心规则：</strong><code>.h</code> 放声明，<code>.c</code> 放实现。</li>
</ul>
</div>

### ② 最底层一句话

```text
头文件 = 给其他 .c 文件看的接口清单；include 后，编译器就知道有哪些函数、类型和外部变量可以使用。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **头文件放声明，源文件放实现；尖括号找系统，双引号找自己；变量共享用 extern，重复包含用保护。**
