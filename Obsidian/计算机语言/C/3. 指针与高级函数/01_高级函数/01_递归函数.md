# 01 | 递归函数：函数自己调用自己

> [!abstract] 核心结论
> 递归函数就是在函数内部直接或间接调用自己。它适合解决“问题可以拆成更小的同类问题”的场景，例如阶乘、斐波那契、树遍历、目录遍历、汉诺塔等。

### ① 底层原理：递归不是循环，而是不断创建新的函数栈帧

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>递归函数的本质：</strong>
<ul>
  <li><strong>自己调用自己：</strong> 函数内部再次调用同一个函数。</li>
  <li><strong>每次调用都是新的栈帧：</strong> 每一层递归都有自己的形参、局部变量和返回地址。</li>
  <li><strong>必须有递归出口：</strong> 没有出口会无限调用，最终栈溢出。</li>
  <li><strong>问题规模必须变小：</strong> 每次递归都应该更接近结束条件。</li>
  <li><strong>返回过程逐层回退：</strong> 最深层结束后，结果一层一层返回到最外层。</li>
</ul>
</div>

### ② 基本结构

```c
返回值类型 函数名(参数) {
    if (递归出口条件) {
        return 结束结果;
    }

    return 函数名(更小规模的问题);
}
```

### ③ 一句话理解

```text
递归 = 把大问题拆成小问题，直到小到能直接解决，再一层层返回结果。
```

> [!tip] 记忆口诀
> **递归两件事：出口要明确，规模要变小。**

# 02 | 递归函数的两个核心条件

> [!abstract] 核心结论
> 一个正确的递归函数必须满足两个条件：第一，有明确的递归出口；第二，每次递归调用都让问题规模变小，并逐渐靠近出口。

### ① 条件一：递归出口

递归出口就是：

```text
什么时候不再继续调用自己。
```

例如阶乘：

```text
1! = 1
0! = 1
```

所以：

```c
if (n == 0 || n == 1) {
    return 1;
}
```

就是出口。

### ② 条件二：问题规模变小

例如：

```c
return n * factorial(n - 1);
```

这里：

```text
n 变成 n - 1。
```

每次都更接近：

```text
n == 1
```

### ③ 错误递归示例

```c
int func(int n) {
    return func(n);
}
```

问题：

```text
没有出口。
参数也没有变化。
会无限递归。
```

### ④ 另一个错误示例

```c
int func(int n) {
    if (n == 1) {
        return 1;
    }

    return func(n + 1);
}
```

问题：

```text
n 不断变大。
离 n == 1 越来越远。
```

### ⑤ 正确递归思路

```text
1. 明确最小问题是什么。
2. 最小问题直接返回。
3. 大问题拆成小问题。
4. 小问题结果返回后，组合成大问题答案。
```

> [!tip] 记忆口诀
> **出口决定能不能停，规模变小决定能不能到出口。**

# 03 | 阶乘递归：最经典的递归函数

> [!abstract] 核心结论
> 阶乘 `n!` 可以定义为 `n * (n - 1)!`，这正好符合递归思想：大问题依赖一个更小的同类问题。

### ① 数学定义

```text
5! = 5 × 4 × 3 × 2 × 1
```

递归定义：

```text
n! = n × (n - 1)!
1! = 1
0! = 1
```

### ② 代码实现

```c
#include <stdio.h>

int factorial(int n) {
    if (n == 0 || n == 1) {
        return 1;
    }

    return n * factorial(n - 1);
}

int main() {
    int result = factorial(5);

    printf("%d\n", result);

    return 0;
}
```

输出：

```text
120
```

### ③ 调用展开

```text
factorial(5)
= 5 * factorial(4)
= 5 * 4 * factorial(3)
= 5 * 4 * 3 * factorial(2)
= 5 * 4 * 3 * 2 * factorial(1)
= 5 * 4 * 3 * 2 * 1
= 120
```

### ④ 栈帧理解

```text
factorial(5) 栈帧
factorial(4) 栈帧
factorial(3) 栈帧
factorial(2) 栈帧
factorial(1) 栈帧
```

当：

```text
factorial(1) 返回 1
```

之后：

```text
factorial(2) 返回 2
factorial(3) 返回 6
factorial(4) 返回 24
factorial(5) 返回 120
```

### ⑤ 重点

```text
递归先不断深入。
到达出口后，再逐层返回。
```

> [!tip] 记忆口诀
> **阶乘递归：n 乘前一个，直到 1 停下。**

# 04 | 递归调用与栈帧变化

> [!abstract] 核心结论
> 每次递归调用都会创建新的函数栈帧。不同层递归中的形参和局部变量互不相同，虽然函数名一样，但每一层调用都是独立的一次执行。

### ① 示例代码

```c
#include <stdio.h>

void print_down(int n) {
    if (n == 0) {
        return;
    }

    printf("进入：n = %d\n", n);
    print_down(n - 1);
    printf("返回：n = %d\n", n);
}

int main() {
    print_down(3);

    return 0;
}
```

输出：

```text
进入：n = 3
进入：n = 2
进入：n = 1
返回：n = 1
返回：n = 2
返回：n = 3
```

### ② 执行过程

```text
print_down(3)
  打印进入 3
  调用 print_down(2)

print_down(2)
  打印进入 2
  调用 print_down(1)

print_down(1)
  打印进入 1
  调用 print_down(0)

print_down(0)
  return
```

然后开始回退：

```text
print_down(1) 继续执行递归调用后面的代码
print_down(2) 继续执行递归调用后面的代码
print_down(3) 继续执行递归调用后面的代码
```

### ③ 栈结构示意

递归深入时：

```text
栈顶
┌────────────────┐
│ print_down(1)  │
├────────────────┤
│ print_down(2)  │
├────────────────┤
│ print_down(3)  │
├────────────────┤
│ main           │
└────────────────┘
栈底
```

递归返回时：

```text
print_down(1) 先释放
print_down(2) 再释放
print_down(3) 最后释放
```

### ④ 重点理解

```text
递归调用不是回到同一个函数空间。
每调用一次，都会开一个新的函数栈帧。
```

### ⑤ 栈帧里通常保存

```text
当前层参数。
当前层局部变量。
返回地址。
临时计算结果。
保存的寄存器。
```

> [!tip] 记忆口诀
> **递归一层一栈帧，进去一层压一层；返回一层弹一层。**

# 05 | 递归的执行顺序：先递进，再回归

> [!abstract] 核心结论
> 递归执行通常分为两个阶段：递进阶段不断调用自己，直到遇到出口；回归阶段从最深层开始逐层返回，并完成剩余计算。

### ① 递进阶段

例如：

```c
factorial(5)
```

会先变成：

```text
factorial(5)
factorial(4)
factorial(3)
factorial(2)
factorial(1)
```

这叫：

```text
递进阶段。
```

也就是：

```text
不断往更小问题深入。
```

### ② 回归阶段

当：

```text
factorial(1) 返回 1
```

之后：

```text
factorial(2) 得到 2 * 1
factorial(3) 得到 3 * 2
factorial(4) 得到 4 * 6
factorial(5) 得到 5 * 24
```

这叫：

```text
回归阶段。
```

### ③ 代码位置影响顺序

```c
void test(int n) {
    if (n == 0) {
        return;
    }

    printf("%d ", n);
    test(n - 1);
}
```

调用：

```c
test(3);
```

输出：

```text
3 2 1
```

因为：

```text
先打印，再递归。
```

### ④ 改变打印位置

```c
void test(int n) {
    if (n == 0) {
        return;
    }

    test(n - 1);
    printf("%d ", n);
}
```

调用：

```c
test(3);
```

输出：

```text
1 2 3
```

因为：

```text
先递归到底，再回退打印。
```

### ⑤ 重点

```text
递归调用前的代码，在递进阶段执行。
递归调用后的代码，在回归阶段执行。
```

> [!tip] 记忆口诀
> **递归前代码先进时做，递归后代码返回时做。**

# 06 | 斐波那契递归：一个函数调用两个自己

> [!abstract] 核心结论
> 斐波那契数列也可以用递归描述：`fib(n) = fib(n - 1) + fib(n - 2)`。它能展示递归思想，但普通递归会产生大量重复计算。

### ① 数列定义

```text
fib(0) = 0
fib(1) = 1
fib(n) = fib(n - 1) + fib(n - 2)
```

数列：

```text
0, 1, 1, 2, 3, 5, 8, 13, ...
```

### ② 代码实现

```c
#include <stdio.h>

int fib(int n) {
    if (n == 0) {
        return 0;
    }

    if (n == 1) {
        return 1;
    }

    return fib(n - 1) + fib(n - 2);
}

int main() {
    printf("%d\n", fib(6));

    return 0;
}
```

输出：

```text
8
```

### ③ 调用展开

```text
fib(6)
= fib(5) + fib(4)
= fib(4) + fib(3) + fib(3) + fib(2)
```

可以看到：

```text
fib(4)
fib(3)
fib(2)
```

会被重复计算很多次。

### ④ 这种递归的问题

```text
代码很直观。
但效率很低。
重复计算多。
n 稍微大一点就很慢。
```

### ⑤ 更适合用循环或记忆化

循环写法：

```c
int fib_loop(int n) {
    if (n == 0) {
        return 0;
    }

    int a = 0;
    int b = 1;

    for (int i = 2; i <= n; i++) {
        int c = a + b;
        a = b;
        b = c;
    }

    return b;
}
```

> [!tip] 记忆口诀
> **斐波那契递归好理解，但重复计算多；能循环就循环，能记忆就记忆。**

# 07 | 递归和循环的区别

> [!abstract] 核心结论
> 递归和循环都能重复执行逻辑。循环靠控制变量反复执行同一段代码；递归靠函数不断调用自己。递归表达更接近数学定义，但会消耗更多栈空间。

### ① 循环写阶乘

```c
int factorial_loop(int n) {
    int result = 1;

    for (int i = 1; i <= n; i++) {
        result *= i;
    }

    return result;
}
```

### ② 递归写阶乘

```c
int factorial_rec(int n) {
    if (n == 0 || n == 1) {
        return 1;
    }

    return n * factorial_rec(n - 1);
}
```

### ③ 对比表

| 对比项 | 循环 | 递归 |
| :--- | :--- | :--- |
| 重复方式 | 代码块反复执行 | 函数反复调用自己 |
| 内存消耗 | 通常较少 | 每层调用都有栈帧 |
| 可读性 | 对线性问题清晰 | 对树形、分治问题清晰 |
| 风险 | 死循环 | 无限递归、栈溢出 |
| 适合场景 | 普通重复计算 | 树、分治、回溯、嵌套结构 |

### ④ 什么时候用循环

```text
简单累加。
简单计数。
线性遍历数组。
固定次数重复。
性能要求高。
```

### ⑤ 什么时候用递归

```text
树结构。
目录结构。
分治算法。
回溯搜索。
问题天然可以拆成同类子问题。
```

> [!tip] 记忆口诀
> **循环省栈适合线性，递归清晰适合嵌套和分治。**

# 08 | 递归常见错误

> [!abstract] 核心结论
> 递归常见错误包括没有递归出口、参数没有向出口靠近、递归层数太深、重复计算过多、返回值没有正确接住。

### ① 错误 1：没有递归出口

```c
void test() {
    test();
}
```

问题：

```text
无限递归。
最终栈溢出。
```

### ② 错误 2：参数没有变化

```c
int sum(int n) {
    if (n == 1) {
        return 1;
    }

    return n + sum(n);
}
```

问题：

```text
一直调用 sum(n)。
永远到不了 n == 1。
```

### ③ 错误 3：方向错了

```c
int sum(int n) {
    if (n == 1) {
        return 1;
    }

    return n + sum(n + 1);
}
```

问题：

```text
n 越来越大。
离出口越来越远。
```

### ④ 错误 4：忘记返回递归结果

错误：

```c
int factorial(int n) {
    if (n == 1) {
        return 1;
    }

    factorial(n - 1);  // 错误，没有 return
}
```

正确：

```c
int factorial(int n) {
    if (n == 1) {
        return 1;
    }

    return n * factorial(n - 1);
}
```

### ⑤ 错误 5：递归层数太深

例如：

```c
factorial(100000);
```

问题：

```text
递归层数太深。
大量栈帧压入栈区。
可能栈溢出。
```

> [!tip] 记忆口诀
> **递归错三类：没出口、不靠近、忘 return；层数太深会爆栈。**

# 09 | 递归适合解决什么问题

> [!abstract] 核心结论
> 递归适合解决可以自然拆分为“当前问题 + 子问题”的场景，尤其是树形结构、分治算法、回溯算法和嵌套结构处理。

### ① 树结构遍历

例如：

```text
遍历一棵树：
先处理当前节点。
再递归处理左子树。
再递归处理右子树。
```

这种结构天然递归。

### ② 目录遍历

```text
遍历文件夹：
遇到普通文件就处理。
遇到子文件夹就递归进入。
```

因为：

```text
文件夹里面还可以有文件夹。
```

### ③ 分治算法

分治思想：

```text
把大问题拆成多个小问题。
分别解决。
再合并结果。
```

典型例子：

```text
归并排序。
快速排序。
二分查找。
```

### ④ 回溯算法

例如：

```text
迷宫搜索。
八皇后。
全排列。
组合问题。
路径搜索。
```

特点：

```text
做一个选择。
递归进入下一层。
不行就撤销选择，尝试别的路。
```

### ⑤ 嵌套结构

例如：

```text
括号匹配。
表达式求值。
JSON / XML 解析。
嵌套菜单。
```

它们天然有层级结构。

> [!tip] 记忆口诀
> **看到树、目录、分治、回溯、嵌套结构，就可以想到递归。**

# 10 | 递归函数总结

> [!abstract] 核心结论
> 递归函数的底层核心是“函数自己调用自己，每次调用创建新栈帧，直到递归出口停止，再逐层返回”。递归写法清晰，但要控制出口、规模和栈深度。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>递归函数核心总结：</strong>
<ul>
  <li><strong>本质：</strong> 函数直接或间接调用自己。</li>
  <li><strong>递归出口：</strong> 决定什么时候停止继续调用。</li>
  <li><strong>规模变小：</strong> 每次递归都要更接近出口。</li>
  <li><strong>栈帧：</strong> 每调用一次函数，就创建一层新的栈帧。</li>
  <li><strong>递进阶段：</strong> 不断调用自己，深入到最小问题。</li>
  <li><strong>回归阶段：</strong> 从最深层开始逐层返回结果。</li>
  <li><strong>典型例子：</strong> 阶乘、斐波那契、树遍历、目录遍历、汉诺塔。</li>
  <li><strong>常见风险：</strong> 无限递归、栈溢出、重复计算、忘记 return。</li>
  <li><strong>和循环：</strong> 循环更省栈，递归更适合树形和分治问题。</li>
  <li><strong>核心判断：</strong> 能否拆成更小的同类问题。</li>
</ul>
</div>

### ② 最底层一句话

```text
递归函数 = 函数不断调用自己，每层保存自己的现场，直到出口返回，再一层层恢复。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **递归先深入，再回退；出口必须有，规模必须小；一层调用一层栈，太深就会爆。**
