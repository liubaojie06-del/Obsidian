# 01 | 函数指针：保存函数入口地址的指针

> [!abstract] 核心结论
> 函数指针本质上是一个指针，只不过它保存的不是普通变量地址，而是函数的入口地址。通过函数指针，可以间接调用函数，也可以把函数作为参数传给另一个函数。

### ① 底层原理：函数名本质上可以代表函数入口地址

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>函数指针的本质：</strong>
<ul>
  <li><strong>函数：</strong> 编译后是一段机器指令，通常存放在代码区。</li>
  <li><strong>函数名：</strong> 在很多表达式中可以表示函数入口地址。</li>
  <li><strong>函数指针：</strong> 保存函数入口地址的指针变量。</li>
  <li><strong>间接调用：</strong> 通过函数指针也可以执行目标函数。</li>
  <li><strong>核心用途：</strong> 回调函数、函数选择表、菜单分发、排序规则传入、状态机跳转。</li>
</ul>
</div>

### ② 基本写法

```c
int (*p)(int, int);
```

读法：

```text
p 先和 * 结合。
所以 p 是指针。

后面的 (int, int) 表示：
p 指向的函数需要两个 int 参数。

前面的 int 表示：
p 指向的函数返回 int。
```

结论：

```text
p 是函数指针。
它可以指向“返回 int，参数是 int、int”的函数。
```

### ③ 一句话理解

```text
函数指针 = 保存函数地址的变量。
```

> [!tip] 记忆口诀
> **函数名是入口，函数指针存入口；指向哪个函数，就能调用哪个函数。**

# 02 | 函数指针的基本使用

> [!abstract] 核心结论
> 函数指针必须和目标函数的返回值类型、参数列表匹配。把函数名赋给函数指针后，就可以通过函数指针调用该函数。

### ① 示例代码

```c
#include <stdio.h>

int add(int a, int b) {
    return a + b;
}

int main() {
    int (*p)(int, int);

    p = add;

    int result = p(10, 20);

    printf("%d\n", result);

    return 0;
}
```

输出：

```text
30
```

### ② 拆开理解

```c
int (*p)(int, int);
```

含义：

```text
p 是函数指针。
它指向的函数返回 int。
它指向的函数有两个 int 参数。
```

```c
p = add;
```

含义：

```text
把 add 函数的入口地址赋给 p。
```

```c
p(10, 20);
```

含义：

```text
通过 p 指向的函数入口，调用 add(10, 20)。
```

### ③ 也可以写成 `(*p)(10, 20)`

```c
int result = (*p)(10, 20);
```

和：

```c
int result = p(10, 20);
```

效果一样。

### ④ 为什么可以不写 `&add`

通常可以写：

```c
p = add;
```

也可以写：

```c
p = &add;
```

因为：

```text
函数名在表达式中通常会转换为函数地址。
```

### ⑤ 重点

```text
函数指针不是调用函数的结果。
函数指针保存的是函数入口地址。
```

> [!tip] 记忆口诀
> **p = add 是存地址，p(参数) 才是调用函数。**

# 03 | 函数指针声明怎么读

> [!abstract] 核心结论
> 读函数指针声明时，先找变量名，再看变量名先和谁结合。`int (*p)(int, int)` 中，括号让 `p` 先和 `*` 结合，所以 `p` 是指针；后面的参数列表说明它指向函数。

### ① 正确声明

```c
int (*p)(int, int);
```

拆解：

```text
p 先和 * 结合：
p 是指针。

(*p) 后面有 (int, int)：
p 指向函数。

最前面 int：
这个函数返回 int。
```

完整含义：

```text
p 是一个函数指针。
指向返回 int、参数为 int 和 int 的函数。
```

### ② 如果少了括号

```c
int *p(int, int);
```

这个不是函数指针。

读法：

```text
p 先和 (int, int) 结合。
所以 p 是函数。

前面 int * 表示：
函数返回 int *。
```

结论：

```text
int *p(int, int) 是指针函数，不是函数指针。
```

### ③ 对比表

| 写法 | 名称 | 本质 | 含义 |
| :--- | :--- | :--- | :--- |
| `int (*p)(int, int)` | 函数指针 | 指针 | 指向返回 `int`、参数为两个 `int` 的函数 |
| `int *p(int, int)` | 指针函数 | 函数 | 返回 `int *` 的函数 |
| `void (*p)(void)` | 函数指针 | 指针 | 指向无参、无返回值函数 |
| `char *func(void)` | 指针函数 | 函数 | 返回 `char *` 的函数 |

### ④ 快速判断技巧

```text
名字先和 * 结合：
它是指针。

名字先和 () 结合：
它是函数。
```

### ⑤ 重点

```text
函数指针的括号不能省。
少了括号，含义就变成指针函数。
```

> [!tip] 记忆口诀
> **函数指针有括号，先是指针后指函数；没括号先成函数。**

# 04 | 函数指针类型必须匹配

> [!abstract] 核心结论
> 函数指针只能安全指向返回值类型和参数列表匹配的函数。类型不匹配会导致调用约定、参数解释、返回值解释出错。

### ① 正确匹配

```c
#include <stdio.h>

int add(int a, int b) {
    return a + b;
}

int main() {
    int (*p)(int, int) = add;

    printf("%d\n", p(10, 20));

    return 0;
}
```

这里：

```text
p 的类型：
返回 int，参数 int、int。

add 的类型：
返回 int，参数 int、int。

完全匹配。
```

### ② 不匹配示例

```c
double div(double a, double b) {
    return a / b;
}

int (*p)(int, int) = div;  // 错误，不匹配
```

问题：

```text
p 期待的是 int、int 参数和 int 返回值。
div 实际是 double、double 参数和 double 返回值。
```

### ③ 参数个数也要匹配

```c
int test(int a) {
    return a;
}

int (*p)(int, int) = test;  // 错误
```

原因：

```text
p 指向的函数应该接收两个 int。
test 只接收一个 int。
```

### ④ 返回值类型也要匹配

```c
void hello(int a, int b) {
    printf("hello\n");
}

int (*p)(int, int) = hello;  // 错误
```

原因：

```text
p 期待返回 int。
hello 返回 void。
```

### ⑤ 为什么严格匹配重要

函数调用底层涉及：

```text
参数放到哪些寄存器。
参数放到栈上哪里。
返回值从哪里取。
调用结束如何恢复现场。
```

类型不匹配会导致：

```text
结果错误。
程序崩溃。
未定义行为。
```

> [!tip] 记忆口诀
> **函数指针看三样：返回值、参数个数、参数类型；三者匹配才能安全调用。**

# 05 | 函数指针作为函数参数：回调函数

> [!abstract] 核心结论
> 函数指针可以作为参数传给另一个函数，这种用法叫回调函数。调用者把“要执行的函数逻辑”传进去，被调用函数在合适的位置反过来调用它。

### ① 基本示例

```c
#include <stdio.h>

int add(int a, int b) {
    return a + b;
}

int sub(int a, int b) {
    return a - b;
}

void calculate(int x, int y, int (*op)(int, int)) {
    int result = op(x, y);
    printf("result = %d\n", result);
}

int main() {
    calculate(10, 20, add);
    calculate(10, 20, sub);

    return 0;
}
```

输出：

```text
result = 30
result = -10
```

### ② 参数解释

```c
void calculate(int x, int y, int (*op)(int, int))
```

含义：

```text
x、y 是普通整数参数。
op 是函数指针参数。
op 可以指向返回 int、参数为 int、int 的函数。
```

### ③ 调用过程

```c
calculate(10, 20, add);
```

底层理解：

```text
把 add 函数入口地址传给 op。
calculate 内部执行 op(10, 20)。
实际调用 add(10, 20)。
```

### ④ 为什么叫回调

```text
调用者把函数传进去。
被调用函数内部再回头调用这个函数。
```

所以叫：

```text
Callback
回调函数
```

### ⑤ 适合场景

```text
排序规则。
菜单处理。
事件处理。
定时器回调。
驱动回调。
状态机动作。
通用算法中传入具体操作。
```

> [!tip] 记忆口诀
> **函数指针当参数，就是把一段函数逻辑传进去；里面再调用，就是回调。**

# 06 | 函数指针数组：一组函数入口地址

> [!abstract] 核心结论
> 函数指针数组是一个数组，数组中的每个元素都是函数指针。它常用于菜单分发、命令表、状态机跳转表等场景。

### ① 基本写法

```c
int (*ops[3])(int, int);
```

读法：

```text
ops 先和 [3] 结合。
所以 ops 是数组。

数组中的每个元素是函数指针。
每个函数指针指向返回 int、参数为 int、int 的函数。
```

结论：

```text
ops 是函数指针数组。
```

### ② 示例代码

```c
#include <stdio.h>

int add(int a, int b) {
    return a + b;
}

int sub(int a, int b) {
    return a - b;
}

int mul(int a, int b) {
    return a * b;
}

int main() {
    int (*ops[3])(int, int) = {add, sub, mul};

    printf("%d\n", ops[0](10, 20));
    printf("%d\n", ops[1](10, 20));
    printf("%d\n", ops[2](10, 20));

    return 0;
}
```

输出：

```text
30
-10
200
```

### ③ 内存理解

```text
ops 函数指针数组：

ops[0] ───► add 函数入口
ops[1] ───► sub 函数入口
ops[2] ───► mul 函数入口
```

### ④ 菜单分发示例

```c
int choice = 0;

printf("%d\n", ops[choice](10, 20));
```

如果：

```text
choice = 0 调 add。
choice = 1 调 sub。
choice = 2 调 mul。
```

### ⑤ 重点

```text
函数指针数组不是保存函数本体。
它保存的是多个函数入口地址。
```

> [!tip] 记忆口诀
> **函数指针数组是一张函数地址表，下标选函数，括号来调用。**

# 07 | typedef 简化函数指针

> [!abstract] 核心结论
> 函数指针声明比较复杂，可以用 `typedef` 给函数指针类型起别名，让代码更清晰，尤其适合回调函数和函数指针数组。

### ① 不用 typedef

```c
int (*p)(int, int);
```

如果作为参数：

```c
void calculate(int x, int y, int (*op)(int, int));
```

看起来比较复杂。

### ② 使用 typedef

```c
typedef int (*Operation)(int, int);
```

含义：

```text
Operation 是一个函数指针类型。
它可以指向返回 int、参数为 int、int 的函数。
```

之后可以写：

```c
Operation p;
```

等价于：

```c
int (*p)(int, int);
```

### ③ 完整示例

```c
#include <stdio.h>

typedef int (*Operation)(int, int);

int add(int a, int b) {
    return a + b;
}

int sub(int a, int b) {
    return a - b;
}

void calculate(int x, int y, Operation op) {
    printf("%d\n", op(x, y));
}

int main() {
    Operation p = add;

    printf("%d\n", p(10, 20));

    calculate(10, 20, sub);

    return 0;
}
```

输出：

```text
30
-10
```

### ④ 函数指针数组简化

不用 typedef：

```c
int (*ops[3])(int, int) = {add, sub, mul};
```

使用 typedef：

```c
Operation ops[3] = {add, sub, mul};
```

更清晰。

### ⑤ 重点

```text
typedef 不是创建新函数。
只是给复杂的函数指针类型起一个名字。
```

> [!tip] 记忆口诀
> **函数指针太绕，用 typedef 起别名；类型清楚，代码好读。**

# 08 | 函数指针和指针函数的区别

> [!abstract] 核心结论
> 函数指针是指向函数的指针；指针函数是返回指针的函数。二者名字很像，但本质完全不同，关键看变量名或函数名先和 `*` 结合，还是先和 `()` 结合。

### ① 函数指针

```c
int (*p)(int, int);
```

判断：

```text
p 先和 * 结合。
所以 p 是指针。

后面有 (int, int)。
说明 p 指向函数。
```

结论：

```text
p 是函数指针。
```

含义：

```text
p 保存函数入口地址。
```

### ② 指针函数

```c
int *func(int, int);
```

判断：

```text
func 先和 (int, int) 结合。
所以 func 是函数。

前面 int *。
说明函数返回 int *。
```

结论：

```text
func 是指针函数。
```

含义：

```text
func 调用后返回一个 int 地址。
```

### ③ 对比表

| 写法 | 名称 | 本质 | 含义 |
| :--- | :--- | :--- | :--- |
| `int (*p)(int, int)` | 函数指针 | 指针 | 指向函数 |
| `int *func(int, int)` | 指针函数 | 函数 | 返回指针 |
| `char (*p)(void)` | 函数指针 | 指针 | 指向返回 `char` 的函数 |
| `char *func(void)` | 指针函数 | 函数 | 返回 `char *` |

### ④ 快速判断

```text
名字先和 * 结合：
它是指针。

名字先和 () 结合：
它是函数。
```

### ⑤ 重点区别

```text
函数指针：
拿函数地址，用来调用函数。

指针函数：
调用函数，得到一个地址。
```

> [!tip] 记忆口诀
> **函数指针先是指针，指针函数先是函数；括号决定身份。**

# 09 | 函数指针常见错误

> [!abstract] 核心结论
> 函数指针常见错误包括声明少括号、函数类型不匹配、调用空函数指针、把函数调用结果赋给函数指针、和指针函数混淆。

### ① 错误 1：少写括号

错误：

```c
int *p(int, int);
```

这不是函数指针，而是：

```text
返回 int * 的函数声明。
```

正确：

```c
int (*p)(int, int);
```

### ② 错误 2：类型不匹配

```c
double div(double a, double b) {
    return a / b;
}

int (*p)(int, int) = div;  // 错误
```

原因：

```text
返回值和参数类型都不匹配。
```

### ③ 错误 3：调用空函数指针

```c
int (*p)(int, int) = NULL;

p(10, 20);  // 错误
```

正确：

```c
if (p != NULL) {
    p(10, 20);
}
```

### ④ 错误 4：把函数调用结果赋给函数指针

错误：

```c
int (*p)(int, int);

p = add(10, 20);
```

问题：

```text
add(10, 20) 是函数调用结果。
它是 int，不是函数地址。
```

正确：

```c
p = add;
```

### ⑤ 错误 5：回调参数不会写

错误：

```c
void calculate(int x, int y, int op(int, int));
```

这个在函数参数中可以退化为函数指针，但初学时容易混乱。

更清晰写法：

```c
void calculate(int x, int y, int (*op)(int, int));
```

> [!tip] 记忆口诀
> **函数指针存函数名，不存函数调用结果；调用前先判空，类型必须匹配。**

# 10 | 函数指针总结

> [!abstract] 核心结论
> 函数指针的底层核心是“保存函数入口地址，并通过这个地址间接调用函数”。它让 C 语言可以实现回调、函数表、菜单分发、状态机等灵活结构。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>函数指针核心总结：</strong>
<ul>
  <li><strong>本质：</strong> 指针。</li>
  <li><strong>保存内容：</strong> 函数入口地址。</li>
  <li><strong>核心写法：</strong> <code>int (*p)(int, int)</code>。</li>
  <li><strong>赋值：</strong> <code>p = add;</code>，把函数地址赋给指针。</li>
  <li><strong>调用：</strong> <code>p(10, 20)</code> 或 <code>(*p)(10, 20)</code>。</li>
  <li><strong>类型匹配：</strong> 返回值类型、参数个数、参数类型必须匹配。</li>
  <li><strong>作为参数：</strong> 可实现回调函数。</li>
  <li><strong>数组形式：</strong> 函数指针数组可实现函数表和菜单分发。</li>
  <li><strong>typedef：</strong> 可简化复杂函数指针类型。</li>
  <li><strong>易混点：</strong> 函数指针是指针，指针函数是函数。</li>
</ul>
</div>

### ② 最底层一句话

```text
函数指针 = 一个保存函数入口地址的指针变量；通过它可以间接调用目标函数。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **函数指针先是指针，里面存函数入口；p 等于函数名，p 加括号就调用。**
