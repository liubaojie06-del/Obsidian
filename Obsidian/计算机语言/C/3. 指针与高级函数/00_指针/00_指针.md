# 01 | C 语言指针：保存地址的变量

> [!abstract] 核心结论
> 指针是 C 语言中用来保存内存地址的变量。普通变量保存数据，指针变量保存地址；通过地址，程序可以间接访问和修改内存中的数据。

### ① 底层原理：指针到底是什么

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>指针的本质：</strong>
<ul>
  <li><strong>普通变量：</strong> 保存具体数据，例如 <code>int a = 10;</code>。</li>
  <li><strong>指针变量：</strong> 保存某块内存的地址，例如 <code>int *p = &amp;a;</code>。</li>
  <li><strong>&amp; 取地址：</strong> 获取变量在内存中的位置。</li>
  <li><strong>* 解引用：</strong> 通过地址访问地址里面的数据。</li>
  <li><strong>指针类型：</strong> 决定通过这个地址一次访问多少字节，以及如何解释这块内存。</li>
</ul>
</div>

### ② 普通变量和指针变量

```c
int a = 10;
int *p = &a;
```

可以理解为：

```text
a 是普通变量，里面保存 10。
p 是指针变量，里面保存 a 的地址。
&p 表示 p 自己的地址。
&a 表示 a 的地址。
*p 表示通过 p 保存的地址，访问 a 里面的值。
```

内存示意：

```text
栈区 Stack

变量 a
┌──────────────┐
│ 10           │
└──────────────┘
地址：0x1000

变量 p
┌──────────────┐
│ 0x1000       │
└──────────────┘
地址：0x2000
```

关系：

```text
p  = &a
*p = a
```

> [!tip] 一句话理解
> **指针不是神秘东西，它就是一个专门存地址的变量。**

# 02 | 指针的基本语法

> [!abstract] 核心结论
> 指针的核心语法只有两个符号：`&` 用来取地址，`*` 在声明时表示指针，在使用时表示解引用。

### ① 定义指针变量

```c
数据类型 *指针名;
```

示例：

```c
int *p;
double *q;
char *s;
```

说明：

```text
int *p 表示 p 是一个指针变量。
它保存的是 int 类型变量的地址。
```

### ② 取地址

```c
int a = 10;
int *p = &a;
```

说明：

```text
&a 表示变量 a 的地址。
p 保存了 a 的地址。
```

### ③ 解引用

```c
printf("%d\n", *p);
```

说明：

```text
*p 表示访问 p 指向的那块内存。
因为 p 指向 a，所以 *p 就是 a 的值。
```

### ④ 完整示例

```c
#include <stdio.h>

int main() {
    int a = 10;
    int *p = &a;

    printf("a = %d\n", a);
    printf("*p = %d\n", *p);
    printf("&a = %p\n", (void*)&a);
    printf("p = %p\n", (void*)p);

    return 0;
}
```

可能输出：

```text
a = 10
*p = 10
&a = 0x7ffeed123abc
p = 0x7ffeed123abc
```

说明：

```text
&a 和 p 的值一样。
因为 p 保存的就是 a 的地址。
```

# 03 | 通过指针修改变量

> [!abstract] 核心结论
> 指针可以通过地址间接修改变量的值。`*p = 20;` 的含义是：找到 p 指向的内存，把 20 写进去。

### ① 示例代码

```c
#include <stdio.h>

int main() {
    int a = 10;
    int *p = &a;

    *p = 20;

    printf("a = %d\n", a);
    printf("*p = %d\n", *p);

    return 0;
}
```

输出：

```text
a = 20
*p = 20
```

### ② 执行过程

```text
1. 栈区创建变量 a，值为 10。
2. 栈区创建指针变量 p。
3. p 保存 a 的地址。
4. 执行 *p = 20。
5. 通过 p 找到 a 的内存。
6. 把 20 写入 a 的内存。
7. 所以 a 的值变成 20。
```

内存变化：

```text
修改前：

a = 10
p = &a

执行 *p = 20 后：

a = 20
p = &a
```

### ③ 指针修改变量的本质

```text
普通赋值：
a = 20;
直接通过变量名修改。

指针赋值：
*p = 20;
通过地址间接修改。
```

> [!tip] 记忆口诀
> **`p` 存地址，`*p` 找内容；改 `*p`，就是改它指向的变量。**

# 04 | 指针类型的作用

> [!abstract] 核心结论
> 指针类型不是随便写的。`int *`、`double *`、`char *` 的区别在于：它们指向的数据类型不同，解引用时读取的字节数不同，指针加一跳过的字节数也不同。

### ① 不同类型指针

```c
int *p1;
double *p2;
char *p3;
```

含义：

```text
p1 指向 int 类型数据。
p2 指向 double 类型数据。
p3 指向 char 类型数据。
```

### ② 指针类型决定解引用方式

```c
int a = 10;
int *p = &a;
```

说明：

```text
p 是 int*。
所以 *p 会按照 int 类型解释 p 指向的内存。
常见情况下，一次读取 4 个字节。
```

如果是：

```c
double x = 3.14;
double *q = &x;
```

说明：

```text
q 是 double*。
所以 *q 会按照 double 类型解释 q 指向的内存。
常见情况下，一次读取 8 个字节。
```

### ③ 指针大小通常和类型无关

```c
#include <stdio.h>

int main() {
    int *p1;
    double *p2;
    char *p3;

    printf("%zu\n", sizeof(p1));
    printf("%zu\n", sizeof(p2));
    printf("%zu\n", sizeof(p3));

    return 0;
}
```

64 位环境常见输出：

```text
8
8
8
```

说明：

```text
指针变量保存的是地址。
同一个平台下，不同类型指针的大小通常相同。
64 位环境地址常见占 8 字节。
```

### ④ 指针类型影响指针加减

```c
int arr[3] = {10, 20, 30};
int *p = arr;

p++;
```

说明：

```text
p 是 int*。
p++ 不是地址加 1 个字节。
而是跳到下一个 int 元素。
如果 int 占 4 字节，p++ 地址实际增加 4。
```

> [!tip] 记忆口诀
> **指针大小看平台，指针步长看类型。**

# 05 | 空指针 NULL

> [!abstract] 核心结论
> 空指针表示“不指向任何有效内存”。当一个指针暂时没有明确指向时，应该初始化为 `NULL`，避免野指针。

### ① 基本写法

```c
int *p = NULL;
```

说明：

```text
p 是一个 int 指针。
但它目前不指向任何有效的 int 变量。
```

### ② 使用前判断

```c
#include <stdio.h>

int main() {
    int *p = NULL;

    if (p != NULL) {
        printf("%d\n", *p);
    } else {
        printf("p 是空指针，不能解引用\n");
    }

    return 0;
}
```

输出：

```text
p 是空指针，不能解引用
```

### ③ 错误写法

```c
int *p = NULL;

printf("%d\n", *p);
```

问题：

```text
NULL 不指向有效内存。
对 NULL 解引用会导致程序崩溃。
```

### ④ 为什么建议初始化为 NULL

```text
未初始化指针可能保存随机地址。
NULL 至少明确表示“不指向任何地方”。
使用前可以判断，安全性更高。
```

> [!tip] 记忆口诀
> **指针不知指向哪，先让它等于 NULL；用前先判断，别直接解引用。**

# 06 | 野指针与悬空指针

> [!abstract] 核心结论
> 野指针是指向未知内存的指针，悬空指针是指向已经失效内存的指针。它们都是 C 语言中非常危险的错误来源。

### ① 野指针

错误写法：

```c
int *p;

*p = 10;
```

问题：

```text
p 没有初始化。
它里面可能是随机地址。
*p = 10 会把 10 写到未知位置，非常危险。
```

正确写法：

```c
int a = 0;
int *p = &a;

*p = 10;
```

或者：

```c
int *p = NULL;
```

### ② 悬空指针

错误示例：

```c
#include <stdlib.h>

int main() {
    int *p = malloc(sizeof(int));

    *p = 10;

    free(p);

    *p = 20;

    return 0;
}
```

问题：

```text
free(p) 后，p 指向的堆空间已经释放。
但 p 里面还保存原来的地址。
此时 p 就是悬空指针。
继续使用 *p 非常危险。
```

正确写法：

```c
#include <stdlib.h>

int main() {
    int *p = malloc(sizeof(int));

    *p = 10;

    free(p);
    p = NULL;

    return 0;
}
```

### ③ 返回局部变量地址

错误写法：

```c
int* test() {
    int a = 10;
    return &a;
}
```

问题：

```text
a 是局部变量，存放在栈区。
函数结束后，a 的空间失效。
返回 &a 会得到悬空指针。
```

### ④ 安全建议

```text
1. 指针定义时尽量初始化。
2. 暂时不用就设为 NULL。
3. malloc 后检查是否申请成功。
4. free 后立刻 p = NULL。
5. 不要返回局部变量地址。
```

> [!tip] 记忆口诀
> **未初始化是野指针，释放后是悬空针；free 之后设 NULL，局部地址别返回。**

# 07 | 指针和数组

> [!abstract] 核心结论
> 数组名在大多数表达式中会退化为首元素地址，因此指针可以用来遍历数组。`arr[i]` 本质上和 `*(arr + i)` 等价。

### ① 数组名和首元素地址

```c
#include <stdio.h>

int main() {
    int arr[3] = {10, 20, 30};

    printf("%p\n", (void*)arr);
    printf("%p\n", (void*)&arr[0]);

    return 0;
}
```

说明：

```text
arr 和 &arr[0] 的地址值通常相同。
arr 通常表示数组首元素地址。
```

### ② 指针遍历数组

```c
#include <stdio.h>

int main() {
    int arr[3] = {10, 20, 30};
    int *p = arr;

    for (int i = 0; i < 3; i++) {
        printf("%d\n", *(p + i));
    }

    return 0;
}
```

输出：

```text
10
20
30
```

### ③ 下标访问和指针访问等价

```c
arr[i]
```

等价于：

```c
*(arr + i)
```

例如：

```c
arr[1]
```

等价于：

```c
*(arr + 1)
```

### ④ 指针自增遍历

```c
#include <stdio.h>

int main() {
    int arr[3] = {10, 20, 30};
    int *p = arr;

    for (int i = 0; i < 3; i++) {
        printf("%d\n", *p);
        p++;
    }

    return 0;
}
```

输出：

```text
10
20
30
```

说明：

```text
p++ 会让 p 指向下一个 int 元素。
```

### ⑤ 数组名不能被赋值

错误写法：

```c
int arr[3] = {10, 20, 30};

arr = arr + 1;
```

原因：

```text
数组名不是普通指针变量。
它不能被重新赋值。
```

> [!tip] 记忆口诀
> **数组名像地址，指向第一个；下标是语法糖，本质是指针加偏移。**

# 08 | 指针和函数

> [!abstract] 核心结论
> 函数参数默认是值传递。如果想让函数修改外部变量，就需要传变量地址，也就是使用指针参数。

### ① 值传递不能修改外部变量

```c
#include <stdio.h>

void change(int x) {
    x = 100;
}

int main() {
    int a = 10;

    change(a);

    printf("%d\n", a);

    return 0;
}
```

输出：

```text
10
```

原因：

```text
change(a) 只是把 a 的值复制给 x。
x 改成 100，不会影响 main 中的 a。
```

### ② 指针参数可以修改外部变量

```c
#include <stdio.h>

void change(int *p) {
    *p = 100;
}

int main() {
    int a = 10;

    change(&a);

    printf("%d\n", a);

    return 0;
}
```

输出：

```text
100
```

执行理解：

```text
&a 把 a 的地址传给 change。
p 保存 a 的地址。
*p = 100 通过地址修改 a。
```

### ③ 使用指针交换两个变量

```c
#include <stdio.h>

void swap(int *pa, int *pb) {
    int temp = *pa;
    *pa = *pb;
    *pb = temp;
}

int main() {
    int a = 10;
    int b = 20;

    swap(&a, &b);

    printf("a = %d\n", a);
    printf("b = %d\n", b);

    return 0;
}
```

输出：

```text
a = 20
b = 10
```

### ④ 数组作为函数参数

```c
#include <stdio.h>

void print_array(int *arr, int len) {
    for (int i = 0; i < len; i++) {
        printf("%d\n", arr[i]);
    }
}

int main() {
    int nums[] = {10, 20, 30};
    int len = sizeof(nums) / sizeof(nums[0]);

    print_array(nums, len);

    return 0;
}
```

说明：

```text
数组作为函数参数时会退化成指针。
所以 int arr[] 和 int *arr 在函数参数中基本等价。
数组长度需要单独传入。
```

> [!tip] 记忆口诀
> **想改外部变量，就把地址传进去；函数拿到地址，才能改原数据。**

# 09 | 指针和字符串

> [!abstract] 核心结论
> C 语言字符串本质是以 `'\0'` 结尾的字符序列。字符指针可以指向字符串常量，字符数组可以保存字符串副本。

### ① 字符指针指向字符串常量

```c
char *p = "Hello";
```

内存理解：

```text
"Hello" 存放在常量区 / 只读数据区。
p 是局部指针变量，通常放在栈区。
p 保存 "Hello" 的首地址。
```

图示：

```text
栈区 Stack
┌──────────────┐
│ p = 地址      │
└──────┬───────┘
       │
       ▼
常量区 Rodata
┌──────────────┐
│ Hello\0      │
└──────────────┘
```

### ② 字符数组保存字符串

```c
char arr[] = "Hello";
```

内存理解：

```text
arr 是字符数组，通常放在栈区。
程序会把 "Hello\0" 复制一份到 arr 中。
```

所以：

```c
arr[0] = 'h';
```

通常是允许的。

但是：

```c
char *p = "Hello";
p[0] = 'h';
```

是危险的，因为字符串常量通常在只读区域。

### ③ 遍历字符串

```c
#include <stdio.h>

int main() {
    char *p = "Hello";

    while (*p != '\0') {
        printf("%c\n", *p);
        p++;
    }

    return 0;
}
```

输出：

```text
H
e
l
l
o
```

说明：

```text
*p 访问当前字符。
p++ 指向下一个字符。
遇到 '\0' 停止。
```

### ④ 推荐写法

如果不打算修改字符串：

```c
const char *p = "Hello";
```

说明：

```text
const 表示不要通过 p 修改字符串内容。
更符合字符串常量只读的语义。
```

如果需要修改字符串：

```c
char arr[] = "Hello";
arr[0] = 'h';
```

> [!tip] 记忆口诀
> **指针指字符串，常量区里放；数组装字符串，栈里有副本可修改。**

# 10 | 指针常见错误与总结

> [!abstract] 核心结论
> 指针最容易出错的地方是：未初始化、空指针解引用、数组越界、free 后继续使用、返回局部变量地址。写指针代码时，必须时刻清楚它指向哪里。

### ① 常见错误

**错误 1：未初始化指针**

```c
int *p;
*p = 10;
```

问题：

```text
p 没有明确指向。
*p = 10 会写入未知地址。
```

正确：

```c
int a = 0;
int *p = &a;
*p = 10;
```

---

**错误 2：空指针解引用**

```c
int *p = NULL;
printf("%d\n", *p);
```

问题：

```text
NULL 不指向有效内存。
不能解引用。
```

正确：

```c
if (p != NULL) {
    printf("%d\n", *p);
}
```

---

**错误 3：数组越界**

```c
int arr[3] = {10, 20, 30};
int *p = arr;

printf("%d\n", *(p + 3));
```

问题：

```text
合法下标是 0、1、2。
p + 3 已经越界。
```

---

**错误 4：free 后继续使用**

```c
int *p = malloc(sizeof(int));
*p = 10;

free(p);

printf("%d\n", *p);
```

问题：

```text
p 指向的堆空间已经释放。
继续使用就是悬空指针问题。
```

正确：

```c
free(p);
p = NULL;
```

---

**错误 5：返回局部变量地址**

```c
int* get_num() {
    int num = 10;
    return &num;
}
```

问题：

```text
num 是局部变量。
函数结束后 num 被销毁。
返回它的地址没有意义。
```

### ② 指针总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>C 语言指针核心总结：</strong>
<ul>
  <li><strong>指针本质：</strong> 保存地址的变量。</li>
  <li><strong>&amp;：</strong> 取变量地址。</li>
  <li><strong>*：</strong> 声明时表示指针，使用时表示解引用。</li>
  <li><strong>指针类型：</strong> 决定解引用方式和指针步长。</li>
  <li><strong>NULL：</strong> 表示不指向任何有效内存。</li>
  <li><strong>数组名：</strong> 大多数情况下会退化为首元素地址。</li>
  <li><strong>函数传参：</strong> 想修改外部变量，就传地址。</li>
  <li><strong>安全原则：</strong> 指针必须先有明确指向，再解引用。</li>
</ul>
</div>

> [!tip] 记忆口诀
> **普通变量存数据，指针变量存地址；`&` 是取地址，`*` 是找内容；指针先指明，再解引用。**
