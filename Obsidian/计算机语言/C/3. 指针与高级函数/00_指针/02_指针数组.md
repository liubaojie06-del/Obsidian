# 01 | 指针数组：数组里面放的是指针

> [!abstract] 核心结论
> 指针数组本质上是数组，只不过数组中的每个元素都是指针。普通数组保存数据，指针数组保存地址。一维指针数组像“一排地址”，二维指针数组像“多排地址表”。

### ① 底层原理：先判断它是不是数组

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>指针数组的本质：</strong>
<ul>
  <li><strong>数组：</strong> 一组连续存放的元素。</li>
  <li><strong>指针：</strong> 保存地址的变量。</li>
  <li><strong>指针数组：</strong> 数组中的每个元素都是指针。</li>
  <li><strong>一维指针数组：</strong> 一排指针，例如 <code>int *p[3]</code>。</li>
  <li><strong>二维指针数组：</strong> 多排指针，例如 <code>int *p[2][3]</code>。</li>
</ul>
</div>

### ② 一维指针数组

```c
int *p[3];
```

判断：

```text
[] 优先级高。
p 先和 [3] 结合。

所以 p 是数组。
数组有 3 个元素。
每个元素类型是 int *。
```

结论：

```text
p 是一维指针数组。
```

### ③ 二维指针数组

```c
int *p[2][3];
```

判断：

```text
p 先和 [2][3] 结合。

所以 p 是二维数组。
这个二维数组有 2 行 3 列。
每个元素类型是 int *。
```

结论：

```text
p 是二维指针数组。
```

### ④ 一句话理解

```text
一维指针数组 = 一排地址。
二维指针数组 = 多排地址。
```

> [!tip] 记忆口诀
> **先看中括号，数组先成立；数组元素是指针，就是指针数组。**

# 02 | 一维指针数组：一排指针

> [!abstract] 核心结论
> 一维指针数组是最常见的指针数组形式。它本质是一个一维数组，数组里的每个元素都保存一个地址。

### ① 基本写法

```c
int *p[3];
```

含义：

```text
p[0] 是 int *
p[1] 是 int *
p[2] 是 int *
```

也就是说：

```text
p[0] 可以保存一个 int 变量地址。
p[1] 可以保存另一个 int 变量地址。
p[2] 也可以保存一个 int 变量地址。
```

### ② 示例代码

```c
#include <stdio.h>

int main() {
    int a = 10;
    int b = 20;
    int c = 30;

    int *p[3] = {&a, &b, &c};

    printf("%d\n", *p[0]);
    printf("%d\n", *p[1]);
    printf("%d\n", *p[2]);

    return 0;
}
```

输出：

```text
10
20
30
```

### ③ 内存示意

```text
p 一维指针数组：

┌────────────┐
│ p[0] = &a  │ ───► a = 10
├────────────┤
│ p[1] = &b  │ ───► b = 20
├────────────┤
│ p[2] = &c  │ ───► c = 30
└────────────┘
```

### ④ 访问规则

```c
p[0]
```

含义：

```text
取出第 0 个指针。
它里面保存的是 a 的地址。
```

```c
*p[0]
```

含义：

```text
通过 p[0] 保存的地址，访问 a 的值。
```

### ⑤ 修改指向的变量

```c
*p[0] = 100;
```

含义：

```text
p[0] 指向 a。
所以 *p[0] = 100;
就是把 a 改成 100。
```

> [!tip] 记忆口诀
> **p[i] 是地址，*p[i] 是地址里的内容。**

# 03 | 一维字符指针数组：保存多个字符串

> [!abstract] 核心结论
> 一维字符指针数组最常用于保存多个字符串。数组中的每个元素都是 `char *`，分别指向不同字符串的首字符。

### ① 基本写法

```c
const char *names[3] = {
    "Tom",
    "Jack",
    "Lucy"
};
```

含义：

```text
names 是一维数组。
names[0] 是 const char *，指向 "Tom"。
names[1] 是 const char *，指向 "Jack"。
names[2] 是 const char *，指向 "Lucy"。
```

### ② 示例代码

```c
#include <stdio.h>

int main() {
    const char *names[3] = {
        "Tom",
        "Jack",
        "Lucy"
    };

    printf("%s\n", names[0]);
    printf("%s\n", names[1]);
    printf("%s\n", names[2]);

    return 0;
}
```

输出：

```text
Tom
Jack
Lucy
```

### ③ 内存理解

```text
names[0] ───► "Tom\0"
names[1] ───► "Jack\0"
names[2] ───► "Lucy\0"
```

其中：

```text
names 数组本身通常在栈区。
字符串常量通常在只读数据区 / 常量区。
```

### ④ 为什么建议加 const

推荐：

```c
const char *names[3]
```

原因：

```text
"Tom"、"Jack"、"Lucy" 是字符串常量。
字符串常量通常不能修改。
使用 const 可以防止误改。
```

### ⑤ 错误示例

```c
char *names[3] = {"Tom", "Jack", "Lucy"};

names[0][0] = 't';
```

问题：

```text
试图修改字符串常量，属于未定义行为。
```

> [!tip] 记忆口诀
> **字符指针数组存字符串地址，不存字符串本体；字符串常量只读，最好加 const。**

# 04 | 一维指针数组保存多个数组首地址

> [!abstract] 核心结论
> 一维指针数组可以保存多个一维数组的首元素地址，用一个数组统一管理多组数据。

### ① 示例代码

```c
#include <stdio.h>

int main() {
    int a[] = {1, 2, 3};
    int b[] = {10, 20, 30};
    int c[] = {100, 200, 300};

    int *p[3] = {a, b, c};

    printf("%d\n", p[0][0]);
    printf("%d\n", p[1][1]);
    printf("%d\n", p[2][2]);

    return 0;
}
```

输出：

```text
1
20
300
```

### ② 为什么 `p[1][1]` 可以访问

```c
p[1]
```

含义：

```text
第 1 个指针。
它保存的是数组 b 的首元素地址。
```

所以：

```c
p[1][1]
```

等价于：

```c
*(p[1] + 1)
```

也就是：

```text
b[1]
```

### ③ 内存示意

```text
p[0] ───► a[0] a[1] a[2]
p[1] ───► b[0] b[1] b[2]
p[2] ───► c[0] c[1] c[2]
```

### ④ 注意：它不记录长度

```c
int *p[3] = {a, b, c};
```

这里只保存：

```text
每个数组的首地址。
```

但是不知道：

```text
每个数组有几个元素。
```

所以实际使用时常配合长度数组：

```c
int lens[3] = {3, 3, 3};
```

### ⑤ 适合场景

```text
多个一维数组统一管理。
多个缓冲区统一管理。
多组数据长度不同。
模拟不规则二维数组。
函数参数批量传递地址。
```

> [!tip] 记忆口诀
> **指针数组能管多组数组，但只管首地址，不管长度。**

# 05 | 二维指针数组：多排指针

> [!abstract] 核心结论
> 二维指针数组本质是一个二维数组，只是每个元素都是指针。例如 `int *p[2][3]` 表示 2 行 3 列，一共 6 个元素，每个元素都是 `int *`。

### ① 基本写法

```c
int *p[2][3];
```

判断：

```text
p 先和 [2][3] 结合。
所以 p 是二维数组。

每个元素类型是 int *。
所以 p 是二维指针数组。
```

含义：

```text
p[0][0] 是 int *
p[0][1] 是 int *
p[0][2] 是 int *

p[1][0] 是 int *
p[1][1] 是 int *
p[1][2] 是 int *
```

### ② 示例代码

```c
#include <stdio.h>

int main() {
    int a = 1;
    int b = 2;
    int c = 3;
    int d = 4;
    int e = 5;
    int f = 6;

    int *p[2][3] = {
        {&a, &b, &c},
        {&d, &e, &f}
    };

    printf("%d\n", *p[0][0]);
    printf("%d\n", *p[0][2]);
    printf("%d\n", *p[1][1]);
    printf("%d\n", *p[1][2]);

    return 0;
}
```

输出：

```text
1
3
5
6
```

### ③ 内存示意

```text
二维指针数组 p：

第 0 行：
p[0][0] ───► a = 1
p[0][1] ───► b = 2
p[0][2] ───► c = 3

第 1 行：
p[1][0] ───► d = 4
p[1][1] ───► e = 5
p[1][2] ───► f = 6
```

### ④ 访问规则

```c
p[1][2]
```

含义：

```text
取第 1 行第 2 列的指针。
```

```c
*p[1][2]
```

含义：

```text
访问这个指针指向的 int 数据。
```

### ⑤ 一句话理解

```text
二维指针数组 = 一个二维表格，表格每一格存一个地址。
```

> [!tip] 记忆口诀
> **二维指针数组像地址表，p[i][j] 是地址，*p[i][j] 是内容。**

# 06 | 二维字符指针数组：多组字符串表

> [!abstract] 核心结论
> 二维字符指针数组可以看作“多组字符串列表”。例如 `const char *menu[2][3]` 表示 2 行 3 列，每个元素都是一个字符串地址。

### ① 示例代码

```c
#include <stdio.h>

int main() {
    const char *menu[2][3] = {
        {"File", "Edit", "View"},
        {"Open", "Save", "Exit"}
    };

    printf("%s\n", menu[0][0]);
    printf("%s\n", menu[0][2]);
    printf("%s\n", menu[1][1]);
    printf("%s\n", menu[1][2]);

    return 0;
}
```

输出：

```text
File
View
Save
Exit
```

### ② 含义

```text
menu 是二维数组。
menu 有 2 行 3 列。
每个元素是 const char *。
每个元素指向一个字符串常量。
```

也就是：

```text
menu[0][0] 指向 "File"
menu[0][1] 指向 "Edit"
menu[0][2] 指向 "View"

menu[1][0] 指向 "Open"
menu[1][1] 指向 "Save"
menu[1][2] 指向 "Exit"
```

### ③ 内存理解

```text
menu[0][0] ───► "File"
menu[0][1] ───► "Edit"
menu[0][2] ───► "View"

menu[1][0] ───► "Open"
menu[1][1] ───► "Save"
menu[1][2] ───► "Exit"
```

### ④ 它和二维字符数组不同

二维字符指针数组：

```c
const char *menu[2][3];
```

每格保存：

```text
字符串地址。
```

二维字符数组：

```c
char menu[2][3][10];
```

每格保存：

```text
真实字符内容。
```

### ⑤ 适合场景

```text
多组菜单。
多语言文本表。
命令列表。
状态字符串表。
错误信息表。
字符串分类表。
```

> [!tip] 记忆口诀
> **二维字符指针数组不是存字符矩阵，而是存一张字符串地址表。**

# 07 | 一维指针数组与二维指针数组对比

> [!abstract] 核心结论
> 一维指针数组是一排指针，二维指针数组是多排指针。它们的元素本质都是指针，只是数组维度不同。

### ① 一维指针数组

```c
int *p[3];
```

结构：

```text
p[0]  p[1]  p[2]
```

每个元素：

```text
都是 int *
```

访问内容：

```c
*p[0]
*p[1]
*p[2]
```

### ② 二维指针数组

```c
int *p[2][3];
```

结构：

```text
p[0][0]  p[0][1]  p[0][2]
p[1][0]  p[1][1]  p[1][2]
```

每个元素：

```text
都是 int *
```

访问内容：

```c
*p[0][0]
*p[1][2]
```

### ③ 对比表

| 写法 | 名称 | 结构 | 元素类型 | 访问指针 | 访问内容 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `int *p[3]` | 一维指针数组 | 1 行 3 个 | `int *` | `p[i]` | `*p[i]` |
| `int *p[2][3]` | 二维指针数组 | 2 行 3 列 | `int *` | `p[i][j]` | `*p[i][j]` |
| `char *s[3]` | 一维字符指针数组 | 1 行 3 个 | `char *` | `s[i]` | `s[i][k]` |
| `char *s[2][3]` | 二维字符指针数组 | 2 行 3 列 | `char *` | `s[i][j]` | `s[i][j][k]` |

### ④ 字符串访问层级

```c
const char *s[2][3];
```

访问：

```c
s[1][2]
```

含义：

```text
第 1 行第 2 列的字符串地址。
```

```c
s[1][2][0]
```

含义：

```text
这个字符串的第 0 个字符。
```

### ⑤ 重点区别

```text
一维：
p[i] 是指针。

二维：
p[i][j] 是指针。
```

> [!tip] 记忆口诀
> **一维用一个下标取指针，二维用两个下标取指针；再加 * 才取内容。**

# 08 | 指针数组和数组指针再次区分

> [!abstract] 核心结论
> 指针数组是数组，数组元素是指针；数组指针是指针，指向数组。判断时先看变量名先和 `[]` 结合，还是先和 `*` 结合。

### ① 一维指针数组

```c
int *p[3];
```

判断：

```text
p 先和 [3] 结合。
所以 p 是数组。
数组元素是 int *。
```

结论：

```text
p 是指针数组。
```

### ② 一维数组指针

```c
int (*p)[3];
```

判断：

```text
p 先和 * 结合。
所以 p 是指针。
这个指针指向 int[3]。
```

结论：

```text
p 是数组指针。
```

### ③ 二维指针数组

```c
int *p[2][3];
```

判断：

```text
p 先和 [2][3] 结合。
所以 p 是二维数组。
每个元素是 int *。
```

结论：

```text
p 是二维指针数组。
```

### ④ 指向二维数组一行的数组指针

```c
int (*p)[3];
```

也常用于：

```c
int arr[2][3];
int (*p)[3] = arr;
```

这里：

```text
p 指向 arr 的一行。
每一行是 int[3]。
```

### ⑤ 对比表

| 写法 | 名称 | 本质 | 含义 |
| :--- | :--- | :--- | :--- |
| `int *p[3]` | 一维指针数组 | 数组 | 3 个元素，每个是 `int *` |
| `int *p[2][3]` | 二维指针数组 | 二维数组 | 2×3 个元素，每个是 `int *` |
| `int (*p)[3]` | 数组指针 | 指针 | 指向一个 `int[3]` 数组 |
| `int (*p)[2][3]` | 二维数组指针 | 指针 | 指向一个 `int[2][3]` 二维数组 |

> [!tip] 记忆口诀
> **指针数组先是数组，数组指针先是指针；括号一加，身份就变。**

# 09 | 指针数组传参：一维和二维

> [!abstract] 核心结论
> 指针数组作为函数参数时，数组名会退化为指针。一维指针数组通常退化为二级指针，二维指针数组通常退化为指向一行指针数组的指针。

### ① 一维指针数组传参

```c
#include <stdio.h>

void print_nums(int *p[], int len) {
    for (int i = 0; i < len; i++) {
        printf("%d\n", *p[i]);
    }
}

int main() {
    int a = 10;
    int b = 20;
    int c = 30;

    int *arr[3] = {&a, &b, &c};

    print_nums(arr, 3);

    return 0;
}
```

这里：

```c
void print_nums(int *p[], int len)
```

等价于：

```c
void print_nums(int **p, int len)
```

因为：

```text
一维指针数组传参时，退化为指向首元素的指针。
首元素类型是 int *。
所以参数类型是 int **。
```

### ② 二维指针数组传参

```c
#include <stdio.h>

void print_nums(int *p[][3], int rows) {
    for (int i = 0; i < rows; i++) {
        for (int j = 0; j < 3; j++) {
            printf("%d ", *p[i][j]);
        }
        printf("\n");
    }
}

int main() {
    int a = 1, b = 2, c = 3;
    int d = 4, e = 5, f = 6;

    int *arr[2][3] = {
        {&a, &b, &c},
        {&d, &e, &f}
    };

    print_nums(arr, 2);

    return 0;
}
```

输出：

```text
1 2 3
4 5 6
```

### ③ 二维指针数组参数等价写法

```c
void print_nums(int *p[][3], int rows)
```

等价于：

```c
void print_nums(int *(*p)[3], int rows)
```

解释：

```text
p 是指针。
p 指向一个数组。
这个数组有 3 个元素。
每个元素是 int *。
```

### ④ 为什么二维指针数组参数要写列数

对于：

```c
int *p[2][3]
```

传参时第一维可以省略：

```c
int *p[][3]
```

但第二维必须写：

```text
因为函数要知道每一行有几个指针。
这样才能正确计算 p[i][j] 的位置。
```

### ⑤ 函数里不要用 sizeof 求元素个数

错误：

```c
void test(int *p[]) {
    int len = sizeof(p) / sizeof(p[0]);
}
```

问题：

```text
p 已经退化成指针。
sizeof(p) 是指针大小，不是数组总大小。
```

正确：

```text
长度要单独传。
```

> [!tip] 记忆口诀
> **一维指针数组传参变二级指针；二维指针数组传参必须保留列数。**

# 10 | 指针数组一维二维总结

> [!abstract] 核心结论
> 指针数组的底层核心是“数组元素保存地址”。一维指针数组是一排地址，二维指针数组是多排地址。访问时先用下标拿到指针，再用 `*` 或字符串下标访问指针指向的内容。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>指针数组一维二维核心总结：</strong>
<ul>
  <li><strong>本质：</strong> 数组，数组元素是指针。</li>
  <li><strong>一维写法：</strong> <code>int *p[3]</code>，一排 3 个 <code>int *</code>。</li>
  <li><strong>二维写法：</strong> <code>int *p[2][3]</code>，2 行 3 列，每格是 <code>int *</code>。</li>
  <li><strong>一维访问指针：</strong> <code>p[i]</code>。</li>
  <li><strong>一维访问内容：</strong> <code>*p[i]</code>。</li>
  <li><strong>二维访问指针：</strong> <code>p[i][j]</code>。</li>
  <li><strong>二维访问内容：</strong> <code>*p[i][j]</code>。</li>
  <li><strong>字符串场景：</strong> <code>const char *s[3]</code> 或 <code>const char *s[2][3]</code>。</li>
  <li><strong>传参：</strong> 一维退化成二级指针，二维要保留列数。</li>
  <li><strong>易混点：</strong> <code>int *p[3]</code> 是指针数组，<code>int (*p)[3]</code> 是数组指针。</li>
</ul>
</div>

### ② 最底层一句话

```text
指针数组 = 用数组批量保存地址；一维是一排地址，二维是一张地址表。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **一维指针数组一排地址，二维指针数组多排地址；p[i] 或 p[i][j] 先拿地址，加星号才取内容。**
