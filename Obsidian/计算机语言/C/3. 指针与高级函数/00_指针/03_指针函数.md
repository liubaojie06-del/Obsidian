# 01 | 指针函数：返回值是指针的函数

> [!abstract] 核心结论
> 指针函数本质上还是函数，只是这个函数的返回值类型是指针。它常用于返回数组首地址、字符串地址、动态内存地址、结构体地址等。

### ① 底层原理：先看它是不是函数

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>指针函数的本质：</strong>
<ul>
  <li><strong>函数：</strong> 一段可以被调用的代码。</li>
  <li><strong>指针：</strong> 保存地址的数据。</li>
  <li><strong>指针函数：</strong> 返回值是指针的函数。</li>
  <li><strong>核心写法：</strong> <code>int *func()</code> 表示 <code>func</code> 是函数，返回值是 <code>int *</code>。</li>
  <li><strong>重点：</strong> 指针函数返回的是地址，不是普通数据值。</li>
</ul>
</div>

### ② 基本写法

```c
int *func();
```

读法：

```text
func 先和 () 结合。
所以 func 是函数。

函数前面是 int *。
所以 func 的返回值是 int *。

结论：
func 是指针函数。
```

### ③ 一句话理解

```text
指针函数 = 一个函数，调用后返回一个地址。
```

例如：

```c
int *get_ptr();
```

含义：

```text
get_ptr 是函数。
它返回 int 类型数据的地址。
```

> [!tip] 记忆口诀
> **先看小括号，是函数；返回值带星号，就是指针函数。**

# 02 | 指针函数的基本示例

> [!abstract] 核心结论
> 指针函数返回一个地址，调用者可以用指针变量接收这个地址，再通过 `*` 解引用访问地址中的内容。

### ① 示例代码

```c
#include <stdio.h>

int global_num = 100;

int *get_num_addr() {
    return &global_num;
}

int main() {
    int *p = get_num_addr();

    printf("%d\n", *p);

    return 0;
}
```

输出：

```text
100
```

### ② 调用过程

```c
int *p = get_num_addr();
```

含义：

```text
调用 get_num_addr 函数。
函数返回 global_num 的地址。
p 接收这个地址。
```

然后：

```c
*p
```

含义：

```text
通过地址访问 global_num 的值。
```

### ③ 内存理解

```text
全局区：
global_num = 100

main 栈帧：
p = &global_num
```

示意：

```text
p ───► global_num = 100
```

### ④ 为什么可以返回全局变量地址

因为：

```text
global_num 是全局变量。
它的生命周期从程序开始到程序结束。
函数返回后它仍然存在。
```

所以：

```text
返回 &global_num 是安全的。
```

### ⑤ 重点

```text
指针函数返回地址。
这个地址必须在函数返回后仍然有效。
```

> [!tip] 记忆口诀
> **函数返回地址，接收用指针；地址有效，才能解引用。**

# 03 | 不能返回局部变量地址

> [!abstract] 核心结论
> 指针函数最常见错误是返回局部变量地址。普通局部变量存放在函数栈帧中，函数返回后栈帧释放，这个地址就失效了。

### ① 错误示例

```c
#include <stdio.h>

int *get_ptr() {
    int num = 100;
    return &num;   // 错误
}

int main() {
    int *p = get_ptr();

    printf("%d\n", *p);

    return 0;
}
```

问题：

```text
num 是 get_ptr 函数内部的局部变量。
get_ptr 返回后，num 所在的栈帧被释放。
p 保存的是已经失效的地址。
```

### ② 栈帧理解

```text
调用 get_ptr 时：

get_ptr 栈帧：
num = 100
返回 &num

get_ptr 返回后：

get_ptr 栈帧释放。
num 不再有效。
```

此时：

```text
p 指向一块已经失效的栈空间。
```

### ③ 后果

可能出现：

```text
输出随机值。
程序崩溃。
看似正常但隐藏 bug。
被后续函数调用覆盖。
```

### ④ 正确做法 1：返回 static 局部变量地址

```c
#include <stdio.h>

int *get_ptr() {
    static int num = 100;
    return &num;
}

int main() {
    int *p = get_ptr();

    printf("%d\n", *p);

    return 0;
}
```

### ⑤ 正确做法 2：返回堆区地址

```c
#include <stdio.h>
#include <stdlib.h>

int *get_ptr() {
    int *p = malloc(sizeof(int));

    if (p == NULL) {
        return NULL;
    }

    *p = 100;
    return p;
}

int main() {
    int *p = get_ptr();

    if (p != NULL) {
        printf("%d\n", *p);
        free(p);
    }

    return 0;
}
```

> [!tip] 记忆口诀
> **局部变量在栈里，函数返回就销毁；别把局部地址带出去。**

# 04 | 返回 static 局部变量地址

> [!abstract] 核心结论
> `static` 局部变量不会随着函数返回而销毁，所以指针函数可以返回 `static` 局部变量的地址。但这种变量会保留状态，多次调用共享同一份数据。

### ① 示例代码

```c
#include <stdio.h>

int *get_count() {
    static int count = 0;

    count++;

    return &count;
}

int main() {
    int *p1 = get_count();
    printf("%d\n", *p1);

    int *p2 = get_count();
    printf("%d\n", *p2);

    int *p3 = get_count();
    printf("%d\n", *p3);

    return 0;
}
```

输出：

```text
1
2
3
```

### ② 为什么安全

```text
static int count
```

存放在：

```text
静态存储区。
```

生命周期：

```text
从程序开始到程序结束。
```

所以：

```text
函数返回后 count 仍然存在。
```

### ③ 注意共享同一份数据

```c
int *p1 = get_count();
int *p2 = get_count();
```

这里：

```text
p1 和 p2 指向的是同一个 static count。
```

不是两份不同数据。

### ④ 内存理解

```text
静态区：
count = 3

main 栈帧：
p1 ─┐
p2 ─┼──► count
p3 ─┘
```

### ⑤ 适合场景

```text
函数内部保存状态。
返回固定缓冲区。
简单计数器。
小型缓存。
```

但要注意：

```text
多线程可能有竞争。
多次调用会覆盖或改变同一份数据。
```

> [!tip] 记忆口诀
> **static 局部变量活得久，可以返回地址；但多次调用共用一份。**

# 05 | 返回堆区地址

> [!abstract] 核心结论
> 指针函数可以通过 `malloc` 在堆区申请内存，然后返回这块内存地址。堆区内存不会随函数返回自动释放，但调用者用完后必须 `free`。

### ① 示例代码

```c
#include <stdio.h>
#include <stdlib.h>

int *create_num(int value) {
    int *p = malloc(sizeof(int));

    if (p == NULL) {
        return NULL;
    }

    *p = value;

    return p;
}

int main() {
    int *p = create_num(100);

    if (p == NULL) {
        printf("内存申请失败\n");
        return 1;
    }

    printf("%d\n", *p);

    free(p);

    return 0;
}
```

输出：

```text
100
```

### ② 为什么可以返回

```text
malloc 申请的是堆区内存。
堆区内存不会因为函数返回就释放。
```

所以：

```text
create_num 返回后，p 指向的堆空间仍然有效。
```

### ③ 谁负责释放

```text
谁拿到 malloc 返回的地址，谁就要负责最终 free。
```

在上面示例中：

```text
main 拿到地址。
main 使用完后 free(p)。
```

### ④ 忘记 free 的后果

如果不释放：

```text
内存泄漏。
```

也就是：

```text
堆区空间一直被占用，程序长期运行会越来越耗内存。
```

### ⑤ 重点

```text
返回堆区地址是安全的。
但必须明确释放责任。
```

> [!tip] 记忆口诀
> **堆区地址能返回，用完必须 free；只 malloc 不 free，就是内存泄漏。**

# 06 | 返回字符串地址

> [!abstract] 核心结论
> 指针函数常用于返回字符串地址。可以返回字符串常量地址、静态字符数组地址、堆区字符串地址，但不能返回普通局部字符数组地址。

### ① 返回字符串常量

```c
#include <stdio.h>

const char *get_msg() {
    return "hello";
}

int main() {
    const char *p = get_msg();

    printf("%s\n", p);

    return 0;
}
```

输出：

```text
hello
```

说明：

```text
"hello" 是字符串常量。
通常存放在只读数据区。
函数返回后仍然存在。
```

推荐返回类型：

```c
const char *
```

因为：

```text
字符串常量不应该被修改。
```

### ② 返回 static 字符数组

```c
#include <stdio.h>
#include <string.h>

char *get_buffer() {
    static char buf[100];

    strcpy(buf, "hello");

    return buf;
}

int main() {
    char *p = get_buffer();

    printf("%s\n", p);

    return 0;
}
```

### ③ 返回堆区字符串

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

char *create_msg() {
    char *p = malloc(20);

    if (p == NULL) {
        return NULL;
    }

    strcpy(p, "hello");

    return p;
}

int main() {
    char *p = create_msg();

    if (p != NULL) {
        printf("%s\n", p);
        free(p);
    }

    return 0;
}
```

### ④ 错误：返回局部字符数组

```c
char *get_msg() {
    char buf[20] = "hello";
    return buf;  // 错误
}
```

原因：

```text
buf 是局部数组。
函数返回后 buf 所在栈帧释放。
返回地址失效。
```

### ⑤ 对比总结

| 返回内容 | 是否安全 | 原因 |
| :--- | :--- | :--- |
| 字符串常量地址 | 安全 | 通常在只读区 |
| static 字符数组地址 | 安全 | 静态存储期 |
| malloc 字符串地址 | 安全 | 堆区，需 free |
| 普通局部数组地址 | 不安全 | 函数返回后销毁 |

> [!tip] 记忆口诀
> **返回字符串要看存哪：常量区、静态区、堆区能返回；栈区不能返回。**

# 07 | 返回数组首地址

> [!abstract] 核心结论
> C 语言函数不能直接返回整个数组，但可以返回数组首元素地址。返回数组地址时，必须保证数组在函数返回后仍然有效。

### ① 错误：返回局部数组

```c
int *get_arr() {
    int arr[3] = {1, 2, 3};
    return arr;  // 错误
}
```

原因：

```text
arr 是局部数组。
函数返回后 arr 被销毁。
返回的首地址失效。
```

### ② 正确：返回 static 数组

```c
#include <stdio.h>

int *get_arr() {
    static int arr[3] = {1, 2, 3};
    return arr;
}

int main() {
    int *p = get_arr();

    printf("%d\n", p[0]);
    printf("%d\n", p[1]);
    printf("%d\n", p[2]);

    return 0;
}
```

输出：

```text
1
2
3
```

### ③ 正确：返回堆区数组

```c
#include <stdio.h>
#include <stdlib.h>

int *create_arr(int len) {
    int *p = malloc(sizeof(int) * len);

    if (p == NULL) {
        return NULL;
    }

    for (int i = 0; i < len; i++) {
        p[i] = i + 1;
    }

    return p;
}

int main() {
    int len = 3;
    int *p = create_arr(len);

    if (p != NULL) {
        for (int i = 0; i < len; i++) {
            printf("%d ", p[i]);
        }

        free(p);
    }

    return 0;
}
```

输出：

```text
1 2 3
```

### ④ 注意长度问题

函数返回数组首地址：

```text
只返回了地址。
没有返回数组长度。
```

所以通常需要：

```text
额外返回长度。
或由调用者传入长度。
或使用结构体一起返回地址和长度。
```

### ⑤ 重点

```text
数组名作为返回值时，本质上返回的是首元素地址。
```

> [!tip] 记忆口诀
> **函数不能直接返回数组，只能返回数组地址；地址有效还不够，长度也要记。**

# 08 | 指针函数和函数指针的区别

> [!abstract] 核心结论
> 指针函数是“返回指针的函数”；函数指针是“指向函数的指针”。二者名字很像，但本质完全不同。

### ① 指针函数

```c
int *func();
```

判断：

```text
func 先和 () 结合。
所以 func 是函数。
函数返回 int *。
```

结论：

```text
func 是指针函数。
```

含义：

```text
调用 func 后得到一个 int 地址。
```

### ② 函数指针

```c
int (*func)();
```

判断：

```text
括号让 func 先和 * 结合。
所以 func 是指针。
这个指针指向一个返回 int 的函数。
```

结论：

```text
func 是函数指针。
```

含义：

```text
func 保存函数入口地址。
可以通过 func 调用函数。
```

### ③ 对比表

| 写法 | 名称 | 本质 | 含义 |
| :--- | :--- | :--- | :--- |
| `int *func()` | 指针函数 | 函数 | 返回 `int *` 的函数 |
| `int (*func)()` | 函数指针 | 指针 | 指向返回 `int` 的函数 |
| `char *get_str()` | 指针函数 | 函数 | 返回 `char *` |
| `char (*p)()` | 函数指针 | 指针 | 指向返回 `char` 的函数 |

### ④ 快速判断

```text
变量名 / 函数名先和 () 结合：
它是函数。

名字先和 * 结合：
它是指针。
```

### ⑤ 记忆

```text
int *func()
```

重点在：

```text
func 是函数。
返回值是 int *。
```

```text
int (*func)()
```

重点在：

```text
func 是指针。
指向函数。
```

> [!tip] 记忆口诀
> **指针函数先是函数，函数指针先是指针；括号决定身份。**

# 09 | 指针函数常见错误

> [!abstract] 核心结论
> 指针函数常见错误包括返回局部变量地址、返回局部数组地址、忘记释放堆区内存、返回字符串常量却用普通 `char *` 接收并修改、混淆函数指针。

### ① 错误 1：返回局部变量地址

```c
int *func() {
    int x = 10;
    return &x;  // 错误
}
```

原因：

```text
x 在栈区，函数返回后失效。
```

### ② 错误 2：返回局部数组地址

```c
char *func() {
    char str[20] = "hello";
    return str;  // 错误
}
```

原因：

```text
str 是局部数组，函数返回后销毁。
```

### ③ 错误 3：忘记 free

```c
int *func() {
    int *p = malloc(sizeof(int));
    *p = 100;
    return p;
}
```

如果调用者不：

```c
free(p);
```

就会：

```text
内存泄漏。
```

### ④ 错误 4：修改字符串常量

```c
char *get_msg() {
    return "hello";
}

int main() {
    char *p = get_msg();
    p[0] = 'H';  // 错误
}
```

更安全写法：

```c
const char *get_msg() {
    return "hello";
}
```

### ⑤ 错误 5：和函数指针混淆

```c
int *func();      // 指针函数
int (*func)();    // 函数指针
```

区别：

```text
前者是函数。
后者是指针。
```

> [!tip] 记忆口诀
> **指针函数最怕返回栈地址；堆区能返回，但用完要释放。**

# 10 | 指针函数总结

> [!abstract] 核心结论
> 指针函数的底层核心是“函数返回一个地址”。这个地址可以来自全局区、静态区、堆区、字符串常量区，但不能来自已经销毁的局部栈区。理解返回地址的生命周期，是掌握指针函数的关键。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>指针函数核心总结：</strong>
<ul>
  <li><strong>本质：</strong> 函数。</li>
  <li><strong>特点：</strong> 返回值是指针。</li>
  <li><strong>基本写法：</strong> <code>int *func()</code>、<code>char *get_str()</code>。</li>
  <li><strong>返回内容：</strong> 返回某块内存的地址。</li>
  <li><strong>安全返回：</strong> 可以返回全局变量地址、static 变量地址、堆区地址、字符串常量地址。</li>
  <li><strong>危险返回：</strong> 不能返回普通局部变量或局部数组地址。</li>
  <li><strong>返回堆区：</strong> 调用者用完后必须 <code>free</code>。</li>
  <li><strong>返回字符串常量：</strong> 推荐使用 <code>const char *</code>。</li>
  <li><strong>和函数指针区别：</strong> <code>int *func()</code> 是指针函数，<code>int (*func)()</code> 是函数指针。</li>
  <li><strong>核心判断：</strong> 先和 <code>()</code> 结合是函数，先和 <code>*</code> 结合是指针。</li>
</ul>
</div>

### ② 最底层一句话

```text
指针函数 = 一个返回地址的函数；关键是返回的地址在函数结束后必须仍然有效。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **指针函数先是函数，返回结果是地址；栈区地址不能回，静态堆区才安全。**
