# 01 | 结构体：把多种数据组合成一个整体

> [!abstract] 核心结论
> 结构体是 C 语言中的自定义复合数据类型，用来把多个不同类型的数据组合成一个整体。它适合表示学生、员工、坐标、日期、设备状态等具有多个属性的对象。

### ① 底层原理

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>结构体的本质：</strong>
<ul>
  <li><strong>组合数据：</strong>把多个变量打包成一个整体。</li>
  <li><strong>成员变量：</strong>结构体内部的每个字段叫成员。</li>
  <li><strong>连续内存：</strong>结构体变量通常占用一整块连续内存。</li>
  <li><strong>成员类型可不同：</strong>可以同时包含 <code>int</code>、<code>char</code>、数组、指针等。</li>
  <li><strong>内存对齐：</strong>结构体大小不一定等于所有成员大小简单相加，可能会有填充字节。</li>
</ul>
</div>

### ② 代码用法

```c
struct Student {
    int id;
    char name[32];
    int age;
};
```

定义变量：

```c
struct Student stu;
```

访问成员：

```c
stu.id = 1;
stu.age = 18;
```

### ③ 使用场景

| 场景 | 结构体设计 |
| :--- | :--- |
| 学生信息 | 学号、姓名、年龄、成绩 |
| 坐标点 | x、y |
| 日期 | 年、月、日 |
| 设备状态 | 编号、状态、温度 |
| 链表节点 | 数据、下一个节点地址 |

> [!tip] 记忆口诀
> **结构体就是数据打包，把多个属性组成一个整体。**

# 02 | 结构体的定义和变量创建

> [!abstract] 核心结论
> 定义结构体类型只是创建一个“模板”，不会立即分配实际数据空间。只有定义结构体变量时，才会真正分配内存。

### ① 定义结构体类型

```c
struct Student {
    int id;
    char name[32];
    int age;
};
```

含义：

```text
struct Student 是一种新类型。
它描述了一个学生应该有哪些字段。
```

注意：

```text
这里只是定义类型模板。
不会创建真正的学生变量。
```

### ② 定义结构体变量

```c
struct Student stu;
```

含义：

```text
创建一个 struct Student 类型变量 stu。
这时才会分配内存。
```

### ③ 定义时直接创建变量

```c
struct Student {
    int id;
    char name[32];
    int age;
} stu1, stu2;
```

含义：

```text
定义 struct Student 类型的同时，创建 stu1 和 stu2 两个变量。
```

### ④ 匿名结构体

```c
struct {
    int x;
    int y;
} point;
```

说明：

```text
这个结构体没有类型名。
只能直接使用 point 这个变量。
后面不方便再创建同类型变量。
```

### ⑤ 推荐写法

初学阶段推荐：

```c
struct Student {
    int id;
    char name[32];
    int age;
};

struct Student stu;
```

这样结构清楚，便于理解。

> [!tip] 记忆口诀
> **定义结构体是画模板，定义变量才造实体。**

# 03 | 结构体初始化

> [!abstract] 核心结论
> 结构体变量可以在定义时初始化。常见方式有顺序初始化、指定成员初始化、整体赋值初始化。

### ① 顺序初始化

```c
struct Student stu = {1, "Tom", 18};
```

对应关系：

```text
1      → id
"Tom"  → name
18     → age
```

完整代码：

```c
#include <stdio.h>

struct Student {
    int id;
    char name[32];
    int age;
};

int main() {
    struct Student stu = {1, "Tom", 18};

    printf("%d %s %d\n", stu.id, stu.name, stu.age);

    return 0;
}
```

输出：

```text
1 Tom 18
```

### ② 指定成员初始化

```c
struct Student stu = {
    .id = 1,
    .name = "Tom",
    .age = 18
};
```

好处：

```text
不依赖成员顺序。
可读性更好。
适合成员较多的结构体。
```

### ③ 部分初始化

```c
struct Student stu = {1};
```

含义：

```text
id 初始化为 1。
其他成员默认初始化为 0。
字符数组会变成空字符串。
```

### ④ 定义后赋值

```c
struct Student stu;

stu.id = 1;
stu.age = 18;
```

注意：

```text
字符数组不能直接用 = 赋值。
```

错误：

```c
stu.name = "Tom";
```

正确：

```c
strcpy(stu.name, "Tom");
```

### ⑤ 使用 strcpy 给字符数组成员赋值

```c
#include <string.h>

strcpy(stu.name, "Tom");
```

> [!tip] 记忆口诀
> **初始化可用大括号，定义后改字符串要用 strcpy。**

# 04 | 访问结构体成员：`.` 和 `->`

> [!abstract] 核心结论
> 普通结构体变量访问成员用 `.`；结构体指针访问成员用 `->`。`p->age` 等价于 `(*p).age`。

### ① 普通变量用点号

```c
struct Student stu;

stu.id = 1;
stu.age = 18;
```

含义：

```text
stu 是结构体变量。
用 stu.id 访问 id 成员。
```

### ② 结构体指针用箭头

```c
struct Student *p = &stu;

p->id = 1;
p->age = 18;
```

含义：

```text
p 保存 stu 的地址。
用 p->id 访问 p 指向的结构体里的 id。
```

### ③ 箭头等价写法

```c
p->age
```

等价于：

```c
(*p).age
```

注意：

```c
*p.age
```

是错误理解，因为 `.` 的优先级比 `*` 高。

### ④ 完整示例

```c
#include <stdio.h>

struct Student {
    int id;
    char name[32];
    int age;
};

int main() {
    struct Student stu = {1, "Tom", 18};
    struct Student *p = &stu;

    printf("%d\n", stu.age);
    printf("%d\n", p->age);
    printf("%d\n", (*p).age);

    return 0;
}
```

输出：

```text
18
18
18
```

### ⑤ 使用规则

| 对象 | 访问方式 | 示例 |
| :--- | :--- | :--- |
| 结构体变量 | `.` | `stu.age` |
| 结构体指针 | `->` | `p->age` |
| 指针解引用后 | `.` | `(*p).age` |

> [!tip] 记忆口诀
> **变量用点，指针用箭头；箭头就是先解引用再点。**

# 05 | 结构体数组

> [!abstract] 核心结论
> 结构体数组是由多个结构体变量组成的数组，适合保存一组相同类型的数据，例如多个学生、多个商品、多个设备信息。

### ① 基本写法

```c
struct Student students[3];
```

含义：

```text
students 是数组。
数组中有 3 个元素。
每个元素都是 struct Student。
```

### ② 初始化

```c
struct Student students[3] = {
    {1, "Tom", 18},
    {2, "Jack", 19},
    {3, "Lucy", 17}
};
```

### ③ 访问元素成员

```c
students[0].id
students[1].name
students[2].age
```

### ④ 完整示例

```c
#include <stdio.h>

struct Student {
    int id;
    char name[32];
    int age;
};

int main() {
    struct Student students[3] = {
        {1, "Tom", 18},
        {2, "Jack", 19},
        {3, "Lucy", 17}
    };

    for (int i = 0; i < 3; i++) {
        printf("%d %s %d\n", students[i].id, students[i].name, students[i].age);
    }

    return 0;
}
```

输出：

```text
1 Tom 18
2 Jack 19
3 Lucy 17
```

### ⑤ 内存理解

```text
students[0]  一个完整 Student
students[1]  一个完整 Student
students[2]  一个完整 Student
```

它们通常连续存放。

> [!tip] 记忆口诀
> **一个结构体存一个对象，结构体数组存一组对象。**

# 06 | 结构体指针

> [!abstract] 核心结论
> 结构体指针保存结构体变量的地址。通过结构体指针，可以避免拷贝整个结构体，也可以在函数中修改外部结构体变量。

### ① 基本写法

```c
struct Student stu = {1, "Tom", 18};

struct Student *p = &stu;
```

含义：

```text
p 是结构体指针。
p 保存 stu 的地址。
```

### ② 通过指针访问成员

```c
p->id = 2;
p->age = 20;
```

等价于：

```c
(*p).id = 2;
(*p).age = 20;
```

### ③ 函数中修改结构体

```c
#include <stdio.h>

struct Student {
    int id;
    char name[32];
    int age;
};

void set_age(struct Student *p, int age) {
    p->age = age;
}

int main() {
    struct Student stu = {1, "Tom", 18};

    set_age(&stu, 20);

    printf("%d\n", stu.age);

    return 0;
}
```

输出：

```text
20
```

### ④ 为什么传指针

如果函数参数是：

```c
void set_age(struct Student s)
```

那么：

```text
传进去的是结构体副本。
修改 s 不会影响外面的 stu。
```

传指针：

```c
void set_age(struct Student *p)
```

可以：

```text
直接修改外部结构体。
避免大结构体拷贝。
```

### ⑤ 重点

```text
结构体变量传参会拷贝整个结构体。
结构体指针传参只拷贝地址。
```

> [!tip] 记忆口诀
> **结构体大，用指针传；想改原对象，也用指针传。**

# 07 | typedef 简化结构体写法

> [!abstract] 核心结论
> `typedef` 可以给结构体类型起别名，避免每次都写 `struct Student`。在工程中常用 `typedef struct {...} Student;`。

### ① 不使用 typedef

```c
struct Student {
    int id;
    char name[32];
    int age;
};

struct Student stu;
```

每次定义变量都要写：

```c
struct Student
```

### ② 使用 typedef

```c
typedef struct Student {
    int id;
    char name[32];
    int age;
} Student;
```

之后可以写：

```c
Student stu;
```

不需要再写：

```c
struct Student stu;
```

### ③ 常见简化写法

```c
typedef struct {
    int id;
    char name[32];
    int age;
} Student;
```

说明：

```text
结构体标签名省略。
直接创建类型别名 Student。
```

### ④ 完整示例

```c
#include <stdio.h>

typedef struct {
    int id;
    char name[32];
    int age;
} Student;

int main() {
    Student stu = {1, "Tom", 18};

    printf("%d %s %d\n", stu.id, stu.name, stu.age);

    return 0;
}
```

### ⑤ 推荐写法

如果需要自引用结构体，例如链表节点：

```c
typedef struct Node {
    int data;
    struct Node *next;
} Node;
```

因为内部要用：

```c
struct Node *next;
```

> [!tip] 记忆口诀
> **typedef 起别名，少写 struct 更清爽。**

# 08 | 结构体内存对齐

> [!abstract] 核心结论
> 结构体大小不一定等于成员大小简单相加。为了让 CPU 更高效访问数据，编译器可能在成员之间或结构体末尾插入填充字节，这叫内存对齐。

### ① 示例

```c
struct A {
    char c;
    int i;
};
```

直觉上：

```text
char 占 1 字节。
int 占 4 字节。
总共应该 5 字节。
```

但实际：

```c
sizeof(struct A)
```

常见结果是：

```text
8
```

### ② 为什么是 8

内存可能类似：

```text
c    填充 填充 填充    i i i i
1字节 3字节填充        4字节 int
```

因为：

```text
int 通常希望放在 4 的倍数地址上。
```

### ③ 成员顺序会影响大小

```c
struct A {
    char c;
    int i;
    char d;
};
```

可能占：

```text
12 字节
```

优化顺序：

```c
struct B {
    int i;
    char c;
    char d;
};
```

可能占：

```text
8 字节
```

### ④ 查看大小

```c
#include <stdio.h>

struct A {
    char c;
    int i;
    char d;
};

int main() {
    printf("%zu\n", sizeof(struct A));
    return 0;
}
```

### ⑤ 重点

```text
结构体成员不是随便紧挨着排。
编译器可能为对齐插入空洞。
```

> [!tip] 记忆口诀
> **结构体会对齐，成员顺序影响大小；大类型放前面，空洞可能少一点。**

# 09 | 结构体和多文件编程

> [!abstract] 核心结论
> 如果多个源文件都需要使用同一个结构体类型，结构体声明应该放在头文件中；函数实现放在功能文件中；主程序通过 include 头文件使用结构体和函数。

### ① student.h

```c
#ifndef STUDENT_H
#define STUDENT_H

typedef struct {
    int id;
    char name[32];
    int age;
} Student;

void print_student(Student s);
void set_student_age(Student *s, int age);

#endif
```

### ② student.c

```c
#include <stdio.h>
#include "student.h"

void print_student(Student s) {
    printf("id = %d\n", s.id);
    printf("name = %s\n", s.name);
    printf("age = %d\n", s.age);
}

void set_student_age(Student *s, int age) {
    s->age = age;
}
```

### ③ main.c

```c
#include "student.h"

int main() {
    Student stu = {1, "Tom", 18};

    set_student_age(&stu, 20);
    print_student(stu);

    return 0;
}
```

### ④ 编译

```bash
gcc main.c student.c -o app
```

### ⑤ 文件分工

```text
student.h：
放 Student 类型和函数声明。

student.c：
放 print_student、set_student_age 的实现。

main.c：
创建 Student 变量并调用函数。
```

> [!tip] 记忆口诀
> **结构体给大家用，就放头文件；函数怎么处理它，就放功能文件。**

# 10 | 结构体总结

> [!abstract] 核心结论
> 结构体的底层核心是“把多个不同类型成员按一定布局组合成一块连续内存”。它让 C 语言可以表达复杂对象，是数组、链表、文件数据、设备状态和工程模块设计的重要基础。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>结构体核心总结：</strong>
<ul>
  <li><strong>关键字：</strong><code>struct</code>。</li>
  <li><strong>本质：</strong>把多个成员变量组合成一个整体类型。</li>
  <li><strong>成员：</strong>结构体内部字段，可以是不同类型。</li>
  <li><strong>变量访问：</strong>普通结构体变量用 <code>.</code>。</li>
  <li><strong>指针访问：</strong>结构体指针用 <code>-></code>。</li>
  <li><strong>结构体数组：</strong>保存多个同类型对象。</li>
  <li><strong>结构体指针：</strong>保存结构体地址，适合函数传参和修改原对象。</li>
  <li><strong>typedef：</strong>给结构体类型起别名，简化写法。</li>
  <li><strong>内存对齐：</strong>结构体大小可能包含填充字节。</li>
  <li><strong>多文件：</strong>公共结构体类型通常放头文件中。</li>
</ul>
</div>

### ② 最底层一句话

```text
结构体 = 把多个相关数据成员打包成一个整体对象，编译器按成员顺序和对齐规则安排它们的内存布局。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **结构体打包数据，变量点号访问，指针箭头访问；typedef 起别名，对齐会补空。**
