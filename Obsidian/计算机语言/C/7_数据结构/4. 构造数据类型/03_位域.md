# 01 | 位域：按“位”来分配结构体成员

> [!abstract] 核心结论
> 位域是 C 语言结构体中的一种特殊成员写法，可以指定某个成员只占用几个二进制位。它常用于节省内存、表示标志位、解析寄存器、描述协议字段等场景。

### ① 底层原理

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>位域的本质：</strong>
<ul>
  <li><strong>依附结构体：</strong>位域只能出现在 <code>struct</code> 或 <code>union</code> 中。</li>
  <li><strong>按位分配：</strong>可以指定成员占用多少个二进制位。</li>
  <li><strong>节省空间：</strong>多个小范围数据可以压缩在一个整数存储单元里。</li>
  <li><strong>常见用途：</strong>状态标志、权限位、硬件寄存器、通信协议字段。</li>
  <li><strong>注意平台差异：</strong>位域在内存中的具体排列顺序和对齐方式与编译器、平台有关。</li>
</ul>
</div>

### ② 代码用法

```c
struct Status {
    unsigned int power : 1;
    unsigned int error : 1;
    unsigned int mode  : 2;
};
```

含义：

```text
power 占 1 位。
error 占 1 位。
mode  占 2 位。
```

### ③ 使用场景

| 场景 | 位域作用 |
| :--- | :--- |
| 状态标志 | 1 位表示开/关 |
| 权限控制 | 读、写、执行各占 1 位 |
| 寄存器映射 | 按硬件手册描述每个位 |
| 通信协议 | 按协议字段长度解析 |
| 内存节省 | 多个小值压缩保存 |

> [!tip] 记忆口诀
> **结构体按字节存，位域按位抠；几位够用，就写几位。**

# 02 | 位域的基本定义

> [!abstract] 核心结论
> 位域成员的格式是：`类型 成员名 : 位数;`。位数表示这个成员占用多少个二进制位。

### ① 基本语法

```c
struct 结构体名 {
    类型 成员名 : 位数;
};
```

例如：

```c
struct Flag {
    unsigned int a : 1;
    unsigned int b : 1;
    unsigned int c : 2;
};
```

### ② 完整示例

```c
#include <stdio.h>

struct Flag {
    unsigned int a : 1;
    unsigned int b : 1;
    unsigned int c : 2;
};

int main() {
    struct Flag f;

    f.a = 1;
    f.b = 0;
    f.c = 3;

    printf("a = %u\n", f.a);
    printf("b = %u\n", f.b);
    printf("c = %u\n", f.c);

    return 0;
}
```

输出：

```text
a = 1
b = 0
c = 3
```

### ③ 位数范围理解

```text
1 位无符号：
可表示 0 ~ 1。

2 位无符号：
可表示 0 ~ 3。

3 位无符号：
可表示 0 ~ 7。

n 位无符号：
可表示 0 ~ 2^n - 1。
```

### ④ 推荐类型

常见写法：

```c
unsigned int flag : 1;
```

比：

```c
int flag : 1;
```

更清晰，因为标志位通常不需要负数。

### ⑤ 重点

```text
位域不是普通成员。
它不能取地址。
因为它可能只占一个整数中的几个 bit。
```

> [!tip] 记忆口诀
> **冒号后面写几位，成员就占几个 bit。**

# 03 | 位域能表示的数值范围

> [!abstract] 核心结论
> 位域的位数决定它能保存的最大值。无符号位域 `n` 位可以表示 `0 ~ 2^n - 1`，如果赋值超过范围，结果可能被截断或产生实现相关行为。

### ① 无符号位域范围

```c
unsigned int mode : 2;
```

2 位可以表示：

```text
00 → 0
01 → 1
10 → 2
11 → 3
```

所以：

```text
mode 的范围是 0 ~ 3。
```

### ② 示例代码

```c
#include <stdio.h>

struct Config {
    unsigned int mode : 2;
};

int main() {
    struct Config cfg;

    cfg.mode = 0;
    printf("%u\n", cfg.mode);

    cfg.mode = 1;
    printf("%u\n", cfg.mode);

    cfg.mode = 2;
    printf("%u\n", cfg.mode);

    cfg.mode = 3;
    printf("%u\n", cfg.mode);

    return 0;
}
```

输出：

```text
0
1
2
3
```

### ③ 超出范围

```c
cfg.mode = 4;
```

因为 2 位放不下 4：

```text
4 的二进制是 100。
低 2 位是 00。
```

某些编译器上可能看到结果变成：

```text
0
```

### ④ 有符号位域

```c
signed int value : 3;
```

3 位有符号能表示的范围和具体实现有关，常见类似：

```text
-4 ~ 3
```

但实际仍要注意实现差异。

### ⑤ 初学建议

```text
表示标志、模式、编号时，优先使用 unsigned int 位域。
少用 signed 位域，避免符号和范围混乱。
```

> [!tip] 记忆口诀
> **无符号 n 位，最大 2 的 n 次方减 1。**

# 04 | 位域和普通结构体成员的区别

> [!abstract] 核心结论
> 普通结构体成员通常按字节分配空间，而位域成员可以按位分配空间。位域更省空间，但可移植性和可控性比普通成员差。

### ① 普通结构体

```c
struct Normal {
    unsigned int power;
    unsigned int error;
    unsigned int mode;
};
```

如果 `unsigned int` 是 4 字节：

```text
power 占 4 字节。
error 占 4 字节。
mode  占 4 字节。
```

总大小可能是：

```text
12 字节。
```

### ② 位域结构体

```c
struct BitField {
    unsigned int power : 1;
    unsigned int error : 1;
    unsigned int mode  : 2;
};
```

这些成员总共只需要：

```text
1 + 1 + 2 = 4 位。
```

但由于存储单元和对齐规则，整个结构体大小通常不会是半个字节，而可能是：

```text
4 字节。
```

### ③ 对比代码

```c
#include <stdio.h>

struct Normal {
    unsigned int power;
    unsigned int error;
    unsigned int mode;
};

struct BitField {
    unsigned int power : 1;
    unsigned int error : 1;
    unsigned int mode  : 2;
};

int main() {
    printf("Normal   = %zu\n", sizeof(struct Normal));
    printf("BitField = %zu\n", sizeof(struct BitField));

    return 0;
}
```

可能输出：

```text
Normal   = 12
BitField = 4
```

### ④ 对比表

| 对比项 | 普通成员 | 位域成员 |
| :--- | :--- | :--- |
| 分配单位 | 字节为主 | 位为主 |
| 是否能取地址 | 可以 | 不可以 |
| 节省空间 | 一般 | 更强 |
| 可移植性 | 较好 | 较差 |
| 适合场景 | 普通数据 | 标志位、寄存器、协议字段 |

### ⑤ 重点

```text
位域节省的是字段空间。
但整个结构体仍会受存储单元和对齐影响。
```

> [!tip] 记忆口诀
> **普通成员按字节，位域成员按 bit；省空间但别滥用。**

# 05 | 位域不能取地址

> [!abstract] 核心结论
> 位域成员不能使用 `&` 取地址，因为它可能只占一个字节中的几个二进制位，而 CPU 地址通常只能定位到字节，不能直接定位单独的 bit。

### ① 错误示例

```c
struct Flag {
    unsigned int a : 1;
};

int main() {
    struct Flag f;

    unsigned int *p = &f.a;  // 错误

    return 0;
}
```

问题：

```text
a 只占 1 个 bit。
它不是完整的可寻址对象。
不能对它取地址。
```

### ② 为什么不能取地址

普通变量：

```text
int x;
```

有完整地址：

```text
&x
```

位域成员：

```text
可能只是某个 int 存储单元里的第几个 bit。
```

硬件地址通常以字节为单位：

```text
无法直接给一个 bit 单独编号为普通指针地址。
```

### ③ 正确做法

如果需要传递值：

```c
unsigned int value = f.a;
```

如果需要修改：

```c
f.a = 1;
```

如果函数要修改整个结构体：

```c
void set_flag(struct Flag *p) {
    p->a = 1;
}
```

### ④ 重点

```text
可以对结构体变量取地址。
不能对位域成员取地址。
```

可以：

```c
struct Flag *p = &f;
```

不可以：

```c
&p->a
```

### ⑤ 使用建议

```text
如果某个成员经常需要取地址、传指针、传给库函数，
就不要把它设计成位域。
```

> [!tip] 记忆口诀
> **地址按字节找，位域按 bit 藏；bit 没法单独取地址。**

# 06 | 位域中的匿名位域和填充

> [!abstract] 核心结论
> 位域可以使用匿名成员作为填充位，也可以使用宽度为 0 的匿名位域强制下一个位域从新的存储单元开始。

### ① 匿名位域

```c
struct Reg {
    unsigned int enable : 1;
    unsigned int        : 3;
    unsigned int mode   : 2;
};
```

说明：

```text
中间的 3 位没有名字。
只能作为填充位使用。
不能访问。
```

### ② 适合场景

硬件寄存器说明中经常有：

```text
bit0      enable
bit1~3    reserved
bit4~5    mode
```

可以写成：

```c
struct Reg {
    unsigned int enable : 1;
    unsigned int        : 3;
    unsigned int mode   : 2;
};
```

### ③ 宽度为 0 的匿名位域

```c
struct Align {
    unsigned int a : 3;
    unsigned int   : 0;
    unsigned int b : 5;
};
```

含义：

```text
:0 强制下一个位域 b 从新的存储单元开始。
```

### ④ 示例理解

```text
a 占当前 unsigned int 存储单元的 3 位。
:0 表示当前存储单元剩余位不用了。
b 从新的 unsigned int 存储单元开始。
```

### ⑤ 重点

| 写法 | 作用 |
| :--- | :--- |
| `unsigned int : 3;` | 跳过 3 位 |
| `unsigned int : 0;` | 强制换到下一个存储单元 |
| 有名字位域 | 可以访问 |
| 无名字位域 | 只能填充，不能访问 |

> [!tip] 记忆口诀
> **匿名位域当空位，冒号零强制换单元。**

# 07 | 位域和硬件寄存器

> [!abstract] 核心结论
> 位域常用于描述硬件寄存器中的各个位字段，让代码看起来更接近硬件手册。但由于位域布局和平台有关，真正工程中要谨慎确认编译器规则。

### ① 假设寄存器布局

```text
bit0      enable
bit1      interrupt
bit2~3    mode
bit4~7    reserved
```

可以写成：

```c
typedef struct {
    unsigned int enable    : 1;
    unsigned int interrupt : 1;
    unsigned int mode      : 2;
    unsigned int reserved  : 4;
} ControlReg;
```

### ② 使用示例

```c
ControlReg reg;

reg.enable = 1;
reg.interrupt = 0;
reg.mode = 3;
```

这样比直接写位操作更直观：

```c
reg_value |= (1u << 0);
reg_value &= ~(1u << 1);
reg_value |= (3u << 2);
```

### ③ 注意 volatile

如果映射硬件寄存器，通常需要：

```c
volatile
```

例如：

```c
volatile ControlReg *reg = (volatile ControlReg *)0x40000000;
```

含义：

```text
这个地址对应硬件寄存器。
每次访问都必须真的读写内存。
不能被编译器随便优化掉。
```

### ④ 重要风险

```text
位域在内存中的排列顺序可能因编译器和平台不同而不同。
硬件寄存器位序非常严格。
所以工程中要查看编译器文档和芯片 SDK 写法。
```

### ⑤ 更稳定写法

很多底层工程更常用：

```c
#define REG_ENABLE_MASK  (1u << 0)
#define REG_INT_MASK     (1u << 1)
#define REG_MODE_MASK    (3u << 2)
```

再用按位运算控制寄存器。

> [!tip] 记忆口诀
> **位域看寄存器很直观，真正上硬件要看平台规则。**

# 08 | 位域和协议字段

> [!abstract] 核心结论
> 位域可以用来描述通信协议中的短字段，例如类型、标志、长度等。但因为字节序、位序和对齐问题，跨平台协议解析中不能盲目直接把字节流强转为位域结构体。

### ① 协议字段示例

假设一个 8 位字段：

```text
bit0      ack
bit1      urgent
bit2~3    type
bit4~7    length
```

可以用位域描述：

```c
typedef struct {
    unsigned int ack    : 1;
    unsigned int urgent : 1;
    unsigned int type   : 2;
    unsigned int length : 4;
} PacketFlag;
```

### ② 访问方式

```c
PacketFlag flag;

flag.ack = 1;
flag.urgent = 0;
flag.type = 2;
flag.length = 10;
```

### ③ 风险

如果收到网络字节流：

```c
unsigned char raw = 0xA5;
```

不建议直接：

```c
PacketFlag *p = (PacketFlag *)&raw;
```

原因：

```text
位域排列顺序可能不同。
网络协议通常有明确字节序。
直接强转可移植性差。
```

### ④ 更稳妥写法

```c
unsigned char raw = 0xA5;

unsigned int ack    = (raw >> 0) & 0x01;
unsigned int urgent = (raw >> 1) & 0x01;
unsigned int type   = (raw >> 2) & 0x03;
unsigned int length = (raw >> 4) & 0x0F;
```

### ⑤ 重点

```text
位域适合表达字段概念。
真正跨平台协议解析，更推荐位运算。
```

> [!tip] 记忆口诀
> **位域画协议很清楚，跨平台解析靠位运算更稳。**

# 09 | 位域常见错误

> [!abstract] 核心结论
> 位域常见错误包括给位域赋超范围值、对位域取地址、依赖位域跨平台布局、误以为结构体大小等于总位数、用 signed 位域造成符号混乱。

### ① 错误 1：赋值超范围

```c
struct Config {
    unsigned int mode : 2;
};

struct Config cfg;
cfg.mode = 4;
```

问题：

```text
2 位无符号最大只能表示 3。
4 超出范围。
```

### ② 错误 2：对位域取地址

```c
unsigned int *p = &cfg.mode;
```

错误原因：

```text
位域不是完整可寻址对象。
不能取地址。
```

### ③ 错误 3：强依赖位域排列顺序

```c
typedef struct {
    unsigned int a : 1;
    unsigned int b : 1;
} Flag;
```

问题：

```text
a 和 b 到底对应低位还是高位，可能受编译器和平台影响。
```

### ④ 错误 4：误算结构体大小

```c
struct Flag {
    unsigned int a : 1;
    unsigned int b : 1;
};
```

错误理解：

```text
sizeof(struct Flag) 应该是 1 位或 1 字节。
```

实际：

```text
通常可能是 4 字节，取决于 unsigned int 存储单元和对齐。
```

### ⑤ 错误 5：滥用 signed 位域

```c
int flag : 1;
```

问题：

```text
有符号 1 位能表示什么，容易产生困惑。
标志位通常用 unsigned 更清楚。
```

> [!tip] 记忆口诀
> **位域别取地址，赋值别越界；布局别乱信，标志用 unsigned。**

# 10 | 位域总结

> [!abstract] 核心结论
> 位域的底层核心是“在结构体中按 bit 分配成员宽度”。它能节省空间、表达标志位和寄存器字段，但具体内存布局和平台相关，工程中要谨慎使用。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>位域核心总结：</strong>
<ul>
  <li><strong>本质：</strong>结构体中的按位成员。</li>
  <li><strong>语法：</strong><code>unsigned int flag : 1;</code></li>
  <li><strong>位数含义：</strong>冒号后面的数字表示占几个 bit。</li>
  <li><strong>范围：</strong>无符号 n 位可表示 <code>0 ~ 2^n - 1</code>。</li>
  <li><strong>优点：</strong>节省内存，表达标志位直观。</li>
  <li><strong>限制：</strong>位域成员不能取地址。</li>
  <li><strong>匿名位域：</strong>可用于填充和强制对齐。</li>
  <li><strong>典型场景：</strong>状态位、权限位、寄存器、协议字段。</li>
  <li><strong>风险：</strong>位域布局、对齐、位序可能和平台有关。</li>
  <li><strong>建议：</strong>标志位可用位域，跨平台协议和硬件寄存器要谨慎。</li>
</ul>
</div>

### ② 最底层一句话

```text
位域 = 在结构体中把成员压缩到指定数量的二进制位里，用更小空间保存有限范围的数据。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **冒号后面写位数，unsigned 更稳妥；节省空间很方便，地址布局要小心。**
