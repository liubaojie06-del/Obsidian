# 01 | 二叉树的基本概念：每个结点最多两个孩子

> [!abstract] 核心结论  
> 二叉树是每个结点最多有两个孩子的树形结构，左右孩子有方向区别。

## ① 底层原理

二叉树可以为空，也可以由根结点、左子树、右子树组成。左子树和右子树本身仍然是二叉树。

---

## ② 基本语法 / 代码用法

```c
typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;
```

---

## ③ 含义 / 说明

```text
TreeNode 是二叉树结点类型。data 保存数据，left 指向左孩子，right 指向右孩子。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

错误理解：二叉树每个结点必须有两个孩子。正确理解：最多两个，可以是 0、1、2 个。

> [!tip] 记忆口诀  
> **最多两个孩子，左右不能乱换。**

---

# 02 | 结点结构：data、left、right 三部分

> [!abstract] 核心结论  
> 链式二叉树的一个结点通常由一个数据域和两个指针域组成。

## ① 底层原理

data 存数据，left 保存左孩子地址，right 保存右孩子地址。孩子不存在时指针为 NULL。

---

## ② 基本语法 / 代码用法

```c
typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;
```

---

## ③ 含义 / 说明

```text
struct TreeNode *left 不能写成 TreeNode *left，因为 typedef 别名此时还没完成定义。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：在结构体内部提前使用 typedef 别名。应写 struct TreeNode *left。

> [!tip] 记忆口诀  
> **一个数据，两根指针。**

---

# 03 | 根结点与空树：root 是整棵树入口

> [!abstract] 核心结论  
> 二叉树通常用 root 指针表示，root == NULL 表示空树。

## ① 底层原理

链式二叉树不是连续空间，必须通过 root 找到根结点，再通过 left 和 right 找到其他结点。

---

## ② 基本语法 / 代码用法

```c
TreeNode *root = NULL;
if (root == NULL) {
    printf("空树\n");
}
```

---

## ③ 含义 / 说明

```text
root 保存根结点地址。初始化为空树时必须写成 NULL，不能让它保存随机地址。
```

---

## ④ 输出

```text
空树
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：TreeNode *root; 只声明不初始化，后续访问可能崩溃。

> [!tip] 记忆口诀  
> **root 找树根，NULL 表空树。**

# 04 | 高度、深度、层数：三个概念别混淆

> [!abstract] 核心结论  
> 层数从根开始数，深度强调从根到当前结点，高度强调从当前结点到最深叶子。

## ① 底层原理

同一个结点既有深度也有高度。根结点深度通常为 1，叶子结点高度通常为 1。

---

## ② 基本语法 / 代码用法

```c
int height(TreeNode *root) {
    if (root == NULL) return 0;
    int l = height(root->left);
    int r = height(root->right);
    return l > r ? l + 1 : r + 1;
}
```

---

## ③ 含义 / 说明

```text
height(NULL) 返回 0；非空树高度等于左右子树高度最大值加 1。
```

---

## ④ 输出

```text
例如三层二叉树高度为 3。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：把高度写成左右高度相加。高度看最长路径，不是结点总数。

> [!tip] 记忆口诀  
> **深度从上数，高度从下看。**

---

# 05 | 叶子结点：左右孩子都为空

> [!abstract] 核心结论  
> 叶子结点是没有任何孩子的结点，判断条件是 left 和 right 都为 NULL。

## ① 底层原理

只要还有一个孩子，就不是叶子。叶子不一定都在同一层，除非是满二叉树等特殊结构。

---

## ② 基本语法 / 代码用法

```c
int countLeaves(TreeNode *root) {
    if (root == NULL) return 0;
    if (root->left == NULL && root->right == NULL) return 1;
    return countLeaves(root->left) + countLeaves(root->right);
}
```

---

## ③ 含义 / 说明

```text
先判断空树，再判断左右孩子是否都为空。是叶子返回 1，否则递归统计左右子树。
```

---

## ④ 输出

```text
返回叶子结点数量。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：使用 || 判断叶子。正确条件是 left == NULL && right == NULL。

> [!tip] 记忆口诀  
> **左右都为空，才算叶子。**

---

# 06 | 满二叉树：每一层都达到最大值

> [!abstract] 核心结论  
> 满二叉树是所有层都填满的二叉树，深度为 k 时共有 2^k - 1 个结点。

## ① 底层原理

满二叉树每个非叶子结点都有两个孩子，所有叶子都在最后一层。

---

## ② 基本语法 / 代码用法

```c
int fullTreeMaxNodes(int k) {
    return (1 << k) - 1;
}
```

---

## ③ 含义 / 说明

```text
1 << k 表示 2 的 k 次方，因此深度 k 的满二叉树最多有 2^k - 1 个结点。
```

---

## ④ 输出

```text
k = 3 时返回 7。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：只要每个结点有两个孩子就是满二叉树。还必须所有叶子在同一层。

> [!tip] 记忆口诀  
> **层层都满，才叫满树。**

---

# 07 | 完全二叉树：最后一层从左到右连续

> [!abstract] 核心结论  
> 完全二叉树除最后一层外都满，最后一层必须从左到右连续排列。

## ① 底层原理

完全二叉树非常适合数组存储，因为父子关系可以用下标计算。

---

## ② 基本语法 / 代码用法

```c
int leftChild(int i) { return 2 * i + 1; }
int rightChild(int i) { return 2 * i + 2; }
int parent(int i) { return (i - 1) / 2; }
```

---

## ③ 含义 / 说明

```text
这里假设根结点下标从 0 开始。左孩子、右孩子、父结点都可以直接由下标算出。
```

---

## ④ 输出

```text
i = 1 时，左孩子下标 3，右孩子下标 4。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：最后一层右边有结点，左边却空着。这不是完全二叉树。

> [!tip] 记忆口诀  
> **完全看最后一层，必须靠左排。**

---

# 08 | 平衡二叉树：左右高度差受限制

> [!abstract] 核心结论  
> 平衡二叉树要求左右子树高度差不能过大，常见要求是不超过 1。

## ① 底层原理

平衡的目的不是让结点数量完全相等，而是控制高度，避免树退化成链表。

---

## ② 基本语法 / 代码用法

```c
int absDiff(int a, int b) {
    return a > b ? a - b : b - a;
}
```

---

## ③ 含义 / 说明

```text
判断平衡时通常比较左右子树高度差。AVL 树要求每个结点高度差不超过 1。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：把平衡理解为左右结点数量一样。真正关注的是高度差。

> [!tip] 记忆口诀  
> **平衡看高度，不看数量。**

---

# 09 | 顺序存储：完全二叉树适合数组

> [!abstract] 核心结论  
> 完全二叉树可以用数组按层序保存，通过下标计算父子关系。

## ① 底层原理

数组存储要求结点位置比较紧凑。如果普通二叉树空洞太多，用数组会浪费空间。

---

## ② 基本语法 / 代码用法

```c
int tree[6] = {1, 2, 3, 4, 5, 6};
printf("%d\n", tree[0]);
```

---

## ③ 含义 / 说明

```text
tree[0] 是根结点，tree[1] 和 tree[2] 是根的左右孩子。
```

---

## ④ 输出

```text
1
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：所有二叉树都强行用数组存。普通二叉树更适合链式存储。

> [!tip] 记忆口诀  
> **完全用数组，普通用指针。**

# 10 | 主题总结：根、左右子树、递归

> [!abstract] 核心结论  
> 二叉树的核心模型是 root、left、right，大多数问题都能拆成左右子树递归处理。

## ① 底层原理

处理二叉树时先考虑空树，再处理当前结点，最后递归处理左右子树。

---

## ② 基本语法 / 代码用法

```c
void solve(TreeNode *root) {
    if (root == NULL) return;
    solve(root->left);
    solve(root->right);
}
```

---

## ③ 含义 / 说明

```text
这是二叉树递归的最小框架。不同问题只是在递归前、中、后加入不同逻辑。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：递归函数忘记写 root == NULL 的出口。

> [!tip] 记忆口诀  
> **二叉树题，先判空，再递归。**
