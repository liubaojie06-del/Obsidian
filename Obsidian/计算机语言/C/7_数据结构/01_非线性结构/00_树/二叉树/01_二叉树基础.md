# 01 | 二叉树基础：每个结点最多只有两个孩子

> [!abstract] 核心结论  
> 二叉树是一种非线性结构。每个结点最多有两个孩子，分别叫左孩子和右孩子。理解二叉树时，要抓住“根结点 + 左子树 + 右子树”的递归结构。

## ① 底层原理

二叉树通常长这样：

```text
        A
       / \
      B   C
     / \
    D   E
```

一棵二叉树可以是空树，也可以由三部分组成：

```text
二叉树 = 根结点 + 左子树 + 右子树
```

二叉树的核心特点：

> - **有根结点**：整棵树从根结点开始。  
> - **最多两个孩子**：每个结点最多有左、右两个孩子。  
> - **左右有区别**：左子树和右子树不能随便交换。  
> - **天然递归**：每一棵子树本身也是一棵二叉树。  

---

## ② 基本概念

| 概念 | 含义 |
|---|---|
| 结点 | 树中保存数据的基本单位 |
| 根结点 | 整棵树的入口结点 |
| 父结点 | 有孩子的结点 |
| 子结点 | 某个结点下面直接连接的结点 |
| 叶子结点 | 没有任何孩子的结点 |
| 度 | 一个结点拥有的孩子数量 |
| 层数 | 根结点为第 1 层，往下递增 |
| 深度 | 从根结点到当前结点经过的层数 |
| 高度 | 从当前结点到最深叶子结点经过的层数 |

---

## ③ 结点定义 / 代码用法

```c
#include <stdio.h>

typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;

int main(void) {
    TreeNode root = {10, NULL, NULL};

    printf("根结点数据: %d\n", root.data);

    return 0;
}
```

> [!note] 代码备注  
> `left` 指向左孩子，`right` 指向右孩子。如果某个孩子不存在，就让对应指针等于 `NULL`。

---

## ④ 特殊二叉树

| 类型 | 特点 |
|---|---|
| 满二叉树 | 每一层都达到最大结点数 |
| 完全二叉树 | 除最后一层外都满，最后一层从左到右连续 |
| 平衡二叉树 | 任意结点左右子树高度差不超过指定范围，常见为不超过 1 |
| 二叉搜索树 | 左子树小于根，右子树大于根 |

满二叉树：

```text
        1
       / \
      2   3
     / \ / \
    4  5 6  7
```

完全二叉树：

```text
        1
       / \
      2   3
     / \ /
    4  5 6
```

## ⑤ 常见性质

| 性质 | 公式 / 说明 |
|---|---|
| 第 `i` 层最多结点数 | `2^(i - 1)` |
| 深度为 `k` 的二叉树最多结点数 | `2^k - 1` |
| 叶子结点数和度为 2 的结点数关系 | `n0 = n2 + 1` |
| 完全二叉树适合存储方式 | 数组顺序存储 |
| 普通二叉树常用存储方式 | 链式存储 |

---

## ⑥ 重点 / 注意 / 常见错误

错误理解：

```text
二叉树中每个结点必须有两个孩子。
```

正确理解：

```text
二叉树中每个结点最多有两个孩子，可以是 0 个、1 个或 2 个。
```

| 易混概念 | 区别 |
|---|---|
| 满二叉树 | 每一层都满 |
| 完全二叉树 | 最后一层可以不满，但必须从左到右连续 |
| 高度 | 从当前结点往下数 |
| 深度 | 从根结点往下数 |

> [!tip] 记忆口诀  
> **一根分左右，左右又成树。**
