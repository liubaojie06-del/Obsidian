# 01 | 路径问题总览：DFS 加回溯

> [!abstract] 核心结论  
> 二叉树路径题一般用 DFS，从根向下走，必要时记录当前路径。

## ① 底层原理

进入结点时加入路径，离开结点时撤销选择，这就是回溯。

---

## ② 基本语法 / 代码用法

```c
void dfs(TreeNode *root) {
    if (root == NULL) return;
    dfs(root->left);
    dfs(root->right);
}
```

---

## ③ 含义 / 说明

```text
路径题会在这个 DFS 框架上加入 path、sum、answer 等变量。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：递归回来后没有恢复路径状态，导致路径混乱。

> [!tip] 记忆口诀  
> **路径向下走，回头要撤销。**

---# 02 | 根到叶路径：必须到叶子才算完整

> [!abstract] 核心结论  
> 根到叶路径要求从根结点开始，到叶子结点结束。

## ① 底层原理

判断路径完成的条件不是 root == NULL，而是当前结点是叶子。

---

## ② 基本语法 / 代码用法

```c
int isLeaf(TreeNode *root) {
    return root != NULL && root->left == NULL && root->right == NULL;
}
```

---

## ③ 含义 / 说明

```text
只有左右孩子都为空时，当前结点才是叶子。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：走到 NULL 就认为找到路径。真正路径结束在叶子结点。

> [!tip] 记忆口诀  
> **根到叶，叶子才结账。**

---# 03 | 路径总和：递归传递剩余目标值

> [!abstract] 核心结论  
> 判断路径和是否等于目标值时，可以每走一个结点就减去当前值。

## ① 底层原理

当走到叶子时，如果叶子值等于剩余目标值，说明找到合法路径。

---

## ② 基本语法 / 代码用法

```c
int hasPathSum(TreeNode *root, int target) {
    if (root == NULL) return 0;
    if (root->left == NULL && root->right == NULL) return root->data == target;
    return hasPathSum(root->left, target - root->data) ||
           hasPathSum(root->right, target - root->data);
}
```

---

## ③ 含义 / 说明

```text
target - root->data 表示剩下的路径还需要凑出的和。
```

---

## ④ 输出

```text
存在返回 1，不存在返回 0。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：中途和等于 target 就返回成功，但题目要求必须到叶子。

> [!tip] 记忆口诀  
> **一路减目标，叶子判相等。**

---# 04 | 输出所有路径：数组保存当前路径

> [!abstract] 核心结论  
> 输出所有根到叶路径时，可以用数组保存当前经过的结点。

## ① 底层原理

每进入一个结点，就把它放入 path[length]，到叶子时打印。

---

## ② 基本语法 / 代码用法

```c
void printPaths(TreeNode *root, int path[], int len) {
    if (root == NULL) return;
    path[len++] = root->data;
    if (root->left == NULL && root->right == NULL) {
        for (int i = 0; i < len; i++) printf("%d ", path[i]);
        printf("\n");
        return;
    }
    printPaths(root->left, path, len);
    printPaths(root->right, path, len);
}
```

---

## ③ 含义 / 说明

```text
len 是值传递，每层递归都有自己的长度，不会互相污染。
```

---

## ④ 输出

```text
每行输出一条根到叶路径。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：使用全局长度时，递归返回后忘记 length--。

> [!tip] 记忆口诀  
> **进来加路径，出去要还原。**

---# 05 | 路径数量：统计满足条件的路径条数

> [!abstract] 核心结论  
> 如果题目要求数量，就让递归函数返回左右子树数量之和。

## ① 底层原理

每条满足条件的路径贡献 1，不满足贡献 0。

---

## ② 基本语法 / 代码用法

```c
int countPathSum(TreeNode *root, int target) {
    if (root == NULL) return 0;
    if (root->left == NULL && root->right == NULL) return root->data == target;
    return countPathSum(root->left, target - root->data) +
           countPathSum(root->right, target - root->data);
}
```

---

## ③ 含义 / 说明

```text
与 hasPathSum 不同，这里不是 ||，而是左右数量相加。
```

---

## ④ 输出

```text
返回满足条件的根到叶路径条数。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：需要统计数量却使用逻辑或，导致只能知道有没有。

> [!tip] 记忆口诀  
> **问有无用或，问数量用加。**

---# 06 | 最大路径和：返回贡献值，更新全局答案

> [!abstract] 核心结论  
> 最大路径和要区分返回给父结点的单边贡献和当前结点形成的完整路径。

## ① 底层原理

向上只能贡献一边；更新答案时可以使用左 + 根 + 右。

---

## ② 基本语法 / 代码用法

```c
int maxGain(TreeNode *root, int *ans) {
    if (root == NULL) return 0;
    int left = maxGain(root->left, ans);
    int right = maxGain(root->right, ans);
    if (left < 0) left = 0;
    if (right < 0) right = 0;
    int cur = root->data + left + right;
    if (cur > *ans) *ans = cur;
    return root->data + (left > right ? left : right);
}
```

---

## ③ 含义 / 说明

```text
负贡献会拖累路径，所以小于 0 时直接丢弃。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：返回 left + root + right 给父结点，导致路径分叉。

> [!tip] 记忆口诀  
> **上交一条边，答案可拐弯。**

---# 08 | 最近公共祖先：左右子树返回目标

> [!abstract] 核心结论  
> LCA 通过递归判断 p 和 q 分别落在哪些子树中。

## ① 底层原理

如果左右递归都返回非空，说明当前结点是最近公共祖先。

---

## ② 基本语法 / 代码用法

```c
TreeNode *lowestCommonAncestor(TreeNode *root, TreeNode *p, TreeNode *q) {
    if (root == NULL || root == p || root == q) return root;
    TreeNode *left = lowestCommonAncestor(root->left, p, q);
    TreeNode *right = lowestCommonAncestor(root->right, p, q);
    if (left != NULL && right != NULL) return root;
    return left != NULL ? left : right;
}
```

---

## ③ 含义 / 说明

```text
返回值表示当前子树中是否找到了 p 或 q，或者已经找到了祖先。
```

---

## ④ 输出

```text
返回最近公共祖先结点地址。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：不理解 left/right 返回值含义，导致错误返回父结点。

> [!tip] 记忆口诀  
> **左右各找到，当前就是祖。**

---# 09 | DFS 与回溯区别：递归不一定等于回溯

> [!abstract] 核心结论  
> DFS 是搜索方式，回溯是恢复状态的动作。

## ① 底层原理

如果递归过程中修改了共享状态，返回时就需要撤销，这才叫回溯。

---

## ② 基本语法 / 代码用法

```c
path[len++] = root->data;
// dfs left and right
len--;
```

---

## ③ 含义 / 说明

```text
如果 len 是全局变量，递归结束后必须 len--。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：所有 DFS 都叫回溯。只有需要撤销选择时才强调回溯。

> [!tip] 记忆口诀  
> **改了共享状态，回来要复原。**
