# 07 | 平衡树与特殊二叉树：AVL、红黑树、堆

> [!abstract] 核心结论  
> 普通二叉搜索树可能退化成链表。平衡树通过旋转或颜色规则控制高度，让查找、插入、删除保持较高效率。堆是一种基于完全二叉树的特殊结构。

## ① 底层原理

| 类型 | 重点 |
|---|---|
| 满二叉树 | 每一层都满 |
| 完全二叉树 | 最后一层从左到右连续 |
| 平衡二叉树 | 左右子树高度差受限制 |
| AVL 树 | 严格平衡的 BST |
| 红黑树 | 近似平衡的 BST |
| 二叉堆 | 完全二叉树 + 堆序性质 |

为什么需要平衡？

```text
普通 BST 如果插入顺序是 1, 2, 3, 4, 5
可能退化成链表，查找效率从 O(log n) 变成 O(n)
```

## ② AVL 树

AVL 树是严格平衡的二叉搜索树。

平衡条件：

```text
任意结点的左右子树高度差不超过 1
```

平衡因子：

```text
平衡因子 = 左子树高度 - 右子树高度
```

| 平衡因子 | 含义 |
|---|---|
| `0` | 左右一样高 |
| `1` | 左边高 1 |
| `-1` | 右边高 1 |
| `> 1` 或 `< -1` | 失衡 |

AVL 树通过旋转恢复平衡：

| 失衡类型 | 处理 |
|---|---|
| LL | 右旋 |
| RR | 左旋 |
| LR | 先左旋再右旋 |
| RL | 先右旋再左旋 |

---

## ③ AVL 结点结构

```c
typedef struct AVLNode {
    int data;
    int height;
    struct AVLNode *left;
    struct AVLNode *right;
} AVLNode;

int height(AVLNode *node) {
    return node == NULL ? 0 : node->height;
}

int maxValue(int a, int b) {
    return a > b ? a : b;
}

int getBalance(AVLNode *node) {
    if (node == NULL) {
        return 0;
    }

    return height(node->left) - height(node->right);
}
```

> [!note] 代码备注  
> AVL 结点通常额外保存 `height`，这样可以快速计算平衡因子。

## ④ 红黑树了解

红黑树是一种近似平衡的二叉搜索树。它不要求像 AVL 那样严格平衡，而是通过颜色规则控制树高。

核心规则可以简单理解为：

> - 每个结点是红色或黑色。  
> - 根结点是黑色。  
> - 红色结点不能连续出现。  
> - 从任意结点到其叶子空结点的黑色结点数量相同。  

常见应用：

| 场景 | 说明 |
|---|---|
| C++ `map` / `set` | 常用红黑树实现 |
| Java `TreeMap` / `TreeSet` | 常用红黑树实现 |
| 操作系统调度 | 某些场景使用红黑树 |

## ⑤ 二叉堆

堆是一种特殊的完全二叉树。

| 类型 | 性质 |
|---|---|
| 大根堆 | 父结点 >= 子结点 |
| 小根堆 | 父结点 <= 子结点 |

如果下标从 `0` 开始：

```text
父结点：(i - 1) / 2
左孩子：2 * i + 1
右孩子：2 * i + 2
```

堆常用于：

```text
优先队列
堆排序
Top K 问题
```

## ⑥ 完全二叉树和满二叉树

| 类型 | 特点 |
|---|---|
| 满二叉树 | 每一层都满 |
| 完全二叉树 | 最后一层从左到右连续 |

关系：

```text
满二叉树一定是完全二叉树。
完全二叉树不一定是满二叉树。
```

---

## ⑦ 重点 / 注意 / 常见错误

错误理解：

```text
平衡树就是左右结点数量完全相同。
```

正确理解：

```text
平衡通常关注高度差，不是简单比较结点数量。
```

| 结构 | 关键词 |
|---|---|
| AVL | 严格平衡、旋转、高度差 |
| 红黑树 | 近似平衡、颜色规则 |
| 堆 | 完全二叉树、父子大小关系 |
| 满二叉树 | 层层满 |
| 完全二叉树 | 左到右连续 |

> [!tip] 记忆口诀  
> **AVL 管高度，红黑管颜色，堆看父子大小。**
