# 01 | 特殊二叉树总览：满、完全、平衡、堆

> [!abstract] 核心结论  
> 特殊二叉树是在普通二叉树基础上增加额外约束，用来提高存储或操作效率。

## ① 底层原理

满二叉树强调层层满，完全二叉树强调靠左连续，平衡树强调高度受控，堆强调父子大小关系。

---# 02 | 满二叉树：所有层都满

> [!abstract] 核心结论  
> 满二叉树深度为 k 时，结点数一定是 2^k - 1。

## ① 底层原理

满二叉树的所有非叶子结点都有两个孩子，所有叶子都在最后一层。

---

## ② 基本语法 / 代码用法

```c
int fullNodes(int k) {
    return (1 << k) - 1;
}
```

---

## ③ 含义 / 说明

```text
1 << k 表示 2^k，因此满二叉树结点数是 2^k - 1。
```

---

## ④ 输出

```text
k = 3 返回 7。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：只看每个结点有两个孩子，不看叶子是否同层。

> [!tip] 记忆口诀  
> **满树层层满。**

---# 03 | 完全二叉树：最后一层靠左

> [!abstract] 核心结论  
> 完全二叉树除了最后一层外都满，最后一层从左到右连续。

## ① 底层原理

完全二叉树适合数组存储，也是堆的基础结构。

---

## ② 基本语法 / 代码用法

```c
int left(int i) { return 2 * i + 1; }
int right(int i) { return 2 * i + 2; }
int parent(int i) { return (i - 1) / 2; }
```

---

## ③ 含义 / 说明

```text
下标从 0 开始时，可以直接计算父子结点位置。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：最后一层中间有空洞仍认为是完全二叉树。

> [!tip] 记忆口诀  
> **完全靠左排。**

---# 04 | 平衡二叉树：高度差不能太大

> [!abstract] 核心结论  
> 平衡二叉树通过控制高度，避免树退化成链表。

## ① 底层原理

常见判断标准是每个结点左右子树高度差不超过 1。

---

## ② 基本语法 / 代码用法

```c
int isBalanceValue(int leftHeight, int rightHeight) {
    int diff = leftHeight - rightHeight;
    if (diff < 0) diff = -diff;
    return diff <= 1;
}
```

---

## ③ 含义 / 说明

```text
这里只判断一个结点的高度差，整棵树还需要递归检查所有结点。
```

---

## ④ 输出

```text
平衡返回 1，不平衡返回 0。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：只检查根结点平衡，不检查子树平衡。

> [!tip] 记忆口诀  
> **每个结点都要平衡。**

---# 05 | AVL 树：严格平衡的 BST

> [!abstract] 核心结论  
> AVL 树是自平衡二叉搜索树，要求任意结点左右子树高度差不超过 1。

## ① 底层原理

插入或删除后如果失衡，需要通过旋转恢复平衡。

---

## ② 基本语法 / 代码用法

```c
typedef struct AVLNode {
    int data;
    int height;
    struct AVLNode *left;
    struct AVLNode *right;
} AVLNode;
```

---

## ③ 含义 / 说明

```text
AVL 结点通常额外保存 height，方便快速计算平衡因子。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：只记 BST 左小右大，忘记 AVL 还要维护高度平衡。

> [!tip] 记忆口诀  
> **AVL 是带高度的 BST。**

---# 06 | AVL 平衡因子：左高减右高

> [!abstract] 核心结论  
> 平衡因子通常定义为左子树高度减右子树高度。

## ① 底层原理

AVL 中平衡因子只能是 -1、0、1，超出范围说明失衡。

---

## ② 基本语法 / 代码用法

```c
int getBalance(AVLNode *node) {
    if (node == NULL) return 0;
    return height(node->left) - height(node->right);
}
```

---

## ③ 含义 / 说明

```text
height(NULL) 通常约定为 0。
```

---

## ④ 输出

```text
左高返回正数，右高返回负数。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：高度更新不及时，导致平衡因子计算错误。

> [!tip] 记忆口诀  
> **左减右，看平衡。**

---# 07 | AVL 旋转：LL、RR、LR、RL

> [!abstract] 核心结论  
> AVL 通过旋转修复失衡，常见四种情况是 LL、RR、LR、RL。

## ① 底层原理

LL 用右旋，RR 用左旋，LR 先左旋再右旋，RL 先右旋再左旋。

---

## ② 基本语法 / 代码用法

```c
// LL: rightRotate(root)
// RR: leftRotate(root)
// LR: root->left = leftRotate(root->left); rightRotate(root)
// RL: root->right = rightRotate(root->right); leftRotate(root)
```

---

## ③ 含义 / 说明

```text
旋转的目标是在保持 BST 顺序的同时降低高度差。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：旋转后没有更新高度。

> [!tip] 记忆口诀  
> **失衡看形状，旋转救高度。**

---# 08 | 红黑树：近似平衡的 BST

> [!abstract] 核心结论  
> 红黑树通过颜色规则保持近似平衡，工程中应用很多。

## ① 底层原理

红黑树不追求 AVL 那样严格平衡，因此插入删除调整次数通常更少。

---

## ② 基本语法 / 代码用法

```c
typedef enum { RED, BLACK } Color;
typedef struct RBNode {
    int data;
    Color color;
    struct RBNode *left;
    struct RBNode *right;
    struct RBNode *parent;
} RBNode;
```

---

## ③ 含义 / 说明

```text
红黑树结点通常保存颜色和父指针，方便调整。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：认为红黑树一定比 AVL 更平衡。AVL 更严格，红黑树更折中。

> [!tip] 记忆口诀  
> **红黑管颜色，平衡靠规则。**

---# 09 | 二叉堆：完全二叉树加堆序性质

> [!abstract] 核心结论  
> 堆是完全二叉树，不是二叉搜索树。大根堆父结点大于等于孩子，小根堆相反。

## ① 底层原理

堆只保证父子之间的大小关系，不保证左子树整体小于右子树。

---

## ② 基本语法 / 代码用法

```c
int heapParent(int i) { return (i - 1) / 2; }
int heapLeft(int i) { return 2 * i + 1; }
int heapRight(int i) { return 2 * i + 2; }
```

---

## ③ 含义 / 说明

```text
堆通常用数组实现，利用完全二叉树下标关系。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：把堆当 BST，以为左小右大。

> [!tip] 记忆口诀  
> **堆看父子，不看左右。**
