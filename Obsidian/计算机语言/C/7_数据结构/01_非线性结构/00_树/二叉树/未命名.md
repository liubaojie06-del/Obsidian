# 01 | 二叉树的基本概念：每个结点最多只有两个孩子

> [!abstract] 核心结论  
> 二叉树是一种非线性结构。它的每个结点最多有两个孩子，分别叫做左孩子和右孩子。学习二叉树时，最重要的是理解“根结点 + 左子树 + 右子树”的递归结构。

## ① 底层原理

二叉树不是像顺序表、栈、队列那样排成一条线，而是从一个根结点向下分叉。

一棵二叉树通常长这样：

```text
        A
       / \
      B   C
     / \
    D   E
```

二叉树的核心特点：

> - **有根结点**：整棵树从根结点开始。  
> - **最多两个孩子**：每个结点最多有左、右两个孩子。  
> - **左右有区别**：左子树和右子树不能随便交换。  
> - **天然递归**：每一棵子树本身也是一棵二叉树。  

二叉树可以分解成：

```text
二叉树 = 根结点 + 左子树 + 右子树
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;

int main(void) {
    TreeNode root = {10, NULL, NULL};

    printf("根结点数据: %d\n", root.data);

    return 0;
}
```

> [!note] 代码备注  
> 这段代码只创建了一个根结点。`left` 和 `right` 都是 `NULL`，说明这个结点暂时没有左孩子和右孩子。

---

## ③ 含义 / 说明

```text
typedef struct TreeNode
定义二叉树结点类型。

int data
数据域，用来保存结点中的数据。

struct TreeNode *left
左指针域，保存左孩子结点的地址。

struct TreeNode *right
右指针域，保存右孩子结点的地址。

left == NULL
表示没有左孩子。

right == NULL
表示没有右孩子。
```

---

## ④ 输出

```text
根结点数据: 10
```

---

## ⑤ 重点 / 注意 / 常见错误

> 二叉树的“二”不是说必须有两个孩子，而是说最多有两个孩子。

| 情况 | 是否是二叉树结点 |
|---|---|
| 没有孩子 | 是 |
| 只有左孩子 | 是 |
| 只有右孩子 | 是 |
| 有左、右两个孩子 | 是 |
| 有三个孩子 | 不是 |

错误理解：

```text
二叉树中每个结点都必须有两个孩子。
```

正确理解：

```text
二叉树中每个结点最多有两个孩子，可以是 0 个、1 个或 2 个。
```

> [!tip] 记忆口诀  
> **一根分左右，左右又成树。**

# 02 | 二叉树结点结构：一个数据域加两个指针域

> [!abstract] 核心结论  
> 二叉树的链式存储通常用结构体实现。一个结点包含一个数据域和两个指针域：`left` 指向左孩子，`right` 指向右孩子。

## ① 底层原理

链式二叉树中的每个结点一般长这样：

```text
+------+--------+---------+
| data | left   | right   |
+------+--------+---------+
```

其中：

> - `data` 保存真正的数据。  
> - `left` 保存左孩子结点的地址。  
> - `right` 保存右孩子结点的地址。  
> - 如果某个孩子不存在，对应指针就是 `NULL`。  

例如：

```text
        10
       /  \
      20   30
```

在内存关系上可以理解为：

```text
root->left  指向 20
root->right 指向 30
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;

int main(void) {
    TreeNode n1 = {10, NULL, NULL};
    TreeNode n2 = {20, NULL, NULL};
    TreeNode n3 = {30, NULL, NULL};

    n1.left = &n2;
    n1.right = &n3;

    printf("根结点: %d\n", n1.data);
    printf("左孩子: %d\n", n1.left->data);
    printf("右孩子: %d\n", n1.right->data);

    return 0;
}
```

> [!note] 代码备注  
> `n1.left = &n2;` 表示让 `n1` 的左孩子指向 `n2`。`n1.right = &n3;` 表示让 `n1` 的右孩子指向 `n3`。

---

## ③ 含义 / 说明

```text
TreeNode n1 = {10, NULL, NULL}
创建一个数据为 10 的结点，并且暂时没有左右孩子。

n1.left = &n2
把 n2 的地址保存到 n1 的 left 中。

n1.right = &n3
把 n3 的地址保存到 n1 的 right 中。

n1.left->data
先通过 left 找到左孩子，再访问左孩子的数据。
```

---

## ④ 输出

```text
根结点: 10
左孩子: 20
右孩子: 30
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
typedef struct TreeNode {
    int data;
    TreeNode *left;
    TreeNode *right;
} TreeNode;
```

> [!note] 代码备注  
> 在结构体还没有完整定义完成时，`TreeNode` 这个别名还不能直接使用。所以结构体内部要写 `struct TreeNode *left;` 和 `struct TreeNode *right;`。

正确写法：

```c
typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;
```

> [!tip] 记忆口诀  
> **结点三部分：数据、左指针、右指针。**

# 03 | 空树和根指针：root 保存整棵二叉树的入口

> [!abstract] 核心结论  
> 二叉树通常用根指针 `root` 表示。`root == NULL` 表示空树，`root != NULL` 表示至少有一个根结点。

## ① 底层原理

链式二叉树不是一个连续数组，而是一批由指针连接起来的结点。

程序要找到整棵二叉树，必须先知道根结点在哪里：

```text
root -> 根结点
          /  \
       左子树  右子树
```

如果没有任何结点：

```text
root -> NULL
```

这就是空树。

二叉树操作通常都从 `root` 开始，例如：

```text
遍历二叉树
统计结点个数
求树的高度
查找某个值
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;

int main(void) {
    TreeNode *root = NULL;

    if (root == NULL) {
        printf("二叉树为空\n");
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
TreeNode *root
定义一个根指针，用来保存根结点地址。

root = NULL
表示当前二叉树中没有任何结点。

root == NULL
判断二叉树是否为空。
```

---

## ④ 输出

```text
二叉树为空
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
TreeNode *root;
```

> [!note] 代码备注  
> 这里只定义了指针，但没有初始化。此时 `root` 可能保存随机地址，后续访问会非常危险。

正确写法：

```c
TreeNode *root = NULL;
```

| 状态 | 含义 |
|---|---|
| `root == NULL` | 空树 |
| `root != NULL` | 非空树 |
| `root->left == NULL` | 没有左子树 |
| `root->right == NULL` | 没有右子树 |

> [!tip] 记忆口诀  
> **root 找树根，NULL 表空树。**

# 04 | 手动创建二叉树：用指针连接左右孩子

> [!abstract] 核心结论  
> 创建二叉树的本质是创建结点，然后让父结点的 `left` 和 `right` 指向对应的孩子结点。

## ① 底层原理

假设要创建下面这棵二叉树：

```text
        1
       / \
      2   3
```

需要做三件事：

```text
1. 创建结点 1、2、3
2. 让 1 的 left 指向 2
3. 让 1 的 right 指向 3
```

也就是：

```text
root->left = node2
root->right = node3
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;

TreeNode *createNode(int value) {
    TreeNode *node = (TreeNode *)malloc(sizeof(TreeNode));

    if (node == NULL) {
        return NULL;
    }

    node->data = value;
    node->left = NULL;
    node->right = NULL;

    return node;
}

int main(void) {
    TreeNode *root = createNode(1);
    TreeNode *node2 = createNode(2);
    TreeNode *node3 = createNode(3);

    root->left = node2;
    root->right = node3;

    printf("根结点: %d\n", root->data);
    printf("左孩子: %d\n", root->left->data);
    printf("右孩子: %d\n", root->right->data);

    free(node3);
    free(node2);
    free(root);

    return 0;
}
```

> [!note] 代码备注  
> `createNode` 负责申请一个新结点，并把左右孩子初始化为 `NULL`。手动创建二叉树时，最重要的是把父子关系连接正确。

---

## ③ 含义 / 说明

```text
createNode(1)
创建数据为 1 的结点。

node->left = NULL
新结点刚创建时，默认没有左孩子。

node->right = NULL
新结点刚创建时，默认没有右孩子。

root->left = node2
把 node2 接到 root 的左边。

root->right = node3
把 node3 接到 root 的右边。
```

---

## ④ 输出

```text
根结点: 1
左孩子: 2
右孩子: 3
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
root->left->data = 2;
```

> [!note] 代码备注  
> 如果 `root->left` 还是 `NULL`，直接访问 `root->left->data` 就是空指针访问。必须先创建左孩子结点，再连接。

正确写法：

```c
TreeNode *node2 = createNode(2);
root->left = node2;
```

> [!tip] 记忆口诀  
> **先造结点，再接左右。**

# 05 | 递归创建二叉树：遇到 -1 返回 NULL

> [!abstract] 核心结论  
> 二叉树天然适合递归创建。可以用一个特殊值表示空结点，例如输入 `-1` 时返回 `NULL`。

## ① 底层原理

因为二叉树可以拆成：

```text
根结点 + 左子树 + 右子树
```

所以创建二叉树时也可以这样做：

```text
1. 先输入根结点
2. 递归创建左子树
3. 递归创建右子树
```

例如输入：

```text
1 2 -1 -1 3 -1 -1
```

表示：

```text
        1
       / \
      2   3
```

其中 `-1` 表示空结点。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;

TreeNode *createTree(void) {
    int value;
    scanf("%d", &value);

    if (value == -1) {
        return NULL;
    }

    TreeNode *node = (TreeNode *)malloc(sizeof(TreeNode));

    if (node == NULL) {
        return NULL;
    }

    node->data = value;
    node->left = createTree();
    node->right = createTree();

    return node;
}

void preorder(TreeNode *root) {
    if (root == NULL) {
        return;
    }

    printf("%d ", root->data);
    preorder(root->left);
    preorder(root->right);
}

int main(void) {
    TreeNode *root = createTree();

    preorder(root);

    return 0;
}
```

> [!note] 代码备注  
> 这段代码按照前序方式输入二叉树：先输入根结点，再输入左子树，最后输入右子树。`-1` 是递归停止的标志。

---

## ③ 含义 / 说明

```text
scanf("%d", &value)
读入当前结点的值。

if (value == -1)
如果读到 -1，说明当前位置没有结点。

node->left = createTree()
递归创建左子树。

node->right = createTree()
递归创建右子树。

return node
返回当前创建好的结点地址。
```

---

## ④ 输入 / 输出

输入：

```text
1 2 -1 -1 3 -1 -1
```

输出：

```text
1 2 3
```

---

## ⑤ 重点 / 注意 / 常见错误

> 递归创建二叉树一定要有结束条件，否则会无限递归。

错误写法：

```c
TreeNode *createTree(void) {
    TreeNode *node = malloc(sizeof(TreeNode));
    node->left = createTree();
    node->right = createTree();
    return node;
}
```

问题是：没有判断什么时候返回 `NULL`。

正确思路：

```c
if (value == -1) {
    return NULL;
}
```

> [!tip] 记忆口诀  
> **遇到空标记，递归就停下。**

# 06 | 前序遍历：先访问根结点，再访问左右子树

> [!abstract] 核心结论  
> 前序遍历的访问顺序是：根 → 左 → 右。它的特点是第一个访问到的一定是当前子树的根结点。

## ① 底层原理

前序遍历顺序：

```text
根结点 -> 左子树 -> 右子树
```

例如二叉树：

```text
        1
       / \
      2   3
     / \
    4   5
```

前序遍历结果：

```text
1 2 4 5 3
```

递归思想：

```text
访问当前根结点
前序遍历左子树
前序遍历右子树
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;

void preorder(TreeNode *root) {
    if (root == NULL) {
        return;
    }

    printf("%d ", root->data);
    preorder(root->left);
    preorder(root->right);
}
```

---

## ③ 含义 / 说明

```text
if (root == NULL)
递归出口，空树不用访问。

printf("%d ", root->data)
先访问当前根结点。

preorder(root->left)
再前序遍历左子树。

preorder(root->right)
最后前序遍历右子树。
```

---

## ④ 输出

对于下面这棵树：

```text
        1
       / \
      2   3
     / \
    4   5
```

输出：

```text
1 2 4 5 3
```

---

## ⑤ 重点 / 注意 / 常见错误

错误理解：

```text
前序遍历就是从左到右访问。
```

正确理解：

```text
前序遍历是先访问根，再处理左子树和右子树。
```

| 遍历方式 | 顺序 |
|---|---|
| 前序遍历 | 根 → 左 → 右 |

> [!tip] 记忆口诀  
> **前序根在前。**

---

# 07 | 中序遍历：先左子树，再根结点，再右子树

> [!abstract] 核心结论  
> 中序遍历的访问顺序是：左 → 根 → 右。对于二叉搜索树，中序遍历可以得到从小到大的有序序列。

## ① 底层原理

中序遍历顺序：

```text
左子树 -> 根结点 -> 右子树
```

例如二叉树：

```text
        1
       / \
      2   3
     / \
    4   5
```

中序遍历结果：

```text
4 2 5 1 3
```

递归思想：

```text
中序遍历左子树
访问当前根结点
中序遍历右子树
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;

void inorder(TreeNode *root) {
    if (root == NULL) {
        return;
    }

    inorder(root->left);
    printf("%d ", root->data);
    inorder(root->right);
}
```

---

## ③ 含义 / 说明

```text
inorder(root->left)
先递归访问左子树。

printf("%d ", root->data)
左子树访问完后，再访问当前根结点。

inorder(root->right)
最后递归访问右子树。
```

---

## ④ 输出

对于下面这棵树：

```text
        1
       / \
      2   3
     / \
    4   5
```

输出：

```text
4 2 5 1 3
```

---

## ⑤ 重点 / 注意 / 常见错误

> 中序遍历不是“中间层遍历”，而是“根结点夹在左、右子树中间”。

| 遍历方式 | 顺序 |
|---|---|
| 中序遍历 | 左 → 根 → 右 |

二叉搜索树中序遍历的特点：

```text
如果左 < 根 < 右，那么中序遍历结果就是升序。
```

> [!tip] 记忆口诀  
> **中序根在中。**

---

# 08 | 后序遍历：先访问左右子树，最后访问根结点

> [!abstract] 核心结论  
> 后序遍历的访问顺序是：左 → 右 → 根。它常用于释放二叉树，因为必须先释放孩子，再释放父结点。

## ① 底层原理

后序遍历顺序：

```text
左子树 -> 右子树 -> 根结点
```

例如二叉树：

```text
        1
       / \
      2   3
     / \
    4   5
```

后序遍历结果：

```text
4 5 2 3 1
```

递归思想：

```text
后序遍历左子树
后序遍历右子树
访问当前根结点
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;

void postorder(TreeNode *root) {
    if (root == NULL) {
        return;
    }

    postorder(root->left);
    postorder(root->right);
    printf("%d ", root->data);
}
```

---

## ③ 含义 / 说明

```text
postorder(root->left)
先访问左子树。

postorder(root->right)
再访问右子树。

printf("%d ", root->data)
最后访问当前根结点。
```

---

## ④ 输出

对于下面这棵树：

```text
        1
       / \
      2   3
     / \
    4   5
```

输出：

```text
4 5 2 3 1
```

---

## ⑤ 重点 / 注意 / 常见错误

释放二叉树时适合用后序遍历：

```c
void freeTree(TreeNode *root) {
    if (root == NULL) {
        return;
    }

    freeTree(root->left);
    freeTree(root->right);
    free(root);
}
```

> [!note] 代码备注  
> 不能先 `free(root)` 再访问 `root->left` 和 `root->right`，否则会访问已经释放的内存。

> [!tip] 记忆口诀  
> **后序根在后，释放最合适。**

# 09 | 层序遍历：用队列从上到下逐层访问

> [!abstract] 核心结论  
> 层序遍历按照从上到下、从左到右的顺序访问结点。它通常需要借助队列实现。

## ① 底层原理

层序遍历顺序：

```text
第 1 层 -> 第 2 层 -> 第 3 层 -> ...
```

例如二叉树：

```text
        1
       / \
      2   3
     / \
    4   5
```

层序遍历结果：

```text
1 2 3 4 5
```

为什么需要队列？

```text
1. 根结点入队
2. 出队一个结点并访问
3. 把它的左孩子、右孩子入队
4. 重复直到队列为空
```

队列保证“先入队的结点先访问”，正好符合逐层访问的要求。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 100

typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;

void levelOrder(TreeNode *root) {
    if (root == NULL) {
        return;
    }

    TreeNode *queue[MAX_SIZE];
    int front = 0;
    int rear = 0;

    queue[rear++] = root;

    while (front < rear) {
        TreeNode *current = queue[front++];

        printf("%d ", current->data);

        if (current->left != NULL) {
            queue[rear++] = current->left;
        }

        if (current->right != NULL) {
            queue[rear++] = current->right;
        }
    }
}
```

> [!note] 代码备注  
> 这里用数组简单模拟队列。`front` 表示队头位置，`rear` 表示队尾后一个位置。

---

## ③ 含义 / 说明

```text
queue[rear++] = root
先把根结点放入队列。

TreeNode *current = queue[front++]
取出队头结点。

printf("%d ", current->data)
访问当前结点。

current->left != NULL
如果有左孩子，就让左孩子入队。

current->right != NULL
如果有右孩子，就让右孩子入队。
```

---

## ④ 输出

对于下面这棵树：

```text
        1
       / \
      2   3
     / \
    4   5
```

输出：

```text
1 2 3 4 5
```

---

## ⑤ 重点 / 注意 / 常见错误

错误理解：

```text
层序遍历也可以直接用递归根左右完成。
```

正确理解：

```text
根左右是前序遍历，不是层序遍历。层序遍历通常依赖队列。
```

| 遍历方式 | 常用工具 |
|---|---|
| 前序 / 中序 / 后序 | 递归或栈 |
| 层序遍历 | 队列 |

> [!tip] 记忆口诀  
> **层序一层层，队列来帮忙。**

# 10 | 统计结点个数：根结点加左右子树结点数

> [!abstract] 核心结论  
> 统计二叉树结点个数的核心公式是：总数 = 1 + 左子树结点数 + 右子树结点数。

## ① 底层原理

对于任意一棵非空二叉树：

```text
结点总数 = 根结点 1 个 + 左子树结点数 + 右子树结点数
```

如果是空树：

```text
结点数 = 0
```

例如：

```text
        1
       / \
      2   3
     / \
    4   5
```

结点总数是：

```text
1 + 左子树 3 个 + 右子树 1 个 = 5
```

---

## ② 基本语法 / 代码用法

```c
typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;

int countNodes(TreeNode *root) {
    if (root == NULL) {
        return 0;
    }

    return 1 + countNodes(root->left) + countNodes(root->right);
}
```

---

## ③ 含义 / 说明

```text
if (root == NULL)
空树没有结点，所以返回 0。

1
表示当前根结点本身。

countNodes(root->left)
统计左子树中的结点个数。

countNodes(root->right)
统计右子树中的结点个数。
```

---

## ④ 输出

对于下面这棵树：

```text
        1
       / \
      2   3
     / \
    4   5
```

返回值：

```text
5
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
return countNodes(root->left) + countNodes(root->right);
```

问题是：漏掉了当前根结点。

正确写法：

```c
return 1 + countNodes(root->left) + countNodes(root->right);
```

> [!tip] 记忆口诀  
> **数结点，别忘根。**

# 11 | 求二叉树高度：左右子树高度取最大再加一

> [!abstract] 核心结论  
> 二叉树高度等于左子树高度和右子树高度的最大值再加 1。空树高度通常记为 0。

## ① 底层原理

二叉树高度可以理解为从根结点到最深叶子结点经过的层数。

例如：

```text
        1        第 1 层
       / \
      2   3      第 2 层
     / \
    4   5        第 3 层
```

这棵树的高度是：

```text
3
```

递归公式：

```text
height(root) = max(height(root->left), height(root->right)) + 1
```

空树：

```text
height(NULL) = 0
```

---

## ② 基本语法 / 代码用法

```c
typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;

int height(TreeNode *root) {
    if (root == NULL) {
        return 0;
    }

    int leftHeight = height(root->left);
    int rightHeight = height(root->right);

    return leftHeight > rightHeight ? leftHeight + 1 : rightHeight + 1;
}
```

---

## ③ 含义 / 说明

```text
height(root->left)
求左子树高度。

height(root->right)
求右子树高度。

leftHeight > rightHeight ? leftHeight + 1 : rightHeight + 1
选择较高的那棵子树，再加上当前根结点这一层。
```

---

## ④ 输出

对于下面这棵树：

```text
        1
       / \
      2   3
     / \
    4   5
```

返回值：

```text
3
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
return height(root->left) + height(root->right) + 1;
```

问题是：高度不是左右相加，而是看更深的一边。

正确写法：

```c
return max(左子树高度, 右子树高度) + 1;
```

| 问题 | 关键点 |
|---|---|
| 空树高度 | 0 |
| 只有根结点 | 1 |
| 高度怎么算 | 左右取最大再加 1 |

> [!tip] 记忆口诀  
> **求高度，看深路；左右比，大的加一。**

# 12 | 叶子结点统计：没有左右孩子的结点才是叶子

> [!abstract] 核心结论  
> 叶子结点是没有任何孩子的结点。统计叶子时，只有 `left == NULL` 且 `right == NULL` 的结点才计数。

## ① 底层原理

叶子结点的判断条件：

```text
root->left == NULL && root->right == NULL
```

例如：

```text
        1
       / \
      2   3
     / \
    4   5
```

叶子结点是：

```text
4 5 3
```

所以叶子结点个数是：

```text
3
```

---

## ② 基本语法 / 代码用法

```c
typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;

int countLeaves(TreeNode *root) {
    if (root == NULL) {
        return 0;
    }

    if (root->left == NULL && root->right == NULL) {
        return 1;
    }

    return countLeaves(root->left) + countLeaves(root->right);
}
```

---

## ③ 含义 / 说明

```text
root == NULL
空树没有叶子。

root->left == NULL && root->right == NULL
当前结点没有左右孩子，所以它是叶子结点。

countLeaves(root->left) + countLeaves(root->right)
如果当前结点不是叶子，就统计左右子树中的叶子数。
```

---

## ④ 输出

对于下面这棵树：

```text
        1
       / \
      2   3
     / \
    4   5
```

返回值：

```text
3
```

---

## ⑤ 重点 / 注意 / 常见错误

错误理解：

```text
只有在最下面一层的结点才一定是叶子。
```

正确理解：

```text
没有孩子的结点就是叶子，不一定非要和其他叶子在同一层。
```

判断叶子的条件：

```c
root->left == NULL && root->right == NULL
```

不是：

```c
root->left == NULL || root->right == NULL
```

> [!tip] 记忆口诀  
> **左右都为空，才算叶子翁。**

# 13 | 满二叉树和完全二叉树：一个每层都满，一个从左到右连续

> [!abstract] 核心结论  
> 满二叉树要求每一层都达到最大结点数。完全二叉树要求除了最后一层外都满，并且最后一层从左到右连续排列。

## ① 底层原理

满二叉树：

```text
        1
       / \
      2   3
     / \ / \
    4  5 6  7
```

特点：

> - 每一层都是满的。  
> - 深度为 `k` 时，总结点数是 `2^k - 1`。  

完全二叉树：

```text
        1
       / \
      2   3
     / \ /
    4  5 6
```

特点：

> - 除最后一层外，前面所有层都满。  
> - 最后一层结点必须从左到右连续。  
> - 堆通常使用完全二叉树实现。  

不是完全二叉树的例子：

```text
        1
       / \
      2   3
       \
        5
```

因为最后一层没有从左到右连续填充。

---

## ② 基本语法 / 代码用法

完全二叉树适合用数组存储。

如果数组下标从 `0` 开始：

```text
父结点下标: (i - 1) / 2
左孩子下标: 2 * i + 1
右孩子下标: 2 * i + 2
```

示例：

```c
#include <stdio.h>

int main(void) {
    int tree[] = {1, 2, 3, 4, 5, 6};
    int i = 1;

    printf("当前结点: %d\n", tree[i]);
    printf("左孩子: %d\n", tree[2 * i + 1]);
    printf("右孩子: %d\n", tree[2 * i + 2]);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
tree[] = {1, 2, 3, 4, 5, 6}
用数组按层序保存完全二叉树。

i = 1
当前结点是 tree[1]，也就是 2。

2 * i + 1
当前结点左孩子的下标。

2 * i + 2
当前结点右孩子的下标。
```

---

## ④ 输出

```text
当前结点: 2
左孩子: 4
右孩子: 5
```

---

## ⑤ 重点 / 注意 / 常见错误

| 类型 | 特点 |
|---|---|
| 满二叉树 | 每一层都满 |
| 完全二叉树 | 前面层都满，最后一层从左到右连续 |

错误理解：

```text
完全二叉树一定是满二叉树。
```

正确理解：

```text
满二叉树一定是完全二叉树，但完全二叉树不一定是满二叉树。
```

> [!tip] 记忆口诀  
> **满树层层满，完全左到右。**

# 14 | 二叉树常见错误：空指针、递归出口、释放顺序

> [!abstract] 核心结论  
> 二叉树代码最容易错在三个地方：没有判断空指针、递归没有出口、释放结点顺序错误。

## ① 底层原理

二叉树大量使用指针和递归，所以常见问题包括：

> - **空指针访问**：`root == NULL` 时还访问 `root->data`。  
> - **递归无出口**：函数一直调用自己，无法停止。  
> - **释放顺序错误**：先释放父结点，再访问左右孩子。  
> - **忘记释放内存**：`malloc` 创建的结点没有 `free`。  

二叉树递归函数通常都应该先写：

```c
if (root == NULL) {
    return;
}
```

如果函数有返回值，则写：

```c
if (root == NULL) {
    return 0;
}
```

---

## ② 基本语法 / 代码用法

```c
#include <stdlib.h>

typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;

void freeTree(TreeNode *root) {
    if (root == NULL) {
        return;
    }

    freeTree(root->left);
    freeTree(root->right);
    free(root);
}
```

> [!note] 代码备注  
> 释放二叉树要使用后序思想：先释放左子树，再释放右子树，最后释放根结点。

---

## ③ 含义 / 说明

```text
if (root == NULL)
空树直接返回，防止访问空指针。

freeTree(root->left)
先释放左子树。

freeTree(root->right)
再释放右子树。

free(root)
左右子树释放完以后，再释放当前根结点。
```

---

## ④ 输出

释放函数通常没有输出。

```text
无输出
```

---

## ⑤ 重点 / 注意 / 常见错误

错误 1：空指针访问

```c
void preorder(TreeNode *root) {
    printf("%d ", root->data);
    preorder(root->left);
    preorder(root->right);
}
```

正确写法：

```c
if (root == NULL) {
    return;
}
```

错误 2：释放顺序错误

```c
free(root);
freeTree(root->left);
freeTree(root->right);
```

> [!note] 代码备注  
> `free(root)` 之后，`root->left` 和 `root->right` 就不能再安全访问了。

正确写法：

```c
freeTree(root->left);
freeTree(root->right);
free(root);
```

> [!tip] 记忆口诀  
> **递归先判空，释放后序走。**

# 15 | 二叉树主题总结：根、左右子树、递归

> [!abstract] 核心结论  
> 二叉树的核心是“根结点、左子树、右子树”。大多数二叉树问题都可以用递归拆解成当前根结点的问题和左右子树的问题。

## ① 底层原理

二叉树要抓住四个关键词：

> - **root**：整棵树的入口。  
> - **left**：左孩子或左子树。  
> - **right**：右孩子或右子树。  
> - **NULL**：表示空树或没有孩子。  

核心模型：

```text
        root
       /    \
   left    right
```

递归处理二叉树时，常用思路是：

```text
1. 如果 root == NULL，直接返回
2. 处理当前根结点
3. 递归处理左子树
4. 递归处理右子树
```

不同遍历方式的区别，只是处理根结点的位置不同。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;

void preorder(TreeNode *root) {
    if (root == NULL) {
        return;
    }

    printf("%d ", root->data);
    preorder(root->left);
    preorder(root->right);
}

int countNodes(TreeNode *root) {
    if (root == NULL) {
        return 0;
    }

    return 1 + countNodes(root->left) + countNodes(root->right);
}

int height(TreeNode *root) {
    if (root == NULL) {
        return 0;
    }

    int leftHeight = height(root->left);
    int rightHeight = height(root->right);

    return leftHeight > rightHeight ? leftHeight + 1 : rightHeight + 1;
}
```

---

## ③ 含义 / 说明

```text
preorder
前序遍历，顺序是根、左、右。

countNodes
统计结点数，公式是 1 + 左子树数量 + 右子树数量。

height
求树高，公式是 max(左高度, 右高度) + 1。

root == NULL
几乎所有递归二叉树函数都需要的出口。
```

---

## ④ 遍历对比

```text
前序遍历：根 -> 左 -> 右
中序遍历：左 -> 根 -> 右
后序遍历：左 -> 右 -> 根
层序遍历：从上到下、从左到右
```

---

## ⑤ 重点 / 注意 / 常见错误

| 问题 | 关键点 |
|---|---|
| 二叉树入口在哪里 | `root` |
| 没有孩子怎么表示 | `NULL` |
| 结点结构是什么 | `data + left + right` |
| 前序遍历 | 根左右 |
| 中序遍历 | 左根右 |
| 后序遍历 | 左右根 |
| 层序遍历 | 队列 |
| 统计结点 | 根 + 左 + 右 |
| 求高度 | 左右取最大 + 1 |
| 释放二叉树 | 后序释放 |

> [!tip] 记忆口诀  
> **二叉树，分左右；递归写，先判空。**
