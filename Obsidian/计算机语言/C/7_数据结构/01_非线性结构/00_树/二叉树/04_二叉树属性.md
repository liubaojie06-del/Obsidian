# 01 | 属性题总览：左右子树结果向上合并

> [!abstract] 核心结论  
> 二叉树属性题通常用递归，先得到左右子树属性，再合并成当前树属性。

## ① 底层原理

属性题的共同框架是：空树给基础值，非空树递归左右，最后合并。

---

## ② 基本语法 / 代码用法

```c
int solve(TreeNode *root) {
    if (root == NULL) return 0;
    int left = solve(root->left);
    int right = solve(root->right);
    return left + right;
}
```

---

## ③ 含义 / 说明

```text
具体问题不同，合并方式不同。结点数量用加法，高度用最大值。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：没有想清楚递归函数返回什么。

> [!tip] 记忆口诀  
> **属性题，先定返回值。**

---# 02 | 最大深度：左右最大值加一

> [!abstract] 核心结论  
> 最大深度是从根到最深叶子结点的层数。

## ① 底层原理

当前树深度等于左右子树最大深度加上当前根这一层。

---

## ② 基本语法 / 代码用法

```c
int maxDepth(TreeNode *root) {
    if (root == NULL) return 0;
    int l = maxDepth(root->left);
    int r = maxDepth(root->right);
    return l > r ? l + 1 : r + 1;
}
```

---

## ③ 含义 / 说明

```text
空树深度为 0，只有根结点的树深度为 1。
```

---

## ④ 输出

```text
三层树返回 3。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：把左右深度相加。深度只看最长的一条路。

> [!tip] 记忆口诀  
> **最大深度，看最长路。**

---# 03 | 最小深度：必须到叶子才算

> [!abstract] 核心结论  
> 最小深度是根到最近叶子结点的层数，空孩子不能当作叶子路径。

## ① 底层原理

如果一个结点只有一边子树，最小深度必须走非空那边。

---

## ② 基本语法 / 代码用法

```c
int minDepth(TreeNode *root) {
    if (root == NULL) return 0;
    if (root->left == NULL && root->right == NULL) return 1;
    if (root->left == NULL) return minDepth(root->right) + 1;
    if (root->right == NULL) return minDepth(root->left) + 1;
    int l = minDepth(root->left), r = minDepth(root->right);
    return l < r ? l + 1 : r + 1;
}
```

---

## ③ 含义 / 说明

```text
先处理叶子和单子树情况，再比较左右最小值。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：直接 min(左深度, 右深度) + 1，会把空子树算进去。

> [!tip] 记忆口诀  
> **最小深度，到叶才停。**

---# 04 | 结点数量：根加左右子树

> [!abstract] 核心结论  
> 结点数量等于当前根 1 个，加左子树数量，加右子树数量。

## ① 底层原理

空树结点数为 0，非空树至少有根结点 1 个。

---

## ② 基本语法 / 代码用法

```c
int countNodes(TreeNode *root) {
    if (root == NULL) return 0;
    return 1 + countNodes(root->left) + countNodes(root->right);
}
```

---

## ③ 含义 / 说明

```text
1 表示当前根结点。左右递归分别统计子树数量。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：忘记加当前根结点。

> [!tip] 记忆口诀  
> **数结点，别忘根。**

---# 05 | 叶子数量：左右都空才计数

> [!abstract] 核心结论  
> 叶子结点没有任何孩子，统计时必须同时满足 left 和 right 都为空。

## ① 底层原理

空树叶子数为 0；如果当前结点是叶子，返回 1。

---

## ② 基本语法 / 代码用法

```c
int countLeaves(TreeNode *root) {
    if (root == NULL) return 0;
    if (root->left == NULL && root->right == NULL) return 1;
    return countLeaves(root->left) + countLeaves(root->right);
}
```

---

## ③ 含义 / 说明

```text
非叶子结点不计数，只统计左右子树中的叶子。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：用 || 判断叶子，导致只有一个孩子的结点也被算作叶子。

> [!tip] 记忆口诀  
> **左右都空，叶子成功。**

---# 06 | 树高：通常等同最大深度

> [!abstract] 核心结论  
> 树高可以理解为从根到最深叶子的层数，常和最大深度同义。

## ① 底层原理

不同教材可能对空树高度定义不同，写代码前要统一约定。

---

## ② 基本语法 / 代码用法

```c
int treeHeight(TreeNode *root) {
    if (root == NULL) return 0;
    int l = treeHeight(root->left);
    int r = treeHeight(root->right);
    return (l > r ? l : r) + 1;
}
```

---

## ③ 含义 / 说明

```text
本笔记约定空树高度为 0，单结点树高度为 1。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：题目按边数定义高度，你却按结点层数返回。注意题目约定。

> [!tip] 记忆口诀  
> **高度定义，先看约定。**

---# 07 | 平衡判断：高度差不超过一

> [!abstract] 核心结论  
> 判断平衡二叉树时，每个结点的左右子树高度差都不能超过 1。

## ① 底层原理

不能只判断根结点平衡，还要保证所有子树都平衡。

---

## ② 基本语法 / 代码用法

```c
int absValue(int x) { return x < 0 ? -x : x; }
int isBalancedSlow(TreeNode *root) {
    if (root == NULL) return 1;
    int diff = absValue(height(root->left) - height(root->right));
    return diff <= 1 && isBalancedSlow(root->left) && isBalancedSlow(root->right);
}
```

---

## ③ 含义 / 说明

```text
这个版本直观但会重复计算高度，效率较低。
```

---

## ④ 输出

```text
平衡返回 1，不平衡返回 0。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：只检查根结点左右高度差，不递归检查子树。

> [!tip] 记忆口诀  
> **平衡要每个结点都平衡。**

---# 08 | 高效平衡判断：不平衡直接返回 -1

> [!abstract] 核心结论  
> 可以把求高度和平衡判断合并，一旦发现不平衡就返回 -1。

## ① 底层原理

递归返回高度；如果某棵子树已经不平衡，就不再继续正常计算。

---

## ② 基本语法 / 代码用法

```c
int checkHeight(TreeNode *root) {
    if (root == NULL) return 0;
    int l = checkHeight(root->left);
    if (l == -1) return -1;
    int r = checkHeight(root->right);
    if (r == -1) return -1;
    int diff = l > r ? l - r : r - l;
    if (diff > 1) return -1;
    return (l > r ? l : r) + 1;
}
```

---

## ③ 含义 / 说明

```text
返回 -1 表示不平衡，否则返回当前子树高度。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：既想返回高度又想返回布尔值，含义混乱。

> [!tip] 记忆口诀  
> **高度正常返，不平衡返负一。**

---# 10 | 属性主题总结：先判空，再合并

> [!abstract] 核心结论  
> 属性类问题的核心是明确空树返回值和左右子树合并方式。

## ① 底层原理

二叉树属性不是靠循环顺序，而是靠递归返回值层层向上汇总。

---

## ② 基本语法 / 代码用法

```c
if (root == NULL) return 0;
int left = solve(root->left);
int right = solve(root->right);
return merge(left, right);
```

---

## ③ 含义 / 说明

```text
merge 可以是相加、取最大、取最小或逻辑判断。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：套模板但不知道 merge 应该怎么写。

> [!tip] 记忆口诀  
> **空树定底，左右合并。**
