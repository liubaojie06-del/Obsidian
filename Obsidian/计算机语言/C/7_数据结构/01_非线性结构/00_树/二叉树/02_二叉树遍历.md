# 01 | 遍历总览：DFS 和 BFS

> [!abstract] 核心结论  
> 前序、中序、后序属于深度优先遍历，层序属于广度优先遍历。

## ① 底层原理

DFS 会沿着一条路径尽量往深处走，BFS 会一层一层向外扩展。

---

## ② 基本语法 / 代码用法

```c
void preorder(TreeNode *root) {
    if (root == NULL) return;
    printf("%d ", root->data);
    preorder(root->left);
    preorder(root->right);
}
```

---

## ③ 含义 / 说明

```text
前序遍历是最典型的 DFS，先访问根，再递归访问左右子树。
```

---

## ④ 输出

```text
按照根、左、右输出。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：把层序遍历写成递归根左右。根左右是前序，不是层序。

> [!tip] 记忆口诀  
> **深度用递归，层序用队列。**

---

# 02 | 前序遍历递归：根左右

> [!abstract] 核心结论  
> 前序遍历先访问根结点，再访问左子树，最后访问右子树。

## ① 底层原理

前序的特点是根结点永远在当前子树遍历结果的最前面。

---

## ② 基本语法 / 代码用法

```c
void preorder(TreeNode *root) {
    if (root == NULL) return;
    printf("%d ", root->data);
    preorder(root->left);
    preorder(root->right);
}
```

---

## ③ 含义 / 说明

```text
printf 放在两个递归调用之前，所以根最先输出。
```

---

## ④ 输出

```text
示例树输出：1 2 4 5 3
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：忘记判断 root == NULL。

> [!tip] 记忆口诀  
> **前序根在前。**

---

# 03 | 中序遍历递归：左根右

> [!abstract] 核心结论  
> 中序遍历先访问左子树，再访问根结点，最后访问右子树。

## ① 底层原理

对二叉搜索树做中序遍历，可以得到从小到大的有序结果。

---

## ② 基本语法 / 代码用法

```c
void inorder(TreeNode *root) {
    if (root == NULL) return;
    inorder(root->left);
    printf("%d ", root->data);
    inorder(root->right);
}
```

---

## ③ 含义 / 说明

```text
printf 放在左递归和右递归中间，所以根在中间输出。
```

---

## ④ 输出

```text
示例树输出：4 2 5 1 3
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：以为中序是按层中间访问。中序指根在左右之间。

> [!tip] 记忆口诀  
> **中序根在中。**

---

# 04 | 后序遍历递归：左右根

> [!abstract] 核心结论  
> 后序遍历先访问左子树和右子树，最后访问根结点。

## ① 底层原理

后序适合释放二叉树，因为必须先处理孩子，再处理父结点。

---

## ② 基本语法 / 代码用法

```c
void postorder(TreeNode *root) {
    if (root == NULL) return;
    postorder(root->left);
    postorder(root->right);
    printf("%d ", root->data);
}
```

---

## ③ 含义 / 说明

```text
printf 放在两个递归调用之后，所以根最后输出。
```

---

## ④ 输出

```text
示例树输出：4 5 2 3 1
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：释放树时先 free(root)，然后再访问 root->left。

> [!tip] 记忆口诀  
> **后序根在后。**

---

# 05 | 层序遍历：队列实现 BFS

> [!abstract] 核心结论  
> 层序遍历按从上到下、从左到右的顺序访问结点。

## ① 底层原理

队列先进先出，正好可以保证先进入的上一层结点先被访问。

---

## ② 基本语法 / 代码用法

```c
#define MAX_SIZE 100
void levelOrder(TreeNode *root) {
    if (root == NULL) return;
    TreeNode *queue[MAX_SIZE];
    int front = 0, rear = 0;
    queue[rear++] = root;
    while (front < rear) {
        TreeNode *cur = queue[front++];
        printf("%d ", cur->data);
        if (cur->left != NULL) queue[rear++] = cur->left;
        if (cur->right != NULL) queue[rear++] = cur->right;
    }
}
```

---

## ③ 含义 / 说明

```text
根先入队。每出队一个结点，就把它的左右孩子入队。
```

---

## ④ 输出

```text
示例树输出：1 2 3 4 5
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：孩子入队前不判断 NULL。

> [!tip] 记忆口诀  
> **层序一层层，队列来帮忙。**

---

# 06 | 前序迭代：栈模拟递归

> [!abstract] 核心结论  
> 前序遍历可以用栈实现，注意先压右孩子再压左孩子。

## ① 底层原理

栈后进先出。为了让左孩子先访问，右孩子必须先入栈。

---

## ② 基本语法 / 代码用法

```c
void preorderIter(TreeNode *root) {
    TreeNode *stack[100]; int top = -1;
    if (root != NULL) stack[++top] = root;
    while (top != -1) {
        TreeNode *cur = stack[top--];
        printf("%d ", cur->data);
        if (cur->right) stack[++top] = cur->right;
        if (cur->left) stack[++top] = cur->left;
    }
}
```

---

## ③ 含义 / 说明

```text
每次弹出当前结点并访问，然后压入右孩子和左孩子。
```

---

## ④ 输出

```text
输出顺序与递归前序一致。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：先压左再压右，导致右子树先访问。

> [!tip] 记忆口诀  
> **栈里先放右，左边先出来。**

---

# 07 | 中序迭代：一路向左入栈

> [!abstract] 核心结论  
> 中序迭代的核心是先把左边界全部入栈，再弹出访问并转向右子树。

## ① 底层原理

中序是左根右，因此要先走到最左，再逐个回退访问根。

---

## ② 基本语法 / 代码用法

```c
void inorderIter(TreeNode *root) {
    TreeNode *stack[100]; int top = -1;
    TreeNode *cur = root;
    while (cur != NULL || top != -1) {
        while (cur != NULL) { stack[++top] = cur; cur = cur->left; }
        cur = stack[top--];
        printf("%d ", cur->data);
        cur = cur->right;
    }
}
```

---

## ③ 含义 / 说明

```text
内部 while 负责一直向左；弹栈后访问当前结点，再处理右子树。
```

---

## ④ 输出

```text
输出顺序与递归中序一致。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：只判断 cur != NULL，导致栈里还有结点时提前退出。

> [!tip] 记忆口诀  
> **中序迭代，先走到最左。**

---

# 08 | 后序迭代：左右根最难

> [!abstract] 核心结论  
> 后序迭代要保证左右孩子都处理完后，才能访问根。

## ① 底层原理

可以用两个栈，也可以记录上一次访问结点。初学阶段先掌握递归更重要。

---

## ② 基本语法 / 代码用法

```c
void postorderSimple(TreeNode *root) {
    if (root == NULL) return;
    postorderSimple(root->left);
    postorderSimple(root->right);
    printf("%d ", root->data);
}
```

---

## ③ 含义 / 说明

```text
后序迭代较复杂，递归版本最直观：左、右、根。
```

---

## ④ 输出

```text
输出顺序与后序定义一致。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：只把前序改一下顺序，却没有保证根最后访问。

> [!tip] 记忆口诀  
> **后序最稳，递归先会。**

---

# 09 | 遍历结果还原理解：根的位置决定类型

> [!abstract] 核心结论  
> 前中后三种遍历的区别，本质是根结点输出位置不同。

## ① 底层原理

根在前是前序，根在中是中序，根在后是后序。

---

## ② 基本语法 / 代码用法

```c
// 前序：printf 在前
// 中序：printf 在中
// 后序：printf 在后
```

---

## ③ 含义 / 说明

```text
同样是递归访问左右子树，打印语句的位置决定遍历类型。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：死记遍历结果，不理解根的位置。

> [!tip] 记忆口诀  
> **看根在哪，就是什么序。**

# 10 | 遍历主题总结：递归出口最重要

> [!abstract] 核心结论  
> 所有递归遍历都必须先处理空树，否则会访问空指针。

## ① 底层原理

root == NULL 是递归终止条件，也是保护空指针的安全边界。

---

## ② 基本语法 / 代码用法

```c
if (root == NULL) {
    return;
}
```

---

## ③ 含义 / 说明

```text
没有这个判断，root->data、root->left、root->right 都可能产生非法访问。
```

---

## ④ 输出

```text
无固定输出，重点看函数返回值或树结构变化。
```

---

## ⑤ 重点 / 注意 / 常见错误

常见错误：直接访问 root->data，不检查 root 是否为空。

> [!tip] 记忆口诀  
> **遍历第一句，先把空树挡住。**
