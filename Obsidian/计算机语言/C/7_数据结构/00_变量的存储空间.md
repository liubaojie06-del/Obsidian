# 01 | 变量的存储空间：变量到底放在哪里

> [!abstract] 核心结论
> C 语言变量不是“凭空存在”的，它们最终都要存放在内存中的某个区域。不同类型的变量会放在不同存储空间中，例如栈区、堆区、全局区、静态区、只读数据区等。

### ① 底层原理

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>变量存储空间的本质：</strong>
<ul>
  <li><strong>变量名：</strong>给一块内存起的名字。</li>
  <li><strong>变量类型：</strong>决定这块内存怎么解释。</li>
  <li><strong>变量值：</strong>存放在这块内存里的二进制数据。</li>
  <li><strong>存储区域：</strong>决定变量什么时候创建、什么时候销毁。</li>
  <li><strong>常见区域：</strong>栈区、堆区、全局区、静态区、只读数据区、代码区。</li>
</ul>
</div>

### ② 代码用法

```c
int g_num = 10;       // 全局区 / .data

int main() {
    int a = 20;       // 栈区
    static int b = 30; // 静态区 / .data

    return 0;
}
```

### ③ 使用场景

| 变量类型 | 常见存储区域 |
| :--- | :--- |
| 局部变量 | 栈区 |
| 全局变量 | 全局区 |
| 静态变量 | 静态区 |
| malloc 申请的空间 | 堆区 |
| 字符串常量 | 只读数据区 |
| 函数机器指令 | 代码区 |

> [!tip] 记忆口诀
> **变量名是门牌号，变量值住内存；不同变量住不同区。**

# 02 | 局部变量：通常存放在栈区

> [!abstract] 核心结论
> 在函数内部定义的普通局部变量，通常存放在栈区。函数调用时创建，函数结束时自动释放。

### ① 底层原理

```c
void test(void) {
    int a = 10;
}
```

这里：

```text
a 是局部变量。
通常在栈区分配空间。
进入 test 函数时创建。
离开 test 函数时销毁。
```

### ② 代码用法

```c
#include <stdio.h>

void test(void) {
    int a = 10;

    printf("a = %d\n", a);
}

int main() {
    test();

    return 0;
}
```

### ③ 栈区特点

```text
自动分配。
自动释放。
速度快。
空间有限。
生命周期短。
函数结束后局部变量失效。
```

### ④ 错误示例

```c
int *test(void) {
    int a = 10;

    return &a;
}
```

问题：

```text
a 是局部变量。
test 函数结束后，a 的栈空间失效。
返回它的地址是危险的。
```

### ⑤ 正确理解

```text
局部变量只在函数执行期间有效。
不要返回局部变量地址。
```

> [!tip] 记忆口诀
> **局部变量住栈区，函数结束就搬走。**

# 03 | 全局变量：存放在全局区

> [!abstract] 核心结论
> 定义在所有函数外部的变量叫全局变量，通常存放在全局区。程序启动时创建，程序结束时销毁。

### ① 代码用法

```c
#include <stdio.h>

int g_num = 100;

int main() {
    printf("%d\n", g_num);

    return 0;
}
```

这里：

```text
g_num 是全局变量。
它不在 main 函数栈帧里。
它通常存放在全局区。
```

### ② 生命周期

```text
程序开始运行时创建。
整个程序运行期间一直存在。
程序结束时释放。
```

### ③ 初始化和未初始化

```c
int g_a = 10;  // 已初始化全局变量，通常在 .data 段
int g_b;       // 未初始化全局变量，通常在 .bss 段
```

说明：

```text
.data：
保存已初始化的全局变量。

.bss：
保存未初始化或初始化为 0 的全局变量。
```

### ④ 默认初始化

```c
int g_num;
```

全局变量没有手动初始化时：

```text
默认初始化为 0。
```

### ⑤ 使用场景

```text
多个函数共享数据。
保存程序级状态。
配置项。
统计计数。
模块间共享变量。
```

> [!tip] 记忆口诀
> **全局变量住全局区，程序不结束它不走。**

# 04 | 静态变量：存放在静态区

> [!abstract] 核心结论
> `static` 修饰的变量通常存放在静态区。它的生命周期贯穿整个程序，但作用域可能被限制在函数内部或当前文件内部。

### ① 局部静态变量

```c
#include <stdio.h>

void test(void) {
    static int count = 0;

    count++;

    printf("count = %d\n", count);
}

int main() {
    test();
    test();
    test();

    return 0;
}
```

输出：

```text
count = 1
count = 2
count = 3
```

### ② 底层理解

```text
count 写在函数内部。
但它不是普通栈变量。
它存放在静态区。
函数结束后不会销毁。
下次调用继续使用原来的值。
```

### ③ 全局静态变量

```c
static int g_num = 10;
```

含义：

```text
g_num 存放在静态区。
但只能在当前 .c 文件内部使用。
其他文件不能 extern 访问它。
```

### ④ 生命周期和作用域

| 类型 | 生命周期 | 作用域 |
| :--- | :--- | :--- |
| 普通局部变量 | 函数调用期间 | 当前代码块 |
| 局部 static 变量 | 整个程序期间 | 当前函数内部 |
| 全局 static 变量 | 整个程序期间 | 当前源文件内部 |
| 普通全局变量 | 整个程序期间 | 可跨文件 extern |

### ⑤ 重点

```text
static 改变生命周期或链接属性。
局部 static 不放栈区，而放静态区。
```

> [!tip] 记忆口诀
> **static 变量寿命长，局部可记值，全局可隐藏。**

# 05 | 堆区变量：malloc 申请的空间

> [!abstract] 核心结论
> 堆区空间由程序员手动申请和释放。`malloc`、`calloc`、`realloc` 申请的内存通常来自堆区，需要使用 `free` 释放。

### ① 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *p = malloc(sizeof(int));

    if (p == NULL) {
        return 1;
    }

    *p = 100;

    printf("%d\n", *p);

    free(p);
    p = NULL;

    return 0;
}
```

### ② 底层理解

```text
p 是局部指针变量。
p 本身通常在栈区。

malloc 申请的 int 空间在堆区。
p 保存这块堆空间的地址。
```

内存关系：

```text
栈区：p
堆区：malloc 申请的 int 空间
```

### ③ 堆区特点

```text
手动申请。
手动释放。
生命周期由程序员控制。
适合大数据和动态大小数据。
如果忘记释放，会造成内存泄漏。
```

### ④ 常见错误

```c
int *p = malloc(sizeof(int));
*p = 10;

// 忘记 free(p)
```

问题：

```text
堆空间一直占用。
程序长时间运行可能内存泄漏。
```

### ⑤ 释放后注意

```c
free(p);
p = NULL;
```

原因：

```text
free 只是释放 p 指向的堆空间。
p 本身仍然保存原来的地址。
把 p 置 NULL 可以减少野指针风险。
```

> [!tip] 记忆口诀
> **malloc 住堆区，free 才搬走；指针在栈上，空间在堆中。**

# 06 | 字符串常量：通常存放在只读数据区

> [!abstract] 核心结论
> 字符串字面量例如 `"hello"` 通常存放在只读数据区。它的内容不应该被修改，否则可能导致程序崩溃。

### ① 代码用法

```c
char *p = "hello";
```

底层理解：

```text
"hello" 存放在只读数据区。
p 是指针变量，通常在栈区。
p 保存字符串常量首地址。
```

### ② 错误修改

```c
char *p = "hello";

p[0] = 'H';  // 危险
```

问题：

```text
试图修改只读数据区内容。
可能导致段错误。
```

### ③ 正确写法

如果想修改字符串：

```c
char arr[] = "hello";

arr[0] = 'H';
```

说明：

```text
arr 是字符数组。
数组内容通常在栈区。
"hello" 初始化时被复制到 arr 中。
arr 可以修改。
```

### ④ 对比

| 写法 | 字符串存储 | 是否可修改 |
| :--- | :--- | :--- |
| `char *p = "hello";` | 字符串常量在只读数据区 | 不建议修改 |
| `char arr[] = "hello";` | 数组内容在栈区 | 可以修改 |

### ⑤ 重点

```text
指针指向字符串常量，不等于字符串在指针里。
数组初始化字符串，会创建一份可修改副本。
```

> [!tip] 记忆口诀
> **指针指常量，别改；数组拷一份，可改。**

# 07 | 代码区：函数机器指令存放的位置

> [!abstract] 核心结论
> 函数本身编译后会变成机器指令，这些指令通常存放在代码区。函数名在很多场景下可以表示函数入口地址。

### ① 代码用法

```c
#include <stdio.h>

void hello(void) {
    printf("hello\n");
}

int main() {
    hello();

    return 0;
}
```

底层理解：

```text
hello 函数的机器指令存放在代码区。
调用 hello 时，CPU 跳转到 hello 的入口地址执行。
```

### ② 函数指针

```c
void (*p)(void) = hello;

p();
```

说明：

```text
p 保存 hello 函数的入口地址。
通过 p() 可以调用函数。
```

### ③ 代码区特点

```text
存放程序机器指令。
通常只读。
程序运行期间一直存在。
多个函数都在代码区有自己的入口地址。
```

### ④ 和变量区别

```text
变量保存数据。
函数保存指令。
```

所以：

```text
变量主要存在栈区、堆区、全局区、静态区。
函数机器码存在代码区。
```

### ⑤ 重点

```text
函数名可以理解为函数入口地址。
函数指针保存的是代码区地址。
```

> [!tip] 记忆口诀
> **变量住数据区，函数住代码区；函数名就是入口。**

# 08 | 变量存储空间和生命周期

> [!abstract] 核心结论
> 存储空间决定变量的生命周期。栈区变量生命周期短，静态区和全局区变量生命周期长，堆区变量生命周期由程序员手动控制。

### ① 生命周期对比

| 存储区域 | 创建时机 | 销毁时机 |
| :--- | :--- | :--- |
| 栈区 | 进入函数或代码块 | 离开函数或代码块 |
| 堆区 | 调用 `malloc` | 调用 `free` |
| 全局区 | 程序启动 | 程序结束 |
| 静态区 | 程序启动 | 程序结束 |
| 只读数据区 | 程序加载 | 程序结束 |
| 代码区 | 程序加载 | 程序结束 |

### ② 示例

```c
int g_num = 10;          // 程序开始到结束

void test(void) {
    int a = 20;          // test 调用期间
    static int b = 30;   // 程序开始到结束
}
```

### ③ 堆区特殊

```c
void test(void) {
    int *p = malloc(sizeof(int));
}
```

问题：

```text
p 是局部变量，函数结束后 p 消失。
但 malloc 申请的堆空间不会自动释放。
如果没 free，就泄漏。
```

正确：

```c
void test(void) {
    int *p = malloc(sizeof(int));

    free(p);
}
```

### ④ 生命周期和作用域区别

```text
生命周期：
变量存在多久。

作用域：
变量名在哪里能被访问。
```

例如：

```c
void test(void) {
    static int count = 0;
}
```

```text
count 生命周期是整个程序。
但作用域只在 test 函数内部。
```

### ⑤ 重点

```text
能不能访问一个变量名，看作用域。
变量内存是否还存在，看生命周期。
```

> [!tip] 记忆口诀
> **作用域管能不能叫名字，生命周期管内存还在不在。**

# 09 | 变量存储空间常见错误

> [!abstract] 核心结论
> 变量存储空间常见错误包括返回局部变量地址、忘记释放堆内存、释放后继续使用、修改字符串常量、混淆指针变量和指向空间的位置。

### ① 错误 1：返回局部变量地址

```c
int *get_num(void) {
    int a = 10;
    return &a;
}
```

问题：

```text
a 在栈区。
函数结束后 a 失效。
返回它的地址会产生野指针。
```

### ② 错误 2：忘记释放堆空间

```c
int *p = malloc(sizeof(int));
*p = 100;
```

问题：

```text
没有 free(p)。
堆空间泄漏。
```

### ③ 错误 3：释放后继续使用

```c
free(p);

printf("%d\n", *p);
```

问题：

```text
p 指向的堆空间已经释放。
继续访问属于悬空指针使用。
```

### ④ 错误 4：修改字符串常量

```c
char *s = "hello";
s[0] = 'H';
```

问题：

```text
"hello" 通常在只读数据区。
不应该修改。
```

### ⑤ 错误 5：混淆指针和堆空间

```c
int *p = malloc(sizeof(int));
```

正确理解：

```text
p 本身是局部变量，通常在栈区。
malloc 申请的空间在堆区。
p 只是保存堆区地址。
```

> [!tip] 记忆口诀
> **栈地址别返回，堆空间要 free；free 后别再用，常量字符串别改。**

# 10 | 变量存储空间总结

> [!abstract] 核心结论
> 变量的存储空间决定了它的创建位置、生命周期、访问方式和安全风险。理解栈区、堆区、全局区、静态区、只读数据区和代码区，是掌握指针、数组、字符串、函数、多文件编程的基础。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>变量存储空间核心总结：</strong>
<ul>
  <li><strong>变量本质：</strong>变量名对应一块内存空间。</li>
  <li><strong>局部变量：</strong>通常在栈区，函数结束自动释放。</li>
  <li><strong>全局变量：</strong>通常在全局区，程序结束才释放。</li>
  <li><strong>静态变量：</strong>通常在静态区，生命周期贯穿整个程序。</li>
  <li><strong>堆区空间：</strong><code>malloc</code> 申请，<code>free</code> 释放。</li>
  <li><strong>字符串常量：</strong>通常在只读数据区，不应修改。</li>
  <li><strong>函数代码：</strong>机器指令存放在代码区。</li>
  <li><strong>.data 段：</strong>保存已初始化的全局变量和静态变量。</li>
  <li><strong>.bss 段：</strong>保存未初始化或初始化为 0 的全局变量和静态变量。</li>
  <li><strong>核心风险：</strong>野指针、内存泄漏、越界访问、修改只读区。</li>
</ul>
</div>

### ② 最底层一句话

```text
变量的存储空间 = 变量对应的内存住在哪个区域；住哪里决定它什么时候创建、什么时候销毁、能不能被安全访问。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **局部住栈，全局静态住全局静态区；malloc 住堆，字符串常量住只读；函数代码住代码区。**
