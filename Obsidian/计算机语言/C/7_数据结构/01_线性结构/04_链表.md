# 01 | 链表的基本概念：用指针把节点串起来

> [!abstract] 核心结论  
> 链表不是靠连续数组存数据，而是靠一个个节点保存数据，再用指针连接下一个节点。

## ① 底层原理

链表的本质是：

> - **节点存数据**：每个节点里保存一个数据。
>     
> - **指针找下一个**：每个节点里还有一个指针，指向下一个节点。
>     
> - **内存不连续**：节点可以分散在内存不同位置。
>     
> - **访问靠遍历**：不能直接用下标跳到某个元素。
>     

链表通常长这样：

```text
head
 ↓
[10 | next] -> [20 | next] -> [30 | NULL]
```

每个节点一般由两部分组成：

```text
数据域：保存真正的数据
指针域：保存下一个节点的地址
```

数组靠下标访问，链表靠指针一路往后找。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

typedef struct Node {
    int data;
    struct Node *next;
} Node;

int main(void) {
    Node n1 = {10, NULL};
    Node n2 = {20, NULL};
    Node n3 = {30, NULL};

    n1.next = &n2;
    n2.next = &n3;

    printf("%d\n", n1.data);
    printf("%d\n", n1.next->data);
    printf("%d\n", n1.next->next->data);

    return 0;
}
```

> [!note] 代码备注  
> 这段代码手动创建了 3 个节点，并用 `next` 指针把它们连起来。`n1.next = &n2;` 表示第一个节点指向第二个节点。

---

## ③ 含义 / 说明

```text
typedef struct Node
定义一个链表节点类型。

int data;
数据域，保存节点里的数据。

struct Node *next;
指针域，保存下一个节点的地址。

n1.next = &n2;
让 n1 指向 n2。

n2.next = &n3;
让 n2 指向 n3。

n1.next->data
先通过 n1.next 找到 n2，再访问 n2 的 data。
```

---

## ④ 输出

```text
10
20
30
```

---

## ⑤ 重点 / 注意 / 常见错误

> 链表不是一个整体连续空间，而是一串用指针连接起来的节点。

|对比项|数组 / 顺序表|链表|
|---|---|---|
|内存是否连续|连续|不一定连续|
|访问方式|下标|指针|
|插入删除|可能移动元素|修改指针|
|随机访问|快|慢|

错误理解：

```text
链表也可以像数组一样用 list[2] 直接访问第 3 个元素。
```

正确理解：

```text
链表必须从头节点开始，一个节点一个节点往后找。
```

> [!tip] 记忆口诀  
> **数组靠下标，链表靠指针。**

# 02 | 链表节点结构：一个节点包含数据和下一个节点地址

> [!abstract] 核心结论  
> 链表的基本单位是节点。每个节点通常包含一个数据域和一个指向下一个节点的指针域。

## ① 底层原理

链表节点通常长这样：

```text
+------+------+
| data | next |
+------+------+
```

其中：

> - `data` 保存真正的数据。
>     
> - `next` 保存下一个节点的地址。
>     
> - 最后一个节点的 `next` 通常是 `NULL`。
>     

为什么结构体里要写 `struct Node *next`？

因为节点里面要保存“下一个同类型节点”的地址。

也就是说：

```text
一个 Node 节点里，放着另一个 Node 节点的地址。
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

typedef struct Node {
    int data;
    struct Node *next;
} Node;

int main(void) {
    Node node;

    node.data = 100;
    node.next = NULL;

    printf("data = %d\n", node.data);

    if (node.next == NULL) {
        printf("这是最后一个节点\n");
    }

    return 0;
}
```

> [!note] 代码备注  
> `node.next = NULL;` 表示这个节点后面没有其他节点。链表中最后一个节点的 `next` 必须指向 `NULL`，否则遍历时不知道哪里结束。

---

## ③ 含义 / 说明

```text
typedef struct Node { ... } Node;
定义节点类型，并给 struct Node 起别名 Node。

int data;
保存节点数据。

struct Node *next;
保存下一个节点的地址。

node.next = NULL;
表示当前节点后面没有节点。
```

---

## ④ 输出

```text
data = 100
这是最后一个节点
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
typedef struct Node {
    int data;
    Node *next;
} Node;
```

> [!note] 代码备注  
> 在结构体还没有完整定义完成时，`Node` 这个别名还不能直接用。所以结构体内部要写 `struct Node *next;`。

正确写法：

```c
typedef struct Node {
    int data;
    struct Node *next;
} Node;
```

> [!note] 代码备注  
> `struct Node *next;` 的意思是：`next` 是一个指针，它指向另一个 `struct Node` 类型的节点。

> [!tip] 记忆口诀  
> **节点两部分：数据加指针。**

# 03 | 动态创建节点：用 malloc 在堆区申请空间

> [!abstract] 核心结论  
> 链表节点通常用 `malloc` 动态创建。因为链表节点数量经常变化，不能完全依赖固定数组。

## ① 底层原理

链表和顺序表最大的区别之一是：

```text
顺序表：提前准备一整块连续数组空间。
链表：需要一个节点，就动态申请一个节点。
```

在 C 语言中，动态内存通常来自堆区。

使用：

```c
malloc(sizeof(Node))
```

可以在堆区申请一个节点大小的空间。

`malloc` 返回的是 `void *`，通常要转换成对应类型指针：

```c
Node *node = (Node *)malloc(sizeof(Node));
```

用完动态内存后，要用 `free` 释放。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int data;
    struct Node *next;
} Node;

int main(void) {
    Node *node = (Node *)malloc(sizeof(Node));

    if (node == NULL) {
        printf("内存申请失败\n");
        return 1;
    }

    node->data = 100;
    node->next = NULL;

    printf("节点数据: %d\n", node->data);

    free(node);

    return 0;
}
```

> [!note] 代码备注  
> `malloc` 申请的是堆区内存，函数结束后不会自动释放。这里最后必须写 `free(node);`，否则会造成内存泄漏。

---

## ③ 含义 / 说明

```text
#include <stdlib.h>
使用 malloc 和 free 必须包含这个头文件。

Node *node
定义一个指向节点的指针。

malloc(sizeof(Node))
申请一块能存放 Node 的内存。

if (node == NULL)
判断内存申请是否失败。

node->data
通过结构体指针访问 data 成员。

free(node)
释放之前申请的堆区内存。
```

---

## ④ 输出

```text
节点数据: 100
```

---

## ⑤ 重点 / 注意 / 常见错误

错误 1：忘记判断 `malloc` 是否成功

```c
Node *node = (Node *)malloc(sizeof(Node));
node->data = 100;
```

> [!note] 代码备注  
> 如果 `malloc` 失败，会返回 `NULL`。这时继续使用 `node->data`，就相当于访问空指针，程序可能崩溃。

正确写法：

```c
Node *node = (Node *)malloc(sizeof(Node));

if (node == NULL) {
    return 1;
}
```

> [!note] 代码备注  
> 先判断指针是否为 `NULL`，确认申请成功后再使用。

错误 2：忘记释放内存

```c
Node *node = (Node *)malloc(sizeof(Node));
node->data = 100;
```

> [!note] 代码备注  
> 这段代码申请了堆区空间，但没有释放。如果这种代码反复执行，程序占用的内存会越来越多。

正确写法：

```c
free(node);
```

> [!note] 代码备注  
> `free` 只能释放通过 `malloc`、`calloc`、`realloc` 申请的堆区内存。

> [!tip] 记忆口诀  
> **malloc 申请，free 归还。**

# 04 | 头指针和空链表：head 保存第一个节点地址

> [!abstract] 核心结论  
> 链表通常用 `head` 指针表示整条链表。`head` 保存第一个节点的地址，如果 `head == NULL`，说明链表为空。

## ① 底层原理

链表不是一个数组变量，而是一串节点。

程序要找到整条链表，必须先知道第一个节点在哪里。

所以需要一个头指针：

```text
head -> 第一个节点 -> 第二个节点 -> 第三个节点 -> NULL
```

如果链表为空：

```text
head -> NULL
```

也就是说：

```text
head 不是节点本身
head 是指向第一个节点的指针
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int data;
    struct Node *next;
} Node;

int main(void) {
    Node *head = NULL;

    if (head == NULL) {
        printf("链表为空\n");
    }

    return 0;
}
```

> [!note] 代码备注  
> `Node *head = NULL;` 表示当前链表没有任何节点。空链表一定要把头指针初始化为 `NULL`，不要让它保存随机地址。

---

## ③ 含义 / 说明

```text
Node *head
定义头指针。

head = NULL
表示链表为空。

if (head == NULL)
判断链表里是否没有节点。
```

---

## ④ 输出

```text
链表为空
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
Node *head;
```

> [!note] 代码备注  
> 这里只定义了一个指针，但没有初始化。此时 `head` 可能保存随机地址，后续访问会非常危险。

正确写法：

```c
Node *head = NULL;
```

> [!note] 代码备注  
> 指针不指向任何有效节点时，应该明确设置为 `NULL`。

|状态|含义|
|---|---|
|`head == NULL`|空链表|
|`head != NULL`|至少有一个节点|
|`head->next == NULL`|只有一个节点|

> [!tip] 记忆口诀  
> **head 找开头，NULL 表空表。**

---

# 05 | 链表尾插：找到最后一个节点，再接上新节点

> [!abstract] 核心结论  
> 链表尾插要先创建新节点，再找到最后一个节点，把最后一个节点的 `next` 指向新节点。

## ① 底层原理

尾插前：

```text
head -> [10 | next] -> [20 | NULL]
```

插入 `30`：

```text
head -> [10 | next] -> [20 | next] -> [30 | NULL]
```

尾插主要做两件事：

```text
1. 创建新节点
2. 让原来的最后一个节点指向新节点
```

最后一个节点的特点是：

```text
current->next == NULL
```

所以要通过循环找到它。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int data;
    struct Node *next;
} Node;

Node *createNode(int value) {
    Node *newNode = (Node *)malloc(sizeof(Node));

    if (newNode == NULL) {
        return NULL;
    }

    newNode->data = value;
    newNode->next = NULL;

    return newNode;
}

void append(Node **head, int value) {
    Node *newNode = createNode(value);

    if (newNode == NULL) {
        return;
    }

    if (*head == NULL) {
        *head = newNode;
        return;
    }

    Node *current = *head;

    while (current->next != NULL) {
        current = current->next;
    }

    current->next = newNode;
}

void printList(Node *head) {
    Node *current = head;

    while (current != NULL) {
        printf("%d ", current->data);
        current = current->next;
    }

    printf("\n");
}

int main(void) {
    Node *head = NULL;

    append(&head, 10);
    append(&head, 20);
    append(&head, 30);

    printList(head);

    return 0;
}
```

> [!note] 代码备注  
> `append(&head, 10);` 传入的是 `head` 的地址，因为当链表为空时，尾插需要修改头指针本身，让 `head` 指向第一个新节点。

---

## ③ 含义 / 说明

```text
Node *createNode(int value)
创建一个新节点。

Node **head
二级指针，用来修改外面的头指针。

if (*head == NULL)
如果链表为空，新节点就是第一个节点。

while (current->next != NULL)
一直走到最后一个节点。

current->next = newNode
把新节点接到链表尾部。
```

---

## ④ 输出

```text
10 20 30
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
void append(Node *head, int value) {
    head = createNode(value);
}
```

> [!note] 代码备注  
> 这里的 `head` 是外部头指针的一份拷贝。修改它不会改变外面的 `head`，所以空链表插入会失败。

正确写法：

```c
void append(Node **head, int value) {
    *head = createNode(value);
}
```

> [!note] 代码备注  
> `Node **head` 保存的是头指针的地址。通过 `*head` 可以修改外面的头指针。

> [!tip] 记忆口诀  
> **尾插找尾巴，next 接新家。**

---

# 06 | 链表头插：新节点指向原头，再更新 head

> [!abstract] 核心结论  
> 头插就是把新节点放到链表最前面。核心操作是：新节点指向原来的头节点，然后让 `head` 指向新节点。

## ① 底层原理

原链表：

```text
head -> [20 | next] -> [30 | NULL]
```

头插 `10`：

```text
newNode -> [10 | next]
newNode->next = head
head = newNode
```

结果：

```text
head -> [10 | next] -> [20 | next] -> [30 | NULL]
```

头插不需要遍历链表，所以效率很高。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int data;
    struct Node *next;
} Node;

Node *createNode(int value) {
    Node *newNode = (Node *)malloc(sizeof(Node));

    if (newNode == NULL) {
        return NULL;
    }

    newNode->data = value;
    newNode->next = NULL;

    return newNode;
}

void pushFront(Node **head, int value) {
    Node *newNode = createNode(value);

    if (newNode == NULL) {
        return;
    }

    newNode->next = *head;
    *head = newNode;
}

void printList(Node *head) {
    Node *current = head;

    while (current != NULL) {
        printf("%d ", current->data);
        current = current->next;
    }

    printf("\n");
}

int main(void) {
    Node *head = NULL;

    pushFront(&head, 30);
    pushFront(&head, 20);
    pushFront(&head, 10);

    printList(head);

    return 0;
}
```

> [!note] 代码备注  
> 头插的顺序不能反。必须先让 `newNode->next = *head;` 保存原链表入口，再让 `*head = newNode;` 更新头指针。

---

## ③ 含义 / 说明

```text
pushFront(&head, 10)
在链表头部插入 10。

newNode->next = *head
让新节点指向原来的第一个节点。

*head = newNode
让头指针改为指向新节点。
```

---

## ④ 输出

```text
10 20 30
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
*head = newNode;
newNode->next = *head;
```

> [!note] 代码备注  
> 这样写会让新节点的 `next` 指向自己，形成死循环。原来的链表入口也丢了。

正确写法：

```c
newNode->next = *head;
*head = newNode;
```

> [!note] 代码备注  
> 先保存原头节点地址，再更新头指针。链表操作里，顺序非常重要。

> [!tip] 记忆口诀  
> **头插两步走：先连旧头，再换新头。**

---

# 07 | 链表遍历：从 head 开始，沿着 next 往后走

> [!abstract] 核心结论  
> 链表不能用下标直接访问，只能从 `head` 开始，通过 `next` 一个节点一个节点往后遍历。

## ① 底层原理

链表遍历过程：

```text
current = head
访问 current
current = current->next
继续访问
直到 current == NULL
```

为什么要判断 `current != NULL`？

因为 `NULL` 表示后面没有节点了。

如果访问：

```c
current->data
```

但此时 `current == NULL`，程序就会访问空指针，可能直接崩溃。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

typedef struct Node {
    int data;
    struct Node *next;
} Node;

void printList(Node *head) {
    Node *current = head;

    while (current != NULL) {
        printf("%d ", current->data);
        current = current->next;
    }

    printf("\n");
}

int main(void) {
    Node n1 = {10, NULL};
    Node n2 = {20, NULL};
    Node n3 = {30, NULL};

    n1.next = &n2;
    n2.next = &n3;

    printList(&n1);

    return 0;
}
```

> [!note] 代码备注  
> `current` 是遍历指针。它一开始指向头节点，每访问完一个节点，就通过 `current = current->next;` 移动到下一个节点。

---

## ③ 含义 / 说明

```text
Node *current = head
从第一个节点开始遍历。

while (current != NULL)
只要当前节点存在，就继续访问。

printf("%d ", current->data)
打印当前节点的数据。

current = current->next
移动到下一个节点。
```

---

## ④ 输出

```text
10 20 30
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
while (current->next != NULL) {
    printf("%d ", current->data);
    current = current->next;
}
```

> [!note] 代码备注  
> 这样会漏掉最后一个节点。因为最后一个节点的 `next` 是 `NULL`，循环条件不成立，最后一个节点不会被打印。

正确写法：

```c
while (current != NULL) {
    printf("%d ", current->data);
    current = current->next;
}
```

> [!note] 代码备注  
> 判断当前节点是否存在，而不是判断下一个节点是否存在。这样才能打印到最后一个节点。

> [!tip] 记忆口诀  
> **遍历看 current，走路靠 next。**

---

# 08 | 链表删除节点：先找到前一个节点，再修改 next

> [!abstract] 核心结论  
> 删除链表节点的关键不是移动数据，而是修改指针。要让前一个节点跳过被删除节点，直接指向后一个节点。

## ① 底层原理

原链表：

```text
head -> [10] -> [20] -> [30] -> NULL
```

删除 `20`：

```text
让 10 的 next 指向 30
释放 20
```

结果：

```text
head -> [10] -> [30] -> NULL
```

核心操作：

```text
prev->next = current->next
free(current)
```

这里：

> - `prev` 是被删节点的前一个节点。
>     
> - `current` 是要删除的节点。
>     
> - `current->next` 是被删节点的后一个节点。
>     

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int data;
    struct Node *next;
} Node;

Node *createNode(int value) {
    Node *newNode = (Node *)malloc(sizeof(Node));

    if (newNode == NULL) {
        return NULL;
    }

    newNode->data = value;
    newNode->next = NULL;

    return newNode;
}

void pushFront(Node **head, int value) {
    Node *newNode = createNode(value);

    if (newNode == NULL) {
        return;
    }

    newNode->next = *head;
    *head = newNode;
}

int deleteValue(Node **head, int value) {
    if (*head == NULL) {
        return 0;
    }

    Node *current = *head;

    if (current->data == value) {
        *head = current->next;
        free(current);
        return 1;
    }

    Node *prev = current;
    current = current->next;

    while (current != NULL) {
        if (current->data == value) {
            prev->next = current->next;
            free(current);
            return 1;
        }

        prev = current;
        current = current->next;
    }

    return 0;
}

void printList(Node *head) {
    Node *current = head;

    while (current != NULL) {
        printf("%d ", current->data);
        current = current->next;
    }

    printf("\n");
}

int main(void) {
    Node *head = NULL;

    pushFront(&head, 30);
    pushFront(&head, 20);
    pushFront(&head, 10);

    deleteValue(&head, 20);

    printList(head);

    return 0;
}
```

> [!note] 代码备注  
> 删除节点时要分两种情况：删除头节点和删除非头节点。删除头节点会改变 `head`，所以函数参数必须使用 `Node **head`。

---

## ③ 含义 / 说明

```text
if (*head == NULL)
空链表不能删除。

if (current->data == value)
如果头节点就是要删除的节点。

*head = current->next
让头指针指向第二个节点。

prev->next = current->next
让前一个节点跳过 current。

free(current)
释放被删除节点的内存。
```

---

## ④ 输出

```text
10 30
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
free(current);
prev->next = current->next;
```

> [!note] 代码备注  
> `free(current)` 之后，`current` 指向的内存已经被释放，再访问 `current->next` 就是危险行为。

正确写法：

```c
prev->next = current->next;
free(current);
```

> [!note] 代码备注  
> 删除前先把链表重新接好，再释放被删除节点。顺序不能反。

> [!tip] 记忆口诀  
> **删除不搬家，改线再释放。**

# 09 | 链表查找节点：从头开始一个个比较

> [!abstract] 核心结论  
> 链表查找不能直接跳到目标位置，只能从头节点开始，沿着 `next` 一个节点一个节点比较。

## ① 底层原理

链表节点不连续，所以不能像数组一样通过下标算地址。

查找过程是：

```text
head -> 10 -> 20 -> 30 -> NULL
```

查找 `30` 时：

```text
先看 10
再看 20
再看 30
找到
```

如果走到 `NULL` 还没找到，说明目标不存在。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

typedef struct Node {
    int data;
    struct Node *next;
} Node;

Node *findValue(Node *head, int value) {
    Node *current = head;

    while (current != NULL) {
        if (current->data == value) {
            return current;
        }

        current = current->next;
    }

    return NULL;
}

int main(void) {
    Node n1 = {10, NULL};
    Node n2 = {20, NULL};
    Node n3 = {30, NULL};

    n1.next = &n2;
    n2.next = &n3;

    Node *result = findValue(&n1, 20);

    if (result != NULL) {
        printf("找到了: %d\n", result->data);
    } else {
        printf("没找到\n");
    }

    return 0;
}
```

> [!note] 代码备注  
> `findValue` 找到目标后返回节点地址，找不到就返回 `NULL`。返回地址的好处是后续可以继续访问或修改这个节点。

---

## ③ 含义 / 说明

```text
Node *findValue(Node *head, int value)
查找值为 value 的节点。

Node *current = head
从头节点开始。

if (current->data == value)
如果当前节点数据等于目标值，就返回当前节点地址。

return NULL
遍历结束还没找到，就返回 NULL。
```

---

## ④ 输出

```text
找到了: 20
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
if (result->data == 20) {
    printf("找到了\n");
}
```

> [!note] 代码备注  
> 如果 `findValue` 没找到，会返回 `NULL`。这时直接访问 `result->data` 就是空指针访问。

正确写法：

```c
if (result != NULL) {
    printf("找到了: %d\n", result->data);
}
```

> [!note] 代码备注  
> 使用查找结果前，必须先判断是不是 `NULL`。

> [!tip] 记忆口诀  
> **链表查找从头走，遇到 NULL 才算无。**

# 10 | 链表释放内存：一个节点一个节点 free

> [!abstract] 核心结论  
> 动态链表用完后，必须逐个释放节点。释放前要先保存下一个节点地址，否则链表会断掉。

## ① 底层原理

链表节点通常来自 `malloc`。

每一个 `malloc` 出来的节点，都应该对应一次 `free`。

释放链表时不能直接写：

```c
free(head);
```

因为这只释放第一个节点，后面的节点会丢失地址，造成内存泄漏。

正确思路：

```text
保存下一个节点
释放当前节点
移动到下一个节点
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int data;
    struct Node *next;
} Node;

Node *createNode(int value) {
    Node *newNode = (Node *)malloc(sizeof(Node));

    if (newNode == NULL) {
        return NULL;
    }

    newNode->data = value;
    newNode->next = NULL;

    return newNode;
}

void pushFront(Node **head, int value) {
    Node *newNode = createNode(value);

    if (newNode == NULL) {
        return;
    }

    newNode->next = *head;
    *head = newNode;
}

void freeList(Node **head) {
    Node *current = *head;

    while (current != NULL) {
        Node *nextNode = current->next;
        free(current);
        current = nextNode;
    }

    *head = NULL;
}

int main(void) {
    Node *head = NULL;

    pushFront(&head, 30);
    pushFront(&head, 20);
    pushFront(&head, 10);

    freeList(&head);

    if (head == NULL) {
        printf("链表已释放\n");
    }

    return 0;
}
```

> [!note] 代码备注  
> `Node *nextNode = current->next;` 必须写在 `free(current);` 前面。因为释放当前节点后，就不能再安全访问 `current->next` 了。

---

## ③ 含义 / 说明

```text
Node *current = *head
从头节点开始释放。

Node *nextNode = current->next
先保存下一个节点地址。

free(current)
释放当前节点。

current = nextNode
移动到下一个节点。

*head = NULL
释放完成后，把头指针置空。
```

---

## ④ 输出

```text
链表已释放
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
while (head != NULL) {
    free(head);
    head = head->next;
}
```

> [!note] 代码备注  
> `free(head)` 之后，再访问 `head->next` 是危险的，因为这块内存已经被释放。

正确写法：

```c
Node *nextNode = current->next;
free(current);
current = nextNode;
```

> [!note] 代码备注  
> 释放前先保存“下一站地址”，这样释放当前节点后还能继续往后走。

> [!tip] 记忆口诀  
> **释放先记路，再 free 当前户。**

---

# 11 | 链表和顺序表对比：一个改指针，一个搬元素

> [!abstract] 核心结论  
> 链表适合频繁插入删除，顺序表适合频繁按下标访问。两者没有绝对好坏，关键看使用场景。

## ① 底层原理

顺序表：

```text
内存连续
可以通过下标直接算地址
插入删除可能要移动元素
```

链表：

```text
内存不一定连续
通过指针连接节点
插入删除主要修改指针
```

顺序表像一排连续座位。

链表像每个人手里拿着下一个人的地址。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

typedef struct Node {
    int data;
    struct Node *next;
} Node;

int main(void) {
    int arr[3] = {10, 20, 30};

    Node n1 = {10, NULL};
    Node n2 = {20, NULL};
    Node n3 = {30, NULL};

    n1.next = &n2;
    n2.next = &n3;

    printf("数组第二个元素: %d\n", arr[1]);
    printf("链表第二个元素: %d\n", n1.next->data);

    return 0;
}
```

> [!note] 代码备注  
> 数组访问第二个元素可以直接写 `arr[1]`。链表访问第二个节点，需要先从 `n1.next` 找到第二个节点，再访问它的 `data`。

---

## ③ 含义 / 说明

```text
arr[1]
数组可以用下标直接访问。

n1.next->data
链表要通过指针访问下一个节点。

数组访问快。
链表插入删除更灵活。
```

---

## ④ 输出

```text
数组第二个元素: 20
链表第二个元素: 20
```

---

## ⑤ 重点 / 注意 / 常见错误

|对比项|顺序表|链表|
|---|---|---|
|底层结构|数组|节点 + 指针|
|内存|连续|不一定连续|
|按下标访问|快|慢|
|插入删除|需要移动元素|修改指针|
|空间开销|较小|需要额外指针|
|实现难度|简单|指针容易出错|

适合顺序表：

```text
经常按下标访问
数据量变化不大
插入删除不频繁
```

适合链表：

```text
频繁插入删除
数据量变化较大
不需要经常随机访问
```

> [!tip] 记忆口诀  
> **顺序表查得快，链表插删快。**

---

# 12 | 链表常见错误：空指针、断链、内存泄漏

> [!abstract] 核心结论  
> 链表最容易错在指针管理。写链表代码时，要重点防止空指针访问、链表断开和忘记释放内存。

## ① 底层原理

链表依赖指针连接。

只要某个指针改错，可能出现：

> - **空指针访问**：访问 `NULL->data`。
>     
> - **断链**：后面的节点地址丢失。
>     
> - **死循环**：某个节点指向自己或前面的节点。
>     
> - **内存泄漏**：`malloc` 后没有 `free`。
>     

C 语言不会自动帮你管理这些问题。

链表灵活，但也更容易写错。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int data;
    struct Node *next;
} Node;

Node *createNode(int value) {
    Node *newNode = (Node *)malloc(sizeof(Node));

    if (newNode == NULL) {
        return NULL;
    }

    newNode->data = value;
    newNode->next = NULL;

    return newNode;
}

int main(void) {
    Node *node = createNode(10);

    if (node == NULL) {
        printf("节点创建失败\n");
        return 1;
    }

    printf("节点数据: %d\n", node->data);

    free(node);
    node = NULL;

    return 0;
}
```

> [!note] 代码备注  
> `free(node);` 之后再写 `node = NULL;`，可以避免野指针继续指向已经释放的内存。虽然内存释放了，但指针变量本身还在。

---

## ③ 含义 / 说明

```text
createNode(10)
创建一个节点。

if (node == NULL)
判断节点是否创建成功。

free(node)
释放节点内存。

node = NULL
避免 node 变成野指针。
```

---

## ④ 输出

```text
节点数据: 10
```

---

## ⑤ 重点 / 注意 / 常见错误

错误 1：空指针访问

```c
Node *node = NULL;
printf("%d\n", node->data);
```

> [!note] 代码备注  
> `node` 没有指向任何有效节点，访问 `node->data` 就是访问空指针，程序很可能崩溃。

正确写法：

```c
if (node != NULL) {
    printf("%d\n", node->data);
}
```

> [!note] 代码备注  
> 访问指针指向的内容前，先确认指针不是 `NULL`。

错误 2：头插顺序写反

```c
*head = newNode;
newNode->next = *head;
```

> [!note] 代码备注  
> 这样会让新节点指向自己，原来的链表入口会丢失，可能造成死循环或内存泄漏。

正确写法：

```c
newNode->next = *head;
*head = newNode;
```

> [!note] 代码备注  
> 先让新节点接住原来的头节点，再更新头指针。

错误 3：释放后继续使用

```c
free(node);
printf("%d\n", node->data);
```

> [!note] 代码备注  
> `free` 后，这块内存已经不属于你了。继续访问属于未定义行为，可能输出旧值，也可能直接崩溃。

正确写法：

```c
free(node);
node = NULL;
```

> [!note] 代码备注  
> 释放后把指针置为 `NULL`，后续可以通过判断 `node == NULL` 避免误用。

> [!tip] 记忆口诀  
> **空指针先判断，改指针防断链，用完记得 free。**

---

# 13 | 链表主题总结：节点存数据，指针连关系

> [!abstract] 核心结论  
> 链表的核心就是节点和指针。节点保存数据，指针保存关系，整条链表靠 `head` 找到入口。

## ① 底层原理

链表要抓住四个关键词：

> - **节点**：链表的基本单位。
>     
> - **指针**：连接节点。
>     
> - **head**：保存第一个节点地址。
>     
> - **NULL**：表示链表结束。
>     

链表核心模型：

```text
head -> 节点 -> 节点 -> 节点 -> NULL
```

和顺序表相比：

```text
顺序表靠数组连续存储
链表靠指针动态连接
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int data;
    struct Node *next;
} Node;

Node *createNode(int value) {
    Node *newNode = (Node *)malloc(sizeof(Node));

    if (newNode == NULL) {
        return NULL;
    }

    newNode->data = value;
    newNode->next = NULL;

    return newNode;
}

void pushFront(Node **head, int value) {
    Node *newNode = createNode(value);

    if (newNode == NULL) {
        return;
    }

    newNode->next = *head;
    *head = newNode;
}

void printList(Node *head) {
    Node *current = head;

    while (current != NULL) {
        printf("%d ", current->data);
        current = current->next;
    }

    printf("\n");
}

void freeList(Node **head) {
    Node *current = *head;

    while (current != NULL) {
        Node *nextNode = current->next;
        free(current);
        current = nextNode;
    }

    *head = NULL;
}

int main(void) {
    Node *head = NULL;

    pushFront(&head, 30);
    pushFront(&head, 20);
    pushFront(&head, 10);

    printList(head);

    freeList(&head);

    return 0;
}
```

> [!note] 代码备注  
> 这段代码串起了链表最基本的流程：创建节点、头插节点、遍历打印、释放链表。学习链表时，先把这四步理解清楚。

---

## ③ 含义 / 说明

```text
createNode
负责动态创建一个节点。

pushFront
负责把新节点插入链表头部。

printList
负责从 head 开始遍历链表。

freeList
负责释放整条链表的堆区内存。

main
负责组织完整流程。
```

---

## ④ 输出

```text
10 20 30
```

---

## ⑤ 重点 / 注意 / 常见错误

|问题|关键点|
|---|---|
|链表入口在哪里|`head`|
|节点怎么连接|`next`|
|链表哪里结束|`NULL`|
|为什么插删方便|改指针即可|
|为什么访问慢|只能从头遍历|
|动态内存怎么处理|`malloc` 后要 `free`|

> [!tip] 记忆口诀  
> **节点存数据，next 找下家；head 管入口，NULL 到尾巴。**
