# 01 | 链式队列的基本概念：用链表实现先进先出

> [!abstract] 核心结论  
> 链式队列就是“链表 + front + rear”。`front` 指向队头，负责出队；`rear` 指向队尾，负责入队，整体遵循先进先出 FIFO。

## ① 底层原理

队列是一种受限制的线性结构：

```text
先进先出 FIFO
```

也就是：先进入队列的元素，先离开队列。

链式队列用节点保存数据，用指针连接节点：

```text
front                     rear
  ↓                        ↓
[10 | next] -> [20 | next] -> [30 | NULL]
```

这里：

> - `front` 指向队头节点。
>     
> - `rear` 指向队尾节点。
>     
> - 入队在队尾进行。
>     
> - 出队在队头进行。
>     

和循环队列相比，链式队列不需要固定数组容量，但需要动态申请和释放内存。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct QueueNode {
    int data;
    struct QueueNode *next;
} QueueNode;

typedef struct {
    QueueNode *front;
    QueueNode *rear;
} LinkQueue;

int main(void) {
    LinkQueue queue;
    queue.front = NULL;
    queue.rear = NULL;

    printf("链式队列初始化完成\n");

    return 0;
}
```

---

## ③ 含义 / 说明

```text
QueueNode
表示链式队列中的一个节点。

int data
节点的数据域，用来保存真正的数据。

struct QueueNode *next
节点的指针域，保存下一个节点的地址。

LinkQueue
表示整个链式队列。

front
指向队头节点。

rear
指向队尾节点。
```

---

## ④ 输出

```text
链式队列初始化完成
```

---

## ⑤ 重点 / 注意 / 常见错误

> 链式队列的核心不是数组下标，而是两个指针：`front` 和 `rear`。

|名称|含义|
|---|---|
|`data`|保存节点数据|
|`next`|连接下一个节点|
|`front`|指向队头节点|
|`rear`|指向队尾节点|
|`NULL`|表示空队列或队尾结束|

> [!tip] 记忆口诀  
> **front 管出队，rear 管入队。**

# 02 | 链式队列初始化：front 和 rear 都设为 NULL

> [!abstract] 核心结论  
> 初始化链式队列时，要把 `front` 和 `rear` 都设置为 `NULL`。二者都为空，表示队列中没有任何节点。

## ① 底层原理

空的链式队列没有队头，也没有队尾：

```text
front -> NULL
rear  -> NULL
```

只要队列为空，`front` 和 `rear` 都应该是 `NULL`。

如果只把其中一个设置为 `NULL`，另一个仍然是野指针，后续操作就可能出错。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct QueueNode {
    int data;
    struct QueueNode *next;
} QueueNode;

typedef struct {
    QueueNode *front;
    QueueNode *rear;
} LinkQueue;

void initQueue(LinkQueue *queue) {
    queue->front = NULL;
    queue->rear = NULL;
}

int main(void) {
    LinkQueue queue;

    initQueue(&queue);

    if (queue.front == NULL && queue.rear == NULL) {
        printf("链式队列为空\n");
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
initQueue(&queue)
初始化链式队列。

queue->front = NULL
队头指针不指向任何节点。

queue->rear = NULL
队尾指针不指向任何节点。
```

---

## ④ 输出

```text
链式队列为空
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
void initQueue(LinkQueue queue) {
    queue.front = NULL;
    queue.rear = NULL;
}
```

这个函数修改的是副本，外面的 `queue` 不会被初始化。

正确写法：

```c
void initQueue(LinkQueue *queue) {
    queue->front = NULL;
    queue->rear = NULL;
}
```

> [!tip] 记忆口诀  
> **初始化双指针，改原队列传地址。**

# 03 | 判断空队列：front 是否等于 NULL

> [!abstract] 核心结论  
> 当 `front == NULL` 时，说明链式队列为空，不能继续出队，也不能读取队头元素。

## ① 底层原理

链式队列靠 `front` 找到第一个节点：

```text
front -> [10] -> [20] -> [30] -> NULL
```

如果 `front` 是 `NULL`：

```text
front -> NULL
```

说明没有任何节点可以访问。

这时如果写：

```c
queue->front->data
```

就是空指针访问，程序可能直接崩溃。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct QueueNode {
    int data;
    struct QueueNode *next;
} QueueNode;

typedef struct {
    QueueNode *front;
    QueueNode *rear;
} LinkQueue;

int isEmpty(LinkQueue *queue) {
    return queue->front == NULL;
}

int main(void) {
    LinkQueue queue = {NULL, NULL};

    if (isEmpty(&queue)) {
        printf("链式队列为空\n");
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
isEmpty(&queue)
判断链式队列是否为空。

queue->front == NULL
如果 front 没有指向任何节点，说明队列为空。

return queue->front == NULL
条件成立返回 1，不成立返回 0。
```

---

## ④ 输出

```text
链式队列为空
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
if (queue->front->data == 0) {
    printf("空队列\n");
}
```

问题是：空队列时 `queue->front` 是 `NULL`，不能继续访问 `data`。

正确写法：

```c
if (queue->front == NULL) {
    printf("空队列\n");
}
```

> [!tip] 记忆口诀  
> **判断空不看数据，只看 front 是否为空。**

# 04 | 创建新节点：malloc 申请空间，填数据，置 next

> [!abstract] 核心结论  
> 链式队列每次入队都需要创建一个新节点。创建节点时，要先 `malloc` 申请空间，再设置 `data` 和 `next`。

## ① 底层原理

链式队列的节点通常在堆区动态创建：

```text
malloc(sizeof(QueueNode))
```

申请成功后，得到一个节点地址：

```text
newNode -> [data | next]
```

然后填入数据：

```text
newNode->data = value
newNode->next = NULL
```

由于新节点会被插入队尾，所以它的 `next` 一开始应该是 `NULL`。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct QueueNode {
    int data;
    struct QueueNode *next;
} QueueNode;

QueueNode *createNode(int value) {
    QueueNode *newNode = (QueueNode *)malloc(sizeof(QueueNode));

    if (newNode == NULL) {
        return NULL;
    }

    newNode->data = value;
    newNode->next = NULL;

    return newNode;
}

int main(void) {
    QueueNode *node = createNode(10);

    if (node != NULL) {
        printf("节点数据: %d\n", node->data);
        free(node);
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
malloc(sizeof(QueueNode))
申请一个队列节点大小的内存空间。

if (newNode == NULL)
判断内存申请是否失败。

newNode->data = value
保存节点数据。

newNode->next = NULL
新节点默认作为队尾，后面没有节点。
```

---

## ④ 输出

```text
节点数据: 10
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
QueueNode *newNode = (QueueNode *)malloc(sizeof(QueueNode));
newNode->data = value;
```

问题是没有判断 `malloc` 是否失败。

正确写法：

```c
if (newNode == NULL) {
    return NULL;
}
```

> [!tip] 记忆口诀  
> **malloc 后先判断，队尾节点 next 置空。**

---

# 05 | 入队 enqueue：新节点接到 rear 后面

> [!abstract] 核心结论  
> 链式队列入队就是尾插。非空队列中，先让原队尾节点指向新节点，再让 `rear` 指向新节点。

## ① 底层原理

原来的链式队列：

```text
front        rear
  ↓           ↓
[10] -> [20] -> NULL
```

入队 `30` 时：

```text
rear->next = newNode
rear = newNode
```

结果：

```text
front                 rear
  ↓                    ↓
[10] -> [20] -> [30] -> NULL
```

注意：如果原队列为空，`front` 和 `rear` 都要指向新节点。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct QueueNode {
    int data;
    struct QueueNode *next;
} QueueNode;

typedef struct {
    QueueNode *front;
    QueueNode *rear;
} LinkQueue;

QueueNode *createNode(int value) {
    QueueNode *newNode = (QueueNode *)malloc(sizeof(QueueNode));

    if (newNode == NULL) {
        return NULL;
    }

    newNode->data = value;
    newNode->next = NULL;

    return newNode;
}

int enqueue(LinkQueue *queue, int value) {
    QueueNode *newNode = createNode(value);

    if (newNode == NULL) {
        return 0;
    }

    if (queue->rear == NULL) {
        queue->front = newNode;
        queue->rear = newNode;
    } else {
        queue->rear->next = newNode;
        queue->rear = newNode;
    }

    return 1;
}

int main(void) {
    LinkQueue queue = {NULL, NULL};

    enqueue(&queue, 10);
    enqueue(&queue, 20);
    enqueue(&queue, 30);

    printf("队头元素: %d\n", queue.front->data);
    printf("队尾元素: %d\n", queue.rear->data);

    return 0;
}
```

> [!note] 代码备注  
> 这段代码为了突出入队操作，没有释放节点。实际完整程序中，用完链式队列后应该逐个 `free`。

---

## ③ 含义 / 说明

```text
enqueue(&queue, 30)
把 30 插入链式队列队尾。

queue->rear == NULL
如果队尾为空，说明原队列是空队列。

queue->front = newNode
空队列入队后，新节点既是队头，也是队尾。

queue->rear->next = newNode
非空队列中，让原队尾连接新节点。

queue->rear = newNode
更新队尾指针。
```

---

## ④ 输出

```text
队头元素: 10
队尾元素: 30
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
queue->rear = newNode;
queue->rear->next = newNode;
```

这样会让新节点指向自己，并且原队尾节点丢失连接。

正确写法：

```c
queue->rear->next = newNode;
queue->rear = newNode;
```

> [!tip] 记忆口诀  
> **入队接尾巴，先连后换 rear。**

# 06 | 出队 dequeue：删除 front 指向的节点

> [!abstract] 核心结论  
> 链式队列出队就是删除队头节点。先保存队头数据和队头节点地址，再让 `front` 后移，最后释放旧队头。

## ① 底层原理

原来的链式队列：

```text
front                 rear
  ↓                    ↓
[10] -> [20] -> [30] -> NULL
```

出队时，要删除 `10`：

```text
value = front->data
oldFront = front
front = front->next
free(oldFront)
```

结果：

```text
front          rear
  ↓             ↓
[20] -> [30] -> NULL
```

如果删除的是最后一个节点，出队后队列为空，必须让 `rear` 也变成 `NULL`。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct QueueNode {
    int data;
    struct QueueNode *next;
} QueueNode;

typedef struct {
    QueueNode *front;
    QueueNode *rear;
} LinkQueue;

int isEmpty(LinkQueue *queue) {
    return queue->front == NULL;
}

int dequeue(LinkQueue *queue, int *value) {
    if (isEmpty(queue)) {
        return 0;
    }

    QueueNode *oldFront = queue->front;
    *value = oldFront->data;
    queue->front = oldFront->next;

    if (queue->front == NULL) {
        queue->rear = NULL;
    }

    free(oldFront);

    return 1;
}

int main(void) {
    QueueNode *n1 = (QueueNode *)malloc(sizeof(QueueNode));
    QueueNode *n2 = (QueueNode *)malloc(sizeof(QueueNode));
    QueueNode *n3 = (QueueNode *)malloc(sizeof(QueueNode));
    int value;

    n1->data = 10;
    n1->next = n2;
    n2->data = 20;
    n2->next = n3;
    n3->data = 30;
    n3->next = NULL;

    LinkQueue queue = {n1, n3};

    if (dequeue(&queue, &value)) {
        printf("出队元素: %d\n", value);
        printf("新的队头: %d\n", queue.front->data);
    }

    while (dequeue(&queue, &value)) {
        /* 清空剩余节点 */
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
QueueNode *oldFront = queue->front
保存原队头节点地址。

*value = oldFront->data
取出队头数据。

queue->front = oldFront->next
让 front 指向下一个节点。

if (queue->front == NULL)
如果删除后没有节点了，说明队列为空。

queue->rear = NULL
空队列中 rear 也必须置空。

free(oldFront)
释放原来的队头节点。
```

---

## ④ 输出

```text
出队元素: 10
新的队头: 20
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
free(queue->front);
queue->front = queue->front->next;
```

问题是释放后又访问 `queue->front->next`，属于危险行为。

正确写法：

```c
QueueNode *oldFront = queue->front;
queue->front = oldFront->next;
free(oldFront);
```

> [!tip] 记忆口诀  
> **出队先存头，front 后移，再释放。**

---

# 07 | 读取队头 peek：只看 front，不删除节点

> [!abstract] 核心结论  
> 读取队头元素只访问 `front->data`，不会修改 `front`，也不会释放节点。

## ① 底层原理

链式队列的队头元素就在 `front` 指向的节点里：

```text
front                 rear
  ↓                    ↓
[10] -> [20] -> [30] -> NULL
```

读取队头就是：

```text
value = front->data
```

读取完成后，队列结构不变。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct QueueNode {
    int data;
    struct QueueNode *next;
} QueueNode;

typedef struct {
    QueueNode *front;
    QueueNode *rear;
} LinkQueue;

int isEmpty(LinkQueue *queue) {
    return queue->front == NULL;
}

int peek(LinkQueue *queue, int *value) {
    if (isEmpty(queue)) {
        return 0;
    }

    *value = queue->front->data;
    return 1;
}

int main(void) {
    QueueNode n1 = {10, NULL};
    QueueNode n2 = {20, NULL};
    QueueNode n3 = {30, NULL};
    int value;

    n1.next = &n2;
    n2.next = &n3;

    LinkQueue queue = {&n1, &n3};

    if (peek(&queue, &value)) {
        printf("队头元素: %d\n", value);
        printf("front 仍然指向: %d\n", queue.front->data);
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
peek(&queue, &value)
读取队头元素。

*value = queue->front->data
把队头节点的数据保存到 value 中。

没有修改 queue->front
说明读取队头不会删除节点。
```

---

## ④ 输出

```text
队头元素: 10
front 仍然指向: 10
```

---

## ⑤ 重点 / 注意 / 常见错误

|操作|是否修改 front|是否修改 rear|是否释放节点|含义|
|---|---|---|---|---|
|`peek`|不修改|不修改|不释放|只看队头|
|`dequeue`|修改|可能修改|释放|取出并删除队头|
|`enqueue`|不修改或初始化时修改|修改|申请新节点|插入队尾|

> [!tip] 记忆口诀  
> **peek 只看头，dequeue 才删头。**

# 08 | 链式队列遍历：从 front 沿着 next 往后走

> [!abstract] 核心结论  
> 链式队列遍历从 `front` 开始，沿着 `next` 一个节点一个节点访问，直到遇到 `NULL`。

## ① 底层原理

链式队列结构：

```text
front -> [10] -> [20] -> [30] -> NULL
```

遍历过程：

```text
current = front
访问 current->data
current = current->next
直到 current == NULL
```

因为链式队列是链表结构，所以不能用数组下标随机访问。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct QueueNode {
    int data;
    struct QueueNode *next;
} QueueNode;

typedef struct {
    QueueNode *front;
    QueueNode *rear;
} LinkQueue;

void printQueue(LinkQueue *queue) {
    QueueNode *current = queue->front;

    while (current != NULL) {
        printf("%d ", current->data);
        current = current->next;
    }

    printf("\n");
}

int main(void) {
    QueueNode n1 = {10, NULL};
    QueueNode n2 = {20, NULL};
    QueueNode n3 = {30, NULL};

    n1.next = &n2;
    n2.next = &n3;

    LinkQueue queue = {&n1, &n3};

    printQueue(&queue);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
QueueNode *current = queue->front
从队头节点开始遍历。

while (current != NULL)
只要当前节点存在，就继续访问。

printf("%d ", current->data)
打印当前节点数据。

current = current->next
移动到下一个节点。
```

---

## ④ 输出

```text
10 20 30
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
while (current->next != NULL) {
    printf("%d ", current->data);
    current = current->next;
}
```

这样会漏掉最后一个节点。

正确写法：

```c
while (current != NULL) {
    printf("%d ", current->data);
    current = current->next;
}
```

> [!tip] 记忆口诀  
> **遍历从 front 走，遇到 NULL 才停。**

---

# 09 | 释放链式队列：逐个 free 节点，最后双指针置空

> [!abstract] 核心结论  
> 链式队列中的节点如果来自 `malloc`，用完后必须逐个 `free`。释放完成后，要把 `front` 和 `rear` 都设置为 `NULL`。

## ① 底层原理

链式队列中的每个节点都可能是动态申请的：

```text
front -> [10] -> [20] -> [30] -> NULL
```

释放时不能只写：

```c
free(queue->front);
```

这样只释放了队头节点，后面的节点地址会丢失，造成内存泄漏。

正确思路是：

```text
保存下一个节点
释放当前节点
移动到下一个节点
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct QueueNode {
    int data;
    struct QueueNode *next;
} QueueNode;

typedef struct {
    QueueNode *front;
    QueueNode *rear;
} LinkQueue;

void clearQueue(LinkQueue *queue) {
    QueueNode *current = queue->front;

    while (current != NULL) {
        QueueNode *nextNode = current->next;
        free(current);
        current = nextNode;
    }

    queue->front = NULL;
    queue->rear = NULL;
}

int main(void) {
    LinkQueue queue = {NULL, NULL};

    clearQueue(&queue);

    if (queue.front == NULL && queue.rear == NULL) {
        printf("链式队列已清空\n");
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
QueueNode *current = queue->front
从队头节点开始释放。

QueueNode *nextNode = current->next
释放当前节点前，先保存下一个节点地址。

free(current)
释放当前节点。

current = nextNode
移动到下一个节点。

queue->front = NULL
queue->rear = NULL
释放完成后，队头队尾都置空。
```

---

## ④ 输出

```text
链式队列已清空
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
while (current != NULL) {
    free(current);
    current = current->next;
}
```

问题是 `free(current)` 后，不能再安全访问 `current->next`。

正确写法：

```c
QueueNode *nextNode = current->next;
free(current);
current = nextNode;
```

> [!tip] 记忆口诀  
> **释放先记路，清完双指针。**

# 10 | 链式队列和循环队列对比：一个靠指针，一个靠数组

> [!abstract] 核心结论  
> 循环队列用数组实现，容量固定但不需要动态内存；链式队列用链表实现，容量更灵活但要管理指针和内存。

## ① 底层原理

循环队列：

```text
数组 + front 下标 + rear 下标 + 取模
```

链式队列：

```text
节点 + next 指针 + front 指针 + rear 指针
```

循环队列像固定大小的环形座位。

链式队列像一串临时接起来的节点，需要多少就申请多少。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int front;
    int rear;
} CircleQueue;

typedef struct QueueNode {
    int data;
    struct QueueNode *next;
} QueueNode;

typedef struct {
    QueueNode *front;
    QueueNode *rear;
} LinkQueue;

int main(void) {
    CircleQueue circleQueue = {{0}, 0, 0};
    LinkQueue linkQueue = {NULL, NULL};

    printf("循环队列 front 下标: %d\n", circleQueue.front);
    printf("链式队列 front 指针: %p\n", (void *)linkQueue.front);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
CircleQueue
循环队列，使用数组保存元素。

LinkQueue
链式队列，使用节点和指针保存元素。

circleQueue.front
循环队列中的 front 是数组下标。

linkQueue.front
链式队列中的 front 是节点地址。
```

---

## ④ 输出

```text
循环队列 front 下标: 0
链式队列 front 指针: 0000000000000000
```

> [!note] 代码备注  
> 指针地址的实际显示形式和编译器、系统有关。空指针通常显示为 `0` 或一串 `0`。

---

## ⑤ 重点 / 注意 / 常见错误

|对比项|循环队列|链式队列|
|---|---|---|
|底层结构|数组|链表|
|队头含义|数组下标|节点指针|
|队尾含义|数组下标|节点指针|
|空队判断|`front == rear`|`front == NULL`|
|满队判断|通常看 `(rear + 1) % MAX_SIZE == front`|通常不固定，主要看 `malloc` 是否成功|
|入队|放到 `rear` 位置|尾插新节点|
|出队|取 `front` 位置|删除队头节点|
|空间特点|容量固定|动态申请|
|主要风险|下标取模错误|空指针、断链、内存泄漏|

> [!tip] 记忆口诀  
> **循环队列看下标，链式队列看指针。**

# 11 | 链式队列主题总结：front 出队，rear 入队

> [!abstract] 核心结论  
> 链式队列的核心就是 `front`、`rear` 和 `next`。`front` 管队头出队，`rear` 管队尾入队，`next` 负责连接节点。

## ① 底层原理

链式队列要抓住四个关键词：

> - **节点**：保存数据和下一个节点地址。
>     
> - **front**：指向队头节点。
>     
> - **rear**：指向队尾节点。
>     
> - **NULL**：表示空队列或队尾结束。
>     

链式队列核心模型：

```text
front -> 节点 -> 节点 -> 节点 -> NULL
                         ↑
                        rear
```

操作规则：

```text
enqueue：队尾插入新节点
dequeue：删除队头节点
peek：读取 front->data
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct QueueNode {
    int data;
    struct QueueNode *next;
} QueueNode;

typedef struct {
    QueueNode *front;
    QueueNode *rear;
} LinkQueue;

void initQueue(LinkQueue *queue) {
    queue->front = NULL;
    queue->rear = NULL;
}

int isEmpty(LinkQueue *queue) {
    return queue->front == NULL;
}

QueueNode *createNode(int value) {
    QueueNode *newNode = (QueueNode *)malloc(sizeof(QueueNode));

    if (newNode == NULL) {
        return NULL;
    }

    newNode->data = value;
    newNode->next = NULL;

    return newNode;
}

int enqueue(LinkQueue *queue, int value) {
    QueueNode *newNode = createNode(value);

    if (newNode == NULL) {
        return 0;
    }

    if (isEmpty(queue)) {
        queue->front = newNode;
        queue->rear = newNode;
    } else {
        queue->rear->next = newNode;
        queue->rear = newNode;
    }

    return 1;
}

int dequeue(LinkQueue *queue, int *value) {
    if (isEmpty(queue)) {
        return 0;
    }

    QueueNode *oldFront = queue->front;
    *value = oldFront->data;
    queue->front = oldFront->next;

    if (queue->front == NULL) {
        queue->rear = NULL;
    }

    free(oldFront);

    return 1;
}

int peek(LinkQueue *queue, int *value) {
    if (isEmpty(queue)) {
        return 0;
    }

    *value = queue->front->data;
    return 1;
}

void printQueue(LinkQueue *queue) {
    QueueNode *current = queue->front;

    while (current != NULL) {
        printf("%d ", current->data);
        current = current->next;
    }

    printf("\n");
}

void clearQueue(LinkQueue *queue) {
    QueueNode *current = queue->front;

    while (current != NULL) {
        QueueNode *nextNode = current->next;
        free(current);
        current = nextNode;
    }

    queue->front = NULL;
    queue->rear = NULL;
}

int main(void) {
    LinkQueue queue;
    int value;

    initQueue(&queue);

    enqueue(&queue, 10);
    enqueue(&queue, 20);
    enqueue(&queue, 30);

    printQueue(&queue);

    if (peek(&queue, &value)) {
        printf("队头元素: %d\n", value);
    }

    if (dequeue(&queue, &value)) {
        printf("出队元素: %d\n", value);
    }

    printQueue(&queue);

    clearQueue(&queue);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
initQueue
初始化链式队列，把 front 和 rear 都设置为 NULL。

enqueue
入队，创建新节点并接到队尾。

dequeue
出队，删除 front 指向的队头节点。

peek
读取队头，只看不删。

printQueue
从 front 开始沿 next 遍历。

clearQueue
释放链式队列中所有动态节点。
```

---

## ④ 输出

```text
10 20 30
队头元素: 10
出队元素: 10
20 30
```

---

## ⑤ 重点 / 注意 / 常见错误

|问题|关键点|
|---|---|
|队头在哪里|`front` 指向的节点|
|队尾在哪里|`rear` 指向的节点|
|空队怎么判断|`front == NULL`|
|入队怎么做|新节点尾插|
|出队怎么做|删除队头节点|
|只剩一个节点出队后|`front` 和 `rear` 都要置空|
|是否需要满队判断|通常不用固定容量判断，但要判断 `malloc` 是否成功|
|用完怎么处理|逐个 `free` 节点|

> [!tip] 记忆口诀  
> **链式队列：front 出头，rear 接尾；空队双空，用完 free。**
