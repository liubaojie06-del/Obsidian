# 01 | 顺序栈概念：用数组实现的“后进先出”结构

> [!abstract] 核心结论  
> 顺序栈本质是用数组实现的栈结构，只能在栈顶进行插入和删除，遵循 LIFO（后进先出）原则。

### ① 底层原理

顺序栈用一段连续内存（数组）保存数据，通过一个 `top` 指针记录栈顶位置。  
所有操作都只发生在数组末端（栈顶），避免了中间移动数据。

### ② 基本语法 / 代码用法

```
#include <stdio.h>#define MAX 5int stack[MAX];int top = -1;
```

### ③ 含义 / 说明

```
stack[MAX]  -> 用数组存数据top = -1    -> 表示当前栈是空的
```

### ⑤ 重点 / 注意 / 常见错误

- 栈只能“先进后出”，不能随意访问中间元素
- top 初始必须是 -1

> [!tip] 记忆口诀  
> **先进后出，栈顶说了算**

# 02 | 顺序栈结构设计：数组 + 栈顶指针 top

> [!abstract] 核心结论  
> 顺序栈的核心就是数组存数据 + top 控制栈顶位置。

### ① 底层原理

top 实际上是“最后一个有效元素的下标”，通过它控制插入和删除位置。

### ② 基本语法 / 代码用法

```
#define MAX 5typedef struct {    int data[MAX];    int top;} Stack;
```

### ③ 含义 / 说明

```
data[MAX]  -> 存储栈元素top        -> 指向当前栈顶元素位置
```

### ⑤ 重点 / 注意 / 常见错误

- 不要把 top 当成“元素个数”
- top=0 不代表空栈

> [!tip] 记忆口诀  
> **top 指位置，不是数量**

# 03 | 初始化顺序栈：让栈从“空状态”开始

> [!abstract] 核心结论  
> 初始化就是把 top 设为 -1，表示栈中没有任何元素。

### ① 底层原理

top=-1 表示数组中没有“合法栈元素”，逻辑上为空。

### ② 基本语法 / 代码用法

```
void init(Stack *s) {    s->top = -1;}
```

### ③ 含义 / 说明

```
s->top = -1  -> 栈为空
```

### ⑤ 重点 / 注意 / 常见错误

- ❌ top = 0（错误理解为空）
- ✅ top = -1 才是空栈

> [!tip] 记忆口诀  
> **-1 才是空，一开始啥都没有**

---

# 04 | 判断空栈：top 是否回到起点

> [!abstract] 核心结论  
> 当 top == -1 时，说明栈为空。

### ① 底层原理

没有任何元素被压入栈时，top 不指向有效位置。

### ② 基本语法 / 代码用法

```
int isEmpty(Stack *s) {    return s->top == -1;}
```

### ③ 含义 / 说明

```
top == -1 -> 没有元素
```

### ⑤ 重点 / 注意 / 常见错误

- 不要用 top==0 判断空栈
- top=0 是“有一个元素”

> [!tip] 记忆口诀  
> **-1 空，0 已经有人**

---

# 05 | 判断满栈：避免数组越界

> [!abstract] 核心结论  
> 当 top == MAX-1 时，栈已满，不能再入栈。

### ① 底层原理

数组是固定长度，top 达到最大下标后无法继续扩展。

### ② 基本语法 / 代码用法

```
int isFull(Stack *s) {    return s->top == MAX - 1;}
```

### ③ 含义 / 说明

```
MAX-1 是最后一个合法下标
```

### ⑤ 重点 / 注意 / 常见错误

- ❌ 忘记判断直接 push → 覆盖内存
- ❌ 越界是 C 语言严重错误

> [!tip] 记忆口诀  
> **满栈到头，不能再加**

---

# 06 | 入栈操作 push：往栈顶压入数据

> [!abstract] 核心结论  
> push 操作先移动 top，再存入数据。

### ① 底层原理

top 先自增，再将值放入数组对应位置。

### ② 基本语法 / 代码用法

```
int push(Stack *s, int x) {    if (isFull(s)) return 0;    s->top++;    s->data[s->top] = x;    return 1;}
```

### ③ 含义 / 说明

```
top++        -> 栈顶上移data[top]=x  -> 存数据
```

### ⑤ 重点 / 注意 / 常见错误

- ❌ 先存再 top++（会覆盖）
- ❌ 不判断满栈

> [!tip] 记忆口诀  
> **先动指针，再放数据**

---

# 07 | 出栈操作 pop：从栈顶弹出元素

> [!abstract] 核心结论  
> pop 先取值，再 top--。

### ① 底层原理

删除只是逻辑删除，不清理内存，只是移动 top。

### ② 基本语法 / 代码用法

```
int pop(Stack *s, int *x) {    if (isEmpty(s)) return 0;    *x = s->data[s->top];    s->top--;    return 1;}
```

### ③ 含义 / 说明

```
先取 top 位置的值再 top--
```

### ⑤ 重点 / 注意 / 常见错误

- ❌ 先 top-- 再取值（会错位）
- ❌ 空栈直接 pop

> [!tip] 记忆口诀  
> **先拿后减，别搞反**

---

# 08 | 常见错误：新手最容易踩的坑

> [!abstract] 核心结论  
> 顺序栈错误大多来自 top 理解错误和边界判断缺失。

### ① 底层原理

错误本质是“数组越界”或“逻辑状态错误”。

### ② 基本语法 / 代码用法

```
错误1：top=0 当空栈错误2：push 不判断满栈错误3：pop 不判断空栈
```

### ⑤ 重点 / 注意 / 常见错误

- ❌ top=0 误当空
- ❌ 忽略边界条件
- ❌ 栈溢出不报错

> [!tip] 记忆口诀  
> **边界不管，程序翻车**

# 09 | 时间复杂度与特点：为什么栈很快

> [!abstract] 核心结论  
> 顺序栈所有操作时间复杂度都是 O(1)，因为只操作栈顶。

### ① 底层原理

没有遍历，没有移动大量数据，只修改 top 和一个数组位置。

### ② 基本语法 / 代码用法

```
push -> O(1)pop  -> O(1)
```

### ⑤ 重点 / 注意 / 常见错误

- 优点：速度快
- 缺点：空间固定，不能动态扩展

> [!tip] 记忆口诀  
> **只动栈顶，快到离谱**

# 10 | 主题总结：顺序栈的本质就是“数组 + top”

> [!abstract] 核心结论  
> 顺序栈用数组存数据，用 top 控制栈顶，实现高效的后进先出结构。

### ① 底层原理

通过一个变量 top 控制数组访问范围，实现“逻辑栈”。

### ② 基本语法 / 代码用法

```
数组存数据 + top 控制边界 = 顺序栈
```

### ⑤ 重点 / 注意 / 常见错误

- 核心永远是 top
- 所有操作围绕栈顶

> [!tip] 记忆口诀  
> **数组存身，top 管命**
