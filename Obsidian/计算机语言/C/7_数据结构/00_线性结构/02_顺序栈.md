# 01 | 顺序栈的基本概念：用数组实现后进先出

> [!abstract] 核心结论  
> 顺序栈就是“数组 + 栈顶下标”。数组负责保存数据，`top` 负责记录当前栈顶元素的位置，所有插入和删除都只能在栈顶进行。

## ① 底层原理

栈是一种受限制的线性结构，它只允许在同一端插入和删除元素，这一端叫做**栈顶**。

顺序栈使用数组保存元素：

```text
下标：  0    1    2    3    4
数据： 10   20   30   ?    ?
                 ↑
                top = 2
```

此时栈顶元素是 `30`。如果继续入栈，新元素会放到 `data[3]`；如果出栈，会先取出 `data[2]`。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int top;
} SqStack;

int main(void) {
    SqStack stack = {{10, 20, 30}, 2};

    printf("栈顶元素: %d\n", stack.data[stack.top]);
    printf("栈顶下标: %d\n", stack.top);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
int data[MAX_SIZE]
保存栈中元素的数组。

int top
记录当前栈顶元素的下标。

SqStack stack = {{10, 20, 30}, 2}
数组前 3 个位置有元素，当前栈顶下标是 2。
```

---

## ④ 输出

```text
栈顶元素: 30
栈顶下标: 2
```

---

## ⑤ 重点 / 注意 / 常见错误

> 顺序栈不是普通数组，而是被 `top` 限制了操作位置的数组。

|名称|含义|
|---|---|
|`data`|保存栈元素|
|`top`|保存栈顶下标|
|`MAX_SIZE`|栈的最大容量|

> [!tip] 记忆口诀  
> **数组负责装，top 指栈顶。**

# 02 | 顺序栈初始化：把 top 设置为 -1

> [!abstract] 核心结论  
> 初始化顺序栈时，最关键的是把 `top` 设置为 `-1`。`top == -1` 表示栈中没有任何有效元素。

## ① 底层原理

如果栈中有一个元素，它会放在数组下标 `0` 的位置：

```text
下标： 0
数据：10
      ↑
     top = 0
```

所以空栈不能用 `top = 0` 表示。空栈应该表示为：

```text
top = -1
```

含义是：当前没有任何合法的栈顶元素。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int top;
} SqStack;

void initStack(SqStack *stack) {
    stack->top = -1;
}

int main(void) {
    SqStack stack;

    initStack(&stack);

    printf("初始化后的 top: %d\n", stack.top);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
void initStack(SqStack *stack)
函数接收顺序栈的地址。

stack->top = -1
把栈顶下标设置为 -1，表示空栈。

initStack(&stack)
把 stack 的地址传给函数，才能修改原来的栈。
```

---

## ④ 输出

```text
初始化后的 top: -1
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
void initStack(SqStack stack) {
    stack.top = -1;
}
```

这个函数改的是副本，外面的 `stack` 不会变。

正确写法：

```c
void initStack(SqStack *stack) {
    stack->top = -1;
}
```

> [!tip] 记忆口诀  
> **空栈 top 负一，改原栈传地址。**

# 03 | 判断空栈：看 top 是否等于 -1

> [!abstract] 核心结论  
> 当 `top == -1` 时，说明顺序栈为空，不能继续出栈，也不能读取栈顶元素。

## ① 底层原理

顺序栈是否为空，只看 `top`：

```text
top = -1    空栈
top = 0     有 1 个元素
top = 1     有 2 个元素
top = 2     有 3 个元素
```

原因是 `top` 保存的是栈顶元素下标，不是元素个数。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int top;
} SqStack;

int isEmpty(SqStack *stack) {
    return stack->top == -1;
}

int main(void) {
    SqStack stack = {{0}, -1};

    if (isEmpty(&stack)) {
        printf("顺序栈为空\n");
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
isEmpty(&stack)
判断顺序栈是否为空。

stack->top == -1
如果 top 是 -1，说明没有任何有效元素。

return stack->top == -1
条件成立返回 1，不成立返回 0。
```

---

## ④ 输出

```text
顺序栈为空
```

---

## ⑤ 重点 / 注意 / 常见错误

错误理解：

```text
top == 0 表示空栈
```

正确理解：

```text
top == 0 表示下标 0 处已经有一个元素。
```

> [!tip] 记忆口诀  
> **-1 才是空，0 已经有人。**

# 04 | 判断满栈：看 top 是否等于 MAX_SIZE - 1

> [!abstract] 核心结论  
> 当 `top == MAX_SIZE - 1` 时，说明数组最后一个位置已经被占用，顺序栈已满，不能继续入栈。

## ① 底层原理

数组下标从 `0` 开始。

如果容量是 `5`，合法下标是：

```text
0  1  2  3  4
```

最后一个合法下标是：

```text
MAX_SIZE - 1
```

所以满栈状态是：

```text
top == MAX_SIZE - 1
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 3

typedef struct {
    int data[MAX_SIZE];
    int top;
} SqStack;

int isFull(SqStack *stack) {
    return stack->top == MAX_SIZE - 1;
}

int main(void) {
    SqStack stack = {{10, 20, 30}, 2};

    if (isFull(&stack)) {
        printf("顺序栈已满\n");
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
MAX_SIZE = 3
说明数组最多保存 3 个元素。

合法下标是 0、1、2。

stack->top == MAX_SIZE - 1
如果 top 等于 2，说明最后一个位置已经有元素。
```

---

## ④ 输出

```text
顺序栈已满
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
return stack->top == MAX_SIZE;
```

问题是 `MAX_SIZE` 不是合法下标。

正确写法：

```c
return stack->top == MAX_SIZE - 1;
```

> [!tip] 记忆口诀  
> **满栈看末位，最大下标减一。**

# 05 | 入栈 push：先让 top 加一，再存入新元素

> [!abstract] 核心结论  
> 入栈就是把新元素放到栈顶。正确顺序是：先判断是否满栈，再 `top++`，最后把新元素存入 `data[top]`。

## ① 底层原理

原来的栈：

```text
下标：  0    1    2    3
数据： 10   20   ?    ?
            ↑
           top = 1
```

入栈 `30`：

```text
top++
data[top] = 30
```

结果：

```text
下标：  0    1    2    3
数据： 10   20   30   ?
                 ↑
                top = 2
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int top;
} SqStack;

int isFull(SqStack *stack) {
    return stack->top == MAX_SIZE - 1;
}

int push(SqStack *stack, int value) {
    if (isFull(stack)) {
        return 0;
    }

    stack->top++;
    stack->data[stack->top] = value;

    return 1;
}

int main(void) {
    SqStack stack = {{10, 20}, 1};

    push(&stack, 30);

    printf("栈顶元素: %d\n", stack.data[stack.top]);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
push(&stack, 30)
把 30 压入栈中。

if (isFull(stack))
入栈前判断是否已经满栈。

stack->top++
让栈顶下标移动到新的位置。

stack->data[stack->top] = value
把新元素放到新的栈顶位置。
```

---

## ④ 输出

```text
栈顶元素: 30
```

---

## ⑤ 重点 / 注意 / 常见错误

> 入栈前必须判断满栈，否则可能数组越界。

错误写法：

```c
stack->data[stack->top] = value;
stack->top++;
```

这样会覆盖当前栈顶元素。

正确写法：

```c
stack->top++;
stack->data[stack->top] = value;
```

> [!tip] 记忆口诀  
> **入栈先上移，再把数据放进去。**

---

# 06 | 出栈 pop：先取出栈顶元素，再让 top 减一

> [!abstract] 核心结论  
> 出栈就是删除栈顶元素。正确顺序是：先判断是否空栈，再取出 `data[top]`，最后 `top--`。

## ① 底层原理

原来的栈：

```text
下标：  0    1    2    3
数据： 10   20   30   ?
                 ↑
                top = 2
```

出栈时，先取出 `data[2]`，再让 `top` 回到 `1`：

```text
value = data[top]
top--
```

注意：数组里的 `30` 可能还在，但它已经不属于有效栈元素。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int top;
} SqStack;

int isEmpty(SqStack *stack) {
    return stack->top == -1;
}

int pop(SqStack *stack, int *value) {
    if (isEmpty(stack)) {
        return 0;
    }

    *value = stack->data[stack->top];
    stack->top--;

    return 1;
}

int main(void) {
    SqStack stack = {{10, 20, 30}, 2};
    int value;

    if (pop(&stack, &value)) {
        printf("出栈元素: %d\n", value);
        printf("新的 top: %d\n", stack.top);
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
pop(&stack, &value)
从栈顶弹出一个元素，并把结果保存到 value 中。

if (isEmpty(stack))
出栈前判断是否空栈。

*value = stack->data[stack->top]
先取出当前栈顶元素。

stack->top--
栈顶下标向下移动一位。
```

---

## ④ 输出

```text
出栈元素: 30
新的 top: 1
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
stack->top--;
*value = stack->data[stack->top];
```

这样会取到原栈顶下面的元素。

正确写法：

```c
*value = stack->data[stack->top];
stack->top--;
```

> [!tip] 记忆口诀  
> **出栈先拿值，再把 top 降下去。**

---

# 07 | 读取栈顶 peek：只看不删

> [!abstract] 核心结论  
> 读取栈顶元素只返回 `data[top]`，不会修改 `top`。它和出栈的区别是：出栈会删除，读取栈顶不会删除。

## ① 底层原理

读取栈顶只是查看当前最上面的元素：

```text
下标：  0    1    2
数据： 10   20   30
                 ↑
                top = 2
```

读取结果是 `30`，但 `top` 仍然等于 `2`。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int top;
} SqStack;

int isEmpty(SqStack *stack) {
    return stack->top == -1;
}

int peek(SqStack *stack, int *value) {
    if (isEmpty(stack)) {
        return 0;
    }

    *value = stack->data[stack->top];
    return 1;
}

int main(void) {
    SqStack stack = {{10, 20, 30}, 2};
    int value;

    if (peek(&stack, &value)) {
        printf("栈顶元素: %d\n", value);
        printf("top 仍然是: %d\n", stack.top);
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
peek(&stack, &value)
读取栈顶元素。

*value = stack->data[stack->top]
把栈顶元素保存到 value 中。

没有 stack->top--
说明读取栈顶不会删除元素。
```

---

## ④ 输出

```text
栈顶元素: 30
top 仍然是: 2
```

---

## ⑤ 重点 / 注意 / 常见错误

|操作|是否修改 top|含义|
|---|---|---|
|`peek`|不修改|只看栈顶|
|`pop`|修改|取出并删除栈顶|
|`push`|修改|压入新元素|

> [!tip] 记忆口诀  
> **peek 只偷看，pop 才弹出。**

---

# 08 | 顺序栈遍历：从 top 向下访问

> [!abstract] 核心结论  
> 顺序栈遍历通常从 `top` 开始向下访问，因为栈顶元素是最先被出栈的元素。

## ① 底层原理

栈的逻辑顺序通常看出栈顺序：

```text
下标：  0    1    2
数据： 10   20   30
                 ↑
                top
```

如果按出栈顺序遍历，应该得到：

```text
30 20 10
```

也就是从 `top` 一直访问到 `0`。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int top;
} SqStack;

void printStack(SqStack *stack) {
    for (int i = stack->top; i >= 0; i--) {
        printf("%d ", stack->data[i]);
    }

    printf("\n");
}

int main(void) {
    SqStack stack = {{10, 20, 30}, 2};

    printStack(&stack);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
for (int i = stack->top; i >= 0; i--)
从栈顶下标开始，向栈底方向遍历。

stack->data[i]
访问当前下标对应的栈元素。

只访问 0 到 top 的范围
top 后面的数组位置不属于有效栈元素。
```

---

## ④ 输出

```text
30 20 10
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
for (int i = 0; i < MAX_SIZE; i++) {
    printf("%d ", stack.data[i]);
}
```

问题是会访问无效区域。

正确写法：

```c
for (int i = stack.top; i >= 0; i--) {
    printf("%d ", stack.data[i]);
}
```

> [!tip] 记忆口诀  
> **看栈按顶到底，别把空位当数据。**

---

# 09 | 顺序栈常见错误：空栈出栈、满栈入栈、top 理解错

> [!abstract] 核心结论  
> 顺序栈最容易错在三个地方：空栈还出栈、满栈还入栈、把 `top` 当成元素个数。

## ① 底层原理

顺序栈的安全边界全部由 `top` 决定：

```text
空栈：top == -1
满栈：top == MAX_SIZE - 1
元素个数：top + 1
```

如果忽略边界判断，就会出现：

> - **空栈出栈**：访问不存在的栈顶元素。
>     
> - **满栈入栈**：写入数组范围之外。
>     
> - **读取无效区**：把数组空位当成栈元素。
>     

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 2

typedef struct {
    int data[MAX_SIZE];
    int top;
} SqStack;

int isEmpty(SqStack *stack) {
    return stack->top == -1;
}

int isFull(SqStack *stack) {
    return stack->top == MAX_SIZE - 1;
}

int push(SqStack *stack, int value) {
    if (isFull(stack)) {
        return 0;
    }

    stack->data[++stack->top] = value;
    return 1;
}

int pop(SqStack *stack, int *value) {
    if (isEmpty(stack)) {
        return 0;
    }

    *value = stack->data[stack->top--];
    return 1;
}

int main(void) {
    SqStack stack = {{0}, -1};
    int value;

    push(&stack, 10);
    push(&stack, 20);

    if (!push(&stack, 30)) {
        printf("入栈失败：顺序栈已满\n");
    }

    pop(&stack, &value);
    pop(&stack, &value);

    if (!pop(&stack, &value)) {
        printf("出栈失败：顺序栈为空\n");
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
MAX_SIZE = 2
说明顺序栈最多保存 2 个元素。

push(&stack, 30)
第三次入栈时已经没有空间，所以失败。

第三次 pop
此时 top 已经是 -1，所以出栈失败。
```

---

## ④ 输出

```text
入栈失败：顺序栈已满
出栈失败：顺序栈为空
```

---

## ⑤ 重点 / 注意 / 常见错误

错误 1：把 `top` 当成元素个数

```text
top = 2 表示有 2 个元素
```

正确理解：

```text
top = 2 表示栈顶下标是 2，实际有 3 个元素。
```

错误 2：入栈不判断满栈

```c
stack->data[++stack->top] = value;
```

正确做法：先判断 `isFull`。

错误 3：出栈不判断空栈

```c
*value = stack->data[stack->top--];
```

正确做法：先判断 `isEmpty`。

> [!tip] 记忆口诀  
> **空前不弹，满后不压，top 是下标不是数量。**

# 10 | 顺序栈主题总结：数组存数据，top 管栈顶

> [!abstract] 核心结论  
> 顺序栈的核心就是 `data` 数组和 `top` 下标。所有操作都围绕栈顶展开，遵循后进先出原则。

## ① 底层原理

顺序栈要抓住四个关键词：

> - **数组**：保存栈中元素。
>     
> - **top**：记录栈顶元素下标。
>     
> - **LIFO**：后进先出。
>     
> - **边界判断**：入栈防满，出栈防空。
>     

顺序栈核心模型：

```text
SqStack = data 数组 + top 栈顶下标
```

空栈和满栈：

```text
空栈：top == -1
满栈：top == MAX_SIZE - 1
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int top;
} SqStack;

void initStack(SqStack *stack) {
    stack->top = -1;
}

int isEmpty(SqStack *stack) {
    return stack->top == -1;
}

int isFull(SqStack *stack) {
    return stack->top == MAX_SIZE - 1;
}

int push(SqStack *stack, int value) {
    if (isFull(stack)) {
        return 0;
    }

    stack->data[++stack->top] = value;
    return 1;
}

int pop(SqStack *stack, int *value) {
    if (isEmpty(stack)) {
        return 0;
    }

    *value = stack->data[stack->top--];
    return 1;
}

int peek(SqStack *stack, int *value) {
    if (isEmpty(stack)) {
        return 0;
    }

    *value = stack->data[stack->top];
    return 1;
}

void printStack(SqStack *stack) {
    for (int i = stack->top; i >= 0; i--) {
        printf("%d ", stack->data[i]);
    }

    printf("\n");
}

int main(void) {
    SqStack stack;
    int value;

    initStack(&stack);

    push(&stack, 10);
    push(&stack, 20);
    push(&stack, 30);

    printStack(&stack);

    if (peek(&stack, &value)) {
        printf("栈顶元素: %d\n", value);
    }

    if (pop(&stack, &value)) {
        printf("出栈元素: %d\n", value);
    }

    printStack(&stack);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
initStack
初始化顺序栈，把 top 设置为 -1。

push
入栈，先判断满栈，再把元素压入栈顶。

pop
出栈，先判断空栈，再取出栈顶元素。

peek
读取栈顶，只看不删。

printStack
从栈顶向栈底打印有效元素。
```

---

## ④ 输出

```text
30 20 10
```

## ⑤ 重点 / 注意 / 常见错误

|问题|关键点|
|---|---|
|数据放在哪里|放在 `data` 数组里|
|栈顶在哪里|看 `top`|
|空栈怎么判断|`top == -1`|
|满栈怎么判断|`top == MAX_SIZE - 1`|
|入栈顺序|先 `top++`，再存数据|
|出栈顺序|先取数据，再 `top--`|
|元素个数|`top + 1`|

> [!tip] 记忆口诀  
> **顺序栈：数组存，top 管；后进先出，防空防满。**
