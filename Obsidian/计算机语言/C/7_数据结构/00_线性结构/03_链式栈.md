# 01 | 链式栈的基本概念：用链表实现后进先出

> [!abstract] 核心结论  
> 链式栈就是“链表 + 栈顶指针”。每个节点保存数据和下一个节点地址，`top` 指向栈顶节点，入栈和出栈都在链表头部完成。

## ① 底层原理

链式栈本质上是链表形式的栈。

它不像顺序栈那样用固定数组保存数据，而是一个节点一个节点动态申请内存：

```text
top
 ↓
[30 | next] -> [20 | next] -> [10 | NULL]
```

这里：

> - `top` 指向栈顶节点。
>     
> - 栈顶节点是最先出栈的节点。
>     
> - 新元素入栈时，插入到链表头部。
>     
> - 出栈时，删除链表头部节点。
>     

链式栈仍然遵循：

```text
后进先出 LIFO
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct StackNode {
    int data;
    struct StackNode *next;
} StackNode;

typedef struct {
    StackNode *top;
} LinkStack;

int main(void) {
    LinkStack stack;
    stack.top = NULL;

    printf("链式栈初始化完成\n");

    return 0;
}
```

---

## ③ 含义 / 说明

```text
StackNode
表示链式栈中的一个节点。

int data
节点的数据域，用来保存真正的数据。

struct StackNode *next
节点的指针域，保存下一个节点的地址。

LinkStack
表示整个链式栈。

StackNode *top
栈顶指针，指向当前栈顶节点。
```

---

## ④ 输出

```text
链式栈初始化完成
```

---

## ⑤ 重点 / 注意 / 常见错误

> 链式栈的核心不是数组下标，而是栈顶指针 `top`。

|名称|含义|
|---|---|
|`data`|保存节点数据|
|`next`|连接下一个节点|
|`top`|指向栈顶节点|
|`NULL`|表示空栈或链表结束|

> [!tip] 记忆口诀  
> **节点存数据，top 指栈顶。**

# 02 | 链式栈初始化：把 top 设置为 NULL

> [!abstract] 核心结论  
> 初始化链式栈时，只需要把 `top` 设置为 `NULL`。`top == NULL` 表示栈中没有任何节点。

## ① 底层原理

空的链式栈没有栈顶节点：

```text
top -> NULL
```

只要 `top` 不指向任何有效节点，就说明栈为空。

这和顺序栈不同：

```text
顺序栈空：top == -1
链式栈空：top == NULL
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct StackNode {
    int data;
    struct StackNode *next;
} StackNode;

typedef struct {
    StackNode *top;
} LinkStack;

void initStack(LinkStack *stack) {
    stack->top = NULL;
}

int main(void) {
    LinkStack stack;

    initStack(&stack);

    if (stack.top == NULL) {
        printf("链式栈为空\n");
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
initStack(&stack)
初始化链式栈。

stack->top = NULL
让栈顶指针不指向任何节点。

stack.top == NULL
判断当前链式栈是否为空。
```

---

## ④ 输出

```text
链式栈为空
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
void initStack(LinkStack stack) {
    stack.top = NULL;
}
```

这个函数修改的是副本，外面的 `stack` 不会被初始化。

正确写法：

```c
void initStack(LinkStack *stack) {
    stack->top = NULL;
}
```

> [!tip] 记忆口诀  
> **链式栈空不看 -1，只看 top 是不是 NULL。**

---

# 03 | 判断空栈：看 top 是否等于 NULL

> [!abstract] 核心结论  
> 当 `top == NULL` 时，说明链式栈为空，不能继续出栈，也不能读取栈顶元素。

## ① 底层原理

链式栈靠 `top` 找到入口：

```text
top -> [30] -> [20] -> [10] -> NULL
```

如果 `top` 是 `NULL`：

```text
top -> NULL
```

说明没有任何节点可以访问。

这时如果写：

```c
stack->top->data
```

就是空指针访问，程序可能直接崩溃。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct StackNode {
    int data;
    struct StackNode *next;
} StackNode;

typedef struct {
    StackNode *top;
} LinkStack;

int isEmpty(LinkStack *stack) {
    return stack->top == NULL;
}

int main(void) {
    LinkStack stack = {NULL};

    if (isEmpty(&stack)) {
        printf("链式栈为空\n");
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
isEmpty(&stack)
判断链式栈是否为空。

stack->top == NULL
如果 top 没有指向任何节点，说明栈为空。

return stack->top == NULL
条件成立返回 1，不成立返回 0。
```

---

## ④ 输出

```text
链式栈为空
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
if (stack->top->data == 0) {
    printf("空栈\n");
}
```

问题是：空栈时 `stack->top` 是 `NULL`，不能继续访问 `data`。

正确写法：

```c
if (stack->top == NULL) {
    printf("空栈\n");
}
```

> [!tip] 记忆口诀  
> **访问节点前，先看指针空不空。**

# 04 | 创建新节点：malloc 申请空间，填数据，置 next

> [!abstract] 核心结论  
> 链式栈每次入栈都需要创建一个新节点。创建节点时，要先 `malloc` 申请空间，再设置 `data` 和 `next`。

## ① 底层原理

链式栈的节点通常在堆区动态创建：

```text
malloc(sizeof(StackNode))
```

申请成功后，得到一个节点地址：

```text
newNode -> [data | next]
```

然后填入数据：

```text
newNode->data = value
newNode->next = NULL
```

如果 `malloc` 失败，会返回 `NULL`，这时不能继续访问节点成员。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct StackNode {
    int data;
    struct StackNode *next;
} StackNode;

StackNode *createNode(int value) {
    StackNode *newNode = (StackNode *)malloc(sizeof(StackNode));

    if (newNode == NULL) {
        return NULL;
    }

    newNode->data = value;
    newNode->next = NULL;

    return newNode;
}

int main(void) {
    StackNode *node = createNode(10);

    if (node != NULL) {
        printf("节点数据: %d\n", node->data);
        free(node);
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
malloc(sizeof(StackNode))
申请一个栈节点大小的内存空间。

if (newNode == NULL)
判断内存申请是否失败。

newNode->data = value
保存节点数据。

newNode->next = NULL
新节点暂时不指向其他节点。

return newNode
返回新节点地址。
```

---

## ④ 输出

```text
节点数据: 10
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
StackNode *newNode = (StackNode *)malloc(sizeof(StackNode));
newNode->data = value;
```

问题是没有判断 `malloc` 是否失败。

正确写法：

```c
if (newNode == NULL) {
    return NULL;
}
```

> [!tip] 记忆口诀  
> **malloc 后先判断，再给节点填数据。**

---

# 05 | 入栈 push：新节点插到 top 前面

> [!abstract] 核心结论  
> 链式栈入栈就是链表头插。先让新节点指向原来的栈顶，再让 `top` 指向新节点。

## ① 底层原理

原来的链式栈：

```text
top
 ↓
[20] -> [10] -> NULL
```

入栈 `30` 时，要把 `30` 放到最前面：

```text
newNode -> [30]
newNode->next = top
top = newNode
```

结果：

```text
top
 ↓
[30] -> [20] -> [10] -> NULL
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct StackNode {
    int data;
    struct StackNode *next;
} StackNode;

typedef struct {
    StackNode *top;
} LinkStack;

StackNode *createNode(int value) {
    StackNode *newNode = (StackNode *)malloc(sizeof(StackNode));

    if (newNode == NULL) {
        return NULL;
    }

    newNode->data = value;
    newNode->next = NULL;

    return newNode;
}

int push(LinkStack *stack, int value) {
    StackNode *newNode = createNode(value);

    if (newNode == NULL) {
        return 0;
    }

    newNode->next = stack->top;
    stack->top = newNode;

    return 1;
}

int main(void) {
    LinkStack stack = {NULL};

    push(&stack, 10);
    push(&stack, 20);
    push(&stack, 30);

    printf("栈顶元素: %d\n", stack.top->data);

    return 0;
}
```

> [!note] 代码备注  
> 这段代码为了突出入栈操作，没有释放节点。实际完整程序中，用完链式栈后应该逐个 `free`。

---

## ③ 含义 / 说明

```text
push(&stack, 30)
把 30 压入链式栈。

newNode->next = stack->top
让新节点指向原来的栈顶节点。

stack->top = newNode
让 top 改为指向新节点。
```

---

## ④ 输出

```text
栈顶元素: 30
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
stack->top = newNode;
newNode->next = stack->top;
```

这样会让新节点指向自己，原来的栈顶节点地址丢失。

正确写法：

```c
newNode->next = stack->top;
stack->top = newNode;
```

> [!tip] 记忆口诀  
> **入栈先连旧顶，再换新顶。**

---

# 06 | 出栈 pop：删除 top 指向的节点

> [!abstract] 核心结论  
> 链式栈出栈就是删除链表头节点。先保存栈顶数据和下一个节点地址，再释放原栈顶节点，最后更新 `top`。

## ① 底层原理

原来的链式栈：

```text
top
 ↓
[30] -> [20] -> [10] -> NULL
```

出栈时，要删除 `30`：

```text
value = top->data
oldTop = top
top = top->next
free(oldTop)
```

结果：

```text
top
 ↓
[20] -> [10] -> NULL
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct StackNode {
    int data;
    struct StackNode *next;
} StackNode;

typedef struct {
    StackNode *top;
} LinkStack;

int isEmpty(LinkStack *stack) {
    return stack->top == NULL;
}

StackNode *createNode(int value) {
    StackNode *newNode = (StackNode *)malloc(sizeof(StackNode));

    if (newNode == NULL) {
        return NULL;
    }

    newNode->data = value;
    newNode->next = NULL;

    return newNode;
}

int push(LinkStack *stack, int value) {
    StackNode *newNode = createNode(value);

    if (newNode == NULL) {
        return 0;
    }

    newNode->next = stack->top;
    stack->top = newNode;

    return 1;
}

int pop(LinkStack *stack, int *value) {
    if (isEmpty(stack)) {
        return 0;
    }

    StackNode *oldTop = stack->top;
    *value = oldTop->data;
    stack->top = oldTop->next;
    free(oldTop);

    return 1;
}

int main(void) {
    LinkStack stack = {NULL};
    int value;

    push(&stack, 10);
    push(&stack, 20);
    push(&stack, 30);

    if (pop(&stack, &value)) {
        printf("出栈元素: %d\n", value);
        printf("新的栈顶: %d\n", stack.top->data);
    }

    while (pop(&stack, &value)) {
        /* 清空剩余节点 */
    }

    return 0;
}
```

> [!note] 代码备注  
> `pop` 中的 `free(oldTop);` 只能释放通过 `malloc` 创建的节点。上面代码的节点都来自 `push`，所以释放是安全的。

---

## ③ 含义 / 说明

```text
StackNode *oldTop = stack->top
保存原栈顶节点地址。

*value = oldTop->data
取出栈顶数据。

stack->top = oldTop->next
让 top 指向下一个节点。

free(oldTop)
释放原来的栈顶节点。
```

---

## ④ 输出

```text
出栈元素: 30
新的栈顶: 20
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
free(stack->top);
stack->top = stack->top->next;
```

问题是释放后又访问 `stack->top->next`，属于危险行为。

正确写法：

```c
StackNode *oldTop = stack->top;
stack->top = oldTop->next;
free(oldTop);
```

> [!tip] 记忆口诀  
> **出栈先存路，再删旧顶。**

# 07 | 读取栈顶 peek：只看 top，不删除节点

> [!abstract] 核心结论  
> 读取栈顶元素只访问 `top->data`，不会修改 `top`，也不会释放节点。

## ① 底层原理

链式栈的栈顶元素就在 `top` 指向的节点里：

```text
top
 ↓
[30] -> [20] -> [10] -> NULL
```

读取栈顶就是：

```text
value = top->data
```

读取完成后，链式栈结构不变。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct StackNode {
    int data;
    struct StackNode *next;
} StackNode;

typedef struct {
    StackNode *top;
} LinkStack;

int isEmpty(LinkStack *stack) {
    return stack->top == NULL;
}

int peek(LinkStack *stack, int *value) {
    if (isEmpty(stack)) {
        return 0;
    }

    *value = stack->top->data;
    return 1;
}

int main(void) {
    StackNode n1 = {10, NULL};
    StackNode n2 = {20, &n1};
    StackNode n3 = {30, &n2};
    LinkStack stack = {&n3};
    int value;

    if (peek(&stack, &value)) {
        printf("栈顶元素: %d\n", value);
        printf("top 仍然指向: %d\n", stack.top->data);
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
peek(&stack, &value)
读取栈顶元素。

*value = stack->top->data
把栈顶节点的数据保存到 value 中。

没有修改 stack->top
说明读取栈顶不会删除节点。
```

---

## ④ 输出

```text
栈顶元素: 30
top 仍然指向: 30
```

---

## ⑤ 重点 / 注意 / 常见错误

|操作|是否修改 top|是否释放节点|含义|
|---|---|---|---|
|`peek`|不修改|不释放|只看栈顶|
|`pop`|修改|释放|取出并删除栈顶|
|`push`|修改|申请新节点|压入新元素|

> [!tip] 记忆口诀  
> **peek 只看顶，pop 才删顶。**

---

# 08 | 链式栈遍历：从 top 沿着 next 往下走

> [!abstract] 核心结论  
> 链式栈遍历从 `top` 开始，沿着 `next` 一个节点一个节点访问，直到遇到 `NULL`。

## ① 底层原理

链式栈结构：

```text
top -> [30] -> [20] -> [10] -> NULL
```

遍历过程：

```text
current = top
访问 current->data
current = current->next
直到 current == NULL
```

因为链式栈是链表结构，所以不能用下标随机访问。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct StackNode {
    int data;
    struct StackNode *next;
} StackNode;

typedef struct {
    StackNode *top;
} LinkStack;

void printStack(LinkStack *stack) {
    StackNode *current = stack->top;

    while (current != NULL) {
        printf("%d ", current->data);
        current = current->next;
    }

    printf("\n");
}

int main(void) {
    StackNode n1 = {10, NULL};
    StackNode n2 = {20, &n1};
    StackNode n3 = {30, &n2};
    LinkStack stack = {&n3};

    printStack(&stack);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
StackNode *current = stack->top
从栈顶节点开始遍历。

while (current != NULL)
只要当前节点存在，就继续访问。

printf("%d ", current->data)
打印当前节点数据。

current = current->next
移动到下一个节点。
```

---

## ④ 输出

```text
30 20 10
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
while (current->next != NULL) {
    printf("%d ", current->data);
    current = current->next;
}
```

这样会漏掉最后一个节点。

正确写法：

```c
while (current != NULL) {
    printf("%d ", current->data);
    current = current->next;
}
```

> [!tip] 记忆口诀  
> **遍历看 current，走路靠 next。**

---

# 09 | 释放链式栈：逐个删除节点，最后 top 置空

> [!abstract] 核心结论  
> 链式栈中的节点如果来自 `malloc`，用完后必须逐个 `free`。释放前要先保存下一个节点地址，最后把 `top` 设置为 `NULL`。

## ① 底层原理

链式栈中的每个节点都可能是动态申请的：

```text
top -> [30] -> [20] -> [10] -> NULL
```

释放时不能只写：

```c
free(stack->top);
```

这样只释放了栈顶节点，后面的节点地址会丢失，造成内存泄漏。

正确思路是：

```text
保存下一个节点
释放当前节点
移动到下一个节点
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct StackNode {
    int data;
    struct StackNode *next;
} StackNode;

typedef struct {
    StackNode *top;
} LinkStack;

void clearStack(LinkStack *stack) {
    StackNode *current = stack->top;

    while (current != NULL) {
        StackNode *nextNode = current->next;
        free(current);
        current = nextNode;
    }

    stack->top = NULL;
}

int main(void) {
    LinkStack stack = {NULL};

    clearStack(&stack);

    if (stack.top == NULL) {
        printf("链式栈已清空\n");
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
StackNode *current = stack->top
从栈顶节点开始释放。

StackNode *nextNode = current->next
释放当前节点前，先保存下一个节点地址。

free(current)
释放当前节点。

current = nextNode
移动到下一个节点。

stack->top = NULL
释放完成后，让栈顶指针置空。
```

---

## ④ 输出

```text
链式栈已清空
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
while (current != NULL) {
    free(current);
    current = current->next;
}
```

问题是 `free(current)` 后，不能再安全访问 `current->next`。

正确写法：

```c
StackNode *nextNode = current->next;
free(current);
current = nextNode;
```

> [!tip] 记忆口诀  
> **释放先记路，再 free 当前户。**

# 10 | 链式栈和顺序栈对比：一个靠指针，一个靠数组

> [!abstract] 核心结论  
> 顺序栈用数组实现，容量固定但实现简单；链式栈用链表实现，容量更灵活但需要管理指针和内存。

## ① 底层原理

顺序栈：

```text
数组 + top 下标
```

链式栈：

```text
节点 + next 指针 + top 指针
```

顺序栈像固定大小的盒子，提前规定最多放多少个元素。

链式栈像一串临时接起来的节点，需要多少就申请多少。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int top;
} SqStack;

typedef struct StackNode {
    int data;
    struct StackNode *next;
} StackNode;

typedef struct {
    StackNode *top;
} LinkStack;

int main(void) {
    SqStack sqStack = {{10, 20, 30}, 2};
    LinkStack linkStack = {NULL};

    printf("顺序栈栈顶下标: %d\n", sqStack.top);
    printf("链式栈栈顶指针: %p\n", (void *)linkStack.top);

    return 0;
}
```

|对比项|顺序栈|链式栈|
|---|---|---|
|底层结构|数组|链表|
|栈顶含义|数组下标|节点指针|

|主要风险|数组越界|空指针、内存泄漏、断链|

# 11 | 链式栈主题总结：top 管入口，next 连节点

> [!abstract] 核心结论  
> 链式栈的核心就是 `top` 和 `next`。`top` 管栈顶入口，`next` 连接下面的节点，入栈头插，出栈删头。

## ① 底层原理

链式栈要抓住四个关键词：

> - **节点**：保存数据和下一个节点地址。
>     
> - **top**：指向栈顶节点。
>     
> - **next**：连接下一个节点。
>     
> - **NULL**：表示空栈或栈底结束。
>     

链式栈核心模型：

```text
top -> 节点 -> 节点 -> 节点 -> NULL
```

操作规则：

```text
push：头插新节点
pop：删除头节点
peek：读取 top->data
```

## ② 基本语法 / 代码用法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct StackNode {
    int data;
    struct StackNode *next;
} StackNode;

typedef struct {
    StackNode *top;
} LinkStack;

void initStack(LinkStack *stack) {
    stack->top = NULL;
}

int isEmpty(LinkStack *stack) {
    return stack->top == NULL;
}

StackNode *createNode(int value) {
    StackNode *newNode = (StackNode *)malloc(sizeof(StackNode));

    if (newNode == NULL) {
        return NULL;
    }

    newNode->data = value;
    newNode->next = NULL;

    return newNode;
}

int push(LinkStack *stack, int value) {
    StackNode *newNode = createNode(value);

    if (newNode == NULL) {
        return 0;
    }

    newNode->next = stack->top;
    stack->top = newNode;

    return 1;
}

int pop(LinkStack *stack, int *value) {
    if (isEmpty(stack)) {
        return 0;
    }

    StackNode *oldTop = stack->top;
    *value = oldTop->data;
    stack->top = oldTop->next;
    free(oldTop);

    return 1;
}

int peek(LinkStack *stack, int *value) {
    if (isEmpty(stack)) {
        return 0;
    }

    *value = stack->top->data;
    return 1;
}

void printStack(LinkStack *stack) {
    StackNode *current = stack->top;

    while (current != NULL) {
        printf("%d ", current->data);
        current = current->next;
    }

    printf("\n");
}

void clearStack(LinkStack *stack) {
    StackNode *current = stack->top;

    while (current != NULL) {
        StackNode *nextNode = current->next;
        free(current);
        current = nextNode;
    }

    stack->top = NULL;
}

int main(void) {
    LinkStack stack;
    int value;

    initStack(&stack);

    push(&stack, 10);
    push(&stack, 20);
    push(&stack, 30);

    printStack(&stack);

    if (peek(&stack, &value)) {
        printf("栈顶元素: %d\n", value);
    }

    if (pop(&stack, &value)) {
        printf("出栈元素: %d\n", value);
    }

    printStack(&stack);

    clearStack(&stack);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
initStack
初始化链式栈，把 top 设置为 NULL。

push
入栈，创建新节点并插到 top 前面。

pop
出栈，删除 top 指向的节点。

peek
读取栈顶，只看不删。

printStack
从 top 开始沿 next 遍历。

clearStack
释放链式栈中所有动态节点。
```

```text
30 20 10
栈顶元素: 30
出栈元素: 30
20 10
```

## ⑤ 重点 / 注意 / 常见错误

|问题|关键点|
|---|---|
|栈顶在哪里|`top` 指向的节点|
|空栈怎么判断|`top == NULL`|
|入栈怎么做|新节点头插|
|出栈怎么做|删除头节点|
|是否需要满栈判断|通常不用固定容量判断，但要判断 `malloc` 是否成功|
|用完怎么处理|逐个 `free` 节点|
|最容易错在哪里|空指针、断链、内存泄漏|

> [!tip] 记忆口诀  
> **链式栈：top 管入口，next 连下家；入栈插头，出栈删头。**
