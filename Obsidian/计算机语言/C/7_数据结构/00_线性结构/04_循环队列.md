# 01 | 循环队列的基本概念：把数组首尾连成一个圈

> [!abstract] 核心结论  
> 循环队列就是“数组 + front + rear + 取模”。它用固定数组存数据，但通过 `% MAX_SIZE` 让下标循环使用，避免普通顺序队列出队后前面空间浪费。

## ① 底层原理

普通队列遵循：

```text
先进先出 FIFO
```

也就是：先进入队列的元素，先离开队列。

如果用普通数组实现队列，出队后前面的空间会空出来：

```text
下标：  0    1    2    3    4
数据：  ?    ?   30   40   50
              ↑         ↑
            front      rear
```

这时数组前面有空位，但尾部可能已经到头了。

循环队列的思路是：

```text
下标到达数组末尾后，再回到 0 号位置继续使用。
```

核心公式：

```text
下一个位置 = (当前位置 + 1) % MAX_SIZE
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int front;
    int rear;
} CircleQueue;

int main(void) {
    CircleQueue queue = {{0}, 0, 0};

    printf("front = %d\n", queue.front);
    printf("rear = %d\n", queue.rear);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
int data[MAX_SIZE]
保存队列元素的数组。

int front
队头下标，指向当前队头元素。

int rear
队尾下标，指向下一个可插入位置。

front == rear
表示循环队列为空。
```

---

## ④ 输出

```text
front = 0
rear = 0
```

---

## ⑤ 重点 / 注意 / 常见错误

> 循环队列里，`rear` 通常不是指向最后一个元素，而是指向“下一个可插入位置”。

|名称|含义|
|---|---|
|`data`|保存队列元素|
|`front`|指向队头元素|
|`rear`|指向下一个插入位置|
|`% MAX_SIZE`|让下标循环回到开头|

> [!tip] 记忆口诀  
> **数组围成圈，front 出，rear 进。**

# 02 | 循环队列初始化：front 和 rear 都设为 0

> [!abstract] 核心结论  
> 初始化循环队列时，把 `front` 和 `rear` 都设置为 `0`。当 `front == rear` 时，表示队列为空。

## ① 底层原理

空队列没有任何有效元素。

初始化状态：

```text
下标：  0    1    2    3    4
数据：  ?    ?    ?    ?    ?
       ↑
 front/rear
```

此时：

```text
front = 0
rear = 0
```

`front` 和 `rear` 指向同一个位置，说明没有元素可出队。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int front;
    int rear;
} CircleQueue;

void initQueue(CircleQueue *queue) {
    queue->front = 0;
    queue->rear = 0;
}

int main(void) {
    CircleQueue queue;

    initQueue(&queue);

    printf("初始化完成: front=%d, rear=%d\n", queue.front, queue.rear);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
initQueue(&queue)
初始化循环队列。

queue->front = 0
队头下标从 0 开始。

queue->rear = 0
队尾插入位置也从 0 开始。
```

---

## ④ 输出

```text
初始化完成: front=0, rear=0
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
void initQueue(CircleQueue queue) {
    queue.front = 0;
    queue.rear = 0;
}
```

这个函数修改的是副本，外面的 `queue` 不会被初始化。

正确写法：

```c
void initQueue(CircleQueue *queue) {
    queue->front = 0;
    queue->rear = 0;
}
```

> [!tip] 记忆口诀  
> **初始化清双指针，改原队列传地址。**

# 03 | 判断空队列：front 等于 rear

> [!abstract] 核心结论  
> 当 `front == rear` 时，循环队列为空，不能继续出队或读取队头元素。

## ① 底层原理

循环队列中：

```text
front 指向队头元素
rear 指向下一个插入位置
```

如果二者相等：

```text
front == rear
```

说明队列中没有有效元素。

空队列示意图：

```text
下标：  0    1    2    3    4
数据：  ?    ?    ?    ?    ?
       ↑
 front/rear
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int front;
    int rear;
} CircleQueue;

int isEmpty(CircleQueue *queue) {
    return queue->front == queue->rear;
}

int main(void) {
    CircleQueue queue = {{0}, 0, 0};

    if (isEmpty(&queue)) {
        printf("循环队列为空\n");
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
isEmpty(&queue)
判断循环队列是否为空。

queue->front == queue->rear
队头位置和插入位置相同，表示没有元素。
```

---

## ④ 输出

```text
循环队列为空
```

---

## ⑤ 重点 / 注意 / 常见错误

错误理解：

```text
front == 0 就是空队列
```

正确理解：

```text
是否为空只看 front 和 rear 是否相等。
```

> [!tip] 记忆口诀  
> **首尾相遇，就是空队。**

# 04 | 判断满队列：牺牲一个数组位置

> [!abstract] 核心结论  
> 常见循环队列会故意空出一个位置，用 `(rear + 1) % MAX_SIZE == front` 判断队满。这样可以区分空队和满队。

## ① 底层原理

如果不空出一个位置，空队和满队都可能出现：

```text
front == rear
```

这样程序分不清到底是空还是满。

所以常用做法是：

```text
牺牲一个数组位置不用。
```

当 `rear` 的下一个位置就是 `front` 时，说明队列已满：

```text
(rear + 1) % MAX_SIZE == front
```

如果 `MAX_SIZE = 5`，实际最多只能存：

```text
MAX_SIZE - 1 = 4 个元素
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int front;
    int rear;
} CircleQueue;

int isFull(CircleQueue *queue) {
    return (queue->rear + 1) % MAX_SIZE == queue->front;
}

int main(void) {
    CircleQueue queue = {{0}, 0, 4};

    if (isFull(&queue)) {
        printf("循环队列已满\n");
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
(queue->rear + 1) % MAX_SIZE
计算 rear 的下一个位置。

== queue->front
如果 rear 的下一个位置追上 front，说明队列已满。

MAX_SIZE = 5
使用牺牲一个位置的写法时，最多存 4 个元素。
```

---

## ④ 输出

```text
循环队列已满
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
return queue->rear == MAX_SIZE - 1;
```

问题是循环队列的 `rear` 到末尾后还会回到 `0`，不能只看是否到数组末尾。

正确写法：

```c
return (queue->rear + 1) % MAX_SIZE == queue->front;
```

> [!tip] 记忆口诀  
> **rear 下一步追上 front，就是满。**

# 05 | 入队 enqueue：先存数据，再移动 rear

> [!abstract] 核心结论  
> 入队就是把新元素放到 `rear` 指向的位置，然后让 `rear` 移动到下一个位置。移动时必须使用 `% MAX_SIZE`。

## ① 底层原理

原来的队列：

```text
下标：  0    1    2    3    4
数据： 10   20   ?    ?    ?
       ↑         ↑
     front      rear
```

入队 `30`：

```text
data[rear] = 30
rear = (rear + 1) % MAX_SIZE
```

结果：

```text
下标：  0    1    2    3    4
数据： 10   20   30   ?    ?
       ↑              ↑
     front           rear
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int front;
    int rear;
} CircleQueue;

int isFull(CircleQueue *queue) {
    return (queue->rear + 1) % MAX_SIZE == queue->front;
}

int enqueue(CircleQueue *queue, int value) {
    if (isFull(queue)) {
        return 0;
    }

    queue->data[queue->rear] = value;
    queue->rear = (queue->rear + 1) % MAX_SIZE;

    return 1;
}

int main(void) {
    CircleQueue queue = {{0}, 0, 0};

    enqueue(&queue, 10);
    enqueue(&queue, 20);
    enqueue(&queue, 30);

    printf("rear = %d\n", queue.rear);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
enqueue(&queue, 30)
把 30 插入循环队列。

if (isFull(queue))
入队前先判断队列是否已满。

queue->data[queue->rear] = value
把新元素放到 rear 指向的位置。

queue->rear = (queue->rear + 1) % MAX_SIZE
rear 循环移动到下一个插入位置。
```

---

## ④ 输出

```text
rear = 3
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
queue->rear++;
```

问题是 `rear` 到达数组末尾后不会自动回到 `0`。

正确写法：

```c
queue->rear = (queue->rear + 1) % MAX_SIZE;
```

> [!tip] 记忆口诀  
> **入队先放 rear，再让 rear 转一圈。**

---

# 06 | 出队 dequeue：先取 front，再移动 front

> [!abstract] 核心结论  
> 出队就是取出 `front` 指向的队头元素，然后让 `front` 移动到下一个位置。移动时同样要使用 `% MAX_SIZE`。

## ① 底层原理

原来的队列：

```text
下标：  0    1    2    3    4
数据： 10   20   30   ?    ?
       ↑              ↑
     front           rear
```

出队时：

```text
value = data[front]
front = (front + 1) % MAX_SIZE
```

结果：

```text
下标：  0    1    2    3    4
数据： 10   20   30   ?    ?
            ↑         ↑
          front      rear
```

注意：`data[0]` 里的 `10` 可能还在，但它已经不属于有效队列元素。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int front;
    int rear;
} CircleQueue;

int isEmpty(CircleQueue *queue) {
    return queue->front == queue->rear;
}

int dequeue(CircleQueue *queue, int *value) {
    if (isEmpty(queue)) {
        return 0;
    }

    *value = queue->data[queue->front];
    queue->front = (queue->front + 1) % MAX_SIZE;

    return 1;
}

int main(void) {
    CircleQueue queue = {{10, 20, 30}, 0, 3};
    int value;

    if (dequeue(&queue, &value)) {
        printf("出队元素: %d\n", value);
        printf("front = %d\n", queue.front);
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
dequeue(&queue, &value)
从队头删除一个元素，并把结果保存到 value 中。

if (isEmpty(queue))
出队前先判断队列是否为空。

*value = queue->data[queue->front]
取出队头元素。

queue->front = (queue->front + 1) % MAX_SIZE
front 循环移动到下一个队头位置。
```

---

## ④ 输出

```text
出队元素: 10
front = 1
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
queue->front++;
```

问题是 `front` 到末尾后不会回到数组开头。

正确写法：

```c
queue->front = (queue->front + 1) % MAX_SIZE;
```

> [!tip] 记忆口诀  
> **出队先取 front，再让 front 转一圈。**

---

# 07 | 读取队头 peek：只看 front，不删除元素

> [!abstract] 核心结论  
> 读取队头元素只访问 `data[front]`，不会修改 `front`，也不会改变队列结构。

## ① 底层原理

队头元素永远在 `front` 指向的位置：

```text
下标：  0    1    2    3    4
数据： 10   20   30   ?    ?
       ↑              ↑
     front           rear
```

读取队头就是：

```text
value = data[front]
```

读取后，`front` 不变。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int front;
    int rear;
} CircleQueue;

int isEmpty(CircleQueue *queue) {
    return queue->front == queue->rear;
}

int peek(CircleQueue *queue, int *value) {
    if (isEmpty(queue)) {
        return 0;
    }

    *value = queue->data[queue->front];
    return 1;
}

int main(void) {
    CircleQueue queue = {{10, 20, 30}, 0, 3};
    int value;

    if (peek(&queue, &value)) {
        printf("队头元素: %d\n", value);
        printf("front 仍然是: %d\n", queue.front);
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
peek(&queue, &value)
读取队头元素。

*value = queue->data[queue->front]
把队头元素保存到 value 中。

没有移动 front
说明读取队头不会删除元素。
```

---

## ④ 输出

```text
队头元素: 10
front 仍然是: 0
```

---

## ⑤ 重点 / 注意 / 常见错误

|操作|是否修改 front|是否修改 rear|含义|
|---|---|---|---|
|`peek`|不修改|不修改|只看队头|
|`dequeue`|修改|不修改|取出并删除队头|
|`enqueue`|不修改|修改|插入队尾|

> [!tip] 记忆口诀  
> **peek 只看头，dequeue 才出头。**

# 08 | 求循环队列长度：rear 减 front 后再取模

> [!abstract] 核心结论  
> 循环队列的元素个数不能简单用 `rear - front`，应该用 `(rear - front + MAX_SIZE) % MAX_SIZE`。

## ① 底层原理

如果 `rear` 在 `front` 后面：

```text
front = 1
rear = 4
元素个数 = 4 - 1 = 3
```

但如果发生了循环：

```text
front = 3
rear = 1
```

说明队列元素跨过了数组末尾，再绕回了开头。

这时不能直接算：

```text
rear - front = 1 - 3 = -2
```

正确公式是：

```text
(rear - front + MAX_SIZE) % MAX_SIZE
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int front;
    int rear;
} CircleQueue;

int size(CircleQueue *queue) {
    return (queue->rear - queue->front + MAX_SIZE) % MAX_SIZE;
}

int main(void) {
    CircleQueue queue = {{0}, 3, 1};

    printf("当前元素个数: %d\n", size(&queue));

    return 0;
}
```

---

## ③ 含义 / 说明

```text
queue->rear - queue->front
计算 rear 和 front 的距离。

+ MAX_SIZE
避免出现负数。

% MAX_SIZE
把结果限制在循环队列范围内。
```

---

## ④ 输出

```text
当前元素个数: 3
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
return queue->rear - queue->front;
```

当 `rear < front` 时，这个结果会变成负数。

正确写法：

```c
return (queue->rear - queue->front + MAX_SIZE) % MAX_SIZE;
```

> [!tip] 记忆口诀  
> **长度怕绕圈，先加容量再取模。**

# 09 | 循环队列遍历：从 front 开始走 size 步

> [!abstract] 核心结论  
> 循环队列遍历时，从 `front` 开始，每次用 `(index + 1) % MAX_SIZE` 前进，一共访问 `size` 个有效元素。

## ① 底层原理

循环队列可能出现“尾部和头部都有元素”的情况：

```text
下标：  0    1    2    3    4
数据： 40   ?    ?   20   30
       ↑         ↑
      rear     front
```

有效元素顺序是：

```text
20 30 40
```

所以遍历不能简单从 `front` 到 `rear - 1`，必须按照循环方式移动。

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int front;
    int rear;
} CircleQueue;

int size(CircleQueue *queue) {
    return (queue->rear - queue->front + MAX_SIZE) % MAX_SIZE;
}

void printQueue(CircleQueue *queue) {
    int count = size(queue);
    int index = queue->front;

    for (int i = 0; i < count; i++) {
        printf("%d ", queue->data[index]);
        index = (index + 1) % MAX_SIZE;
    }

    printf("\n");
}

int main(void) {
    CircleQueue queue = {{40, 0, 0, 20, 30}, 3, 1};

    printQueue(&queue);

    return 0;
}
```

---

## ③ 含义 / 说明

```text
int count = size(queue)
先计算有效元素个数。

int index = queue->front
从队头位置开始访问。

index = (index + 1) % MAX_SIZE
每次移动到下一个循环位置。
```

---

## ④ 输出

```text
20 30 40
```

---

## ⑤ 重点 / 注意 / 常见错误

错误写法：

```c
for (int i = queue->front; i < queue->rear; i++) {
    printf("%d ", queue->data[i]);
}
```

当 `rear < front` 时，这个循环无法正确遍历。

正确思路：

```text
从 front 开始，循环走 size 步。
```

> [!tip] 记忆口诀  
> **遍历不看大小关系，只从 front 走步数。**

# 10 | 循环队列常见错误：忘记取模、满队判断错、容量理解错

> [!abstract] 核心结论  
> 循环队列最容易错在三个地方：移动下标忘记取模、把 `rear == MAX_SIZE - 1` 当满队、忘记牺牲一个存储位置。

## ① 底层原理

循环队列的关键是“循环”。

只要移动 `front` 或 `rear`，都应该写：

```text
(index + 1) % MAX_SIZE
```

如果直接 `++`，下标到达末尾后就会越界。

另外，采用牺牲一个位置的写法时：

```text
数组容量是 MAX_SIZE
实际最多存 MAX_SIZE - 1 个元素
```

---

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 3

typedef struct {
    int data[MAX_SIZE];
    int front;
    int rear;
} CircleQueue;

int isEmpty(CircleQueue *queue) {
    return queue->front == queue->rear;
}

int isFull(CircleQueue *queue) {
    return (queue->rear + 1) % MAX_SIZE == queue->front;
}

int enqueue(CircleQueue *queue, int value) {
    if (isFull(queue)) {
        return 0;
    }

    queue->data[queue->rear] = value;
    queue->rear = (queue->rear + 1) % MAX_SIZE;

    return 1;
}

int main(void) {
    CircleQueue queue = {{0}, 0, 0};

    enqueue(&queue, 10);
    enqueue(&queue, 20);

    if (!enqueue(&queue, 30)) {
        printf("入队失败：循环队列已满\n");
    }

    return 0;
}
```

---

## ③ 含义 / 说明

```text
MAX_SIZE = 3
数组长度是 3。

采用牺牲一个位置的写法
最多只能存 2 个元素。

第三次 enqueue
队列已满，所以入队失败。
```

---

## ④ 输出

```text
入队失败：循环队列已满
```

---

## ⑤ 重点 / 注意 / 常见错误

错误 1：移动下标忘记取模

```c
queue->rear++;
```

正确写法：

```c
queue->rear = (queue->rear + 1) % MAX_SIZE;
```

错误 2：满队判断错

```c
queue->rear == MAX_SIZE - 1
```

正确写法：

```c
(queue->rear + 1) % MAX_SIZE == queue->front
```

错误 3：容量理解错

```text
MAX_SIZE = 5，所以一定能存 5 个元素
```

采用牺牲一个位置的写法时，实际只能存 `4` 个元素。

# 11 | 循环队列主题总结：front 出队，rear 入队，取模循环

> [!abstract] 核心结论  
> 循环队列的核心就是 `front`、`rear` 和 `% MAX_SIZE`。`front` 管出队，`rear` 管入队，取模负责让下标循环回到数组开头。

## ① 底层原理

循环队列要抓住四个关键词：

> - **front**：指向队头元素。
>     
> - **rear**：指向下一个插入位置。
>     
> - **取模**：让下标首尾循环。
>     
> - **牺牲一个位置**：区分队空和队满。
>     

核心判断：

```text
空队：front == rear
满队：(rear + 1) % MAX_SIZE == front
长度：(rear - front + MAX_SIZE) % MAX_SIZE
```

## ② 基本语法 / 代码用法

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int front;
    int rear;
} CircleQueue;

void initQueue(CircleQueue *queue) {
    queue->front = 0;
    queue->rear = 0;
}

int isEmpty(CircleQueue *queue) {
    return queue->front == queue->rear;
}

int isFull(CircleQueue *queue) {
    return (queue->rear + 1) % MAX_SIZE == queue->front;
}

int enqueue(CircleQueue *queue, int value) {
    if (isFull(queue)) {
        return 0;
    }

    queue->data[queue->rear] = value;
    queue->rear = (queue->rear + 1) % MAX_SIZE;

    return 1;
}

int dequeue(CircleQueue *queue, int *value) {
    if (isEmpty(queue)) {
        return 0;
    }

    *value = queue->data[queue->front];
    queue->front = (queue->front + 1) % MAX_SIZE;

    return 1;
}

int peek(CircleQueue *queue, int *value) {
    if (isEmpty(queue)) {
        return 0;
    }

    *value = queue->data[queue->front];
    return 1;
}

int size(CircleQueue *queue) {
    return (queue->rear - queue->front + MAX_SIZE) % MAX_SIZE;
}

void printQueue(CircleQueue *queue) {
    int count = size(queue);
    int index = queue->front;

    for (int i = 0; i < count; i++) {
        printf("%d ", queue->data[index]);
        index = (index + 1) % MAX_SIZE;
    }

    printf("\n");
}

int main(void) {
    CircleQueue queue;
    int value;

    initQueue(&queue);

    enqueue(&queue, 10);
    enqueue(&queue, 20);
    enqueue(&queue, 30);

    printQueue(&queue);

    if (peek(&queue, &value)) {
        printf("队头元素: %d\n", value);
    }

    if (dequeue(&queue, &value)) {
        printf("出队元素: %d\n", value);
    }

    printQueue(&queue);

    return 0;
}
```

## ③ 含义 / 说明

```text
initQueue
初始化循环队列，把 front 和 rear 都设为 0。

enqueue
入队，把元素放到 rear，再循环移动 rear。

dequeue
出队，取出 front，再循环移动 front。

peek
读取队头，只看不删。

size
计算当前有效元素个数。

printQueue
从 front 开始循环遍历有效元素。
```

## ⑤ 重点 / 注意 / 常见错误

|问题|关键点|
|---|---|
|队头在哪里|`front` 指向的位置|
|从哪里入队|`rear` 指向的位置|
|空队怎么判断|`front == rear`|
|满队怎么判断|`(rear + 1) % MAX_SIZE == front`|
|下标怎么移动|`(index + 1) % MAX_SIZE`|
|实际容量是多少|牺牲一个位置时是 `MAX_SIZE - 1`|
|长度怎么算|`(rear - front + MAX_SIZE) % MAX_SIZE`|
