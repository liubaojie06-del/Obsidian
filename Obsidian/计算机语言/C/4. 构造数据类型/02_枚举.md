# 01 | 枚举：给一组整数常量起名字

> [!abstract] 核心结论
> 枚举 `enum` 是 C 语言中用来定义一组命名整数常量的语法。它可以让代码从“直接写数字”变成“写有意义的名字”，常用于状态、选项、错误码、菜单编号、设备模式等场景。

### ① 底层原理

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>枚举的本质：</strong>
<ul>
  <li><strong>关键字：</strong><code>enum</code>。</li>
  <li><strong>本质：</strong>给一组整数常量起名字。</li>
  <li><strong>默认值：</strong>第一个枚举常量默认是 <code>0</code>，后面依次加 <code>1</code>。</li>
  <li><strong>可手动赋值：</strong>可以给枚举常量指定具体整数值。</li>
  <li><strong>核心作用：</strong>提高代码可读性，减少魔法数字。</li>
</ul>
</div>

### ② 代码用法

```c
enum Color {
    RED,
    GREEN,
    BLUE
};
```

默认对应：

```text
RED   = 0
GREEN = 1
BLUE  = 2
```

定义变量：

```c
enum Color color = GREEN;
```

### ③ 使用场景

| 场景 | 枚举示例 |
| :--- | :--- |
| 颜色 | `RED`、`GREEN`、`BLUE` |
| 状态 | `STOPPED`、`RUNNING`、`PAUSED` |
| 错误码 | `SUCCESS`、`ERROR`、`TIMEOUT` |
| 菜单编号 | `MENU_ADD`、`MENU_DELETE`、`MENU_EXIT` |
| 设备模式 | `MODE_AUTO`、`MODE_MANUAL` |

> [!tip] 记忆口诀
> **枚举就是给整数起名字，让数字有含义。**

# 02 | 枚举的定义和基本使用

> [!abstract] 核心结论
> 定义枚举类型只是创建一组命名整数常量。使用枚举变量时，可以让变量只表达某一组固定选项，代码更清楚。

### ① 定义枚举类型

```c
enum Week {
    MON,
    TUE,
    WED,
    THU,
    FRI,
    SAT,
    SUN
};
```

默认值：

```text
MON = 0
TUE = 1
WED = 2
THU = 3
FRI = 4
SAT = 5
SUN = 6
```

### ② 定义枚举变量

```c
enum Week today = MON;
```

含义：

```text
today 是 enum Week 类型变量。
它当前的值是 MON。
```

### ③ 完整示例

```c
#include <stdio.h>

enum Week {
    MON,
    TUE,
    WED,
    THU,
    FRI,
    SAT,
    SUN
};

int main() {
    enum Week today = FRI;

    printf("%d\n", today);

    return 0;
}
```

输出：

```text
4
```

### ④ 为什么输出 4

因为：

```text
MON 从 0 开始。
FRI 是第 5 个枚举常量。
对应整数值是 4。
```

### ⑤ 重点

```text
枚举常量本质上是整数常量。
枚举变量内部保存的也是对应整数值。
```

> [!tip] 记忆口诀
> **枚举从零排队走，后面默认加一走。**

# 03 | 手动指定枚举值

> [!abstract] 核心结论
> 枚举常量可以手动指定值。如果某个枚举常量手动赋值，后面的枚举常量默认从它的值继续递增。

### ① 手动赋值

```c
enum Status {
    SUCCESS = 0,
    ERROR = -1,
    TIMEOUT = 100
};
```

对应：

```text
SUCCESS = 0
ERROR   = -1
TIMEOUT = 100
```

### ② 中间赋值

```c
enum Code {
    A,
    B = 10,
    C,
    D
};
```

对应：

```text
A = 0
B = 10
C = 11
D = 12
```

### ③ 完整示例

```c
#include <stdio.h>

enum Code {
    A,
    B = 10,
    C,
    D
};

int main() {
    printf("A = %d\n", A);
    printf("B = %d\n", B);
    printf("C = %d\n", C);
    printf("D = %d\n", D);

    return 0;
}
```

输出：

```text
A = 0
B = 10
C = 11
D = 12
```

### ④ 可以出现相同值

```c
enum Result {
    OK = 0,
    SUCCESS = 0,
    FAIL = 1
};
```

说明：

```text
OK 和 SUCCESS 都代表 0。
这种写法可以用于别名。
```

### ⑤ 使用场景

```text
状态码要和协议一致。
错误码要和系统规范一致。
菜单编号要从 1 开始。
某些值需要特殊含义。
```

> [!tip] 记忆口诀
> **手动赋值定起点，后面默认接着加。**

# 04 | 枚举和 switch 搭配

> [!abstract] 核心结论
> 枚举非常适合和 `switch` 搭配使用。枚举负责表达有限状态，`switch` 根据不同状态执行不同逻辑。

### ① 定义枚举

```c
enum State {
    STATE_IDLE,
    STATE_RUNNING,
    STATE_ERROR
};
```

### ② switch 使用

```c
switch (state) {
    case STATE_IDLE:
        printf("空闲状态\n");
        break;
    case STATE_RUNNING:
        printf("运行状态\n");
        break;
    case STATE_ERROR:
        printf("错误状态\n");
        break;
    default:
        printf("未知状态\n");
        break;
}
```

### ③ 完整示例

```c
#include <stdio.h>

enum State {
    STATE_IDLE,
    STATE_RUNNING,
    STATE_ERROR
};

int main() {
    enum State state = STATE_RUNNING;

    switch (state) {
        case STATE_IDLE:
            printf("空闲状态\n");
            break;
        case STATE_RUNNING:
            printf("运行状态\n");
            break;
        case STATE_ERROR:
            printf("错误状态\n");
            break;
        default:
            printf("未知状态\n");
            break;
    }

    return 0;
}
```

输出：

```text
运行状态
```

### ④ 好处

```text
比直接写 0、1、2 更清楚。
状态含义一眼能看懂。
后期维护更方便。
```

### ⑤ 对比

不推荐：

```c
if (state == 1) {
    printf("running\n");
}
```

推荐：

```c
if (state == STATE_RUNNING) {
    printf("running\n");
}
```

> [!tip] 记忆口诀
> **枚举管状态，switch 管分支；名字比数字更清楚。**

# 05 | typedef 简化枚举写法

> [!abstract] 核心结论
> `typedef` 可以给枚举类型起别名，避免每次定义变量时都写 `enum 类型名`。工程中常写成 `typedef enum {...} TypeName;`。

### ① 不使用 typedef

```c
enum Color {
    RED,
    GREEN,
    BLUE
};

enum Color color = RED;
```

每次定义变量都要写：

```c
enum Color
```

### ② 使用 typedef

```c
typedef enum Color {
    RED,
    GREEN,
    BLUE
} Color;
```

之后可以写：

```c
Color color = RED;
```

### ③ 常见简化写法

```c
typedef enum {
    RED,
    GREEN,
    BLUE
} Color;
```

说明：

```text
省略 enum 标签名。
直接给枚举类型起别名 Color。
```

### ④ 完整示例

```c
#include <stdio.h>

typedef enum {
    RED,
    GREEN,
    BLUE
} Color;

int main() {
    Color color = BLUE;

    printf("%d\n", color);

    return 0;
}
```

输出：

```text
2
```

### ⑤ 推荐写法

学习阶段可以写：

```c
typedef enum {
    MODE_AUTO,
    MODE_MANUAL
} Mode;
```

这样：

```text
Mode 是类型名。
MODE_AUTO 和 MODE_MANUAL 是枚举常量。
```

> [!tip] 记忆口诀
> **typedef 给 enum 起别名，定义变量少写 enum。**

# 06 | 枚举做菜单编号

> [!abstract] 核心结论
> 枚举适合给菜单选项编号。相比直接写数字，枚举名称更能表达菜单含义，后续修改编号也更方便。

### ① 菜单枚举

```c
typedef enum {
    MENU_EXIT = 0,
    MENU_ADD = 1,
    MENU_DELETE = 2,
    MENU_FIND = 3,
    MENU_PRINT = 4
} MenuChoice;
```

### ② 使用 switch

```c
switch (choice) {
    case MENU_ADD:
        printf("添加\n");
        break;
    case MENU_DELETE:
        printf("删除\n");
        break;
    case MENU_FIND:
        printf("查找\n");
        break;
    case MENU_PRINT:
        printf("显示\n");
        break;
    case MENU_EXIT:
        printf("退出\n");
        break;
    default:
        printf("无效选择\n");
        break;
}
```

### ③ 完整示例

```c
#include <stdio.h>

typedef enum {
    MENU_EXIT = 0,
    MENU_ADD = 1,
    MENU_DELETE = 2,
    MENU_FIND = 3,
    MENU_PRINT = 4
} MenuChoice;

int main() {
    int choice = MENU_ADD;

    switch (choice) {
        case MENU_ADD:
            printf("添加学生\n");
            break;
        case MENU_DELETE:
            printf("删除学生\n");
            break;
        case MENU_FIND:
            printf("查找学生\n");
            break;
        case MENU_PRINT:
            printf("显示学生\n");
            break;
        case MENU_EXIT:
            printf("退出程序\n");
            break;
        default:
            printf("无效选择\n");
            break;
    }

    return 0;
}
```

输出：

```text
添加学生
```

### ④ 好处

```text
菜单编号集中管理。
case 分支含义清楚。
避免魔法数字。
```

### ⑤ 重点

```text
枚举常量可以直接当整数用。
菜单输入通常仍然用 int 接收，再和枚举常量比较。
```

> [!tip] 记忆口诀
> **菜单编号用枚举，数字背后有名字。**

# 07 | 枚举做状态机

> [!abstract] 核心结论
> 枚举非常适合描述程序状态。状态机中，每个状态用枚举常量表示，程序根据当前状态执行不同逻辑，并切换到下一个状态。

### ① 状态枚举

```c
typedef enum {
    STATE_INIT,
    STATE_IDLE,
    STATE_RUNNING,
    STATE_ERROR,
    STATE_EXIT
} AppState;
```

### ② 状态切换

```c
AppState state = STATE_INIT;

switch (state) {
    case STATE_INIT:
        state = STATE_IDLE;
        break;
    case STATE_IDLE:
        state = STATE_RUNNING;
        break;
    case STATE_RUNNING:
        state = STATE_EXIT;
        break;
    case STATE_ERROR:
        state = STATE_EXIT;
        break;
    case STATE_EXIT:
        break;
}
```

### ③ 完整示例

```c
#include <stdio.h>

typedef enum {
    STATE_INIT,
    STATE_IDLE,
    STATE_RUNNING,
    STATE_ERROR,
    STATE_EXIT
} AppState;

int main() {
    AppState state = STATE_INIT;

    while (state != STATE_EXIT) {
        switch (state) {
            case STATE_INIT:
                printf("初始化\n");
                state = STATE_IDLE;
                break;
            case STATE_IDLE:
                printf("空闲\n");
                state = STATE_RUNNING;
                break;
            case STATE_RUNNING:
                printf("运行\n");
                state = STATE_EXIT;
                break;
            case STATE_ERROR:
                printf("错误\n");
                state = STATE_EXIT;
                break;
            case STATE_EXIT:
                break;
        }
    }

    printf("退出\n");

    return 0;
}
```

输出：

```text
初始化
空闲
运行
退出
```

### ④ 使用场景

```text
菜单流程。
设备运行状态。
通信协议状态。
工控流程状态。
游戏状态。
任务调度状态。
```

### ⑤ 重点

```text
枚举负责定义状态集合。
switch 负责根据状态执行动作。
state 变量负责保存当前状态。
```

> [!tip] 记忆口诀
> **枚举列状态，switch 跑流程，state 记当前。**

# 08 | 枚举和宏定义的区别

> [!abstract] 核心结论
> 枚举和宏都能表示常量，但宏是预处理文本替换，枚举是语言层面的命名整数常量。枚举更适合一组相关整数常量，宏更适合编译开关、字符串、表达式宏等。

### ① 宏定义

```c
#define RED 0
#define GREEN 1
#define BLUE 2
```

特点：

```text
预处理阶段文本替换。
没有类型概念。
宏名全局替换，容易冲突。
```

### ② 枚举

```c
enum Color {
    RED,
    GREEN,
    BLUE
};
```

特点：

```text
语言层面的命名整数常量。
适合表达一组相关选项。
代码结构更清楚。
```

### ③ 对比表

| 对比项 | 宏定义 | 枚举 |
| :--- | :--- | :--- |
| 本质 | 文本替换 | 命名整数常量 |
| 处理阶段 | 预处理阶段 | 编译阶段 |
| 是否适合一组状态 | 一般 | 很适合 |
| 是否有类型语义 | 弱 | 更强 |
| 常见用途 | 编译开关、常量、宏函数 | 状态、选项、错误码 |

### ④ 推荐场景

用枚举：

```text
状态集合。
菜单选项。
错误码。
模式编号。
协议命令类型。
```

用宏：

```text
头文件保护。
条件编译。
字符串常量。
函数宏。
编译期开关。
```

### ⑤ 重点

```text
一组相关整数常量，优先考虑 enum。
```

> [!tip] 记忆口诀
> **一组整数用 enum，编译开关用 define。**

# 09 | 枚举常见错误

> [!abstract] 核心结论
> 枚举常见错误包括以为枚举只能从 1 开始、忘记默认从 0 开始、枚举名冲突、把枚举当成强类型限制、没有处理 default 分支。

### ① 错误 1：以为默认从 1 开始

```c
enum Color {
    RED,
    GREEN,
    BLUE
};
```

实际：

```text
RED = 0
GREEN = 1
BLUE = 2
```

### ② 错误 2：枚举常量名冲突

```c
enum Color {
    RED,
    GREEN
};

enum Light {
    RED,
    YELLOW
};
```

问题：

```text
同一作用域中，RED 重复定义。
```

可以加前缀：

```c
COLOR_RED
LIGHT_RED
```

### ③ 错误 3：以为枚举变量绝对不能赋其他整数

```c
enum Color color;

color = 100;
```

在 C 中通常是允许的，但业务上可能无效。

所以使用时要注意：

```text
枚举提高可读性，但不是完全强类型保护。
```

### ④ 错误 4：switch 没有 default

```c
switch (state) {
    case STATE_INIT:
        break;
    case STATE_RUNNING:
        break;
}
```

如果出现未知值：

```text
没有兜底处理。
```

更推荐：

```c
default:
    printf("未知状态\n");
    break;
```

### ⑤ 错误 5：枚举值重复但忘记说明

```c
enum Result {
    OK = 0,
    SUCCESS = 0
};
```

这可以用，但要明确：

```text
OK 和 SUCCESS 是同一个值。
```

> [!tip] 记忆口诀
> **枚举默认从零起，名字同域不能重；switch 分支加 default，更稳。**
