# 01 | 联合体：多个成员共用同一块内存

> [!abstract] 核心结论
> 联合体 `union` 是 C 语言中的一种复合数据类型。它和结构体很像，都可以包含多个成员，但结构体的成员各自占用内存，而联合体的所有成员共用同一块内存。

### ① 底层原理

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>联合体的本质：</strong>
<ul>
  <li><strong>关键字：</strong><code>union</code>。</li>
  <li><strong>共用内存：</strong>所有成员从同一个起始地址开始存放。</li>
  <li><strong>大小规则：</strong>联合体大小至少等于最大成员的大小，并可能受内存对齐影响。</li>
  <li><strong>一次主要存一个值：</strong>同一时间通常只应该使用最后一次写入的那个成员。</li>
  <li><strong>核心用途：</strong>节省内存、表示多种可能类型、解析底层数据、协议数据共用存储。</li>
</ul>
</div>

### ② 代码用法

```c
union Data {
    int i;
    float f;
    char c;
};
```

定义变量：

```c
union Data data;
```

访问成员：

```c
data.i = 100;
printf("%d\n", data.i);
```

### ③ 使用场景

| 场景 | 作用 |
| :--- | :--- |
| 节省内存 | 多个成员共用一块空间 |
| 协议解析 | 同一段数据按不同类型解释 |
| 嵌入式开发 | 访问寄存器、字节、位段 |
| 变体数据 | 一个变量可能保存不同类型 |
| 底层调试 | 观察数据在内存中的字节表现 |

> [!tip] 记忆口诀
> **结构体成员各住各屋，联合体成员共住一屋。**

# 02 | 联合体的定义和变量创建

> [!abstract] 核心结论
> 定义联合体类型只是创建一个模板，不会立即分配数据空间。只有定义联合体变量时，才会真正分配内存。

### ① 定义联合体类型

```c
union Data {
    int i;
    float f;
    char c;
};
```

含义：

```text
union Data 是一种新类型。
它内部有 i、f、c 三个成员。
这些成员共用同一块内存。
```

注意：

```text
这里只是定义类型模板。
还没有真正创建变量。
```

### ② 定义联合体变量

```c
union Data data;
```

含义：

```text
创建一个 union Data 类型变量 data。
这时才会分配内存。
```

### ③ 定义时直接创建变量

```c
union Data {
    int i;
    float f;
    char c;
} data1, data2;
```

含义：

```text
定义 union Data 类型的同时，创建 data1 和 data2 两个变量。
```

### ④ 匿名联合体

```c
union {
    int i;
    float f;
} value;
```

说明：

```text
这个联合体没有类型名。
只能直接使用 value 这个变量。
后面不方便再创建同类型变量。
```

### ⑤ 推荐写法

初学阶段推荐：

```c
union Data {
    int i;
    float f;
    char c;
};

union Data data;
```

这样结构清楚，容易理解。

> [!tip] 记忆口诀
> **定义 union 是画模板，定义变量才占内存。**

# 03 | 联合体的内存大小

> [!abstract] 核心结论
> 联合体所有成员共用同一块内存，所以联合体大小不是所有成员大小相加，而是由最大成员决定，并且可能因为对齐规则变大。

### ① 示例代码

```c
#include <stdio.h>

union Data {
    int i;
    float f;
    char c;
};

int main() {
    printf("%zu\n", sizeof(union Data));

    return 0;
}
```

常见输出：

```text
4
```

原因：

```text
int 通常占 4 字节。
float 通常占 4 字节。
char 占 1 字节。
最大成员大小是 4。
所以 union Data 通常占 4 字节。
```

### ② 不是成员大小相加

```text
如果是结构体：
int + float + char 可能至少 9 字节，甚至因为对齐变成 12 字节。

如果是联合体：
所有成员共用内存，通常只需要最大成员大小。
```

### ③ 更大的成员

```c
union Value {
    char c;
    int i;
    double d;
};
```

常见：

```text
char   1 字节
int    4 字节
double 8 字节
```

所以：

```c
sizeof(union Value)
```

常见结果：

```text
8
```

### ④ 对齐影响

联合体大小还要满足：

```text
最大对齐要求。
```

所以有时联合体大小可能不是简单最大成员大小，而是对齐后的大小。

### ⑤ 重点

```text
联合体大小 = 能装下最大成员 + 满足对齐要求。
```

> [!tip] 记忆口诀
> **联合体不累加，看最大；还要考虑对齐。**

# 04 | 联合体成员共用同一地址

> [!abstract] 核心结论
> 联合体的所有成员从同一个地址开始存放，所以打印各成员地址时，通常会看到它们的地址相同。

### ① 示例代码

```c
#include <stdio.h>

union Data {
    int i;
    float f;
    char c;
};

int main() {
    union Data data;

    printf("&data   = %p\n", (void *)&data);
    printf("&data.i = %p\n", (void *)&data.i);
    printf("&data.f = %p\n", (void *)&data.f);
    printf("&data.c = %p\n", (void *)&data.c);

    return 0;
}
```

输出示例：

```text
&data   = 0x7ffc1234
&data.i = 0x7ffc1234
&data.f = 0x7ffc1234
&data.c = 0x7ffc1234
```

### ② 底层理解

```text
data.i、data.f、data.c 都从联合体起始地址开始。
```

内存示意：

```text
同一块 4 字节空间：

┌───────────────┐
│ i / f / c 共用 │
└───────────────┘
```

### ③ 写入一个成员

```c
data.i = 100;
```

表示：

```text
把这块共用内存按 int 方式写入。
```

再写：

```c
data.f = 3.14f;
```

表示：

```text
把同一块内存改按 float 方式写入。
原来的 int 内容被覆盖。
```

### ④ 重点

```text
联合体不是每个成员都有一份空间。
它们只是用不同类型解释同一块空间。
```

### ⑤ 和结构体对比

结构体：

```text
成员地址通常不同。
```

联合体：

```text
成员地址通常相同。
```

> [!tip] 记忆口诀
> **联合体成员同起点，写谁就覆盖这块空间。**

# 05 | 联合体一次主要使用一个成员

> [!abstract] 核心结论
> 联合体虽然可以有多个成员，但因为它们共用内存，所以一次通常只应该保存一种有效数据。最后写入哪个成员，就应该按哪个成员来读取。

### ① 示例代码

```c
#include <stdio.h>

union Data {
    int i;
    float f;
    char c;
};

int main() {
    union Data data;

    data.i = 100;
    printf("i = %d\n", data.i);

    data.f = 3.14f;
    printf("f = %f\n", data.f);

    data.c = 'A';
    printf("c = %c\n", data.c);

    return 0;
}
```

输出示例：

```text
i = 100
f = 3.140000
c = A
```

### ② 注意覆盖

执行：

```c
data.i = 100;
```

这时：

```text
data.i 是有效的。
```

再执行：

```c
data.f = 3.14f;
```

这时：

```text
data.f 是有效的。
data.i 原来的内容已经被覆盖。
```

### ③ 错误理解

```c
data.i = 100;
data.f = 3.14f;

printf("%d\n", data.i);
```

问题：

```text
最后写入的是 f。
再用 i 读取同一块内存，结果没有正常业务意义。
```

### ④ 正确理解

```text
联合体里不是同时保存 i、f、c 三个值。
它只提供一块内存，让你用不同成员名访问。
```

### ⑤ 重点

```text
最后写哪个成员，通常就读哪个成员。
```

> [!tip] 记忆口诀
> **联合体一块内存多种看法，最后写谁就按谁读。**

# 06 | 联合体和结构体的区别

> [!abstract] 核心结论
> 结构体用于同时保存多个成员，联合体用于多个成员共用一块内存。结构体大小通常接近成员大小总和，联合体大小通常由最大成员决定。

### ① 结构体

```c
struct S {
    int i;
    float f;
    char c;
};
```

含义：

```text
i、f、c 都各自拥有存储空间。
可以同时保存三个值。
```

### ② 联合体

```c
union U {
    int i;
    float f;
    char c;
};
```

含义：

```text
i、f、c 共用同一块存储空间。
通常一次只保存一个有效值。
```

### ③ 对比代码

```c
#include <stdio.h>

struct S {
    int i;
    float f;
    char c;
};

union U {
    int i;
    float f;
    char c;
};

int main() {
    printf("struct S = %zu\n", sizeof(struct S));
    printf("union U  = %zu\n", sizeof(union U));

    return 0;
}
```

常见输出：

```text
struct S = 12
union U  = 4
```

### ④ 对比表

| 对比项 | 结构体 struct | 联合体 union |
| :--- | :--- | :--- |
| 内存方式 | 每个成员独立存储 | 所有成员共用内存 |
| 是否能同时保存多个值 | 可以 | 通常不可以 |
| 大小规则 | 约等于成员大小总和 + 对齐 | 最大成员大小 + 对齐 |
| 典型用途 | 表示对象的多个属性 | 表示多种可能之一 |
| 成员地址 | 通常不同 | 通常相同 |

### ⑤ 重点

```text
结构体表示“同时拥有多个字段”。
联合体表示“同一块数据可能有多种解释”。
```

> [!tip] 记忆口诀
> **结构体是并列存，联合体是共用存。**

# 07 | 联合体配合结构体：标记联合体

> [!abstract] 核心结论
> 联合体常和结构体配合使用，通过一个类型标记字段记录当前联合体里到底保存的是哪种数据。这种写法叫标记联合体，能避免读错成员。

### ① 为什么需要标记

联合体本身不会记住：

```text
当前有效成员是谁。
```

所以我们需要额外字段记录类型。

### ② 示例代码

```c
#include <stdio.h>

typedef enum {
    TYPE_INT,
    TYPE_FLOAT,
    TYPE_CHAR
} ValueType;

typedef struct {
    ValueType type;

    union {
        int i;
        float f;
        char c;
    } data;
} Value;

void print_value(Value v) {
    switch (v.type) {
        case TYPE_INT:
            printf("int = %d\n", v.data.i);
            break;
        case TYPE_FLOAT:
            printf("float = %f\n", v.data.f);
            break;
        case TYPE_CHAR:
            printf("char = %c\n", v.data.c);
            break;
    }
}

int main() {
    Value v;

    v.type = TYPE_INT;
    v.data.i = 100;
    print_value(v);

    v.type = TYPE_FLOAT;
    v.data.f = 3.14f;
    print_value(v);

    return 0;
}
```

输出：

```text
int = 100
float = 3.140000
```

### ③ 底层理解

结构体里有两个部分：

```text
type：
说明当前 data 里面是什么类型。

data：
联合体，真正保存数据。
```

### ④ 好处

```text
避免读错成员。
可以表示多种数据类型。
适合解释器、协议字段、配置项、消息系统。
```

### ⑤ 重点

```text
联合体负责节省存储。
枚举标记负责说明当前类型。
```

> [!tip] 记忆口诀
> **union 存数据，enum 记类型；标记联合体最安全。**

# 08 | 联合体在字节解析中的应用

> [!abstract] 核心结论
> 联合体可以把同一块内存按不同方式观察，例如把一个 `int` 同时看成整数和字节数组。不过这种做法和平台字节序、类型别名规则有关，学习底层时可以理解，工程中要谨慎。

### ① 示例代码

```c
#include <stdio.h>

union IntBytes {
    int value;
    unsigned char bytes[4];
};

int main() {
    union IntBytes data;

    data.value = 0x12345678;

    printf("%x\n", data.bytes[0]);
    printf("%x\n", data.bytes[1]);
    printf("%x\n", data.bytes[2]);
    printf("%x\n", data.bytes[3]);

    return 0;
}
```

可能输出：

```text
78
56
34
12
```

### ② 为什么可能这样输出

如果机器是小端序：

```text
低地址存低字节。
```

所以：

```text
0x12345678
```

在内存中可能是：

```text
78 56 34 12
```

### ③ 这能看出什么

```text
可以观察数据的字节布局。
可以理解大小端。
可以辅助学习内存表示。
```

### ④ 注意事项

```text
不同平台结果可能不同。
不同编译器优化规则下需要谨慎。
工程中更稳妥的字节处理常用 memcpy。
```

### ⑤ 更稳妥思路

如果只是想安全复制字节：

```c
memcpy(bytes, &value, sizeof(value));
```

比直接用联合体做类型重解释更稳妥。

> [!tip] 记忆口诀
> **联合体能换角度看内存，但字节序和平台差异要小心。**

# 09 | 联合体常见错误

> [!abstract] 核心结论
> 联合体常见错误包括以为它能同时保存所有成员、读错最后写入的成员、忽略大小端差异、把联合体大小当成成员大小总和、没有类型标记。

### ① 错误 1：以为能同时保存多个值

错误理解：

```c
union Data data;

data.i = 100;
data.f = 3.14f;
data.c = 'A';
```

以为：

```text
i、f、c 都保存着有效值。
```

实际：

```text
它们共用同一块内存。
后写的会覆盖先写的。
```

### ② 错误 2：读错成员

```c
data.i = 100;
printf("%f\n", data.f);
```

问题：

```text
写的是 int。
却按 float 读取。
结果没有正常业务意义。
```

### ③ 错误 3：没有类型标记

```c
union Value {
    int i;
    float f;
};
```

如果没有额外记录当前类型：

```text
后面很容易不知道该读 i 还是 f。
```

更推荐：

```text
enum + union 组成标记联合体。
```

### ④ 错误 4：误算 sizeof

错误理解：

```text
sizeof(union Data) = sizeof(int) + sizeof(float) + sizeof(char)
```

正确：

```text
sizeof(union Data) 主要看最大成员和对齐。
```

### ⑤ 错误 5：跨平台直接依赖字节布局

```text
不同机器大小端可能不同。
不同平台对齐规则可能不同。
所以底层解析要谨慎。
```

> [!tip] 记忆口诀
> **联合体怕三错：以为能同时存，读错成员，不记当前类型。**

# 10 | 联合体总结

> [!abstract] 核心结论
> 联合体的底层核心是“多个成员共用同一块内存”。它适合表示多种可能之一的数据，能节省内存，也能用于底层数据解释。实际使用时最好配合枚举类型记录当前有效成员。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>联合体核心总结：</strong>
<ul>
  <li><strong>关键字：</strong><code>union</code>。</li>
  <li><strong>本质：</strong>多个成员共用同一块内存。</li>
  <li><strong>大小规则：</strong>由最大成员大小和对齐规则决定。</li>
  <li><strong>成员地址：</strong>所有成员通常具有相同起始地址。</li>
  <li><strong>使用规则：</strong>通常最后写哪个成员，就按哪个成员读取。</li>
  <li><strong>和结构体区别：</strong>结构体成员分别存，联合体成员共用存。</li>
  <li><strong>典型组合：</strong><code>enum</code> + <code>union</code> 组成标记联合体。</li>
  <li><strong>适合场景：</strong>节省内存、多类型数据、协议解析、嵌入式寄存器。</li>
  <li><strong>底层观察：</strong>可以观察同一数据的不同字节表现。</li>
  <li><strong>常见风险：</strong>读错成员、忽略字节序、没有类型标记。</li>
</ul>
</div>

### ② 最底层一句话

```text
联合体 = 多个成员共享同一块内存，用不同成员名以不同类型解释同一段数据。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **union 共用一块内存，大小看最大成员；后写覆盖先写，enum 标记才稳。**
