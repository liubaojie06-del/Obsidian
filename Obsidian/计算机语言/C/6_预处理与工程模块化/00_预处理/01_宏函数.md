# 01 | 宏函数：看起来像函数，本质是文本替换

> [!abstract] 核心结论
> 宏函数是带参数的宏定义，使用形式像函数调用，但它不是函数。宏函数在预处理阶段完成文本替换，不会产生普通函数调用过程，因此速度可能更快，但也更容易出现优先级、副作用和类型安全问题。

### ① 底层原理：宏函数不是函数调用

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>宏函数的本质：</strong>
<ul>
  <li><strong>定义方式：</strong> 使用 <code>#define</code> 定义，并带有参数。</li>
  <li><strong>处理阶段：</strong> 在预处理阶段展开，早于真正编译。</li>
  <li><strong>本质行为：</strong> 把宏名和参数替换成对应表达式。</li>
  <li><strong>不是函数：</strong> 没有形参压栈、返回地址、函数栈帧等普通函数调用过程。</li>
  <li><strong>主要风险：</strong> 宏只是文本替换，参数可能被重复计算，括号不当会改变运算结果。</li>
</ul>
</div>

### ② 基本语法

```c
#define 宏函数名(参数) 替换表达式
```

例如：

```c
#define SQUARE(x) ((x) * (x))
```

使用：

```c
int result = SQUARE(5);
```

预处理后类似：

```c
int result = ((5) * (5));
```

### ③ 一句话理解

```text
宏函数 = 带参数的文本替换模板。
```

> [!tip] 记忆口诀
> **宏函数像函数，实质是替换；编译前展开，不是真的调用。**

# 02 | 宏函数的基本用法

> [!abstract] 核心结论
> 宏函数可以把常用表达式封装起来，例如求平方、求最大值、求最小值、数组元素个数等。使用时看起来像函数调用，但预处理后会直接替换成表达式。

### ① 求平方

```c
#include <stdio.h>

#define SQUARE(x) ((x) * (x))

int main() {
    int a = 5;

    printf("%d\n", SQUARE(a));

    return 0;
}
```

输出：

```text
25
```

### ② 求最大值

```c
#define MAX(a, b) ((a) > (b) ? (a) : (b))
```

使用：

```c
int m = MAX(10, 20);
```

预处理后类似：

```c
int m = ((10) > (20) ? (10) : (20));
```

结果：

```text
20
```

### ③ 求最小值

```c
#define MIN(a, b) ((a) < (b) ? (a) : (b))
```

使用：

```c
int m = MIN(10, 20);
```

结果：

```text
10
```

### ④ 数组元素个数

```c
#define ARRAY_SIZE(arr) (sizeof(arr) / sizeof((arr)[0]))
```

示例：

```c
int nums[] = {1, 2, 3, 4, 5};

int len = ARRAY_SIZE(nums);
```

结果：

```text
5
```

### ⑤ 重点

```text
宏函数适合短小、简单、没有副作用风险的表达式。
```

> [!tip] 记忆口诀
> **短表达式可用宏，复杂逻辑别硬塞。**

# 03 | 宏函数为什么要大量加括号

> [!abstract] 核心结论
> 宏函数的参数和整体表达式都应该加括号。因为宏展开只是文本替换，如果不加括号，替换后的表达式可能被运算符优先级改变含义。

### ① 错误写法

```c
#define SQUARE(x) x * x
```

使用：

```c
int result = SQUARE(1 + 2);
```

预处理后：

```c
int result = 1 + 2 * 1 + 2;
```

实际计算：

```text
1 + 2 * 1 + 2 = 5
```

不是想要的：

```text
9
```

### ② 正确写法

```c
#define SQUARE(x) ((x) * (x))
```

使用：

```c
int result = SQUARE(1 + 2);
```

预处理后：

```c
int result = ((1 + 2) * (1 + 2));
```

结果：

```text
9
```

### ③ 整体表达式也要加括号

错误：

```c
#define DOUBLE(x) (x) + (x)
```

使用：

```c
int result = 10 * DOUBLE(2);
```

预处理后：

```c
int result = 10 * (2) + (2);
```

结果：

```text
22
```

### ④ 更安全写法

```c
#define DOUBLE(x) ((x) + (x))
```

预处理后：

```c
int result = 10 * ((2) + (2));
```

结果：

```text
40
```

### ⑤ 规则总结

```text
参数出现的地方加括号。
整个宏表达式也加括号。
```

> [!tip] 记忆口诀
> **宏参数套括号，整体再套一层；不怕表达式复杂，就怕优先级乱套。**

# 04 | 宏函数的重复求值问题

> [!abstract] 核心结论
> 宏函数会把实参表达式原样替换到宏体中。如果参数在宏体中出现多次，那么带有副作用的表达式也可能被执行多次。

### ① 问题示例

```c
#include <stdio.h>

#define SQUARE(x) ((x) * (x))

int main() {
    int a = 3;

    int result = SQUARE(a++);

    printf("result = %d\n", result);
    printf("a = %d\n", a);

    return 0;
}
```

预处理后类似：

```c
int result = ((a++) * (a++));
```

问题：

```text
a++ 被展开了两次。
```

结果不是简单的：

```text
3 * 3
```

而是可能变成：

```text
3 * 4
```

并且这种多次修改同一变量的表达式要避免。

### ② MAX 宏也有类似问题

```c
#define MAX(a, b) ((a) > (b) ? (a) : (b))
```

使用：

```c
int m = MAX(x++, y++);
```

展开后类似：

```c
int m = ((x++) > (y++) ? (x++) : (y++));
```

问题：

```text
x++ 或 y++ 可能被执行不止一次。
```

### ③ 普通函数不会这样

```c
int square(int x) {
    return x * x;
}
```

调用：

```c
square(a++);
```

这里：

```text
a++ 只在传参时执行一次。
函数内部使用的是形参 x 的副本。
```

### ④ 宏函数使用建议

不要传入：

```text
i++
++i
函数调用
赋值表达式
带副作用的表达式
```

推荐传入：

```text
变量
常量
简单表达式
```

### ⑤ 重点

```text
宏函数不是先计算参数再传入。
而是先把表达式复制到宏体里。
```

> [!tip] 记忆口诀
> **宏参数出现几次，实参就可能复制几次；带副作用别传宏。**

# 05 | 宏函数和普通函数的区别

> [!abstract] 核心结论
> 宏函数在预处理阶段展开，没有普通函数调用开销，但没有严格类型检查，也容易出现重复求值。普通函数更安全、更清晰，但可能存在函数调用开销。

### ① 宏函数

```c
#define ADD(a, b) ((a) + (b))
```

使用：

```c
int x = ADD(10, 20);
```

预处理后：

```c
int x = ((10) + (20));
```

### ② 普通函数

```c
int add(int a, int b) {
    return a + b;
}
```

调用：

```c
int x = add(10, 20);
```

底层会有：

```text
传参。
跳转到函数入口。
执行函数体。
返回结果。
回到调用处。
```

### ③ 对比表

| 对比项 | 宏函数 | 普通函数 |
| :--- | :--- | :--- |
| 本质 | 文本替换 | 函数调用 |
| 处理阶段 | 预处理阶段 | 编译 / 运行阶段 |
| 类型检查 | 弱 | 强 |
| 参数求值 | 可能重复 | 通常只求值一次 |
| 调用开销 | 无普通调用开销 | 有调用过程 |
| 调试 | 较困难 | 较方便 |
| 安全性 | 较低 | 较高 |

### ④ 什么时候用宏函数

```text
非常短小的表达式。
希望避免函数调用开销。
需要适配多种类型。
条件编译或编译期处理。
```

### ⑤ 什么时候用普通函数

```text
逻辑较复杂。
需要类型检查。
需要调试清晰。
参数可能有副作用。
代码可维护性优先。
```

> [!tip] 记忆口诀
> **宏函数快但险，普通函数稳但有调用；简单可宏，复杂用函数。**

# 06 | 宏函数和 inline 函数

> [!abstract] 核心结论
> `inline` 函数可以在保持类型检查和函数语义的同时，建议编译器进行内联优化。很多情况下，`static inline` 比复杂宏函数更安全。

### ① 宏函数写法

```c
#define SQUARE(x) ((x) * (x))
```

优点：

```text
简单。
没有普通函数调用开销。
可适配多种类型。
```

缺点：

```text
没有类型检查。
参数可能重复求值。
调试不方便。
```

### ② inline 函数写法

```c
static inline int square_int(int x) {
    return x * x;
}
```

使用：

```c
int result = square_int(5);
```

特点：

```text
有类型检查。
参数只求值一次。
代码像普通函数。
编译器可能直接内联展开。
```

### ③ 示例对比

```c
#define SQUARE_MACRO(x) ((x) * (x))

static inline int square_func(int x) {
    return x * x;
}
```

使用：

```c
int a = 3;

int x = SQUARE_MACRO(a++);
int y = square_func(a++);
```

区别：

```text
宏可能让 a++ 执行多次。
inline 函数只会在传参时执行一次。
```

### ④ 为什么是 static inline

在头文件中定义小函数时，常写：

```c
static inline int max_int(int a, int b) {
    return a > b ? a : b;
}
```

这样：

```text
每个源文件内部拥有自己的内联函数定义。
减少链接冲突。
```

### ⑤ 选择建议

```text
简单常量：可以用宏或 enum。
简单表达式：可以用宏，但要小心。
复杂逻辑：优先 inline 或普通函数。
类型安全重要：优先 inline。
```

> [!tip] 记忆口诀
> **宏是替换，inline 是函数；想安全又想快，优先 static inline。**

# 07 | 多语句宏函数：do while(0) 写法

> [!abstract] 核心结论
> 如果宏函数内部有多条语句，推荐使用 `do { ... } while (0)` 包裹。这样可以让宏在语法上像一条普通语句，避免 `if else` 结构出错。

### ① 错误多语句宏

```c
#define PRINT_TWO(x) printf("%d\n", x); printf("done\n")
```

使用：

```c
if (flag)
    PRINT_TWO(10);
else
    printf("else\n");
```

展开后可能导致：

```c
if (flag)
    printf("%d\n", 10);
printf("done\n");
else
    printf("else\n");
```

语法混乱。

### ② 推荐写法

```c
#define PRINT_TWO(x)              \
    do {                          \
        printf("%d\n", (x));      \
        printf("done\n");         \
    } while (0)
```

使用：

```c
if (flag)
    PRINT_TWO(10);
else
    printf("else\n");
```

展开后仍然像一条语句。

### ③ 为什么 while(0)

```text
do while(0) 只执行一次。
但语法上形成一个完整语句块。
```

### ④ 注意反斜杠

多行宏每一行末尾要写：

```c
\
```

表示：

```text
宏定义还没结束，下一行继续。
```

最后一行通常不加反斜杠。

### ⑤ 多语句宏适合场景

```text
日志宏。
错误检查宏。
资源释放宏。
调试辅助宏。
简单封装多条语句。
```

> [!tip] 记忆口诀
> **多语句宏套 do while，像单句一样安全用。**

# 08 | 宏函数中的 # 和 ##

> [!abstract] 核心结论
> 宏函数中 `#` 可以把参数变成字符串，`##` 可以把两个记号拼接成一个新记号。它们常用于日志、调试和自动生成名称。

### ① # 字符串化

```c
#define PRINT_EXPR(x) printf(#x " = %d\n", (x))
```

使用：

```c
int a = 10;
PRINT_EXPR(a + 5);
```

预处理后类似：

```c
printf("a + 5" " = %d\n", (a + 5));
```

输出：

```text
a + 5 = 15
```

### ② 完整示例

```c
#include <stdio.h>

#define PRINT_EXPR(x) printf(#x " = %d\n", (x))

int main() {
    int a = 10;

    PRINT_EXPR(a);
    PRINT_EXPR(a + 5);

    return 0;
}
```

输出：

```text
a = 10
a + 5 = 15
```

### ③ ## 记号拼接

```c
#define MAKE_VAR(name) var_##name
```

使用：

```c
int MAKE_VAR(age) = 18;
```

预处理后：

```c
int var_age = 18;
```

### ④ 完整示例

```c
#include <stdio.h>

#define MAKE_VAR(name) var_##name

int main() {
    int MAKE_VAR(score) = 100;

    printf("%d\n", var_score);

    return 0;
}
```

输出：

```text
100
```

### ⑤ 重点

```text
# 和 ## 是预处理器功能。
不是普通 C 运算符。
```

> [!tip] 记忆口诀
> **# 参数变字符串，## 记号粘一起。**

# 09 | 宏函数常见错误

> [!abstract] 核心结论
> 宏函数常见错误包括不加括号、传入带副作用表达式、把宏函数当真正函数调试、定义多语句宏不使用 `do while(0)`、宏名与变量或函数名冲突。

### ① 错误 1：参数不加括号

```c
#define SQUARE(x) x * x
```

问题：

```text
SQUARE(1 + 2) 得到 5，不是 9。
```

正确：

```c
#define SQUARE(x) ((x) * (x))
```

### ② 错误 2：传入副作用表达式

```c
int result = SQUARE(a++);
```

问题：

```text
a++ 可能执行多次。
```

### ③ 错误 3：多语句宏没包裹

```c
#define LOG(x) printf("%d\n", x); printf("done\n")
```

问题：

```text
放进 if else 时容易破坏语法结构。
```

正确：

```c
#define LOG(x)              \
    do {                    \
        printf("%d\n", x);  \
        printf("done\n");   \
    } while (0)
```

### ④ 错误 4：宏名冲突

```c
#define max 100
```

如果后面有变量或函数也叫：

```c
max
```

可能被意外替换。

### ⑤ 错误 5：用宏函数写复杂逻辑

```text
宏展开后可读性差。
调试困难。
错误提示不直观。
维护成本高。
```

复杂逻辑更推荐：

```text
普通函数。
static inline 函数。
```

> [!tip] 记忆口诀
> **宏函数三防：防优先级，防副作用，防多语句乱展开。**

# 10 | 宏函数总结

> [!abstract] 核心结论
> 宏函数的底层核心是“带参数的预处理文本替换”。它适合封装短小表达式、调试输出和条件编译辅助，但不是普通函数，使用时必须注意括号、重复求值和类型安全问题。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>宏函数核心总结：</strong>
<ul>
  <li><strong>定义：</strong> 使用 <code>#define NAME(args) body</code> 定义。</li>
  <li><strong>本质：</strong> 预处理阶段的带参数文本替换。</li>
  <li><strong>不是函数：</strong> 没有普通函数调用、栈帧和类型检查。</li>
  <li><strong>优点：</strong> 展开后没有普通函数调用开销，写法灵活。</li>
  <li><strong>缺点：</strong> 容易出现优先级问题、重复求值问题、调试困难。</li>
  <li><strong>括号规则：</strong> 参数和整体表达式都要加括号。</li>
  <li><strong>副作用风险：</strong> 不要传入 <code>i++</code>、函数调用等复杂表达式。</li>
  <li><strong>多语句宏：</strong> 推荐使用 <code>do { ... } while (0)</code> 包裹。</li>
  <li><strong>#：</strong> 参数字符串化。</li>
  <li><strong>##：</strong> 记号拼接。</li>
</ul>
</div>

### ② 最底层一句话

```text
宏函数 = 预处理器把带参数的宏展开成代码文本；它像函数，但不是函数。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **宏函数先展开，编译器后判断；参数整体加括号，副作用表达式别乱传；多语句宏用 do while。**
