# 01 | 条件编译：让代码按条件参与编译

> [!abstract] 核心结论
> 条件编译是 C 语言预处理器提供的功能，用来决定某一段代码是否进入真正的编译阶段。它常用于调试开关、平台适配、功能裁剪、头文件保护等场景。

### ① 底层原理

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>条件编译的本质：</strong>
<ul>
  <li><strong>处理阶段：</strong>发生在预处理阶段，比真正编译更早。</li>
  <li><strong>核心作用：</strong>根据宏是否定义、宏值是否满足条件，决定代码是否保留。</li>
  <li><strong>常用指令：</strong><code>#if</code>、<code>#ifdef</code>、<code>#ifndef</code>、<code>#else</code>、<code>#elif</code>、<code>#endif</code>。</li>
  <li><strong>不是运行时判断：</strong>被排除的代码不会参与编译，也不会生成机器指令。</li>
  <li><strong>典型用途：</strong>调试日志、跨平台代码、头文件保护、功能开关。</li>
</ul>
</div>

### ② 代码用法

```c
#define DEBUG

#ifdef DEBUG
printf("debug mode\n");
#endif
```

预处理理解：

```text
如果 DEBUG 已定义：
保留 printf("debug mode\n");

如果 DEBUG 没定义：
删除这段代码，不参与编译。
```

### ③ 使用场景

| 指令 | 含义 | 常见用途 |
| :--- | :--- | :--- |
| `#ifdef` | 如果宏已定义 | 调试开关 |
| `#ifndef` | 如果宏未定义 | 头文件保护 |
| `#if` | 如果表达式为真 | 根据宏值选择代码 |
| `#elif` | 否则如果 | 多条件选择 |
| `#else` | 否则 | 备用代码 |
| `#endif` | 结束条件编译 | 必须配套出现 |

> [!tip] 记忆口诀
> **条件编译看宏值，预处理先裁剪；留下才编译，删掉不运行。**

# 02 | `#ifdef`：如果宏已经定义

> [!abstract] 核心结论
> `#ifdef` 用来判断某个宏是否已经被定义。如果已经定义，就保留中间的代码；如果没有定义，就跳过中间的代码。

### ① 底层原理

```c
#ifdef DEBUG
    // 只有 DEBUG 被定义时，这里才参与编译
#endif
```

等价理解：

```text
如果 DEBUG 存在：
保留代码。

如果 DEBUG 不存在：
删除代码。
```

### ② 代码用法

```c
#include <stdio.h>

#define DEBUG

int main() {
#ifdef DEBUG
    printf("调试模式开启\n");
#endif

    printf("程序正常运行\n");

    return 0;
}
```

输出：

```text
调试模式开启
程序正常运行
```

如果去掉：

```c
#define DEBUG
```

输出：

```text
程序正常运行
```

### ③ 使用场景

```text
调试日志开关。
临时代码开关。
根据宏启用某个功能。
编译不同版本程序。
```

> [!tip] 记忆口诀
> **ifdef 看有没有，有就编译，没有就删。**

# 03 | `#ifndef`：如果宏没有定义

> [!abstract] 核心结论
> `#ifndef` 用来判断某个宏是否没有定义。如果宏不存在，就保留中间代码；如果宏已经存在，就跳过中间代码。它最常用于头文件保护。

### ① 底层原理

```c
#ifndef CONFIG_H
#define CONFIG_H

// 头文件内容

#endif
```

第一次包含：

```text
CONFIG_H 没定义。
进入代码。
定义 CONFIG_H。
```

第二次包含：

```text
CONFIG_H 已经定义。
跳过代码。
```

### ② 代码用法

```c
#include <stdio.h>

// #define DEBUG

int main() {
#ifndef DEBUG
    printf("普通模式\n");
#endif

    return 0;
}
```

输出：

```text
普通模式
```

如果打开：

```c
#define DEBUG
```

那么：

```text
#ifndef DEBUG 中的代码会被跳过。
```

### ③ 使用场景

| 场景 | 作用 |
| :--- | :--- |
| 头文件保护 | 防止重复包含 |
| 默认配置 | 没定义时使用默认值 |
| 关闭调试 | 未开启 DEBUG 时执行 |
| 兼容处理 | 如果平台未提供某宏，就自己定义 |

> [!tip] 记忆口诀
> **ifndef 看有没有，没有才进去。**

# 04 | `#if`：根据宏表达式判断

> [!abstract] 核心结论
> `#if` 可以根据预处理阶段能计算的整数常量表达式决定是否保留代码。它比 `#ifdef` 更灵活，可以判断宏值大小、版本号、功能开关等。

### ① 底层原理

```c
#define VERSION 2

#if VERSION == 1
    // 版本 1 代码
#elif VERSION == 2
    // 版本 2 代码
#else
    // 其他版本代码
#endif
```

预处理器会根据：

```text
VERSION 的值
```

决定保留哪一段。

### ② 代码用法

```c
#include <stdio.h>

#define VERSION 2

int main() {
#if VERSION == 1
    printf("版本 1\n");
#elif VERSION == 2
    printf("版本 2\n");
#else
    printf("其他版本\n");
#endif

    return 0;
}
```

输出：

```text
版本 2
```

### ③ 使用场景

```text
根据版本号编译不同代码。
根据平台宏启用不同实现。
根据功能开关裁剪模块。
根据编译配置生成不同程序。
```

### ④ 注意

`#if` 后面必须是预处理器能处理的表达式，例如：

```c
#define ENABLE 1

#if ENABLE
    // 可以
#endif
```

不能直接用运行时变量：

```c
int flag = 1;

#if flag
    // 错误，预处理阶段不知道 flag 的运行时值
#endif
```

> [!tip] 记忆口诀
> **if 是运行时判断，#if 是编译前判断；变量不能给 #if 用。**

# 05 | `#elif` 和 `#else`：多分支条件编译

> [!abstract] 核心结论
> `#elif` 和 `#else` 用于多分支条件编译。它们让预处理器从多段代码中选择一段保留下来。

### ① 底层原理

```c
#if 条件1
    // 条件1成立时保留
#elif 条件2
    // 条件1不成立，条件2成立时保留
#else
    // 前面都不成立时保留
#endif
```

只会选择：

```text
第一个满足条件的分支。
```

### ② 代码用法

```c
#include <stdio.h>

#define PLATFORM 2

int main() {
#if PLATFORM == 1
    printf("Windows 平台\n");
#elif PLATFORM == 2
    printf("Linux 平台\n");
#elif PLATFORM == 3
    printf("macOS 平台\n");
#else
    printf("未知平台\n");
#endif

    return 0;
}
```

输出：

```text
Linux 平台
```

### ③ 使用场景

| 宏值 | 编译代码 |
| :--- | :--- |
| `PLATFORM == 1` | Windows 版本 |
| `PLATFORM == 2` | Linux 版本 |
| `PLATFORM == 3` | macOS 版本 |
| 其他 | 默认版本 |

### ④ 注意

```text
#if 必须用 #endif 结束。
#elif 和 #else 不能单独存在。
```

> [!tip] 记忆口诀
> **多条件用 elif，兜底分支用 else，最后别忘 endif。**

# 06 | `defined` 运算符：在 `#if` 中判断宏是否存在

> [!abstract] 核心结论
> `defined` 是预处理表达式中的运算符，用来判断宏是否已经定义。它常和 `#if`、`#elif` 配合使用，比多个 `#ifdef` 更灵活。

### ① 底层原理

```c
#if defined(DEBUG)
    // DEBUG 已定义时保留
#endif
```

等价于：

```c
#ifdef DEBUG
    // DEBUG 已定义时保留
#endif
```

### ② 代码用法

```c
#include <stdio.h>

#define DEBUG
#define LOG_ENABLE

int main() {
#if defined(DEBUG) && defined(LOG_ENABLE)
    printf("调试日志开启\n");
#endif

    return 0;
}
```

输出：

```text
调试日志开启
```

### ③ 常见写法

```c
#if defined(_WIN32)
    // Windows
#elif defined(__linux__)
    // Linux
#elif defined(__APPLE__)
    // macOS
#else
    // Unknown
#endif
```

### ④ 使用场景

```text
多个宏组合判断。
跨平台判断。
调试开关和功能开关组合。
复杂编译条件控制。
```

### ⑤ 写法对比

| 写法 | 含义 |
| :--- | :--- |
| `#ifdef DEBUG` | 判断 DEBUG 是否定义 |
| `#if defined(DEBUG)` | 判断 DEBUG 是否定义 |
| `#if defined(A) && defined(B)` | 同时判断多个宏 |

> [!tip] 记忆口诀
> **ifdef 判断一个宏，defined 能组合多个宏。**

# 07 | 条件编译做调试开关

> [!abstract] 核心结论
> 条件编译常用于控制调试代码是否参与编译。发布版本可以关闭调试宏，让调试输出代码完全不生成。

### ① 底层原理

```c
#define DEBUG 1

#if DEBUG
#define LOG(fmt, ...) printf("[DEBUG] " fmt, ##__VA_ARGS__)
#else
#define LOG(fmt, ...)
#endif
```

如果：

```text
DEBUG 为 1
```

`LOG` 会展开成 `printf`。

如果：

```text
DEBUG 为 0
```

`LOG` 会展开成空代码。

### ② 代码用法

```c
#include <stdio.h>

#define DEBUG 1

#if DEBUG
#define LOG(fmt, ...) printf("[DEBUG] " fmt, ##__VA_ARGS__)
#else
#define LOG(fmt, ...)
#endif

int main() {
    int x = 10;

    LOG("x = %d\n", x);

    printf("main running\n");

    return 0;
}
```

输出：

```text
[DEBUG] x = 10
main running
```

如果改成：

```c
#define DEBUG 0
```

输出：

```text
main running
```

### ③ 使用场景

```text
开发版打开日志。
发布版关闭日志。
性能测试关闭调试输出。
嵌入式程序裁剪无用代码。
```

### ④ 重点

```text
条件编译关闭的代码不会进入最终编译。
这和 if (DEBUG) 不完全一样。
```

运行时判断：

```c
if (DEBUG) {
    printf("debug\n");
}
```

这段代码通常仍然会被编译器看到。

条件编译：

```c
#if DEBUG
printf("debug\n");
#endif
```

如果 DEBUG 为 0，预处理阶段就删掉。

> [!tip] 记忆口诀
> **调试用条件编译，关掉就不进程序。**

# 08 | 条件编译做平台适配

> [!abstract] 核心结论
> 不同操作系统、编译器、芯片平台可能需要不同代码。条件编译可以根据平台宏选择对应实现。

### ① 底层原理

不同平台通常会定义不同宏，例如：

```text
_WIN32      Windows 常见宏
__linux__   Linux 常见宏
__APPLE__   Apple 平台常见宏
```

可以用：

```c
#if defined(...)
```

选择平台代码。

### ② 代码用法

```c
#include <stdio.h>

int main() {
#if defined(_WIN32)
    printf("Windows 平台\n");
#elif defined(__linux__)
    printf("Linux 平台\n");
#elif defined(__APPLE__)
    printf("Apple 平台\n");
#else
    printf("未知平台\n");
#endif

    return 0;
}
```

### ③ 使用场景

| 平台差异 | 示例 |
| :--- | :--- |
| 头文件不同 | Windows 用 `windows.h`，Linux 用 `unistd.h` |
| 系统调用不同 | 睡眠、文件、网络接口 |
| 编译器差异 | GCC、MSVC、Clang |
| 芯片差异 | ARM、x86、RISC-V |

### ④ 示例：不同头文件

```c
#if defined(_WIN32)
#include <windows.h>
#else
#include <unistd.h>
#endif
```

### ⑤ 重点

```text
平台适配代码要尽量集中管理。
不要在项目里到处散落复杂条件编译。
```

> [!tip] 记忆口诀
> **平台不同代码不同，条件编译来分流。**

# 09 | 条件编译做头文件保护

> [!abstract] 核心结论
> 头文件保护是条件编译最经典的应用。它能防止同一个头文件被重复包含，避免重复定义和编译错误。

### ① 底层原理

```c
#ifndef STUDENT_H
#define STUDENT_H

// 头文件内容

#endif
```

第一次包含：

```text
STUDENT_H 未定义。
进入头文件内容。
定义 STUDENT_H。
```

第二次包含：

```text
STUDENT_H 已定义。
跳过整个头文件内容。
```

### ② 代码用法

student.h：

```c
#ifndef STUDENT_H
#define STUDENT_H

void print_student(void);

#endif
```

student.c：

```c
#include <stdio.h>
#include "student.h"

void print_student(void) {
    printf("student\n");
}
```

main.c：

```c
#include "student.h"

int main() {
    print_student();

    return 0;
}
```

### ③ 为什么需要

如果没有头文件保护，复杂项目中可能出现：

```text
同一个头文件被多次 include。
结构体重复定义。
类型重复定义。
编译报错。
```

### ④ 另一种写法

```c
#pragma once
```

作用：

```text
让头文件只被包含一次。
```

但传统通用写法仍然是：

```c
#ifndef
#define
#endif
```

### ⑤ 命名建议

头文件保护宏一般用：

```text
文件名大写 + 下划线
```

例如：

```c
STUDENT_H
CONFIG_H
MY_PROJECT_UTILS_H
```

> [!tip] 记忆口诀
> **ifndef 开门，define 上锁，endif 关门；头文件只进一次。**

# 10 | 条件编译总结

> [!abstract] 核心结论
> 条件编译的底层核心是“预处理器根据宏条件决定代码是否进入编译器”。它适合处理调试开关、平台差异、功能裁剪和头文件保护，但不应滥用，否则代码可读性会下降。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>条件编译核心总结：</strong>
<ul>
  <li><strong>本质：</strong>预处理阶段根据条件保留或删除代码。</li>
  <li><strong>不是运行时判断：</strong>被排除的代码不会参与编译。</li>
  <li><strong><code>#ifdef</code>：</strong>宏已定义时保留代码。</li>
  <li><strong><code>#ifndef</code>：</strong>宏未定义时保留代码。</li>
  <li><strong><code>#if</code>：</strong>根据宏表达式值判断。</li>
  <li><strong><code>#elif</code>：</strong>多条件分支。</li>
  <li><strong><code>#else</code>：</strong>兜底分支。</li>
  <li><strong><code>#endif</code>：</strong>结束条件编译块。</li>
  <li><strong><code>defined</code>：</strong>在 <code>#if</code> 中判断宏是否定义。</li>
  <li><strong>典型用途：</strong>调试开关、平台适配、功能裁剪、头文件保护。</li>
</ul>
</div>

### ② 最底层一句话

```text
条件编译 = 预处理器在编译前根据宏条件裁剪代码，保留的才会真正进入编译器。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **ifdef 看定义，ifndef 看未定义；if 看宏值，elif 多分支；endif 来收尾，编译前先裁剪。**
