# 01 | 不定参数宏：参数数量可以变化的宏函数

> [!abstract] 核心结论
> 不定参数宏是 C 语言宏函数的一种高级写法，允许宏接收不固定数量的参数。它常用于日志宏、调试宏、封装 `printf`、错误输出等场景。

### ① 底层原理

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>不定参数宏的本质：</strong>
<ul>
  <li><strong>定义方式：</strong>使用 <code>...</code> 表示可变参数。</li>
  <li><strong>参数接收：</strong>使用 <code>__VA_ARGS__</code> 表示传入的所有可变参数。</li>
  <li><strong>处理阶段：</strong>发生在预处理阶段，不是运行时函数调用。</li>
  <li><strong>核心作用：</strong>让宏像 <code>printf</code> 一样接收不同数量的参数。</li>
  <li><strong>典型用途：</strong>日志宏、调试宏、错误提示宏、统一输出格式。</li>
</ul>
</div>

### ② 代码用法

```c
#define LOG(fmt, ...) printf(fmt, __VA_ARGS__)
```

示例：

```c
#include <stdio.h>

#define LOG(fmt, ...) printf(fmt, __VA_ARGS__)

int main() {
    int a = 10;
    int b = 20;

    LOG("a = %d\n", a);
    LOG("a + b = %d\n", a + b);

    return 0;
}
```

输出：

```text
a = 10
a + b = 30
```

### ③ 使用场景

| 写法 | 含义 |
| :--- | :--- |
| `...` | 表示宏可以接收不定数量参数 |
| `__VA_ARGS__` | 代表传入的所有可变参数 |
| `LOG("x=%d\n", x)` | 像 printf 一样传参 |

> [!tip] 记忆口诀
> **三个点收参数，VA_ARGS 取全部；宏像 printf，参数可多可少。**

# 02 | `...` 和 `__VA_ARGS__`

> [!abstract] 核心结论
> `...` 用来声明可变参数位置，`__VA_ARGS__` 用来在宏体中展开这些可变参数。二者通常成对出现。

### ① 底层原理

```c
#define 宏名(固定参数, ...) 宏体中使用 __VA_ARGS__
```

其中：

```text
...：
表示从这里开始，后面的参数数量不固定。

__VA_ARGS__：
表示实际传入的所有可变参数。
```

### ② 代码用法

```c
#include <stdio.h>

#define PRINT_INFO(fmt, ...) printf(fmt, __VA_ARGS__)

int main() {
    PRINT_INFO("name = %s, age = %d\n", "Tom", 18);

    return 0;
}
```

输出：

```text
name = Tom, age = 18
```

### ③ 展开理解

```c
PRINT_INFO("name = %s, age = %d\n", "Tom", 18);
```

预处理后类似：

```c
printf("name = %s, age = %d\n", "Tom", 18);
```

### ④ 注意点

```text
__VA_ARGS__ 不是普通变量。
它是预处理器在宏展开时替换出来的参数文本。
```

### ⑤ 常见结构

```c
#define DEBUG(fmt, ...) printf("[DEBUG] " fmt, __VA_ARGS__)
#define ERROR(fmt, ...) fprintf(stderr, "[ERROR] " fmt, __VA_ARGS__)
```

> [!tip] 记忆口诀
> **点点点负责收，VA_ARGS 负责放。**

# 03 | 封装 printf 的日志宏

> [!abstract] 核心结论
> 不定参数宏最常见用途就是封装 `printf`，给输出统一加上日志级别、文件名、行号等信息。

### ① 底层原理

```c
#define LOG(fmt, ...) printf("[LOG] " fmt, __VA_ARGS__)
```

这里：

```text
"[LOG] " fmt：
利用字符串字面量自动拼接。

__VA_ARGS__：
展开为 printf 后面的实际参数。
```

### ② 代码用法

```c
#include <stdio.h>

#define LOG(fmt, ...) printf("[LOG] " fmt, __VA_ARGS__)

int main() {
    int id = 1001;
    int score = 95;

    LOG("id = %d\n", id);
    LOG("id = %d, score = %d\n", id, score);

    return 0;
}
```

输出：

```text
[LOG] id = 1001
[LOG] id = 1001, score = 95
```

### ③ 展开理解

```c
LOG("id = %d\n", id);
```

预处理后类似：

```c
printf("[LOG] " "id = %d\n", id);
```

相邻字符串自动拼接：

```c
printf("[LOG] id = %d\n", id);
```

### ④ 使用场景

```text
调试输出。
错误输出。
统一日志前缀。
自动附加文件名和行号。
减少重复 printf。
```

### ⑤ 重点

```text
fmt 本身通常是格式字符串。
__VA_ARGS__ 是格式字符串后面的参数。
```

> [!tip] 记忆口诀
> **日志宏套 printf，前缀固定，参数灵活。**

# 04 | 带文件名和行号的不定参数宏

> [!abstract] 核心结论
> 不定参数宏可以配合 `__FILE__` 和 `__LINE__`，自动输出日志所在的文件名和行号，非常适合调试定位。

### ① 底层原理

```c
#define DEBUG(fmt, ...) \
    printf("[DEBUG] %s:%d: " fmt, __FILE__, __LINE__, __VA_ARGS__)
```

其中：

```text
__FILE__：
当前源文件名。

__LINE__：
当前代码行号。

fmt：
用户传入的格式字符串。

__VA_ARGS__：
用户传入的格式参数。
```

### ② 代码用法

```c
#include <stdio.h>

#define DEBUG(fmt, ...) \
    printf("[DEBUG] %s:%d: " fmt, __FILE__, __LINE__, __VA_ARGS__)

int main() {
    int a = 10;
    int b = 20;

    DEBUG("a = %d\n", a);
    DEBUG("a + b = %d\n", a + b);

    return 0;
}
```

输出示例：

```text
[DEBUG] main.c:9: a = 10
[DEBUG] main.c:10: a + b = 30
```

### ③ 使用场景

| 信息 | 作用 |
| :--- | :--- |
| `__FILE__` | 定位哪个文件输出日志 |
| `__LINE__` | 定位哪一行输出日志 |
| `fmt` | 用户自定义格式 |
| `__VA_ARGS__` | 用户自定义参数 |

### ④ 重点

```text
这种宏不需要每次手动写文件名和行号。
预处理器会自动替换。
```

### ⑤ 常见用途

```text
调试程序。
定位错误。
输出变量状态。
检查函数执行路径。
```

> [!tip] 记忆口诀
> **日志加 FILE 和 LINE，错误位置自动显。**

# 05 | 没有可变参数时的逗号问题

> [!abstract] 核心结论
> 不定参数宏如果有时只传格式字符串、不传额外参数，普通 `__VA_ARGS__` 写法可能出现多余逗号问题。常见解决方式是使用 `##__VA_ARGS__` 或 C23 的 `__VA_OPT__`。

### ① 问题示例

```c
#define LOG(fmt, ...) printf(fmt, __VA_ARGS__)
```

调用：

```c
LOG("hello\n");
```

可能展开成：

```c
printf("hello\n", );
```

问题：

```text
多了一个逗号，语法错误。
```

### ② GCC 常见写法

```c
#define LOG(fmt, ...) printf(fmt, ##__VA_ARGS__)
```

这样：

```c
LOG("hello\n");
```

可以展开为：

```c
printf("hello\n");
```

而：

```c
LOG("x = %d\n", x);
```

展开为：

```c
printf("x = %d\n", x);
```

### ③ C23 写法

```c
#define LOG(fmt, ...) printf(fmt __VA_OPT__(,) __VA_ARGS__)
```

含义：

```text
如果 __VA_ARGS__ 不为空，就插入逗号。
如果 __VA_ARGS__ 为空，就不插入逗号。
```

### ④ 推荐理解

```text
##__VA_ARGS__：
常见编译器扩展写法。

__VA_OPT__：
更新标准中的更规范写法。
```

### ⑤ 初学建议

如果你使用 GCC / Clang，常见写法：

```c
#define LOG(fmt, ...) printf(fmt, ##__VA_ARGS__)
```

更适合日常学习和 Linux 编程。

> [!tip] 记忆口诀
> **可变参数可能为空，逗号要特殊处理。**

# 06 | `##__VA_ARGS__`：吞掉多余逗号

> [!abstract] 核心结论
> `##__VA_ARGS__` 是 GCC / Clang 常见扩展写法。当可变参数为空时，它可以删除前面多余的逗号，让日志宏既支持带参数，也支持不带参数。

### ① 底层原理

```c
#define LOG(fmt, ...) printf(fmt, ##__VA_ARGS__)
```

当调用：

```c
LOG("hello\n");
```

时：

```text
__VA_ARGS__ 为空。
## 会让前面的逗号被删除。
```

展开为：

```c
printf("hello\n");
```

### ② 代码用法

```c
#include <stdio.h>

#define LOG(fmt, ...) printf("[LOG] " fmt, ##__VA_ARGS__)

int main() {
    int x = 10;

    LOG("program start\n");
    LOG("x = %d\n", x);

    return 0;
}
```

输出：

```text
[LOG] program start
[LOG] x = 10
```

### ③ 展开理解

```c
LOG("program start\n");
```

展开为：

```c
printf("[LOG] " "program start\n");
```

```c
LOG("x = %d\n", x);
```

展开为：

```c
printf("[LOG] " "x = %d\n", x);
```

### ④ 注意

```text
##__VA_ARGS__ 是常见扩展。
在 GCC / Clang 中常用。
如果追求严格标准兼容，需要了解 __VA_OPT__。
```

### ⑤ 适合场景

```text
Linux C 编程。
GCC 编译环境。
Clang 编译环境。
调试日志宏。
嵌入式日志宏。
```

> [!tip] 记忆口诀
> **双井号遇空参数，前面逗号可吞掉。**

# 07 | `__VA_OPT__`：标准化处理空参数

> [!abstract] 核心结论
> `__VA_OPT__` 用于在可变参数不为空时插入指定内容。它可以优雅解决不定参数宏中的多余逗号问题，是较新的标准写法。

### ① 底层原理

```c
#define LOG(fmt, ...) printf(fmt __VA_OPT__(,) __VA_ARGS__)
```

含义：

```text
如果 __VA_ARGS__ 有内容：
__VA_OPT__(,) 展开成逗号。

如果 __VA_ARGS__ 为空：
__VA_OPT__(,) 展开为空。
```

### ② 代码用法

```c
#include <stdio.h>

#define LOG(fmt, ...) printf("[LOG] " fmt __VA_OPT__(,) __VA_ARGS__)

int main() {
    int x = 10;

    LOG("program start\n");
    LOG("x = %d\n", x);

    return 0;
}
```

输出：

```text
[LOG] program start
[LOG] x = 10
```

### ③ 展开理解

```c
LOG("program start\n");
```

展开：

```c
printf("[LOG] " "program start\n");
```

```c
LOG("x = %d\n", x);
```

展开：

```c
printf("[LOG] " "x = %d\n", x);
```

### ④ 对比 `##__VA_ARGS__`

| 写法 | 特点 |
| :--- | :--- |
| `##__VA_ARGS__` | GCC / Clang 常见扩展 |
| `__VA_OPT__` | 更新标准写法，更规范 |
| 普通 `__VA_ARGS__` | 空参数时容易多逗号 |

### ⑤ 初学建议

```text
学 Linux / GCC：
先掌握 ##__VA_ARGS__。

学标准 C / 新编译器：
再掌握 __VA_OPT__。
```

> [!tip] 记忆口诀
> **VA_OPT 看参数空不空，不空才放逗号。**

# 08 | 不定参数宏和字符串化运算符结合

> [!abstract] 核心结论
> 不定参数宏可以和字符串化运算符 `#` 配合，打印表达式文本和值，也可以生成更灵活的调试信息。

### ① 底层原理

```c
#define SHOW(expr, fmt, ...) \
    printf(#expr " = " fmt "\n", (expr), ##__VA_ARGS__)
```

这里：

```text
#expr：
把表达式变成字符串。

(expr)：
计算表达式本身。

fmt：
用户指定输出格式。

__VA_ARGS__：
额外参数。
```

### ② 简单调试宏

```c
#include <stdio.h>

#define SHOW_INT(expr) printf(#expr " = %d\n", (expr))

int main() {
    int a = 10;
    int b = 20;

    SHOW_INT(a + b);

    return 0;
}
```

输出：

```text
a + b = 30
```

### ③ 带日志前缀

```c
#include <stdio.h>

#define LOG_EXPR(expr) \
    printf("[EXPR] " #expr " = %d\n", (expr))

int main() {
    int x = 5;

    LOG_EXPR(x * x);

    return 0;
}
```

输出：

```text
[EXPR] x * x = 25
```

### ④ 使用场景

```text
调试变量。
打印表达式和值。
封装日志。
错误追踪。
自动格式化输出。
```

### ⑤ 注意

```text
表达式如果有副作用，例如 i++，宏里可能产生重复求值风险。
调试宏中也要谨慎。
```

> [!tip] 记忆口诀
> **不定参数管格式，字符串化管名字；组合起来就是调试利器。**

# 09 | 不定参数宏常见错误

> [!abstract] 核心结论
> 不定参数宏常见错误包括空参数导致多余逗号、格式字符串和参数不匹配、把宏当函数调试、复杂参数重复求值、宏体多语句没有用 `do while(0)` 包裹。

### ① 错误 1：空参数多逗号

```c
#define LOG(fmt, ...) printf(fmt, __VA_ARGS__)

LOG("hello\n");
```

可能展开：

```c
printf("hello\n", );
```

正确：

```c
#define LOG(fmt, ...) printf(fmt, ##__VA_ARGS__)
```

或：

```c
#define LOG(fmt, ...) printf(fmt __VA_OPT__(,) __VA_ARGS__)
```

### ② 错误 2：格式和参数不匹配

```c
LOG("x = %d\n", "hello");
```

问题：

```text
%d 需要 int。
但传入的是字符串地址。
```

### ③ 错误 3：把宏当函数

```text
不定参数宏不是函数。
它没有真正的参数类型检查。
错误往往在展开后的代码里体现。
```

### ④ 错误 4：宏体多语句没包裹

错误：

```c
#define LOG_AND_COUNT(fmt, ...) printf(fmt, ##__VA_ARGS__); count++
```

更安全：

```c
#define LOG_AND_COUNT(fmt, ...)        \
    do {                               \
        printf(fmt, ##__VA_ARGS__);    \
        count++;                       \
    } while (0)
```

### ⑤ 错误 5：副作用表达式

```c
#define DEBUG_VALUE(x) printf(#x " = %d\n", (x))

DEBUG_VALUE(i++);
```

虽然这里通常只计算一次，但如果宏体多次使用 `(x)`，就可能造成重复执行。

> [!tip] 记忆口诀
> **空参数防逗号，格式参数要配套；宏不是函数，复杂逻辑要小心。**

# 10 | 不定参数宏总结

> [!abstract] 核心结论
> 不定参数宏的底层核心是“用 `...` 接收可变参数，用 `__VA_ARGS__` 在宏体中展开参数文本”。它非常适合封装日志和调试输出，但要注意空参数逗号、格式匹配和宏展开副作用。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>不定参数宏核心总结：</strong>
<ul>
  <li><strong>本质：</strong>预处理阶段的可变参数文本替换。</li>
  <li><strong>定义方式：</strong><code>#define LOG(fmt, ...) ...</code>。</li>
  <li><strong>接收参数：</strong><code>...</code> 表示不定参数。</li>
  <li><strong>展开参数：</strong><code>__VA_ARGS__</code> 表示传入的所有可变参数。</li>
  <li><strong>典型用途：</strong>封装 <code>printf</code>、日志宏、调试宏、错误输出宏。</li>
  <li><strong>文件行号：</strong>可配合 <code>__FILE__</code> 和 <code>__LINE__</code> 定位输出位置。</li>
  <li><strong>空参数问题：</strong>普通 <code>__VA_ARGS__</code> 可能产生多余逗号。</li>
  <li><strong>常见解决：</strong><code>##__VA_ARGS__</code> 或 <code>__VA_OPT__</code>。</li>
  <li><strong>注意风险：</strong>格式字符串和参数必须匹配。</li>
  <li><strong>多语句宏：</strong>建议用 <code>do { ... } while (0)</code> 包裹。</li>
</ul>
</div>

### ② 最底层一句话

```text
不定参数宏 = 用 ... 收集可变参数，再用 __VA_ARGS__ 把这些参数展开到宏体里。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **三个点收参数，VA_ARGS 放参数；日志宏最常见，空参逗号要处理。**
