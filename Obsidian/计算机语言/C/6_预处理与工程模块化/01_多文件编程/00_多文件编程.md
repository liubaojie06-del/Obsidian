# 01 | 多文件编程：把一个程序拆成多个模块

> [!abstract] 核心结论
> 多文件编程就是把一个 C 程序拆成多个 `.c` 源文件和 `.h` 头文件。`.h` 负责声明接口，`.c` 负责实现功能，最后通过 GCC 编译和链接生成一个完整程序。

### ① 底层原理

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>多文件编程的本质：</strong>
<ul>
  <li><strong>头文件 .h：</strong>放函数声明、类型声明、宏定义、extern 变量声明。</li>
  <li><strong>源文件 .c：</strong>放函数定义、全局变量定义、模块内部实现。</li>
  <li><strong>编译阶段：</strong>每个 .c 文件先分别编译成 .o 目标文件。</li>
  <li><strong>链接阶段：</strong>链接器把多个 .o 文件合并，解决函数和全局变量引用。</li>
  <li><strong>核心思想：</strong>声明让别人认识，实现负责真正干活。</li>
</ul>
</div>

### ② 代码用法

一个最小项目可以拆成：

```text
project/
├── main.c
├── math_utils.c
└── math_utils.h
```

含义：

```text
main.c：
程序入口，调用功能模块。

math_utils.h：
声明函数接口。

math_utils.c：
实现函数功能。
```

### ③ 使用场景

| 文件 | 作用 | 放什么 |
| :--- | :--- | :--- |
| `.h` | 对外声明 | 函数原型、结构体声明、宏、extern 声明 |
| `.c` | 具体实现 | 函数定义、全局变量定义、static 内部函数 |
| `.o` | 编译产物 | 单个源文件编译后的目标文件 |
| 可执行文件 | 最终程序 | 多个目标文件链接后的结果 |

> [!tip] 记忆口诀
> **头文件放声明，源文件放实现；先编译成目标，再链接成程序。**

# 02 | 基础框架：main.c + 模块 .h + 模块 .c

> [!abstract] 核心结论
> 最常见的多文件框架是：`main.c` 负责主流程，模块头文件负责声明，模块源文件负责实现。其他文件只需要 `#include` 头文件，就能调用模块功能。

### ① 项目结构

```text
project/
├── main.c
├── calc.c
└── calc.h
```

### ② calc.h：声明接口

```c
#ifndef CALC_H
#define CALC_H

int add(int a, int b);
int sub(int a, int b);

#endif
```

说明：

```text
这里只声明函数。
告诉其他 .c 文件：
有 add 和 sub 这两个函数可以用。
```

### ③ calc.c：实现功能

```c
#include "calc.h"

int add(int a, int b) {
    return a + b;
}

int sub(int a, int b) {
    return a - b;
}
```

说明：

```text
这里写真正的函数定义。
```

### ④ main.c：调用模块

```c
#include <stdio.h>
#include "calc.h"

int main() {
    int x = 10;
    int y = 20;

    printf("add = %d\n", add(x, y));
    printf("sub = %d\n", sub(x, y));

    return 0;
}
```

### ⑤ 编译运行

```bash
gcc main.c calc.c -o app
./app
```

输出：

```text
add = 30
sub = -10
```

> [!tip] 记忆口诀
> **main 管流程，h 管声明，c 管实现。**

# 03 | 头文件保护：防止重复包含

> [!abstract] 核心结论
> 每个 `.h` 头文件都应该写头文件保护，防止同一个头文件被重复包含，避免结构体、类型、声明重复引发编译问题。

### ① 底层原理

头文件保护写法：

```c
#ifndef CALC_H
#define CALC_H

// 头文件内容

#endif
```

第一次包含：

```text
CALC_H 未定义。
进入头文件内容。
定义 CALC_H。
```

第二次包含：

```text
CALC_H 已经定义。
跳过头文件内容。
```

### ② 标准模板

```c
#ifndef 文件名大写_H
#define 文件名大写_H

// 声明内容

#endif
```

例如：

```c
#ifndef STUDENT_H
#define STUDENT_H

void print_student(void);

#endif
```

### ③ 另一种写法

```c
#pragma once
```

说明：

```text
让编译器保证该头文件只被包含一次。
写法简单，但传统宏保护更通用。
```

### ④ 推荐规则

```text
学习阶段：
优先使用 #ifndef / #define / #endif。

工程中：
很多项目也会使用 #pragma once。
```

### ⑤ 使用场景

```text
所有自己写的 .h 文件都建议加头文件保护。
```

> [!tip] 记忆口诀
> **ifndef 开门，define 上锁，endif 关门；头文件只进一次。**

# 04 | extern：跨文件共享全局变量

> [!abstract] 核心结论
> 多文件编程中，如果一个全局变量要被多个 `.c` 文件使用，应该在一个 `.c` 文件中定义，在 `.h` 文件中使用 `extern` 声明，其他文件通过包含头文件来使用。

### ① 项目结构

```text
project/
├── main.c
├── config.c
└── config.h
```

### ② config.h：外部声明

```c
#ifndef CONFIG_H
#define CONFIG_H

extern int g_debug_mode;

void print_config(void);

#endif
```

说明：

```text
extern int g_debug_mode;
只是声明这个变量在别的地方定义。
不分配存储空间。
```

### ③ config.c：真正定义

```c
#include <stdio.h>
#include "config.h"

int g_debug_mode = 1;

void print_config(void) {
    printf("g_debug_mode = %d\n", g_debug_mode);
}
```

说明：

```text
int g_debug_mode = 1;
这里才是真正定义。
真正分配全局变量空间。
```

### ④ main.c：使用变量

```c
#include "config.h"

int main() {
    print_config();

    g_debug_mode = 0;

    print_config();

    return 0;
}
```

### ⑤ 编译运行

```bash
gcc main.c config.c -o app
./app
```

输出：

```text
g_debug_mode = 1
g_debug_mode = 0
```

> [!tip] 记忆口诀
> **变量定义放一个 c，extern 声明放 h；别人 include 后再用它。**

# 05 | static：隐藏模块内部变量和函数

> [!abstract] 核心结论
> `static` 用在全局变量或函数前，可以限制它们只在当前 `.c` 文件内部可见。多文件编程中，模块内部不想暴露的变量和函数应该加 `static`。

### ① 底层原理

```text
普通全局变量 / 函数：
其他 .c 文件可以通过 extern 或函数声明访问。

static 全局变量 / 函数：
只在当前 .c 文件内部可见。
链接器不会把它暴露给其他文件。
```

### ② 模块内部变量

counter.c：

```c
#include "counter.h"

static int g_count = 0;

void counter_add(void) {
    g_count++;
}

int counter_get(void) {
    return g_count;
}
```

说明：

```text
g_count 是模块内部状态。
外部文件不能直接访问。
只能通过 counter_add 和 counter_get 操作。
```

### ③ counter.h：只暴露接口

```c
#ifndef COUNTER_H
#define COUNTER_H

void counter_add(void);
int counter_get(void);

#endif
```

### ④ main.c

```c
#include <stdio.h>
#include "counter.h"

int main() {
    counter_add();
    counter_add();

    printf("%d\n", counter_get());

    return 0;
}
```

### ⑤ 编译运行

```bash
gcc main.c counter.c -o app
./app
```

输出：

```text
2
```

> [!tip] 记忆口诀
> **extern 往外找，static 往里藏；模块内部变量别乱暴露。**

# 06 | 推荐工程目录结构

> [!abstract] 核心结论
> 稍微规范一点的 C 项目，通常会把头文件放到 `include/`，源文件放到 `src/`，编译产物放到 `build/`，这样结构清楚，方便管理。

### ① 推荐结构

```text
project/
├── include/
│   ├── calc.h
│   ├── student.h
│   └── config.h
├── src/
│   ├── main.c
│   ├── calc.c
│   ├── student.c
│   └── config.c
└── build/
```

### ② include 放什么

```text
.h 头文件。
对外接口声明。
结构体类型声明。
宏定义。
extern 全局变量声明。
```

### ③ src 放什么

```text
.c 源文件。
函数实现。
模块内部 static 函数。
模块内部 static 变量。
真正的全局变量定义。
```

### ④ build 放什么

```text
.o 目标文件。
最终可执行文件。
中间编译产物。
```

### ⑤ 编译命令

```bash
gcc src/main.c src/calc.c src/student.c src/config.c -Iinclude -o build/app
```

说明：

```text
-Iinclude：
告诉 GCC 去 include 目录找头文件。
```

运行：

```bash
./build/app
```

> [!tip] 记忆口诀
> **include 放声明，src 放实现，build 放产物。**

# 07 | GCC 编译流程：从 .c 到可执行文件

> [!abstract] 核心结论
> 多文件项目编译时，每个 `.c` 文件都会先变成 `.o` 目标文件，最后链接成一个可执行程序。头文件不会单独编译，它只是被 `#include` 展开到 `.c` 文件中。

### ① 一步编译

```bash
gcc main.c calc.c student.c -o app
```

说明：

```text
GCC 自动完成预处理、编译、汇编、链接。
```

### ② 分步编译

```bash
gcc -c main.c -o main.o
gcc -c calc.c -o calc.o
gcc -c student.c -o student.o
gcc main.o calc.o student.o -o app
```

说明：

```text
-c：
只编译生成 .o，不链接。
```

### ③ 加 include 路径

如果头文件在 include 目录：

```bash
gcc -c src/main.c -Iinclude -o build/main.o
gcc -c src/calc.c -Iinclude -o build/calc.o
gcc build/main.o build/calc.o -o build/app
```

### ④ 常用编译选项

```bash
gcc src/*.c -Iinclude -Wall -Wextra -g -o build/app
```

含义：

```text
-Wall：
开启常见警告。

-Wextra：
开启更多警告。

-g：
生成调试信息。

-Iinclude：
指定头文件搜索路径。

-o build/app：
指定输出文件名。
```

### ⑤ 运行

```bash
./build/app
```

> [!tip] 记忆口诀
> **每个 c 变 o，多个 o 再链接；h 不单独编译，只被 include 展开。**

# 08 | Makefile 多文件编译框架

> [!abstract] 核心结论
> 当源文件变多后，不适合每次手写 GCC 命令。可以使用 Makefile 管理编译规则，实现一条 `make` 自动编译整个项目。

### ① 项目结构

```text
project/
├── include/
│   └── calc.h
├── src/
│   ├── main.c
│   └── calc.c
├── build/
└── Makefile
```

### ② Makefile 模板

```makefile
CC = gcc
CFLAGS = -Wall -Wextra -g -Iinclude

SRC = src/main.c src/calc.c
OBJ = build/main.o build/calc.o
TARGET = build/app

$(TARGET): $(OBJ)
	$(CC) $(OBJ) -o $(TARGET)

build/main.o: src/main.c include/calc.h
	$(CC) $(CFLAGS) -c src/main.c -o build/main.o

build/calc.o: src/calc.c include/calc.h
	$(CC) $(CFLAGS) -c src/calc.c -o build/calc.o

clean:
	rm -f build/*.o $(TARGET)
```

### ③ 使用命令

```bash
make
```

运行：

```bash
./build/app
```

清理：

```bash
make clean
```

### ④ 注意缩进

Makefile 中命令行前面必须是：

```text
Tab
```

不是普通空格。

例如：

```makefile
	$(CC) $(OBJ) -o $(TARGET)
```

### ⑤ 使用场景

```text
源文件变多。
编译命令太长。
只想重新编译改动过的文件。
需要统一管理编译参数。
```

> [!tip] 记忆口诀
> **文件多了用 make，规则写好一键编译。**

# 09 | 多文件编程常见错误

> [!abstract] 核心结论
> 多文件编程常见错误包括头文件里定义全局变量、忘记头文件保护、函数只声明不定义、忘记一起编译 `.c` 文件、static 符号被外部访问。

### ① 错误 1：头文件里定义全局变量

错误：

```c
// global.h
int g_num = 10;
```

问题：

```text
多个 .c 文件 include 后，会导致重复定义。
```

正确：

```c
// global.h
extern int g_num;
```

```c
// global.c
int g_num = 10;
```

### ② 错误 2：忘记头文件保护

错误：

```c
// calc.h
int add(int a, int b);
```

更推荐：

```c
#ifndef CALC_H
#define CALC_H

int add(int a, int b);

#endif
```

### ③ 错误 3：只声明不定义

```c
// calc.h
int add(int a, int b);
```

但没有任何 `.c` 文件写：

```c
int add(int a, int b) {
    return a + b;
}
```

链接时可能报：

```text
undefined reference to `add`
```

### ④ 错误 4：忘记一起编译

错误：

```bash
gcc main.c -o app
```

如果 `add` 在 `calc.c` 里，应该：

```bash
gcc main.c calc.c -o app
```

### ⑤ 错误 5：外部访问 static 函数

calc.c：

```c
static int helper(void) {
    return 1;
}
```

main.c 不能调用：

```c
helper();
```

因为：

```text
static 函数只在 calc.c 内部可见。
```

> [!tip] 记忆口诀
> **头文件别定义变量，声明定义要配套；源文件要一起编，static 外面找不到。**

# 10 | 多文件编程总结

> [!abstract] 核心结论
> 多文件编程的底层核心是“头文件提供声明，源文件提供定义，编译器分别编译，链接器统一连接”。掌握 `.h`、`.c`、`extern`、`static`、GCC 编译和 Makefile，就能搭建基本 C 项目框架。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>多文件编程核心总结：</strong>
<ul>
  <li><strong>.h 文件：</strong>放声明，不放普通全局变量定义。</li>
  <li><strong>.c 文件：</strong>放实现和真正定义。</li>
  <li><strong>main.c：</strong>程序入口，负责主流程。</li>
  <li><strong>模块 .h：</strong>暴露函数接口、类型声明、extern 声明。</li>
  <li><strong>模块 .c：</strong>实现模块功能。</li>
  <li><strong>extern：</strong>声明外部全局变量。</li>
  <li><strong>static：</strong>隐藏模块内部变量和函数。</li>
  <li><strong>头文件保护：</strong>防止重复包含。</li>
  <li><strong>GCC：</strong>多个 .c 文件一起编译链接。</li>
  <li><strong>Makefile：</strong>文件多时自动化编译。</li>
</ul>
</div>

### ② 最底层一句话

```text
多文件编程 = .h 让别人知道你有什么，.c 真正实现你怎么做，链接器最后把所有模块拼成程序。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **头文件写声明，源文件写真身；extern 向外找，static 往里藏；多个 c 先编译，多个 o 再链接。**
