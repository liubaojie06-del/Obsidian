# 01 | 回调函数：把函数当参数传进去，等需要时再调用

> [!abstract] 核心结论
> 回调函数本质上是通过函数指针实现的“函数参数”。调用者把一个函数地址传给另一个函数，被调用函数在合适的时机再调用这个函数，这种机制就叫回调。

### ① 底层原理：传进去的不是结果，而是函数入口地址

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>回调函数的本质：</strong>
<ul>
  <li><strong>函数名：</strong> 在很多表达式中可以表示函数入口地址。</li>
  <li><strong>函数指针：</strong> 用来保存函数入口地址。</li>
  <li><strong>回调函数：</strong> 被当作参数传入另一个函数，并由另一个函数在内部调用。</li>
  <li><strong>调用者：</strong> 决定传入哪个函数。</li>
  <li><strong>被调用者：</strong> 不关心具体函数名，只通过函数指针调用它。</li>
</ul>
</div>

### ② 一句话理解

```text
回调函数 = 把一段“以后要执行的逻辑”先传给别人，等别人需要时再帮你调用。
```

例如：

```text
你把 add 函数传给 calculate。
calculate 内部通过函数指针 op 调用 add。
这就是回调。
```

### ③ 基本形式

```c
返回值类型 函数名(参数列表);

void 外层函数(函数指针参数) {
    函数指针参数(...);
}
```

### ④ 最小示例

```c
#include <stdio.h>

void hello() {
    printf("hello callback\n");
}

void run(void (*func)()) {
    func();
}

int main() {
    run(hello);

    return 0;
}
```

输出：

```text
hello callback
```

### ⑤ 重点理解

```text
run(hello);
```

不是：

```text
立刻在 main 里执行 hello。
```

而是：

```text
把 hello 的函数入口地址传给 run。
由 run 内部决定什么时候调用。
```

> [!tip] 记忆口诀
> **函数名当地址传，别人内部再调用；传的是逻辑，不是结果。**

# 02 | 回调函数和函数指针的关系

> [!abstract] 核心结论
> 回调函数依赖函数指针实现。函数指针负责保存函数入口地址，回调函数是函数指针作为参数使用时形成的一种编程模式。

### ① 函数指针

```c
int (*p)(int, int);
```

含义：

```text
p 是函数指针。
它可以指向返回 int，参数为 int、int 的函数。
```

### ② 指向一个函数

```c
int add(int a, int b) {
    return a + b;
}

int (*p)(int, int) = add;
```

含义：

```text
p 保存 add 函数入口地址。
```

调用：

```c
int result = p(10, 20);
```

等价于：

```c
int result = add(10, 20);
```

### ③ 回调函数参数

```c
void calculate(int x, int y, int (*op)(int, int)) {
    int result = op(x, y);
    printf("%d\n", result);
}
```

这里：

```text
op 是函数指针参数。
外部传入哪个函数，calculate 内部就调用哪个函数。
```

### ④ 完整示例

```c
#include <stdio.h>

int add(int a, int b) {
    return a + b;
}

int sub(int a, int b) {
    return a - b;
}

void calculate(int x, int y, int (*op)(int, int)) {
    int result = op(x, y);
    printf("result = %d\n", result);
}

int main() {
    calculate(10, 20, add);
    calculate(10, 20, sub);

    return 0;
}
```

输出：

```text
result = 30
result = -10
```

### ⑤ 重点区别

```text
函数指针：
一种变量类型，保存函数地址。

回调函数：
函数指针作为参数传递后，由别的函数内部调用的用法。
```

> [!tip] 记忆口诀
> **函数指针是工具，回调函数是用法；指针存地址，回调传逻辑。**

# 03 | 为什么需要回调函数

> [!abstract] 核心结论
> 回调函数可以让一个通用函数不固定具体操作，而是把具体逻辑交给调用者决定。这样代码更灵活，复用性更强。

### ① 不用回调的问题

假设要写一个计算函数：

```c
int calculate_add(int a, int b) {
    return a + b;
}

int calculate_sub(int a, int b) {
    return a - b;
}

int calculate_mul(int a, int b) {
    return a * b;
}
```

问题：

```text
每种操作都要写一个函数。
重复代码多。
扩展新操作不方便。
```

### ② 使用回调

```c
int calculate(int a, int b, int (*op)(int, int)) {
    return op(a, b);
}
```

这样：

```text
calculate 不关心具体是加法、减法还是乘法。
具体操作由 op 决定。
```

### ③ 调用者决定逻辑

```c
calculate(10, 20, add);
calculate(10, 20, sub);
calculate(10, 20, mul);
```

同一个 `calculate` 可以执行不同逻辑。

### ④ 好处

```text
减少重复代码。
提高函数复用性。
把固定流程和变化逻辑分离。
适合事件处理。
适合排序规则。
适合菜单分发。
适合状态机。
```

### ⑤ 一句话理解

```text
回调函数让“流程固定，具体动作可替换”。
```

> [!tip] 记忆口诀
> **流程写一遍，动作传进去；谁来干活，由回调决定。**

# 04 | 回调函数基本写法

> [!abstract] 核心结论
> 写回调函数通常分三步：先定义符合要求的普通函数，再在外层函数中写函数指针参数，最后调用时把函数名传进去。

### ① 第一步：定义回调函数

```c
int add(int a, int b) {
    return a + b;
}
```

这个函数以后会被当作回调传入。

### ② 第二步：定义接收回调的函数

```c
void calculate(int x, int y, int (*callback)(int, int)) {
    int result = callback(x, y);
    printf("%d\n", result);
}
```

其中：

```c
int (*callback)(int, int)
```

表示：

```text
callback 是函数指针。
它指向返回 int，参数为 int、int 的函数。
```

### ③ 第三步：传入函数名

```c
calculate(10, 20, add);
```

含义：

```text
把 add 函数入口地址传给 callback。
```

### ④ 完整代码

```c
#include <stdio.h>

int add(int a, int b) {
    return a + b;
}

void calculate(int x, int y, int (*callback)(int, int)) {
    int result = callback(x, y);
    printf("result = %d\n", result);
}

int main() {
    calculate(10, 20, add);

    return 0;
}
```

输出：

```text
result = 30
```

### ⑤ 注意不是这样写

错误：

```c
calculate(10, 20, add(10, 20));
```

原因：

```text
add(10, 20) 是函数调用结果。
它是 int。
不是函数地址。
```

正确：

```c
calculate(10, 20, add);
```

> [!tip] 记忆口诀
> **定义函数，声明函数指针参数，调用时传函数名。**

# 05 | 回调函数作为排序规则

> [!abstract] 核心结论
> 回调函数最典型的用途之一是把“比较规则”传给排序函数。排序流程固定，但升序、降序、按年龄、按成绩等规则可以通过回调函数替换。

### ① 升序比较函数

```c
int asc(int a, int b) {
    return a > b;
}
```

含义：

```text
如果 a > b，说明需要交换。
```

### ② 降序比较函数

```c
int desc(int a, int b) {
    return a < b;
}
```

含义：

```text
如果 a < b，说明需要交换。
```

### ③ 排序函数接收回调

```c
void sort(int arr[], int len, int (*cmp)(int, int)) {
    for (int i = 0; i < len - 1; i++) {
        for (int j = 0; j < len - 1 - i; j++) {
            if (cmp(arr[j], arr[j + 1])) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}
```

### ④ 完整示例

```c
#include <stdio.h>

int asc(int a, int b) {
    return a > b;
}

int desc(int a, int b) {
    return a < b;
}

void sort(int arr[], int len, int (*cmp)(int, int)) {
    for (int i = 0; i < len - 1; i++) {
        for (int j = 0; j < len - 1 - i; j++) {
            if (cmp(arr[j], arr[j + 1])) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}

void print_arr(int arr[], int len) {
    for (int i = 0; i < len; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
}

int main() {
    int nums[] = {5, 2, 9, 1, 7};
    int len = sizeof(nums) / sizeof(nums[0]);

    sort(nums, len, asc);
    print_arr(nums, len);

    sort(nums, len, desc);
    print_arr(nums, len);

    return 0;
}
```

输出：

```text
1 2 5 7 9
9 7 5 2 1
```

### ⑤ 重点

```text
sort 只负责排序流程。
cmp 负责决定谁应该排前面。
```

> [!tip] 记忆口诀
> **排序流程固定，比较规则变化；规则传进去，就是回调。**

# 06 | qsort 中的回调函数

> [!abstract] 核心结论
> C 标准库中的 `qsort` 就是回调函数的经典应用。它不知道你排序的是整数、结构体还是字符串，所以需要你传入比较函数。

### ① qsort 原型

```c
void qsort(
    void *base,
    size_t num,
    size_t size,
    int (*compar)(const void *, const void *)
);
```

含义：

```text
base：数组首地址。
num：元素个数。
size：每个元素大小。
compar：比较函数，也就是回调函数。
```

### ② 整数比较函数

```c
int cmp_int(const void *a, const void *b) {
    int x = *(const int *)a;
    int y = *(const int *)b;

    return x - y;
}
```

更稳妥写法：

```c
int cmp_int(const void *a, const void *b) {
    int x = *(const int *)a;
    int y = *(const int *)b;

    if (x > y) return 1;
    if (x < y) return -1;
    return 0;
}
```

### ③ 完整示例

```c
#include <stdio.h>
#include <stdlib.h>

int cmp_int(const void *a, const void *b) {
    int x = *(const int *)a;
    int y = *(const int *)b;

    if (x > y) return 1;
    if (x < y) return -1;
    return 0;
}

int main() {
    int nums[] = {5, 2, 9, 1, 7};
    int len = sizeof(nums) / sizeof(nums[0]);

    qsort(nums, len, sizeof(nums[0]), cmp_int);

    for (int i = 0; i < len; i++) {
        printf("%d ", nums[i]);
    }

    return 0;
}
```

输出：

```text
1 2 5 7 9
```

### ④ 为什么参数是 const void *

因为：

```text
qsort 要支持任意类型数组。
它不知道数组元素是什么类型。
所以用 void * 表示通用地址。
```

比较函数内部需要：

```text
把 void * 转回真实类型。
```

### ⑤ qsort 的回调逻辑

```text
qsort 负责排序流程。
cmp_int 负责比较两个元素大小。
qsort 在排序过程中反复调用 cmp_int。
```

> [!tip] 记忆口诀
> **qsort 管排序，cmp 管比较；元素类型你知道，比较逻辑你来写。**

# 07 | 回调函数用于菜单分发

> [!abstract] 核心结论
> 回调函数可以把菜单选项和处理函数绑定起来。用户选择不同编号时，程序通过函数指针调用对应功能，避免大量 `if else` 或 `switch`。

### ① 普通写法

```c
if (choice == 0) {
    start();
} else if (choice == 1) {
    stop();
} else if (choice == 2) {
    pause();
}
```

问题：

```text
选项多时，代码很长。
维护不方便。
```

### ② 函数指针数组写法

```c
void (*menu[])() = {start, stop, pause};
```

含义：

```text
menu 是函数指针数组。
每个元素保存一个函数入口地址。
```

### ③ 完整示例

```c
#include <stdio.h>

void start() {
    printf("start\n");
}

void stop() {
    printf("stop\n");
}

void pause_app() {
    printf("pause\n");
}

int main() {
    void (*menu[])() = {start, stop, pause_app};

    int choice = 1;

    menu[choice]();

    return 0;
}
```

输出：

```text
stop
```

### ④ 加安全检查

```c
int count = sizeof(menu) / sizeof(menu[0]);

if (choice >= 0 && choice < count) {
    menu[choice]();
} else {
    printf("invalid choice\n");
}
```

### ⑤ 重点理解

```text
menu[0] 保存 start 地址。
menu[1] 保存 stop 地址。
menu[2] 保存 pause_app 地址。

menu[choice]() 调用对应函数。
```

> [!tip] 记忆口诀
> **函数地址放数组，下标选择功能；菜单分发少写一堆判断。**

# 08 | typedef 简化回调函数写法

> [!abstract] 核心结论
> 回调函数参数的函数指针声明通常比较复杂，可以用 `typedef` 给函数指针类型起别名，让代码更容易阅读和维护。

### ① 不使用 typedef

```c
void calculate(int x, int y, int (*op)(int, int)) {
    printf("%d\n", op(x, y));
}
```

这段：

```c
int (*op)(int, int)
```

可读性较差。

### ② 使用 typedef

```c
typedef int (*Operation)(int, int);
```

含义：

```text
Operation 是一种函数指针类型。
它可以指向返回 int，参数为 int、int 的函数。
```

之后可以写：

```c
void calculate(int x, int y, Operation op) {
    printf("%d\n", op(x, y));
}
```

### ③ 完整示例

```c
#include <stdio.h>

typedef int (*Operation)(int, int);

int add(int a, int b) {
    return a + b;
}

int sub(int a, int b) {
    return a - b;
}

void calculate(int x, int y, Operation op) {
    int result = op(x, y);
    printf("%d\n", result);
}

int main() {
    calculate(10, 20, add);
    calculate(10, 20, sub);

    return 0;
}
```

输出：

```text
30
-10
```

### ④ 函数指针数组也更清晰

不用 typedef：

```c
int (*ops[2])(int, int) = {add, sub};
```

使用 typedef：

```c
Operation ops[2] = {add, sub};
```

### ⑤ 重点

```text
typedef 不改变底层原理。
只是给复杂类型起名字。
```

> [!tip] 记忆口诀
> **函数指针太绕，typedef 起外号；回调参数更清楚。**

# 09 | 回调函数常见错误

> [!abstract] 核心结论
> 回调函数常见错误包括把函数调用结果当作函数地址传入、函数指针类型不匹配、调用空函数指针、回调函数访问已经失效的数据。

### ① 错误 1：传了函数调用结果

错误：

```c
calculate(10, 20, add(10, 20));
```

问题：

```text
add(10, 20) 是 int 结果。
不是函数地址。
```

正确：

```c
calculate(10, 20, add);
```

### ② 错误 2：函数类型不匹配

```c
double div(double a, double b) {
    return a / b;
}

calculate(10, 20, div);  // 错误
```

原因：

```text
calculate 需要 int (*)(int, int)。
div 是 double (*)(double, double)。
类型不匹配。
```

### ③ 错误 3：调用空函数指针

```c
void run(void (*callback)()) {
    callback();  // 如果 callback 是 NULL，会崩溃
}
```

更安全：

```c
void run(void (*callback)()) {
    if (callback != NULL) {
        callback();
    }
}
```

### ④ 错误 4：回调中使用无效地址

```c
int *p;

void callback() {
    printf("%d\n", *p);
}
```

如果：

```text
p 没初始化，或者指向已经释放的内存。
```

回调执行时会出错。

### ⑤ 错误 5：忘记回调函数由别人调用

```text
回调函数不一定马上执行。
它可能稍后执行。
所以回调里用到的数据必须在执行时仍然有效。
```

> [!tip] 记忆口诀
> **传函数名，不传调用结果；类型要匹配，调用前判空，数据别失效。**

# 10 | 回调函数总结

> [!abstract] 核心结论
> 回调函数的底层核心是“把函数入口地址作为参数传递”。它让一个通用流程可以在关键步骤调用外部传入的具体逻辑，是 C 语言实现灵活扩展的重要方式。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>回调函数核心总结：</strong>
<ul>
  <li><strong>本质：</strong> 通过函数指针把函数作为参数传递。</li>
  <li><strong>函数名：</strong> 可以作为函数入口地址传入。</li>
  <li><strong>接收方式：</strong> 使用函数指针参数接收。</li>
  <li><strong>调用方式：</strong> 在被调用函数内部通过函数指针执行。</li>
  <li><strong>典型场景：</strong> 排序规则、菜单分发、事件处理、状态机、驱动接口。</li>
  <li><strong>qsort：</strong> C 标准库中使用回调比较函数的典型例子。</li>
  <li><strong>typedef：</strong> 可简化复杂函数指针类型。</li>
  <li><strong>类型要求：</strong> 回调函数的返回值和参数列表必须匹配。</li>
  <li><strong>安全要求：</strong> 调用前最好判断函数指针是否为 NULL。</li>
  <li><strong>核心价值：</strong> 固定流程和可变逻辑分离，提高代码复用性。</li>
</ul>
</div>

### ② 最底层一句话

```text
回调函数 = 把函数地址传给另一个函数，让另一个函数在内部按需要调用它。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **回调就是传函数，别人内部再调用；流程固定，动作可换，函数指针来实现。**
