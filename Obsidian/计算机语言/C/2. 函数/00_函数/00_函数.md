# 01 | 函数：把一段代码封装成可重复调用的模块

> [!abstract] 核心结论
> 函数是 C 语言中最重要的代码组织方式。它把一段具有明确功能的代码封装起来，需要时通过函数名调用，可以减少重复代码、提升可读性，并让程序结构更清晰。

### ① 底层原理：函数不是“消失的代码”，而是一次跳转执行

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>函数的底层本质：</strong>
<ul>
  <li><strong>函数是一段代码：</strong> 编译后函数会变成机器指令，存放在代码区。</li>
  <li><strong>调用函数：</strong> CPU 会跳转到函数入口地址执行。</li>
  <li><strong>参数传递：</strong> 调用者把数据传给函数，函数内部用形参接收。</li>
  <li><strong>返回值：</strong> 函数执行完后，把结果返回给调用者。</li>
  <li><strong>栈帧：</strong> 函数调用时通常会在栈区创建临时空间，保存局部变量、返回地址等信息。</li>
</ul>
</div>

### ② 基本结构

```c
返回值类型 函数名(参数列表) {
    函数体;
    return 返回值;
}
```

示例：

```c
int add(int a, int b) {
    return a + b;
}
```

含义：

```text
int：函数返回 int 类型结果
add：函数名
int a, int b：函数参数
return a + b：返回计算结果
```

### ③ 调用函数

```c
#include <stdio.h>

int add(int a, int b) {
    return a + b;
}

int main() {
    int result = add(10, 20);

    printf("%d\n", result);

    return 0;
}
```

输出：

```text
30
```

> [!tip] 记忆口诀
> **函数就是代码块，传参进去，执行一遍，结果返回。**

# 02 | 函数声明、函数定义与函数调用

> [!abstract] 核心结论
> 函数声明告诉编译器“有这个函数”；函数定义给出函数的真实代码；函数调用才是真正让函数执行。

### ① 函数声明

函数声明也叫函数原型。

```c
int add(int a, int b);
```

作用：

```text
告诉编译器：
add 是一个函数。
它接收两个 int 参数。
它返回 int。
```

函数声明通常放在：

```text
源文件开头
头文件 .h 中
```

### ② 函数定义

```c
int add(int a, int b) {
    return a + b;
}
```

作用：

```text
真正写出函数要执行的代码。
```

### ③ 函数调用

```c
int result = add(10, 20);
```

作用：

```text
让程序跳到 add 函数执行。
执行完后，把返回值赋给 result。
```

### ④ 完整示例

```c
#include <stdio.h>

int add(int a, int b);  // 函数声明

int main() {
    int result = add(10, 20);  // 函数调用

    printf("%d\n", result);

    return 0;
}

int add(int a, int b) {  // 函数定义
    return a + b;
}
```

输出：

```text
30
```

### ⑤ 为什么要声明

如果函数定义写在 `main` 后面，而前面没有声明：

```text
编译器在 main 中看到 add 时，可能还不知道 add 是什么。
```

所以提前声明可以让编译器知道：

```text
函数名
参数类型
返回值类型
```

> [!tip] 记忆口诀
> **声明是报名字，定义是真代码，调用才会执行。**

# 03 | 形参和实参

> [!abstract] 核心结论
> 实参是调用函数时传进去的真实数据，形参是函数定义中用来接收数据的变量。函数调用时，实参的值会传给形参。

### ① 实参

调用函数时传入的值叫实参。

```c
int result = add(10, 20);
```

这里：

```text
10 是实参
20 是实参
```

### ② 形参

函数定义中接收数据的变量叫形参。

```c
int add(int a, int b) {
    return a + b;
}
```

这里：

```text
a 是形参
b 是形参
```

### ③ 调用时发生什么

```c
add(10, 20);
```

可以理解为：

```text
a = 10
b = 20
```

然后函数内部执行：

```c
return a + b;
```

### ④ 示例代码

```c
#include <stdio.h>

void print_num(int x) {
    printf("x = %d\n", x);
}

int main() {
    int a = 100;

    print_num(a);

    return 0;
}
```

输出：

```text
x = 100
```

### ⑤ 重点理解

```text
实参在调用处。
形参在函数定义处。
实参把值传给形参。
```

> [!tip] 记忆口诀
> **调用传进去的是实参，函数接住的是形参。**

# 04 | 值传递：普通变量传参不会改原变量

> [!abstract] 核心结论
> C 语言函数参数默认是值传递。普通变量作为参数传入函数时，函数拿到的是一份拷贝，修改形参不会改变外部原变量。

### ① 示例代码

```c
#include <stdio.h>

void change(int x) {
    x = 100;
}

int main() {
    int a = 10;

    change(a);

    printf("a = %d\n", a);

    return 0;
}
```

输出：

```text
a = 10
```

### ② 为什么没有修改成功

调用：

```c
change(a);
```

底层理解：

```text
把 a 的值 10 拷贝一份给 x。
```

函数内部：

```c
x = 100;
```

只是改了形参 `x`，不是改外面的 `a`。

### ③ 内存理解

```text
main 函数栈帧：
a = 10

change 函数栈帧：
x = 10  →  改成 100
```

`a` 和 `x` 是两个不同变量。

### ④ 想改外部变量怎么办

要传地址：

```c
void change(int *p) {
    *p = 100;
}
```

调用：

```c
change(&a);
```

### ⑤ 指针修改示例

```c
#include <stdio.h>

void change(int *p) {
    *p = 100;
}

int main() {
    int a = 10;

    change(&a);

    printf("a = %d\n", a);

    return 0;
}
```

输出：

```text
a = 100
```

> [!tip] 记忆口诀
> **普通传参传副本，函数里改形参，外面不受影响；想改原变量，就传地址。**

# 05 | 返回值：函数把结果带回调用处

> [!abstract] 核心结论
> 返回值是函数执行完成后交给调用者的结果。`return` 会结束当前函数，并把指定结果返回。

### ① 有返回值函数

```c
int add(int a, int b) {
    return a + b;
}
```

调用：

```c
int result = add(10, 20);
```

含义：

```text
add 执行后返回 30。
30 赋值给 result。
```

### ② 无返回值函数

```c
void print_hello() {
    printf("hello\n");
}
```

含义：

```text
void 表示没有返回值。
```

调用：

```c
print_hello();
```

### ③ return 的作用

```c
return 表达式;
```

作用：

```text
1. 结束当前函数。
2. 把表达式结果返回给调用者。
```

例如：

```c
int max(int a, int b) {
    if (a > b) {
        return a;
    }

    return b;
}
```

### ④ void 函数中的 return

```c
void test() {
    printf("before\n");

    return;

    printf("after\n");
}
```

输出：

```text
before
```

说明：

```text
return 会直接结束函数。
```

### ⑤ 常见错误

错误：

```c
int get_num() {
    int a = 10;
}
```

问题：

```text
函数声明返回 int，但没有 return。
```

正确：

```c
int get_num() {
    int a = 10;
    return a;
}
```

> [!tip] 记忆口诀
> **return 一出现，函数立刻停；有值带值回，无值直接退。**

# 06 | 函数调用与栈帧

> [!abstract] 核心结论
> 函数调用时，系统通常会在栈区为该函数创建一块临时空间，称为栈帧。栈帧中保存形参、局部变量、返回地址等信息。函数结束后，栈帧释放。

### ① 什么是栈帧

```text
栈帧 = 函数调用时在栈上创建的临时工作区。
```

一个函数栈帧里可能有：

```text
形参
局部变量
返回地址
保存的寄存器
临时数据
```

### ② 调用过程

```text
main 调用 add
      ↓
保存返回地址
      ↓
传递参数
      ↓
创建 add 的栈帧
      ↓
执行 add
      ↓
返回结果
      ↓
释放 add 的栈帧
      ↓
回到 main 继续执行
```

### ③ 示例

```c
#include <stdio.h>

int add(int a, int b) {
    int c = a + b;
    return c;
}

int main() {
    int result = add(10, 20);

    printf("%d\n", result);

    return 0;
}
```

栈帧理解：

```text
main 栈帧：
result

add 栈帧：
a
b
c
返回地址
```

### ④ 函数结束后局部变量消失

```c
int *test() {
    int a = 10;
    return &a;
}
```

这是错误写法。

原因：

```text
a 是局部变量。
函数结束后 test 的栈帧释放。
&a 指向的空间已经失效。
```

### ⑤ 正确理解

```text
局部变量通常随函数调用创建。
函数结束后释放。
不能返回局部变量地址。
```

> [!tip] 记忆口诀
> **函数一调用，栈帧就创建；函数一返回，栈帧就释放。**

# 07 | 数组作为函数参数

> [!abstract] 核心结论
> 数组作为函数参数时，数组名会退化为指向首元素的指针。所以函数无法通过 `sizeof` 得到数组元素个数，必须额外传入长度。

### ① 一维数组传参

```c
void print_arr(int arr[], int len)
```

等价于：

```c
void print_arr(int *arr, int len)
```

因为：

```text
数组参数会退化为指针。
```

### ② 示例代码

```c
#include <stdio.h>

void print_arr(int arr[], int len) {
    for (int i = 0; i < len; i++) {
        printf("%d ", arr[i]);
    }

    printf("\n");
}

int main() {
    int nums[5] = {1, 2, 3, 4, 5};

    print_arr(nums, 5);

    return 0;
}
```

输出：

```text
1 2 3 4 5
```

### ③ 函数里不能用 sizeof 求长度

错误：

```c
void print_arr(int arr[]) {
    int len = sizeof(arr) / sizeof(arr[0]);
}
```

原因：

```text
arr 在函数参数中已经退化成 int *。
sizeof(arr) 得到的是指针大小，不是数组总大小。
```

### ④ 二维数组传参

```c
void print_arr(int arr[][4], int rows)
```

等价于：

```c
void print_arr(int (*arr)[4], int rows)
```

第二维必须写清楚：

```text
因为函数要知道一行有几个元素。
```

### ⑤ 二维数组示例

```c
#include <stdio.h>

void print_arr(int arr[][4], int rows) {
    for (int i = 0; i < rows; i++) {
        for (int j = 0; j < 4; j++) {
            printf("%d ", arr[i][j]);
        }

        printf("\n");
    }
}

int main() {
    int nums[2][4] = {
        {1, 2, 3, 4},
        {5, 6, 7, 8}
    };

    print_arr(nums, 2);

    return 0;
}
```

输出：

```text
1 2 3 4
5 6 7 8
```

> [!tip] 记忆口诀
> **数组传参会退化，一维传长度，二维写列数。**

# 08 | 函数与指针参数

> [!abstract] 核心结论
> 指针参数可以让函数访问外部变量或数组。普通值传递只能改副本，指针传参可以通过地址修改原数据。

### ① 修改一个变量

```c
#include <stdio.h>

void set_value(int *p) {
    *p = 100;
}

int main() {
    int a = 10;

    set_value(&a);

    printf("%d\n", a);

    return 0;
}
```

输出：

```text
100
```

### ② 交换两个变量

```c
#include <stdio.h>

void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main() {
    int x = 10;
    int y = 20;

    swap(&x, &y);

    printf("x = %d, y = %d\n", x, y);

    return 0;
}
```

输出：

```text
x = 20, y = 10
```

### ③ 传数组本质也是传地址

```c
void change_first(int arr[]) {
    arr[0] = 100;
}
```

等价于：

```c
void change_first(int *arr) {
    arr[0] = 100;
}
```

所以：

```text
函数可以修改数组元素。
```

### ④ 使用 const 防止修改

如果函数只读数据，不想修改：

```c
void print_arr(const int *arr, int len) {
    for (int i = 0; i < len; i++) {
        printf("%d ", arr[i]);
    }
}
```

作用：

```text
防止函数内部误改数组内容。
```

### ⑤ 指针参数常见风险

```text
传入 NULL。
传入野指针。
传入已经释放的地址。
数组长度不够。
忘记传长度。
```

> [!tip] 记忆口诀
> **想让函数改外面，就把地址传进去；只读不改，加 const 更安全。**

# 09 | 函数声明放头文件，多文件编程

> [!abstract] 核心结论
> 在大型程序中，函数通常分散在多个 `.c` 文件中。函数声明放在 `.h` 头文件里，函数定义放在 `.c` 源文件里，其他文件通过 `#include` 引入声明后即可调用。

### ① add.h

```c
#ifndef ADD_H
#define ADD_H

int add(int a, int b);

#endif
```

### ② add.c

```c
#include "add.h"

int add(int a, int b) {
    return a + b;
}
```

### ③ main.c

```c
#include <stdio.h>
#include "add.h"

int main() {
    int result = add(10, 20);

    printf("%d\n", result);

    return 0;
}
```

### ④ 编译

```bash
gcc main.c add.c -o app
```

运行：

```bash
./app
```

输出：

```text
30
```

### ⑤ 为什么要用头文件

```text
.h 放声明。
.c 放定义。
其他文件 include 头文件后，就知道函数怎么调用。
```

### ⑥ 头文件保护

```c
#ifndef ADD_H
#define ADD_H

// 内容

#endif
```

作用：

```text
防止同一个头文件被重复包含。
```

> [!tip] 记忆口诀
> **头文件放声明，源文件放定义；include 先认识，链接再找实现。**

# 10 | 函数总结

> [!abstract] 核心结论
> 函数的底层核心是“代码区中的一段指令 + 调用时创建栈帧 + 参数传递 + 返回值”。函数让程序可以模块化组织代码，减少重复，提高可读性。

### ① 核心总结

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #27AE60; color: #333; margin-bottom: 10px;">
<strong>函数核心总结：</strong>
<ul>
  <li><strong>本质：</strong> 一段可以被重复调用的代码。</li>
  <li><strong>声明：</strong> 告诉编译器函数名、参数和返回值。</li>
  <li><strong>定义：</strong> 写出函数真正执行的代码。</li>
  <li><strong>调用：</strong> 让程序跳到函数执行。</li>
  <li><strong>形参：</strong> 函数定义中接收数据的变量。</li>
  <li><strong>实参：</strong> 调用函数时传入的真实数据。</li>
  <li><strong>值传递：</strong> 普通变量传参是拷贝，函数里改形参不影响外部变量。</li>
  <li><strong>指针传参：</strong> 传地址，可以修改外部变量。</li>
  <li><strong>栈帧：</strong> 函数调用时创建，函数返回后释放。</li>
  <li><strong>数组传参：</strong> 会退化为指针，长度需要单独传。</li>
</ul>
</div>

### ② 最底层一句话

```text
函数 = 程序中一段有入口、有参数、有局部栈帧、可返回结果的代码块。
```

### ③ 最终记忆口诀

> [!tip] 记忆口诀
> **函数封装一段活，实参传给形参；普通传值改副本，指针传址改原身；调用建栈帧，返回就释放。**
