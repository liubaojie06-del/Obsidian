# 01｜FOR 固定次数循环

> [!NOTE] 卡片定位  
> `FOR` 是 SCL 中最常用的循环语句。  
> 它适合**循环次数明确**的场景。

> [!IMPORTANT] 底层原理  
> `FOR` 会让循环变量从起始值变化到结束值。  
> 每变化一次，就执行一次 `DO` 到 `END_FOR` 之间的代码。

```scl
FOR i := 1 TO 10 DO
    // 循环执行 10 次
END_FOR;
```

| 适合场景 | 说明 |
|---|---|
| 数组清零 | 批量清除数据 |
| 报警扫描 | 检查多个报警位 |
| 产品统计 | 遍历多个工位 |
| 数据复制 | 批量搬运数组 |
| 配方处理 | 遍历配方参数 |

```scl
// 场景：清空 10 个工位的产品检测状态
// StationHasProduct[1..10]：每个工位是否有产品

FOR i := 1 TO 10 DO
    StationHasProduct[i] := FALSE;
END_FOR;
```

```scl
// 场景：统计 10 个工位中有多少个工位有产品

ProductStationCount := 0;

FOR i := 1 TO 10 DO
    IF StationHasProduct[i] = TRUE THEN
        ProductStationCount := ProductStationCount + 1;
    END_IF;
END_FOR;
```

```scl
// 场景：扫描 16 个报警位，只要有一个报警，就置位总报警

GeneralAlarm := FALSE;

FOR i := 1 TO 16 DO
    IF AlarmList[i] = TRUE THEN
        GeneralAlarm := TRUE;
    END_IF;
END_FOR;
```

> [!WARNING] 工程注意  
> PLC 是周期扫描执行。  
> `FOR` 循环太大，会增加单个扫描周期时间，严重时可能导致 PLC 扫描超时。

> [!IMPORTANT] 工控口诀  
> **次数明确用 FOR。**  
> **数组批量处理最常用 FOR。**

# 02｜TO 循环范围

> [!NOTE] 卡片定位  
> `TO` 用于指定 `FOR` 循环的结束值。  
> 它表示循环变量从起始值运行到某个目标值。

> [!IMPORTANT] 底层原理  
> `FOR i := 1 TO 10 DO` 表示：  
> `i` 从 1 开始，每次默认加 1，直到 10 结束。

```scl
FOR i := StartValue TO EndValue DO
    // 循环体
END_FOR;
```

| 写法 | 循环次数 |
|---|---:|
| `FOR i := 1 TO 5 DO` | 5 次 |
| `FOR i := 0 TO 9 DO` | 10 次 |
| `FOR i := 1 TO 16 DO` | 16 次 |

```scl
// 场景：扫描 8 个按钮输入状态
// Button[1..8]：按钮数组
// AnyButtonPressed：任意按钮按下

AnyButtonPressed := FALSE;

FOR i := 1 TO 8 DO
    IF Button[i] = TRUE THEN
        AnyButtonPressed := TRUE;
    END_IF;
END_FOR;
```

```scl
// 场景：把 20 个称重数据求和

TotalWeight := 0.0;

FOR i := 1 TO 20 DO
    TotalWeight := TotalWeight + WeightValue[i];
END_FOR;

AverageWeight := TotalWeight / 20.0;
```

> [!WARNING] 下标注意  
> 数组下标必须和声明范围一致。  
> 如果数组是 `[0..9]`，循环就不要写 `1 TO 10`。

错误写法：

```scl
// 数组如果是 WeightValue[0..9]
// 这样会访问不存在的 WeightValue[10]

FOR i := 1 TO 10 DO
    TotalWeight := TotalWeight + WeightValue[i];
END_FOR;
```

正确写法：

```scl
FOR i := 0 TO 9 DO
    TotalWeight := TotalWeight + WeightValue[i];
END_FOR;
```

> [!IMPORTANT] 工控口诀  
> **TO 决定循环终点。**  
> **数组循环，范围必须对齐数组下标。**

# 03｜BY 步长循环

> [!NOTE] 卡片定位  
> `BY` 用于指定 `FOR` 循环变量每次变化的步长。  
> 不写 `BY` 时，默认步长通常按 1 递增。

> [!IMPORTANT] 底层原理  
> `BY 2` 表示每次加 2。  
> `BY -1` 表示每次减 1。  
> 步长决定循环变量变化速度。

```scl
FOR i := 1 TO 10 BY 2 DO
    // i = 1, 3, 5, 7, 9
END_FOR;
```

| 写法 | 变量变化 |
|---|---|
| `BY 1` | 每次加 1 |
| `BY 2` | 每次加 2 |
| `BY 5` | 每次加 5 |
| `BY -1` | 每次减 1 |

```scl
// 场景：只扫描奇数工位
// 工位 1、3、5、7、9 是上料工位

FOR i := 1 TO 9 BY 2 DO
    IF StationHasProduct[i] = TRUE THEN
        LoadStationCount := LoadStationCount + 1;
    END_IF;
END_FOR;
```

```scl
// 场景：倒序检查缓存区
// 从最后一个缓存位往前找第一个有数据的位置

LastValidIndex := 0;

FOR i := 10 TO 1 BY -1 DO
    IF BufferValid[i] = TRUE THEN
        LastValidIndex := i;
        EXIT;
    END_IF;
END_FOR;
```

```scl
// 场景：每隔 5 个采样点取一次数据

SampleIndex := 0;

FOR i := 1 TO 100 BY 5 DO
    SampleIndex := SampleIndex + 1;
    ReducedSample[SampleIndex] := RawSample[i];
END_FOR;
```

> [!WARNING] 使用注意  
> 步长不能写错方向。  
> 起始值小于结束值时，一般用正步长。  
> 起始值大于结束值时，一般用负步长。

> [!IMPORTANT] 工控口诀  
> **BY 是步长。**  
> **跳着扫用 BY，倒着扫用 BY -1。**

# 04｜WHILE 条件前置循环

> [!NOTE] 卡片定位  
> `WHILE` 是条件前置循环。  
> 它适合循环次数不固定，但需要根据条件持续执行的场景。

> [!IMPORTANT] 底层原理  
> `WHILE` 会先判断条件。  
> 条件为 `TRUE` 才执行循环体。  
> 条件一开始就是 `FALSE`，循环一次也不执行。

```scl
WHILE 条件 DO
    // 条件成立时重复执行
END_WHILE;
```

| 特点 | 说明 |
|---|---|
| 先判断 | 条件成立才进入 |
| 次数不固定 | 由条件决定 |
| 可能一次不执行 | 条件初始为 FALSE |
| 必须改变条件 | 否则可能死循环 |

```scl
// 场景：从缓存中查找第一个有效产品
// 找到后退出循环

i := 1;
FoundProduct := FALSE;

WHILE i <= 10 AND FoundProduct = FALSE DO

    IF BufferValid[i] = TRUE THEN
        FoundProduct := TRUE;
        ProductIndex := i;
    END_IF;

    i := i + 1;

END_WHILE;
```

```scl
// 场景：查找第一个报警通道

i := 1;
FirstAlarmIndex := 0;

WHILE i <= 32 AND FirstAlarmIndex = 0 DO

    IF AlarmList[i] = TRUE THEN
        FirstAlarmIndex := i;
    END_IF;

    i := i + 1;

END_WHILE;
```

> [!WARNING] 死循环风险  
> `WHILE` 内部必须让条件有机会变成 `FALSE`。  
> 如果忘记 `i := i + 1`，PLC 可能一直卡在循环里。

错误写法：

```scl
i := 1;

WHILE i <= 10 DO
    Total := Total + Value[i];
    // 忘记 i := i + 1
END_WHILE;
```

正确写法：

```scl
i := 1;

WHILE i <= 10 DO
    Total := Total + Value[i];
    i := i + 1;
END_WHILE;
```

> [!IMPORTANT] 工控口诀  
> **条件不确定用 WHILE。**  
> **WHILE 必须让条件变化，否则容易死循环。**

# 05｜REPEAT 后置循环

> [!NOTE] 卡片定位  
> `REPEAT` 是条件后置循环。  
> 它会先执行一次循环体，再判断 `UNTIL` 条件。

> [!IMPORTANT] 底层原理  
> `REPEAT` 至少执行一次。  
> 当 `UNTIL` 后面的条件为 `TRUE` 时，循环结束。

```scl
REPEAT
    // 至少执行一次
UNTIL 条件
END_REPEAT;
```

| 特点 | 说明 |
|---|---|
| 先执行 | 循环体先执行 |
| 后判断 | 到 UNTIL 才判断 |
| 至少一次 | 不管条件如何，至少执行一次 |
| 条件 TRUE 结束 | 与 WHILE 不同 |

```scl
// 场景：至少扫描一次缓存，直到找到有效数据或扫描到末尾

i := 1;
FoundData := FALSE;

REPEAT

    IF BufferValid[i] = TRUE THEN
        FoundData := TRUE;
        DataIndex := i;
    END_IF;

    i := i + 1;

UNTIL FoundData = TRUE OR i > 10
END_REPEAT;
```

```scl
// 场景：从 1 号配方开始查找第一个启用配方

RecipeIndex := 1;
RecipeFound := FALSE;

REPEAT

    IF RecipeEnable[RecipeIndex] = TRUE THEN
        RecipeFound := TRUE;
        ActiveRecipeNo := RecipeIndex;
    END_IF;

    RecipeIndex := RecipeIndex + 1;

UNTIL RecipeFound = TRUE OR RecipeIndex > 20
END_REPEAT;
```

> [!WARNING] 使用注意  
> `REPEAT` 至少执行一次。  
> 如果数组可能为空，或者下标可能无效，使用前要先确认范围。

> [!IMPORTANT] 工控口诀  
> **至少执行一次，用 REPEAT。**  
> **UNTIL 条件成立，循环结束。**

# 06｜EXIT 跳出循环

> [!NOTE] 卡片定位  
> `EXIT` 用于提前退出当前循环。  
> 它常用于找到目标后立即停止循环，减少扫描时间。

> [!IMPORTANT] 底层原理  
> 当程序执行到 `EXIT` 时，会立刻跳出当前 `FOR`、`WHILE` 或 `REPEAT` 循环。  
> 循环后面的代码继续执行。

```scl
FOR i := 1 TO 10 DO
    IF 条件 THEN
        EXIT;
    END_IF;
END_FOR;
```

| 使用场景 | 说明 |
|---|---|
| 找到第一个报警 | 找到后不再继续扫 |
| 找到第一个空位 | 用于缓存写入 |
| 找到目标产品 | 立即退出搜索 |
| 异常中断 | 条件不安全时退出 |

```scl
// 场景：查找第一个报警通道
// 找到后立即退出循环

FirstAlarmIndex := 0;

FOR i := 1 TO 32 DO

    IF AlarmList[i] = TRUE THEN
        FirstAlarmIndex := i;
        EXIT;
    END_IF;

END_FOR;
```

```scl
// 场景：查找第一个空缓存位，用于写入新产品数据

FreeIndex := 0;

FOR i := 1 TO 20 DO

    IF BufferValid[i] = FALSE THEN
        FreeIndex := i;
        EXIT;
    END_IF;

END_FOR;

IF FreeIndex <> 0 THEN
    BufferValue[FreeIndex] := NewProductValue;
    BufferValid[FreeIndex] := TRUE;
END_IF;
```

```scl
// 场景：扫描工位时发现急停，立即退出循环

FOR i := 1 TO 10 DO

    IF EmergencyStop = TRUE THEN
        EXIT;
    END_IF;

    StationEnable[i] := TRUE;

END_FOR;
```

> [!WARNING] 使用注意  
> `EXIT` 只跳出当前这一层循环。  
> 如果有多层嵌套循环，只会退出最内层。

> [!IMPORTANT] 工控口诀  
> **找到就走，用 EXIT。**  
> **减少无效扫描，提高程序效率。**

# 07｜CONTINUE 跳过本轮

> [!NOTE] 卡片定位  
> `CONTINUE` 用于跳过当前这一轮循环，直接进入下一轮循环。  
> 它适合跳过无效数据、禁用工位、空缓存位。

> [!IMPORTANT] 底层原理  
> 程序执行到 `CONTINUE` 后，本轮剩下的代码不再执行。  
> 循环变量进入下一次循环。

```scl
FOR i := 1 TO 10 DO
    IF 跳过条件 THEN
        CONTINUE;
    END_IF;

    // 只有未跳过时才执行
END_FOR;
```

| 使用场景 | 说明 |
|---|---|
| 跳过禁用工位 | 工位未启用不处理 |
| 跳过无效数据 | 数据无效不计算 |
| 跳过空缓存 | 空位置不统计 |
| 跳过报警通道 | 被屏蔽的不处理 |

```scl
// 场景：统计启用工位的产品数量
// 未启用工位直接跳过

EnabledProductCount := 0;

FOR i := 1 TO 10 DO

    IF StationEnable[i] = FALSE THEN
        CONTINUE;
    END_IF;

    IF StationHasProduct[i] = TRUE THEN
        EnabledProductCount := EnabledProductCount + 1;
    END_IF;

END_FOR;
```

```scl
// 场景：计算有效称重数据平均值
// 无效数据直接跳过

TotalWeight := 0.0;
ValidCount := 0;

FOR i := 1 TO 20 DO

    IF WeightValid[i] = FALSE THEN
        CONTINUE;
    END_IF;

    TotalWeight := TotalWeight + WeightValue[i];
    ValidCount := ValidCount + 1;

END_FOR;

IF ValidCount > 0 THEN
    AverageWeight := TotalWeight / INT_TO_REAL(ValidCount);
ELSE
    AverageWeight := 0.0;
END_IF;
```

> [!WARNING] 工程注意  
> 如果当前 TIA Portal / CPU 固件版本不支持 `CONTINUE`，可以用 `IF` 包住后续逻辑来实现同样效果。

等价写法：

```scl
FOR i := 1 TO 10 DO

    IF StationEnable[i] = TRUE THEN

        IF StationHasProduct[i] = TRUE THEN
            EnabledProductCount := EnabledProductCount + 1;
        END_IF;

    END_IF;

END_FOR;
```

> [!IMPORTANT] 工控口诀  
> **本轮不处理，用 CONTINUE。**  
> **不支持 CONTINUE，就用 IF 包起来。**

# 08｜RETURN 退出当前块

> [!NOTE] 卡片定位  
> `RETURN` 用于提前退出当前程序块。  
> 它不是普通循环语句，但经常用于循环前的安全检查或异常中断。

> [!IMPORTANT] 底层原理  
> `RETURN` 执行后，当前 FC / FB 后面的代码不再执行。  
> 程序直接返回调用它的上一级程序块。

```scl
IF 异常条件 THEN
    RETURN;
END_IF;
```

| 使用场景 | 说明 |
|---|---|
| 急停触发 | 当前块后续逻辑不执行 |
| 参数非法 | 直接退出计算 |
| 数组长度错误 | 防止越界 |
| 设备未使能 | 跳过控制逻辑 |

```scl
// 场景：急停时，当前控制块直接退出
// 后续运动逻辑不再执行

IF EmergencyStop = TRUE THEN
    MotorRun := FALSE;
    CylinderExtend := FALSE;
    RETURN;
END_IF;

// 后续自动控制逻辑
IF AutoMode = TRUE THEN
    MotorRun := TRUE;
END_IF;
```

```scl
// 场景：没有有效数据时，不执行平均值计算

IF ValidCount <= 0 THEN
    AverageWeight := 0.0;
    RETURN;
END_IF;

AverageWeight := TotalWeight / INT_TO_REAL(ValidCount);
```

```scl
// 场景：设备未使能时，退出当前功能块

IF Enable = FALSE THEN
    Busy := FALSE;
    Done := FALSE;
    RETURN;
END_IF;
```

> [!WARNING] 使用注意  
> `RETURN` 会直接结束当前块。  
> 如果后面还有必须执行的清理逻辑，不要随便提前 `RETURN`。

> [!IMPORTANT] 工控口诀  
> **当前块不再执行，用 RETURN。**  
> **异常保护可以提前 RETURN。**

# 09｜LOWER_BOUND 数组下限

> [!NOTE] 卡片定位  
> `LOWER_BOUND()` 用于获取数组的下标下限。  
> 它常和循环一起使用，避免数组下标写死。

> [!IMPORTANT] 底层原理  
> 数组不一定都是从 1 开始。  
> 有的数组是 `[0..9]`，有的是 `[1..10]`。  
> `LOWER_BOUND()` 可以读取数组真实起始下标。

```scl
LowIndex := LOWER_BOUND(ARR := ArrayName, DIM := 1);
```

| 好处 | 说明 |
|---|---|
| 不写死下标 | 修改数组范围后程序更安全 |
| 防止越界 | 循环范围自动跟随数组 |
| 便于复用 | 同一段逻辑适配不同数组 |

```scl
// 场景：根据数组真实下限和上限统计报警数量

LowIndex := LOWER_BOUND(ARR := AlarmList, DIM := 1);
HighIndex := UPPER_BOUND(ARR := AlarmList, DIM := 1);

AlarmCount := 0;

FOR i := LowIndex TO HighIndex DO
    IF AlarmList[i] = TRUE THEN
        AlarmCount := AlarmCount + 1;
    END_IF;
END_FOR;
```

```scl
// 场景：清空产品缓存数组

LowIndex := LOWER_BOUND(ARR := BufferValid, DIM := 1);
HighIndex := UPPER_BOUND(ARR := BufferValid, DIM := 1);

FOR i := LowIndex TO HighIndex DO
    BufferValid[i] := FALSE;
END_FOR;
```

> [!WARNING] 使用注意  
> `DIM := 1` 表示第一维数组。  
> 多维数组时，需要明确第几维。

> [!IMPORTANT] 工控口诀  
> **数组循环别写死下标。**  
> **下限用 LOWER_BOUND。**

# 10｜UPPER_BOUND 数组上限

> [!NOTE] 卡片定位  
> `UPPER_BOUND()` 用于获取数组的下标上限。  
> 它常和 `LOWER_BOUND()` 配合，用于安全遍历数组。

> [!IMPORTANT] 底层原理  
> `UPPER_BOUND()` 返回数组指定维度的最大下标。  
> 这样数组长度变化时，循环范围也能自动适配。

```scl
HighIndex := UPPER_BOUND(ARR := ArrayName, DIM := 1);
```

| 使用场景 | 说明 |
|---|---|
| 遍历数组 | 自动得到最大下标 |
| 清空数组 | 不怕数组长度变化 |
| 报警统计 | 自动扫描全部报警 |
| 数据求和 | 自动扫描全部数据 |

```scl
// 场景：计算有效重量数据平均值
// 使用 LOWER_BOUND 和 UPPER_BOUND 自动适配数组范围

LowIndex := LOWER_BOUND(ARR := WeightValue, DIM := 1);
HighIndex := UPPER_BOUND(ARR := WeightValue, DIM := 1);

TotalWeight := 0.0;
ValidCount := 0;

FOR i := LowIndex TO HighIndex DO

    IF WeightValid[i] = TRUE THEN
        TotalWeight := TotalWeight + WeightValue[i];
        ValidCount := ValidCount + 1;
    END_IF;

END_FOR;

IF ValidCount > 0 THEN
    AverageWeight := TotalWeight / INT_TO_REAL(ValidCount);
ELSE
    AverageWeight := 0.0;
END_IF;
```

```scl
// 场景：批量复位所有工位报警

LowIndex := LOWER_BOUND(ARR := StationAlarm, DIM := 1);
HighIndex := UPPER_BOUND(ARR := StationAlarm, DIM := 1);

FOR i := LowIndex TO HighIndex DO
    StationAlarm[i] := FALSE;
END_FOR;
```

> [!WARNING] 工程建议  
> 数组项目中，推荐使用 `LOWER_BOUND()` 和 `UPPER_BOUND()`。  
> 不推荐长期写死 `1 TO 10`，后期数组扩展容易漏改。

> [!IMPORTANT] 工控口诀  
> **上限用 UPPER_BOUND。**  
> **数组遍历最稳写法：LOWER_BOUND 到 UPPER_BOUND。**
