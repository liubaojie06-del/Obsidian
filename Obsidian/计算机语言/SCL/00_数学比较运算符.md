# 01｜= 等于

> [!NOTE] 卡片定位  
> `=` 是 SCL 中的**等于比较运算符**，用于判断左边和右边是否相等。  
> 它只负责判断，不负责写入数据。

> [!IMPORTANT] 底层原理  
> `=` 运算完成后，结果只有两种：  
> `TRUE`：左边等于右边  
> `FALSE`：左边不等于右边

```scl
IF A = B THEN
    Result := TRUE;
ELSE
    Result := FALSE;
END_IF;
```

> [!WARNING] 关键区别  
> `=` 是比较。  
> `:=` 是赋值。  
> 这两个符号不能混用。

```scl
// 比较
IF Mode = 1 THEN
    AutoMode := TRUE;
END_IF;

// 赋值
MotorRun := TRUE;
```

| 判断对象 | 示例 | 说明 |
|---|---|---|
| 模式号 | `Mode = 1` | 判断是否自动模式 |
| 状态码 | `DeviceState = 2` | 判断是否运行状态 |
| 故障码 | `FaultCode = 0` | 判断是否无故障 |
| 按钮 | `StartButton = TRUE` | 判断按钮是否按下 |
| 到位信号 | `SensorOK = TRUE` | 判断传感器是否有效 |
| 时间 | `RunTime = T#10s` | 判断时间是否刚好等于设定值 |

> [!TIP] 适合使用 `=` 的场景  
> 固定值判断、状态码判断、模式判断、布尔信号判断。

```scl
// 场景：设备处于自动模式，并且按下启动按钮时，启动输送线
// Mode = 1：自动模式
// StartButton = TRUE：启动按钮按下
// ConveyorRun：输送线运行输出

IF Mode = 1 THEN
    IF StartButton = TRUE THEN
        ConveyorRun := TRUE;
    ELSE
        ConveyorRun := FALSE;
    END_IF;
ELSE
    ConveyorRun := FALSE;
END_IF;
```

```scl
// 场景：设备状态码为 2 时，认为设备正在运行
// DeviceState = 0：停止
// DeviceState = 1：待机
// DeviceState = 2：运行
// DeviceState = 3：报警

IF DeviceState = 2 THEN
    MachineRunning := TRUE;
ELSE
    MachineRunning := FALSE;
END_IF;
```

```scl
// 场景：故障码为 0，允许设备启动
// FaultCode = 0：无故障
// FaultCode 非 0：有故障

IF FaultCode = 0 THEN
    FaultOK := TRUE;
ELSE
    FaultOK := FALSE;
END_IF;
```

> [!WARNING] 小数比较注意  
> `REAL`、`LREAL` 这类小数不建议直接用 `=` 判断到位。  
> 因为模拟量、位置值、计算值可能存在精度误差。

不推荐：

```scl
IF ActualPosition = TargetPosition THEN
    InPosition := TRUE;
END_IF;
```

推荐：

```scl
IF ABS(ActualPosition - TargetPosition) <= 0.2 THEN
    InPosition := TRUE;
ELSE
    InPosition := FALSE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **固定状态用 `=`，模拟量到位别用 `=`。**  
> **判断用 `=`，赋值用 `:=`。**

# 02｜<> 不等于

> [!NOTE] 卡片定位  
> `<>` 是 SCL 中的**不等于比较运算符**，用于判断左边和右边是否不同。

> [!IMPORTANT] 底层原理  
> 当两个值不相等时，结果为 `TRUE`。  
> 当两个值相等时，结果为 `FALSE`。

```scl
IF A <> B THEN
    Result := TRUE;
ELSE
    Result := FALSE;
END_IF;
```

| 判断对象 | 示例 | 说明 |
|---|---|---|
| 故障码 | `FaultCode <> 0` | 判断是否有故障 |
| 模式变化 | `Mode <> LastMode` | 判断模式是否改变 |
| 状态异常 | `DeviceState <> 2` | 判断不在运行状态 |
| 计数非零 | `ProductCount <> 0` | 判断数量不是 0 |
| 按钮未按下 | `StartButton <> TRUE` | 判断按钮不是 TRUE |

> [!TIP] 适合使用 `<>` 的场景  
> 故障判断、变化检测、异常判断、非零判断、状态不一致判断。

```scl
// 场景：变频器故障码判断
// FaultCode = 0：正常
// FaultCode <> 0：存在故障

IF FaultCode <> 0 THEN
    InverterAlarm := TRUE;
    MotorRun := FALSE;
ELSE
    InverterAlarm := FALSE;
END_IF;
```

```scl
// 场景：检测操作模式是否发生变化
// 当 Mode 与 LastMode 不一致时，重新初始化步序

IF Mode <> LastMode THEN
    InitRequest := TRUE;
    StepNo := 0;
ELSE
    InitRequest := FALSE;
END_IF;

LastMode := Mode;
```

```scl
// 场景：设备不是运行状态时，关闭运行指示灯
// DeviceState = 2 表示运行

IF DeviceState <> 2 THEN
    RunLamp := FALSE;
ELSE
    RunLamp := TRUE;
END_IF;
```

```scl
// 场景：批次数量不是 0 时，允许执行清零动作

IF BatchCount <> 0 THEN
    ResetEnable := TRUE;
ELSE
    ResetEnable := FALSE;
END_IF;
```

> [!WARNING] 常见误区  
> `<>` 不是“大于或小于”的组合判断，而是一个完整的“不等于”判断。  
> 它只关心两个值是否不同，不关心谁大谁小。

```scl
// 只判断不同
IF ActualSpeed <> SetSpeed THEN
    SpeedDifferent := TRUE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **0 多数代表正常，非 0 多数代表异常。**  
> **判断状态变化，用 `<>` 最直接。**

# 03｜> 大于

> [!NOTE] 卡片定位  
> `>` 是 SCL 中的**大于比较运算符**，用于判断左边是否严格大于右边。

> [!IMPORTANT] 底层原理  
> `>` 不包含等于。  
> 只有左边真正超过右边时，条件才成立。

```scl
IF A > B THEN
    Result := TRUE;
ELSE
    Result := FALSE;
END_IF;
```

```scl
// Temp = 80.0 时不成立
IF Temp > 80.0 THEN
    HighTemp := TRUE;
END_IF;
```

| 判断对象 | 示例 | 说明 |
|---|---|---|
| 温度 | `Temp > HighLimit` | 温度超过上限 |
| 压力 | `Pressure > MaxPressure` | 压力超过最大值 |
| 电流 | `MotorCurrent > RatedCurrent` | 电流超过额定值 |
| 速度 | `Speed > MaxSpeed` | 速度过快 |
| 液位 | `TankLevel > HighLevel` | 液位过高 |
| 偏差 | `ErrorValue > Tolerance` | 正向偏差过大 |

> [!TIP] 适合使用 `>` 的场景  
> 超限报警、超过才动作、过载预警、越界判断。

```scl
// 场景：液压站压力超过 16.0MPa 时，立即停止油泵
// OilPressure：当前油压
// PumpRun：油泵运行
// PressureAlarm：压力报警

IF OilPressure > 16.0 THEN
    PumpRun := FALSE;
    PressureAlarm := TRUE;
ELSE
    PressureAlarm := FALSE;
END_IF;
```

```scl
// 场景：电机电流大于 12A 时，触发过载预警
// MotorCurrent：电机当前电流
// CurrentWarning：电流预警

IF MotorCurrent > 12.0 THEN
    CurrentWarning := TRUE;
ELSE
    CurrentWarning := FALSE;
END_IF;
```

```scl
// 场景：输送线速度超过最大允许速度时，强制降速
// ConveyorSpeed：当前速度
// MaxSpeed：最大允许速度
// SpeedLimitActive：限速标志

IF ConveyorSpeed > MaxSpeed THEN
    ConveyorSpeedSet := MaxSpeed;
    SpeedLimitActive := TRUE;
ELSE
    ConveyorSpeedSet := ConveyorSpeed;
    SpeedLimitActive := FALSE;
END_IF;
```

```scl
// 场景：水箱液位超过 90%，触发高液位报警

IF TankLevel > 90.0 THEN
    HighLevelAlarm := TRUE;
    FillPumpRun := FALSE;
ELSE
    HighLevelAlarm := FALSE;
END_IF;
```

> [!WARNING] `>` 与 `>=` 的区别  
> `>`：超过才动作。  
> `>=`：到达就动作。

```scl
// Temp = 80.0 时不动作
IF Temp > 80.0 THEN
    Alarm := TRUE;
END_IF;

// Temp = 80.0 时动作
IF Temp >= 80.0 THEN
    Alarm := TRUE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **超过才动作，用 `>`。**  
> **到达就动作，用 `>=`。**

# 04｜< 小于

> [!NOTE] 卡片定位  
> `<` 是 SCL 中的**小于比较运算符**，用于判断左边是否严格小于右边。

> [!IMPORTANT] 底层原理  
> `<` 不包含等于。  
> 只有左边真正低于右边时，条件才成立。

```scl
IF A < B THEN
    Result := TRUE;
ELSE
    Result := FALSE;
END_IF;
```

```scl
// Level = 20.0 时不成立
IF Level < 20.0 THEN
    LowLevel := TRUE;
END_IF;
```

| 判断对象 | 示例 | 说明 |
|---|---|---|
| 液位 | `TankLevel < LowLevel` | 液位过低 |
| 压力 | `AirPressure < MinPressure` | 气压不足 |
| 温度 | `Temp < LowTemp` | 温度过低 |
| 速度 | `Speed < MinSpeed` | 速度过低 |
| 电压 | `Voltage < MinVoltage` | 电压不足 |
| 剩余时间 | `TimeLeft < 1` | 时间快结束 |

> [!TIP] 适合使用 `<` 的场景  
> 低限预警、低于才动作、压力不足、液位不足、温度未达到。

```scl
// 场景：水箱液位低于 20% 时，不允许启动水泵
// TankLevel：水箱液位百分比
// PumpEnable：水泵启动允许
// LowLevelAlarm：低液位报警

IF TankLevel < 20.0 THEN
    LowLevelAlarm := TRUE;
    PumpEnable := FALSE;
ELSE
    LowLevelAlarm := FALSE;
    PumpEnable := TRUE;
END_IF;
```

```scl
// 场景：气源压力低于 0.45MPa 时，禁止气缸动作
// AirPressure：气源压力
// CylinderEnable：气缸动作允许

IF AirPressure < 0.45 THEN
    AirPressureOK := FALSE;
    CylinderEnable := FALSE;
ELSE
    AirPressureOK := TRUE;
    CylinderEnable := TRUE;
END_IF;
```

```scl
// 场景：加热炉温度低于 60℃ 时，不允许开始生产
// ActualTemp：实际温度
// ProductionEnable：生产允许

IF ActualTemp < 60.0 THEN
    TempReady := FALSE;
    ProductionEnable := FALSE;
ELSE
    TempReady := TRUE;
    ProductionEnable := TRUE;
END_IF;
```

```scl
// 场景：变频器速度低于最低运行速度时，认为速度异常

IF MotorSpeed < MinRunSpeed THEN
    SpeedLowAlarm := TRUE;
ELSE
    SpeedLowAlarm := FALSE;
END_IF;
```

> [!WARNING] `<` 与 `<=` 的区别  
> `<`：低于才动作。  
> `<=`：低到临界值就动作。

```scl
// Level = 30.0 时不动作
IF Level < 30.0 THEN
    PumpRun := TRUE;
END_IF;

// Level = 30.0 时动作
IF Level <= 30.0 THEN
    PumpRun := TRUE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **低于才动作，用 `<`。**  
> **低到就动作，用 `<=`。**

# 05｜>= 大于等于

> [!NOTE] 卡片定位  
> `>=` 是 SCL 中的**大于等于比较运算符**，用于判断左边是否大于或等于右边。

> [!IMPORTANT] 底层原理  
> `>=` 包含临界点。  
> 左边大于右边，条件成立。  
> 左边等于右边，条件也成立。

```scl
IF A >= B THEN
    Result := TRUE;
ELSE
    Result := FALSE;
END_IF;
```

```scl
// Length = CutLength 时成立
IF Length >= CutLength THEN
    CutterStart := TRUE;
END_IF;
```

| 判断对象 | 示例 | 说明 |
|---|---|---|
| 长度 | `Length >= CutLength` | 长度到达 |
| 数量 | `Count >= TargetCount` | 数量达标 |
| 温度 | `Temp >= SetTemp` | 温度达到 |
| 压力 | `Pressure >= HighLimit` | 压力达到高限 |
| 时间 | `RunTime >= SetTime` | 时间到达 |
| 液位 | `Level >= HighLevel` | 液位达到高位 |

> [!TIP] 适合使用 `>=` 的场景  
> 到达设定值就动作、累计值达标、上限保护、时间到达、长度到达。

```scl
// 场景：输送材料长度达到设定切割长度后，停止输送并启动切刀
// Length_mm：当前长度
// CutLength_mm：设定切割长度
// ConveyorRun：输送带运行
// CutterStart：切刀启动

IF Length_mm >= CutLength_mm THEN
    ConveyorRun := FALSE;
    CutterStart := TRUE;
ELSE
    ConveyorRun := TRUE;
    CutterStart := FALSE;
END_IF;
```

```scl
// 场景：产量达到目标数量后，自动停止生产线
// ProductCount：当前产量
// TargetCount：目标产量
// BatchFinish：批次完成

IF ProductCount >= TargetCount THEN
    LineRun := FALSE;
    BatchFinish := TRUE;
ELSE
    BatchFinish := FALSE;
END_IF;
```

```scl
// 场景：加热炉温度达到设定值后，允许进入生产步骤
// ActualTemp：实际温度
// SetTemp：设定温度
// TempReady：温度到达

IF ActualTemp >= SetTemp THEN
    TempReady := TRUE;
ELSE
    TempReady := FALSE;
END_IF;
```

```scl
// 场景：电机运行时间达到维护时间后，提示保养
// MotorRunHour：电机累计运行小时
// MaintainHour：设定保养小时

IF MotorRunHour >= MaintainHour THEN
    MaintainRequest := TRUE;
ELSE
    MaintainRequest := FALSE;
END_IF;
```

> [!WARNING] 到达判断不要用 `=`  
> 位置、长度、速度、模拟量经常不会刚好等于目标值。  
> PLC 一个扫描周期后，数值可能直接越过目标点。

错误写法：

```scl
IF Length_mm = CutLength_mm THEN
    CutterStart := TRUE;
END_IF;
```

正确写法：

```scl
IF Length_mm >= CutLength_mm THEN
    CutterStart := TRUE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **长度到达、数量到达、时间到达，优先用 `>=`。**  
> **到达就动作，用 `>=`。**

# 06｜<= 小于等于

> [!NOTE] 卡片定位  
> `<=` 是 SCL 中的**小于等于比较运算符**，用于判断左边是否小于或等于右边。

> [!IMPORTANT] 底层原理  
> `<=` 包含临界点。  
> 左边小于右边，条件成立。  
> 左边等于右边，条件也成立。

```scl
IF A <= B THEN
    Result := TRUE;
ELSE
    Result := FALSE;
END_IF;
```

```scl
// Level = 30.0 时成立
IF Level <= 30.0 THEN
    PumpRun := TRUE;
END_IF;
```

| 判断对象 | 示例 | 说明 |
|---|---|---|
| 液位 | `Level <= LowLevel` | 液位低到下限 |
| 压力 | `Pressure <= MinPressure` | 压力低到下限 |
| 温度 | `Temp <= LowTemp` | 温度低到下限 |
| 时间 | `TimeLeft <= 0` | 倒计时结束 |
| 电压 | `Voltage <= MinVoltage` | 电压低到保护值 |
| 误差 | `ABS(Error) <= Tolerance` | 误差在允许范围内 |

> [!TIP] 适合使用 `<=` 的场景  
> 低限保护、低位启动、倒计时结束、误差范围判断、低到临界值就动作。

```scl
// 场景：水箱液位低于或等于 30% 时，启动补水泵
// TankLevel：水箱液位百分比
// FillPumpRun：补水泵运行

IF TankLevel <= 30.0 THEN
    FillPumpRun := TRUE;
END_IF;

IF TankLevel >= 80.0 THEN
    FillPumpRun := FALSE;
END_IF;
```

```scl
// 场景：搅拌时间倒计时结束后，自动停止搅拌机
// MixTimeLeft_s：剩余搅拌时间
// MixerRun：搅拌机运行
// MixFinish：搅拌完成

IF MixTimeLeft_s <= 0 THEN
    MixerRun := FALSE;
    MixFinish := TRUE;
ELSE
    MixerRun := TRUE;
    MixFinish := FALSE;
END_IF;
```

```scl
// 场景：气源压力低到 0.45MPa 或以下时，禁止设备启动
// AirPressure：气源压力
// MachineEnable：设备允许

IF AirPressure <= 0.45 THEN
    LowPressureAlarm := TRUE;
    MachineEnable := FALSE;
ELSE
    LowPressureAlarm := FALSE;
    MachineEnable := TRUE;
END_IF;
```

```scl
// 场景：伺服轴位置误差在 ±0.2mm 内，认为到位
// ActualPosition：实际位置
// TargetPosition：目标位置
// Tolerance：允许误差

PositionError := ActualPosition - TargetPosition;

IF ABS(PositionError) <= Tolerance THEN
    AxisInPosition := TRUE;
ELSE
    AxisInPosition := FALSE;
END_IF;
```

> [!WARNING] 低限判断选型  
> `<` 表示低于才动作。  
> `<=` 表示低到临界值就动作。  
> 在保护逻辑里，通常更常用 `<=`。

```scl
// 低液位保护，推荐包含临界点
IF TankLevel <= LowLevelLimit THEN
    PumpRun := FALSE;
    LowLevelAlarm := TRUE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **低位启动、高位停止，中间保持。**  
> **低到就动作，用 `<=`。**

# 07｜ABS() 绝对值比较

> [!NOTE] 卡片定位  
> `ABS()` 不是比较符，而是 SCL 中常用的数学函数。  
> 但它经常和 `<=` 配合，用来做**误差比较**、**到位判断**、**稳定判断**。

> [!IMPORTANT] 底层原理  
> `ABS()` 会把负数变成正数，正数保持不变。  
> 它只关心偏差大小，不关心偏差方向。

```scl
ABS(-5.0) = 5.0
ABS(5.0)  = 5.0
```

```scl
ErrorAbs := ABS(ActualValue - SetValue);
```

| 判断对象 | 推荐写法 | 说明 |
|---|---|---|
| 位置到位 | `ABS(PositionError) <= 0.2` | 误差小于等于 0.2mm |
| 温度到达 | `ABS(TempError) <= 1.0` | 温差小于等于 1℃ |
| 压力稳定 | `ABS(PressureError) <= 0.05` | 压力偏差允许 |
| 速度稳定 | `ABS(SpeedError) <= 5.0` | 速度接近目标 |
| 重量合格 | `ABS(WeightError) <= 0.1` | 重量在公差内 |

> [!TIP] 适合使用 `ABS()` 的场景  
> 模拟量接近目标、伺服位置到位、温度稳定、压力稳定、称重合格。

```scl
// 场景：伺服轴目标位置为 TargetPosition
// 实际位置与目标位置误差在 ±0.2mm 内，认为到位
// ActualPosition：实际位置
// TargetPosition：目标位置
// PositionError：位置误差
// AxisInPosition：轴到位

PositionError := ActualPosition - TargetPosition;

IF ABS(PositionError) <= 0.2 THEN
    AxisInPosition := TRUE;
ELSE
    AxisInPosition := FALSE;
END_IF;
```

```scl
// 场景：实际温度与设定温度误差在 ±1.5℃ 内，允许进入下一步
// ActualTemp：实际温度
// SetTemp：设定温度
// TempReady：温度到达

TempError := ActualTemp - SetTemp;

IF ABS(TempError) <= 1.5 THEN
    TempReady := TRUE;
ELSE
    TempReady := FALSE;
END_IF;
```

```scl
// 场景：称重结果在目标重量 ±0.05kg 内，认为合格
// ActualWeight：实际重量
// TargetWeight：目标重量
// WeightOK：重量合格

WeightError := ActualWeight - TargetWeight;

IF ABS(WeightError) <= 0.05 THEN
    WeightOK := TRUE;
ELSE
    WeightOK := FALSE;
END_IF;
```

> [!WARNING] 不推荐直接比较小数相等  
> 小数计算、模拟量采集、位置反馈都可能存在波动。  
> 到位判断不要死等于。

错误写法：

```scl
IF ActualTemp = SetTemp THEN
    TempReady := TRUE;
END_IF;
```

正确写法：

```scl
IF ABS(ActualTemp - SetTemp) <= 1.5 THEN
    TempReady := TRUE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **模拟量别死等，误差判断用 `ABS()`。**  
> **到位判断公式：`ABS(实际值 - 目标值) <= 允许误差`。**

# 08｜LIMIT() 限幅比较

> [!NOTE] 卡片定位  
> `LIMIT()` 是 SCL 中常用的限幅函数，用于把输入值限制在最小值和最大值之间。  
> 它本质上是比较运算符的封装。

> [!IMPORTANT] 底层原理  
> 输入值小于下限，输出下限。  
> 输入值大于上限，输出上限。  
> 输入值在范围内，输出原值。

```scl
OutValue := LIMIT(
    MN := MinValue,
    IN := InValue,
    MX := MaxValue
);
```

等价逻辑：

```scl
IF InValue < MinValue THEN
    OutValue := MinValue;
ELSIF InValue > MaxValue THEN
    OutValue := MaxValue;
ELSE
    OutValue := InValue;
END_IF;
```

| 应用对象 | 示例 | 说明 |
|---|---|---|
| 速度设定 | `10.0 ~ 60.0` | 防止速度过低或过高 |
| 温度设定 | `40.0 ~ 180.0` | 防止温度设定越界 |
| 压力设定 | `0.2 ~ 0.7` | 防止压力不安全 |
| 模拟量输出 | `0.0 ~ 100.0` | 防止输出超范围 |
| PID输出 | `0.0 ~ 50.0` | 限制变频器频率 |

> [!TIP] 适合使用 `LIMIT()` 的场景  
> 操作员输入保护、HMI 参数保护、模拟量输出保护、PID 输出限制。

```scl
// 场景：操作员输入输送带速度
// 要求速度不能低于 10.0，也不能高于 60.0
// OperatorSpeedSet：操作员输入速度
// ConveyorSpeedSet：最终速度设定

ConveyorSpeedSet := LIMIT(
    MN := 10.0,
    IN := OperatorSpeedSet,
    MX := 60.0
);
```

```scl
// 场景：加热炉温度设定范围为 40℃ 到 180℃
// 操作员输入超出范围时，自动限制到安全范围内

SafeTempSet := LIMIT(
    MN := 40.0,
    IN := OperatorTempSet,
    MX := 180.0
);
```

```scl
// 场景：PID 输出控制变频器频率
// 频率范围限制在 0Hz 到 50Hz

FanFrequencySet := LIMIT(
    MN := 0.0,
    IN := PID_Output,
    MX := 50.0
);
```

> [!WARNING] LIMIT() 不是报警  
> `LIMIT()` 只负责把值限制住。  
> 如果需要提示操作员参数越界，还要额外写报警逻辑。

```scl
OperatorSpeedSet_Limited := LIMIT(
    MN := 10.0,
    IN := OperatorSpeedSet,
    MX := 60.0
);

IF OperatorSpeedSet < 10.0 OR OperatorSpeedSet > 60.0 THEN
    SpeedSetAlarm := TRUE;
ELSE
    SpeedSetAlarm := FALSE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **怕参数越界，用 `LIMIT()` 卡上下限。**  
> **要报警，另外写 `<` 和 `>` 判断。**

# 09｜MIN() 最小值比较

> [!NOTE] 卡片定位  
> `MIN()` 是取最小值函数，用于从两个或多个数值中取出较小值。  
> 它的底层逻辑就是连续做“小于比较”。

> [!IMPORTANT] 底层原理  
> `MIN(A, B)` 返回 A 和 B 中较小的那个值。

```scl
OutValue := MIN(
    IN1 := A,
    IN2 := B
);
```

等价逻辑：

```scl
IF A < B THEN
    OutValue := A;
ELSE
    OutValue := B;
END_IF;
```

| 应用对象 | 示例 | 说明 |
|---|---|---|
| 速度限制 | `MIN(设定速度, 安全限速)` | 防止速度过高 |
| 压力限制 | `MIN(设定压力, 最大压力)` | 防止压力过高 |
| 温度限制 | `MIN(设定温度, 最高温度)` | 防止温度过高 |
| 输出限制 | `MIN(PID输出, 最大输出)` | 防止输出过大 |

> [!TIP] 适合使用 `MIN()` 的场景  
> 需要防止数值过大时，优先考虑 `MIN()`。

```scl
// 场景：输送线速度受到操作员设定和安全限速共同限制
// OperatorSpeed：操作员设定速度
// SafetySpeedLimit：安全限速
// FinalSpeed：最终速度

FinalSpeed := MIN(
    IN1 := OperatorSpeed,
    IN2 := SafetySpeedLimit
);
```

```scl
// 场景：温度预警时，速度不能超过 30.0
// 如果操作员设定 50.0，最终也只能输出 30.0

IF Temp >= 80.0 THEN
    FinalSpeed := MIN(
        IN1 := OperatorSpeed,
        IN2 := 30.0
    );
ELSE
    FinalSpeed := OperatorSpeed;
END_IF;
```

```scl
// 场景：压力设定不能超过设备允许最大压力
// OperatorPressureSet：操作员输入压力
// MaxSafePressure：设备安全压力上限

PressureSet := MIN(
    IN1 := OperatorPressureSet,
    IN2 := MaxSafePressure
);
```

> [!WARNING] MIN() 的方向不要用反  
> `MIN()` 是取小值，所以常用于限制最大值。  
> 如果要保证最低值，应该用 `MAX()`。

```scl
// 限制最大速度
FinalSpeed := MIN(
    IN1 := OperatorSpeed,
    IN2 := MaxSpeed
);
```

> [!IMPORTANT] 工控口诀  
> **防止过大，用 `MIN()`。**  
> **最大不能超过谁，就和谁取 `MIN()`。**

# 10｜MAX() 最大值比较

> [!IMPORTANT] 底层原理  
> `MAX(A, B)` 返回 A 和 B 中较大的那个值。

```scl
OutValue := MAX(
    IN1 := A,
    IN2 := B
);
```

等价逻辑：

```scl
IF A > B THEN

    OutValue := B;
```

| 最小开度 | `MAX(阀门输出, 最小开度)` | 防止阀门全关 |

> [!TIP] 适合使用 `MAX()` 的场景  
> 需要防止数值过小时，优先考虑 `MAX()`。

```scl
// 场景：输送带运行时，速度不能低于 10.0

FinalSpeed := MAX(
```

```scl
// 场景：风机运行时，变频器频率不能低于 15Hz
// PID_Output：PID计算输出频率
// FanFrequencySet：风机最终频率设定

IF FanRun = TRUE THEN
```

```scl
// 场景：阀门运行时，最小开度不能低于 5%

IF ValveEnable = TRUE THEN
    ValveOutput := MAX(
        IN1 := PID_ValveOutput,
```

> [!WARNING] MAX() 的方向不要用反  
> `MAX()` 是取大值，所以常用于限制最小值。

```scl
// 保证最低速度
FinalSpeed := MAX(

```

> [!IMPORTANT] 工控口诀

# 11｜IN_RANGE 范围内比较

> [!NOTE] 卡片定位  
> `IN_RANGE` 这里表示“范围内比较逻辑”。  
> 它不是所有 SCL 项目都自带的固定函数，但在工程中非常常用。  
> 可以用 `>=`、`<=` 组合实现。

> [!IMPORTANT] 底层原理  
> 判断一个值是否在下限和上限之间：

```scl
Value >= LowLimit AND Value <= HighLimit
```

| 判断对象 | 推荐写法 | 说明 |
|---|---|---|
| 温度正常 | `Temp >= LowTemp AND Temp <= HighTemp` | 温度在允许范围 |
| 压力正常 | `Pressure >= LowPressure AND Pressure <= HighPressure` | 压力正常 |
| 液位正常 | `Level >= LowLevel AND Level <= HighLevel` | 液位正常 |
| 位置允许 | `Position >= MinPos AND Position <= MaxPos` | 位置没有越界 |
| 电流正常 | `Current >= MinCurrent AND Current <= MaxCurrent` | 电流在范围内 |

> [!TIP] 适合使用范围内比较的场景  
> 正常范围判断、启动允许判断、工艺参数合格判断。

```scl
// 场景：气源压力必须在 0.45MPa 到 0.70MPa 之间
// AirPressure：当前气源压力
// AirPressureOK：气压正常
// AirPressureAlarm：气压报警

IF AirPressure >= 0.45 AND AirPressure <= 0.70 THEN
    AirPressureOK := TRUE;
    AirPressureAlarm := FALSE;
ELSE
    AirPressureOK := FALSE;
    AirPressureAlarm := TRUE;
END_IF;
```

```scl
// 场景：设备温度在 60℃ 到 80℃ 之间，才允许生产
// Temp：当前温度
// ProductionEnable：生产允许

IF Temp >= 60.0 AND Temp <= 80.0 THEN
    TempOK := TRUE;
    ProductionEnable := TRUE;
ELSE
    TempOK := FALSE;
    ProductionEnable := FALSE;
END_IF;
```

```scl
// 场景：伺服实际位置必须在机械允许范围内
// ActualPosition：实际位置
// MinPosition：最小位置
// MaxPosition：最大位置

IF ActualPosition >= MinPosition AND ActualPosition <= MaxPosition THEN
    PositionRangeOK := TRUE;
ELSE
    PositionRangeOK := FALSE;
END_IF;
```

```scl
// 场景：称重值在合格范围内，允许放行产品
// Weight：实际重量
// LowWeight：重量下限
// HighWeight：重量上限

IF Weight >= LowWeight AND Weight <= HighWeight THEN
    ProductOK := TRUE;
    RejectCylinder := FALSE;
ELSE
    ProductOK := FALSE;
    RejectCylinder := TRUE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **判断正常范围：下限用 `>=`，上限用 `<=`。**  
> **范围内比较公式：`Value >= LowLimit AND Value <= HighLimit`。**

# 12｜OUT_RANGE 范围外比较

> [!NOTE] 卡片定位  
> `OUT_RANGE` 这里表示“范围外比较逻辑”。  
> 它不是所有 SCL 项目都自带的固定函数，但在报警保护中非常常用。  
> 可以用 `<`、`>` 组合实现。

> [!IMPORTANT] 底层原理  
> 判断一个值是否超出上下限：

```scl
Value < LowLimit OR Value > HighLimit
```

| 判断对象 | 推荐写法 | 说明 |
|---|---|---|
| 温度异常 | `Temp < LowTemp OR Temp > HighTemp` | 温度越界 |
| 压力异常 | `Pressure < LowPressure OR Pressure > HighPressure` | 压力越界 |
| 液位异常 | `Level < LowLevel OR Level > HighLevel` | 液位越界 |
| 位置越界 | `Position < MinPos OR Position > MaxPos` | 位置超范围 |
| 电压异常 | `Voltage < MinVoltage OR Voltage > MaxVoltage` | 电压异常 |

> [!TIP] 适合使用范围外比较的场景  
> 越界报警、参数非法、设备保护、禁止启动。

```scl
// 场景：压力低于 0.45MPa 或高于 0.70MPa 都报警
// Pressure：当前压力
// PressureAlarm：压力报警
// MachineEnable：设备允许运行

IF Pressure < 0.45 OR Pressure > 0.70 THEN
    PressureAlarm := TRUE;
    MachineEnable := FALSE;
ELSE
    PressureAlarm := FALSE;
    MachineEnable := TRUE;
END_IF;
```

```scl
// 场景：伺服实际位置超出允许范围时，立即禁止运动
// ActualPosition：实际位置
// MinPosition：最小允许位置
// MaxPosition：最大允许位置

IF ActualPosition < MinPosition OR ActualPosition > MaxPosition THEN
    AxisEnable := FALSE;
    PositionLimitAlarm := TRUE;
ELSE
    AxisEnable := TRUE;
    PositionLimitAlarm := FALSE;
END_IF;
```

```scl
// 场景：温度低于 50℃ 或高于 90℃，禁止设备进入自动运行

IF Temp < 50.0 OR Temp > 90.0 THEN
    TempRangeAlarm := TRUE;
    AutoRunPermit := FALSE;
ELSE
    TempRangeAlarm := FALSE;
    AutoRunPermit := TRUE;
END_IF;
```

```scl
// 场景：操作员输入的目标数量不允许小于 1，也不允许大于 9999

IF TargetCount < 1 OR TargetCount > 9999 THEN
    TargetCountAlarm := TRUE;
    StartPermit := FALSE;
ELSE
    TargetCountAlarm := FALSE;
    StartPermit := TRUE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **判断异常范围：低于下限用 `<`，高于上限用 `>`。**  
> **范围外比较公式：`Value < LowLimit OR Value > HighLimit`。**

# 13｜HYSTERESIS 滞回比较

> [!NOTE] 卡片定位  
> `HYSTERESIS` 表示滞回比较，也叫死区比较。  
> 它不是单个 SCL 运算符，而是比较运算符在工控中的重要组合用法。

> [!IMPORTANT] 底层原理  
> 使用两个不同的比较点，避免输出在临界值附近频繁跳变。

```scl
低于下限：启动
高于上限：停止
中间区域：保持原状态
```

| 应用对象 | 启动点 | 停止点 | 作用 |
|---|---:|---:|---|
| 水箱补水 | `<= 30%` | `>= 80%` | 防止水泵频繁启停 |
| 温度加热 | `<= 58℃` | `>= 62℃` | 防止加热器抖动 |
| 空压机 | `<= 0.45MPa` | `>= 0.70MPa` | 防止压缩机频繁启动 |
| 风机散热 | `>= 70℃` | `<= 60℃` | 防止风机反复启停 |

> [!TIP] 适合使用滞回比较的场景  
> 水泵、加热器、风机、空压机、电磁阀等不适合频繁启停的设备。

```scl
// 场景：水箱液位低于等于 30% 时开泵
// 液位高于等于 80% 时停泵
// 中间区域保持水泵原状态，防止频繁启停

IF TankLevel <= 30.0 THEN
    FillPumpRun := TRUE;

ELSIF TankLevel >= 80.0 THEN
    FillPumpRun := FALSE;

END_IF;
```

```scl
// 场景：温度低于 58℃ 开启加热
// 温度高于 62℃ 关闭加热
// 58℃ 到 62℃ 之间保持原状态

IF ActualTemp <= 58.0 THEN
    HeaterRun := TRUE;

ELSIF ActualTemp >= 62.0 THEN
    HeaterRun := FALSE;

END_IF;
```

```scl
// 场景：空压机压力低于等于 0.45MPa 时启动
// 压力高于等于 0.70MPa 时停止

IF AirPressure <= 0.45 THEN
    CompressorRun := TRUE;

ELSIF AirPressure >= 0.70 THEN
    CompressorRun := FALSE;

END_IF;
```

```scl
// 场景：柜内温度高于等于 70℃ 时启动风机
// 温度低于等于 60℃ 时停止风机
// 中间区域保持原状态

IF CabinetTemp >= 70.0 THEN
    FanRun := TRUE;

ELSIF CabinetTemp <= 60.0 THEN
    FanRun := FALSE;

END_IF;
```

> [!WARNING] 常见错误  
> 不要只用一个比较点控制启停，否则设备容易在临界值附近反复动作。

错误写法：

```scl
IF ActualTemp < 60.0 THEN
    HeaterRun := TRUE;
ELSE
    HeaterRun := FALSE;
END_IF;
```

正确写法：

```scl
IF ActualTemp <= 58.0 THEN
    HeaterRun := TRUE;
ELSIF ActualTemp >= 62.0 THEN
    HeaterRun := FALSE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **低位启动，高位停止，中间保持。**  
> **滞回控制的关键是两个比较点。**

# 14｜COMPARE_RULES 比较选型规则

> [!NOTE] 卡片定位  
> `COMPARE_RULES` 用来汇总 SCL 数学比较运算符的选型规则。  
> 写程序时先判断需求，再选择比较符。

| 需求 | 推荐写法 | 说明 |
|---|---|---|
| 判断固定值 | `=` | 模式、状态、故障码 |
| 判断不同 | `<>` | 状态变化、异常码 |
| 超过才报警 | `>` | 严格超过上限 |
| 低于才报警 | `<` | 严格低于下限 |
| 到达就动作 | `>=` | 长度、数量、时间 |
| 低到就动作 | `<=` | 低液位、低压力 |
| 判断正常范围 | `>= AND <=` | 上下限范围 |
| 判断异常范围 | `< OR >` | 越界报警 |
| 判断模拟量到位 | `ABS() <= 公差` | 位置、温度、压力 |
| 限制上下限 | `LIMIT()` | HMI 输入、PID 输出 |
| 防止过大 | `MIN()` | 最大速度、最大压力 |
| 防止过小 | `MAX()` | 最低速度、最低频率 |
| 防止频繁启停 | `HYSTERESIS` | 死区、滞回 |

> [!IMPORTANT] 最终口诀  
> **固定值判断用 `=`**  
> **异常变化用 `<>`**  
> **超过才动作用 `>`**  
> **低于才动作用 `<`**  
> **到达就动作用 `>=`**  
> **低到就动作用 `<=`**  
> **模拟量到位用 `ABS()`**  
> **参数限幅用 `LIMIT()`**  
> **防止过大用 `MIN()`**  
> **防止过小用 `MAX()`**  
> **频繁启停用滞回比较**

```scl
// 固定值判断
IF Mode = 1 THEN
    AutoMode := TRUE;
END_IF;

// 异常判断
IF FaultCode <> 0 THEN
    Alarm := TRUE;
END_IF;

// 高限报警
IF Temp > HighLimit THEN
    HighAlarm := TRUE;
END_IF;

// 低限报警
IF Pressure < LowLimit THEN
    LowAlarm := TRUE;
END_IF;

// 到达判断
IF Length_mm >= CutLength_mm THEN
    CutterStart := TRUE;
END_IF;

// 低到判断
IF TankLevel <= LowLevel THEN
    FillPumpRun := TRUE;
END_IF;

// 正常范围
IF Value >= LowLimit AND Value <= HighLimit THEN
    ValueOK := TRUE;
ELSE
    ValueOK := FALSE;
END_IF;

// 异常范围
IF Value < LowLimit OR Value > HighLimit THEN
    ValueAlarm := TRUE;
ELSE
    ValueAlarm := FALSE;
END_IF;

// 模拟量到位
IF ABS(ActualPosition - TargetPosition) <= Tolerance THEN
    InPosition := TRUE;
ELSE
    InPosition := FALSE;
END_IF;

// 参数限幅
SafeSpeedSet := LIMIT(
    MN := MinSpeed,
    IN := OperatorSpeedSet,
    MX := MaxSpeed
);

// 防止过大
FinalSpeed := MIN(
    IN1 := OperatorSpeed,
    IN2 := SafetySpeedLimit
);

// 防止过小
FinalFrequency := MAX(
    IN1 := PID_Output,
    IN2 := MinFrequency
);

// 滞回控制
IF TankLevel <= 30.0 THEN
    FillPumpRun := TRUE;
ELSIF TankLevel >= 80.0 THEN
    FillPumpRun := FALSE;
END_IF;
```
