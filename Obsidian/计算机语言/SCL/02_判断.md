# 01｜IF 条件判断

> [!NOTE] 卡片定位  
> `IF` 是 SCL 中最常用的判断语句。  
> 当条件成立时，执行 `THEN` 后面的程序。

> [!IMPORTANT] 底层原理  
> `IF` 后面的条件表达式最终必须得到一个 `BOOL` 结果。  
> `TRUE`：执行程序。  
> `FALSE`：跳过程序。

```scl
IF 条件 THEN
    // 条件成立时执行
END_IF;
```

| 判断类型 | 示例 |
|---|---|
| 按钮判断 | `StartButton = TRUE` |
| 模式判断 | `Mode = 1` |
| 高限判断 | `Temp >= 80.0` |
| 低限判断 | `Pressure <= 0.45` |
| 无故障判断 | `FaultCode = 0` |

```scl
// 场景：启动按钮按下，并且没有故障时，启动电机

IF StartButton = TRUE AND FaultCode = 0 THEN
    MotorRun := TRUE;
END_IF;
```

```scl
// 场景：温度超过 90℃，触发高温报警

IF ActualTemp >= 90.0 THEN
    HighTempAlarm := TRUE;
END_IF;
```

> [!WARNING] 使用注意  
> `IF` 只判断条件是否成立。  
> 如果条件不成立，又没有写 `ELSE`，变量会保持之前的状态。

```scl
// 只置位，不复位，Alarm 可能一直保持 TRUE

IF ActualTemp >= 90.0 THEN
    HighTempAlarm := TRUE;
END_IF;
```

更完整写法：

```scl
IF ActualTemp >= 90.0 THEN
    HighTempAlarm := TRUE;
ELSE
    HighTempAlarm := FALSE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **有判断，用 IF。**  
> **需要明确开关状态，就写完整 IF ELSE。**

# 02｜THEN 执行入口

> [!NOTE] 卡片定位  
> `THEN` 表示条件成立后的执行入口。  
> `IF` 判断为 `TRUE` 时，PLC 才会执行 `THEN` 后面的代码。

> [!IMPORTANT] 底层原理  
> `IF 条件 THEN` 可以理解为：  
> **如果条件成立，那么执行下面动作。**

```scl
IF 条件 THEN
    动作;
END_IF;
```

| 结构 | 含义 |
|---|---|
| `IF` | 开始判断 |
| `条件` | 得到 TRUE 或 FALSE |
| `THEN` | TRUE 时进入执行 |
| `END_IF` | 判断结束 |

```scl
// 场景：气压正常时，允许气缸动作

IF AirPressure >= 0.45 THEN
    CylinderEnable := TRUE;
END_IF;
```

```scl
// 场景：产品到位后，启动夹紧气缸

IF ProductInPosition = TRUE THEN
    ClampCylinder := TRUE;
END_IF;
```

```scl
// 场景：产量达到目标值后，批次完成

IF ProductCount >= TargetCount THEN
    BatchFinish := TRUE;
    LineRun := FALSE;
END_IF;
```

> [!WARNING] 常见错误  
> `THEN` 后面不要直接写条件。  
> 条件应该写在 `IF` 和 `THEN` 中间。

错误写法：

```scl
IF THEN StartButton = TRUE
    MotorRun := TRUE;
END_IF;
```

正确写法：

```scl
IF StartButton = TRUE THEN
    MotorRun := TRUE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **IF 后面写条件，THEN 后面写动作。**

# 03｜ELSE 否则判断

> [!NOTE] 卡片定位  
> `ELSE` 表示条件不成立时执行的分支。  
> 它常用于让输出状态明确，不让变量保持旧状态。

> [!IMPORTANT] 底层原理  
> `IF` 条件为 `TRUE` 时，执行 `THEN` 分支。  
> `IF` 条件为 `FALSE` 时，执行 `ELSE` 分支。

```scl
IF 条件 THEN
    // 条件成立
ELSE
    // 条件不成立
END_IF;
```

| 用法 | 说明 |
|---|---|
| 有条件运行 | 条件成立开，条件不成立关 |
| 报警判断 | 异常报警，正常清除 |
| 允许判断 | 满足允许，不满足禁止 |
| 状态切换 | 是一种状态，否则另一种状态 |

```scl
// 场景：安全门关闭时允许运行，否则禁止运行

IF SafetyDoorClosed = TRUE THEN
    MachineEnable := TRUE;
ELSE
    MachineEnable := FALSE;
END_IF;
```

```scl
// 场景：压力低于下限时报警，否则清除报警

IF AirPressure < 0.45 THEN
    LowPressureAlarm := TRUE;
ELSE
    LowPressureAlarm := FALSE;
END_IF;
```

```scl
// 场景：自动模式时运行自动逻辑，否则关闭自动运行

IF Mode = 1 THEN
    AutoMode := TRUE;
ELSE
    AutoMode := FALSE;
    AutoRun := FALSE;
END_IF;
```

> [!WARNING] 工程建议  
> 控制输出、报警、允许位，建议写 `ELSE`。  
> 这样状态更明确，避免变量残留。

> [!IMPORTANT] 工控口诀  
> **要么开，要么关，就写 ELSE。**  
> **没有 ELSE，变量可能保持上一次状态。**

# 04｜ELSIF 多条件判断

> [!NOTE] 卡片定位  
> `ELSIF` 用于多个条件分支判断。  
> 它适合做温度等级、报警等级、设备步骤、范围分段。

> [!IMPORTANT] 底层原理  
> `ELSIF` 从上往下判断。  
> 只要某一个条件成立，执行对应分支后，后面的分支不再判断。

```scl
IF 条件1 THEN
    动作1;
ELSIF 条件2 THEN
    动作2;
ELSIF 条件3 THEN
    动作3;
ELSE
    默认动作;
END_IF;
```

| 场景 | 适合用法 |
|---|---|
| 温度分级 | 高温、预警、正常、低温 |
| 电流分级 | 严重过载、预警、正常 |
| 液位分级 | 高液位、正常、低液位 |
| 速度分级 | 超速、正常、低速 |

```scl
// 场景：温度分级判断
// >= 90：高温停机
// >= 80：预警降速
// >= 60：正常运行
// < 60：等待升温

IF ActualTemp >= 90.0 THEN
    TempLevel := 3;
    MachineRun := FALSE;
    HighTempAlarm := TRUE;

ELSIF ActualTemp >= 80.0 THEN
    TempLevel := 2;
    MachineRun := TRUE;
    SpeedSet := 30.0;
    HighTempAlarm := FALSE;

ELSIF ActualTemp >= 60.0 THEN
    TempLevel := 1;
    MachineRun := TRUE;
    SpeedSet := 50.0;
    HighTempAlarm := FALSE;

ELSE
    TempLevel := 0;
    MachineRun := FALSE;
    HeaterRun := TRUE;
    HighTempAlarm := FALSE;

END_IF;
```

> [!WARNING] 顺序很重要  
> 多级判断通常从严重到轻微、从大到小写。

错误写法：

```scl
IF ActualTemp >= 60.0 THEN
    TempLevel := 1;
ELSIF ActualTemp >= 80.0 THEN
    TempLevel := 2;
ELSIF ActualTemp >= 90.0 THEN
    TempLevel := 3;
END_IF;
```

正确写法：

```scl
IF ActualTemp >= 90.0 THEN
    TempLevel := 3;
ELSIF ActualTemp >= 80.0 THEN
    TempLevel := 2;
ELSIF ActualTemp >= 60.0 THEN
    TempLevel := 1;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **多级判断用 ELSIF。**  
> **从严重到轻微，从大到小写。**

# 05｜CASE 状态判断

> [!NOTE] 卡片定位  
> `CASE` 用于固定值判断。  
> 它特别适合模式号、状态码、步序号、故障码。

> [!IMPORTANT] 底层原理  
> `CASE` 会根据变量当前的值，进入对应分支。  
> 它适合判断离散值，不适合判断范围。

```scl
CASE 变量 OF
    值1:
        动作1;
    值2:
        动作2;
    值3:
        动作3;
ELSE
    默认动作;
END_CASE;
```

| 判断对象 | 示例 |
|---|---|
| 模式号 | `0 手动 / 1 自动 / 2 维修` |
| 状态码 | `0 停止 / 1 待机 / 2 运行 / 3 报警` |
| 步序号 | `10 初始化 / 20 夹紧 / 30 加工` |
| 故障码 | `1001 超温 / 1002 低压` |

```scl
// 场景：设备状态码判断

CASE DeviceState OF

    0:
        MotorRun := FALSE;
        GreenLamp := FALSE;
        YellowLamp := FALSE;
        RedLamp := FALSE;

    1:
        MotorRun := FALSE;
        GreenLamp := FALSE;
        YellowLamp := TRUE;
        RedLamp := FALSE;

    2:
        MotorRun := TRUE;
        GreenLamp := TRUE;
        YellowLamp := FALSE;
        RedLamp := FALSE;

    3:
        MotorRun := FALSE;
        GreenLamp := FALSE;
        YellowLamp := FALSE;
        RedLamp := TRUE;

ELSE
    MotorRun := FALSE;
    RedLamp := TRUE;

END_CASE;
```

```scl
// 场景：步序控制

CASE StepNo OF

    0:
        CylinderExtend := FALSE;
        CylinderRetract := TRUE;

        IF HomeSensor = TRUE THEN
            StepNo := 10;
        END_IF;

    10:
        CylinderRetract := FALSE;
        CylinderExtend := TRUE;

        IF ExtendSensor = TRUE THEN
            StepNo := 20;
        END_IF;

    20:
        ProcessFinish := TRUE;

ELSE
    StepNo := 0;

END_CASE;
```

> [!WARNING] IF 和 CASE 选型  
> 范围判断用 `IF / ELSIF`。  
> 固定值判断用 `CASE`。

> [!IMPORTANT] 工控口诀  
> **范围用 IF，状态用 CASE。**  
> **CASE 一定要写 ELSE，防止异常状态。**

# 06｜AND 多条件同时判断

> [!NOTE] 卡片定位  
> `AND` 用于多个条件同时成立的判断。  
> 它常用于启动允许、安全联锁、动作许可。

> [!IMPORTANT] 底层原理  
> 所有条件都是 `TRUE`，最终才是 `TRUE`。  
> 任意一个条件为 `FALSE`，最终就是 `FALSE`。

```scl
IF 条件1 AND 条件2 AND 条件3 THEN
    动作;
END_IF;
```

| 判断场景 | 常用条件 |
|---|---|
| 启动允许 | 自动模式 + 启动按钮 + 无故障 |
| 气缸伸出 | 有料 + 气压正常 + 后退到位 |
| 伺服运动 | 伺服就绪 + 无报警 + 未超限 |
| 生产允许 | 温度正常 + 压力正常 + 安全门关闭 |

```scl
// 场景：输送线启动允许判断

StartPermit :=
    Mode = 1
    AND EmergencyStop = FALSE
    AND SafetyDoorClosed = TRUE
    AND AirPressure >= 0.45
    AND FaultCode = 0;

IF StartPermit = TRUE AND StartButton = TRUE THEN
    ConveyorRun := TRUE;
ELSE
    ConveyorRun := FALSE;
END_IF;
```

```scl
// 场景：气缸伸出判断

IF ProductExist = TRUE
   AND AirPressureOK = TRUE
   AND CylinderRetracted = TRUE
   AND SafetyDoorClosed = TRUE THEN

    CylinderExtend := TRUE;

ELSE
    CylinderExtend := FALSE;
END_IF;
```

> [!WARNING] 工程建议  
> 复杂条件建议先计算成中间变量。  
> 例如 `StartPermit`、`MovePermit`、`SafetyOK`。

> [!IMPORTANT] 工控口诀  
> **启动允许用 AND。**  
> **条件缺一不可，就用 AND。**

# 07｜OR 任一条件判断

> [!NOTE] 卡片定位  
> `OR` 用于任意一个条件成立的判断。  
> 它常用于报警、停机、异常、多个来源命令。

> [!IMPORTANT] 底层原理  
> 只要有一个条件是 `TRUE`，最终就是 `TRUE`。  
> 所有条件都是 `FALSE`，最终才是 `FALSE`。

```scl
IF 条件1 OR 条件2 OR 条件3 THEN
    动作;
END_IF;
```

| 判断场景 | 常用条件 |
|---|---|
| 总报警 | 任意报警 |
| 停机判断 | 停止按钮 / 急停 / 故障 |
| 启动请求 | 本地启动 / 远程启动 |
| 越界报警 | 低于下限 / 高于上限 |

```scl
// 场景：任意故障触发总报警

GeneralAlarm :=
    HighTempAlarm
    OR LowPressureAlarm
    OR MotorOverload
    OR EmergencyStop;
```

```scl
// 场景：停止按钮、急停、故障任意一个触发，设备停止

IF StopButton = TRUE
   OR EmergencyStop = TRUE
   OR FaultCode <> 0 THEN

    MachineRun := FALSE;

END_IF;
```

```scl
// 场景：参数超出范围报警

IF SpeedSet < 10.0 OR SpeedSet > 60.0 THEN
    SpeedSetAlarm := TRUE;
ELSE
    SpeedSetAlarm := FALSE;
END_IF;
```

> [!WARNING] 使用注意  
> 启动条件一般不要用 `OR` 拼太多安全条件。  
> 启动许可大多要求全部满足，应使用 `AND`。

> [!IMPORTANT] 工控口诀  
> **报警停机用 OR。**  
> **有一个就算，就用 OR。**

# 08｜NOT 反向判断

> [!NOTE] 卡片定位  
> `NOT` 用于反向判断。  
> 它把 `TRUE` 变成 `FALSE`，把 `FALSE` 变成 `TRUE`。

> [!IMPORTANT] 底层原理  
> `NOT Fault` 表示没有故障。  
> `NOT Alarm` 表示没有报警。  
> `NOT Sensor` 表示传感器未触发。

```scl
IF NOT 条件 THEN
    动作;
END_IF;
```

| 写法 | 含义 |
|---|---|
| `NOT Fault` | 无故障 |
| `NOT Alarm` | 无报警 |
| `NOT EmergencyStop` | 急停未触发 |
| `NOT ProductSensor` | 未检测到产品 |
| `NOT SafetyDoorClosed` | 安全门未关闭 |

```scl
// 场景：没有故障时，允许启动

IF NOT Fault THEN
    FaultOK := TRUE;
ELSE
    FaultOK := FALSE;
END_IF;
```

```scl
// 场景：安全门没有关闭时报警

IF NOT SafetyDoorClosed THEN
    DoorAlarm := TRUE;
    MachineEnable := FALSE;
ELSE
    DoorAlarm := FALSE;
    MachineEnable := TRUE;
END_IF;
```

```scl
// 场景：启动允许判断

StartPermit :=
    AutoMode
    AND NOT EmergencyStop
    AND SafetyDoorClosed
    AND NOT Fault;
```

> [!WARNING] 命名建议  
> 尽量不要写双重否定。  
> 例如 `NOT NoFault` 很容易看错。

不推荐：

```scl
IF NOT NoFault THEN
    Alarm := TRUE;
END_IF;
```

推荐：

```scl
IF Fault THEN
    Alarm := TRUE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **NOT 是取反。**  
> **无故障写 `NOT Fault`，但变量命名要清楚。**

# 09｜ABS 误差判断

> [!NOTE] 卡片定位  
> `ABS()` 常用于模拟量判断。  
> 它适合判断位置、温度、压力、速度是否接近目标值。

> [!IMPORTANT] 底层原理  
> `ABS()` 取绝对值，只看误差大小，不看正负方向。  
> 到位判断常用公式：

```scl
ABS(实际值 - 目标值) <= 允许误差
```

| 场景 | 推荐判断 |
|---|---|
| 伺服到位 | `ABS(PositionError) <= 0.2` |
| 温度到达 | `ABS(TempError) <= 1.5` |
| 压力稳定 | `ABS(PressureError) <= 0.05` |
| 速度稳定 | `ABS(SpeedError) <= 5.0` |

```scl
// 场景：伺服轴位置到位判断
// 允许误差 ±0.2mm

PositionError := ActualPosition - TargetPosition;

IF ABS(PositionError) <= 0.2 THEN
    AxisInPosition := TRUE;
ELSE
    AxisInPosition := FALSE;
END_IF;
```

```scl
// 场景：实际温度接近设定温度，允许进入下一步
// 允许误差 ±1.5℃

TempError := ActualTemp - SetTemp;

IF ABS(TempError) <= 1.5 THEN
    TempReady := TRUE;
ELSE
    TempReady := FALSE;
END_IF;
```

```scl
// 场景：称重合格判断
// 实际重量在目标重量 ±0.05kg 内，认为合格

WeightError := ActualWeight - TargetWeight;

IF ABS(WeightError) <= 0.05 THEN
    WeightOK := TRUE;
ELSE
    WeightOK := FALSE;
END_IF;
```

> [!WARNING] 常见错误  
> 模拟量、位置值、温度值不建议直接用 `=` 判断。

错误写法：

```scl
IF ActualPosition = TargetPosition THEN
    AxisInPosition := TRUE;
END_IF;
```

正确写法：

```scl
IF ABS(ActualPosition - TargetPosition) <= 0.2 THEN
    AxisInPosition := TRUE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **模拟量别死等于。**  
> **到位判断用 ABS。**

# 10｜JUDGE_RULES 判断选型规则

> [!NOTE] 卡片定位  
> `JUDGE_RULES` 用于总结 SCL 判断语句选型。  
> 写程序前先判断需求，再选择判断结构。

| 需求 | 推荐写法 |
|---|---|
| 单个条件判断 | `IF` |
| 条件不成立也要处理 | `IF ELSE` |
| 多级范围判断 | `IF ELSIF ELSE` |
| 固定状态码判断 | `CASE` |
| 多个条件全部满足 | `AND` |
| 任意一个条件满足 | `OR` |
| 反向判断 | `NOT` |
| 模拟量到位 | `ABS() <= 公差` |
| 越界报警 | `< 下限 OR > 上限` |
| 正常范围 | `>= 下限 AND <= 上限` |

```scl
// 单条件判断
IF StartButton THEN
    StartRequest := TRUE;
END_IF;

// IF ELSE 判断
IF AirPressure >= 0.45 THEN
    AirPressureOK := TRUE;
ELSE
    AirPressureOK := FALSE;
END_IF;

// 多级判断
IF Temp >= 90.0 THEN
    TempLevel := 3;
ELSIF Temp >= 80.0 THEN
    TempLevel := 2;
ELSIF Temp >= 60.0 THEN
    TempLevel := 1;
ELSE
    TempLevel := 0;
END_IF;

// CASE 状态判断
CASE DeviceState OF
    0:
        MachineRun := FALSE;
    1:
        Standby := TRUE;
    2:
        MachineRun := TRUE;
    3:
        MachineRun := FALSE;
        Alarm := TRUE;
ELSE
    MachineRun := FALSE;
    Alarm := TRUE;
END_CASE;

// 正常范围判断
IF Pressure >= 0.45 AND Pressure <= 0.70 THEN
    PressureOK := TRUE;
ELSE
    PressureOK := FALSE;
END_IF;

// 异常范围判断
IF Pressure < 0.45 OR Pressure > 0.70 THEN
    PressureAlarm := TRUE;
ELSE
    PressureAlarm := FALSE;
END_IF;

// 模拟量到位判断
IF ABS(ActualPosition - TargetPosition) <= 0.2 THEN
    AxisInPosition := TRUE;
ELSE
    AxisInPosition := FALSE;
END_IF;
```

> [!IMPORTANT] 最终口诀  
> **单条件用 IF。**  
> **多级范围用 ELSIF。**  
> **固定状态用 CASE。**  
> **启动联锁用 AND。**  
> **报警停机用 OR。**  
> **无故障用 NOT。**  
> **模拟量到位用 ABS。**
