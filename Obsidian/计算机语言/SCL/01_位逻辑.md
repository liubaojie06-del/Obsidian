# 01｜BOOL 布尔位

> [!NOTE] 卡片定位  
> `BOOL` 是 SCL 位逻辑的基础类型。  
> 它只有两个状态：`TRUE` 和 `FALSE`。

> [!IMPORTANT] 底层原理  
> `BOOL` 本质是一个二进制位。  
> `TRUE` 可以理解为 1。  
> `FALSE` 可以理解为 0。

| 状态 | 含义 | 工控含义 |
|---|---|---|
| `TRUE` | 真 | 信号有效 |
| `FALSE` | 假 | 信号无效 |

```scl
StartButton : BOOL;
StopButton  : BOOL;
MotorRun    : BOOL;
Alarm       : BOOL;
```

```scl
// 场景：启动按钮按下，电机运行

IF StartButton = TRUE THEN
    MotorRun := TRUE;
ELSE
    MotorRun := FALSE;
END_IF;
```

```scl
// 场景：急停按下，设备停止
// EmergencyStop = TRUE 表示急停被按下

IF EmergencyStop = TRUE THEN
    MotorRun := FALSE;
    Alarm := TRUE;
END_IF;
```

> [!WARNING] 工程注意  
> `TRUE` 不一定代表安全，要看变量含义。  
> 例如：  
> `SafetyDoorClosed = TRUE` 表示安全门关闭，是安全状态。  
> `EmergencyStop = TRUE` 表示急停按下，是危险状态。

> [!IMPORTANT] 工控口诀  
> **位逻辑就是 TRUE / FALSE。**  
> **看清变量名，再判断 TRUE 代表安全还是危险。**

# 02｜AND 与逻辑

> [!NOTE] 卡片定位  
> `AND` 表示“与逻辑”。  
> 多个条件必须全部成立，结果才成立。

> [!IMPORTANT] 底层原理  
> `AND` 的规则是：  
> **只要有一个条件为 FALSE，整体就是 FALSE。**

| A | B | A AND B |
|---|---|---|
| TRUE | TRUE | TRUE |
| TRUE | FALSE | FALSE |
| FALSE | TRUE | FALSE |
| FALSE | FALSE | FALSE |

> [!TIP] 使用场景  
> 启动允许、安全联锁、运行条件、动作前置条件。

```scl
// 场景：自动模式、启动按钮、无故障、急停正常，电机才允许运行

IF AutoMode = TRUE
   AND StartButton = TRUE
   AND Fault = FALSE
   AND EmergencyStop = FALSE THEN

    MotorRun := TRUE;

ELSE
    MotorRun := FALSE;
END_IF;
```

```scl
// 场景：气缸伸出联锁
// 必须有料、气压正常、安全门关闭，才允许气缸伸出

CylinderExtendPermit :=
    ProductExist = TRUE
    AND AirPressureOK = TRUE
    AND SafetyDoorClosed = TRUE
    AND CylinderRetracted = TRUE;

CylinderExtend := ExtendCmd AND CylinderExtendPermit;
```

> [!WARNING] 常见错误  
> `AND` 是缺一不可。  
> 如果是任意一个故障触发报警，应使用 `OR`，不是 `AND`。

错误写法：

```scl
IF HighTempAlarm AND LowPressureAlarm THEN
    GeneralAlarm := TRUE;
END_IF;
```

正确写法：

```scl
IF HighTempAlarm OR LowPressureAlarm THEN
    GeneralAlarm := TRUE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **启动联锁用 AND。**  
> **AND 是全部满足，缺一不可。**

# 03｜OR 或逻辑

> [!NOTE] 卡片定位  
> `OR` 表示“或逻辑”。  
> 多个条件中只要有一个成立，整体就成立。

> [!IMPORTANT] 底层原理  
> `OR` 的规则是：  
> **只要有一个条件为 TRUE，整体就是 TRUE。**

| A | B | A OR B |
|---|---|---|
| TRUE | TRUE | TRUE |
| TRUE | FALSE | TRUE |
| FALSE | TRUE | TRUE |
| FALSE | FALSE | FALSE |

> [!TIP] 使用场景  
> 多故障报警、任一条件停机、多按钮控制、多传感器检测。

```scl
// 场景：任意一个故障发生，触发总报警

GeneralAlarm :=
    HighTempAlarm = TRUE
    OR LowPressureAlarm = TRUE
    OR MotorOverload = TRUE
    OR EmergencyStop = TRUE;
```

```scl
// 场景：停止按钮、急停、故障任意一个触发，电机停止

IF StopButton = TRUE
   OR EmergencyStop = TRUE
   OR Fault = TRUE THEN

    MotorRun := FALSE;

END_IF;
```

```scl
// 场景：本地启动或远程启动，任意一个有效就产生启动请求

IF LocalStartButton = TRUE OR RemoteStartCmd = TRUE THEN
    StartRequest := TRUE;
ELSE
    StartRequest := FALSE;
END_IF;
```

> [!WARNING] 常见错误  
> 启动条件一般不要乱用 `OR`，否则容易少条件也启动。

错误写法：

```scl
IF AutoMode OR StartButton OR SafetyDoorClosed THEN
    MotorRun := TRUE;
END_IF;
```

正确写法：

```scl
IF AutoMode AND StartButton AND SafetyDoorClosed THEN
    MotorRun := TRUE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **报警停机用 OR。**  
> **OR 是有一个就算。**

# 04｜NOT 非逻辑

> [!NOTE] 卡片定位  
> `NOT` 表示“非逻辑”。  
> 它用于取反，把 `TRUE` 变 `FALSE`，把 `FALSE` 变 `TRUE`。

> [!IMPORTANT] 底层原理  
> `NOT` 的规则是取反。

| A | NOT A |
|---|---|
| TRUE | FALSE |
| FALSE | TRUE |

> [!TIP] 使用场景  
> 判断无故障、未报警、未按下、未到位、未触发。

```scl
// 场景：没有故障时，允许运行

IF NOT Fault THEN
    RunPermit := TRUE;
ELSE
    RunPermit := FALSE;
END_IF;
```

```scl
// 场景：安全门未关闭时报警

IF NOT SafetyDoorClosed THEN
    DoorAlarm := TRUE;
    MachineEnable := FALSE;
ELSE
    DoorAlarm := FALSE;
    MachineEnable := TRUE;
END_IF;
```

```scl
// 场景：启动允许统一计算

StartPermit :=
    AutoMode
    AND NOT EmergencyStop
    AND SafetyDoorClosed
    AND NOT Fault;
```

> [!WARNING] 命名建议  
> 尽量使用正向变量名，避免多重否定。

不推荐：

```scl
IF NOT NoFault THEN
    Alarm := TRUE;
END_IF;
```

推荐：

```scl
IF Fault THEN
    Alarm := TRUE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **NOT 是取反。**  
> **无故障可以写 `NOT Fault`。**

# 05｜XOR 异或逻辑

> [!NOTE] 卡片定位  
> `XOR` 表示“异或逻辑”。  
> 两个条件不一样时，结果为 `TRUE`。  
> 两个条件一样时，结果为 `FALSE`。

> [!IMPORTANT] 底层原理  
> `XOR` 的规则是：  
> **相同为 FALSE，不同为 TRUE。**

| A | B | A XOR B |
|---|---|---|
| TRUE | TRUE | FALSE |
| TRUE | FALSE | TRUE |
| FALSE | TRUE | TRUE |
| FALSE | FALSE | FALSE |

> [!TIP] 使用场景  
> 双信号不一致检测、互斥按钮判断、二选一有效判断、状态变化检测。

```scl
// 场景：两个安全门开关状态不一致时报警
// 正常情况下 DoorSwitchA 和 DoorSwitchB 应该一致

IF DoorSwitchA XOR DoorSwitchB THEN
    DoorSwitchFault := TRUE;
ELSE
    DoorSwitchFault := FALSE;
END_IF;
```

```scl
// 场景：正转按钮和反转按钮只能有一个有效

IF ForwardButton XOR ReverseButton THEN
    DirectionCmdValid := TRUE;
ELSE
    DirectionCmdValid := FALSE;
END_IF;
```

```scl
// 场景：检测两个传感器状态是否不同

SensorDifferent := SensorA XOR SensorB;
```

> [!WARNING] 使用注意  
> `XOR` 普通启停逻辑中用得少。  
> 它更适合“不一致检测”和“二选一有效”。

> [!IMPORTANT] 工控口诀  
> **相同为假，不同为真。**  
> **判断两个信号不一致，用 XOR。**

# 06｜:= 位赋值

> [!NOTE] 卡片定位  
> `:=` 是 SCL 的赋值符号。  
> 在位逻辑中，它相当于把逻辑结果写入线圈、变量、M 位、Q 点或 DB 位。

> [!IMPORTANT] 底层原理  
> 右边表达式先计算。  
> 计算结果写入左边变量。

```scl
MotorRun := StartButton;
```

等价于：

```scl
IF StartButton = TRUE THEN
    MotorRun := TRUE;
ELSE
    MotorRun := FALSE;
END_IF;
```

| 写法 | 含义 |
|---|---|
| `MotorRun := TRUE` | 电机运行位置位 |
| `MotorRun := FALSE` | 电机运行位复位 |
| `%Q0.0 := MotorRun` | 把运行状态输出到 Q0.0 |
| `"DB_Alarm".HighTemp := TRUE` | DB 报警位置位 |

```scl
// 场景：启动允许成立时，直接赋值给电机运行

MotorRun :=
    AutoMode
    AND StartButton
    AND NOT Fault
    AND NOT EmergencyStop;
```

```scl
// 场景：多个报警合成总报警

GeneralAlarm :=
    HighTempAlarm
    OR LowPressureAlarm
    OR MotorOverload
    OR EmergencyStop;
```

```scl
// 场景：最终输出到物理地址

%Q0.0 := MotorRun;
%Q0.1 := GeneralAlarm;
```

> [!WARNING] 常见错误  
> `:=` 是赋值。  
> `=` 是比较。  
> 两个符号不能混用。

错误写法：

```scl
MotorRun = TRUE;
```

正确写法：

```scl
MotorRun := TRUE;
```

> [!IMPORTANT] 工控口诀  
> **比较用 `=`，赋值用 `:=`。**  
> **右边算结果，左边接结果。**

# 07｜SET_RESET 置位复位

> [!NOTE] 卡片定位  
> `SET_RESET` 表示置位和复位逻辑。  
> 在 SCL 中通常用 `IF` 把某个位写成 `TRUE` 或 `FALSE`。

> [!IMPORTANT] 底层原理  
> 置位：把位写成 `TRUE`。  
> 复位：把位写成 `FALSE`。  
> 置位后如果没有复位条件，变量会一直保持。

| 动作 | 写法 | 含义 |
|---|---|---|
| 置位 | `Bit := TRUE` | 打开 / 保持 |
| 复位 | `Bit := FALSE` | 关闭 / 清除 |

```scl
// 场景：电机启停自保持
// 启动按钮置位，停止 / 急停 / 故障复位

IF StartButton = TRUE
   AND AutoMode = TRUE
   AND Fault = FALSE
   AND EmergencyStop = FALSE THEN

    MotorRunLatch := TRUE;

END_IF;

IF StopButton = TRUE
   OR EmergencyStop = TRUE
   OR Fault = TRUE THEN

    MotorRunLatch := FALSE;

END_IF;

MotorRun := MotorRunLatch;
```

```scl
// 场景：报警保持
// 故障出现后报警保持，故障消失并按复位按钮后清除

IF MotorOverload = TRUE THEN
    MotorOverloadAlarm := TRUE;
END_IF;

IF ResetButton = TRUE AND MotorOverload = FALSE THEN
    MotorOverloadAlarm := FALSE;
END_IF;
```

> [!WARNING] 优先级规则  
> 运行保持通常复位优先：停止、急停、故障优先。  
> 报警保持通常置位优先：故障还在，复位无效。

```scl
// 复位优先：适合运行保持
IF StartButton AND StartPermit THEN
    MotorRunLatch := TRUE;
END_IF;

IF StopButton OR EmergencyStop OR Fault THEN
    MotorRunLatch := FALSE;
END_IF;
```

```scl
// 置位优先：适合报警保持
IF ResetButton AND NOT FaultSignal THEN
    FaultAlarmLatch := FALSE;
END_IF;

IF FaultSignal THEN
    FaultAlarmLatch := TRUE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **置位就是写 TRUE。**  
> **复位就是写 FALSE。**  
> **运行保持复位优先，报警保持置位优先。**

# 08｜R_TRIG 上升沿

> [!NOTE] 卡片定位  
> `R_TRIG` 是上升沿检测功能块。  
> 它用于检测信号从 `FALSE` 变成 `TRUE` 的瞬间。

> [!IMPORTANT] 底层原理  
> 上一周期为 `FALSE`，当前周期为 `TRUE`，只输出一个扫描周期的脉冲。

| 上周期 | 当前周期 | R_TRIG.Q |
|---|---|---|
| FALSE | FALSE | FALSE |
| FALSE | TRUE | TRUE |
| TRUE | TRUE | FALSE |
| TRUE | FALSE | FALSE |

> [!TIP] 使用场景  
> 按钮单次触发、产品计数、启动脉冲、复位脉冲、步序推进。

```scl
// 场景：启动按钮按下一次，只产生一个扫描周期启动脉冲

StartEdge(CLK := StartButton);

IF StartEdge.Q = TRUE THEN
    StartPulse := TRUE;
ELSE
    StartPulse := FALSE;
END_IF;
```

```scl
// 场景：光电检测到产品上升沿，产品数量加 1
// 避免光电一直遮挡时每个扫描周期都加 1

ProductEdge(CLK := ProductSensor);

IF ProductEdge.Q = TRUE THEN
    ProductCount := ProductCount + 1;
END_IF;
```

```scl
// 场景：步骤完成信号上升沿，步序加 1

StepFinishEdge(CLK := StepFinish);

IF StepFinishEdge.Q THEN
    StepNo := StepNo + 1;
END_IF;
```

> [!WARNING] 常见错误  
> 用持续信号计数，会导致数量飞涨。

错误写法：

```scl
IF ProductSensor = TRUE THEN
    ProductCount := ProductCount + 1;
END_IF;
```

正确写法：

```scl
ProductEdge(CLK := ProductSensor);

IF ProductEdge.Q THEN
    ProductCount := ProductCount + 1;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **按一次只执行一次，用 R_TRIG。**  
> **计数、步进、单次触发都用上升沿。**

# 09｜F_TRIG 下降沿

> [!NOTE] 卡片定位  
> `F_TRIG` 是下降沿检测功能块。  
> 它用于检测信号从 `TRUE` 变成 `FALSE` 的瞬间。

> [!IMPORTANT] 底层原理  
> 上一周期为 `TRUE`，当前周期为 `FALSE`，只输出一个扫描周期的脉冲。

| 上周期 | 当前周期 | F_TRIG.Q |
|---|---|---|
| FALSE | FALSE | FALSE |
| FALSE | TRUE | FALSE |
| TRUE | TRUE | FALSE |
| TRUE | FALSE | TRUE |

> [!TIP] 使用场景  
> 按钮松开检测、产品离开检测、信号断开检测、到位信号丢失检测。

```scl
// 场景：按钮松开时触发一次动作

ButtonFallEdge(CLK := Button);

IF ButtonFallEdge.Q = TRUE THEN
    ButtonReleasePulse := TRUE;
ELSE
    ButtonReleasePulse := FALSE;
END_IF;
```

```scl
// 场景：产品离开光电传感器时，触发一次后续动作

ProductLeaveEdge(CLK := ProductSensor);

IF ProductLeaveEdge.Q THEN
    ProductLeavePulse := TRUE;
ELSE
    ProductLeavePulse := FALSE;
END_IF;
```

```scl
// 场景：气缸到位信号消失瞬间，锁存报警

CylinderInPosFall(CLK := CylinderInPosition);

IF CylinderInPosFall.Q THEN
    CylinderPositionLostAlarm := TRUE;
END_IF;
```

> [!WARNING] 常见误区  
> 下降沿不是“信号为 FALSE”。  
> 它只表示从 TRUE 变 FALSE 的那个瞬间。

错误理解：

```scl
IF Sensor = FALSE THEN
    Pulse := TRUE;
END_IF;
```

正确理解：

```scl
SensorFallEdge(CLK := Sensor);

IF SensorFallEdge.Q THEN
    Pulse := TRUE;
END_IF;
```

> [!IMPORTANT] 工控口诀  
> **松开瞬间、离开瞬间、断开瞬间，用 F_TRIG。**

# 10｜MASK 位掩码

> [!NOTE] 卡片定位  
> `MASK` 表示位掩码逻辑。  
> 它常用于从 `WORD`、`DWORD` 状态字中取出某一位，或对控制字的某一位进行置位、清零、翻转。

> [!IMPORTANT] 底层原理  
> `AND Mask`：取某一位。  
> `OR Mask`：置某一位。  
> `AND NOT Mask`：清某一位。  
> `XOR Mask`：翻转某一位。

| 操作 | 写法 | 作用 |
|---|---|---|
| 取位 | `Word AND Mask` | 判断某位是否为 1 |
| 置位 | `Word OR Mask` | 把某位写成 1 |
| 清位 | `Word AND NOT Mask` | 把某位写成 0 |
| 翻转 | `Word XOR Mask` | 0 变 1，1 变 0 |

| 掩码 | 二进制位置 | 常见写法 |
|---|---|---|
| bit0 | 第 0 位 | `16#0001` |
| bit1 | 第 1 位 | `16#0002` |
| bit2 | 第 2 位 | `16#0004` |
| bit3 | 第 3 位 | `16#0008` |
| bit4 | 第 4 位 | `16#0010` |

```scl
// 场景：判断驱动器状态字 bit0 是否为 1
// bit0 = Ready

IF (StatusWord AND 16#0001) <> 0 THEN
    DriveReady := TRUE;
ELSE
    DriveReady := FALSE;
END_IF;
```

```scl
// 场景：判断驱动器状态字 bit3 是否为 1
// bit3 = Fault

IF (StatusWord AND 16#0008) <> 0 THEN
    DriveFault := TRUE;
ELSE
    DriveFault := FALSE;
END_IF;
```

```scl
// 场景：置位控制字 bit0
// bit0 = Enable

ControlWord := ControlWord OR 16#0001;
```

```scl
// 场景：清除控制字 bit2
// bit2 = ResetFault

ControlWord := ControlWord AND NOT 16#0004;
```

```scl
// 场景：翻转控制字 bit1

ControlWord := ControlWord XOR 16#0002;
```

> [!WARNING] 使用注意  
> 掩码必须和设备手册位定义一致。  
> 不要凭感觉写 `16#0004`、`16#0008`。  
> 控制字写错位，可能造成设备误动作。

> [!IMPORTANT] 工控口诀  
> **取位用 AND。**  
> **置位用 OR。**  
> **清位用 AND NOT。**  
> **翻转用 XOR。**
