#  GCC 编译器安装教程

> [!abstract] 核心结论
> GCC 是 Linux / Unix 系统中最常用的 C / C++ 编译器套件。学习 Vim 编写 C/C++ 程序之前，先安装 GCC，才能完成“写代码 → 编译 → 运行”的完整闭环。

### ① 底层原理

<div style="background-color: #F7F9FC; padding: 15px; border-radius: 8px; border-left: 5px solid #2C3E50; color: #333; margin-bottom: 10px;">
<strong>GCC 的作用：</strong>
<ul>
  <li><strong>gcc：</strong> 主要用于编译 C 语言程序。</li>
  <li><strong>g++：</strong> 主要用于编译 C++ 程序。</li>
  <li><strong>make：</strong> 用于按照 Makefile 自动化编译项目。</li>
  <li><strong>build-essential：</strong> Ubuntu / Debian 中常用的编译工具合集。</li>
</ul>
</div>

### ② 安装方法

| 系统 | 安装命令 |
| :--- | :--- |
| Ubuntu / Debian | `sudo apt update && sudo apt install -y build-essential` |
| CentOS / RHEL | `sudo yum groupinstall -y "Development Tools"` |
| Fedora | `sudo dnf groupinstall -y "Development Tools"` |
| Arch Linux | `sudo pacman -S base-devel` |
| macOS | `xcode-select --install` |
| Windows | 先安装 MSYS2，然后执行 `pacman -S mingw-w64-ucrt-x86_64-gcc` |

### ③ 验证安装

| 指令 | 作用 |
| :--- | :--- |
| `gcc --version` | 查看 C 编译器版本 |
| `g++ --version` | 查看 C++ 编译器版本 |
| `make --version` | 查看 make 工具版本 |
| `which gcc` | 查看 gcc 所在路径 |
| `which g++` | 查看 g++ 所在路径 |

### ④ 编译 C 程序

先创建文件：

| 文件名 | 内容 |
| :--- | :--- |
| `hello.c` | C 语言源文件 |

```c
#include <stdio.h>

int main() {
    printf("Hello, GCC!\n");
    return 0;
}
```

然后编译运行：

| 指令 | 作用 |
| :--- | :--- |
| `gcc hello.c -o hello` | 把 `hello.c` 编译成可执行文件 `hello` |
| `./hello` | 运行当前目录下的 `hello` 程序 |
| `gcc hello.c` | 不指定输出名，默认生成 `a.out` |
| `./a.out` | 运行默认生成的程序 |

### ⑤ 编译 C++ 程序

先创建文件：

| 文件名 | 内容 |
| :--- | :--- |
| `hello.cpp` | C++ 源文件 |

```cpp
#include <iostream>
using namespace std;

int main() {
    cout << "Hello, G++!" << endl;
    return 0;
}
```

然后编译运行：

| 指令 | 作用 |
| :--- | :--- |
| `g++ hello.cpp -o hello` | 把 `hello.cpp` 编译成可执行文件 `hello` |
| `./hello` | 运行当前目录下的 `hello` 程序 |
| `g++ hello.cpp` | 不指定输出名，默认生成 `a.out` |
| `./a.out` | 运行默认生成的程序 |

### ⑥ 常用编译参数

| 参数 | 作用 |
| :--- | :--- |
| `-o name` | 指定输出文件名 |
| `-Wall` | 开启常见警告 |
| `-Wextra` | 开启更多警告 |
| `-g` | 生成调试信息，方便 GDB 调试 |
| `-O2` | 开启二级优化 |
| `-std=c11` | 使用 C11 标准 |
| `-std=c++17` | 使用 C++17 标准 |
| `-std=c++20` | 使用 C++20 标准 |

### ⑦ 推荐编译命令

| 语言 | 推荐命令 |
| :--- | :--- |
| C 语言 | `gcc main.c -o main -Wall -Wextra -g` |
| C++17 | `g++ main.cpp -o main -Wall -Wextra -g -std=c++17` |
| C++20 | `g++ main.cpp -o main -Wall -Wextra -g -std=c++20` |

### ⑧ Vim 联动用法

| 指令 | 作用 |
| :--- | :--- |
| `vim main.c` | 用 Vim 打开 / 创建 C 文件 |
| `vim main.cpp` | 用 Vim 打开 / 创建 C++ 文件 |
| `:w` | 在 Vim 中保存代码 |
| `:!gcc main.c -o main` | 在 Vim 中调用 GCC 编译 C 文件 |
| `:!g++ main.cpp -o main` | 在 Vim 中调用 G++ 编译 C++ 文件 |
| `:!./main` | 在 Vim 中运行编译后的程序 |
| `:w \| !gcc % -o main && ./main` | 保存当前 C 文件、编译并运行 |
| `:w \| !g++ % -o main && ./main` | 保存当前 C++ 文件、编译并运行 |

### ⑨ 最小闭环流程

| 步骤 | 指令 | 说明 |
| :--- | :--- | :--- |
| 1 | `vim main.c` | 创建并编辑 C 文件 |
| 2 | `:w` | 保存代码 |
| 3 | `:!gcc main.c -o main` | 编译生成可执行文件 |
| 4 | `:!./main` | 运行程序 |
| 5 | `:w \| !gcc % -o main && ./main` | 熟练后可一键保存、编译、运行 |

> [!tip] 记忆口诀
> **先装 GCC，再写 C；`gcc` 编 C，`g++` 编 C++；`-o` 起名字，`./` 来运行。**
