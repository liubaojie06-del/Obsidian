# 01 中型PLC总览

> [!info] 核心结论
> 中型PLC适合 **中等规模设备、产线控制、多轴运动、较多I/O、通讯联网和模块化扩展** 的自动化项目。

## ① 分类/公式/规则

> [!note] 核心理解
> 小型PLC偏向单机设备和简单逻辑控制；  
> 中型PLC更适合 **模块化扩展、复杂工艺、分布式I/O、伺服/变频/阀岛通讯**。

> [!warning] 选型公式
> **中型PLC选型 = I/O点数 + 通讯协议 + 程序容量 + 扫描周期 + 运动控制 + 扩展模块 + 诊断能力**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **控制规模** | 中型设备、单条产线、工艺单元 |
| **I/O能力** | 本地I/O + 远程I/O扩展 |
| **通讯能力** | PROFINET、EtherNet/IP、Modbus、CANopen等 |
| **运动控制** | 伺服、步进、变频器、多轴同步 |
| **程序容量** | 大于小型PLC，适合复杂程序 |
| **典型应用** | 包装机、装配线、输送线、测试设备、工艺控制 |

## ③ 计算/选型示例

- 已知：  
  设备有120点I/O、6台伺服、4台变频器、1个HMI。
- 公式：  
  **点数多 + 通讯多 + 轴控多 → 中型PLC优先**
- 计算：  
  小型PLC可能I/O、通讯和程序容量不足。
- 结论：  
  应选择 **中型模块化PLC**，并预留20%左右扩展余量。

> [!tip] 记忆口诀
> 小设备用小PLC，中型产线看中型PLC；点数多、轴数多、通讯多，就别硬上小PLC。

# 02 底层原理

> [!info] 核心结论
> 中型PLC通过循环扫描程序，读取输入、执行逻辑、刷新输出，并通过通讯控制现场设备。

## ① 分类/公式/规则

> [!note] 工作链路
> PLC先读取传感器输入，再执行程序逻辑，最后输出控制信号给电磁阀、接触器、伺服、变频器、阀岛等设备。

> [!warning] 扫描流程
> **输入采样 → 程序执行 → 通讯处理 → 输出刷新 → 诊断监控**

## ② 参考表

| 环节 | 作用 |
|---|---|
| **输入采样** | 读取按钮、传感器、开关 |
| **程序执行** | 运算逻辑、互锁、顺控 |
| **通讯处理** | 与HMI、伺服、远程I/O交换数据 |
| **输出刷新** | 控制继电器、电磁阀、指示灯 |
| **故障诊断** | 检查模块、通讯、电源、程序异常 |
| **数据存储** | 保存配方、报警、运行参数 |

## ③ 计算/选型示例

- 已知：  
  设备要求传感器触发后100ms内动作。
- 公式：  
  **响应时间 = 输入滤波 + 扫描周期 + 输出响应 + 执行机构响应**
- 计算：  
  若PLC扫描周期过长或通讯周期过慢，动作会延迟。
- 结论：  
  高速动作场景需校核PLC扫描周期和I/O响应时间。

> [!tip] 记忆口诀
> PLC不是瞬间反应，输入、程序、输出都要时间。

# 03 中型PLC vs 小型PLC

> [!info] 核心结论
> 中型PLC比小型PLC更适合扩展、联网、多轴控制和复杂程序。

## ① 分类/公式/规则

> [!note] 快速区别
> 小型PLC：成本低、结构简单、适合单机。  
> 中型PLC：模块化强、通讯强、诊断强、适合产线和复杂设备。

> [!warning] 选择规则
> **简单单机 → 小型PLC**  
> **中型设备/产线/多通讯 → 中型PLC**

## ② 参考表

| 对比项 | 小型PLC | 中型PLC |
|---|---|---|
| 控制规模 | 小型单机 | **中型设备/产线** |
| 扩展能力 | 较有限 | **更强** |
| 通讯能力 | 基础通讯 | **多协议/多网络** |
| 运动控制 | 简单轴控 | **多轴控制更强** |
| 程序容量 | 较小 | **较大** |
| 诊断维护 | 基础 | **更完善** |

## ③ 计算/选型示例

- 已知：  
  项目初期只有50点I/O，但后期可能增加远程I/O、伺服和扫码枪。
- 公式：  
  **后期扩展需求强 → 中型PLC更稳妥**
- 计算：  
  小型PLC后期可能受扩展槽、通讯口和程序容量限制。
- 结论：  
  建议直接选中型PLC，避免后期换平台。

> [!tip] 记忆口诀
> 只看当前选小型，看未来扩展选中型。

# 04 系统组成

> [!info] 核心结论
> 中型PLC系统通常由 **CPU、电源、I/O模块、通讯模块、功能模块、远程I/O和HMI** 组成。

## ① 分类/公式/规则

> [!note] 组成逻辑
> CPU负责运算；  
> I/O模块连接现场；  
> 通讯模块连接设备；  
> 功能模块处理高速计数、定位、模拟量等特殊任务。

> [!warning] 系统公式
> **中型PLC系统 = CPU + 电源 + I/O + 通讯 + 功能模块 + HMI + 远程站**

## ② 参考表

| 模块 | 作用 |
|---|---|
| **CPU模块** | 程序执行和通讯管理 |
| **电源模块** | 给PLC机架和模块供电 |
| **DI/DO模块** | 开关量输入输出 |
| **AI/AO模块** | 模拟量采集和控制 |
| **通讯模块** | 连接变频器、伺服、上位机 |
| **功能模块** | 高速计数、定位、温控等 |
| **远程I/O** | 分布式现场接线 |

## ③ 计算/选型示例

- 已知：  
  现场有80DI、64DO、8AI、4AO。
- 公式：  
  **模块数量 = 点数 ÷ 单模块通道数**
- 计算：  
  若DI模块16点/块，则80DI需要5块DI模块。
- 结论：  
  选型时需统计每类I/O点数，并预留扩展空间。

> [!tip] 记忆口诀
> CPU是大脑，I/O是手脚，通讯是嘴巴，HMI是脸面。

# 05 CPU选型

> [!info] 核心结论
> CPU是中型PLC的核心，选型重点看 **程序容量、扫描速度、通讯口、运动控制能力和扩展能力**。

## ① 分类/公式/规则

> [!warning] CPU选型公式
> **CPU等级 = 程序复杂度 + I/O规模 + 通讯数量 + 轴控需求 + 数据处理需求**

## ② 参考表

| 参数 | 选型意义 |
|---|---|
| **程序容量** | 决定可写程序规模 |
| **数据存储区** | 配方、报警、运行数据 |
| **扫描速度** | 影响设备响应 |
| **通讯接口** | 以太网、串口、总线 |
| **最大扩展模块** | 决定系统扩展上限 |
| **运动控制能力** | 决定可控伺服轴数 |
| **诊断功能** | 便于维护排故 |

## ③ 计算/选型示例

- 已知：  
  设备有复杂顺控、配方管理、6轴伺服和多台变频器。
- 公式：  
  **复杂程序 + 多通讯 + 多轴 → 中高性能CPU**
- 计算：  
  低端CPU可能程序容量和通讯负载不足。
- 结论：  
  应选择性能更高的CPU，并核对最大轴数和通讯连接数。

> [!tip] 记忆口诀
> CPU别只看价格，程序、通讯、轴控都要算。

# 06 I/O模块选型

> [!info] 核心结论
> I/O模块选型必须匹配现场信号类型、电压等级、PNP/NPN、通道数和响应速度。

## ① 分类/公式/规则

> [!note] I/O分类
> DI接收开关量信号；  
> DO控制电磁阀、继电器、指示灯；  
> AI读取压力、温度、位移等连续量；  
> AO控制比例阀、变频器频率等连续输出。

> [!warning] I/O匹配公式
> **I/O可用 = 信号类型 + 电压等级 + 接线方式 + 通道数量 + 电流容量匹配**

## ② 参考表

| I/O类型 | 常见对象 |
|---|---|
| **DI** | 按钮、限位、接近开关 |
| **DO** | 电磁阀、继电器、指示灯 |
| **AI** | 压力、温度、流量、位移 |
| **AO** | 比例阀、变频器给定 |
| **高速输入** | 编码器、计数器 |
| **高速输出** | 脉冲、步进、定位 |

## ③ 计算/选型示例

- 已知：  
  现场传感器是PNP输出，PLC输入模块是NPN型。
- 公式：  
  **传感器输出类型必须匹配PLC输入类型**
- 计算：  
  PNP/NPN不匹配可能导致PLC读不到信号。
- 结论：  
  应更换匹配输入模块或使用信号转换模块。

> [!tip] 记忆口诀
> 输入先分PNP/NPN，输出先看电流够不够。

# 07 通讯能力

> [!info] 核心结论
> 中型PLC的关键优势之一是通讯能力强，能连接HMI、伺服、变频器、阀岛、机器人、扫码枪和上位系统。

## ① 分类/公式/规则

> [!warning] 通讯选型公式
> **通讯方案 = 设备协议 + 实时性要求 + 数据量 + 站点数量 + 诊断需求**

## ② 参考表

| 通讯协议 | 常见用途 |
|---|---|
| **PROFINET** | 西门子系统、远程I/O、阀岛 |
| **EtherCAT** | 多轴伺服高速同步 |
| **EtherNet/IP** | 工业以太网设备 |
| **Modbus TCP/RTU** | 仪表、变频器、第三方设备 |
| **RS485/RS232** | 扫码枪、仪表、老设备 |
| **OPC UA** | 上位机、MES、数据平台 |
| **IO-Link** | 智能传感器和小型设备 |

## ③ 计算/选型示例

- 已知：  
  PLC需要连接8台变频器、1台HMI、1套远程I/O。
- 公式：  
  **通讯连接数 ≥ 设备数量**
- 计算：  
  至少需要支持这些站点数量，并校核通讯刷新周期。
- 结论：  
  选CPU和通讯模块时必须核对最大连接数和协议支持。

> [!tip] 记忆口诀
> 协议对不上，设备连不上；连接数不够，后期扩不了。

# 08 运动控制

> [!info] 核心结论
> 中型PLC可用于伺服、步进、变频器控制，适合定位、同步、插补和简单运动控制场景。

## ① 分类/公式/规则

> [!note] 控制方式
> 简单定位可用脉冲；  
> 多轴同步优先工业以太网总线；  
> 高性能插补和电子凸轮需校核PLC运动控制功能。

> [!warning] 运动选型公式
> **运动能力 = 轴数 + 控制方式 + 同步精度 + 插补需求 + 总线周期**

## ② 参考表

| 运动需求 | 推荐关注点 |
|---|---|
| **单轴定位** | 脉冲或总线伺服 |
| **多轴同步** | 实时总线 |
| **直线插补** | PLC运动库支持 |
| **电子齿轮** | 主从同步 |
| **电子凸轮** | 包装、飞剪、追剪 |
| **回原点** | 原点开关、限位、回零方式 |

## ③ 计算/选型示例

- 已知：  
  设备有4个伺服轴，需要同步动作。
- 公式：  
  **多轴同步 → 总线运动控制优先**
- 计算：  
  脉冲接线复杂，同步精度和诊断能力有限。
- 结论：  
  可选择支持总线伺服控制的中型PLC。

> [!tip] 记忆口诀
> 单轴定位脉冲够，多轴同步看总线。

# 09 程序容量与扫描周期

> [!info] 核心结论
> 中型PLC程序越复杂、通讯越多、数据处理越多，越要关注程序容量和扫描周期。

## ① 分类/公式/规则

> [!note] 工程理解
> 扫描周期太长，设备响应会慢；  
> 程序容量不足，后期功能无法继续扩展；  
> 通讯任务过多，也会影响CPU负载。

> [!warning] 响应公式
> **设备响应时间 ≈ 输入响应 + PLC扫描周期 + 输出响应 + 执行机构响应**

## ② 参考表

| 参数 | 影响 |
|---|---|
| **程序容量** | 可开发功能多少 |
| **数据区容量** | 配方、报警、记录 |
| **扫描周期** | 控制响应速度 |
| **通讯负载** | 影响CPU资源 |
| **中断任务** | 高速事件处理 |
| **周期任务** | 固定周期控制 |

## ③ 计算/选型示例

- 已知：  
  PLC扫描周期为20ms，输入滤波10ms，输出响应5ms。
- 公式：  
  **最小响应时间 ≈ 10 + 20 + 5**
- 计算：  
  最小响应时间约 **35ms**
- 结论：  
  若设备要求10ms级高速响应，需要使用高速输入、中断或更高性能CPU。

> [!tip] 记忆口诀
> 程序越大越要看容量，动作越快越要看周期。

# 10 远程I/O

> [!info] 核心结论
> 远程I/O适合现场分散、点数多、设备距离远的中型PLC系统，可减少电柜到现场的长距离布线。

## ① 分类/公式/规则

> [!note] 使用场景
> 传感器和执行器分布在设备各工位时，可把远程I/O安装在现场附近，再用总线连接PLC。

> [!warning] 远程I/O选型公式
> **远程I/O选型 = 现场点数 + 通讯协议 + 防护等级 + 响应周期 + 诊断需求**

## ② 参考表

| 优点 | 说明 |
|---|---|
| **减少布线** | 现场短线接入远程站 |
| **模块化强** | 工位独立维护 |
| **诊断方便** | 掉站、断线可诊断 |
| **扩展灵活** | 后期加站点方便 |
| **适合产线** | 多工位分布式控制 |
| **注意周期** | 高速信号需校核响应 |

## ③ 计算/选型示例

- 已知：  
  一条产线有5个工位，每个工位约30个I/O点。
- 公式：  
  **分散点位多 → 远程I/O优先**
- 计算：  
  若所有线都拉回主柜，线缆数量多、维护困难。
- 结论：  
  可每个工位配置一个远程I/O站点。

> [!tip] 记忆口诀
> 点多分散用远程，线少柜清维护省。

# 11 HMI与数据功能

> [!info] 核心结论
> 中型PLC常与HMI配合，实现参数设定、状态显示、报警记录、配方管理和数据追溯。

## ① 分类/公式/规则

> [!note] HMI作用
> HMI不是只做按钮画面，还要显示设备状态、报警原因、手动调试、配方参数和产量统计。

> [!warning] 数据设计公式
> **HMI功能 = 操作画面 + 参数设定 + 报警记录 + 配方管理 + 趋势显示**

## ② 参考表

| 功能 | 作用 |
|---|---|
| **状态显示** | 设备运行、停止、报警 |
| **手动操作** | 调试单动作 |
| **参数设定** | 速度、时间、压力等 |
| **报警记录** | 便于追溯故障 |
| **配方管理** | 多产品快速切换 |
| **趋势曲线** | 压力、温度、速度监控 |

## ③ 计算/选型示例

- 已知：  
  设备有10种产品，每种产品参数不同。
- 公式：  
  **多产品切换 → 配方管理**
- 计算：  
  若每次人工修改参数，容易出错。
- 结论：  
  PLC和HMI应设计配方功能，保存不同产品参数。

> [!tip] 记忆口诀
> HMI不只是画面，更是调试、报警、配方和追溯入口。

# 12 安全与联锁

> [!info] 核心结论
> 中型PLC负责普通逻辑联锁，但涉及人身安全的急停、安全门、光栅、STO等应按安全规范设计。

## ① 分类/公式/规则

> [!note] 安全理解
> 普通PLC程序不能替代安全回路。  
> 安全功能需根据风险评估选择安全继电器、安全PLC、安全I/O和安全执行元件。

> [!warning] 安全规则
> **普通互锁由PLC处理，安全保护按安全系统设计**

## ② 参考表

| 安全项 | 说明 |
|---|---|
| **急停** | 紧急停止危险动作 |
| **安全门** | 开门禁止危险运动 |
| **安全光栅** | 防止人员进入危险区 |
| **STO** | 伺服安全转矩关闭 |
| **安全继电器** | 基础安全回路 |
| **安全PLC** | 复杂安全逻辑 |
| **复位逻辑** | 故障解除后安全确认 |

## ③ 计算/选型示例

- 已知：  
  伺服轴在安全门打开时必须停止输出转矩。
- 公式：  
  **安全门打开 → STO触发**
- 计算：  
  不能只靠普通PLC程序停轴。
- 结论：  
  应把安全门接入安全回路，并控制伺服STO，具体安全等级需按规范校核。

> [!tip] 记忆口诀
> PLC管逻辑，安全回路管人命。

# 13 安装与电柜设计

> [!info] 核心结论
> 中型PLC安装要重点关注电源、接地、散热、模块间距、屏蔽、强弱电分离和柜内维护空间。

## ① 分类/公式/规则

> [!note] 安装原则
> PLC不只是接线正确就行，还要保证抗干扰、散热和维护方便。  
> 通讯线、模拟量线、编码器线应远离动力线。

> [!warning] 电柜设计公式
> **稳定运行 = 稳定电源 + 良好接地 + 合理散热 + 规范布线 + 清晰标识**

## ② 参考表

| 项目 | 注意 |
|---|---|
| **24V电源** | 容量和冗余 |
| **接地** | PE、屏蔽层处理 |
| **散热** | 柜内温度、风道 |
| **强弱电分离** | 降低干扰 |
| **端子标识** | 方便维护 |
| **模块间距** | 按厂家要求 |
| **备用空间** | 方便扩展 |

## ③ 计算/选型示例

- 已知：  
  模拟量压力信号波动明显，附近有变频器动力线。
- 公式：  
  **模拟量波动 = 干扰 + 接地/屏蔽不良**
- 计算：  
  模拟量线与动力线并行过长，易受干扰。
- 结论：  
  应分开走线，使用屏蔽线，并检查接地方式。

> [!tip] 记忆口诀
> PLC怕干扰，线缆要分开；柜内要散热，接地要可靠。

# 14 常见故障排查

> [!info] 核心结论
> 中型PLC故障排查按 **电源、CPU状态、模块状态、通讯、I/O、程序、现场设备** 的顺序进行。

## ① 分类/公式/规则

> [!note] 排查顺序
> 先看PLC是否运行；  
> 再看模块是否报警；  
> 再看通讯是否掉站；  
> 最后对比程序监控和现场电压。

> [!warning] 排错公式
> **PLC异常 = 电源异常 + CPU停机 + 模块故障 + 通讯掉站 + I/O错误 + 程序逻辑问题 + 现场设备故障**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **CPU不运行** | 电源、程序错误、硬件故障 |
| **模块报警** | 接线、短路、模块损坏 |
| **通讯掉站** | 网线、地址、协议、从站故障 |
| **输入无信号** | 传感器、接线、PNP/NPN不匹配 |
| **输出不动作** | 输出点、电源、负载、公共端 |
| **自动不运行** | 条件未满足、互锁未解除 |
| **数据异常** | 地址映射、量程、通讯字节序 |

## ③ 计算/选型示例

- 已知：  
  程序里输出点已经ON，但现场电磁阀不动作。
- 公式：  
  **程序ON不等于端子有电**
- 计算：  
  需在输出端子测量电压，再检查公共端、电磁阀线圈和保险。
- 结论：  
  排查PLC输出问题必须结合现场实测。

> [!tip] 记忆口诀
> 程序亮看端子，端子有电看负载，负载不动查现场。

# 15 中型PLC快速选型表

> [!info] 核心结论
> 中型PLC选型要先统计控制规模，再确认CPU、I/O、通讯、轴控、HMI、安全和扩展余量。

## ① 分类/公式/规则

> [!note] 十步选型法
> ① 统计DI/DO/AI/AO点数  
> ② 确认伺服、变频器、阀岛数量  
> ③ 确认通讯协议和站点数量  
> ④ 估算程序复杂度和数据量  
> ⑤ 确认扫描周期和响应要求  
> ⑥ 选择CPU等级  
> ⑦ 选择本地I/O和远程I/O  
> ⑧ 选择HMI和数据功能  
> ⑨ 设计安全联锁和急停方案  
> ⑩ 预留20%左右扩展余量

> [!warning] 最终选型公式
> **推荐PLC = CPU性能 + I/O容量 + 通讯协议 + 轴控能力 + 扩展余量 + 诊断维护能力**

## ② 参考表

| 需求 | 推荐关注点 |
|---|---|
| 中型设备控制 | 模块化PLC |
| 多工位产线 | 远程I/O |
| 多轴伺服 | 运动控制能力 |
| 多变频器 | 工业以太网/现场总线 |
| 数据追溯 | HMI/数据库/OPC UA |
| 后期扩展 | 预留模块和通讯口 |
| 安全保护 | 安全继电器/安全PLC |

## ③ 计算/选型示例

- 已知：  
  项目有96DI、64DO、12AI、4AO、6轴伺服、8台变频器、1个HMI。
- 公式：  
  **PLC需求 = I/O点数 + 轴数 + 通讯站点 + 扩展余量**
- 计算：  
  I/O总点数较多，通讯和运动设备也较多，应按中型PLC系统设计。
- 结论：  
  推荐选择 **中型模块化PLC + 远程I/O + 工业以太网通讯 + HMI**，具体型号需按厂家手册校核。

> [!tip] 选型口诀
> 先数点，再数轴；再看通讯和周期；小型不够别硬撑，中型PLC更稳妥。
