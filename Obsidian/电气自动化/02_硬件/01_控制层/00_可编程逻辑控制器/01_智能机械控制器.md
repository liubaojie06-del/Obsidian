# 01 智能机械控制器总览

> [!info] 核心结论
> 智能机械控制器是在传统PLC控制基础上，集成 **逻辑控制、运动控制、通讯、数据处理、视觉/AI接口和设备诊断** 的综合控制平台。

## ① 分类/公式/规则

> [!note] 核心理解
> 普通PLC主要负责“逻辑顺序控制”；  
> 智能机械控制器更像设备大脑，既要控制动作，也要处理数据、通讯、算法和设备状态。

> [!warning] 选型公式
> **智能机械控制器选型 = I/O数量 + 轴数 + 控制周期 + 通讯协议 + 算法能力 + 数据接口 + 安全需求 + 扩展能力**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **逻辑控制** | 顺序、互锁、报警、工艺流程 |
| **运动控制** | 伺服、步进、插补、电子凸轮 |
| **通讯能力** | PROFINET、EtherCAT、Modbus、OPC UA等 |
| **数据处理** | 采集、缓存、统计、上传 |
| **智能算法** | 视觉、AI、预测维护、质量判断 |
| **人机交互** | HMI、Web界面、远程诊断 |
| **安全联锁** | 急停、安全门、STO、安全I/O |

## ③ 计算/选型示例

- 已知：  
  一台设备有12个伺服轴、80点I/O、视觉检测和MES数据上传需求。
- 公式：  
  **控制器能力 ≥ 轴数 + I/O + 通讯 + 数据处理**
- 计算：  
  普通小型PLC可能难以同时承担多轴运动、视觉通讯和数据上传。
- 结论：  
  应选择支持多轴运动控制、工业以太网和数据接口的 **智能机械控制器**，具体性能需按厂家手册校核。

> [!tip] 记忆口诀
> 普通PLC管动作，智能控制器管动作、数据和决策。

# 02 底层原理

> [!info] 核心结论
> 智能机械控制器的底层逻辑，是实时采集设备状态，按控制程序和算法计算输出，再驱动执行机构动作。

## ① 分类/公式/规则

> [!note] 工作链路
> 控制器读取传感器和通讯数据，执行逻辑、运动和算法任务，然后输出控制指令给伺服、变频器、电磁阀、机器人或视觉系统。

> [!warning] 控制链路
> **输入采集 → 程序运算 → 运动/逻辑/算法处理 → 输出控制 → 反馈校正 → 数据上传**

## ② 参考表

| 环节 | 作用 |
|---|---|
| **输入层** | 传感器、按钮、编码器、视觉结果 |
| **控制层** | 逻辑、运动、算法、互锁 |
| **输出层** | 伺服、阀岛、变频器、机器人 |
| **通讯层** | 设备联网、总线控制、MES对接 |
| **数据层** | 采集、记录、分析、上传 |
| **诊断层** | 报警、状态、寿命、维护提醒 |

## ③ 计算/选型示例

- 已知：  
  传感器检测到工件到位后，伺服轴要在20ms内响应。
- 公式：  
  **响应时间 = 输入采样 + 程序扫描 + 通讯周期 + 伺服响应**
- 计算：  
  若控制周期、总线周期或程序扫描过慢，就会影响动作响应。
- 结论：  
  高速设备需重点校核控制器实时性和总线周期。

> [!tip] 记忆口诀
> 输入要快，运算要稳，输出要准，反馈要及时。

# 03 与PLC的区别

> [!info] 核心结论
> PLC擅长逻辑控制，智能机械控制器更强调 **多轴运动、数据处理、算法集成和设备联网**。

## ① 分类/公式/规则

> [!note] 快速区别
> PLC是自动化设备的基础控制器；  
> 智能机械控制器通常是在PLC能力上增强运动、计算、通讯和智能化功能。

> [!warning] 选择规则
> **简单逻辑设备 → PLC即可**  
> **多轴、高速、数据化、智能化设备 → 智能机械控制器优先**

## ② 参考表

| 对比项 | PLC | 智能机械控制器 |
|---|---|---|
| 逻辑控制 | **强** | **强** |
| 多轴运动 | 视型号而定 | **更强** |
| 数据处理 | 一般 | **更强** |
| 算法能力 | 较弱 | **较强** |
| 视觉/AI接口 | 需扩展 | **更适合** |
| 联网能力 | 视型号而定 | **更强调开放接口** |

## ③ 计算/选型示例

- 已知：  
  设备只有简单气缸顺序动作和少量I/O。
- 公式：  
  **简单顺控 + 少量I/O → 普通PLC可满足**
- 计算：  
  不涉及高速轴控制、视觉、数据分析和MES接口。
- 结论：  
  普通PLC即可；不必过度配置智能机械控制器。

> [!tip] 记忆口诀
> 只做顺序用PLC，要做高速和智能看机械控制器。

# 04 系统架构

> [!info] 核心结论
> 智能机械控制器通常位于设备控制核心，向下连接执行层，向上连接HMI、MES、SCADA或云平台。

## ① 分类/公式/规则

> [!note] 架构理解
> 它既要完成现场实时控制，也要把设备状态、产量、报警、质量数据上传到上位系统。

> [!warning] 标准架构
> **传感器/执行器 → 控制器 → HMI/SCADA/MES → 数据平台/云端**

## ② 参考表

| 层级 | 典型对象 |
|---|---|
| **现场层** | 传感器、阀岛、伺服、变频器 |
| **控制层** | 智能机械控制器 |
| **操作层** | HMI、触摸屏、按钮盒 |
| **管理层** | SCADA、MES、数据库 |
| **分析层** | 质量分析、预测维护、能耗分析 |
| **远程层** | VPN、远程诊断、云平台 |

## ③ 计算/选型示例

- 已知：  
  设备需要本地HMI操作，同时上传产量和报警到MES。
- 公式：  
  **本地控制 + 上位通讯 = 控制器需具备多接口能力**
- 计算：  
  控制器需同时支持HMI通讯和MES数据接口。
- 结论：  
  应选择支持工业以太网、OPC UA或数据库网关的控制器，具体接口需按项目要求校核。

> [!tip] 记忆口诀
> 下控设备，上连系统；中间控制器，既要实时也要联网。

# 05 硬件组成

> [!info] 核心结论
> 智能机械控制器硬件由CPU、I/O、通讯接口、运动控制接口、存储、电源和扩展模块组成。

## ① 分类/公式/规则

> [!note] 硬件理解
> CPU决定运算和实时能力；  
> I/O决定现场接点数量；  
> 通讯决定能连接多少设备；  
> 存储决定数据记录和程序容量。

> [!warning] 硬件选型公式
> **硬件能力 = CPU性能 + I/O点数 + 通讯端口 + 轴控能力 + 存储容量 + 扩展槽位**

## ② 参考表

| 硬件模块 | 作用 |
|---|---|
| **CPU单元** | 程序运算和任务调度 |
| **DI/DO模块** | 开关量输入输出 |
| **AI/AO模块** | 模拟量采集和控制 |
| **高速计数** | 编码器、脉冲输入 |
| **运动控制接口** | 伺服、步进、多轴控制 |
| **通讯接口** | 以太网、串口、现场总线 |
| **存储单元** | 数据记录、配方、程序 |

## ③ 计算/选型示例

- 已知：  
  设备需要48点输入、32点输出、8路模拟量和6轴伺服。
- 公式：  
  **控制器配置 ≥ 实际I/O + 预留余量**
- 计算：  
  推荐至少预留10%~20% I/O和通讯扩展余量。
- 结论：  
  选型时不要刚好卡满点数，后期调试和改造会很困难。

> [!tip] 记忆口诀
> CPU看速度，I/O看点数，通讯看协议，扩展看余量。

# 06 软件与编程

> [!info] 核心结论
> 智能机械控制器的软件能力决定设备开发效率，重点看编程语言、运动库、诊断工具和数据功能。

## ① 分类/公式/规则

> [!note] 软件理解
> 除了梯形图，智能机械控制器还可能支持结构化文本、功能块、运动控制库、脚本、C/C++、Python或边缘应用环境。

> [!warning] 软件选型公式
> **软件能力 = 编程语言 + 调试工具 + 运动库 + 通讯库 + 数据接口 + 版本管理**

## ② 参考表

| 软件能力 | 作用 |
|---|---|
| **梯形图LD** | 逻辑控制直观 |
| **结构化文本ST** | 复杂算法更方便 |
| **功能块FB** | 模块化复用 |
| **运动库** | 插补、电子凸轮、同步控制 |
| **通讯库** | 对接机器人、视觉、MES |
| **诊断工具** | 报警、趋势、在线监控 |
| **配方管理** | 多产品工艺参数切换 |

## ③ 计算/选型示例

- 已知：  
  设备有复杂配方、数据统计和多轴联动。
- 公式：  
  **复杂设备 → 需要模块化程序 + 数据管理**
- 计算：  
  仅用简单梯形图维护困难，应使用功能块和结构化程序。
- 结论：  
  控制器应支持模块化编程和配方数据管理。

> [!tip] 记忆口诀
> 小程序看梯形图，大系统看模块化。

# 07 运动控制能力

> [!info] 核心结论
> 智能机械控制器常用于多轴设备，运动控制能力是选型关键。

## ① 分类/公式/规则

> [!note] 运动功能
> 常见功能包括点位运动、速度控制、回原点、插补、电子齿轮、电子凸轮、同步轴、飞剪、追剪等。

> [!warning] 运动选型公式
> **运动控制能力 = 轴数 + 同步精度 + 插补类型 + 总线周期 + 运动库功能**

## ② 参考表

| 功能 | 应用 |
|---|---|
| **点位控制** | 滑台、转盘、送料 |
| **速度控制** | 输送线、卷绕 |
| **直线插补** | XY平台 |
| **圆弧插补** | 涂胶、切割 |
| **电子齿轮** | 主从轴同步 |
| **电子凸轮** | 包装、飞剪、追剪 |
| **回原点** | 建立坐标基准 |

## ③ 计算/选型示例

- 已知：  
  设备有X/Y/Z三轴，需要走直线轨迹。
- 公式：  
  **多轴轨迹运动 → 需要插补控制**
- 计算：  
  单轴点动无法保证三轴协调运动。
- 结论：  
  应选择支持多轴插补的智能机械控制器。

> [!tip] 记忆口诀
> 单轴看定位，多轴看同步；轨迹要漂亮，插补少不了。

# 08 I/O与现场设备

> [!info] 核心结论
> I/O是控制器连接现场设备的基础，必须匹配传感器、按钮、电磁阀、继电器和模拟量设备。

## ① 分类/公式/规则

> [!note] I/O理解
> DI接收开关量信号，DO控制电磁阀和继电器；  
> AI读取压力、温度、位移等连续量；  
> AO控制比例阀、变频器频率等连续输出。

> [!warning] I/O匹配公式
> **I/O可用 = 信号类型 + 电压等级 + PNP/NPN + 电流容量 + 响应速度匹配**

## ② 参考表

| I/O类型 | 典型应用 |
|---|---|
| **DI** | 按钮、传感器、限位 |
| **DO** | 电磁阀、指示灯、继电器 |
| **AI** | 压力、温度、位移传感器 |
| **AO** | 比例阀、变频器给定 |
| **高速输入** | 编码器、计数器 |
| **高速输出** | 脉冲控制、步进驱动 |

## ③ 计算/选型示例

- 已知：  
  现场传感器为PNP输出，但控制器输入模块为NPN型。
- 公式：  
  **传感器输出类型必须匹配输入模块**
- 计算：  
  PNP/NPN不匹配会导致输入无法正确识别。
- 结论：  
  应更换输入模块、传感器或增加信号转换模块。

> [!tip] 记忆口诀
> 输入看PNP/NPN，输出看电流容量。

# 09 通讯协议

> [!info] 核心结论
> 智能机械控制器的通讯能力决定它能否连接伺服、阀岛、视觉、机器人、HMI和MES系统。

## ① 分类/公式/规则

> [!note] 通讯理解
> 实时总线用于高速控制，开放协议用于数据交换。  
> 运动控制看实时性，MES对接看开放性。

> [!warning] 通讯选型公式
> **通讯方案 = 设备协议 + 实时性要求 + 数据量 + 拓扑结构 + 诊断需求**

## ② 参考表

| 协议 | 常见用途 |
|---|---|
| **EtherCAT** | 多轴运动、高实时性 |
| **PROFINET** | 西门子系统集成 |
| **EtherNet/IP** | 工业以太网控制 |
| **Modbus TCP/RTU** | 通用设备通讯 |
| **IO-Link** | 传感器/小型设备参数化 |
| **OPC UA** | 上位系统、MES、数据交换 |
| **CANopen** | 运动控制、嵌入式设备 |

## ③ 计算/选型示例

- 已知：  
  设备有16个伺服轴，要求高速同步。
- 公式：  
  **多轴同步 → 需要高实时性总线**
- 计算：  
  普通以太网通讯可能无法满足同步周期和抖动要求。
- 结论：  
  可优先评估 **EtherCAT等实时运动总线**，具体周期按设备节拍校核。

> [!tip] 记忆口诀
> 控运动看实时，传数据看开放；协议不匹配，设备难集成。

# 10 数据采集与追溯

> [!info] 核心结论
> 智能机械控制器不仅控制设备，还要记录产量、报警、节拍、质量、能耗和关键工艺参数。

## ① 分类/公式/规则

> [!note] 数据理解
> 数据采集用于设备分析、质量追溯、生产统计和预测维护。  
> 关键是采什么、多久采一次、存多久、传给谁。

> [!warning] 数据设计公式
> **数据方案 = 数据点 + 采样周期 + 存储容量 + 上传方式 + 追溯规则**

## ② 参考表

| 数据类型 | 用途 |
|---|---|
| **产量数据** | 统计生产数量 |
| **节拍数据** | 分析设备效率 |
| **报警记录** | 故障追溯 |
| **质量数据** | 产品追溯 |
| **工艺参数** | 配方和过程记录 |
| **能耗数据** | 节能分析 |
| **维护数据** | 寿命和保养提醒 |

## ③ 计算/选型示例

- 已知：  
  每秒采集100个变量，每个变量4字节，连续存储1小时。
- 公式：  
  **存储量 = 变量数 × 字节数 × 采样次数**
- 计算：  
  存储量 = 100 × 4 × 3600 = **1,440,000字节 ≈ 1.44MB**
- 结论：  
  存储量不大，但长期追溯需考虑文件管理、时间戳和数据上传机制。

> [!tip] 记忆口诀
> 数据不是越多越好，关键是能追溯、能分析、能定位问题。

# 11 视觉与AI集成

> [!info] 核心结论
> 智能机械控制器可与视觉系统、AI检测和边缘计算模块配合，实现定位、识别、测量、缺陷检测和工艺优化。

## ① 分类/公式/规则

> [!note] 集成理解
> 视觉系统负责看，控制器负责判断和执行。  
> AI模型可以用于质量分类、缺陷识别、异常预测，但必须与设备节拍匹配。

> [!warning] 集成公式
> **视觉/AI可用 = 检测时间 + 通讯时间 + 控制响应 ≤ 设备节拍**

## ② 参考表

| 功能 | 应用 |
|---|---|
| **视觉定位** | 机器人抓取、对位 |
| **尺寸测量** | 长度、宽度、角度检测 |
| **缺陷检测** | 划伤、缺料、脏污 |
| **条码识别** | 产品追溯 |
| **AI分类** | OK/NG判定 |
| **边缘计算** | 本地推理、低延迟处理 |

## ③ 计算/选型示例

- 已知：  
  设备节拍为500ms，视觉拍照和算法处理需要350ms，通讯需要80ms。
- 公式：  
  **总检测时间 = 350 + 80**
- 计算：  
  总检测时间 = **430ms**
- 结论：  
  理论可满足500ms节拍，但还需留控制输出和机构动作余量。

> [!tip] 记忆口诀
> 视觉负责看，AI负责判，控制器负责动；节拍不够，一切白忙。

# 12 安全控制

> [!info] 核心结论
> 智能机械控制器必须与安全回路配合，处理急停、安全门、光栅、STO和安全复位。

## ① 分类/公式/规则

> [!note] 安全理解
> 普通控制程序不能替代安全系统。  
> 涉及人身安全的急停、安全门、光栅、双手按钮等，需按实际安全规范设计。

> [!warning] 安全公式
> **安全控制 = 风险评估 + 安全输入 + 安全逻辑 + 安全输出 + 复位确认**

## ② 参考表

| 安全元件 | 作用 |
|---|---|
| **急停按钮** | 紧急停止设备 |
| **安全门开关** | 防止开门运行 |
| **安全光栅** | 防止人员进入危险区 |
| **STO** | 伺服安全转矩关闭 |
| **安全继电器** | 基础安全逻辑 |
| **安全PLC** | 复杂安全控制 |
| **安全阀** | 气动/液压能量切断 |

## ③ 计算/选型示例

- 已知：  
  设备有伺服轴，开门后必须防止轴继续输出转矩。
- 公式：  
  **安全门打开 → STO触发 → 伺服停止输出转矩**
- 计算：  
  不能只靠普通程序停轴，必须接入安全链路。
- 结论：  
  应配置安全门输入、STO和安全复位逻辑，安全等级需按实际规范校核。

> [!tip] 记忆口诀
> 普通程序管动作，安全回路管人命。

# 13 冗余与可靠性

> [!info] 核心结论
> 对关键设备，智能机械控制器要考虑电源、通讯、程序、数据和故障恢复的可靠性。

## ① 分类/公式/规则

> [!note] 可靠性理解
> 可靠性不是只看控制器品牌，还要看供电、接地、散热、备份、抗干扰和维护方案。

> [!warning] 可靠性公式
> **系统可靠 = 稳定供电 + 良好接地 + 合理散热 + 程序备份 + 通讯诊断 + 故障恢复**

## ② 参考表

| 项目 | 重点 |
|---|---|
| **电源冗余** | 防止单电源故障 |
| **程序备份** | 便于快速恢复 |
| **参数备份** | 配方、轴参数、通讯参数 |
| **通讯诊断** | 快速定位掉站 |
| **UPS** | 防止突然断电丢数据 |
| **散热设计** | 控制柜温度管理 |
| **接地屏蔽** | 抗干扰能力 |

## ③ 计算/选型示例

- 已知：  
  设备断电后配方数据偶尔丢失。
- 公式：  
  **数据可靠 = 非易失存储 + 正常写入 + 掉电保护**
- 计算：  
  频繁写存储或断电瞬间写入失败都可能导致数据异常。
- 结论：  
  应优化数据保存机制，并考虑UPS或掉电保护策略。

> [!tip] 记忆口诀
> 控制器再强，也怕电源乱、接地差、参数没备份。

# 14 调试流程

> [!info] 核心结论
> 智能机械控制器调试要按“硬件检查、通讯联调、单点测试、单轴测试、单动作、联动、节拍、安全”的顺序进行。

## ① 分类/公式/规则

> [!note] 调试原则
> 不要一上来跑自动程序。  
> 先把每个输入、输出、轴、阀、传感器、通讯设备逐一验证，再做联动。

> [!warning] 调试流程
> **硬件接线 → I/O点检 → 通讯测试 → 单轴点动 → 单动作 → 联动 → 自动循环 → 安全验证**

## ② 参考表

| 步骤 | 检查内容 |
|---|---|
| **接线检查** | 电源、接地、端子、屏蔽 |
| **I/O点检** | 输入输出地址和逻辑 |
| **通讯测试** | 伺服、阀岛、视觉、HMI |
| **轴调试** | 方向、原点、限位、软限位 |
| **动作调试** | 气缸、夹爪、模组、机构 |
| **联动调试** | 顺序、互锁、超时 |
| **安全测试** | 急停、门锁、STO、复位 |

## ③ 计算/选型示例

- 已知：  
  自动运行时动作混乱，但手动单动作还没测试完。
- 公式：  
  **自动异常 = 单点未验证 + 联动逻辑未确认**
- 计算：  
  未确认单个动作和反馈，就直接跑自动，问题难定位。
- 结论：  
  应退回手动模式，先完成单点和单动作调试。

> [!tip] 记忆口诀
> 先单点，再单轴；先单动，再联动；最后才自动。

# 15 常见故障排查

> [!info] 核心结论
> 智能机械控制器故障排查按 **电源、通讯、I/O、程序、轴控制、现场设备、数据接口** 的顺序定位。

## ① 分类/公式/规则

> [!note] 排查顺序
> 先看控制器是否正常运行，再看通讯站点；  
> 先看I/O状态，再看程序逻辑；  
> 先单独测试设备，再排查联动逻辑。

> [!warning] 排错公式
> **系统异常 = 电源问题 + 通讯掉站 + I/O错误 + 程序逻辑错误 + 参数错误 + 现场设备故障**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **控制器不运行** | 电源、CPU故障、程序错误 |
| **通讯掉站** | 网线、地址、协议、从站故障 |
| **输入无信号** | 接线、PNP/NPN、传感器故障 |
| **输出不动作** | 输出点、继电器、电磁阀、负载故障 |
| **伺服报警** | 参数、限位、编码器、过载 |
| **自动不循环** | 条件未满足、互锁未解除 |
| **数据不上报** | 网络、协议、权限、字段映射错误 |

## ③ 计算/选型示例

- 已知：  
  程序输出已经ON，但电磁阀不动作。
- 公式：  
  **输出ON不动作 = 输出模块 + 接线 + 负载 + 电源任一异常**
- 计算：  
  应在端子处实测电压，再检查电磁阀线圈和公共端。
- 结论：  
  不要只看程序监控，必须结合现场电压和负载状态判断。

> [!tip] 记忆口诀
> 程序亮不代表现场动，端子有电才算真正输出。

# 16 智能机械控制器快速选型表

> [!info] 核心结论
> 智能机械控制器选型要先从设备功能出发，统一校核轴数、I/O、通讯、周期、数据、安全和扩展能力。

## ① 分类/公式/规则

> [!note] 十步选型法
> ① 列出设备动作和工艺流程  
> ② 统计DI/DO/AI/AO点数  
> ③ 统计伺服、步进、变频器、阀岛数量  
> ④ 确认运动控制轴数和同步要求  
> ⑤ 确认通讯协议和网络拓扑  
> ⑥ 确认HMI、MES、SCADA数据接口  
> ⑦ 确认控制周期和实时性  
> ⑧ 确认视觉、AI、边缘计算需求  
> ⑨ 确认安全回路和STO需求  
> ⑩ 预留扩展、备份和维护方案

> [!warning] 最终选型公式
> **推荐控制器 = CPU性能 + I/O容量 + 轴控能力 + 通讯协议 + 数据接口 + 安全能力 + 扩展余量**

## ② 参考表

| 需求 | 推荐关注点 |
|---|---|
| 简单顺序控制 | PLC即可 |
| 多轴高速同步 | 实时总线 + 运动控制器 |
| 视觉定位 | 视觉通讯和坐标转换 |
| 数据追溯 | 存储、数据库、OPC UA |
| MES对接 | 以太网、API、数据结构 |
| 设备诊断 | 报警记录、远程维护 |
| 安全控制 | 安全I/O、STO、安全PLC |

## ③ 计算/选型示例

- 已知：  
  设备有10轴伺服、64DI、48DO、4AI、2AO，需要EtherCAT运动控制和OPC UA上传数据。
- 公式：  
  **控制器需求 = 10轴 + 114点I/O + EtherCAT + OPC UA + 扩展余量**
- 计算：  
  需选择支持10轴以上实时运动控制、足够I/O扩展和OPC UA通讯的控制器。
- 结论：  
  推荐选择 **高性能智能机械控制器/运动控制器**，并预留轴数、I/O和通讯扩展余量。

> [!tip] 选型口诀
> 先数点，再数轴；再看协议和周期；要追溯看数据，要安全看STO；余量不留，后期发愁。
