# 01 通用运动控制卡总览

> [!info] 核心结论
> 通用运动控制卡用于实现 **多轴伺服/步进控制、点位运动、插补运动、电子齿轮、电子凸轮、回零、限位和轨迹规划**，常用于PC-Based自动化设备。

## ① 分类/公式/规则

> [!note] 核心理解
> 普通PLC主要做逻辑控制；  
> 运动控制卡专门负责运动轴的控制计算和高速脉冲/总线输出。  
> 它通常安装在工控机中，或通过以太网/USB/总线与上位机连接。

> [!warning] 选型公式
> **运动控制卡选型 = 轴数 + 控制方式 + 插补需求 + 脉冲频率 + 编码器反馈 + I/O数量 + 软件接口 + 实时性**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **控制对象** | 伺服电机、步进电机、直线电机、模组 |
| **控制方式** | 脉冲、模拟量、EtherCAT、CANopen等 |
| **核心功能** | 点位、速度、回零、插补、同步 |
| **安装方式** | PCI、PCIe、USB、Ethernet、独立式 |
| **常见应用** | 点胶、切割、雕刻、检测、贴装、搬运 |
| **重点参数** | 轴数、频率、插补、接口、实时性、开发库 |

## ③ 计算/选型示例

- 已知：  
  设备有4个伺服轴，需要X/Y插补运动和Z轴升降。
- 公式：  
  **轴数需求 ≥ 实际轴数 + 预留轴数**
- 计算：  
  至少需要4轴控制卡，若后期可能增加旋转轴，可选6轴或8轴。
- 结论：  
  推荐选择 **4轴以上通用运动控制卡**，并确认支持XY直线/圆弧插补。

> [!tip] 记忆口诀
> PLC管逻辑，运动卡管轨迹；轴数先数清，插补要确认。

# 02 底层原理

> [!info] 核心结论
> 运动控制卡的底层原理，是根据目标位置、速度、加速度和运动轨迹，实时生成控制指令驱动电机运动。

## ① 分类/公式/规则

> [!note] 工作链路
> 上位机或程序发出运动命令，运动控制卡完成轨迹规划，输出脉冲、模拟量或总线指令，驱动器控制电机运动，编码器或传感器反馈状态。

> [!warning] 控制链路
> **上位机程序 → 运动控制卡 → 轨迹规划 → 驱动器 → 电机 → 机构运动 → 反馈检测**

## ② 参考表

| 环节 | 作用 |
|---|---|
| **上位机** | 下发运动命令和工艺参数 |
| **运动控制卡** | 规划轨迹、输出控制信号 |
| **驱动器** | 驱动伺服/步进电机 |
| **电机** | 输出转矩和速度 |
| **机械机构** | 丝杠、同步带、齿轮、模组 |
| **反馈元件** | 编码器、限位、原点、传感器 |

## ③ 计算/选型示例

- 已知：  
  目标位置100mm，脉冲当量0.001mm/pulse。
- 公式：  
  **所需脉冲数 = 目标位移 ÷ 脉冲当量**
- 计算：  
  100 ÷ 0.001 = **100000 pulse**
- 结论：  
  控制卡需要输出100000个脉冲，轴理论移动100mm。

> [!tip] 记忆口诀
> 程序给目标，控制卡算轨迹，驱动器带电机，机构完成动作。

# 03 常见类型

> [!info] 核心结论
> 通用运动控制卡按接口和控制方式可分为 **PCI/PCIe卡、以太网控制卡、USB控制卡、独立式控制器、总线运动控制卡**。

## ① 分类/公式/规则

> [!warning] 类型选择规则
> **工控机内部安装 → PCI/PCIe卡**  
> **远距离布置 → 以太网运动控制卡**  
> **便携调试/小型设备 → USB控制卡**  
> **多轴实时总线 → EtherCAT运动控制卡**

## ② 参考表

| 类型 | 特点 |
|---|---|
| **PCI运动控制卡** | 传统工控机插槽，实时性较好 |
| **PCIe运动控制卡** | 新型工控机常用，带宽更高 |
| **以太网运动控制卡** | 布线方便，距离更灵活 |
| **USB运动控制卡** | 接入方便，适合小型设备 |
| **独立式控制器** | 可脱离PC部分运行 |
| **总线控制卡** | 适合多轴伺服和远程I/O |

## ③ 计算/选型示例

- 已知：  
  设备使用工控机，机箱有PCIe插槽，需要稳定控制6轴。
- 公式：  
  **工控机插槽类型 = 控制卡接口类型**
- 计算：  
  若工控机没有PCI槽，就不能选择PCI卡。
- 结论：  
  应选择 **PCIe接口多轴运动控制卡**。

> [!tip] 记忆口诀
> 先看装在哪里，再看怎么控轴。

# 04 脉冲控制方式

> [!info] 核心结论
> 脉冲控制是通用运动控制卡最常见的方式，通过脉冲数量控制位置，通过脉冲频率控制速度。

## ① 分类/公式/规则

> [!warning] 基本公式
> **位移 = 脉冲数 × 脉冲当量**  
> **速度 = 脉冲频率 × 脉冲当量**

## ② 参考表

| 脉冲模式 | 说明 |
|---|---|
| **PUL/DIR** | 脉冲 + 方向，最常见 |
| **CW/CCW** | 正转脉冲 + 反转脉冲 |
| **AB相脉冲** | 类似编码器相位输出 |
| **差分脉冲** | 抗干扰更好 |
| **集电极开路** | 接线简单，但抗干扰较弱 |
| **高速脉冲** | 需校核最高输出频率 |

## ③ 计算/选型示例

- 已知：  
  脉冲当量0.002mm/pulse，目标速度200mm/s。
- 公式：  
  **频率 = 速度 ÷ 脉冲当量**
- 计算：  
  频率 = 200 ÷ 0.002 = **100000Hz = 100kHz**
- 结论：  
  控制卡单轴脉冲输出频率必须高于100kHz，并留余量。

> [!tip] 记忆口诀
> 脉冲数量定位置，脉冲频率定速度。

# 05 模拟量控制方式

> [!info] 核心结论
> 模拟量控制常用于速度或转矩给定，通常配合编码器反馈或驱动器闭环控制。

## ① 分类/公式/规则

> [!note] 核心理解
> 模拟量控制常见信号为 ±10V、0-10V、4-20mA。  
> 运动控制卡输出模拟量给驱动器，驱动器按电压或电流大小控制速度或转矩。

> [!warning] 换算公式
> **输出速度 = 模拟量比例 × 速度满量程**

## ② 参考表

| 信号类型 | 应用 |
|---|---|
| **±10V** | 伺服速度/转矩控制常见 |
| **0-10V** | 单方向速度给定 |
| **4-20mA** | 工业过程控制常见 |
| **编码器反馈** | 位置闭环或速度反馈 |
| **模拟量输出AO** | 给驱动器或比例阀 |
| **模拟量输入AI** | 读取传感器或反馈 |

## ③ 计算/选型示例

- 已知：  
  ±10V对应电机速度±3000rpm，输出5V。
- 公式：  
  **速度 = 5 ÷ 10 × 3000**
- 计算：  
  速度 = **1500rpm**
- 结论：  
  输出5V时，理论速度约为1500rpm。

> [!tip] 记忆口诀
> 脉冲控位置直观，模拟量控速度和力矩常见。

# 06 总线控制方式

> [!info] 核心结论
> 总线运动控制通过工业网络控制多个驱动器，适合多轴、高速同步和诊断要求高的设备。

## ① 分类/公式/规则

> [!warning] 总线选型公式
> **总线方案 = 协议 + 轴数 + 周期时间 + 同步精度 + 驱动器兼容性 + 诊断需求**

## ② 参考表

| 总线协议 | 特点 |
|---|---|
| **EtherCAT** | 高速实时，多轴同步常用 |
| **CANopen** | 运动设备和嵌入式设备常见 |
| **MECHATROLINK** | 运动控制领域常见 |
| **RTEX** | 部分伺服系统使用 |
| **PROFINET IRT** | 高实时工业以太网场景 |
| **EtherNet/IP Motion** | 特定平台运动控制 |

## ③ 计算/选型示例

- 已知：  
  设备有16轴伺服，需要同步运动和报警诊断。
- 公式：  
  **多轴同步 + 诊断 → 总线控制优先**
- 计算：  
  脉冲接线数量大，诊断信息少；总线可减少接线并读取驱动状态。
- 结论：  
  可选择支持EtherCAT等实时总线的运动控制卡。

> [!tip] 记忆口诀
> 轴少脉冲够，轴多看总线；要诊断，也看总线。

# 07 轴数与扩展

> [!info] 核心结论
> 运动控制卡轴数必须覆盖当前设备需求，并预留后期扩展空间。

## ① 分类/公式/规则

> [!warning] 轴数选型公式
> **控制卡轴数 ≥ 实际运动轴数 + 预留轴数**

## ② 参考表

| 轴数 | 常见应用 |
|---|---|
| **1轴** | 单滑台、送料轴 |
| **2轴** | XY平台、简单定位 |
| **3轴** | XYZ平台、点胶、检测 |
| **4轴** | XYZ + 旋转轴 |
| **6轴** | 多工位设备、机器人辅助轴 |
| **8轴及以上** | 复杂自动化设备、多轴联动 |

## ③ 计算/选型示例

- 已知：  
  当前设备有X/Y/Z/R四轴，后期可能增加夹爪旋转轴。
- 公式：  
  **推荐轴数 ≥ 4 + 1**
- 计算：  
  至少需要5轴，实际产品常见可选6轴或8轴。
- 结论：  
  推荐选择 **6轴或8轴运动控制卡**，便于扩展。

> [!tip] 记忆口诀
> 轴数别刚好，后期加轴最怕没余量。

# 08 点位运动

> [!info] 核心结论
> 点位运动是最基础的运动控制方式，用于让某一轴从当前位置移动到指定位置。

## ① 分类/公式/规则

> [!note] 点位逻辑
> 点位运动通常需要设置目标位置、速度、加速度、减速度和运动模式。  
> 适合滑台定位、送料、压装、取放等动作。

> [!warning] 点位运动参数
> **点位运动 = 目标位置 + 速度 + 加速度 + 减速度 + 到位判断**

## ② 参考表

| 参数 | 作用 |
|---|---|
| **目标位置** | 要运动到哪里 |
| **速度** | 运动快慢 |
| **加速度** | 起步快慢 |
| **减速度** | 停止快慢 |
| **S曲线** | 减小冲击和振动 |
| **到位信号** | 判断动作完成 |

## ③ 计算/选型示例

- 已知：  
  滑台从0mm移动到150mm，脉冲当量0.001mm/pulse。
- 公式：  
  **脉冲数 = 150 ÷ 0.001**
- 计算：  
  脉冲数 = **150000 pulse**
- 结论：  
  运动控制卡应输出150000个脉冲完成该点位移动。

> [!tip] 记忆口诀
> 点位运动就是“从这里走到那里”。

# 09 插补运动

> [!info] 核心结论
> 插补运动用于多轴协调走出直线、圆弧或空间轨迹，是运动控制卡的重要能力。

## ① 分类/公式/规则

> [!note] 插补理解
> 插补不是让各轴各走各的，而是让多轴按统一轨迹同步运动。  
> 常见于点胶、切割、雕刻、涂布、检测平台。

> [!warning] 选择规则
> **轨迹加工 / XY平台 / 多轴路径控制 → 必须确认插补功能**

## ② 参考表

| 插补类型 | 应用 |
|---|---|
| **直线插补** | XY直线路径 |
| **圆弧插补** | 圆弧轨迹 |
| **螺旋插补** | 复杂加工轨迹 |
| **空间直线插补** | XYZ同步运动 |
| **连续插补** | 多段轨迹平滑衔接 |
| **前瞻控制** | 提高轨迹速度和平滑性 |

## ③ 计算/选型示例

- 已知：  
  XY平台需要从A点到B点走直线轨迹。
- 公式：  
  **轨迹运动 = 多轴同步比例运动**
- 计算：  
  X轴和Y轴必须同时开始、同时结束，速度比例由路径方向决定。
- 结论：  
  需要使用 **直线插补功能**，不能简单分别启动X轴和Y轴。

> [!tip] 记忆口诀
> 多轴一起走轨迹，就是插补。

# 10 回零与限位

> [!info] 核心结论
> 运动控制系统必须配置原点、正负限位和软限位，防止位置丢失和机械撞机。

## ① 分类/公式/规则

> [!warning] 安全位置逻辑
> **原点建立坐标，软限位限制范围，硬限位防止撞机**

## ② 参考表

| 信号 | 作用 |
|---|---|
| **原点信号ORG** | 建立机械坐标零点 |
| **正限位+EL** | 防止正向超程 |
| **负限位-EL** | 防止反向超程 |
| **软限位** | 程序内限制运动范围 |
| **急停信号EMG** | 立即停止危险动作 |
| **伺服报警ALM** | 驱动器异常反馈 |
| **伺服到位INP** | 驱动器定位完成反馈 |

## ③ 计算/选型示例

- 已知：  
  丝杠有效行程500mm。
- 公式：  
  **软限位范围 < 机械极限范围**
- 计算：  
  可设置软件行程为0~480mm，预留机械缓冲空间。
- 结论：  
  不建议把软件限位设置到机械极限位置。

> [!tip] 记忆口诀
> 先回零，再运动；有软限位还要有硬限位。

# 11 编码器反馈

> [!info] 核心结论
> 编码器反馈用于读取实际位置和速度，可用于闭环检测、位置比较、跟随控制和报警判断。

## ① 分类/公式/规则

> [!warning] 反馈判断公式
> **跟随误差 = 指令位置 - 实际反馈位置**

## ② 参考表

| 编码器类型 | 说明 |
|---|---|
| **增量式编码器** | 输出A/B/Z相 |
| **绝对值编码器** | 可读取绝对位置 |
| **线性光栅尺** | 高精度直线反馈 |
| **电机编码器** | 伺服电机自带 |
| **外部编码器** | 检测机构真实位置 |
| **编码器计数器** | 控制卡读取反馈脉冲 |

## ③ 计算/选型示例

- 已知：  
  指令位置100000 pulse，反馈位置99800 pulse。
- 公式：  
  **跟随误差 = 100000 - 99800**
- 计算：  
  跟随误差 = **200 pulse**
- 结论：  
  若超过允许误差，应报警或减速检查机构、驱动和负载。

> [!tip] 记忆口诀
> 指令是想去哪，反馈是实际到哪；两者差值就是跟随误差。

# 12 I/O接口

> [!info] 核心结论
> 运动控制卡通常带有专用运动I/O和通用I/O，用于限位、原点、报警、到位、气缸、夹爪和外部触发。

## ① 分类/公式/规则

> [!warning] I/O选型公式
> **控制卡I/O需求 = 每轴专用I/O + 通用DI/DO + 高速触发 + 安全信号**

## ② 参考表

| I/O类型 | 用途 |
|---|---|
| **ORG原点** | 回零使用 |
| **EL限位** | 正负行程保护 |
| **ALM报警** | 驱动器报警反馈 |
| **INP到位** | 伺服定位完成 |
| **SVON使能** | 伺服使能输出 |
| **ERC清除偏差** | 驱动器偏差清除 |
| **通用DI/DO** | 传感器、电磁阀、夹爪控制 |
| **高速触发DI** | 视觉拍照、飞拍、位置锁存 |

## ③ 计算/选型示例

- 已知：  
  4轴系统，每轴需要原点、正限位、负限位、报警输入。
- 公式：  
  **专用输入点 = 4轴 × 4点**
- 计算：  
  专用输入点 = **16点**
- 结论：  
  控制卡需要至少16个轴专用输入，或配套端子板满足接线。

> [!tip] 记忆口诀
> 运动轴不是只有脉冲线，原点限位报警也要接。

# 13 轨迹规划与加减速

> [!info] 核心结论
> 运动控制卡通过梯形加减速或S曲线加减速控制速度变化，减少冲击、振动和失步风险。

## ① 分类/公式/规则

> [!note] 加减速理解
> 速度不能从0瞬间变到最大值。  
> 加速度过大，电机可能过载、步进可能失步、机构可能振动。

> [!warning] 运动规划规则
> **速度要满足节拍，加速度要满足负载和机构刚性**

## ② 参考表

| 加减速方式 | 特点 |
|---|---|
| **梯形速度曲线** | 简单常用，效率高 |
| **S曲线** | 冲击小，适合精密设备 |
| **前瞻控制** | 多段轨迹平滑衔接 |
| **加速度限制** | 防止过载 |
| **减速度限制** | 防止急停冲击 |
| **Jerk限制** | 控制加速度变化率 |

## ③ 计算/选型示例

- 已知：  
  运动速度设得很高，启动瞬间伺服过载报警。
- 公式：  
  **峰值推力/转矩与加速度相关**
- 计算：  
  加速度太大时，驱动器需要瞬间输出很大转矩。
- 结论：  
  应降低加速度或使用S曲线，必要时增大电机和驱动器规格。

> [!tip] 记忆口诀
> 速度决定节拍，加速度决定冲击。

# 15 常见故障排查

> [!info] 核心结论
> 运动控制卡故障排查按 **供电、驱动、接线、参数、I/O、软件、机构** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **运动异常 = 控制卡未初始化 + 脉冲未输出 + 驱动未使能 + 接线错误 + 参数错误 + 限位报警 + 机构卡滞**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **轴不动** | 控制卡未初始化、驱动未使能、脉冲线错误 |
| **方向反了** | DIR方向信号反、参数方向设置错 |
| **运动距离不对** | 脉冲当量、电子齿轮比设置错误 |
| **高速丢步** | 频率过高、加速度太大、负载过重 |
| **伺服报警** | 限位、过载、编码器、跟随误差 |
| **回零失败** | 原点接线、回零方向、限位状态错误 |
| **插补轨迹异常** | 坐标参数、轴映射、单位换算错误 |
| **软件连不上卡** | 驱动程序、设备号、接口、权限问题 |

## ③ 计算/选型示例

- 已知：  
  设定移动100mm，实际只移动50mm。
- 公式：  
  **实际位移异常 = 脉冲当量 / 电子齿轮比 / 丝杠导程设置错误**
- 计算：  
  若脉冲当量设置比真实值小一半，实际运动距离就会偏差。
- 结论：  
  应检查丝杠导程、驱动器电子齿轮比和控制卡脉冲当量。

> [!tip] 记忆口诀
> 不动先查使能和脉冲，距离不对查当量，方向不对查DIR。

# 16 通用运动控制卡快速选型表

> [!info] 核心结论
> 通用运动控制卡选型先确定轴数和控制方式，再校核脉冲频率、插补功能、I/O、软件接口和实时性。

## ① 分类/公式/规则

> [!note] 十步选型法
> ① 统计伺服/步进轴数  
> ② 确认控制方式：脉冲、模拟量、总线  
> ③ 计算最高脉冲频率或总线周期  
> ④ 确认插补类型：直线、圆弧、空间插补  
> ⑤ 确认是否需要电子齿轮/电子凸轮  
> ⑥ 确认原点、限位、报警、到位I/O  
> ⑦ 确认编码器反馈需求  
> ⑧ 确认接口形式：PCI、PCIe、USB、Ethernet  
> ⑨ 确认SDK语言和上位机系统  
> ⑩ 预留轴数、I/O和软件扩展余量

> [!warning] 最终选型公式
> **推荐运动控制卡 = 轴数 + 控制方式 + 频率/周期 + 插补功能 + I/O接口 + 编码器反馈 + SDK支持**

## ② 参考表

| 需求 | 推荐关注点 |
|---|---|
| 单轴送料 | 1轴/2轴脉冲卡 |
| XY平台 | 2轴直线/圆弧插补 |
| XYZ点胶 | 3轴插补 + 高速触发 |
| 多轴同步 | EtherCAT总线控制卡 |
| 高速视觉定位 | 位置锁存/高速触发 |
| PC-Based设备 | PCIe/以太网控制卡 |
| 测试测量设备 | SDK和上位机兼容性 |
| 后期扩展 | 预留轴数和I/O |

## ③ 计算/选型示例

- 已知：  
  设备有4轴伺服，脉冲当量0.001mm/pulse，最高速度500mm/s，需要XY圆弧插补。
- 公式：  
  **脉冲频率 = 速度 ÷ 脉冲当量**
- 计算：  
  频率 = 500 ÷ 0.001 = **500kHz**
- 结论：  
  推荐选择 **4轴以上、单轴脉冲频率高于500kHz、支持直线/圆弧插补的运动控制卡**，并确认SDK与上位机软件兼容。

> [!tip] 选型口诀
> 先数轴，再算频率；要轨迹看插补，要同步看总线；硬件能控，软件也要会用。
