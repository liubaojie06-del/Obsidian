# 01 DTU / RTU 总览

> [!info] 核心结论
> DTU主要负责 **数据透明传输和远程联网**；RTU主要负责 **现场数据采集、控制逻辑、遥测遥控和远程上报**。  
> 简单理解：**DTU偏“传数据”，RTU偏“采数据 + 控现场 + 传数据”**。

## ① 分类/公式/规则

> [!warning] 选型公式
> **DTU/RTU选型 = 现场设备接口 + 通讯方式 + 采集点数 + 控制需求 + 协议支持 + 供电方式 + 环境等级 + 平台接口**

## ② 参考表

| 类型 | 核心作用 |
|---|---|
| **DTU** | 串口/网口数据通过4G/5G/NB-IoT等网络传到平台 |
| **RTU** | 采集DI/AI/脉冲/串口数据，并可执行远程控制 |
| **4G DTU** | 常见远程透明传输设备 |
| **工业RTU** | 常用于水利、电力、环保、油气、泵站、管网 |
| **遥测终端** | 采集现场状态并周期上报 |
| **远程控制终端** | 接收平台命令并控制继电器、阀门、泵等设备 |

## ③ 计算/选型示例

- 已知：  
  现场有一台RS485电表，只需要把数据传给远端服务器。
- 公式：  
  **只传输串口数据 → DTU可用**
- 计算：  
  DTU负责把RS485数据通过4G网络传给平台。
- 结论：  
  可选择 **4G DTU**；如果还要本地采集DI/AI并控制泵阀，则更适合RTU。

> [!tip] 记忆口诀
> DTU像“无线网线”，RTU像“远程小PLC + 通讯终端”。

# 02 底层原理

> [!info] 核心结论
> DTU/RTU的底层逻辑，是把现场数据采集或接入后，通过蜂窝网络、有线网络或专网传输到远端平台。

## ① 分类/公式/规则

> [!note] 工作链路
> DTU通常把串口/以太网数据封装成TCP/UDP/MQTT等网络数据转发。  
> RTU则先采集现场I/O、模拟量、脉冲量或串口设备数据，再进行处理、存储、报警判断和远程上传。

> [!warning] 数据链路
> **现场设备/传感器 → DTU/RTU → 4G/5G/以太网 → 服务器/SCADA/云平台 → 远程监控**

## ② 参考表

| 环节 | 作用 |
|---|---|
| **现场采集** | DI、DO、AI、AO、脉冲、RS485、传感器 |
| **数据处理** | 协议解析、工程量换算、报警判断 |
| **数据封装** | TCP、UDP、MQTT、HTTP、Modbus TCP等 |
| **网络传输** | 4G、5G、NB-IoT、LoRa、以太网 |
| **远程平台** | SCADA、云平台、数据中心、调度中心 |
| **远程控制** | 平台下发开关量、参数、控制命令 |

## ③ 计算/选型示例

- 已知：  
  水位传感器输出4-20mA，RTU采集后上传平台。
- 公式：  
  **传感器信号 → RTU换算 → 平台显示工程值**
- 计算：  
  RTU读取电流值，换算成水位高度，再通过4G上传。
- 结论：  
  RTU不仅传数据，还负责采集和换算。

> [!tip] 记忆口诀
> DTU把数据送出去，RTU先把现场看明白再送出去。

# 03 DTU是什么

> [!info] 核心结论
> DTU是数据传输终端，核心任务是把本地串口或网口数据通过无线或有线网络传到远端。

## ① 分类/公式/规则

> [!warning] DTU核心公式
> **DTU功能 = 串口/网口接入 + 数据封装 + 网络传输 + 远程服务器连接**

## ② 参考表

| 功能 | 说明 |
|---|---|
| **串口透传** | RS232/RS485数据透明传输 |
| **TCP Client** | 主动连接服务器 |
| **TCP Server** | 等待远端连接 |
| **UDP传输** | 简单低开销传输 |
| **MQTT上传** | 物联网平台常见 |
| **短信/心跳** | 部分型号支持状态上报 |
| **断线重连** | 网络异常后自动恢复连接 |

## ③ 计算/选型示例

- 已知：  
  串口仪表原来接本地电脑，现在要接远端服务器。
- 公式：  
  **串口设备远程联网 → DTU**
- 计算：  
  DTU把串口数据通过4G网络发给服务器IP和端口。
- 结论：  
  该场景适合DTU，不一定需要RTU。

> [!tip] 记忆口诀
> DTU重点不是控制，而是把本地数据送到远方。

# 04 RTU是什么

> [!info] 核心结论
> RTU是远程终端单元，用于现场遥测、遥信、遥控、遥调，是远程监控系统中的现场采集与控制节点。

## ① 分类/公式/规则

> [!warning] RTU核心公式
> **RTU功能 = 现场采集 + 本地逻辑 + 数据存储 + 远程通讯 + 远程控制**

## ② 参考表

| 功能 | 说明 |
|---|---|
| **遥测** | 采集水位、压力、温度、流量等模拟量 |
| **遥信** | 采集开关状态、门磁、故障、运行状态 |
| **遥控** | 远程控制泵、阀、继电器、开关 |
| **遥调** | 调节参数、输出模拟量或设定值 |
| **本地报警** | 超限、断线、掉电、通信异常报警 |
| **数据记录** | 本地缓存历史数据 |
| **远程上传** | 上传SCADA、云平台或调度中心 |

## ③ 计算/选型示例

- 已知：  
  泵站需要采集水位、流量、泵运行状态，并可远程启停水泵。
- 公式：  
  **采集 + 控制 + 远程上传 → RTU**
- 计算：  
  需要AI采水位、DI采运行状态、DO控制水泵，同时4G上传平台。
- 结论：  
  该场景应选择RTU，而不是单纯DTU。

> [!tip] 记忆口诀
> RTU能采、能控、能存、能传，是远程现场小站点。

# 05 DTU vs RTU

> [!info] 核心结论
> DTU偏通信，RTU偏采控；DTU适合已有设备数据透传，RTU适合现场没有控制器或需要独立采集控制的场景。

## ① 分类/公式/规则

> [!warning] 选择规则
> **只需要把串口/网口数据传出去 → DTU**  
> **需要采集I/O、模拟量、控制设备、报警缓存 → RTU**

## ② 参考表

| 对比项 | DTU | RTU |
|---|---|---|
| 核心定位 | 数据传输 | 远程采集与控制 |
| 是否带I/O | 一般较少或没有 | **通常有DI/DO/AI/AO** |
| 是否可本地逻辑 | 较弱 | **较强** |
| 是否可远程控制 | 依赖外部设备 | **可直接控制现场输出** |
| 典型接口 | RS232/RS485/网口 | I/O + 串口 + 网口 + 无线 |
| 常见场景 | 串口设备联网 | 泵站、水文、电力、环保遥测 |

## ③ 计算/选型示例

- 已知：  
  现场已有PLC，PLC负责所有控制，只需要把PLC数据传到云平台。
- 公式：  
  **已有PLC控制 + 只需传输 → DTU或工业网关**
- 计算：  
  若只是透传PLC串口数据，DTU即可。
- 结论：  
  不需要重复配置RTU；除非还需要额外现场采集和控制。

> [!tip] 记忆口诀
> 有设备会采控，只差联网，用DTU；现场要自己采控，用RTU。

# 06 DTU/RTU vs 工业网关

> [!info] 核心结论
> 工业网关更强调协议转换、点表采集和上云；DTU/RTU更常见于远程站点、遥测遥控和蜂窝网络传输。

## ① 分类/公式/规则

> [!warning] 选择规则
> **多协议采集、点表、MQTT/OPC UA上云 → 工业网关**  
> **偏远站点遥测、远程控制、低功耗采集 → RTU**  
> **简单串口透传 → DTU**

## ② 参考表

| 对比项 | DTU | RTU | 工业网关 |
|---|---|---|---|
| 数据透传 | 强 | 可支持 | 可支持 |
| I/O采集 | 弱 | **强** | 视型号 |
| 协议转换 | 一般 | 中等 | **强** |
| 点表管理 | 较弱 | 中等 | **强** |
| 边缘计算 | 弱 | 中等 | **较强** |
| 远程站点 | 常见 | **非常常见** | 常见 |
| 上云接口 | 视型号 | 视型号 | **常见** |

## ③ 计算/选型示例

- 已知：  
  工厂内有多台PLC，需要统一采集1000个点上传MES。
- 公式：  
  **多PLC + 大点表 + MES接口 → 工业网关**
- 计算：  
  工业网关更适合点表管理、协议转换和平台对接。
- 结论：  
  这种场景优先工业网关，而不是普通DTU。

> [!tip] 记忆口诀
> DTU传输简单，RTU现场遥控，网关协议集成。

# 07 常见应用场景

> [!info] 核心结论
> DTU/RTU常用于分散、偏远、无人值守、需要远程采集和控制的工业现场。

## ① 分类/公式/规则

> [!warning] 应用判断规则
> **现场分散 + 远程监控 + 通讯不方便 → DTU/RTU可评估**

## ② 参考表

| 场景 | 常见需求 |
|---|---|
| **水利水文** | 水位、雨量、流量、闸门状态 |
| **泵站监控** | 水泵运行、液位、压力、远程启停 |
| **环保监测** | 水质、废气、噪声、排放数据 |
| **电力配电** | 电表、开关状态、遥测遥控 |
| **油气管网** | 压力、流量、阀门状态 |
| **农业灌溉** | 土壤湿度、阀门、泵控制 |
| **智慧井盖/路灯** | 状态监测、故障报警、远程控制 |

## ③ 计算/选型示例

- 已知：  
  山区水位站没有宽带，需要上传水位和雨量数据。
- 公式：  
  **偏远无人站 + 传感器采集 + 无线通讯 → RTU**
- 计算：  
  RTU采集水位、雨量，使用4G/NB-IoT上传。
- 结论：  
  该场景适合低功耗遥测RTU。

> [!tip] 记忆口诀
> 越偏远、越分散、越无人值守，越容易用到DTU/RTU。

# 08 通讯方式

> [!info] 核心结论
> DTU/RTU常见通讯方式包括4G、5G、NB-IoT、LoRa、以太网、Wi-Fi、短信和专线网络。

## ① 分类/公式/规则

> [!warning] 通讯选择公式
> **通讯方式 = 覆盖条件 + 数据量 + 实时性 + 功耗 + 成本 + 平台接口**

## ② 参考表

| 通讯方式 | 适合场景 |
|---|---|
| **4G** | 通用远程站点，数据量中等 |
| **5G** | 大带宽、低延迟、视频或大数据 |
| **NB-IoT** | 小数据、低功耗、广覆盖 |
| **LoRa** | 局部区域低功耗远距离 |
| **以太网** | 有线网络稳定场景 |
| **Wi-Fi** | 有无线覆盖的厂区或园区 |
| **短信** | 低频报警或备用通道 |

## ③ 计算/选型示例

- 已知：  
  设备每小时只上传一次少量状态数据，电池供电。
- 公式：  
  **小数据 + 低功耗 → NB-IoT/LoRa可评估**
- 计算：  
  4G实时性更好，但功耗和费用可能更高。
- 结论：  
  可优先评估NB-IoT或LoRa RTU。

> [!tip] 记忆口诀
> 数据大看4G/5G，功耗低看NB-IoT/LoRa。

# 09 串口与以太网接口

> [!info] 核心结论
> DTU/RTU常通过RS232、RS485、RS422或以太网连接现场设备，接口必须与设备匹配。

## ① 分类/公式/规则

> [!warning] 接口匹配公式
> **接口可用 = 接口类型 + 通讯参数 + 协议 + 接线方式一致**

## ② 参考表

| 接口 | 常见对象 |
|---|---|
| **RS232** | 单台仪表、扫码器、老设备 |
| **RS485** | 电表、水表、温控器、Modbus仪表 |
| **RS422** | 差分串口设备 |
| **以太网** | PLC、摄像头、网关、上位机 |
| **DI输入** | 开关状态、门磁、液位开关 |
| **DO输出** | 继电器、阀门、水泵控制 |
| **AI输入** | 水位、压力、温度、流量 |

## ③ 计算/选型示例

- 已知：  
  现场有8台Modbus RTU电表，全部挂在RS485总线上。
- 公式：  
  **多台串口仪表 → RS485接口**
- 计算：  
  DTU/RTU需要支持RS485和Modbus RTU。
- 结论：  
  应选择带RS485接口的DTU/RTU，并确保站号不重复。

> [!tip] 记忆口诀
> 设备走什么口，DTU/RTU就要有什么口。

# 10 数据采集点位

> [!info] 核心结论
> RTU选型必须统计采集点位，包括DI、DO、AI、AO、脉冲量、串口设备数量和通讯点数。

## ① 分类/公式/规则

> [!warning] 点位选型公式
> **RTU点位需求 = DI + DO + AI + AO + 脉冲输入 + 串口设备 + 预留余量**

## ② 参考表

| 点位类型 | 典型信号 |
|---|---|
| **DI** | 开关状态、门禁、液位、故障 |
| **DO** | 继电器输出、阀门、水泵启停 |
| **AI** | 4-20mA、0-10V传感器 |
| **AO** | 比例控制、模拟量给定 |
| **脉冲输入** | 流量计、电表脉冲 |
| **RS485点表** | 电表、仪表寄存器 |
| **内部变量** | 电池电压、信号强度、设备状态 |

## ③ 计算/选型示例

- 已知：  
  泵站有6DI、4DO、3AI、1路脉冲流量计。
- 公式：  
  **RTU配置 ≥ 6DI + 4DO + 3AI + 1PI**
- 计算：  
  选择RTU时应至少满足这些点位，并预留10%~20%。
- 结论：  
  可选择8DI、4DO、4AI、带脉冲输入的RTU。

> [!tip] 记忆口诀
> RTU先数点，再看网；点不够，功能再强也没用。

# 11 模拟量采集

> [!info] 核心结论
> RTU常采集4-20mA、0-10V等模拟量，用于水位、压力、温度、流量等连续变量监测。

## ① 分类/公式/规则

> [!warning] 模拟量换算公式
> **工程值 = (输入值 - 下限信号) ÷ (上限信号 - 下限信号) × 工程量量程 + 工程量下限**

## ② 参考表

| 信号类型 | 常见应用 |
|---|---|
| **4-20mA** | 水位、压力、流量、液位 |
| **0-10V** | 压力、位移、速度反馈 |
| **热电偶** | 温度采集 |
| **热电阻** | PT100/PT1000温度 |
| **脉冲量** | 流量累计、电量累计 |
| **开关量** | 液位高低、设备运行状态 |

## ③ 计算/选型示例

- 已知：  
  液位传感器4-20mA对应0-5米，RTU采到12mA。
- 公式：  
  **液位 = (12 - 4) ÷ (20 - 4) × 5**
- 计算：  
  液位 = 8 ÷ 16 × 5 = **2.5米**
- 结论：  
  当前液位约2.5米，RTU或平台需配置正确量程。

> [!tip] 记忆口诀
> 模拟量不换算，只是一串电流电压，不是工程值。

# 12 远程控制

> [!info] 核心结论
> RTU可以接收平台命令，远程控制继电器、泵、阀门、照明、报警器等现场设备。

## ① 分类/公式/规则

> [!warning] 远程控制规则
> **远程控制 = 权限确认 + 命令下发 + 本地联锁 + 输出执行 + 状态反馈**

## ② 参考表

| 控制对象 | 注意 |
|---|---|
| **水泵** | 启停联锁、过载保护、液位条件 |
| **电磁阀** | 电源、线圈电流、动作反馈 |
| **闸门** | 开到位、关到位、超时保护 |
| **照明** | 定时/远程控制 |
| **报警器** | 声光提示 |
| **继电器输出** | 负载电流不足时需中间继电器 |
| **模拟量输出** | 远程调节阀门开度或设定值 |

## ③ 计算/选型示例

- 已知：  
  平台远程启动水泵，但液位低于最低允许值。
- 公式：  
  **远程命令 ≠ 一定执行，本地联锁优先**
- 计算：  
  RTU应判断液位条件，不满足时拒绝启动并上报原因。
- 结论：  
  远程控制必须有本地安全联锁，不能盲目执行平台命令。

> [!tip] 记忆口诀
> 平台能下命令，现场RTU要判断能不能执行。

# 13 数据上报方式

> [!info] 核心结论
> DTU/RTU数据上报可采用周期上报、变化上报、事件上报、报警上报和远程召测。

## ① 分类/公式/规则

> [!warning] 上报策略公式
> **上报策略 = 周期数据 + 变化数据 + 报警事件 + 平台召测**

## ② 参考表

| 上报方式 | 适合场景 |
|---|---|
| **周期上报** | 定时上传水位、压力、电量 |
| **变化上报** | 数据变化超过阈值时上传 |
| **事件上报** | 门开、设备启停、故障发生 |
| **报警上报** | 超限、掉电、断线、通信异常 |
| **平台召测** | 平台主动读取现场数据 |
| **历史补传** | 网络恢复后补传缓存数据 |

## ③ 计算/选型示例

- 已知：  
  水位变化很慢，每分钟上传一次即可。
- 公式：  
  **慢变量 → 长周期上报**
- 计算：  
  如果每秒上传，会增加流量和平台压力。
- 结论：  
  可设置1分钟或更长周期，并对超限报警立即上报。

> [!tip] 记忆口诀
> 平时慢慢报，报警马上报。

# 14 协议与平台对接

> [!info] 核心结论
> DTU/RTU能否接入平台，关键看支持的上行协议、数据格式、认证方式和平台接口。

## ① 分类/公式/规则

> [!warning] 平台对接公式
> **对接成功 = 上行协议一致 + 数据格式正确 + 认证通过 + 心跳在线**

## ② 参考表

| 协议/方式 | 常见用途 |
|---|---|
| **TCP透传** | 自定义平台、老系统 |
| **UDP透传** | 简单低开销上报 |
| **MQTT** | 物联网平台常见 |
| **HTTP/HTTPS** | Web接口上传 |
| **Modbus TCP** | SCADA/主站读取 |
| **IEC 60870-5-104** | 电力远动常见 |
| **私有协议** | 厂家平台或行业平台 |

## ③ 计算/选型示例

- 已知：  
  云平台要求MQTT接入，且需要用户名、密码和设备ID。
- 公式：  
  **平台MQTT → DTU/RTU需支持MQTT认证**
- 计算：  
  设备需配置Broker地址、端口、Topic、账号和密码。
- 结论：  
  应选择支持MQTT和平台认证机制的DTU/RTU。

> [!tip] 记忆口诀
> 设备能上网不代表能进平台，协议和认证也要对。

# 15 本地存储与断点续传

> [!info] 核心结论
> 远程站点网络不稳定时，RTU需要本地存储和断点续传，保证数据不因断网丢失。

## ① 分类/公式/规则

> [!warning] 缓存容量公式
> **缓存容量 ≥ 单次数据量 × 上报次数 × 断网时长**

## ② 参考表

| 功能 | 作用 |
|---|---|
| **本地缓存** | 网络中断时保存数据 |
| **历史记录** | 保存周期数据和报警事件 |
| **断点续传** | 网络恢复后补传 |
| **时间戳** | 保留真实采集时间 |
| **去重机制** | 避免重复入库 |
| **存储介质** | Flash、SD卡、EEPROM等 |

## ③ 计算/选型示例

- 已知：  
  每次上报数据2KB，每分钟上报一次，可能断网24小时。
- 公式：  
  **缓存容量 = 2KB × 60 × 24**
- 计算：  
  缓存容量 = **2880KB ≈ 2.88MB**
- 结论：  
  RTU本地存储至少要大于2.88MB，并留余量。

> [!tip] 记忆口诀
> 网络可以断，数据不能丢；关键看缓存和补传。

# 16 供电与低功耗

> [!info] 核心结论
> DTU/RTU常用于无人值守现场，供电方式可能是市电、DC24V、太阳能、电池或后备电源，低功耗非常关键。

## ① 分类/公式/规则

> [!warning] 续航估算公式
> **续航时间 = 电池容量Wh ÷ 平均功耗W**

## ② 参考表

| 供电方式 | 适合场景 |
|---|---|
| **AC市电** | 泵站、机房、固定站点 |
| **DC24V** | 工业控制柜 |
| **太阳能** | 野外水文、农业、环保监测 |
| **锂电池** | 低功耗远程采集 |
| **铅酸电池** | 备用电源、太阳能系统 |
| **UPS** | 防止断电导致数据丢失 |
| **休眠唤醒** | 低功耗RTU常见 |

## ③ 计算/选型示例

- 已知：  
  RTU平均功耗1W，电池容量120Wh。
- 公式：  
  **续航 = 120 ÷ 1**
- 计算：  
  理论续航 = **120小时 ≈ 5天**
- 结论：  
  如果要求连续运行30天，需要更大电池或太阳能补电。

> [!tip] 记忆口诀
> 远程站点先问有没有电，再谈怎么通信。

# 17 天线与信号

> [!info] 核心结论
> 蜂窝DTU/RTU的稳定性很大程度取决于信号强度、天线位置、运营商覆盖和现场遮挡。

## ① 分类/公式/规则

> [!warning] 信号稳定公式
> **无线稳定 = 信号强度 + 天线位置 + 运营商覆盖 + 供电稳定 + 网络重连机制**

## ② 参考表

| 项目 | 注意 |
|---|---|
| **外置天线** | 金属柜内必须重点考虑 |
| **高增益天线** | 弱信号区域可评估 |
| **天线馈线** | 过长会有损耗 |
| **SIM卡运营商** | 不同区域覆盖不同 |
| **信号指示** | 用于现场调试 |
| **断线重连** | 蜂窝网络必须关注 |
| **双卡切换** | 提高通信可靠性 |

## ③ 计算/选型示例

- 已知：  
  RTU装在金属柜内，4G频繁掉线。
- 公式：  
  **金属柜屏蔽 → 天线外置**
- 计算：  
  天线闷在柜内会导致信号衰减。
- 结论：  
  应把天线引到柜外，必要时更换高增益天线或换运营商。

> [!tip] 记忆口诀
> RTU可以在柜里，天线不要闷在铁柜里。

# 18 安全与权限

> [!info] 核心结论
> DTU/RTU涉及远程控制和数据上传，必须做好平台认证、加密传输、权限控制、白名单和操作日志。

## ① 分类/公式/规则

> [!warning] 安全公式
> **远程安全 = 认证 + 加密 + 权限 + 白名单 + 日志 + 本地联锁**

## ② 参考表

| 安全项 | 作用 |
|---|---|
| **设备ID** | 识别唯一终端 |
| **账号密码** | 平台认证 |
| **证书认证** | 提高连接安全 |
| **VPN/专网** | 隔离公网风险 |
| **访问白名单** | 限制允许访问来源 |
| **操作日志** | 记录远程控制行为 |
| **本地联锁** | 防止远程误操作造成危险 |

## ③ 计算/选型示例

- 已知：  
  平台可远程控制水泵启停。
- 公式：  
  **远程控制高风险 → 权限 + 日志 + 本地保护**
- 计算：  
  误操作可能导致干抽、溢流或设备损坏。
- 结论：  
  必须设置权限分级、操作记录和现场联锁条件。

> [!tip] 记忆口诀
> 能远程控制，就必须能追踪、能限制、能保护。

# 19 常见故障排查

> [!info] 核心结论
> DTU/RTU故障排查按 **供电、SIM卡、信号、网络、串口/I/O、协议、平台、天线** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **DTU/RTU异常 = 供电异常 + 信号弱 + SIM卡异常 + 网络不通 + 参数错误 + 协议不匹配 + 平台认证失败**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **设备不上电** | 电源无电、极性接错、保险损坏 |
| **无法上线** | SIM卡欠费、APN错误、信号弱 |
| **频繁掉线** | 天线差、运营商网络不稳、电源波动 |
| **串口无数据** | 波特率、校验位、接线、站号错误 |
| **I/O无响应** | 传感器、供电、公共端、负载故障 |
| **数据异常** | 点表、倍率、数据类型、字节序错误 |
| **平台收不到** | IP端口、Topic、账号密码、证书错误 |
| **远程控制失败** | 权限不足、联锁条件不满足、输出故障 |

## ③ 计算/选型示例

- 已知：  
  RTU显示4G在线，但平台没有数据。
- 公式：  
  **网络在线 ≠ 平台接入成功**
- 计算：  
  可能是服务器地址、端口、协议、Topic或认证错误。
- 结论：  
  应检查平台接入参数和上报日志，而不是只看4G信号。

> [!tip] 记忆口诀
> 信号在线只是能上网，平台收到才算数据通。

# 20 DTU/RTU快速选型表

> [!info] 核心结论
> DTU/RTU选型要先判断是“只传数据”还是“采集控制”，再确认接口、点数、网络、协议、供电、缓存和安全。

## ① 分类/公式/规则

> [!note] 十步选型法
> ① 判断需求：DTU透传还是RTU采控  
> ② 确认现场接口：RS232、RS485、以太网、DI/DO/AI/AO  
> ③ 统计采集点数和控制点数  
> ④ 确认通讯方式：4G、5G、NB-IoT、LoRa、以太网  
> ⑤ 确认上行协议：TCP、UDP、MQTT、HTTP、Modbus TCP等  
> ⑥ 确认平台接口、设备ID、认证和数据格式  
> ⑦ 确认上报周期、事件上报和报警策略  
> ⑧ 确认是否需要本地缓存和断点续传  
> ⑨ 确认供电方式、功耗、太阳能或备用电池  
> ⑩ 确认防护等级、温度、天线、安装和安全权限

> [!warning] 最终选型公式
> **推荐方案 = DTU/RTU类型 + 接口点数 + 通讯网络 + 上行协议 + 缓存能力 + 供电方式 + 安全策略 + 环境等级**

## ② 参考表

| 需求 | 推荐方向 |
|---|---|
| 串口设备远程透传 | DTU |
| PLC数据简单传输 | DTU或工业网关 |
| 泵站采集控制 | RTU |
| 水文雨量遥测 | 低功耗RTU |
| 电表远程采集 | RS485 DTU/RTU |
| 偏远无网络站点 | 4G/5G/NB-IoT RTU |
| 断网不丢数据 | 本地缓存 + 断点续传 |
| 远程控制设备 | RTU + 权限 + 本地联锁 |
| 多协议上云 | 工业网关或网关型RTU |

## ③ 计算/选型示例

- 已知：  
  某泵站需要采集2路液位、1路压力、6个开关状态，远程控制2台水泵，并通过4G上传平台。
- 公式：  
  **需求 = 3AI + 6DI + 2DO + 4G + 平台协议**
- 计算：  
  需要RTU至少支持3路AI、6路DI、2路DO、4G通信和平台上传协议，并支持本地联锁。
- 结论：  
  推荐选择 **4G工业RTU，配置不少于4AI、8DI、4DO，支持本地缓存、断点续传、远程控制权限和天线外置安装**。

> [!tip] 选型口诀
> 只传数据选DTU，现场采控选RTU；远程站点看4G/5G，低功耗看NB/LoRa；能控设备就要权限和联锁。
