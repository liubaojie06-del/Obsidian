# 01 串口服务器总览

> [!info] 核心结论
> 串口服务器用于把 **RS232、RS485、RS422串口设备接入以太网**，实现串口设备联网、远程访问、数据透传和协议转换。

## ① 分类/公式/规则

> [!warning] 选型公式
> **串口服务器选型 = 串口类型 + 串口数量 + 网络接口 + 通讯模式 + 协议需求 + 波特率 + 供电方式 + 安装环境**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **RS232** | 点对点短距离串口通讯 |
| **RS485** | 多站总线通讯，仪表常见 |
| **RS422** | 差分通讯，抗干扰较好 |
| **以太网口** | 接入交换机、路由器、工控机 |
| **TCP/UDP** | 网络传输方式 |
| **虚拟串口** | 让电脑软件像访问本地COM口一样访问远程串口 |
| **典型设备** | 仪表、电表、扫码枪、PLC、变频器、传感器 |

## ③ 计算/选型示例

- 已知：  
  现场有10台RS485电表，需要把数据接入工控机。
- 公式：  
  **串口设备联网 → 串口服务器**
- 计算：  
  串口服务器连接RS485总线，再通过以太网把数据传给工控机。
- 结论：  
  可选择 **RS485转以太网串口服务器**，并确认支持对应波特率和Modbus协议。

> [!tip] 记忆口诀
> 老设备走串口，新系统走网口，中间靠串口服务器。

# 02 底层原理

> [!info] 核心结论
> 串口服务器的底层原理，是把串口数据封装成网络数据包发送出去，再在另一端还原为串口数据。

## ① 分类/公式/规则

> [!note] 工作链路
> 串口设备发出的数据进入串口服务器，串口服务器把数据转换为TCP/UDP数据，通过以太网传给上位机或服务器。

> [!warning] 数据链路
> **串口设备 → RS232/RS485/RS422 → 串口服务器 → TCP/UDP网络 → 上位机/服务器**

## ② 参考表

| 环节 | 作用 |
|---|---|
| **串口端** | 接收或发送RS232/RS485/RS422数据 |
| **串口参数** | 波特率、数据位、停止位、校验位 |
| **协议封装** | 把串口数据打包为TCP/UDP数据 |
| **网口端** | 通过以太网传输 |
| **上位机软件** | 接收、解析、记录数据 |
| **虚拟串口驱动** | 把网络串口模拟成本地COM口 |

## ③ 计算/选型示例

- 已知：  
  RS485仪表发送温度数据，上位机在办公室读取。
- 公式：  
  **串口数据 → 网络封装 → 远程读取**
- 计算：  
  仪表数据通过串口服务器进入网络，上位机通过IP地址和端口访问。
- 结论：  
  串口服务器解决了串口设备远距离联网访问问题。

> [!tip] 记忆口诀
> 串口服务器不是改变数据内容，而是改变数据走的路。

# 03 串口服务器 vs 工业网关

> [!info] 核心结论
> 串口服务器主要做串口透传和联网，工业网关更强调协议解析、数据采集、边缘计算和上云。

## ① 分类/公式/规则

> [!warning] 选择规则
> **只需要串口转网络 → 串口服务器**  
> **需要协议解析、点表、MQTT/OPC UA上传 → 工业网关**

## ② 参考表

| 对比项 | 串口服务器 | 工业网关 |
|---|---|---|
| 核心功能 | 串口转以太网 | 协议转换、数据采集 |
| 数据处理 | 较少，偏透传 | **较强** |
| 点表配置 | 通常简单或没有 | **常见** |
| 上云能力 | 一般较弱 | **常见** |
| 适合对象 | 老串口设备联网 | 多设备数据采集和系统集成 |
| 应用重点 | 通讯通道 | 数据可用性 |

## ③ 计算/选型示例

- 已知：  
  上位机原软件只能读COM口，现在仪表距离很远。
- 公式：  
  **原软件不改 + 远程串口 → 串口服务器 + 虚拟串口**
- 计算：  
  上位机通过虚拟COM口访问远端仪表。
- 结论：  
  这种场景串口服务器比工业网关更直接。

> [!tip] 记忆口诀
> 串口服务器管“串口怎么联网”，网关管“数据怎么理解和上传”。

# 04 RS232、RS485、RS422区别

> [!info] 核心结论
> 串口服务器选型必须先确认现场设备使用的是RS232、RS485还是RS422，不同接口不能随便混接。

## ① 分类/公式/规则

> [!warning] 接口匹配规则
> **设备是什么串口类型，串口服务器就要支持什么串口类型**

## ② 参考表

| 类型 | 特点 |
|---|---|
| **RS232** | 点对点通讯，距离较短，常见DB9接口 |
| **RS485** | 差分总线，距离较远，支持多站 |
| **RS422** | 差分通讯，常用于较高速或较远距离 |
| **半双工** | 常见RS485，两线收发共用 |
| **全双工** | RS232/RS422常见，可同时收发 |
| **终端电阻** | RS485/RS422长距离总线常需考虑 |

## ③ 计算/选型示例

- 已知：  
  现场有多台Modbus RTU仪表挂在同一条总线上。
- 公式：  
  **多站串口总线 → RS485常见**
- 计算：  
  RS232一般不适合多站总线。
- 结论：  
  应选择支持RS485的串口服务器。

> [!tip] 记忆口诀
> RS232一对一，RS485一拖多，RS422差分更稳。

# 05 通讯模式

> [!info] 核心结论
> 串口服务器常见模式有TCP Server、TCP Client、UDP、虚拟串口和Modbus网关模式。

## ① 分类/公式/规则

> [!warning] 模式选择规则
> **上位机主动连接设备 → TCP Server**  
> **串口服务器主动连接平台 → TCP Client**  
> **原软件只认COM口 → 虚拟串口**

## ② 参考表

| 模式 | 说明 |
|---|---|
| **TCP Server** | 串口服务器等待上位机连接 |
| **TCP Client** | 串口服务器主动连接远程服务器 |
| **UDP模式** | 无连接传输，开销低但可靠性弱 |
| **虚拟串口** | 在电脑上生成COM口访问远端串口 |
| **Real COM** | 类似虚拟串口，厂家驱动实现 |
| **Modbus网关** | Modbus RTU与Modbus TCP转换 |

## ③ 计算/选型示例

- 已知：  
  电脑上的老软件只能选择COM1、COM2，不能填IP地址。
- 公式：  
  **老软件只支持COM口 → 虚拟串口模式**
- 计算：  
  安装虚拟串口驱动，把远端串口映射成本地COM口。
- 结论：  
  应选择支持虚拟串口驱动的串口服务器。

> [!tip] 记忆口诀
> 软件能填IP就走TCP，软件只认COM就用虚拟串口。

# 06 TCP Server模式

> [!info] 核心结论
> TCP Server模式下，串口服务器作为服务器端监听端口，上位机主动连接它。

## ① 分类/公式/规则

> [!warning] 连接规则
> **上位机 → 连接串口服务器IP和端口 → 访问串口设备**

## ② 参考表

| 参数 | 说明 |
|---|---|
| **本机IP** | 串口服务器网络地址 |
| **监听端口** | 上位机连接的TCP端口 |
| **客户端数量** | 是否允许多个客户端连接 |
| **超时断开** | 空闲多久断开 |
| **串口参数** | 与现场设备一致 |
| **网络防火墙** | 端口需允许访问 |

## ③ 计算/选型示例

- 已知：  
  串口服务器IP为192.168.1.50，监听端口为4001。
- 公式：  
  **上位机连接 = IP + 端口**
- 计算：  
  上位机连接192.168.1.50:4001。
- 结论：  
  连接成功后，上位机即可通过网络访问该串口通道。

> [!tip] 记忆口诀
> TCP Server是“我在这等你连”。

# 07 TCP Client模式

> [!info] 核心结论
> TCP Client模式下，串口服务器主动连接远程服务器，适合数据主动上传或跨网络场景。

## ① 分类/公式/规则

> [!warning] 连接规则
> **串口服务器 → 主动连接远程服务器IP和端口**

## ② 参考表

| 参数 | 说明 |
|---|---|
| **远程服务器IP** | 数据接收服务器地址 |
| **远程端口** | 服务器监听端口 |
| **重连机制** | 断线后自动重连 |
| **心跳包** | 保持连接或检测掉线 |
| **串口数据触发** | 有数据时发送 |
| **网络可达性** | 服务器地址必须可访问 |

## ③ 计算/选型示例

- 已知：  
  现场串口设备需要把数据主动发到云端服务器。
- 公式：  
  **主动上传 → TCP Client**
- 计算：  
  串口服务器配置云端IP和端口，自动建立连接。
- 结论：  
  可使用TCP Client模式，并开启断线重连。

> [!tip] 记忆口诀
> TCP Client是“我主动去找服务器”。

# 08 UDP模式

> [!info] 核心结论
> UDP模式传输开销小、速度快，但没有TCP的连接确认和重传机制，可靠性相对弱。

## ① 分类/公式/规则

> [!warning] 选择规则
> **要求可靠传输 → TCP优先**  
> **简单广播、低延迟、允许少量丢包 → UDP可评估**

## ② 参考表

| 对比项 | TCP | UDP |
|---|---|---|
| 是否连接 | 有连接 | 无连接 |
| 可靠性 | **较高** | 较低 |
| 重传机制 | 有 | 无 |
| 开销 | 较大 | 较小 |
| 适合场景 | 可靠数据采集 | 简单透传、广播、低延迟 |
| 调试难度 | 较易 | 需注意丢包 |

## ③ 计算/选型示例

- 已知：  
  数据不能丢，一条记录都要保存。
- 公式：  
  **不允许丢数据 → TCP优先**
- 计算：  
  UDP不保证送达，丢包后应用层需要自己处理。
- 结论：  
  应选择TCP模式，而不是UDP模式。

> [!tip] 记忆口诀
> TCP稳，UDP快；工业数据采集多数优先稳。

# 09 虚拟串口

> [!info] 核心结论
> 虚拟串口可以把网络上的串口服务器映射成本地COM口，让旧软件无需修改即可访问远程串口设备。

## ① 分类/公式/规则

> [!warning] 虚拟串口链路
> **旧软件COM口 → 虚拟串口驱动 → 网络 → 串口服务器 → 串口设备**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **虚拟COM号** | COM3、COM4等 |
| **驱动软件** | 厂家提供或通用虚拟串口软件 |
| **IP与端口绑定** | 虚拟COM对应远程串口服务器 |
| **多通道映射** | 多个串口可映射多个COM |
| **断线重连** | 提高稳定性 |
| **系统兼容性** | Windows版本需确认 |

## ③ 计算/选型示例

- 已知：  
  原系统使用COM3读取仪表，现在仪表改为远程网络连接。
- 公式：  
  **COM3 → 映射到远程串口服务器**
- 计算：  
  在虚拟串口软件中把COM3绑定到串口服务器IP和端口。
- 结论：  
  原软件仍然选择COM3，不需要大改程序。

> [!tip] 记忆口诀
> 虚拟串口就是把远端串口伪装成本地串口。

# 10 Modbus RTU转Modbus TCP

> [!info] 核心结论
> 串口服务器常用于Modbus RTU与Modbus TCP转换，把RS485仪表接入以太网系统。

## ① 分类/公式/规则

> [!warning] 转换链路
> **Modbus TCP主站 → 串口服务器 → Modbus RTU从站**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **Modbus RTU** | 串口协议，常见于仪表和电表 |
| **Modbus TCP** | 以太网协议，常见于上位机和PLC |
| **站号** | RTU从站地址 |
| **功能码** | 读线圈、读寄存器、写寄存器等 |
| **寄存器地址** | 数据点地址 |
| **网关模式** | 串口服务器负责RTU/TCP封装转换 |

## ③ 计算/选型示例

- 已知：  
  电表为Modbus RTU，SCADA只支持Modbus TCP。
- 公式：  
  **RTU设备 → 串口服务器 → TCP系统**
- 计算：  
  SCADA访问串口服务器的IP和502端口，串口服务器转发到RS485电表。
- 结论：  
  选择支持Modbus网关模式的串口服务器。

> [!tip] 记忆口诀
> RTU在串口上跑，TCP在网线上跑，串口服务器负责中间转换。

# 11 串口参数

> [!info] 核心结论
> 串口通讯必须保证波特率、数据位、停止位、校验位一致，否则会乱码或完全不通。

## ① 分类/公式/规则

> [!warning] 串口成功公式
> **串口通讯成功 = 波特率一致 + 数据位一致 + 停止位一致 + 校验位一致 + 接线正确**

## ② 参考表

| 参数 | 常见值 |
|---|---|
| **波特率** | 9600、19200、38400、115200 |
| **数据位** | 7位、8位 |
| **停止位** | 1位、2位 |
| **校验位** | None、Even、Odd |
| **流控** | None、RTS/CTS、XON/XOFF |
| **站号** | Modbus等多站协议需要 |

## ③ 计算/选型示例

- 已知：  
  仪表参数为9600、8、N、1。
- 公式：  
  **串口服务器参数必须一致**
- 计算：  
  串口服务器也应设置9600波特率、8数据位、无校验、1停止位。
- 结论：  
  参数不一致会导致数据错误或无响应。

> [!tip] 记忆口诀
> 串口不通，先查9600、8、N、1这些基础参数。

# 12 RS485接线与终端电阻

> [!info] 核心结论
> RS485通讯要注意A/B线、总线拓扑、终端电阻、屏蔽接地和站号冲突。

## ① 分类/公式/规则

> [!warning] RS485规则
> **RS485稳定 = A/B正确 + 总线型布线 + 终端电阻 + 屏蔽接地 + 站号唯一**

## ② 参考表

| 项目 | 注意 |
|---|---|
| **A/B线** | 接反会导致通讯失败 |
| **总线拓扑** | 尽量避免星型乱接 |
| **终端电阻** | 长距离两端常用120Ω |
| **屏蔽线** | 强干扰环境建议使用 |
| **站号唯一** | 多从站不能重复 |
| **共地问题** | 远距离需注意参考地和隔离 |
| **偏置电阻** | 部分总线需保证空闲状态稳定 |

## ③ 计算/选型示例

- 已知：  
  RS485总线上有5台仪表，其中两台站号都为3。
- 公式：  
  **多站通讯 = 站号必须唯一**
- 计算：  
  主站访问3号时，两台仪表可能同时应答，导致数据冲突。
- 结论：  
  应修改其中一台仪表站号。

> [!tip] 记忆口诀
> RS485不怕距离，怕接反、乱接、站号重复。

# 13 多串口通道

> [!info] 核心结论
> 多串口服务器可同时连接多个串口设备或多条RS485总线，适合设备多、协议多、分区采集的场景。

## ① 分类/公式/规则

> [!warning] 通道选型公式
> **串口数量 ≥ 现场独立串口链路数量 + 预留通道**

## ② 参考表

| 通道数量 | 应用 |
|---|---|
| **1口** | 单台设备或一条RS485总线 |
| **2口** | 两条独立串口链路 |
| **4口** | 多仪表、多区域采集 |
| **8口** | 中大型串口设备联网 |
| **16口及以上** | 集中机房、老系统改造 |
| **独立配置** | 每个串口可单独设置参数 |

## ③ 计算/选型示例

- 已知：  
  现场有3条RS485总线，每条总线参数不同。
- 公式：  
  **不同参数总线 → 独立串口通道**
- 计算：  
  至少需要3个串口通道。
- 结论：  
  可选择4口串口服务器，并预留1路备用。

> [!tip] 记忆口诀
> 一条总线一个通道，参数不同不要硬并在一起。

# 14 隔离与防雷

> [!info] 核心结论
> 工业现场串口容易受雷击、浪涌、地电位差和电磁干扰影响，重要场景应选隔离型串口服务器。

## ① 分类/公式/规则

> [!warning] 防护选择规则
> **长距离RS485 / 室外线路 / 强干扰环境 → 隔离型 + 防雷浪涌优先**

## ② 参考表

| 防护项 | 作用 |
|---|---|
| **串口隔离** | 减少地电位差影响 |
| **电源隔离** | 提高抗干扰能力 |
| **浪涌保护** | 抵抗瞬态高压 |
| **防雷保护** | 室外线路重点考虑 |
| **ESD保护** | 抗静电冲击 |
| **金属外壳** | 屏蔽和散热较好 |
| **接地端子** | 需要可靠接地 |

## ③ 计算/选型示例

- 已知：  
  RS485线缆跨厂区铺设，雷雨天后设备频繁损坏。
- 公式：  
  **跨区域长线 → 防雷隔离**
- 计算：  
  长距离室外线缆容易引入浪涌。
- 结论：  
  应选隔离防雷型串口服务器，并增加浪涌保护和良好接地。

> [!tip] 记忆口诀
> 串口长线跨区域，隔离防雷不能省。

# 15 供电与安装

> [!info] 核心结论
> 串口服务器常安装在控制柜内，常用DC12V、DC24V或宽压输入，工业现场优先选择导轨安装和宽温型号。

## ① 分类/公式/规则

> [!warning] 稳定安装公式
> **稳定运行 = 稳定电源 + 可靠接地 + 规范布线 + 合理散热 + 清晰标识**

## ② 参考表

| 项目 | 注意 |
|---|---|
| **DC24V供电** | 工业柜内常见 |
| **宽压输入** | 适应电源波动 |
| **DIN导轨安装** | 控制柜安装方便 |
| **金属外壳** | 抗干扰和散热较好 |
| **工作温度** | 高温或室外场景需关注 |
| **串口端子** | 螺钉端子、凤凰端子、DB9等 |
| **网口标识** | 便于维护排查 |

## ③ 计算/选型示例

- 已知：  
  串口服务器功耗5W，使用DC24V供电。
- 公式：  
  **电流 = 功率 ÷ 电压**
- 计算：  
  电流 = 5 ÷ 24 ≈ **0.21A**
- 结论：  
  电源容量很小即可满足，但仍需留余量并保证供电稳定。

> [!tip] 记忆口诀
> 串口服务器功耗不大，但电源不稳照样掉线。

# 16 网络配置

> [!info] 核心结论
> 串口服务器网络配置重点是IP地址、子网掩码、网关、端口号、DHCP和防火墙。

## ① 分类/公式/规则

> [!warning] 网络可达公式
> **网络可达 = IP正确 + 网段一致/路由正确 + 端口开放 + 防火墙允许**

## ② 参考表

| 参数 | 说明 |
|---|---|
| **IP地址** | 串口服务器网络地址 |
| **子网掩码** | 判断同网段范围 |
| **网关** | 跨网段访问需要 |
| **端口号** | TCP/UDP访问入口 |
| **DHCP** | 自动获取IP，工业现场不一定推荐 |
| **静态IP** | 工业设备更常用 |
| **MAC地址** | 网络设备硬件地址 |

## ③ 计算/选型示例

- 已知：  
  工控机IP为192.168.1.100，串口服务器IP为192.168.2.50。
- 公式：  
  **不同网段通信 → 需要路由或改同网段**
- 计算：  
  如果没有路由，两者可能无法直接通信。
- 结论：  
  可把串口服务器改为192.168.1.x，或配置网关路由。

> [!tip] 记忆口诀
> 串口参数管串口通不通，IP参数管网络通不通。

# 17 常见故障排查

> [!info] 核心结论
> 串口服务器故障排查按 **供电、网口、IP、端口、串口参数、接线、协议、上位机软件** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **通讯异常 = 电源异常 + 网络不通 + 端口不通 + 串口参数错误 + A/B接反 + 协议不匹配 + 站号冲突**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **设备不上电** | 电源无电、极性接错、设备损坏 |
| **Ping不通** | IP错误、网线问题、网段不一致 |
| **端口连不上** | 模式错误、防火墙阻断、端口号错误 |
| **串口乱码** | 波特率、校验位、数据位不一致 |
| **RS485无响应** | A/B接反、站号错误、终端电阻问题 |
| **虚拟串口失败** | 驱动异常、COM号冲突、权限不足 |
| **Modbus读不到** | 功能码、地址、站号、寄存器偏移错误 |
| **偶发掉线** | 电源波动、干扰、网络不稳、线缆松动 |

## ③ 计算/选型示例

- 已知：  
  上位机能Ping通串口服务器，但读不到仪表数据。
- 公式：  
  **Ping通 ≠ 串口协议通**
- 计算：  
  网络层没问题，重点查串口参数、RS485接线、站号和协议地址。
- 结论：  
  应从串口侧继续排查，而不是只看网络。

> [!tip] 记忆口诀
> 网络通只是进了门，串口通才读得到设备。

# 18 串口服务器快速选型表

> [!info] 核心结论
> 串口服务器选型要先确认现场串口类型和应用模式，再确认串口数量、网络模式、协议转换、防护和安装方式。

## ① 分类/公式/规则

> [!note] 十步选型法
> ① 确认串口类型：RS232、RS485、RS422  
> ② 确认串口数量：1口、2口、4口、8口等  
> ③ 确认串口参数：波特率、数据位、停止位、校验位  
> ④ 确认应用模式：TCP Server、TCP Client、UDP、虚拟串口  
> ⑤ 确认是否需要Modbus RTU/TCP转换  
> ⑥ 确认网络接口和IP规划  
> ⑦ 确认是否需要隔离、防雷、浪涌保护  
> ⑧ 确认供电方式：DC12V、DC24V、宽压输入  
> ⑨ 确认安装方式：导轨、壁挂、机架  
> ⑩ 确认上位机软件、驱动、系统兼容性

> [!warning] 最终选型公式
> **推荐串口服务器 = 串口类型 + 串口数量 + 网络模式 + 协议功能 + 隔离防护 + 工业安装环境**

## ② 参考表

| 需求 | 推荐关注点 |
|---|---|
| 单台RS232设备联网 | 1口RS232串口服务器 |
| 多台RS485仪表采集 | RS485串口服务器 |
| 老软件继续用COM口 | 虚拟串口支持 |
| Modbus仪表接SCADA | RTU转TCP网关模式 |
| 多条总线独立采集 | 多串口通道 |
| 强干扰环境 | 隔离型、金属外壳 |
| 室外长距离 | 防雷、浪涌、屏蔽接地 |
| 工业柜安装 | DC24V、DIN导轨、宽温 |

## ③ 计算/选型示例

- 已知：  
  现场有2条RS485总线，每条接8台Modbus RTU仪表，上位机通过以太网采集数据。
- 公式：  
  **需求 = 2路RS485 + Modbus RTU/TCP + 以太网**
- 计算：  
  至少需要2个独立RS485通道，并支持Modbus网关或TCP透传。
- 结论：  
  推荐选择 **2口或4口RS485工业串口服务器，支持Modbus RTU转TCP、DC24V供电、导轨安装和隔离防护**。

> [!tip] 选型口诀
> 先看串口类型，再看几路通道；老软件看虚拟串口，Modbus看RTU转TCP；距离长、干扰强，就选隔离防雷。
