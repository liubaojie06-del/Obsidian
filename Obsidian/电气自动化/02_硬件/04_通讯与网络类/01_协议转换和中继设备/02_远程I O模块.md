# 01 远程I/O模块总览

> [!info] 核心结论
> 远程I/O模块用于把现场传感器、按钮、阀岛、电磁阀、模拟量仪表等信号就近接入，再通过通讯网络传回PLC或控制器，适合 **点位分散、距离较远、多工位、模块化设备和减少布线** 的场景。

## ① 分类/公式/规则

> [!warning] 选型公式
> **远程I/O选型 = I/O点数 + 信号类型 + 通讯协议 + 响应周期 + 供电方式 + 防护等级 + 安装环境 + 诊断能力**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **DI输入** | 接按钮、传感器、限位、开关量信号 |
| **DO输出** | 控制电磁阀、继电器、指示灯、阀岛 |
| **AI输入** | 采集压力、温度、流量、位移等模拟量 |
| **AO输出** | 控制比例阀、变频器给定、模拟量执行器 |
| **通讯协议** | PROFINET、EtherCAT、EtherNet/IP、Modbus等 |
| **安装方式** | 柜内DIN导轨、柜外IP65/IP67模块 |
| **核心价值** | 减少布线、便于扩展、提高维护效率 |

## ③ 计算/选型示例

- 已知：  
  一条产线有5个工位，每个工位有16DI和12DO。
- 公式：  
  **点位分散 + 工位独立 → 远程I/O优先**
- 计算：  
  如果全部拉线回主柜，线缆数量大、排查困难。
- 结论：  
  可每个工位配置一个远程I/O站，通过工业以太网连接PLC。

> [!tip] 记忆口诀
> 点位集中进柜，点位分散远程；线越长，越该考虑远程I/O。

# 02 底层原理

> [!info] 核心结论
> 远程I/O模块的底层原理，是在现场采集输入信号、驱动输出负载，再通过现场总线或工业以太网与PLC交换数据。

## ① 分类/公式/规则

> [!note] 工作链路
> 远程I/O模块本身通常不负责复杂逻辑判断，主要负责 **信号采集、信号输出、通讯交换和状态诊断**。  
> 真正的控制逻辑一般仍由PLC、PAC或运动控制器完成。

> [!warning] 控制链路
> **传感器/执行器 → 远程I/O模块 → 通讯网络 → PLC/控制器 → 程序逻辑 → 远程I/O输出**

## ② 参考表

| 环节 | 作用 |
|---|---|
| **现场输入** | 采集传感器、按钮、限位状态 |
| **现场输出** | 驱动电磁阀、继电器、指示灯 |
| **信号转换** | 把电信号转换为通讯数据 |
| **通讯交换** | 与PLC周期性刷新输入输出数据 |
| **诊断检测** | 检测断线、短路、掉站、低电压 |
| **地址映射** | 在PLC程序中对应输入输出地址 |

## ③ 计算/选型示例

- 已知：  
  远程I/O输入点检测到工件到位。
- 公式：  
  **现场信号 → I/O模块采集 → PLC程序判断**
- 计算：  
  远程I/O把输入状态上传给PLC，PLC判断后再控制输出动作。
- 结论：  
  远程I/O不是独立“大脑”，它是PLC延伸到现场的“手脚”。

> [!tip] 记忆口诀
> PLC是大脑，远程I/O是分布在现场的眼睛和手脚。

# 03 远程I/O vs 本地I/O

> [!info] 核心结论
> 本地I/O安装在PLC附近，远程I/O安装在现场工位附近；两者本质都是I/O，只是安装位置和通讯方式不同。

## ① 分类/公式/规则

> [!warning] 选择规则
> **点位集中、距离短 → 本地I/O**  
> **点位分散、距离远、线缆多 → 远程I/O**

## ② 参考表

| 对比项 | 本地I/O | 远程I/O |
|---|---|---|
| 安装位置 | PLC附近 | 现场工位附近 |
| 接线方式 | 集中回柜 | 就近接线 |
| 扩展方式 | PLC机架或本地模块 | 现场总线/工业以太网扩展 |
| 线缆数量 | 较多 | **较少** |
| 维护位置 | 控制柜内 | 现场就近 |
| 适合场景 | 点位集中 | 多工位、长距离、模块化产线 |
| 通讯依赖 | 较低 | **依赖网络通讯** |

## ③ 计算/选型示例

- 已知：  
  设备所有传感器距离PLC控制柜都在2米以内。
- 公式：  
  **距离短 + 点位集中 → 本地I/O即可**
- 计算：  
  直接接入PLC本地I/O布线简单、成本低。
- 结论：  
  不必强行使用远程I/O。

> [!tip] 记忆口诀
> 本地I/O近，远程I/O远；不是越高级越好，适合现场布局才好。

# 04 远程I/O vs 分布式I/O

> [!info] 核心结论
> 远程I/O强调“离PLC远”，分布式I/O强调“按区域分散布置”；实际项目中两者经常指同一类系统。

## ① 分类/公式/规则

> [!note] 概念理解
> 远程I/O通常通过通讯网络连接PLC。  
> 分布式I/O通常按工位、设备模块、生产单元分散布置。  
> 一个系统可以同时是远程I/O，也是分布式I/O。

> [!warning] 使用判断
> **按距离看叫远程I/O，按布局看叫分布式I/O**

## ② 参考表

| 名称 | 关注重点 |
|---|---|
| **远程I/O** | 与PLC距离远，通过通讯连接 |
| **分布式I/O** | 多个I/O站点分散在设备各处 |
| **柜内远程I/O** | 安装在电柜内，但与PLC通过总线连接 |
| **柜外远程I/O** | 安装在现场，常见IP65/IP67 |
| **模块化I/O站** | 按工位组合不同I/O模块 |
| **总线I/O岛** | 一个通讯头带多个I/O模块 |

## ③ 计算/选型示例

- 已知：  
  包装线有上料、检测、封口、下料4个独立工位。
- 公式：  
  **多工位独立 → 分布式I/O**
- 计算：  
  每个工位配置一个I/O站，主PLC统一控制。
- 结论：  
  这种方案既是分布式I/O，也是远程I/O。

> [!tip] 记忆口诀
> 远程看距离，分布看布局；现场说法不同，核心都是就近接入。

# 05 系统组成

> [!info] 核心结论
> 远程I/O系统通常由通讯耦合器、电源模块、DI/DO/AI/AO模块、端子模块、网络线缆和PLC组态组成。

## ① 分类/公式/规则

> [!warning] 系统公式
> **远程I/O站 = 通讯头/耦合器 + 电源 + I/O模块 + 端子 + 通讯线 + PLC组态**

## ② 参考表

| 部件 | 作用 |
|---|---|
| **通讯耦合器** | 与PLC通讯，连接现场总线 |
| **电源模块** | 给I/O站和负载供电 |
| **DI模块** | 采集开关量输入 |
| **DO模块** | 输出开关量控制负载 |
| **AI模块** | 采集模拟量信号 |
| **AO模块** | 输出模拟量控制信号 |
| **端子模块** | 接入现场线缆 |
| **配置文件** | GSD、ESI、EDS等设备描述文件 |

## ③ 计算/选型示例

- 已知：  
  某工位需要16DI、16DO、4AI。
- 公式：  
  **I/O站配置 = 通讯耦合器 + DI模块 + DO模块 + AI模块**
- 计算：  
  若每块DI/DO为16点，AI为4通道，则需要1块DI、1块DO、1块AI。
- 结论：  
  选型时不只选I/O点数，还要选对应通讯耦合器和电源模块。

> [!tip] 记忆口诀
> 远程I/O不是一块模块，而是一整套站点配置。

# 06 通讯协议

> [!info] 核心结论
> 远程I/O必须与PLC或控制器的通讯协议匹配，协议不匹配就无法正常组态和通信。

## ① 分类/公式/规则

> [!warning] 通讯选型公式
> **通讯可用 = PLC支持协议 + I/O站支持协议 + 配置文件 + 网络拓扑正确**

## ② 参考表

| 协议 | 常见场景 |
|---|---|
| **PROFINET** | 西门子系统、远程I/O、阀岛 |
| **EtherCAT** | 高速I/O、运动控制系统 |
| **EtherNet/IP** | 罗克韦尔及部分工业设备 |
| **Modbus TCP** | 通用工业以太网设备 |
| **Modbus RTU** | 串口远程I/O、仪表采集 |
| **CANopen** | 部分运动和嵌入式设备 |
| **CC-Link** | 三菱系统常见 |
| **IO-Link** | 智能传感器接入 |

## ③ 计算/选型示例

- 已知：  
  PLC系统为西门子PROFINET。
- 公式：  
  **PLC协议 = 远程I/O协议**
- 计算：  
  应选择支持PROFINET的远程I/O耦合器，并导入对应GSD文件。
- 结论：  
  优先选择与PLC原生协议一致的远程I/O模块。

> [!tip] 记忆口诀
> 协议对了才能连，点数够了才好用。

# 07 DI输入模块

> [!info] 核心结论
> DI模块用于采集按钮、接近开关、光电开关、磁性开关、限位、压力开关等开关量输入信号。

## ① 分类/公式/规则

> [!warning] DI匹配公式
> **DI可用 = 电压等级 + PNP/NPN类型 + 输入响应时间 + 公共端接法**

## ② 参考表

| 输入对象 | 说明 |
|---|---|
| **按钮** | 启动、停止、复位 |
| **接近开关** | 检测金属或机构位置 |
| **光电开关** | 检测工件有无 |
| **磁性开关** | 检测气缸伸出/缩回 |
| **限位开关** | 行程位置保护 |
| **压力开关** | 气压、液压到位检测 |
| **液位开关** | 液位高低报警 |

## ③ 计算/选型示例

- 已知：  
  一个工位有18个PNP传感器。
- 公式：  
  **DI点数 ≥ 实际输入点数 + 预留余量**
- 计算：  
  若预留20%，18 × 1.2 = **21.6点**
- 结论：  
  可选择24点DI输入容量，并确认PNP接线方式匹配。

> [!tip] 记忆口诀
> 输入先看PNP/NPN，再看点数和响应速度。

# 08 DO输出模块

> [!info] 核心结论
> DO模块用于控制电磁阀、继电器、指示灯、蜂鸣器、阀岛和小型执行器，重点校核单点电流和公共端总电流。

## ① 分类/公式/规则

> [!warning] DO匹配公式
> **DO可用 = 负载电压 + 单点输出电流 + 公共端总电流 + 输出类型**

## ② 参考表

| 输出对象 | 注意 |
|---|---|
| **电磁阀** | 注意线圈电流和浪涌 |
| **继电器** | 可用于隔离和扩流 |
| **指示灯** | 电流较小 |
| **蜂鸣器** | 注意声光报警器电流 |
| **阀岛** | 可用DO或总线方式控制 |
| **接触器** | 建议中间继电器隔离 |
| **小型执行器** | 校核启动电流和保护 |

## ③ 计算/选型示例

- 已知：  
  DO模块一个公共端下接10个电磁阀，每个0.12A，最多同时动作8个。
- 公式：  
  **公共端电流 = 单个电流 × 同时动作数量**
- 计算：  
  公共端电流 = 0.12 × 8 = **0.96A**
- 结论：  
  公共端总电流必须大于0.96A并留余量。

> [!tip] 记忆口诀
> 单点看单个负载，公共端看同时动作总电流。

# 09 AI模拟量输入模块

> [!info] 核心结论
> AI模块用于采集压力、温度、流量、液位、位移、电流、电压等连续信号。

## ① 分类/公式/规则

> [!warning] 模拟量换算公式
> **工程值 = 原始数字量 ÷ 满量程数字量 × 工程量满量程**

## ② 参考表

| AI信号类型 | 常见应用 |
|---|---|
| **0-10V** | 压力、位移、速度反馈 |
| **0-5V** | 部分传感器输出 |
| **4-20mA** | 压力、流量、液位、工业仪表 |
| **0-20mA** | 工业模拟量信号 |
| **热电偶TC** | 温度采集 |
| **热电阻RTD** | PT100、PT1000精密温度 |
| **称重信号** | 需专用称重模块 |

## ③ 计算/选型示例

- 已知：  
  压力传感器4-20mA对应0-1.0MPa，当前电流12mA。
- 公式：  
  **压力 = (12 - 4) ÷ (20 - 4) × 1.0**
- 计算：  
  压力 = 8 ÷ 16 × 1.0 = **0.5MPa**
- 结论：  
  当前压力约0.5MPa，程序中需做工程量换算。

> [!tip] 记忆口诀
> 开关量看ON/OFF，模拟量一定要看量程和换算。

# 10 AO模拟量输出模块

> [!info] 核心结论
> AO模块用于输出连续控制信号，常见于比例阀、变频器频率给定、调节阀、压力控制和速度给定。

## ① 分类/公式/规则

> [!warning] AO输出公式
> **输出信号 = 目标工程值 ÷ 满量程工程值 × 满量程信号**

## ② 参考表

| AO信号类型 | 常见应用 |
|---|---|
| **0-10V** | 变频器频率给定、比例阀 |
| **±10V** | 伺服速度/转矩给定 |
| **4-20mA** | 调节阀、工业仪表 |
| **0-20mA** | 部分工业控制设备 |
| **PWM输出** | 简单比例控制 |
| **专用控制模块** | 温控、称重、定位等 |

## ③ 计算/选型示例

- 已知：  
  0-10V对应变频器0-50Hz，希望输出30Hz。
- 公式：  
  **电压 = 30 ÷ 50 × 10**
- 计算：  
  电压 = **6V**
- 结论：  
  AO模块输出6V时，理论给定频率为30Hz。

> [!tip] 记忆口诀
> AI是把现场数值读进来，AO是把控制目标送出去。

# 11 柜内远程I/O

> [!info] 核心结论
> 柜内远程I/O安装在控制柜内，通过通讯连接PLC，适合控制柜分区、扩展模块较多或多柜系统。

## ① 分类/公式/规则

> [!warning] 选择规则
> **多柜体 / 多控制单元 / 需要模块化扩展 → 柜内远程I/O可评估**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **安装位置** | 控制柜内DIN导轨 |
| **防护要求** | 依赖控制柜防护 |
| **接线方式** | 端子排、弹簧端子、螺钉端子 |
| **优点** | 维护集中、模块保护好 |
| **缺点** | 现场线缆仍需拉回柜内 |
| **适合场景** | 多柜系统、中型产线、扩展I/O站 |

## ③ 计算/选型示例

- 已知：  
  一条产线有主控柜和两个分控柜，每个柜内都有大量I/O。
- 公式：  
  **多柜分区控制 → 柜内远程I/O**
- 计算：  
  每个分控柜配置远程I/O站，通过总线回主PLC。
- 结论：  
  可减少跨柜多芯线缆，提高系统模块化程度。

> [!tip] 记忆口诀
> 柜内远程I/O适合多柜分区，不一定必须装在设备外面。

# 12 柜外远程I/O

> [!info] 核心结论
> 柜外远程I/O安装在现场设备附近，常用M8/M12快插和IP65/IP67防护，适合现场点位分散和模块化设备。

## ① 分类/公式/规则

> [!warning] 柜外I/O选型公式
> **柜外I/O选型 = 防护等级 + 接插件类型 + 通讯协议 + 供电容量 + 点数 + 环境适应性**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **安装位置** | 设备机架、工位、现场模块 |
| **防护等级** | 常见IP65/IP67 |
| **接线方式** | M8/M12预制线缆 |
| **优点** | 线缆少、快插维护、模块化强 |
| **缺点** | 模块和接插件成本较高 |
| **适合场景** | 输送线、机器人周边、装配线、夹具模块 |

## ③ 计算/选型示例

- 已知：  
  远端夹具模块有8DI、8DO，距离主柜20米。
- 公式：  
  **远端点位 + 模块化夹具 → 柜外远程I/O**
- 计算：  
  使用IP67远程I/O可减少大量信号线回柜。
- 结论：  
  可选IP67柜外I/O模块，使用M12快插线缆。

> [!tip] 记忆口诀
> 柜外I/O靠近现场，线少、快换、维护直观。

# 13 供电设计

> [!info] 核心结论
> 远程I/O供电要区分模块逻辑电源、传感器电源和输出负载电源，输出负载多时必须分组保护。

## ① 分类/公式/规则

> [!warning] 供电校核公式
> **电源容量 ≥ 模块自身耗电 + 传感器电流 + 输出负载电流 × 同时系数 + 余量**

## ② 参考表

| 电源类型 | 作用 |
|---|---|
| **逻辑电源** | 给远程I/O模块内部电路供电 |
| **传感器电源** | 给接近开关、光电开关供电 |
| **负载电源** | 给电磁阀、继电器、指示灯供电 |
| **分组保险** | 防止单路短路影响整站 |
| **电源诊断** | 检测欠压、短路、过载 |
| **双电源输入** | 关键站点可提高可靠性 |

## ③ 计算/选型示例

- 已知：  
  一个远程I/O站有12个传感器，每个30mA；8个电磁阀，每个0.12A，最多同时动作6个。
- 公式：  
  **总电流 = 传感器电流 + 负载电流**
- 计算：  
  传感器电流 = 12 × 0.03 = **0.36A**  
  负载电流 = 6 × 0.12 = **0.72A**  
  合计约 **1.08A**
- 结论：  
  24V电源至少要大于1.08A，并留余量。

> [!tip] 记忆口诀
> 模块用电、传感器用电、负载用电要分开算。

# 14 通讯周期与响应时间

> [!info] 核心结论
> 远程I/O的响应时间不仅由PLC扫描周期决定，还受输入滤波、总线周期、I/O刷新周期和输出响应影响。

## ① 分类/公式/规则

> [!warning] 响应时间公式
> **响应时间 ≈ 输入滤波 + I/O采样 + 通讯周期 + PLC扫描周期 + 输出刷新 + 执行机构响应**

## ② 参考表

| 因素 | 影响 |
|---|---|
| **输入滤波** | 抗干扰，但会增加延迟 |
| **总线周期** | 决定I/O数据刷新速度 |
| **PLC扫描周期** | 影响逻辑判断速度 |
| **输出响应** | 输出模块动作延迟 |
| **网络负载** | 负载过高可能增加抖动 |
| **协议实时性** | EtherCAT通常更适合高速场景 |

## ③ 计算/选型示例

- 已知：  
  输入滤波5ms，总线周期2ms，PLC扫描周期10ms，输出响应3ms。
- 公式：  
  **控制侧响应 ≈ 5 + 2 + 10 + 3**
- 计算：  
  控制侧响应约 **20ms**
- 结论：  
  若设备要求10ms内响应，需要降低滤波、优化周期或使用高速I/O方案。

> [!tip] 记忆口诀
> 远程I/O不是瞬间反应，输入、网络、PLC、输出都要时间。

# 15 地址映射与组态

> [!info] 核心结论
> 远程I/O必须在PLC工程中正确组态，地址、模块顺序、设备名称和配置文件错误都会导致通信或点位异常。

## ① 分类/公式/规则

> [!warning] 组态规则
> **远程I/O可用 = 设备描述文件 + 模块顺序正确 + 地址映射正确 + 设备名称/IP正确**

## ② 参考表

| 配置项 | 说明 |
|---|---|
| **设备名称** | PROFINET设备名等 |
| **IP地址** | 工业以太网设备地址 |
| **站号/节点号** | 总线网络中设备编号 |
| **模块顺序** | 与现场实际插入顺序一致 |
| **输入地址** | PLC读取现场输入 |
| **输出地址** | PLC控制现场输出 |
| **诊断地址** | 读取模块状态和报警 |

## ③ 计算/选型示例

- 已知：  
  PLC程序中I0.0应该是工件到位，但现场触发后I0.1变化。
- 公式：  
  **点位错乱 = 地址映射 / 模块顺序 / 接线错误**
- 计算：  
  可能组态模块顺序与现场安装不一致。
- 结论：  
  应核对PLC组态、端子接线和变量表。

> [!tip] 记忆口诀
> 远程I/O点位不对，先查组态，再查接线。

# 16 诊断功能

> [!info] 核心结论
> 现代远程I/O模块通常支持掉站、短路、断线、欠压、过载和通道级诊断，能显著提升排故效率。

## ① 分类/公式/规则

> [!warning] 诊断价值公式
> **诊断能力越强 → 故障定位越快 → 停机时间越短**

## ② 参考表

| 诊断项 | 说明 |
|---|---|
| **模块掉站** | 通讯中断或设备离线 |
| **通道短路** | 输出负载或传感器供电短路 |
| **断线检测** | 部分AI、AO和智能模块支持 |
| **低电压报警** | 模块或负载供电不足 |
| **过载报警** | 输出负载电流过大 |
| **模块温度报警** | 环境温度或负载异常 |
| **通道级诊断** | 精确到某一个输入/输出通道 |

## ③ 计算/选型示例

- 已知：  
  某DO通道频繁报过载。
- 公式：  
  **DO过载 = 负载电流过大 / 短路 / 线缆损坏**
- 计算：  
  可直接定位该通道对应的电磁阀或线缆。
- 结论：  
  通道级诊断能减少大量人工查线时间。

> [!tip] 记忆口诀
> 没诊断靠人找，有诊断直接报点位。

# 17 IO-Link远程I/O

> [!info] 核心结论
> IO-Link远程I/O可以连接智能传感器和执行器，实现参数读取、远程配置、诊断和设备更换快速恢复。

## ① 分类/公式/规则

> [!warning] IO-Link价值公式
> **智能传感器 + IO-Link主站 = 数据采集 + 参数化 + 诊断能力**

## ② 参考表

| 功能 | 作用 |
|---|---|
| **传感器参数化** | 远程设置检测距离、阈值等 |
| **状态诊断** | 读取污染、过温、故障等状态 |
| **过程数据** | 读取测量值，不只是ON/OFF |
| **设备识别** | 读取厂家、型号、序列号 |
| **快速更换** | 替换传感器后自动下发参数 |
| **布线标准化** | 常用M12连接 |

## ③ 计算/选型示例

- 已知：  
  现场传感器经常更换，每次都要手动调参数。
- 公式：  
  **频繁更换 + 参数复杂 → IO-Link可评估**
- 计算：  
  IO-Link主站可保存参数并自动下发给新传感器。
- 结论：  
  可选带IO-Link主站功能的远程I/O模块。

> [!tip] 记忆口诀
> 普通DI只知道开关，IO-Link还能知道状态和参数。

# 18 安装与接线

> [!info] 核心结论
> 远程I/O安装接线要重点关注接地、屏蔽、强弱电分离、端子标识、M12锁紧、未用口防护和线缆固定。

## ① 分类/公式/规则

> [!warning] 稳定安装公式
> **远程I/O稳定 = 电源稳定 + 通讯可靠 + 接地正确 + 线缆规范 + 防护到位**

## ② 参考表

| 项目 | 注意 |
|---|---|
| **DIN导轨固定** | 柜内模块常用 |
| **M12接插件** | 柜外模块常用，需锁紧 |
| **未用接口防护帽** | 防止进水、进油、进尘 |
| **通讯线屏蔽** | 强干扰环境重要 |
| **模拟量屏蔽线** | 防止模拟量波动 |
| **强弱电分离** | 通讯线远离动力线 |
| **线号标识** | 后期维护必需 |

## ③ 计算/选型示例

- 已知：  
  柜外远程I/O雨后部分点异常。
- 公式：  
  **雨后异常 = 接插件进水 / 防护帽缺失 / 密封不良**
- 计算：  
  未用M12口如果没有防护帽，容易进水短路。
- 结论：  
  应检查接插件锁紧、防护帽和线缆密封。

> [!tip] 记忆口诀
> 柜外I/O最怕水进接头，未用接口一定要封好。

# 19 常见故障排查

> [!info] 核心结论
> 远程I/O故障排查按 **电源、通讯、组态、地址、I/O接线、负载、模块诊断、网络干扰** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **远程I/O异常 = 电源异常 + 通讯掉站 + 组态错误 + 地址映射错 + 接线错误 + 负载短路 + 模块故障**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **整站掉线** | 电源、网线、站名/IP、协议配置错误 |
| **部分输入无信号** | 传感器、电源、PNP/NPN、接线错误 |
| **部分输出不动作** | 负载电源、公共端、输出通道、负载故障 |
| **模块报警** | 短路、过载、欠压、模块损坏 |
| **点位错乱** | 模块顺序、地址映射、线号错误 |
| **模拟量波动** | 屏蔽接地差、干扰、量程配置错误 |
| **频繁掉站** | 网线差、交换机异常、干扰、供电不稳 |
| **PLC无法组态** | 缺少GSD/ESI/EDS文件或版本不匹配 |

## ③ 计算/选型示例

- 已知：  
  PLC显示远程I/O掉站。
- 公式：  
  **掉站 = 电源 / 通讯线 / 站名IP / 组态 / 模块故障**
- 计算：  
  先看远程I/O电源灯和通讯灯，再Ping设备或检查总线诊断。
- 结论：  
  掉站优先查电源和通讯，不要一开始就改程序。

> [!tip] 记忆口诀
> 整站掉线查网络和电源，单点异常查接线和负载。

# 20 远程I/O模块快速选型表

> [!info] 核心结论
> 远程I/O选型要从现场布局出发，先确认点数和信号类型，再确认协议、防护、供电、周期、诊断和扩展能力。

## ① 分类/公式/规则

> [!note] 十步选型法
> ① 统计DI、DO、AI、AO点数  
> ② 判断点位集中还是分散  
> ③ 确认柜内安装还是柜外安装  
> ④ 确认PLC或控制器通讯协议  
> ⑤ 确认PNP/NPN、电压等级和负载电流  
> ⑥ 确认模拟量类型、分辨率和精度  
> ⑦ 确认通讯周期和响应时间要求  
> ⑧ 确认供电分组和电源容量  
> ⑨ 确认诊断功能、防护等级和接插件  
> ⑩ 预留10%~20%点位和通讯扩展余量

> [!warning] 最终选型公式
> **推荐远程I/O方案 = 通讯协议 + I/O点数 + 信号类型 + 安装位置 + 防护等级 + 供电分组 + 诊断能力 + 扩展余量**

## ② 参考表

| 需求 | 推荐关注点 |
|---|---|
| 多工位产线 | 分布式远程I/O |
| 西门子PLC系统 | PROFINET远程I/O |
| 高速运动系统 | EtherCAT远程I/O |
| 普通仪表采集 | Modbus远程I/O |
| 柜外设备模块 | IP67远程I/O |
| 智能传感器 | IO-Link主站模块 |
| 模拟量采集 | AI精度、量程、屏蔽 |
| 高维护效率 | 通道级诊断、M12快插 |
| 高可靠要求 | 冗余电源、环网、诊断报警 |

## ③ 计算/选型示例

- 已知：  
  某设备远端工位有24DI、16DO、4AI，使用PROFINET通讯，安装在现场机架上，环境有粉尘和油污。
- 公式：  
  **需求 = 24DI + 16DO + 4AI + PROFINET + 柜外防护**
- 计算：  
  需要PROFINET远程I/O站，点数满足并预留余量；现场安装需IP65/IP67和M12接插件。
- 结论：  
  推荐选择 **PROFINET柜外远程I/O模块，配置24DI以上、16DO以上、4AI以上，支持IP67防护、M12快插、通道诊断和24V分组供电**。

> [!tip] 选型口诀
> 先数点，再看信号；远了用远程，分散用分布；协议要匹配，供电要分组，柜外要防护，诊断要到点。
