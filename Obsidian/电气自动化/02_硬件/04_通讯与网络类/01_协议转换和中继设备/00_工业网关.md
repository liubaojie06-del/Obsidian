# 01 工业网关总览

> [!info] 核心结论
> 工业网关用于实现 **设备数据采集、协议转换、边缘计算、数据上云、远程监控和系统集成**，常见于 PLC、仪表、传感器、MES、SCADA、云平台之间的数据连接。

## ① 分类/公式/规则

> [!warning] 选型公式
> **工业网关选型 = 采集对象 + 通讯协议 + 数据点数 + 上传方式 + 边缘计算 + 网络方式 + 安全要求 + 安装环境**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **采集对象** | PLC、仪表、变频器、传感器、机器人、CNC |
| **下行协议** | Modbus、S7、EtherNet/IP、串口协议等 |
| **上行协议** | MQTT、OPC UA、HTTP、数据库、云平台接口 |
| **核心功能** | 协议转换、数据采集、缓存、转发 |
| **边缘计算** | 本地计算、报警判断、数据清洗 |
| **远程维护** | 部分网关支持VPN或远程访问 |
| **典型场景** | 设备上云、MES采集、能耗监测、预测维护 |

## ③ 计算/选型示例

- 已知：  
  现场有多台PLC，需要把产量、报警、温度、压力数据上传MES。
- 公式：  
  **多设备数据采集 + MES上传 → 工业网关**
- 计算：  
  网关向下读取PLC数据，向上通过OPC UA、MQTT或数据库接口上传。
- 结论：  
  推荐使用 **工业网关**，并确认PLC协议和MES接口匹配。

> [!tip] 记忆口诀
> 网关就是工业数据翻译官，下面读设备，上面传系统。

# 02 底层原理

> [!info] 核心结论
> 工业网关的底层原理，是把不同设备、不同协议、不同格式的数据采集进来，再转换成上位系统能理解的数据格式。

## ① 分类/公式/规则

> [!note] 工作链路
> 工业网关通过以太网、串口、4G/5G、Wi-Fi等接口连接现场设备，按照对应协议读取数据，再进行格式转换、缓存、计算和上传。

> [!warning] 数据链路
> **现场设备 → 工业网关采集 → 协议解析 → 数据处理 → 缓存/转发 → MES/SCADA/云平台**

## ② 参考表

| 环节 | 作用 |
|---|---|
| **设备连接** | 连接PLC、仪表、传感器 |
| **协议解析** | 读取Modbus、S7、OPC UA等数据 |
| **数据映射** | 把寄存器地址映射为变量名 |
| **数据处理** | 换算、过滤、报警判断 |
| **数据缓存** | 网络断开时临时保存 |
| **数据上传** | MQTT、HTTP、OPC UA、数据库等 |
| **远程管理** | 参数配置、状态监控、固件升级 |

## ③ 计算/选型示例

- 已知：  
  温度仪表使用Modbus RTU，云平台只接受MQTT数据。
- 公式：  
  **Modbus RTU → 工业网关 → MQTT**
- 计算：  
  网关读取串口仪表寄存器，再把温度值打包成MQTT消息。
- 结论：  
  工业网关实现了 **协议转换和数据上传**。

> [!tip] 记忆口诀
> 设备说设备语言，平台说平台语言，中间靠网关翻译。

# 03 工业网关 vs 工业路由器

> [!info] 核心结论
> 工业路由器主要解决联网和远程访问，工业网关主要解决协议转换和数据采集。

## ① 分类/公式/规则

> [!warning] 选择规则
> **只需要联网/VPN远程 → 工业路由器**  
> **需要读PLC/仪表数据并上传 → 工业网关**

## ② 参考表

| 对比项 | 工业路由器 | 工业网关 |
|---|---|---|
| 核心功能 | 网络连接、VPN、NAT、防火墙 | **协议转换、数据采集、边缘处理** |
| 是否解析PLC数据 | 通常不解析 | **通常支持** |
| 远程维护 | 强 | 视型号 |
| 数据上云 | 部分支持 | **常见** |
| 通讯协议 | 网络协议为主 | **工业协议更多** |
| 应用重点 | 让设备连上网络 | 让设备数据被系统使用 |

## ③ 计算/选型示例

- 已知：  
  设备只需要工程师远程访问PLC。
- 公式：  
  **远程访问PLC → 工业路由器更合适**
- 计算：  
  不需要解析PLC数据，只需要VPN通道。
- 结论：  
  可选工业路由器；如果还要采集数据上传平台，再选工业网关或路由网关一体机。

> [!tip] 记忆口诀
> 路由器负责“连得上”，网关负责“读得懂、传得出”。

# 04 工业网关 vs 工业交换机

> [!info] 核心结论
> 工业交换机负责同一网络内设备互联，工业网关负责不同协议和系统之间的数据转换。

## ① 分类/公式/规则

> [!warning] 区别规则
> **多台设备组网 → 工业交换机**  
> **设备数据转换上传 → 工业网关**

## ② 参考表

| 对比项 | 工业交换机 | 工业网关 |
|---|---|---|
| 主要作用 | 网络转发 | **协议转换与数据采集** |
| 工作重点 | MAC转发、端口连接 | 数据读取、解析、上传 |
| 是否处理变量 | 不处理 | **处理变量** |
| 是否需要配置点表 | 通常不需要 | **通常需要** |
| 典型对象 | PLC、HMI、相机、IO联网 | PLC、仪表、MES、云平台对接 |
| 本质 | 网络设备 | 数据设备 |

## ③ 计算/选型示例

- 已知：  
  PLC、HMI、工控机要接到同一网络。
- 公式：  
  **同网段设备互联 → 工业交换机**
- 计算：  
  交换机即可完成网络连接，不需要解析数据。
- 结论：  
  只联网用交换机；要采集数据上传系统才需要网关。

> [!tip] 记忆口诀
> 交换机管线通不通，网关管数据懂不懂。

# 05 工业网关 vs 工控机

> [!info] 核心结论
> 工控机计算能力强、软件灵活；工业网关更轻量、更稳定、更适合固定协议采集和设备上云。

## ① 分类/公式/规则

> [!warning] 选择规则
> **复杂软件、视觉、数据库、本地界面 → 工控机**  
> **标准化采集、协议转换、上云 → 工业网关**

## ② 参考表

| 对比项 | 工控机 | 工业网关 |
|---|---|---|
| 计算能力 | 强 | 中等或轻量 |
| 软件自由度 | 高 | 取决于厂家平台 |
| 部署复杂度 | 较高 | **较低** |
| 稳定性 | 取决于系统维护 | **更偏嵌入式稳定运行** |
| 数据采集 | 需软件开发 | **内置采集功能常见** |
| 维护方式 | 系统、软件、驱动维护 | Web配置、远程平台维护 |

## ③ 计算/选型示例

- 已知：  
  现场只需要采集100个PLC变量上传云平台。
- 公式：  
  **标准点位采集 → 工业网关更简洁**
- 计算：  
  不需要复杂上位机软件和显示界面。
- 结论：  
  工业网关比工控机更适合轻量数据采集场景。

> [!tip] 记忆口诀
> 要复杂计算用工控机，要稳定采集用工业网关。

# 06 常见类型

> [!info] 核心结论
> 工业网关按功能可分为协议转换网关、边缘计算网关、物联网网关、远程维护网关、串口网关和安全网关。

## ① 分类/公式/规则

> [!warning] 类型选择规则
> **协议不通 → 协议转换网关**  
> **数据上云 → 物联网网关**  
> **现场本地判断 → 边缘计算网关**  
> **串口设备联网 → 串口网关**

## ② 参考表

| 类型 | 适合场景 |
|---|---|
| **协议转换网关** | Modbus转OPC UA、S7转MQTT等 |
| **物联网网关** | 设备数据上云、远程监控 |
| **边缘计算网关** | 本地计算、报警判断、数据清洗 |
| **串口网关** | RS232/RS485设备联网 |
| **远程维护网关** | VPN远程访问PLC/HMI |
| **安全网关** | 网络隔离、防火墙、白名单 |
| **能源网关** | 电表、水表、气表、能耗采集 |

## ③ 计算/选型示例

- 已知：  
  老旧仪表只有RS485接口，需要接入以太网系统。
- 公式：  
  **串口设备 → 串口网关 → 以太网**
- 计算：  
  网关读取RS485数据，再转换为Modbus TCP或MQTT。
- 结论：  
  推荐使用带RS485接口的工业串口网关。

> [!tip] 记忆口诀
> 协议不通找转换，设备上云找物联网，现场计算找边缘。

# 07 下行采集协议

> [!info] 核心结论
> 下行协议是工业网关读取现场设备数据的协议，必须与PLC、仪表、驱动器或控制器匹配。

## ① 分类/公式/规则

> [!warning] 下行协议匹配公式
> **采集成功 = 设备协议 + 地址参数 + 通讯接口 + 点表配置正确**

## ② 参考表

| 下行协议 | 常见对象 |
|---|---|
| **Modbus RTU** | 串口仪表、电表、温控器 |
| **Modbus TCP** | PLC、控制器、仪表 |
| **Siemens S7** | 西门子PLC |
| **EtherNet/IP** | 罗克韦尔PLC、部分设备 |
| **OPC UA Client** | 读取OPC UA服务器数据 |
| **CANopen** | 部分驱动器和设备 |
| **自定义串口协议** | 老设备、专用仪表 |

## ③ 计算/选型示例

- 已知：  
  网关要读取西门子PLC数据。
- 公式：  
  **PLC品牌型号 → 网关需支持对应协议**
- 计算：  
  若PLC使用S7通讯，网关必须支持S7协议或通过OPC UA读取。
- 结论：  
  选网关前必须确认PLC型号和开放的通讯方式。

> [!tip] 记忆口诀
> 采集前先问设备说什么协议。

# 08 上行上传协议

> [!info] 核心结论
> 上行协议是工业网关把数据发送给MES、SCADA、数据库、云平台或第三方系统的方式。

## ① 分类/公式/规则

> [!warning] 上行协议选择公式
> **上传方式 = 目标系统接口 + 实时性 + 数据格式 + 安全要求**

## ② 参考表

| 上行协议 | 常见用途 |
|---|---|
| **MQTT** | 物联网平台、云平台 |
| **OPC UA Server** | SCADA、MES、上位系统读取 |
| **HTTP/HTTPS API** | Web平台、MES接口 |
| **TCP/UDP Socket** | 自定义系统通讯 |
| **数据库写入** | SQL Server、MySQL、PostgreSQL |
| **FTP/SFTP** | 文件上传、报表上传 |
| **AMQP/Kafka** | 大规模数据平台 |

## ③ 计算/选型示例

- 已知：  
  云平台要求设备数据通过MQTT上传。
- 公式：  
  **云平台MQTT → 网关需支持MQTT Client**
- 计算：  
  网关把采集到的变量打包成JSON或平台要求格式后发布。
- 结论：  
  应选择支持MQTT、Topic配置和认证加密的工业网关。

> [!tip] 记忆口诀
> 下行看设备协议，上行看平台接口。

# 09 点表配置

> [!info] 核心结论
> 点表是工业网关采集数据的核心配置，定义采什么地址、叫什么变量、什么类型、多久采一次、怎么换算。

## ① 分类/公式/规则

> [!warning] 点表配置公式
> **点位 = 设备地址 + 寄存器地址 + 数据类型 + 采样周期 + 工程量换算**

## ② 参考表

| 点表字段 | 说明 |
|---|---|
| **设备名称** | PLC1、电表1、温控器1 |
| **变量名称** | 温度、压力、产量、报警 |
| **寄存器地址** | Modbus地址、PLC地址 |
| **数据类型** | Bool、Int、DInt、Float、String |
| **字节序** | 大端、小端、字交换 |
| **采样周期** | 1s、5s、10s等 |
| **单位/量程** | ℃、MPa、kWh等 |

## ③ 计算/选型示例

- 已知：  
  Modbus寄存器40001为温度原始值，读数为253，倍率0.1。
- 公式：  
  **实际温度 = 原始值 × 倍率**
- 计算：  
  实际温度 = 253 × 0.1 = **25.3℃**
- 结论：  
  点表中应配置倍率换算，否则上传数据不是真实工程值。

> [!tip] 记忆口诀
> 点表配错，数据全错；地址、类型、字节序一个都不能错。

# 10 数据类型与字节序

> [!info] 核心结论
> 工业网关采集数据时，数据类型和字节序非常关键，配置错误会导致数值异常。

## ① 分类/公式/规则

> [!warning] 数据解析规则
> **数值正确 = 地址正确 + 类型正确 + 字节序正确 + 倍率正确**

## ② 参考表

| 数据类型 | 说明 |
|---|---|
| **Bool** | 开关量、报警位 |
| **UInt16/Int16** | 16位整数 |
| **UInt32/Int32** | 32位整数 |
| **Float** | 浮点数，常用于温度、压力 |
| **Double** | 双精度浮点 |
| **String** | 字符串、批号 |
| **Bit位** | 一个寄存器中的单个位 |

## ③ 计算/选型示例

- 已知：  
  网关读取压力值，显示为极大异常数。
- 公式：  
  **异常数值 = 数据类型错误 / 字节序错误**
- 计算：  
  PLC实际存Float，但网关按Int读取，就会显示错误。
- 结论：  
  应检查变量类型、寄存器顺序和高低字节设置。

> [!tip] 记忆口诀
> 地址对了不一定数值对，类型和字节序也要对。

# 11 数据采样周期

> [!info] 核心结论
> 采样周期决定数据刷新频率，周期太长数据不及时，周期太短会增加PLC、网关和网络负载。

## ① 分类/公式/规则

> [!warning] 采样周期选择规则
> **关键快速变量 → 短周期**  
> **普通统计变量 → 长周期**  
> **不要所有点都用最快周期**

## ② 参考表

| 数据类型 | 采样周期趋势 |
|---|---|
| **报警状态** | 较短 |
| **设备运行状态** | 较短 |
| **温度压力** | 中等 |
| **电表能耗** | 较长 |
| **产量计数** | 中等 |
| **工艺配方** | 变化时采集即可 |
| **历史报表** | 可按分钟或小时 |

## ③ 计算/选型示例

- 已知：  
  500个变量，每个变量每秒采集1次。
- 公式：  
  **采集负载 = 点数 × 采样频率**
- 计算：  
  采集负载 = 500 × 1 = **500点/秒**
- 结论：  
  需要确认PLC通讯能力、网关性能和网络带宽是否能承受。

> [!tip] 记忆口诀
> 不是所有数据都要秒级采集，采得太快也会拖垮系统。

# 12 边缘计算

> [!info] 核心结论
> 边缘计算网关可以在现场本地完成数据清洗、公式计算、报警判断、统计汇总和异常检测。

## ① 分类/公式/规则

> [!warning] 边缘计算公式
> **上传结果 = 原始数据 → 本地计算 → 过滤/压缩 → 关键结果上传**

## ② 参考表

| 功能 | 应用 |
|---|---|
| **数据清洗** | 去除异常值、空值 |
| **工程换算** | 原始值转工程值 |
| **报警判断** | 超限、本地联动 |
| **统计汇总** | 平均值、最大值、最小值 |
| **能耗计算** | 班次能耗、单件能耗 |
| **状态判断** | 开机、停机、待机、故障 |
| **脚本计算** | JavaScript、Python、Lua等视平台支持 |

## ③ 计算/选型示例

- 已知：  
  设备每秒上传1000个原始点，但平台只需要每分钟平均值。
- 公式：  
  **边缘汇总 → 减少上传数据**
- 计算：  
  网关在本地计算每分钟平均值，再上传1条结果。
- 结论：  
  边缘计算可降低网络压力和平台存储压力。

> [!tip] 记忆口诀
> 能在现场算完的，就别把原始数据全往上传。

# 13 数据缓存与断点续传

> [!info] 核心结论
> 工业现场网络不稳定时，网关需要本地缓存和断点续传，防止数据丢失。

## ① 分类/公式/规则

> [!warning] 可靠上传公式
> **可靠数据上传 = 本地缓存 + 时间戳 + 网络恢复补传 + 重复数据处理**

## ② 参考表

| 功能 | 作用 |
|---|---|
| **本地缓存** | 断网时保存数据 |
| **时间戳** | 保留真实采集时间 |
| **断点续传** | 网络恢复后补传 |
| **去重机制** | 防止重复写入 |
| **缓存容量** | 决定可断网多久 |
| **存储介质** | 内置Flash、SD卡、SSD |
| **上传状态** | 已上传/未上传标记 |

## ③ 计算/选型示例

- 已知：  
  每分钟产生1MB数据，现场可能断网6小时。
- 公式：  
  **缓存容量 ≥ 1MB/min × 60 × 6**
- 计算：  
  缓存容量 ≥ **360MB**
- 结论：  
  网关本地存储至少要大于360MB，并留余量。

> [!tip] 记忆口诀
> 网络断了不可怕，网关能缓存才可靠。

# 14 时间戳与时钟同步

> [!info] 核心结论
> 数据追溯必须有准确时间戳，工业网关应支持NTP、RTC或平台时间同步。

## ① 分类/公式/规则

> [!warning] 时间规则
> **追溯数据 = 数据值 + 准确时间戳 + 设备来源**

## ② 参考表

| 时间功能 | 说明 |
|---|---|
| **RTC时钟** | 断电后保持时间 |
| **NTP同步** | 从网络时间服务器同步 |
| **平台授时** | 云平台或上位系统下发时间 |
| **本地时区** | 防止时间偏移 |
| **采集时间** | 数据产生时间 |
| **上传时间** | 数据上传到平台时间 |
| **断网补传时间** | 要保留原始采集时间 |

## ③ 计算/选型示例

- 已知：  
  断网期间数据补传后，平台显示时间全部集中在恢复网络时刻。
- 公式：  
  **补传数据应使用采集时间，不应只用上传时间**
- 计算：  
  否则无法判断故障实际发生时间。
- 结论：  
  网关应记录原始采集时间戳，并随数据一起上传。

> [!tip] 记忆口诀
> 没有时间戳，数据只能看数值，不能做追溯。

# 15 网络接口与联网方式

> [!info] 核心结论
> 工业网关常用以太网、RS232、RS485、CAN、Wi-Fi、4G/5G等接口连接设备和平台。

## ① 分类/公式/规则

> [!warning] 接口选型公式
> **接口需求 = 下行设备接口 + 上行网络接口 + 备用维护接口**

## ② 参考表

| 接口 | 常见用途 |
|---|---|
| **LAN以太网** | PLC、控制器、交换机 |
| **WAN以太网** | 上联工厂网络或互联网 |
| **RS485** | Modbus RTU仪表、电表 |
| **RS232** | 老设备、扫码器、仪表 |
| **CAN/CANopen** | 驱动器、车载、嵌入式设备 |
| **Wi-Fi** | 无线接入 |
| **4G/5G** | 远程站点、无有线网络场景 |

## ③ 计算/选型示例

- 已知：  
  现场有2台PLC以太网接口、10台RS485电表，需要上传云平台。
- 公式：  
  **网关接口 ≥ 以太网 + RS485 + 上行网络**
- 计算：  
  需要至少1个以太网口、1个RS485口和上行网络接口。
- 结论：  
  推荐选择带多网口和RS485的工业网关。

> [!tip] 记忆口诀
> 下接设备看接口，上接平台看网络。

# 16 安全与权限

> [!info] 核心结论
> 工业网关连接现场设备和上位系统，必须关注账号权限、数据加密、访问控制、证书认证和日志审计。

## ① 分类/公式/规则

> [!warning] 安全公式
> **安全网关 = 加密传输 + 强密码 + 权限分级 + 白名单 + 日志记录 + 固件更新**

## ② 参考表

| 安全项 | 作用 |
|---|---|
| **账号权限** | 区分管理员、工程师、只读用户 |
| **HTTPS/TLS** | 加密Web配置和数据上传 |
| **MQTT证书** | 云平台认证 |
| **防火墙** | 限制不必要访问 |
| **白名单** | 只允许指定IP或设备访问 |
| **日志审计** | 记录登录、配置、上传状态 |
| **固件升级** | 修复漏洞和功能更新 |

## ③ 计算/选型示例

- 已知：  
  网关使用默认密码上线运行。
- 公式：  
  **默认密码 = 高安全风险**
- 计算：  
  外部人员可能通过默认账号登录网关。
- 结论：  
  上线前必须修改默认密码，并关闭不必要服务。

> [!tip] 记忆口诀
> 网关连着设备和平台，安全不能只靠默认配置。

# 17 安装与供电

> [!info] 核心结论
> 工业网关通常安装在控制柜内，常用DIN导轨安装和DC24V供电，要注意接地、散热、线缆、天线和电源稳定性。

## ① 分类/公式/规则

> [!warning] 稳定安装公式
> **稳定运行 = 稳定电源 + 良好接地 + 合理散热 + 规范布线 + 天线位置正确**

## ② 参考表

| 项目 | 注意 |
|---|---|
| **DC24V供电** | 工业控制柜常见 |
| **宽压输入** | 抗电源波动 |
| **DIN导轨安装** | 方便柜内安装 |
| **接地端子** | 提高抗干扰能力 |
| **串口屏蔽** | RS485远距离需关注 |
| **网线质量** | 工业现场建议用可靠线缆 |
| **外置天线** | 4G/5G或Wi-Fi不要被金属柜屏蔽 |

## ③ 计算/选型示例

- 已知：  
  4G网关装在金属控制柜内，信号很弱。
- 公式：  
  **金属柜屏蔽 → 天线外置**
- 计算：  
  柜体会衰减无线信号，导致上传不稳定。
- 结论：  
  应把4G天线引到柜外，并检查信号强度。

> [!tip] 记忆口诀
> 网关可以在柜里，天线不要闷在铁柜里。

# 18 常见故障排查

> [!info] 核心结论
> 工业网关故障排查按 **供电、设备通讯、点表、网络、平台接口、缓存、权限、安全策略** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **网关异常 = 供电异常 + 设备协议不通 + 点表错误 + 网络异常 + 平台认证失败 + 权限不足 + 防火墙阻断**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **网关不上电** | 电源无电、接线错误、设备故障 |
| **读不到PLC** | 协议不匹配、IP错误、端口未开、PLC未运行 |
| **读不到仪表** | RS485接反、波特率错误、站号错误 |
| **数值异常** | 地址错、数据类型错、字节序错、倍率错 |
| **上传失败** | 网络断开、平台地址错、证书错误 |
| **数据延迟大** | 采样周期太长、网络慢、点数过多 |
| **断网丢数据** | 未开启缓存或缓存容量不足 |
| **远程配置失败** | 权限、防火墙、平台连接异常 |

## ③ 计算/选型示例

- 已知：  
  网关能Ping通PLC，但读不到数据。
- 公式：  
  **Ping通 ≠ 协议可读**
- 计算：  
  可能PLC通讯端口未开放、协议不支持、机架槽号错误或访问权限不足。
- 结论：  
  应检查PLC协议配置、端口、访问权限和网关驱动设置。

> [!tip] 记忆口诀
> 网络通只是第一步，协议通才是真的能采集。

# 19 工业网关快速选型表

> [!info] 核心结论
> 工业网关选型要先确认下行设备和上行系统，再确定协议、点数、采样周期、缓存、安全、供电和安装环境。

## ① 分类/公式/规则

> [!note] 十步选型法
> ① 确认采集对象：PLC、仪表、变频器、机器人、CNC  
> ② 确认下行协议：Modbus、S7、EtherNet/IP、OPC UA等  
> ③ 统计采集点数和采样周期  
> ④ 确认数据类型、倍率、单位和点表格式  
> ⑤ 确认上行接口：MQTT、OPC UA、HTTP、数据库等  
> ⑥ 确认是否需要边缘计算、报警判断和数据清洗  
> ⑦ 确认是否需要断网缓存和断点续传  
> ⑧ 确认联网方式：以太网、Wi-Fi、4G/5G  
> ⑨ 确认安全认证、加密、权限和日志  
> ⑩ 确认供电、安装、温度、接地和天线

> [!warning] 最终选型公式
> **推荐工业网关 = 下行协议 + 上行协议 + 点数容量 + 边缘计算 + 缓存能力 + 联网方式 + 安全能力 + 工业环境等级**

## ② 参考表

| 需求 | 推荐关注点 |
|---|---|
| PLC数据采集 | PLC协议支持、点表配置 |
| 仪表采集 | RS485、Modbus RTU、站号管理 |
| 设备上云 | MQTT、HTTP、证书认证 |
| MES/SCADA对接 | OPC UA、数据库、API |
| 网络不稳定 | 本地缓存、断点续传 |
| 本地报警判断 | 边缘计算、脚本功能 |
| 多设备接入 | 多协议、多通道、多点数 |
| 远程维护 | VPN或远程平台功能 |
| 高安全要求 | 加密、白名单、日志审计 |

## ③ 计算/选型示例

- 已知：  
  项目有3台PLC、12台RS485电表、共800个采集点，要求每5秒上传一次到云平台，协议为MQTT。
- 公式：  
  **网关需求 = 多PLC协议 + RS485采集 + 800点 + 5秒周期 + MQTT上传**
- 计算：  
  网关需要支持多设备连接、足够点数容量、MQTT上传、缓存和时间戳。
- 结论：  
  推荐选择 **支持PLC协议、Modbus RTU、MQTT、断网缓存、边缘计算和工业导轨安装的工业物联网网关**。

> [!tip] 选型口诀
> 先看下面读什么，再看上面传给谁；点数决定容量，周期决定性能；断网要缓存，上云要安全。
