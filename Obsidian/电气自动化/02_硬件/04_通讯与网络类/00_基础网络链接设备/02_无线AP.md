# 01 无线AP / 网桥总览

> [!info] 核心结论
> 无线AP用于把有线网络扩展成无线网络；无线网桥用于把两个有线网络通过无线方式连接起来，常用于 **移动设备联网、厂区无线覆盖、两地设备互联、AGV/机器人通信、无法布线场景**。

## ① 分类/公式/规则

> [!warning] 选型公式
> **无线AP/网桥选型 = 通讯距离 + 带宽需求 + 频段 + 天线 + 安装环境 + 供电方式 + 安全加密 + 漫游需求**

## ② 参考表

| 类型 | 作用 |
|---|---|
| **无线AP** | 提供Wi-Fi热点，让终端无线接入网络 |
| **无线网桥** | 两点之间无线桥接，相当于无线网线 |
| **客户端模式** | 让有线设备通过无线接入网络 |
| **中继模式** | 扩大无线覆盖范围 |
| **Mesh无线** | 多节点无线组网 |
| **工业无线AP** | 适合车间、仓库、设备现场等工业环境 |

## ③ 计算/选型示例

- 已知：  
  一台设备在车间移动，无法长期接网线，但需要访问PLC数据。
- 公式：  
  **移动设备 + 网络接入 = 无线AP/客户端方案**
- 计算：  
  固定网络用网线更稳定，移动设备可通过Wi-Fi接入。
- 结论：  
  可在车间布置工业无线AP，移动设备通过无线连接网络。

> [!tip] 记忆口诀
> AP负责“让设备连无线”，网桥负责“让两地像接了一根无线网线”。

# 02 底层原理

> [!info] 核心结论
> 无线AP/网桥的底层原理，是把以太网数据转换成无线射频信号发送，再由另一端接收并还原成网络数据。

## ① 分类/公式/规则

> [!note] 工作链路
> 有线网络数据进入AP或网桥后，被调制成无线信号，通过天线发射；  
> 接收端天线收到无线信号后，再解调成以太网数据。

> [!warning] 通讯链路
> **有线数据 → 无线设备 → 射频信号 → 天线传播 → 接收设备 → 有线/无线终端**

## ② 参考表

| 环节 | 作用 |
|---|---|
| **以太网口** | 接入PLC、交换机、工控机等有线网络 |
| **无线射频模块** | 负责Wi-Fi信号收发 |
| **天线** | 决定覆盖方向和距离 |
| **加密认证** | 控制谁可以接入 |
| **无线信道** | 决定使用的频率资源 |
| **无线协议** | 决定速率、兼容性和稳定性 |
| **网络管理** | 配置SSID、IP、功率、信道等参数 |

## ③ 计算/选型示例

- 已知：  
  两栋厂房之间无法拉网线，但需要网络互通。
- 公式：  
  **两地网络互通 + 无法布线 = 无线网桥**
- 计算：  
  A厂房网桥连接交换机，B厂房网桥连接交换机，两端无线对接。
- 结论：  
  可用一对无线网桥建立点对点链路。

> [!tip] 记忆口诀
> 看不见的是无线信号，本质传的还是网络数据。

# 03 无线AP vs 无线网桥

> [!info] 核心结论
> AP主要服务无线终端接入，网桥主要服务两个固定网络之间的无线互联。

## ① 分类/公式/规则

> [!warning] 选择规则
> **手机、PDA、平板、笔记本接入 → 无线AP**  
> **两个车间、两栋楼、两台设备远距离互联 → 无线网桥**

## ② 参考表

| 对比项 | 无线AP | 无线网桥 |
|---|---|---|
| 主要作用 | 提供Wi-Fi接入 | 两点网络互联 |
| 连接对象 | 手机、PDA、工控机、移动终端 | 交换机、PLC、摄像头、远端网络 |
| 覆盖方式 | 面覆盖 | 点对点/点对多点 |
| 天线类型 | 全向或定向 | 多用定向天线 |
| 常见场景 | 车间Wi-Fi覆盖 | 跨楼、跨道路、远距离联网 |
| 重点参数 | 覆盖、并发、漫游 | 距离、视距、带宽、稳定性 |

## ③ 计算/选型示例

- 已知：  
  仓库有多台手持终端需要联网。
- 公式：  
  **多终端无线接入 → 无线AP**
- 计算：  
  手持终端需要连接Wi-Fi热点，不是两个固定网络互联。
- 结论：  
  应选择工业无线AP，而不是单纯点对点网桥。

> [!tip] 记忆口诀
> 人和移动终端连无线，用AP；两个固定点连网络，用网桥。

# 04 无线AP vs 工业路由器

> [!info] 核心结论
> 无线AP主要负责无线接入，工业路由器负责不同网络互联、上网、VPN和NAT。

## ① 分类/公式/规则

> [!warning] 区别规则
> **扩展Wi-Fi覆盖 → 无线AP**  
> **4G/5G上网、VPN远程维护、网络隔离 → 工业路由器**

## ② 参考表

| 对比项 | 无线AP | 工业路由器 |
|---|---|---|
| 无线接入 | 强 | 视型号 |
| 路由功能 | 较弱或无 | **强** |
| NAT | 通常无 | **常见** |
| VPN | 通常无 | **常见** |
| 4G/5G | 通常无 | **常见** |
| 典型用途 | Wi-Fi覆盖 | 远程维护、上云、网络隔离 |

## ③ 计算/选型示例

- 已知：  
  现场已有工业路由器联网，但车间无线覆盖不够。
- 公式：  
  **已有路由 + 需要无线覆盖 = 增加AP**
- 计算：  
  路由器负责上外网，AP负责扩展无线接入。
- 结论：  
  可在路由器或交换机后面接入无线AP。

> [!tip] 记忆口诀
> 路由器负责“出门”，AP负责“进无线”。

# 05 AP模式

> [!info] 核心结论
> AP模式是最常见模式，用于创建Wi-Fi热点，让无线终端接入有线网络。

## ① 分类/公式/规则

> [!warning] AP模式链路
> **交换机/路由器 → AP → Wi-Fi终端**

## ② 参考表

| 参数 | 说明 |
|---|---|
| **SSID** | Wi-Fi名称 |
| **密码** | 无线接入认证 |
| **信道** | 无线频率通道 |
| **发射功率** | 影响覆盖范围 |
| **加密方式** | WPA2/WPA3等 |
| **接入人数** | 同时连接终端数量 |
| **IP地址** | AP管理地址 |

## ③ 计算/选型示例

- 已知：  
  车间有20台手持终端需要连接MES。
- 公式：  
  **多无线终端接入 = AP模式**
- 计算：  
  AP接入工厂网络，手持终端通过Wi-Fi访问MES。
- 结论：  
  应选择支持足够并发数量和工业环境的无线AP。

> [!tip] 记忆口诀
> AP模式就是“发Wi-Fi”。

# 06 网桥模式

> [!info] 核心结论
> 网桥模式用于把两个有线网络通过无线方式连接起来，适合无法布线的固定点位。

## ① 分类/公式/规则

> [!warning] 网桥链路
> **交换机A → 网桥A )))) 无线链路 (((( 网桥B → 交换机B**

## ② 参考表

| 类型 | 说明 |
|---|---|
| **点对点网桥** | 两个固定点互联 |
| **点对多点网桥** | 一个中心点连接多个远端点 |
| **透明网桥** | 两端网络像同一条网线连接 |
| **定向天线网桥** | 距离远、方向性强 |
| **室外网桥** | 防水、防雷、耐温 |
| **工业网桥** | 适合设备现场和工业网络 |

## ③ 计算/选型示例

- 已知：  
  A车间与B车间相距200米，中间无法铺设网线。
- 公式：  
  **固定两点 + 无法布线 = 点对点网桥**
- 计算：  
  两端安装网桥并对准天线，即可建立无线链路。
- 结论：  
  可选择一对室外工业无线网桥。

> [!tip] 记忆口诀
> 网桥像一根看不见的网线。

# 07 客户端模式

> [!info] 核心结论
> 客户端模式让没有Wi-Fi功能的有线设备，通过无线方式接入现有AP网络。

## ① 分类/公式/规则

> [!warning] 客户端模式链路
> **PLC/工控机 → 客户端无线设备 → Wi-Fi AP → 网络**

## ② 参考表

| 应用 | 说明 |
|---|---|
| **PLC无线接入** | PLC只有网口，没有Wi-Fi |
| **工控机无线接入** | 避免拉网线 |
| **移动小车联网** | AGV、移动平台常见 |
| **远程IO接入** | 特殊场景使用 |
| **扫码设备联网** | 固定扫码器无线接入 |
| **临时调试** | 临时把有线设备接入无线网络 |

## ③ 计算/选型示例

- 已知：  
  一台PLC只有RJ45网口，但现场只能接Wi-Fi。
- 公式：  
  **有线设备接Wi-Fi → 客户端模式**
- 计算：  
  无线客户端设备连接AP，再通过网线接PLC。
- 结论：  
  可使用支持Client模式的工业无线设备。

> [!tip] 记忆口诀
> 设备没无线，就给它加一个无线客户端。

# 08 中继与Mesh

> [!info] 核心结论
> 中继和Mesh用于扩大无线覆盖范围，但每增加一级无线转发，都可能增加延迟并降低带宽。

## ① 分类/公式/规则

> [!warning] 覆盖规则
> **能拉网线回传优先有线回传；不能拉线再考虑无线中继/Mesh**

## ② 参考表

| 方式 | 特点 |
|---|---|
| **无线中继** | 简单扩展覆盖，但带宽会受影响 |
| **Mesh组网** | 多节点自动组网，部署灵活 |
| **有线回传AP** | 稳定性更好 |
| **无线回传AP** | 布线少，但受干扰影响 |
| **多AP覆盖** | 适合大面积车间 |
| **漫游切换** | 移动终端跨AP时保持连接 |

## ③ 计算/选型示例

- 已知：  
  仓库中间区域Wi-Fi信号弱，但可以拉网线到该区域。
- 公式：  
  **可拉网线 → 增加有线AP优先**
- 计算：  
  有线回传比无线中继更稳定，带宽损耗更小。
- 结论：  
  推荐新增一台有线接入AP，而不是简单加中继。

> [!tip] 记忆口诀
> 能有线回传就别无线中继，稳定性差很多。

# 09 频段：2.4GHz与5GHz

> [!info] 核心结论
> 2.4GHz覆盖更远、穿透较好但干扰多；5GHz速率更高、干扰相对少但覆盖距离较短。

## ① 分类/公式/规则

> [!warning] 频段选择规则
> **远距离、穿墙多 → 2.4GHz可评估**  
> **高速率、低干扰 → 5GHz优先**  
> **工业关键通讯 → 现场实测最重要**

## ② 参考表

| 频段 | 特点 |
|---|---|
| **2.4GHz** | 覆盖广，兼容性好，干扰多 |
| **5GHz** | 速率高，信道多，覆盖较短 |
| **双频AP** | 同时支持2.4G和5G |
| **6GHz** | 新标准场景，需终端支持 |
| **自动频段选择** | 方便但不一定最优 |
| **固定频段规划** | 工业现场更可控 |

## ③ 计算/选型示例

- 已知：  
  工业相机需要无线传输较大数据量。
- 公式：  
  **大带宽需求 → 5GHz或更高性能无线优先**
- 计算：  
  2.4GHz容易拥堵，实际吞吐可能不足。
- 结论：  
  可优先评估5GHz工业无线方案，但关键视觉数据仍建议优先有线。

> [!tip] 记忆口诀
> 2.4G跑得远，5G跑得快；现场干扰决定最终效果。

# 10 无线标准与速率

> [!info] 核心结论
> Wi-Fi标准影响理论速率、并发能力和稳定性，但现场实际速率通常低于标称速率。

## ① 分类/公式/规则

> [!warning] 实际速率规则
> **实际吞吐 < 理论速率**  
> **无线速率受距离、干扰、信号强度、终端能力影响**

## ② 参考表

| 标准 | 常见名称 |
|---|---|
| **802.11n** | Wi-Fi 4 |
| **802.11ac** | Wi-Fi 5 |
| **802.11ax** | Wi-Fi 6 |
| **802.11be** | Wi-Fi 7 |
| **MIMO** | 多天线提升吞吐和稳定性 |
| **带宽20/40/80MHz** | 信道宽度越大，速率越高但更易受干扰 |

## ③ 计算/选型示例

- 已知：  
  系统需要稳定传输100Mbps以上数据。
- 公式：  
  **实际吞吐需大于业务带宽**
- 计算：  
  不能只看AP标称速率，还要看现场实测吞吐。
- 结论：  
  应选择更高规格AP，并在现场做无线带宽测试。

> [!tip] 记忆口诀
> 标称速率是理想值，现场速率看实测。

# 11 覆盖范围与信号强度

> [!info] 核心结论
> 无线覆盖不是越远越好，关键是目标区域信号稳定、丢包低、延迟可接受。

## ① 分类/公式/规则

> [!warning] 覆盖质量公式
> **无线质量 = 信号强度 + 信噪比 + 丢包率 + 延迟 + 漫游稳定性**

## ② 参考表

| 影响因素 | 说明 |
|---|---|
| **距离** | 距离越远信号越弱 |
| **墙体/金属** | 会明显衰减信号 |
| **设备遮挡** | 机床、货架、钢结构影响大 |
| **天线高度** | 合理高度可改善覆盖 |
| **发射功率** | 过高也可能造成干扰 |
| **终端能力** | 终端天线差也会影响连接 |
| **环境干扰** | 其他Wi-Fi、蓝牙、变频器等 |

## ③ 计算/选型示例

- 已知：  
  AP显示信号满格，但通信仍然不稳定。
- 公式：  
  **信号强 ≠ 通信一定稳定**
- 计算：  
  可能存在干扰、丢包、信道拥塞或终端漫游问题。
- 结论：  
  应测试丢包率、延迟和信道占用，而不是只看信号格数。

> [!tip] 记忆口诀
> 无线不是看满格就够，要看丢不丢包、延迟稳不稳。

# 12 天线类型

> [!info] 核心结论
> 天线决定无线覆盖方向和距离，全向天线适合区域覆盖，定向天线适合远距离点对点。

## ① 分类/公式/规则

> [!warning] 天线选择规则
> **区域覆盖 → 全向天线**  
> **远距离网桥 → 定向天线**  
> **金属柜内设备 → 外置天线优先**

## ② 参考表

| 天线类型 | 适合场景 |
|---|---|
| **全向天线** | 周围区域覆盖 |
| **定向天线** | 两点远距离传输 |
| **面板天线** | 定向覆盖 |
| **吸盘天线** | 柜外引出常用 |
| **高增益天线** | 提高方向性和距离 |
| **MIMO天线** | 提高速率和稳定性 |

## ③ 计算/选型示例

- 已知：  
  AP装在金属控制柜内部，终端连接不稳定。
- 公式：  
  **金属柜屏蔽 → 外置天线**
- 计算：  
  金属柜会衰减无线信号。
- 结论：  
  应把天线引到柜外，或把AP安装在合适位置。

> [!tip] 记忆口诀
> 全向管一片，定向管一线；金属柜里别闷天线。

# 13 信道规划

> [!info] 核心结论
> 多个AP同时使用时，必须规划信道，避免同频干扰和相邻信道干扰。

## ① 分类/公式/规则

> [!warning] 信道规则
> **相邻AP尽量错开信道**  
> **2.4GHz常用1/6/11非重叠信道规划**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **信道冲突** | 多AP同信道互相影响 |
| **相邻信道干扰** | 频率重叠造成干扰 |
| **自动信道** | 方便但可能不稳定 |
| **固定信道** | 工业现场更可控 |
| **信道宽度** | 宽度越大，吞吐高但干扰风险增加 |
| **频谱扫描** | 用于查找干扰源 |

## ③ 计算/选型示例

- 已知：  
  车间有3个2.4GHz AP，覆盖区域相邻。
- 公式：  
  **相邻AP信道错开**
- 计算：  
  可分别设置为1、6、11信道。
- 结论：  
  合理信道规划可降低AP之间互相干扰。

> [!tip] 记忆口诀
> AP多了不是越强越好，信道不规划反而互相吵。

# 14 漫游切换

> [!info] 核心结论
> 移动终端在多个AP之间移动时，需要稳定漫游，否则会出现掉线、延迟增加或业务中断。

## ① 分类/公式/规则

> [!warning] 漫游规则
> **移动终端业务连续 = 覆盖重叠 + 同一SSID + 合理功率 + 漫游优化**

## ② 参考表

| 参数 | 作用 |
|---|---|
| **同一SSID** | 便于终端在AP之间切换 |
| **覆盖重叠** | 避免中间盲区 |
| **功率控制** | 避免终端粘连远端AP |
| **802.11k/v/r** | 辅助快速漫游 |
| **控制器管理** | 统一管理多个AP |
| **终端漫游能力** | 终端本身也很关键 |

## ③ 计算/选型示例

- 已知：  
  AGV经过不同区域时，偶尔通信中断。
- 公式：  
  **移动通信中断 = 漫游切换不稳定 / 覆盖盲区 / 干扰**
- 计算：  
  AGV可能一直连接远处AP，不及时切换到近处AP。
- 结论：  
  应优化AP位置、功率、信道和漫游参数。

> [!tip] 记忆口诀
> 固定设备看信号，移动设备看漫游。

# 15 安全加密

> [!info] 核心结论
> 工业无线网络必须做好加密、密码、权限和网络隔离，避免未授权设备接入控制网络。

## ① 分类/公式/规则

> [!warning] 安全规则
> **工业无线 = 强密码 + 加密认证 + 访问控制 + 网络隔离**

## ② 参考表

| 安全项 | 作用 |
|---|---|
| **WPA2/WPA3** | 无线加密 |
| **强密码** | 防止弱口令破解 |
| **隐藏SSID** | 只能降低暴露度，不能代替加密 |
| **MAC过滤** | 限制设备接入 |
| **VLAN隔离** | 控制网、访客网分开 |
| **访客网络** | 防止外来设备进入控制网 |
| **日志审计** | 记录接入行为 |

## ③ 计算/选型示例

- 已知：  
  现场维修人员手机也能连接设备Wi-Fi。
- 公式：  
  **无权限隔离 → 控制网风险增加**
- 计算：  
  非授权设备接入可能影响网络安全和稳定性。
- 结论：  
  应配置独立访客网络或限制访问PLC网络。

> [!tip] 记忆口诀
> 无线越方便，安全越要管住。

# 16 PoE供电

> [!info] 核心结论
> 许多无线AP和网桥支持PoE供电，可通过一根网线同时传输数据和供电，便于高处安装。

## ① 分类/公式/规则

> [!warning] PoE规则
> **PoE供电成功 = 交换机/电源支持PoE + AP功率匹配 + 网线质量合格**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **PoE** | 网线供电 |
| **PoE+** | 支持更高功率 |
| **PoE注入器** | 非PoE交换机也可给AP供电 |
| **供电距离** | 受网线距离和压降影响 |
| **室外AP** | PoE安装更方便 |
| **总功率预算** | 多AP时必须校核 |

## ③ 计算/选型示例

- 已知：  
  4台AP，每台功率15W。
- 公式：  
  **PoE总功率 = 4 × 15**
- 计算：  
  PoE总功率 = **60W**
- 结论：  
  PoE交换机或电源总功率应大于60W并留余量。

> [!tip] 记忆口诀
> PoE少拉电源线，但一定要算功率。

# 17 室外安装与防雷

> [!info] 核心结论
> 室外无线AP/网桥必须考虑防水、防晒、防雷、接地、风载、安装高度和天线对准。

## ① 分类/公式/规则

> [!warning] 室外安装规则
> **室外无线可靠 = 高防护设备 + 防雷接地 + 天线对准 + 线缆防水 + 稳固支架**

## ② 参考表

| 项目 | 注意 |
|---|---|
| **防护等级** | IP65/IP67常见 |
| **防雷器** | 室外网线和天线需考虑 |
| **接地** | 防雷和抗干扰关键 |
| **防水接头** | 防止雨水沿线进入 |
| **安装高度** | 避免遮挡，保证视距 |
| **天线对准** | 网桥稳定性关键 |
| **抗风结构** | 高处安装需稳固 |

## ③ 计算/选型示例

- 已知：  
  两栋楼之间网桥雨天后频繁掉线。
- 公式：  
  **雨后掉线 = 防水不良 / 接头进水 / 信号衰减**
- 计算：  
  网线接头、设备密封和天线罩都可能进水。
- 结论：  
  应检查防水接头、防雷接地和设备外壳密封。

> [!tip] 记忆口诀
> 室外无线，防水和防雷比速率更重要。

# 18 常见故障排查

> [!info] 核心结论
> 无线AP/网桥故障排查按 **供电、网线、SSID、密码、IP、信号、信道、干扰、天线、终端** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **无线异常 = 供电异常 + 配置错误 + 信号弱 + 信道干扰 + 天线问题 + 终端漫游异常 + 网络阻断**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **AP不亮** | 电源/PoE异常、设备损坏 |
| **搜不到Wi-Fi** | SSID关闭、频段不支持、AP未启动 |
| **连不上Wi-Fi** | 密码错、认证方式不匹配、黑名单 |
| **能连但不能上网** | IP、网关、DHCP、路由问题 |
| **通信时断时续** | 信号弱、干扰、漫游异常 |
| **网桥不通** | 两端未配对、天线未对准、距离过远 |
| **速度很慢** | 信道拥堵、信号差、带宽不足 |
| **移动设备掉线** | AP覆盖不连续、漫游参数不合理 |

## ③ 计算/选型示例

- 已知：  
  无线终端能连接AP，但无法访问PLC。
- 公式：  
  **Wi-Fi连接成功 ≠ PLC可达**
- 计算：  
  可能是IP网段不对、防火墙限制、VLAN隔离或PLC网关错误。
- 结论：  
  应检查终端IP、PLC IP、网关、VLAN和访问权限。

> [!tip] 记忆口诀
> 连上Wi-Fi只是第一步，能不能访问设备还要看网络配置。

# 19 无线AP / 网桥快速选型表

> [!info] 核心结论
> 无线AP/网桥选型要先判断是无线覆盖还是两点桥接，再确认距离、带宽、频段、天线、供电、安全、漫游和环境。

## ① 分类/公式/规则

> [!note] 十步选型法
> ① 明确用途：无线覆盖、点对点桥接、移动设备接入  
> ② 确认通讯距离和是否有遮挡  
> ③ 确认带宽需求和实时性要求  
> ④ 确认频段：2.4GHz、5GHz、双频  
> ⑤ 确认天线：全向、定向、外置、高增益  
> ⑥ 确认供电：DC电源、PoE、PoE+  
> ⑦ 确认安装环境：室内、室外、粉尘、潮湿、高温  
> ⑧ 确认安全：WPA2/WPA3、密码、VLAN、访问控制  
> ⑨ 确认是否需要漫游、Mesh、集中管理  
> ⑩ 现场测试信号强度、丢包率、延迟和吞吐量

> [!warning] 最终选型公式
> **推荐无线方案 = AP/网桥类型 + 频段 + 带宽 + 天线 + 供电 + 防护等级 + 安全策略 + 现场测试结果**

## ② 参考表

| 需求 | 推荐方向 |
|---|---|
| 手持终端联网 | 工业无线AP |
| AGV移动联网 | 多AP覆盖 + 漫游优化 |
| 两栋楼互联 | 点对点无线网桥 |
| 多远端设备接入 | 点对多点网桥 |
| 高速相机数据 | 优先有线，必要时5GHz高带宽无线 |
| 室外安装 | 高防护AP/网桥 + 防雷接地 |
| 金属柜内设备 | 外置天线 |
| 高安全要求 | WPA2/WPA3 + VLAN + 白名单 |

## ③ 计算/选型示例

- 已知：  
  仓库有20台手持终端、2台AGV，需要覆盖约80米长区域，并接入MES系统。
- 公式：  
  **需求 = 多终端接入 + 移动漫游 + 工业环境覆盖**
- 计算：  
  单个AP可能覆盖不足，AGV移动还需要漫游稳定。
- 结论：  
  推荐使用 **多台工业无线AP有线回传，统一SSID，合理信道规划，支持漫游优化和WPA2/WPA3安全加密**。

> [!tip] 选型口诀
> 终端接入用AP，两点互联用网桥；远距离看天线，移动设备看漫游；工业现场别只看速率，稳定才是第一。
