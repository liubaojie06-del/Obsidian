# 01 工业路由器总览

> [!info] 核心结论
> 工业路由器用于实现 **工业设备联网、不同网段互通、4G/5G远程通信、VPN远程维护、NAT地址转换、防火墙隔离和数据上云**。

## ① 分类/公式/规则

> [!warning] 选型公式
> **工业路由器选型 = 上网方式 + LAN/WAN端口 + VPN需求 + 4G/5G需求 + 防火墙/NAT + 协议支持 + 安装环境 + 远程运维平台**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **联网方式** | 有线宽带、4G、5G、Wi-Fi、双卡 |
| **核心功能** | 路由、NAT、防火墙、VPN、远程维护 |
| **连接对象** | PLC、HMI、工控机、摄像头、传感器、网关 |
| **远程运维** | 工程师远程访问PLC/HMI/工控机 |
| **数据上传** | 设备数据上传MES、SCADA、云平台 |
| **安全隔离** | 内外网隔离、端口访问控制 |
| **安装方式** | DIN导轨、壁挂、机柜安装 |

## ③ 计算/选型示例

- 已知：  
  设备在客户现场，工程师需要远程访问PLC调试程序。
- 公式：  
  **远程维护 + PLC联网 = 工业路由器 + VPN**
- 计算：  
  路由器连接PLC内网，并通过4G/5G或有线网络连接外网。
- 结论：  
  推荐使用 **支持VPN远程访问的工业路由器**。

> [!tip] 记忆口诀
> 交换机负责连设备，路由器负责连网络；要远程维护，就离不开路由器。

# 02 底层原理

> [!info] 核心结论
> 工业路由器的底层作用，是在不同网络之间转发数据包，并根据IP地址、路由表和安全规则决定数据能不能通过。

## ① 分类/公式/规则

> [!note] 工作逻辑
> 路由器连接两个或多个不同网段。  
> 它根据目标IP地址查找路由表，把数据转发到正确的出口，同时可进行NAT、防火墙过滤和VPN加密。

> [!warning] 数据链路
> **工业设备 → LAN口 → 路由判断 → NAT/防火墙/VPN → WAN/4G/5G → 远程平台**

## ② 参考表

| 环节 | 作用 |
|---|---|
| **LAN口** | 连接设备内网 |
| **WAN口** | 连接外部网络 |
| **路由表** | 决定数据从哪个出口走 |
| **NAT** | 内网地址转换成外网可访问地址 |
| **防火墙** | 控制哪些数据允许通过 |
| **VPN** | 加密远程访问通道 |
| **SIM卡模块** | 提供4G/5G无线联网 |

## ③ 计算/选型示例

- 已知：  
  PLC地址为192.168.1.10，远程电脑不在同一网段。
- 公式：  
  **不同网段通信 → 需要路由**
- 计算：  
  远程电脑不能像本地交换机一样直接访问PLC，必须通过路由器转发。
- 结论：  
  工业路由器用于解决不同网络之间的数据转发和远程访问问题。

> [!tip] 记忆口诀
> 同网段靠交换，不同网段靠路由。

# 03 工业路由器 vs 工业交换机

> [!info] 核心结论
> 工业交换机主要连接同一局域网设备，工业路由器主要连接不同网络，并实现远程、安全和上网功能。

## ① 分类/公式/规则

> [!warning] 区别规则
> **同一局域网内多设备互联 → 工业交换机**  
> **不同网段互通 / 上外网 / VPN远程 → 工业路由器**

## ② 参考表

| 对比项 | 工业交换机 | 工业路由器 |
|---|---|---|
| 主要作用 | 局域网设备互联 | 不同网络互联 |
| 工作对象 | MAC地址 | IP地址 |
| 是否上网 | 通常不负责 | **负责** |
| 是否NAT | 通常没有 | **常见** |
| 是否VPN | 通常没有 | **常见** |
| 是否防火墙 | 简单或无 | **较常见** |
| 典型场景 | PLC、HMI、IO组网 | 远程维护、设备上云 |

## ③ 计算/选型示例

- 已知：  
  PLC、HMI、远程IO在一个控制柜内互联。
- 公式：  
  **同网段设备互联 → 交换机**
- 计算：  
  如果只是让这些设备互相通信，不需要路由器。
- 结论：  
  用工业交换机即可；若要远程访问或上云，再加工业路由器。

> [!tip] 记忆口诀
> 交换机管“内部联系”，路由器管“对外连接”。

# 04 工业路由器 vs 工业网关

> [!info] 核心结论
> 工业路由器偏网络连接和远程访问，工业网关偏协议转换、数据采集和边缘计算。

## ① 分类/公式/规则

> [!warning] 选择规则
> **设备联网/VPN远程 → 工业路由器**  
> **协议转换/数据采集/上云 → 工业网关**

## ② 参考表

| 对比项 | 工业路由器 | 工业网关 |
|---|---|---|
| 核心功能 | 网络连接、路由、VPN | 协议转换、数据采集 |
| 常见协议 | TCP/IP、VPN、NAT | Modbus、OPC UA、MQTT等 |
| 数据处理 | 较弱或基础 | **更强** |
| 远程访问 | **强** | 视型号 |
| 上云能力 | 部分支持 | **常见** |
| 典型用途 | 远程维护PLC | 采集PLC数据上传平台 |

## ③ 计算/选型示例

- 已知：  
  现场需要把PLC数据采集后通过MQTT上传云平台。
- 公式：  
  **协议采集 + MQTT上云 → 工业网关优先**
- 计算：  
  普通工业路由器只负责联网，不一定能解析PLC协议。
- 结论：  
  可选择工业网关，或选择带网关功能的工业路由器。

> [!tip] 记忆口诀
> 路由器解决“怎么连上”，网关解决“数据怎么读、怎么传”。

# 05 LAN口与WAN口

> [!info] 核心结论
> 工业路由器通常有LAN口和WAN口，LAN连接设备内网，WAN连接外部网络或上级网络。

## ① 分类/公式/规则

> [!warning] 接口规则
> **LAN = 设备侧内网**  
> **WAN = 外部网络侧**

## ② 参考表

| 接口 | 作用 |
|---|---|
| **LAN口** | 连接PLC、HMI、交换机、工控机 |
| **WAN口** | 连接工厂网络、宽带、上级路由器 |
| **4G/5G口** | 通过SIM卡联网 |
| **Wi-Fi口** | 无线接入或热点 |
| **串口** | 部分型号可接仪表或PLC |
| **DI/DO** | 远程报警或状态输入输出 |
| **天线接口** | 连接蜂窝网络或Wi-Fi天线 |

## ③ 计算/选型示例

- 已知：  
  设备内部PLC网段为192.168.10.x，工厂外网为10.10.1.x。
- 公式：  
  **LAN和WAN应分属不同网段**
- 计算：  
  LAN可设置192.168.10.1，WAN连接10.10.1.x网络。
- 结论：  
  LAN/WAN网段不要混乱，否则容易导致路由异常。

> [!tip] 记忆口诀
> LAN接设备，WAN接外网；两个网段别乱套。

# 06 4G/5G工业路由器

> [!info] 核心结论
> 4G/5G工业路由器通过SIM卡联网，适合无有线网络、移动设备、偏远站点和远程维护场景。

## ① 分类/公式/规则

> [!warning] 无线选型公式
> **无线可用 = 运营商信号 + SIM卡流量 + 天线位置 + 网络制式 + VPN平台**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **4G路由器** | 成本适中，远程维护常用 |
| **5G路由器** | 速度高，适合大数据和视频 |
| **单卡** | 基础联网 |
| **双卡** | 提高网络可靠性 |
| **外置天线** | 改善信号质量 |
| **APN设置** | 专网卡或运营商要求 |
| **流量套餐** | 远程视频和图片上传需重点核算 |

## ③ 计算/选型示例

- 已知：  
  现场没有网线，但PLC需要远程维护。
- 公式：  
  **无有线网络 + 远程维护 → 4G/5G工业路由器**
- 计算：  
  路由器通过SIM卡联网，工程师通过VPN访问设备内网。
- 结论：  
  可选4G工业路由器；若需要视频或大量数据，可评估5G。

> [!tip] 记忆口诀
> 没网线，用蜂窝；普通远程4G够，大流量低延迟看5G。

# 07 VPN远程维护

> [!info] 核心结论
> VPN可建立加密通道，让工程师像在现场一样访问PLC、HMI、工控机和摄像头。

## ① 分类/公式/规则

> [!warning] VPN使用公式
> **远程访问成功 = 路由器在线 + VPN连接成功 + 目标IP可达 + 权限允许**

## ② 参考表

| VPN类型 | 说明 |
|---|---|
| **OpenVPN** | 常见远程访问方案 |
| **IPsec VPN** | 企业网络常见 |
| **L2TP/PPTP** | 部分老系统使用 |
| **SSL VPN** | 浏览器或客户端访问 |
| **厂家云VPN** | 通过厂家平台远程维护 |
| **点对点VPN** | 工程师电脑直连设备网络 |
| **站点到站点VPN** | 两个工厂网络互联 |

## ③ 计算/选型示例

- 已知：  
  工程师需要远程下载PLC程序。
- 公式：  
  **远程PLC调试 → VPN通道 + PLC软件访问**
- 计算：  
  VPN建立后，工程师电脑才能访问PLC所在网段。
- 结论：  
  工业路由器应支持稳定VPN，并设置访问权限。

> [!tip] 记忆口诀
> VPN就是给远程工程师开一条加密专线。

# 08 NAT地址转换

> [!info] 核心结论
> NAT用于把内网地址转换成外网可访问地址，也可隐藏设备内网，提高网络隔离性。

## ① 分类/公式/规则

> [!warning] NAT规则
> **内网设备访问外网 → 源地址NAT**  
> **外网访问内网设备 → 端口映射或VPN**

## ② 参考表

| NAT类型 | 用途 |
|---|---|
| **源NAT** | 内网设备访问外网 |
| **目的NAT** | 外部访问内部指定设备 |
| **端口映射** | 把外部端口转到内部设备 |
| **一对一NAT** | 内外地址固定映射 |
| **NAT隔离** | 隐藏内网真实结构 |
| **多设备冲突处理** | 解决相同PLC网段冲突 |

## ③ 计算/选型示例

- 已知：  
  远程需要访问PLC的192.168.1.10，但现场内网不允许直接暴露。
- 公式：  
  **安全远程访问 → VPN优先，端口映射谨慎使用**
- 计算：  
  端口映射直接暴露服务，安全风险更高。
- 结论：  
  推荐通过VPN访问PLC，减少公网暴露风险。

> [!tip] 记忆口诀
> NAT能转地址，VPN更适合安全远程维护。

# 09 防火墙与访问控制

> [!info] 核心结论
> 工业路由器防火墙用于限制哪些设备、端口和服务可以访问，减少外部攻击和误访问风险。

## ① 分类/公式/规则

> [!warning] 安全规则
> **默认拒绝不必要访问，只开放必须端口和目标设备**

## ② 参考表

| 功能 | 作用 |
|---|---|
| **IP过滤** | 限制允许访问的IP |
| **端口过滤** | 限制允许访问的服务端口 |
| **MAC过滤** | 限制指定设备接入 |
| **访问白名单** | 只允许授权访问 |
| **黑名单** | 禁止指定来源 |
| **防扫描** | 减少异常访问 |
| **日志记录** | 追踪访问行为 |

## ③ 计算/选型示例

- 已知：  
  只允许远程工程师访问PLC，不允许访问工控机。
- 公式：  
  **最小权限访问 = 只放行PLC IP**
- 计算：  
  防火墙规则可限制远程VPN客户端只能访问PLC地址。
- 结论：  
  应配置访问白名单，不要把整个设备网络完全开放。

> [!tip] 记忆口诀
> 能少开就少开，能白名单就别全放通。

# 10 双网口与多网段隔离

> [!info] 核心结论
> 工业路由器常用于隔离设备内网和工厂外网，避免控制网络直接暴露到办公网络或互联网。

## ① 分类/公式/规则

> [!warning] 网络隔离规则
> **控制网、办公网、外网尽量分层隔离，不要随意混接**

## ② 参考表

| 网络 | 说明 |
|---|---|
| **设备控制网** | PLC、HMI、远程IO、伺服 |
| **设备数据网** | 工控机、视觉、数据库 |
| **工厂网络** | MES、SCADA、服务器 |
| **办公网络** | 办公网电脑 |
| **互联网** | 远程访问和云平台 |
| **隔离区DMZ** | 工业安全中常见中间区域 |

## ③ 计算/选型示例

- 已知：  
  PLC网络和办公网络直接接在同一个交换机上。
- 公式：  
  **控制网直接暴露 → 安全风险增加**
- 计算：  
  办公网病毒、广播流量或误操作可能影响PLC网络。
- 结论：  
  应通过工业路由器、防火墙或网关隔离控制网和办公网。

> [!tip] 记忆口诀
> PLC控制网不是办公网，能隔离就隔离。

# 11 串口转网络

> [!info] 核心结论
> 部分工业路由器带RS232/RS485接口，可实现串口设备联网、远程透传或Modbus RTU转TCP。

## ① 分类/公式/规则

> [!warning] 串口通讯公式
> **串口通讯成功 = 波特率 + 数据位 + 停止位 + 校验位 + 地址一致**

## ② 参考表

| 功能 | 说明 |
|---|---|
| **RS232透传** | 单设备串口远程访问 |
| **RS485透传** | 多站串口设备联网 |
| **Modbus RTU转TCP** | 串口仪表接入以太网 |
| **透明传输** | 数据原样转发 |
| **协议转换** | 视路由器型号支持 |
| **远程参数配置** | 远程访问串口设备 |

## ③ 计算/选型示例

- 已知：  
  现场有一台RS485电表，需要远程读取数据。
- 公式：  
  **串口设备联网 → 串口转网络功能**
- 计算：  
  路由器通过RS485读取或透传电表数据，再通过网络传到上位系统。
- 结论：  
  可选带RS485接口和Modbus转换功能的工业路由器。

> [!tip] 记忆口诀
> 老设备走串口，新系统走网络；中间靠串口联网功能。

# 12 数据上云

> [!info] 核心结论
> 工业路由器可为设备提供上云通道，部分型号还支持MQTT、边缘采集或与云平台直连。

## ① 分类/公式/规则

> [!warning] 上云链路
> **设备数据 → PLC/网关采集 → 路由器联网 → 云平台/MES/SCADA**

## ② 参考表

| 上云方式 | 说明 |
|---|---|
| **透明联网** | 路由器只提供网络 |
| **MQTT上传** | 部分路由器支持 |
| **HTTP/API上传** | 视软件功能 |
| **OPC UA转发** | 部分网关型路由器支持 |
| **云平台VPN** | 远程维护和数据访问 |
| **边缘缓存** | 网络断开时暂存数据 |

## ③ 计算/选型示例

- 已知：  
  设备只需要联网，数据采集由PLC和上位机完成。
- 公式：  
  **只需要网络 → 普通工业路由器即可**
- 计算：  
  路由器不必解析PLC协议，只负责稳定上网。
- 结论：  
  若需要协议采集和MQTT上云，则应选网关型工业路由器。

> [!tip] 记忆口诀
> 只联网用路由器，要采数据看网关能力。

# 13 无线Wi-Fi功能

> [!info] 核心结论
> 带Wi-Fi的工业路由器可作为无线接入点、客户端或无线桥接，适合移动设备、调试终端和无线传感器接入。

## ① 分类/公式/规则

> [!warning] Wi-Fi选型公式
> **Wi-Fi可用 = 覆盖范围 + 频段 + 天线 + 干扰环境 + 安全加密**

## ② 参考表

| 模式 | 说明 |
|---|---|
| **AP模式** | 路由器作为无线热点 |
| **Client模式** | 路由器连接上级Wi-Fi |
| **桥接模式** | 两个网络无线互联 |
| **2.4GHz** | 覆盖广，干扰多 |
| **5GHz** | 速率高，覆盖稍弱 |
| **WPA/WPA2/WPA3** | 无线加密方式 |

## ③ 计算/选型示例

- 已知：  
  维修人员希望在设备旁用笔记本无线连接PLC。
- 公式：  
  **本地无线调试 → 路由器AP模式**
- 计算：  
  笔记本连接路由器Wi-Fi，再访问PLC内网。
- 结论：  
  可启用工业路由器Wi-Fi热点，但要设置密码和访问权限。

> [!tip] 记忆口诀
> Wi-Fi方便调试，但安全密码和权限不能省。

# 14 SIM卡与流量

> [!info] 核心结论
> 蜂窝工业路由器使用SIM卡联网，必须确认运营商制式、信号强度、流量套餐和APN配置。

## ① 分类/公式/规则

> [!warning] 流量估算公式
> **月流量 = 单次数据量 × 次数 × 天数 + 远程维护流量**

## ② 参考表

| 项目 | 注意 |
|---|---|
| **运营商制式** | 路由器频段需匹配 |
| **SIM卡类型** | 普通卡、物联网卡、专网卡 |
| **APN** | 专网卡常需配置 |
| **信号强度** | 影响稳定性和速度 |
| **流量套餐** | 图片、视频、远程桌面耗流量大 |
| **双卡切换** | 提高可靠性 |
| **外置天线** | 弱信号现场很重要 |

## ③ 计算/选型示例

- 已知：  
  设备每天上传100MB数据，远程维护每月约2GB。
- 公式：  
  **月流量 = 100MB × 30 + 2GB**
- 计算：  
  100MB × 30 = 3000MB ≈ 3GB  
  月流量 ≈ **5GB**
- 结论：  
  SIM卡套餐应大于5GB，并留余量。

> [!tip] 记忆口诀
> 远程调试不费太多，视频图片最费流量。

# 15 供电与安装

> [!info] 核心结论
> 工业路由器通常安装在控制柜内，常用DC12V/DC24V供电，需关注宽压输入、接地、天线布置和散热。

## ① 分类/公式/规则

> [!warning] 稳定安装公式
> **稳定运行 = 稳定供电 + 良好接地 + 合理天线 + 强弱电分离 + 散热空间**

## ② 参考表

| 项目 | 注意 |
|---|---|
| **DC24V供电** | 工业现场常见 |
| **宽压输入** | 适应电源波动 |
| **DIN导轨安装** | 控制柜常用 |
| **天线位置** | 避免被金属柜体屏蔽 |
| **接地端子** | 提高抗干扰能力 |
| **工作温度** | 高温柜体需关注 |
| **状态指示灯** | 网络、电源、SIM、VPN状态判断 |

## ③ 计算/选型示例

- 已知：  
  4G路由器装在金属控制柜内部，信号很弱。
- 公式：  
  **金属柜体屏蔽 → 外置天线**
- 计算：  
  蜂窝信号被柜体遮挡，导致掉线或网速慢。
- 结论：  
  应把天线引到柜外，或使用外置高增益天线。

> [!tip] 记忆口诀
> 路由器可以装柜内，天线最好别闷在金属柜里。

# 16 远程运维安全

> [!info] 核心结论
> 工业路由器远程维护很方便，但必须做好账号权限、VPN认证、访问控制、日志和关闭不必要端口。

## ① 分类/公式/规则

> [!warning] 安全公式
> **安全远程维护 = VPN加密 + 强密码 + 权限分级 + 白名单 + 日志审计**

## ② 参考表

| 安全项 | 作用 |
|---|---|
| **强密码** | 防止弱口令入侵 |
| **VPN证书** | 提高认证安全性 |
| **访问白名单** | 限制允许访问对象 |
| **关闭公网端口** | 减少暴露面 |
| **日志记录** | 追踪谁何时访问 |
| **权限分级** | 操作员、工程师、管理员区分 |
| **固件更新** | 修复安全漏洞 |

## ③ 计算/选型示例

- 已知：  
  路由器直接把PLC编程端口映射到公网。
- 公式：  
  **公网暴露工业端口 → 高风险**
- 计算：  
  外部人员可能扫描到端口并尝试访问PLC。
- 结论：  
  不建议直接公网映射PLC端口，应使用VPN和访问白名单。

> [!tip] 记忆口诀
> 远程不是越开放越好，越关键越要收紧权限。

# 17 常见故障排查

> [!info] 核心结论
> 工业路由器故障排查按 **电源、SIM/网线、信号、IP地址、路由、VPN、防火墙、平台状态** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **路由器异常 = 供电异常 + 网络未连接 + IP冲突 + 路由错误 + VPN未建立 + 防火墙阻断 + SIM卡异常**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **路由器不亮** | 电源无电、接线错误、设备故障 |
| **4G/5G不上线** | SIM卡、APN、信号弱、欠费 |
| **VPN连不上** | 平台离线、账号错误、网络阻断 |
| **能上网但访问不了PLC** | LAN地址错误、路由缺失、防火墙限制 |
| **远程很慢** | 信号弱、流量限速、带宽不足 |
| **频繁掉线** | 天线位置差、电源波动、运营商网络不稳 |
| **IP冲突** | LAN/WAN网段重复 |
| **端口映射无效** | 防火墙、NAT规则或公网IP问题 |

## ③ 计算/选型示例

- 已知：  
  VPN显示已连接，但无法Ping到PLC。
- 公式：  
  **VPN连上 ≠ PLC可达**
- 计算：  
  可能是PLC网关没设、路由器LAN地址不对、防火墙没放行或PLC不在线。
- 结论：  
  应逐步检查PLC IP、网关、路由器LAN口、防火墙规则和网线连接。

> [!tip] 记忆口诀
> VPN连上只是进了门，能不能到PLC还要看路由和权限。

# 18 工业路由器快速选型表

> [!info] 核心结论
> 工业路由器选型要先明确是远程维护、设备上云还是网络隔离，再确认联网方式、VPN、安全、端口和环境。

## ① 分类/公式/规则

> [!note] 十步选型法
> ① 明确用途：远程维护、上云、隔离、无线联网  
> ② 确认联网方式：WAN、4G、5G、Wi-Fi  
> ③ 确认LAN口数量和是否需要接交换机  
> ④ 确认是否需要VPN远程访问  
> ⑤ 确认是否需要NAT、端口映射和防火墙  
> ⑥ 确认是否需要双卡、双WAN或链路备份  
> ⑦ 确认是否需要串口、DI/DO、协议转换  
> ⑧ 确认是否需要云平台、MQTT或边缘采集  
> ⑨ 确认供电、安装、温度、天线和防护  
> ⑩ 确认账号权限、日志、固件更新和运维平台

> [!warning] 最终选型公式
> **推荐工业路由器 = WAN/4G/5G + LAN端口 + VPN + 防火墙/NAT + 远程平台 + 工业环境等级**

## ② 参考表

| 需求 | 推荐关注点 |
|---|---|
| PLC远程维护 | VPN、远程平台、LAN口 |
| 偏远设备联网 | 4G/5G、外置天线、双卡 |
| 设备数据上云 | MQTT/网关功能或配合工业网关 |
| 网络隔离 | 防火墙、NAT、多网段 |
| 多设备接入 | LAN口数量或外接交换机 |
| 高可靠联网 | 双卡、双WAN、链路切换 |
| 串口设备联网 | RS232/RS485透传 |
| 高安全要求 | VPN证书、白名单、日志审计 |

## ③ 计算/选型示例

- 已知：  
  设备现场没有有线网络，需要远程维护PLC和HMI，并上传少量运行数据。
- 公式：  
  **需求 = 4G/5G联网 + VPN远程 + LAN连接PLC/HMI + 数据上传**
- 计算：  
  至少需要蜂窝网络、VPN功能、2个以上LAN口或外接交换机。
- 结论：  
  推荐选择 **4G/5G工业路由器，支持VPN远程维护、NAT防火墙、外置天线、DC24V供电和工业导轨安装**。

> [!tip] 选型口诀
> 远程维护看VPN，没网线看4G/5G；要安全看防火墙，要上云看网关功能；信号不好先看天线。
