# 01 总线编码器总览

> [!info] 核心结论
> 总线编码器通过工业通讯总线直接输出 **位置值、速度值、状态信息和诊断数据**，比普通脉冲编码器更适合数字化控制系统。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> 普通增量编码器输出A/B/Z脉冲；  
> 总线编码器直接通过总线通讯，把当前位置以数字数据传给PLC、运动控制器或上位机。

> [!warning] 🟧 通用选型公式
> **总线编码器选型 = 通讯协议 + 单圈/多圈 + 分辨率 + 轴型结构 + 电源电压 + 防护等级**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **类型** | 总线型编码器 |
| **常见功能** | 位置、速度、方向、状态、诊断 |
| **常见协议** | PROFINET、EtherCAT、CANopen、PROFIBUS、Modbus等 |
| **常见结构** | 实心轴、空心轴、盲孔轴 |
| **常见分类** | 单圈绝对值、多圈绝对值 |
| **重点参数** | 分辨率、通讯周期、地址、设备文件 |

## ③ 计算/选型示例

- 已知：  
  设备需要PLC直接读取旋转轴的绝对位置，并希望断电后位置不丢失。
- 公式：  
  **编码器需求 = 绝对位置 + 总线通讯 + 掉电保持**
- 计算：  
  增量编码器断电后通常需要回零；总线绝对值编码器可直接提供当前位置。
- 结论：  
  可选择 **总线型绝对值编码器**，通讯协议需与PLC系统匹配。

> [!tip] 记忆口诀
> 增量数脉冲，总线传数据；要绝对位置，总线更省事。

# 02 底层原理

> [!info] 核心结论
> 总线编码器的本质，是把机械角度转换成数字位置值，再通过工业总线发送给控制系统。

## ① 分类/公式/规则

> [!note] 🟦 工作流程
> 编码器内部检测轴的角度位置，经过电子电路转换成数字量，再按通讯协议打包发送到PLC或控制器。

> [!warning] 🟧 总线编码逻辑
> **轴旋转 → 位置检测 → 数字编码 → 通讯协议打包 → PLC读取位置值**

## ② 参考表

| 部分 | 作用 |
|---|---|
| **检测单元** | 获取轴的位置 |
| **编码电路** | 将位置转换为数字值 |
| **通讯接口** | 与PLC或控制器通信 |
| **设备文件** | 让PLC识别设备参数 |
| **诊断功能** | 提供故障、状态、报警信息 |

## ③ 计算/选型示例

- 已知：  
  编码器单圈分辨率为 **4096**。
- 公式：  
  **每计数角度 = 360° ÷ 分辨率**
- 计算：  
  每计数角度 = 360° ÷ 4096 ≈ **0.088°**
- 结论：  
  该编码器每增加1个位置值，轴大约转过 **0.088°**。

> [!tip] 记忆口诀
> 先把角度变数字，再把数字走总线。

# 03 总线编码器 vs 增量编码器

> [!info] 核心结论
> 增量编码器适合高速计数和相对位置，总线编码器适合绝对位置、诊断和网络化控制。

## ① 分类/公式/规则

> [!note] 🟦 核心区别
> 增量编码器输出的是脉冲，需要PLC自己计数；  
> 总线编码器输出的是数字位置，PLC直接读取数据。

> [!warning] 🟧 区分公式
> **增量编码器 = 脉冲 + 计数 + 回零**  
> **总线编码器 = 位置值 + 通讯 + 诊断**

## ② 参考表

| 对比项 | 增量编码器 | 总线编码器 |
|---|---|---|
| 输出信号 | A/B/Z脉冲 | **数字通讯数据** |
| 断电位置 | 通常丢失 | 多数可保持 |
| 接线 | 脉冲线较多 | 总线线缆 |
| PLC要求 | 高速计数输入 | 通讯接口 |
| 诊断能力 | 较少 | **较强** |
| 适合场景 | 测速、测长 | 绝对定位、联网控制 |

## ③ 计算/选型示例

- 已知：  
  设备断电后重新上电，要求立刻知道当前位置。
- 公式：  
  **上电可用性 = 是否保留绝对位置**
- 计算：  
  增量编码器一般需要回零；总线绝对值编码器可直接读取当前位置。
- 结论：  
  优先选择 **总线绝对值编码器**。

> [!tip] 记忆口诀
> 增量要回零，总线读位置；想省回零，用绝对值。

# 04 常见通讯协议

> [!info] 核心结论
> 总线编码器选型第一步，是确认它的通讯协议是否与PLC或控制器兼容。

## ① 分类/公式/规则

> [!note] 🟦 协议优先原则
> 编码器再好，协议不匹配也无法直接通讯。  
> 例如西门子系统常见PROFINET，运动控制常见EtherCAT，通用现场总线可能使用CANopen或PROFIBUS。

> [!warning] 🟧 协议匹配公式
> **通讯成功 = PLC支持协议 + 编码器协议一致 + 设备文件正确 + 地址参数正确**

## ② 参考表

| 协议 | 常见场景 |
|---|---|
| **PROFINET** | 西门子PLC、工业以太网 |
| **EtherCAT** | 高速运动控制 |
| **CANopen** | 设备网络、移动机械 |
| **PROFIBUS** | 老项目、传统现场总线 |
| **Modbus RTU/TCP** | 通用仪表和控制系统 |
| **SSI** | 点对点绝对值位置传输 |

## ③ 计算/选型示例

- 已知：  
  项目PLC为西门子S7-1500，现场网络为PROFINET。
- 公式：  
  **推荐协议 = 控制器主协议**
- 计算：  
  S7-1500项目中PROFINET集成更方便。
- 结论：  
  优先选择 **PROFINET总线编码器**，并导入对应GSDML设备文件。

> [!tip] 记忆口诀
> 总线先看协议，协议不对全白费。

# 05 单圈与多圈

> [!info] 核心结论
> 单圈编码器只记录一圈内的位置，多圈编码器还能记录转过多少圈。

## ① 分类/公式/规则

> [!note] 🟦 单圈/多圈区别
> 单圈：0～360°内的位置。  
> 多圈：除了0～360°内的位置，还记录旋转圈数。

> [!warning] 🟧 位置范围公式
> **总位置数 = 单圈分辨率 × 多圈圈数**

## ② 参考表

| 类型 | 适合场景 |
|---|---|
| **单圈编码器** | 旋转角度不超过一圈 |
| **多圈编码器** | 升降、卷绕、丝杆、长行程 |
| **机械多圈** | 通过齿轮机构记圈 |
| **电子多圈** | 通过电子方式记圈 |
| **绝对值输出** | 上电可读当前位置 |

## ③ 计算/选型示例

- 已知：  
  编码器单圈分辨率为4096，多圈圈数为4096圈。
- 公式：  
  **总位置数 = 4096 × 4096**
- 计算：  
  总位置数 = **16,777,216**
- 结论：  
  该多圈编码器可表示非常大的位置范围，适合长行程位置检测。

> [!tip] 记忆口诀
> 一圈以内用单圈，超过一圈用多圈。

# 06 分辨率与精度

> [!info] 核心结论
> 分辨率表示位置划分得有多细，精度表示测得准不准，两者不是完全相同的概念。

## ① 分类/公式/规则

> [!note] 🟦 关键区别
> 分辨率高，代表能分得更细；  
> 精度高，代表实际位置误差更小。  
> 选型时不能只看位数，还要看实际精度、重复精度和机械安装误差。

> [!warning] 🟧 分辨率公式
> **单圈分辨率 = 2ⁿ**  
> n：编码器单圈位数

## ② 参考表

| 位数 | 单圈分辨率 |
|---|---|
| **10位** | 1024 |
| **12位** | 4096 |
| **13位** | 8192 |
| **14位** | 16384 |
| **16位** | 65536 |

## ③ 计算/选型示例

- 已知：  
  编码器为 **12位单圈**。
- 公式：  
  **单圈分辨率 = 2¹²**
- 计算：  
  2¹² = **4096**
- 结论：  
  12位单圈编码器一圈可分成 **4096份**。

> [!tip] 记忆口诀
> 位数决定细分，精度还看误差。

# 07 地址与设备文件

> [!info] 核心结论
> 总线编码器接入PLC前，通常需要设置地址、设备名或节点号，并导入设备文件。

## ① 分类/公式/规则

> [!note] 🟦 工程配置重点
> 不同总线有不同配置方式：  
> PROFINET常看设备名和IP；  
> CANopen常看Node ID和波特率；  
> EtherCAT常看从站配置；  
> PROFIBUS常看站地址。

> [!warning] 🟧 组态成功公式
> **组态成功 = 设备文件正确 + 地址唯一 + 参数匹配 + 通讯状态正常**

## ② 参考表

| 项目 | 作用 |
|---|---|
| **GSD/GSDML** | PROFIBUS/PROFINET设备描述 |
| **ESI文件** | EtherCAT设备描述 |
| **EDS文件** | CANopen设备描述 |
| **IP地址** | 工业以太网定位 |
| **设备名** | PROFINET设备识别 |
| **节点号** | CANopen/现场总线设备地址 |

## ③ 计算/选型示例

- 已知：  
  PROFINET编码器已经接线，但PLC诊断显示设备名不匹配。
- 公式：  
  **PROFINET识别 = 设备名一致 + IP正确 + 网络连通**
- 计算：  
  设备名不一致时，PLC可能无法把实际设备和项目组态对应起来。
- 结论：  
  需要重新分配PROFINET设备名，并下载硬件组态。

> [!tip] 记忆口诀
> 总线设备要上线，地址名称先配全。

# 08 输出数据与PLC读取

> [!info] 核心结论
> 总线编码器通常会向PLC发送位置值、速度值、状态字和诊断字，PLC程序需要按数据格式正确解析。

## ① 分类/公式/规则

> [!note] 🟦 数据类型
> 常见数据包括当前位置、实际速度、状态位、报警位、方向位、零点偏移、预置值等。

> [!warning] 🟧 PLC读取公式
> **实际工程值 = 原始位置值 × 比例系数 + 零点偏移**

## ② 参考表

| 数据 | 说明 |
|---|---|
| **位置值** | 当前绝对位置 |
| **速度值** | 当前旋转速度 |
| **状态字** | 运行、错误、方向等状态 |
| **诊断字** | 通讯、硬件、参数错误 |
| **预置值** | 设置当前位置为指定值 |
| **零点偏移** | 调整机械零点 |

## ③ 计算/选型示例

- 已知：  
  单圈分辨率为4096，PLC读取当前位置值为1024。
- 公式：  
  **角度 = 位置值 ÷ 分辨率 × 360°**
- 计算：  
  角度 = 1024 ÷ 4096 × 360° = **90°**
- 结论：  
  当前编码器位置约为 **90°**。

> [!tip] 记忆口诀
> 读到的是原始数，换算后才是工程量。

# 09 预置与零点

> [!info] 核心结论
> 总线编码器通常支持预置值功能，可把当前机械位置设为指定零点或基准值。

## ① 分类/公式/规则

> [!note] 🟦 预置功能
> 预置不是改变机械位置，而是改变编码器输出的位置参考。  
> 常用于设备调试、原点校准、换型调整和机械基准建立。

> [!warning] 🟧 预置公式
> **显示位置 = 原始位置 + 零点偏移**

## ② 参考表

| 功能 | 作用 |
|---|---|
| **预置值** | 将当前位置设为指定数值 |
| **零点偏移** | 修正机械零点 |
| **回零** | 建立机械基准 |
| **参数保存** | 防止断电丢失设置 |
| **权限控制** | 防止误改零点 |

## ③ 计算/选型示例

- 已知：  
  当前机械位置为实际零点，但编码器显示为2350。
- 公式：  
  **零点校准 = 当前实际零点 → 输出设为0**
- 计算：  
  通过预置功能将当前位置设置为0。
- 结论：  
  后续PLC读取的位置值就以该机械位置作为零点。

> [!tip] 记忆口诀
> 机械不动，数值可归零；预置用得好，调试少烦恼。

# 10 安装与抗干扰

> [!info] 核心结论
> 总线编码器不仅要机械安装可靠，还要保证通讯线缆、屏蔽接地、终端电阻和网络拓扑正确。

## ① 分类/公式/规则

> [!note] 🟦 安装重点
> 机械侧要防止联轴器偏心、轴向冲击和振动；  
> 通讯侧要注意屏蔽、接地、线缆长度、节点地址和网络终端。

> [!warning] 🟧 可靠运行公式
> **可靠运行 = 机械同轴 + 供电稳定 + 屏蔽接地 + 通讯参数正确 + 网络无冲突**

## ② 参考表

| 检查项 | 建议 |
|---|---|
| **联轴器** | 避免偏心和硬连接冲击 |
| **轴承负载** | 不要超过编码器允许值 |
| **供电电压** | 与型号一致 |
| **通讯线缆** | 使用对应总线专用线 |
| **屏蔽接地** | 按规范处理 |
| **地址冲突** | 每个设备地址唯一 |

## ③ 计算/选型示例

- 已知：  
  编码器偶发掉线，现场线缆与变频器动力线同槽敷设。
- 公式：  
  **通讯干扰风险 = 强电干扰 + 屏蔽不良 + 线缆距离过近**
- 计算：  
  变频器动力线可能对通讯线产生干扰，导致掉线或数据异常。
- 结论：  
  应将编码器通讯线远离动力线，并检查屏蔽接地和总线终端。

> [!tip] 记忆口诀
> 机械要同轴，通讯要屏蔽；地址不冲突，网络才稳定。

# 11 常见故障排查

> [!info] 核心结论
> 总线编码器故障排查，优先检查协议、设备文件、地址、供电、通讯线缆和PLC参数映射。

## ① 分类/公式/规则

> [!note] 🟦 排查顺序
> 先查是否在线，再查地址和协议；  
> 先查设备文件，再查数据映射；  
> 先查通讯状态，再查机械连接。

> [!warning] 🟧 排错公式
> **总线编码器异常 = 协议不匹配 + 地址冲突 + 设备文件错误 + 参数错误 + 通讯干扰 + 供电异常**

## ② 参考表

| 问题 | 可能原因 |
|---|---|
| **PLC找不到设备** | 协议不对、地址错误、设备名错误 |
| **设备在线但无数据** | 数据映射或参数配置错误 |
| **位置值跳变** | 通讯干扰、机械松动、参数异常 |
| **方向相反** | 安装方向或方向参数设置反 |
| **掉线报警** | 线缆、屏蔽、终端、电源问题 |
| **预置失败** | 权限、参数保存或通讯命令错误 |

## ③ 计算/选型示例

- 已知：  
  PLC可以扫描到编码器，但读取位置一直为0。
- 公式：  
  **数据读取正常 = 设备在线 + 输入地址正确 + 数据格式正确**
- 计算：  
  设备在线说明通讯基本存在，但程序可能读取了错误地址或解析格式不对。
- 结论：  
  应检查PLC硬件组态中的输入地址、数据长度、字节顺序和程序变量映射。

> [!tip] 记忆口诀
> 找不到查网络，读不对查映射，跳不稳查干扰。

# 12 总线编码器快速选型表

> [!info] 核心结论
> 总线编码器选型先确认协议，再确认单圈/多圈、分辨率、机械结构和PLC组态支持。

## ① 分类/公式/规则

> [!note] 🟦 七步选型法
> ① 看PLC支持什么总线  
> ② 看是否需要绝对位置  
> ③ 看单圈还是多圈  
> ④ 看分辨率和精度  
> ⑤ 看轴型和安装尺寸  
> ⑥ 看防护等级和环境  
> ⑦ 看设备文件和参数功能

> [!warning] 🟧 最终选型公式
> **推荐型号 = 通讯协议 + 单圈/多圈 + 分辨率 + 轴型 + 电源 + 防护等级 + 设备文件**

## ② 参考表

| 需求 | 推荐关注点 |
|---|---|
| 西门子PLC | PROFINET、PROFIBUS |
| 高速运动控制 | EtherCAT |
| 通用设备网络 | CANopen、Modbus |
| 断电保位置 | 绝对值总线编码器 |
| 长行程位置 | 多圈编码器 |
| 高精度角度 | 高位数单圈分辨率 |
| 强干扰环境 | 屏蔽、接地、总线线缆 |

## ③ 计算/选型示例

- 已知：  
  项目使用PROFINET网络，需要检测一根升降轴的位置，断电后不能丢失。
- 公式：  
  **型号需求 = PROFINET + 多圈绝对值 + 合适分辨率 + 机械轴型匹配**
- 计算：  
  升降轴超过一圈行程，需要多圈；断电不丢位置，需要绝对值；PLC网络为PROFINET。
- 结论：  
  可选择 **PROFINET多圈绝对值总线编码器**，并导入GSDML文件完成组态。

> [!tip] 选型口诀
> 协议先匹配，单圈看角度，多圈看行程；组态文件别忘记。
