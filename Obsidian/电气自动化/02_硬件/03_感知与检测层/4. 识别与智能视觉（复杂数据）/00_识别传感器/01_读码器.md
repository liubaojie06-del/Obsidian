# 01 读码器总览

> [!info] 核心结论
> 读码器用于识别 **一维码、二维码、DataMatrix码、QR码** 等编码信息，常用于产品追溯、工序防错、物料识别和自动分拣。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> 读码器不是简单拍照，而是通过图像采集、图像处理、码制解析，把标签上的编码转换成PLC或上位机可用的数据。

> [!warning] 🟧 通用选型公式
> **读码器选型 = 码制类型 + 码尺寸 + 读取距离 + 运动速度 + 光源方式 + 通讯接口 + 安装环境**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **读码器** | 识别条码、二维码、DPM码 |
| **一维码** | 条形码，如Code128、EAN等 |
| **二维码** | QR、DataMatrix等 |
| **DPM码** | 直接打标在零件表面的码 |
| **通讯接口** | Ethernet、RS232、USB、PROFINET等 |
| **典型应用** | 追溯、防错、分拣、入库、出库 |

## ③ 计算/选型示例

- 已知：  
  产线上每个产品都有二维码，需要扫码后上传产品编号。
- 公式：  
  **识别需求 = 二维码 + 在线读取 + 数据上传 + 通讯接口**
- 计算：  
  需要读码器识别二维码，并把结果传给PLC或MES系统。
- 结论：  
  可选择 **工业二维码读码器**，具体分辨率、光源、通讯协议需按厂家手册校核。

> [!tip] 记忆口诀
> 条码管编号，二维码管信息，DPM管零件追溯。

# 02 底层原理

> [!info] 核心结论
> 读码器的本质，是把图像中的黑白、明暗或刻印特征转换成数字数据。

## ① 分类/公式/规则

> [!note] 🟦 工作流程
> 读码器先通过镜头和传感器采集图像，再通过算法定位码区，最后解析码制内容，并把结果输出给PLC或上位机。

> [!warning] 🟧 读码逻辑
> **采集图像 → 定位码区 → 图像增强 → 码制解码 → 输出数据**

## ② 参考表

| 环节 | 作用 |
|---|---|
| **镜头** | 决定视野和清晰度 |
| **光源** | 提高码与背景对比度 |
| **图像传感器** | 采集码图像 |
| **解码算法** | 识别码制和内容 |
| **通讯模块** | 输出字符串或状态 |

## ③ 计算/选型示例

- 已知：  
  读码器拍到二维码，但经常解码失败。
- 公式：  
  **解码成功 = 图像清晰 + 对比度足够 + 码区完整 + 算法支持码制**
- 计算：  
  如果图像模糊、反光严重或码区缺失，算法就难以解码。
- 结论：  
  应先优化光源、焦距、曝光和安装角度，再检查通讯和程序。

> [!tip] 记忆口诀
> 读码先看图，图清才能读；图不清，算法再强也难行。

# 03 一维码读码

> [!info] 核心结论
> 一维码适合简单编号识别，信息量较小，但读取速度快、应用广。

## ① 分类/公式/规则

> [!note] 🟦 一维码特点
> 一维码由粗细不同的条纹组成，通常用于物料编号、产品编号、包装箱编号等场景。

> [!warning] 🟧 一维码适用公式
> **适合一维码 = 编号简单 + 标签空间足够 + 读取距离稳定 + 表面清晰**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **信息容量** | 较小 |
| **读取速度** | 较快 |
| **标签长度** | 内容越多越长 |
| **常见码制** | Code128、Code39、EAN等 |
| **应用场景** | 包装、仓储、物流、零件编号 |
| **注意** | 污损、折皱会影响读取 |

## ③ 计算/选型示例

- 已知：  
  包装箱只需要识别一个产品编号。
- 公式：  
  **码制需求 = 简单编号 + 快速读取**
- 计算：  
  一维码已经能满足编号识别，不一定需要二维码。
- 结论：  
  可使用 **一维码读码器**，但标签位置和条码清晰度需保证。

> [!tip] 记忆口诀
> 信息少用一维码，简单快速又直观。

# 04 二维码读码

> [!info] 核心结论
> 二维码适合存储更多信息，常用于产品追溯、工艺信息、批次信息和防错识别。

## ① 分类/公式/规则

> [!note] 🟦 二维码特点
> 二维码通过横向和纵向图形同时编码，单位面积信息量更高，常见类型有QR码和DataMatrix码。

> [!warning] 🟧 二维码适用公式
> **适合二维码 = 信息量较大 + 空间有限 + 需要追溯 + 码制稳定**

## ② 参考表

| 码制 | 常见用途 |
|---|---|
| **QR码** | 通用二维码、产品信息 |
| **DataMatrix** | 工业零件追溯常见 |
| **PDF417** | 证件、物流、票据 |
| **Micro QR** | 小空间标签 |
| **DPM码** | 激光、打刻、喷码在零件表面 |

## ③ 计算/选型示例

- 已知：  
  标签上要包含产品型号、批次号、生产日期和流水号。
- 公式：  
  **信息容量需求 = 字符数量 + 纠错需求 + 标签空间**
- 计算：  
  信息较多，一维码可能过长，二维码更合适。
- 结论：  
  可选择 **二维码读码器**，并按码尺寸、读取距离和速度校核。

> [!tip] 记忆口诀
> 信息多、地方小，二维码更可靠。

# 05 DPM直接打标码

> [!info] 核心结论
> DPM码直接刻在零件表面，适合长期追溯，但对光源、角度和算法要求更高。

## ① 分类/公式/规则

> [!note] 🟦 DPM特点
> DPM是 Direct Part Marking，常见于金属件、塑料件、汽车零件、电子零件上，通过激光、打点、蚀刻等方式形成编码。

> [!warning] 🟧 DPM识别公式
> **DPM稳定识别 = 合适光源 + 合适角度 + 表面反差 + 算法支持 + 码质量合格**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **标记方式** | 激光、打点、蚀刻、喷码 |
| **常见码制** | DataMatrix |
| **优点** | 不易脱落，适合全生命周期追溯 |
| **难点** | 反光、划伤、低对比度 |
| **光源要求** | 同轴光、环形光、低角度光等 |
| **应用行业** | 汽车、电子、机械加工、医疗器械 |

## ③ 计算/选型示例

- 已知：  
  金属零件上有激光打标DataMatrix码，但读码不稳定。
- 公式：  
  **识别稳定性 = 码对比度 + 表面反光控制 + 光源匹配**
- 计算：  
  金属表面反光强，普通光源可能导致局部过曝。
- 结论：  
  应选择支持 **DPM识别** 的读码器，并重点调试光源角度和曝光。

> [!tip] 记忆口诀
> 标签码看清晰，DPM看光源。

# 06 分辨率与视野

> [!info] 核心结论
> 读码器能不能读小码，关键看视野内每个码单元能占多少像素。

## ① 分类/公式/规则

> [!note] 🟦 像素理解
> 码越小、距离越远，对读码器分辨率和镜头要求越高。  
> 码单元太小，图像里看不清黑白格，解码就不稳定。

> [!warning] 🟧 分辨率判断公式
> **每个码单元像素数 = 码单元实际尺寸 ÷ 单像素对应尺寸**

## ② 参考表

| 项目 | 影响 |
|---|---|
| **分辨率** | 越高越能看小码 |
| **视野FOV** | 越大，单个像素对应实际尺寸越大 |
| **读取距离** | 越远，码越小 |
| **镜头焦距** | 影响视野和清晰度 |
| **码尺寸** | 决定最小可读要求 |
| **运动速度** | 影响曝光和拖影 |

## ③ 计算/选型示例

- 已知：  
  读码器水平分辨率为1280像素，实际视野宽度为160mm。
- 公式：  
  **单像素尺寸 = 视野宽度 ÷ 像素数**
- 计算：  
  单像素尺寸 = 160 ÷ 1280 = **0.125mm/像素**
- 结论：  
  若码单元尺寸很小，需要确认每个码单元有足够像素，否则需缩小视野或提高分辨率。

> [!tip] 记忆口诀
> 视野越大看得广，码越小越要高分辨率。

# 07 光源与安装角度

> [!info] 核心结论
> 读码稳定性很大程度取决于光源，尤其是反光、透明、曲面和DPM场景。

## ① 分类/公式/规则

> [!note] 🟦 光源作用
> 光源的目的不是越亮越好，而是让码和背景形成稳定对比，减少反光、阴影和过曝。

> [!warning] 🟧 光源选择公式
> **稳定图像 = 合适亮度 + 合适角度 + 反光控制 + 码背景反差**

## ② 参考表

| 光源类型 | 适合场景 |
|---|---|
| **内置光源** | 普通标签码 |
| **环形光** | 常规平面标签 |
| **同轴光** | 平面反光材料 |
| **低角度光** | 刻印、凹凸、DPM码 |
| **背光** | 轮廓或透明物辅助 |
| **偏振光** | 抑制高反光 |

## ③ 计算/选型示例

- 已知：  
  塑料膜标签表面反光，读码器经常过曝。
- 公式：  
  **反光问题 = 光源直射 + 表面高反射 + 角度不合理**
- 计算：  
  光线直接反射回镜头，会导致局部发白，码信息丢失。
- 结论：  
  可调整安装角度、降低亮度、加偏振片或更换光源方案。

> [!tip] 记忆口诀
> 读码不稳先调光，光不对，码难读。

# 08 运动读码

> [!info] 核心结论
> 在线高速读码时，要同时考虑触发时机、曝光时间、运动速度和读码器处理速度。

## ① 分类/公式/规则

> [!note] 🟦 运动影响
> 产品运动越快，图像越容易拖影；码停留在视野中的时间越短，对触发和处理速度要求越高。

> [!warning] 🟧 运动读码公式
> **可读取时间 = 视野长度 ÷ 物体速度**

## ② 参考表

| 参数 | 影响 |
|---|---|
| **运动速度** | 越快越容易拖影 |
| **曝光时间** | 越长越容易拖影 |
| **触发信号** | 决定拍照时机 |
| **视野长度** | 影响可读取时间 |
| **处理时间** | 决定能否跟上节拍 |
| **补光亮度** | 亮度足够才能缩短曝光 |

## ③ 计算/选型示例

- 已知：  
  读码视野长度为100mm，产品速度为500mm/s。
- 公式：  
  **可读取时间 = 100 ÷ 500**
- 计算：  
  可读取时间 = **0.2s**
- 结论：  
  读码器必须在0.2s内完成触发、拍照、解码和输出，否则可能漏读。

> [!tip] 记忆口诀
> 静态看清晰，动态看速度；高速读码，曝光要短。

# 09 通讯与PLC/MES对接

> [!info] 核心结论
> 读码器输出的不只是“读到/没读到”，更重要的是把码内容正确传给PLC、上位机或MES。

## ① 分类/公式/规则

> [!note] 🟦 通讯内容
> 常见输出包括读码结果、字符串内容、读码成功标志、失败标志、质量等级、错误码和触发状态。

> [!warning] 🟧 通讯成功公式
> **系统读取正常 = 通讯接口匹配 + 字符串格式正确 + 触发逻辑正确 + 数据解析正确**

## ② 参考表

| 通讯方式 | 常见用途 |
|---|---|
| **Ethernet TCP/IP** | 上位机、MES、PLC通讯 |
| **PROFINET** | 西门子PLC集成 |
| **EtherNet/IP** | 罗克韦尔系统 |
| **RS232/RS485** | 串口输出字符串 |
| **USB** | 调试、配置、简单连接 |
| **IO输出** | OK/NG、触发、完成信号 |

## ③ 计算/选型示例

- 已知：  
  读码器能显示读码成功，但PLC没有收到字符串。
- 公式：  
  **PLC收码成功 = 通讯连通 + 数据格式一致 + 程序解析正确**
- 计算：  
  可能是PLC没有配置字符串接收区，或结束符、长度、字节顺序不一致。
- 结论：  
  应检查通讯协议、IP/端口、字符串格式、结束符和PLC接收逻辑。

> [!tip] 记忆口诀
> 读到码不算完，传到系统才闭环。

# 10 常见故障排查

> [!info] 核心结论
> 读码异常要按“图像质量 → 码质量 → 触发时机 → 通讯解析”的顺序排查。

## ① 分类/公式/规则

> [!note] 🟦 排查顺序
> 先看图像是否清楚，再看码是否完整；  
> 先看读码器是否读到，再看PLC是否收到；  
> 先低速调稳，再高速验证。

> [!warning] 🟧 排错公式
> **读码失败 = 图像模糊 + 光源不合适 + 码损坏 + 焦距错误 + 触发错误 + 通讯错误**

## ② 参考表

| 问题 | 可能原因 |
|---|---|
| **读不到码** | 焦距、光源、码制、距离错误 |
| **偶发漏读** | 运动过快、触发偏移、曝光过长 |
| **反光过曝** | 光源角度不合理 |
| **读到错误码** | 视野内有多个码 |
| **PLC无数据** | 通讯参数或解析错误 |
| **换批次不稳定** | 印刷质量或码尺寸变化 |

## ③ 计算/选型示例

- 已知：  
  静止时能读码，运动时漏读。
- 公式：  
  **运动漏读 = 曝光拖影 + 触发延迟 + 处理时间不足**
- 计算：  
  产品运动后图像模糊或码区不在最佳位置。
- 结论：  
  应缩短曝光、增强光源、优化触发位置，必要时选择更高速读码器。

> [!tip] 记忆口诀
> 静态能读不算稳，在线节拍能读才算准。

# 11 读码器快速选型表

> [!info] 核心结论
> 读码器选型先看码制和码尺寸，再看距离、速度、光源、通讯和环境。

## ① 分类/公式/规则

> [!note] 🟦 八步选型法
> ① 看码制类型  
> ② 看码尺寸和单元大小  
> ③ 看读取距离  
> ④ 看产品运动速度  
> ⑤ 看表面材质和反光  
> ⑥ 看是否需要DPM  
> ⑦ 看通讯接口  
> ⑧ 看防护等级

> [!warning] 🟧 最终选型公式
> **推荐读码器 = 码制 + 分辨率 + 镜头 + 光源 + 读取距离 + 通讯接口 + 防护等级**

## ② 参考表

| 需求 | 推荐关注点 |
|---|---|
| 一维条码 | 一维码读码器或通用读码器 |
| 二维码/QR | 二维读码器 |
| DataMatrix | 工业二维码读码器 |
| 金属DPM码 | DPM专用算法和光源 |
| 高速产线 | 曝光、处理速度、触发 |
| 接PLC | PROFINET、TCP/IP、IO |
| 接MES | Ethernet、TCP/IP、数据库接口 |

## ③ 计算/选型示例

- 已知：  
  产品表面为金属件，码为激光打标DataMatrix，产线高速运行，需要上传MES。
- 公式：  
  **型号需求 = DPM能力 + 高速解码 + 合适光源 + Ethernet通讯**
- 计算：  
  金属DPM码需要专用光源和算法，高速运行需要短曝光和快速处理。
- 结论：  
  应选择 **支持DPM的工业读码器**，并校核光源、镜头、处理速度和通讯接口。

> [!tip] 选型口诀
> 先看码，再看光；静态看清晰，动态看速度；读码要闭环，通讯要对上。
