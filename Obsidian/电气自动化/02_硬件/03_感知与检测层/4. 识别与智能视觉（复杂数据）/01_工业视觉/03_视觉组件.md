# 01 图像采集产品总览

> [!info] 核心结论
> 图像采集产品负责把相机图像稳定、快速、完整地送到视觉软件或控制器，是机器视觉系统的“图像入口”。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> 相机负责拍图，图像采集产品负责 **接收图像、触发同步、数据传输、缓存处理、接口转换**。

> [!warning] 🟧 通用选型公式
> **图像采集产品选型 = 相机接口 + 分辨率 + 帧率 + 数据带宽 + 触发方式 + 软件兼容性**

## ② 参考表

| 类型 | 作用 |
|---|---|
| **图像采集卡** | 高速接收相机图像 |
| **接口卡** | 扩展GigE、USB3.0、Camera Link等接口 |
| **采集盒** | 外置式图像采集与转换 |
| **I/O触发模块** | 触发相机、接收传感器信号 |
| **光源控制器** | 控制频闪光源 |
| **视觉控制器** | 采图、处理、输出一体化 |

## ③ 计算/选型示例

- 已知：  
  4台工业相机需要同时采图，并送到视觉软件处理。
- 公式：  
  **采集需求 = 相机数量 × 单相机数据量 × 帧率**
- 计算：  
  相机数量越多、分辨率越高、帧率越快，对采集带宽和处理能力要求越高。
- 结论：  
  需要选择带宽足够、接口匹配、支持多相机同步的图像采集方案。

> [!tip] 记忆口诀
> 相机拍得好，还要采得稳；带宽不够，再清楚也掉帧。

# 02 底层原理

> [!info] 核心结论
> 图像采集的本质，是把相机传出的像素数据按正确时序传输到控制器或工控机。

## ① 分类/公式/规则

> [!note] 🟦 工作流程
> 相机曝光后生成图像数据，通过接口传输到采集卡、视觉控制器或工控机，软件再进行图像处理和结果判断。

> [!warning] 🟧 采集逻辑
> **触发 → 曝光 → 传输图像 → 缓存 → 软件处理 → 输出结果**

## ② 参考表

| 环节 | 作用 |
|---|---|
| **触发信号** | 决定什么时候拍照 |
| **曝光** | 决定图像亮度和拖影 |
| **传输接口** | 决定带宽和距离 |
| **缓存** | 防止短时数据拥堵 |
| **驱动/SDK** | 让软件读取图像 |
| **处理平台** | 工控机或视觉控制器 |

## ③ 计算/选型示例

- 已知：  
  相机拍照正常，但软件偶尔收不到图像。
- 公式：  
  **采集稳定 = 触发稳定 + 接口带宽足够 + 驱动正常 + 缓存不溢出**
- 计算：  
  若触发频率过快或带宽不足，图像可能丢帧。
- 结论：  
  应检查触发频率、相机帧率、接口带宽、驱动和缓存设置。

> [!tip] 记忆口诀
> 图像采集看链路，触发、曝光、传输一个都不能乱。

# 03 常见图像采集产品

> [!info] 核心结论
> 不同采集产品解决的问题不同：采集卡管高速，接口卡管扩展，I/O模块管同步，光源控制器管打光。

## ① 分类/公式/规则

> [!note] 🟦 分类主线
> 高速相机看采集卡；  
> 多相机接入看接口卡；  
> 高速定位看触发模块；  
> 运动拍照看编码器同步；  
> 短曝光防拖影看频闪光源控制器。

> [!warning] 🟧 产品匹配公式
> **采集方案 = 相机接口 + 触发同步 + 光源控制 + 软件平台**

## ② 参考表

| 产品 | 适合场景 |
|---|---|
| **采集卡** | 高速、高分辨率相机 |
| **GigE网卡** | 多台GigE相机 |
| **USB3扩展卡** | 多台USB3相机 |
| **Camera Link采集卡** | 传统高速工业相机 |
| **I/O模块** | 触发、完成、OK/NG信号 |
| **光源控制器** | 频闪、亮度、通道控制 |

## ③ 计算/选型示例

- 已知：  
  高速产线运动拍照有拖影，需要缩短曝光。
- 公式：  
  **清晰图像 = 短曝光 + 足够亮度 + 稳定触发**
- 计算：  
  曝光缩短后图像会变暗，需要频闪光源提高瞬时亮度。
- 结论：  
  可增加 **光源控制器 + 频闪光源**，并与相机触发同步。

> [!tip] 记忆口诀
> 高速看采集，运动看触发，拖影看频闪。

# 04 相机接口与采集卡

> [!info] 核心结论
> 相机接口决定传输带宽、传输距离、稳定性和采集硬件选择。

## ① 分类/公式/规则

> [!note] 🟦 常见接口
> GigE适合较长距离和多相机；  
> USB3适合短距离高速采集；  
> Camera Link、CoaXPress常用于高带宽工业视觉；  
> 接口选择必须与采集卡或控制器匹配。

> [!warning] 🟧 接口匹配公式
> **采集成功 = 相机接口 + 采集硬件接口 + 驱动/SDK + 软件平台一致**

## ② 参考表

| 接口 | 特点 |
|---|---|
| **GigE** | 距离较长，工业常见 |
| **USB3.0** | 速度快，距离较短 |
| **Camera Link** | 高速稳定，需采集卡 |
| **CoaXPress** | 高带宽，适合高速高分辨率 |
| **MIPI** | 嵌入式、板级相机常见 |
| **HDMI/SDI** | 显示和视频类应用常见 |

## ③ 计算/选型示例

- 已知：  
  相机为Camera Link接口，工控机没有对应接口。
- 公式：  
  **可采集 = Camera Link相机 + Camera Link采集卡**
- 计算：  
  普通网口或USB口无法直接采集Camera Link相机图像。
- 结论：  
  需要安装匹配的 **Camera Link图像采集卡**，并确认驱动和视觉软件支持。

> [!tip] 记忆口诀
> 相机什么口，采集就配什么口；接口不对，图像进不来。

# 05 触发同步

> [!info] 核心结论
> 触发同步决定图像拍摄时机，是运动检测、定位检测和多相机系统稳定性的关键。

## ① 分类/公式/规则

> [!note] 🟦 触发来源
> 触发信号可来自光电传感器、PLC、编码器、运动控制器或相机内部定时器。

> [!warning] 🟧 触发同步公式
> **稳定拍照 = 触发位置固定 + 曝光时间合适 + 光源同步 + 处理节拍满足**

## ② 参考表

| 触发方式 | 适合场景 |
|---|---|
| **软触发** | 软件控制拍照 |
| **硬触发** | 光电/PLC触发，稳定性高 |
| **编码器触发** | 线阵相机、定距采图 |
| **多相机同步** | 多角度同时拍照 |
| **外部频闪** | 高速运动防拖影 |
| **触发延时** | 补偿传感器与拍照点距离 |

## ③ 计算/选型示例

- 已知：  
  产品经过光电后 **50mm** 到达拍照位置，输送速度 **500mm/s**。
- 公式：  
  **触发延时 = 距离 ÷ 速度**
- 计算：  
  触发延时 = 50 ÷ 500 = **0.1s**
- 结论：  
  PLC或视觉软件可设置约 **100ms触发延时**，让产品到达视野中心再拍照。

> [!tip] 记忆口诀
> 拍早拍晚都不行，触发延时要算清。

# 06 图像带宽计算

> [!info] 核心结论
> 图像采集是否稳定，必须校核数据带宽，尤其是多相机、高分辨率、高帧率场景。

## ① 分类/公式/规则

> [!warning] 🟧 带宽公式
> **图像带宽 = 水平像素 × 垂直像素 × 位深 × 帧率**

> [!note] 🟦 单位提醒
> 位深通常用bit表示，8bit = 1Byte。  
> 实际还需考虑协议开销、缓存、网卡性能和硬盘写入速度。

## ② 参考表

| 参数 | 影响 |
|---|---|
| **分辨率** | 越高数据越大 |
| **帧率** | 越高带宽越大 |
| **位深** | 8bit、10bit、12bit等 |
| **相机数量** | 多相机带宽叠加 |
| **接口带宽** | 限制最大传输能力 |
| **存图需求** | 影响硬盘写入速度 |

## ③ 计算/选型示例

- 已知：  
  相机分辨率为 **1920×1080**，8bit灰度图，帧率 **60fps**。
- 公式：  
  **带宽 = 1920 × 1080 × 1Byte × 60**
- 计算：  
  带宽 ≈ 124,416,000Byte/s ≈ **124MB/s**
- 结论：  
  单台相机约需 **124MB/s**，多相机时要叠加，并校核接口和工控机带宽。

> [!tip] 记忆口诀
> 分辨率乘帧率，带宽马上见高低。

# 07 视觉控制器总览

> [!info] 核心结论
> 视觉控制器是专用的视觉处理设备，用于连接相机、运行视觉算法、管理I/O通讯，并把结果输出给PLC或机器人。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> 视觉控制器可以理解为“专门做视觉的工业电脑”。  
> 它比普通相机更能处理多相机任务，比工控机部署更集成。

> [!warning] 🟧 视觉控制器选型公式
> **视觉控制器选型 = 相机数量 + 算法复杂度 + 处理节拍 + 通讯接口 + 存储需求 + 软件平台**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **相机接口** | GigE、USB3、Camera Link等 |
| **处理能力** | CPU/GPU/专用视觉处理 |
| **I/O接口** | 触发、OK/NG、完成信号 |
| **通讯接口** | PROFINET、EtherNet/IP、TCP/IP等 |
| **存储** | 图片、日志、配方、结果 |
| **软件** | 视觉流程、算法、调试平台 |

## ③ 计算/选型示例

- 已知：  
  一个工位有3台相机，需要检测多个角度并统一给PLC结果。
- 公式：  
  **控制器需求 = 多相机输入 + 统一处理 + PLC通讯**
- 计算：  
  多台智能相机各自输出可能管理复杂，视觉控制器可集中处理。
- 结论：  
  可选择 **多相机视觉控制器**，并校核接口数量、处理速度和PLC通讯协议。

> [!tip] 记忆口诀
> 单相机简单用智能，多相机集中用控制器。

# 08 视觉控制器 vs 工控机 vs 智能相机

> [!info] 核心结论
> 智能相机适合简单单点检测，视觉控制器适合多相机集成，工控机适合复杂算法和高扩展性系统。

## ① 分类/公式/规则

> [!note] 🟦 快速选择
> 单点OK/NG：智能相机。  
> 多相机标准检测：视觉控制器。  
> AI、多算法、大量数据：工控机或服务器。

> [!warning] 🟧 平台选择公式
> **平台选择 = 检测复杂度 + 相机数量 + 算力需求 + 维护方式**

## ② 参考表

| 平台 | 适合场景 |
|---|---|
| **智能相机** | 单工位、简单检测 |
| **视觉控制器** | 多相机、标准视觉流程 |
| **工控机** | 复杂算法、深度学习、多接口 |
| **服务器** | 大数据、集中管理、AI训练 |
| **嵌入式平台** | 小型设备、低功耗 |
| **云平台** | 远程分析和追溯 |

## ③ 计算/选型示例

- 已知：  
  项目有8台相机，需要AI缺陷检测和大量图片保存。
- 公式：  
  **算力需求 = 相机数量 × 图像大小 × 算法复杂度 × 节拍**
- 计算：  
  多相机+AI通常对CPU/GPU和存储要求高。
- 结论：  
  普通视觉控制器可能不够，建议评估 **高性能工控机或GPU视觉平台**。

> [!tip] 记忆口诀
> 简单用相机，标准用控制器，复杂上工控。

# 09 与PLC通讯

> [!info] 核心结论
> 图像采集系统和视觉控制器必须与PLC做好触发、完成、OK/NG、数据和复位握手。

## ① 分类/公式/规则

> [!note] 🟦 通讯内容
> 常见信号包括：触发Trigger、忙碌Busy、完成Done、结果OK/NG、坐标数据、尺寸数据、错误码、复位Reset。

> [!warning] 🟧 通讯握手公式
> **PLC触发 → 视觉采图 → 视觉处理 → 输出Done和结果 → PLC读取 → 视觉复位等待下一拍**

## ② 参考表

| 信号/数据 | 作用 |
|---|---|
| **Trigger** | PLC通知拍照 |
| **Busy** | 视觉正在处理 |
| **Done** | 视觉处理完成 |
| **OK/NG** | 检测结果 |
| **Data** | 坐标、尺寸、角度、条码 |
| **Error** | 通讯或检测异常 |

## ③ 计算/选型示例

- 已知：  
  视觉处理时间为 **120ms**，PLC工位节拍为 **150ms**。
- 公式：  
  **节拍余量 = PLC节拍 - 视觉处理时间**
- 计算：  
  节拍余量 = 150 - 120 = **30ms**
- 结论：  
  理论可满足节拍，但还需考虑触发延时、相机曝光、通讯时间和执行机构动作时间。

> [!tip] 记忆口诀
> 触发要清楚，完成要握手；PLC没读完，视觉别乱走。

# 10 常见故障排查

> [!info] 核心结论
> 图像采集和视觉控制器异常时，按“相机在线 → 触发正常 → 图像稳定 → 带宽足够 → 软件输出”的顺序排查。

## ① 分类/公式/规则

> [!note] 🟦 排查顺序
> 先查供电和连接，再查驱动和IP；  
> 先看是否采到图，再看是否处理正确；  
> 先查视觉本体结果，再查PLC通讯。

> [!warning] 🟧 排错公式
> **采集异常 = 相机离线 + 触发错误 + 带宽不足 + 驱动异常 + 缓存溢出 + 通讯失败**

## ② 参考表

| 问题 | 可能原因 |
|---|---|
| **搜索不到相机** | IP、驱动、网线、供电异常 |
| **偶发丢帧** | 带宽不足、缓存溢出 |
| **图像黑屏** | 曝光、光源、触发模式错误 |
| **触发无图** | 触发极性、触发源设置错误 |
| **处理卡顿** | 算力不足、图片过大 |
| **PLC无结果** | 通讯地址、握手逻辑错误 |

## ③ 计算/选型示例

- 已知：  
  相机单帧数据约10MB，连续采集30fps。
- 公式：  
  **采集带宽 = 单帧大小 × 帧率**
- 计算：  
  采集带宽 = 10 × 30 = **300MB/s**
- 结论：  
  若接口或硬盘写入速度低于该带宽，可能出现丢帧、卡顿或保存失败。

> [!tip] 记忆口诀
> 搜不到查网络，丢帧查带宽，黑图查曝光，没结果查握手。

# 11 快速选型表

> [!info] 核心结论
> 图像采集产品和视觉控制器选型时，先看相机数量和数据带宽，再看触发同步、算法能力和PLC通讯。

## ① 分类/公式/规则

> [!note] 🟦 八步选型法
> ① 看相机接口  
> ② 看相机数量  
> ③ 算图像带宽  
> ④ 看触发同步方式  
> ⑤ 看是否需要光源控制  
> ⑥ 看算法复杂度  
> ⑦ 看PLC/机器人/MES通讯  
> ⑧ 看存储和维护方式

> [!warning] 🟧 最终选型公式
> **推荐方案 = 采集接口 + 带宽能力 + 触发I/O + 视觉软件 + 控制器算力 + 通讯接口**

## ② 参考表

| 需求 | 推荐方向 |
|---|---|
| 单相机简单检测 | 智能相机可评估 |
| 多相机集中处理 | 视觉控制器 |
| 高速高分辨率 | 采集卡 / 高带宽接口 |
| 线阵连续扫描 | 编码器同步 + 高行频采集 |
| 频闪拍照 | 光源控制器 |
| 复杂AI检测 | 工控机/GPU平台 |
| PLC集成 | PROFINET、EtherNet/IP、IO等 |

## ③ 计算/选型示例

- 已知：  
  产线有4台GigE相机，每台约60MB/s，需要同时采集。
- 公式：  
  **总带宽 = 单相机带宽 × 相机数量**
- 计算：  
  总带宽 = 60 × 4 = **240MB/s**
- 结论：  
  需要校核网卡、交换机、视觉控制器或工控机是否能稳定承载 **240MB/s以上** 数据量，并留有余量。

> [!tip] 选型口诀
> 相机定接口，帧率定带宽；多相机看控制器，高速采集看同步。
