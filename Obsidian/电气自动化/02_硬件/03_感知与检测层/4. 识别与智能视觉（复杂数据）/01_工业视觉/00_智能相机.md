# 01 智能相机总览

> [!info] 核心结论
> 智能相机是一种集 **图像采集、图像处理、结果判断、通讯输出** 于一体的工业视觉设备。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> 普通工业相机主要负责拍照，需要外部工控机处理图像；  
> 智能相机内部自带处理器，可直接完成检测并输出 OK/NG、坐标、尺寸、条码等结果。

> [!warning] 🟧 通用选型公式
> **智能相机选型 = 检测任务 + 视野范围 + 精度要求 + 处理速度 + 光源镜头 + 通讯接口 + 安装环境**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **智能相机** | 相机 + 处理器 + 软件算法 |
| **检测内容** | 有无、定位、尺寸、缺陷、读码 |
| **输出结果** | OK/NG、坐标、角度、数值、字符串 |
| **通讯接口** | IO、Ethernet、PROFINET、TCP/IP等 |
| **典型应用** | 装配防错、外观检测、定位引导、读码追溯 |
| **重点参数** | 分辨率、视野、帧率、算法能力、光源 |

## ③ 计算/选型示例

- 已知：  
  需要检测产品是否漏装螺丝，并把OK/NG信号给PLC。
- 公式：  
  **视觉需求 = 图像采集 + 螺丝区域识别 + OK/NG输出**
- 计算：  
  智能相机可拍摄产品图像，判断螺丝是否存在，再通过IO或通讯输出结果。
- 结论：  
  可选择 **智能相机 + 合适光源 + 镜头**，具体分辨率和算法需按现场样品测试校核。

> [!tip] 记忆口诀
> 普通相机只拍图，智能相机能判断。

# 02 底层原理

> [!info] 核心结论
> 智能相机的本质，是把真实物体转换成图像，再通过算法提取特征并输出判断结果。

## ① 分类/公式/规则

> [!note] 🟦 工作流程
> 智能相机先通过镜头和图像传感器采集图像，再由内部算法进行定位、测量、识别、判断，最后把结果传给PLC、机器人或上位机。

> [!warning] 🟧 视觉检测逻辑
> **打光 → 成像 → 定位 → 提取特征 → 算法判断 → 输出结果**

## ② 参考表

| 环节 | 作用 |
|---|---|
| **光源** | 提高目标与背景对比度 |
| **镜头** | 决定视野、放大倍率、清晰度 |
| **图像传感器** | 采集图像 |
| **视觉算法** | 查找、测量、识别、判断 |
| **通讯模块** | 输出结果给PLC或上位机 |
| **触发信号** | 控制拍照时机 |

## ③ 计算/选型示例

- 已知：  
  相机能拍到产品，但检测结果不稳定。
- 公式：  
  **检测稳定 = 图像稳定 + 特征明显 + 算法参数正确**
- 计算：  
  如果光源不稳定、产品反光、位置偏移大，算法就容易误判。
- 结论：  
  应先优化成像质量，再调整算法参数。

> [!tip] 记忆口诀
> 视觉先看图，图稳才好判。

# 03 智能相机 vs 普通工业相机

> [!info] 核心结论
> 智能相机适合单机检测和快速部署，普通工业相机更适合复杂视觉系统和高算力应用。

## ① 分类/公式/规则

> [!note] 🟦 核心区别
> 智能相机内部完成处理；  
> 普通工业相机需要连接工控机、视觉软件或控制器完成处理。

> [!warning] 🟧 选择公式
> **简单检测 / 单工位 → 智能相机**  
> **复杂算法 / 多相机 / 高算力 → 工业相机 + 工控机**

## ② 参考表

| 对比项 | 智能相机 | 普通工业相机 |
|---|---|---|
| 图像处理 | **相机内部完成** | 外部工控机完成 |
| 部署难度 | 较低 | 较高 |
| 系统成本 | 小项目较省 | 大系统更灵活 |
| 算法扩展 | 有限制 | 更强 |
| 多相机同步 | 一般 | 更适合 |
| 典型应用 | 单点检测、防错、读码 | 复杂检测、AI视觉、多相机 |

## ③ 计算/选型示例

- 已知：  
  一个工位只需要判断标签是否贴正。
- 公式：  
  **任务复杂度 = 单相机 + 简单定位 + OK/NG输出**
- 计算：  
  不需要复杂工控机系统，智能相机即可完成。
- 结论：  
  该场景优先选择 **智能相机**。

> [!tip] 记忆口诀
> 单点检测选智能，多点复杂上工控。

# 04 典型检测功能

> [!info] 核心结论
> 智能相机常用于有无检测、尺寸测量、定位引导、缺陷检测、颜色判断和读码追溯。

## ① 分类/公式/规则

> [!note] 🟦 常见功能
> 智能相机不是只能拍照，它可以输出检测结论、坐标、角度、尺寸、面积、数量、码内容等数据。

> [!warning] 🟧 功能选择公式
> **视觉功能 = 检测对象 + 判断标准 + 输出结果**

## ② 参考表

| 功能 | 示例 |
|---|---|
| **有无检测** | 螺丝、标签、零件是否存在 |
| **尺寸测量** | 长度、宽度、孔径、间距 |
| **定位检测** | 坐标X/Y、角度θ |
| **缺陷检测** | 划伤、污点、破损、毛刺 |
| **颜色识别** | 色差、混料、防错 |
| **读码识别** | 条码、二维码、DataMatrix |

## ③ 计算/选型示例

- 已知：  
  需要检测零件孔位是否偏移。
- 公式：  
  **偏移量 = 实际孔中心坐标 - 标准孔中心坐标**
- 计算：  
  相机定位孔中心后，与标准位置比较，超过公差则输出NG。
- 结论：  
  可使用智能相机的 **定位 + 测量工具** 完成孔位检测。

> [!tip] 记忆口诀
> 有无看存在，测量看尺寸，定位看坐标，缺陷看差异。

# 05 分辨率与视野

> [!info] 核心结论
> 分辨率决定图像细节，视野决定能看到多大范围，两者共同决定检测精度。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> 视野越大，同样分辨率下每个像素代表的实际尺寸越大，检测精度会下降。  
> 小缺陷、小尺寸检测通常需要更高分辨率或更小视野。

> [!warning] 🟧 单像素尺寸公式
> **单像素尺寸 = 视野宽度 ÷ 水平像素数**

## ② 参考表

| 参数 | 影响 |
|---|---|
| **分辨率** | 图像细节 |
| **视野FOV** | 拍摄范围 |
| **像素尺寸** | 每个像素代表多少mm |
| **检测精度** | 通常需要多个像素支撑 |
| **镜头焦距** | 影响视野和放大倍率 |
| **安装距离** | 影响视野和清晰度 |

## ③ 计算/选型示例

- 已知：  
  相机水平像素为 **1280px**，视野宽度为 **160mm**。
- 公式：  
  **单像素尺寸 = 160 ÷ 1280**
- 计算：  
  单像素尺寸 = **0.125mm/px**
- 结论：  
  如果要检测0.1mm级别变化，该视野和分辨率可能不足，需要缩小视野或提高分辨率。

> [!tip] 记忆口诀
> 视野越大看得广，精度越高视野要收小。

# 06 光源与镜头

> [!info] 核心结论
> 机器视觉稳定性的核心不是相机越贵越好，而是光源、镜头、角度和算法匹配。

## ① 分类/公式/规则

> [!note] 🟦 光源作用
> 光源的目标是让被检测特征更明显，让背景干扰更少。  
> 镜头负责把目标清晰成像到相机传感器上。

> [!warning] 🟧 成像稳定公式
> **好图像 = 合适光源 + 合适镜头 + 合适距离 + 合适角度**

## ② 参考表

| 光源/镜头 | 适合场景 |
|---|---|
| **环形光** | 常规表面检测 |
| **条形光** | 边缘、轮廓、侧向补光 |
| **背光源** | 外形尺寸、轮廓检测 |
| **同轴光** | 平面反光表面 |
| **低角度光** | 划伤、刻印、凹凸缺陷 |
| **远心镜头** | 高精度尺寸测量 |

## ③ 计算/选型示例

- 已知：  
  需要检测金属表面划痕，普通环形光下看不清。
- 公式：  
  **缺陷可见性 = 光照角度 + 表面反射差异**
- 计算：  
  划痕属于微小表面变化，低角度光更容易把划痕打亮或形成阴影。
- 结论：  
  可尝试 **低角度光源**，必要时配合偏振片或改变拍摄角度。

> [!tip] 记忆口诀
> 视觉不稳定，先别怪相机，先看光和镜头。

# 07 触发与运动拍照

> [!info] 核心结论
> 在线检测时，触发时机决定拍照位置；曝光时间决定运动图像是否拖影。

## ① 分类/公式/规则

> [!note] 🟦 运动检测重点
> 产品在运动中经过相机时，通常由光电、编码器或PLC触发拍照。  
> 如果触发晚了，产品不在视野中心；如果曝光太长，图像会拖影。

> [!warning] 🟧 可拍摄时间公式
> **可拍摄时间 = 视野长度 ÷ 物体速度**

## ② 参考表

| 参数 | 影响 |
|---|---|
| **触发信号** | 决定拍照时刻 |
| **触发延时** | 补偿传感器到拍照点距离 |
| **曝光时间** | 影响亮度和拖影 |
| **频闪光源** | 可冻结高速运动 |
| **处理时间** | 决定检测节拍 |
| **输出延时** | 影响PLC动作时机 |

## ③ 计算/选型示例

- 已知：  
  相机视野长度为 **80mm**，产品速度为 **400mm/s**。
- 公式：  
  **可拍摄时间 = 80 ÷ 400**
- 计算：  
  可拍摄时间 = **0.2s**
- 结论：  
  相机需要在0.2s内完成拍照、处理和输出，否则可能跟不上节拍。

> [!tip] 记忆口诀
> 静态看清晰，动态看触发；速度越快，曝光越短。

# 08 与PLC通讯

> [!info] 核心结论
> 智能相机接PLC时，要明确输出的是OK/NG信号、坐标数据、测量值还是字符串结果。

## ① 分类/公式/规则

> [!note] 🟦 通讯方式
> 简单检测可用IO输出OK/NG；  
> 需要传坐标、尺寸、条码内容时，通常使用Ethernet、TCP/IP、PROFINET、EtherNet/IP等通讯方式。

> [!warning] 🟧 通讯成功公式
> **PLC读取正常 = 通讯协议匹配 + 数据格式一致 + 触发握手正确 + 地址映射正确**

## ② 参考表

| 通讯方式 | 适合用途 |
|---|---|
| **数字IO** | OK/NG、触发、完成信号 |
| **Ethernet TCP/IP** | 字符串、坐标、测量值 |
| **PROFINET** | 西门子PLC集成 |
| **EtherNet/IP** | 罗克韦尔系统 |
| **RS232/RS485** | 简单串口数据 |
| **FTP/MES** | 图片或追溯数据上传 |

## ③ 计算/选型示例

- 已知：  
  相机需要把零件坐标X、Y、角度θ发送给PLC。
- 公式：  
  **数据通讯需求 = X + Y + θ + 状态位**
- 计算：  
  单纯IO只能传OK/NG，不适合传多个数值。
- 结论：  
  应选择支持以太网通讯的智能相机，并在PLC中解析坐标数据。

> [!tip] 记忆口诀
> OK/NG用IO，坐标数据走通讯。

# 09 标定与坐标换算

> [!info] 核心结论
> 如果智能相机要输出真实尺寸或机器人坐标，就必须进行标定。

## ① 分类/公式/规则

> [!note] 🟦 标定作用
> 相机看到的是像素坐标；  
> 设备需要的是毫米坐标、工件坐标或机器人坐标。  
> 标定就是建立像素和实际坐标之间的关系。

> [!warning] 🟧 坐标换算公式
> **实际尺寸 = 像素数量 × 单像素实际尺寸**

## ② 参考表

| 标定类型 | 用途 |
|---|---|
| **像素标定** | 像素转毫米 |
| **畸变校正** | 修正镜头变形 |
| **手眼标定** | 相机坐标转机器人坐标 |
| **平面标定** | 2D定位和测量 |
| **高度标定** | 位移或3D测量 |
| **基准点标定** | 建立设备坐标原点 |

## ③ 计算/选型示例

- 已知：  
  标定后 1px = **0.05mm**，测得两个边缘相距 **300px**。
- 公式：  
  **实际尺寸 = 300 × 0.05**
- 计算：  
  实际尺寸 = **15mm**
- 结论：  
  该目标宽度约为 **15mm**。

> [!tip] 记忆口诀
> 相机看像素，设备要毫米；不做标定，坐标没意义。

# 10 安装与防护

> [!info] 核心结论
> 智能相机安装要保证视野稳定、焦距稳定、光源固定、线缆可靠，并防止粉尘、油污和振动影响图像。

## ① 分类/公式/规则

> [!note] 🟦 安装重点
> 相机支架要刚性好，镜头焦距不能松动，光源位置要固定，现场振动大时要加固支撑。

> [!warning] 🟧 安装可靠公式
> **稳定检测 = 相机固定 + 光源固定 + 焦距固定 + 触发稳定 + 环境防护**

## ② 参考表

| 检查项 | 重点 |
|---|---|
| **相机支架** | 防振、防松 |
| **镜头锁紧** | 防止焦距变化 |
| **光源固定** | 防止角度变化 |
| **防尘防油** | 必要时加防护罩 |
| **线缆固定** | 避免拉扯和干扰 |
| **散热空间** | 防止长期高温 |

## ③ 计算/选型示例

- 已知：  
  设备运行一段时间后，检测位置逐渐偏移。
- 公式：  
  **位置漂移 = 相机松动 + 光源偏移 + 工装定位变化**
- 计算：  
  若相机支架松动，图像坐标会整体偏移。
- 结论：  
  应检查相机支架、镜头锁紧、工装定位和光源固定状态。

> [!tip] 记忆口诀
> 相机一松，全图都偏；光源一变，结果也变。

# 11 常见故障排查

> [!info] 核心结论
> 智能相机异常排查，要按“图像质量 → 算法参数 → 触发时机 → 通讯输出”的顺序检查。

## ① 分类/公式/规则

> [!note] 🟦 排查顺序
> 先看图像是否清楚，再看特征是否明显；  
> 先低速调稳定，再高速验证；  
> 先确认相机结果，再检查PLC接收。

> [!warning] 🟧 排错公式
> **视觉异常 = 图像模糊 + 光源不稳 + 焦距错误 + 产品偏移 + 参数错误 + 通讯异常**

## ② 参考表

| 问题 | 可能原因 |
|---|---|
| **误判** | 光照变化、阈值不合理、样品差异大 |
| **漏检** | 特征不明显、视野偏移、算法区域错误 |
| **图像模糊** | 焦距不准、运动拖影、镜头污染 |
| **触发不到** | 触发信号错误或延时不对 |
| **PLC无结果** | 通讯、地址、握手逻辑错误 |
| **换批次异常** | 产品颜色、位置、反光变化 |

## ③ 计算/选型示例

- 已知：  
  相机本体显示检测OK，但PLC没有收到OK信号。
- 公式：  
  **PLC接收正常 = 相机输出 + 接线/通讯 + PLC地址 + 握手逻辑**
- 计算：  
  相机检测结果正常，问题可能在输出映射、通讯格式或PLC程序。
- 结论：  
  应检查IO接线、通讯协议、数据地址和触发完成信号。

> [!tip] 记忆口诀
> 相机判得对，还要PLC收得到。

# 12 智能相机快速选型表

> [!info] 核心结论
> 智能相机选型先看检测任务和精度，再看视野、速度、光源镜头、通讯和环境。

## ① 分类/公式/规则

> [!note] 🟦 八步选型法
> ① 看检测任务  
> ② 看视野范围  
> ③ 看最小缺陷或最小尺寸  
> ④ 看分辨率和镜头  
> ⑤ 看产品速度和节拍  
> ⑥ 看光源方案  
> ⑦ 看输出通讯  
> ⑧ 看安装环境和防护

> [!warning] 🟧 最终选型公式
> **推荐型号 = 分辨率 + 处理速度 + 算法功能 + 镜头光源 + 通讯接口 + 防护等级**

## ② 参考表

| 需求 | 推荐关注点 |
|---|---|
| 有无检测 | 普通智能相机即可评估 |
| 尺寸测量 | 分辨率、镜头、标定 |
| 缺陷检测 | 光源和算法能力 |
| 读码追溯 | 读码算法和通讯 |
| 机器人定位 | 坐标输出和手眼标定 |
| 高速产线 | 帧率、曝光、处理时间 |
| 恶劣环境 | 防护罩、防尘、防油 |

## ③ 计算/选型示例

- 已知：  
  视野宽度为100mm，需要检测最小 **0.2mm** 的缺陷，相机水平像素为1280px。
- 公式：  
  **单像素尺寸 = 100 ÷ 1280**
- 计算：  
  单像素尺寸 ≈ **0.078mm/px**  
  0.2mm缺陷约占 0.2 ÷ 0.078 ≈ **2.6px**
- 结论：  
  像素占比偏少，检测可能不稳定；建议提高分辨率、缩小视野或优化光源。

> [!tip] 选型口诀
> 先看任务，再看视野；精度看像素，稳定看光源；结果给PLC，通讯要匹配。
