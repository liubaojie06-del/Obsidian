# 01 视觉软件平台总览

> [!info] 核心结论
> 视觉软件平台用于完成 **相机采图、图像处理、检测判断、数据通讯、结果追溯**，是机器视觉系统的大脑。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> 相机负责“看见”，光源负责“看清”，视觉软件负责“判断”。  
> 视觉软件平台通常把相机、镜头、光源、算法、PLC、数据库、MES连接成完整检测系统。

> [!warning] 🟧 通用选型公式
> **视觉软件平台选型 = 检测任务 + 相机数量 + 算法能力 + 通讯接口 + 数据追溯 + 部署方式**

## ② 参考表

| 模块 | 作用 |
|---|---|
| **相机管理** | 连接相机、设置曝光、触发 |
| **图像处理** | 滤波、增强、二值化、边缘 |
| **视觉算法** | 定位、测量、缺陷、读码 |
| **流程编辑** | 搭建检测逻辑 |
| **通讯输出** | 连接PLC、机器人、MES |
| **数据追溯** | 保存图片、结果、报表 |

## ③ 计算/选型示例

- 已知：  
  产线需要检测产品有无缺件，并把OK/NG结果发给PLC。
- 公式：  
  **视觉系统 = 相机采图 + 软件判断 + PLC接收结果**
- 计算：  
  软件平台完成图像定位和缺件判断，最后输出OK/NG信号。
- 结论：  
  可选择支持 **相机采集、检测流程编辑、PLC通讯** 的视觉软件平台。

> [!tip] 记忆口诀
> 相机负责看，光源负责清，软件负责判。

# 02 底层原理

> [!info] 核心结论
> 视觉软件平台的本质，是把图像里的像素变化转化成可判断的工程结果。

## ① 分类/公式/规则

> [!note] 🟦 工作流程
> 视觉软件先获取图像，再通过算法提取边缘、颜色、灰度、轮廓、模板、码制等特征，最后根据规则输出检测结果。

> [!warning] 🟧 视觉处理逻辑
> **采图 → 预处理 → 定位 → 特征提取 → 判断 → 输出结果 → 保存追溯**

## ② 参考表

| 环节 | 说明 |
|---|---|
| **采图** | 从相机获取图像 |
| **预处理** | 提升图像质量 |
| **定位** | 找到产品或基准点 |
| **检测** | 测量尺寸、识别缺陷 |
| **判断** | 与阈值或标准比较 |
| **输出** | 发送OK/NG或数据 |

## ③ 计算/选型示例

- 已知：  
  产品位置每次有轻微偏移，需要检测孔位是否合格。
- 公式：  
  **稳定检测 = 先定位基准 + 再测量孔位**
- 计算：  
  如果不先定位，产品偏移会导致测量区域错位。
- 结论：  
  软件流程应先做 **定位工具**，再做 **测量工具**。

> [!tip] 记忆口诀
> 先定位，再检测；不定位，结果容易偏。

# 03 常见算法模块

> [!info] 核心结论
> 视觉软件平台常用算法包括定位、测量、匹配、读码、颜色、缺陷检测和深度学习。

## ① 分类/公式/规则

> [!note] 🟦 算法分类
> 规则清晰的检测，用传统算法更稳定；  
> 形态复杂、缺陷多变的检测，可评估深度学习算法。

> [!warning] 🟧 算法选择公式
> **算法选择 = 检测目标 + 图像特征 + 稳定性要求 + 节拍要求**

## ② 参考表

| 算法 | 适合任务 |
|---|---|
| **模板匹配** | 定位、找产品 |
| **边缘检测** | 尺寸、孔位、间距 |
| **Blob分析** | 面积、数量、有无 |
| **灰度检测** | 明暗差异判断 |
| **颜色识别** | 色差、混料、防错 |
| **读码算法** | 条码、二维码、DPM |
| **深度学习** | 复杂缺陷、外观检测 |

## ③ 计算/选型示例

- 已知：  
  需要检测产品表面随机划伤，划伤形状和位置不固定。
- 公式：  
  **复杂缺陷 = 样本变化大 + 规则难描述**
- 计算：  
  传统阈值和边缘算法可能难以覆盖全部缺陷形态。
- 结论：  
  可评估 **深度学习缺陷检测模块**，但需准备足够样本并现场验证。

> [!tip] 记忆口诀
> 规则清楚用传统，缺陷复杂看AI。

# 04 流程编辑与工具链

> [!info] 核心结论
> 视觉软件平台通常通过流程图或工具链方式，把采图、处理、判断、输出串起来。

## ① 分类/公式/规则

> [!note] 🟦 流程结构
> 一个完整视觉流程通常包括：  
> 触发采图、图像预处理、产品定位、检测测量、结果判断、通讯输出、图片保存。

> [!warning] 🟧 流程搭建公式
> **稳定流程 = 采图稳定 + 定位稳定 + 检测稳定 + 输出稳定**

## ② 参考表

| 工具 | 作用 |
|---|---|
| **采集工具** | 获取相机图像 |
| **ROI工具** | 限定检测区域 |
| **定位工具** | 修正产品偏移 |
| **测量工具** | 输出尺寸或坐标 |
| **逻辑工具** | 判断OK/NG |
| **通讯工具** | 发送结果给PLC/MES |

## ③ 计算/选型示例

- 已知：  
  产品每次进入相机视野时位置偏差约±5mm。
- 公式：  
  **检测区域 = 定位结果 + 相对偏移**
- 计算：  
  先找到产品基准点，再让检测ROI跟随基准点移动。
- 结论：  
  应使用 **定位补偿 + 跟随ROI**，避免固定检测框误判。

> [!tip] 记忆口诀
> 流程先搭骨架，ROI跟着产品走。

# 05 标定与坐标转换

> [!info] 核心结论
> 如果视觉软件要输出真实尺寸、机器人坐标或运动补偿，就必须做标定。

## ① 分类/公式/规则

> [!note] 🟦 标定作用
> 相机看到的是像素坐标，设备需要的是毫米坐标或机器人坐标。  
> 标定就是建立 **像素 → 毫米 → 机械坐标** 的换算关系。

> [!warning] 🟧 坐标换算公式
> **实际尺寸 = 像素数量 × 单像素尺寸**

## ② 参考表

| 标定类型 | 用途 |
|---|---|
| **像素标定** | 像素转毫米 |
| **畸变校正** | 修正镜头变形 |
| **九点标定** | 建立平面坐标 |
| **手眼标定** | 相机坐标转机器人坐标 |
| **基准点标定** | 建立设备零点 |
| **高度标定** | 3D或位移测量 |

## ③ 计算/选型示例

- 已知：  
  标定后 1px = **0.04mm**，软件测得两边缘距离为 **250px**。
- 公式：  
  **实际尺寸 = 250 × 0.04**
- 计算：  
  实际尺寸 = **10mm**
- 结论：  
  该产品宽度约为 **10mm**。

> [!tip] 记忆口诀
> 相机看像素，设备看毫米；不做标定，尺寸没意义。

# 06 配方与换型管理

> [!info] 核心结论
> 多产品共线时，视觉软件平台必须支持配方管理，避免换型后参数混乱。

## ① 分类/公式/规则

> [!note] 🟦 配方内容
> 配方通常包含相机参数、ROI位置、检测阈值、模板文件、输出格式、保存路径和产品标准值。

> [!warning] 🟧 配方管理公式
> **稳定换型 = 产品型号 + 对应配方 + 参数锁定 + 权限管理**

## ② 参考表

| 配方项 | 说明 |
|---|---|
| **曝光/增益** | 不同产品表面反光不同 |
| **ROI区域** | 检测位置不同 |
| **模板** | 产品外形不同 |
| **尺寸标准** | 公差不同 |
| **输出格式** | PLC/MES字段不同 |
| **保存路径** | 图片和数据归档 |

## ③ 计算/选型示例

- 已知：  
  A产品孔距标准为20mm，B产品孔距标准为25mm。
- 公式：  
  **检测标准 = 当前产品型号对应配方参数**
- 计算：  
  如果换型后仍使用A产品配方检测B产品，会误判NG。
- 结论：  
  换型时应由PLC、MES或操作员选择正确配方，并限制误改参数。

> [!tip] 记忆口诀
> 一种产品一套配方，换型先换参数。

# 07 与PLC通讯

> [!info] 核心结论
> 视觉软件平台接PLC时，要明确触发、忙碌、完成、OK/NG、数据结果和复位信号。

## ① 分类/公式/规则

> [!note] 🟦 通讯内容
> 简单项目可用IO传OK/NG；  
> 复杂项目需要通过TCP/IP、PROFINET、EtherNet/IP、Modbus等方式传坐标、尺寸、字符串和状态码。

> [!warning] 🟧 通讯握手公式
> **PLC触发 → 软件采图检测 → 输出完成 → PLC读取结果 → 复位等待下一次**

## ② 参考表

| 信号/数据 | 作用 |
|---|---|
| **Trigger触发** | PLC通知相机拍照 |
| **Busy忙碌** | 软件正在处理 |
| **Done完成** | 检测完成 |
| **OK/NG** | 判断结果 |
| **Data数据** | 坐标、尺寸、角度、码内容 |
| **Error错误码** | 通讯或检测异常 |

## ③ 计算/选型示例

- 已知：  
  软件检测时间为80ms，PLC节拍要求100ms。
- 公式：  
  **节拍余量 = PLC节拍 - 软件检测时间**
- 计算：  
  节拍余量 = 100 - 80 = **20ms**
- 结论：  
  理论上可满足节拍，但还需考虑触发延时、通讯时间和设备动作时间。

> [!tip] 记忆口诀
> 视觉和PLC要握手，触发完成别乱走。

# 08 与MES/数据库追溯

> [!info] 核心结论
> 视觉软件平台不仅能判断OK/NG，还能保存图片、结果、条码和时间信息，用于质量追溯。

## ① 分类/公式/规则

> [!note] 🟦 追溯内容
> 常见追溯数据包括产品码、检测时间、检测结果、测量值、缺陷图片、工位号、批次号、操作员和设备号。

> [!warning] 🟧 追溯公式
> **完整追溯 = 产品唯一ID + 检测结果 + 图片证据 + 时间戳 + 数据上传**

## ② 参考表

| 数据 | 用途 |
|---|---|
| **产品ID** | 关联单件产品 |
| **OK/NG结果** | 质量判定 |
| **测量值** | 尺寸趋势分析 |
| **缺陷图** | 复查问题 |
| **时间戳** | 追踪生产时间 |
| **批次号** | 批量质量分析 |

## ③ 计算/选型示例

- 已知：  
  每件产品都需要保存1张图片，单张图片大小为2MB，每天检测10000件。
- 公式：  
  **每日存储量 = 单张图片大小 × 产品数量**
- 计算：  
  每日存储量 = 2MB × 10000 = **20000MB ≈ 20GB**
- 结论：  
  每天约需 **20GB** 存储空间，需规划硬盘容量、保存周期和压缩策略。

> [!tip] 记忆口诀
> 追溯不是只存结果，图片数据都要管。

# 09 权限与参数保护

> [!info] 核心结论
> 视觉软件平台必须做好权限管理，防止操作员误改阈值、模板、配方和通讯参数。

## ① 分类/公式/规则

> [!note] 🟦 权限分层
> 常见权限包括操作员、工艺员、工程师、管理员。  
> 操作员只允许切换配方和查看结果；工程师才允许修改算法和参数。

> [!warning] 🟧 权限管理公式
> **稳定生产 = 参数锁定 + 分级权限 + 修改记录 + 配方备份**

## ② 参考表

| 权限项 | 建议 |
|---|---|
| **操作员** | 启停、查看结果、切换配方 |
| **工艺员** | 调整公差和少量参数 |
| **工程师** | 修改视觉流程 |
| **管理员** | 用户管理、系统配置 |
| **日志记录** | 记录谁改了什么 |
| **备份恢复** | 参数异常时可还原 |

## ③ 计算/选型示例

- 已知：  
  现场检测突然大量误判，发现阈值被误改。
- 公式：  
  **误判风险 = 参数可随意修改 + 无修改记录**
- 计算：  
  没有权限控制时，人员误改参数很难追溯。
- 结论：  
  应启用权限管理和参数备份，关键参数只允许授权人员修改。

> [!tip] 记忆口诀
> 参数不加锁，现场容易乱。

# 10 部署方式

> [!info] 核心结论
> 视觉软件平台可部署在智能相机、视觉控制器、工控机或服务器上，选择取决于任务复杂度和系统规模。

## ① 分类/公式/规则

> [!note] 🟦 部署选择
> 单工位简单检测可用智能相机内置软件；  
> 多相机、高算力、复杂算法、深度学习任务通常需要工控机或服务器。

> [!warning] 🟧 部署选择公式
> **部署方式 = 相机数量 + 算法复杂度 + 数据量 + 通讯对象 + 维护方式**

## ② 参考表

| 部署方式 | 适合场景 |
|---|---|
| **智能相机内置** | 单工位、简单检测 |
| **视觉控制器** | 多相机中等复杂度 |
| **工控机软件** | 复杂算法、多相机 |
| **服务器平台** | 大量数据、AI训练、集中管理 |
| **边缘计算盒** | 小型AI视觉部署 |
| **云平台** | 远程分析和大数据管理 |

## ③ 计算/选型示例

- 已知：  
  一个项目有8台相机，需要同时检测并保存图片。
- 公式：  
  **算力需求 = 相机数量 × 图像分辨率 × 检测复杂度 × 节拍**
- 计算：  
  多相机同时运行对CPU、内存、网卡和硬盘写入速度要求较高。
- 结论：  
  建议使用 **工控机视觉软件平台**，并校核硬件算力和网络带宽。

> [!tip] 记忆口诀
> 单机简单用智能，多机复杂上工控。

# 11 常见故障排查

> [!info] 核心结论
> 视觉软件平台异常时，按“相机连接 → 图像质量 → 算法结果 → 通讯输出 → 数据保存”的顺序排查。

## ① 分类/公式/规则

> [!note] 🟦 排查顺序
> 先看相机是否在线，再看图像是否正常；  
> 先确认软件检测结果，再检查PLC/MES是否收到；  
> 先查配置，再查硬件。

> [!warning] 🟧 排错公式
> **视觉异常 = 相机掉线 + 图像不稳 + 算法参数错误 + 通讯失败 + 存储异常**

## ② 参考表

| 问题 | 可能原因 |
|---|---|
| **相机离线** | IP、驱动、网线、供电异常 |
| **图像过暗/过亮** | 曝光、光源、增益不合适 |
| **误判/漏判** | ROI、阈值、模板、样品变化 |
| **PLC无结果** | 通讯地址、握手、数据格式错误 |
| **图片没保存** | 路径、权限、硬盘空间不足 |
| **运行卡顿** | 算力不足、图片过大、后台任务多 |

## ③ 计算/选型示例

- 已知：  
  软件检测正常，但图片保存失败。
- 公式：  
  **保存成功 = 路径存在 + 权限足够 + 硬盘空间足够**
- 计算：  
  如果硬盘满了或保存路径无权限，检测可正常但无法保存图片。
- 结论：  
  应检查保存路径、磁盘容量、用户权限和文件命名规则。

> [!tip] 记忆口诀
> 检测异常先看图，通讯异常看握手，保存异常看硬盘。

# 12 视觉软件平台快速选型表

> [!info] 核心结论
> 视觉软件平台选型先看检测任务和系统规模，再看算法能力、通讯能力、追溯能力和维护便利性。

## ① 分类/公式/规则

> [!note] 🟦 八步选型法
> ① 看检测任务  
> ② 看相机数量  
> ③ 看算法复杂度  
> ④ 看节拍速度  
> ⑤ 看PLC/机器人/MES接口  
> ⑥ 看图片和数据追溯  
> ⑦ 看权限和配方管理  
> ⑧ 看部署硬件和维护方式

> [!warning] 🟧 最终选型公式
> **推荐平台 = 算法模块 + 相机支持 + 通讯接口 + 配方管理 + 数据追溯 + 部署方式**

## ② 参考表

| 需求 | 推荐关注点 |
|---|---|
| 简单OK/NG检测 | 流程式视觉平台 |
| 尺寸测量 | 标定、测量工具 |
| 机器人定位 | 坐标转换、手眼标定 |
| 缺陷检测 | 传统算法或深度学习 |
| 读码追溯 | 读码模块、数据库接口 |
| 多相机系统 | 工控机平台、带宽和算力 |
| 换型频繁 | 配方管理和权限控制 |

## ③ 计算/选型示例

- 已知：  
  产线有4台相机，需完成定位、尺寸检测、OK/NG输出，并保存NG图片。
- 公式：  
  **平台需求 = 多相机 + 定位测量 + PLC通讯 + 图片追溯**
- 计算：  
  单一相机客户端可能只适合调试，不适合完整生产流程。
- 结论：  
  应选择支持 **多相机管理、视觉流程编辑、PLC通讯、NG图片保存** 的视觉软件平台。

> [!tip] 选型口诀
> 任务定算法，相机定规模；通讯接PLC，追溯看数据库；换型多，配方不能少。
