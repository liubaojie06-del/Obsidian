# 01 移动终端 / 手持终端总览

> [!info] 核心结论
> 工业移动终端 / 手持终端用于现场 **扫码、RFID读写、数据采集、设备巡检、仓储管理、生产报工、质量追溯和移动运维**。

## ① 分类/公式/规则

> [!warning] 选型公式
> **手持终端选型 = 应用场景 + 扫码/RFID能力 + 系统平台 + 通讯方式 + 防护等级 + 续航 + 软件兼容性**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **条码扫描** | 一维码、二维码、DPM码 |
| **RFID读写** | 高频HF、超高频UHF等 |
| **数据采集** | 工单、物料、批次、设备状态 |
| **移动通讯** | Wi-Fi、4G/5G、蓝牙、NFC |
| **系统平台** | Android、Windows、专用系统 |
| **防护能力** | 防摔、防尘、防水、耐油污 |
| **典型系统** | MES、WMS、ERP、CMMS、SCADA移动端 |

## ③ 计算/选型示例

- 已知：  
  仓库需要扫描物料二维码，并实时上传到WMS。
- 公式：  
  **扫码采集 + 实时上传 → 手持终端**
- 计算：  
  普通手机扫码稳定性、防护和工业软件适配可能不足。
- 结论：  
  推荐使用 **工业手持终端PDA**，并确认扫码头、Wi-Fi和WMS接口。

> [!tip] 记忆口诀
> 现场数据要移动采，扫码、联网、耐摔是关键。

# 02 底层原理

> [!info] 核心结论
> 手持终端的本质，是把现场条码、RFID、人工录入和传感数据采集后，通过无线网络传给后台系统。

## ① 分类/公式/规则

> [!note] 工作链路
> 操作员用终端扫描条码或读取RFID，终端软件解析数据，再通过Wi-Fi、4G/5G或蓝牙传到服务器、MES、WMS或ERP。

> [!warning] 数据链路
> **现场标识 → 终端采集 → 软件校验 → 无线传输 → 后台系统 → 业务确认**

## ② 参考表

| 环节 | 作用 |
|---|---|
| **条码/RFID** | 物料、产品、设备身份标识 |
| **扫描模块** | 读取一维码、二维码、DPM码 |
| **RFID模块** | 读取或写入电子标签 |
| **终端软件** | 解析、校验、录入业务数据 |
| **无线网络** | 上传和下载任务 |
| **后台系统** | 存储、追溯、统计、管理 |

## ③ 计算/选型示例

- 已知：  
  现场扫码成功，但后台没有记录。
- 公式：  
  **后台记录成功 = 扫码成功 + 软件提交成功 + 网络正常 + 系统写入成功**
- 计算：  
  不能只看扫码声音，还要看是否上传成功。
- 结论：  
  应检查网络、接口、账号权限和后台返回信息。

> [!tip] 记忆口诀
> 扫到码只是第一步，传到系统才算完成。

# 03 手持终端 vs 普通手机

> [!info] 核心结论
> 普通手机适合轻量移动应用，工业手持终端更适合高频扫码、恶劣环境、长续航和工业系统集成。

## ① 分类/公式/规则

> [!warning] 选择规则
> **低频扫码、环境普通 → 手机可评估**  
> **高频作业、仓储生产、恶劣环境 → 工业手持终端优先**

## ② 参考表

| 对比项 | 普通手机 | 工业手持终端 |
|---|---|---|
| 扫码效率 | 依赖摄像头 | **专业扫码头更快** |
| 防护能力 | 一般 | **防摔、防尘、防水更强** |
| 按键操作 | 少 | **可有实体按键** |
| 续航 | 一般 | **可换电池/大电池** |
| 工业接口 | 较少 | **可扩展RFID、NFC、蓝牙等** |
| 长期稳定性 | 普通消费级 | **工业现场更适合** |

## ③ 计算/选型示例

- 已知：  
  仓库每天要扫码上千次。
- 公式：  
  **高频扫码 → 专业扫码头优先**
- 计算：  
  手机摄像头扫码速度和瞄准体验不如专业扫码模组。
- 结论：  
  应选择工业手持终端，而不是普通手机。

> [!tip] 记忆口诀
> 偶尔扫用手机，天天扫用PDA。

# 04 常见类型

> [!info] 核心结论
> 手持终端可按形态分为工业PDA、手持扫码枪、RFID手持机、工业平板和车载终端。

## ① 分类/公式/规则

> [!warning] 类型选择规则
> **扫码+业务录入 → 工业PDA**  
> **只扫码输入电脑 → 扫码枪**  
> **远距离批量识别 → RFID手持机**  
> **大屏移动操作 → 工业平板**

## ② 参考表

| 类型 | 适合场景 |
|---|---|
| **工业PDA** | 仓储、生产、巡检、追溯 |
| **手持扫码枪** | 收银、工位扫码、固定电脑输入 |
| **RFID手持机** | 资产盘点、物流、服装、托盘识别 |
| **工业平板** | 大屏表单、移动点检、维修指导 |
| **车载终端** | 叉车、AGV调度、物流作业 |
| **防爆手持终端** | 化工、油气等危险区域 |

## ③ 计算/选型示例

- 已知：  
  需要在仓库盘点托盘，不想逐个对准条码扫描。
- 公式：  
  **批量非接触识别 → RFID手持机**
- 计算：  
  RFID可以在一定距离内读取标签，适合批量盘点。
- 结论：  
  推荐选择UHF RFID手持终端。

> [!tip] 记忆口诀
> 一个个扫看条码，一批批读看RFID。

# 05 条码扫描能力

> [!info] 核心结论
> 条码扫描能力是手持终端核心参数，重点看支持码制、扫码距离、识读速度、景深和对低质量条码的识别能力。

## ① 分类/公式/规则

> [!warning] 扫码选型公式
> **扫码能力 = 条码类型 + 条码尺寸 + 扫描距离 + 码面质量 + 环境光 + 扫码频率**

## ② 参考表

| 条码类型 | 说明 |
|---|---|
| **一维码** | 物流、物料、包装常见 |
| **二维码** | 容量大，追溯常用 |
| **DPM码** | 直接打标在金属/塑料表面 |
| **污损码** | 需较强算法和照明 |
| **反光码** | 需注意角度和光源 |
| **小尺寸码** | 需要高分辨率扫码头 |

## ③ 计算/选型示例

- 已知：  
  金属零件上激光打了DPM二维码。
- 公式：  
  **DPM码 → 需要支持DPM识读的扫码模组**
- 计算：  
  普通扫码头可能无法稳定读取低对比度打标码。
- 结论：  
  应选择支持DPM的工业手持终端。

> [!tip] 记忆口诀
> 纸箱码普通扫码头够，金属打标码要看DPM能力。

# 06 RFID功能

> [!info] 核心结论
> RFID手持终端可实现非接触识别和批量读取，适合资产管理、仓储盘点、物流追踪和工装夹具管理。

## ① 分类/公式/规则

> [!warning] RFID选型公式
> **RFID选型 = 频段 + 标签类型 + 读取距离 + 读取数量 + 环境材料 + 系统接口**

## ② 参考表

| RFID类型 | 特点 |
|---|---|
| **低频LF** | 距离短，抗液体金属影响相对好 |
| **高频HF/NFC** | 近距离识别，门禁、工卡常见 |
| **超高频UHF** | 距离较远，批量读取常用 |
| **无源标签** | 不带电池，成本低 |
| **有源标签** | 距离远，成本高 |
| **抗金属标签** | 适合金属资产表面 |

## ③ 计算/选型示例

- 已知：  
  要在1~3米范围内盘点仓库货架标签。
- 公式：  
  **中远距离批量读取 → UHF RFID**
- 计算：  
  HF/NFC读取距离较短，不适合远距离盘点。
- 结论：  
  推荐选择UHF RFID手持终端，并匹配UHF标签。

> [!tip] 记忆口诀
> 近距离刷卡看HF，远距离盘点看UHF。

# 07 通讯方式

> [!info] 核心结论
> 手持终端必须稳定联网，常见通讯方式有Wi-Fi、4G/5G、蓝牙、NFC、USB和串口底座。

## ① 分类/公式/规则

> [!warning] 通讯选型公式
> **通讯方式 = 作业范围 + 实时性 + 网络覆盖 + 数据量 + 后台系统接口**

## ② 参考表

| 通讯方式 | 适合场景 |
|---|---|
| **Wi-Fi** | 工厂、仓库内部实时联网 |
| **4G/5G** | 外勤、物流、厂外移动作业 |
| **蓝牙** | 连接打印机、电子秤、外设 |
| **NFC** | 近距离读卡、身份识别 |
| **USB** | 充电、调试、数据同步 |
| **底座通讯** | 批量充电、数据传输、固定工位 |

## ③ 计算/选型示例

- 已知：  
  终端在仓库角落经常上传失败。
- 公式：  
  **上传失败 = 网络覆盖不足 / 信号弱 / 漫游不稳定**
- 计算：  
  仓库货架和金属环境会影响Wi-Fi信号。
- 结论：  
  应优化AP覆盖，必要时使用4G/5G或离线缓存机制。

> [!tip] 记忆口诀
> 手持终端好不好用，网络覆盖很关键。

# 08 系统平台

> [!info] 核心结论
> 手持终端常见系统有Android、Windows和专用系统，选型要优先看业务软件是否兼容。

## ① 分类/公式/规则

> [!warning] 平台选择公式
> **系统平台 = 业务软件兼容性 + 开发语言 + 外设驱动 + 维护习惯 + 安全策略**

## ② 参考表

| 平台 | 特点 |
|---|---|
| **Android** | 主流手持终端常见，生态丰富 |
| **Windows CE/Windows Mobile** | 老旧系统，部分存量项目仍在用 |
| **Windows平板** | 适合PC软件、复杂表单 |
| **Linux/专用系统** | 特定设备或定制终端 |
| **浏览器应用** | 依赖Web系统，部署方便 |
| **原生App** | 性能和外设控制更好 |

## ③ 计算/选型示例

- 已知：  
  现有WMS移动端只支持Android。
- 公式：  
  **软件平台决定终端系统**
- 计算：  
  选择Windows终端会导致软件无法直接安装。
- 结论：  
  应选择Android工业手持终端，并确认系统版本兼容。

> [!tip] 记忆口诀
> 先看软件跑在哪，再选终端用什么系统。

# 09 防护等级与耐摔

> [!info] 核心结论
> 工业手持终端必须适应跌落、粉尘、水汽、油污、低温和高温等现场环境。

## ① 分类/公式/规则

> [!warning] 防护选型公式
> **环境适应性 = IP等级 + 跌落高度 + 工作温度 + 防尘防水 + 抗振抗冲击**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **IP54** | 基础防尘防溅 |
| **IP65/IP67** | 更适合工业现场 |
| **跌落等级** | 常见1.2m、1.5m、1.8m等 |
| **宽温范围** | 冷库、户外需关注 |
| **防油污** | 车间环境常见需求 |
| **屏幕防刮** | 高频触控和现场使用重要 |

## ③ 计算/选型示例

- 已知：  
  终端经常在仓库叉车和地面之间使用，容易掉落。
- 公式：  
  **高跌落风险 → 工业防摔终端**
- 计算：  
  普通手机跌落损坏概率高。
- 结论：  
  应选择具备工业防摔等级的手持终端，并配置保护套或手绳。

> [!tip] 记忆口诀
> 手持终端天天拿在手里，最先考验的就是耐摔。

# 10 电池与续航

> [!info] 核心结论
> 手持终端续航决定现场连续作业能力，重点看电池容量、可更换电池、快充和充电底座。

## ① 分类/公式/规则

> [!warning] 续航估算公式
> **理论续航 = 电池容量Wh ÷ 平均功耗W**

## ② 参考表

| 项目 | 作用 |
|---|---|
| **电池容量** | 决定续航基础 |
| **可拆卸电池** | 适合多班倒换电 |
| **快充功能** | 减少等待时间 |
| **充电底座** | 批量管理终端 |
| **低电报警** | 防止作业中断 |
| **热插拔电池** | 部分型号支持不断电换电 |

## ③ 计算/选型示例

- 已知：  
  电池容量为20Wh，平均功耗为4W。
- 公式：  
  **续航 = 20 ÷ 4**
- 计算：  
  理论续航 = **5小时**
- 结论：  
  如果现场需要8小时连续作业，应选择更大电池或配置备用电池。

> [!tip] 记忆口诀
> 单班看容量，多班看换电和充电底座。

# 11 屏幕与按键

> [!info] 核心结论
> 手持终端屏幕和按键影响现场操作效率，强光、手套、油污和高频扫码都要考虑。

## ① 分类/公式/规则

> [!warning] 操作设计规则
> **现场操作 = 屏幕可读 + 按键顺手 + 手套可用 + 扫码快捷**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **屏幕尺寸** | 4寸~6寸常见，工业平板更大 |
| **亮度** | 强光环境需高亮 |
| **触摸方式** | 电容屏、手套模式等 |
| **实体按键** | 高频扫码和确认更方便 |
| **扫码键** | 左右两侧或正面快捷键 |
| **防误触** | 现场移动作业重要 |

## ③ 计算/选型示例

- 已知：  
  操作员戴手套，频繁扫码并确认数量。
- 公式：  
  **手套操作 + 高频扫码 → 实体按键更可靠**
- 计算：  
  纯触摸操作在手套、油污环境下效率可能低。
- 结论：  
  推荐选择带实体扫码键和确认键的工业PDA。

> [!tip] 记忆口诀
> 办公看触摸体验，现场看能不能戴手套快速操作。

# 12 软件与业务系统

> [!info] 核心结论
> 手持终端价值主要来自业务软件，常对接MES、WMS、ERP、CMMS、QMS和资产管理系统。

## ① 分类/公式/规则

> [!warning] 软件集成公式
> **终端可用 = 业务App + 账号权限 + 数据接口 + 离线机制 + 外设驱动**

## ② 参考表

| 系统 | 典型功能 |
|---|---|
| **WMS** | 入库、出库、盘点、移库 |
| **MES** | 报工、追溯、工序采集 |
| **ERP** | 物料、订单、库存联动 |
| **CMMS** | 巡检、点检、维修工单 |
| **QMS** | 质量检验、缺陷记录 |
| **资产系统** | 设备、工装、固定资产盘点 |

## ③ 计算/选型示例

- 已知：  
  终端硬件扫码正常，但业务系统不能提交工单。
- 公式：  
  **业务提交 = 账号权限 + 工单状态 + 接口正常**
- 计算：  
  硬件正常不代表业务逻辑允许提交。
- 结论：  
  应检查账号权限、工单状态、网络接口和后台日志。

> [!tip] 记忆口诀
> 终端是工具，业务系统才决定流程怎么走。

# 13 离线缓存

> [!info] 核心结论
> 工业现场网络不稳定时，手持终端需要离线缓存，避免数据丢失和作业中断。

## ① 分类/公式/规则

> [!warning] 离线机制公式
> **可靠移动作业 = 本地缓存 + 网络恢复上传 + 重复提交防护 + 数据校验**

## ② 参考表

| 功能 | 作用 |
|---|---|
| **本地缓存** | 无网时保存数据 |
| **断点续传** | 网络恢复后继续上传 |
| **重复提交防护** | 防止同一单据多次入账 |
| **时间戳** | 记录实际作业时间 |
| **同步状态** | 显示已上传/未上传 |
| **异常提示** | 提醒用户补传失败记录 |

## ③ 计算/选型示例

- 已知：  
  巡检区域网络差，巡检记录经常丢失。
- 公式：  
  **弱网环境 → 离线缓存必需**
- 计算：  
  如果App只支持在线提交，网络中断会导致记录失败。
- 结论：  
  应选择支持离线巡检和补传机制的软件方案。

> [!tip] 记忆口诀
> 网络不稳不可怕，可怕的是没缓存。

# 14 安全与权限

> [!info] 核心结论
> 手持终端涉及生产、库存和质量数据，必须做好账号权限、数据加密、设备管理和防误操作。

## ① 分类/公式/规则

> [!warning] 安全公式
> **移动数据安全 = 账号权限 + 操作日志 + 数据加密 + 设备锁定 + 远程管理**

## ② 参考表

| 安全项 | 作用 |
|---|---|
| **账号登录** | 区分操作人员 |
| **角色权限** | 限制不同操作 |
| **操作日志** | 追溯谁在何时做了什么 |
| **数据加密** | 防止敏感数据泄露 |
| **设备锁定** | 防止丢失后被滥用 |
| **MDM管理** | 批量管理终端、远程配置 |

## ③ 计算/选型示例

- 已知：  
  终端丢失，里面保存了未上传的库存数据。
- 公式：  
  **移动设备丢失 → 需要远程锁定/擦除**
- 计算：  
  没有设备管理机制时，数据泄露风险高。
- 结论：  
  多终端项目建议配置MDM或远程管理方案。

> [!tip] 记忆口诀
> 手持终端会移动，权限和数据安全不能省。

# 16 常见故障排查

> [!info] 核心结论
> 手持终端故障排查按 **电量、扫码/RFID、网络、App、账号、后台接口、硬件损坏** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **终端异常 = 电量不足 + 扫码失败 + 网络异常 + App错误 + 权限不足 + 后台接口异常 + 硬件故障**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **无法开机** | 电池没电、电池接触不良、主板故障 |
| **扫码不识别** | 码污损、距离不对、扫码头故障、码制未开启 |
| **RFID读不到** | 频段不匹配、标签方向、距离、金属干扰 |
| **无法联网** | Wi-Fi弱、IP异常、SIM卡/流量问题 |
| **App闪退** | 版本不兼容、权限不足、系统异常 |
| **提交失败** | 后台接口、账号权限、网络中断 |
| **续航很短** | 电池老化、后台程序耗电、屏幕亮度过高 |

## ③ 计算/选型示例

- 已知：  
  同一个条码，部分终端能扫，部分终端不能扫。
- 公式：  
  **部分终端异常 = 扫码配置 / 码制支持 / 扫码头差异**
- 计算：  
  可能有些终端未开启该码制，或扫码头型号不同。
- 结论：  
  应检查扫码配置、App权限、硬件型号和固件版本。

> [!tip] 记忆口诀
> 扫不出先看码和扫码头，传不上先看网络和接口。

# 17 移动终端 / 手持终端快速选型表

> [!info] 核心结论
> 手持终端选型要从业务流程出发，先确定扫码/RFID和软件系统，再确认网络、防护、续航和外设。

## ① 分类/公式/规则

> [!note] 十步选型法
> ① 明确应用：仓储、生产、巡检、资产、物流  
> ② 确认识别方式：条码、二维码、DPM、RFID、NFC  
> ③ 确认业务系统：MES、WMS、ERP、CMMS等  
> ④ 确认系统平台：Android、Windows或Web  
> ⑤ 确认通讯方式：Wi-Fi、4G/5G、蓝牙  
> ⑥ 确认防护等级和跌落等级  
> ⑦ 确认电池续航和是否可换电池  
> ⑧ 确认屏幕、按键、手套操作需求  
> ⑨ 确认离线缓存、权限和数据安全  
> ⑩ 确认充电底座、打印机、保护套等外设

> [!warning] 最终选型公式
> **推荐手持终端 = 扫码/RFID能力 + 系统平台 + 通讯方式 + 防护等级 + 续航 + 软件兼容 + 外设配置**

## ② 参考表

| 需求 | 推荐关注点 |
|---|---|
| 仓储出入库 | 二维扫码、Wi-Fi、WMS |
| 资产盘点 | UHF RFID、长续航 |
| 生产追溯 | 条码/DPM、MES接口 |
| 设备巡检 | 大电池、离线缓存、防摔 |
| 冷库作业 | 宽温、电池低温性能 |
| 物流外勤 | 4G/5G、GPS、拍照 |
| 现场打印 | 蓝牙打印机 |
| 多终端管理 | MDM、权限、远程配置 |

## ③ 计算/选型示例

- 已知：  
  工厂需要30台终端用于仓库扫码、MES报工和设备巡检，要求Wi-Fi联网、可离线缓存、8小时续航。
- 公式：  
  **终端需求 = 扫码 + Wi-Fi + MES/WMS软件 + 离线缓存 + 8小时续航**
- 计算：  
  终端数量多，还要考虑充电底座、备用电池、账号权限和设备管理。
- 结论：  
  推荐选择 **工业PDA手持终端 + 专业二维扫码头 + Android系统 + Wi-Fi + 大容量电池 + 充电底座 + MDM管理方案**。

> [!tip] 选型口诀
> 先看业务，再看识别；条码看扫码头，资产看RFID；网络要覆盖，电池要撑班，软件要能对接。
