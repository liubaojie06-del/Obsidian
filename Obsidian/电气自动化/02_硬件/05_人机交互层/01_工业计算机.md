# 01 工业计算机总览

> [!info] 核心结论
> 工业计算机用于工业现场的 **数据采集、上位机监控、机器视觉、运动控制、边缘计算、设备联网和数据追溯**，相比普通电脑更强调稳定性、抗干扰、长期运行和工业接口。

## ① 分类/公式/规则

> [!warning] 选型公式
> **工业计算机选型 = CPU性能 + 内存容量 + 存储容量 + 接口数量 + 扩展槽 + 系统环境 + 防护等级 + 软件平台**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **IPC工控机** | 工业现场常用计算平台 |
| **嵌入式工控机** | 体积小、低功耗、适合设备集成 |
| **工业平板电脑** | 显示与计算一体化 |
| **边缘计算机** | 本地数据处理、AI推理、设备联网 |
| **视觉工控机** | 连接相机、光源、采集卡、算法软件 |
| **上位机** | 监控、报表、数据记录、工艺管理 |

## ③ 计算/选型示例

- 已知：  
  设备需要连接2台工业相机、1套PLC、1个数据库，并运行视觉检测软件。
- 公式：  
  **视觉 + 数据 + 通讯 → 工业计算机优先**
- 计算：  
  普通PLC不适合做复杂图像处理和数据库操作。
- 结论：  
  推荐选择 **工业计算机/视觉工控机**，并重点校核CPU、内存、相机接口和软件兼容性。

> [!tip] 记忆口诀
> PLC负责实时控制，工控机负责计算、视觉、数据和联网。

# 02 底层原理

> [!info] 核心结论
> 工业计算机的本质是工业级计算平台，通过操作系统和软件程序处理现场设备数据，再通过通讯接口与PLC、相机、仪表、数据库和上位系统交互。

## ① 分类/公式/规则

> [!note] 工作链路
> 工业计算机读取现场数据，运行上位机、视觉、算法、数据库或通讯软件，然后把结果传给PLC或MES系统。

> [!warning] 控制链路
> **现场设备 → 工业计算机采集 → 软件算法处理 → 输出结果 → PLC/设备执行 → 数据记录/上传**

## ② 参考表

| 环节 | 作用 |
|---|---|
| **输入数据** | 相机图像、传感器数据、PLC变量 |
| **软件处理** | 视觉算法、数据分析、报表统计 |
| **通讯接口** | 以太网、串口、USB、现场总线 |
| **输出结果** | OK/NG、坐标、报警、报表 |
| **数据存储** | 图片、日志、数据库、CSV文件 |
| **上位连接** | MES、SCADA、云平台、数据库 |

## ③ 计算/选型示例

- 已知：  
  相机拍照后，工控机需要判断产品OK/NG，并把结果发送给PLC。
- 公式：  
  **检测流程 = 采图 + 算法 + 结果通讯**
- 计算：  
  工控机完成图像处理，PLC根据OK/NG控制剔除机构。
- 结论：  
  工控机不一定直接控制执行机构，常作为计算和判断单元与PLC配合。

> [!tip] 记忆口诀
> 工控机负责算，PLC负责动；算完结果，再让设备执行。

# 03 工业计算机 vs 普通电脑

> [!info] 核心结论
> 工业计算机和普通电脑都能运行操作系统和软件，但工业计算机更适合长期运行、恶劣环境、工业接口和设备集成。

## ① 分类/公式/规则

> [!warning] 选择规则
> **办公应用 → 普通电脑**  
> **工业现场长期运行 → 工业计算机**

## ② 参考表

| 对比项 | 普通电脑 | 工业计算机 |
|---|---|---|
| 使用环境 | 办公环境 | 工业现场 |
| 稳定性 | 一般 | **更适合长期运行** |
| 抗干扰 | 较弱 | **更强** |
| 接口 | 办公接口为主 | **工业接口更多** |
| 扩展槽 | 视型号 | **可支持PCIe、采集卡等** |
| 维护周期 | 日常办公 | 工业设备长期运行 |
| 安装方式 | 桌面/机箱 | 机柜、DIN导轨、嵌入式 |

## ③ 计算/选型示例

- 已知：  
  设备需要7×24小时运行，并安装在控制柜内。
- 公式：  
  **长期运行 + 工业环境 → 工业计算机**
- 计算：  
  普通电脑散热、防尘、抗振和供电适应性可能不足。
- 结论：  
  推荐选择工控机，而不是普通办公电脑。

> [!tip] 记忆口诀
> 办公电脑怕现场，工业计算机为现场而生。

# 04 常见类型

> [!info] 核心结论
> 工业计算机常见有机架式工控机、嵌入式工控机、工业平板电脑、无风扇工控机、边缘计算机和视觉工控机。

## ① 分类/公式/规则

> [!warning] 类型选择规则
> **高扩展 → 机架式工控机**  
> **空间紧凑 → 嵌入式工控机**  
> **人机一体 → 工业平板电脑**  
> **粉尘环境 → 无风扇工控机**  
> **AI/数据处理 → 边缘计算机**

## ② 参考表

| 类型 | 适合场景 |
|---|---|
| **机架式工控机** | 需要多扩展卡、多硬盘、大功率计算 |
| **嵌入式工控机** | 设备内部集成，体积小 |
| **工业平板电脑** | 触控操作、HMI、上位机一体 |
| **无风扇工控机** | 粉尘、油污、维护困难环境 |
| **视觉工控机** | 工业相机、图像处理 |
| **边缘计算机** | AI推理、数据网关、设备联网 |

## ③ 计算/选型示例

- 已知：  
  设备需要安装图像采集卡和运动控制卡。
- 公式：  
  **需要扩展卡 → 选择带PCIe扩展槽的工控机**
- 计算：  
  嵌入式小主机可能没有足够扩展槽。
- 结论：  
  可选择机架式工控机或带PCIe扩展的工业计算机。

> [!tip] 记忆口诀
> 要扩展选机架，要紧凑选嵌入，要触控选平板。

# 05 CPU与算力

> [!info] 核心结论
> CPU决定工业计算机的运算能力，视觉检测、数据分析、AI推理和多任务软件对CPU性能要求更高。

## ① 分类/公式/规则

> [!warning] CPU选型公式
> **CPU性能需求 = 软件复杂度 + 图像处理量 + 数据处理量 + 并发任务数量 + 实时响应要求**

## ② 参考表

| 应用 | CPU关注点 |
|---|---|
| **简单上位机** | 中低端CPU即可 |
| **SCADA监控** | 稳定性和多任务 |
| **机器视觉** | 多核性能、主频、缓存 |
| **AI推理** | CPU/GPU/NPU协同 |
| **数据库记录** | 多任务和磁盘性能 |
| **多相机检测** | 高性能CPU或GPU |

## ③ 计算/选型示例

- 已知：  
  设备有4台相机同时检测，每个节拍500ms。
- 公式：  
  **多相机 + 短节拍 → 高算力需求**
- 计算：  
  若单次算法处理时间过长，会拖慢设备节拍。
- 结论：  
  应选择较高性能CPU，必要时增加GPU或选择视觉专用工控机。

> [!tip] 记忆口诀
> 画面越多、算法越重、节拍越快，CPU越不能省。

# 06 内存与存储

> [!info] 核心结论
> 内存影响软件运行和图像处理缓存，存储影响系统启动、数据记录、图片保存和长期追溯。

## ① 分类/公式/规则

> [!warning] 存储估算公式
> **数据量 = 单个文件大小 × 文件数量 × 保存周期**

## ② 参考表

| 项目 | 作用 |
|---|---|
| **内存RAM** | 软件运行、图像缓存、多任务 |
| **SSD固态硬盘** | 系统和数据存储，抗振较好 |
| **工业级SSD** | 更适合长期运行和宽温环境 |
| **双硬盘** | 系统盘和数据盘分离 |
| **RAID** | 提高可靠性或容量 |
| **外部存储** | U盘、移动硬盘、NAS、服务器 |

## ③ 计算/选型示例

- 已知：  
  每张图片2MB，每小时保存1000张，每天运行10小时。
- 公式：  
  **日数据量 = 2MB × 1000 × 10**
- 计算：  
  日数据量 = **20000MB ≈ 20GB**
- 结论：  
  若保存30天，需要约600GB空间，还需考虑压缩、清理策略和备份。

> [!tip] 记忆口诀
> 内存不够软件卡，硬盘不够数据丢。

# 07 工业接口

> [!info] 核心结论
> 工业计算机接口越丰富，越容易连接PLC、相机、扫码枪、传感器、运动卡、采集卡和上位系统。

## ① 分类/公式/规则

> [!warning] 接口选型公式
> **接口需求 = 设备数量 + 通讯类型 + 带宽要求 + 扩展余量**

## ② 参考表

| 接口 | 常见用途 |
|---|---|
| **LAN以太网** | PLC、相机、MES、SCADA |
| **USB** | 加密狗、相机、键鼠、U盘 |
| **RS232** | 扫码枪、打印机、仪表 |
| **RS485** | 变频器、仪表、Modbus设备 |
| **HDMI/DP/VGA** | 显示器 |
| **PCIe** | 采集卡、运动控制卡、网卡 |
| **GPIO/DIO** | 简单开关量输入输出 |

## ③ 计算/选型示例

- 已知：  
  设备需要连接2个网口相机、1台PLC和1个MES网络。
- 公式：  
  **网口数量 ≥ 2 + 1 + 1**
- 计算：  
  至少需要4个网口，或使用工业交换机合理规划网络。
- 结论：  
  应选择多网口工控机，或配置独立工业交换机。

> [!tip] 记忆口诀
> 接口不够，现场最麻烦；选型时先把设备都数清。

# 08 工业通讯

> [!info] 核心结论
> 工业计算机常通过工业协议与PLC、控制器、仪表、数据库和MES通讯。

## ① 分类/公式/规则

> [!warning] 通讯成功公式
> **通讯成功 = 协议一致 + 网络正常 + 地址正确 + 数据格式匹配 + 权限允许**

## ② 参考表

| 协议 | 常见对象 |
|---|---|
| **Modbus TCP/RTU** | 仪表、PLC、变频器 |
| **OPC UA** | PLC、SCADA、MES、数据平台 |
| **TCP/UDP Socket** | 自定义设备通讯 |
| **Profinet/EtherNet/IP** | 工业控制网络，需要对应软硬件支持 |
| **MQTT** | 边缘数据上传 |
| **数据库接口** | SQL Server、MySQL、SQLite等 |
| **HTTP/API** | MES、Web服务、数据平台 |

## ③ 计算/选型示例

- 已知：  
  工控机需要读取PLC数据，并上传到MES。
- 公式：  
  **下位通讯 + 上位通讯 = 双向数据接口**
- 计算：  
  工控机向下可用OPC UA或Modbus读取PLC，向上可用数据库或API上传MES。
- 结论：  
  选型时要确认工控机软件和系统支持对应通讯协议。

> [!tip] 记忆口诀
> 工控机是数据中转站，下面连设备，上面连系统。

# 09 操作系统与软件平台

> [!info] 核心结论
> 工业计算机常用Windows、Linux或实时系统，选型要根据上位机软件、视觉软件、驱动程序和现场维护能力决定。

## ① 分类/公式/规则

> [!warning] 系统选择公式
> **操作系统 = 软件兼容性 + 驱动支持 + 实时要求 + 维护习惯 + 安全策略**

## ② 参考表

| 系统 | 特点 |
|---|---|
| **Windows** | 兼容上位机、视觉、数据库软件多 |
| **Windows IoT** | 工业嵌入式场景常见 |
| **Linux** | 稳定、开放、适合边缘计算和服务器应用 |
| **实时Linux** | 对实时任务更友好 |
| **专用实时系统** | 用于高实时控制场景 |
| **虚拟化/容器** | 适合复杂软件部署和隔离 |

## ③ 计算/选型示例

- 已知：  
  视觉软件只支持Windows系统。
- 公式：  
  **软件平台决定操作系统**
- 计算：  
  如果选择Linux工控机，视觉软件无法直接运行。
- 结论：  
  应选择支持Windows的工业计算机，并确认授权和驱动兼容。

> [!tip] 记忆口诀
> 先看软件跑在哪，再选工控机装什么系统。

# 10 机器视觉应用

> [!info] 核心结论
> 工业计算机是机器视觉系统的核心计算平台，负责图像采集、算法处理、结果输出和图片保存。

## ① 分类/公式/规则

> [!warning] 视觉节拍公式
> **视觉总时间 = 曝光时间 + 传输时间 + 算法时间 + 通讯输出时间**

## ② 参考表

| 视觉任务 | 工控机作用 |
|---|---|
| **尺寸测量** | 计算长度、角度、间隙 |
| **缺陷检测** | 划伤、脏污、缺料、破损 |
| **定位引导** | 输出X/Y/角度补正 |
| **条码识别** | 读取二维码、条形码 |
| **OCR识别** | 识别字符和批号 |
| **图片追溯** | 保存OK/NG图片和结果 |

## ③ 计算/选型示例

- 已知：  
  设备节拍800ms，视觉曝光50ms，图像传输80ms，算法处理300ms，结果通讯50ms。
- 公式：  
  **视觉总时间 = 50 + 80 + 300 + 50**
- 计算：  
  视觉总时间 = **480ms**
- 结论：  
  理论满足800ms节拍，但还要留机械动作和PLC响应余量。

> [!tip] 记忆口诀
> 视觉不是只看算法，曝光、传输、通讯都占时间。

# 11 数据采集与追溯

> [!info] 核心结论
> 工业计算机适合做数据采集和追溯，能记录设备状态、报警、产量、质量、图片、工艺参数和操作日志。

## ① 分类/公式/规则

> [!warning] 数据追溯公式
> **追溯数据 = 时间戳 + 产品编号 + 工艺参数 + 检测结果 + 操作记录**

## ② 参考表

| 数据类型 | 用途 |
|---|---|
| **报警记录** | 故障分析 |
| **生产记录** | 产量统计 |
| **质量结果** | OK/NG追溯 |
| **工艺参数** | 追踪生产条件 |
| **图片数据** | 视觉检测证据 |
| **操作日志** | 追踪人为修改 |
| **设备状态** | 稼动率、停机分析 |

## ③ 计算/选型示例

- 已知：  
  每个产品保存1条记录，每天生产50000件，每条记录200字节。
- 公式：  
  **日数据量 = 50000 × 200**
- 计算：  
  日数据量 = **10,000,000字节 ≈ 10MB**
- 结论：  
  纯文本记录不大，但如果保存图片，存储量会快速增加。

> [!tip] 记忆口诀
> 追溯不能只存结果，还要存时间、编号和关键参数。

# 12 边缘计算与AI

> [!info] 核心结论
> 边缘工业计算机可在设备现场完成AI推理、异常检测、数据压缩、协议转换和本地决策。

## ① 分类/公式/规则

> [!warning] 边缘计算公式
> **边缘价值 = 本地快速处理 + 减少上传数据 + 提高设备自治能力**

## ② 参考表

| 功能 | 应用 |
|---|---|
| **AI视觉检测** | 缺陷识别、分类判断 |
| **预测维护** | 振动、电流、温度趋势分析 |
| **协议网关** | 不同设备协议转换 |
| **数据清洗** | 过滤异常值、压缩数据 |
| **本地缓存** | 网络异常时暂存数据 |
| **实时报警** | 本地快速判断异常 |

## ③ 计算/选型示例

- 已知：  
  设备每秒产生大量原始数据，但MES只需要统计结果和报警。
- 公式：  
  **本地处理 → 上传结果**
- 计算：  
  工业计算机先在本地计算平均值、最大值、报警状态，再上传摘要。
- 结论：  
  边缘计算可降低网络负载和服务器压力。

> [!tip] 记忆口诀
> 能在设备边上算完的，就别全丢到云端。

# 13 安装与散热

> [!info] 核心结论
> 工业计算机长期稳定运行，关键看安装方式、散热、电源、接地、防尘和抗振。

## ① 分类/公式/规则

> [!warning] 稳定运行公式
> **稳定运行 = 合理散热 + 稳定电源 + 可靠接地 + 防尘防振 + 线缆固定**

## ② 参考表

| 项目 | 注意 |
|---|---|
| **安装方式** | 机柜、壁挂、DIN导轨、嵌入式 |
| **散热方式** | 风扇散热、无风扇散热 |
| **环境温度** | 高温需降额或加散热 |
| **电源输入** | DC24V或AC供电，需稳定 |
| **接地** | 防静电和抗干扰 |
| **防尘** | 粉尘环境优先无风扇 |
| **抗振** | 固态硬盘和可靠固定 |

## ③ 计算/选型示例

- 已知：  
  工控机安装在密闭电柜内，柜内温度长期偏高。
- 公式：  
  **温度过高 → 寿命下降 + 死机风险增加**
- 计算：  
  CPU、SSD、电源都会受温度影响。
- 结论：  
  应增加柜内散热、选择宽温工控机，必要时使用无风扇低功耗型号。

> [!tip] 记忆口诀
> 工控机怕热、怕灰、怕电源不稳。

# 14 可靠性与维护

> [!info] 核心结论
> 工业计算机维护重点是系统备份、数据备份、硬盘健康、风扇状态、软件授权、杀毒白名单和远程维护。

## ① 分类/公式/规则

> [!warning] 维护公式
> **可恢复系统 = 系统镜像 + 软件安装包 + 参数备份 + 数据备份 + 授权备份**

## ② 参考表

| 维护项 | 作用 |
|---|---|
| **系统镜像** | 系统崩溃后快速恢复 |
| **软件备份** | 防止重装困难 |
| **参数备份** | 工艺参数、相机标定、通讯配置 |
| **数据库备份** | 防止追溯数据丢失 |
| **硬盘健康检测** | 提前发现SSD老化 |
| **风扇清理** | 防止过热 |
| **远程维护** | 降低现场排障成本 |

## ③ 计算/选型示例

- 已知：  
  工控机硬盘损坏后，现场无法恢复视觉软件和参数。
- 公式：  
  **无备份 = 恢复时间不可控**
- 计算：  
  重新安装系统、软件、相机驱动和标定参数可能耗时很长。
- 结论：  
  新设备交付时必须做系统镜像和参数备份。

> [!tip] 记忆口诀
> 工控机最怕不是坏，而是坏了没备份。

# 15 常见故障排查

> [!info] 核心结论
> 工业计算机故障排查按 **电源、显示、系统、接口、通讯、软件、硬件温度、存储** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **工控机异常 = 电源异常 + 系统故障 + 驱动错误 + 通讯异常 + 软件崩溃 + 温度过高 + 硬盘问题**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **不开机** | 电源、主板、内存、硬盘故障 |
| **开机无显示** | 显示线、显卡、分辨率、显示器问题 |
| **系统卡顿** | CPU占用高、内存不足、硬盘异常 |
| **软件打不开** | 授权、路径、依赖库、系统权限 |
| **相机掉线** | 网线、IP冲突、供电、驱动 |
| **PLC通讯失败** | IP、协议、端口、防火墙 |
| **频繁死机** | 过热、电源波动、硬盘故障 |
| **数据丢失** | 存储满、数据库损坏、未备份 |

## ③ 计算/选型示例

- 已知：  
  工控机能开机，但视觉软件运行一段时间后死机。
- 公式：  
  **运行后死机 = 温度过高 / 内存泄漏 / 硬盘异常 / 软件异常**
- 计算：  
  如果刚开机正常、运行后异常，优先查看CPU温度、内存占用和软件日志。
- 结论：  
  应检查散热、系统日志、软件版本、驱动和硬盘健康状态。

> [!tip] 记忆口诀
> 开不了查电源，运行卡查资源，通讯断查网络，运行久死机先查温度。

# 16 工业计算机快速选型表

> [!info] 核心结论
> 工业计算机选型要从软件需求和现场环境出发，再确认CPU、内存、硬盘、接口、扩展、系统和维护方案。

## ① 分类/公式/规则

> [!note] 十步选型法
> ① 明确用途：上位机、视觉、数据、边缘计算  
> ② 确认软件系统：Windows、Linux、实时系统  
> ③ 确认CPU性能需求  
> ④ 确认内存容量  
> ⑤ 确认硬盘容量和工业级要求  
> ⑥ 统计以太网、USB、串口、显示接口数量  
> ⑦ 确认是否需要PCIe扩展卡  
> ⑧ 确认安装方式和尺寸  
> ⑨ 确认温度、防尘、抗振和供电条件  
> ⑩ 规划系统镜像、数据备份和远程维护

> [!warning] 最终选型公式
> **推荐工业计算机 = CPU + 内存 + SSD + 接口 + 扩展槽 + 操作系统 + 安装方式 + 环境等级**

## ② 参考表

| 需求 | 推荐关注点 |
|---|---|
| 简单上位机 | 中低配嵌入式工控机 |
| 机器视觉 | 高性能CPU、多网口、大内存 |
| AI检测 | GPU/NPU边缘计算机 |
| 多扩展卡 | 机架式工控机 |
| 粉尘环境 | 无风扇工控机 |
| 触控操作 | 工业平板电脑 |
| 数据追溯 | 大容量SSD、数据库、备份 |
| 长期运行 | 工业级电源、散热和系统镜像 |

## ③ 计算/选型示例

- 已知：  
  项目需要2台网口相机、1台PLC、1个MES网络、保存检测图片30天，每天约20GB。
- 公式：  
  **网口数量 ≥ 2相机 + 1 PLC + 1 MES**  
  **存储容量 ≥ 20GB × 30天**
- 计算：  
  网口至少4个；存储至少600GB，建议选1TB以上SSD并留余量。
- 结论：  
  推荐选择 **多网口、高性能CPU、16GB以上内存、1TB以上SSD的工业计算机**，并做好图片清理和备份策略。

> [!tip] 选型口诀
> 先看软件，再看算力；先数接口，再算存储；现场热就重散热，数据重要就做备份。
