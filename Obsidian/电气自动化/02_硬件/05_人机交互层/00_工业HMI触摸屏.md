# 01 工业HMI触摸屏总览

> [!info] 核心结论
> 工业HMI触摸屏用于实现 **设备操作、状态显示、参数设定、报警提示、配方管理、数据记录和PLC通讯**，是操作人员与自动化设备交互的窗口。

## ① 分类/公式/规则

> [!warning] 选型公式
> **HMI选型 = 屏幕尺寸 + 分辨率 + 通讯协议 + PLC品牌 + I/O/数据需求 + 防护等级 + 安装环境 + 软件功能**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **显示功能** | 显示设备状态、数据、报警 |
| **操作功能** | 启动、停止、复位、手动动作 |
| **参数功能** | 设置速度、时间、温度、压力 |
| **报警功能** | 显示故障原因和处理提示 |
| **配方功能** | 多产品参数切换 |
| **通讯功能** | 与PLC、变频器、仪表、控制器通讯 |
| **数据功能** | 记录产量、报警、趋势和工艺参数 |

## ③ 计算/选型示例

- 已知：  
  一台设备需要显示运行状态、设置参数、查看报警，并连接PLC。
- 公式：  
  **设备操作 + 参数设定 + PLC通讯 → 工业HMI**
- 计算：  
  普通按钮和指示灯无法方便显示复杂参数和报警信息。
- 结论：  
  应配置工业HMI触摸屏，并确认通讯协议与PLC匹配。

> [!tip] 记忆口诀
> PLC负责控制，HMI负责让人看得懂、调得动、查得到。

# 02 底层原理

> [!info] 核心结论
> HMI本质上是通过通讯读取和写入PLC或控制器的数据寄存器，再把这些数据用画面、按钮、数值框、报警和趋势显示出来。

## ① 分类/公式/规则

> [!note] 工作链路
> HMI读取PLC变量，显示设备状态；  
> 操作人员点击按钮或输入参数后，HMI把数据写入PLC；  
> PLC根据这些数据执行控制逻辑。

> [!warning] 控制链路
> **操作人员 → HMI画面 → 通讯写入PLC → PLC程序执行 → 设备动作 → HMI显示反馈**

## ② 参考表

| 环节 | 作用 |
|---|---|
| **画面按钮** | 给PLC写入启动、停止、复位等信号 |
| **数值显示** | 读取PLC实时数据 |
| **参数输入** | 修改PLC内部设定值 |
| **报警显示** | 根据PLC报警位提示故障 |
| **趋势曲线** | 记录并显示连续变量变化 |
| **通讯驱动** | 实现HMI与PLC数据交换 |

## ③ 计算/选型示例

- 已知：  
  HMI画面上点击“启动”，设备没有动作。
- 公式：  
  **HMI按钮有效 = 画面变量地址正确 + 通讯正常 + PLC逻辑允许**
- 计算：  
  需要检查HMI按钮写入地址、PLC是否收到信号、启动条件是否满足。
- 结论：  
  HMI按钮不动作时，不能只查画面，也要查通讯和PLC程序互锁。

> [!tip] 记忆口诀
> HMI只是发命令，PLC才决定能不能执行。

# 03 工业HMI与普通屏幕区别

> [!info] 核心结论
> 工业HMI不是普通显示器，它带有工业通讯、变量绑定、报警、配方、数据记录和现场抗干扰能力。

## ① 分类/公式/规则

> [!warning] 区别规则
> **普通显示器 = 显示图像**  
> **工业HMI = 显示 + 操作 + 通讯 + 数据 + 报警**

## ② 参考表

| 对比项 | 普通显示器 | 工业HMI |
|---|---|---|
| 显示 | 有 | 有 |
| 触控操作 | 视型号 | 常见 |
| PLC通讯 | 通常没有 | **有** |
| 报警管理 | 没有 | **有** |
| 配方管理 | 没有 | **有** |
| 工业防护 | 较弱 | **较强** |
| 现场抗干扰 | 较弱 | **更适合工业现场** |

## ③ 计算/选型示例

- 已知：  
  设备需要直接读写PLC寄存器。
- 公式：  
  **读写PLC数据 → 需要工业HMI或上位机软件**
- 计算：  
  普通显示器不能直接和PLC进行变量通讯。
- 结论：  
  需要工业HMI触摸屏，而不是普通电脑显示器。

> [!tip] 记忆口诀
> 普通屏只能看，HMI能看、能按、能和PLC说话。

# 04 屏幕尺寸与分辨率

> [!info] 核心结论
> HMI尺寸决定操作空间，分辨率决定画面细腻度，选型时要兼顾可读性、按钮大小和安装空间。

## ① 分类/公式/规则

> [!warning] 尺寸选择规则
> **参数少、按钮少 → 小尺寸**  
> **数据多、趋势多、画面复杂 → 大尺寸高分辨率**

## ② 参考表

| 尺寸 | 常见应用 |
|---|---|
| **4.3寸** | 简单小设备、少量参数 |
| **7寸** | 单机设备常用 |
| **10寸** | 中型设备、画面较多 |
| **12寸以上** | 产线、复杂设备、数据监控 |
| **宽屏** | 趋势、流程图显示更方便 |
| **高分辨率** | 适合复杂画面和精细图表 |

## ③ 计算/选型示例

- 已知：  
  设备有20多个参数、10多个报警页面，还要显示趋势曲线。
- 公式：  
  **画面复杂 → 大尺寸HMI**
- 计算：  
  7寸屏可能显示拥挤，按钮和文字过小。
- 结论：  
  可优先选择 **10寸或以上HMI**。

> [!tip] 记忆口诀
> 屏幕小省钱，屏幕大好操作；参数越多，屏幕越不能太小。

# 05 通讯协议

> [!info] 核心结论
> HMI能否正常工作，关键看是否支持PLC或控制器的通讯协议。

## ① 分类/公式/规则

> [!warning] 通讯成功公式
> **通讯成功 = 协议一致 + 地址正确 + 站号正确 + 参数一致 + 接线正确**

## ② 参考表

| 通讯协议 | 常见对象 |
|---|---|
| **Modbus RTU** | 仪表、变频器、小型PLC |
| **Modbus TCP** | 以太网PLC、控制器、网关 |
| **PROFINET/S7协议** | 西门子PLC |
| **EtherNet/IP** | 罗克韦尔、部分工业设备 |
| **Mitsubishi协议** | 三菱PLC |
| **Omron FINS** | 欧姆龙PLC |
| **串口协议** | RS232/RS485设备 |
| **OPC UA** | 上位系统、数据平台 |

## ③ 计算/选型示例

- 已知：  
  PLC支持Modbus TCP，HMI也支持Modbus TCP。
- 公式：  
  **协议匹配 → 可通讯**
- 计算：  
  还需设置PLC IP、端口号、站号和寄存器地址。
- 结论：  
  通讯协议匹配只是第一步，地址和参数也必须正确。

> [!tip] 记忆口诀
> 通讯不通，先查协议，再查地址、站号、线缆和端口。

# 06 变量与地址绑定

> [!info] 核心结论
> HMI画面上的按钮、指示灯、数值框都必须绑定PLC变量地址，地址错误会导致显示或操作异常。

## ① 分类/公式/规则

> [!warning] 变量绑定规则
> **HMI对象 = 画面控件 + PLC变量地址 + 数据类型 + 读写权限**

## ② 参考表

| HMI对象 | 绑定内容 |
|---|---|
| **按钮** | 写入PLC启动、停止、复位位 |
| **指示灯** | 读取PLC状态位 |
| **数值显示** | 读取PLC寄存器 |
| **数值输入** | 写入PLC参数寄存器 |
| **报警列表** | 读取PLC报警位 |
| **趋势曲线** | 读取连续变量 |
| **配方表** | 读写多组参数 |

## ③ 计算/选型示例

- 已知：  
  HMI显示温度为异常大数。
- 公式：  
  **显示异常 = 地址错误 / 数据类型错误 / 字节序错误 / 量程换算错误**
- 计算：  
  例如PLC是浮点数，HMI按整数读取，就会显示错误。
- 结论：  
  应检查变量地址、数据类型、字节顺序和工程量换算。

> [!tip] 记忆口诀
> 画面只是外壳，变量地址才是灵魂。

# 07 按钮与权限

> [!info] 核心结论
> HMI按钮用于操作设备，必须设计权限和确认机制，防止误操作造成设备异常或安全风险。

## ① 分类/公式/规则

> [!warning] 操作安全规则
> **危险操作 = 权限限制 + 二次确认 + PLC互锁条件**

## ② 参考表

| 操作类型 | 建议设计 |
|---|---|
| **启动** | 检查安全门、急停、原点、报警 |
| **停止** | 可直接操作 |
| **复位** | 报警清除前需确认条件 |
| **手动动作** | 需要手动模式和互锁 |
| **参数修改** | 需要权限 |
| **清零数据** | 需要二次确认 |
| **维护操作** | 需要工程师权限 |

## ③ 计算/选型示例

- 已知：  
  操作员误点“清除产量”，导致生产统计丢失。
- 公式：  
  **关键数据操作 → 权限 + 二次确认**
- 计算：  
  清零按钮应弹出确认窗口，并限制权限。
- 结论：  
  HMI设计时不要把关键操作直接暴露给所有用户。

> [!tip] 记忆口诀
> 普通操作给按钮，危险操作加权限，关键操作要确认。

# 08 报警显示

> [!info] 核心结论
> HMI报警系统应显示故障名称、发生时间、当前状态、处理提示和复位条件。

## ① 分类/公式/规则

> [!warning] 报警设计公式
> **报警信息 = 报警位 + 报警文本 + 时间戳 + 处理建议 + 复位逻辑**

## ② 参考表

| 报警内容 | 作用 |
|---|---|
| **报警名称** | 告诉发生了什么 |
| **报警时间** | 追溯发生时刻 |
| **报警等级** | 区分提示、警告、停机 |
| **报警状态** | 当前、已确认、历史 |
| **处理建议** | 指导维修人员排查 |
| **复位条件** | 防止未处理就复位 |
| **历史记录** | 分析重复故障 |

## ③ 计算/选型示例

- 已知：  
  设备只显示“故障”，没有具体原因。
- 公式：  
  **报警无细分 → 排查效率低**
- 计算：  
  操作员不知道是气压不足、伺服报警还是安全门打开。
- 结论：  
  HMI报警应按具体故障点显示，例如“气缸伸出超时”“真空不到位”。

> [!tip] 记忆口诀
> 好报警不是只说坏了，而是告诉哪里坏、怎么查。

# 09 参数设定

> [!info] 核心结论
> HMI参数设定用于修改设备运行参数，必须设置范围限制、单位说明和权限保护。

## ① 分类/公式/规则

> [!warning] 参数保护规则
> **参数可修改 = 合理范围 + 单位清晰 + 权限控制 + 保存机制**

## ② 参考表

| 参数类型 | 示例 |
|---|---|
| **时间参数** | 延时、超时时间 |
| **速度参数** | 伺服速度、输送速度 |
| **压力参数** | 夹紧压力、真空阈值 |
| **温度参数** | 设定温度、报警温度 |
| **计数参数** | 目标数量、批次数 |
| **位置参数** | 取料点、放料点、补偿量 |
| **工艺参数** | 配方、比例、补偿值 |

## ③ 计算/选型示例

- 已知：  
  气缸伸出超时时间正常应为3s，操作员误设为0.1s。
- 公式：  
  **参数范围不合理 → 误报警**
- 计算：  
  0.1s小于气缸实际动作时间，PLC会频繁超时报警。
- 结论：  
  HMI参数输入应限制范围，例如1~10s。

> [!tip] 记忆口诀
> 参数能改是方便，范围不控就是隐患。

# 10 配方管理

> [!info] 核心结论
> 配方管理用于保存不同产品的工艺参数，实现快速切换、减少人工输入错误。

## ① 分类/公式/规则

> [!warning] 配方公式
> **配方 = 产品编号 + 多个工艺参数 + 下载到PLC + 当前配方确认**

## ② 参考表

| 配方内容 | 示例 |
|---|---|
| **产品编号** | A产品、B产品、C产品 |
| **位置参数** | 取放点、补偿值 |
| **速度参数** | 运行速度、加速度 |
| **时间参数** | 延时、保压时间 |
| **温度参数** | 设定温度 |
| **压力参数** | 夹紧压力 |
| **计数参数** | 每批数量 |

## ③ 计算/选型示例

- 已知：  
  一台设备生产5种产品，每种产品有20个参数。
- 公式：  
  **参数数量 = 5 × 20**
- 计算：  
  共需管理 **100个参数值**。
- 结论：  
  应使用HMI配方功能，而不是让操作员逐项手动输入。

> [!tip] 记忆口诀
> 产品多、参数多，就上配方；切换快，出错少。

# 11 趋势曲线

> [!info] 核心结论
> 趋势曲线用于观察温度、压力、流量、速度、电流等连续变量随时间变化，适合调试和故障分析。

## ① 分类/公式/规则

> [!warning] 趋势设计公式
> **趋势曲线 = 变量 + 采样周期 + 时间范围 + 上下限 + 数据保存**

## ② 参考表

| 趋势变量 | 用途 |
|---|---|
| **温度** | 观察升温、保温、超温 |
| **压力** | 判断气压、液压是否稳定 |
| **真空值** | 判断吸附稳定性 |
| **速度** | 分析节拍和运动状态 |
| **电流** | 判断电机负载 |
| **流量** | 判断耗气、耗水或泄漏 |

## ③ 计算/选型示例

- 已知：  
  需要记录压力变化，每秒采样1次，显示10分钟。
- 公式：  
  **采样点数 = 采样频率 × 时间**
- 计算：  
  1 × 600 = **600点**
- 结论：  
  HMI趋势曲线至少需要保存600个采样点。

> [!tip] 记忆口诀
> 报警告诉你出事了，趋势告诉你怎么出事的。

# 12 数据记录与导出

> [!info] 核心结论
> HMI可记录报警、产量、工艺参数和趋势数据，部分型号支持U盘、SD卡、以太网或数据库导出。

## ① 分类/公式/规则

> [!warning] 数据记录公式
> **数据量 = 变量数量 × 单个数据大小 × 采样次数**

## ② 参考表

| 数据类型 | 作用 |
|---|---|
| **报警历史** | 故障追溯 |
| **生产数量** | 产量统计 |
| **工艺参数** | 质量追溯 |
| **趋势数据** | 分析过程变化 |
| **操作记录** | 追踪人为修改 |
| **配方记录** | 产品参数管理 |
| **导出文件** | CSV、Excel、数据库等 |

## ③ 计算/选型示例

- 已知：  
  每秒记录20个变量，每个变量4字节，记录8小时。
- 公式：  
  **数据量 = 20 × 4 × 28800**
- 计算：  
  数据量 = **2,304,000字节 ≈ 2.3MB**
- 结论：  
  数据量不大，但长期保存要考虑存储容量、文件数量和导出方式。

> [!tip] 记忆口诀
> 数据不只是显示，还要能保存、能导出、能追溯。

# 13 画面设计

> [!info] 核心结论
> HMI画面设计要做到清晰、分层、少误操作、易排查，不能只追求炫酷。

## ① 分类/公式/规则

> [!warning] 画面设计规则
> **好画面 = 状态清楚 + 操作明确 + 报警醒目 + 参数有单位 + 权限有边界**

## ② 参考表

| 画面类型 | 作用 |
|---|---|
| **首页总览** | 显示设备总体状态 |
| **手动画面** | 单动作调试 |
| **自动画面** | 显示流程和节拍 |
| **参数画面** | 设置运行参数 |
| **报警画面** | 查看当前和历史报警 |
| **趋势画面** | 查看连续变量变化 |
| **维护画面** | I/O监控、版本、权限、备份 |

## ③ 计算/选型示例

- 已知：  
  操作员经常找不到报警复位按钮。
- 公式：  
  **高频操作 → 首页或固定区域显示**
- 计算：  
  常用按钮不应藏在多级菜单深处。
- 结论：  
  启动、停止、复位、报警入口应放在明显位置。

> [!tip] 记忆口诀
> HMI画面不是给工程师炫技，是给现场人员快速操作和排故。

# 14 安装与防护

> [!info] 核心结论
> 工业HMI通常安装在电柜门、操作台或设备面板上，选型要关注防护等级、安装开孔、供电和环境温度。

## ① 分类/公式/规则

> [!warning] 安装规则
> **可靠安装 = 开孔尺寸正确 + 面板密封良好 + 接地可靠 + 电源稳定 + 通讯线规范**

## ② 参考表

| 项目 | 注意 |
|---|---|
| **安装方式** | 嵌入式面板安装常见 |
| **开孔尺寸** | 必须按厂家尺寸加工 |
| **防护等级** | 前面板常见IP65或更高 |
| **供电电压** | 常见DC24V |
| **接地** | 防止干扰和静电 |
| **温度范围** | 高温柜体需散热 |
| **触摸环境** | 油污、粉尘、手套操作需考虑 |

## ③ 计算/选型示例

- 已知：  
  HMI安装在设备外壳，现场有水雾。
- 公式：  
  **潮湿环境 → 高防护前面板 + 密封安装**
- 计算：  
  普通低防护面板可能进水导致触控失灵。
- 结论：  
  应选择适合工业现场防护等级的HMI，并保证密封圈安装正确。

> [!tip] 记忆口诀
> HMI前面防水靠等级，后面防护靠电柜。

# 15 常见故障排查

> [!info] 核心结论
> HMI故障排查按 **电源、屏幕、触摸、通讯、变量地址、PLC状态、程序逻辑** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **HMI异常 = 电源异常 + 触摸失灵 + 通讯失败 + 地址错误 + 权限限制 + PLC逻辑未满足**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **屏幕不亮** | 无电源、保险断、电源故障 |
| **触摸无效** | 触摸屏损坏、校准异常、画面锁定 |
| **通讯失败** | IP、站号、协议、线缆、端口错误 |
| **数据显示错误** | 地址错、数据类型错、字节序错 |
| **按钮无效** | 权限不足、写入地址错、PLC互锁 |
| **报警不显示** | 报警位未绑定、文本未配置 |
| **参数不保存** | 掉电保持区或配方下载逻辑错误 |

## ③ 计算/选型示例

- 已知：  
  HMI显示PLC通讯中断。
- 公式：  
  **通讯中断 = 协议/IP/站号/线缆/PLC状态异常**
- 计算：  
  先Ping PLC，再检查HMI通讯参数和PLC是否运行。
- 结论：  
  通讯故障优先查网络连接和参数，不要直接怀疑屏幕损坏。

> [!tip] 记忆口诀
> 屏不亮查电源，数不动查通讯，数不对查地址。

# 16 工业HMI触摸屏快速选型表

> [!info] 核心结论
> HMI选型要先确认PLC和通讯，再确定尺寸、功能、数据、配方、防护和安装环境。

## ① 分类/公式/规则

> [!note] 十步选型法
> ① 确认PLC或控制器品牌型号  
> ② 确认通讯协议和接口：以太网/RS485/RS232  
> ③ 确认屏幕尺寸和分辨率  
> ④ 确认画面数量和复杂程度  
> ⑤ 确认是否需要报警历史  
> ⑥ 确认是否需要趋势曲线  
> ⑦ 确认是否需要配方管理  
> ⑧ 确认是否需要数据记录和导出  
> ⑨ 确认防护等级和安装开孔  
> ⑩ 确认软件授权、工程备份和维护方式

> [!warning] 最终选型公式
> **推荐HMI = 尺寸 + 分辨率 + 通讯协议 + 数据功能 + 配方功能 + 防护等级 + 安装方式**

## ② 参考表

| 需求 | 推荐关注点 |
|---|---|
| 简单单机设备 | 4.3寸或7寸HMI |
| 参数较多设备 | 7寸或10寸HMI |
| 复杂产线监控 | 10寸以上HMI |
| 多产品切换 | 配方功能 |
| 过程数据分析 | 趋势曲线 |
| 质量追溯 | 数据记录和导出 |
| 恶劣环境 | 高防护等级和工业面板 |
| 多PLC通讯 | 多协议、多连接能力 |

## ③ 计算/选型示例

- 已知：  
  设备需要连接PLC，显示30个参数，记录报警历史，支持5种产品配方。
- 公式：  
  **HMI需求 = PLC通讯 + 参数画面 + 报警记录 + 配方管理**
- 计算：  
  屏幕过小会操作拥挤，建议选择7寸以上，并确认支持报警历史和配方。
- 结论：  
  推荐选择 **7寸或10寸工业HMI触摸屏**，通讯协议必须与PLC匹配。

> [!tip] 选型口诀
> 先看PLC协议，再看屏幕大小；参数多要大屏，产品多要配方，故障多要报警历史。
