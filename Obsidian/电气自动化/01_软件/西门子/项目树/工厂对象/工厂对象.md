## 01 `工厂对象` | 按现场结构管理工程对象

> [!info] 核心结论
> **工厂对象**用于在工厂视图中按产线、区域、工位、设备单元来组织 PLC、HMI、驱动、通信和数据对象。

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">核心规则</strong><br>
工厂对象不是程序逻辑本身，而是用来把工程对象按现场结构整理清楚。
</div>

### ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| 所属位置 | **工厂视图** |
| 名称 | **工厂对象** |
| 核心作用 | 按现场层级组织项目对象 |
| 常见层级 | 工厂 → 产线 → 区域 → 工位 → 设备 |
| 适用场景 | 大型项目、多设备项目、复杂产线项目 |
| 注意事项 | 具体可挂载对象需按实际 TIA Portal 版本校核 |

### ② 参考表

| 工厂对象层级 | 示例 |
|---|---|
| 工厂 | 一期工厂 / 二期工厂 |
| 产线 | 1号线 / 2号线 |
| 区域 | 上料区 / 加工区 / 下料区 |
| 工位 | 工位 01 / 工位 02 |
| 设备 | 输送机 / 机器人 / 检测站 |
| 控制对象 | PLC / HMI / 驱动 / IO站 |

### ③ 计算/选型示例

- 已知：项目中有 2 条产线，每条产线包含上料、加工、检测、下料区域
- 公式/规则：现场结构复杂时，用工厂对象分层管理
- 计算/操作：  
  `工厂 → 产线 → 区域 → 工位 → 设备`
- 结论：使用工厂对象后，工程结构更接近现场，查找和维护更方便

> [!tip] 记忆口诀
> **现场怎么分，工厂对象就怎么建。**


## 02 `工厂对象层级` | 从大到小搭结构

> [!info] 核心结论
> 建工厂对象时，建议按“由大到小”的现场层级搭建，避免项目后期混乱。

<div style="background:#F7F9FC;border-left:6px solid #FF6B35;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#FF6B35;">选型口诀</strong><br>
先建产线，再分区域；区域下面放工位，工位下面挂设备。
</div>

### ① 分类/公式/规则

| 层级 | 作用 |
|---|---|
| 第 1 层 | 工厂 / 项目总区域 |
| 第 2 层 | 产线 |
| 第 3 层 | 功能区域 |
| 第 4 层 | 工位 |
| 第 5 层 | 具体设备或控制对象 |

### ② 参考表

| 现场对象 | 建议命名 |
|---|---|
| 产线 | `Line_01` / `Line_02` |
| 区域 | `Loading` / `Process` / `Testing` |
| 工位 | `Station_01` / `Station_02` |
| 设备 | `Conveyor_01` / `Robot_01` |
| 控制器 | `PLC_Line01` / `HMI_Main` |

### ③ 计算/选型示例

- 已知：要整理一条自动装配线
- 公式/规则：按现场工艺流向建立层级
- 计算/操作：  
  `Line_01 → 上料区 → 上料工位 → 上料机械手`
- 结论：后续查找设备、报警、程序对象时更直观

> [!tip] 记忆口诀
> **先大后小，先线后站；对象分清，维护不乱。**


## 03 `工厂对象` | 选型速记

> [!info] 核心结论
> 工厂对象的核心是：**用现场结构管理工程对象，让项目树更像真实产线。**

<div style="background:#F7F9FC;border-left:6px solid #27AE60;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#27AE60;">总口诀</strong><br>
项目多用工厂对象，按线、区、站、设备分层摆放。
</div>

### ① 分类/公式/规则

| 需求 | 推荐做法 |
|---|---|
| 项目对象很多 | 使用 **工厂对象** |
| 多条产线 | 按 **产线** 分层 |
| 每条线多个区域 | 按 **区域** 分层 |
| 每个区域多个工位 | 按 **工位** 分层 |
| 多设备维护 | 设备挂到对应工位下 |
| 多人协作 | 统一命名规范 |

### ② 参考表

| 场景 | 是否建议用工厂对象 |
|---|---|
| 单 PLC 小项目 | 可不用 |
| 多 PLC 项目 | **建议使用** |
| 多 HMI 项目 | **建议使用** |
| 多产线项目 | **强烈建议使用** |
| 设备很多、维护复杂 | **强烈建议使用** |

### ③ 计算/选型示例

- 已知：一个工厂项目包含 3 条生产线，每条线有 PLC、HMI、远程 IO 和驱动
- 公式/规则：对象多、层级多时，用工厂对象统一组织
- 计算/操作：  
  `工厂 → 产线 → 工位 → PLC/HMI/IO/驱动`
- 结论：项目结构清晰，查找设备和维护程序更高效

> [!tip] 记忆口诀
> **小项目看项目树，大项目靠工厂对象。**
