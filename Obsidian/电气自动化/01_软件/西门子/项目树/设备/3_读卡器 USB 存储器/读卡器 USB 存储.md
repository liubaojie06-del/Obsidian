# 01 读卡器/USB 存储器总览

> [!info] 核心结论
> **读卡器/USB 存储器** 用于访问外部存储介质，常用于项目文件、设备数据、备份文件或存储卡内容的读取与传输。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> 它不是PLC、HMI或驱动设备，而是电脑访问外部存储介质的入口。

> [!warning] 🟧 使用判断公式
> **是否使用存储器 = 需要读取文件 + 需要备份数据 + 需要传输项目 + 有外部存储介质**

## ② 参考表

| 类型 | 主要用途 |
|---|---|
| **读卡器** | 读取存储卡、工程卡、设备数据卡 |
| **USB存储器** | 拷贝项目、备份文件、传输数据 |
| **存储卡** | 可能用于设备程序、参数或归档 |
| **外部介质** | 项目转移、现场备份、数据导出 |

## ③ 计算/选型示例

- 已知：  
  现场需要把项目备份文件从调试电脑转移到另一台电脑。
- 公式：  
  **文件转移 = 项目文件 + USB存储器 + 正确路径 + 完整校验**
- 计算：  
  将项目归档文件复制到USB存储器，再在另一台电脑中解压或打开。
- 结论：  
  可使用 **USB存储器** 完成项目转移，但必须确认文件完整、版本正确、无病毒风险。

> [!tip] 记忆口诀
> 项目要带走，U盘来帮手；存储卡要读，先找读卡器。

# 02 读卡器

> [!info] 核心结论
> **读卡器** 用于读取存储卡内容，适合查看、备份或恢复某些设备相关数据。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> 读卡器相当于电脑和存储卡之间的桥梁，把存储卡内容显示到工程软件或系统中。

> [!warning] 🟧 读卡流程公式
> **读卡成功 = 存储卡正常 + 读卡器兼容 + 驱动正常 + 软件可识别**

## ② 参考表

| 检查项 | 重点 |
|---|---|
| **存储卡类型** | 是否支持该读卡器 |
| **读卡器驱动** | 电脑是否识别 |
| **文件系统** | 是否能被系统读取 |
| **数据完整性** | 是否损坏或缺失 |
| **操作权限** | 是否允许读写 |

## ③ 计算/选型示例

- 已知：  
  需要读取一张设备存储卡中的备份数据。
- 公式：  
  **读取条件 = 卡正常 + 读卡器正常 + 软件识别**
- 计算：  
  先确认电脑能识别读卡器，再确认软件中能看到对应存储介质。
- 结论：  
  若无法识别，应检查读卡器、驱动、存储卡类型和存储卡状态。

> [!tip] 记忆口诀
> 卡没坏、器能读、软件识别才算通。

# 03 USB 存储器

> [!info] 核心结论
> **USB存储器** 常用于项目拷贝、归档备份、日志导出、参数文件传输等现场工程操作。

## ① 分类/公式/规则

> [!note] 🟦 常见用途
> USB存储器常用于：  
> ① 项目备份  
> ② 文件传输  
> ③ 日志导出  
> ④ 参数文件保存  
> ⑤ 现场交付资料转移

> [!warning] 🟧 USB使用公式
> **安全使用USB = 格式正确 + 容量足够 + 文件完整 + 病毒检查 + 正确弹出**

## ② 参考表

| 项目 | 建议 |
|---|---|
| **容量** | 满足项目和日志大小 |
| **格式** | 按设备或系统要求选择 |
| **速度** | 大项目建议用高速U盘 |
| **可靠性** | 重要项目不要用劣质U盘 |
| **安全** | 插入前建议查杀病毒 |
| **拔出** | 使用前后正确弹出 |

## ③ 计算/选型示例

- 已知：  
  项目归档文件大小为 3.5GB，U盘剩余容量为 4GB。
- 公式：  
  **剩余容量 ≥ 文件大小 × 1.2**
- 计算：  
  推荐容量 = 3.5GB × 1.2 = **4.2GB**
- 结论：  
  当前U盘剩余4GB略不足，建议更换更大容量U盘，避免复制失败或文件损坏。

> [!tip] 记忆口诀
> U盘容量留余量，拷贝项目不慌张。

# 04 常见操作场景

> [!info] 核心结论
> 读卡器和USB存储器最常见的价值，是帮助工程师进行 **备份、转移、恢复和归档**。

## ① 分类/公式/规则

> [!note] 🟦 四类典型操作
> **备份项目、转移文件、读取数据、恢复资料** 是最常见的四类场景。

> [!warning] 🟧 文件管理公式
> **可靠文件操作 = 正确文件 + 正确路径 + 正确版本 + 完整校验**

## ② 参考表

| 场景 | 推荐操作 |
|---|---|
| **项目备份** | 导出归档后复制到U盘 |
| **项目转移** | 用U盘拷贝到目标电脑 |
| **读取存储卡** | 使用读卡器读取 |
| **日志导出** | 保存到USB存储器 |
| **参数恢复** | 从备份文件恢复 |
| **客户交付** | 使用只读或归档文件 |

## ③ 计算/选型示例

- 已知：  
  调试完成后，需要把最终项目交付给客户。
- 公式：  
  **交付文件 = 项目归档 + 版本说明 + 备份文件 + 校验确认**
- 计算：  
  将最终版项目归档，复制到USB存储器，并确认文件可打开。
- 结论：  
  交付前不要只复制项目文件夹，建议使用项目归档方式，并保留正式版本说明。

> [!tip] 记忆口诀
> 交付别乱拷，归档最可靠。

# 05 使用注意事项

> [!info] 核心结论
> 使用读卡器或USB存储器时，最怕文件损坏、版本混乱、病毒感染和误删数据。

## ① 分类/公式/规则

> [!note] 🟦 关键注意
> 不要直接拔出U盘，不要覆盖唯一备份，不要把测试版误当正式版交付。

> [!warning] 🟧 风险公式
> **数据风险 = 误删 + 覆盖 + 病毒 + 拔出中断 + 版本混乱**

## ② 参考表

| 风险 | 预防方法 |
|---|---|
| **文件损坏** | 复制完成后重新打开验证 |
| **版本混乱** | 文件名包含日期和版本号 |
| **病毒感染** | 插入前后进行安全检查 |
| **误覆盖** | 先备份再替换 |
| **拔出中断** | 正确弹出设备 |
| **唯一备份丢失** | 至少保留两份备份 |

## ③ 计算/选型示例

- 已知：  
  U盘里同时有“最新版”“最终版”“现场版”三个项目。
- 公式：  
  **版本风险 = 文件数量 × 命名混乱程度**
- 计算：  
  名称越模糊，越容易下载错项目。
- 结论：  
  应统一命名，例如：`项目名_日期_V1.0_Release`，避免误用错误版本。

> [!tip] 记忆口诀
> 文件不怕多，就怕名字看不懂。

# 06 常见问题排查

> [!info] 核心结论
> 读卡器或USB无法使用时，优先检查电脑识别、驱动状态、文件格式、权限和介质本身。

## ① 分类/公式/规则

> [!note] 🟦 排查顺序
> 先看电脑能不能识别，再看TIA能不能访问，最后检查文件和权限。

> [!warning] 🟧 排错公式
> **无法访问 = 介质故障 + 驱动异常 + 格式不支持 + 权限不足 + 文件损坏**

## ② 参考表

| 问题 | 可能原因 |
|---|---|
| **看不到U盘** | USB口、U盘或驱动异常 |
| **看不到存储卡** | 读卡器不兼容或卡损坏 |
| **文件打不开** | 文件损坏或版本不兼容 |
| **无法复制** | 容量不足或权限限制 |
| **复制很慢** | U盘速度低或文件过大 |
| **项目打不开** | TIA版本不匹配 |

## ③ 计算/选型示例

- 已知：  
  U盘能被电脑识别，但项目文件打不开。
- 公式：  
  **项目打开条件 = 文件完整 + TIA版本支持 + 路径正常**
- 计算：  
  如果项目来自更高版本TIA，低版本TIA可能无法打开。
- 结论：  
  应确认项目使用的TIA版本，并检查文件是否完整复制。

> [!tip] 记忆口诀
> 电脑能识别，不代表项目能打开。

# 07 快速检查表

> [!info] 核心结论
> 使用外部存储介质前后，都要检查容量、版本、完整性和安全性。

## ① 分类/公式/规则

> [!note] 🟦 五步检查法
> ① 介质是否识别  
> ② 容量是否足够  
> ③ 文件版本是否正确  
> ④ 文件是否能打开  
> ⑤ 是否安全弹出

> [!warning] 🟧 最终检查公式
> **可靠存储 = 识别正常 + 容量足够 + 版本明确 + 文件完整 + 安全拔出**

## ② 参考表

| 检查项 | 建议 |
|---|---|
| **介质识别** | 插入后先确认电脑识别 |
| **剩余容量** | 至少预留20%余量 |
| **文件命名** | 包含项目名、日期、版本 |
| **完整验证** | 拷贝后重新打开检查 |
| **安全检查** | 注意病毒和权限 |
| **安全弹出** | 不要直接拔出 |

## ③ 计算/选型示例

- 已知：  
  需要复制一个 8GB 的项目备份到U盘。
- 公式：  
  **建议剩余容量 ≥ 文件大小 × 1.2**
- 计算：  
  8GB × 1.2 = **9.6GB**
- 结论：  
  U盘剩余容量建议不少于 **10GB**，并在复制完成后重新打开验证。

> [!tip] 选型口诀
> 读卡先识别，U盘看容量；文件要归档，拔出要正常。
