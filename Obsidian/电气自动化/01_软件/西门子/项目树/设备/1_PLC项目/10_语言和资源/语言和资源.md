# 01 语言和资源总览

> [!info] 核心结论
> **语言和资源** 用于统一管理项目语言、项目文本和多语言显示，是HMI国际化和项目标准化的重要入口。

## ① 分类/公式/规则

> [!note] 🟦 功能主线
> 语言和资源主要包含两部分：  
> **项目语言**：决定项目支持哪些语言；  
> **项目文本**：集中管理画面文字、报警文本、提示文本等内容。

> [!warning] 🟧 多语言配置公式
> **多语言项目 = 项目语言 + 项目文本 + 翻译内容 + HMI语言切换 + 下载验证**

## ② 参考表

| 项目 | 主要作用 |
|---|---|
| **语言和资源** | 管理项目语言和文本资源 |
| **项目语言** | 设置项目支持的语言 |
| **项目文本** | 管理文本、报警、提示、标签 |
| **多语言显示** | HMI切换不同语言 |
| **文本维护** | 统一修改、导入、导出翻译 |

## ③ 计算/选型示例

- 已知：  
  一台设备需要中文和英文两种HMI界面。
- 公式：  
  **双语HMI = 中文项目语言 + 英文项目语言 + 文本翻译 + 切换按钮**
- 计算：  
  需要先添加两种项目语言，再为按钮、报警、提示等文本填写对应翻译。
- 结论：  
  应在 **语言和资源** 中配置项目语言和项目文本，并在HMI中实现语言切换。

> [!tip] 记忆口诀
> 语言先添加，文本再翻译；HMI要双语，资源要统一。

# 02 项目语言

> [!info] 核心结论
> **项目语言** 用于设置整个TIA项目支持的语言，是多语言HMI和多语言文本的基础。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> 项目语言决定项目中可以使用哪些语言版本，例如中文、英文、德文、日文等。

> [!warning] 🟧 项目语言配置公式
> **项目语言配置 = 默认语言 + 目标语言 + 编辑语言 + 运行语言**

## ② 参考表

| 配置项 | 说明 |
|---|---|
| **默认语言** | 项目主要编辑语言 |
| **目标语言** | 需要翻译和显示的语言 |
| **编辑语言** | 工程师当前编辑使用的语言 |
| **运行语言** | HMI运行时可切换的语言 |
| **语言数量** | 按项目交付需求确定 |

## ③ 计算/选型示例

- 已知：  
  项目需要交付国内客户和海外客户，HMI需要中文、英文两种语言。
- 公式：  
  **语言需求 = 客户语言 + 操作员语言 + 维护人员语言**
- 计算：  
  国内操作员使用中文，海外客户或维护人员可能需要英文。
- 结论：  
  项目语言中至少应配置 **中文 + 英文**，具体语言包和显示效果需按TIA版本及HMI支持情况校核。

> [!tip] 记忆口诀
> 先定谁来用，再定用什么语言。

# 03 项目文本

> [!info] 核心结论
> **项目文本** 用于集中管理项目中的文字内容，例如按钮名称、报警文本、提示信息和单位说明。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> 项目文本相当于项目的“文字数据库”，方便统一修改、翻译和复用。

> [!warning] 🟧 项目文本管理公式
> **项目文本 = 画面文字 + 报警文字 + 提示文字 + 单位文字 + 多语言翻译**

## ② 参考表

| 文本类型 | 示例 |
|---|---|
| **按钮文本** | 启动、停止、复位 |
| **状态文本** | 运行中、待机、故障 |
| **报警文本** | 电机过载、气压低 |
| **提示文本** | 请确认安全门已关闭 |
| **单位文本** | ℃、bar、rpm、mm |
| **菜单文本** | 参数设置、报警记录 |

## ③ 计算/选型示例

- 已知：  
  HMI中有“启动”“停止”“复位”三个按钮，需要支持英文。
- 公式：  
  **文本翻译 = 原始文本 + 目标语言文本 + 显示验证**
- 计算：  
  - 启动 → Start  
  - 停止 → Stop  
  - 复位 → Reset
- 结论：  
  应在 **项目文本** 中统一维护翻译，避免每个画面单独修改导致不一致。

> [!tip] 记忆口诀
> 文本集中管，翻译不跑偏。

# 04 多语言HMI配置

> [!info] 核心结论
> HMI多语言配置的关键，是让项目语言、项目文本和运行时语言切换保持一致。

## ① 分类/公式/规则

> [!note] 🟦 配置步骤
> ① 添加项目语言  
> ② 填写项目文本翻译  
> ③ 设置HMI运行语言  
> ④ 添加语言切换按钮  
> ⑤ 下载到HMI验证显示

> [!warning] 🟧 多语言HMI公式
> **多语言生效 = 语言已添加 + 文本已翻译 + HMI支持 + 切换逻辑正确**

## ② 参考表

| 步骤 | 检查内容 |
|---|---|
| **1 项目语言** | 是否添加目标语言 |
| **2 项目文本** | 是否完成翻译 |
| **3 HMI画面** | 文本是否引用项目文本 |
| **4 切换按钮** | 是否能切换运行语言 |
| **5 下载验证** | HMI实际显示是否正确 |

## ③ 计算/选型示例

- 已知：  
  HMI运行时中文显示正常，但切换英文后部分按钮仍显示中文。
- 公式：  
  **显示异常 = 目标语言缺失 + 文本未翻译 + 文本未引用**
- 计算：  
  某些按钮可能直接写死中文，没有使用项目文本资源。
- 结论：  
  应检查项目文本中对应英文是否填写，并确认画面对象正确引用多语言文本。

> [!tip] 记忆口诀
> 语言能切换，不代表文本全翻译。

# 05 翻译导入导出

> [!info] 核心结论
> 项目文本可用于批量维护翻译，适合大量画面、报警和提示文本的多语言项目。

## ① 分类/公式/规则

> [!note] 🟦 使用场景
> 当项目文本较多时，建议导出文本进行统一翻译，再导入项目，减少手动逐项修改。

> [!warning] 🟧 批量翻译公式
> **高效翻译 = 文本导出 + 表格翻译 + 格式不变 + 导入校验**

## ② 参考表

| 操作 | 重点 |
|---|---|
| **导出文本** | 保留原始格式 |
| **翻译文本** | 不改变量名和标识 |
| **导入文本** | 检查语言对应关系 |
| **编译项目** | 确认无文本错误 |
| **运行验证** | 检查换行、长度、显示 |

## ③ 计算/选型示例

- 已知：  
  项目有300条报警文本需要翻译成英文。
- 公式：  
  **翻译工作量 = 文本数量 × 目标语言数量**
- 计算：  
  300条 × 1种目标语言 = 300条翻译内容，手动逐项输入效率低。
- 结论：  
  建议通过项目文本导出后批量翻译，再导入TIA项目，并逐项抽查显示效果。

> [!tip] 记忆口诀
> 文本多别手敲，导出翻译效率高。

# 06 常见问题排查

> [!info] 核心结论
> 多语言显示异常，通常是语言未添加、文本未翻译、对象未引用或HMI未正确下载。

## ① 分类/公式/规则

> [!note] 🟦 排查顺序
> 先查项目语言，再查项目文本，最后查HMI对象和下载结果。

> [!warning] 🟧 排错公式
> **语言异常 = 语言缺失 + 翻译缺失 + 引用错误 + 下载不完整**

## ② 参考表

| 问题 | 可能原因 |
|---|---|
| **切换后仍是原语言** | HMI未配置运行语言 |
| **部分文本未翻译** | 项目文本未填写目标语言 |
| **报警不翻译** | 报警文本未维护多语言 |
| **文字显示不全** | 翻译后文本过长 |
| **乱码或符号异常** | 字体或语言支持问题 |
| **下载后无变化** | 未重新编译或下载 |

## ③ 计算/选型示例

- 已知：  
  英文界面中有些报警仍显示中文。
- 公式：  
  **报警翻译检查 = 报警文本 + 项目文本 + 目标语言列**
- 计算：  
  若报警文本没有填写英文列，HMI切换后仍可能显示默认语言。
- 结论：  
  应补全报警文本的目标语言，并重新编译下载HMI项目。

> [!tip] 记忆口诀
> 多语言出问题，先查文本表。

# 07 语言和资源快速检查表

> [!info] 核心结论
> 多语言项目交付前，必须检查语言、文本、报警、画面和HMI运行切换是否完整。

## ① 分类/公式/规则

> [!note] 🟦 交付前五查
> ① 项目语言是否完整  
> ② 项目文本是否翻译  
> ③ 报警文本是否翻译  
> ④ HMI切换是否正常  
> ⑤ 文本显示是否超出画面

> [!warning] 🟧 最终检查公式
> **多语言交付 = 语言完整 + 翻译完整 + 显示正常 + 切换正常 + 下载验证**

## ② 参考表

| 检查项 | 建议 |
|---|---|
| **项目语言** | 按客户要求配置 |
| **项目文本** | 统一维护 |
| **报警文本** | 必须翻译完整 |
| **按钮文本** | 检查长度和换行 |
| **单位符号** | 注意国际单位习惯 |
| **HMI下载** | 编译后重新下载验证 |

## ③ 计算/选型示例

- 已知：  
  项目准备交付海外客户，需要英文界面。
- 公式：  
  **交付检查 = 英文语言 + 英文文本 + 英文报警 + HMI切换**
- 计算：  
  如果只翻译按钮，不翻译报警，客户维护时仍会看不懂故障信息。
- 结论：  
  交付前应完整检查所有项目文本、报警文本和HMI显示效果。

> [!tip] 选型口诀
> 按钮要翻译，报警更要翻译；界面能切换，交付才放心。
