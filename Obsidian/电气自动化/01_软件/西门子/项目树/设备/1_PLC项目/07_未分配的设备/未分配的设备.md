# 01 未分配的设备总览

> [!info] 核心结论
> **未分配的设备** 是项目中已经存在，但还没有正确分配到控制器、网络、站点或系统结构中的设备。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> 未分配的设备不一定是错误，但表示该设备还没有完成归属关系、网络关系或控制关系的配置。

> [!warning] 🟧 判断公式
> **未分配设备 = 已添加设备 + 未建立归属 + 未完成通讯/控制关系**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **显示位置** | 项目树 / 设备区域 |
| **常见对象** | HMI、驱动、I/O、PC系统、第三方设备 |
| **常见原因** | 未连接网络、未绑定PLC、未设置接口 |
| **影响** | 可能无法通讯、无法下载、无法联动 |
| **处理方式** | 分配控制器、连接网络、设置地址 |

## ③ 计算/选型示例

- 已知：  
  项目中添加了一台HMI，但还没有连接PLC。
- 公式：  
  **HMI分配 = 选择PLC + 建立连接 + 设置IP + 下载项目**
- 计算：  
  HMI没有通讯对象时，可能被视为未分配或未建立完整关系。
- 结论：  
  需要在设备和网络中把HMI连接到对应PLC，并检查通讯参数。

> [!tip] 记忆口诀
> 设备进项目不算完，分配清楚才算稳。

# 02 未分配 vs 未分组

> [!info] 核心结论
> **未分组**偏向项目管理结构，**未分配**偏向设备归属和通讯关系。

## ① 分类/公式/规则

> [!note] 🟦 区别口诀
> 未分组：设备没有整理到文件夹或设备组。  
> 未分配：设备没有绑定到正确控制器、网络或系统对象。

> [!warning] 🟧 区分公式
> **未分组看管理，未分配看关系**

## ② 参考表

| 对比项 | 未分组的设备 | 未分配的设备 |
|---|---|---|
| 重点 | 项目树整理 | **设备归属关系** |
| 是否影响通讯 | 不一定 | 可能影响 |
| 处理方式 | 分组、命名 | **分配、联网、绑定** |
| 常见对象 | 新增设备 | HMI、驱动、I/O |
| 核心问题 | 不好管理 | 关系不完整 |

## ③ 计算/选型示例

- 已知：  
  一台HMI在项目中存在，但没有连接到PLC。
- 公式：  
  **通讯完整性 = 设备存在 + 网络连接 + 通讯对象**
- 计算：  
  设备已添加，但没有通讯对象，属于关系不完整。
- 结论：  
  这类问题更接近 **未分配**，应重点检查设备连接和通讯对象。

> [!tip] 记忆口诀
> 未分组是“没归档”，未分配是“没对象”。

# 03 常见出现原因

> [!info] 核心结论
> 未分配设备通常出现在新建、导入、复制或修改网络结构之后。

## ① 分类/公式/规则

> [!note] 🟦 常见触发情况
> 新增设备、导入设备、复制设备、删除原控制器、修改网络接口后，都可能出现未分配设备。

> [!warning] 🟧 出现原因公式
> **未分配原因 = 新设备 + 旧关系丢失 + 网络未连 + 控制器未绑定**

## ② 参考表

| 情况 | 可能原因 |
|---|---|
| **新增HMI** | 未选择PLC连接 |
| **新增驱动** | 未连接PROFINET/PROFIBUS |
| **新增远程I/O** | 未分配给IO控制器 |
| **复制设备** | 原项目关系丢失 |
| **删除PLC** | 下级设备失去归属 |
| **修改网络** | 接口关系断开 |

## ③ 计算/选型示例

- 已知：  
  从旧项目复制了一台驱动器到新项目。
- 公式：  
  **复制后检查 = 设备型号 + 网络接口 + 控制对象 + 地址参数**
- 计算：  
  复制设备可能只复制硬件对象，没有复制完整网络和控制关系。
- 结论：  
  需要重新连接网络，并分配给对应PLC或控制器。

> [!tip] 记忆口诀
> 复制设备别只看图标，关系和地址也要补好。

# 04 如何分配设备

> [!info] 核心结论
> 分配设备的关键，是让设备找到自己的控制器、网络接口、地址和通讯对象。

## ① 分类/公式/规则

> [!note] 🟦 分配步骤
> ① 确认设备类型  
> ② 连接到正确网络  
> ③ 设置IP或站地址  
> ④ 指定控制器或通讯对象  
> ⑤ 编译检查

> [!warning] 🟧 分配公式
> **设备分配 = 网络连接 + 地址设置 + 控制器绑定 + 编译下载**

## ② 参考表

| 设备类型 | 分配重点 |
|---|---|
| **HMI** | 连接对应PLC |
| **驱动器** | 分配给PLC或运动控制器 |
| **远程I/O** | 分配给IO控制器 |
| **PC系统** | 设置通讯对象和运行软件 |
| **第三方设备** | 确认协议和接口映射 |

## ③ 计算/选型示例

- 已知：  
  新增一个远程I/O站，需要让PLC控制它。
- 公式：  
  **远程I/O分配 = IO站 + PROFINET网络 + PLC IO控制器 + 设备名**
- 计算：  
  远程I/O必须接入PLC所在网络，并分配给对应IO控制器。
- 结论：  
  在网络视图中完成连接和分配后，再编译下载硬件组态。

> [!tip] 记忆口诀
> 先联网，再分配；先设地址，再下载。

# 05 与网络配置的关系

> [!info] 核心结论
> 大多数未分配问题，本质上都和网络连接、地址规划或控制器归属有关。

## ① 分类/公式/规则

> [!note] 🟦 网络关键点
> 设备能不能正常工作，除了型号正确，还要看是否连接到正确网络，并且地址不冲突。

> [!warning] 🟧 通讯成功公式
> **通讯成功 = 网络一致 + 地址唯一 + 设备名正确 + 控制器分配正确**

## ② 参考表

| 配置项 | 检查重点 |
|---|---|
| **网络接口** | PN/IE、PROFIBUS是否选对 |
| **IP地址** | 是否同网段、是否冲突 |
| **设备名** | PROFINET名称是否一致 |
| **IO控制器** | 远程I/O是否分配给PLC |
| **通讯连接** | HMI是否指向正确PLC |

## ③ 计算/选型示例

- 已知：  
  远程I/O已经添加，但PLC在线诊断中找不到该站。
- 公式：  
  **IO站识别 = 网络连接 + 设备名 + IP地址 + IO控制器分配**
- 计算：  
  如果没有把IO站分配给PLC，PLC就无法作为IO控制器管理它。
- 结论：  
  应检查网络视图中的分配关系和设备名，必要时重新下载设备名和硬件组态。

> [!tip] 记忆口诀
> IP管地址，设备名管身份，控制器分配管归属。

# 06 常见问题排查

> [!info] 核心结论
> 排查未分配设备时，要按“设备是否存在 → 网络是否连接 → 地址是否正确 → 是否绑定控制器”的顺序检查。

## ① 分类/公式/规则

> [!note] 🟦 排查顺序
> 先确认设备型号和接口，再检查网络连接，最后检查控制器或通讯对象是否正确。

> [!warning] 🟧 排错公式
> **未分配异常 = 设备未联网 + 地址未设 + 控制器未选 + 组态未下载**

## ② 参考表

| 问题 | 可能原因 |
|---|---|
| **设备无法通讯** | 网络未连接或地址错误 |
| **远程I/O不工作** | 未分配给IO控制器 |
| **HMI无数据** | 未连接正确PLC |
| **驱动无法控制** | 未分配到控制系统 |
| **编译报错** | 设备关系不完整 |
| **下载失败** | 接口或目标设备错误 |

## ③ 计算/选型示例

- 已知：  
  HMI画面能打开，但变量全部显示异常。
- 公式：  
  **HMI变量正常 = HMI连接 + PLC在线 + 变量地址正确 + 下载正确**
- 计算：  
  如果HMI没有正确连接PLC，即使画面正常，也无法读到变量。
- 结论：  
  检查HMI连接对象是否为正确PLC，并确认项目已下载到正确HMI。

> [!tip] 记忆口诀
> 画面能开不代表通讯正常，变量能读才算连接成功。

# 07 快速检查表

> [!info] 核心结论
> 看到未分配设备时，不要急着删除，先确认它是否是正式设备，再补全网络和归属关系。

## ① 分类/公式/规则

> [!note] 🟦 五步检查法
> ① 是否正式设备  
> ② 是否连接网络  
> ③ 是否设置地址  
> ④ 是否绑定控制器  
> ⑤ 是否编译下载

> [!warning] 🟧 最终检查公式
> **设备可用 = 型号正确 + 网络正确 + 地址正确 + 分配正确 + 编译无错**

## ② 参考表

| 检查项 | 建议 |
|---|---|
| **设备是否需要保留** | 不用则删除或备注 |
| **设备名称是否清楚** | 使用统一命名规则 |
| **网络接口是否正确** | PN/IE、PROFIBUS等 |
| **地址是否唯一** | 防止IP或站地址冲突 |
| **控制器是否分配** | PLC、HMI、驱动关系明确 |
| **项目是否编译通过** | 下载前必须检查 |

## ③ 计算/选型示例

- 已知：  
  项目树中出现一个未分配的驱动设备。
- 公式：  
  **驱动可用性 = 驱动型号 + 网络连接 + 控制器分配 + 参数下载**
- 计算：  
  如果该驱动是正式设备，就要接入网络并分配给对应控制器；如果是测试残留，应删除或单独备注。
- 结论：  
  未分配设备要么补全配置，要么清理掉，避免后期误连、误下载或维护混乱。

> [!tip] 选型口诀
> 未分配先别删，确认用途再判断；正式设备补关系，临时设备及时清。
