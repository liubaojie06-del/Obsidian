# 01｜组织块 OB 是什么

> [!info] 核心结论
> **组织块 OB 是 PLC 程序的入口和调度中心，CPU 会按照事件或周期自动调用对应 OB。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
OB 就像 PLC 程序的“入口门”，CPU 发生什么事件，就从对应的 OB 进去执行程序。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **英文名称** | Organization Block |
| **中文名称** | 组织块 |
| **作用** | 决定程序从哪里开始执行 |
| **调用方式** | CPU 自动调用 |
| **常见代表** | `OB1`、启动 OB、循环中断 OB、诊断错误 OB |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>核心规则</strong><br>
FC / FB 需要被调用才会执行；OB 是 CPU 根据运行事件自动调用的入口。
</div>

## ② 参考表

| 块类型 | 谁调用 | 主要用途 |
|---|---|---|
| **OB** | CPU 自动调用 | 程序入口 / 事件响应 |
| **FB** | OB / FC / FB 调用 | 带实例数据的功能块 |
| **FC** | OB / FB / FC 调用 | 不带固定实例数据的函数 |
| **DB** | 被程序访问 | 存储数据 |

## ③ 计算/选型示例

- **已知：** 一个电机启停程序写在 `FC1` 中  
- **公式：** FC 不会自动执行，必须被 OB 调用  
- **计算：**
  - CPU 周期执行 `OB1`
  - 在 `OB1` 中调用 `FC1`
  - `FC1` 内部电机逻辑才会运行
- **结论：** 程序逻辑写好后，必须放到 OB 的调用链里。

> [!tip] 记忆口诀
> **OB 是入口，FB/FC 是功能；入口不调用，功能不执行。**

# 02｜OB1 主循环组织块

> [!info] 核心结论
> **OB1 是最常用的主循环组织块，PLC 在 RUN 状态下会不断循环执行 OB1。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
OB1 就是 PLC 的“主程序循环”，大多数普通逻辑都从这里开始。
</div>

## ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| **块名** | `OB1` |
| **类型** | 程序循环 OB |
| **调用条件** | CPU 处于 RUN |
| **执行方式** | 扫描循环 |
| **常放内容** | 主控制逻辑、FC/FB 调用、状态处理 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>扫描周期公式感</strong><br>
一个扫描周期 ≈ 读取输入 → 执行 OB1 程序 → 刷新输出
</div>

## ② 参考表

| 内容 | 是否适合放 OB1 | 说明 |
|---|---|---|
| **普通启停逻辑** | 适合 | 最常见 |
| **状态机调用** | 适合 | 推荐结构化 |
| **通信轮询** | 适合 | 注意节拍 |
| **高速精准定时** | 不太适合 | 建议用中断 OB |
| **一次性初始化** | 不建议直接放 | 更适合启动 OB |

## ③ 计算/选型示例

- **已知：** PLC 需要持续检测启动按钮并控制电机  
- **公式：** 周期性普通逻辑 → 放入 `OB1` 或由 `OB1` 调用  
- **计算：**
  - 启动按钮每个扫描周期读取
  - 程序每个扫描周期判断
  - 输出每个扫描周期刷新
- **结论：** 电机启停这种普通逻辑适合放在 `OB1` 调用链中。

> [!tip] 记忆口诀
> **普通逻辑放 OB1，周期扫描不停歇。**

# 03｜启动 OB

> [!info] 核心结论
> **启动 OB 用于 CPU 从 STOP 切换到 RUN 时执行一次，适合做初始化、清零、参数准备。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
启动 OB 就像设备开机前的“准备动作”，只在启动阶段执行。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **类型** | Startup OB |
| **调用时机** | CPU 启动 / STOP 到 RUN |
| **执行次数** | 启动时执行一次或按启动事件执行 |
| **常见用途** | 初始化变量、复位状态、准备参数 |
| **注意** | 不适合写长期循环逻辑 |

## ② 参考表

| 适合内容 | 说明 |
|---|---|
| **初始化状态机** | 把步骤清到初始状态 |
| **清报警锁存** | 按工艺决定是否清除 |
| **加载默认参数** | 准备运行参数 |
| **复位临时变量** | 避免上电脏数据 |
| **建立初始标志** | 生成首扫条件 |

## ③ 计算/选型示例

- **已知：** 设备每次上电后，步骤号必须回到 `0`  
- **公式：** 上电初始化 → 启动 OB  
- **计算：**
  - CPU 从 STOP 到 RUN
  - 启动 OB 被调用
  - 将 `Step := 0`
- **结论：** 上电初始化变量适合放在启动 OB 中。

> [!tip] 记忆口诀
> **开机只做一次，用启动 OB 最合适。**

# 04｜循环中断 OB

> [!info] 核心结论
> **循环中断 OB 按固定时间间隔执行，适合对节拍要求稳定的任务。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
循环中断 OB 是 PLC 的“定时闹钟”，到时间就执行一次。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **类型** | Cyclic Interrupt OB |
| **调用方式** | 固定周期自动调用 |
| **常见用途** | PID、采样、周期计算、运动辅助 |
| **关键参数** | 中断周期 |
| **注意** | 周期不能设置得过短，需按 CPU 性能校核 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点规则</strong><br>
周期越短，CPU 负载越高；中断任务太重，会影响主循环。
</div>

## ② 参考表

| 任务 | 是否适合循环中断 OB | 原因 |
|---|---|---|
| **PID 计算** | 适合 | 需要固定采样周期 |
| **速度计算** | 适合 | 时间基准稳定 |
| **普通按钮逻辑** | 不必 | OB1 足够 |
| **大批量通信** | 谨慎 | 可能占用过多周期 |
| **高速脉冲处理** | 不一定 | 可能需硬件高速功能 |

## ③ 计算/选型示例

- **已知：** 每 `100ms` 计算一次流量平均值  
- **公式：** 固定周期任务 → 循环中断 OB  
- **计算：**
  - 设置中断周期：`100ms`
  - 每次进入 OB 时采集一次数据
  - 更新平均值
- **结论：** 需要稳定时间基准的任务适合用循环中断 OB。

> [!tip] 记忆口诀
> **定时要稳定，循环中断来执行。**

# 05｜时间中断 OB

> [!info] 核心结论
> **时间中断 OB 按指定日期、时间或延时事件触发，适合做定时任务或计划任务。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
时间中断 OB 更像“预约任务”，到了指定时间自动执行。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **类型** | Time-of-day / Time-delay Interrupt OB |
| **调用条件** | 到达指定时间或延时到期 |
| **常见用途** | 定时记录、定时切换、延时事件 |
| **依赖条件** | CPU 时间设置正确 |
| **注意** | 时间相关功能需按 CPU 支持情况校核 |

## ② 参考表

| 应用 | 说明 |
|---|---|
| **每天定时记录** | 固定时间保存数据 |
| **延时执行动作** | 触发后延迟执行 |
| **班次切换** | 根据时间切换状态 |
| **定时清零** | 每天/每班清统计 |
| **定时报警检查** | 周期性维护提醒 |

## ③ 计算/选型示例

- **已知：** 每天 8:00 需要清零班次产量  
- **公式：** 固定时间任务 → 时间中断 OB  
- **计算：**
  - 设置触发时间为 8:00
  - 到点执行清零逻辑
  - 重新开始统计
- **结论：** 与实际时钟绑定的任务适合用时间中断 OB。

> [!tip] 记忆口诀
> **固定周期用循环中断，固定时刻用时间中断。**

# 06｜硬件中断 OB

> [!info] 核心结论
> **硬件中断 OB 由硬件事件触发，适合快速响应外部信号变化或模块事件。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
硬件中断 OB 是 PLC 对外部事件的“快速反应通道”。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **类型** | Hardware Interrupt OB |
| **触发来源** | 输入模块、工艺模块、特殊事件 |
| **常见用途** | 快速计数、边沿捕捉、事件记录 |
| **优势** | 比普通 OB1 扫描更及时 |
| **注意** | 触发源和模块能力需按手册校核 |

## ② 参考表

| 场景 | 是否适合硬件中断 | 说明 |
|---|---|---|
| **高速边沿捕捉** | 适合 | 避免扫描漏检 |
| **普通按钮** | 不必 | OB1 足够 |
| **事件时间戳** | 适合 | 需要快速响应 |
| **频繁抖动信号** | 谨慎 | 可能频繁触发 |
| **高速计数** | 看硬件功能 | 可能用 HSC 更合适 |

## ③ 计算/选型示例

- **已知：** 一个传感器脉冲很短，OB1 可能扫描不到  
- **公式：** 短脉冲事件 → 硬件中断或高速计数  
- **计算：**
  - OB1 扫描周期可能大于脉冲宽度
  - 普通扫描可能漏掉
  - 使用硬件中断可在事件发生时立即响应
- **结论：** 短脉冲或快速事件应考虑硬件中断或高速计数功能。

> [!tip] 记忆口诀
> **慢信号 OB1 看，快事件中断管。**

# 07｜诊断错误 OB

> [!info] 核心结论
> **诊断错误 OB 用于响应模块故障、通道错误、设备诊断等事件，常用于故障记录和安全处理。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>避坑提醒</strong><br>
没有对应错误 OB 时，某些故障可能导致 CPU 停机；具体行为需按 CPU 和故障类型校核。
</div>

## ① 分类/公式/规则

| 类型 | 作用 |
|---|---|
| **诊断错误 OB** | 响应模块/通道诊断事件 |
| **插拔模块 OB** | 响应模块插拔或丢失 |
| **机架/站故障 OB** | 响应分布式站点异常 |
| **编程错误 OB** | 响应程序访问错误等 |
| **时间错误 OB** | 响应周期超时等问题 |

## ② 参考表

| 故障现象 | 可能触发 |
|---|---|
| **模块掉线** | 站故障 / 模块故障 OB |
| **AI断线** | 诊断错误 OB |
| **访问不存在地址** | 编程错误 OB |
| **循环时间过长** | 时间错误 OB |
| **通信设备离线** | 诊断 / 站故障相关 OB |

## ③ 计算/选型示例

- **已知：** 一个远程 I/O 站掉线后，PLC 不希望直接停机，而是记录报警并进入安全状态  
- **公式：** 设备故障事件 → 错误 OB 响应  
- **计算：**
  - 创建对应诊断/站故障 OB
  - 在 OB 中记录故障标志
  - 主程序根据故障标志进入安全逻辑
- **结论：** 错误 OB 可用于故障捕捉和降级处理，具体 OB 类型需按 PLC 型号和诊断事件校核。

> [!tip] 记忆口诀
> **故障来了别硬停，错误 OB 先接住。**

# 08｜OB 执行优先级

> [!info] 核心结论
> **多个 OB 可能同时满足执行条件，CPU 会按优先级调度，高优先级 OB 可以打断低优先级 OB。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
OB 不是谁写在前面谁先执行，而是 CPU 根据事件和优先级调度。
</div>

## ① 分类/公式/规则

| OB 类型 | 一般特点 |
|---|---|
| **启动 OB** | 启动阶段执行 |
| **OB1** | 主循环，持续执行 |
| **循环中断 OB** | 固定周期触发 |
| **硬件中断 OB** | 外部事件触发 |
| **错误 OB** | 故障事件触发 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点规则</strong><br>
高优先级 OB 逻辑不要写太重，否则可能影响其他任务执行。
</div>

## ② 参考表

| 设计项 | 建议 |
|---|---|
| **OB1** | 放普通主流程 |
| **中断 OB** | 放短小、确定的逻辑 |
| **错误 OB** | 记录故障、置位标志 |
| **启动 OB** | 做初始化 |
| **复杂算法** | 尽量由 OB 调用 FC/FB，结构化处理 |

## ③ 计算/选型示例

- **已知：** 循环中断 OB 每 `10ms` 执行，但里面计算任务耗时过长  
- **公式：** 中断周期必须大于任务执行时间并留裕量  
- **计算：**
  - 中断周期：`10ms`
  - 任务执行：如果接近或超过 `10ms`
  - 会造成周期堆积或影响主循环
- **结论：** 中断 OB 中应放短逻辑，复杂运算需优化或降低频率，需按 CPU 性能校核。

> [!tip] 记忆口诀
> **中断要短，主循环要稳；优先级越高，逻辑越要轻。**

# 09｜OB 与 FC / FB / DB 的关系

> [!info] 核心结论
> **OB 是程序入口，FC / FB 是功能逻辑，DB 是数据存储；大型项目通常由 OB 调用 FC / FB 来组织程序。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>结构口诀</strong><br>
OB 管调度，FB 管设备，FC 管算法，DB 管数据。
</div>

## ① 分类/公式/规则

| 块 | 定位 | 示例 |
|---|---|---|
| **OB** | 入口/调度 | `OB1` |
| **FB** | 带状态的功能 | 电机控制 FB |
| **FC** | 无固定状态函数 | 数据换算 FC |
| **DB** | 数据存储 | 参数 DB、实例 DB |

## ② 参考表

| 项目结构 | 推荐做法 |
|---|---|
| **OB1** | 只做主调度 |
| **设备控制** | 写成 FB |
| **公共计算** | 写成 FC |
| **参数/配方** | 放 DB |
| **报警记录** | DB + FB 管理 |

## ③ 计算/选型示例

- **已知：** 项目有 10 台电机需要控制  
- **公式：** 多个相同设备 → 用 FB + 多实例/实例 DB  
- **计算：**
  - 写一个电机控制 FB
  - 每台电机一个实例
  - OB1 中调用 10 次
- **结论：** OB 不建议塞满重复逻辑，应通过 FB/FC 结构化程序。

> [!tip] 记忆口诀
> **OB 只管叫人干活，FB/FC 才是真干活。**

# 10｜一页速记总卡

> [!info] 核心结论
> **组织块 OB 的核心就是“CPU 什么时候执行哪段程序”，它决定 PLC 程序的运行入口、事件响应和调度方式。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
OB 是入口，OB1 是主循环；启动做初始化，中断做快响应，错误 OB 接故障。
</div>

## ① 分类/公式/规则

| OB 类型 | 一句话 |
|---|---|
| **OB1** | 主循环入口 |
| **启动 OB** | 上电/启动初始化 |
| **循环中断 OB** | 固定周期执行 |
| **时间中断 OB** | 指定时间执行 |
| **硬件中断 OB** | 外部事件触发 |
| **错误 OB** | 故障事件响应 |

## ② 参考表

| 需求 | 推荐 OB |
|---|---|
| **普通控制逻辑** | `OB1` |
| **上电清零初始化** | 启动 OB |
| **固定周期采样** | 循环中断 OB |
| **每天定时任务** | 时间中断 OB |
| **快速外部事件** | 硬件中断 OB |
| **模块故障处理** | 诊断/错误 OB |

## ③ 计算/选型示例

- **已知：** 一个项目包含主流程、初始化、周期采样和故障记录  
- **公式：** 不同任务 → 放入不同 OB  
- **计算：**
  - 主流程：`OB1`
  - 初始化：启动 OB
  - 每 `100ms` 采样：循环中断 OB
  - 模块故障：诊断错误 OB
- **结论：** 合理使用 OB 可以让 PLC 程序结构更清晰、响应更稳定、故障更容易排查。

> [!tip] 记忆口诀
> **主程序找 OB1，初始化找启动，中断找周期，故障找错误 OB。**
