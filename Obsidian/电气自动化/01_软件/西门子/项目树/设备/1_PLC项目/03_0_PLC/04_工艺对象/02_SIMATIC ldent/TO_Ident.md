# 01｜TO_Ident 是什么

> [!info] 核心结论
> **TO_Ident 是 TIA Portal 中用于被控对象辨识的功能对象，常用于 PID 调试前识别系统特性，帮助后续整定控制参数。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>图片识别</strong><br>
截图中的 <strong>TO_Ident V1.2</strong> 可以理解为 PID 调试相关的“过程辨识对象”，用于分析温度、压力、流量、液位等系统的响应特性。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **名称** | `TO_Ident` |
| **版本** | `V1.2` |
| **中文理解** | 被控对象辨识 / 过程辨识 |
| **常见用途** | PID 参数整定前的系统识别 |
| **典型对象** | 温度、压力、流量、液位、速度 |
| **核心目的** | 获取过程增益、滞后、时间常数等特性 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
TO_Ident 不是直接控制设备，而是先“摸清系统脾气”，再帮助 PID 调得更稳。
</div>

## ② 参考表

| 对象 | 作用 | 重点 |
|---|---|---|
| **PID_Compact** | 通用连续量 PID | 常配合辨识/整定 |
| **PID_3Step** | 三步执行器 PID | 电动阀、风门 |
| **PID_Temp** | 温度专用 PID | 加热/冷却 |
| **TO_Ident** | 过程辨识 | 分析系统响应 |

## ③ 计算/选型示例

- **已知：** 温控系统升温慢、超调明显，PID 参数不好调  
- **公式：** PID 调不稳 → 先识别过程特性  
- **计算：**
  - 给系统一个小阶跃输出
  - 观察温度反馈变化
  - 识别响应速度、滞后和增益
  - 再根据结果调整 PID 参数
- **结论：** 调 PID 前用 TO_Ident 做过程辨识，可以减少盲目试参数。

> [!tip] 记忆口诀
> **先辨识，再整定；先懂对象，再调 PID。**

# 02｜它和 PID 的关系

> [!info] 核心结论
> **TO_Ident 通常用于 PID 控制前期调试，它负责识别对象特性，PID 负责闭环调节。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>系统链路</strong><br>
过程对象 → TO_Ident 辨识 → 得到对象特性 → PID 参数整定 → PID 闭环控制
</div>

## ① 分类/公式/规则

| 阶段 | 使用对象 | 目的 |
|---|---|---|
| **辨识阶段** | `TO_Ident` | 获取对象响应特性 |
| **整定阶段** | PID 调试工具 / 参数整定 | 计算或优化 PID 参数 |
| **运行阶段** | `PID_Compact` / `PID_Temp` / `PID_3Step` | 正常闭环控制 |
| **维护阶段** | 趋势曲线 / 诊断状态 | 检查控制效果 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>核心区别</strong><br>
TO_Ident 偏“分析系统”，PID 偏“控制系统”。
</div>

## ② 参考表

| 功能 | TO_Ident | PID |
|---|---|---|
| **是否直接闭环控制** | 一般不是 | 是 |
| **核心任务** | 识别过程特性 | 稳定控制输出 |
| **关注点** | 响应曲线 | 设定值与反馈偏差 |
| **输出结果** | 过程模型/辨识信息 | 控制输出 |
| **使用阶段** | 调试/整定前 | 正常运行 |

## ③ 计算/选型示例

- **已知：** 水箱液位控制要用 PID，但不知道 Kp、Ti 怎么设  
- **公式：** 参数未知 → 先做对象辨识  
- **计算：**
  - 先用小输出变化观察液位响应
  - 记录液位上升速度和延迟
  - 根据辨识结果估算 PID 参数
- **结论：** TO_Ident 是 PID 参数调试的辅助工具，不是普通启停控制块。

> [!tip] 记忆口诀
> **TO_Ident 负责看脾气，PID 负责管脾气。**

# 03｜辨识原理

> [!info] 核心结论
> **过程辨识通常通过观察输出变化后过程值的响应曲线，估算对象的增益、滞后和时间常数。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点公式</strong><br>
过程增益 K ≈ 过程值变化量 ΔPV ÷ 输出变化量 ΔOUT
</div>

## ① 分类/公式/规则

| 参数 | 中文含义 | 工程理解 |
|---|---|---|
| **ΔOUT** | 输出变化量 | 给执行器的变化 |
| **ΔPV** | 过程值变化量 | 反馈实际变化 |
| **K** | 过程增益 | 输出变一点，过程量变多少 |
| **Tu** | 滞后时间 | 输出后多久才开始响应 |
| **Tg / T** | 时间常数 | 过程变化快慢 |
| **Noise** | 噪声 | 反馈波动干扰辨识 |

## ② 参考表

| 对象特性 | 表现 | PID 调试影响 |
|---|---|---|
| **增益大** | 小输出引起大变化 | PID 不能太猛 |
| **增益小** | 输出变化不明显 | 需要更强输出 |
| **滞后大** | 反应很慢 | 易超调 |
| **时间常数大** | 上升慢 | 调参要保守 |
| **噪声大** | 曲线抖动 | 需滤波 |

## ③ 计算/选型示例

- **已知：**
  - 输出从 `30%` 增加到 `40%`
  - 温度从 `50℃` 稳定到 `60℃`
- **公式：** `K = ΔPV ÷ ΔOUT`
- **计算：**
  - `ΔOUT = 40% - 30% = 10%`
  - `ΔPV = 60℃ - 50℃ = 10℃`
  - `K ≈ 10℃ ÷ 10% = 1℃/%`
- **结论：** 输出每增加 `1%`，温度最终约升高 `1℃`，实际需按稳定曲线校核。

> [!tip] 记忆口诀
> **看输出变多少，再看反馈变多少；曲线一出来，对象就明白。**

# 04｜关键输入输出

> [!info] 核心结论
> **TO_Ident 需要实际过程值、控制输出、启动/停止辨识命令、状态和错误信息，具体接口以实际块为准。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>信号链路</strong><br>
执行器输出 → 过程对象变化 → 传感器反馈 → TO_Ident 分析响应
</div>

## ① 分类/公式/规则

| 信号类别 | 典型含义 | 作用 |
|---|---|---|
| **Start / Execute** | 开始辨识 | 启动识别流程 |
| **PV / Input** | 过程值 | 实际反馈量 |
| **Output / LMN** | 控制输出 | 给对象的激励 |
| **Mode** | 模式选择 | 手动/自动/辨识 |
| **Busy** | 正在辨识 | 不要重复触发 |
| **Done** | 辨识完成 | 可查看结果 |
| **Error / Status** | 错误诊断 | 判断异常原因 |

> [!warning]
> `TO_Ident V1.2` 的具体接口名称、数据类型和支持功能，需按 TIA Portal 实际块接口与 Siemens 帮助文档校核。

## ② 参考表

| 参数 | 作用 | 注意 |
|---|---|---|
| **过程值 PV** | 被分析的反馈量 | 必须先换算成工程量 |
| **输出变化** | 激励过程对象 | 幅度不能过大 |
| **采样周期** | 记录响应曲线 | 周期要稳定 |
| **输出限幅** | 保护设备 | 防止过热/超压 |
| **状态码** | 诊断辨识状态 | 结合 Error 判断 |

## ③ 计算/选型示例

- **已知：** 压力传感器反馈为原始模拟量 `0~27648`  
- **公式：** PID/辨识前先换算工程量  
- **计算：**
  - 原始值不能直接作为 `bar`
  - 需要换算成压力工程量
  - 再送入 TO_Ident 或 PID
- **结论：** TO_Ident 分析的是过程对象，输入反馈必须真实、稳定、单位正确。

> [!tip] 记忆口诀
> **辨识先看反馈，反馈不准，模型不准。**

# 05｜典型使用流程

> [!info] 核心结论
> **TO_Ident 的典型流程是：准备过程对象、切到安全状态、施加测试输出、采集响应、完成辨识、再整定 PID。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>调试流程</strong><br>
确认安全 → 稳定初始状态 → 开始辨识 → 观察曲线 → 得到结果 → 调整 PID
</div>

## ① 分类/公式/规则

| 步骤 | 动作 | 重点 |
|---|---|---|
| **1** | 检查传感器 | PV 必须可信 |
| **2** | 检查执行器 | 输出方向正确 |
| **3** | 设置安全限幅 | 防止超温/超压 |
| **4** | 稳定初始状态 | 不要一边波动一边辨识 |
| **5** | 启动 TO_Ident | 触发辨识 |
| **6** | 等待 Done | 期间不要乱改对象 |
| **7** | 根据结果整定 PID | 再进入自动控制 |

## ② 参考表

| 条件 | 为什么重要 |
|---|---|
| **反馈稳定** | 曲线不乱 |
| **输出方向正确** | 不会越调越远 |
| **执行器正常** | 激励有效 |
| **对象处于安全范围** | 防止工艺风险 |
| **采样周期稳定** | 辨识结果可信 |
| **无外部扰动** | 减少误差 |

## ③ 计算/选型示例

- **已知：** 要对加热炉做辨识  
- **公式：** 温度对象辨识 = 小输出阶跃 + 温度响应曲线  
- **计算：**
  - 初始输出设为 `20%`
  - 稳定后阶跃到 `30%`
  - 记录温度上升过程
  - 得到滞后、斜率、稳定变化量
- **结论：** 加热炉惯性大，辨识时要耐心等待温度响应，不要频繁修改输出。

> [!tip] 记忆口诀
> **先稳态，再阶跃；先采曲线，再调参数。**

# 06｜适合与不适合场景

> [!info] 核心结论
> **TO_Ident 适合连续过程控制对象，不适合普通开关逻辑、气缸动作、伺服定位这类离散控制。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>选型判断</strong><br>
有连续反馈、有连续输出、有动态过程，才适合做辨识。
</div>

## ① 分类/公式/规则

| 场景 | 是否适合 | 原因 |
|---|---|---|
| **温度控制** | 适合 | 慢过程、有反馈 |
| **压力控制** | 适合 | 连续量调节 |
| **流量控制** | 适合 | 阀门/泵调节 |
| **液位控制** | 适合 | 有过程响应 |
| **电机启停** | 不适合 | 开关逻辑 |
| **气缸伸缩** | 不适合 | 离散动作 |
| **伺服定位** | 不适合 | 运动控制对象 |

## ② 参考表

| 控制对象 | 推荐工具 |
|---|---|
| **温度闭环** | `PID_Temp` + 辨识/整定 |
| **压力/流量/液位** | `PID_Compact` + 辨识 |
| **三步电动阀** | `PID_3Step` |
| **伺服轴** | `TO_PositioningAxis` |
| **多步运动** | `TO_CommandTable` |
| **开关量设备** | FB 顺序控制 |

## ③ 计算/选型示例

- **已知：** 一个气缸只有伸出和缩回两个动作  
- **公式：** 离散动作 ≠ 连续过程辨识  
- **计算：**
  - 没有连续输出百分比
  - 没有连续过程响应曲线
  - 不属于 PID 闭环对象
- **结论：** 气缸控制不适合用 TO_Ident，应使用 FB 顺序控制。

> [!tip] 记忆口诀
> **连续过程可辨识，开关动作不用它。**

# 07｜常见问题排查

> [!info] 核心结论
> **TO_Ident 辨识异常时，优先检查反馈是否可信、输出是否有效、方向是否正确、过程是否有扰动。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>排查顺序</strong><br>
反馈值 → 输出方向 → 执行器动作 → 安全限幅 → 采样周期 → 外部扰动 → Status。
</div>

## ① 分类/公式/规则

| 问题 | 常见原因 |
|---|---|
| **辨识不启动** | 启动信号无上升沿、模式不对 |
| **Busy 一直为 1** | 过程未稳定、响应太慢 |
| **Error=1** | 参数非法或状态不满足 |
| **结果不可信** | 反馈噪声大、外部扰动大 |
| **过程无变化** | 执行器未动作或输出方向错误 |
| **超限停止** | 输出幅度过大或安全边界触发 |

## ② 参考表

| 现象 | 排查方向 | 解决建议 |
|---|---|---|
| **PV 不变化** | 传感器/量程/执行器 | 先修正反馈和输出 |
| **PV 反向变化** | 控制方向错 | 先手动确认方向 |
| **曲线抖动很大** | 噪声/干扰 | 加滤波，查接地 |
| **辨识时间过长** | 对象惯性大 | 延长等待或降低要求 |
| **频繁中断** | 安全限幅太紧 | 合理设置边界 |
| **Status 异常** | 块状态错误 | 按帮助文档查码 |

## ③ 计算/选型示例

- **已知：** 输出从 `30%` 到 `40%`，但 PV 一直不变  
- **公式：** 输出变化后 PV 无变化 → 激励无效或反馈无效  
- **计算：**
  - 执行器可能没动作
  - 输出通道可能没接好
  - 传感器可能断线
  - 过程对象可能被其他控制干扰
- **结论：** 先排查硬件和反馈链路，再判断 TO_Ident 本身。

> [!tip] 记忆口诀
> **输出变了反馈不变，先查执行器和传感器。**

# 08｜一页速记总卡

> [!info] 核心结论
> **TO_Ident 的核心价值是识别过程对象，让 PID 参数整定不再完全靠试。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
先看反馈准不准，再看输出动不动；先辨识对象，再整定 PID。
</div>

## ① 分类/公式/规则

| 主轴 | 关键词 |
|---|---|
| **对象** | `TO_Ident V1.2` |
| **用途** | 被控对象辨识 |
| **输入** | 过程值 PV |
| **输出/激励** | 控制输出变化 |
| **结果** | 过程增益、滞后、时间常数等 |
| **用途延伸** | PID 参数整定 |

## ② 参考表

| 需求 | 推荐 |
|---|---|
| **温度 PID 调试** | `TO_Ident` + `PID_Temp` |
| **压力/液位 PID 调试** | `TO_Ident` + `PID_Compact` |
| **三步阀调节** | `PID_3Step` |
| **普通设备启停** | FB 控制 |
| **伺服轴定位** | TO / MC 运动控制 |
| **公式换算** | FC |

## ③ 计算/选型示例

- **已知：** 一个液位 PID 长期调不稳  
- **公式：** PID 难调 = 先确认对象特性  
- **计算：**
  - 检查液位反馈是否稳定
  - 用小输出变化观察液位响应
  - 通过 TO_Ident 获取对象响应信息
  - 再优化 PID 参数
- **结论：** TO_Ident 适合用于 PID 调试前的“摸底测试”。

> [!tip] 记忆口诀
> **不懂对象别乱调，TO_Ident 先帮你看一看。**
