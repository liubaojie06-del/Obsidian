# 01｜PID_3Step 是什么

> [!info] 核心结论
> **PID_3Step 是 TIA Portal 中用于三步执行器的 PID 控制块，常用于电动调节阀、电动风门、执行机构的“开-停-关”闭环调节。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>图片识别</strong><br>
截图中的 <strong>PID_3Step V2.3</strong> 是西门子 PLC 里的三步控制 PID 功能块。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **指令名称** | `PID_3Step` |
| **版本** | `V2.3` |
| **中文含义** | 三步 PID 控制器 |
| **控制对象** | 电动阀、电动风门、电动执行器 |
| **输出形式** | 开方向输出 / 关方向输出 |
| **控制特点** | 不是连续模拟量，而是通过开关脉冲调节执行器位置 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
PID_3Step 不直接输出 0-10V，而是告诉阀门：开一点、停一下、关一点。
</div>

## ② 参考表

| 对比项 | PID_Compact | PID_3Step |
|---|---|---|
| **输出类型** | 连续量输出 | 开/关两路输出 |
| **典型执行器** | 变频器、调节阀、加热器 | 电动阀、电动风门 |
| **输出信号** | `0~100%`、模拟量、PWM | `Open` / `Close` |
| **适合对象** | 连续可调设备 | 三位式执行机构 |
| **控制效果** | 连续调节 | 分步调节 |

## ③ 计算/选型示例

- **已知：** 一个电动调节阀只有“开阀”和“关阀”两个数字量输入，没有模拟量输入  
- **公式：** 执行器只有开/关方向控制 → 选三步控制  
- **计算：**
  - PLC 输出 `OpenCmd`：阀门向开方向动作
  - PLC 输出 `CloseCmd`：阀门向关方向动作
  - 两个都不输出：阀门停止
- **结论：** 这种电动阀适合使用 `PID_3Step`，不适合直接用普通模拟量 PID 输出。

> [!tip] 记忆口诀
> **模拟量阀用 Compact，开关型电动阀用 3Step。**

# 02｜三步控制原理

> [!info] 核心结论
> **三步控制的本质是根据偏差大小，决定执行器向“开方向”“关方向”动作，或保持停止。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>三步口诀</strong><br>
偏低就开，偏高就关，到位就停。
</div>

## ① 分类/公式/规则

| 偏差状态 | 控制动作 | 输出状态 |
|---|---|---|
| **实际值低于设定值** | 增大控制量 | `Open = 1` |
| **实际值高于设定值** | 减小控制量 | `Close = 1` |
| **接近设定值** | 保持当前位置 | `Open = 0`，`Close = 0` |
| **故障/禁止** | 停止动作 | 输出关闭 |

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>控制链路</strong><br>
设定值 → 实际值反馈 → PID_3Step → 开/关脉冲 → 电动执行器 → 过程量变化
</div>

## ② 参考表

| 信号 | 中文含义 | 工程理解 |
|---|---|---|
| **Setpoint** | 设定值 | 想要达到的目标 |
| **Input / PV** | 实际值 | 传感器反馈 |
| **Error** | 偏差 | 目标与实际的差 |
| **Open 输出** | 开方向命令 | 阀门开大 |
| **Close 输出** | 关方向命令 | 阀门关小 |
| **Deadband** | 死区 | 小偏差不动作 |

## ③ 计算/选型示例

- **已知：**
  - 温度设定：`80℃`
  - 当前温度：`75℃`
- **公式：** 偏差 = `设定值 - 实际值`
- **计算：**
  - 偏差 = `80 - 75 = 5℃`
  - 温度偏低，需要增加加热或开大热水阀
  - PID_3Step 可能输出开阀脉冲
- **结论：** 当实际值低于目标值时，控制器会让执行器朝“增大过程量”的方向动作。

> [!tip] 记忆口诀
> **差多少算多少，开多久看偏差；快到目标就停下。**

# 03｜适合什么设备

> [!info] 核心结论
> **PID_3Step 适合只能通过开/关方向信号调节位置的执行器，尤其是电动调节阀和电动风门。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程定位</strong><br>
只要执行器不是模拟量控制，而是“开一点、关一点、停住”，就可以考虑 PID_3Step。
</div>

## ① 分类/公式/规则

| 设备 | 是否适合 PID_3Step | 原因 |
|---|---|---|
| **电动调节阀** | 适合 | 常见开/关方向控制 |
| **电动风门** | 适合 | 通过开度调节风量 |
| **三位式执行器** | 适合 | 开-停-关 |
| **模拟量调节阀** | 不优先 | 更适合 `PID_Compact` |
| **变频器调速** | 不适合 | 更适合连续量 PID |
| **普通气缸** | 不适合 | 不是连续调节对象 |

## ② 参考表

| 应用场景 | 控制目标 | 执行器动作 |
|---|---|---|
| **温度控制** | 控制热水阀开度 | 开阀/关阀 |
| **压力控制** | 控制泄压阀或进气阀 | 开/关调节 |
| **风量控制** | 控制风门角度 | 开大/关小 |
| **液位控制** | 控制进水阀开度 | 开/关调节 |
| **流量控制** | 调节阀门开度 | 分步调整 |

## ③ 计算/选型示例

- **已知：** 风门执行器有三根控制线：公共端、开方向、关方向  
- **公式：** 三线开关型执行器 → 三步控制  
- **计算：**
  - 开方向得电：风门开大
  - 关方向得电：风门关小
  - 两路都不得电：风门停止
- **结论：** 该风门属于典型三步执行器，适合用 `PID_3Step` 控制。

> [!tip] 记忆口诀
> **有开有关有停止，就是三步执行器。**

# 04｜关键输入输出

> [!info] 核心结论
> **PID_3Step 的核心接口是设定值、实际值、手自动模式、开方向输出、关方向输出、状态和错误诊断。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>信号链路</strong><br>
Setpoint / Input → PID_3Step → Open / Close → 执行器动作 → 过程值反馈
</div>

## ① 分类/公式/规则

| 接口类别 | 典型含义 | 作用 |
|---|---|---|
| **设定输入** | Setpoint | 目标值 |
| **反馈输入** | Input / PV | 实际测量值 |
| **模式控制** | Auto / Manual | 自动或手动 |
| **手动控制** | ManualOpen / ManualClose | 手动开关阀 |
| **控制输出** | Open / Close | 控制执行器方向 |
| **诊断输出** | Error / Status | 故障判断 |

> [!warning]
> 不同 CPU、TIA Portal 版本和 `PID_3Step` 版本中接口名称可能略有差异，实际项目需按块接口和帮助文档校核。

## ② 参考表

| 参数 | 作用 | 注意 |
|---|---|---|
| **Setpoint** | 目标值 | 单位要和反馈一致 |
| **Input** | 实际值 | 需先做工程量换算 |
| **Open 输出** | 开方向动作 | 不能与 Close 同时有效 |
| **Close 输出** | 关方向动作 | 需要互锁保护 |
| **Manual** | 手动模式 | 调试时先验证方向 |
| **Error** | 错误状态 | 结合状态码判断 |

## ③ 计算/选型示例

- **已知：** 液位设定值为 `500mm`，液位实际值为 `450mm`
- **公式：** 偏差 = `Setpoint - Input`
- **计算：**
  - 偏差 = `500 - 450 = 50mm`
  - 液位偏低
  - 进水阀应朝开方向动作
- **结论：** 若控制对象是进水阀，PID_3Step 应输出开阀方向脉冲。

> [!tip] 记忆口诀
> **设定和反馈同单位，开关输出才有意义。**

# 05｜死区与脉冲控制

> [!info] 核心结论
> **PID_3Step 通常会设置死区、最小脉冲时间、最小暂停时间，防止阀门频繁来回动作。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点规则</strong><br>
三步执行器不能一直频繁点动，否则容易磨损执行机构或造成控制振荡。
</div>

## ① 分类/公式/规则

| 参数 | 中文含义 | 作用 |
|---|---|---|
| **Deadband** | 死区 | 小偏差不动作 |
| **Min Pulse Time** | 最小脉冲时间 | 避免输出太短无效 |
| **Min Pause Time** | 最小暂停时间 | 避免频繁动作 |
| **Actuator Runtime** | 执行器全行程时间 | 估算开度变化 |
| **Hysteresis** | 回差 | 防止临界点抖动 |

## ② 参考表

| 设置项 | 太小的风险 | 太大的影响 |
|---|---|---|
| **死区太小** | 阀门频繁动作 | 控制更精细但易抖 |
| **死区太大** | 控制不够精确 | 稳定但偏差大 |
| **脉冲太短** | 执行器不响应 | 输出无效 |
| **暂停太短** | 频繁开关 | 机械磨损 |
| **全行程时间错误** | 开度估算不准 | 控制效果变差 |

## ③ 计算/选型示例

- **已知：**
  - 执行器从全关到全开时间：`60s`
  - 输出开阀脉冲：`3s`
- **公式：** 开度变化 ≈ `脉冲时间 ÷ 全行程时间 × 100%`
- **计算：**
  - `3 ÷ 60 × 100% = 5%`
- **结论：** 一个 `3s` 开阀脉冲大约让阀门增加 `5%` 开度，实际需按执行器特性校核。

> [!tip] 记忆口诀
> **小偏差不动作，大偏差才点动；脉冲别太短，暂停别太急。**

# 06｜执行器全行程时间

> [!info] 核心结论
> **执行器全行程时间是 PID_3Step 的关键参数，它决定控制器如何估算开阀或关阀动作对过程量的影响。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
全行程时间就是阀门从 0% 开到 100% 需要多久。
</div>

## ① 分类/公式/规则

| 参数 | 说明 |
|---|---|
| **全开时间** | 从全关到全开所需时间 |
| **全关时间** | 从全开到全关所需时间 |
| **执行速度** | 单位时间开度变化 |
| **开度估算** | 根据输出时间估算阀位 |
| **实际校准** | 需按执行器铭牌或实测校核 |

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点公式</strong><br>
开度变化百分比 ≈ 动作时间 ÷ 全行程时间 × 100%
</div>

## ② 参考表

| 全行程时间 | 1s 动作约等于 | 说明 |
|---:|---:|---|
| **30s** | `3.33%` | 动作较快 |
| **60s** | `1.67%` | 常见中速 |
| **120s** | `0.83%` | 动作较慢 |
| **180s** | `0.56%` | 慢速执行器 |

## ③ 计算/选型示例

- **已知：**
  - 执行器全行程时间：`120s`
  - 关阀输出时间：`6s`
- **公式：** 开度变化 = `动作时间 ÷ 全行程时间 × 100%`
- **计算：**
  - `6 ÷ 120 × 100% = 5%`
- **结论：** `6s` 关阀动作大约减少 `5%` 开度，实际开度需按阀门反馈或现场标定确认。

> [!tip] 记忆口诀
> **阀门慢，脉冲长；阀门快，脉冲短。**

# 07｜手动与自动调试

> [!info] 核心结论
> **PID_3Step 调试必须先手动验证开/关方向，再切自动闭环，否则方向反了会越调越偏。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>避坑提醒</strong><br>
开阀输出和关阀输出绝不能同时动作，程序和硬件都要做好互锁。
</div>

## ① 分类/公式/规则

| 调试步骤 | 目的 |
|---|---|
| **手动开阀** | 确认 Open 输出方向正确 |
| **手动关阀** | 确认 Close 输出方向正确 |
| **观察过程值** | 判断过程量变化方向 |
| **检查限位** | 防止全开/全关过冲 |
| **切自动** | 进入闭环调节 |

## ② 参考表

| 现象 | 可能原因 | 处理 |
|---|---|---|
| **输出开阀，过程值反而下降** | 控制方向反 | 调整控制方向 |
| **开关同时输出** | 逻辑错误 | 增加互锁 |
| **阀门不动作** | 接线/电源/执行器故障 | 查输出和执行器 |
| **动作很频繁** | 死区太小 | 增大死区/暂停 |
| **动作太慢** | 脉冲太短或执行器慢 | 调整参数 |

## ③ 计算/选型示例

- **已知：** 手动开阀后，压力不是升高而是降低  
- **公式：** 手动方向测试 = 验证控制作用方向  
- **计算：**
  - 预期：开阀后压力升高
  - 实际：压力降低
  - 说明阀门作用方向或工艺理解可能相反
- **结论：** 自动运行前必须先修正方向，否则 PID 会越调越远。

> [!tip] 记忆口诀
> **先手动看方向，再自动看稳定。**

# 08｜参数整定

> [!info] 核心结论
> **PID_3Step 的整定目标是让过程量稳定接近设定值，同时减少阀门频繁动作和超调。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>整定口诀</strong><br>
先稳住，再调快；先少动，再精准。
</div>

## ① 分类/公式/规则

| 参数 | 影响 | 调整方向 |
|---|---|---|
| **Kp** | 响应速度 | 太小慢，太大振 |
| **Ti** | 静差消除 | 太强容易频繁动作 |
| **Td** | 抑制趋势 | 噪声大时慎用 |
| **Deadband** | 动作频率 | 太小频繁，太大偏差 |
| **Min Pulse** | 动作有效性 | 太短可能阀不动 |

## ② 参考表

| 现象 | 可能原因 | 调整方向 |
|---|---|---|
| **响应太慢** | Kp 偏小 / 脉冲太短 | 增大 Kp 或脉冲 |
| **来回振荡** | Kp 太大 / 死区太小 | 降 Kp 或增死区 |
| **阀门频繁点动** | 死区太小 / 积分太强 | 增大死区，减弱积分 |
| **长期有偏差** | 积分不足 | 增强积分作用 |
| **输出不动作** | 最小脉冲不合理 | 调整最小脉冲时间 |

## ③ 计算/选型示例

- **已知：** 阀门每隔几秒就开一下、关一下，过程值在设定值附近抖动  
- **公式：** 频繁动作 = 死区太小或参数太激进  
- **计算：**
  - 增大死区
  - 适当减小 Kp
  - 检查最小脉冲和暂停时间
- **结论：** 三步控制要兼顾精度和执行器寿命，不能只追求偏差最小。

> [!tip] 记忆口诀
> **频繁动作加死区，振荡明显降增益。**

# 09｜常见应用场景

> [!info] 核心结论
> **PID_3Step 适合温度、压力、流量、液位等连续过程，但执行器必须是三步式开关执行机构。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>应用口诀</strong><br>
温压流液都能调，关键看执行器是不是三步式。
</div>

## ① 分类/公式/规则

| 场景 | 过程值 | 执行器 |
|---|---|---|
| **温度控制** | 温度 | 热水阀 / 冷却阀 |
| **压力控制** | 压力 | 电动调节阀 |
| **流量控制** | 流量 | 电动阀 |
| **液位控制** | 液位 | 进水阀 / 出水阀 |
| **风量控制** | 风量/风压 | 电动风门 |

## ② 参考表

| 不适合场景 | 原因 |
|---|---|
| **普通电磁阀开关** | 只有开/关，不适合连续调节 |
| **变频器调速** | 更适合连续量 PID |
| **模拟量调节阀** | 更适合 `PID_Compact` |
| **气缸动作** | 离散顺序控制 |
| **伺服定位** | 应用运动控制指令 |

## ③ 计算/选型示例

- **已知：** 一台电动阀用于控制热水流量，阀门有开阀和关阀两个输入  
- **公式：** 热水流量连续调节 + 三步执行器 → PID_3Step  
- **计算：**
  - 温度低时开大热水阀
  - 温度高时关小热水阀
  - 接近设定值时停止动作
- **结论：** 该场景适合使用 `PID_3Step` 做温度闭环控制。

> [!tip] 记忆口诀
> **过程是连续的，执行器是三步的，就用 PID_3Step。**

# 10｜常见故障排查

> [!info] 核心结论
> **PID_3Step 排查顺序：先看反馈量程，再看开关方向，再看死区脉冲，最后再调 PID 参数。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>排查顺序</strong><br>
反馈值 → 输出方向 → 开关互锁 → 全行程时间 → 死区脉冲 → PID 参数。
</div>

## ① 分类/公式/规则

| 问题 | 常见原因 |
|---|---|
| **阀门不动** | 输出未接通、电源故障、执行器故障 |
| **越调越远** | 开/关方向反了 |
| **频繁动作** | 死区太小、参数太激进 |
| **控制很慢** | 执行器全行程太长、脉冲太短 |
| **到设定值附近抖动** | 死区、回差、Kp 不合理 |
| **Error 报错** | 参数非法或块状态异常 |

## ② 参考表

| 现象 | 排查方向 | 解决建议 |
|---|---|---|
| **Open 有输出但阀不动** | 输出模块/接线/执行器 | 查 DO 和电源 |
| **Close 后过程值反向变化** | 方向或工艺理解错误 | 手动验证方向 |
| **Open/Close 同时亮** | 逻辑互锁错误 | 增加硬件/软件互锁 |
| **过程值不准** | 传感器量程错误 | 先修正反馈 |
| **动作太频繁** | 死区太小 | 增大死区 |
| **长时间不到目标** | 执行器能力不足 | 查阀门、工艺和参数 |

## ③ 计算/选型示例

- **已知：** 设定压力 `5bar`，实际压力一直显示 `0bar`
- **公式：** 反馈异常时，不能先调 PID
- **计算：**
  - 传感器可能断线
  - 模拟量通道量程可能错误
  - 工程量换算可能错误
- **结论：** 先修正压力反馈，再整定 PID_3Step。

> [!tip] 记忆口诀
> **三步 PID 不稳，先查反馈和方向。**

# 11｜一页速记总卡

> [!info] 核心结论
> **PID_3Step 的核心是：用 PID 算法控制三步执行器，通过 Open / Close 两路输出逐步调节过程量。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
三步控制三句话：偏低开一点，偏高关一点，接近就停住。
</div>

## ① 分类/公式/规则

| 主轴 | 关键词 |
|---|---|
| **对象** | `PID_3Step V2.3` |
| **控制方式** | 三步式开/停/关 |
| **输入** | Setpoint、Input |
| **输出** | Open、Close |
| **关键参数** | Kp、Ti、Td、Deadband、Pulse、Runtime |
| **典型对象** | 电动阀、电动风门、三位执行器 |

## ② 参考表

| 需求 | 推荐 |
|---|---|
| **模拟量连续输出** | `PID_Compact` |
| **三步电动阀** | `PID_3Step` |
| **普通开关阀** | 顺序控制 / FB |
| **变频器调速** | `PID_Compact` |
| **伺服定位** | 运动控制 TO / MC 指令 |
| **风门开度调节** | `PID_3Step` |

## ③ 计算/选型示例

- **已知：** 一个电动风门全行程 `90s`，希望每次微调约 `2%`
- **公式：** 脉冲时间 = `全行程时间 × 目标开度变化`
- **计算：**
  - `90s × 2% = 1.8s`
- **结论：** 可把最小有效动作时间初步设在约 `1.8s` 附近，最终需按执行器实际响应和厂家手册校核。

> [!tip] 记忆口诀
> **Compact 出连续量，3Step 出开关量；阀门慢慢调，死区保护好。**
