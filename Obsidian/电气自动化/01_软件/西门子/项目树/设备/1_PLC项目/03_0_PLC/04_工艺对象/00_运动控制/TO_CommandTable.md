# 01｜TO_CommandTable 是什么

> [!info] 核心结论
> **TO_CommandTable 是 TIA Portal 运动控制里的“命令表工艺对象”，用于把一组运动命令按顺序整理成表格，让轴按预设步骤自动执行。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>图片识别</strong><br>
截图中的 <strong>TO_CommandTable V8.0</strong> 是命令表 Technology Object，通常配合定位轴和运动控制指令使用。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **英文名称** | `TO_CommandTable` |
| **中文含义** | 命令表工艺对象 |
| **版本** | `V8.0` |
| **所属类别** | TIA Portal 运动控制 / Technology Object |
| **核心作用** | 预先配置一串运动命令 |
| **常见搭配** | `TO_PositioningAxis`、`MC_CommandTable`、`MC_Power`、`MC_Home` |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
TO_CommandTable 就是“运动步骤表”：第 1 步去哪，第 2 步等多久，第 3 步再去哪。
</div>

## ② 参考表

| 对象 | 作用 | 关系 |
|---|---|---|
| **TO_PositioningAxis** | 定义轴 | 负责轴参数 |
| **TO_CommandTable** | 定义动作表 | 负责动作顺序 |
| **MC_CommandTable** | 执行命令表 | 负责启动执行 |
| **MC_Power** | 轴使能 | 执行前必须满足 |
| **MC_Home** | 回零 | 绝对定位前通常需要 |

## ③ 计算/选型示例

- **已知：** 滑台需要自动执行“到 100mm → 到 200mm → 回 0mm”
- **公式：** 多步固定动作流程 → 命令表
- **计算：**
  - 第 1 行：绝对定位到 `100mm`
  - 第 2 行：绝对定位到 `200mm`
  - 第 3 行：绝对定位到 `0mm`
  - 程序中用 `MC_CommandTable` 启动执行
- **结论：** 多个固定运动步骤适合用 `TO_CommandTable` 管理。

> [!tip] 记忆口诀
> **单步运动用 Move，多步流程用 CommandTable。**

# 02｜它和定位轴的关系

> [!info] 核心结论
> **TO_CommandTable 本身不直接驱动电机，它需要绑定或配合一个轴对象，由轴对象负责真正运动。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>系统链路</strong><br>
命令表 → 运动控制指令 → 定位轴 → 驱动器 → 电机 → 机械负载
</div>

## ① 分类/公式/规则

| 层级 | 对象 | 作用 |
|---|---|---|
| **命令层** | `TO_CommandTable` | 保存动作步骤 |
| **执行层** | `MC_CommandTable` | 启动/执行命令表 |
| **轴对象层** | `TO_PositioningAxis` | 管理轴参数 |
| **驱动层** | 伺服/步进/驱动器 | 输出运动 |
| **机械层** | 丝杆/同步带/转盘 | 实际动作 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>核心规则</strong><br>
命令表只负责“动作顺序”，轴对象负责“怎么动”，驱动器负责“实际输出”。
</div>

## ② 参考表

| 条件 | 是否必须 | 说明 |
|---|---|---|
| **轴已组态** | **是** | 需要定位轴对象 |
| **轴已使能** | **是** | 通常先执行 `MC_Power` |
| **轴已回零** | 视情况 | 绝对定位通常需要 |
| **命令表已配置** | **是** | 至少要有有效命令 |
| **安全条件满足** | **是** | 急停、限位、安全门等 |

## ③ 计算/选型示例

- **已知：** 命令表中第一步是移动到 `100mm`
- **公式：** 绝对定位 = 轴已参考 + 目标位置
- **计算：**
  - 若轴未回零，当前位置基准不确定
  - 执行绝对定位可能报错或不允许执行
  - 应先 `MC_Home` 建立坐标
- **结论：** 使用命令表前，要先确认轴状态满足执行条件。

> [!tip] 记忆口诀
> **表里写动作，轴上去执行；轴没准备好，表也跑不了。**

# 03｜命令表能做什么

> [!info] 核心结论
> **命令表适合把多个运动步骤按顺序执行，例如定位、相对移动、速度运行、等待、停止等。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>使用定位</strong><br>
它更像“运动流程表”，适合固定节拍、重复动作、简单工艺路径。
</div>

## ① 分类/公式/规则

| 命令类型 | 中文理解 | 常见用途 |
|---|---|---|
| **绝对定位** | 去指定坐标 | 到 `100mm`、到 `0°` |
| **相对定位** | 基于当前位置移动 | 再走 `20mm` |
| **速度运行** | 按速度连续运行 | 输送、点动 |
| **等待命令** | 等时间或条件 | 停顿、节拍 |
| **停止/暂停** | 中断当前动作 | 安全或流程控制 |
| **结束命令** | 表执行完成 | 流程收尾 |

> [!warning]
> 不同 CPU、TIA Portal 版本和工艺对象版本支持的具体命令类型可能不同，需按实际帮助文档和厂家手册校核。

## ② 参考表

| 工艺需求 | 推荐方式 |
|---|---|
| **单次去一个位置** | `MC_MoveAbsolute` |
| **当前位置再走一段** | `MC_MoveRelative` |
| **连续按速度运行** | `MC_MoveVelocity` |
| **固定多步动作** | `TO_CommandTable` |
| **复杂条件流程** | 状态机 + MC 指令 |

## ③ 计算/选型示例

- **已知：** 一根轴需要固定执行 5 个动作点
- **公式：** 固定多点动作 → 命令表优先
- **计算：**
  - 点位 1：`0mm`
  - 点位 2：`50mm`
  - 点位 3：`100mm`
  - 点位 4：`50mm`
  - 点位 5：`0mm`
- **结论：** 这种固定轨迹可用命令表减少程序里大量重复 Move 指令。

> [!tip] 记忆口诀
> **点位少用 Move，点位多用表。**

# 04｜命令表关键参数

> [!info] 核心结论
> **命令表里的每一行命令通常都要关注命令类型、目标值、速度、加速度、减速度、等待条件等关键参数。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>参数口诀</strong><br>
类型定动作，位置定目标，速度定快慢，加减速定平稳。
</div>

## ① 分类/公式/规则

| 参数 | 中文含义 | 作用 |
|---|---|---|
| **Command** | 命令类型 | 决定做什么 |
| **Position** | 目标位置 | 决定去哪里 |
| **Distance** | 相对距离 | 决定再走多少 |
| **Velocity** | 速度 | 决定运动快慢 |
| **Acceleration** | 加速度 | 决定启动快慢 |
| **Deceleration** | 减速度 | 决定停止快慢 |
| **Jerk** | 加加速度 | 决定运动柔顺性 |
| **Wait Time / Condition** | 等待时间/条件 | 决定何时进入下一步 |

## ② 参考表

| 参数设置 | 过大风险 | 过小影响 |
|---|---|---|
| **速度过大** | 冲击、跟随误差 | 节拍慢 |
| **加速度过大** | 抖动、过流 | 起步慢 |
| **减速度过大** | 过压、冲击 | 停止慢 |
| **等待时间过短** | 机构未稳定 | 节拍变长 |
| **目标位置错误** | 跑偏/撞机 | 动作不到位 |

## ③ 计算/选型示例

- **已知：**
  - 目标速度：`300 mm/s`
  - 加速时间：`0.6 s`
- **公式：** 加速度 = `速度 ÷ 时间`
- **计算：**
  - `a = 300 ÷ 0.6`
  - `a = 500 mm/s²`
- **结论：** 命令表中加速度可先按 `500 mm/s²` 估算，最终需按机械能力和驱动器能力校核。

> [!tip] 记忆口诀
> **先低速试表，再逐步提速；参数不稳，动作必抖。**

# 05｜典型执行流程

> [!info] 核心结论
> **命令表执行前要先完成轴使能、回零、安全确认，然后再用执行指令启动命令表。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>执行流程</strong><br>
安全确认 → 轴使能 → 回零 → 启动命令表 → 监控状态 → 异常复位
</div>

## ① 分类/公式/规则

| 步骤 | 动作 | 重点 |
|---|---|---|
| **1** | 创建定位轴 | 配好轴参数 |
| **2** | 创建命令表 | 填写动作步骤 |
| **3** | 轴使能 | `MC_Power` |
| **4** | 回零 | `MC_Home` |
| **5** | 启动表 | `MC_CommandTable` |
| **6** | 监控状态 | Busy、Done、Error |
| **7** | 故障处理 | Stop / Reset |

## ② 参考表

| 信号 | 含义 | 处理 |
|---|---|---|
| **Execute** | 启动命令表 | 用上升沿 |
| **Busy** | 正在执行 | 不要重复启动 |
| **Done** | 表执行完成 | 可进入下一流程 |
| **Error** | 出错 | 查 ErrorID |
| **ErrorID** | 错误编号 | 按帮助诊断 |
| **CommandIndex** | 当前执行行 | 用于定位故障步骤 |

> [!warning]
> 实际接口名称可能因 TIA 版本、CPU 类型和指令版本略有差异，需按实际块接口校核。

## ③ 计算/选型示例

- **已知：** 命令表执行过程中卡在第 3 步
- **公式：** 故障定位 = 当前命令行 + ErrorID + 轴状态
- **计算：**
  - 查看当前执行行号
  - 查看 `ErrorID`
  - 查看轴是否超限、未回零、驱动报警
- **结论：** 命令表调试时要监控当前步骤号，否则很难知道卡在哪一行。

> [!tip] 记忆口诀
> **执行看 Busy，完成看 Done，故障看 ErrorID，卡步看行号。**

# 06｜命令表示例

> [!info] 核心结论
> **命令表最典型的用法，是把一组固定点位动作做成表格，然后一次启动自动执行。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>示例场景</strong><br>
滑台执行：回到 0mm → 到 100mm → 等 1s → 到 200mm → 回 0mm
</div>

## ① 分类/公式/规则

| 行号 | 命令 | 参数 |
|---:|---|---|
| **1** | 绝对定位 | `0 mm` |
| **2** | 绝对定位 | `100 mm` |
| **3** | 等待 | `1 s` |
| **4** | 绝对定位 | `200 mm` |
| **5** | 绝对定位 | `0 mm` |
| **6** | 结束 | 表完成 |

## ② 参考表

| 行号 | 速度 | 加速度 | 说明 |
|---:|---:|---:|---|
| **1** | `100 mm/s` | `300 mm/s²` | 回起点 |
| **2** | `200 mm/s` | `500 mm/s²` | 去工位 1 |
| **3** | - | - | 等待工艺 |
| **4** | `200 mm/s` | `500 mm/s²` | 去工位 2 |
| **5** | `150 mm/s` | `400 mm/s²` | 回原点 |

## ③ 计算/选型示例

- **已知：**
  - 从 `100mm` 到 `200mm`
  - 速度 `200 mm/s`
- **公式：** 匀速阶段估算时间 = `距离 ÷ 速度`
- **计算：**
  - 距离：`200 - 100 = 100mm`
  - 时间：`100 ÷ 200 = 0.5s`
  - 实际还要加上加减速时间
- **结论：** 理论匀速时间约 `0.5s`，实际节拍需加上加减速和等待时间。

> [!tip] 记忆口诀
> **表格排动作，程序管启动；点位写清楚，节拍好维护。**

# 07｜适合与不适合场景

> [!info] 核心结论
> **TO_CommandTable 适合固定动作流程，不适合特别复杂、强条件分支、多轴插补的运动逻辑。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>选型判断</strong><br>
流程固定、点位固定、顺序固定，用命令表很舒服；逻辑多变，建议用状态机。
</div>

## ① 分类/公式/规则

| 场景 | 是否适合命令表 | 原因 |
|---|---|---|
| **固定点位搬运** | 适合 | 顺序稳定 |
| **往复运动** | 适合 | 动作重复 |
| **测试循环** | 适合 | 表格易维护 |
| **简单送料流程** | 适合 | 点位明确 |
| **复杂条件分支** | 不太适合 | 状态机更清晰 |
| **多轴轨迹插补** | 不适合或需专用功能 | 需专门运动控制 |

## ② 参考表

| 需求 | 推荐方式 |
|---|---|
| **固定多点位** | `TO_CommandTable` |
| **单次定位** | `MC_MoveAbsolute` |
| **复杂工艺判断** | 状态机 + MC 指令 |
| **多轴同步** | 同步/插补类工艺对象 |
| **临时可变目标** | 程序动态指令更灵活 |

## ③ 计算/选型示例

- **已知：** 轴要根据扫码结果去不同位置，位置组合很多
- **公式：** 路径不固定 → 状态机或动态 Move 更灵活
- **计算：**
  - 每次目标位置由扫码决定
  - 命令表固定路径不够灵活
  - 程序中用 `MC_MoveAbsolute` 动态给位置更合适
- **结论：** 动态目标多的项目，不一定适合用命令表。

> [!tip] 记忆口诀
> **固定动作查表跑，变化逻辑程序管。**

# 08｜常见问题排查

> [!info] 核心结论
> **命令表执行异常时，优先检查轴是否使能、是否回零、表是否有效、目标是否超限、当前卡在哪一行。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>排查顺序</strong><br>
轴使能 → 回零状态 → 表配置 → 目标位置 → 限位范围 → ErrorID → 当前行号。
</div>

## ① 分类/公式/规则

| 问题 | 常见原因 |
|---|---|
| **表不启动** | Execute 没上升沿、轴未使能 |
| **启动就报错** | 未回零、命令参数非法 |
| **运行中断** | 超限、驱动报警、安全条件断开 |
| **位置不对** | 机械参数/单位换算错误 |
| **卡在某一步** | 等待条件不满足或目标到不了 |
| **完成信号不来** | 未到位、Busy 未结束、Error 已发生 |

## ② 参考表

| 现象 | 排查方向 | 解决建议 |
|---|---|---|
| **Execute 无反应** | 是否上升沿 | 加边沿触发 |
| **Busy 一直为 1** | 当前行是否卡住 | 查行号和条件 |
| **Error=1** | 查 ErrorID | 对照帮助文档 |
| **轴不动** | MC_Power、驱动报警 | 先使能轴 |
| **绝对定位失败** | 未 Home | 先回零 |
| **撞限位** | 目标位置/方向错误 | 低速校验方向 |

## ③ 计算/选型示例

- **已知：** 命令表第一步目标位置为 `600mm`，软件正限位为 `500mm`
- **公式：** 目标位置必须在软件限位范围内
- **计算：**
  - 目标：`600mm`
  - 正限位：`500mm`
  - 目标超出范围
- **结论：** 命令表会报错或不允许执行，应修改目标位置或限位参数，需按机械实际行程校核。

> [!tip] 记忆口诀
> **表跑不动先看轴，轴没问题再看行。**

# 09｜一页速记总卡

> [!info] 核心结论
> **TO_CommandTable 的核心价值是把固定多步运动流程做成表格，减少程序中重复写多个 Move 指令。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
TO 建表，MC 执行；轴先使能，位置先回零；表按行跑，故障看行号。
</div>

## ① 分类/公式/规则

| 主轴 | 关键词 |
|---|---|
| **对象** | 命令表工艺对象 |
| **版本** | `V8.0` |
| **作用** | 多步运动流程 |
| **依赖** | 定位轴、轴使能、回零 |
| **执行** | `MC_CommandTable` |
| **诊断** | Busy、Done、Error、ErrorID、当前行 |

## ② 参考表

| 需求 | 推荐 |
|---|---|
| **多点固定定位** | `TO_CommandTable` |
| **单点定位** | `MC_MoveAbsolute` |
| **相对移动** | `MC_MoveRelative` |
| **连续速度运行** | `MC_MoveVelocity` |
| **复杂条件流程** | 状态机 |
| **故障恢复** | `MC_Stop` / `MC_Reset` |

## ③ 计算/选型示例

- **已知：** 一根轴要循环执行 4 个固定点位  
- **公式：** 固定点位序列 = 命令表  
- **计算：**
  - 在 TO_CommandTable 中写入 4 行动作
  - 轴使能并回零
  - 用 `MC_CommandTable` 启动
  - 监控 Done / Error
- **结论：** 这种固定多步定位流程非常适合用 TO_CommandTable。

> [!tip] 记忆口诀
> **单动调用指令，多动交给表；表管顺序，轴管执行。**
