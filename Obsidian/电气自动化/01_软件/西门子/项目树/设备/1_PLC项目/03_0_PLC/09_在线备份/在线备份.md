# 01｜在线备份是什么

> [!info] 核心结论
> **在线备份是 TIA Portal 从在线 PLC 设备中读取当前程序、硬件组态或设备数据并保存成备份，用于现场维护、版本留存和故障恢复。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>图片识别</strong><br>
截图中的 <strong>在线备份</strong> 是 TIA Portal 项目树中的维护功能，常用于把现场 PLC 当前运行状态保存下来。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **中文名称** | 在线备份 |
| **英文理解** | Online Backup |
| **作用对象** | PLC、CPU、程序块、组态、部分在线数据 |
| **主要用途** | 保存现场当前版本 |
| **典型场景** | 改程序前、换 CPU 前、设备验收前 |
| **注意** | 不等于完整工程归档 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
在线备份就是给现场 PLC 做一份“当前版本快照”，防止改坏后没有退路。
</div>

## ② 参考表

| 对比项 | 在线备份 | 项目归档 |
|---|---|---|
| **来源** | 在线 PLC 设备 | 工程项目文件 |
| **用途** | 保存现场当前状态 | 保存完整工程 |
| **适合场景** | 现场维护前 | 项目交付/长期保存 |
| **是否包含注释** | 需按实际情况校核 | 通常完整 |
| **恢复价值** | 高 | 更高 |

## ③ 计算/选型示例

- **已知：** 现场准备修改 PLC 程序  
- **公式：** 改程序前 = `在线备份 + 项目归档 + 当前版本记录`
- **计算：**
  - 先在线连接 PLC
  - 执行在线备份
  - 再归档当前 TIA 项目
  - 最后开始修改程序
- **结论：** 在线备份是现场改程序前的安全动作。

> [!tip] 记忆口诀
> **先备份，再修改；有备份，才敢动。**

# 02｜在线备份能备什么

> [!info] 核心结论
> **在线备份通常用于保存 PLC 当前下载到设备中的程序与组态信息，但是否完整保留符号、注释、源文件等内容，要按 CPU、TIA 版本和项目设置校核。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程理解</strong><br>
在线备份更像“从设备里捞当前版本”，不是百分百等同于原始工程文件。
</div>

## ① 分类/公式/规则

| 内容 | 是否可能包含 | 注意 |
|---|---|---|
| **程序块** | 通常可以 | 受保护块可能受限 |
| **硬件组态** | 可能可以 | 需按设备支持校核 |
| **DB 当前值** | 视操作方式 | 与初始值不同 |
| **符号名** | 视情况 | 取决于下载内容 |
| **注释** | 不一定完整 | 原始工程更可靠 |
| **源文件** | 不一定 | 外部源文件需单独保存 |
| **密码/保护块** | 可能无法读取 | 需权限 |

## ② 参考表

| 备份对象 | 作用 | 风险点 |
|---|---|---|
| **OB / FB / FC** | 程序逻辑 | 受保护块可能无法查看 |
| **DB** | 参数和数据 | 当前值/初始值要分清 |
| **硬件组态** | CPU 和模块配置 | 与现场硬件一致性 |
| **变量表** | 地址符号 | 可能不如原项目完整 |
| **工艺对象** | 轴/PID等配置 | 需实际校核 |
| **HMI项目** | 画面工程 | 通常需单独备份 |

## ③ 计算/选型示例

- **已知：** 现场只有 PLC，没有原始 TIA 项目  
- **公式：** 无原始项目 → 尽快做在线备份  
- **计算：**
  - 连接 PLC
  - 上传/备份在线程序
  - 检查是否有受保护块
  - 保存备份文件并记录时间
- **结论：** 在线备份可以救急，但不能替代完整项目归档。

> [!tip] 记忆口诀
> **在线能救急，归档才完整。**

# 03｜什么时候必须做在线备份

> [!info] 核心结论
> **只要准备改程序、改参数、换设备、下载新项目或排查重大故障，都应该先做在线备份。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>备份口诀</strong><br>
改前备，换前备，下载前也要备。
</div>

## ① 分类/公式/规则

| 场景 | 是否建议备份 | 原因 |
|---|---|---|
| **修改程序前** | **必须** | 防止改坏 |
| **下载程序前** | **必须** | 防止覆盖现场版本 |
| **更换 CPU 前** | **必须** | 便于恢复 |
| **修改参数前** | 建议 | 防止参数丢失 |
| **设备验收前** | 建议 | 固化交付版本 |
| **故障排查前** | 建议 | 保留故障现场 |

## ② 参考表

| 操作 | 备份重点 |
|---|---|
| **改 PLC 逻辑** | 程序块 |
| **改硬件组态** | 设备组态 |
| **改 DB 参数** | 当前值和初始值 |
| **改驱动参数** | 驱动器参数需单独备份 |
| **改 HMI** | HMI 工程需单独归档 |
| **换 CPU** | PLC 程序、组态、IP、设备名 |

## ③ 计算/选型示例

- **已知：** 现场要下载一个新版本程序  
- **公式：** 下载前安全动作 = `在线备份 + 在线比较 + 项目归档`
- **计算：**
  - 备份当前 PLC
  - 比较当前项目与在线设备差异
  - 确认下载内容
  - 再执行下载
- **结论：** 下载前不备份，出问题后可能无法恢复现场版本。

> [!tip] 记忆口诀
> **下载一时快，不备份就危险。**

# 04｜在线备份典型流程

> [!info] 核心结论
> **在线备份流程建议按：确认设备 → 在线连接 → 读取/备份 → 保存命名 → 验证备份 → 记录版本。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>标准流程</strong><br>
连设备 → 查版本 → 做备份 → 存文件 → 写记录 → 再修改
</div>

## ① 分类/公式/规则

| 步骤 | 动作 | 重点 |
|---|---|---|
| **1** | 确认 PLC 型号和 IP | 避免连错设备 |
| **2** | 在线连接 CPU | 选择正确网卡 |
| **3** | 查看在线状态 | 确认设备正常 |
| **4** | 执行在线备份 | 保存当前版本 |
| **5** | 命名备份文件 | 时间+设备+版本 |
| **6** | 检查备份内容 | 是否有报错 |
| **7** | 记录版本说明 | 便于追溯 |

## ② 参考表

| 命名字段 | 示例 |
|---|---|
| **日期** | `2026-04-29` |
| **设备名** | `Line1_PLC` |
| **CPU型号** | `S7-1500` |
| **版本** | `Before_Modify_V1` |
| **人员** | `Zhang` |
| **说明** | `改程序前备份` |

## ③ 计算/选型示例

- **已知：** 1号线 PLC 改程序前备份  
- **公式：** 文件名 = `日期_设备_动作_版本`
- **计算：**
  - 日期：`2026-04-29`
  - 设备：`Line1_PLC`
  - 动作：`BeforeModify`
  - 版本：`V1`
- **结论：** 可命名为 `2026-04-29_Line1_PLC_BeforeModify_V1`。

> [!tip] 记忆口诀
> **备份不命名，等于没备清。**

# 05｜在线备份与上传的区别

> [!info] 核心结论
> **在线备份偏向保存当前设备状态，上传偏向把在线设备内容读取到工程中；两者都不是完整项目归档的替代品。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
备份是留快照，上传是拿回来编辑；归档是完整保存项目。
</div>

## ① 分类/公式/规则

| 操作 | 中文理解 | 典型用途 |
|---|---|---|
| **在线备份** | 保存设备当前版本 | 改程序前留底 |
| **上传设备** | 从 PLC 读取项目内容 | 没有原项目时救急 |
| **在线比较** | 比较项目和设备差异 | 下载前确认 |
| **项目归档** | 保存完整工程 | 交付和长期保存 |
| **下载设备** | 把项目写入 PLC | 更新现场程序 |

## ② 参考表

| 需求 | 推荐操作 |
|---|---|
| **改程序前留底** | 在线备份 |
| **没有原程序，想取回 PLC 内容** | 上传设备 |
| **判断项目和现场是否一致** | 在线比较 |
| **交付项目给客户** | 项目归档 |
| **更新现场程序** | 下载设备 |
| **保存 HMI 工程** | HMI 项目归档 |

## ③ 计算/选型示例

- **已知：** 工程师不知道电脑项目是不是现场运行版本  
- **公式：** 不确定版本 → 先在线比较，再决定备份或下载  
- **计算：**
  - 在线连接 PLC
  - 做在线比较
  - 如果不同，先在线备份现场版本
  - 再决定是否下载电脑项目
- **结论：** 不确定版本时，不要直接下载。

> [!tip] 记忆口诀
> **不确定，先比较；要修改，先备份。**

# 06｜备份文件管理

> [!info] 核心结论
> **备份文件必须按设备、时间、版本、修改内容分类保存，否则后期恢复时容易拿错版本。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>管理口诀</strong><br>
一机一夹，一改一备，一版一记。
</div>

## ① 分类/公式/规则

| 管理项 | 建议 |
|---|---|
| **文件夹** | 按产线/设备分类 |
| **文件名** | 日期+设备+版本+说明 |
| **修改记录** | 写清改动原因 |
| **负责人** | 记录操作人员 |
| **备份位置** | 本地+服务器/云盘双份 |
| **恢复验证** | 备份后检查能否打开 |

## ② 参考表

| 文件夹结构 | 示例 |
|---|---|
| **项目根目录** | `PLC_Backup` |
| **产线目录** | `Line1` |
| **设备目录** | `S7_1500_CPU` |
| **日期目录** | `2026-04-29` |
| **版本目录** | `Before_Modify` |
| **说明文件** | `ChangeLog.md` |

## ③ 计算/选型示例

- **已知：** 一个月内对同一 PLC 改了 5 次程序  
- **公式：** 每次改动 = 一份备份 + 一条记录  
- **计算：**
  - 第 1 次：`V1_Before`
  - 第 2 次：`V2_After`
  - 第 3 次：`V3_Before`
  - 每次写明修改内容
- **结论：** 备份不只是文件，还要有版本说明。

> [!tip] 记忆口诀
> **没有记录的备份，恢复时最危险。**

# 07｜常见错误与风险

> [!info] 核心结论
> **在线备份常见风险包括备错设备、备份不完整、受保护块无法读取、忘记保存项目、恢复版本拿错。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>避坑提醒</strong><br>
在线备份前一定确认设备名称、IP 地址、CPU 型号和产线位置，避免把别的 PLC 当成目标设备。
</div>

## ① 分类/公式/规则

| 风险 | 原因 |
|---|---|
| **备错设备** | IP 或设备名没确认 |
| **备份不完整** | 受保护块、权限不足 |
| **注释丢失** | 在线设备中不一定有完整工程信息 |
| **版本混乱** | 文件名无日期和说明 |
| **恢复失败** | CPU 型号/固件不匹配 |
| **覆盖现场程序** | 未比较就下载 |

## ② 参考表

| 现象 | 排查方向 | 建议 |
|---|---|---|
| **备份后打不开** | 软件版本 | 用对应 TIA 版本 |
| **部分块无法查看** | 块保护 | 找原始项目或权限 |
| **变量名不完整** | 在线上传限制 | 优先找原项目 |
| **恢复后设备异常** | 硬件不一致 | 查设备组态 |
| **不知道哪个版本对** | 命名混乱 | 查时间和记录 |
| **下载后停机** | 未校核差异 | 下载前先比较 |

## ③ 计算/选型示例

- **已知：** 现场有两台 PLC，IP 分别是 `192.168.0.1` 和 `192.168.0.2`
- **公式：** 在线备份前 = 确认目标设备
- **计算：**
  - 核对设备名称
  - 核对 IP 地址
  - 核对 CPU 指示灯和在线闪烁功能
  - 再执行备份
- **结论：** 多 PLC 现场最容易备错设备，必须先确认目标。

> [!tip] 记忆口诀
> **先确认设备，再谈备份。**

# 08｜一页速记总卡

> [!info] 核心结论
> **在线备份的核心价值是保存现场 PLC 当前版本，给后续修改、下载、换机和故障恢复留退路。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
改前先备份，下载先比较；备份要命名，恢复看版本。
</div>

## ① 分类/公式/规则

| 主轴 | 关键词 |
|---|---|
| **对象** | 在线备份 |
| **用途** | 保存现场当前版本 |
| **场景** | 改程序、换 CPU、下载前 |
| **重点** | 设备确认、版本记录 |
| **风险** | 备错设备、备份不完整 |
| **补充** | 不能替代完整项目归档 |

## ② 参考表

| 需求 | 推荐做法 |
|---|---|
| **改程序前** | 在线备份 |
| **下载前** | 在线比较 + 在线备份 |
| **项目交付** | 项目归档 |
| **无原程序救急** | 上传设备 + 在线备份 |
| **换 CPU** | 在线备份 + 参数记录 |
| **版本长期保存** | 备份文件 + 修改记录 |

## ③ 计算/选型示例

- **已知：** 准备对生产线 PLC 做一次程序优化  
- **公式：** 安全修改 = `确认设备 + 在线备份 + 项目归档 + 在线比较 + 修改下载`
- **计算：**
  - 确认 PLC 名称和 IP
  - 备份现场程序
  - 归档当前工程
  - 比较差异
  - 修改并下载
- **结论：** 在线备份是工程师现场改程序前的必做动作。

> [!tip] 记忆口诀
> **备份是后悔药，没备份就别乱动。**
