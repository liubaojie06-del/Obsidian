# 01｜PLC 报警文本列表是什么

> [!info] 核心结论
> **PLC 报警文本列表是 TIA Portal 中用于集中管理 PLC 报警编号、报警文本、多语言显示和报警说明的工程对象。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>图片识别</strong><br>
截图中的 <strong>PLC 报警文本列表</strong> 是 PLC 报警消息的文本管理入口，常用于给报警编号配置可读的中文/英文报警内容。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **中文名称** | PLC 报警文本列表 |
| **英文理解** | PLC Alarm Text List |
| **核心作用** | 把报警编号转换成可读报警文本 |
| **典型用途** | HMI 报警显示、诊断、维护提示 |
| **常见内容** | 报警号、报警文本、帮助文本、语言版本 |
| **是否直接控制设备** | 否，只负责报警文本和显示信息 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
PLC 报警文本列表就是报警系统的“翻译表”：报警号负责识别，文本负责让人看懂。
</div>

## ② 参考表

| 对象 | 作用 |
|---|---|
| **报警触发条件** | 程序中判断故障是否发生 |
| **报警编号** | 唯一标识某条报警 |
| **报警文本列表** | 给报警编号配置文字说明 |
| **HMI 报警窗口** | 显示报警内容 |
| **报警历史** | 保存发生和确认记录 |
| **维护人员** | 根据报警文本排查故障 |

## ③ 计算/选型示例

- **已知：** 电机过载信号触发报警  
- **公式：** 报警显示 = `报警编号 + 报警文本`
- **计算：**
  - 报警编号：`1001`
  - 报警文本：`1号电机过载，请检查热继电器或驱动器报警`
  - HMI 显示该文本
- **结论：** 报警文本列表让报警从“代码”变成“人能看懂的信息”。

> [!tip] 记忆口诀
> **报警号给系统看，报警文本给人看。**

# 02｜它解决什么问题

> [!info] 核心结论
> **报警文本列表解决的是报警信息标准化问题，让故障显示、维护提示、多语言文本集中管理。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程理解</strong><br>
没有报警文本列表，现场只能看到零散变量或状态码；有了文本列表，HMI 可以直接显示清晰报警原因和处理建议。
</div>

## ① 分类/公式/规则

| 问题 | 报警文本列表的作用 |
|---|---|
| **报警太多不好管理** | 集中编号和文本 |
| **HMI 显示不清楚** | 提供可读报警内容 |
| **多语言项目维护难** | 统一管理中文/英文等文本 |
| **维修不知道怎么处理** | 增加原因和处理建议 |
| **报警编号混乱** | 建立统一编码规则 |
| **项目交接困难** | 报警含义清晰可查 |

## ② 参考表

| 报警信息 | 不推荐 | 推荐 |
|---|---|---|
| **电机报警** | `Alarm1001` | `1号电机过载` |
| **气缸报警** | `Fault_23` | `A气缸伸出超时` |
| **通讯报警** | `CommErr` | `扫码枪通信中断` |
| **模拟量报警** | `AI_High` | `压力超过上限` |
| **安全报警** | `SafeErr` | `安全门未关闭` |

## ③ 计算/选型示例

- **已知：** 一条产线有 200 条报警  
- **公式：** 报警多 → 必须编号化、文本化、分类化  
- **计算：**
  - 电机报警：`1000~1999`
  - 气缸报警：`2000~2999`
  - 通信报警：`3000~3999`
  - 安全报警：`9000~9999`
- **结论：** 报警文本列表适合做长期维护的报警数据库。

> [!tip] 记忆口诀
> **报警越多，越要编号；维护越久，越要文本。**

# 03｜报警文本列表通常包含什么

> [!info] 核心结论
> **报警文本列表通常包含报警编号、报警名称、报警正文、附加说明、语言版本和分类信息，具体字段需按 TIA 版本校核。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
一条好报警，不只告诉你“坏了”，还要告诉你“哪里坏、为什么、怎么查”。
</div>

## ① 分类/公式/规则

| 字段 | 作用 |
|---|---|
| **Alarm ID** | 报警编号 |
| **Alarm Text** | 报警正文 |
| **Alarm Class** | 报警类别 |
| **Priority** | 报警优先级 |
| **Help Text** | 维修帮助文本 |
| **Language** | 多语言文本 |
| **Acknowledge** | 是否需要确认 |
| **Source** | 报警来源对象 |

> [!warning]
> 不同 TIA Portal 版本、CPU 类型、HMI 类型支持的字段可能不同，实际项目需按软件界面和厂家手册校核。

## ② 参考表

| 字段 | 示例 |
|---|---|
| **编号** | `1001` |
| **报警名称** | `Motor1_Overload` |
| **报警文本** | `1号电机过载` |
| **类别** | `故障报警` |
| **优先级** | `高` |
| **帮助文本** | `检查热继电器、驱动器报警和机械卡滞` |
| **语言** | `zh-CN / en-US` |

## ③ 计算/选型示例

- **已知：** 要建立一条气缸伸出超时报警  
- **公式：** 好报警 = `对象 + 动作 + 异常 + 建议`
- **计算：**
  - 对象：`A气缸`
  - 动作：`伸出`
  - 异常：`超时`
  - 建议：`检查气源、磁性开关、电磁阀`
- **结论：** 报警文本可写成：`A气缸伸出超时，请检查气源、伸出到位开关和电磁阀。`

> [!tip] 记忆口诀
> **报警文本四要素：谁、怎么了、为什么、怎么查。**

# 04｜报警编号规则

> [!info] 核心结论
> **报警编号要提前规划，建议按设备类别、工位、故障类型分段管理，避免后期重复和混乱。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点规则</strong><br>
报警编号一旦用于 HMI、MES、报警历史和维修文档，后期不要随意改。
</div>

## ① 分类/公式/规则

| 编号段 | 建议用途 |
|---:|---|
| **1000~1999** | 电机 / 驱动器报警 |
| **2000~2999** | 气缸 / 阀门报警 |
| **3000~3999** | 通信 / 扫码 / 视觉报警 |
| **4000~4999** | 模拟量 / 工艺参数报警 |
| **5000~5999** | 运动轴 / 伺服报警 |
| **9000~9999** | 安全 / 急停 / 门锁报警 |

> [!warning]
> 上表是工程规划示例，实际编号规则需按公司标准、项目规范或客户要求校核。

## ② 参考表

| 报警编号 | 报警文本 |
|---:|---|
| **1001** | 1号电机过载 |
| **1002** | 1号电机运行反馈丢失 |
| **2001** | A气缸伸出超时 |
| **2002** | A气缸缩回超时 |
| **3001** | 扫码枪通信中断 |
| **9001** | 急停按钮按下 |

## ③ 计算/选型示例

- **已知：** 新增 5 条扫码枪报警  
- **公式：** 通信类报警 → 放入 `3000~3999`
- **计算：**
  - `3001`：扫码枪通信中断
  - `3002`：扫码内容为空
  - `3003`：扫码格式错误
  - `3004`：扫码超时
  - `3005`：扫码结果与配方不匹配
- **结论：** 分类编号能让报警维护更有秩序。

> [!tip] 记忆口诀
> **编号先分段，报警不混乱。**

# 05｜报警文本怎么写

> [!info] 核心结论
> **报警文本要短、准、可执行，最好直接指出设备对象、故障现象和排查方向。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>写法模板</strong><br>
设备对象 + 故障现象 + 处理建议
</div>

## ① 分类/公式/规则

| 写法 | 示例 |
|---|---|
| **对象明确** | `1号电机过载` |
| **动作明确** | `A气缸伸出超时` |
| **条件明确** | `压力低于下限` |
| **建议明确** | `请检查气源压力和传感器` |
| **避免模糊** | 不要只写 `故障`、`异常` |

## ② 参考表

| 不推荐 | 推荐 |
|---|---|
| **报警1** | `1号电机过载` |
| **气缸异常** | `A气缸伸出超时` |
| **通信故障** | `扫码枪通信中断` |
| **压力错误** | `主气源压力低于下限` |
| **传感器坏** | `原点传感器未检测到信号` |
| **设备故障** | `伺服轴1驱动器报警` |

## ③ 计算/选型示例

- **已知：** 传送带启动后 3 秒内没有运行反馈  
- **公式：** 报警文本 = `设备 + 现象 + 时间条件`
- **计算：**
  - 设备：`传送带电机`
  - 现象：`运行反馈未到`
  - 时间：`启动后3秒`
- **结论：** 推荐文本：`传送带电机启动后3秒内无运行反馈，请检查接触器、变频器和反馈信号。`

> [!tip] 记忆口诀
> **报警别写玄学，维修要能照着查。**

# 06｜与 HMI 报警的关系

> [!info] 核心结论
> **PLC 报警文本列表偏 PLC 侧报警文本管理，HMI 报警窗口负责显示、确认、记录和筛选报警。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>关系链路</strong><br>
PLC 报警条件 → 报警编号/文本 → HMI 报警控件 → 操作员确认 → 报警历史记录
</div>

## ① 分类/公式/规则

| 对象 | 作用 |
|---|---|
| **PLC 程序** | 判断报警条件 |
| **报警文本列表** | 管理报警文字 |
| **HMI 报警视图** | 显示当前报警 |
| **报警确认** | 操作员确认已知晓 |
| **报警历史** | 保存发生、消失、确认时间 |
| **报警帮助** | 指导维修排查 |

## ② 参考表

| HMI 显示内容 | 来源 |
|---|---|
| **报警时间** | HMI/PLC 系统时间 |
| **报警文本** | 报警文本列表 |
| **报警状态** | PLC 报警触发位 |
| **确认状态** | HMI 或 PLC 报警系统 |
| **报警类别** | 报警配置 |
| **帮助信息** | 报警帮助文本 |

## ③ 计算/选型示例

- **已知：** 操作员看到报警 `2001 A气缸伸出超时`  
- **公式：** HMI 显示 = `报警编号 + 报警文本 + 时间 + 状态`
- **计算：**
  - 编号：`2001`
  - 文本：`A气缸伸出超时`
  - 状态：`已发生，未确认`
  - 处理：检查气源、电磁阀、伸出到位开关
- **结论：** 报警文本列表决定 HMI 上显示得是否清楚。

> [!tip] 记忆口诀
> **PLC 负责报警逻辑，HMI 负责报警展示。**

# 07｜多语言报警文本

> [!info] 核心结论
> **多语言项目中，报警文本列表可以统一维护中文、英文等报警文本，避免 HMI 多处重复翻译。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程理解</strong><br>
出口设备或跨国项目中，报警文本必须标准化翻译，否则现场维护会非常困难。
</div>

## ① 分类/公式/规则

| 语言项 | 示例 |
|---|---|
| **中文 zh-CN** | `1号电机过载` |
| **英文 en-US** | `Motor 1 overload` |
| **德文 de-DE** | 需按项目翻译 |
| **越南语/泰语等** | 需按客户要求 |
| **默认语言** | 需按 HMI 设置 |

## ② 参考表

| 中文报警 | 英文示例 |
|---|---|
| **急停按钮按下** | `Emergency stop pressed` |
| **安全门未关闭** | `Safety door not closed` |
| **气源压力低** | `Air pressure low` |
| **扫码枪通信中断** | `Barcode scanner communication lost` |
| **伺服驱动器报警** | `Servo drive fault` |

## ③ 计算/选型示例

- **已知：** 设备需要中英文报警显示  
- **公式：** 一条报警 = 一个编号 + 多语言文本  
- **计算：**
  - 编号：`9001`
  - 中文：`急停按钮按下`
  - 英文：`Emergency stop pressed`
- **结论：** 多语言报警应共用同一报警编号，避免不同语言版本编号不一致。

> [!tip] 记忆口诀
> **编号只一套，文本多语言。**

# 08｜常见错误与排查

> [!info] 核心结论
> **PLC 报警文本列表常见问题是编号重复、文本不清、HMI 不显示、语言未同步、报警变量和文本不匹配。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>避坑提醒</strong><br>
报警文本写错不会让设备停机，但会让维修人员排错方向完全错误。
</div>

## ① 分类/公式/规则

| 问题 | 常见原因 |
|---|---|
| **HMI 不显示报警文本** | 报警列表未同步或未下载 |
| **显示编号不显示文字** | 文本缺失或语言未配置 |
| **报警内容不对应** | 报警编号和触发条件错配 |
| **重复报警号** | 编号规划混乱 |
| **维修看不懂** | 文本太短或太泛 |
| **语言显示错误** | 多语言文本未维护 |

## ② 参考表

| 现象 | 排查方向 | 解决建议 |
|---|---|---|
| **只显示报警号** | 文本列表是否下载 | 重新编译下载 |
| **报警文字不对** | 编号是否匹配 | 检查报警 ID |
| **HMI 无报警** | 触发位是否有效 | 在线监控报警条件 |
| **无法确认报警** | 确认机制 | 查报警类别和确认设置 |
| **英文为空** | 多语言文本 | 补全翻译 |
| **报警太多刷屏** | 分类和优先级 | 优化报警等级 |

## ③ 计算/选型示例

- **已知：** PLC 触发的是报警编号 `2001`，HMI 显示为 `1号电机过载`  
- **公式：** 报警编号错配 → 文本显示错误  
- **计算：**
  - `2001` 本应是气缸报警
  - 文本却显示电机报警
  - 说明编号或文本表配置错误
- **结论：** 报警排查时要同时核对触发条件、报警编号和文本列表。

> [!tip] 记忆口诀
> **报警显示不对，先查编号映射。**

# 09｜一页速记总卡

> [!info] 核心结论
> **PLC 报警文本列表的核心价值是把 PLC 报警标准化、文本化、多语言化，让故障能被快速理解和处理。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
报警先编号，文本写清楚；中文给现场，英文给出口；触发要准确，处理要可查。
</div>

## ① 分类/公式/规则

| 主轴 | 关键词 |
|---|---|
| **对象** | PLC 报警文本列表 |
| **用途** | 报警编号与文本管理 |
| **显示对象** | HMI、诊断、报警历史 |
| **核心内容** | 编号、文本、类别、语言、帮助 |
| **设计重点** | 分类编号、文本清晰、多语言同步 |
| **风险点** | 编号错配、文本不清、未同步下载 |

## ② 参考表

| 需求 | 推荐做法 |
|---|---|
| **电机报警** | 编号 `1000` 段 |
| **气缸报警** | 编号 `2000` 段 |
| **通信报警** | 编号 `3000` 段 |
| **安全报警** | 编号 `9000` 段 |
| **维修提示** | 加 Help Text |
| **出口设备** | 做多语言文本 |
| **HMI 显示** | 同步报警文本列表 |

## ③ 计算/选型示例

- **已知：** 要给一台设备建立报警系统  
- **公式：** 报警系统 = `报警条件 + 报警编号 + 报警文本 + HMI显示 + 历史记录`
- **计算：**
  - PLC 程序判断报警条件
  - 报警文本列表配置文本
  - HMI 报警窗口显示报警
  - 操作员确认并记录历史
- **结论：** 报警文本列表是 PLC 报警系统长期维护的核心基础。

> [!tip] 记忆口诀
> **报警不是只亮灯，而是要让人知道怎么修。**
