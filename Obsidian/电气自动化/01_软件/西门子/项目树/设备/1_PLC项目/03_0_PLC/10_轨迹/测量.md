# 01｜测量 Measurements 是什么

> [!info] 核心结论
> **“测量”是 TIA Portal 中用于保存、查看和分析 Trace 采集结果的区域，重点不是实时监控，而是对已采集数据进行复盘分析。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>图片识别</strong><br>
截图中的 <strong>测量</strong> 通常与 <strong>轨迹 Trace</strong> 配合使用，用来管理采集后的测量曲线和数据记录。
</div>

## ① 分类/公式/规则

| 对象 | 中文理解 | 作用 |
|---|---|---|
| **轨迹 Trace** | 采集配置 | 决定采哪些变量、周期、触发条件 |
| **测量 Measurements** | 采集结果 | 保存某一次采集到的数据 |
| **曲线数据** | 波形记录 | 分析变量随时间变化 |
| **导出数据** | CSV/表格数据 | 便于离线分析 |
| **对比分析** | 多次测量对比 | 比较调参前后效果 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
Trace 负责“抓数据”，测量负责“存数据、看曲线、做复盘”。
</div>

## ② 参考表

| 项目 | Trace 轨迹 | 测量 Measurements |
|---|---|---|
| **定位** | 采集任务 | 采集结果 |
| **作用** | 设置变量和触发 | 保存波形数据 |
| **使用阶段** | 采集前/采集中 | 采集后分析 |
| **是否实时** | 偏实时 | 偏历史记录 |
| **典型用途** | 抓故障 | 复盘故障 |

## ③ 计算/选型示例

- **已知：** 调试 PID 后，需要比较调参前后的温度曲线  
- **公式：** 调参对比 = `测量1曲线 - 测量2曲线`
- **计算：**
  - 第一次测量：旧参数曲线
  - 第二次测量：新参数曲线
  - 对比超调、稳定时间、输出波动
- **结论：** “测量”适合保存每次 Trace 结果，便于后续对比。

> [!tip] 记忆口诀
> **Trace 是抓拍，测量是相册；抓完数据，放进测量。**

# 02｜测量能干什么

> [!info] 核心结论
> **测量可以保存变量曲线、查看历史采集数据、对比不同测试结果，并辅助分析 PID、轴控、通信、模拟量等问题。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程理解</strong><br>
测量不是用来替代监控表，而是用来把“故障发生过程”留存下来，方便慢慢分析。
</div>

## ① 分类/公式/规则

| 功能 | 作用 |
|---|---|
| **保存曲线** | 保留某次 Trace 数据 |
| **查看波形** | 分析变量变化过程 |
| **对比数据** | 比较调试前后差异 |
| **导出记录** | 给报告、复盘、存档使用 |
| **故障复盘** | 看报警前后的变量状态 |
| **参数优化** | 辅助 PID 和运动控制调试 |

## ② 参考表

| 应用场景 | 推荐保存的测量内容 |
|---|---|
| **PID 调试** | SP、PV、Output、Error |
| **伺服定位** | 目标位置、实际位置、速度、误差 |
| **通信故障** | REQ、BUSY、DONE、ERROR、STATUS |
| **模拟量波动** | Raw、Real、Filtered |
| **气缸超时** | Cmd、HomeFB、WorkFB、Timeout |
| **电机异常** | RunCmd、RunFB、Fault、Current |

## ③ 计算/选型示例

- **已知：** 电机启动后偶发过载  
- **公式：** 过载分析 = `启动命令 + 电流 + 速度 + 故障位`
- **计算：**
  - 保存启动过程测量
  - 查看启动瞬间电流是否过高
  - 对比速度爬升是否异常
  - 查看故障位出现时间点
- **结论：** 测量可以帮助判断过载是机械卡滞、加速太猛，还是驱动参数问题。

> [!tip] 记忆口诀
> **故障别只看结果，要看故障前后过程。**

# 03｜测量与监控表的区别

> [!info] 核心结论
> **监控表适合看当前值，测量适合看历史曲线；一个看“现在”，一个看“过程”。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>对比口诀</strong><br>
监控看实时，测量看历史；监控看数值，测量看曲线。
</div>

## ① 分类/公式/规则

| 工具 | 主要用途 | 适合问题 |
|---|---|---|
| **监控表** | 看实时变量 | 信号有没有 |
| **强制表** | 临时干预变量 | 测试硬件输出 |
| **轨迹 Trace** | 在线采集变量 | 抓变化过程 |
| **测量** | 保存采集结果 | 复盘和对比 |
| **诊断缓冲区** | 看故障事件 | 查报警历史 |

## ② 参考表

| 问题 | 推荐工具 |
|---|---|
| **按钮有没有信号** | 监控表 |
| **输出模块能不能动作** | 强制表，谨慎使用 |
| **PID 为什么振荡** | Trace + 测量 |
| **伺服为什么超调** | Trace + 测量 |
| **PLC 为什么停机** | 诊断缓冲区 |
| **通信状态怎么变化** | Trace + 测量 |

## ③ 计算/选型示例

- **已知：** 一个信号只闪一下，监控表很难看到  
- **公式：** 瞬时变化 → Trace 采集 → 测量保存  
- **计算：**
  - 在 Trace 中添加该 Bool 变量
  - 设置较短采样周期
  - 触发后保存测量
  - 查看信号变化时间点
- **结论：** 闪变信号不适合靠眼睛盯监控表，应该用 Trace 和测量。

> [!tip] 记忆口诀
> **当前值用监控，瞬间变化用测量。**

# 04｜测量数据怎么看

> [!info] 核心结论
> **看测量数据时，不要只看单条曲线，要看变量之间的先后关系、时间差、峰值、稳定时间和异常点。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>分析口诀</strong><br>
先看时间点，再看先后顺序；先看峰值，再看稳定过程。
</div>

## ① 分类/公式/规则

| 分析项 | 看什么 |
|---|---|
| **触发点** | 故障从什么时候开始 |
| **时间差** | 命令到反馈用了多久 |
| **峰值** | 最大压力、最大速度、最大误差 |
| **稳定时间** | 多久进入稳定状态 |
| **超调量** | 实际值超过目标多少 |
| **异常跳变** | 信号是否突然变化 |

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点公式</strong><br>
响应时间 = 反馈到达时间 - 命令发出时间
</div>

## ② 参考表

| 曲线现象 | 可能原因 |
|---|---|
| **反馈晚很多** | 机械慢、传感器延迟、气压不足 |
| **实际值超目标** | 参数过猛、惯性大、减速不足 |
| **输出一直满值** | 执行器能力不足或反馈异常 |
| **信号频繁抖动** | 干扰、接线、滤波不足 |
| **状态码突然变化** | 通信或设备状态异常 |
| **曲线断续** | 采样设置或在线连接问题 |

## ③ 计算/选型示例

- **已知：**
  - 开阀命令发出时间：`1.20s`
  - 开到位反馈出现时间：`2.85s`
- **公式：** 响应时间 = `2.85 - 1.20`
- **计算：**
  - `2.85s - 1.20s = 1.65s`
- **结论：** 阀门从发出开命令到开到位用了约 `1.65s`。

> [!tip] 记忆口诀
> **曲线不只看高低，更要看先后和时间。**

# 05｜测量在 PID 调试中的用法

> [!info] 核心结论
> **PID 调试时保存测量数据，可以对比不同参数下的响应速度、超调量、稳定时间和输出波动。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>PID 测量四件套</strong><br>
设定值 SP、实际值 PV、控制输出 Output、偏差 Error。
</div>

## ① 分类/公式/规则

| 指标 | 含义 | 调试价值 |
|---|---|---|
| **超调量** | PV 超过 SP 的最大值 | 判断参数是否过猛 |
| **上升时间** | PV 接近 SP 所需时间 | 判断响应快慢 |
| **稳定时间** | PV 稳定在范围内的时间 | 判断系统稳定性 |
| **稳态误差** | 稳定后 PV 与 SP 差值 | 判断积分作用 |
| **输出波动** | Output 是否频繁变化 | 判断执行器压力 |

## ② 参考表

| 测量现象 | 可能调整方向 |
|---|---|
| **响应慢** | 适当增大 Kp 或检查执行器 |
| **超调大** | 降低 Kp，减弱积分 |
| **长期有偏差** | 检查积分 Ti 或输出限幅 |
| **输出频繁跳动** | 增加滤波或降低参数激进程度 |
| **PV 不变化** | 先查传感器和执行器 |

## ③ 计算/选型示例

- **已知：**
  - 温度设定值：`80℃`
  - 测量曲线峰值：`88℃`
- **公式：** 超调量 = `峰值 - 设定值`
- **计算：**
  - `88 - 80 = 8℃`
- **结论：** 本次 PID 测量超调约 `8℃`，参数可能偏激进或系统热惯性较大。

> [!tip] 记忆口诀
> **PID 调参要留曲线，不留曲线难复盘。**

# 06｜测量在运动控制中的用法

> [!info] 核心结论
> **运动控制中，测量常用于分析定位误差、速度曲线、跟随误差、加减速冲击和 Done 信号时序。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>轴控测量三条线</strong><br>
目标位置、实际位置、跟随误差。
</div>

## ① 分类/公式/规则

| 变量 | 作用 |
|---|---|
| **TargetPosition** | 目标位置 |
| **ActualPosition** | 实际位置 |
| **ActualVelocity** | 实际速度 |
| **FollowingError** | 跟随误差 |
| **Busy** | 运动中 |
| **Done** | 运动完成 |
| **ErrorID** | 故障编号 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点公式</strong><br>
跟随误差 = 目标位置 - 实际位置
</div>

## ② 参考表

| 测量现象 | 可能原因 |
|---|---|
| **实际位置滞后大** | 加速度过大、负载重、伺服响应慢 |
| **实际位置超目标** | 减速不足、惯量大 |
| **速度曲线抖动** | 机械松动、增益不合适 |
| **Done 过晚** | 到位窗口太小 |
| **Error 瞬间出现** | 限位、跟随误差、驱动报警 |

## ③ 计算/选型示例

- **已知：**
  - 目标位置：`100.0mm`
  - 实际位置峰值：`101.5mm`
- **公式：** 超调量 = `实际峰值 - 目标位置`
- **计算：**
  - `101.5 - 100.0 = 1.5mm`
- **结论：** 本次定位超调约 `1.5mm`，需检查减速度、负载惯量和驱动参数。

> [!tip] 记忆口诀
> **轴动得准不准，看测量曲线最清楚。**

# 07｜测量数据保存与命名

> [!info] 核心结论
> **测量数据一定要按设备、时间、工况、参数版本命名，否则后期很难知道哪条曲线对应哪次调试。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>命名口诀</strong><br>
设备名 + 日期 + 工况 + 参数版本。
</div>

## ① 分类/公式/规则

| 命名字段 | 示例 |
|---|---|
| **设备名** | `Axis1`、`TempLoop1` |
| **日期** | `2026-04-29` |
| **工况** | `Move100mm`、`Heat80C` |
| **参数版本** | `PID_V1`、`ServoGain_V2` |
| **结果说明** | `Overshoot`、`Stable` |

## ② 参考表

| 测量名称示例 | 含义 |
|---|---|
| **Axis1_2026-04-29_Move100mm_V1** | 轴 1 移动 100mm 第 1 版参数 |
| **Temp1_2026-04-29_SP80_PIDV2** | 温控 80℃，PID 第 2 版 |
| **Comm_2026-04-29_ErrorTest** | 通信错误测试 |
| **CylinderA_Timeout_Test1** | 气缸 A 超时分析 |
| **AI_Pressure_NoiseTest** | 压力模拟量噪声测试 |

## ③ 计算/选型示例

- **已知：** 今天测试 1 号温控回路，设定值 `120℃`，PID 参数版本 `V3`
- **公式：** 测量名 = `对象_日期_工况_参数版本`
- **计算：**
  - 对象：`TempLoop1`
  - 日期：`2026-04-29`
  - 工况：`SP120`
  - 版本：`PIDV3`
- **结论：** 可命名为 `TempLoop1_2026-04-29_SP120_PIDV3`。

> [!tip] 记忆口诀
> **曲线不命名，复盘两眼懵。**

# 08｜常见错误与排查

> [!info] 核心结论
> **测量常见问题是曲线太短、采样太慢、变量选错、未保存结果、没有记录工况，导致后期无法复盘。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>避坑提醒</strong><br>
测量数据不是越多越好，关键是变量选对、周期合适、命名清楚。
</div>

## ① 分类/公式/规则

| 问题 | 常见原因 |
|---|---|
| **看不到关键瞬间** | 采样周期太长 |
| **记录时间不够** | 点数太少 |
| **曲线无意义** | 变量选错 |
| **无法对比** | 没记录参数版本 |
| **数据丢失** | 没保存测量 |
| **复盘困难** | 名称和说明不清 |

## ② 参考表

| 现象 | 排查方向 | 解决建议 |
|---|---|---|
| **曲线太粗** | 采样周期 | 缩短周期 |
| **只看到后半段** | 触发/预触发 | 设置预触发 |
| **没有故障前数据** | 触发位置 | 提前记录 |
| **数值单位不清** | 注释不足 | 写明单位 |
| **多次测量混乱** | 命名不规范 | 加日期和工况 |
| **导出后看不懂** | 缺少变量说明 | 保存变量清单 |

## ③ 计算/选型示例

- **已知：** 要看一个 `50ms` 的输出脉冲，但测量采样周期是 `200ms`
- **公式：** 采样周期应小于关键信号持续时间
- **计算：**
  - 脉冲宽度：`50ms`
  - 采样周期：`200ms`
  - 很可能漏采
- **结论：** 需要缩短采样周期，具体数值需按 CPU 性能和 Trace 支持能力校核。

> [!tip] 记忆口诀
> **快信号短周期，慢过程长时间。**

# 09｜一页速记总卡

> [!info] 核心结论
> **测量的核心价值是保存 Trace 采集结果，让工程师可以对故障、调参和动作过程进行复盘。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
Trace 抓过程，测量存结果；曲线看趋势，数据做复盘。
</div>

## ① 分类/公式/规则

| 主轴 | 关键词 |
|---|---|
| **对象** | 测量 Measurements |
| **来源** | Trace 采集结果 |
| **用途** | 保存、查看、对比、导出 |
| **适合分析** | PID、轴控、模拟量、通信、动作时序 |
| **核心公式** | `记录总时长 = 采样周期 × 采样点数` |
| **重点风险** | 变量错、周期错、命名乱 |

## ② 参考表

| 需求 | 推荐做法 |
|---|---|
| **调 PID** | 保存 SP、PV、Output 曲线 |
| **调伺服轴** | 保存目标、实际、误差曲线 |
| **查通信故障** | 保存 REQ、BUSY、DONE、ERROR、STATUS |
| **查模拟量波动** | 保存 Raw、Real、Filtered |
| **查气缸超时** | 保存 Cmd、到位、Timeout |
| **写调试报告** | 导出测量数据 |

## ③ 计算/选型示例

- **已知：** 要记录 `120s` 的液位变化，采样周期为 `500ms`
- **公式：** 采样点数 = `记录时长 ÷ 采样周期`
- **计算：**
  - `120s ÷ 0.5s = 240`
  - 至少需要 `240` 个采样点
- **结论：** 该液位测量至少需要约 `240` 点，建议留一定余量。

> [!tip] 记忆口诀
> **要调得准，先量得清；要能复盘，测量要存。**
