# 01｜外部源文件是什么

> [!info] 核心结论
> **外部源文件是 TIA Portal 中用于存放、导入、编辑和生成 PLC 程序块的文本源文件区域，常用于 SCL 源码、UDT、DB、FC、FB 等程序结构的版本管理。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>图片识别</strong><br>
截图中的 <strong>外部源文件</strong> 是 TIA Portal 项目树里的源码管理入口，不是 OB / FB / FC / DB 这种直接运行的程序块。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **中文名称** | 外部源文件 |
| **英文理解** | External Source Files |
| **核心作用** | 用文本源码生成 PLC 块 |
| **常见内容** | SCL 源文件、UDT、DB、函数/函数块源码 |
| **主要用途** | 导入、导出、版本管理、批量生成块 |
| **是否直接执行** | 否，需要生成 PLC 块后才参与运行 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
外部源文件就像 PLC 程序的“源码草稿区”，真正运行的是由它生成出来的 OB / FB / FC / DB。
</div>

## ② 参考表

| 对象 | 是否运行 | 作用 |
|---|---|---|
| **外部源文件** | 否 | 保存源码文本 |
| **OB** | 是 | 程序入口 |
| **FB** | 是 | 带实例数据的功能块 |
| **FC** | 是 | 无状态函数 |
| **DB** | 否，但被程序访问 | 保存数据 |
| **UDT** | 否 | 定义数据结构类型 |

## ③ 计算/选型示例

- **已知：** 你有一份 `MotorControl.scl` 源码  
- **公式：** 外部源文件 → 生成 PLC 块 → 编译下载  
- **计算：**
  - 将 `.scl` 文件导入“外部源文件”
  - 右键生成块
  - TIA 生成对应 `FB_MotorControl`
  - 编译后下载到 PLC
- **结论：** 外部源文件本身不执行，必须先生成程序块。

> [!tip] 记忆口诀
> **源码放外部源，运行靠生成块。**

# 02｜它和 PLC 块的关系

> [!info] 核心结论
> **外部源文件是“源码”，PLC 块是“生成结果”；源码改了，不代表 PLC 块自动更新，通常需要重新生成和编译。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>关系链路</strong><br>
外部源文件 → 生成 OB / FB / FC / DB / UDT → 编译 → 下载到 PLC
</div>

## ① 分类/公式/规则

| 阶段 | 内容 | 作用 |
|---|---|---|
| **源文件阶段** | `.scl` / 源码文本 | 便于编辑和管理 |
| **生成阶段** | 从源文件生成块 | 变成 TIA 项目块 |
| **编译阶段** | 检查语法和类型 | 确认可下载 |
| **下载阶段** | 写入 PLC | PLC 执行新程序 |
| **运行阶段** | PLC 执行块 | 源文件不直接参与扫描 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>核心规则</strong><br>
改源文件后，记得重新生成块；改块后，源文件不一定自动同步。
</div>

## ② 参考表

| 操作 | 影响源文件 | 影响 PLC 块 |
|---|---|---|
| **导入源文件** | 增加源码 | 不一定立即生成块 |
| **生成块** | 源文件不变 | 新建或更新块 |
| **修改源文件** | 源码改变 | 块需重新生成 |
| **修改已生成块** | 源文件不一定变 | 块改变 |
| **下载到 PLC** | 源文件不下载 | 下载已编译块 |

## ③ 计算/选型示例

- **已知：** 修改了外部源文件中的电机启动逻辑  
- **公式：** 修改源码 ≠ PLC 自动更新  
- **计算：**
  - 修改 `.scl` 文件
  - 重新生成 `FB_Motor`
  - 编译项目
  - 下载到 PLC
- **结论：** 只改外部源文件而不生成块，PLC 程序不会真正变化。

> [!tip] 记忆口诀
> **源文件改完要生成，生成之后再编译。**

# 03｜常见文件类型

> [!info] 核心结论
> **外部源文件常见于 SCL 源码、数据块源文件、UDT 类型定义和部分文本化程序文件，具体支持格式需按 TIA 版本和 CPU 类型校核。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程理解</strong><br>
外部源文件最常见的价值，是让 PLC 程序像普通代码一样可以导入、导出和版本管理。
</div>

## ① 分类/公式/规则

| 类型 | 说明 | 常见用途 |
|---|---|---|
| **SCL 源文件** | 结构化控制语言源码 | 生成 FC / FB |
| **DB 源文件** | 数据块定义 | 生成数据块 |
| **UDT 源文件** | 用户自定义类型 | 生成结构类型 |
| **块源文件** | 程序块文本表示 | 迁移和复用 |
| **库/模板源文件** | 标准化代码 | 项目复用 |

> [!warning]
> 不同 TIA Portal 版本、PLC 系列和项目设置支持的源文件格式可能不同，需按实际软件帮助和项目环境校核。

## ② 参考表

| 文件用途 | 推荐放法 | 注意 |
|---|---|---|
| **算法函数** | SCL 源文件 | 适合版本管理 |
| **设备模板 FB** | SCL 源文件 | 接口要稳定 |
| **结构体定义** | UDT 源文件 | 先生成类型 |
| **通信缓冲区** | DB 源文件 | 地址和类型要清楚 |
| **批量标准块** | 外部源文件管理 | 命名要规范 |

## ③ 计算/选型示例

- **已知：** 你要把 10 个项目都用的模拟量换算函数统一管理  
- **公式：** 通用函数源码 → 外部源文件  
- **计算：**
  - 建立 `FC_Scale.scl`
  - 每个项目导入该源文件
  - 生成 `FC_Scale`
  - 后期统一维护源码版本
- **结论：** 通用函数适合用外部源文件做标准化管理。

> [!tip] 记忆口诀
> **通用逻辑做源码，多个项目好复用。**

# 04｜为什么要用外部源文件

> [!info] 核心结论
> **外部源文件最大的价值是：方便版本管理、批量生成、代码复用、文本对比和跨项目迁移。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>选型口诀</strong><br>
项目小可直接写块，项目大要管理源码；多人协作，外部源更清楚。
</div>

## ① 分类/公式/规则

| 优点 | 说明 |
|---|---|
| **版本管理** | 可用 Git / SVN 管理文本 |
| **代码复用** | 标准块可跨项目导入 |
| **批量生成** | 多个块可从源码生成 |
| **文本对比** | 容易看出修改差异 |
| **标准化开发** | 统一命名、接口、模板 |
| **备份迁移** | 源码文件更便于保存 |

## ② 参考表

| 场景 | 是否推荐外部源文件 | 原因 |
|---|---|---|
| **小型单机项目** | 可选 | 直接写块也可以 |
| **大型项目** | 推荐 | 便于维护 |
| **多人协作** | 推荐 | 方便版本控制 |
| **标准库开发** | 推荐 | 便于复用 |
| **临时测试** | 不一定 | 直接新建块更快 |
| **长期知识积累** | 推荐 | 源码可沉淀 |

## ③ 计算/选型示例

- **已知：** 一个标准电机 FB 要在 20 条产线复用  
- **公式：** 标准代码复用 = 外部源文件 + 版本号管理  
- **计算：**
  - 将 `FB_Motor.scl` 作为标准源文件
  - 每次修改记录版本
  - 各项目导入对应版本
- **结论：** 外部源文件适合企业标准块和长期维护项目。

> [!tip] 记忆口诀
> **一次写好，多处生成；源码管理，项目更稳。**

# 05｜典型使用流程

> [!info] 核心结论
> **外部源文件的典型流程是：创建或导入源码 → 检查语法 → 生成块 → 编译项目 → 下载测试。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>使用流程</strong><br>
导入源文件 → 生成块 → 编译 → 调试 → 下载 → 归档源码
</div>

## ① 分类/公式/规则

| 步骤 | 操作 | 重点 |
|---|---|---|
| **1** | 新建或导入源文件 | 文件格式正确 |
| **2** | 检查源码内容 | 块名、接口、类型 |
| **3** | 生成 PLC 块 | 注意是否覆盖旧块 |
| **4** | 编译项目 | 检查语法和引用 |
| **5** | 下载 PLC | 只下载生成后的块 |
| **6** | 在线测试 | 验证逻辑 |
| **7** | 版本归档 | 保存源码版本 |

## ② 参考表

| 操作点 | 注意事项 |
|---|---|
| **块名** | 不要和已有块冲突 |
| **变量类型** | 必须和项目中类型一致 |
| **UDT 引用** | 先生成 UDT，再生成使用它的块 |
| **DB 结构** | 通信项目中不要随意改偏移 |
| **覆盖生成** | 小心覆盖手工改动 |
| **版本号** | 建议写入注释区 |

## ③ 计算/选型示例

- **已知：** `FB_Valve.scl` 引用了 `UDT_Valve`  
- **公式：** 被引用类型必须先存在  
- **计算：**
  - 先导入并生成 `UDT_Valve`
  - 再导入并生成 `FB_Valve`
  - 最后编译整个项目
- **结论：** 外部源文件生成顺序很重要，依赖类型要先生成。

> [!tip] 记忆口诀
> **先类型，后功能；先生成，后编译。**

# 06｜外部源文件与版本管理

> [!info] 核心结论
> **外部源文件是 PLC 项目进行 Git / SVN 版本管理的关键入口，因为它是可读、可比较、可追踪的文本文件。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程价值</strong><br>
TIA 工程文件整体不容易文本对比，但外部源文件可以像普通代码一样管理修改记录。
</div>

## ① 分类/公式/规则

| 版本管理项 | 建议 |
|---|---|
| **文件命名** | 块名 + 功能 + 版本 |
| **注释头** | 写作者、日期、版本、修改说明 |
| **目录结构** | 按功能分类 |
| **变更记录** | 每次修改写清原因 |
| **兼容性说明** | 标注 TIA 版本和 CPU |
| **测试记录** | 记录已验证项目 |

## ② 参考表

| 文件命名示例 | 用途 |
|---|---|
| **FB_Motor_V1_0.scl** | 电机控制模板 |
| **FC_ScaleAnalog_V1_2.scl** | 模拟量换算 |
| **UDT_Valve_V1_0.scl** | 阀门结构 |
| **DB_CommMap_V1_1.scl** | 通信映射 |
| **FB_ModbusMaster_V2_0.scl** | 通信功能块 |

## ③ 计算/选型示例

- **已知：** 你修改了电机 FB 的报警复位逻辑  
- **公式：** 代码变化 = 版本号 + 修改记录  
- **计算：**
  - 原版本：`V1.0`
  - 新版本：`V1.1`
  - 修改说明：优化报警复位条件
- **结论：** 外部源文件要配合版本说明，后期排故才知道改了什么。

> [!tip] 记忆口诀
> **源码不写版本，后期排故很痛苦。**

# 07｜常见错误与排查

> [!info] 核心结论
> **外部源文件常见问题集中在语法错误、块名冲突、数据类型缺失、生成顺序错误和覆盖旧块。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>避坑提醒</strong><br>
从外部源文件生成块时，可能覆盖已有块；生成前先确认是否有手工修改未保存。
</div>

## ① 分类/公式/规则

| 问题 | 常见原因 |
|---|---|
| **无法生成块** | 源码语法错误 |
| **类型找不到** | UDT 或 DB 未先生成 |
| **块名冲突** | 项目中已有同名块 |
| **接口报错** | 参数类型不匹配 |
| **生成后逻辑不对** | 源码版本不是最新 |
| **通信地址错位** | DB 结构被修改 |
| **下载后无变化** | 只改源文件未重新生成块 |

## ② 参考表

| 现象 | 排查方向 | 解决建议 |
|---|---|---|
| **生成失败** | 看错误列表 | 修正语法 |
| **变量类型红色** | 查 UDT / DB 是否存在 | 先生成依赖 |
| **块被覆盖** | 查同名块 | 生成前备份 |
| **程序无变化** | 是否重新生成和下载 | 完整编译下载 |
| **HMI 变量错** | DB 地址是否变动 | 校核 HMI 变量 |
| **多人版本冲突** | 源码版本不一致 | 用版本管理合并 |

## ③ 计算/选型示例

- **已知：** 修改了 `DB_Comm.scl`，新增一个 `Real` 变量放在 DB 前面  
- **公式：** 前面新增变量 → 后续地址可能偏移  
- **计算：**
  - `Real` 常占 `4 Byte`
  - 后面的通信字段可能整体后移
  - 上位机若仍读旧地址，会读错数据
- **结论：** 通信用 DB 源文件修改结构时，必须同步检查通信地址表。

> [!tip] 记忆口诀
> **生成失败看语法，数据错位看地址，逻辑没变看是否重新生成。**

# 08｜一页速记总卡

> [!info] 核心结论
> **外部源文件的核心价值是把 PLC 程序文本化、标准化、可复用化，适合长期项目积累和版本管理。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
源码放外部源，块由源码生成；先生成再编译，先测试再下载。
</div>

## ① 分类/公式/规则

| 主轴 | 关键词 |
|---|---|
| **对象** | 外部源文件 |
| **作用** | 存放源码 |
| **生成结果** | OB / FB / FC / DB / UDT |
| **优势** | 版本管理、复用、迁移 |
| **注意** | 源文件不直接运行 |
| **风险** | 覆盖块、地址偏移、依赖缺失 |

## ② 参考表

| 需求 | 推荐做法 |
|---|---|
| **复用标准块** | 用外部源文件 |
| **多人协作** | 用源码版本管理 |
| **简单临时逻辑** | 可直接写块 |
| **批量生成块** | 用外部源文件 |
| **通信 DB 固定地址** | 慎改源文件结构 |
| **长期知识积累** | 保存源码和版本说明 |

## ③ 计算/选型示例

- **已知：** 你要建立自己的 PLC 标准库  
- **公式：** 标准库 = `UDT + FB + FC + DB 源文件 + 版本说明`
- **计算：**
  - UDT 定义设备结构
  - FB 封装设备逻辑
  - FC 封装公共算法
  - DB 定义参数和通信映射
  - 源文件统一归档
- **结论：** 外部源文件非常适合 PLC 学习笔记、企业标准块和项目长期维护。

> [!tip] 记忆口诀
> **小项目看块，大项目看源码；会用外部源，程序更专业。**
