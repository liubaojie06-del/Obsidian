# 01｜合并测量是什么

> [!info] 核心结论
> **合并测量是 TIA Portal 中用于把多次 Trace 测量结果放到一起对比分析的工具，适合查看调参前后、故障前后、不同工况下的变量曲线差异。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>图片识别</strong><br>
截图中的 <strong>合并测量</strong> 通常用于管理和对比多次测量数据，不是新的 PLC 程序块，也不是实时控制指令。
</div>

## ① 分类/公式/规则

| 对象 | 中文理解 | 作用 |
|---|---|---|
| **轨迹 Trace** | 采集配置 | 决定采哪些变量 |
| **测量 Measurements** | 单次采集结果 | 保存一次曲线 |
| **合并测量** | 多次结果对比 | 把多组曲线放一起看 |
| **变量曲线** | 数据变化过程 | 分析时间、峰值、趋势 |
| **对比分析** | 调参复盘 | 比较不同参数效果 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
测量是“单次曲线”，合并测量是“多次曲线叠在一起比较”。
</div>

## ② 参考表

| 功能 | 重点 | 典型用途 |
|---|---|---|
| **单次测量** | 看一次动作过程 | 分析某次故障 |
| **合并测量** | 看多次曲线差异 | 对比调参效果 |
| **曲线对比** | 峰值、延迟、稳定时间 | PID / 轴控调试 |
| **数据复盘** | 留存测试结果 | 写调试报告 |
| **趋势分析** | 多工况对比 | 找最佳参数 |

## ③ 计算/选型示例

- **已知：** 伺服轴调了两版加速度参数，需要比较哪版更平稳  
- **公式：** 对比分析 = `测量1 + 测量2 + 同变量曲线`
- **计算：**
  - 测量 1：旧加速度参数
  - 测量 2：新加速度参数
  - 合并查看目标位置、实际位置、速度、误差
- **结论：** 合并测量可以直观看出新参数是否降低超调、缩短定位时间。

> [!tip] 记忆口诀
> **一次测量看过程，多次合并看差异。**

# 02｜它和 Trace / 测量的关系

> [!info] 核心结论
> **Trace 负责采集，测量负责保存单次结果，合并测量负责把多个结果放在一起分析。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>关系链路</strong><br>
Trace 设置变量 → 采集生成测量 → 多个测量合并 → 对比曲线差异
</div>

## ① 分类/公式/规则

| 层级 | 作用 | 类比 |
|---|---|---|
| **Trace** | 配置采集任务 | 相机设置 |
| **测量** | 保存一次采集结果 | 一张照片 |
| **合并测量** | 对比多次采集结果 | 多张照片叠加对比 |
| **导出数据** | 离线分析 | 保存证据 |
| **调试报告** | 记录结论 | 复盘文档 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>核心规则</strong><br>
没有测量结果，就没有合并测量；合并测量的前提是已经采集过多组数据。
</div>

## ② 参考表

| 需求 | 推荐工具 |
|---|---|
| **看当前变量值** | 监控表 |
| **抓变量变化过程** | Trace |
| **保存一次曲线** | 测量 |
| **比较两次调试效果** | 合并测量 |
| **查 PLC 停机原因** | 诊断缓冲区 |
| **测试输出点** | 强制表，谨慎使用 |

## ③ 计算/选型示例

- **已知：** 温控 PID 参数从 V1 改到 V2  
- **公式：** 调参效果 = `V1测量曲线` 对比 `V2测量曲线`
- **计算：**
  - V1：记录 SP、PV、Output
  - V2：记录同样变量
  - 合并测量中比较超调和稳定时间
- **结论：** 只有使用相同变量和相近工况采集，合并对比才有意义。

> [!tip] 记忆口诀
> **Trace 负责抓，测量负责存，合并负责比。**

# 03｜适合分析什么问题

> [!info] 核心结论
> **合并测量适合分析“变化前后差异”，尤其适合 PID 调参、伺服调试、模拟量滤波、通信故障复盘。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程定位</strong><br>
只要你想回答“改之前和改之后有什么不同”，就可以用合并测量。
</div>

## ① 分类/公式/规则

| 场景 | 合并对比内容 |
|---|---|
| **PID 调试** | SP、PV、Output、Error |
| **伺服定位** | 目标位置、实际位置、速度、误差 |
| **模拟量滤波** | 原始值、滤波值、工程量 |
| **气缸动作** | 命令、到位、超时 |
| **通信排查** | REQ、BUSY、DONE、ERROR、STATUS |
| **电机启动** | 命令、反馈、电流、故障 |

## ② 参考表

| 对比目标 | 重点指标 |
|---|---|
| **谁响应更快** | 上升时间 / 到位时间 |
| **谁更稳定** | 波动幅度 / 振荡次数 |
| **谁超调更小** | 峰值 - 目标值 |
| **谁误差更小** | 目标值 - 实际值 |
| **谁动作更顺** | 速度曲线 / 加减速曲线 |
| **谁故障更少** | Error 出现时间和次数 |

## ③ 计算/选型示例

- **已知：**
  - 旧参数下温度峰值：`88℃`
  - 新参数下温度峰值：`83℃`
  - 设定温度：`80℃`
- **公式：** 超调量 = `峰值 - 设定值`
- **计算：**
  - 旧参数超调：`88 - 80 = 8℃`
  - 新参数超调：`83 - 80 = 3℃`
- **结论：** 新参数把超调从 `8℃` 降到 `3℃`，控温效果更稳。

> [!tip] 记忆口诀
> **改参数别凭感觉，合并曲线看结果。**

# 04｜合并测量的关键前提

> [!info] 核心结论
> **合并测量要有可比性，必须保证变量一致、工况接近、采样周期合理、命名清楚。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>对比规则</strong><br>
变量不一致，不能硬比；工况不一致，结论不准。
</div>

## ① 分类/公式/规则

| 前提 | 说明 |
|---|---|
| **变量一致** | 两次测量记录同一组变量 |
| **工况一致** | 负载、速度、设定值尽量相同 |
| **采样合理** | 周期足够捕捉变化 |
| **时间对齐** | 关注相同触发点后的变化 |
| **命名规范** | 知道每条曲线对应什么条件 |
| **参数记录** | 保存调参版本 |

## ② 参考表

| 项目 | 好的做法 | 不推荐做法 |
|---|---|---|
| **变量选择** | 两次都选 SP/PV/Output | 每次变量不同 |
| **采样周期** | 保持一致或记录清楚 | 随便设置 |
| **测试条件** | 同一设定值、同一负载 | 工况变化太大 |
| **曲线命名** | 带日期、参数版本 | 只叫“测量1” |
| **结论记录** | 写明哪个参数更好 | 只保存曲线不说明 |

## ③ 计算/选型示例

- **已知：** 要比较两次轴定位效果  
- **公式：** 可比数据 = `同目标位置 + 同速度 + 同采样变量`
- **计算：**
  - 两次都移动 `100mm`
  - 两次都记录 `TargetPosition`、`ActualPosition`、`FollowingError`
  - 两次都使用相近采样周期
- **结论：** 这样合并出来的曲线才适合判断哪次定位更好。

> [!tip] 记忆口诀
> **要比较，先统一；变量统一，工况统一。**

# 05｜合并测量怎么看

> [!info] 核心结论
> **看合并测量时，重点看曲线的峰值、延迟、响应时间、稳定时间、误差和异常点。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>看图口诀</strong><br>
先看谁先动，再看谁超高；先看谁稳定，再看谁误差小。
</div>

## ① 分类/公式/规则

| 指标 | 说明 |
|---|---|
| **响应时间** | 从命令到反馈变化的时间 |
| **超调量** | 实际值超过目标值多少 |
| **稳定时间** | 进入允许范围所需时间 |
| **峰值** | 最大值或最小值 |
| **误差** | 目标值与实际值差 |
| **波动幅度** | 稳定后上下摆动范围 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点公式</strong><br>
响应时间 = 反馈变化时间 - 命令发出时间
</div>

## ② 参考表

| 曲线现象 | 可能说明 |
|---|---|
| **新曲线更快到达目标** | 响应速度提升 |
| **新曲线峰值更低** | 超调减小 |
| **新曲线波动更小** | 稳定性更好 |
| **新曲线输出更平滑** | 执行器压力更小 |
| **新曲线误差更小** | 控制精度更高 |
| **新曲线故障点消失** | 排故有效 |

## ③ 计算/选型示例

- **已知：**
  - 旧测量到位时间：`1.8s`
  - 新测量到位时间：`1.2s`
- **公式：** 时间缩短 = `旧时间 - 新时间`
- **计算：**
  - `1.8 - 1.2 = 0.6s`
- **结论：** 新参数让动作到位时间缩短了约 `0.6s`，但还要确认是否增加超调或冲击。

> [!tip] 记忆口诀
> **快不一定好，还要看稳不稳。**

# 06｜PID 合并测量示例

> [!info] 核心结论
> **PID 调参后用合并测量对比 SP、PV、Output，可以判断响应速度、超调和稳定性是否改善。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>PID 对比三件套</strong><br>
设定值 SP、过程值 PV、控制输出 Output。
</div>

## ① 分类/公式/规则

| 曲线 | 看什么 |
|---|---|
| **SP** | 目标是否一致 |
| **PV** | 是否跟得上目标 |
| **Output** | 输出是否过大或频繁 |
| **Error** | 偏差是否减少 |
| **Mode** | 是否都在自动模式 |

## ② 参考表

| 合并对比结果 | 结论方向 |
|---|---|
| **PV 更快接近 SP** | 响应改善 |
| **PV 超调更小** | 参数更稳 |
| **Output 不再频繁跳动** | 执行器更轻松 |
| **Error 稳态更小** | 精度提高 |
| **PV 仍然波动大** | 参数仍需优化 |

## ③ 计算/选型示例

- **已知：**
  - 设定值：`100℃`
  - V1 峰值：`112℃`
  - V2 峰值：`104℃`
- **公式：** 超调量 = `峰值 - 设定值`
- **计算：**
  - V1 超调：`112 - 100 = 12℃`
  - V2 超调：`104 - 100 = 4℃`
- **结论：** V2 参数明显降低超调，更适合当前温控系统。

> [!tip] 记忆口诀
> **PID 调得好不好，合并曲线最知道。**

# 07｜运动控制合并测量示例

> [!info] 核心结论
> **运动控制调试中，合并测量适合对比不同速度、加速度、减速度或伺服增益下的定位效果。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>轴控对比三条线</strong><br>
目标位置、实际位置、跟随误差。
</div>

## ① 分类/公式/规则

| 对比变量 | 作用 |
|---|---|
| **TargetPosition** | 目标位置是否一致 |
| **ActualPosition** | 实际运动轨迹 |
| **ActualVelocity** | 速度曲线是否平滑 |
| **FollowingError** | 跟随误差大小 |
| **Busy / Done** | 动作时间 |
| **ErrorID** | 是否产生故障 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点公式</strong><br>
跟随误差 = 目标位置 - 实际位置
</div>

## ② 参考表

| 合并对比结果 | 可能说明 |
|---|---|
| **新曲线误差更小** | 跟随性更好 |
| **新曲线速度更平滑** | 加减速更合理 |
| **新曲线超调更小** | 减速或增益更合适 |
| **新曲线 Done 更早** | 节拍提升 |
| **新曲线 Error 消失** | 参数改善有效 |

## ③ 计算/选型示例

- **已知：**
  - V1 最大跟随误差：`2.5mm`
  - V2 最大跟随误差：`1.0mm`
- **公式：** 误差降低量 = `V1误差 - V2误差`
- **计算：**
  - `2.5 - 1.0 = 1.5mm`
- **结论：** V2 让最大跟随误差降低约 `1.5mm`，运动控制效果更好。

> [!tip] 记忆口诀
> **轴控不只看 Done，更要看误差曲线。**

# 08｜常见错误与排查

> [!info] 核心结论
> **合并测量常见问题是测量不具备可比性、曲线变量不一致、采样周期不同、命名混乱、结论误判。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>避坑提醒</strong><br>
合并测量不是简单把曲线叠一起，必须确认测试条件相近，否则容易得出错误结论。
</div>

## ① 分类/公式/规则

| 问题 | 常见原因 |
|---|---|
| **曲线无法比较** | 变量不一致 |
| **时间轴难对齐** | 触发条件不同 |
| **结果差异很大** | 工况不同 |
| **看不懂哪条曲线** | 命名不规范 |
| **结论误判** | 只看速度，不看超调 |
| **数据缺失** | 测量未保存完整 |

## ② 参考表

| 现象 | 排查方向 | 解决建议 |
|---|---|---|
| **两条曲线起点不同** | 触发条件 | 使用相同触发 |
| **曲线变量不同** | 变量配置 | 统一 Trace 变量 |
| **一条曲线很粗糙** | 采样周期 | 统一采样设置 |
| **不知道参数版本** | 命名/记录缺失 | 增加版本说明 |
| **新参数更快但超调大** | 只看速度 | 综合判断稳定性 |
| **无法复现结果** | 工况未记录 | 记录负载和设定值 |

## ③ 计算/选型示例

- **已知：** 两次测量中，一次速度设定 `100mm/s`，另一次 `300mm/s`
- **公式：** 工况不同 → 不能直接比较定位时间
- **计算：**
  - 速度不同会直接影响到位时间
  - 若只看谁更快，会误判参数好坏
- **结论：** 比较控制参数时，应尽量保持速度、目标位置、负载一致。

> [!tip] 记忆口诀
> **合并前先问一句：这两次真的能比吗？**

# 09｜一页速记总卡

> [!info] 核心结论
> **合并测量的核心价值是对比多次测量曲线，帮助工程师判断调参、排故、优化是否真的有效。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
单次测量看过程，合并测量看变化；变量要一致，工况要接近，结论才可信。
</div>

## ① 分类/公式/规则

| 主轴 | 关键词 |
|---|---|
| **对象** | 合并测量 |
| **来源** | 多次 Trace 测量结果 |
| **用途** | 对比、复盘、优化 |
| **适合分析** | PID、轴控、通信、模拟量 |
| **核心指标** | 响应时间、超调、稳定时间、误差 |
| **风险点** | 工况不一致、变量不一致 |

## ② 参考表

| 需求 | 合并对比内容 |
|---|---|
| **PID 参数优化** | SP、PV、Output、Error |
| **伺服参数优化** | 目标、实际、速度、误差 |
| **滤波效果对比** | Raw、Filtered、Real |
| **通信故障复盘** | REQ、BUSY、DONE、ERROR、STATUS |
| **气缸节拍优化** | Cmd、到位、Timeout |
| **电机启动优化** | RunCmd、RunFB、Current、Fault |

## ③ 计算/选型示例

- **已知：** 要比较滤波前后压力信号波动  
- **公式：** 波动幅度 = `最大值 - 最小值`
- **计算：**
  - 滤波前：最大 `5.8bar`，最小 `4.2bar`，波动 `1.6bar`
  - 滤波后：最大 `5.2bar`，最小 `4.8bar`，波动 `0.4bar`
- **结论：** 滤波后压力波动明显减小，但还要确认响应是否变慢。

> [!tip] 记忆口诀
> **对比看曲线，结论看数据；快、稳、准都要看。**
