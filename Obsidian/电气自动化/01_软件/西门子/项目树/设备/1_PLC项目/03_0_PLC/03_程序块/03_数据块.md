# 01｜数据块 DB 是什么

> [!info] 核心结论
> **DB 是 PLC 用来存储数据的程序块，本身不执行逻辑，只负责保存变量、参数、状态、配方和实例数据。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
DB 就是 PLC 的“数据仓库”，程序负责计算，DB 负责记住。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **英文名称** | Data Block |
| **中文名称** | 数据块 |
| **简称** | `DB` |
| **核心作用** | 存储变量和数据 |
| **是否执行逻辑** | 不执行 |
| **常见用途** | 参数、配方、状态、报警、实例数据 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>核心规则</strong><br>
OB / FB / FC 负责执行逻辑，DB 负责保存数据。
</div>

## ② 参考表

| 块类型 | 主要作用 | 是否存数据 |
|---|---|---|
| **OB** | 程序入口 / 调度 | 少量系统相关 |
| **FB** | 带状态的功能块 | 通过实例 DB 存 |
| **FC** | 无状态函数 | 通常不存 |
| **DB** | 数据存储 | **是** |

## ③ 计算/选型示例

- **已知：** 设备需要保存温度设定值、速度设定值、报警状态  
- **公式：** 需要集中保存数据 → 建立 DB  
- **计算：**
  - 温度设定值放入 `DB_Para.TempSet`
  - 速度设定值放入 `DB_Para.SpeedSet`
  - 报警状态放入 `DB_Status.Alarm`
- **结论：** 需要长期保存或集中管理的数据，适合放在 DB 中。

> [!tip] 记忆口诀
> **程序负责干活，DB 负责记账。**

# 02｜全局 DB

> [!info] 核心结论
> **全局 DB 是可以被多个 OB、FB、FC 共同访问的数据块，适合存放公共参数、状态、配方和工艺数据。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
全局 DB 就像项目里的“公共资料库”，谁需要都可以来读写。
</div>

## ① 分类/公式/规则

| 数据类型 | 适合放全局 DB 吗 | 说明 |
|---|---|---|
| **工艺参数** | 适合 | 温度、压力、速度设定 |
| **配方数据** | 适合 | 产品型号参数 |
| **设备状态** | 适合 | 运行、报警、模式 |
| **统计数据** | 适合 | 产量、节拍、不良数 |
| **临时计算值** | 不太适合 | 更适合 Temp |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>使用规则</strong><br>
公共数据放全局 DB，设备内部状态优先放实例 DB。
</div>

## ② 参考表

| 全局 DB 示例 | 存放内容 |
|---|---|
| **DB_Para** | 设备参数 |
| **DB_Recipe** | 产品配方 |
| **DB_Status** | 设备状态 |
| **DB_Alarm** | 报警信息 |
| **DB_HMI** | HMI 交互变量 |
| **DB_Comm** | 通信发送/接收数据 |

## ③ 计算/选型示例

- **已知：** HMI 需要修改设备运行速度设定值  
- **公式：** HMI 与 PLC 共享参数 → 全局 DB  
- **计算：**
  - 建立 `DB_HMI`
  - 创建变量 `SpeedSet`
  - HMI 写入 `DB_HMI.SpeedSet`
  - PLC 程序读取该变量参与控制
- **结论：** HMI 与 PLC 共用的变量适合放在全局 DB。

> [!tip] 记忆口诀
> **多人共用放全局，设备私有放实例。**

# 03｜实例 DB

> [!info] 核心结论
> **实例 DB 是 FB 的专属数据区，用来保存某一次 FB 调用的静态变量、定时器、状态和内部数据。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
FB 是逻辑模板，实例 DB 是这一次调用的“专属记忆本”。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **对应对象** | FB 调用 |
| **保存内容** | Static 变量、定时器、内部状态 |
| **典型用途** | 电机 FB、阀门 FB、通信 FB |
| **数据归属** | 某一个 FB 实例 |
| **是否建议手动乱改** | 不建议随意修改 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点公式</strong><br>
FB 调用 = FB 逻辑 + 实例 DB 数据
</div>

## ② 参考表

| 场景 | 实例 DB 作用 |
|---|---|
| **电机控制 FB** | 保存运行状态、故障锁存 |
| **阀门控制 FB** | 保存开关步骤、超时状态 |
| **气缸控制 FB** | 保存动作阶段、到位状态 |
| **通信 FB** | 保存发送步骤、错误码 |
| **定时器** | 保存计时状态 |

## ③ 计算/选型示例

- **已知：** 同一个 `FB_Motor` 控制 3 台电机  
- **公式：** 独立设备 = 独立实例 DB  
- **计算：**
  - 电机 1：`DB_Motor_1`
  - 电机 2：`DB_Motor_2`
  - 电机 3：`DB_Motor_3`
- **结论：** 同一个 FB 可以重复调用，但每台设备必须有自己的实例数据。

> [!tip] 记忆口诀
> **FB 可以共用，实例 DB 不能乱共用。**

# 04｜DB 变量类型

> [!info] 核心结论
> **DB 可以存放 BOOL、INT、REAL、WORD、ARRAY、STRUCT、UDT 等多种数据类型，是 PLC 数据组织的核心。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>变量口诀</strong><br>
位信号用 BOOL，整数用 INT，工程量用 REAL，成组数据用 STRUCT。
</div>

## ① 分类/公式/规则

| 数据类型 | 中文含义 | 常见用途 |
|---|---|---|
| **Bool** | 布尔量 | 启停、报警、状态位 |
| **Byte** | 字节 | 通信数据 |
| **Word** | 字 | 状态字、控制字 |
| **Int / DInt** | 整数 | 计数、编号 |
| **Real** | 浮点数 | 温度、压力、速度 |
| **Array** | 数组 | 批量数据 |
| **Struct** | 结构体 | 设备数据组 |
| **UDT** | 用户自定义类型 | 标准化数据结构 |

## ② 参考表

| 需求 | 推荐类型 | 注意 |
|---|---|---|
| **报警位** | `Bool` | 位信号 |
| **控制字** | `Word` | 常用于通信 |
| **温度值** | `Real` | 工程量 |
| **产品编号** | `DInt` / `String` | 看需求 |
| **发送缓冲区** | `Array of Byte` | 通信常用 |
| **电机数据包** | `Struct` / `UDT` | 结构清晰 |

## ③ 计算/选型示例

- **已知：** 需要保存 10 个字节的 TCP 发送数据  
- **公式：** 通信发送缓冲区 → `Array of Byte`  
- **计算：**
  - 建立变量：`SendData`
  - 类型：`Array[0..9] of Byte`
  - 发送长度：`10 Byte`
- **结论：** 通信数据区适合用 Byte 数组，便于按字节发送。

> [!tip] 记忆口诀
> **单个信号用 Bool，连续报文用 Byte 数组。**

# 05｜优化访问与非优化访问

> [!info] 核心结论
> **TIA Portal 中 DB 分为优化访问和非优化访问；优化访问适合符号编程，非优化访问适合绝对地址和兼容通信。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点区别</strong><br>
优化 DB 重视符号访问；非优化 DB 可以看到固定地址，如 <code>DB1.DBW0</code>。
</div>

## ① 分类/公式/规则

| 类型 | 特点 | 适合场景 |
|---|---|---|
| **优化访问 DB** | 符号访问、效率高 | S7-1200/1500 常规程序 |
| **非优化访问 DB** | 有固定偏移地址 | 通信、旧项目、HMI兼容 |
| **符号访问** | 用变量名访问 | `DB_Para.SpeedSet` |
| **绝对访问** | 用地址访问 | `DB1.DBW0` |

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>避坑提醒</strong><br>
如果第三方设备、HMI、S7 通信要按绝对地址访问 DB，通常需要确认 DB 是否允许非优化访问，需按项目规范校核。
</div>

## ② 参考表

| 场景 | 推荐方式 | 注意 |
|---|---|---|
| **新项目内部程序** | 优化访问 | 推荐符号编程 |
| **S7 通信访问 DB 地址** | 非优化或允许访问 | 地址要固定 |
| **第三方上位机读 DB** | 非优化常见 | 需权限与地址 |
| **HMI 符号变量** | 优化可用 | 看 HMI 支持 |
| **旧项目移植** | 非优化常见 | 注意地址变化 |

## ③ 计算/选型示例

- **已知：** 上位机要读取 `DB1.DBW0` 作为速度设定值  
- **公式：** 绝对地址访问 → DB 地址必须固定  
- **计算：**
  - 上位机访问地址：`DB1.DBW0`
  - DB 需支持该地址访问
  - 变量位置不能随意变动
- **结论：** 这种场景要谨慎使用优化 DB，需按通信方式和厂家手册校核。

> [!tip] 记忆口诀
> **内部程序用符号，外部通信看地址。**

# 06｜保持性 Retain

> [!info] 核心结论
> **DB 变量可以设置保持性 Retain，让 PLC 断电后仍保留关键数据，例如配方、计数和设定值。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
普通变量断电可能清零，Retain 变量断电后还能记住。
</div>

## ① 分类/公式/规则

| 数据 | 是否建议保持 | 说明 |
|---|---|---|
| **配方参数** | 建议 | 断电不能丢 |
| **累计产量** | 建议 | 需保存 |
| **报警历史** | 可选 | 看需求 |
| **临时状态** | 不建议 | 上电应重新初始化 |
| **动作输出命令** | 不建议 | 防止上电误动作 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>保持性规则</strong><br>
该记住的才 Retain，不该记住的不要 Retain。
</div>

## ② 参考表

| 变量 | 是否 Retain | 原因 |
|---|---|---|
| **SpeedSet** | 是 | 设定值需保留 |
| **ProductCount** | 是 | 累计值需保留 |
| **RunCmd** | 否 | 上电不应自动运行 |
| **Step** | 看工艺 | 有些设备需回初始 |
| **AlarmLatch** | 看需求 | 有些报警需保留 |

## ③ 计算/选型示例

- **已知：** 设备断电后，产品累计数量不能丢失  
- **公式：** 断电需要保存 → 设置 Retain  
- **计算：**
  - 变量：`ProductCount`
  - 放入 DB
  - 设置为保持性变量
- **结论：** 累计产量适合设置 Retain，但需考虑存储寿命和项目规范。

> [!tip] 记忆口诀
> **设定值要记住，动作命令别保持。**

# 07｜DB 与 HMI

> [!info] 核心结论
> **HMI 常通过 DB 与 PLC 交换数据，DB 是触摸屏参数设定、状态显示、报警交互的重要数据区。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程理解</strong><br>
HMI 写 DB 的设定值，PLC 读 DB 执行控制；PLC 写 DB 的状态，HMI 读 DB 显示画面。
</div>

## ① 分类/公式/规则

| 数据方向 | 说明 |
|---|---|
| **HMI → PLC** | 参数设定、按钮命令、配方下载 |
| **PLC → HMI** | 状态显示、报警信息、实时数值 |
| **双向数据** | 模式切换、配方确认、操作权限 |
| **建议做法** | HMI 变量集中放 DB，便于管理 |

## ② 参考表

| DB 变量 | HMI 用途 | 注意 |
|---|---|---|
| **StartButton** | 启动按钮 | 建议做脉冲或确认 |
| **SpeedSet** | 速度设定 | 限幅处理 |
| **TempActual** | 温度显示 | 单位清楚 |
| **AlarmCode** | 报警显示 | 与报警表对应 |
| **RecipeNo** | 配方编号 | 防止越界 |

## ③ 计算/选型示例

- **已知：** HMI 输入速度设定值，范围要求 `0~1500 rpm`  
- **公式：** HMI 输入值必须限幅  
- **计算：**
  - HMI 写入 `DB_HMI.SpeedSet`
  - PLC 判断小于 0 则取 0
  - 大于 1500 则取 1500
- **结论：** HMI 写入 DB 的参数必须在 PLC 侧做边界保护。

> [!tip] 记忆口诀
> **HMI 写设定，PLC 做校验；显示走 DB，控制更清晰。**

# 08｜DB 与通信

> [!info] 核心结论
> **通信收发数据通常会放在 DB 中，例如 TCP、Modbus、PROFINET、串口报文都需要发送区和接收区。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>通信口诀</strong><br>
发送数据放 Send DB，接收数据放 Receive DB；长度对不上，通信就乱套。
</div>

## ① 分类/公式/规则

| 通信数据区 | 作用 |
|---|---|
| **SendBuffer** | 发送缓冲区 |
| **RecvBuffer** | 接收缓冲区 |
| **ControlWord** | 控制字 |
| **StatusWord** | 状态字 |
| **RegisterMap** | Modbus 寄存器映射 |
| **PZD_Data** | PROFINET 过程数据 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点公式</strong><br>
通信字节数 = 数据元素数量 × 单个元素字节数
</div>

## ② 参考表

| 通信场景 | DB 设计建议 |
|---|---|
| **TCP发送** | `Array of Byte` |
| **Modbus寄存器** | `Array of Word` |
| **PROFINET PZD** | `Array of Word` 或结构体 |
| **串口ASCII** | Byte数组 / String |
| **报文解析** | 原始区 + 解析区 |

## ③ 计算/选型示例

- **已知：** Modbus 需要映射 20 个保持寄存器  
- **公式：** 1 个寄存器 = 1 Word = 2 Byte  
- **计算：**
  - 寄存器数量：`20`
  - 数据大小：`20 × 2 = 40 Byte`
  - 可建立 `Array[0..19] of Word`
- **结论：** Modbus 寄存器映射适合用 Word 数组放在 DB 中。

> [!tip] 记忆口诀
> **通信先建缓冲区，收发长度要对齐。**

# 09｜DB 常见错误

> [!info] 核心结论
> **DB 常见错误包括地址错位、优化访问不匹配、实例 DB 共用、变量类型错误、Retain 设置不当。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>避坑提醒</strong><br>
DB 不是“随便放变量”的地方，变量结构一变，通信地址和 HMI 变量可能都要同步检查。
</div>

## ① 分类/公式/规则

| 问题 | 典型原因 |
|---|---|
| **数据错位** | DB 结构变了，外部地址没改 |
| **HMI 显示异常** | 变量类型或地址不一致 |
| **通信读不到** | 优化访问/权限/地址问题 |
| **设备状态串台** | 多设备共用同一实例 DB |
| **断电数据丢失** | 未设置 Retain |
| **上电误动作** | 动作命令设置了 Retain |

## ② 参考表

| 现象 | 排查方向 | 解决建议 |
|---|---|---|
| **HMI 数值乱码** | 数据类型 | 检查 Real/Int/Word |
| **上位机读错值** | DB 地址偏移 | 固定地址并校核 |
| **FB 状态混乱** | 实例 DB | 每个设备独立实例 |
| **配方丢失** | Retain | 设置保持性 |
| **通信长度错误** | Buffer 长度 | 校核字节数 |
| **下载后变量变红** | 接口变化 | 更新调用和 HMI 变量 |

## ③ 计算/选型示例

- **已知：** 上位机读取 `DB1.DBW4`，但你在 DB 前面新增了一个 `Real` 变量  
- **公式：** 非优化 DB 地址会随结构变化产生偏移  
- **计算：**
  - `Real` 占 4 Byte
  - 后续变量地址整体可能后移
  - 上位机仍读旧地址会读错
- **结论：** 通信用 DB 结构不要随意插入变量，必要时预留地址或使用固定映射表。

> [!tip] 记忆口诀
> **通信 DB 少改结构，改了结构查地址。**

# 10｜一页速记总卡

> [!info] 核心结论
> **DB 的核心价值是保存数据：全局 DB 存公共数据，实例 DB 存 FB 状态，通信 DB 存收发缓冲。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
DB 管数据，FB 管状态，FC 管计算，OB 管入口。
</div>

## ① 分类/公式/规则

| DB 类型 | 一句话 |
|---|---|
| **全局 DB** | 公共数据仓库 |
| **实例 DB** | FB 的专属记忆 |
| **参数 DB** | 存工艺设定 |
| **配方 DB** | 存产品参数 |
| **通信 DB** | 存收发报文 |
| **HMI DB** | 存画面交互 |

## ② 参考表

| 需求 | 推荐 DB |
|---|---|
| **设备参数** | 参数 DB |
| **产品配方** | 配方 DB |
| **电机 FB 状态** | 实例 DB |
| **HMI 按钮和显示** | HMI DB |
| **TCP发送区** | 通信 DB |
| **Modbus寄存器映射** | 通信 DB / Register DB |
| **累计产量保持** | Retain DB |

## ③ 计算/选型示例

- **已知：** 项目需要管理设备参数、报警、HMI 交互、通信数据  
- **公式：** 不同数据用途 → 分 DB 管理  
- **计算：**
  - 参数：`DB_Para`
  - 报警：`DB_Alarm`
  - HMI：`DB_HMI`
  - 通信：`DB_Comm`
- **结论：** 按用途拆分 DB，可以让项目结构更清晰、维护更方便。
