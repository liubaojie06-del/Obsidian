# 01｜轨迹 Trace 是什么

> [!info] 核心结论
> **轨迹 Trace 是 TIA Portal 中用于在线采集 PLC 变量变化曲线的调试工具，适合分析速度、位置、温度、压力、PID 输出、通信状态等随时间变化的过程。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>图片识别</strong><br>
截图中的 <strong>轨迹</strong> 是 TIA Portal 项目树里的 Trace 功能入口，用来记录变量随时间变化的波形。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **中文名称** | 轨迹 |
| **英文名称** | Trace |
| **核心作用** | 记录变量随时间变化 |
| **常见对象** | PLC 变量、DB 变量、轴位置、PID 数据 |
| **显示形式** | 趋势曲线 / 波形图 |
| **主要用途** | 调试、整定、排故、过程分析 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
监控表只能看“当前值”，轨迹 Trace 能看“变化过程”。
</div>

## ② 参考表

| 工具 | 适合看什么 | 特点 |
|---|---|---|
| **监控表** | 当前变量值 | 实时看数值 |
| **强制表** | 临时强制信号 | 调试风险较高 |
| **轨迹 Trace** | 变量变化曲线 | 适合分析过程 |
| **诊断缓冲区** | 故障历史 | 查报警事件 |
| **在线趋势** | 工艺变化 | 看长期趋势 |

## ③ 计算/选型示例

- **已知：** 伺服轴定位时偶尔超调，监控表看不出原因  
- **公式：** 动态问题 → 用 Trace 看曲线  
- **计算：**
  - 记录目标位置
  - 记录实际位置
  - 记录速度
  - 记录 Busy / Done / Error 状态
- **结论：** Trace 可以看出轴是加速太猛、减速太晚，还是位置反馈异常。

> [!tip] 记忆口诀
> **静态看监控，动态看轨迹。**

# 02｜轨迹能记录什么

> [!info] 核心结论
> **Trace 可以记录 Bool、Int、DInt、Real、Word 等变量，也可用于分析运动控制、PID、模拟量和通信状态。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程理解</strong><br>
只要变量会随时间变化，而且你想知道“它怎么变”，就可以考虑放进 Trace。
</div>

## ① 分类/公式/规则

| 变量类型 | 适合记录吗 | 典型用途 |
|---|---|---|
| **Bool** | 适合 | 启动、到位、报警、触发信号 |
| **Int / DInt** | 适合 | 计数、位置、状态码 |
| **Real** | 适合 | 温度、压力、速度、PID 输出 |
| **Word** | 适合 | 控制字、状态字 |
| **Array** | 视情况 | 通信缓冲区局部分析 |
| **String** | 不常用 | 不适合曲线分析 |

## ② 参考表

| 应用场景 | 推荐记录变量 |
|---|---|
| **伺服定位** | 目标位置、实际位置、速度、Error |
| **PID 调试** | Setpoint、PV、Output、Error |
| **模拟量波动** | Raw、Real、滤波后值 |
| **通信排查** | REQ、BUSY、DONE、ERROR、STATUS |
| **气缸动作** | Cmd、到位信号、超时报警 |
| **电机启停** | RunCmd、RunFB、Fault |

## ③ 计算/选型示例

- **已知：** PID 温度控制波动较大  
- **公式：** PID 分析 = `设定值 + 实际值 + 输出值`
- **计算：**
  - 记录 `Temp_Set`
  - 记录 `Temp_PV`
  - 记录 `PID_Output`
  - 观察实际温度是否滞后、输出是否频繁变化
- **结论：** PID 调试时，Trace 比单纯看当前值更有价值。

> [!tip] 记忆口诀
> **看 PID 三条线：设定、反馈、输出。**

# 03｜轨迹参数

> [!info] 核心结论
> **Trace 的关键参数是采样周期、记录变量、触发条件、记录长度和存储方式。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>参数口诀</strong><br>
采样周期定精细，记录长度定范围，触发条件定时机。
</div>

## ① 分类/公式/规则

| 参数 | 作用 | 注意 |
|---|---|---|
| **采样周期** | 多久记录一次 | 太慢看不清，太快占资源 |
| **记录变量** | 要抓哪些数据 | 不要一次放太多 |
| **触发条件** | 什么时候开始记录 | 适合抓偶发故障 |
| **记录长度** | 记录多少点 | 影响时间范围 |
| **预触发** | 触发前也保留数据 | 便于看故障前状态 |
| **存储位置** | 在线或导出 | 便于复盘分析 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点公式</strong><br>
记录总时长 = 采样周期 × 采样点数
</div>

## ② 参考表

| 需求 | 采样周期建议 |
|---|---|
| **开关量动作顺序** | 较短周期 |
| **伺服位置曲线** | 较短周期 |
| **PID 温度曲线** | 中等或较长周期 |
| **液位变化** | 较长周期 |
| **通信状态跳变** | 按通信周期校核 |
| **偶发报警** | 配合触发条件 |

> [!warning]
> 具体采样周期要按 CPU 性能、变量数量、过程速度和 TIA 支持能力校核。

## ③ 计算/选型示例

- **已知：**
  - 采样周期：`100ms`
  - 采样点数：`1000`
- **公式：** 记录总时长 = `采样周期 × 采样点数`
- **计算：**
  - `100ms × 1000 = 100000ms`
  - `100000ms = 100s`
- **结论：** 该 Trace 可记录约 `100s` 的变量变化过程。

> [!tip] 记忆口诀
> **周期越短看得越细，点数越多看得越久。**

# 04｜触发条件

> [!info] 核心结论
> **触发条件用于在某个事件发生时自动开始记录，特别适合抓偶发故障和瞬间状态变化。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
触发条件就像 Trace 的“自动拍照按钮”：信号一到，马上开始记录。
</div>

## ① 分类/公式/规则

| 触发方式 | 说明 | 示例 |
|---|---|---|
| **手动启动** | 人工开始记录 | 调试观察 |
| **上升沿触发** | FALSE → TRUE | 启动命令 |
| **下降沿触发** | TRUE → FALSE | 信号断开 |
| **数值超过阈值** | 大于或小于某值 | 压力超限 |
| **错误信号触发** | Error=TRUE | 抓故障现场 |
| **状态码触发** | Status 等于某值 | 通信异常 |

## ② 参考表

| 问题 | 推荐触发条件 |
|---|---|
| **伺服偶发报警** | `Axis_Error = TRUE` |
| **通信偶发失败** | `ERROR = TRUE` |
| **压力突然升高** | `Pressure > HighLimit` |
| **气缸超时** | `Timeout_Alarm = TRUE` |
| **输出突然断开** | `RunCmd` 下降沿 |
| **PID 超调** | `PV > SP + 偏差阈值` |

## ③ 计算/选型示例

- **已知：** 通信偶发报错，平时正常  
- **公式：** 偶发故障 → 用 Error 触发 Trace  
- **计算：**
  - 触发变量：`Comm_Error`
  - 触发条件：上升沿
  - 记录变量：`REQ`、`BUSY`、`DONE`、`ERROR`、`STATUS`
- **结论：** 当通信错误出现时，Trace 会保留错误前后的状态，方便定位原因。

> [!tip] 记忆口诀
> **偶发问题靠触发，手动盯守不可靠。**

# 05｜轨迹与 PID 调试

> [!info] 核心结论
> **PID 调试时，Trace 可以同时观察设定值、反馈值、输出值和偏差，是整定参数的重要工具。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>PID 看图口诀</strong><br>
响应慢看 P，长期偏差看 I，超调振荡看参数太猛。
</div>

## ① 分类/公式/规则

| 曲线 | 含义 | 看什么 |
|---|---|---|
| **SP** | 设定值 | 目标是否变化 |
| **PV** | 实际值 | 是否跟随目标 |
| **Output** | 输出量 | 是否过大或频繁波动 |
| **Error** | 偏差 | 是否长期不消失 |
| **Mode** | 手自动 | 模式是否正确 |

## ② 参考表

| 曲线现象 | 可能原因 | 调整方向 |
|---|---|---|
| **PV 很慢接近 SP** | Kp 偏小 / 输出受限 | 增大响应或查执行器 |
| **PV 超过 SP 很多** | 参数太激进 / 惯性大 | 降 Kp 或减弱积分 |
| **PV 来回振荡** | Kp 太大 / I 太强 | 降 Kp、增 Ti |
| **Output 一直 100%** | 执行器能力不足 / 反馈异常 | 查硬件 |
| **PV 不变化** | 传感器或执行器异常 | 查输入输出链路 |

## ③ 计算/选型示例

- **已知：** 温度设定 `80℃`，实际温度超过 `90℃` 后才回落  
- **公式：** 超调量 = `峰值 - 设定值`
- **计算：**
  - `90 - 80 = 10℃`
  - 超调量为 `10℃`
- **结论：** 通过 Trace 可量化超调幅度，再调整 PID 参数或输出限幅。

> [!tip] 记忆口诀
> **PID 不看曲线调，基本都是瞎调。**

# 06｜轨迹与运动控制

> [!info] 核心结论
> **运动控制调试时，Trace 可以记录目标位置、实际位置、速度、加速度、跟随误差和轴状态。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程理解</strong><br>
轴动得准不准，不能只看 Done，要看位置曲线和误差曲线。
</div>

## ① 分类/公式/规则

| 变量 | 作用 |
|---|---|
| **TargetPosition** | 目标位置 |
| **ActualPosition** | 实际位置 |
| **ActualVelocity** | 实际速度 |
| **FollowingError** | 跟随误差 |
| **Busy** | 是否运动中 |
| **Done** | 是否完成 |
| **ErrorID** | 错误编号 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点公式</strong><br>
跟随误差 = 目标位置 - 实际位置
</div>

## ② 参考表

| 现象 | 可能原因 | 排查方向 |
|---|---|---|
| **实际位置滞后大** | 负载重 / 加速度大 | 降加速度或查机械 |
| **位置超调** | 减速不足 / 参数激进 | 调减速度和增益 |
| **速度曲线抖动** | 机械间隙 / 参数不稳 | 查机械和伺服 |
| **Done 很晚出现** | 到位窗口小 | 查定位容差 |
| **Error 突然出现** | 超限/跟随误差 | 查 ErrorID |

## ③ 计算/选型示例

- **已知：**
  - 目标位置：`100.0mm`
  - 实际位置最大到：`101.2mm`
- **公式：** 超调量 = `实际峰值 - 目标位置`
- **计算：**
  - `101.2 - 100.0 = 1.2mm`
- **结论：** 该轴定位超调约 `1.2mm`，需检查减速度、负载惯量和伺服参数。

> [!tip] 记忆口诀
> **轴控看三线：目标、实际、误差。**

# 07｜轨迹使用流程

> [!info] 核心结论
> **Trace 的典型流程是：新建轨迹 → 添加变量 → 设置采样 → 设置触发 → 在线启动 → 观察曲线 → 保存数据。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>标准流程</strong><br>
选变量 → 定周期 → 设触发 → 启动采集 → 分析曲线 → 保存结果
</div>

## ① 分类/公式/规则

| 步骤 | 操作 | 重点 |
|---|---|---|
| **1** | 新建 Trace | 命名清楚 |
| **2** | 添加变量 | 只加关键变量 |
| **3** | 设置采样周期 | 匹配过程速度 |
| **4** | 设置触发条件 | 抓关键事件 |
| **5** | 在线启动 | PLC 需在线 |
| **6** | 查看曲线 | 对比变量关系 |
| **7** | 保存/导出 | 便于复盘 |

## ② 参考表

| 命名示例 | 用途 |
|---|---|
| **Trace_PID_Temp** | 温度 PID 调试 |
| **Trace_Axis1_Move** | 轴 1 定位分析 |
| **Trace_Comm_Error** | 通信故障抓取 |
| **Trace_AI_Noise** | 模拟量噪声分析 |
| **Trace_Cylinder_Timeout** | 气缸超时排查 |

## ③ 计算/选型示例

- **已知：** 要分析气缸动作超时  
- **公式：** 气缸排查 = `命令 + 两端到位 + 超时报警`
- **计算：**
  - 记录 `ExtendCmd`
  - 记录 `HomeFB`
  - 记录 `WorkFB`
  - 记录 `TimeoutAlarm`
- **结论：** Trace 可以看出是命令没发出、到位信号慢，还是超时时间设置过短。

> [!tip] 记忆口诀
> **先选对变量，再谈看曲线。**

# 08｜常见错误与排查

> [!info] 核心结论
> **Trace 常见问题包括采样周期不合适、变量选错、触发条件不对、记录时间太短、曲线看不出重点。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>避坑提醒</strong><br>
Trace 不是变量越多越好，变量太多反而看不清重点，也可能增加 CPU 和通信负担。
</div>

## ① 分类/公式/规则

| 问题 | 常见原因 |
|---|---|
| **曲线太粗糙** | 采样周期太长 |
| **记录时间太短** | 点数太少或周期太短 |
| **抓不到故障** | 触发条件不合理 |
| **曲线无变化** | 变量选错或程序未写入 |
| **数据看不懂** | 单位、量程未标清 |
| **CPU 负载高** | 变量太多或周期太短 |

## ② 参考表

| 现象 | 排查方向 | 解决建议 |
|---|---|---|
| **波形断断续续** | 采样/通信压力 | 减少变量或放慢周期 |
| **触发不了** | 条件是否满足 | 改触发变量 |
| **记录不到故障前状态** | 无预触发 | 增加预触发比例 |
| **数值不合理** | 类型/单位 | 查变量类型和换算 |
| **趋势太平滑** | 采样太慢或滤波太强 | 缩短周期或查滤波 |
| **无法在线采集** | PLC连接/权限 | 查在线状态 |

## ③ 计算/选型示例

- **已知：** 一个信号脉冲宽度约 `20ms`，Trace 采样周期为 `100ms`
- **公式：** 采样周期必须小于关键变化时间
- **计算：**
  - 信号变化：`20ms`
  - 采样周期：`100ms`
  - 可能错过脉冲
- **结论：** 要捕捉 `20ms` 脉冲，采样周期需要明显小于 `20ms`，具体需按 CPU 性能校核。

> [!tip] 记忆口诀
> **变化快，采样短；变化慢，记录长。**

# 09｜一页速记总卡

> [!info] 核心结论
> **轨迹 Trace 的核心价值是把变量变化过程画成曲线，帮助你看清“故障发生前后到底发生了什么”。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
监控看现在，轨迹看过程；触发抓故障，曲线找原因。
</div>

## ① 分类/公式/规则

| 主轴 | 关键词 |
|---|---|
| **对象** | 轨迹 Trace |
| **用途** | 记录变量曲线 |
| **关键参数** | 变量、周期、点数、触发 |
| **适合分析** | PID、轴控、模拟量、通信 |
| **核心公式** | `记录总时长 = 采样周期 × 采样点数` |
| **风险点** | 周期不合适、变量选错 |

## ② 参考表

| 需求 | 推荐 Trace 变量 |
|---|---|
| **PID 调试** | SP、PV、Output、Error |
| **轴定位分析** | Target、Actual、Velocity、Error |
| **通信故障** | REQ、BUSY、DONE、ERROR、STATUS |
| **模拟量噪声** | Raw、Real、Filtered |
| **气缸超时** | Cmd、HomeFB、WorkFB、Timeout |
| **电机异常** | RunCmd、RunFB、Fault、Current |

## ③ 计算/选型示例

- **已知：** 要记录 `60s` 的温度曲线，采样周期为 `500ms`
- **公式：** 采样点数 = `记录时长 ÷ 采样周期`
- **计算：**
  - `60s ÷ 0.5s = 120`
  - 至少需要 `120` 个采样点
- **结论：** 若想记录 60 秒过程，采样点数至少设置约 `120` 点，建议留一定余量。

> [!tip] 记忆口诀
> **要看瞬间，周期要短；要看趋势，时间要长。**
