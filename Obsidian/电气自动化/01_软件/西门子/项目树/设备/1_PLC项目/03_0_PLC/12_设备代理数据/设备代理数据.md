# 01｜设备代理数据是什么

> [!info] 核心结论
> **设备代理数据是 TIA Portal 中用于把某个 PLC / CPU 的工程数据导出成代理文件，供其它项目或 HMI 工程引用的中间数据对象。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>图片识别</strong><br>
截图中的 <strong>设备代理数据</strong> 是 TIA Portal 项目树里的工程交换功能，常用于跨项目引用 PLC 数据。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **中文名称** | 设备代理数据 |
| **英文名称** | Device Proxy Data |
| **核心作用** | 导出 PLC 工程数据供其它项目使用 |
| **常见格式** | IPE 相关工程交换数据 |
| **常见用途** | HMI 项目引用 PLC、跨项目协同、多人分工 |
| **是否是程序块** | 不是程序块，不直接运行 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
设备代理数据就像 PLC 工程的“名片包”：把 PLC 的接口、变量、块信息导出来，给别的项目识别和连接。
</div>

## ② 参考表

| 对象 | 作用 | 类比 |
|---|---|---|
| **PLC 项目** | 原始控制器工程 | 数据源 |
| **设备代理数据** | PLC 数据快照 | 名片包 |
| **HMI 项目** | 使用 PLC 数据做画面 | 数据使用者 |
| **IPE 文件** | 跨项目交换文件 | 中转文件 |
| **设备代理** | 在其它项目中代表 PLC | PLC 的影子对象 |

## ③ 计算/选型示例

- **已知：** PLC 工程和 HMI 工程分别由两个人维护  
- **公式：** PLC 数据共享 = `PLC工程 → 导出设备代理数据 → HMI工程导入使用`
- **计算：**
  - PLC 工程师生成设备代理数据
  - HMI 工程师导入代理数据
  - HMI 可引用 PLC 变量和接口
- **结论：** 设备代理数据适合 PLC 与 HMI 分项目开发。

> [!tip] 记忆口诀
> **PLC 出代理，HMI 来引用；不用同项目，也能连数据。**

# 02｜它解决什么问题

> [!info] 核心结论
> **设备代理数据主要解决“PLC 工程和 HMI 工程不在同一个项目里，但 HMI 仍要访问 PLC 数据”的问题。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程理解</strong><br>
大型项目中，PLC 和 HMI 经常由不同工程师、不同项目文件维护，设备代理数据就是两边交换接口的桥。
</div>

## ① 分类/公式/规则

| 问题 | 设备代理数据的作用 |
|---|---|
| **PLC 与 HMI 分开开发** | 让 HMI 引用 PLC 接口 |
| **多个项目需要同一 PLC 数据** | 导出一次，多项目使用 |
| **不想复制完整 PLC 项目** | 只给代理数据 |
| **保护 PLC 原工程** | HMI 侧不用拿完整控制程序 |
| **变量接口同步** | 用代理数据更新 HMI 变量 |

## ② 参考表

| 场景 | 是否适合设备代理数据 |
|---|---|
| **PLC/HMI 同一个 TIA 项目** | 不一定需要 |
| **PLC/HMI 分项目开发** | **适合** |
| **HMI 只需要访问 PLC 变量** | **适合** |
| **需要完整修改 PLC 程序** | 不适合，只用原 PLC 项目 |
| **多人协同开发** | 适合 |
| **现场临时监控变量** | 不一定需要 |

## ③ 计算/选型示例

- **已知：** HMI 工程师只需要画面变量，不需要 PLC 源程序  
- **公式：** 只共享接口，不共享完整程序 → 设备代理数据  
- **计算：**
  - PLC 工程师导出代理数据
  - HMI 工程师导入
  - HMI 根据代理变量建立画面
- **结论：** 既能协同开发，又能避免完整 PLC 程序到处复制。

> [!tip] 记忆口诀
> **只要接口，不要源码；用设备代理最省事。**

# 03｜设备代理数据包含什么

> [!info] 核心结论
> **设备代理数据通常包含供其它项目引用的 PLC 接口信息，例如变量、程序块信息、通信相关数据等，具体内容取决于导出时选择项。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
导出时勾选什么，代理数据里才有什么。
</div>

## ① 分类/公式/规则

| 可能内容 | 作用 |
|---|---|
| **PLC 变量** | 给 HMI 或其它项目引用 |
| **程序块接口** | 让外部项目识别块和接口 |
| **DB 变量** | 供画面、通信、上位系统使用 |
| **数据类型** | 保持变量类型一致 |
| **设备信息** | 识别 CPU / PLC 对象 |
| **安全/通信相关信息** | 视项目配置而定 |

> [!warning]
> 实际可导出的内容与 TIA Portal 版本、CPU 类型、项目权限和导出选项有关，需按实际软件界面校核。

## ② 参考表

| 数据类型 | 是否常用于代理 |
|---|---|
| **PLC Tags** | 常用 |
| **DB 变量** | 常用 |
| **UDT 数据类型** | 常用 |
| **程序块接口** | 视需要 |
| **完整 PLC 程序逻辑** | 不一定完整 |
| **HMI 画面** | 不属于 PLC 代理数据 |

## ③ 计算/选型示例

- **已知：** HMI 需要访问 `DB_HMI.SpeedSet` 和 `DB_HMI.MotorStatus`
- **公式：** HMI 需要访问 DB 变量 → 导出时包含相关 DB 数据
- **计算：**
  - 检查 DB 是否允许 HMI 访问
  - 生成设备代理数据
  - HMI 项目导入后选择对应变量
- **结论：** 代理数据必须包含 HMI 需要的变量，否则画面无法正确绑定。

> [!tip] 记忆口诀
> **HMI 要什么，代理就导什么。**

# 04｜设备代理数据与 HMI 的关系

> [!info] 核心结论
> **HMI 可以通过设备代理引用 PLC 数据，即使 HMI 工程不在 PLC 原始项目中，也能建立变量连接。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>关系链路</strong><br>
PLC 工程 → 设备代理数据 → HMI 工程 → HMI 变量 → 画面对象
</div>

## ① 分类/公式/规则

| 对象 | 作用 |
|---|---|
| **PLC 工程** | 提供真实变量和数据结构 |
| **设备代理数据** | 导出 PLC 接口信息 |
| **HMI 工程** | 导入代理数据 |
| **HMI 变量** | 绑定 PLC 代理变量 |
| **画面对象** | 显示或写入变量 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>核心规则</strong><br>
代理数据更新后，HMI 项目也要同步更新，否则变量结构可能还是旧版本。
</div>

## ② 参考表

| HMI 需求 | 设备代理数据作用 |
|---|---|
| **显示运行状态** | 提供状态变量 |
| **写入速度设定** | 提供可写参数变量 |
| **显示报警代码** | 提供报警变量 |
| **读取生产计数** | 提供统计变量 |
| **调用数据结构** | 提供 UDT/DB 结构 |
| **连接 PLC 通信** | 提供设备接口参考 |

## ③ 计算/选型示例

- **已知：** PLC 工程新增了 `DB_HMI.AlarmCode`
- **公式：** PLC 新增变量 → 重新导出代理数据 → HMI 重新导入/更新
- **计算：**
  - PLC 新增变量
  - 生成新的设备代理数据
  - HMI 工程更新代理
  - 画面绑定 `AlarmCode`
- **结论：** PLC 接口变化后，设备代理数据必须更新。

> [!tip] 记忆口诀
> **PLC 变量一改，代理数据要更新。**

# 05｜典型使用流程

> [!info] 核心结论
> **设备代理数据的典型流程是：在源 PLC 项目中创建代理数据 → 选择导出内容 → 导出文件 → 在目标项目中导入使用。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>标准流程</strong><br>
源项目导出 → 目标项目导入 → 建立连接 → 绑定变量 → 联调测试
</div>

## ① 分类/公式/规则

| 步骤 | 动作 | 重点 |
|---|---|---|
| **1** | 打开源 PLC 项目 | 找到对应 CPU |
| **2** | 新建设备代理数据 | 在“设备代理数据”下创建 |
| **3** | 选择导出内容 | 变量、块、数据类型等 |
| **4** | 导出代理文件 | 保存到工程目录 |
| **5** | 目标项目导入 | HMI 或其它工程使用 |
| **6** | 建立连接 | 选择代理设备 |
| **7** | 测试变量 | 在线确认读写正确 |

## ② 参考表

| 操作点 | 注意事项 |
|---|---|
| **源项目版本** | 必须是当前 PLC 接口版本 |
| **变量权限** | HMI/OPC UA 可访问性要校核 |
| **DB 结构** | 改结构后要重新导出 |
| **代理文件命名** | 带日期和版本 |
| **目标项目更新** | 旧代理要同步替换 |
| **联调测试** | 读写都要验证 |

## ③ 计算/选型示例

- **已知：** PLC 接口版本从 V1 改为 V2  
- **公式：** 接口版本变化 = 代理数据版本变化  
- **计算：**
  - 旧代理：`PLCProxy_V1`
  - 新代理：`PLCProxy_V2`
  - HMI 工程必须切换到 V2
- **结论：** 代理数据也要做版本管理，避免 HMI 使用旧接口。

> [!tip] 记忆口诀
> **源项目变了，代理重导；目标项目变了，变量重绑。**

# 06｜设备代理数据与 OPC UA 的区别

> [!info] 核心结论
> **设备代理数据是工程阶段的数据交换对象；OPC UA 是运行阶段的通信协议，两者不是一回事。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点区别</strong><br>
设备代理数据用于工程组态共享，OPC UA 用于运行时数据通信。
</div>

## ① 分类/公式/规则

| 对比项 | 设备代理数据 | OPC UA 通信 |
|---|---|---|
| **阶段** | 工程组态阶段 | 运行通信阶段 |
| **作用** | 共享 PLC 接口信息 | 在线读写数据 |
| **对象** | 工程文件 / 代理对象 | Server / Client |
| **是否实时通信** | 否 | 是 |
| **典型用途** | HMI 分项目开发 | MES / SCADA 采集 |
| **核心风险** | 版本不同步 | 证书、权限、节点错误 |

## ② 参考表

| 需求 | 推荐 |
|---|---|
| **HMI 项目引用 PLC 变量** | 设备代理数据 |
| **MES 在线读取 PLC 数据** | OPC UA |
| **跨项目共享 CPU 接口** | 设备代理数据 |
| **运行时订阅状态变化** | OPC UA |
| **保护 PLC 原项目不外发** | 设备代理数据 |
| **跨平台数据接口** | OPC UA |

## ③ 计算/选型示例

- **已知：** HMI 工程师需要离线做画面，不需要现场实时连接 PLC  
- **公式：** 离线工程引用 → 设备代理数据  
- **计算：**
  - PLC 工程导出代理数据
  - HMI 工程导入
  - HMI 离线完成画面变量绑定
- **结论：** 这种不是 OPC UA 通信需求，而是设备代理数据的典型用途。

> [!tip] 记忆口诀
> **工程共享用代理，运行通信用 OPC UA。**

# 07｜版本管理与命名

> [!info] 核心结论
> **设备代理数据必须做好版本管理，否则 HMI 或目标项目很容易引用到旧变量、旧 DB 结构。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>命名口诀</strong><br>
设备名 + 日期 + 接口版本 + 导出说明。
</div>

## ① 分类/公式/规则

| 命名字段 | 示例 |
|---|---|
| **设备名** | `Line1_CPU1511` |
| **日期** | `2026-04-29` |
| **接口版本** | `IF_V2_0` |
| **导出内容** | `HMI_Tags` |
| **工程版本** | `PLC_V3_1` |

## ② 参考表

| 文件名示例 | 含义 |
|---|---|
| **Line1_CPU1511_2026-04-29_IF_V1_0** | 1号线 CPU 接口 V1.0 |
| **PackingPLC_HMIProxy_V2_1** | 包装机 HMI 代理 V2.1 |
| **RobotCell_CPUProxy_BeforeModify** | 修改前代理数据 |
| **PLCProxy_AfterDBChange_V3** | DB 改动后的代理数据 |

## ③ 计算/选型示例

- **已知：** 今天 PLC 工程新增了 5 个 HMI 变量  
- **公式：** 接口变更 = 代理版本 +1  
- **计算：**
  - 旧版本：`IF_V1_2`
  - 新增变量后：`IF_V1_3`
  - 导出新代理并通知 HMI 工程师
- **结论：** 每次 PLC 对外接口变化，都应该更新代理版本。

> [!tip] 记忆口诀
> **接口一变，代理升版；不升版本，后期混乱。**

# 08｜常见错误与排查

> [!info] 核心结论
> **设备代理数据常见问题包括代理未更新、变量丢失、DB 结构变化、目标项目引用旧版本、权限或可访问性未开启。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>避坑提醒</strong><br>
HMI 变量找不到，不一定是 HMI 问题，可能是 PLC 没把新变量导进代理数据。
</div>

## ① 分类/公式/规则

| 问题 | 常见原因 |
|---|---|
| **HMI 找不到变量** | 代理数据未更新 |
| **变量类型不一致** | DB 或 UDT 结构改了 |
| **画面读写异常** | 变量路径变化 |
| **目标项目报错** | 代理文件版本不匹配 |
| **导入失败** | TIA 版本或工程环境不兼容 |
| **变量不可访问** | HMI/OPC UA 可访问性未配置 |

## ② 参考表

| 现象 | 排查方向 | 解决建议 |
|---|---|---|
| **变量列表缺少新变量** | 代理是否重新导出 | 重新生成代理数据 |
| **HMI 编译报错** | 变量路径是否变更 | 重新绑定变量 |
| **数据类型红色** | UDT 是否同步 | 更新 UDT/代理 |
| **读写值异常** | DB 结构是否改动 | 检查变量映射 |
| **导入代理失败** | TIA 版本 | 用兼容版本 |
| **上线后通信失败** | 连接/证书/权限 | 按通信链路排查 |

## ③ 计算/选型示例

- **已知：** PLC 中 `SpeedSet` 从 `Int` 改成 `Real`，HMI 仍按旧代理使用  
- **公式：** 数据类型变化 = 代理数据必须更新  
- **计算：**
  - 旧代理：`SpeedSet : Int`
  - 新 PLC：`SpeedSet : Real`
  - HMI 类型不匹配
- **结论：** 会导致编译错误或运行数据异常，必须重新导出并更新代理数据。

> [!tip] 记忆口诀
> **找不到变量查代理，类型不对查版本。**

# 09｜一页速记总卡

> [!info] 核心结论
> **设备代理数据的核心价值是把 PLC 工程接口独立导出，让其它项目能引用 PLC 数据而不必拿完整 PLC 工程。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
PLC 导代理，HMI 用代理；接口变了重导出，变量变了重同步。
</div>

## ① 分类/公式/规则

| 主轴 | 关键词 |
|---|---|
| **对象** | 设备代理数据 |
| **用途** | 跨项目工程交换 |
| **来源** | PLC / CPU 工程 |
| **目标** | HMI 或其它项目 |
| **核心文件** | IPE / 代理数据 |
| **风险点** | 版本不同步、变量缺失、类型不匹配 |

## ② 参考表

| 需求 | 推荐做法 |
|---|---|
| **PLC/HMI 分项目开发** | 使用设备代理数据 |
| **HMI 引用 PLC 变量** | 导入代理 |
| **PLC 接口变更** | 重新导出代理 |
| **HMI 变量报错** | 检查代理版本 |
| **运行时数据通信** | 用通信连接 / OPC UA |
| **完整 PLC 修改** | 使用原始 PLC 工程 |

## ③ 计算/选型示例

- **已知：** 一个项目由 PLC 工程师和 HMI 工程师分开开发  
- **公式：** 分工协同 = `PLC项目 + 设备代理数据 + HMI项目`
- **计算：**
  - PLC 工程师维护 CPU、程序、DB
  - 定期导出设备代理数据
  - HMI 工程师导入代理并绑定变量
  - 联调时检查变量一致性
- **结论：** 设备代理数据是 TIA Portal 跨项目协同开发的重要工具。

> [!tip] 记忆口诀
> **代理不是程序，代理是接口；接口同步好，项目协同稳。**
