# 01｜本地模块是什么

> [!info] 核心结论
> **本地模块是直接安装在 PLC CPU 本体或本地机架上的扩展模块，用于增加 DI、DQ、AI、AQ、通信、电源等硬件能力。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>图片识别</strong><br>
截图中的 <strong>本地模块</strong> 是 TIA Portal 硬件组态中的模块分类入口，通常用于管理直接挂在 CPU 旁边的扩展模块。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **中文名称** | 本地模块 |
| **英文理解** | Local Modules |
| **安装位置** | CPU 本体旁边 / 本地机架 |
| **核心作用** | 扩展 PLC 的输入、输出、模拟量、通信能力 |
| **典型模块** | DI、DQ、AI、AQ、通信模块、电源模块 |
| **区别对象** | 分布式 I/O、远程站、现场总线设备 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
本地模块就是直接插在 PLC 身边的“扩展硬件”，离 CPU 最近，地址也直接由 CPU 管理。
</div>

## ② 参考表

| 模块类型 | 中文含义 | 作用 |
|---|---|---|
| **DI** | 数字量输入 | 接按钮、传感器、限位 |
| **DQ / DO** | 数字量输出 | 控制继电器、电磁阀、指示灯 |
| **AI** | 模拟量输入 | 采集压力、温度、液位 |
| **AQ / AO** | 模拟量输出 | 控制阀门、变频器给定 |
| **CM / CP** | 通信模块 | 扩展串口、以太网、现场总线 |
| **PM / PS** | 电源模块 | 给模块或系统供电 |

## ③ 计算/选型示例

- **已知：** 设备有 16 个按钮输入、8 个电磁阀输出、2 路压力传感器  
- **公式：** I/O 点数需求 → 选择本地扩展模块  
- **计算：**
  - DI：至少 `16` 点
  - DQ：至少 `8` 点
  - AI：至少 `2` 路
- **结论：** 可选择 16DI、8DQ、2AI 或更大容量模块，最终需按 CPU 支持数量和模块手册校核。

> [!tip] 记忆口诀
> **按钮传感进 DI，电磁阀灯出 DQ，压力温度走 AI。**

# 02｜本地模块与分布式 I/O 区别

> [!info] 核心结论
> **本地模块直接挂在 CPU 旁边，分布式 I/O 通过 PROFINET、PROFIBUS 等网络远程连接。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程理解</strong><br>
距离 CPU 近、点数不多，优先本地模块；设备分散、线缆很长，优先分布式 I/O。
</div>

## ① 分类/公式/规则

| 对比项 | 本地模块 | 分布式 I/O |
|---|---|---|
| **安装位置** | CPU 旁边 | 现场设备附近 |
| **连接方式** | 背板总线 / 本机扩展 | 网络通信 |
| **典型对象** | CPU 扩展模块 | ET 200SP、ET 200MP、远程 IO |
| **布线距离** | 适合短距离 | 适合远距离 |
| **维护特点** | 集中维护 | 分散维护 |
| **网络依赖** | 较少 | 依赖 PROFINET 等网络 |

## ② 参考表

| 场景 | 推荐方式 | 原因 |
|---|---|---|
| **小型设备，I/O 集中** | 本地模块 | 接线简单 |
| **控制柜内信号** | 本地模块 | 近距离方便 |
| **现场点位分散** | 分布式 I/O | 减少长线 |
| **产线多个工位** | 分布式 I/O | 网络扩展方便 |
| **高速本地信号** | 本地模块 | 响应直接 |
| **远程阀岛/传感器** | 分布式 I/O | 就近接线 |

## ③ 计算/选型示例

- **已知：** 传感器距离 PLC 控制柜 `80m`，数量很多  
- **公式：** 信号远且多 → 分布式 I/O 更合适  
- **计算：**
  - 若全部拉回控制柜，线缆多、干扰大
  - 使用远程 I/O，可就近接传感器
  - 只需通过网络回 PLC
- **结论：** 远距离大量信号不建议全部做本地模块，需考虑分布式 I/O。

> [!tip] 记忆口诀
> **点近用本地，点远用远程；线少好维护，系统更稳定。**

# 03｜常见本地模块分类

> [!info] 核心结论
> **本地模块主要分为数字量、模拟量、通信、电源和特殊功能模块。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>分类口诀</strong><br>
开关量看 DI/DQ，连续量看 AI/AQ，协议扩展看通信模块。
</div>

## ① 分类/公式/规则

| 类别 | 模块 | 典型信号 |
|---|---|---|
| **数字量输入** | DI | 按钮、光电、接近开关 |
| **数字量输出** | DQ / DO | 继电器、阀、灯 |
| **模拟量输入** | AI | 0-10V、4-20mA、热电偶、RTD |
| **模拟量输出** | AQ / AO | 0-10V、4-20mA |
| **通信模块** | CM / CP | RS232、RS485、以太网 |
| **特殊模块** | 高速计数、定位、称重 | 编码器、脉冲、称重传感器 |

## ② 参考表

| 需求 | 选择模块 |
|---|---|
| **接按钮** | DI |
| **接接近开关** | DI |
| **控制电磁阀** | DQ |
| **控制继电器线圈** | DQ |
| **采集压力变送器** | AI |
| **给变频器频率信号** | AQ |
| **接扫码枪串口** | CM RS232 / RS485 |
| **接编码器高速脉冲** | 高速计数模块或 CPU 高速输入 |

## ③ 计算/选型示例

- **已知：** 变频器需要 PLC 给 `0-10V` 速度给定  
- **公式：** 连续电压输出 → AQ 模块  
- **计算：**
  - 信号类型：`0-10V`
  - 方向：PLC 输出到变频器
  - 所需模块：AQ / AO
- **结论：** 该场景应选择支持 `0-10V` 输出的模拟量输出模块。

> [!tip] 记忆口诀
> **输入看现场信号，输出看控制对象。**

# 04｜地址分配

> [!info] 核心结论
> **本地模块插入硬件组态后，TIA Portal 会为模块分配 I/Q 地址，程序通过地址或 PLC 变量访问这些信号。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点规则</strong><br>
硬件地址决定程序读写哪里，地址错了，变量名再好也会读错信号。
</div>

## ① 分类/公式/规则

| 地址类型 | 含义 | 示例 |
|---|---|---|
| **I** | 输入区 | `I0.0`、`IW64` |
| **Q** | 输出区 | `Q0.0`、`QW80` |
| **Bit 地址** | 单点开关量 | `I0.0` |
| **Byte 地址** | 字节 | `IB0` |
| **Word 地址** | 模拟量或状态字 | `IW64` |
| **DWord 地址** | 双字数据 | `ID100` |

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点公式</strong><br>
1 Byte = 8 bit；1 Word = 2 Byte；1 DWord = 4 Byte
</div>

## ② 参考表

| 模块 | 常见地址形式 | 说明 |
|---|---|---|
| **8DI** | `I0.0 ~ I0.7` | 8 个输入点 |
| **16DI** | `I0.0 ~ I1.7` | 16 个输入点 |
| **8DQ** | `Q0.0 ~ Q0.7` | 8 个输出点 |
| **AI 4路** | `IW64、IW66、IW68、IW70` | 每路通常占 1 Word |
| **AQ 2路** | `QW80、QW82` | 每路通常占 1 Word |

> [!warning]
> 上表为常见理解示例，实际地址由硬件组态、模块类型和起始地址决定，必须以 TIA Portal 实际分配为准。

## ③ 计算/选型示例

- **已知：** 一个 16 点 DI 模块起始地址为 `I2.0`  
- **公式：** 16 点 = 2 Byte = 16 bit  
- **计算：**
  - 第 1 点：`I2.0`
  - 第 8 点：`I2.7`
  - 第 9 点：`I3.0`
  - 第 16 点：`I3.7`
- **结论：** 16 点数字量模块会占用连续 2 个输入字节。

> [!tip] 记忆口诀
> **8点占一字节，16点占两字节。**

# 05｜本地模块选型规则

> [!info] 核心结论
> **本地模块选型要同时看信号类型、点数、电压等级、电流能力、模块数量限制、供电能力和安全裕量。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>选型口诀</strong><br>
先分输入输出，再数点位；先看电压电流，再留备用。
</div>

## ① 分类/公式/规则

| 选型项 | 说明 |
|---|---|
| **信号方向** | 输入还是输出 |
| **信号类型** | 数字量、模拟量、通信 |
| **电压等级** | 24VDC、230VAC 等 |
| **点数数量** | 当前点数 + 备用点 |
| **输出电流** | 是否能直接带负载 |
| **隔离需求** | 是否需要通道隔离 |
| **模块数量限制** | CPU 支持几个本地模块 |
| **供电能力** | 电源模块是否够用 |

## ② 参考表

| 选型问题 | 重点检查 |
|---|---|
| **DI 选型** | 输入电压、漏型/源型、滤波时间 |
| **DQ 选型** | 晶体管/继电器、输出电流、频率 |
| **AI 选型** | 电压/电流/热电偶/RTD、分辨率 |
| **AQ 选型** | 0-10V / 4-20mA、负载能力 |
| **通信模块** | RS232 / RS485 / Ethernet、协议 |
| **特殊模块** | 高速计数、PTO、称重、定位 |

## ③ 计算/选型示例

- **已知：** 当前需要 22 个 DI 点，项目希望预留 20%  
- **公式：** 选型点数 = `当前点数 × 1.2`
- **计算：**
  - `22 × 1.2 = 26.4`
  - 向上取整至少 `27` 点
  - 可选择 32DI 或多个 DI 模块组合
- **结论：** DI 点数建议选到 32 点左右，预留后期扩展空间。

> [!tip] 记忆口诀
> **点数别卡满，扩展要留量。**

# 06｜本地模块供电与接线

> [!info] 核心结论
> **本地模块不仅要组态正确，还要供电正确、公共端正确、输入输出类型匹配，否则程序正常也可能信号不动作。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>避坑提醒</strong><br>
PLC 程序没问题，不代表模块接线没问题；本地模块故障很多来自电源、公共端和负载接线。
</div>

## ① 分类/公式/规则

| 接线项 | 说明 |
|---|---|
| **L+ / M** | 24VDC 电源正负 |
| **公共端 COM** | 输入或输出公共端 |
| **传感器供电** | 传感器是否有 24V |
| **负载供电** | 输出模块是否给负载供电 |
| **屏蔽接地** | 模拟量和通信线需关注 |
| **隔离分组** | 不同电源组不要混接 |
| **保险保护** | 输出回路应有保护 |

## ② 参考表

| 现象 | 可能接线原因 |
|---|---|
| **DI 无信号** | 传感器无电、公共端错、NPN/PNP 不匹配 |
| **DQ 不动作** | 输出电源没接、负载无电、输出类型不匹配 |
| **AI 数值跳动** | 屏蔽不好、接地不良、量程错误 |
| **AQ 无输出** | 负载不匹配、通道未使能 |
| **模块报错** | 电源不足、模块型号不匹配 |
| **通信失败** | A/B 线反、终端电阻、波特率错误 |

## ③ 计算/选型示例

- **已知：** 一个 DQ 模块每点最大输出 `0.5A`，电磁阀线圈电流 `0.7A`  
- **公式：** 负载电流 ≤ 输出允许电流  
- **计算：**
  - 输出能力：`0.5A`
  - 负载需求：`0.7A`
  - `0.7A > 0.5A`
- **结论：** 不能直接驱动该电磁阀，应加中间继电器或选择更大电流输出模块，需按模块手册校核。

> [!tip] 记忆口诀
> **模块能不能带，先看电流够不够。**

# 07｜硬件组态与下载

> [!info] 核心结论
> **本地模块必须在 TIA Portal 硬件组态中正确添加、排列、设置地址和参数，并下载到 CPU 后才能正常识别。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>标准流程</strong><br>
选模块 → 插到正确位置 → 设置参数 → 分配地址 → 编译 → 下载 → 在线诊断
</div>

## ① 分类/公式/规则

| 步骤 | 内容 | 重点 |
|---|---|---|
| **1** | 添加模块 | 型号必须一致 |
| **2** | 放入机架位置 | 顺序与现场一致 |
| **3** | 设置参数 | 滤波、量程、诊断等 |
| **4** | 分配地址 | I/Q 地址不要冲突 |
| **5** | 编译硬件 | 检查错误 |
| **6** | 下载到 CPU | 更新硬件组态 |
| **7** | 在线诊断 | 查看模块状态 |

## ② 参考表

| 设置项 | 常见影响 |
|---|---|
| **DI 滤波时间** | 影响输入响应和抗干扰 |
| **DQ 输出类型** | 晶体管/继电器动作特性 |
| **AI 量程** | 决定原始值解释 |
| **AI 滤波** | 影响稳定性和响应速度 |
| **AQ 输出范围** | 决定电压/电流给定 |
| **诊断使能** | 是否报告断线/短路等 |

## ③ 计算/选型示例

- **已知：** AI 模块接入 `4-20mA` 压力变送器，但组态设成 `0-10V`  
- **公式：** 模块量程必须与现场信号一致  
- **计算：**
  - 现场信号：电流型 `4-20mA`
  - 组态信号：电压型 `0-10V`
  - 类型不匹配，数值会异常
- **结论：** 模拟量模块一定要按现场信号类型设置量程。

> [!tip] 记忆口诀
> **硬件型号要一致，现场信号和组态要一致。**

# 08｜常见故障排查

> [!info] 核心结论
> **本地模块故障排查要按：模块状态 → 组态型号 → 供电 → 地址 → 接线 → 程序变量的顺序检查。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>排查顺序</strong><br>
先看模块在线，再看型号组态；先查电源公共端，再查程序变量。
</div>

## ① 分类/公式/规则

| 问题 | 常见原因 |
|---|---|
| **模块灰色/离线** | 型号不对、插槽不对、未下载组态 |
| **DI 不亮** | 接线、电源、公共端、传感器类型 |
| **DQ 不输出** | 负载电源、输出损坏、程序未写 TRUE |
| **AI 数值异常** | 量程错、接线错、屏蔽差 |
| **AQ 无信号** | 通道未使能、负载不匹配 |
| **地址读错** | I/Q 地址或变量绑定错误 |

## ② 参考表

| 现象 | 排查方向 | 解决建议 |
|---|---|---|
| **模块诊断报警** | 在线诊断 | 查看具体错误 |
| **输入点不变化** | 现场电压和地址 | 用万用表 + 监控表 |
| **输出点亮但负载不动** | 负载电源 | 查电磁阀/继电器 |
| **模拟量跳动** | 屏蔽接地 | 检查线缆和滤波 |
| **变量值不对** | 地址绑定 | 核对 PLC 变量表 |
| **下载后模块报错** | 组态不一致 | 核对模块订货号 |

## ③ 计算/选型示例

- **已知：** 程序监控 `Start_PB` 无变化，但按钮实际有 24V 信号  
- **公式：** 现场有信号但程序无变化 → 查地址和公共端  
- **计算：**
  - 按钮端有 24V
  - 模块对应输入灯是否亮
  - 变量 `Start_PB` 是否绑定正确地址
- **结论：** 若模块灯亮但变量不变，多半是变量地址绑定错误；若模块灯不亮，多半是接线或公共端问题。

> [!tip] 记忆口诀
> **灯亮变量不变查地址，灯不亮先查接线。**

# 09｜一页速记总卡

> [!info] 核心结论
> **本地模块的核心价值是直接扩展 PLC 本体 I/O 和通信能力，是小型设备和柜内信号最常用的硬件扩展方式。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
本地模块贴 CPU，输入输出直接接；型号地址要一致，供电接线要校核。
</div>

## ① 分类/公式/规则

| 主轴 | 关键词 |
|---|---|
| **对象** | 本地模块 |
| **位置** | CPU 本体旁 / 本地机架 |
| **类型** | DI、DQ、AI、AQ、CM、PM |
| **核心配置** | 型号、插槽、地址、参数 |
| **核心风险** | 型号错、地址错、供电错、量程错 |
| **适合场景** | 小型设备、控制柜内信号、短距离 I/O |

## ② 参考表

| 需求 | 推荐模块 |
|---|---|
| **按钮/限位输入** | DI |
| **电磁阀/指示灯输出** | DQ |
| **压力/液位采集** | AI |
| **阀门/变频器给定** | AQ |
| **串口仪表通信** | CM RS232 / RS485 |
| **高速脉冲/编码器** | 高速计数或定位模块 |
| **模块供电扩展** | 电源模块 |

## ③ 计算/选型示例

- **已知：** 一个小型设备需要 12DI、10DQ、2AI、1AQ  
- **公式：** 模块选型 = 当前需求 + 备用余量  
- **计算：**
  - DI 可选 16DI
  - DQ 可选 16DQ
  - AI 可选 4AI
  - AQ 可选 2AQ
- **结论：** 选模块时不要刚好卡满点数，应留一定备用点，最终按 CPU 支持模块数量和厂家手册校核。

> [!tip] 记忆口诀
> **本地模块三步选：先点数，后类型，再校核供电和地址。**
