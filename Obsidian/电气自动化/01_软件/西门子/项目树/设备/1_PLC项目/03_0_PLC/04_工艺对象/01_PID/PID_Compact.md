# 01｜PID_Compact 是什么

> [!info] 核心结论
> **PID_Compact 是 TIA Portal 中的紧凑型 PID 控制功能块，用于温度、压力、流量、液位、速度等连续量闭环控制。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>图片识别</strong><br>
截图中的 <strong>PID_Compact V2.3</strong> 是西门子 PLC 常用的 PID 闭环控制块。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **指令名称** | `PID_Compact` |
| **版本** | `V2.3` |
| **中文含义** | 紧凑型 PID 控制器 |
| **控制类型** | 连续量闭环控制 |
| **常见对象** | 温度、压力、流量、液位、转速 |
| **典型输出** | 模拟量、PWM、百分比控制量 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
PID_Compact 就是让 PLC 根据“设定值”和“实际值”的偏差，自动算出一个控制输出。
</div>

## ② 参考表

| 名称 | 含义 | 工程理解 |
|---|---|---|
| **Setpoint** | 设定值 | 想要达到的目标 |
| **Input / PV** | 过程值 | 传感器测到的实际值 |
| **Error** | 偏差 | 目标值 - 实际值 |
| **Output / LMN** | 输出值 | 给执行器的控制量 |
| **Kp** | 比例增益 | 反应快慢 |
| **Ti** | 积分时间 | 消除静差 |
| **Td** | 微分时间 | 抑制变化趋势 |

## ③ 计算/选型示例

- **已知：** 需要把水箱液位稳定在 `500mm`
- **公式：** 闭环控制 = `设定值 - 实际值 → PID计算 → 输出调节`
- **计算：**
  - 设定值：`500mm`
  - 实际值：液位传感器反馈
  - 偏差大：PID 输出增大
  - 偏差小：PID 输出减小
- **结论：** 液位、温度、压力这类连续调节场景适合使用 `PID_Compact`。

> [!tip] 记忆口诀
> **有目标，有反馈，有调节，就能想到 PID。**

# 02｜PID 闭环原理

> [!info] 核心结论
> **PID 的本质是根据偏差大小、偏差累计和偏差变化趋势，计算控制输出。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点公式</strong><br>
输出 = P比例 + I积分 + D微分
</div>

## ① 分类/公式/规则

$$
u(t)=K_p\left[e(t)+\frac{1}{T_i}\int e(t)dt+T_d\frac{de(t)}{dt}\right]
$$

| 项 | 中文含义 | 作用 |
|---|---|---|
| **P** | 比例 | 偏差越大，输出越大 |
| **I** | 积分 | 消除长期静差 |
| **D** | 微分 | 预测变化，抑制超调 |

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>三项口诀</strong><br>
P 管现在，I 管过去，D 管趋势。
</div>

## ② 参考表

| 参数 | 调大后效果 | 调太大的风险 |
|---|---|---|
| **Kp** | 响应更快 | 振荡、超调 |
| **Ti** | 积分作用变弱 | 静差消除慢 |
| **Ti 变小** | 积分作用变强 | 超调、振荡 |
| **Td** | 抑制变化 | 对噪声敏感 |
| **采样周期** | 影响稳定性 | 太慢控制差 |

## ③ 计算/选型示例

- **已知：** 温度设定 `80℃`，实际温度 `70℃`
- **公式：** 偏差 = `设定值 - 实际值`
- **计算：**
  - 偏差 = `80 - 70 = 10℃`
  - PID 根据 `10℃` 偏差计算加热输出
- **结论：** 偏差越大，控制器通常会给出更大的调节输出。

> [!tip] 记忆口诀
> **偏差大就加力，偏差小就收力，快到目标要稳住。**

# 03｜PID_Compact 信号结构

> [!info] 核心结论
> **PID_Compact 的核心信号就是：设定值、实际值、模式、手动值、输出值、状态和错误信息。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>信号链路</strong><br>
Setpoint → PID_Compact → Output → 执行器 → 过程对象 → 传感器反馈 → Input
</div>

## ① 分类/公式/规则

| 信号类别 | 典型名称 | 作用 |
|---|---|---|
| **设定输入** | `Setpoint` | 目标值 |
| **反馈输入** | `Input` / `Input_PER` | 实际值 |
| **控制输出** | `Output` / `Output_PER` / `Output_PWM` | 控制执行器 |
| **模式控制** | 自动 / 手动 / 调试 | 控制运行方式 |
| **状态诊断** | `State`、`Error`、状态码 | 判断运行状态 |

> [!warning]
> 不同 CPU、TIA Portal 版本、PID_Compact 版本中接口名称可能略有差异，实际项目需按块接口和厂家帮助文档校核。

## ② 参考表

| 参数 | 作用 | 注意 |
|---|---|---|
| **Setpoint** | 目标设定值 | 单位要和实际值一致 |
| **Input** | 工程量实际值 | 如 ℃、bar、mm |
| **Input_PER** | 外设输入值 | 常用于模拟量原始值 |
| **Output** | 工程量输出 | 常为百分比 |
| **Output_PER** | 外设输出值 | 可接模拟量输出 |
| **ManualValue** | 手动输出值 | 调试时常用 |
| **Error** | 错误标志 | 需结合状态码 |

## ③ 计算/选型示例

- **已知：** 温度控制输出需要控制加热器功率百分比
- **公式：** 输出百分比 = `0% ~ 100%`
- **计算：**
  - 输出 `0%`：不加热
  - 输出 `50%`：半功率
  - 输出 `100%`：满功率
- **结论：** 加热控制常把 PID 输出理解为功率百分比。

> [!tip] 记忆口诀
> **设定值给目标，实际值给反馈，输出值管执行。**

# 04｜输入量程与工程量

> [!info] 核心结论
> **PID 控制前必须先把传感器原始值换算成工程量，否则 PID 的设定值和实际值单位会不一致。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点公式</strong><br>
工程量 = 原始值 ÷ 满量程原始值 × 工程量范围
</div>

## ① 分类/公式/规则

| 信号 | 示例 | 说明 |
|---|---|---|
| **模拟量原始值** | `0~27648` | PLC 模拟量常见范围 |
| **工程量** | `0~100℃` | 实际控制单位 |
| **设定值** | `80℃` | 与工程量一致 |
| **反馈值** | `实际温度℃` | 与设定值同单位 |

## ② 参考表

| 输入类型 | 原始值 | 工程量 |
|---|---:|---:|
| **0-10V** | `0~27648` | 按量程换算 |
| **4-20mA** | 通常需按模块规则换算 | 按量程换算 |
| **热电偶/RTD** | 可能直接为温度 | 按模块设置 |
| **压力传感器** | 原始量 | bar / MPa |
| **液位传感器** | 原始量 | mm / % |

## ③ 计算/选型示例

- **已知：**
  - 模拟量原始值：`13824`
  - 满量程原始值：`27648`
  - 温度范围：`0~100℃`
- **公式：** 温度 = `原始值 ÷ 27648 × 100`
- **计算：**
  - `13824 ÷ 27648 × 100 = 50℃`
- **结论：** 当前反馈温度为 `50℃`，可作为 PID 的实际值输入。

> [!tip] 记忆口诀
> **PID 之前先换算，单位不对全白干。**

# 05｜输出控制方式

> [!info] 核心结论
> **PID_Compact 的输出可以用于模拟量调节、PWM 控制、阀门开度、变频器频率给定等场景。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>输出理解</strong><br>
PID 算出来的不是“开关命令”，而是一个连续调节量。
</div>

## ① 分类/公式/规则

| 输出方式 | 适合对象 | 说明 |
|---|---|---|
| **模拟量输出** | 调节阀、变频器 | `0-10V` / `4-20mA` |
| **PWM 输出** | 加热器、固态继电器 | 按占空比控制 |
| **百分比输出** | 上位逻辑处理 | `0~100%` |
| **频率给定** | 变频器 | 输出换算成 Hz |
| **阀门开度** | 电动调节阀 | 输出换算成开度 |

## ② 参考表

| 控制对象 | 常见输出 | 注意 |
|---|---|---|
| **加热器** | PWM / SSR | 周期需合理 |
| **变频器** | 模拟量 / 通信给定 | 方向和范围 |
| **调节阀** | 4-20mA | 阀门正反作用 |
| **风机** | 0-10V / 通信 | 最小频率限制 |
| **水泵** | 频率给定 | 防止低频失速 |

## ③ 计算/选型示例

- **已知：** PID 输出为 `60%`，变频器最大频率 `50Hz`
- **公式：** 频率给定 = `输出百分比 × 最大频率`
- **计算：**
  - `60% × 50Hz = 30Hz`
- **结论：** PID 输出 `60%` 可换算为变频器频率给定 `30Hz`。

> [!tip] 记忆口诀
> **PID 出百分比，后面再换电压、电流或频率。**

# 06｜手动与自动模式

> [!info] 核心结论
> **PID_Compact 通常支持手动和自动模式，调试时先用手动验证执行器方向和输出范围，再切自动闭环。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>调试顺序</strong><br>
先手动，后自动；先小输出，后大输出；先看方向，再调参数。
</div>

## ① 分类/公式/规则

| 模式 | 含义 | 适合场景 |
|---|---|---|
| **手动模式** | 人为给输出 | 调试执行器 |
| **自动模式** | PID 自动调节 | 正常运行 |
| **停止/禁用** | 控制器不输出 | 维护或停机 |
| **调试/整定** | 参数优化 | 调试阶段 |

## ② 参考表

| 检查项 | 手动模式怎么用 |
|---|---|
| **输出方向** | 给小输出看过程值是否朝目标变化 |
| **输出范围** | 验证 `0~100%` 是否合理 |
| **执行器响应** | 看阀门/变频器/加热器是否动作 |
| **安全边界** | 防止输出过大造成危险 |
| **切自动条件** | 反馈正常、方向正确、参数合理 |

## ③ 计算/选型示例

- **已知：** 温度低于设定值，但 PID 输出越大温度反而下降
- **公式：** 输出方向错误 = 正反作用可能反了
- **计算：**
  - 加热系统：输出增大应升温
  - 实际却降温
  - 可能输出方向、执行器接线或控制作用设置错误
- **结论：** 自动调节前必须先用手动模式验证控制方向。

> [!tip] 记忆口诀
> **手动看方向，自动看稳定。**

# 07｜参数整定

> [!info] 核心结论
> **PID 参数整定就是调整 Kp、Ti、Td，让系统既能快速到达目标，又不明显超调和振荡。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>整定口诀</strong><br>
先调 P 看反应，再加 I 消静差，最后少量 D 抑超调。
</div>

## ① 分类/公式/规则

| 参数 | 调节目标 | 经验方向 |
|---|---|---|
| **Kp** | 响应速度 | 太小慢，太大振 |
| **Ti** | 消除静差 | 太小易超调 |
| **Td** | 抑制趋势 | 噪声大时慎用 |
| **采样周期** | 控制节奏 | 与过程动态匹配 |
| **输出限幅** | 防止过控 | 设置上下限 |

## ② 参考表

| 现象 | 可能原因 | 调整方向 |
|---|---|---|
| **响应很慢** | Kp 太小 | 适当增大 Kp |
| **来回振荡** | Kp 太大 / Ti 太强 | 降 Kp 或增 Ti |
| **长期有静差** | I 作用不足 | 减小 Ti 或启用积分 |
| **超调明显** | P/I 过强 | 降低 Kp，减弱积分 |
| **输出跳动** | 反馈噪声 / D过强 | 滤波或减小 D |

## ③ 计算/选型示例

- **已知：** 温度始终比设定值低 `3℃`
- **公式：** 长期静差 → 增强积分作用
- **计算：**
  - 只靠 P 可能留下偏差
  - 加入 I 可以累计偏差
  - 控制器逐步提高输出消除静差
- **结论：** 稳态偏差明显时，应检查积分功能和 Ti 参数。

> [!tip] 记忆口诀
> **慢就加 P，差就加 I，抖就减 P 或弱 I。**

# 08｜调用周期与采样时间

> [!info] 核心结论
> **PID_Compact 应按稳定周期调用，采样周期太慢会控制迟钝，太快会增加 CPU 负载并放大噪声。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>周期理解</strong><br>
PID 是按节拍计算的，节拍不稳，控制也容易不稳。
</div>

## ① 分类/公式/规则

| 调用位置 | 说明 |
|---|---|
| **循环中断 OB** | 推荐用于固定周期控制 |
| **OB1** | 可用但周期可能波动 |
| **采样时间** | 需要与过程响应匹配 |
| **慢过程** | 温度、液位，周期可较长 |
| **快过程** | 压力、速度，周期需更短 |

> [!warning]
> 具体调用周期需按 CPU 性能、过程对象响应速度和 Siemens 帮助文档校核。

## ② 参考表

| 控制对象 | 过程特点 | 周期建议 |
|---|---|---|
| **温度** | 慢过程 | 可较慢 |
| **液位** | 中慢过程 | 中等 |
| **压力** | 较快 | 较短 |
| **流量** | 较快 | 较短 |
| **电机速度** | 快过程 | 需更严格 |

## ③ 计算/选型示例

- **已知：** PID 放在 OB1 中，OB1 周期在 `5ms~80ms` 波动
- **公式：** PID 计算周期波动大 → 控制稳定性下降
- **计算：**
  - 有时 5ms 算一次
  - 有时 80ms 才算一次
  - 积分和微分计算节奏不一致
- **结论：** 建议把 PID 放到固定周期的循环中断 OB 中调用。

> [!tip] 记忆口诀
> **PID 要稳，调用周期先稳。**

# 09｜常见应用场景

> [!info] 核心结论
> **PID_Compact 最适合连续调节场景，不适合简单开关控制或强逻辑顺序控制。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>应用口诀</strong><br>
温压流液速，闭环常用 PID。
</div>

## ① 分类/公式/规则

| 场景 | 控制量 | 执行器 |
|---|---|---|
| **温度控制** | ℃ | 加热器、冷却阀 |
| **压力控制** | bar / MPa | 泵、阀门 |
| **流量控制** | L/min | 调节阀、变频泵 |
| **液位控制** | mm / % | 水泵、阀门 |
| **速度控制** | rpm / Hz | 变频器、伺服 |
| **张力控制** | N | 电机/制动器 |

## ② 参考表

| 对象 | 适合 PID 吗 | 说明 |
|---|---|---|
| **加热炉温度** | 适合 | 慢过程 |
| **水箱液位** | 适合 | 连续调节 |
| **气缸伸缩** | 不适合 | 离散动作 |
| **电机启停** | 不适合 | 开关控制 |
| **流量阀开度** | 适合 | 模拟调节 |
| **报警判断** | 不适合 | 逻辑判断 |

## ③ 计算/选型示例

- **已知：** 需要控制气缸伸出和缩回
- **公式：** 离散动作控制 ≠ PID 控制
- **计算：**
  - 气缸只有伸出/缩回
  - 没有连续调节输出
  - 应使用顺序控制或 FB 封装
- **结论：** 气缸控制不适合用 PID_Compact。

> [!tip] 记忆口诀
> **连续调节用 PID，开关动作别乱用。**

# 10｜常见故障排查

> [!info] 核心结论
> **PID 故障排查顺序：先看反馈值是否正确，再看输出方向，再看模式，最后再调参数。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>排查顺序</strong><br>
反馈量程 → 输出方向 → 手自动模式 → 输出限幅 → PID参数 → 执行器状态。
</div>

## ① 分类/公式/规则

| 问题 | 常见原因 |
|---|---|
| **输出不变** | 未使能、手动模式、限幅 |
| **越调越远** | 正反作用错误 |
| **一直满输出** | 反馈不变、设定过高、输出限幅 |
| **振荡严重** | Kp 太大、积分太强 |
| **长期达不到设定值** | 输出能力不足、积分不足 |
| **反馈跳动** | 传感器干扰、滤波不足 |

## ② 参考表

| 现象 | 排查方向 | 解决建议 |
|---|---|---|
| **PV 不变化** | 传感器/量程 | 查模拟量输入 |
| **输出 100% 不升温** | 执行器问题 | 查加热器/阀门 |
| **输出越大 PV 越小** | 方向反 | 检查正反作用 |
| **自动模式不动作** | 模式未切换 | 查 Mode/Enable |
| **频繁振荡** | 参数太激进 | 降 Kp 或减弱 I |
| **有 Error** | 状态码 | 按帮助诊断 |

## ③ 计算/选型示例

- **已知：** 温度设定 `80℃`，实际值一直显示 `0℃`
- **公式：** 反馈值异常 → 不要先调 PID
- **计算：**
  - 传感器可能断线
  - 模拟量量程可能错误
  - 工程量换算可能错误
  - PID 输入值不可信
- **结论：** 先修正反馈值，再进行 PID 整定。

> [!tip] 记忆口诀
> **PID 不稳先别调，先看反馈准不准。**

## ③ 计算/选型示例

- **已知：** 要做一个水箱液位 PID
- **公式：** 液位控制 = `液位设定值 - 液位反馈值 → PID输出 → 调节阀/水泵`
- **计算：**
  - 设定值：`500mm`
  - 反馈值：液位传感器工程量
  - 输出：阀门开度或水泵频率
  - 先手动确认阀门方向，再切自动
- **结论：** PID_Compact 适合水箱液位这种连续闭环调节，但前提是反馈、输出、方向和量程都正确。
