# 01｜函数 FC 是什么

> [!info] 核心结论
> **FC 是不带固定实例数据的程序块，适合做计算、转换、判断、复用逻辑。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
FC 就像一个“公式工具箱”，输入数据，执行计算，输出结果，本身不长期记状态。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **英文名称** | Function |
| **中文名称** | 函数 |
| **简称** | `FC` |
| **核心特点** | 不自动生成实例 DB |
| **适合场景** | 数学计算、数据转换、条件判断、公共逻辑 |
| **调用方式** | 由 `OB` / `FB` / `FC` 调用 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>核心规则</strong><br>
FC 不会自己保存长期状态；如果需要记忆状态，优先考虑 FB 或外部 DB。
</div>

## ② 参考表

| 块类型 | 是否带实例 DB | 典型用途 |
|---|---|---|
| **OB** | 否，系统自动调度 | 程序入口 |
| **FB** | **有实例 DB** | 电机、阀门、状态机 |
| **FC** | **无实例 DB** | 计算、换算、判断 |
| **DB** | 数据存储 | 参数、配方、记录 |

## ③ 计算/选型示例

- **已知：** 要把模拟量原始值 `0~27648` 换算成温度 `0~100℃`
- **公式：** 纯计算逻辑 → 优先用 FC
- **计算：**
  - 输入原始值
  - 按比例换算
  - 输出工程量
  - 不需要保存上一次状态
- **结论：** 这种工程量换算适合写成 FC。

> [!tip] 记忆口诀
> **只算结果用 FC，要记状态用 FB。**

# 02｜FC 的核心特点

> [!info] 核心结论
> **FC 的特点是轻量、通用、可复用，但内部变量默认不保存跨扫描周期状态。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程理解</strong><br>
FC 适合做“拿来就算”的功能，不适合做“需要记住上次状态”的设备对象。
</div>

## ① 分类/公式/规则

| 特点 | 说明 |
|---|---|
| **无实例 DB** | 调用时不自动生成专属数据块 |
| **执行快、结构轻** | 适合公共算法 |
| **结果依赖输入** | 输入变化，输出跟着变化 |
| **不适合状态机** | 状态保存不方便 |
| **可重复调用** | 多处可调用同一个 FC |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点规则</strong><br>
FC 内部 Temp 变量只在本次调用中有效，扫描结束后不适合用来保存状态。
</div>

## ② 参考表

| 变量类型 | 是否适合 FC | 说明 |
|---|---|---|
| **Input** | 适合 | 外部传入参数 |
| **Output** | 适合 | 输出计算结果 |
| **InOut** | 适合但谨慎 | 双向读写外部变量 |
| **Temp** | 适合 | 临时中间变量 |
| **Static** | FC 中通常不使用 | 需要状态更适合 FB |

## ③ 计算/选型示例

- **已知：** 需要判断一个压力值是否超过报警上限
- **公式：** 判断结果 = 压力值 > 上限值
- **计算：**
  - 输入：当前压力
  - 输入：报警上限
  - 输出：是否超限
- **结论：** 这种简单判断逻辑适合封装成 FC。

> [!tip] 记忆口诀
> **FC 做判断，FB 做过程；FC 管计算，FB 管状态。**

# 03｜FC 接口区

> [!info] 核心结论
> **FC 接口区用于定义输入、输出、输入输出和临时变量，是 FC 与外部程序交换数据的通道。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>接口口诀</strong><br>
Input 接输入，Output 给结果，InOut 双向改，Temp 临时算。
</div>

## ① 分类/公式/规则

| 接口区 | 中文含义 | 作用 |
|---|---|---|
| **Input** | 输入参数 | 外部给 FC 的数据 |
| **Output** | 输出参数 | FC 计算后的结果 |
| **InOut** | 输入输出参数 | 外部变量传入并被修改 |
| **Temp** | 临时变量 | 本次调用中的中间值 |
| **Return** | 返回值 | FC 的函数返回结果，按语言/使用方式配置 |

## ② 参考表

| 接口 | 适合放什么 | 注意 |
|---|---|---|
| **Input** | 原始值、设定值、使能条件 | 不在 FC 内长期保存 |
| **Output** | 工程量、判断结果、状态输出 | 调用后给外部使用 |
| **InOut** | 需要直接修改的外部变量 | 慎用，避免误改数据 |
| **Temp** | 中间计算值 | 不能当保持变量 |
| **Return** | 单一计算结果 | 适合简单函数 |

## ③ 计算/选型示例

- **已知：** 写一个 FC 计算电流 `I = P ÷ U`
- **公式：** `I = P ÷ U`
- **计算：**
  - `P` 放 Input
  - `U` 放 Input
  - `I` 放 Output 或 Return
- **结论：** 纯公式计算非常适合 FC。

> [!tip] 记忆口诀
> **外面给数据，FC 算结果；临时变量别当记忆。**

# 04｜FC 与 FB 的区别

> [!info] 核心结论
> **FC 适合无记忆计算，FB 适合有状态控制；两者最大的区别是 FB 有实例 DB，而 FC 没有。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>选型口诀</strong><br>
有状态选 FB，无状态选 FC；设备控制选 FB，公式换算选 FC。
</div>

## ① 分类/公式/规则

| 对比项 | FC | FB |
|---|---|---|
| **实例 DB** | 无 | 有 |
| **保存状态** | 不适合 | 适合 |
| **调用方式** | 直接调用 | 调用时需实例 |
| **典型用途** | 计算、换算、判断 | 电机、阀门、气缸、状态机 |
| **复用方式** | 多处直接调用 | 多实例或多个实例 DB |

## ② 参考表

| 需求 | 推荐块 | 原因 |
|---|---|---|
| **模拟量换算** | FC | 纯计算 |
| **温度单位转换** | FC | 不需要状态 |
| **电机启停控制** | FB | 需要运行状态 |
| **阀门超时报警** | FB | 需要定时和锁存 |
| **字符串处理** | FC | 多为计算/转换 |
| **通信状态机** | FB | 需要步骤和状态 |

## ③ 计算/选型示例

- **已知：** 要做一个气缸控制逻辑，包含伸出、缩回、到位、超时报警
- **公式：** 需要保存动作过程和报警状态 → 选 FB
- **计算：**
  - 动作步骤要记住
  - 定时器状态要保存
  - 报警要锁存
- **结论：** 这类逻辑不适合 FC，更适合 FB。

> [!tip] 记忆口诀
> **FC 像公式，FB 像设备。**

# 05｜FC 典型用途

> [!info] 核心结论
> **FC 最适合做项目中的公共小工具，例如工程量换算、限幅、比较判断、数据打包、字节拆分。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程理解</strong><br>
一个好的 FC 就像一把标准工具，多处调用，逻辑统一，维护方便。
</div>

## ① 分类/公式/规则

| 用途 | 示例 |
|---|---|
| **工程量换算** | `0~27648` → `0~100℃` |
| **限幅处理** | 超过上限取上限，低于下限取下限 |
| **单位转换** | mm ↔ pulse，Hz ↔ rpm |
| **数据拆分** | Word 拆 Byte，高低字节处理 |
| **数据组合** | Byte 合 Word，Word 合 DWord |
| **条件判断** | 超限、范围判断、状态判断 |

## ② 参考表

| FC 模板 | 作用 |
|---|---|
| **FC_Scale** | 模拟量比例换算 |
| **FC_Limit** | 数值限幅 |
| **FC_ByteSwap** | 高低字节交换 |
| **FC_PulseToMM** | 脉冲转位移 |
| **FC_RPMCalc** | 脉冲测速 |
| **FC_CheckRange** | 范围检查 |

## ③ 计算/选型示例

- **已知：** 模拟量原始值 `13824`，对应量程 `0~100℃`
- **公式：** 工程量 = 原始值 ÷ `27648` × 量程
- **计算：**
  - `13824 ÷ 27648 × 100 = 50`
- **结论：** 当前温度约为 `50℃`，该换算逻辑适合封装成 FC。

> [!tip] 记忆口诀
> **重复计算别复制，封装 FC 更清晰。**

# 06｜FC 调用方式

> [!info] 核心结论
> **FC 不会自动运行，必须被 OB、FB 或其他 FC 调用后才会执行。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>调用链理解</strong><br>
CPU 自动执行 OB，OB 再调用 FC，FC 才真正参与程序运行。
</div>

## ① 分类/公式/规则

| 调用位置 | 说明 |
|---|---|
| **OB1 调用 FC** | 最常见 |
| **FB 内调用 FC** | 设备逻辑中调用公共算法 |
| **FC 内调用 FC** | 算法分层 |
| **启动 OB 调用 FC** | 初始化计算 |
| **中断 OB 调用 FC** | 周期计算，需注意执行时间 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>核心规则</strong><br>
FC 写好了但没有被调用，就不会执行。
</div>

## ② 参考表

| 项目结构 | 推荐做法 |
|---|---|
| **OB1** | 负责主流程调度 |
| **FC** | 放通用计算 |
| **FB** | 放设备控制 |
| **DB** | 放参数和数据 |
| **FC 调用层级** | 不宜过深，方便维护 |

## ③ 计算/选型示例

- **已知：** 已创建 `FC_Scale`，但在线监控发现工程量不变化
- **公式：** FC 执行前提 = 被调用且调用条件成立
- **计算：**
  - 检查 `OB1` 是否调用 `FC_Scale`
  - 检查调用网络是否有能流
  - 检查输入参数是否连接正确
- **结论：** FC 不执行时，优先检查是否被调用。

> [!tip] 记忆口诀
> **FC 不调用，逻辑不生效。**

# 07｜FC 编程模板

> [!info] 核心结论
> **一个好的 FC 应该输入清楚、输出明确、内部不乱改外部状态，适合做稳定可复用的小功能。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>模板思路</strong><br>
输入参数 → 检查范围 → 执行计算 → 输出结果 → 返回状态。
</div>

## ① 分类/公式/规则

| 模块 | 内容 |
|---|---|
| **输入区** | 原始数据、上下限、使能条件 |
| **计算区** | 比例换算、逻辑判断、数据处理 |
| **保护区** | 除零保护、范围限制 |
| **输出区** | 计算结果、有效标志、错误标志 |
| **注释区** | 写清单位、范围、使用条件 |

## ② 参考表

| 模板项 | 建议 |
|---|---|
| **变量命名** | 清晰表达单位和含义 |
| **输入范围** | 需要边界检查 |
| **除法计算** | 防止除数为 0 |
| **输出结果** | 明确数据类型 |
| **错误标志** | 复杂 FC 建议增加 |

## ③ 计算/选型示例

- **已知：** 写一个限幅 FC，要求输出不超过上下限
- **公式：**
  - 若 `Value > HighLimit`，输出 `HighLimit`
  - 若 `Value < LowLimit`，输出 `LowLimit`
  - 否则输出 `Value`
- **计算：**
  - 输入值：`120`
  - 上限：`100`
  - 下限：`0`
  - 输出：`100`
- **结论：** 限幅逻辑适合封装为通用 FC。

> [!tip] 记忆口诀
> **输入要干净，输出要明确；边界要保护，复用才可靠。**

# 08｜FC 常见错误

> [!info] 核心结论
> **FC 常见错误是：以为 Temp 会记忆、未连接输出、没被调用、InOut 误改外部变量、除数为 0。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>避坑提醒</strong><br>
FC 里的 Temp 变量不要当成状态寄存器使用，下一次调用可能不是你想象的值。
</div>

## ① 分类/公式/规则

| 问题 | 原因 |
|---|---|
| **结果不变化** | FC 没被调用或输出没接 |
| **状态丢失** | 把状态放到 Temp |
| **数据被误改** | InOut 使用不当 |
| **运行报错** | 除数为 0 或类型错误 |
| **结果不准** | 数据类型、比例、范围错误 |
| **多处调用异常** | 误以为 FC 内部能独立保存状态 |

## ② 参考表

| 现象 | 排查方向 | 解决建议 |
|---|---|---|
| **输出为 0** | 输入是否有值 | 在线监控参数 |
| **计算错误** | 数据类型 | 检查 Int/Real 转换 |
| **偶发异常** | 除零或越界 | 增加保护判断 |
| **变量被改乱** | InOut 接错 | 减少 InOut 使用 |
| **状态不保持** | Temp 用错 | 改用 FB/DB |
| **无法编译** | 接口变化 | 更新调用点 |

## ③ 计算/选型示例

- **已知：** FC 中做 `Result = A / B`，偶发 CPU 报错
- **公式：** 除法前必须判断除数
- **计算：**
  - 当 `B = 0` 时
  - 除法非法
  - 可能引发运行错误
- **结论：** FC 中涉及除法时，必须加入 `B <> 0` 的保护。

> [!tip] 记忆口诀
> **FC 做计算，边界要先判；除数不为零，程序才安稳。**

# 09｜FC 与 DB 的配合

> [!info] 核心结论
> **FC 本身不保存实例状态，但可以通过读写外部 DB 来使用参数、配方和结果数据。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
FC 没有自己的记事本，但可以读写别人准备好的 DB 数据。
</div>

## ① 分类/公式/规则

| 数据来源 | 用法 |
|---|---|
| **Input 参数** | 外部传入当前值 |
| **Output 参数** | 输出计算结果 |
| **InOut 参数** | 修改外部变量 |
| **全局 DB** | 存放参数、配方、结果 |
| **PLC Tag** | 直接连接 I/O 或变量 |

## ② 参考表

| 场景 | 推荐方式 |
|---|---|
| **参数固定** | 放全局 DB |
| **多处复用** | 通过 Input 传入 |
| **结果要保存** | 输出到 DB |
| **结构化数据** | 用 Struct / UDT |
| **临时计算** | 用 Temp |

## ③ 计算/选型示例

- **已知：** 多个模拟量通道都要使用同一个换算 FC，但量程不同
- **公式：** 通用 FC + 外部参数 DB
- **计算：**
  - 每个通道的上限、下限放在 DB
  - 调用 FC 时传入对应参数
  - 输出工程量回写到 DB
- **结论：** FC 配合 DB 可以做出通用、可维护的参数化功能。

> [!tip] 记忆口诀
> **FC 管算法，DB 管参数；算法通用，参数外放。**

# 10｜一页速记总卡

> [!info] 核心结论
> **FC 的核心价值是封装无状态、可复用的公共逻辑，让 PLC 程序更简洁、更标准。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
FC 做计算，FB 做设备；OB 调用入口，DB 存数据。
</div>

## ① 分类/公式/规则

| 主轴 | 关键词 |
|---|---|
| **本质** | 无实例 DB 的函数 |
| **适合** | 计算、换算、判断、转换 |
| **不适合** | 长期状态、复杂设备对象 |
| **接口** | Input / Output / InOut / Temp |
| **调用** | 由 OB / FB / FC 调用 |
| **配合** | 可读写外部 DB |

## ② 参考表

| 需求 | 推荐块 |
|---|---|
| **模拟量换算** | FC |
| **单位转换** | FC |
| **数据限幅** | FC |
| **字节交换** | FC |
| **电机控制** | FB |
| **阀门状态机** | FB |
| **程序入口** | OB |
| **参数存储** | DB |

## ③ 计算/选型示例

- **已知：** 项目里很多地方都要判断数值是否在范围内
- **公式：** 重复判断逻辑 → 封装成 FC
- **计算：**
  - 输入：当前值、上限、下限
  - 输出：是否超上限、是否低于下限、是否在范围内
  - 多处重复调用
- **结论：** 通用判断逻辑封装成 FC，可以减少重复代码，提高程序一致性。

> [!tip] 记忆口诀
> **重复算法封 FC，设备对象封 FB；入口进 OB，数据进 DB。**
