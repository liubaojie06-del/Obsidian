# 01｜监控与强制表是什么

> [!info] 核心结论
> **监控与强制表是 TIA Portal 中用于在线查看、修改、强制 PLC 变量和 I/O 信号的调试工具。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>图片识别</strong><br>
截图中的 <strong>监控与强制表</strong> 是 TIA Portal 项目树里的调试入口，常用于现场查信号、改设定、测试输出、排查逻辑。
</div>

## ① 分类/公式/规则

| 工具 | 中文含义 | 作用 |
|---|---|---|
| **监控表** | Watch Table | 在线查看变量当前值 |
| **修改变量** | Modify | 临时写入变量值 |
| **强制表** | Force Table | 强制 I/O 或变量保持指定值 |
| **在线监控** | Online Monitor | 实时观察 PLC 状态 |
| **测试调试** | Commissioning | 调试设备动作和信号 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
监控表是“看变量”，强制表是“硬插手变量”；前者偏观察，后者有风险。
</div>

## ② 参考表

| 项目 | 监控表 | 强制表 |
|---|---|---|
| **主要用途** | 查看变量值 | 强制变量或 I/O |
| **风险等级** | 较低 | **较高** |
| **是否影响设备动作** | 通常不直接影响 | 可能直接影响 |
| **常用场景** | 查信号、查状态 | 临时测试、排故 |
| **使用建议** | 可长期观察 | 不要长期强制 |

## ③ 计算/选型示例

- **已知：** 电机不启动，需要判断是按钮没信号还是程序条件不满足  
- **公式：** 排查顺序 = `监控输入 → 监控中间条件 → 监控输出`
- **计算：**
  - 看 `Start_PB` 是否为 TRUE
  - 看 `AutoMode`、`Safety_OK` 是否满足
  - 看 `Motor_RunCmd` 是否输出
- **结论：** 先用监控表观察信号链，再决定是否需要强制测试。

> [!tip] 记忆口诀
> **先监控，后修改；能不强制，就不强制。**

# 02｜监控表 Watch Table

> [!info] 核心结论
> **监控表用于在线查看 PLC 变量、I/O 地址、DB 数据、状态位和模拟量数值，是最常用的调试表。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程理解</strong><br>
监控表就像 PLC 的“实时仪表盘”，可以把你关心的变量集中放在一张表里观察。
</div>

## ① 分类/公式/规则

| 可监控对象 | 示例 |
|---|---|
| **输入点** | `I0.0`、`Start_PB` |
| **输出点** | `Q0.0`、`Motor_RunCmd` |
| **中间变量** | `M0.0`、`AutoMode` |
| **DB变量** | `DB_Para.SpeedSet` |
| **模拟量** | `IW64`、`Temp_Raw` |
| **状态字** | `Drive_StatusWord` |

## ② 参考表

| 监控内容 | 看什么 | 用途 |
|---|---|---|
| **Bool变量** | TRUE / FALSE | 查开关状态 |
| **Int/DInt** | 整数值 | 查计数、位置 |
| **Real** | 工程量 | 查温度、压力 |
| **Word** | 十六进制/二进制 | 查状态字 |
| **Array** | 多个元素 | 查通信缓冲区 |
| **DB结构体** | 成组变量 | 查设备状态 |

## ③ 计算/选型示例

- **已知：** 压力传感器模拟量原始值为 `13824`，量程为 `0~10bar`
- **公式：** 压力 = `Raw ÷ 27648 × 10`
- **计算：**
  - `13824 ÷ 27648 × 10 = 5bar`
- **结论：** 在监控表中同时观察 `Pressure_Raw` 和 `Pressure_Real`，可以快速判断换算是否正确。

> [!tip] 记忆口诀
> **变量集中看，故障更好判。**

# 03｜修改变量 Modify

> [!info] 核心结论
> **修改变量用于在线给变量写入一个临时值，常用于调试设定值、复位标志、测试中间条件。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点区别</strong><br>
修改变量通常是“写一次值”；强制变量是“持续压住值”。两者风险不同。
</div>

## ① 分类/公式/规则

| 修改对象 | 是否推荐 | 注意 |
|---|---|---|
| **参数设定值** | 推荐 | 如速度、温度设定 |
| **报警复位位** | 可用 | 确认现场安全 |
| **中间测试位** | 可用 | 调试后恢复 |
| **真实输出点** | 谨慎 | 可能动作设备 |
| **安全相关信号** | 不推荐 | 需按安全规范 |
| **运动控制命令** | 谨慎 | 防止误动作 |

## ② 参考表

| 场景 | 可修改变量 | 注意 |
|---|---|---|
| **调试温度设定** | `Temp_Set` | 检查上下限 |
| **清除报警锁存** | `Alarm_Reset` | 确认故障已排除 |
| **测试条件满足** | `Test_Enable` | 调试后关闭 |
| **通信发送触发** | `Send_Req` | 用上升沿 |
| **轴运动命令** | `Move_Req` | 必须低速安全 |

## ③ 计算/选型示例

- **已知：** HMI 暂时没做好，需要测试变频器速度给定  
- **公式：** 临时设定值 → 可用监控表修改  
- **计算：**
  - 在监控表添加 `Speed_Set`
  - 修改为 `30.0Hz`
  - 观察变频器实际频率
- **结论：** 参数类变量适合用修改功能测试，但要设置合理范围。

> [!tip] 记忆口诀
> **改参数可以，改输出谨慎；改完记得恢复。**

# 04｜强制表 Force Table

> [!info] 核心结论
> **强制表用于让 PLC 某些信号保持指定值，常用于硬件排查和现场调试，但风险比监控表高很多。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>安全警告</strong><br>
强制输出可能直接让电机、阀门、气缸、加热器动作；使用前必须确认现场无人、机械安全、急停有效。
</div>

## ① 分类/公式/规则

| 强制对象 | 风险 | 说明 |
|---|---|---|
| **输入点 I** | 中 | 可模拟传感器信号 |
| **输出点 Q** | 高 | 可能直接动作设备 |
| **中间变量 M** | 中 | 可能绕过逻辑条件 |
| **DB变量** | 中 | 可能改变流程状态 |
| **安全信号** | 极高 | 不建议随意强制 |
| **运动轴命令** | 极高 | 可能造成撞机 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>核心规则</strong><br>
强制不是正常程序逻辑，只能用于临时调试和排故，结束后必须取消强制。
</div>

## ② 参考表

| 使用场景 | 是否适合强制 | 注意 |
|---|---|---|
| **模拟输入信号** | 可用 | 确认不会误动作 |
| **测试输出模块** | 谨慎 | 断开负载更安全 |
| **绕过安全条件** | 不建议 | 风险极高 |
| **检查接线方向** | 谨慎 | 低速、低风险 |
| **长期运行保持** | 不允许 | 应改程序逻辑 |
| **紧急生产临时处理** | 极谨慎 | 需审批和记录 |

## ③ 计算/选型示例

- **已知：** 电机输出 `Q0.0` 不动作，需要判断是程序没输出还是输出模块/接线问题  
- **公式：** 先监控程序输出，再谨慎强制硬件输出  
- **计算：**
  - 监控 `Motor_RunCmd`
  - 若程序输出为 FALSE，先查逻辑
  - 若需要测试输出模块，确保现场安全后短时强制 `Q0.0`
- **结论：** 强制输出只能作为短时硬件测试手段，不能代替程序调试。

> [!tip] 记忆口诀
> **强制一时爽，忘关事故长。**

# 05｜监控表常用排查思路

> [!info] 核心结论
> **用监控表排故时，要沿着“输入 → 条件 → 中间状态 → 输出 → 反馈”的链路逐级检查。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>排查链路</strong><br>
输入信号 → 联锁条件 → 程序状态 → 输出命令 → 现场反馈
</div>

## ① 分类/公式/规则

| 层级 | 检查内容 |
|---|---|
| **输入层** | 按钮、传感器、限位 |
| **条件层** | 自动模式、安全条件、联锁 |
| **程序层** | 步骤号、状态机、报警位 |
| **输出层** | 电机、阀门、气缸命令 |
| **反馈层** | 运行反馈、到位信号 |
| **诊断层** | Error、Status、报警码 |

## ② 参考表

| 设备不动作 | 优先监控 |
|---|---|
| **电机不转** | Start、Safety_OK、RunCmd、RunFB |
| **阀门不开** | OpenCmd、OpenFB、CloseFB、Fault |
| **气缸不伸** | ExtendCmd、HomeFB、WorkFB、AirOK |
| **伺服不动** | Enable、HomeDone、MoveReq、Error |
| **通信不发** | REQ、BUSY、DONE、ERROR、STATUS |

## ③ 计算/选型示例

- **已知：** 阀门不打开  
- **公式：** 阀门排查 = `开命令 + 联锁 + 输出 + 到位反馈`
- **计算：**
  - 看 `Valve_OpenCmd` 是否为 TRUE
  - 看 `Air_OK`、`Safety_OK` 是否满足
  - 看输出点是否为 TRUE
  - 看 `Valve_OpenFB` 是否到位
- **结论：** 监控表可以快速判断故障在程序逻辑、输出回路还是现场阀门。

> [!tip] 记忆口诀
> **设备不动别乱猜，顺着信号链往下排。**

# 06｜强制使用安全流程

> [!info] 核心结论
> **强制前必须做安全确认，强制中必须有人监护，强制后必须取消并复查。**

<div class="card red" style="background:#FFF4F3;border-left:6px solid #E74C3C;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>强制三原则</strong><br>
确认安全再强制，短时测试不长期，测试结束必须取消。
</div>

## ① 分类/公式/规则

| 步骤 | 动作 | 目的 |
|---|---|---|
| **1** | 通知现场人员 | 防止误伤 |
| **2** | 确认设备状态 | 防止误动作 |
| **3** | 确认急停有效 | 保留安全手段 |
| **4** | 低风险信号先测 | 逐步验证 |
| **5** | 短时强制 | 不长期保持 |
| **6** | 取消强制 | 恢复程序控制 |
| **7** | 记录结果 | 便于追溯 |

## ② 参考表

| 强制对象 | 安全建议 |
|---|---|
| **输出电机** | 先确认机械无人员 |
| **气缸阀** | 防夹伤，低压调试 |
| **加热器** | 防超温，设保护 |
| **伺服轴** | 低速、小距离、限位有效 |
| **安全信号** | 原则上不要强制 |
| **通信变量** | 注意对端动作 |

## ③ 计算/选型示例

- **已知：** 需要强制气缸伸出测试电磁阀  
- **公式：** 强制气缸 = 机械安全 + 气源确认 + 限位确认  
- **计算：**
  - 确认气缸运动范围无人
  - 确认急停有效
  - 短时强制伸出输出
  - 观察伸出到位反馈
  - 立即取消强制
- **结论：** 气缸类输出强制必须重点防夹伤和误动作。

> [!tip] 记忆口诀
> **强制先喊人，动作先看机械，结束先取消。**

# 07｜常见错误与排查

> [!info] 核心结论
> **监控与强制表常见问题包括变量选错、地址不对、数据类型不匹配、强制未取消、在线状态错误。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>避坑提醒</strong><br>
看到变量值不对，不一定是程序错，也可能是变量地址、数据类型或监控对象选错。
</div>

## ① 分类/公式/规则

| 问题 | 常见原因 |
|---|---|
| **变量值不变化** | 地址错、未在线、变量未被程序写入 |
| **修改无效** | 程序下一扫描又覆盖 |
| **强制无效** | 对象不支持强制或未激活 |
| **输出异常动作** | 强制未取消 |
| **数值乱码** | 数据类型选错 |
| **HMI 与监控值不同** | HMI 变量路径不同 |

## ② 参考表

| 现象 | 排查方向 | 解决建议 |
|---|---|---|
| **监控显示灰色** | 是否在线 | 重新连接 PLC |
| **修改后马上变回去** | 程序覆盖 | 查写入逻辑 |
| **强制后设备不动** | 输出模块/负载/安全回路 | 查硬件 |
| **取消强制后异常** | 程序状态未恢复 | 复位流程 |
| **数值不合理** | 类型/字节序 | 查数据类型 |
| **忘记强制点** | 查强制表状态 | 全部取消强制 |

## ③ 计算/选型示例

- **已知：** 修改 `Motor_RunCmd` 为 TRUE，但下一秒又变成 FALSE  
- **公式：** 修改变量是临时写入，程序扫描会继续赋值  
- **计算：**
  - 你写 TRUE
  - PLC 程序下一扫描根据条件重新计算
  - 条件不满足，又写回 FALSE
- **结论：** 修改变量不能绕过长期逻辑，若要测试条件，应查清程序赋值来源。

> [!tip] 记忆口诀
> **改了又变回，说明程序在写它。**

# 08｜一页速记总卡

> [!info] 核心结论
> **监控与强制表的核心价值是调试排故：监控用于看清信号，修改用于临时测试，强制用于短时验证硬件。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
监控看状态，修改试参数，强制测硬件；强制要短时，结束要取消。
</div>

## ① 分类/公式/规则

| 工具 | 一句话 |
|---|---|
| **监控表** | 在线看变量 |
| **修改变量** | 临时写变量 |
| **强制表** | 持续压变量 |
| **变量地址** | 决定看哪里 |
| **数据类型** | 决定怎么看 |
| **状态记录** | 调试后要恢复 |

## ② 参考表

| 需求 | 推荐方式 |
|---|---|
| **查按钮信号** | 监控表 |
| **查程序条件** | 监控表 |
| **临时改设定值** | 修改变量 |
| **清报警测试** | 修改变量 |
| **测试输出模块** | 短时强制 |
| **长期改变逻辑** | 改程序，不用强制 |
| **安全相关信号** | 不建议强制 |

## ③ 计算/选型示例

- **已知：** 设备不动作，需要快速定位问题  
- **公式：** 快速排故 = `监控输入 → 监控条件 → 监控输出 → 必要时短时强制`
- **计算：**
  - 先看输入是否到 PLC
  - 再看联锁条件是否满足
  - 再看输出命令是否生成
  - 最后才考虑强制输出测试硬件
- **结论：** 正确顺序是先观察再干预，避免一上来强制导致误动作。

> [!tip] 记忆口诀
> **调试先看，排故再改；强制最后用，用完马上关。**
