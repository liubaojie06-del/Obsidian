# 01 计划任务总览

> [!info] 核心结论
> **计划任务** 用于让HMI/SCADA在指定时间、指定周期或指定条件下自动执行操作，例如 **数据记录、报表生成、文件导出、数据清理、配方备份、变量写入、脚本执行和系统维护**。

## ① 分类/公式/规则

> [!warning] 计划任务公式
> **计划任务 = 触发时间/周期 + 执行动作 + 执行对象 + 运行条件 + 日志记录**

## ② 参考表

| 任务类型 | 典型作用 |
|---|---|
| **周期任务** | 每秒、每分钟、每小时执行一次 |
| **定时任务** | 每天8:00、每周一、每月1号执行 |
| **报表任务** | 定时生成日报、班报、月报 |
| **归档任务** | 定时保存趋势、报警、生产数据 |
| **清理任务** | 删除过期记录，防止磁盘满 |
| **备份任务** | 备份配方、数据库、工程参数 |
| **脚本任务** | 定时执行C/VB脚本或系统函数 |

## ③ 计算/选型示例

- 已知：  
  每天0点需要自动生成前一天生产日报。
- 公式：  
  **定时触发 + 报表生成动作 = 计划任务**
- 计算：  
  计划任务在每天00:00执行，读取昨日产量、报警、停机数据并生成报表。
- 结论：  
  这种场景适合用计划任务，而不是人工每天导出。

> [!tip] 记忆口诀
> 人每天重复做的固定工作，优先考虑交给计划任务。

# 02 计划任务的作用

> [!info] 核心结论
> 计划任务的核心作用是让系统自动完成固定时间或固定周期的维护、记录、统计、备份和导出工作。

## ① 分类/公式/规则

> [!warning] 使用判断规则
> **固定时间执行 + 重复性强 + 规则明确 → 适合计划任务**

## ② 参考表

| 作用 | 说明 |
|---|---|
| **减少人工操作** | 自动导出、自动备份、自动清理 |
| **保证执行一致性** | 每天、每班、每小时按规则执行 |
| **提高追溯能力** | 定期保存关键数据 |
| **降低漏操作风险** | 避免人工忘记导出或备份 |
| **维护系统稳定** | 定期清理旧数据，防止磁盘满 |
| **支持报表统计** | 自动生成生产、质量、报警报表 |
| **支持无人值守** | 设备夜间或远程站点自动运行 |

## ③ 计算/选型示例

- 已知：  
  操作员每天下班前都要手动导出生产记录。
- 公式：  
  **重复人工导出 → 计划任务自动导出**
- 计算：  
  可设置每天20:00自动导出当天生产记录。
- 结论：  
  计划任务可以减少漏导出和人工操作错误。

> [!tip] 记忆口诀
> 固定时间、固定动作、固定规则，就是计划任务的典型场景。

# 03 周期任务

> [!info] 核心结论
> 周期任务按固定间隔重复执行，例如每1秒、10秒、1分钟、1小时执行一次。

## ① 分类/公式/规则

> [!warning] 周期任务公式
> **执行次数 = 运行时间 ÷ 周期时间**

## ② 参考表

| 周期 | 适合任务 |
|---|---|
| **100ms~1s** | 实时刷新、特殊监控，不宜过多 |
| **5s~10s** | 常规状态检查 |
| **1min** | 产量统计、慢变量记录 |
| **5min~10min** | 能耗统计、系统状态巡检 |
| **1h** | 小时报表、数据汇总 |
| **1天** | 日报、备份、数据清理 |
| **1月** | 月报、长期归档 |

## ③ 计算/选型示例

- 已知：  
  一个任务每10秒执行一次，系统连续运行24小时。
- 公式：  
  **执行次数 = 86400 ÷ 10**
- 计算：  
  执行次数 = **8640次/天**
- 结论：  
  周期越短，任务执行次数越多，对系统负载影响越大。

> [!tip] 记忆口诀
> 周期任务不要设得太快，够用就好。

# 04 定时任务

> [!info] 核心结论
> 定时任务在指定时间点执行，常用于日报生成、班次结算、数据库备份、文件导出和数据清理。

## ① 分类/公式/规则

> [!warning] 定时任务公式
> **定时任务 = 指定时间点 + 指定动作**

## ② 参考表

| 时间触发 | 示例 |
|---|---|
| **每天固定时间** | 每天00:00生成日报 |
| **每班结束时** | 08:00、16:00、24:00生成班报 |
| **每周固定日** | 每周一备份数据 |
| **每月固定日** | 每月1号生成月报 |
| **开机后延时** | Runtime启动后60秒执行初始化 |
| **整点任务** | 每小时整点汇总数据 |
| **停机前任务** | 关闭系统前保存数据 |

## ③ 计算/选型示例

- 已知：  
  工厂三班制，班次结束时间为8:00、16:00、24:00。
- 公式：  
  **班报任务 = 班次结束时间触发**
- 计算：  
  设置三个定时任务，分别生成对应班次报表。
- 结论：  
  班次类统计适合按固定时间触发。

> [!tip] 记忆口诀
> 到点就做，用定时任务；隔多久做一次，用周期任务。

# 05 启动任务

> [!info] 核心结论
> 启动任务在HMI Runtime启动时执行，常用于初始化变量、检查连接、加载默认配方、同步时间和刷新系统状态。

## ① 分类/公式/规则

> [!warning] 启动任务规则
> **启动任务 = Runtime启动 + 初始化动作 + 状态检查**

## ② 参考表

| 启动任务 | 作用 |
|---|---|
| **初始化变量** | 清除临时变量、设置默认值 |
| **检查PLC连接** | 判断PLC是否在线 |
| **加载默认画面** | 打开首页或登录页 |
| **加载默认配方** | 读取上次使用配方 |
| **时间同步** | NTP或服务器校时 |
| **检查数据库** | 判断记录功能是否可用 |
| **启动日志** | 记录Runtime启动时间 |

## ③ 计算/选型示例

- 已知：  
  HMI开机后需要等待PLC启动完成再读取数据。
- 公式：  
  **启动延时 ≥ PLC启动时间**
- 计算：  
  如果PLC约30秒后通讯稳定，可设置启动任务延时30~60秒。
- 结论：  
  启动任务不要过早执行，否则可能因PLC未就绪而失败。

> [!tip] 记忆口诀
> 系统刚启动时，先初始化，再通讯，再运行。

# 06 班次任务

> [!info] 核心结论
> 班次任务用于按生产班次统计产量、良率、报警、停机、能耗和操作记录。

## ① 分类/公式/规则

> [!warning] 班次统计公式
> **班次数据 = 班次结束值 - 班次开始值**

## ② 参考表

| 班次任务 | 说明 |
|---|---|
| **班次产量统计** | 统计本班总产量 |
| **良品/不良品统计** | 计算良率和不良率 |
| **报警统计** | 统计本班报警次数 |
| **停机统计** | 统计故障停机时间 |
| **能耗统计** | 统计本班用电、气、水 |
| **班报生成** | 自动生成班次报表 |
| **班次清零** | 切换班次时清零班次计数 |

## ③ 计算/选型示例

- 已知：  
  班次开始产量为1200，班次结束产量为1850。
- 公式：  
  **班次产量 = 1850 - 1200**
- 计算：  
  班次产量 = **650件**
- 结论：  
  计划任务可在班次结束时自动计算并保存班次产量。

> [!tip] 记忆口诀
> 班次任务看开始值和结束值，差值就是本班数据。

# 07 报表任务

> [!info] 核心结论
> 报表任务用于自动生成日报、班报、月报、报警报表、质量报表和能耗报表。

## ① 分类/公式/规则

> [!warning] 报表任务公式
> **报表 = 数据范围 + 数据源 + 模板 + 生成时间 + 保存路径**

## ② 参考表

| 报表类型 | 内容 |
|---|---|
| **日报** | 当日产量、报警、停机、良率 |
| **班报** | 每班产量、节拍、不良品 |
| **月报** | 月度汇总数据 |
| **报警报表** | 报警次数、持续时间、等级 |
| **质量报表** | 检测结果、良率、不良率 |
| **能耗报表** | 电、水、气、压缩空气消耗 |
| **维护报表** | 设备运行时间、保养提醒 |

## ③ 计算/选型示例

- 已知：  
  每天23:59需要生成当天生产日报。
- 公式：  
  **日报 = 当天生产数据 + 报表模板**
- 计算：  
  计划任务在23:59读取当天数据并生成PDF或Excel报表。
- 结论：  
  报表生成非常适合计划任务自动执行。

> [!tip] 记忆口诀
> 报表任务要确定三件事：数据从哪来、用哪个模板、保存到哪里。

# 08 数据导出任务

> [!info] 核心结论
> 数据导出任务用于定期把报警、趋势、操作、生产、配方等记录导出到文件或服务器。

## ① 分类/公式/规则

> [!warning] 导出任务公式
> **导出文件 = 数据类型 + 时间范围 + 文件格式 + 文件路径**

## ② 参考表

| 导出内容 | 用途 |
|---|---|
| **报警记录导出** | 故障分析 |
| **趋势数据导出** | 工艺分析 |
| **操作记录导出** | 人员操作追溯 |
| **生产记录导出** | 产量和质量统计 |
| **配方记录导出** | 换型追溯 |
| **参数记录导出** | 工艺变化分析 |
| **系统记录导出** | 维护诊断 |

## ③ 计算/选型示例

- 已知：  
  每天导出前一天报警记录，文件名按日期命名。
- 公式：  
  **文件名 = Alarm + 日期**
- 计算：  
  2026-05-05的报警文件可命名为Alarm_2026-05-05.csv。
- 结论：  
  文件命名规则清晰，后期查询更方便。

> [!tip] 记忆口诀
> 自动导出要带日期，不要每天覆盖同一个文件。

# 09 数据备份任务

> [!info] 核心结论
> 数据备份任务用于定时备份配方、报警记录、趋势数据、数据库、参数文件和工程配置。

## ① 分类/公式/规则

> [!warning] 备份任务公式
> **可恢复系统 = 原始数据 + 备份文件 + 备份时间 + 恢复验证**

## ② 参考表

| 备份内容 | 说明 |
|---|---|
| **配方备份** | 防止工艺参数丢失 |
| **报警备份** | 保存故障追溯数据 |
| **趋势备份** | 保存过程数据 |
| **数据库备份** | 保护长期记录 |
| **参数备份** | 保存设备设定值 |
| **用户权限备份** | 保护账号和角色 |
| **工程配置备份** | 保护项目文件 |

## ③ 计算/选型示例

- 已知：  
  每天产生100MB记录，需要每天凌晨备份。
- 公式：  
  **每日备份量 = 100MB**
- 计算：  
  一个月约需100MB × 30 = **3GB**备份空间。
- 结论：  
  备份任务要同时规划备份周期和备份存储容量。

> [!tip] 记忆口诀
> 备份不是存一次，而是按周期持续存。

# 10 数据清理任务

> [!info] 核心结论
> 数据清理任务用于定期删除或归档过期记录，防止磁盘、数据库或HMI存储空间被占满。

## ① 分类/公式/规则

> [!warning] 清理规则
> **清理任务 = 保留周期 + 删除范围 + 备份确认 + 权限保护**

## ② 参考表

| 清理对象 | 说明 |
|---|---|
| **过期报警记录** | 删除超过保留期的报警 |
| **过期趋势数据** | 删除或压缩旧趋势 |
| **过期操作日志** | 按保存周期清理 |
| **临时导出文件** | 防止临时目录爆满 |
| **旧报表文件** | 保留最近几个月 |
| **调试日志** | 验收后可清理 |
| **缓存文件** | 网络恢复后可清理 |

## ③ 计算/选型示例

- 已知：  
  每天产生200MB记录，磁盘剩余空间20GB。
- 公式：  
  **可保存天数 = 20000MB ÷ 200MB**
- 计算：  
  可保存约 **100天**
- 结论：  
  如果要求长期运行，应设置90天左右自动清理或归档。

> [!tip] 记忆口诀
> 记录要会存，也要会清；只存不清，迟早磁盘满。

# 11 变量写入任务

> [!info] 核心结论
> 变量写入任务用于按计划自动写入某些HMI或PLC变量，例如班次清零、定时复位、启用标志、统计变量归零。

## ① 分类/公式/规则

> [!warning] 写入安全公式
> **自动写变量 = 写入对象明确 + 权限/条件满足 + PLC允许 + 日志记录**

## ② 参考表

| 写入任务 | 示例 |
|---|---|
| **班次清零** | 班次产量归零 |
| **日报清零** | 日产量计数归零 |
| **状态复位** | 清除HMI内部临时状态 |
| **统计变量更新** | 写入汇总值 |
| **定时开关** | 定时启用或禁用功能 |
| **维护提醒** | 达到时间写入保养标志 |
| **心跳变量** | 定时写入通讯心跳 |

## ③ 计算/选型示例

- 已知：  
  每天0点需要把日产量清零。
- 公式：  
  **0点触发 + 日产量变量写0**
- 计算：  
  计划任务到点执行，将DailyCount写为0。
- 结论：  
  自动清零前应先保存当天记录，防止数据丢失。

> [!tip] 记忆口诀

# 12 脚本执行任务

> [!info] 核心结论
> 脚本执行任务用于按计划运行C/VB脚本或系统函数，实现更复杂的计算、判断、导出、备份和维护逻辑。

## ① 分类/公式/规则

> [!warning] 脚本任务公式
> **计划任务触发 → 执行脚本 → 读写变量/文件/数据库 → 记录结果**

## ② 参考表

| 脚本任务 | 示例 |
|---|---|
| **报表生成脚本** | 读取数据并生成文件 |
| **数据清理脚本** | 删除过期文件 |
| **配方备份脚本** | 导出配方文件 |
| **统计计算脚本** | 计算班次产量、良率 |
| **数据库写入脚本** | 写入汇总数据 |
| **通讯检查脚本** | 检查连接状态 |
| **报警统计脚本** | 统计报警次数和时长 |

## ③ 计算/选型示例

- 已知：  
  每小时需要统计一次设备运行率。
- 公式：  
  **运行率 = 运行时间 ÷ 总时间 × 100%**
- 计算：  
  如果本小时运行45分钟，运行率 = 45 ÷ 60 × 100% = **75%**
- 结论：  
  可用计划任务每小时执行脚本计算并记录运行率。

> [!tip] 记忆口诀
> 简单动作直接配置，复杂动作交给脚本。

# 13 任务触发条件

> [!info] 核心结论
> 计划任务不一定只按时间执行，也可以结合运行状态、用户权限、设备模式、连接状态和变量条件来决定是否执行。

## ① 分类/公式/规则

> [!warning] 条件任务公式
> **任务执行 = 时间条件满足 + 运行条件满足**

## ② 参考表

| 条件 | 示例 |
|---|---|
| **运行模式** | 只有自动模式下执行 |
| **设备状态** | 设备停止时执行备份 |
| **用户状态** | 管理员登录才允许执行 |
| **连接状态** | PLC在线才读取数据 |
| **变量条件** | BatchEnd=1时生成批次报表 |
| **报警条件** | 无严重报警才执行某任务 |
| **磁盘状态** | 空间不足时执行清理 |

## ③ 计算/选型示例

- 已知：  
  配方备份任务只允许在设备停止时执行。
- 公式：  
  **执行条件 = 到达备份时间 + 设备停止**
- 计算：  
  如果到点时设备正在运行，则延后或跳过备份。
- 结论：  
  关键任务应增加运行条件，避免影响生产。

> [!tip] 记忆口诀
> 到时间不一定马上做，还要看现场条件允不允许。

# 14 任务优先级

> [!info] 核心结论
> 多个计划任务同时触发时，应考虑任务优先级，避免报表、备份、清理、采集等任务互相影响。

## ① 分类/公式/规则

> [!warning] 优先级规则
> **安全和记录优先，报表和清理次之，低优先级任务可延后**

## ② 参考表

| 优先级 | 任务 |
|---|---|
| **最高** | 安全相关、报警记录、关键数据保存 |
| **高** | 当前生产记录、批次结束记录 |
| **中** | 报表生成、班次统计 |
| **低** | 历史文件导出、旧数据清理 |
| **可延后** | 大文件备份、非关键统计 |
| **不建议冲突** | 备份与清理同时执行 |
| **需排队** | 大量数据库操作 |

## ③ 计算/选型示例

- 已知：  
  0点同时执行日报生成、数据库备份和旧数据清理。
- 公式：  
  **多个重任务同时执行 → 系统负载升高**
- 计算：  
  可把日报放0:00，备份放0:10，清理放0:30。
- 结论：  
  重任务应错峰执行。

> [!tip] 记忆口诀
> 任务多了要错开，别都挤在同一分钟。

# 15 任务日志

> [!info] 核心结论
> 任务日志用于记录计划任务是否执行、执行时间、执行结果和错误原因，便于排查自动任务失败问题。

## ① 分类/公式/规则

> [!warning] 任务日志公式
> **任务日志 = 任务名称 + 开始时间 + 结束时间 + 执行结果 + 错误信息**

## ② 参考表

| 日志字段 | 说明 |
|---|---|
| **任务名称** | 哪个任务执行了 |
| **开始时间** | 任务开始时间 |
| **结束时间** | 任务结束时间 |
| **执行结果** | 成功、失败、跳过 |
| **错误原因** | 路径错误、权限不足、数据库异常 |
| **执行用户** | 系统或具体用户 |
| **影响对象** | 文件、变量、数据库、报表 |

## ③ 计算/选型示例

- 已知：  
  昨天没有生成日报。
- 公式：  
  **任务异常 → 查询任务日志**
- 计算：  
  查看日报任务是否执行、是否报错、是否路径无权限。
- 结论：  
  计划任务必须保留执行日志，否则失败后很难排查。

> [!tip] 记忆口诀
> 自动任务不等于一定成功，任务日志必须有。

# 16 任务失败处理

> [!info] 核心结论
> 计划任务可能因为通讯中断、数据库异常、路径错误、权限不足、磁盘满等原因失败，应配置失败提示、重试和记录。

## ① 分类/公式/规则

> [!warning] 失败处理公式
> **任务可靠 = 错误检测 + 失败记录 + 重试机制 + 报警提示**

## ② 参考表

| 失败原因 | 处理建议 |
|---|---|
| **PLC离线** | 跳过任务并记录通讯异常 |
| **数据库断开** | 重试或缓存本地 |
| **文件路径错误** | 报警提示并记录日志 |
| **权限不足** | 提示权限配置问题 |
| **磁盘空间不足** | 触发磁盘空间报警 |
| **脚本错误** | 记录错误代码 |
| **任务超时** | 中止任务并报警 |

## ③ 计算/选型示例

- 已知：  
  自动导出任务失败，原因是目标文件夹不存在。
- 公式：  
  **路径不存在 → 导出失败**
- 计算：  
  系统应记录错误，并提示“导出路径无效”。
- 结论：  
  文件类计划任务必须检查路径是否存在。

> [!tip] 记忆口诀
> 任务失败不可怕，怕的是失败了没人知道。

# 17 任务与权限

> [!info] 核心结论
> 计划任务可能涉及删除、清空、导出、写入PLC变量和修改文件，应根据动作风险设置权限和保护。

## ① 分类/公式/规则

> [!warning] 权限规则
> **任务权限 = 动作风险 + 数据重要性 + 用户角色**

## ② 参考表

| 任务动作 | 推荐权限 |
|---|---|
| **查看任务** | 操作员及以上 |
| **手动执行任务** | 班组长或工程师 |
| **修改任务周期** | 工程师 |
| **启用/禁用任务** | 工程师或管理员 |
| **删除任务** | 管理员 |
| **清理历史记录** | 管理员 + 二次确认 |
| **写PLC关键变量** | 工程师 + PLC联锁 |

## ③ 计算/选型示例

- 已知：  
  普通操作员可以修改数据清理任务周期。
- 公式：  
  **清理任务影响历史数据 → 高权限控制**
- 计算：  
  如果误设为每天清理当天数据，会造成记录丢失。
- 结论：  
  清理和删除类任务应限制管理员权限。

> [!tip] 记忆口诀
> 自动任务越能改数据，权限越要严格。

# 18 任务与系统性能

> [!info] 核心结论
> 计划任务会占用CPU、内存、磁盘和数据库资源，任务过多、周期过短或同时执行重任务会导致HMI卡顿。

## ① 分类/公式/规则

> [!warning] 任务负载公式
> **系统负载 = 任务数量 × 执行频率 × 单次任务耗时**

## ② 参考表

| 影响项 | 说明 |
|---|---|
| **任务数量** | 越多管理越复杂 |
| **执行频率** | 越快负载越高 |
| **脚本复杂度** | 复杂计算会占CPU |
| **文件大小** | 大文件导出占磁盘I/O |
| **数据库写入** | 高频写入影响数据库 |
| **同时执行** | 多个任务叠加负载 |
| **网络访问** | 远程路径可能拖慢任务 |

## ③ 计算/选型示例

- 已知：  
  一个脚本任务每秒执行一次，每次耗时0.6秒。
- 公式：  
  **任务占用率 = 0.6 ÷ 1 × 100%**
- 计算：  
  占用率 = **60%**
- 结论：  
  该任务负载很高，应优化脚本或延长周期。

> [!tip] 记忆口诀
> 计划任务不是越密越好，系统跑得稳才重要。

# 19 常见故障排查

> [!info] 核心结论
> 计划任务异常排查按 **启用状态、触发时间、运行条件、脚本动作、路径权限、数据库连接、任务日志、系统负载** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **计划任务异常 = 未启用 + 时间错误 + 条件不满足 + 权限不足 + 路径错误 + 脚本失败**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **任务不执行** | 未启用、时间未到、Runtime未运行 |
| **任务跳过** | 运行条件不满足 |
| **报表未生成** | 模板缺失、路径错误、数据源异常 |
| **导出失败** | 文件夹无权限、磁盘满 |
| **备份失败** | 目标路径不存在、文件被占用 |
| **清理无效** | 清理条件设置错误 |
| **任务重复执行** | 周期设置过短或触发条件重复 |
| **系统卡顿** | 任务太多、脚本太重、同时执行 |

## ③ 计算/选型示例

- 已知：  
  每天0点的备份任务没有执行。
- 公式：  
  **任务未执行 = 未启用 / 时间设置错 / Runtime未运行 / 条件不满足**
- 计算：  
  先检查任务是否启用，再查系统时间和任务日志。
- 结论：  
  定时任务异常时，先查时间和日志。

> [!tip] 记忆口诀
> 任务没跑，先查启用和时间；任务跑错，先查条件和脚本。

# 20 计划任务快速检查表

> [!info] 核心结论
> 计划任务检查的目标是确保任务按时执行、动作正确、失败可追踪、数据不丢失、系统不卡顿。

## ① 分类/公式/规则

> [!note] 十步检查法
> ① 确认任务名称是否清晰  
> ② 确认任务是否启用  
> ③ 检查触发方式是周期、定时还是条件触发  
> ④ 检查系统时间和时区是否正确  
> ⑤ 检查任务执行动作是否正确  
> ⑥ 检查路径、数据库、变量和脚本权限  
> ⑦ 检查任务失败是否有报警或日志  
> ⑧ 检查多个重任务是否错峰执行  
> ⑨ 检查清理、删除、写入类任务是否有权限保护  
> ⑩ 上线前测试正常执行、失败处理、断电重启和磁盘不足场景

> [!warning] 最终检查公式
> **合格计划任务 = 时间正确 + 条件正确 + 动作正确 + 日志完整 + 权限安全 + 性能稳定**

## ② 参考表

| 检查项 | 目标 |
|---|---|
| **触发时间** | 按计划执行 |
| **执行条件** | 不误执行 |
| **动作内容** | 做正确的事 |
| **日志** | 失败能追踪 |
| **权限** | 防止误改误删 |
| **路径** | 文件可保存 |
| **数据库** | 数据可写入 |
| **性能** | 不影响Runtime运行 |

## ③ 计算/选型示例

- 已知：  
  设备需要每天生成日报、每周备份数据库、每月清理过期记录。
- 公式：  
  **计划任务组 = 日报任务 + 周备份任务 + 月清理任务**
- 计算：  
  日报每天00:05执行，备份每周日01:00执行，清理每月1日02:00执行。
- 结论：  
  多个计划任务应错峰配置，并保留执行日志和失败报警。

> [!tip] 选型口诀
> 到点执行看定时，循环执行看周期；报表备份要错峰，删除清理要权限；任务失败要报警，执行结果要留痕。
