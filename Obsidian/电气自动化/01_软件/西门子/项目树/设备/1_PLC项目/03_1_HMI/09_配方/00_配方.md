# 01 配方总览

> [!info] 核心结论
> **配方** 用于管理不同产品、型号、工艺对应的一组参数，实现快速切换产品、减少人工输入错误、保证工艺参数一致。

## ① 分类/公式/规则

> [!warning] 配方公式
> **配方 = 产品编号 + 参数组 + 保存/加载 + 下载到PLC + 当前配方确认**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **配方名称** | 产品A、产品B、型号001等 |
| **配方参数** | 速度、位置、时间、温度、压力、阈值 |
| **配方保存** | 把当前参数保存为一个配方 |
| **配方加载** | 从配方文件或数据库读取参数 |
| **配方下载** | 把配方参数写入PLC |
| **配方上传** | 从PLC读取当前参数到HMI |
| **当前配方** | 当前设备正在使用的产品参数组 |

## ③ 计算/选型示例

- 已知：  
  设备生产5种产品，每种产品有20个参数。
- 公式：  
  **配方参数总数 = 产品数量 × 每种参数数量**
- 计算：  
  5 × 20 = **100个参数值**
- 结论：  
  应使用配方管理，不建议让操作员每次手动输入100个参数。

> [!tip] 记忆口诀
> 产品多、参数多，就用配方；切换快，出错少。

# 02 配方的作用

> [!info] 核心结论
> 配方的核心作用是把一组工艺参数集中管理，避免产品切换时漏改、错改、重复输入。

## ① 分类/公式/规则

> [!warning] 使用规则
> **频繁换型 + 参数较多 + 工艺要求一致 → 必须考虑配方**

## ② 参考表

| 作用 | 说明 |
|---|---|
| **快速换型** | 一键切换产品参数 |
| **减少错误** | 避免人工逐项输入 |
| **工艺一致** | 同一产品使用同一参数组 |
| **参数追溯** | 知道生产时用了哪套参数 |
| **权限控制** | 防止低权限人员随意改配方 |
| **版本管理** | 不同工艺版本可区分 |
| **质量稳定** | 参数一致有利于产品一致性 |

## ③ 计算/选型示例

- 已知：  
  操作员每次换型要手动输入30个参数。
- 公式：  
  **人工输入次数越多 → 出错概率越高**
- 计算：  
  只要其中一个关键参数输入错误，就可能导致产品不良。
- 结论：  
  应把30个参数做成配方，一次加载并下载。

> [!tip] 记忆口诀
> 配方不是为了好看，是为了防止换型出错。

# 03 配方参数

> [!info] 核心结论
> 配方参数是组成配方的具体工艺数据，常见包括速度、位置、时间、温度、压力、检测阈值和产量目标。

## ① 分类/公式/规则

> [!warning] 配方参数公式
> **配方参数 = 参数名称 + 数据类型 + 单位 + 范围 + 默认值**

## ② 参考表

| 参数类型 | 示例 |
|---|---|
| **速度参数** | 输送速度、伺服速度、加速度 |
| **位置参数** | 取料位、放料位、补偿值 |
| **时间参数** | 延时、保压、冷却、超时 |
| **温度参数** | 设定温度、报警温度 |
| **压力参数** | 夹紧压力、真空阈值 |
| **检测参数** | 尺寸上限、尺寸下限、视觉阈值 |
| **计数参数** | 目标产量、批次数量 |

## ③ 计算/选型示例

- 已知：  
  产品A速度300mm/s，产品B速度500mm/s。
- 公式：  
  **不同产品参数不同 → 放入配方**
- 计算：  
  切换到产品B时，速度设定自动变为500mm/s。
- 结论：  
  速度这类产品相关参数适合纳入配方。

> [!tip] 记忆口诀
> 跟产品型号有关的参数，优先考虑放配方。

# 04 配方记录

> [!info] 核心结论
> 一条配方记录通常对应一个产品或工艺版本，里面包含该产品运行所需的全部参数。

## ① 分类/公式/规则

> [!warning] 配方记录公式
> **一条配方记录 = 一个产品/型号 + 一组完整参数**

## ② 参考表

| 字段 | 说明 |
|---|---|
| **配方编号** | 唯一识别配方 |
| **配方名称** | 产品名称或型号 |
| **版本号** | 工艺参数版本 |
| **创建人** | 谁创建了配方 |
| **修改人** | 谁修改了配方 |
| **修改时间** | 最近修改时间 |
| **参数列表** | 该产品所有工艺参数 |

## ③ 计算/选型示例

- 已知：  
  产品A有V1和V2两版工艺参数。
- 公式：  
  **同一产品不同工艺 → 不同配方版本**
- 计算：  
  可建立ProductA_V1和ProductA_V2两条配方。
- 结论：  
  不同版本不要混用，否则追溯困难。

> [!tip] 记忆口诀
> 一个产品可以有多个配方版本，版本要写清楚。

# 05 配方变量

> [!info] 核心结论
> 配方变量是配方与HMI变量、PLC变量之间的桥梁，配方参数最终要通过变量写入PLC或控制器。

## ① 分类/公式/规则

> [!warning] 配方变量公式
> **配方参数 → HMI配方变量 → PLC设定变量**

## ② 参考表

| 配方变量 | 用途 |
|---|---|
| **RecipeName** | 当前配方名称 |
| **RecipeNo** | 当前配方编号 |
| **Speed_SP** | 速度设定 |
| **Position_SP** | 位置设定 |
| **Temp_SP** | 温度设定 |
| **Pressure_SP** | 压力设定 |
| **Limit_High/Low** | 检测上下限 |

## ③ 计算/选型示例

- 已知：  
  配方中有一个温度设定值Temp_SP。
- 公式：  
  **配方Temp_SP → PLC Temp_SP变量**
- 计算：  
  下载配方时，HMI把配方中的温度设定写入PLC对应变量。
- 结论：  
  配方变量必须与PLC设定变量一一对应。

> [!tip] 记忆口诀
> 配方存在HMI里，真正生效要写到PLC里。

# 06 配方保存

> [!info] 核心结论
> 配方保存是把当前参数值保存为一条配方记录，通常需要权限、确认和参数校验。

## ① 分类/公式/规则

> [!warning] 保存规则
> **配方可保存 = 权限满足 + 参数合法 + 配方名唯一 + 用户确认**

## ② 参考表

| 保存项 | 说明 |
|---|---|
| **保存当前参数** | 把当前HMI/PLC参数保存到配方 |
| **另存为** | 保存为新配方 |
| **覆盖保存** | 覆盖已有配方 |
| **参数校验** | 检查范围和数据类型 |
| **权限判断** | 工程师或管理员权限 |
| **保存时间** | 记录保存时间 |
| **保存用户** | 记录谁保存 |

## ③ 计算/选型示例

- 已知：  
  当前压力设定允许范围0.2~0.8MPa，操作员输入1.0MPa后保存配方。
- 公式：  
  **1.0 > 0.8 → 参数非法**
- 计算：  
  保存前应拒绝，并提示压力设定超限。
- 结论：  
  配方保存前必须做参数范围校验。

> [!tip] 记忆口诀
> 配方保存前先校验，不能把错误参数保存成标准。

# 07 配方加载

> [!info] 核心结论
> 配方加载是从配方库中读取某条配方参数到HMI，加载不等于已经写入PLC。

## ① 分类/公式/规则

> [!warning] 加载规则
> **加载配方 ≠ 配方已生效**  
> **生效通常还需要下载到PLC**

## ② 参考表

| 加载动作 | 说明 |
|---|---|
| **选择配方** | 从列表中选中目标配方 |
| **读取参数** | 把配方参数读到HMI显示区 |
| **显示预览** | 操作员确认参数 |
| **权限检查** | 限制谁能加载 |
| **差异检查** | 与当前参数比较 |
| **等待下载** | 需要进一步写入PLC |
| **加载记录** | 记录谁加载了配方 |

## ③ 计算/选型示例

- 已知：  
  操作员在HMI上选择了产品B配方，但没有点击下载。
- 公式：  
  **只加载不下载 → PLC仍可能使用旧参数**
- 计算：  
  HMI显示区变成产品B参数，但PLC内部参数可能没更新。
- 结论：  
  配方加载后必须确认是否已下载到PLC。

> [!tip] 记忆口诀
> 选中配方只是拿出来看，下载到PLC才算真正使用。

# 08 配方下载

> [!info] 核心结论
> 配方下载是把选中的配方参数写入PLC或控制器，是产品切换真正生效的关键步骤。

## ① 分类/公式/规则

> [!warning] 下载公式
> **配方生效 = 选择配方 + 下载到PLC + PLC确认接收**

## ② 参考表

| 下载项 | 说明 |
|---|---|
| **目标PLC** | 参数写入的控制器 |
| **目标变量** | PLC中的设定参数 |
| **下载确认** | 防止误下载 |
| **权限判断** | 防止低权限人员切换 |
| **写入完成** | HMI完成变量写入 |
| **PLC确认** | PLC反馈配方已接收 |
| **当前配方号** | PLC内当前运行配方编号 |

## ③ 计算/选型示例

- 已知：  
  产品A切换到产品B，需要写入20个参数。
- 公式：  
  **下载参数数量 = 20**
- 计算：  
  HMI需要将产品B的20个参数逐项写入PLC。
- 结论：  
  下载完成后应读取PLC当前配方号确认是否成功。

> [!tip] 记忆口诀
> 配方下载要有反馈，不能只看按钮点没点。

# 09 配方上传

> [!info] 核心结论
> 配方上传是从PLC读取当前运行参数到HMI或配方库，常用于保存现场调试后的参数。

## ① 分类/公式/规则

> [!warning] 上传公式
> **PLC当前参数 → HMI配方显示区 → 保存为配方**

## ② 参考表

| 上传用途 | 说明 |
|---|---|
| **读取现场参数** | 从PLC读取当前实际设定 |
| **调试后保存** | 把调试好的参数保存为配方 |
| **参数备份** | 防止PLC参数丢失 |
| **配方比对** | 与库中配方比较 |
| **版本更新** | 保存为新版本 |
| **工程师操作** | 通常需要较高权限 |
| **上传记录** | 记录上传时间和用户 |

## ③ 计算/选型示例

- 已知：  
  工程师现场调试后把速度从300改到320，效果更好。
- 公式：  
  **现场优化参数 → 上传保存新配方**
- 计算：  
  从PLC上传当前参数，并保存为ProductA_V2。
- 结论：  
  现场调试后的有效参数应及时上传并保存。

> [!tip] 记忆口诀
> 下载是HMI给PLC，上传是PLC给HMI。

# 10 当前配方确认

> [!info] 核心结论
> 当前配方确认用于判断PLC当前实际运行的配方是否与HMI选择的配方一致。

## ① 分类/公式/规则

> [!warning] 确认公式
> **当前配方一致 = HMI选择配方号 = PLC当前配方号**

## ② 参考表

| 确认项 | 说明 |
|---|---|
| **HMI选择配方** | 操作员当前选中的配方 |
| **PLC当前配方** | PLC实际使用的配方编号 |
| **下载完成位** | PLC反馈已接收参数 |
| **参数校验位** | PLC确认参数合法 |
| **配方一致提示** | 显示当前配方正常 |
| **配方不一致报警** | 防止错料生产 |
| **产品型号显示** | 给操作员明确提示 |

## ③ 计算/选型示例

- 已知：  
  HMI选择配方号为5，PLC当前配方号为4。
- 公式：  
  **5 ≠ 4 → 配方不一致**
- 计算：  
  设备可能仍按旧产品参数运行。
- 结论：  
  应提示“配方未下载或下载失败”，禁止直接生产。

> [!tip] 记忆口诀
> HMI选了不算，PLC确认了才算。

# 11 配方权限

> [!info] 核心结论
> 配方操作会直接影响设备工艺和产品质量，因此必须根据操作类型设置权限。

## ① 分类/公式/规则

> [!warning] 权限公式
> **配方操作允许 = 当前用户角色 + 操作类型 + 设备状态**

## ② 参考表

| 操作 | 推荐权限 |
|---|---|
| **查看配方** | 操作员及以上 |
| **加载配方** | 操作员或班组长 |
| **下载配方** | 班组长或工程师 |
| **保存配方** | 工程师 |
| **覆盖配方** | 工程师或管理员 |
| **删除配方** | 管理员 |
| **修改关键参数** | 工程师或管理员 |

## ③ 计算/选型示例

- 已知：  
  普通操作员可以删除配方。
- 公式：  
  **删除配方 = 高风险操作**
- 计算：  
  删除后可能无法恢复产品工艺参数。
- 结论：  
  删除配方应限制管理员权限，并加二次确认。

> [!tip] 记忆口诀
> 看配方权限低，改配方权限高，删配方最严格。

# 12 配方参数范围

> [!info] 核心结论
> 配方参数必须设置上下限，防止错误参数被保存、下载或运行。

## ① 分类/公式/规则

> [!warning] 范围公式
> **参数合法 = 最小值 ≤ 参数值 ≤ 最大值**

## ② 参考表

| 参数 | 范围示例 |
|---|---|
| **速度** | 100~800mm/s |
| **温度** | 20~180℃ |
| **压力** | 0.2~0.8MPa |
| **延时** | 0.1~10s |
| **位置补偿** | -5~5mm |
| **真空阈值** | -80~-30kPa |
| **检测上限** | 必须大于检测下限 |

## ③ 计算/选型示例

- 已知：  
  检测下限为10mm，检测上限为8mm。
- 公式：  
  **上限必须 > 下限**
- 计算：  
  8 < 10，参数逻辑错误。
- 结论：  
  配方不仅要检查单个范围，还要检查参数之间的逻辑关系。

> [!tip] 记忆口诀
> 参数要看上下限，也要看互相关系。

# 13 配方与产品切换

> [!info] 核心结论
> 产品切换时，配方必须与物料、工装、程序、视觉参数和PLC运行状态匹配，不能只切换HMI显示。

## ① 分类/公式/规则

> [!warning] 换型公式
> **安全换型 = 停机状态 + 选择配方 + 下载参数 + 确认工装/物料 + 试运行确认**

## ② 参考表

| 换型检查项 | 说明 |
|---|---|
| **设备状态** | 是否允许切换 |
| **产品编号** | 是否选对产品 |
| **工装型号** | 是否与产品匹配 |
| **物料批次** | 是否正确 |
| **PLC配方号** | 是否下载成功 |
| **视觉程序** | 是否切换一致 |
| **首件确认** | 切换后验证产品质量 |

## ③ 计算/选型示例

- 已知：  
  HMI切换到产品B，但现场工装仍是产品A工装。
- 公式：  
  **配方与工装不一致 → 换型风险**
- 计算：  
  即使参数正确，工装不匹配仍可能导致撞机或不良。
- 结论：  
  配方切换应联动工装确认和首件确认。

> [!tip] 记忆口诀
> 换配方不等于换型完成，物料、工装、程序都要对。

# 14 配方与报警

> [!info] 核心结论
> 配方相关报警用于提示配方未下载、配方不一致、参数超限、下载失败、配方文件缺失等问题。

## ① 分类/公式/规则

> [!warning] 配方报警公式
> **配方异常 = 未选择 + 未下载 + 不一致 + 参数非法 + 文件异常**

## ② 参考表

| 报警 | 含义 |
|---|---|
| **未选择配方** | 当前无有效配方 |
| **配方未下载** | HMI选择但未写入PLC |
| **配方不一致** | HMI与PLC配方号不同 |
| **参数超限** | 配方参数不在允许范围 |
| **下载失败** | 参数写入PLC失败 |
| **配方文件缺失** | 找不到配方记录 |
| **配方版本错误** | 版本与设备程序不匹配 |

## ③ 计算/选型示例

- 已知：  
  PLC当前配方号为0，表示无有效配方。
- 公式：  
  **当前配方号=0 → 未下载有效配方**
- 计算：  
  设备应禁止自动运行或提示选择配方。
- 结论：  
  没有有效配方时，不应允许直接生产。

> [!tip] 记忆口诀
> 配方错误不是小问题，可能直接导致产品不良。

# 15 配方与数据库

> [!info] 核心结论
> 配方可以保存在HMI本地文件、数据库或服务器中；大型项目更适合使用数据库集中管理。

## ① 分类/公式/规则

> [!warning] 存储选择规则
> **单机小项目 → 本地配方文件**  
> **多设备、多用户、需追溯 → 数据库配方**

## ② 参考表

| 存储方式 | 特点 |
|---|---|
| **HMI本地文件** | 简单、易部署 |
| **CSV/Excel文件** | 易导入导出 |
| **SQLite** | 单机本地数据库 |
| **SQL Server/MySQL** | 多设备集中管理 |
| **MES配方** | 与生产计划联动 |
| **云端配方** | 远程维护和统一发布 |
| **PLC存储** | 关键参数可保留在PLC内 |

## ③ 计算/选型示例

- 已知：  
  10台设备都要使用同一套产品配方。
- 公式：  
  **多设备共享配方 → 数据库或服务器管理**
- 计算：  
  如果每台HMI单独维护，容易版本不一致。
- 结论：  
  多设备项目建议使用集中式配方管理。

> [!tip] 记忆口诀
> 一台设备本地配方，多台设备集中配方。

# 16 配方备份与恢复

> [!info] 核心结论
> 配方是关键工艺资产，必须定期备份，并支持设备更换、HMI损坏或误删后的恢复。

## ① 分类/公式/规则

> [!warning] 备份公式
> **可恢复配方 = 配方文件 + 参数说明 + 版本记录 + 备份日期**

## ② 参考表

| 备份内容 | 说明 |
|---|---|
| **配方文件** | 实际参数数据 |
| **配方数据库** | 集中存储时备份数据库 |
| **参数说明** | 每个参数含义、单位、范围 |
| **版本记录** | 配方修改历史 |
| **导出文件** | CSV、Excel等 |
| **恢复流程** | 换屏或换机后如何恢复 |
| **权限备份** | 保护配方修改权限 |

## ③ 计算/选型示例

- 已知：  
  HMI损坏后没有配方备份。
- 公式：  
  **无配方备份 → 工艺恢复困难**
- 计算：  
  只能靠人工回忆或重新调试参数，风险很高。
- 结论：  
  配方文件必须随工程一起备份交付。

> [!tip] 记忆口诀
> 程序要备份，配方更要备份。

# 17 配方版本管理

> [!info] 核心结论
> 配方版本管理用于记录配方修改历史，防止现场使用旧参数或无法追溯参数变化。

## ① 分类/公式/规则

> [!warning] 版本公式
> **配方版本 = 产品编号 + 版本号 + 修改内容 + 修改人 + 修改时间**

## ② 参考表

| 版本字段 | 说明 |
|---|---|
| **版本号** | V1.0、V1.1、V2.0 |
| **修改人** | 谁修改了配方 |
| **修改时间** | 什么时候修改 |
| **修改原因** | 为什么修改 |
| **旧值新值** | 参数变化内容 |
| **审核人** | 谁确认版本 |
| **启用状态** | 当前是否允许使用 |

## ③ 计算/选型示例

- 已知：  
  产品A配方速度从300改到350，但没有记录原因。
- 公式：  
  **无修改记录 → 质量问题难追溯**
- 计算：  
  后续产品异常时无法判断参数变化是否相关。
- 结论：  
  关键配方修改必须记录旧值、新值和修改原因。

> [!tip] 记忆口诀
> 配方改过不可怕，怕的是不知道谁改了什么。

# 18 配方导入导出

> [!info] 核心结论
> 配方导入导出用于批量维护、备份、迁移和离线编辑配方，但必须保持字段、单位和数据类型一致。

## ① 分类/公式/规则

> [!warning] 导入导出规则
> **可导入配方 = 字段正确 + 类型正确 + 单位一致 + 参数范围合法**

## ② 参考表

| 操作 | 注意 |
|---|---|
| **导出配方** | 用于备份或离线编辑 |
| **导入配方** | 导入前校验格式 |
| **Excel维护** | 防止单元格格式变更 |
| **CSV维护** | 注意编码和分隔符 |
| **字段映射** | 参数名必须匹配 |
| **单位一致** | mm、m、s、ms不能混用 |
| **导入日志** | 记录导入人和时间 |

## ③ 计算/选型示例

- 已知：  
  Excel中把时间参数单位从秒改成毫秒，但HMI仍按秒解释。
- 公式：  
  **单位不一致 → 参数错误**
- 计算：  
  500ms如果按500s写入，会造成严重异常。
- 结论：  
  配方导入必须校验单位和范围。

> [!tip] 记忆口诀
> 配方表能导入，不代表数据一定安全。

# 19 配方常见故障排查

> [!info] 核心结论
> 配方故障排查按 **配方选择、权限、参数范围、变量绑定、下载状态、PLC确认、文件/数据库、版本** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **配方异常 = 未选择 + 权限不足 + 参数非法 + 变量错绑 + 下载失败 + PLC未确认 + 文件缺失**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **配方列表为空** | 配方文件缺失、数据库连接异常 |
| **无法保存配方** | 权限不足、文件只读、参数非法 |
| **无法下载配方** | PLC连接异常、变量地址错 |
| **下载后不生效** | PLC未接收、当前配方号未更新 |
| **参数显示错误** | 数据类型错、倍率错、单位错 |
| **配方不一致** | HMI选择与PLC当前不同 |
| **切换产品异常** | 工装、物料、程序未同步 |
| **历史配方丢失** | 未备份或误删除 |

## ③ 计算/选型示例

- 已知：  
  配方下载后，设备仍按旧速度运行。
- 公式：  
  **下载按钮点击 ≠ PLC参数已更新**
- 计算：  
  可能变量地址错、PLC未接收、配方号未确认。
- 结论：  
  应检查PLC设定变量和当前配方确认位。

> [!tip] 记忆口诀
> 配方不生效，先查有没有真正写到PLC。

# 20 配方快速检查表

> [!info] 核心结论
> 配方检查的目标是确保配方能正确保存、加载、下载、确认、追溯、备份，并防止错误参数进入生产。

## ① 分类/公式/规则

> [!note] 十步检查法
> ① 检查配方名称和编号是否唯一  
> ② 检查配方参数是否完整  
> ③ 检查参数单位、范围和数据类型  
> ④ 检查保存、加载、下载、上传功能  
> ⑤ 检查HMI变量与PLC变量绑定是否正确  
> ⑥ 检查下载后PLC当前配方号是否更新  
> ⑦ 检查配方权限和二次确认  
> ⑧ 检查配方修改日志和版本记录  
> ⑨ 检查配方文件/数据库备份  
> ⑩ 上线前测试换型、断电、重启和异常恢复

> [!warning] 最终检查公式
> **合格配方系统 = 参数完整 + 范围安全 + 下载可靠 + PLC确认 + 权限清楚 + 记录可追溯**

## ② 参考表

| 检查项 | 目标 |
|---|---|
| **配方名称** | 不重复、不混乱 |
| **参数范围** | 防止错误参数 |
| **变量绑定** | 保证写入正确PLC地址 |
| **下载确认** | 确认PLC真正使用 |
| **权限** | 防止误改误删 |
| **版本** | 参数变化可追溯 |
| **备份** | 配方丢失可恢复 |
| **换型测试** | 产品切换可靠 |

## ③ 计算/选型示例

- 已知：  
  一台设备有8种产品，每种产品包含速度、位置、温度、压力、检测上下限等40个参数。
- 公式：  
  **配方总参数值 = 8 × 40**
- 计算：  
  总参数值 = **320个**
- 结论：  
  必须使用配方管理，并建立权限、版本、备份和下载确认机制。

> [!tip] 选型口诀
> 配方先命名，参数要完整；保存要校验，下载要确认；修改留日志，删除要权限；换型看一致，备份不能少。
