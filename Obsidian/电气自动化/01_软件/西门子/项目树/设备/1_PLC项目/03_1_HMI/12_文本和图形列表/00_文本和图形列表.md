# 01 文本和图形列表总览

> [!info] 核心结论
> **文本和图形列表** 用于把PLC或HMI变量的数值状态转换成对应的 **文字、图标、图片或状态显示**。  
> 简单理解：PLC给的是数字，HMI通过列表把数字翻译成人能看懂的文字或图形。

## ① 分类/公式/规则

> [!warning] 显示映射公式
> **变量值 → 文本列表/图形列表 → 画面显示内容**

## ② 参考表

| 类型 | 作用 |
|---|---|
| **文本列表** | 把数值转换为文字，例如0=停止，1=运行 |
| **图形列表** | 把数值转换为图片，例如0=灰色电机，1=绿色电机 |
| **状态文本** | 显示设备模式、状态、报警说明 |
| **状态图标** | 显示阀门、电机、泵、传感器状态 |
| **符号I/O域** | 常调用文本列表显示文字状态 |
| **图形I/O域** | 常调用图形列表显示图片状态 |
| **多语言文本** | 文本列表可配合语言资源实现多语言 |

## ③ 计算/选型示例

- 已知：  
  PLC状态变量State=2。
- 公式：  
  **State=2 → 文本列表映射 → 报警**
- 计算：  
  HMI画面显示“报警”。
- 结论：  
  文本列表能把PLC状态数字转换为现场人员能理解的文字。

> [!tip] 记忆口诀
> PLC用数字表达状态，HMI用文本和图形让人看懂状态。

# 04 文本列表 vs 图形列表

> [!info] 核心结论
> 文本列表适合表达明确含义，图形列表适合快速识别状态；复杂画面中两者经常配合使用。

## ① 分类/公式/规则

> [!warning] 选择规则
> **需要详细说明 → 文本列表**  
> **需要快速识别 → 图形列表**  
> **重要状态 → 文本 + 图形同时显示**

## ② 参考表

| 对比项 | 文本列表 | 图形列表 |
|---|---|---|
| 表达方式 | 文字 | 图片/图标 |
| 优点 | 含义清楚 | 直观醒目 |
| 缺点 | 阅读速度较慢 | 需要图标含义统一 |
| 常用对象 | 状态文本、模式显示 | 电机、阀门、报警图标 |
| 多语言 | 需要翻译 | 图形通常不需要翻译 |
| 适合场景 | 参数页、报警页 | 总览页、流程图 |

## ③ 计算/选型示例

- 已知：  
  总览画面需要显示阀门状态。
- 公式：  
  **快速识别状态 → 图形列表优先**
- 计算：  
  阀门开显示打开图标，阀门关显示关闭图标，故障显示红色图标。
- 结论：  
  总览画面适合用图形列表，详情页可同时显示文本说明。

> [!tip] 记忆口诀
> 总览看图标，详情看文字。

# 05 状态变量映射

> [!info] 核心结论
> 文本和图形列表通常由一个状态变量驱动，该变量可以来自PLC、HMI内部变量、脚本计算或系统变量。

## ① 分类/公式/规则

> [!warning] 状态映射公式
> **状态变量值 = 列表索引 / 映射值**

## ② 参考表

| 状态变量来源 | 示例 |
|---|---|
| **PLC状态字** | 设备状态、模式状态 |
| **PLC故障码** | 伺服报警码、变频器故障码 |
| **HMI内部变量** | 当前页面状态、语言状态 |
| **脚本计算结果** | 综合设备状态 |
| **系统变量** | 当前用户、当前语言 |
| **通讯状态变量** | 在线、离线、异常 |
| **配方状态变量** | 未选择、已加载、已下载 |

## ③ 计算/选型示例

- 已知：  
  设备状态变量State：0停止，1运行，2报警。
- 公式：  
  **State → 文本列表/图形列表**
- 计算：  
  State=2时，文本显示“报警”，图形显示红色报警图标。
- 结论：  
  一个状态变量可以同时驱动文本和图形显示。

> [!tip] 记忆口诀
> 状态变量是源头，文本图形是表现。

# 06 默认项与未知状态

> [!info] 核心结论
> 文本和图形列表必须设置默认项或未知状态，防止变量值超出定义范围时画面空白或误导操作员。

## ① 分类/公式/规则

> [!warning] 默认项规则
> **变量值无匹配项 → 显示未知/异常/未定义**

## ② 参考表

| 异常情况 | 推荐显示 |
|---|---|
| **变量值未定义** | 未知状态 |
| **通讯中断** | 通讯异常 |
| **状态码超范围** | 未定义状态 |
| **图片缺失** | 默认故障图标 |
| **文本缺失** | 未配置文本 |
| **状态冲突** | 状态异常 |
| **变量质量坏** | 数据无效 |

## ③ 计算/选型示例

- 已知：  
  文本列表只定义0~5，但变量值为9。
- 公式：  
  **9无匹配项 → 默认显示未知状态**
- 计算：  
  HMI显示“未知状态”，而不是空白。
- 结论：  
  列表必须处理未定义值。

> [!tip] 记忆口诀
> 列表要有默认项，异常状态不能空着。

# 07 文本列表多语言

> [!info] 核心结论
> 文本列表中的文字应纳入语言资源管理，方便中文、英文或其他语言切换。

## ① 分类/公式/规则

> [!warning] 多语言公式
> **文本项 = 文本ID + 当前运行语言对应文本**

## ② 参考表

| 中文 | 英文 |
|---|---|
| **停止** | Stop |
| **运行** | Running |
| **待机** | Standby |
| **报警** | Alarm |
| **手动** | Manual |
| **自动** | Auto |
| **未知状态** | Unknown |

## ③ 计算/选型示例

- 已知：  
  文本列表中状态1为“运行”。
- 公式：  
  **运行语言=英文 → 显示Running**
- 计算：  
  切换英文后，状态1不应继续显示中文“运行”。
- 结论：  
  多语言项目中，文本列表必须接入语言资源。

> [!tip] 记忆口诀
> 列表文字也要翻译，不能只翻译画面按钮。

# 08 图形列表资源管理

> [!info] 核心结论
> 图形列表依赖图片资源，必须保证图标风格统一、文件存在、尺寸合适、含义清楚。

## ① 分类/公式/规则

> [!warning] 图形资源规则
> **图形列表可用 = 图片存在 + 路径正确 + 尺寸合适 + 状态含义统一**

## ② 参考表

| 图形资源项 | 注意 |
|---|---|
| **运行图标** | 建议统一绿色或动态样式 |
| **停止图标** | 建议灰色 |
| **报警图标** | 建议红色或警告符号 |
| **维护图标** | 可用扳手或蓝色提示 |
| **未知图标** | 可用问号或灰色异常图 |
| **图片尺寸** | 避免过大导致画面卡顿 |
| **图片命名** | 按状态和对象命名 |

## ③ 计算/选型示例

- 已知：  
  电机图形列表使用4K图片作为图标。
- 公式：  
  **图片过大 → 工程变大 + 加载变慢**
- 计算：  
  小尺寸HMI图标不需要高分辨率原图。
- 结论：  
  图形列表图片应按画面实际尺寸优化。

> [!tip] 记忆口诀
> 图形列表图标要小而清楚，不是越大越好。

# 15 列表索引与变量值

> [!info] 核心结论
> 配置文本或图形列表时，要确认变量值与列表索引是否一一对应，避免显示错位。

## ① 分类/公式/规则

> [!warning] 映射正确公式
> **变量值 = 正确的列表项编号**

## ② 参考表

| 常见错误 | 后果 |
|---|---|
| **索引从0开始但按1配置** | 所有状态错位 |
| **缺少某个状态值** | 显示空白或未知 |
| **变量值超范围** | 找不到对应项 |
| **PLC状态定义改了** | HMI显示不再匹配 |
| **列表顺序被改动** | 图文显示错乱 |
| **默认项未设置** | 异常状态无提示 |
| **多语言项漏翻** | 切换语言后显示异常 |

## ③ 计算/选型示例

- 已知：  
  PLC定义0=停止，1=运行，但HMI列表配置1=停止，2=运行。
- 公式：  
  **PLC定义与HMI列表不一致 → 显示错位**
- 计算：  
  PLC发1时，HMI显示停止而不是运行。
- 结论：  
  配置列表前必须确认PLC状态定义。

> [!tip] 记忆口诀
> 列表不是随便排，必须跟PLC状态表一致。

  工程师不知道它用于电机状态、报警等级还是运行模式。
- 结论：  
  应命名为能表达用途的名称，例如MotorState_TextList。

> [!tip] 记忆口诀
> 列表名字要说明它翻译的是谁的状态。

# 17 列表复用

> [!info] 核心结论
> 同类设备或状态应尽量复用同一个文本或图形列表，避免多个列表内容不一致。

## ① 分类/公式/规则

> [!warning] 复用规则
> **同一类状态 = 同一套列表**

## ② 参考表

| 可复用列表 | 应用 |
|---|---|
| **电机状态列表** | 多个电机对象 |
| **阀门状态列表** | 多个阀门对象 |
| **报警等级列表** | 报警页面、状态栏 |
| **通讯状态列表** | PLC、网关、服务器连接 |
| **运行模式列表** | 首页、状态栏、参数页 |
| **配方状态列表** | 配方画面、自动运行画面 |
| **用户等级列表** | 权限画面、状态栏 |

## ③ 计算/选型示例

- 已知：  
  工程中有20个电机，状态定义相同。
- 公式：  
  **20个电机 → 复用1套电机状态列表**
- 计算：  
  所有电机都调用同一个MotorState_TextList和GraphicList。
- 结论：  
  复用列表能保证显示统一，后期修改也方便。

> [!tip] 记忆口诀
> 状态定义一样，列表就不要重复建。

# 18 常见故障排查

> [!info] 核心结论
> 文本和图形列表异常排查按 **变量值、列表映射、默认项、图形资源、多语言文本、控件绑定** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **列表异常 = 变量值异常 + 映射错误 + 文本缺失 + 图片缺失 + 控件未绑定**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **显示空白** | 没有对应列表项、文本为空 |
| **显示错误文字** | 变量值与列表映射错位 |
| **图标不显示** | 图片资源缺失或路径错误 |
| **图标显示不对** | 图形列表项配置错误 |
| **切语言后仍是中文** | 文本列表未接入语言资源 |
| **状态不变化** | 变量未刷新或控件未绑定 |
| **显示未知状态** | 变量值超出定义范围 |
| **画面卡顿** | 图形资源过大或图标过多 |

## ③ 计算/选型示例

- 已知：  
  设备正在运行，但HMI显示“停止”。
- 公式：  
  **显示错 = 变量值错 / 列表映射错 / 状态定义不一致**
- 计算：  
  先在线监控变量值，再检查列表中该值对应的文本。
- 结论：  
  列表显示异常时，先确认变量值是否正确。

> [!tip] 记忆口诀
> 列表显示错，先看变量值，再看映射表。

# 19 文本和图形列表快速检查表

> [!info] 核心结论
> 文本和图形列表检查的目标是确保变量值能准确转换为文字或图形，显示清楚、统一、可维护。

## ① 分类/公式/规则

> [!note] 十步检查法
> ① 确认列表用于文本还是图形  
> ② 确认变量值定义来自PLC还是HMI  
> ③ 检查每个变量值是否有对应项  
> ④ 检查默认项或未知状态是否配置  
> ⑤ 检查文本是否清楚、统一、可翻译  
> ⑥ 检查图形资源是否存在、大小合适  
> ⑦ 检查控件是否绑定正确列表  
> ⑧ 检查多语言切换是否正常  
> ⑨ 检查多个画面是否复用统一列表  
> ⑩ 上线前测试所有状态值和异常值

> [!warning] 最终检查公式
> **合格列表 = 变量值正确 + 映射正确 + 文本清楚 + 图形完整 + 默认项可靠**

## ② 参考表

| 检查项 | 目标 |
|---|---|
| **变量值** | 状态来源正确 |
| **映射关系** | 不错位 |
| **文本项** | 人能看懂 |
| **图形项** | 一眼识别 |
| **默认项** | 异常有提示 |
| **多语言** | 切换正常 |
| **复用** | 显示统一 |
| **资源** | 图片不丢失 |

## ③ 计算/选型示例

- 已知：  
  设备状态有停止、运行、待机、报警、维护5种状态。
- 公式：  
  **5种状态 → 至少5个列表项 + 1个默认项**
- 计算：  
  建议配置0~4对应状态，再配置“未知状态”作为默认项。
- 结论：  
  列表配置要覆盖正常状态，也要覆盖异常状态。

> [!tip] 选型口诀
> 数字变文字用文本列表，数字变图片用图形列表；状态先定义，列表再映射；默认项要有，资源要完整。
