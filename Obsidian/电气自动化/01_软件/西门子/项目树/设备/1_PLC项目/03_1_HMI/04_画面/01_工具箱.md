# 01 选项 / 对象工具箱总览

> [!info] 核心结论
> “选项 / 对象工具箱”是HMI、SCADA或WinCC画面编辑器中用于插入画面对象的工具面板，常包含 **基本对象、元素、控件、图形库、文本、按钮、输入输出域、报警、趋势、用户管理、图形资源** 等内容。

## ① 分类/公式/规则

> [!warning] 使用公式
> **画面对象 = 图形外观 + 变量绑定 + 属性设置 + 动作事件 + 权限控制**

## ② 参考表

| 类别 | 作用 |
|---|---|
| **基本对象** | 线条、圆形、矩形、文本、图片等基础绘图对象 |
| **元素** | 数值显示、按钮、输入域、图形列表、状态显示等常用HMI对象 |
| **控件** | 报警控件、趋势控件、用户管理、网页/诊断等高级对象 |
| **图形库** | 系统图形、用户图形、设备图标、符号库 |
| **属性设置** | 颜色、位置、大小、边框、字体、可见性 |
| **事件设置** | 点击、按下、释放、打开画面、变量变化 |
| **动态设置** | 通过变量控制颜色、闪烁、显示、移动、数值 |

## ③ 计算/选型示例

- 已知：  
  画面需要显示电机运行状态，并允许点击进入电机详情页。
- 公式：  
  **图形对象 + 状态变量 + 跳转事件 = 可交互设备图标**
- 计算：  
  电机图标绑定运行/报警变量，点击事件设置为跳转详情画面。
- 结论：  
  画面对象不是单纯图形，而是图形、变量和动作的组合。

> [!tip] 记忆口诀
> 工具箱负责“放对象”，属性负责“调样式”，变量负责“动起来”。

# 02 基本对象

> [!info] 核心结论
> 基本对象用于绘制画面基础结构，例如线条、圆形、矩形、文本和图片，是HMI画面布局和静态图形的基础。

## ① 分类/公式/规则

> [!warning] 基本对象公式
> **基础画面 = 线条 + 形状 + 文本 + 图片 + 颜色布局**

## ② 参考表

| 基本对象 | 用途 |
|---|---|
| **直线** | 分隔区域、流程线、指示线 |
| **圆形/椭圆** | 指示灯、状态点、圆形按钮 |
| **矩形** | 区域背景、设备外框、按钮底图 |
| **文本** | 标题、标签、单位、说明 |
| **图片** | 设备图、工艺图、LOGO |
| **多边形** | 特殊图形、箭头、设备外形 |
| **组合对象** | 多个基础对象组合成设备图标 |

## ③ 计算/选型示例

- 已知：  
  需要在画面上画一个设备区域，并标注“输送线1”。
- 公式：  
  **区域显示 = 矩形背景 + 文本标签**
- 计算：  
  用矩形作为区域底图，用文本显示设备名称。
- 结论：  
  简单结构优先用基本对象，不一定需要复杂控件。

> [!tip] 记忆口诀
> 基本对象负责画外形，动态对象负责显示状态。

# 03 线条对象

> [!info] 核心结论
> 线条对象常用于流程连接、区域分隔、管道示意、信号路径和设备结构线。

## ① 分类/公式/规则

> [!warning] 线条设计规则
> **线条用途 = 连接关系 + 流向提示 + 区域分隔**

## ② 参考表

| 设置项 | 说明 |
|---|---|
| **线宽** | 控制线条粗细 |
| **颜色** | 区分状态或区域 |
| **线型** | 实线、虚线、点划线 |
| **箭头** | 表示流向或运动方向 |
| **动态颜色** | 通过变量显示运行/报警状态 |
| **层级** | 避免被背景遮挡 |
| **对齐** | 保持画面整齐 |

## ③ 计算/选型示例

- 已知：  
  管道正常运行显示绿色，报警显示红色。
- 公式：  
  **线条颜色 = 设备状态变量**
- 计算：  
  当运行位ON时显示绿色，报警位ON时显示红色。
- 结论：  
  线条也可以动态化，用于显示工艺流向和状态。

> [!tip] 记忆口诀
> 线条不只是装饰，也可以表达流程和状态。

## ③ 计算/选型示例

# 05 文本对象

> [!info] 核心结论
> 文本对象用于显示标题、标签、单位、按钮文字、状态说明和操作提示，是画面可读性的关键。

## ① 分类/公式/规则

> [!warning] 文本设计规则
> **文本清晰 = 字体合适 + 单位明确 + 颜色统一 + 多语言可管理**

## ② 参考表

| 文本类型 | 示例 |
|---|---|
| **标题文本** | 自动运行、参数设置 |
| **标签文本** | 温度、压力、速度 |
| **单位文本** | ℃、MPa、mm/s |
| **状态文本** | 运行中、待机、报警 |
| **提示文本** | 请关闭安全门 |
| **按钮文本** | 启动、停止、复位 |
| **报警说明** | 气压不足，请检查气源 |

## ③ 计算/选型示例

- 已知：  
  画面显示压力数值，但没有单位。
- 公式：  
  **完整数据显示 = 数值 + 单位**
- 计算：  
  0.5如果不标注单位，无法判断是MPa、bar还是kPa。
- 结论：  
  数值显示旁必须配单位文本。

> [!tip] 记忆口诀
> 没单位的数值，现场人员容易误解。

# 06 图片对象

> [!info] 核心结论
> 图片对象用于显示设备照片、工艺流程图、图标、LOGO和背景图，但图片过大会影响画面加载速度。

## ① 分类/公式/规则

> [!warning] 图片使用规则
> **图片资源 = 清晰够用 + 尺寸合适 + 文件不臃肿 + 风格统一**

## ② 参考表

| 图片类型 | 应用 |
|---|---|
| **设备图片** | 显示真实设备外观 |
| **流程图** | 显示工艺路径 |
| **图标** | 电机、阀门、泵、传感器 |
| **状态图片** | 开/关、运行/停止 |
| **背景图** | 画面装饰或区域底图 |
| **LOGO** | 厂家或项目标识 |
| **帮助图片** | 维护说明、接线提示 |

## ③ 计算/选型示例

- 已知：  
  7寸HMI导入了一张4K设备图片。
- 公式：  
  **图片过大 → 工程变大 + 加载变慢**
- 计算：  
  小屏显示不需要4K原图，应压缩到合适尺寸。
- 结论：  
  图片资源要按屏幕分辨率优化。

> [!tip] 记忆口诀
> HMI图片够清楚就好，不是越大越好。

# 07 元素对象

> [!info] 核心结论
> 元素对象是HMI画面中最常用的交互和显示对象，例如输入输出域、按钮、列表、状态显示、图形列表和棒图。

## ① 分类/公式/规则

> [!warning] 元素对象公式
> **元素对象 = 显示/输入功能 + 变量地址 + 数据类型 + 属性设置**

## ② 参考表

| 元素 | 作用 |
|---|---|
| **I/O域** | 显示或输入数值 |
| **按钮** | 启动、停止、复位、跳转 |
| **符号I/O域** | 用文本显示不同状态 |
| **图形I/O域** | 用图片显示不同状态 |
| **棒图** | 显示液位、压力、进度 |
| **滑块/调节对象** | 调节设定值 |
| **日期时间显示** | 显示系统时间或过程时间 |

## ③ 计算/选型示例

- 已知：  
  需要显示当前温度，并允许工程师修改设定温度。
- 公式：  
  **显示值用输出域，设定值用输入域**
- 计算：  
  当前温度只读，设定温度可写并加权限。
- 结论：  
  同样是数值框，也要区分只读和可写。

> [!tip] 记忆口诀
> 元素对象是画面和PLC变量交互的主要入口。

# 08 I/O域

> [!info] 核心结论
> I/O域用于显示或输入变量数值，是HMI中最常用的数据对象之一。

## ① 分类/公式/规则

> [!warning] I/O域配置公式
> **I/O域可用 = 变量地址 + 数据类型 + 显示格式 + 权限 + 范围限制**

## ② 参考表

| 设置项 | 说明 |
|---|---|
| **变量地址** | PLC或内部变量地址 |
| **数据类型** | Int、DInt、Real、String等 |
| **显示格式** | 小数位、单位、前后缀 |
| **输入范围** | 最小值、最大值 |
| **读写权限** | 只读、可写、工程师可写 |
| **颜色动态** | 超限变色、报警闪烁 |
| **确认方式** | 输入后确认或立即写入 |

## ③ 计算/选型示例

- 已知：  
  压力设定范围为0.2~0.8MPa。
- 公式：  
  **输入值必须满足 0.2 ≤ 设定值 ≤ 0.8**
- 计算：  
  输入1.0MPa应拒绝保存或提示超限。
- 结论：  
  I/O输入域必须配置范围保护。

> [!tip] 记忆口诀
> I/O域能输入，就必须限制范围。

# 09 按钮对象

> [!info] 核心结论
> 按钮对象用于执行启动、停止、复位、页面跳转、参数保存、配方下载等操作。

## ① 分类/公式/规则

> [!warning] 按钮安全公式
> **按钮动作 = 权限允许 + 条件满足 + 事件触发 + PLC确认**

## ② 参考表

| 按钮类型 | 应用 |
|---|---|
| **瞬时按钮** | 点动、启动脉冲 |
| **保持按钮** | 模式开关、启用禁用 |
| **跳转按钮** | 打开页面 |
| **确认按钮** | 保存、下载、执行 |
| **复位按钮** | 报警复位、计数清零 |
| **弹窗按钮** | 打开确认窗口 |
| **带状态按钮** | 显示当前ON/OFF状态 |

## ③ 计算/选型示例

- 已知：  
  清零产量按钮点击后会删除统计数据。
- 公式：  
  **关键操作 = 权限 + 二次确认**
- 计算：  
  清零前应弹窗确认，并要求班组长或工程师权限。
- 结论：  
  危险按钮不能直接执行，必须防误操作。

> [!tip] 记忆口诀
> 普通按钮给操作，危险按钮加权限和确认。

# 10 符号I/O域

> [!info] 核心结论
> 符号I/O域用于把变量数值转换为文字状态，例如0显示“停止”，1显示“运行”，2显示“报警”。

## ① 分类/公式/规则

> [!warning] 符号显示公式
> **状态文本 = 数值变量 → 文本列表映射**

## ② 参考表

| 数值 | 显示文本 |
|---|---|
| **0** | 停止 |
| **1** | 运行 |
| **2** | 待机 |
| **3** | 报警 |
| **4** | 手动 |
| **5** | 自动 |
| **其他** | 未知状态 |

## ③ 计算/选型示例

- 已知：  
  PLC状态字为3。
- 公式：  
  **状态字3 → 报警**
- 计算：  
  符号I/O域根据文本列表显示“报警”。
- 结论：  
  符号I/O域适合显示枚举状态。

> [!tip] 记忆口诀
> 数值给PLC看，文字给人看。

# 12 棒图 / 柱状显示

> [!info] 核心结论
> 棒图用于显示连续变量的大小，例如液位、压力、温度、进度、电流和产量完成率。

## ① 分类/公式/规则

> [!warning] 棒图比例公式
> **显示比例 = 当前值 ÷ 满量程 × 100%**

## ② 参考表

| 变量 | 棒图用途 |
|---|---|
| **液位** | 水箱、料仓高度 |
| **压力** | 气压、液压大小 |
| **温度** | 加热状态 |
| **电流** | 电机负载 |
| **产量** | 当前完成进度 |
| **速度** | 运行速度百分比 |
| **真空值** | 吸附状态 |

## ③ 计算/选型示例

- 已知：  
  水箱液位为60cm，满量程100cm。
- 公式：  
  **显示比例 = 60 ÷ 100 × 100%**
- 计算：  
  显示比例 = **60%**
- 结论：  
  棒图可直观显示当前液位占满量程的比例。

> [!tip] 记忆口诀
> 棒图适合看比例，一眼看出高低多少。

## ① 分类/公式/规则

> [!warning] 控件使用公式

## ② 参考表

| 控件 | 作用 |
|---|---|
| **报警控件** | 当前报警、历史报警、确认复位 |
| **趋势控件** | 曲线显示、历史查询 |
| **用户管理控件** | 登录、权限、用户切换 |
| **配方控件** | 配方选择、保存、下载 |
| **报表控件** | 数据表格、统计记录 |
| **Web控件** | 显示网页或设备页面 |
| **诊断控件** | 通讯状态、系统信息 |

## ③ 计算/选型示例

- 已知：  
  需要显示最近24小时温度变化曲线。
- 公式：  
  **历史曲线 = 趋势控件 + 数据记录**
- 计算：  
  只放一个数值显示不够，需要趋势控件和历史存储。
- 结论：  
  复杂数据分析应使用专用控件。

> [!tip] 记忆口诀
> 简单显示用元素，复杂功能用控件。

# 14 报警控件

> [!info] 核心结论
> 报警控件用于显示当前报警、历史报警、报警等级、时间戳、确认状态和处理信息。

## ① 分类/公式/规则

> [!warning] 报警控件公式
> **报警显示 = 报警变量 + 报警文本 + 时间戳 + 确认状态 + 历史记录**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **当前报警** | 当前仍存在的故障 |
| **历史报警** | 已发生过的报警 |
| **报警等级** | 提示、警告、停机、急停 |
| **确认状态** | 是否被操作员确认 |
| **时间戳** | 发生时间、恢复时间 |
| **报警文本** | 具体故障说明 |
| **处理建议** | 指导维修排查 |

## ③ 计算/选型示例

- 已知：  
  HMI只显示“故障”，没有具体内容。
- 公式：  
  **报警不具体 → 排查效率低**
- 计算：  
  操作员不知道是气压不足、伺服报警还是安全门打开。
- 结论：  
  报警控件应显示具体报警文本和时间。

> [!tip] 记忆口诀
> 报警控件不是摆设，要能帮现场快速定位故障。

# 15 趋势控件

> [!info] 核心结论
> 趋势控件用于显示连续变量随时间变化，常用于温度、压力、流量、液位、电流和真空值分析。

## ① 分类/公式/规则

> [!warning] 趋势控件公式
> **趋势显示 = 变量 + 采样周期 + 时间范围 + 历史数据源**

## ② 参考表

| 设置项 | 说明 |
|---|---|
| **趋势变量** | 温度、压力、电流等 |
| **采样周期** | 多久记录一次 |
| **显示范围** | 最近10分钟、1小时、24小时 |
| **上下限线** | 报警阈值参考 |
| **历史查询** | 查询过去数据 |
| **曲线缩放** | 放大局部波动 |
| **数据导出** | CSV、Excel、数据库 |

## ③ 计算/选型示例

- 已知：  
  压力每秒采样1次，显示最近30分钟。
- 公式：  
  **采样点数 = 1 × 1800**
- 计算：  
  采样点数 = **1800点**
- 结论：  
  趋势控件和数据记录必须支持足够采样点。

> [!tip] 记忆口诀
> 报警看结果，趋势看过程。

# 20 常见故障排查

> [!info] 核心结论
> 对象工具箱相关问题排查按 **对象是否放置、属性是否正确、变量是否绑定、权限是否允许、事件是否触发、通讯是否正常** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **对象异常 = 属性错误 + 变量错绑 + 数据类型错 + 权限不足 + 事件未配置 + 通讯中断**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **按钮无效** | 事件未配置、权限不足、变量地址错 |
| **数值不变** | 通讯异常、变量地址错、刷新未启用 |
| **显示乱码** | 数据类型错误、字符串长度错误 |
| **图形不切换** | 图形列表映射错误、状态变量错误 |
| **报警不显示** | 报警变量或报警控件配置错误 |
| **趋势无曲线** | 采样未启用、变量未绑定 |
| **对象看不到** | 被遮挡、可见性为False、层级错误 |
| **画面卡顿** | 对象太多、刷新太快、图片太大 |

## ③ 计算/选型示例

- 已知：  
  按钮点击后没有任何反应。
- 公式：  
  **按钮有效 = 事件动作 + 变量写入 + 权限允许**
- 计算：  
  先检查按钮是否设置了点击事件，再检查写入变量和权限。
- 结论：  
  按钮问题不一定是PLC问题，很多时候是事件或变量配置错误。

> [!tip] 记忆口诀
> 对象不动，先查属性；数据不对，先查变量。

# 21 对象工具箱快速检查表

> [!info] 核心结论
> 使用对象工具箱开发画面时，要检查对象类别、变量绑定、动态属性、事件动作、权限和画面风格是否统一。

## ① 分类/公式/规则

> [!note] 十步检查法
> ① 基本对象是否用于静态结构  
> ② 元素对象是否正确绑定变量  
> ③ 控件对象是否配置数据源  
> ④ 图形库资源是否统一管理  
> ⑤ 按钮事件是否正确触发  
> ⑥ 输入域是否有范围和权限  
> ⑦ 指示状态是否绑定反馈变量  
> ⑧ 报警和趋势控件是否记录历史  
> ⑨ 对象层级是否正确，重要提示不被遮挡  
> ⑩ 页面风格、颜色、字体和单位是否统一

> [!warning] 最终检查公式
> **合格画面对象 = 放置正确 + 属性正确 + 变量正确 + 动作正确 + 权限正确**

## ② 参考表

| 检查项 | 目标 |
|---|---|
| **对象类型** | 选择合适控件 |
| **变量绑定** | 地址和类型正确 |
| **动态效果** | 状态显示清楚 |
| **事件动作** | 点击、跳转、写入有效 |
| **权限限制** | 防止误操作 |
| **视觉统一** | 风格一致 |
| **资源管理** | 图片不臃肿 |
| **运行性能** | 不因对象过多卡顿 |

## ③ 计算/选型示例

- 已知：  
  需要做一个气缸手动控制区域。
- 公式：  
  **气缸控制区 = 伸出按钮 + 缩回按钮 + 到位指示 + 超时报警 + 权限互锁**
- 计算：  
  不能只放两个按钮，还要显示伸出到位、缩回到位和报警状态。
- 结论：  
  工具箱对象要组合使用，才能形成完整可用的操作单元。

> [!tip] 选型口诀
> 画静态用基本对象，看数值用I/O域，切状态用图形/符号域，看历史用趋势，查故障用报警，管权限用用户控件。
