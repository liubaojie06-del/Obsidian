# 01 指令面板总览

> [!info] 核心结论
> “指令”面板用于辅助编写HMI/SCADA脚本，常见内容包括 **代码模板、函数列表、脚本片段、系统函数、对象函数和变量处理函数**。

## ① 分类/公式/规则

> [!warning] 使用公式
> **脚本功能 = 触发事件 + 代码模板 + 函数调用 + 变量读写 + 执行结果**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **代码模板** | 提供常用脚本结构，减少重复编写 |
| **函数列表** | 提供可调用函数，方便查找和插入 |
| **C脚本** | 使用C风格语法编写逻辑 |
| **VB脚本** | 使用VB风格语法编写逻辑 |
| **事件脚本** | 按钮点击、画面打开、变量变化时执行 |
| **对象脚本** | 与按钮、输入框、报警控件等对象绑定 |
| **系统函数** | 调用画面跳转、用户登录、变量读写等系统能力 |

## ③ 计算/选型示例

- 已知：  
  点击按钮后需要跳转到参数画面。
- 公式：  
  **按钮点击事件 + 画面切换函数 = 页面跳转**
- 计算：  
  在按钮的“点击”事件中调用画面切换相关函数。
- 结论：  
  指令面板可以帮助快速插入对应函数，减少手动输入错误。

> [!tip] 记忆口诀
> 画面靠对象，动作靠事件，复杂逻辑靠脚本指令。

# 02 代码模板

> [!info] 核心结论
> 代码模板用于快速生成常用脚本结构，适合反复使用的判断、循环、函数调用和变量处理逻辑。

## ① 分类/公式/规则

> [!warning] 截图规则
> **代码模板仅可用于 C 和 VB 脚本**

## ② 参考表

| 模板类型 | 用途 |
|---|---|
| **条件判断模板** | if/else逻辑 |
| **循环模板** | 批量处理对象或变量 |
| **函数调用模板** | 快速调用已有函数 |
| **变量读写模板** | 读取或写入HMI/PLC变量 |
| **错误处理模板** | 异常判断和提示 |
| **注释模板** | 统一脚本说明 |
| **结构化模板** | 统一代码格式和维护规范 |

## ③ 计算/选型示例

- 已知：  
  需要判断当前用户是否有权限修改参数。
- 公式：  
  **权限变量 + 条件判断模板 = 操作许可判断**
- 计算：  
  如果权限满足，则允许写入参数；否则弹出提示。
- 结论：  
  条件判断模板可以减少重复写权限判断代码。

> [!tip] 记忆口诀
> 常写的逻辑做模板，少写错，也更统一。

# 06 事件脚本

> [!info] 核心结论
> 事件脚本是在特定动作发生时执行的脚本，例如按钮点击、画面打开、画面关闭、变量变化和定时触发。

## ① 分类/公式/规则

> [!warning] 事件脚本公式
> **事件脚本 = 触发条件 + 执行动作**

## ② 参考表

| 事件 | 典型用途 |
|---|---|
| **点击事件** | 按钮操作、画面跳转 |
| **按下事件** | 点动按钮、临时输出 |
| **释放事件** | 松开按钮后复位输出 |
| **画面打开** | 初始化变量、刷新状态 |
| **画面关闭** | 清除临时变量 |
| **变量变化** | 状态变化后触发逻辑 |
| **定时事件** | 周期刷新、周期计算 |

## ③ 计算/选型示例

- 已知：  
  点动按钮按下时电机运行，松开时停止。
- 公式：  
  **按下写1，释放写0**
- 计算：  
  按下事件写入启动位，释放事件清除启动位。
- 结论：  
  点动类按钮必须同时配置按下和释放事件。

> [!tip] 记忆口诀
> 按钮不是只有点击，按下和释放也很重要。

# 09 变量读写

> [!info] 核心结论
> 脚本常用于读取和写入HMI变量、PLC变量、内部变量和系统变量，是画面与控制逻辑交互的核心。

## ① 分类/公式/规则

> [!warning] 变量读写公式
> **变量操作成功 = 地址正确 + 类型正确 + 权限允许 + 通讯正常**

## ② 参考表

| 变量类型 | 用途 |
|---|---|
| **PLC变量** | 启停、复位、状态、参数 |
| **HMI内部变量** | 页面状态、临时计算 |
| **系统变量** | 当前用户、时间、语言 |
| **报警变量** | 报警位、报警字 |
| **配方变量** | 工艺参数组 |
| **数据库变量** | 历史记录、报表数据 |
| **脚本变量** | 临时判断和计算 |

## ③ 计算/选型示例

- 已知：  
  点击“复位”按钮后报警没有复位。
- 公式：  
  **复位有效 = 复位变量写入 + PLC复位条件满足**
- 计算：  
  先检查脚本是否写入正确复位变量，再检查PLC是否允许复位。
- 结论：  
  变量写入成功不代表动作一定成功，还要看PLC逻辑。

> [!tip] 记忆口诀
> 脚本写变量，PLC做判断。

# 10 参数校验

> [!info] 核心结论
> 参数校验用于防止输入超限、类型错误、空值或非法参数，是运行系统安全的重要保护。

## ① 分类/公式/规则

> [!warning] 参数校验公式
> **参数可保存 = 最小值 ≤ 输入值 ≤ 最大值 + 权限满足**

## ② 参考表

| 校验项 | 说明 |
|---|---|
| **最小值** | 防止参数过小 |
| **最大值** | 防止参数过大 |
| **数据类型** | 整数、浮点数、字符串 |
| **空值检查** | 防止未输入 |
| **单位检查** | 防止单位误解 |
| **权限检查** | 防止低权限修改 |
| **保存确认** | 防止误保存 |

## ③ 计算/选型示例

- 已知：  
  输送速度允许范围为100~800mm/s，用户输入1000mm/s。
- 公式：  
  **1000 > 800 → 参数非法**
- 计算：  
  脚本应拒绝保存，并提示“速度超出范围”。
- 结论：  
  参数输入必须通过脚本或控件属性进行范围保护。

> [!tip] 记忆口诀
> 能输入的地方，就要有校验。

# 14 配方相关指令

> [!info] 核心结论
> 配方指令用于保存、加载、下载、上传和切换不同产品的工艺参数。

## ① 分类/公式/规则

> [!warning] 配方操作公式
> **配方切换 = 选择配方 + 权限确认 + 下载到PLC + 校验当前配方**

## ② 参考表

| 指令用途 | 说明 |
|---|---|
| **保存配方** | 保存当前参数组 |
| **加载配方** | 从文件或数据库读取 |
| **下载配方** | 写入PLC或控制器 |
| **上传配方** | 从PLC读取当前参数 |
| **删除配方** | 删除旧参数组 |
| **复制配方** | 新建类似产品参数 |
| **确认配方** | 校验当前运行配方 |

## ③ 计算/选型示例

- 已知：  
  设备生产A产品，需要切换到B产品。
- 公式：  
  **B配方加载 + 下载PLC + 当前配方确认**
- 计算：  
  如果只在HMI显示B配方，但没有写入PLC，设备仍按旧参数运行。
- 结论：  
  配方切换后必须确认PLC实际参数已更新。

> [!tip] 记忆口诀
> 配方不是选中就完事，必须下载并确认。

# 15 日志与记录指令

> [!info] 核心结论
> 日志指令用于记录用户操作、参数修改、报警事件、配方切换和系统状态变化。

## ① 分类/公式/规则

> [!warning] 日志记录公式
> **有效日志 = 时间 + 用户 + 操作 + 对象 + 旧值 + 新值**

## ② 参考表

| 记录类型 | 示例 |
|---|---|
| **操作日志** | 用户点击启动、停止、复位 |
| **参数日志** | 修改速度、压力、温度设定 |
| **报警日志** | 故障发生、确认、恢复 |
| **配方日志** | 配方切换、保存、下载 |
| **登录日志** | 用户登录、注销 |
| **系统日志** | 通讯中断、数据库异常 |
| **维护日志** | 工程师进入维护画面 |

## ③ 计算/选型示例

- 已知：  
  伺服速度从300改为600，设备节拍异常。
- 公式：  
  **参数追溯 = 修改人 + 修改时间 + 旧值新值**
- 计算：  
  日志应记录谁在什么时候把300改成600。
- 结论：  
  关键参数修改必须记录日志。

> [!tip] 记忆口诀
> 没日志，出了问题就只能靠猜。

# 16 函数封装

> [!info] 核心结论
> 函数封装用于把重复逻辑写成可复用函数，提升脚本一致性和维护效率。

## ① 分类/公式/规则

> [!warning] 函数封装公式
> **通用函数 = 输入参数 + 处理逻辑 + 返回结果**

## ② 参考表

| 可封装功能 | 示例 |
|---|---|
| **权限判断** | 判断当前用户是否允许操作 |
| **参数范围检查** | 判断输入值是否超限 |
| **报警提示** | 统一弹窗格式 |
| **页面跳转** | 带权限的跳转函数 |
| **日志记录** | 统一记录操作日志 |
| **配方校验** | 校验配方参数完整性 |
| **工程量换算** | 原始值转实际值 |

## ③ 计算/选型示例

- 已知：  
  多个参数页面都要检查输入范围。
- 公式：  
  **重复校验逻辑 → 封装函数**
- 计算：  
  写一个通用范围检查函数，各页面调用。
- 结论：  
  函数封装能减少重复代码和维护成本。

> [!tip] 记忆口诀
> 一段代码复制三次，就该考虑封装函数。

# 17 脚本命名与注释

> [!info] 核心结论
> 脚本命名和注释决定后期能否快速理解、修改和排查脚本逻辑。

## ① 分类/公式/规则

> [!warning] 命名公式
> **脚本名称 = 功能对象 + 动作 + 用途**

## ② 参考表

| 类型 | 示例 |
|---|---|
| **按钮脚本** | Btn_Start_Click |
| **页面脚本** | Page_Main_Open |
| **参数脚本** | Param_Save_Check |
| **配方脚本** | Recipe_Download |
| **报警脚本** | Alarm_Reset |
| **权限脚本** | User_CheckRole |
| **日志脚本** | Log_WriteOperation |

## ③ 计算/选型示例

- 已知：  
  一个脚本名字叫“script1”。
- 公式：  
  **命名不清 → 后期难维护**
- 计算：  
  维护人员不知道它属于哪个按钮、哪个页面、做什么动作。
- 结论：  
  应改成能表达功能的名称，例如Btn_ResetAlarm_Click。

> [!tip] 记忆口诀
> 脚本名字写清楚，半年后自己也看得懂。

# 18 脚本调试

> [!info] 核心结论
> 脚本调试要按语法错误、变量错误、权限错误、事件错误和通讯错误逐级排查。

## ① 分类/公式/规则

> [!warning] 调试公式
> **脚本异常 = 语法错误 + 变量错误 + 事件未触发 + 权限不足 + 通讯异常**

## ② 参考表

| 调试项 | 检查内容 |
|---|---|
| **语法** | 括号、分号、关键字是否正确 |
| **变量** | 地址、类型、读写权限是否正确 |
| **事件** | 脚本是否绑定到正确事件 |
| **权限** | 当前用户是否允许操作 |
| **通讯** | PLC、数据库、服务器是否在线 |
| **返回值** | 函数执行结果是否正确 |
| **日志** | 是否记录错误信息 |

## ③ 计算/选型示例

- 已知：  
  按钮点击后脚本没有执行。
- 公式：  
  **脚本未执行 = 事件未绑定 / 按钮被遮挡 / 权限禁用**
- 计算：  
  先检查按钮点击事件是否绑定脚本，再检查按钮是否被禁用。
- 结论：  
  不要一开始就怀疑PLC，先确认脚本事件是否触发。

> [!tip] 记忆口诀
> 脚本不跑，先查事件；脚本跑错，先查变量。

# 19 脚本安全

> [!info] 核心结论
> 脚本可以直接影响设备操作和参数写入，因此必须做好权限、范围、确认和PLC互锁。

## ① 分类/公式/规则

> [!warning] 安全公式
> **安全脚本 = 权限判断 + 参数校验 + 二次确认 + PLC安全联锁**

## ② 参考表

| 安全项 | 作用 |
|---|---|
| **权限判断** | 防止低权限操作 |
| **范围校验** | 防止参数超限 |
| **二次确认** | 防止误点危险按钮 |
| **操作日志** | 追溯谁执行了操作 |
| **PLC互锁** | 最终安全保护 |
| **错误提示** | 说明为什么操作失败 |
| **禁止默认密码** | 防止权限失效 |

## ③ 计算/选型示例

- 已知：  
  脚本允许远程打开阀门。
- 公式：  
  **远程控制 = 权限 + 条件 + 联锁 + 反馈**
- 计算：  
  只有权限通过、现场条件满足、PLC允许动作时才执行。
- 结论：  
  远程控制类脚本必须谨慎设计。

> [!tip] 记忆口诀
> 脚本能写变量，安全就不能只靠界面。

# 20 常见故障排查

> [!info] 核心结论
> 指令和脚本相关问题排查按 **代码模板、函数调用、事件绑定、变量连接、权限、通讯、日志** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **脚本问题 = 模板用错 + 函数选错 + 参数错 + 事件错 + 变量错 + 权限错**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **脚本编译失败** | 语法错误、括号缺失、函数名错误 |
| **函数无法调用** | 函数不存在、参数类型错误 |
| **按钮无反应** | 未绑定事件、权限不足、对象被遮挡 |
| **变量不变化** | 地址错、类型错、通讯异常 |
| **参数不保存** | 未写入目标变量、数据库失败 |
| **页面不跳转** | 目标画面名错误、权限不足 |
| **报警不复位** | 故障未消除、复位变量错误 |
| **运行卡顿** | 脚本循环过重、刷新太快 |

## ③ 计算/选型示例

- 已知：  
  脚本调用函数时报参数错误。
- 公式：  
  **函数调用正确 = 函数名正确 + 参数数量正确 + 参数类型正确**
- 计算：  
  重点检查函数列表中的函数说明和参数格式。
- 结论：  
  函数调用错误多数来自参数数量、顺序或类型不匹配。

> [!tip] 记忆口诀
> 函数错先查参数，按钮错先查事件，数据错先查变量。

# 21 指令面板快速检查表

> [!info] 核心结论
> 使用指令面板编写脚本时，要检查模板适用语言、函数来源、事件绑定、变量读写、权限判断和运行测试。

## ① 分类/公式/规则

> [!note] 十步检查法
> ① 确认当前脚本语言是C还是VB  
> ② 确认代码模板是否适用于当前脚本  
> ③ 确认函数列表中的函数是否适用  
> ④ 检查函数参数数量和类型  
> ⑤ 检查脚本是否绑定到正确事件  
> ⑥ 检查变量地址、类型和读写方向  
> ⑦ 检查权限判断和二次确认  
> ⑧ 检查参数范围和异常处理  
> ⑨ 检查操作日志和错误提示  
> ⑩ 上线前做正常、异常、断电、权限和通讯测试

> [!warning] 最终检查公式
> **合格脚本 = 语言正确 + 函数正确 + 变量正确 + 事件正确 + 权限正确 + 测试通过**

## ② 参考表

| 检查项 | 目标 |
|---|---|
| **代码模板** | 减少重复和语法错误 |
| **函数列表** | 快速找到可用函数 |
| **事件绑定** | 确保脚本能被触发 |
| **变量读写** | 确保数据正确交互 |
| **权限判断** | 防止误操作 |
| **参数校验** | 防止超限 |
| **日志记录** | 便于追溯 |
| **调试测试** | 确保运行可靠 |

## ③ 计算/选型示例

- 已知：  
  需要做一个“保存参数”按钮。
- 公式：  
  **保存参数 = 权限判断 + 参数校验 + 写入变量 + 保存日志 + 提示结果**
- 计算：  
  不能只写入变量，还要判断权限、检查范围、记录修改。
- 结论：  
  推荐用代码模板建立统一保存逻辑，并通过函数列表调用通用函数。

> [!tip] 选型口诀
> 模板管结构，函数管能力；事件负责触发，变量负责数据；权限先判断，参数先校验，日志要留下。
