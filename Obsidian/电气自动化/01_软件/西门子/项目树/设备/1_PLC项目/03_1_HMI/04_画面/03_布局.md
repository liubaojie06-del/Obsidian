# 01 布局总览

> [!info] 核心结论
> “布局”用于管理HMI/SCADA画面对象的 **层级、对齐、网格、位置、尺寸、可见性、画面边界和超出范围对象**，目的是让画面整齐、易维护、不卡顿、不误触。

## ① 分类/公式/规则

> [!warning] 布局公式
> **好布局 = 图层清楚 + 网格统一 + 对齐规范 + 间距一致 + 边界不超出 + 操作不拥挤**

## ② 参考表

| 布局项 | 作用 |
|---|---|
| **层** | 管理对象前后顺序和显示隐藏 |
| **根画面** | 当前画面的最底层容器 |
| **网格** | 辅助对象对齐和排版 |
| **线对齐** | 按参考线或对象边缘对齐 |
| **网格对齐** | 按固定网格吸附对象 |
| **无对齐** | 自由移动对象 |
| **超出范围对象** | 检查对象是否跑出画面边界 |

## ③ 计算/选型示例

- 已知：  
  画面上有按钮、文本、状态灯、设备图形和弹窗对象。
- 公式：  
  **多对象画面 → 必须做布局管理**
- 计算：  
  如果没有层级和对齐规则，后期对象容易遮挡、错位、难维护。
- 结论：  
  画面编辑时应优先规划图层、网格和公共区域。

> [!tip] 记忆口诀
> 控件能用是功能，排得整齐才是工程。

# 02 层 / 图层管理

> [!info] 核心结论
> 层用于管理画面对象的前后顺序和显示范围，常用于背景、设备图形、状态对象、按钮、弹窗和报警提示的分层显示。

## ① 分类/公式/规则

> [!warning] 图层规则
> **背景层在下，操作层在中，弹窗报警层在上**

## ② 参考表

| 图层类型 | 常见内容 |
|---|---|
| **背景层** | 背景色、区域框、装饰线 |
| **设备层** | 电机、气缸、阀门、管道、流程图 |
| **状态层** | 指示灯、状态文字、动画对象 |
| **操作层** | 按钮、输入框、选择开关 |
| **数据层** | 数值显示、趋势入口、统计信息 |
| **报警层** | 报警条、报警弹窗、红色提示 |
| **维护层** | 调试对象、隐藏诊断对象 |

## ③ 计算/选型示例

- 已知：  
  一个按钮被背景矩形挡住，运行时无法点击。
- 公式：  
  **按钮层级必须高于背景层**
- 计算：  
  背景矩形在上层时，会覆盖按钮点击区域。
- 结论：  
  应把背景对象下移图层，或把按钮上移图层。

> [!tip] 记忆口诀
> 看不见查可见性，点不了查层级和遮挡。

# 03 根画面

> [!info] 核心结论
> 根画面是当前页面的基础容器，所有对象、图层、网格和边界检查都基于根画面展开。

## ① 分类/公式/规则

> [!warning] 根画面规则
> **所有对象都必须位于根画面的有效显示范围内**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **画面尺寸** | 当前HMI/SCADA页面宽高 |
| **根画面边界** | 对象显示的有效区域 |
| **对象坐标** | 相对于画面左上角的位置 |
| **图层列表** | 根画面下的多个层 |
| **公共对象** | 可能附着在根画面或模板上 |
| **超出范围检查** | 检查对象是否跑出画面 |

## ③ 计算/选型示例

- 已知：  
  HMI分辨率为800×480，一个按钮X坐标为820。
- 公式：  
  **对象X坐标 > 画面宽度 → 超出画面**
- 计算：  
  820 > 800，按钮超出右侧边界。
- 结论：  
  该按钮在运行时可能不可见或不可操作，应移动回画面内。

> [!tip] 记忆口诀
> 根画面是舞台，控件不能跑出演出区域。

# 04 网格

> [!info] 核心结论
> 网格用于辅助对象整齐排列，常见设置包括显示网格、X间距、Y间距和网格对齐。

## ① 分类/公式/规则

> [!warning] 网格公式
> **对象坐标 ≈ 网格间距的整数倍**

## ② 参考表

| 网格设置 | 作用 |
|---|---|
| **显示网格** | 在编辑区显示辅助线 |
| **X间距** | 水平方向网格间隔 |
| **Y间距** | 垂直方向网格间隔 |
| **网格对齐** | 移动对象时自动吸附网格 |
| **小网格** | 适合精细排版 |
| **大网格** | 适合快速布局 |
| **隐藏网格** | 运行时不显示，仅编辑辅助 |

## ③ 计算/选型示例

- 已知：  
  网格X=8，Y=8，对象坐标为X=103，Y=57。
- 公式：  
  **对齐坐标 ≈ 最近的8的倍数**
- 计算：  
  X=104，Y=56更接近网格点。
- 结论：  
  开启网格对齐后，对象会更容易排整齐。

> [!tip] 记忆口诀
> 网格是尺子，对齐是规矩。

# 05 布局模式

> [!info] 核心结论
> 布局模式决定对象移动时是否自动吸附到线、网格或完全自由移动。

## ① 分类/公式/规则

> [!warning] 布局模式选择
> **需要整齐排版 → 线对齐或网格对齐**  
> **需要精细微调 → 无对齐**

## ② 参考表

| 布局模式 | 适合场景 |
|---|---|
| **线对齐** | 多对象边缘对齐、表格式布局 |
| **网格对齐** | 按固定间距快速排版 |
| **无** | 精细调整、特殊图形位置 |
| **显示网格** | 辅助判断间距 |
| **关闭网格** | 查看最终画面效果 |
| **临时切换模式** | 排版时常用 |

## ③ 计算/选型示例

- 已知：  
  需要把10个按钮做成整齐的一列。
- 公式：  
  **规则排列 → 网格对齐 / 线对齐**
- 计算：  
  使用网格对齐可以保证按钮X坐标一致，Y间距一致。
- 结论：  
  多按钮、多输入框、多指示灯布局时，建议开启对齐模式。

> [!tip] 记忆口诀
> 做表格用对齐，做动画用微调。

# 06 线对齐

> [!info] 核心结论
> 线对齐用于让对象边缘、中心线或参考线保持一致，适合按钮组、参数表、状态栏和导航栏排版。

## ① 分类/公式/规则

> [!warning] 线对齐规则
> **同一组对象应左边对齐、中心对齐或右边对齐，不要混乱排列**

## ② 参考表

| 对齐方式 | 说明 |
|---|---|
| **左对齐** | 多个对象左边缘一致 |
| **右对齐** | 多个对象右边缘一致 |
| **顶部对齐** | 多个对象上边缘一致 |
| **底部对齐** | 多个对象下边缘一致 |
| **水平居中** | 对象中心在同一竖线 |
| **垂直居中** | 对象中心在同一横线 |
| **参考线对齐** | 按页面辅助线排列 |

## ③ 计算/选型示例

- 已知：  
  三个按钮X坐标分别为100、103、98。
- 公式：  
  **左对齐 → X坐标一致**
- 计算：  
  可统一设置为X=100。
- 结论：  
  对齐后按钮组更整齐，误触概率更低。

> [!tip] 记忆口诀
> 一组按钮一条线，一组数据一个基准。

# 07 网格对齐

> [!info] 核心结论
> 网格对齐用于让对象按固定间距吸附，适合快速搭建整齐的HMI页面。

## ① 分类/公式/规则

> [!warning] 网格对齐公式
> **对象位置 = 网格间距 × 整数**

## ② 参考表

| 网格间距 | 适合用途 |
|---|---|
| **4像素** | 精细控件微调 |
| **8像素** | 常用HMI排版 |
| **10像素** | 十进制习惯排版 |
| **16像素** | 大屏卡片式布局 |
| **20像素** | 工业触摸屏大按钮 |
| **自定义间距** | 按项目规范设置 |

## ③ 计算/选型示例

- 已知：  
  网格间距为8像素，按钮宽度为96像素。
- 公式：  
  **96 ÷ 8 = 12**
- 计算：  
  按钮宽度正好占12个网格单位。
- 结论：  
  使用8像素网格时，按钮尺寸更容易整齐排列。

> [!tip] 记忆口诀
> 尺寸最好是网格倍数，排版最省心。

# 08 无对齐模式

> [!info] 核心结论
> 无对齐模式允许对象自由移动，适合精细调整图形、动画轨迹、设备轮廓和不规则布局。

## ① 分类/公式/规则

> [!warning] 使用规则
> **无对齐适合微调，不适合大量控件排版**

## ② 参考表

| 适合场景 | 说明 |
|---|---|
| **设备轮廓微调** | 图片、图标位置精确调整 |
| **动画对象** | 需要精确起点和终点 |
| **管线连接** | 细节位置调整 |
| **特殊图形** | 不规则图形组合 |
| **临时修正** | 对齐后局部微调 |
| **小屏排版** | 需要节省每个像素 |

## ③ 计算/选型示例

- 已知：  
  一个气缸动画对象需要精确移动到X=137的位置。
- 公式：  
  **精确坐标 → 无对齐模式**
- 计算：  
  如果开启8像素网格对齐，可能自动吸附到136或144。
- 结论：  
  动画对象精调时可临时关闭对齐。

> [!tip] 记忆口诀
> 大布局靠对齐，小细节靠微调。

# 09 层可见性

> [!info] 核心结论
> 层可见性用于显示或隐藏某一层的对象，便于编辑复杂画面，也可减少误选对象。

## ① 分类/公式/规则

> [!warning] 编辑规则
> **编辑某一类对象时，可隐藏无关层，避免误操作**

## ② 参考表

| 场景 | 操作 |
|---|---|
| **编辑背景** | 显示背景层，隐藏操作层 |
| **编辑按钮** | 锁定或隐藏背景层 |
| **检查报警对象** | 只显示报警层 |
| **检查动画对象** | 显示设备层和动画层 |
| **防止误移动** | 锁定背景或模板层 |
| **查看最终效果** | 打开全部可见层 |

## ③ 计算/选型示例

- 已知：  
  编辑按钮时经常误选中背景矩形。
- 公式：  
  **误选背景 → 隐藏或锁定背景层**
- 计算：  
  背景层隐藏后，编辑区域只剩按钮和文字对象。
- 结论：  
  图层可见性可以提高编辑效率。

> [!tip] 记忆口诀
> 不编辑的层就隐藏或锁定。

# 10 层顺序

> [!info] 核心结论
> 层顺序决定对象显示前后，弹窗、报警、按钮必须高于背景和装饰对象。

## ① 分类/公式/规则

> [!warning] 层顺序规则
> **背景 < 设备图形 < 数据状态 < 操作按钮 < 弹窗报警**

## ② 参考表

| 层级 | 内容 |
|---|---|
| **最低层** | 背景、区域色块 |
| **较低层** | 管道、设备轮廓 |
| **中间层** | 状态灯、数值、动画 |
| **较高层** | 按钮、输入域、菜单 |
| **最高层** | 弹窗、报警、安全提示 |

## ③ 计算/选型示例

- 已知：  
  报警提示被设备图片盖住。
- 公式：  
  **报警层级必须高于设备层**
- 计算：  
  把报警对象移动到更高层即可显示在最上方。
- 结论：  
  重要提示必须放在高层。

> [!tip] 记忆口诀
> 越重要，越靠上；越背景，越靠下。

# 11 对象锁定

> [!info] 核心结论
> 对象锁定用于防止背景、模板、导航栏和公共元素被误移动或误删除。

## ① 分类/公式/规则

> [!warning] 锁定规则
> **固定不改的对象应锁定，频繁编辑的对象不要锁定**

## ② 参考表

| 建议锁定对象 | 原因 |
|---|---|
| **背景图** | 防止误移动 |
| **页面边框** | 保持布局稳定 |
| **公共导航栏** | 防止误删按钮 |
| **公司LOGO** | 保持统一 |
| **状态栏** | 防止错位 |
| **模板对象** | 避免局部页面破坏统一风格 |
| **隐藏热区** | 防止误操作 |

## ③ 计算/选型示例

- 已知：  
  修改参数输入框时，不小心把整个背景拖偏。
- 公式：  
  **背景对象应锁定**
- 计算：  
  背景锁定后，鼠标操作不会选中和移动背景。
- 结论：  
  大型画面中建议锁定背景层和模板层。

> [!tip] 记忆口诀
> 背景一旦定，最好锁起来。

# 12 对象分组

> [!info] 核心结论
> 对象分组可以把多个控件作为一个整体移动、复制、对齐和管理，适合设备组件、按钮组和参数区域。

## ① 分类/公式/规则

> [!warning] 分组公式
> **组件 = 多个对象组合成一个功能单元**

## ② 参考表

| 分组对象 | 示例 |
|---|---|
| **电机组件** | 图标 + 状态灯 + 名称 + 数值 |
| **气缸组件** | 图形 + 伸出按钮 + 缩回按钮 + 到位灯 |
| **参数行** | 标签 + 输入框 + 单位 |
| **报警条** | 图标 + 文本 + 时间 |
| **按钮组** | 启动 + 停止 + 复位 |
| **导航组** | 首页 + 自动 + 手动 + 报警 |
| **趋势区域** | 标题 + 曲线 + 图例 |

## ③ 计算/选型示例

- 已知：  
  一个参数行包括“速度”文本、输入框和“mm/s”单位。
- 公式：  
  **参数行 = 标签 + 数值框 + 单位**
- 计算：  
  分组后可整体移动，不会导致单位和输入框错位。
- 结论：  
  重复结构建议分组或做成模板对象。

> [!tip] 记忆口诀
> 一个功能单元，最好作为一个整体管理。

# 13 对齐与分布

> [!info] 核心结论
> 对齐保证对象边缘整齐，分布保证对象间距一致，是工业画面专业度的重要体现。

## ① 分类/公式/规则

> [!warning] 等距分布公式
> **间距 = (总区域宽度 - 对象总宽度) ÷ 间隔数量**

## ② 参考表

| 操作 | 作用 |
|---|---|
| **左对齐** | 表格左边整齐 |
| **右对齐** | 数值框右边整齐 |
| **水平居中** | 对象中心线一致 |
| **垂直居中** | 对象高度居中 |
| **水平等距** | 横向按钮间距一致 |
| **垂直等距** | 参数行间距一致 |
| **统一大小** | 按钮或输入框尺寸一致 |

## ③ 计算/选型示例

- 已知：  
  一行放4个按钮，总区域宽度500，每个按钮宽100。
- 公式：  
  **间距 = (500 - 4×100) ÷ 3**
- 计算：  
  间距 = 100 ÷ 3 ≈ **33.3像素**
- 结论：  
  可以设置约33像素间距，让按钮均匀分布。

> [!tip] 记忆口诀
> 对齐看边，分布看间距。

# 14 超出范围的对象

> [!info] 核心结论
> 超出范围对象是指对象位置超出画面有效边界，可能导致运行时看不见、点不到或影响画面性能。

## ① 分类/公式/规则

> [!warning] 边界规则
> **对象左上角和右下角都应在画面有效区域内**

## ② 参考表

| 超出方向 | 影响 |
|---|---|
| **左侧超出** | 对象部分不可见 |
| **右侧超出** | 按钮或文本被裁切 |
| **上方超出** | 标题或控件看不到 |
| **下方超出** | 底部按钮不可操作 |
| **隐藏对象超出** | 后期维护难找 |
| **动画终点超出** | 动画运行后对象消失 |
| **弹窗超出** | 弹窗无法完整显示 |

## ③ 计算/选型示例

- 已知：  
  画面高度480，某对象Y=460，高度50。
- 公式：  
  **对象底部 = Y + 高度**
- 计算：  
  对象底部 = 460 + 50 = **510**，510 > 480。
- 结论：  
  对象下方超出画面30像素，应上移或缩小。

> [!tip] 记忆口诀
> 对象不只看起点，还要看宽高加起来有没有越界。

# 15 画面边距

> [!info] 核心结论
> 画面边距用于避免控件贴边，防止文字被裁切、按钮误触和显示不完整。

## ① 分类/公式/规则

> [!warning] 边距规则
> **重要控件不要贴着画面边缘，至少留出安全边距**

## ② 参考表

| 边距类型 | 作用 |
|---|---|
| **左边距** | 防止文字贴边 |
| **右边距** | 防止按钮被裁切 |
| **上边距** | 留给状态栏或标题 |
| **下边距** | 留给导航栏或系统栏 |
| **控件间距** | 防止误触 |
| **组内边距** | 保持参数区域清晰 |
| **弹窗边距** | 防止弹窗贴屏幕边缘 |

## ③ 计算/选型示例

- 已知：  
  触摸屏按钮紧贴屏幕边缘，现场容易误触。
- 公式：  
  **触摸控件 = 按钮尺寸 + 安全边距**
- 计算：  
  按钮周围应留出足够空白区域。
- 结论：  
  触摸按钮不要贴边，也不要彼此过近。

> [!tip] 记忆口诀
> 工业触摸屏不是手机界面，按钮要大，边距要够。

# 16 布局与触摸操作

> [!info] 核心结论
> HMI布局必须考虑手指、手套、油污和现场震动，按钮和输入框不能太小、太密、太靠边。

## ① 分类/公式/规则

> [!warning] 触摸布局规则
> **高频按钮要大，危险按钮要远离普通按钮**

## ② 参考表

| 布局项 | 注意 |
|---|---|
| **按钮尺寸** | 戴手套时要更大 |
| **按钮间距** | 防止误触 |
| **危险按钮位置** | 避免与普通按钮相邻 |
| **确认弹窗** | 清零、删除、下载等操作 |
| **状态反馈** | 点击后要有反馈 |
| **灰色禁用** | 不可用按钮应明显置灰 |
| **固定导航** | 常用页面入口固定 |

## ③ 计算/选型示例

- 已知：  
  “启动”和“清零产量”按钮挨得很近。
- 公式：  
  **危险按钮应远离高频按钮**
- 计算：  
  生产中频繁点启动，容易误触清零。
- 结论：  
  清零按钮应放到统计或维护区域，并加确认和权限。

> [!tip] 记忆口诀
> 高频操作放顺手，危险操作放稳妥。

# 17 布局与维护

> [!info] 核心结论
> 好布局不仅方便操作，也方便工程师后期维护、查找对象、修改变量和排查故障。

## ① 分类/公式/规则

> [!warning] 维护公式
> **可维护布局 = 图层清楚 + 命名清楚 + 分组清楚 + 对齐清楚**

## ② 参考表

| 维护问题 | 布局优化 |
|---|---|
| **对象难找** | 按图层和功能分组 |
| **误选背景** | 锁定背景层 |
| **变量混乱** | 对象命名与变量一致 |
| **页面卡顿** | 减少重叠对象和复杂图片 |
| **重复修改多处** | 使用公共模板 |
| **弹窗遮挡** | 明确层级优先级 |

## ③ 计算/选型示例

- 已知：

- 公式：  
  **重复公共区域 → 模板化**
- 计算：  
  后期修改状态栏需要改20处，容易漏改。
- 结论：  
  公共状态栏应放入模板或公共层管理。

# 18 常见故障排查

> [!info] 核心结论
> 布局问题排查按 **层级、可见性、锁定、坐标、尺寸、网格、边界、模板** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **布局异常 = 层级遮挡 + 对象隐藏 + 坐标越界 + 尺寸错误 + 模板覆盖 + 锁定误操作**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **对象看不到** | 被隐藏、超出范围、被遮挡 |
| **对象点不了** | 被透明对象覆盖、权限禁用、层级错误 |
| **对象移不动** | 对象或图层被锁定 |
| **对象总是吸附** | 网格对齐开启 |
| **对象位置不准** | 对齐模式影响或坐标输入错误 |
| **弹窗被挡住** | 弹窗层级太低 |
| **画面边缘缺失** | 对象超出画面范围 |
| **页面风格不统一** | 未使用模板或网格规范 |

## ③ 计算/选型示例

- 已知：  
  运行时按钮显示正常，但点击无反应。
- 公式：  
  **显示正常但点不了 = 可能被透明对象覆盖**
- 计算：  
  上层透明矩形可能挡住按钮点击区域。
- 结论：  
  应检查图层顺序和透明对象范围。

> [!tip] 记忆口诀
> 看得见不一定点得到，点不到先查上面有没有东西挡着。

# 19 布局快速检查表

> [!info] 核心结论
> 布局检查的目标是确保画面对象整齐、层级正确、边界安全、触摸可靠、维护方便。

## ① 分类/公式/规则

> [!note] 十步检查法
> ① 检查对象是否按功能分层  
> ② 检查背景层是否锁定  
> ③ 检查按钮、输入框是否对齐  
> ④ 检查网格间距是否统一  
> ⑤ 检查对象是否超出画面范围  
> ⑥ 检查弹窗和报警是否在最高层  
> ⑦ 检查危险按钮是否加权限和确认  
> ⑧ 检查触摸按钮尺寸和间距是否足够  
> ⑨ 检查隐藏对象和透明对象是否遮挡操作  
> ⑩ 检查不同页面布局风格是否一致

> [!warning] 最终检查公式
> **合格布局 = 不乱 + 不挡 + 不越界 + 不误触 + 好维护**

## ② 参考表

| 检查项 | 目标 |
|---|---|
| **层** | 前后顺序清楚 |
| **网格** | 对象排布整齐 |
| **对齐** | 页面专业统一 |
| **边界** | 不超出画面 |
| **按钮** | 易点、不误触 |
| **弹窗** | 不被遮挡 |
| **模板** | 多页面一致 |
| **维护** | 后期容易修改 |

## ③ 计算/选型示例

- 已知：  
  一个页面从20个对象增加到200个对象。
- 公式：  
  **对象数量增加 → 布局管理必须加强**
- 计算：  
  对象越多，层级遮挡、超出范围、误选和卡顿风险越高。
- 结论：  
  大型页面必须使用图层、网格、分组、锁定和边界检查。

> [!tip] 选型口诀
> 先分层，再对齐；先定网格，再放控件；背景锁住，弹窗置顶；对象不越界，按钮不误触。
