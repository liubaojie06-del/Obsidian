# 01 库总览

> [!info] 核心结论
> “库”用于集中管理HMI/SCADA工程中的 **标准对象、按钮、开关、监控组件、图形模板、脚本函数、文档模板和复用资源**，目的是提高开发效率、统一画面风格、减少重复绘制。

## ① 分类/公式/规则

> [!warning] 库使用公式
> **库对象 = 标准外观 + 标准变量接口 + 标准事件动作 + 可复用模板**

## ② 参考表

| 库类型 | 作用 |
|---|---|
| **项目库** | 当前项目专用对象和模板 |
| **全局库** | 多个项目可复用的公共库 |
| **Buttons-and-Switches** | 按钮、开关、选择器类对象 |
| **Long Functions** | 长函数、脚本函数、通用逻辑 |
| **Monitoring-and-control-objects** | 监控与控制对象 |
| **Documentation templates** | 文档模板、说明模板 |
| **状态/版本** | 管理库对象状态和版本 |

## ③ 计算/选型示例

- 已知：  
  多个画面都要用相同风格的启动、停止、复位按钮。
- 公式：  
  **重复对象 → 放入库复用**
- 计算：  
  从库中拖入标准按钮，可以保证颜色、尺寸、变量接口和事件一致。
- 结论：  
  常用对象应标准化到库中，不建议每个画面重新画。

> [!tip] 记忆口诀
> 画一次能复用，就放库；每次重画，迟早不统一。

# 02 项目库

> [!info] 核心结论
> 项目库用于保存当前工程专用的画面对象、组件、模板、图标和脚本资源，只服务于当前项目。

## ① 分类/公式/规则

> [!warning] 项目库规则
> **项目专用、只在本项目使用的对象 → 放项目库**

## ② 参考表

| 适合放入项目库的内容 | 示例 |
|---|---|
| **项目专用按钮组** | 当前设备启动/停止/复位按钮组 |
| **设备组件** | 本项目专用电机、气缸、阀门组件 |
| **工位模板** | 上料工位、检测工位、下料工位 |
| **项目图标** | 特定设备照片、客户LOGO |
| **项目脚本** | 当前项目特有计算逻辑 |
| **报警提示模板** | 当前设备故障提示对象 |
| **配方对象** | 当前产品参数组控件 |

## ③ 计算/选型示例

- 已知：  
  某设备有专用的“压装工位状态组件”，其他项目很少使用。
- 公式：  
  **项目专用对象 → 项目库**
- 计算：  
  放到项目库后，当前工程多个画面都能复用。
- 结论：  
  项目库适合管理本项目内部标准对象。

> [!tip] 记忆口诀
> 项目库服务当前项目，全局库服务多个项目。

# 03 全局库

> [!info] 核心结论
> 全局库用于保存跨项目复用的标准对象，例如公司统一按钮、报警组件、电机组件、阀门组件、图形模板和脚本函数。

## ① 分类/公式/规则

> [!warning] 全局库规则
> **多个项目都会用到的对象 → 放全局库**

## ② 参考表

| 适合放入全局库的内容 | 示例 |
|---|---|
| **标准按钮** | 启动、停止、复位、确认、返回 |
| **标准开关** | 手动/自动、本地/远程 |
| **标准状态灯** | 运行、停止、报警、待机 |
| **标准设备组件** | 电机、气缸、阀门、泵 |
| **标准报警条** | 当前报警提示栏 |
| **标准导航栏** | 首页、自动、手动、报警、维护 |
| **标准脚本函数** | 权限判断、参数校验、日志记录 |

## ③ 计算/选型示例

- 已知：  
  公司所有HMI项目都要求启动按钮为绿色、停止按钮为红色、复位按钮为黄色。
- 公式：  
  **公司统一规范 → 全局库**
- 计算：  
  在全局库中维护一套标准按钮，所有项目统一调用。
- 结论：  
  全局库适合建立企业级HMI标准。

> [!tip] 记忆口诀
> 全局库管标准，项目库管个性。

# 04 项目库 vs 全局库

> [!info] 核心结论
> 项目库偏当前工程专用，全局库偏跨项目共享；选错位置会导致复用困难或库资源混乱。

## ① 分类/公式/规则

> [!warning] 判断规则
> **只给当前项目用 → 项目库**  
> **以后多个项目都要用 → 全局库**

## ② 参考表

| 对比项 | 项目库 | 全局库 |
|---|---|---|
| 使用范围 | 当前项目 | 多个项目 |
| 维护对象 | 项目专用对象 | 公司/团队标准对象 |
| 复用能力 | 项目内复用 | 跨项目复用 |
| 版本影响 | 只影响当前项目 | 可能影响多个项目 |
| 适合内容 | 特定设备组件 | 通用按钮、图标、模板 |
| 维护要求 | 相对灵活 | 更需要版本控制 |

## ③ 计算/选型示例

- 已知：  
  “伺服轴状态组件”在多台设备项目中都要使用。
- 公式：  
  **跨项目复用对象 → 全局库**
- 计算：  
  放在全局库后，新项目可直接调用，不必重新制作。
- 结论：  
  通用组件应沉淀到全局库。

> [!tip] 记忆口诀
> 当前项目用项目库，多个项目用全局库。

# 05 Buttons-and-Switches

> [!info] 核心结论
> Buttons-and-Switches通常包含按钮、开关、选择器、切换按钮和操作类控件，是HMI画面最常用的交互对象库。

## ① 分类/公式/规则

> [!warning] 按钮对象公式
> **按钮/开关 = 外观 + 变量写入 + 状态反馈 + 权限判断**

## ② 参考表

| 对象 | 用途 |
|---|---|
| **启动按钮** | 写入启动命令 |
| **停止按钮** | 写入停止命令 |
| **复位按钮** | 报警或设备复位 |
| **确认按钮** | 确认操作或保存参数 |
| **返回按钮** | 页面跳转 |
| **切换开关** | 手动/自动、本地/远程 |
| **选择按钮** | 模式选择、配方选择 |

## ③ 计算/选型示例

- 已知：  
  需要制作一个手动/自动切换按钮。
- 公式：  
  **模式按钮 = 模式变量 + 当前状态显示 + 权限**
- 计算：  
  当前为自动时显示“自动”，点击后写入手动/自动切换命令。
- 结论：  
  模式类按钮应同时显示当前状态和写入目标状态。

> [!tip] 记忆口诀
> 按钮不只是能点，还要显示状态、限制权限、防止误操作。

# 06 Monitoring-and-control-objects

> [!info] 核心结论
> Monitoring-and-control-objects通常包含监控与控制对象，例如电机、阀门、泵、气缸、仪表、状态灯、报警条和工艺单元组件。

## ① 分类/公式/规则

> [!warning] 监控控制对象公式
> **设备组件 = 图形显示 + 状态变量 + 控制按钮 + 报警反馈 + 详情入口**

## ② 参考表

| 对象 | 常见内容 |
|---|---|
| **电机对象** | 运行、停止、故障、电流、启停 |
| **阀门对象** | 开到位、关到位、开关控制 |
| **泵对象** | 运行状态、故障、压力/流量 |
| **气缸对象** | 伸出、缩回、到位、超时 |
| **仪表对象** | 温度、压力、流量、液位 |
| **报警对象** | 当前报警、报警等级、处理提示 |
| **工位对象** | 工位状态、节拍、产量、报警 |

## ③ 计算/选型示例

- 已知：  
  一个电机对象需要显示运行、故障，并可点击进入详情。
- 公式：  
  **电机组件 = 电机图标 + 运行变量 + 故障变量 + 跳转事件**
- 计算：  
  运行显示绿色，故障显示红色闪烁，点击打开电机详情页。
- 结论：  
  监控对象最好做成标准组件，方便全项目统一使用。

> [!tip] 记忆口诀
> 监控对象要能看状态，控制对象要能安全操作。

# 07 Long Functions

> [!info] 核心结论
> Long Functions通常用于存放较长脚本、通用函数、复杂逻辑和可复用代码片段，适合跨画面调用。

## ① 分类/公式/规则

> [!warning] 函数库公式
> **通用逻辑 = 输入参数 + 处理过程 + 返回结果 + 多处调用**

## ② 参考表

| 函数类型 | 示例 |
|---|---|
| **权限判断函数** | 判断当前用户是否允许操作 |
| **参数校验函数** | 判断输入值是否超限 |
| **日志记录函数** | 记录用户、时间、动作 |
| **页面跳转函数** | 带权限的页面跳转 |
| **报警处理函数** | 报警确认、报警复位 |
| **配方处理函数** | 保存、下载、校验配方 |
| **工程量换算函数** | 原始值转实际工程值 |

## ③ 计算/选型示例

- 已知：  
  多个参数页面都需要判断输入值是否在范围内。
- 公式：  
  **重复逻辑 → 封装成长函数**
- 计算：  
  写一个通用参数校验函数，各参数页面调用。
- 结论：  
  Long Functions适合沉淀通用脚本逻辑，减少重复代码。

> [!tip] 记忆口诀
> 复制三次的脚本，就该考虑做成函数。

# 09 库对象状态

> [!info] 核心结论
> 库对象状态用于表示对象是否可用、是否修改、是否同步、是否需要更新或是否存在版本差异。

## ① 分类/公式/规则

> [!warning] 状态检查规则
> **库对象使用前先看状态，状态异常先确认再调用**

## ② 参考表

| 状态 | 含义 |
|---|---|
| **正常** | 对象可直接使用 |
| **已修改** | 当前对象与库版本可能不同 |
| **待更新** | 有新版本可用 |
| **未同步** | 项目库与全局库不同步 |
| **缺失资源** | 图片、变量或脚本缺失 |
| **不可用** | 版本不兼容或对象损坏 |
| **未验证** | 对象尚未经过项目测试 |

## ③ 计算/选型示例

- 已知：  
  从全局库拖入的按钮在当前项目中显示异常。
- 公式：  
  **库对象异常 = 版本不兼容 / 资源缺失 / 变量接口不匹配**
- 计算：  
  先检查对象状态、资源引用和变量连接。
- 结论：  
  使用库对象后仍需要项目内测试。

> [!tip] 记忆口诀
> 库对象是模板，不是免测试对象。

# 10 库对象版本

> [!info] 核心结论
> 版本用于管理库对象的修改历史，避免不同项目使用不同版本导致显示、变量或脚本不一致。

## ① 分类/公式/规则

> [!warning] 版本管理公式
> **库版本 = 对象编号 + 版本号 + 修改说明 + 测试状态**

## ② 参考表

| 版本信息 | 说明 |
|---|---|
| **版本号** | V1.0、V1.1、V2.0 |
| **修改日期** | 记录更新时间 |
| **修改人** | 记录责任人 |
| **修改内容** | 外观、变量、脚本、权限变化 |
| **兼容性** | 是否兼容旧项目 |
| **测试状态** | 未测试、已测试、已发布 |
| **回退版本** | 出现问题时可恢复 |

## ③ 计算/选型示例

- 已知：  
  标准电机组件从V1.0升级到V2.0，增加了电流显示。
- 公式：  
  **功能变化 → 版本号更新**
- 计算：  
  旧项目如果没有电流变量，直接替换可能报错或显示空白。
- 结论：  
  库对象升级必须说明新增接口和兼容性。

> [!tip] 记忆口诀
> 库对象一旦多人用，就必须管版本。

> 库对象命名应体现对象类别、功能、版本或适用场景，方便搜索、调用和维护。

## ① 分类/公式/规则

> [!warning] 命名公式
> **库对象名称 = 类别 + 功能 + 状态/版本**

## ② 参考表

| 类型 | 示例 |
|---|---|

| **阀门组件** | Obj_Valve_OpenClose_V1 |
| **报警条** | Obj_Alarm_Banner |
| **导航栏** | Nav_Main_Menu |

## ③ 计算/选型示例

- 已知：  
  一个库对象名称叫“对象1”。
- 公式：  
  **名称不清 → 后期难查**
- 计算：  
  维护人员不知道该对象是按钮、阀门还是参数组件。
- 结论：  
  应改为能表达功能的名称，例如Obj_Valve_OpenClose_V1。

> [!tip] 记忆口诀

# 12 库对象变量接口

> [!info] 核心结论
> 复杂库对象通常需要变量接口，例如运行状态、故障状态、命令位、反馈位、数值变量和权限变量。

## ① 分类/公式/规则

> [!warning] 接口公式
> **库组件可复用 = 变量接口清晰 + 命名统一 + 类型一致**

## ② 参考表

| 接口类型 | 示例 |
|---|---|
| **命令变量** | StartCmd、StopCmd、ResetCmd |
| **状态变量** | RunFB、StopFB、Ready |
| **报警变量** | Alarm、FaultCode |
| **反馈变量** | OpenFB、CloseFB、HomeFB |
| **数值变量** | Speed、Pressure、Current |
| **权限变量** | UserLevel、EnableEdit |
| **显示变量** | DeviceName、ModeText |

## ③ 计算/选型示例

- 已知：  
  标准阀门组件需要开命令、关命令、开到位、关到位和报警。
- 公式：  
  **阀门组件接口 = OpenCmd + CloseCmd + OpenFB + CloseFB + Alarm**
- 计算：  
  每次拖入组件后，只需连接这几个变量。
- 结论：  
  标准接口越清楚，库对象越容易复用。

> [!tip] 记忆口诀
> 库对象要好用，变量接口必须标准化。

# 13 库对象拖放使用

> [!info] 核心结论
> 库对象通常可直接拖入画面使用，但拖入后必须检查变量绑定、权限、脚本、层级和显示效果。

## ① 分类/公式/规则

> [!warning] 拖放使用公式
> **拖入库对象 → 绑定变量 → 检查属性 → 运行测试**

## ② 参考表

| 检查项 | 说明 |
|---|---|
| **变量绑定** | 是否连接到当前设备变量 |
| **对象名称** | 是否改成当前设备名称 |
| **权限设置** | 是否符合当前项目权限 |
| **脚本事件** | 是否正常执行 |
| **图层位置** | 是否被遮挡 |
| **尺寸比例** | 是否适合当前画面 |
| **运行测试** | 点击、显示、报警是否正常 |

## ③ 计算/选型示例

- 已知：  
  从库中拖入一个电机组件后，显示“Motor_01”。
- 公式：  
  **库默认名称 → 项目实际名称**
- 计算：  
  应修改为当前设备名称，例如“输送电机1”。
- 结论：  
  库对象拖入后要做项目化配置。

> [!tip] 记忆口诀
> 从库里拿出来，还要接到项目变量上。

# 14 库对象导入导出

> [!info] 核心结论
> 导入导出用于在不同工程、不同电脑或团队之间共享库对象，但必须注意版本、依赖资源和兼容性。

## ① 分类/公式/规则

> [!warning] 导入导出公式
> **可移植库对象 = 对象本体 + 图片资源 + 脚本函数 + 变量说明 + 版本信息**

## ② 参考表

| 导入导出内容 | 注意 |
|---|---|
| **对象本体** | 图形、控件、组合对象 |
| **图片资源** | 图标、背景、设备图片 |
| **脚本函数** | 对象使用的函数 |
| **文本资源** | 多语言文本 |
| **变量接口说明** | 方便接入项目 |
| **版本说明** | 防止错用旧版本 |
| **测试记录** | 确认对象可用 |

## ③ 计算/选型示例

- 已知：  
  导入库对象后图标丢失。
- 公式：  
  **资源缺失 = 对象导出时未包含图片资源**
- 计算：  
  对象本体存在，但引用的图片路径无效。
- 结论：  
  导出库对象时应同时导出依赖资源。

> [!tip] 记忆口诀
> 导对象别只导外壳，图片、脚本、文本也要带走。

# 15 库对象更新

> [!info] 核心结论
> 库对象更新可以把标准组件的新版本应用到项目中，但更新前必须确认是否会覆盖项目内修改。

## ① 分类/公式/规则

> [!warning] 更新规则
> **更新库对象前 = 备份工程 + 查看差异 + 确认兼容 + 运行测试**

## ② 参考表

| 更新风险 | 说明 |
|---|---|
| **覆盖本地修改** | 项目中手动改过的对象可能被覆盖 |
| **变量接口变化** | 新版本变量名或数量不同 |
| **脚本函数变化** | 旧项目函数可能不兼容 |
| **外观变化** | 页面风格可能改变 |
| **权限变化** | 操作权限可能被改变 |
| **多语言缺失** | 新文本未翻译 |
| **尺寸变化** | 可能影响布局 |

## ③ 计算/选型示例

- 已知：  
  全局库中的按钮样式更新，项目内已有按钮做过局部修改。
- 公式：  
  **库更新可能覆盖项目修改**
- 计算：  
  更新前应比较差异，确认是否保留本地修改。
- 结论：  
  库更新不是简单点确认，要先备份和测试。

> [!tip] 记忆口诀
> 库更新能统一标准，也可能覆盖个性修改。

# 16 库对象标准化

> [!info] 核心结论
> 库对象标准化能让不同工程师开发的画面保持一致，减少样式混乱、变量混乱和操作习惯不一致。

## ① 分类/公式/规则

> [!warning] 标准化公式
> **标准库对象 = 统一尺寸 + 统一颜色 + 统一命名 + 统一接口 + 统一权限**

## ② 参考表

| 标准项 | 示例 |
|---|---|
| **颜色标准** | 绿色运行、红色报警、灰色停止 |

| **命名标准** | Btn_、Obj_、Nav_前缀 |

| **权限标准** | 操作员、工程师、管理员 |
| **弹窗标准** | 确认、警告、提示统一样式 |

## ③ 计算/选型示例

- 已知：  
  两个工程师分别制作了不同样式的“复位按钮”。
- 公式：  
  **样式不统一 → 使用标准库按钮**
- 计算：  
  从标准库中统一调用复位按钮对象。
- 结论：  
  标准库能提升项目一致性和专业度。

> [!tip] 记忆口诀
> 标准化不是限制发挥，是减少混乱。

# 18 常见故障排查

> [!info] 核心结论
> 库相关问题排查按 **库位置、对象状态、版本、资源依赖、变量接口、脚本函数、权限、显示层级** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **库对象异常 = 版本不一致 + 资源缺失 + 变量未绑定 + 函数缺失 + 权限错误 + 层级遮挡**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **库对象拖不进画面** | 版本不兼容或库未加载 |
| **对象显示不完整** | 图片资源缺失 |
| **按钮无反应** | 事件脚本未绑定或函数缺失 |
| **状态不变化** | 变量接口未连接 |
| **颜色不对** | 动态规则或状态变量错误 |
| **对象被遮挡** | 图层顺序错误 |
| **权限异常** | 当前项目权限组不匹配 |
| **多语言缺失** | 文本资源未导入 |

## ③ 计算/选型示例

- 已知：  
  从全局库导入的报警条只显示图标，不显示文字。
- 公式：  
  **文字缺失 = 文本资源缺失 / 变量未绑定**
- 计算：  
  检查报警文本资源和报警变量连接。
- 结论：  
  库对象异常时，不只看对象本体，还要看依赖资源。

> [!tip] 记忆口诀
> 库对象坏了，先查依赖：图片、文本、变量、脚本。

# 19 库使用快速检查表

> [!info] 核心结论
> 使用库时，要确认对象来源、适用范围、版本、变量接口、资源依赖和项目测试结果。

## ① 分类/公式/规则

> [!note] 十步检查法
> ① 确认对象来自项目库还是全局库  
> ② 确认对象版本是否适合当前项目  
> ③ 检查对象状态是否正常  
> ④ 检查图片、文本、脚本等依赖资源  
> ⑤ 检查变量接口说明  
> ⑥ 拖入画面后绑定项目变量  
> ⑦ 检查权限和事件动作  
> ⑧ 检查图层、尺寸和布局  
> ⑨ 运行测试正常、异常、报警状态  
> ⑩ 修改后记录版本并备份

> [!warning] 最终检查公式
> **合格库使用 = 来源明确 + 版本正确 + 接口清楚 + 资源完整 + 测试通过**

## ② 参考表

| 检查项 | 目标 |
|---|---|
| **项目库** | 当前项目复用 |
| **全局库** | 跨项目复用 |
| **状态** | 判断对象是否可用 |
| **版本** | 防止新旧混乱 |
| **变量接口** | 保证连接正确 |
| **资源依赖** | 防止图片/脚本丢失 |
| **权限** | 防止误操作 |
| **测试** | 确认现场可用 |

## ③ 计算/选型示例

- 已知：  
  需要为新项目快速搭建标准HMI画面。
- 公式：  
  **新项目开发 = 全局库标准对象 + 项目库专用对象**
- 计算：  
  通用按钮、导航、报警条用全局库；项目专用工位组件放项目库。
- 结论：  
  正确使用项目库和全局库，可以显著提升开发效率和画面一致性。

> [!tip] 选型口诀
> 通用对象进全局，项目对象进项目；拖入先绑变量，使用必须测试；改了记版本，交付要备份。
