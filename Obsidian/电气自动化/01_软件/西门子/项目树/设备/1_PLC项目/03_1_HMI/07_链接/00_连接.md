# 01 连接总览

> [!info] 核心结论
> **连接** 是HMI/SCADA与PLC、控制器、网关、仪表、数据库或服务器之间建立通讯关系的配置入口。  
> HMI变量要读取PLC数据，必须先有正确的连接。

## ① 分类/公式/规则

> [!warning] 连接公式
> **HMI通讯成功 = 通讯驱动 + 连接对象 + IP/站号 + 协议参数 + 变量地址 + 网络正常**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **连接名称** | 给PLC或设备连接起一个清晰名称 |
| **通讯驱动** | 选择PLC品牌或协议类型 |
| **通讯对象** | PLC、网关、仪表、控制器等 |
| **网络参数** | IP地址、端口、子网、网关 |
| **协议参数** | 机架号、槽号、站号、设备ID等 |
| **变量绑定** | HMI变量通过连接访问PLC地址 |
| **诊断状态** | 判断连接是否在线、异常或中断 |

## ③ 计算/选型示例

- 已知：  
  HMI要读取PLC里的电机运行状态。
- 公式：  
  **PLC连接 → HMI变量 → 画面对象**
- 计算：  
  先创建PLC连接，再建立变量Motor1_RunFB，最后绑定到指示灯。
- 结论：  
  没有连接，HMI变量就无法访问PLC数据。

> [!tip] 记忆口诀
> 连接是路，变量是车，画面是仪表盘；路不通，数据到不了。

# 02 连接的作用

> [!info] 核心结论
> 连接用于定义HMI和外部设备之间“怎么通信、找谁通信、用什么协议通信”。

## ① 分类/公式/规则

> [!warning] 连接作用公式
> **连接 = 通讯对象 + 通讯协议 + 通讯参数**

## ② 参考表

| 作用 | 说明 |
|---|---|
| **确定设备** | HMI要连接哪台PLC或设备 |
| **确定协议** | 使用S7、Modbus、OPC UA等协议 |
| **确定地址** | IP、站号、端口、机架槽号 |
| **支撑变量** | 外部变量通过连接访问设备地址 |
| **支撑报警** | 报警变量来自连接设备 |
| **支撑趋势** | 趋势变量通过连接采集 |
| **支撑控制** | 按钮命令通过连接写入PLC |

## ③ 计算/选型示例

- 已知：  
  HMI上有一个启动按钮，需要控制PLC中的启动位。
- 公式：  
  **按钮 → HMI变量 → PLC连接 → PLC地址**
- 计算：  
  按钮写入变量StartCmd，变量通过PLC连接写入PLC对应地址。
- 结论：  
  按钮动作能否生效，取决于连接、变量和PLC逻辑都正确。

> [!tip] 记忆口诀
> 画面按钮不是直接控制设备，而是通过连接写变量。

# 03 连接对象

> [!info] 核心结论
> 连接对象是HMI要通讯的外部设备，常见包括PLC、工业网关、远程I/O、变频器、伺服、仪表和服务器。

## ① 分类/公式/规则

> [!warning] 对象选择规则
> **连接对象是谁，通讯驱动和参数就按谁来选**

## ② 参考表

| 连接对象 | 常见用途 |
|---|---|
| **PLC** | 启停、状态、报警、参数、配方 |
| **工业网关** | 多设备数据采集和协议转换 |
| **远程I/O** | 输入输出点状态 |
| **变频器** | 频率、电流、故障码 |
| **伺服驱动器** | 位置、速度、报警、使能 |
| **仪表** | 温度、压力、流量、液位 |
| **数据库/服务器** | 生产数据、报表、历史记录 |

## ③ 计算/选型示例

- 已知：  
  HMI需要同时连接一台PLC和一台工业网关。
- 公式：  
  **不同设备 → 建立不同连接**
- 计算：  
  PLC连接用于控制变量，网关连接用于采集第三方设备数据。
- 结论：  
  多设备项目通常需要多个连接。

> [!tip] 记忆口诀
> 一个连接对应一个通讯对象或一类通讯通道。

# 04 通讯驱动

> [!info] 核心结论
> 通讯驱动决定HMI使用什么协议与设备通信，驱动选错会导致变量无法读取。

## ① 分类/公式/规则

> [!warning] 驱动匹配规则
> **PLC品牌/协议 = HMI通讯驱动**

## ② 参考表

| 通讯驱动/协议 | 常见对象 |
|---|---|
| **Siemens S7** | 西门子PLC |
| **Modbus TCP** | 通用PLC、网关、仪表 |
| **Modbus RTU** | 串口仪表、远程I/O |
| **OPC UA** | SCADA、网关、服务器 |
| **EtherNet/IP** | 罗克韦尔PLC和部分设备 |
| **PROFINET** | 工业设备与控制系统 |
| **自定义协议** | 专用设备或老设备 |

## ③ 计算/选型示例

- 已知：  
  PLC是西门子S7-1200。
- 公式：  
  **西门子PLC → 选择S7通讯驱动**
- 计算：  
  若误选Modbus TCP，变量地址格式和通讯方式都可能不匹配。
- 结论：  
  连接第一步就是选对通讯驱动。

> [!tip] 记忆口诀
> 驱动选错，IP写对也没用。

# 05 连接名称

> [!info] 核心结论
> 连接名称用于区分不同PLC或设备，命名应清晰表达设备对象、区域和用途。

## ① 分类/公式/规则

> [!warning] 命名公式
> **连接名称 = 设备对象 + 区域/编号 + 用途**

## ② 参考表

| 命名 | 含义 |
|---|---|
| **PLC_Main** | 主PLC连接 |
| **PLC_Station01** | 1号工位PLC |
| **Gateway_Energy** | 能耗网关 |
| **PLC_PackingLine** | 包装线PLC |
| **Modbus_Meter_RS485** | RS485电表连接 |
| **OPCUA_Server_MES** | MES OPC UA服务器连接 |
| **PLC_Safety** | 安全PLC连接 |

## ③ 计算/选型示例

- 已知：  
  工程里有3台PLC，如果都叫“PLC_1、PLC_2、PLC_3”。
- 公式：  
  **命名不清 → 后期易绑错变量**
- 计算：  
  工程师可能不知道PLC_2对应哪个设备区域。
- 结论：  
  建议命名为PLC_Loading、PLC_Testing、PLC_Unloading等。

> [!tip] 记忆口诀
> 连接名称要让人一眼知道连的是谁。

# 06 IP地址与网络参数

> [!info] 核心结论
> 以太网连接必须配置正确的IP地址、子网掩码、网关和端口，网络不通时HMI无法读取PLC数据。

## ① 分类/公式/规则

> [!warning] 网络可达公式
> **网络可达 = IP正确 + 网段正确 + 网线正常 + 交换机正常 + 防火墙允许**

## ② 参考表

| 参数 | 说明 |
|---|---|
| **IP地址** | PLC或设备的网络地址 |
| **子网掩码** | 判断是否同一网段 |
| **网关** | 跨网段通讯需要 |
| **端口号** | 通讯服务端口 |
| **MAC地址** | 网络硬件地址 |
| **DNS** | 域名访问时使用 |
| **防火墙** | 可能阻挡通讯端口 |

## ③ 计算/选型示例

- 已知：  
  HMI IP为192.168.1.100，PLC IP为192.168.2.10，没有路由。
- 公式：  
  **不同网段 + 无路由 → 通讯失败**
- 计算：  
  192.168.1.x和192.168.2.x不在同一网段。
- 结论：  
  应改成同网段，或配置网关路由。

> [!tip] 记忆口诀
> IP不对连不上，网段不对也连不上。

# 07 机架号与槽号

> [!info] 核心结论
> 部分PLC连接需要配置机架号和槽号，尤其是西门子PLC通讯中常见。

## ① 分类/公式/规则

> [!warning] 机架槽号规则
> **通讯参数必须与PLC实际CPU位置匹配**

## ② 参考表

| 参数 | 说明 |
|---|---|
| **机架号 Rack** | PLC所在机架编号 |
| **槽号 Slot** | CPU所在槽位 |
| **连接资源** | PLC允许的通讯连接数量 |
| **访问权限** | 是否允许HMI访问 |
| **优化访问** | 部分变量访问方式相关 |
| **符号访问** | 通过变量名访问 |
| **绝对地址访问** | 通过DB/M/I/Q地址访问 |

## ③ 计算/选型示例

- 已知：  
  HMI能Ping通PLC，但变量读不到。
- 公式：  
  **Ping通 ≠ PLC协议参数正确**
- 计算：  
  可能是机架号、槽号或PLC访问权限设置错误。
- 结论：  
  网络通后，还要检查PLC协议参数。

> [!tip] 记忆口诀
> Ping通只是网通，机架槽号对了才可能数据通。

# 08 站号与设备地址

> [!info] 核心结论
> 串口或总线通讯中常用站号、设备地址、从站地址来区分不同设备。

## ① 分类/公式/规则

> [!warning] 站号规则
> **同一总线上的设备站号不能重复**

## ② 参考表

| 参数 | 常见场景 |
|---|---|
| **Modbus站号** | RS485仪表、电表、温控器 |
| **从站地址** | 总线设备编号 |
| **设备ID** | 网关或协议设备编号 |
| **节点号** | CANopen、现场总线 |
| **串口号** | COM口或网关通道 |
| **波特率** | 串口通讯速度 |
| **校验位** | 串口通讯参数 |

## ③ 计算/选型示例

- 已知：  
  RS485总线上有两台仪表站号都为5。
- 公式：  
  **站号重复 → 应答冲突**
- 计算：  
  HMI访问5号站时，两台设备可能同时响应。
- 结论：  
  必须修改其中一台设备站号。

> [!tip] 记忆口诀
> 一条总线上，站号必须唯一。

# 09 串口连接

> [!info] 核心结论
> 串口连接常用于HMI连接仪表、电表、温控器、串口服务器或老旧PLC，重点是串口参数必须一致。

## ① 分类/公式/规则

> [!warning] 串口成功公式
> **串口通讯成功 = 波特率一致 + 数据位一致 + 停止位一致 + 校验位一致 + 接线正确**

## ② 参考表

| 参数 | 常见值 |
|---|---|
| **接口类型** | RS232、RS485、RS422 |
| **波特率** | 9600、19200、38400、115200 |
| **数据位** | 7、8 |
| **停止位** | 1、2 |
| **校验位** | None、Even、Odd |
| **站号** | Modbus RTU等协议需要 |
| **超时时间** | 等待设备响应时间 |

## ③ 计算/选型示例

- 已知：  
  仪表通讯参数为9600、8、N、1。
- 公式：  
  **HMI串口参数必须一致**
- 计算：  
  HMI连接也应设置9600波特率、8数据位、无校验、1停止位。
- 结论：  
  串口参数不一致时，容易无响应或乱码。

> [!tip] 记忆口诀
> 串口不通，先查9600、8、N、1。

# 10 以太网连接

> [!info] 核心结论
> 以太网连接是HMI与PLC、网关、服务器通讯最常见方式，优点是速度快、布线方便、扩展性好。

## ① 分类/公式/规则

> [!warning] 以太网连接公式
> **以太网连接成功 = IP可达 + 协议端口开放 + 驱动匹配 + 地址正确**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **网线** | 物理连接 |
| **交换机** | 多设备组网 |
| **IP地址** | 设备网络地址 |
| **端口号** | 协议服务入口 |
| **通讯协议** | S7、Modbus TCP、OPC UA等 |
| **防火墙** | 可能阻止端口 |
| **网络诊断** | Ping、端口测试、通讯状态 |

## ③ 计算/选型示例

- 已知：  
  HMI能Ping通PLC，但Modbus TCP读不到数据。
- 公式：  
  **Ping通 ≠ 端口和协议通**
- 计算：  
  可能PLC未开启Modbus TCP服务，或502端口被阻挡。
- 结论：  
  网络层通后，还要检查协议端口和设备服务。

> [!tip] 记忆口诀
> Ping通只是第一步，协议通才是真的连接成功。

# 11 连接与HMI变量

> [!info] 核心结论
> HMI外部变量必须选择对应连接，变量地址才知道要去哪个PLC或设备读取。

## ① 分类/公式/规则

> [!warning] 变量连接公式
> **外部变量 = 连接名称 + 设备地址 + 数据类型**

## ② 参考表

| 变量配置项 | 说明 |
|---|---|
| **变量名** | HMI内部使用名称 |
| **连接** | 指向哪台PLC或设备 |
| **地址** | PLC地址、寄存器地址、节点 |
| **数据类型** | Bool、Int、Real、String |
| **采集周期** | 多久刷新一次 |
| **读写属性** | 只读、只写、读写 |
| **注释** | 变量说明 |

## ③ 计算/选型示例

- 已知：  
  两台PLC都有DB10.DBD0地址。
- 公式：  
  **同一地址 + 不同连接 = 不同数据**
- 计算：  
  PLC1.DB10.DBD0和PLC2.DB10.DBD0不是同一个变量。
- 结论：  
  多PLC工程中，变量必须选对连接。

> [!tip] 记忆口诀
> 地址相同不代表变量相同，还要看连接是哪台设备。

# 12 连接与报警

> [!info] 核心结论
> 报警变量通常来自PLC连接，连接中断时可能导致报警不刷新、通讯报警触发或报警状态异常。

## ① 分类/公式/规则

> [!warning] 报警链路
> **PLC报警位 → HMI连接 → 报警变量 → 报警控件**

## ② 参考表

| 报警相关项 | 说明 |
|---|---|
| **报警位** | PLC中的Bool报警变量 |
| **报警字** | Word中的多个报警位 |
| **故障码** | 数值对应故障类型 |
| **连接状态** | 离线时报警可能失效 |
| **通讯报警** | HMI与PLC连接中断 |
| **时间戳** | 报警发生时间 |
| **报警文本** | 故障说明和处理建议 |

## ③ 计算/选型示例

- 已知：  
  PLC断线后，HMI报警画面没有更新。
- 公式：  
  **连接中断 → 外部报警变量无法刷新**
- 计算：  
  HMI无法继续读取PLC报警位。
- 结论：  
  应配置通讯中断报警，提示PLC连接异常。

> [!tip] 记忆口诀
> 设备报警靠变量，通讯报警靠连接状态。

# 13 连接与趋势

> [!info] 核心结论
> 趋势数据依赖连接周期性读取变量，连接不稳定会导致趋势曲线断点、数据缺失或历史记录异常。

## ① 分类/公式/规则

> [!warning] 趋势链路
> **PLC过程值 → HMI连接 → 趋势变量 → 历史记录**

## ② 参考表

| 趋势问题 | 可能原因 |
|---|---|
| **曲线断点** | 连接中断 |
| **数据不刷新** | 变量地址或连接错误 |
| **数值异常** | 数据类型不匹配 |
| **刷新过慢** | 采集周期太长 |
| **系统卡顿** | 变量太多、周期太快 |
| **历史缺失** | 记录服务异常 |
| **时间错乱** | 系统时间不同步 |

## ③ 计算/选型示例

- 已知：  
  压力趋势每秒记录一次，但连接每5秒刷新一次。
- 公式：  
  **趋势有效刷新受连接周期限制**
- 计算：  
  即使趋势设置1秒采样，变量本身5秒才更新一次。
- 结论：  
  连接刷新周期和趋势采样周期要协调。

> [!tip] 记忆口诀
> 趋势采得再快，连接不刷新也没用。

# 14 连接数量

> [!info] 核心结论
> HMI可建立多个连接，但连接数量过多会增加通讯负载，PLC也有最大连接资源限制。

## ① 分类/公式/规则

> [!warning] 连接负载公式
> **通讯负载 = 连接数量 × 变量数量 × 刷新频率**

## ② 参考表

| 项目 | 影响 |
|---|---|
| **连接数量** | 多设备通讯需求 |
| **变量数量** | 采集点越多负载越大 |
| **刷新周期** | 刷新越快负载越高 |
| **PLC连接资源** | PLC允许的HMI连接数量有限 |
| **网络带宽** | 大量数据可能占用带宽 |
| **HMI性能** | 变量过多会影响刷新 |
| **通讯分组** | 可优化读取效率 |

## ③ 计算/选型示例

- 已知：  
  HMI连接5台PLC，每台PLC读取500个变量。
- 公式：  
  **总变量数 = 5 × 500**
- 计算：  
  总变量数 = **2500个变量**
- 结论：  
  应评估HMI性能、PLC通讯资源和刷新周期设置。

> [!tip] 记忆口诀
> 连接越多、变量越多、刷新越快，通讯压力越大。

# 15 连接状态诊断

> [!info] 核心结论
> 连接状态诊断用于判断HMI与PLC或设备是否在线，是排查通讯故障的重要依据。

## ① 分类/公式/规则

> [!warning] 诊断公式
> **连接诊断 = 在线状态 + 错误码 + 最后通讯时间 + 重连状态**

## ② 参考表

| 诊断项 | 说明 |
|---|---|
| **在线/离线** | 当前连接是否正常 |
| **错误码** | 通讯失败原因 |
| **最后通讯时间** | 最近一次成功通讯时间 |
| **重连次数** | 网络不稳定判断 |
| **超时次数** | 设备无响应次数 |
| **变量质量** | 数据有效、无效、坏质量 |
| **通讯日志** | 记录连接异常 |

## ③ 计算/选型示例

- 已知：  
  画面数值显示为“---”。
- 公式：  
  **数值无效 = 连接离线 / 变量质量坏 / 地址错误**
- 计算：  
  先检查连接状态是否在线，再查变量地址。
- 结论：  
  连接诊断能快速判断是通讯问题还是变量问题。

> [!tip] 记忆口诀
> 数据不显示，先看连接在线不在线。

# 16 连接超时与重连

> [!info] 核心结论
> 超时和重连参数决定连接异常时HMI多久判定失败、多久尝试恢复通讯。

## ① 分类/公式/规则

> [!warning] 超时规则
> **超时时间太短易误报，太长故障发现慢**

## ② 参考表

| 参数 | 说明 |
|---|---|
| **通讯超时** | 等待设备响应的最长时间 |
| **重试次数** | 通讯失败后再次尝试 |
| **重连周期** | 离线后多久重新连接 |
| **心跳检测** | 判断连接是否仍然有效 |
| **离线报警延时** | 防止瞬间网络波动误报警 |
| **恢复检测** | 连接恢复后更新状态 |
| **日志记录** | 记录断线和恢复时间 |

## ③ 计算/选型示例

- 已知：  
  现场网络偶尔波动100ms，HMI超时设置为50ms。
- 公式：  
  **超时设置 < 网络波动 → 易误报通讯中断**
- 计算：  
  50ms过短，可能频繁报离线。
- 结论：  
  超时时间应结合现场网络质量设置。

> [!tip] 记忆口诀
> 超时太短容易假报警，太长又发现慢。

# 18 连接安全

> [!info] 核心结论
> HMI连接可能具备写入PLC参数和命令的能力，因此必须关注访问权限、网络隔离、写入保护和操作日志。

## ① 分类/公式/规则

> [!warning] 连接安全公式
> **安全连接 = 网络隔离 + 权限控制 + 写入限制 + 操作日志 + PLC互锁**

## ② 参考表

| 安全项 | 作用 |
|---|---|
| **控制网隔离** | 防止无关网络访问PLC |
| **用户权限** | 限制谁能写变量 |
| **只读变量** | 防止误写反馈值 |
| **参数范围** | 防止写入危险值 |
| **操作日志** | 记录谁写了什么 |
| **PLC互锁** | 最终安全判断 |
| **防火墙** | 限制通讯来源 |

## ③ 计算/选型示例

- 已知：  
  HMI可以直接写入伺服位置参数。
- 公式：  
  **关键参数写入 = 权限 + 范围 + 日志 + PLC校验**
- 计算：  
  如果没有限制，误输入可能导致机构撞机。
- 结论：  
  写入类连接变量必须加权限和范围保护。

> [!tip] 记忆口诀
> 连接能读数据，也能写命令；能写就必须管安全。

# 19 常见故障排查

> [!info] 核心结论
> 连接故障排查按 **物理网络、IP网段、通讯驱动、协议参数、PLC权限、变量地址、数据类型、防火墙** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **连接异常 = 网线问题 + IP错误 + 驱动错误 + 参数错误 + 权限不足 + 地址错误**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **连接离线** | 网线、IP、PLC掉电、交换机异常 |
| **Ping不通** | IP错误、网段错误、防火墙 |
| **Ping通但读不到变量** | 驱动、协议参数、PLC访问权限 |
| **部分变量异常** | 地址错、类型错、字节序错 |
| **按钮写入无效** | 写入地址错、权限不足、PLC互锁 |
| **频繁掉线** | 网络不稳、超时太短、交换机问题 |
| **趋势断点** | 连接中断或变量质量差 |
| **报警不刷新** | 报警变量连接异常 |

## ③ 计算/选型示例

- 已知：  
  HMI能Ping通PLC，但所有变量都显示无效。
- 公式：  
  **Ping通 + 变量无效 = 协议层问题**
- 计算：  
  重点检查通讯驱动、机架槽号、PLC访问权限和变量地址格式。
- 结论：  
  不要只看网络通不通，要继续检查协议和变量配置。

> [!tip] 记忆口诀
> 网络通只是第一关，协议通和变量对才是真通。

# 20 连接快速检查表

> [!info] 核心结论
> 连接检查的目标是确保HMI能稳定、正确、安全地读取和写入外部设备数据。

## ① 分类/公式/规则

> [!note] 十步检查法
> ① 确认连接对象是谁  
> ② 确认通讯驱动是否匹配  
> ③ 检查IP地址、端口、站号或串口参数  
> ④ 检查PLC或设备是否允许HMI访问  
> ⑤ 检查HMI与设备是否同网段或路由可达  
> ⑥ 检查连接名称是否清晰  
> ⑦ 检查变量是否选择正确连接  
> ⑧ 检查变量地址和数据类型是否正确  
> ⑨ 检查连接状态诊断和通讯报警  
> ⑩ 上线前做断网、重连、读写和报警测试

> [!warning] 最终检查公式
> **合格连接 = 驱动正确 + 参数正确 + 网络可达 + 变量正确 + 诊断清楚 + 安全可控**

## ② 参考表

| 检查项 | 目标 |
|---|---|
| **驱动** | 与PLC/设备协议匹配 |
| **IP/站号** | 能找到正确设备 |
| **端口/串口** | 通讯参数一致 |
| **变量连接** | 指向正确设备 |
| **数据类型** | 显示解析正确 |
| **读写权限** | 防止误操作 |
| **诊断** | 故障能快速定位 |
| **测试** | 读写和断线恢复正常 |

## ③ 计算/选型示例

- 已知：  
  新建HMI工程需要连接主PLC、能耗网关和MES服务器。
- 公式：  
  **三类对象 → 三类连接**
- 计算：  
  主PLC连接用于控制和状态；能耗网关连接用于采集电表数据；MES连接用于上传生产数据。
- 结论：  
  应分别建立清晰命名的连接，并按连接分组管理变量。

> [!tip] 选型口诀
> 先建连接，再建变量；先查网络，再查协议；显示看反馈，写入看权限；多设备要分组，断线要报警。
