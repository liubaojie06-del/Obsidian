# 01 用户管理总览

> [!info] 核心结论
> **用户管理** 用于管理HMI/SCADA系统中的账号、密码、角色、权限、登录、注销、自动退出和操作记录，核心目的是 **防止误操作、限制关键功能、追溯用户行为**。

## ① 分类/公式/规则

> [!warning] 用户管理公式
> **用户管理 = 用户账号 + 角色分组 + 权限等级 + 登录验证 + 操作日志**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **用户账号** | 操作员、工程师、管理员等登录账号 |
| **密码管理** | 设置密码、修改密码、密码策略 |
| **角色管理** | 按岗位分配权限 |
| **权限控制** | 限制页面、按钮、参数、配方等操作 |
| **登录注销** | 用户进入和退出系统权限 |
| **自动注销** | 长时间无操作后退出登录 |
| **操作日志** | 记录谁在什么时候做了什么 |
| **安全策略** | 防止默认密码、弱密码和越权操作 |

## ③ 计算/选型示例

- 已知：  
  设备参数画面允许修改伺服速度、位置和加速度。
- 公式：  
  **关键参数修改 = 用户登录 + 工程师权限 + 操作记录**
- 计算：  
  普通操作员只能查看，工程师登录后才允许修改。
- 结论：  
  用户管理是HMI安全运行的重要基础。

> [!tip] 记忆口诀
> 谁能看、谁能改、谁改过，都要靠用户管理。

# 02 用户账号

> [!info] 核心结论
> 用户账号用于区分不同操作人员和维护人员，每个账号应对应真实岗位或责任人，避免多人共用高权限账号。

## ① 分类/公式/规则

> [!warning] 账号规则
> **一个账号对应一个用户或一个明确岗位，不建议多人共用管理员账号**

## ② 参考表

| 账号类型 | 典型用途 |
|---|---|
| **操作员账号** | 日常启停、复位、查看报警 |
| **班组长账号** | 部分参数修改、报警确认 |
| **工程师账号** | 调试、参数、配方、维护 |
| **管理员账号** | 用户管理、权限配置、系统设置 |
| **维护账号** | I/O诊断、通讯诊断、手动调试 |
| **只读账号** | 只查看数据，不允许操作 |
| **审计账号** | 查看日志、历史记录和操作追溯 |

## ③ 计算/选型示例

- 已知：  
  现场所有人都使用同一个管理员账号。
- 公式：  
  **共用管理员账号 → 无法追溯具体责任人**
- 计算：  
  参数被修改后，只能看到管理员账号操作，无法知道是谁改的。
- 结论：  
  关键项目应避免多人共用高权限账号。

> [!tip] 记忆口诀
> 账号越共用，追溯越困难。

# 03 用户角色

> [!info] 核心结论
> 角色用于按岗位分配权限，比单独给每个用户设置权限更清晰、更容易维护。

## ① 分类/公式/规则

> [!warning] 角色权限公式
> **用户权限 = 用户所属角色的权限集合**

## ② 参考表

| 角色 | 典型权限 |
|---|---|
| **操作员** | 启动、停止、复位、查看当前报警 |
| **班组长** | 报警确认、部分参数修改、班次数据查看 |
| **工程师** | 手动调试、参数设置、配方下载、I/O诊断 |
| **管理员** | 用户管理、权限配置、数据清理、系统设置 |
| **维护人员** | 设备诊断、手动动作、通讯检查 |
| **质量人员** | 查询记录、导出报表、查看配方版本 |
| **访客/只读** | 查看状态，不允许控制 |

## ③ 计算/选型示例

- 已知：  
  需要给5名操作员相同权限。
- 公式：  
  **相同权限 → 建立一个操作员角色**
- 计算：  
  5名用户全部加入“操作员”角色即可。
- 结论：  
  角色管理比逐个账号配置权限更高效。

> [!tip] 记忆口诀
> 用户负责登录，角色负责权限。

# 04 权限等级

> [!info] 核心结论
> 权限等级用于控制用户能否访问画面、点击按钮、修改参数、下载配方、清空记录和进入维护功能。

## ① 分类/公式/规则

> [!warning] 权限判断公式
> **操作允许 = 当前用户权限 ≥ 功能要求权限**

## ② 参考表

| 功能 | 推荐权限 |
|---|---|
| **查看首页** | 所有用户 |
| **启动/停止设备** | 操作员及以上 |
| **报警确认** | 操作员及以上 |
| **报警复位** | 班组长或工程师 |
| **参数修改** | 工程师及以上 |
| **配方删除** | 管理员 |
| **清空历史记录** | 管理员 + 二次确认 |
| **用户管理** | 管理员 |

## ③ 计算/选型示例

- 已知：  
  某页面要求权限等级3，当前用户权限等级1。
- 公式：  
  **1 < 3 → 禁止访问**
- 计算：  
  系统应拒绝进入页面，或弹出登录窗口。
- 结论：  
  权限等级可以防止低权限用户进入高风险页面。

> [!tip] 记忆口诀
> 权限不够，按钮不能按，页面不能进。

# 05 登录与注销

> [!info] 核心结论
> 登录用于获取对应权限，注销用于退出当前权限，防止人员离开后其他人继续使用高权限操作。

## ① 分类/公式/规则

> [!warning] 登录公式
> **登录成功 = 用户名正确 + 密码正确 + 账号有效 + 权限加载成功**

## ② 参考表

| 操作 | 说明 |
|---|---|
| **用户登录** | 输入用户名和密码 |
| **用户注销** | 退出当前账号权限 |
| **切换用户** | 从一个账号切换到另一个账号 |
| **登录失败提示** | 用户名或密码错误 |
| **账号禁用** | 被停用账号不能登录 |
| **登录记录** | 记录登录用户和时间 |
| **注销记录** | 记录退出时间 |

## ③ 计算/选型示例

- 已知：  
  工程师登录后修改完参数，没有注销就离开现场。
- 公式：  
  **高权限未注销 → 误操作风险**
- 计算：  
  其他人可能继续使用工程师权限修改参数。
- 结论：  
  高权限用户操作完成后应及时注销。

> [!tip] 记忆口诀
> 登录拿权限，离开要注销。

# 06 自动注销

> [!info] 核心结论
> 自动注销用于在用户长时间无操作后自动退出登录，防止高权限账号长时间停留在HMI上。

## ① 分类/公式/规则

> [!warning] 自动注销公式
> **无操作时间 ≥ 自动注销时间 → 用户自动退出**

## ② 参考表

| 设置项 | 说明 |
|---|---|
| **无操作时间** | 多久没有触摸或点击 |
| **自动注销时间** | 例如5分钟、10分钟、30分钟 |
| **注销后权限** | 回到默认操作员或访客权限 |
| **页面返回** | 可返回首页或登录页 |
| **提示弹窗** | 注销前提醒 |
| **高权限专用** | 工程师和管理员建议更短 |
| **日志记录** | 记录自动注销事件 |

## ③ 计算/选型示例

- 已知：  
  自动注销时间设置为10分钟，工程师登录后12分钟未操作。
- 公式：  
  **12分钟 ≥ 10分钟 → 自动注销**
- 计算：  
  系统退出工程师账号，恢复默认权限。
- 结论：  
  自动注销能降低高权限遗留风险。

> [!tip] 记忆口诀
> 人走权限退，自动注销来保护。

# 07 密码管理

> [!info] 核心结论
> 密码管理用于防止弱密码、默认密码和长期不变密码带来的安全风险。

## ① 分类/公式/规则

> [!warning] 密码安全公式
> **安全密码 = 长度足够 + 不易猜 + 定期更换 + 不共用**

## ② 参考表

| 密码策略 | 说明 |
|---|---|
| **最小长度** | 建议不少于6~8位 |
| **复杂度要求** | 字母、数字、符号组合 |
| **禁止默认密码** | 上线前必须修改默认密码 |
| **定期修改** | 按项目安全要求更新 |
| **错误次数限制** | 多次失败可锁定账号 |
| **密码隐藏** | 输入时不明文显示 |
| **管理员重置** | 忘记密码时由管理员处理 |

## ③ 计算/选型示例

- 已知：  
  管理员账号密码仍是默认值“123456”。
- 公式：  
  **默认密码 = 高风险**
- 计算：  
  任何知道默认密码的人都可能进入系统修改关键配置。
- 结论：  
  设备上线前必须修改默认密码。

> [!tip] 记忆口诀
> 默认密码不能留，管理员密码要管住。

# 08 页面权限

> [!info] 核心结论
> 页面权限用于限制不同用户能否进入某些画面，例如参数画面、手动画面、维护画面、系统设置画面。

## ① 分类/公式/规则

> [!warning] 页面访问公式
> **页面可进入 = 当前用户权限满足页面要求**

## ② 参考表

| 页面 | 推荐权限 |
|---|---|
| **首页** | 所有用户 |
| **自动运行画面** | 操作员及以上 |
| **报警画面** | 操作员及以上 |
| **趋势画面** | 操作员及以上 |
| **参数画面** | 工程师及以上 |
| **手动画面** | 工程师或维护人员 |
| **系统设置画面** | 管理员 |
| **用户管理画面** | 管理员 |

## ③ 计算/选型示例

- 已知：  
  普通操作员可以进入伺服调试页面。
- 公式：  
  **危险调试页面 → 高权限限制**
- 计算：  
  操作员误操作可能导致轴移动或设备损坏。
- 结论：  
  伺服、手动、系统设置等页面必须加权限。

> [!tip] 记忆口诀
> 生产页面给操作员，调试页面给工程师，系统页面给管理员。

# 09 按钮权限

> [!info] 核心结论
> 按钮权限用于限制用户能否执行具体动作，例如复位、保存、下载、清零、删除、手动动作等。

## ① 分类/公式/规则

> [!warning] 按钮权限公式
> **按钮可操作 = 用户权限满足 + 设备条件满足 + 按钮未禁用**

## ② 参考表

| 按钮 | 推荐权限 |
|---|---|
| **启动按钮** | 操作员及以上 |
| **停止按钮** | 操作员及以上 |
| **报警复位** | 班组长或工程师 |
| **参数保存** | 工程师 |
| **配方下载** | 班组长或工程师 |
| **配方删除** | 管理员 |
| **清空记录** | 管理员 + 二次确认 |
| **手动动作按钮** | 工程师或维护人员 |

## ③ 计算/选型示例

- 已知：  
  “清空产量”按钮所有人都能点击。
- 公式：  
  **清空类操作 = 权限 + 二次确认**
- 计算：  
  误点会导致统计数据丢失。
- 结论：  
  清空、删除、覆盖类按钮必须加权限和确认弹窗。

> [!tip] 记忆口诀
> 危险按钮不裸奔，权限确认都要有。

# 10 参数权限

> [!info] 核心结论
> 参数权限用于防止低权限人员修改速度、位置、温度、压力、配方等关键工艺参数。

## ① 分类/公式/规则

> [!warning] 参数保护公式
> **参数可修改 = 用户权限 + 参数范围 + 保存确认 + 修改记录**

## ② 参考表

| 参数类型 | 推荐权限 |
|---|---|
| **普通生产参数** | 班组长或工程师 |
| **速度参数** | 工程师 |
| **位置参数** | 工程师 |
| **温度压力参数** | 工程师 |
| **检测阈值** | 工程师或质量人员 |
| **配方参数** | 工程师 |
| **安全相关参数** | 管理员或授权工程师 |

## ③ 计算/选型示例

- 已知：  
  操作员把伺服速度从300改成800。
- 公式：  
  **关键参数误改 → 设备异常风险**
- 计算：  
  速度过高可能导致撞机、报警或产品不良。
- 结论：  
  关键参数应限制工程师权限，并记录旧值新值。

> [!tip] 记忆口诀
> 参数能改是方便，权限不管就是风险。

# 11 配方权限

> [!info] 核心结论
> 配方权限用于控制谁能查看、加载、下载、保存、覆盖、删除配方。

## ① 分类/公式/规则

> [!warning] 配方权限公式
> **配方操作允许 = 用户角色 + 操作类型 + 配方状态**

## ② 参考表

| 配方操作 | 推荐权限 |
|---|---|
| **查看配方** | 操作员及以上 |
| **加载配方** | 操作员或班组长 |
| **下载配方** | 班组长或工程师 |
| **保存配方** | 工程师 |
| **覆盖配方** | 工程师或管理员 |
| **删除配方** | 管理员 |
| **导入导出配方** | 工程师或管理员 |

## ③ 计算/选型示例

- 已知：  
  操作员可以删除产品配方。
- 公式：  
  **删除配方 = 高风险操作**
- 计算：  
  删除后可能无法恢复产品工艺参数。
- 结论：  
  删除配方必须限制管理员权限，并加二次确认。

> [!tip] 记忆口诀

> [!info] 核心结论
> 报警权限用于限制报警确认、复位、清空历史、修改报警阈值等操作。

## ① 分类/公式/规则

> [!warning] 报警权限公式
> **报警操作权限 = 报警等级 + 操作类型 + 用户角色**

## ② 参考表

| 报警操作 | 推荐权限 |
|---|---|
| **查看当前报警** | 所有用户 |
| **查看历史报警** | 操作员及以上 |
| **确认报警** | 操作员及以上 |
| **普通报警复位** | 班组长或工程师 |
| **安全报警复位** | 工程师或管理员 |
| **清空报警历史** | 管理员 |
| **修改报警阈值** | 工程师或管理员 |

## ③ 计算/选型示例

- 已知：  
  普通操作员可以清空报警历史。
- 公式：  
  **清空报警历史 → 追溯数据丢失**
- 计算：  
  后期无法分析故障发生时间和责任。
- 结论：  
  清空报警历史应限制管理员权限。

# 13 记录与数据权限

> [!info] 核心结论
> 记录权限用于控制谁能查看、筛选、导出、删除报警记录、操作记录、趋势记录和生产记录。

## ① 分类/公式/规则

> [!warning] 数据权限公式
> **数据操作权限 = 数据重要性 + 操作风险 + 用户角色**

## ② 参考表

| 数据操作 | 推荐权限 |
|---|---|
| **查看记录** | 操作员及以上 |

| **导出记录** | 班组长、工程师或质量人员 |
| **删除记录** | 管理员 |
| **清空记录** | 管理员 + 二次确认 |
| **修改保存周期** | 管理员 |
| **修改记录路径** | 工程师或管理员 |

## ③ 计算/选型示例

- 已知：  
  质量追溯记录可以被普通用户删除。
- 公式：  
  **追溯数据可删除 → 高风险**
- 计算：  
  删除后无法确认生产过程和质量责任。
- 结论：  
  记录删除类操作必须高权限控制。

> [!tip] 记忆口诀
> 记录是证据，删除要慎重。

# 14 操作日志

> [!info] 核心结论
> 操作日志用于记录用户登录、注销、按钮点击、参数修改、配方操作、报警确认和系统设置变更。

## ① 分类/公式/规则

> [!warning] 操作日志公式
> **操作日志 = 时间 + 用户 + 操作对象 + 操作内容 + 操作结果**

## ② 参考表

| 日志类型 | 示例 |
|---|---|
| **登录日志** | 用户登录、注销、登录失败 |
| **设备操作日志** | 启动、停止、复位 |
| **参数修改日志** | 修改速度、温度、压力 |
| **配方日志** | 加载、下载、保存、删除配方 |
| **报警日志** | 确认、复位、清空报警 |
| **数据操作日志** | 导出、删除、清空记录 |
| **系统设置日志** | 修改通讯、权限、任务设置 |

## ③ 计算/选型示例

- 已知：  
  温度设定从60℃改为90℃，但没有日志。
- 公式：  
  **无日志 → 无法追溯谁修改**
- 计算：  
  质量异常后无法判断修改责任。
- 结论：  
  关键参数修改必须写入操作日志。

> [!tip] 记忆口诀
> 用户管理不只是限制操作，还要记录操作。

# 15 用户组

> [!info] 核心结论
> 用户组用于把多个用户归为一类，统一分配权限，适合多操作员、多班组、多部门项目。

## ① 分类/公式/规则

> [!warning] 用户组规则
> **同岗位、同权限、同管理要求 → 放入同一用户组**

## ② 参考表

| 用户组 | 适合人员 |
|---|---|
| **Operators** | 一线操作员 |
| **ShiftLeaders** | 班组长 |
| **Engineers** | 工艺、电气、设备工程师 |
| **Maintenance** | 维修人员 |
| **Administrators** | 系统管理员 |
| **Quality** | 质量人员 |
| **ViewOnly** | 访客或只读用户 |

## ③ 计算/选型示例

- 已知：  
  10名操作员权限完全一致。
- 公式：  
  **10名用户 → 1个操作员用户组**
- 计算：  
  只需配置一次操作员组权限。
- 结论：  
  用户组能减少重复配置和权限混乱。

> [!tip] 记忆口诀
> 用户多了用组管，别一个一个单独配。

# 16 用户管理与画面对象

> [!info] 核心结论
> 用户权限可以控制画面对象的可见性、可操作性、颜色状态和弹窗提示。

## ① 分类/公式/规则

> [!warning] 对象权限公式
> **对象状态 = 当前用户权限 → 显示/隐藏/启用/禁用**

## ② 参考表

| 对象控制 | 示例 |
|---|---|
| **隐藏按钮** | 工程师登录后才显示调试按钮 |
| **禁用按钮** | 权限不足时按钮置灰 |
| **弹出登录** | 点击高权限按钮时弹出登录窗口 |
| **显示当前用户** | 顶部状态栏显示登录用户 |
| **权限提示** | 无权限时提示“权限不足” |
| **动态颜色** | 高权限状态显示特殊颜色 |
| **自动返回首页** | 注销后返回主画面 |

## ③ 计算/选型示例

- 已知：  
  “伺服调试”按钮只允许工程师使用。
- 公式：  
  **当前用户权限 < 工程师 → 按钮隐藏或禁用**
- 计算：  
  操作员登录时看不到或不能点击该按钮。
- 结论：  
  用户管理应与画面对象动态属性联动。

> [!tip] 记忆口诀
> 权限不仅管页面，也管按钮、输入框和弹窗。

# 17 默认用户与访客权限

> [!info] 核心结论
> 默认用户是系统未登录时的权限状态，通常应设置为最低权限或只读权限。

## ① 分类/公式/规则

> [!warning] 默认权限规则
> **未登录状态 = 最低权限**

## ② 参考表

| 默认状态 | 建议 |
|---|---|
| **未登录** | 只允许查看首页和当前报警 |
| **访客权限** | 只读，不允许控制 |
| **默认操作员** | 小型设备可使用，但要谨慎 |
| **默认高权限** | 不推荐 |
| **启动后默认页面** | 首页或登录页 |
| **自动注销后** | 回到访客或只读状态 |
| **异常重启后** | 不应保留高权限 |

## ③ 计算/选型示例

- 已知：  
  HMI开机后默认进入管理员权限。
- 公式：  
  **默认高权限 = 安全风险**
- 计算：  
  任何人开机后都能修改系统设置。
- 结论：  
  系统启动后应默认低权限，必要时再登录高权限账号。

> [!tip] 记忆口诀
> 默认权限越低，系统越安全。

# 18 用户管理安全风险

> [!info] 核心结论
> 用户管理常见风险包括默认密码、共用账号、权限过大、无自动注销、无操作日志和普通用户可清空记录。

## ① 分类/公式/规则

> [!warning] 安全风险公式
> **权限风险 = 弱密码 + 共用账号 + 高权限长期登录 + 无日志**

## ② 参考表

| 风险 | 后果 |
|---|---|
| **默认密码未改** | 容易被随意登录 |
| **多人共用账号** | 无法追溯责任 |
| **权限过大** | 普通人员可误改关键功能 |
| **无自动注销** | 高权限遗留 |
| **无操作日志** | 修改无法追踪 |
| **可删除记录** | 追溯证据丢失 |
| **用户离职未禁用** | 账号继续可用 |

## ③ 计算/选型示例

- 已知：  
  离职工程师账号仍可登录HMI。
- 公式：  
  **无效账号未禁用 → 安全隐患**
- 计算：  
  该账号仍可能进入系统修改参数。
- 结论：  
  人员变动后应及时禁用或删除账号。

> [!tip] 记忆口诀
> 用户变动要同步改权限，离职账号不能留。

# 19 常见故障排查

> [!info] 核心结论
> 用户管理异常排查按 **账号状态、密码、角色权限、页面权限、按钮权限、自动注销、日志记录** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **用户管理异常 = 账号错误 + 密码错误 + 权限不足 + 角色未分配 + 功能未授权**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **无法登录** | 用户名错误、密码错误、账号被禁用 |
| **登录后权限不对** | 角色分配错误 |
| **页面进不去** | 页面权限要求过高 |
| **按钮点不了** | 按钮权限不足或设备条件不满足 |
| **参数不能修改** | 参数对象权限限制 |
| **自动退出太快** | 自动注销时间过短 |
| **操作无日志** | 日志功能未启用 |
| **清空记录无提示** | 未配置二次确认和权限 |

## ③ 计算/选型示例

- 已知：  
  工程师登录后仍无法进入参数画面。
- 公式：  
  **登录成功 ≠ 页面权限一定正确**
- 计算：  
  可能工程师账号没有加入工程师角色，或页面权限配置错误。
- 结论：  
  先查用户所属角色，再查页面权限要求。

> [!tip] 记忆口诀
> 登录成功只是第一步，权限配置才决定能做什么。

# 20 用户管理快速检查表

> [!info] 核心结论
> 用户管理检查的目标是确保账号清晰、权限合理、密码安全、操作可追溯、关键功能不被误操作。

## ① 分类/公式/规则

> [!note] 十步检查法
> ① 检查是否修改默认管理员密码  
> ② 检查用户账号是否对应真实人员或岗位  
> ③ 检查用户角色是否分配清楚  
> ④ 检查页面权限是否合理  
> ⑤ 检查按钮、参数、配方和记录权限  
> ⑥ 检查自动注销时间是否合适  
> ⑦ 检查登录、注销和操作日志是否启用  
> ⑧ 检查删除、清空、覆盖类操作是否有二次确认  
> ⑨ 检查离职或无效账号是否禁用  
> ⑩ 上线前测试操作员、工程师、管理员三类权限

> [!warning] 最终检查公式
> **合格用户管理 = 账号明确 + 角色清楚 + 权限合理 + 密码安全 + 日志完整 + 自动注销**

## ② 参考表

| 检查项 | 目标 |
|---|---|
| **账号** | 谁登录能追溯 |
| **角色** | 权限分配清楚 |
| **密码** | 防止随意登录 |
| **页面权限** | 防止越权访问 |
| **按钮权限** | 防止误操作 |
| **参数权限** | 防止误改工艺 |
| **日志** | 修改可追溯 |
| **自动注销** | 防止高权限遗留 |

## ③ 计算/选型示例

- 已知：  
  一套设备有操作员、班组长、工程师和管理员4类人员。
- 公式：  
  **用户管理结构 = 4类角色 + 对应页面/按钮/参数权限**
- 计算：  
  操作员负责生产操作，班组长负责报警和部分参数，工程师负责调试和配方，管理员负责用户和系统设置。
- 结论：  
  推荐按岗位建立角色，再把用户加入对应角色。

> [!tip] 选型口诀
> 操作员管生产，工程师管参数，管理员管系统；登录拿权限，注销退权限；关键操作留日志，危险操作加确认。
