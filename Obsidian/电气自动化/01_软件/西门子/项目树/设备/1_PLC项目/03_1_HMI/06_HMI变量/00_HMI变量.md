# 01 HMI变量总览

> [!info] 核心结论
> **HMI变量** 是HMI/SCADA画面与PLC、控制器、数据库、脚本、报警、趋势、配方之间交换数据的核心对象。  
> 简单理解：**画面对象想显示或控制设备，必须通过变量连接到真实数据。**

## ① 分类/公式/规则

> [!warning] HMI变量公式
> **HMI变量 = 变量名称 + 数据类型 + 数据来源 + 地址/连接 + 采集周期 + 读写属性**

## ② 参考表

| 类型 | 说明 |
|---|---|
| **外部变量** | 连接PLC、控制器、网关等外部设备 |
| **内部变量** | HMI内部使用，不直接连接PLC |
| **系统变量** | 当前用户、时间、语言、系统状态等 |
| **报警变量** | 触发报警、报警字、报警位 |
| **趋势变量** | 用于曲线、历史数据记录 |
| **配方变量** | 用于产品参数组保存和下载 |
| **脚本变量** | 用于临时计算、判断和控制 |

## ③ 计算/选型示例

- 已知：  
  画面上需要显示PLC中的电机运行状态。
- 公式：  
  **PLC运行位 → HMI变量 → 指示灯对象**
- 计算：  
  HMI变量绑定PLC运行位，指示灯再绑定HMI变量。
- 结论：  
  没有HMI变量，画面对象无法与PLC数据联动。

> [!tip] 记忆口诀
> PLC里有数据，HMI靠变量拿；画面要动，变量先通。

# 02 外部变量

> [!info] 核心结论
> 外部变量连接PLC、远程I/O、网关、仪表、驱动器等现场设备，是HMI读取和写入设备数据的主要方式。

## ① 分类/公式/规则

> [!warning] 外部变量公式
> **外部变量可用 = 通讯连接正常 + 地址正确 + 数据类型正确 + 读写权限正确**

## ② 参考表

| 连接对象 | 常见变量 |
|---|---|
| **PLC** | 启停、状态、报警、参数、计数 |
| **远程I/O** | 输入输出状态 |
| **变频器** | 频率、电流、运行状态、故障码 |
| **伺服驱动器** | 位置、速度、报警、使能状态 |
| **工业网关** | 采集点位、设备数据 |
| **仪表** | 温度、压力、流量、液位 |
| **数据库/服务器** | 生产数据、配方、报表 |

## ③ 计算/选型示例

- 已知：  
  HMI要显示PLC中DB10.DBD0的压力值。
- 公式：  
  **PLC地址 → 外部变量 → 数值显示**
- 计算：  
  建立外部变量Pressure_PV，地址绑定DB10.DBD0，类型为Real。
- 结论：  
  外部变量地址和数据类型必须与PLC一致。

> [!tip] 记忆口诀
> 外部变量连设备，地址错了数据就错。

# 03 内部变量

> [!info] 核心结论
> 内部变量只在HMI内部使用，常用于页面状态、临时计算、弹窗控制、语言切换、权限判断和脚本中间值。

## ① 分类/公式/规则

> [!warning] 内部变量规则
> **不需要PLC参与、只在HMI内部使用的数据 → 用内部变量**

## ② 参考表

| 应用 | 示例 |
|---|---|
| **页面状态** | 当前页面编号 |
| **弹窗控制** | 是否显示确认窗口 |
| **临时计算** | 中间值、百分比、格式化数值 |
| **语言切换** | 当前语言编号 |
| **权限显示** | 当前用户等级 |
| **画面动画** | 临时动画状态 |
| **脚本逻辑** | 临时判断标志 |

## ③ 计算/选型示例

- 已知：  
  点击帮助按钮后显示帮助弹窗，不需要通知PLC。
- 公式：  
  **HMI内部显示逻辑 → 内部变量**
- 计算：  
  建立HelpPopupVisible变量，控制弹窗可见性。
- 结论：  
  这种场景不需要占用PLC变量。

> [!tip] 记忆口诀
> 设备逻辑用PLC变量，画面逻辑用HMI内部变量。

# 04 系统变量

> [!info] 核心结论
> 系统变量由HMI运行系统提供，用于读取当前时间、当前用户、语言、系统状态、画面状态等信息。

## ① 分类/公式/规则

> [!warning] 系统变量规则
> **系统状态类信息优先使用系统变量，不建议自己重复造变量**

## ② 参考表

| 系统信息 | 用途 |
|---|---|
| **当前时间** | 日志、报警、报表时间戳 |
| **当前用户** | 权限显示、操作日志 |
| **用户等级** | 按钮权限、页面权限 |
| **当前语言** | 多语言切换 |
| **当前画面** | 页面状态记录 |
| **运行状态** | Runtime状态 |
| **系统诊断** | 通讯、错误、运行信息 |

## ③ 计算/选型示例

- 已知：  
  状态栏需要显示当前登录用户。
- 公式：  
  **当前用户系统变量 → 文本显示**
- 计算：  
  读取系统当前用户变量，显示在画面顶部。
- 结论：  
  用户信息应从系统变量获取，而不是手动输入。

> [!tip] 记忆口诀
> 用户、时间、语言这些系统信息，优先找系统变量。

# 05 变量命名规则

> [!info] 核心结论
> 变量命名必须清晰、统一、可维护，最好体现设备、功能、信号类型和方向。

## ① 分类/公式/规则

> [!warning] 命名公式
> **变量名称 = 设备对象 + 功能 + 信号类型**

## ② 参考表

| 命名 | 含义 |
|---|---|
| **Motor1_RunFB** | 电机1运行反馈 |
| **Motor1_StartCmd** | 电机1启动命令 |
| **Valve1_OpenFB** | 阀门1开到位 |
| **Cyl1_ExtendCmd** | 气缸1伸出命令 |
| **Cyl1_ExtendFB** | 气缸1伸出到位 |
| **Press1_PV** | 压力1当前值 |
| **Temp1_SP** | 温度1设定值 |

## ③ 计算/选型示例

- 已知：  
  一个变量名叫Tag_001。
- 公式：  
  **命名不清 → 后期难维护**
- 计算：  
  维护人员不知道它对应哪个设备、哪个功能。
- 结论：  
  应改为类似Motor1_RunFB这样的功能化名称。

> [!tip] 记忆口诀
> 变量名要让半年后的自己也看得懂。

# 06 数据类型

> [!info] 核心结论
> 数据类型决定HMI如何解析变量值，类型错误会导致显示异常、控制失败或脚本报错。

## ① 分类/公式/规则

> [!warning] 类型匹配公式
> **显示正确 = PLC数据类型 = HMI变量数据类型**

## ② 参考表

| 数据类型 | 常见用途 |
|---|---|
| **Bool** | 启停、到位、报警、开关状态 |
| **Byte/Word** | 状态字、报警字、模式字 |
| **Int** | 整数参数、计数 |
| **DInt** | 大范围计数、累计值 |
| **Real/Float** | 温度、压力、速度、流量 |
| **String** | 产品名称、批号、条码 |
| **DateTime** | 时间戳、记录时间 |

## ③ 计算/选型示例

- 已知：  
  PLC中压力是Real类型，HMI按Int类型读取。
- 公式：  
  **类型不匹配 → 数值异常**
- 计算：  
  浮点数被当整数解释，显示可能变成乱码或异常大数。
- 结论：  
  HMI变量类型必须与PLC变量类型一致。

> [!tip] 记忆口诀
> 地址对了还不够，类型也要对。

# 07 Bool变量

> [!info] 核心结论
> Bool变量用于表示开关状态，常用于按钮命令、设备反馈、报警位、模式位和条件判断。

## ① 分类/公式/规则

> [!warning] Bool变量规则
> **Bool变量只有0/1两种状态，适合表示ON/OFF**

## ② 参考表

| Bool变量 | 用途 |
|---|---|
| **StartCmd** | 启动命令 |
| **StopCmd** | 停止命令 |
| **ResetCmd** | 复位命令 |
| **RunFB** | 运行反馈 |
| **Alarm** | 报警状态 |
| **AutoMode** | 自动模式 |
| **DoorClosed** | 安全门关闭 |

## ③ 计算/选型示例

- 已知：  
  HMI按钮需要给PLC发送一次启动命令。
- 公式：  
  **按钮按下 → Bool变量=1，释放/脉冲后 → Bool变量=0**
- 计算：  
  启动命令通常不应长时间保持ON，需按PLC逻辑设计。
- 结论：  
  启动、复位类命令常用瞬时Bool变量。

> [!tip] 记忆口诀
> 状态看反馈，命令给脉冲；别把命令当反馈用。

# 08 数值变量

> [!info] 核心结论
> 数值变量用于显示和设置温度、压力、速度、位置、时间、计数、产量等连续或离散数值。

## ① 分类/公式/规则

> [!warning] 数值变量规则
> **数值变量 = 数据类型 + 单位 + 小数位 + 范围限制**

## ② 参考表

| 数值变量 | 示例 |
|---|---|
| **温度当前值** | Temp_PV |
| **温度设定值** | Temp_SP |
| **压力当前值** | Pressure_PV |
| **速度设定值** | Speed_SP |
| **伺服位置** | Axis_Pos |
| **产量计数** | Product_Count |
| **超时时间** | Timeout_Set |

## ③ 计算/选型示例

- 已知：  
  速度设定范围100~800mm/s。
- 公式：  
  **100 ≤ Speed_SP ≤ 800**
- 计算：  
  输入900mm/s时，HMI应提示超限并拒绝写入。
- 结论：  
  可写数值变量必须设置范围保护。

> [!tip] 记忆口诀
> 能输入的数值，一定要有限制。

## ① 分类/公式/规则

> [!warning] 字符串变量规则

# 10 变量地址

> [!info] 核心结论
> 外部变量必须绑定正确的PLC或设备地址，地址错误会导致数据显示错位、按钮写错目标或报警误触发。

## ① 分类/公式/规则

> [!warning] 地址规则
> **地址正确 = 设备地址 + 区域 + 偏移量 + 数据类型一致**

## ② 参考表

| 地址类型 | 示例 |
|---|---|
| **输入区** | I0.0、I1.2 |
| **输出区** | Q0.0、Q1.5 |
| **存储区** | M10.0、MW20 |
| **数据块** | DB10.DBX0.0、DB10.DBD4 |
| **寄存器** | 40001、Holding Register |
| **OPC节点** | 设备变量节点 |
| **网关点位** | 点表变量名 |

## ③ 计算/选型示例

- 已知：  
  HMI复位按钮误写到了启动变量地址。
- 公式：  
  **按钮地址错误 → 操作异常**
- 计算：  
  操作员点复位，PLC却收到启动命令。
- 结论：  
  按钮类变量地址必须重点复核。

> [!tip] 记忆口诀
> 显示错地址只是看错，写错地址可能误动作。

# 11 变量读写属性

> [!info] 核心结论
> HMI变量应区分只读、只写和读写，防止把状态反馈误写、把命令位误读或把关键参数误改。

## ① 分类/公式/规则

> [!warning] 读写规则
> **反馈变量只读，命令变量可写，参数变量读写并加权限**

## ② 参考表

| 变量类型 | 推荐属性 |
|---|---|
| **运行反馈** | 只读 |
| **报警状态** | 只读 |
| **启动命令** | 写入或读写 |
| **复位命令** | 写入或读写 |
| **设定参数** | 读写 |
| **实际值** | 只读 |
| **系统状态** | 只读 |

## ③ 计算/选型示例

- 已知：  
  操作员在HMI上可以修改“实际压力值”。
- 公式：  
  **实际值来自传感器 → 应只读**
- 计算：  
  修改实际值不会改变真实压力，还可能误导判断。
- 结论：  
  PV实际值应只读，SP设定值才可写。

> [!tip] 记忆口诀
> PV看现场，SP给设定；反馈别乱写，命令别乱读。

# 12 采集周期

> [!info] 核心结论
> 采集周期决定HMI多久刷新一次变量，周期太慢显示滞后，周期太快会增加PLC和网络负载。

## ① 分类/公式/规则

> [!warning] 采集负载公式
> **变量刷新负载 = 变量数量 × 刷新频率**

## ② 参考表

| 变量类型 | 推荐趋势 |
|---|---|
| **按钮反馈** | 较快 |
| **运行状态** | 较快 |
| **报警状态** | 较快 |
| **温度压力** | 中等 |
| **产量统计** | 中等或较慢 |
| **报表数据** | 较慢 |
| **隐藏页面变量** | 可降低刷新或按需刷新 |

## ③ 计算/选型示例

- 已知：  
  画面有500个变量，每个变量100ms刷新一次。
- 公式：  
  **每秒刷新次数 = 500 × 10**
- 计算：  
  每秒刷新5000次变量，负载较高。
- 结论：  
  应按变量重要性分级设置采集周期。

> [!tip] 记忆口诀
> 不是所有变量都要刷得很快。

# 13 变量组

> [!info] 核心结论
> 变量组用于按设备、工位、功能、页面或通讯对象管理变量，方便查找、维护和批量配置。

## ① 分类/公式/规则

> [!warning] 分组规则
> **变量多时必须分组，分组方式应与设备结构一致**

## ② 参考表

| 分组方式 | 示例 |
|---|---|
| **按设备** | Motor、Valve、Cylinder |
| **按工位** | Station01、Station02 |
| **按功能** | Alarm、Recipe、Trend |
| **按页面** | MainPage、ManualPage |
| **按通讯连接** | PLC1、PLC2、Gateway |
| **按数据类型** | Bool、Real、String |
| **按项目模块** | 上料、检测、下料 |

## ③ 计算/选型示例

- 已知：  
  工程中有3000个HMI变量。
- 公式：  
  **变量数量大 → 分组管理**
- 计算：  
  若不分组，查找一个变量会很困难。
- 结论：  
  大项目必须建立变量组和命名规范。

> [!tip] 记忆口诀
> 变量少靠搜索，变量多靠分组。

# 14 变量与画面对象绑定

> [!info] 核心结论
> 画面对象通过变量绑定实现显示、输入、颜色变化、移动动画、报警提示和按钮操作。

## ① 分类/公式/规则

> [!warning] 绑定公式
> **画面对象有效 = 对象属性 + 变量绑定 + 动态规则**

## ② 参考表

| 画面对象 | 常见绑定变量 |
|---|---|
| **按钮** | 命令变量 |
| **指示灯** | 状态反馈变量 |
| **数值框** | 当前值或设定值 |
| **棒图** | 液位、压力、进度 |
| **趋势曲线** | 连续变量 |
| **报警列表** | 报警变量 |
| **动画对象** | 位置、状态、模式变量 |

## ③ 计算/选型示例

- 已知：  
  电机图标需要运行时变绿，报警时变红。
- 公式：  
  **运行反馈 + 报警变量 → 外观动态**
- 计算：  
  RunFB=1时绿色，Alarm=1时红色闪烁。
- 结论：  
  对象显示应优先绑定反馈变量，而不是命令变量。

> [!tip] 记忆口诀
> 显示看反馈，操作写命令。

# 15 变量与报警

> [!info] 核心结论
> 报警系统通常通过Bool报警位、报警字、故障码或状态字触发，HMI变量负责连接报警源。

## ① 分类/公式/规则

> [!warning] 报警变量公式
> **报警显示 = 报警变量 + 报警文本 + 时间戳 + 确认/复位逻辑**

## ② 参考表

| 报警变量形式 | 说明 |
|---|---|
| **单个Bool位** | 一个报警对应一个变量 |
| **报警字Word** | 一个字包含多个报警位 |
| **故障码Int** | 数值对应不同故障 |
| **状态字** | 状态位组合 |
| **通讯报警变量** | PLC/设备离线 |
| **模拟量超限** | 温度、压力超过阈值 |
| **系统报警变量** | Runtime、数据库、网络异常 |

## ③ 计算/选型示例

- 已知：  
  一个Word报警字有16位，每位代表一个报警。
- 公式：  
  **1个Word = 16个Bool报警位**
- 计算：  
  通过位解析可显示16条不同报警。
- 结论：  
  报警字适合节省变量数量，但报警映射必须清楚。

> [!tip] 记忆口诀
> 报警变量要能追到具体故障点。

# 16 变量与趋势

> [!info] 核心结论
> 趋势变量用于记录连续数据变化，必须配置采样周期、保存周期、单位、量程和历史数据源。

## ① 分类/公式/规则

> [!warning] 趋势数据公式
> **趋势数据量 = 变量数量 × 单点大小 × 采样次数**

## ② 参考表

| 趋势变量 | 用途 |
|---|---|
| **温度** | 升温、保温、超温分析 |
| **压力** | 气压、液压稳定性 |

| **速度** | 运动状态分析 |
| **液位** | 储罐、水箱状态 |
| **流量** | 工艺流量变化 |
| **真空值** | 吸附稳定性 |

## ③ 计算/选型示例

- 已知：  
  10个Real变量，每秒记录1次，每个4字节，保存1小时。
- 公式：  
  **数据量 = 10 × 4 × 3600**
- 计算：  
  数据量 = **144000字节 ≈ 144KB**
- 结论：  
  单小时不大，但长期保存要规划存储和清理。

# 20 变量注释

|---|---|

# 21 HMI变量快速检查表

> [!info] 核心结论
> HMI变量检查的目标是确保变量名称清楚、地址正确、类型匹配、刷新合理、读写安全、画面绑定可靠。

## ① 分类/公式/规则

> [!note] 十步检查法
> ① 确认变量是外部变量、内部变量还是系统变量  
> ② 检查变量名称是否符合命名规范  
> ③ 检查数据类型是否与PLC一致  
> ④ 检查地址和通讯连接是否正确  
> ⑤ 检查读写属性是否合理  
> ⑥ 检查采集周期是否合适  
> ⑦ 检查画面对象是否绑定正确变量  
> ⑧ 检查报警、趋势、配方是否引用正确  
> ⑨ 检查变量注释、单位和范围是否完整  
> ⑩ 上线前做在线监控、写入测试和断电重启测试

> [!warning] 最终检查公式
> **合格HMI变量 = 名称清楚 + 类型正确 + 地址正确 + 刷新合理 + 读写安全 + 注释完整**

## ② 参考表

| 检查项 | 目标 |
|---|---|
| **名称** | 容易理解和搜索 |
| **类型** | 与PLC数据一致 |
| **地址** | 不错位、不误写 |
| **连接** | 通讯设备正确 |
| **读写** | 防止误操作 |
| **周期** | 平衡实时性和负载 |
| **绑定** | 画面显示正确 |
| **注释** | 后期维护清楚 |

## ③ 计算/选型示例

- 已知：  
  HMI画面显示电机运行，但现场电机没动。
- 公式：  
  **画面状态应绑定运行反馈，不应只绑定启动命令**
- 计算：  
  如果指示灯绑定StartCmd，按钮一按就显示运行，但不代表电机真实运行。
- 结论：  
  状态显示必须绑定反馈变量，控制按钮才绑定命令变量。

> [!tip] 选型口诀
> 命令变量写PLC，反馈变量看现场；参数变量要范围，报警变量要文本；变量多要分组，交付前要在线验证。
