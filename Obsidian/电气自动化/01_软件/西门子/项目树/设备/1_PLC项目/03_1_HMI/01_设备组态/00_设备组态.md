# 01｜设备组态是什么

> [!info] 核心结论
> **设备组态是 TIA Portal 中配置 PLC、I/O 模块、通信模块、驱动器、HMI、PROFINET 设备等硬件结构和参数的核心入口。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
设备组态就是告诉 PLC：现场有哪些硬件、插在哪、地址是多少、怎么通信。
</div>

## ① 分类/公式/规则

| 项目 | 作用 | 关键词 |
|---|---|---|
| **硬件组态** | 添加 CPU、模块、设备 | 型号、订货号 |
| **网络组态** | 配置 PROFINET / PROFIBUS | IP、设备名 |
| **I/O 地址** | 分配输入输出地址 | I区、Q区 |
| **参数设置** | 设置模块工作方式 | 滤波、量程、诊断 |
| **设备下载** | 把组态下载到 PLC | 编译、下载 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>核心规则</strong><br>
项目里的设备组态必须和现场真实硬件一致，否则容易出现模块不匹配、设备不上线、I/O 地址错误。
</div>

## ② 参考表

| 检查项 | 重点 | 常见错误 |
|---|---|---|
| **CPU 型号** | 与实物一致 | 订货号不匹配 |
| **模块槽位** | 插槽正确 | 模块位置错 |
| **I/O 地址** | 程序访问依据 | 地址重复/错位 |
| **IP 地址** | 网络通信 | IP 冲突 |
| **设备名称** | PROFINET 识别 | 名称不一致 |
| **GSDML** | 第三方设备识别 | 文件版本错误 |

## ③ 计算/选型示例

- **已知：** 现场有一台 S7-1200 CPU、一个 DI 模块、一个 DQ 模块  
- **公式：** 设备组态 = `CPU + 扩展模块 + I/O 地址 + 参数下载`
- **计算：**
  - 添加对应 CPU 型号
  - 在右侧插入 DI / DQ 模块
  - 检查输入输出地址
  - 编译无误后下载到 PLC
- **结论：** 设备组态完成后，PLC 才知道现场模块如何工作。

> [!tip] 记忆口诀
> **先组硬件，再配地址；先编译，再下载。**

# 02｜硬件组态

> [!info] 核心结论
> **硬件组态用于搭建 PLC 实际硬件结构，包括 CPU、信号模块、通信模块、电源模块和分布式 I/O。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程理解</strong><br>
硬件组态相当于在软件里搭一套“虚拟控制柜”，必须和现场实物一一对应。
</div>

## ① 分类/公式/规则

| 硬件对象 | 作用 | 注意 |
|---|---|---|
| **CPU** | 控制核心 | 型号/固件版本 |
| **DI 模块** | 数字量输入 | 输入地址 |
| **DQ 模块** | 数字量输出 | 输出地址 |
| **AI 模块** | 模拟量输入 | 量程/滤波 |
| **AQ 模块** | 模拟量输出 | 电压/电流类型 |
| **通信模块** | 串口/以太网扩展 | 参数匹配 |

## ② 参考表

| 模块类型 | 常见参数 | 重点 |
|---|---|---|
| **DI** | 滤波时间 | 防抖动 |
| **DQ** | 输出类型 | 继电器/晶体管 |
| **AI** | 电压/电流/温度 | 量程一致 |
| **AQ** | 0-10V / 4-20mA | 负载匹配 |
| **CM** | 波特率/协议 | 通信参数一致 |
| **TM** | 高速计数/运动 | 功能参数校核 |

## ③ 计算/选型示例

- **已知：** 一个 4-20mA 压力传感器接入 PLC  
- **公式：** 传感器信号类型 = AI 模块通道类型  
- **计算：**
  - 传感器输出：`4-20mA`
  - AI 通道必须配置为电流输入
  - 程序中再做工程量换算
- **结论：** 模拟量模块不只要接线正确，组态量程也必须正确。

> [!tip] 记忆口诀
> **模块型号要对，通道参数要配，地址程序要一致。**

# 03｜网络组态

> [!info] 核心结论
> **网络组态用于配置 PLC 与 HMI、远程 I/O、驱动器、网关、传感器等设备之间的通信关系。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>网络口诀</strong><br>
IP 管通信，名称管识别，GSD 管设备，地址管数据。
</div>

## ① 分类/公式/规则

| 网络项 | 作用 |
|---|---|
| **IP 地址** | 设备网络地址 |
| **子网掩码** | 判断同网段 |
| **设备名称** | PROFINET 设备识别 |
| **PROFINET IO 系统** | PLC 与 IO Device 关系 |
| **GSDML 文件** | 第三方设备描述文件 |
| **更新时间** | 周期数据刷新速度 |

## ② 参考表

| 检查项 | 正常要求 | 异常表现 |
|---|---|---|
| **IP 地址** | 同网段、不冲突 | 搜不到/冲突 |
| **Device Name** | 与组态一致 | PN 设备不上线 |
| **GSDML** | 版本正确 | 找不到设备 |
| **端口连接** | 网线正常 | 设备离线 |
| **I/O长度** | 与设备一致 | 数据错位 |

## ③ 计算/选型示例

- **已知：**
  - PLC IP：`192.168.0.1`
  - 远程 I/O IP：`192.168.0.10`
  - 电脑 IP：`192.168.1.100`
- **公式：** 同网段通信 = 前三段一致，且掩码匹配
- **计算：**
  - PLC 与远程 I/O：同属 `192.168.0.x`
  - 电脑属于 `192.168.1.x`
  - 电脑可能无法直接在线访问
- **结论：** 调试电脑应改到同网段，例如 `192.168.0.100`，需按现场网络规划校核。

> [!tip] 记忆口诀
> **PN 不上线，先查名字；搜不到设备，先查网段。**

# 04｜I/O 地址分配

> [!info] 核心结论
> **I/O 地址是 PLC 程序访问现场信号的入口，设备组态中的地址必须和程序变量对应。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点规则</strong><br>
输入用 I 区，输出用 Q 区；地址一错，程序逻辑就会读错或写错。
</div>

## ① 分类/公式/规则

| 地址类型 | 中文含义 | 示例 |
|---|---|---|
| **I** | 输入区 | `I0.0`、`IW64` |
| **Q** | 输出区 | `Q0.0`、`QW64` |
| **M** | 存储区 | `M0.0`、`MW10` |
| **DB** | 数据块 | `DB1.DBW0` |
| **PI/PQ** | 外设输入/输出 | 直接外设访问 |

## ② 参考表

| 信号类型 | 常用地址 | 注意 |
|---|---|---|
| **数字量输入** | `I0.0` | 按位访问 |
| **数字量输出** | `Q0.0` | 按位输出 |
| **模拟量输入** | `IW64` | 常为 Word |
| **模拟量输出** | `QW64` | 常为 Word |
| **PROFINET 数据** | I/Q 字节区 | 按设备映射 |

## ③ 计算/选型示例

- **已知：** 一个 DI 模块起始地址为 `I0.0`，第 5 个输入点接急停反馈  
- **公式：** 第 n 点地址 = 起始位 + 点号偏移  
- **计算：**
  - 第 1 点：`I0.0`
  - 第 2 点：`I0.1`
  - 第 5 点：`I0.4`
- **结论：** 程序中急停反馈应读取 `I0.4`。

> [!tip] 记忆口诀
> **看信号先看地址，地址错了全都错。**

# 05｜参数设置

> [!info] 核心结论
> **参数设置决定模块和设备怎么工作，例如模拟量量程、数字量滤波、通信速率、诊断使能等。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
同一个模块，参数不同，工作方式就可能完全不同。
</div>

## ① 分类/公式/规则

| 参数类型 | 作用 | 示例 |
|---|---|---|
| **通道参数** | 配置信号类型 | 电压/电流 |
| **滤波参数** | 抑制抖动/干扰 | DI滤波时间 |
| **诊断参数** | 开启故障检测 | 断线诊断 |
| **通信参数** | 设置通信方式 | 波特率/IP |
| **量程参数** | 定义测量范围 | 0-10V/4-20mA |

## ② 参考表

| 参数 | 设置错误后果 | 处理 |
|---|---|---|
| **AI量程** | 数值不准 | 匹配传感器 |
| **DI滤波** | 抖动或响应慢 | 按现场调整 |
| **通信速率** | 通信失败 | 主从一致 |
| **诊断使能** | 无报警或误报警 | 按需求开启 |
| **输出类型** | 动作异常 | 匹配负载 |

## ③ 计算/选型示例

- **已知：** AI 模块配置为 `0-10V`，现场传感器输出为 `4-20mA`  
- **公式：** 模块参数必须匹配现场信号  
- **计算：**
  - 模块期待电压信号
  - 现场给的是电流信号
  - 采集值会异常或不可用
- **结论：** 应把 AI 通道改为 `4-20mA` 电流输入，具体范围需按模块手册校核。

> [!tip] 记忆口诀
> **接线只是第一步，参数配对才算数。**

# 06｜编译与下载

> [!info] 核心结论
> **设备组态修改后必须编译并下载到 PLC，现场设备才能按新的硬件和网络参数运行。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>下载口诀</strong><br>
改了硬件要编译，改了组态要下载，下载前先核对在线设备。
</div>

## ① 分类/公式/规则

| 操作 | 作用 |
|---|---|
| **编译硬件** | 检查组态是否合法 |
| **编译软件** | 检查程序逻辑 |
| **下载到设备** | 写入 PLC |
| **在线比较** | 比较项目与设备差异 |
| **重启设备** | 部分参数生效需要 |

## ② 参考表

| 操作场景 | 是否需要下载 | 注意 |
|---|---|---|
| **改 IP** | 是 | 可能影响连接 |
| **加模块** | 是 | 需停机风险评估 |
| **改 I/O 地址** | 是 | 程序需同步修改 |
| **改 AI 量程** | 是 | 现场值会变化 |
| **改变量注释** | 通常不一定 | 看下载内容 |

## ③ 计算/选型示例

- **已知：** 新增一个 DQ 模块，但程序下载后模块仍不工作  
- **公式：** 新硬件生效 = 硬件组态下载 + 程序地址正确  
- **计算：**
  - 检查是否只下载了程序，没下载硬件组态
  - 检查模块槽位是否正确
  - 检查 Q 地址是否被程序使用
- **结论：** 新增硬件后必须确认硬件组态已正确下载到 PLC。

> [!tip] 记忆口诀
> **只改项目不下载，现场设备不知道。**

# 07｜常见故障排查

> [!info] 核心结论
> **设备组态故障排查顺序：型号、槽位、地址、参数、网络、下载状态。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>排查顺序</strong><br>
订货号 → 槽位 → 地址 → 参数 → IP/名称 → 编译下载 → 在线诊断。
</div>

## ① 分类/公式/规则

| 层级 | 检查重点 |
|---|---|
| **硬件层** | CPU/模块型号 |
| **安装层** | 插槽、供电、背板 |
| **地址层** | I/Q 地址是否正确 |
| **参数层** | 通道量程、滤波、诊断 |
| **网络层** | IP、设备名称、GSDML |
| **下载层** | 项目是否已下载 |

## ② 参考表

| 现象 | 可能原因 | 处理方向 |
|---|---|---|
| **模块红灯** | 型号/参数/接线问题 | 查模块信息 |
| **设备不上线** | IP/名称/GSD错误 | 分配名称和IP |
| **程序读不到信号** | I地址错误 | 查地址分配 |
| **输出不动作** | Q地址错/模块未启用 | 查输出区 |
| **模拟量不准** | 量程或接线错误 | 查通道参数 |
| **下载失败** | 连接/权限/版本问题 | 查在线访问 |

## ③ 计算/选型示例

- **已知：** 程序读取 `IW64`，但实际 AI 模块起始地址是 `IW80`  
- **公式：** 程序地址必须等于组态地址  
- **计算：**
  - 程序读取：`IW64`
  - 实际模块：`IW80`
  - 地址不一致
- **结论：** 程序会读错数据，应修改程序地址或重新分配模块地址。

> [!tip] 记忆口诀
> **模块对不上查订货号，数据对不上查地址。**

# 08｜一页速记总卡

> [!info] 核心结论
> **设备组态的核心就是：把现场真实硬件、网络关系、I/O 地址和参数，准确映射到 TIA Portal 项目中。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
硬件要对应，地址要一致；参数要匹配，下载才生效。
</div>

## ① 分类/公式/规则

| 主轴 | 关键词 |
|---|---|
| **设备** | CPU、模块、HMI、驱动器 |
| **网络** | IP、Device Name、GSDML |
| **地址** | I区、Q区、DB区 |
| **参数** | 量程、滤波、诊断 |
| **下载** | 编译、在线、下载 |
| **诊断** | 模块信息、在线状态 |

## ② 参考表

| 问题 | 优先检查 |
|---|---|
| **设备不在线** | IP / 设备名称 |
| **模块不匹配** | 订货号 / 槽位 |
| **I/O 不对** | 地址分配 |
| **模拟量异常** | 通道量程 |
| **通信失败** | GSDML / 网络 |
| **修改未生效** | 是否下载硬件组态 |

## ③ 计算/选型示例

- **已知：** 现场更换了一个型号不同的 I/O 模块  
- **公式：** 现场硬件变化 = 项目硬件组态也要同步变化  
- **计算：**
  - 核对新模块订货号
  - 在设备组态中替换模块
  - 检查地址和参数
  - 编译并下载
- **结论：** 现场硬件变了，TIA 项目不改，PLC 诊断大概率会报警。

> [!tip] 记忆口诀
> **设备组态四件事：型号、地址、参数、下载。**
