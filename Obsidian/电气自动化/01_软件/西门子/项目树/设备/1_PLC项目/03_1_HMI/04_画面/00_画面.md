# 01 画面总览

> [!info] 核心结论
> “画面”是HMI、SCADA、上位机或工业触摸屏中的可视化页面，用于实现 **设备监控、操作控制、参数设置、报警显示、趋势记录、配方管理、权限管理和维护诊断**。

## ① 分类/公式/规则

> [!warning] 画面设计公式
> **好画面 = 状态清楚 + 操作明确 + 报警醒目 + 参数有单位 + 权限有边界 + 数据可追溯**

## ② 参考表

| 画面类型 | 作用 |
|---|---|
| **首页/总览画面** | 显示设备整体状态 |
| **自动运行画面** | 显示自动流程、节拍、产量 |
| **手动画面** | 单独控制气缸、电机、阀门等动作 |
| **参数画面** | 设置速度、时间、温度、压力等参数 |
| **报警画面** | 显示当前报警和历史报警 |
| **趋势画面** | 显示温度、压力、电流等曲线 |
| **配方画面** | 管理不同产品工艺参数 |
| **维护画面** | I/O监控、通讯状态、版本信息 |

## ③ 计算/选型示例

- 已知：  
  一台设备需要启动停止、查看报警、修改参数、查看产量和手动调试。
- 公式：  
  **设备操作需求 → 多功能HMI画面**
- 计算：  
  至少需要首页、自动、手动、参数、报警、维护等页面。
- 结论：  
  画面不能只做一个首页，应按功能分层设计。

> [!tip] 记忆口诀
> 画面不是“好看就行”，而是让现场人员看得懂、按得准、查得快。

# 02 底层原理

> [!info] 核心结论
> 画面的底层原理，是把PLC、控制器、数据库或网关的数据变量绑定到按钮、指示灯、数值框、曲线、报警列表等控件上。

## ① 分类/公式/规则

> [!warning] 画面运行链路
> **PLC/控制器变量 → 通讯驱动 → HMI变量 → 画面控件 → 操作人员显示/操作**

## ② 参考表

| 环节 | 作用 |
|---|---|
| **PLC变量** | 设备真实控制数据 |
| **通讯驱动** | 负责HMI与PLC交换数据 |
| **HMI变量** | 画面内部使用的数据标签 |
| **画面控件** | 按钮、灯、数值框、曲线、报警 |
| **操作输入** | 人点击按钮、输入参数 |
| **写入PLC** | HMI把操作结果写回控制器 |
| **状态反馈** | PLC运行结果再显示到画面 |

## ③ 计算/选型示例

- 已知：  
  HMI上按“启动”按钮，设备没有动作。
- 公式：  
  **按钮有效 = 控件地址正确 + 通讯正常 + PLC允许启动**
- 计算：  
  即使按钮写入成功，PLC内部急停、安全门、报警未复位等条件不满足，也不会启动。
- 结论：  
  画面只是交互入口，真正动作由PLC逻辑决定。

> [!tip] 记忆口诀
> 画面负责“显示和下命令”，PLC负责“判断和执行”。

# 03 画面分类

> [!info] 核心结论
> 工业软件画面应按功能分区，常见结构是：首页、自动、手动、参数、报警、趋势、配方、维护、权限和系统设置。

## ① 分类/公式/规则

> [!warning] 页面结构规则
> **常用操作放一级页面，低频维护放二级页面，危险操作加权限和确认**

## ② 参考表

| 页面 | 典型内容 |
|---|---|
| **首页** | 设备状态、运行模式、报警、产量 |
| **自动画面** | 自动流程、节拍、工序状态 |
| **手动画面** | 单动作、点动、回原点、测试 |
| **参数画面** | 时间、速度、位置、温度、压力 |
| **报警画面** | 当前报警、历史报警、处理建议 |
| **趋势画面** | 工艺变量曲线 |
| **配方画面** | 产品参数组管理 |
| **维护画面** | I/O、通讯、版本、日志 |

## ③ 计算/选型示例

- 已知：  
  设备操作员每天只用启动、停止、复位和查看报警。
- 公式：  
  **高频功能 → 首页或固定按钮区**
- 计算：  
  常用按钮不应藏在多级菜单里。
- 结论：  
  启动、停止、复位、报警入口应放在明显位置。

> [!tip] 记忆口诀
> 高频操作要少点几次，低频维护可以藏深一点。

# 04 首页 / 总览画面

> [!info] 核心结论
> 首页是操作人员最常看的页面，应显示设备总体状态、运行模式、报警状态、产量、节拍和关键工艺数据。

## ① 分类/公式/规则

> [!warning] 首页设计公式
> **首页 = 设备状态 + 模式 + 报警 + 产量 + 关键参数 + 快捷入口**

## ② 参考表

| 内容 | 说明 |
|---|---|
| **设备状态** | 运行、停止、待机、报警、急停 |
| **运行模式** | 手动、自动、调试、维护 |
| **产量统计** | 当前产量、目标产量、良品/不良品 |
| **节拍显示** | 当前节拍、平均节拍 |
| **报警提示** | 当前最重要报警 |
| **关键数据** | 温度、压力、速度、电流等 |
| **快捷按钮** | 启动、停止、复位、报警入口 |

## ③ 计算/选型示例

- 已知：  
  操作员需要快速判断设备是否正常运行。
- 公式：  
  **首页核心 = 一眼判断状态**
- 计算：  
  颜色、文字和状态灯要统一，例如绿色运行、红色报警、黄色等待。
- 结论：  
  首页不要堆太多细节，重点是快速判断设备是否正常。

> [!tip] 记忆口诀
> 首页不是资料页，是设备状态仪表盘。

# 05 自动运行画面

> [!info] 核心结论
> 自动运行画面用于显示设备自动流程、工序状态、动作步骤、节拍、产量和自动条件。

## ① 分类/公式/规则

> [!warning] 自动画面公式
> **自动运行 = 自动条件满足 + 流程步骤执行 + 状态反馈正常 + 异常报警处理**

## ② 参考表

| 内容 | 作用 |
|---|---|
| **流程步骤** | 显示当前执行到哪一步 |
| **工位状态** | 上料、检测、加工、下料等 |
| **自动条件** | 急停复位、安全门关闭、气压正常 |
| **节拍数据** | 当前循环时间、平均循环时间 |
| **产量数据** | 目标、已完成、良品、不良品 |
| **暂停原因** | 缺料、等待、报警、互锁 |
| **自动控制按钮** | 启动、暂停、停止、复位 |

## ③ 计算/选型示例

- 已知：  
  设备自动运行时卡在某一步不动。
- 公式：  
  **卡步骤 = 当前步骤等待条件未满足**
- 计算：  
  应显示当前步骤名称和等待的输入条件，例如“等待气缸伸出到位”。
- 结论：  
  自动画面应能帮助现场快速定位流程卡点。

> [!tip] 记忆口诀
> 自动画面最重要的是告诉你“现在跑到哪、为什么停在这”。

# 06 手动画面

> [!info] 核心结论
> 手动画面用于调试和维修，允许工程师单独控制气缸、电机、阀门、夹爪、输送线等动作。

## ① 分类/公式/规则

> [!warning] 手动操作规则
> **手动动作 = 手动模式 + 权限允许 + 安全条件满足 + 单动作互锁**

## ② 参考表

| 手动对象 | 常见操作 |
|---|---|
| **气缸** | 伸出、缩回 |
| **电机** | 点动正转、点动反转 |
| **伺服轴** | 回原点、点动、移动到指定位置 |
| **电磁阀** | 打开、关闭 |
| **夹爪** | 夹紧、松开 |
| **真空吸盘** | 吸真空、破真空 |
| **输送线** | 启动、停止、点动 |

## ③ 计算/选型示例

- 已知：  
  手动画面允许气缸伸出。
- 公式：  
  **气缸伸出 = 输出动作 + 到位反馈 + 超时报警**
- 计算：  
  按下伸出按钮后，应显示伸出输出、伸出到位、缩回到位状态。
- 结论：  
  手动画面不能只给按钮，还要显示动作反馈和互锁条件。

> [!tip] 记忆口诀
> 手动画面是维修工具，必须有状态反馈和安全互锁。

# 07 参数画面

> [!info] 核心结论
> 参数画面用于设置设备运行参数，必须设计单位、范围、默认值、权限和保存机制。

## ① 分类/公式/规则

> [!warning] 参数保护公式
> **安全参数 = 单位清楚 + 范围限制 + 权限控制 + 修改记录 + 掉电保存**

## ② 参考表

| 参数类型 | 示例 |
|---|---|
| **速度参数** | 输送速度、伺服速度、加速度 |
| **时间参数** | 延时、超时、保压时间 |
| **温度参数** | 设定温度、报警温度 |
| **压力参数** | 气压、液压、真空阈值 |
| **位置参数** | 取料点、放料点、补偿量 |
| **计数参数** | 目标数量、批次数 |
| **工艺参数** | 配方参数、检测阈值 |

## ③ 计算/选型示例

- 已知：  
  气缸动作超时时间正常范围为1~10秒。
- 公式：  
  **参数输入范围 = 最小值 ~ 最大值**
- 计算：  
  若操作员输入0.1秒，应禁止保存或提示超出范围。
- 结论：  
  参数画面必须限制范围，避免误设造成频繁报警。

> [!tip] 记忆口诀
> 参数页面不是随便填数字，必须有范围、单位和权限。

# 08 报警画面

> [!info] 核心结论
> 报警画面用于显示当前报警、历史报警、报警等级、发生时间、确认状态和处理建议。

## ① 分类/公式/规则

> [!warning] 报警信息公式
> **报警信息 = 报警位 + 文本 + 时间戳 + 等级 + 处理建议 + 复位条件**

## ② 参考表

| 内容 | 作用 |
|---|---|
| **当前报警** | 当前仍存在的故障 |
| **历史报警** | 已发生过的报警记录 |
| **报警等级** | 提示、警告、停机、急停 |
| **发生时间** | 追溯故障时间 |
| **确认状态** | 操作员是否已确认 |
| **处理建议** | 指导排查 |
| **复位按钮** | 故障消除后复位 |

## ③ 计算/选型示例

- 已知：  
  画面只显示“设备故障”。
- 公式：  
  **报警不具体 → 排查效率低**
- 计算：  
  操作员不知道是气压不足、伺服报警、门未关还是真空不到位。
- 结论：  
  报警画面应具体到故障点，并提供处理提示。

> [!tip] 记忆口诀
> 好报警要告诉你哪里坏、什么时候坏、怎么查。

# 09 趋势画面

> [!info] 核心结论
> 趋势画面用于显示连续变量随时间变化，适合分析温度、压力、流量、液位、电流、速度和真空值。

## ① 分类/公式/规则

> [!warning] 趋势设置公式
> **趋势曲线 = 变量 + 采样周期 + 时间范围 + 上下限 + 历史保存**

## ② 参考表

| 趋势变量 | 用途 |
|---|---|
| **温度** | 升温、保温、超温分析 |
| **压力** | 气压、液压稳定性 |
| **流量** | 工艺流量变化 |
| **液位** | 水箱、料仓液位趋势 |
| **电流** | 电机负载分析 |
| **真空值** | 吸附稳定性 |
| **速度** | 运动和输送节拍分析 |

## ③ 计算/选型示例

- 已知：  
  压力每秒采样1次，显示最近10分钟。
- 公式：  
  **采样点数 = 采样频率 × 时间**
- 计算：  
  1 × 600 = **600点**
- 结论：  
  趋势画面至少要支持600个采样点缓存。

> [!tip] 记忆口诀
> 报警告诉你结果，趋势告诉你过程。

# 10 配方画面

> [!info] 核心结论
> 配方画面用于管理不同产品对应的参数组，实现快速切换产品、减少人工输入错误。

## ① 分类/公式/规则

> [!warning] 配方公式
> **配方 = 产品编号 + 参数组 + 下载到PLC + 当前配方确认**

## ② 参考表

| 配方内容 | 示例 |
|---|---|
| **产品型号** | A型、B型、C型 |
| **位置参数** | 取放点、补偿量 |
| **速度参数** | 伺服速度、输送速度 |
| **时间参数** | 延时、保压、冷却时间 |
| **温度参数** | 设定温度、报警阈值 |
| **压力参数** | 夹紧压力、真空阈值 |
| **检测参数** | 尺寸上下限、视觉阈值 |

## ③ 计算/选型示例

- 已知：  
  设备有6种产品，每种产品有30个参数。
- 公式：  
  **参数总数 = 产品数 × 每种参数数**
- 计算：  
  6 × 30 = **180个参数值**
- 结论：  
  应使用配方画面管理，不应让操作员逐项手动输入。

> [!tip] 记忆口诀
> 产品多、参数多，就做配方；切换快，出错少。

# 11 I/O监控画面

> [!info] 核心结论
> I/O监控画面用于查看PLC输入输出点状态，是调试、维护和排故的重要页面。

## ① 分类/公式/规则

> [!warning] I/O监控规则
> **I/O状态 = 点位名称 + 地址 + 当前状态 + 物理含义**

## ② 参考表

| 内容 | 作用 |
|---|---|
| **DI输入状态** | 查看传感器、按钮、限位是否有效 |
| **DO输出状态** | 查看电磁阀、指示灯、继电器输出 |
| **AI模拟量** | 查看压力、温度、流量等输入 |
| **AO模拟量** | 查看比例阀、变频器给定 |
| **地址显示** | 对应PLC点位 |
| **点位名称** | 便于现场排查 |
| **强制提示** | 显示是否有强制或调试状态 |

## ③ 计算/选型示例

- 已知：  
  传感器亮灯，但HMI上输入点无信号。
- 公式：  
  **现场亮灯 ≠ PLC输入有效**
- 计算：  
  可能是接线、公共端、PNP/NPN、输入模块或地址映射问题。
- 结论：  
  I/O监控画面应配合万用表和PLC输入灯一起排查。

> [!tip] 记忆口诀
> 现场查线看电，程序查点看I/O监控。

# 12 通讯诊断画面

> [!info] 核心结论
> 通讯诊断画面用于显示PLC、远程I/O、变频器、伺服、网关、数据库和MES等连接状态。

## ① 分类/公式/规则

> [!warning] 通讯诊断公式
> **通讯状态 = 在线/离线 + 错误码 + 最后通讯时间 + 重连状态**

## ② 参考表

| 通讯对象 | 诊断内容 |
|---|---|
| **PLC** | 在线状态、读写错误 |
| **远程I/O** | 掉站、模块报警 |
| **伺服/变频器** | 通讯报警、设备状态 |
| **工业网关** | 点表采集、上传状态 |
| **数据库** | 连接状态、写入状态 |
| **MES/SCADA** | API、OPC UA、MQTT状态 |
| **网络设备** | IP、端口、延迟、丢包 |

## ③ 计算/选型示例

- 已知：  
  数据库写入失败，但PLC通讯正常。
- 公式：  
  **PLC正常 ≠ 系统全部正常**
- 计算：  
  可能是数据库账号、网络、防火墙或磁盘空间问题。
- 结论：  
  通讯诊断画面应分别显示控制通讯和数据通讯状态。

> [!tip] 记忆口诀
> 通讯诊断要分对象，不要只显示一个“系统正常”。

# 13 弹窗画面

> [!info] 核心结论
> 弹窗画面用于二次确认、参数输入、报警提示、登录验证和临时操作，不能滥用。

## ① 分类/公式/规则

> [!warning] 弹窗使用规则
> **关键操作用弹窗确认，普通操作不要频繁打断**

## ② 参考表

| 弹窗类型 | 应用 |
|---|---|
| **确认弹窗** | 清零、删除、复位、切换配方 |
| **输入弹窗** | 参数输入、密码输入 |
| **报警弹窗** | 重要故障强提示 |
| **登录弹窗** | 用户切换、权限验证 |
| **提示弹窗** | 操作成功、保存成功 |
| **维护弹窗** | 调试说明、版本信息 |
| **帮助弹窗** | 故障处理步骤 |

## ③ 计算/选型示例

- 已知：  
  操作员误点“清空报警历史”。
- 公式：  
  **不可逆操作 → 二次确认**
- 计算：  
  删除历史记录会影响追溯，应要求确认和权限。
- 结论：  
  清零、删除、配方覆盖等操作必须加确认弹窗。

> [!tip] 记忆口诀
> 危险操作要确认，普通操作别弹太多。

# 14 导航与菜单

> [!info] 核心结论
> 画面导航决定用户能否快速找到所需功能，应清晰分层、固定入口、统一风格。

## ① 分类/公式/规则

> [!warning] 导航设计规则
> **一级导航放常用功能，二级导航放维护功能，返回首页要明显**

## ② 参考表

| 导航项 | 说明 |
|---|---|
| **首页按钮** | 一键回总览 |
| **自动页面** | 生产运行 |
| **手动页面** | 调试维护 |
| **参数页面** | 工艺设置 |
| **报警页面** | 故障查看 |
| **趋势页面** | 数据曲线 |
| **配方页面** | 产品切换 |
| **系统页面** | 权限、通讯、备份 |

## ③ 计算/选型示例

- 已知：  
  操作员经常找不到报警页面。
- 公式：  
  **高频页面 → 固定导航入口**
- 计算：  
  报警入口应在每个主画面都能快速进入。
- 结论：  
  导航栏应固定显示报警入口和当前报警状态。

> [!tip] 记忆口诀
> 页面多不可怕，导航乱才可怕。

# 15 权限与登录画面

> [!info] 核心结论
> 权限画面用于管理用户登录、角色权限、密码修改、自动注销和操作日志。

## ① 分类/公式/规则

> [!warning] 权限公式
> **功能可操作 = 用户登录 + 角色权限 + 操作条件满足**

## ② 参考表

| 角色 | 典型权限 |
|---|---|
| **操作员** | 启停、复位、查看状态 |
| **班组长** | 部分参数修改、报警确认 |
| **工程师** | 手动调试、配方、工艺参数 |
| **管理员** | 用户管理、系统设置 |
| **维护人员** | I/O、通讯、诊断 |
| **只读用户** | 只能查看不能操作 |
| **审计用户** | 查看日志和记录 |

## ③ 计算/选型示例

- 已知：  
  普通操作员误改了伺服位置参数。
- 公式：  
  **关键参数 = 工程师权限**
- 计算：  
  操作员只应能查看或修改有限范围内的生产参数。
- 结论：  
  权限画面应按角色限制参数和危险操作。

> [!tip] 记忆口诀
> 谁能看、谁能改、谁改过，都要能管理。

# 16 变量绑定

> [!info] 核心结论
> 变量绑定是画面功能的核心，按钮、指示灯、数值框、曲线、报警都必须绑定正确变量。

## ① 分类/公式/规则

> [!warning] 控件绑定公式
> **控件有效 = 变量地址正确 + 数据类型正确 + 读写权限正确 + 通讯正常**

## ② 参考表

| 控件 | 常见绑定 |
|---|---|
| **按钮** | 写入启动、停止、复位位 |
| **指示灯** | 读取运行、报警、到位状态 |
| **数值显示** | 读取温度、压力、速度 |
| **数值输入** | 写入设定值 |
| **报警列表** | 读取报警位或报警变量 |
| **趋势曲线** | 绑定连续变量 |
| **动画对象** | 绑定状态或位置变量 |

## ③ 计算/选型示例

- 已知：  
  温度显示为异常大数。
- 公式：  
  **显示异常 = 地址错误 / 类型错误 / 字节序错误 / 倍率错误**
- 计算：  
  PLC中是浮点数，HMI按整数读，就会显示错误。
- 结论：  
  变量绑定时必须核对地址、类型、字节序和工程量换算。

> [!tip] 记忆口诀
> 画面控件是外壳，变量地址才是灵魂。

# 17 画面动画

> [!info] 核心结论
> 画面动画用于直观显示设备状态，例如颜色变化、位置移动、阀门开关、液位高度和电机运行。

## ① 分类/公式/规则

> [!warning] 动画设计规则
> **动画要帮助理解状态，不要为了炫酷增加干扰**

## ② 参考表

| 动画类型 | 应用 |
|---|---|
| **颜色变化** | 运行、停止、报警、禁用 |
| **闪烁** | 报警、等待处理 |
| **位置移动** | 气缸、滑台、机械手状态 |
| **旋转动画** | 电机、风机、泵运行 |
| **液位动画** | 水箱、料仓、油箱 |
| **开关状态** | 阀门、门锁、夹爪 |
| **流程高亮** | 当前工序或流向 |

## ③ 计算/选型示例

- 已知：  
  电机运行时画面需要显示旋转状态。
- 公式：  
  **动画触发 = 电机运行反馈ON**
- 计算：  
  不应只用启动命令触发动画，因为启动命令ON不代表电机实际运行。
- 结论：  
  动画应优先绑定反馈状态，而不是只绑定操作命令。

> [!tip] 记忆口诀
> 动画要显示真实状态，别只显示按钮命令。

# 18 分辨率与布局

> [!info] 核心结论
> 画面布局必须匹配屏幕尺寸、分辨率、触摸方式和现场观看距离。

## ① 分类/公式/规则

> [!warning] 布局规则
> **屏幕越小，画面越要简洁；触摸操作按钮不能太小**

## ② 参考表

| 项目 | 注意 |
|---|---|
| **屏幕尺寸** | 7寸、10寸、15寸等 |
| **分辨率** | 800×480、1024×600、1920×1080等 |
| **字体大小** | 现场距离越远字体越大 |
| **按钮尺寸** | 戴手套操作需更大 |
| **留白** | 避免过度拥挤 |
| **颜色数量** | 不宜过多 |
| **固定区域** | 顶部状态栏、底部导航栏 |

## ③ 计算/选型示例

- 已知：  
  7寸触摸屏上放了大量参数输入框。
- 公式：  
  **小屏 + 多参数 = 页面拥挤**
- 计算：  
  按钮和文字过小会导致误触。
- 结论：  
  小屏应分页显示参数，大量参数更适合10寸以上屏幕。

> [!tip] 记忆口诀
> 屏小就少放，屏大才展开；现场操作优先清楚。

# 19 多语言与单位

> [!info] 核心结论
> 多语言和单位设置影响现场人员理解，出口设备、跨区域产线和国际项目尤其重要。

## ① 分类/公式/规则

> [!warning] 多语言规则
> **多语言画面 = 文本统一管理 + 单位统一 + 报警翻译一致**

## ② 参考表

| 内容 | 注意 |
|---|---|
| **按钮文字** | 启动、停止、复位等 |
| **报警文本** | 必须准确翻译 |
| **参数名称** | 与说明书一致 |
| **单位** | ℃、MPa、mm、inch等 |
| **日期时间格式** | 不同地区格式不同 |
| **小数点格式** | 部分地区用逗号 |
| **字体兼容** | 中英文、特殊字符显示 |

## ③ 计算/选型示例

- 已知：  
  同一设备需要中英文切换。
- 公式：  
  **多语言 = 文本变量化**
- 计算：  
  不建议把中文文字直接画死在图片里。
- 结论：  
  应使用文本库或多语言表管理画面文字。

> [!tip] 记忆口诀
> 多语言不是简单翻译，是整套画面、报警、单位都要一致。

# 20 常见故障排查

> [!info] 核心结论
> 画面故障排查按 **通讯、变量、权限、控件、脚本、数据库、报警配置、分辨率** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **画面异常 = 通讯异常 + 变量绑定错 + 权限不足 + 控件配置错 + 脚本错误 + 数据库异常**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **按钮无效** | 权限不足、地址错、PLC互锁 |
| **指示灯不变** | 变量地址错、通讯中断 |
| **数值异常** | 数据类型、字节序、倍率错误 |
| **报警不显示** | 报警位未配置、文本未绑定 |
| **趋势无曲线** | 变量未绑定、采样未启用 |
| **参数不保存** | 未写入PLC/数据库、权限不足 |
| **画面卡顿** | 变量太多、刷新太快、脚本过重 |
| **触摸偏移** | 校准错误、分辨率不匹配 |

## ③ 计算/选型示例

- 已知：  
  点击启动按钮后，画面按钮变色，但设备没启动。
- 公式：  
  **画面变化 ≠ PLC动作成功**
- 计算：  
  按钮可能只是本地状态变化，未真正写入PLC，或PLC启动条件未满足。
- 结论：  
  应检查按钮写入地址、PLC输入状态和启动互锁条件。

> [!tip] 记忆口诀
> 画面变了不代表设备动了，要看PLC是否真的收到并执行。

# 21 画面设计快速检查表

> [!info] 核心结论
> 画面交付前要检查导航、变量、权限、报警、趋势、参数、配方、数据记录和异常恢复。

## ① 分类/公式/规则

> [!note] 十步检查法
> ① 首页能一眼看出设备状态  
> ② 启动、停止、复位按钮位置明显  
> ③ 手动画面有互锁和反馈  
> ④ 参数画面有单位、范围、权限  
> ⑤ 报警画面有文本、时间、处理建议  
> ⑥ 趋势画面采样和历史保存正常  
> ⑦ 配方能保存、下载、切换和确认  
> ⑧ I/O监控地址与现场一致  
> ⑨ 通讯诊断能显示关键设备状态  
> ⑩ 权限、日志、备份和恢复经过测试

> [!warning] 最终设计公式
> **合格画面 = 好操作 + 好排故 + 好追溯 + 好维护 + 好防误操作**

## ② 参考表

| 检查项 | 目标 |
|---|---|
| **状态显示** | 一眼判断设备情况 |
| **操作按钮** | 明确、安全、防误触 |
| **报警提示** | 具体、及时、可追溯 |
| **参数输入** | 有范围、有权限、有保存 |
| **数据趋势** | 能看过程变化 |
| **配方管理** | 能快速切换产品 |
| **权限管理** | 不同人员权限清楚 |
| **维护诊断** | 故障能快速定位 |

## ③ 计算/选型示例

- 已知：  
  设备交付前需要验证画面是否可用。
- 公式：  
  **画面验收 = 正常操作测试 + 异常故障测试 + 断电恢复测试**
- 计算：  
  不能只测试正常生产，还要测试报警、权限、通讯中断、参数保存和重启恢复。
- 结论：  
  画面验收应覆盖正常、异常和维护三类场景。

> [!tip] 选型口诀
> 首页看状态，自动看流程，手动看反馈，参数看范围，报警看原因，趋势看过程，维护看诊断。
