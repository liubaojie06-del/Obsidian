# 01｜在线和诊断是什么

> [!info] 核心结论
> **“在线和诊断”是 TIA Portal 中用于连接 PLC、查看设备状态、读取故障信息、排查网络与硬件问题的调试入口。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
在线和诊断不是 PLC 指令，而是工程师现场调试 PLC 的“体检入口”。
</div>

## ① 分类/公式/规则

| 项目 | 说明 |
|---|---|
| **位置** | TIA Portal 设备视图 / 项目树中 |
| **对象** | PLC、HMI、驱动器、IO 模块、PROFINET 设备 |
| **用途** | 在线连接、查看状态、诊断故障、下载/维护 |
| **核心价值** | 快速判断设备是否在线、是否报错、哪里异常 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>核心规则</strong><br>
先确认能否在线，再看诊断信息；先看设备状态，再查程序逻辑。
</div>

## ② 参考表

| 功能 | 作用 | 常见用途 |
|---|---|---|
| **在线访问** | 连接真实设备 | 查 PLC 是否在线 |
| **诊断缓冲区** | 查看历史故障 | 找停机原因 |
| **模块信息** | 查看硬件状态 | 查模块报警 |
| **网络设置** | 查看 IP / 设备名 | 查 PROFINET 问题 |
| **维护功能** | 复位、设置、更新 | 现场维护 |

## ③ 计算/选型示例

- **已知：** PLC 运行异常，HMI 显示通信报警  
- **公式：** 排查顺序 = `在线连接 → 设备状态 → 诊断缓冲区 → 网络参数`
- **计算：**
  - 先进入“在线和诊断”
  - 查看 PLC 是否在线
  - 查看是否有模块故障或通信故障
  - 再看诊断缓冲区中的错误记录
- **结论：** “在线和诊断”适合作为 PLC 故障排查的第一入口。

> [!tip] 记忆口诀
> **先上线，再诊断；先看硬件，再查程序。**

# 02｜在线连接

> [!info] 核心结论
> **在线连接用于让 TIA Portal 与现场 PLC 或设备建立通信，是所有诊断操作的前提。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程理解</strong><br>
只有软件真正连上设备，才能读取状态、下载程序、查看报警和执行维护操作。
</div>

## ① 分类/公式/规则

| 连接项 | 说明 |
|---|---|
| **PG/PC 接口** | 电脑使用哪个网卡连接 PLC |
| **接口类型** | PN/IE、MPI、PROFIBUS 等 |
| **IP 地址** | PLC 与电脑需在可通信网段 |
| **设备名称** | PROFINET 设备识别的重要条件 |
| **在线状态** | 判断设备是否可访问 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>上线口诀</strong><br>
网卡选对，IP 同段，设备能搜到，才能谈诊断。
</div>

## ② 参考表

| 检查项 | 正常表现 | 异常表现 |
|---|---|---|
| **网卡选择** | 能搜索到 PLC | 搜不到设备 |
| **IP 网段** | 能通信 | 无法访问 |
| **网线/交换机** | 链路灯正常 | 端口不亮 |
| **设备名称** | 与项目一致 | PROFINET 设备不上线 |
| **防火墙** | 不阻止通信 | 搜索失败 |

## ③ 计算/选型示例

- **已知：**
  - PLC IP：`192.168.0.1`
  - 电脑 IP：`192.168.1.100`
- **公式：** 通信前提 = 同网段或正确路由
- **计算：**
  - PLC 网段：`192.168.0.x`
  - 电脑网段：`192.168.1.x`
  - 两者不在同一网段
- **结论：** 电脑 IP 应改到同网段，例如 `192.168.0.100`，具体需按现场网络规划校核。

> [!tip] 记忆口诀
> **搜不到先查网卡，连不上先查 IP。**

# 03｜诊断缓冲区

> [!info] 核心结论
> **诊断缓冲区是 PLC 的“故障日记本”，会记录停机、模块故障、通信中断、程序错误等事件。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>一句话理解</strong><br>
设备什么时候出错、哪里出错、为什么停机，很多线索都在诊断缓冲区里。
</div>

## ① 分类/公式/规则

| 信息 | 作用 |
|---|---|
| **时间戳** | 判断故障发生时间 |
| **事件类型** | 停机、报警、通信故障等 |
| **故障对象** | CPU、模块、网络、程序块 |
| **事件描述** | 故障原因提示 |
| **事件顺序** | 判断先后关系 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>排查规则</strong><br>
不要只看最后一条报警，要看报警发生前后的事件链。
</div>

## ② 参考表

| 记录内容 | 重点看什么 | 用途 |
|---|---|---|
| **CPU STOP** | 停机原因 | 查程序或硬件错误 |
| **模块故障** | 哪个槽位异常 | 查 I/O 模块 |
| **通信中断** | 哪个设备掉线 | 查 PROFINET |
| **访问错误** | DB / 地址问题 | 查程序寻址 |
| **电源事件** | 上电/掉电 | 查供电问题 |

## ③ 计算/选型示例

- **已知：** PLC 昨晚自动停机，现场已恢复运行  
- **公式：** 历史故障 = 诊断缓冲区按时间查找
- **计算：**
  - 打开在线和诊断
  - 进入诊断缓冲区
  - 按时间查找停机前后的记录
  - 找到最早触发故障的事件
- **结论：** 诊断缓冲区能帮助追溯“故障源头”，而不是只看当前状态。

> [!tip] 记忆口诀
> **故障不怕过夜，缓冲区里有线索。**

# 04｜模块信息

> [!info] 核心结论
> **模块信息用于查看 CPU、信号模块、通信模块、分布式 I/O 等硬件的实际运行状态。**

<div class="card purple" style="background:#F8F1FF;border-left:6px solid #9B59B6;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>现场理解</strong><br>
模块信息就像 PLC 硬件体检表：哪个模块在线、哪个模块报警、哪个模块参数不一致。
</div>

## ① 分类/公式/规则

| 模块信息 | 说明 |
|---|---|
| **订货号** | 实际硬件型号 |
| **固件版本** | CPU / 模块版本 |
| **运行状态** | 正常、报警、故障 |
| **槽位信息** | 模块安装位置 |
| **诊断状态** | 是否有硬件报警 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>核对规则</strong><br>
项目里的硬件组态，必须与现场实际模块型号、槽位、版本基本一致。
</div>

## ② 参考表

| 异常现象 | 可能原因 | 处理方向 |
|---|---|---|
| **模块不一致** | 现场型号与组态不符 | 核对订货号 |
| **模块报警** | 接线/参数/硬件故障 | 看诊断详情 |
| **模块丢失** | 未插好/断电/总线故障 | 查供电和背板 |
| **通道故障** | 断线/短路/超量程 | 查现场接线 |
| **版本不兼容** | 固件或 GSD 不匹配 | 按手册校核 |

## ③ 计算/选型示例

- **已知：** TIA 显示某个 AI 模块报警  
- **公式：** 模块报警 = 模块状态 + 通道诊断 + 现场信号
- **计算：**
  - 打开模块信息
  - 查看哪个通道报警
  - 检查该通道是否断线、超量程或参数不一致
  - 再查现场传感器和接线
- **结论：** 模块报警不要只看模块本体，要追到具体通道和现场信号。

> [!tip] 记忆口诀
> **模块红不等于模块坏，先查通道和接线。**

# 05｜网络与设备名称

> [!info] 核心结论
> **PROFINET 设备上线不仅看 IP，还要看设备名称，名称不匹配时设备可能能 ping 通但仍然不上线。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>重点提醒</strong><br>
Ping 通 ≠ PROFINET 正常；PROFINET 很看重 Device Name。
</div>

## ① 分类/公式/规则

| 项目 | 作用 |
|---|---|
| **IP 地址** | 网络层通信 |
| **设备名称** | PROFINET 识别设备 |
| **MAC 地址** | 物理设备唯一标识 |
| **GSDML** | 第三方设备描述文件 |
| **子网/网关** | 跨网段通信相关 |

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>PROFINET 上线三件套</strong><br>
GSDML 要对，Device Name 要对，IP 地址要对。
</div>

## ② 参考表

| 检查项 | 错误后果 | 处理 |
|---|---|---|
| **Device Name** | 设备不上线 | 重新分配名称 |
| **IP 冲突** | 通信不稳定 | 修改 IP |
| **GSDML 错误** | 设备无法组态 | 导入正确文件 |
| **端口断开** | 设备离线 | 查网线交换机 |
| **拓扑不符** | 诊断报警 | 按项目要求校核 |

## ③ 计算/选型示例

- **已知：** 设备可以 ping 通，但 TIA 中 PROFINET 设备仍显示未连接  
- **公式：** Ping 通只说明 IP 层正常，不代表 PROFINET 名称匹配
- **计算：**
  - IP 能通信
  - 但 Device Name 可能不等于项目组态名称
  - 重新在线分配设备名称
- **结论：** 这种情况优先检查 PROFINET 设备名称，而不是只改 IP。

> [!tip] 记忆口诀
> **IP 让你找到它，名称让 PN 认得它。**

# 06｜在线监控与状态判断

> [!info] 核心结论
> **在线监控用于实时查看 PLC 和设备状态，但判断故障时要结合状态位、诊断缓冲区和硬件指示灯。**

<div class="card green" style="background:#F1FBF5;border-left:6px solid #27AE60;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>工程经验</strong><br>
在线监控能看到“现在发生了什么”，诊断缓冲区能看到“之前发生了什么”。
</div>

## ① 分类/公式/规则

| 监控对象 | 说明 |
|---|---|
| **CPU 状态** | RUN / STOP / ERROR |
| **模块状态** | 正常 / 报警 / 缺失 |
| **通信状态** | 在线 / 离线 / 中断 |
| **变量状态** | 程序变量实时值 |
| **报警状态** | 当前报警或历史报警 |

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>判断规则</strong><br>
不要只看一个状态灯，要把 PLC 状态、模块诊断、网络状态一起看。
</div>

## ② 参考表

| 状态 | 可能含义 | 下一步 |
|---|---|---|
| **RUN** | CPU 运行中 | 查模块/程序 |
| **STOP** | CPU 停止 | 查诊断缓冲区 |
| **ERROR** | 有故障 | 查具体对象 |
| **MAINT** | 维护需求 | 查诊断详情 |
| **离线** | 网络/设备问题 | 查连接和名称 |

## ③ 计算/选型示例

- **已知：** CPU 是 RUN，但设备仍然不能动作  
- **公式：** CPU RUN ≠ 系统全正常
- **计算：**
  - CPU 程序在运行
  - 但输出模块可能故障
  - 或驱动器可能报警
  - 或现场安全回路未满足
- **结论：** 设备不动作时，不能只看 CPU RUN，要查完整控制链。

> [!tip] 记忆口诀
> **CPU 在跑，不代表现场在好。**

# 07｜常见故障排查

> [!info] 核心结论
> **“在线和诊断”排故顺序建议：先连上设备，再看状态，再读缓冲区，再查模块与网络。**

<div class="card orange" style="background:#FFF7F3;border-left:6px solid #FF6B35;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>排查顺序</strong><br>
网线 → 网卡 → IP → 设备名称 → 设备状态 → 诊断缓冲区 → 模块信息 → 程序逻辑。
</div>

## ① 分类/公式/规则

| 层级 | 检查重点 |
|---|---|
| **物理层** | 网线、交换机、端口灯 |
| **网络层** | IP、网段、网卡、防火墙 |
| **PROFINET层** | 设备名称、GSDML、拓扑 |
| **硬件层** | 模块、通道、供电 |
| **程序层** | 地址、变量、逻辑条件 |
| **现场层** | 传感器、执行器、安全回路 |

## ② 参考表

| 现象 | 可能原因 | 解决方向 |
|---|---|---|
| **搜不到 PLC** | 网卡选错 / 网线问题 | 查 PG/PC 接口 |
| **能 ping 但不上线** | 设备名称不匹配 | 分配 Device Name |
| **CPU STOP** | 程序错误 / 硬件错误 | 查诊断缓冲区 |
| **模块红灯** | 接线/参数/模块故障 | 查模块信息 |
| **IO 无动作** | 地址错 / 条件不满足 | 在线监控变量 |
| **设备间歇掉线** | 干扰/交换机/线缆 | 查网络质量 |

## ③ 计算/选型示例

- **已知：** 现场 PROFINET 远程 I/O 偶发掉线  
- **公式：** 偶发掉线 = 网络质量 + 供电 + 接地 + 拓扑综合排查
- **计算：**
  - 查诊断缓冲区掉线时间
  - 查交换机端口记录
  - 查远程 I/O 供电是否波动
  - 查网线屏蔽和接地
- **结论：** 偶发通信故障通常不是单一参数问题，要结合网络、供电和现场干扰排查。

> [!tip] 记忆口诀
> **一直不通查配置，偶发掉线查干扰。**

# 08｜一页速记总卡

> [!info] 核心结论
> **在线和诊断的核心价值是：快速判断 PLC 是否在线、设备是否正常、故障从哪里来。**

<div class="card blue" style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px 14px;border-radius:12px;margin:10px 0;">
<strong>总口诀</strong><br>
上线看连接，故障看缓冲；模块看状态，网络看名称。
</div>

## ① 分类/公式/规则

| 功能 | 记忆点 |
|---|---|
| **在线连接** | 先确认能不能连上 |
| **诊断缓冲区** | 看历史故障 |
| **模块信息** | 看硬件和通道 |
| **网络设置** | 看 IP 和设备名称 |
| **在线监控** | 看实时状态 |
| **故障排查** | 从硬件到程序逐层查 |

## ② 参考表

| 问题 | 优先入口 |
|---|---|
| **PLC 搜不到** | 在线访问 |
| **PLC 停机** | 诊断缓冲区 |
| **模块报警** | 模块信息 |
| **PN 设备不上线** | 分配设备名称 |
| **变量不变化** | 在线监控 |
| **通信不稳定** | 网络诊断 |

## ③ 计算/选型示例

- **已知：** 设备现场突然停机，需要快速判断原因  
- **公式：** 快速诊断 = `上线 → 缓冲区 → 模块状态 → 网络状态 → 程序条件`
- **计算：**
  - 先确认 PLC 在线
  - 查看 CPU 是否 STOP
  - 查看诊断缓冲区第一条关键故障
  - 查看模块和网络设备是否报警
- **结论：** 用“在线和诊断”可以快速缩小故障范围，减少盲目改程序。

> [!tip] 记忆口诀
> **调试先上线，排故看诊断；硬件先确认，程序后判断。**
