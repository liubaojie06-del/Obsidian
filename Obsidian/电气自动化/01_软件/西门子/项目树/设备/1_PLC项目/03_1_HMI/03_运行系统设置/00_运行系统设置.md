# 01 运行系统设置总览

> [!info] 核心结论
> 运行系统设置用于配置工业软件或HMI/SCADA系统在运行时的 **启动方式、通讯连接、用户权限、报警记录、数据存储、语言时间、画面显示、备份恢复和安全策略**。

## ① 分类/公式/规则

> [!warning] 配置公式
> **运行系统设置 = 启动配置 + 通讯参数 + 权限管理 + 数据记录 + 报警设置 + 安全策略 + 维护备份**

## ② 参考表

| 设置项 | 作用 |
|---|---|
| **启动设置** | 开机自启、默认画面、运行模式 |
| **通讯设置** | PLC、网关、数据库、服务器连接 |
| **用户权限** | 操作员、工程师、管理员分级 |
| **报警设置** | 当前报警、历史报警、报警确认 |
| **数据记录** | 产量、参数、趋势、日志保存 |
| **画面设置** | 分辨率、语言、主题、触摸校准 |
| **备份恢复** | 参数、配方、数据库、工程备份 |
| **安全设置** | 密码、锁屏、远程访问、操作日志 |

## ③ 计算/选型示例

- 已知：  
  一套设备运行软件需要开机自动启动、自动连接PLC、保存报警历史，并限制普通操作员修改参数。
- 公式：  
  **稳定运行 = 自启动 + 自动通讯 + 权限控制 + 数据记录**
- 计算：  
  如果缺少权限和日志，误操作后难以追溯。
- 结论：  
  运行系统设置必须在设备交付前完整配置和测试。

> [!tip] 记忆口诀
> 程序能跑只是开始，运行设置决定系统能不能长期稳定跑。

# 02 启动设置

> [!info] 核心结论
> 启动设置决定系统上电后如何进入运行状态，包括开机自启、默认页面、登录状态和异常恢复。

## ① 分类/公式/规则

> [!warning] 启动规则
> **可靠启动 = 系统开机 + 软件自启 + 通讯初始化 + 默认画面 + 状态检查**

## ② 参考表

| 启动项 | 说明 |
|---|---|
| **开机自启** | 工控机/HMI上电后自动启动运行软件 |
| **默认画面** | 启动后进入首页、总览页或登录页 |
| **自动登录** | 特定权限自动进入系统，需谨慎 |
| **启动延时** | 等待PLC、数据库、网络就绪 |
| **异常恢复** | 软件崩溃后自动重启 |
| **全屏运行** | 防止操作员误关软件 |
| **看门狗** | 监控软件运行状态 |

## ③ 计算/选型示例

- 已知：  
  工控机开机后PLC启动较慢，HMI软件立即连接PLC会报警。
- 公式：  
  **启动延时 ≥ PLC启动时间**
- 计算：  
  如果PLC约30秒后通讯就绪，可设置HMI启动延时30~60秒。
- 结论：  
  启动设置应考虑PLC、网络和数据库的启动顺序。

> [!tip] 记忆口诀
> 系统启动不是越快越好，关键是各模块都准备好。

# 03 通讯设置

> [!info] 核心结论
> 通讯设置用于配置HMI、SCADA或上位机与PLC、网关、仪表、数据库和服务器之间的数据连接。

## ① 分类/公式/规则

> [!warning] 通讯成功公式
> **通讯成功 = 协议一致 + IP/端口正确 + 站号正确 + 地址正确 + 网络正常**

## ② 参考表

| 通讯对象 | 常见设置 |
|---|---|
| **PLC** | IP地址、协议、站号、机架槽号 |
| **远程I/O** | 设备名、IP、总线周期 |
| **工业网关** | IP、端口、协议、点表 |
| **数据库** | 地址、账号、密码、库名 |
| **MES/SCADA** | API、OPC UA、MQTT、Socket |
| **串口设备** | 波特率、数据位、停止位、校验位 |
| **网络设备** | 网关、DNS、子网掩码 |

## ③ 计算/选型示例

- 已知：  
  HMI显示PLC通讯中断。
- 公式：  
  **通讯中断 = 网络问题 / 协议错误 / 地址错误 / PLC未运行**
- 计算：  
  先Ping PLC，再检查通讯驱动和变量地址。
- 结论：  
  通讯问题应按网络层、协议层、变量层逐级排查。

> [!tip] 记忆口诀
> 网络通不代表数据通，数据通还要看协议和地址。

# 04 用户与权限管理

> [!info] 核心结论
> 用户权限用于限制不同人员能操作哪些功能，防止误改参数、误清数据和误执行危险操作。

## ① 分类/公式/规则

> [!warning] 权限规则
> **操作权限 = 用户身份 + 角色等级 + 功能授权 + 操作日志**

## ② 参考表

| 角色 | 典型权限 |
|---|---|
| **操作员** | 启停、复位、查看状态 |
| **班组长** | 参数微调、报警确认 |
| **工程师** | 配方、工艺参数、调试画面 |
| **管理员** | 用户管理、系统设置、备份恢复 |
| **维护人员** | I/O监控、手动动作、诊断 |
| **只读用户** | 查看数据，不允许操作 |
| **审计账号** | 查看日志和历史记录 |

## ③ 计算/选型示例

- 已知：  
  操作员误改伺服速度参数，导致设备节拍异常。
- 公式：  
  **关键参数 = 权限限制 + 修改记录**
- 计算：  
  普通操作员不应直接修改核心工艺参数。
- 结论：  
  参数设置页面应加权限，并记录修改人、时间和旧值新值。

> [!tip] 记忆口诀
> 谁能看、谁能改、谁改了什么，都要能查。

# 05 参数设置

> [!info] 核心结论
> 参数设置用于管理设备运行参数，必须配置范围限制、单位说明、保存机制和权限保护。

## ① 分类/公式/规则

> [!warning] 参数保护公式
> **安全参数 = 合理范围 + 单位清楚 + 权限控制 + 掉电保存 + 修改日志**

## ② 参考表

| 参数类型 | 示例 |
|---|---|
| **速度参数** | 输送速度、伺服速度、加速度 |
| **时间参数** | 延时、超时、保压时间 |
| **温度参数** | 设定温度、报警温度 |
| **压力参数** | 气压、液压、真空阈值 |
| **位置参数** | 取放点、补偿量、限位范围 |
| **计数参数** | 目标数量、批次数 |
| **工艺参数** | 配方号、工艺模式、比例系数 |

## ③ 计算/选型示例

- 已知：  
  气缸伸出超时时间正常为3秒，误设为0.1秒。
- 公式：  
  **参数异常 → 设备误报警**
- 计算：  
  0.1秒小于气缸实际动作时间，系统会频繁报警。
- 结论：  
  参数输入应限制范围，例如1~10秒。

> [!tip] 记忆口诀
> 参数能改是方便，范围不控就是风险。

# 06 报警设置

> [!info] 核心结论
> 报警设置决定系统如何显示、确认、记录和处理设备故障。

## ① 分类/公式/规则

> [!warning] 报警配置公式
> **报警 = 报警条件 + 报警文本 + 等级 + 时间戳 + 处理建议 + 复位条件**

## ② 参考表

| 报警设置 | 作用 |
|---|---|
| **报警等级** | 提示、警告、停机、急停 |
| **报警文本** | 显示具体故障原因 |
| **报警确认** | 操作员确认已知晓 |
| **报警复位** | 故障消除后复位 |
| **报警历史** | 追溯故障发生记录 |
| **报警声音** | 蜂鸣器或声光报警 |
| **报警过滤** | 防止瞬时抖动误报警 |

## ③ 计算/选型示例

- 已知：  
  温度超过80℃报警，但瞬间波动到81℃后又恢复。
- 公式：  
  **稳定报警 = 超限 + 持续时间**
- 计算：  
  可设置温度超过80℃持续5秒才触发报警。
- 结论：  
  报警设置应避免瞬时波动造成误停机。

> [!tip] 记忆口诀
> 好报警不是只说“故障”，而是告诉哪里故障、怎么处理。

# 07 数据记录设置

> [!info] 核心结论
> 数据记录用于保存产量、报警、参数、趋势、操作日志和质量数据，是设备追溯和故障分析的基础。

## ① 分类/公式/规则

> [!warning] 数据量公式
> **数据量 = 变量数量 × 单点大小 × 采样次数 × 保存周期**

## ② 参考表

| 数据类型 | 作用 |
|---|---|
| **报警记录** | 故障追溯 |
| **操作日志** | 人员操作追踪 |
| **参数修改记录** | 工艺变化追踪 |
| **趋势数据** | 温度、压力、电流变化 |
| **产量数据** | 班次、批次、日统计 |
| **质量数据** | OK/NG、检测结果 |
| **设备状态** | 运行、待机、故障、停机 |

## ③ 计算/选型示例

- 已知：  
  每秒记录50个变量，每个变量4字节，保存24小时。
- 公式：  
  **数据量 = 50 × 4 × 86400**
- 计算：  
  数据量 = **17,280,000字节 ≈ 17.3MB**
- 结论：  
  单日数据不大，但长期保存需要规划清理、导出和备份策略。

> [!tip] 记忆口诀
> 没记录就没追溯，没备份就可能丢证据。

# 08 趋势曲线设置

> [!info] 核心结论
> 趋势曲线用于观察连续变量变化，常用于温度、压力、流量、电流、速度、真空值和液位分析。

## ① 分类/公式/规则

> [!warning] 趋势设置公式
> **趋势曲线 = 变量 + 采样周期 + 显示范围 + 保存时间 + 上下限**

## ② 参考表

| 设置项 | 说明 |
|---|---|
| **采样周期** | 多久采一次数据 |
| **显示时间范围** | 最近10分钟、1小时、24小时 |
| **曲线变量** | 温度、压力、电流等 |
| **上下限线** | 显示报警阈值 |
| **历史查询** | 查看过去数据 |
| **数据导出** | CSV、Excel、数据库 |
| **曲线缩放** | 放大局部变化 |

## ③ 计算/选型示例

- 已知：  
  压力曲线每秒采样1次，显示最近30分钟。
- 公式：  
  **采样点数 = 采样频率 × 时间**
- 计算：  
  1 × 1800 = **1800点**
- 结论：  
  趋势缓存至少需要支持1800个采样点。

> [!tip] 记忆口诀
> 报警告诉你结果，趋势告诉你过程。

# 09 配方设置

> [!info] 核心结论
> 配方设置用于保存不同产品、工艺或型号对应的一组参数，减少人工切换错误。

## ① 分类/公式/规则

> [!warning] 配方公式
> **配方 = 产品编号 + 参数组 + 下载到控制器 + 当前配方确认**

## ② 参考表

| 配方内容 | 示例 |
|---|---|
| **产品编号** | A型、B型、C型 |
| **位置参数** | 取料点、放料点、补偿值 |
| **速度参数** | 输送速度、伺服速度 |
| **时间参数** | 延时、保压、冷却时间 |
| **温度参数** | 设定温度、上下限 |
| **压力参数** | 夹紧压力、真空阈值 |
| **质量参数** | 检测阈值、尺寸范围 |

## ③ 计算/选型示例

- 已知：  
  设备生产8种产品，每种产品有25个工艺参数。
- 公式：  
  **参数总数 = 产品数 × 每种参数数**
- 计算：  
  参数总数 = 8 × 25 = **200个参数值**
- 结论：  
  应使用配方管理，而不是每次人工逐项输入。

> [!tip] 记忆口诀
> 产品多、参数多，就用配方；切换快，出错少。

# 10 画面与显示设置

> [!info] 核心结论
> 画面设置决定运行界面的显示效果、操作体验和现场可读性。

## ① 分类/公式/规则

> [!warning] 画面设置规则
> **好界面 = 状态清楚 + 操作明确 + 字体可读 + 报警醒目 + 权限分明**

## ② 参考表

| 设置项 | 说明 |
|---|---|
| **分辨率** | 匹配显示器或HMI屏幕 |
| **全屏模式** | 防止误切系统桌面 |
| **主题颜色** | 区分运行、报警、维护状态 |
| **字体大小** | 适合现场观看距离 |
| **语言切换** | 中文、英文、多语言 |
| **按钮布局** | 常用操作放明显位置 |
| **触摸校准** | 防止触摸偏移 |

## ③ 计算/选型示例

- 已知：  
  现场操作员戴手套，按钮太小经常误触。
- 公式：  
  **现场触摸操作 → 按钮尺寸要足够大**
- 计算：  
  高频按钮应放在明显位置，并避免过密排列。
- 结论：  
  运行画面要按现场人员操作习惯优化。

> [!tip] 记忆口诀
> 运行画面不是给工程师看的，是给现场人员快速操作和排故用的。

# 11 时间与语言设置

> [!info] 核心结论
> 时间设置影响报警、日志、数据追溯和报表；语言设置影响操作人员理解和跨区域使用。

## ① 分类/公式/规则

> [!warning] 时间规则
> **追溯数据 = 数值 + 准确时间戳 + 设备来源**

## ② 参考表

| 设置项 | 作用 |
|---|---|
| **系统时间** | 报警、日志、数据记录基础 |
| **NTP校时** | 与服务器统一时间 |
| **时区设置** | 防止跨区域时间偏移 |
| **日期格式** | 年月日显示格式 |
| **语言设置** | 中文、英文、多语言 |
| **单位制** | ℃、MPa、mm、inch等 |
| **班次时间** | 产量统计和报表计算 |

## ③ 计算/选型示例

- 已知：  
  报警历史时间比实际时间慢10分钟。
- 公式：  
  **时间不准 → 追溯不准**
- 计算：  
  故障分析会出现顺序错误或班次统计错误。
- 结论：  
  运行系统应启用NTP或定期校时。

> [!tip] 记忆口诀
> 时间不准，日志就不可信。

# 12 数据库设置

> [!info] 核心结论
> 数据库设置用于配置生产数据、报警、配方、质量记录和报表的保存位置与访问方式。

## ① 分类/公式/规则

> [!warning] 数据库连接公式
> **数据库可用 = 地址正确 + 账号权限 + 表结构正确 + 网络正常 + 存储空间足够**

## ② 参考表

| 设置项 | 说明 |
|---|---|
| **数据库类型** | SQLite、MySQL、SQL Server、PostgreSQL |
| **服务器地址** | 本机或远程服务器 |
| **账号密码** | 数据库访问权限 |
| **表结构** | 字段、类型、索引 |
| **写入周期** | 实时、周期、事件触发 |
| **备份策略** | 定时备份、异地备份 |
| **清理策略** | 定期删除旧数据 |

## ③ 计算/选型示例

- 已知：  
  每天产生500MB数据，需要保存90天。
- 公式：  
  **存储容量 = 每日数据量 × 保存天数**
- 计算：  
  500MB × 90 = **45GB**
- 结论：  
  数据库磁盘至少应大于45GB，并预留索引、备份和增长空间。

> [!tip] 记忆口诀
> 数据库不怕写，怕没人管容量、备份和清理。

# 13 网络设置

> [!info] 核心结论
> 网络设置影响设备与PLC、服务器、MES、云平台、远程维护系统之间的连接稳定性。

## ① 分类/公式/规则

> [!warning] 网络可达公式
> **网络可达 = IP正确 + 网段正确 + 网关正确 + DNS正确 + 防火墙允许**

## ② 参考表

| 设置项 | 说明 |
|---|---|
| **IP地址** | 设备网络地址 |
| **子网掩码** | 判断同网段范围 |
| **网关** | 跨网段通信需要 |
| **DNS** | 域名访问需要 |
| **端口号** | 通讯服务入口 |
| **防火墙** | 控制访问权限 |
| **双网卡** | 控制网与数据网隔离 |

## ③ 计算/选型示例

- 已知：  
  工控机有两个网口，一个接PLC网，一个接MES网。
- 公式：  
  **控制网与信息网分离 → 双网口配置**
- 计算：  
  PLC网不应直接暴露到办公或互联网。
- 结论：  
  应正确配置双网卡、路由和防火墙规则。

> [!tip] 记忆口诀
> IP不对连不上，网关不对出不去，防火墙不放行也不通。

# 14 备份与恢复设置

> [!info] 核心结论
> 备份恢复设置用于保护工程、参数、配方、数据库、日志和系统镜像，保证故障后能快速恢复。

## ① 分类/公式/规则

> [!warning] 备份公式
> **可恢复系统 = 工程备份 + 参数备份 + 配方备份 + 数据库备份 + 系统镜像**

## ② 参考表

| 备份内容 | 作用 |
|---|---|
| **工程文件** | HMI/SCADA/上位机项目 |
| **参数文件** | 设备运行参数 |
| **配方文件** | 产品工艺参数 |
| **数据库** | 生产、报警、质量数据 |
| **用户权限** | 账号、角色、密码策略 |
| **系统镜像** | 工控机系统快速恢复 |
| **授权文件** | 软件授权、加密狗信息 |

## ③ 计算/选型示例

- 已知：  
  工控机硬盘损坏，现场没有工程备份。
- 公式：  
  **无备份 = 恢复时间不可控**
- 计算：  
  重新配置通讯、画面、数据库和参数可能耗费大量时间。
- 结论：  
  设备交付时必须提供完整备份和恢复流程。

> [!tip] 记忆口诀
> 备份不是可选项，是设备交付的一部分。

# 15 安全设置

> [!info] 核心结论
> 安全设置用于保护运行系统不被误操作、非法访问、恶意修改或未经授权远程连接。

## ① 分类/公式/规则

> [!warning] 安全公式
> **系统安全 = 强密码 + 权限分级 + 操作日志 + 白名单 + 关闭不必要服务**

## ② 参考表

| 安全项 | 作用 |
|---|---|
| **密码策略** | 防止弱口令 |
| **账号权限** | 限制操作范围 |
| **锁屏超时** | 防止无人值守误操作 |
| **操作日志** | 追踪关键操作 |
| **远程访问控制** | 限制远程调试入口 |
| **端口白名单** | 只允许必要通讯 |
| **软件防篡改** | 防止工程被非法修改 |

## ③ 计算/选型示例

- 已知：  
  运行系统仍使用默认管理员密码。
- 公式：  
  **默认密码 = 高安全风险**
- 计算：  
  任何知道默认密码的人都可能进入系统修改参数。
- 结论：  
  上线前必须修改默认密码，并建立权限分级。

> [!tip] 记忆口诀
> 系统越容易远程维护，越要做好权限和日志。

# 16 远程维护设置

> [!info] 核心结论
> 远程维护设置用于让工程师远程查看、诊断、更新和维护系统，但必须受权限和安全策略控制。

## ① 分类/公式/规则

> [!warning] 远程维护公式
> **远程维护可用 = 网络在线 + VPN/平台连接 + 权限允许 + 日志记录**

## ② 参考表

| 远程项 | 说明 |
|---|---|
| **VPN连接** | 安全远程访问 |
| **远程桌面** | 维护工控机或上位机 |
| **远程下载** | 更新PLC/HMI/软件程序 |
| **远程日志** | 查看报警、运行、通讯记录 |
| **远程备份** | 上传工程和数据库备份 |
| **访问权限** | 限制可访问设备 |
| **维护窗口** | 规定允许维护时间 |

## ③ 计算/选型示例

- 已知：  
  工程师能远程进入系统，但无法访问PLC。
- 公式：  
  **远程连接成功 ≠ PLC可达**
- 计算：  
  可能是路由、防火墙、PLC网关或VPN权限未配置。
- 结论：  
  远程维护要同时检查系统入口和目标设备访问权限。

> [!tip] 记忆口诀
> 远程连上只是进门，能不能到PLC还要看路由和权限。

# 17 常见故障排查

> [!info] 核心结论
> 运行系统设置故障排查按 **启动、权限、通讯、参数、报警、数据库、网络、备份、安全** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **运行异常 = 启动失败 + 通讯异常 + 权限不足 + 参数错误 + 数据库异常 + 网络阻断 + 配置丢失**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **软件不自启** | 启动项未配置、权限不足、路径错误 |
| **登录失败** | 密码错误、账号禁用、权限配置错误 |
| **PLC通讯中断** | IP、协议、网络、PLC状态异常 |
| **参数不保存** | 保存路径、数据库、权限或掉电保持异常 |
| **报警不记录** | 报警历史未启用、数据库异常 |
| **趋势无数据** | 变量未绑定、采样周期错误 |
| **远程连不上** | VPN、路由、防火墙、账号权限问题 |
| **数据丢失** | 未备份、磁盘满、数据库损坏 |

## ③ 计算/选型示例

- 已知：  
  参数修改后重启系统又恢复旧值。
- 公式：  
  **参数未保存 = 写入位置错误 / 权限不足 / 保存动作未执行**
- 计算：  
  可能只修改了画面临时变量，没有写入PLC或数据库。
- 结论：  
  应检查参数保存逻辑、掉电保持区和数据库写入状态。

> [!tip] 记忆口诀
> 运行系统异常，先看设置，再看通讯，最后看程序逻辑。

# 18 运行系统设置快速检查表

> [!info] 核心结论
> 运行系统设置交付前必须逐项检查，确保设备断电重启、权限切换、报警记录、数据保存和远程维护都能正常工作。

## ① 分类/公式/规则

> [!note] 十步检查法
> ① 检查开机自启和默认画面  
> ② 检查PLC/网关/数据库通讯  
> ③ 检查用户权限和密码策略  
> ④ 检查参数范围、保存和修改日志  
> ⑤ 检查报警显示、确认、复位和历史记录  
> ⑥ 检查趋势曲线和数据记录  
> ⑦ 检查配方下载、保存和切换  
> ⑧ 检查时间同步、语言和单位  
> ⑨ 检查网络、防火墙和远程维护  
> ⑩ 检查工程、数据库、参数和系统镜像备份

> [!warning] 最终配置公式
> **稳定运行系统 = 自动启动 + 通讯可靠 + 权限清晰 + 数据可追溯 + 备份可恢复 + 安全可控**

## ② 参考表

| 需求 | 推荐检查项 |
|---|---|
| 开机自动运行 | 启动项、延时、自恢复 |
| PLC稳定通讯 | IP、协议、变量地址 |
| 防止误操作 | 权限、确认弹窗、范围限制 |
| 故障追溯 | 报警历史、操作日志 |
| 质量追溯 | 数据记录、时间戳、数据库 |
| 多产品切换 | 配方管理 |
| 远程维护 | VPN、账号、访问白名单 |
| 故障恢复 | 备份、镜像、恢复流程 |

## ③ 计算/选型示例

- 已知：  
  一套设备需要无人值守运行，异常时报警记录，断网时本地保存数据，恢复后继续上传。
- 公式：  
  **无人值守 = 自启动 + 报警记录 + 本地缓存 + 自动恢复**
- 计算：  
  运行系统设置必须覆盖启动、报警、数据库、网络和备份。
- 结论：  
  推荐建立完整的运行系统设置清单，并在交付前做断电、断网、重启、权限和报警测试。

> [!tip] 选型口诀
> 能自启，能通讯；有权限，有日志；参数能保存，报警能追溯；坏了能恢复，远程能维护。
