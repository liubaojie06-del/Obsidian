# 01 任务面板总览

> [!info] 核心结论
> “任务”面板用于处理HMI/SCADA工程中的辅助工作，例如 **查找和替换、语言切换、资源管理、文本翻译、多语言检查和工程内容定位**。

## ① 分类/公式/规则

> [!warning] 使用公式
> **任务面板 = 查找替换 + 语言资源 + 工程文本管理 + 多语言维护**

## ② 参考表

| 功能区 | 作用 |
|---|---|
| **查找和替换** | 查找对象、变量、文本、地址并批量替换 |
| **语言和资源** | 设置编辑语言、参考语言和多语言资源 |
| **编辑语言** | 当前正在编辑的语言文本 |
| **参考语言** | 用于对照翻译或核对的语言 |
| **资源管理** | 管理文本、图形、语言包等资源 |
| **多语言检查** | 检查翻译缺失、文本不一致 |
| **工程维护** | 快速定位画面、变量、脚本和文本 |

## ③ 计算/选型示例

- 已知：  
  HMI工程里有大量按钮文字、报警文本和参数名称，需要从中文扩展到英文。
- 公式：  
  **多语言工程 = 编辑语言 + 参考语言 + 文本资源管理**
- 计算：  
  以中文为参考语言，编辑英文语言内容。
- 结论：  
  应使用“语言和资源”进行统一维护，而不是逐个画面手动修改。

> [!tip] 记忆口诀
> 页面对象靠画面管理，文本翻译靠语言资源，批量修改靠查找替换。

# 02 查找和替换

> [!info] 核心结论
> 查找和替换用于在工程中快速定位并批量修改文本、变量名、对象名、地址、脚本内容和报警信息。

## ① 分类/公式/规则

> [!warning] 查找替换公式
> **批量修改 = 查找范围 + 查找内容 + 替换内容 + 结果确认**

## ② 参考表

| 查找对象 | 示例 |
|---|---|
| **画面文本** | 启动、停止、复位 |
| **报警文本** | 气压不足、安全门未关 |
| **变量名称** | Motor_Run、Valve_Open |
| **PLC地址** | DB地址、M区、I/O地址 |
| **对象名称** | Button_01、Text_Alarm |
| **脚本内容** | 函数名、变量调用 |
| **配方字段** | 产品名称、参数名称 |

## ③ 计算/选型示例

- 已知：  
  工程中所有“启动”按钮要改成“运行启动”。
- 公式：  
  **查找“启动” → 替换“运行启动”**
- 计算：  
  替换前应检查结果，避免把报警文本或说明文本误改。
- 结论：  
  批量替换要谨慎，尤其是通用词汇。

> [!tip] 记忆口诀
> 查找能省时间，替换要防误伤。

# 03 查找范围

> [!info] 核心结论
> 查找范围决定搜索会影响哪些工程内容，范围越大越全面，但误替换风险也越高。

## ① 分类/公式/规则

> [!warning] 范围选择规则
> **只改某类内容 → 限定查找范围**  
> **不确定位置 → 全工程查找，手动确认结果**

## ② 参考表

| 查找范围 | 适合场景 |
|---|---|
| **当前画面** | 只修改当前页面文本 |
| **全部画面** | 查找所有页面对象 |
| **变量表** | 查找变量名或地址 |
| **报警文本** | 修改报警说明 |
| **脚本区域** | 查找函数或变量调用 |
| **配方资源** | 查找配方字段 |
| **整个工程** | 不确定目标位置时使用 |

## ③ 计算/选型示例

- 已知：  
  只想修改参数画面中的“速度”文字。
- 公式：  
  **目标明确 → 当前画面或参数页面范围**
- 计算：  
  如果全工程替换“速度”，可能影响报警、趋势和配方文本。
- 结论：  
  应优先限定查找范围。

> [!tip] 记忆口诀
> 范围越小越安全，范围越大越要确认。

# 04 替换风险

> [!info] 核心结论
> 替换操作可能影响画面文本、变量引用、脚本函数和报警描述，必须先备份再执行。

## ① 分类/公式/规则

> [!warning] 替换安全公式
> **安全替换 = 工程备份 + 搜索预览 + 单条确认 + 修改后测试**

## ② 参考表

| 风险 | 说明 |
|---|---|
| **误改变量名** | 导致变量绑定失效 |
| **误改脚本函数** | 导致脚本编译失败 |
| **误改报警文本** | 导致报警含义错误 |
| **误改单位** | 导致现场误解 |
| **误改对象名** | 影响脚本引用 |
| **误改配方字段** | 影响配方读取 |
| **误改多语言文本** | 导致翻译不一致 |

## ③ 计算/选型示例

- 已知：  
  把“Motor”批量替换成“Servo”。
- 公式：  
  **通用词替换 → 高风险**
- 计算：  
  可能把Motor_Run、Motor_Stop、Motor_Alarm等变量全部改名，导致PLC变量不匹配。
- 结论：  
  涉及变量和脚本时，替换前必须备份并逐条确认。

> [!tip] 记忆口诀
> 文本替换影响显示，变量替换可能影响功能。

# 05 语言和资源

> [!info] 核心结论
> 语言和资源用于管理工程中的文本语言、翻译内容、参考语言、图形资源和多语言显示。

## ① 分类/公式/规则

> [!warning] 多语言管理公式
> **多语言工程 = 文本资源库 + 编辑语言 + 参考语言 + 运行语言切换**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **编辑语言** | 当前正在编辑的语言 |
| **参考语言** | 用于翻译对照的语言 |
| **文本资源** | 按钮、标题、报警、参数文本 |
| **图形资源** | 图标、图片、设备图 |
| **语言包** | 不同语言对应文本 |
| **翻译状态** | 已翻译、未翻译、需更新 |
| **运行语言** | 设备运行时显示的语言 |

## ③ 计算/选型示例

- 已知：  
  当前编辑语言为中文，参考语言也为中文。
- 公式：  
  **单语言工程 → 编辑语言 = 参考语言**
- 计算：  
  如果只做中文工程，这样设置可以正常编辑。
- 结论：  
  若要做英文版本，应把编辑语言切到英文，参考语言保留中文用于对照。

> [!tip] 记忆口诀
> 编辑语言是你正在改的，参考语言是拿来对照的。

# 06 编辑语言

> [!info] 核心结论
> 编辑语言决定当前修改的是哪一套语言文本，例如中文、英文、德文、日文等。

## ① 分类/公式/规则

> [!warning] 编辑语言规则
> **当前编辑语言 = 当前输入和修改的目标语言**

## ② 参考表

| 编辑语言场景 | 说明 |
|---|---|
| **中文编辑** | 修改中文界面文本 |
| **英文编辑** | 修改英文界面文本 |
| **多语言翻译** | 切换目标语言逐项翻译 |
| **报警文本编辑** | 当前语言下的报警内容 |
| **按钮文本编辑** | 当前语言下的按钮名称 |
| **参数名称编辑** | 当前语言下的参数说明 |
| **帮助文本编辑** | 当前语言下的操作提示 |

## ③ 计算/选型示例

- 已知：  
  需要把中文按钮“启动”翻译成英文“Start”。
- 公式：  
  **编辑语言切到英文 → 修改该按钮文本为Start**
- 计算：  
  中文文本保持“启动”，英文文本显示“Start”。
- 结论：  
  多语言工程不要直接覆盖中文，应在对应编辑语言下修改。

> [!tip] 记忆口诀
> 改哪种语言，先切到哪种编辑语言。

# 07 参考语言

> [!info] 核心结论
> 参考语言用于在翻译或编辑时提供对照，通常选择最完整、最准确的原始语言。

## ① 分类/公式/规则

> [!warning] 参考语言规则
> **参考语言 = 翻译依据，不是当前被修改的语言**

## ② 参考表

| 参考语言用途 | 说明 |
|---|---|
| **翻译对照** | 根据原文翻译目标语言 |
| **缺失检查** | 判断目标语言是否漏翻 |
| **文本一致性** | 保持多语言含义一致 |
| **报警核对** | 避免报警翻译错误 |
| **参数核对** | 确认单位和含义 |
| **帮助说明核对** | 保证操作指导一致 |
| **版本更新对照** | 新增文本后检查其他语言是否补齐 |

## ③ 计算/选型示例

- 已知：  
  中文文本最完整，英文文本需要补全。
- 公式：  
  **参考语言=中文，编辑语言=英文**
- 计算：  
  一边看中文，一边填写英文。
- 结论：  
  这种设置最适合中英双语工程翻译。

> [!tip] 记忆口诀
> 参考语言看原文，编辑语言写译文。

# 08 多语言资源表

> [!info] 核心结论
> 多语言资源表集中管理工程文本，避免每个画面单独维护，适合按钮、标题、报警、参数、帮助说明等内容。

## ① 分类/公式/规则

> [!warning] 文本资源公式
> **文本资源 = 文本ID + 中文 + 英文 + 其他语言 + 使用位置**

## ② 参考表

| 字段 | 说明 |
|---|---|
| **文本ID** | 唯一标识一条文本 |
| **中文文本** | 中文显示内容 |
| **英文文本** | 英文显示内容 |
| **使用对象** | 按钮、标签、报警、弹窗 |
| **使用画面** | 文本所在页面 |
| **翻译状态** | 已翻译/未翻译 |
| **备注说明** | 特殊语境或单位说明 |

## ③ 计算/选型示例

- 已知：  
  多个页面都使用“报警复位”文本。
- 公式：  
  **重复文本 → 文本资源统一管理**
- 计算：  
  修改资源表中的英文翻译，所有引用处同步更新。
- 结论：  
  常用文本应放入资源库，避免多处不一致。

> [!tip] 记忆口诀
> 文本统一进资源表，翻译维护少出错。

# 09 报警文本多语言

> [!info] 核心结论
> 报警文本多语言最容易影响现场排故，必须翻译准确，并包含设备、原因和处理建议。

## ① 分类/公式/规则

> [!warning] 报警文本公式
> **报警文本 = 故障对象 + 故障原因 + 处理建议**

## ② 参考表

| 报警内容 | 示例 |
|---|---|
| **故障对象** | 气缸1、伺服轴A、安全门 |
| **故障原因** | 未到位、通讯中断、气压不足 |
| **处理建议** | 检查传感器、检查气源、复位驱动器 |
| **报警等级** | 提示、警告、停机 |
| **发生时间** | 用于追溯 |
| **恢复条件** | 故障消除后复位 |
| **多语言文本** | 中文、英文等 |

## ③ 计算/选型示例

- 已知：  
  中文报警为“气压不足，请检查气源”。
- 公式：  
  **报警翻译要保留原因和建议**
- 计算：  
  英文不能只写“Alarm”，应表达低气压和检查气源。
- 结论：  
  报警文本翻译不能过度简化，否则影响维护。

> [!tip] 记忆口诀
> 报警翻译不是文学翻译，是现场排故说明。

# 10 按钮与菜单多语言

> [!info] 核心结论
> 按钮和菜单文本必须短、清晰、一致，避免不同页面同一功能使用不同说法。

## ① 分类/公式/规则

> [!warning] 按钮文本规则
> **同一功能 = 同一术语 + 同一翻译**

## ② 参考表

| 中文 | 常见英文 |
|---|---|
| **启动** | Start |
| **停止** | Stop |
| **复位** | Reset |
| **确认** | Confirm |
| **取消** | Cancel |
| **保存** | Save |
| **下载** | Download |
| **返回** | Back |

## ③ 计算/选型示例

- 已知：  
  有些页面把“复位”翻译为Reset，有些页面翻译为Recover。
- 公式：  
  **术语不统一 → 操作理解风险**
- 计算：  
  操作员可能以为两个按钮功能不同。
- 结论：  
  常用按钮术语应建立统一翻译表。

> [!tip] 记忆口诀
> 一个功能一个词，别一会儿一个说法。

# 11 参数名称与单位多语言

> [!info] 核心结论
> 参数名称翻译必须保留工程含义和单位，单位不能随意省略或改变。

## ① 分类/公式/规则

> [!warning] 参数文本公式
> **完整参数显示 = 参数名称 + 数值 + 单位 + 范围**

## ② 参考表

| 参数 | 注意 |
|---|---|
| **速度** | mm/s、m/min、rpm要区分 |
| **压力** | MPa、bar、kPa要区分 |
| **温度** | ℃、℉要区分 |
| **时间** | s、ms、min要区分 |
| **位置** | mm、degree、pulse要区分 |
| **电流** | A、mA要区分 |
| **流量** | L/min、m³/h要区分 |

## ③ 计算/选型示例

- 已知：  
  中文参数为“输送速度 mm/s”。
- 公式：  
  **翻译 = Conveyor Speed + mm/s**
- 计算：  
  不能只翻译为Speed，否则不知道是哪一个速度。
- 结论：  
  参数翻译要具体，并保留单位。

> [!tip] 记忆口诀
> 参数少了单位，现场就容易设错。

# 12 图形资源

> [!info] 核心结论
> 图形资源包括图标、图片、背景、设备图、报警图标和按钮图形，多语言工程中应避免把文字直接做进图片里。

## ① 分类/公式/规则

> [!warning] 图形资源规则
> **图片尽量不包含固定文字，文字应由语言资源控制**

## ② 参考表

| 图形资源 | 注意 |
|---|---|
| **设备图标** | 风格统一 |
| **报警图标** | 含义清晰 |
| **按钮图片** | 不建议直接带文字 |
| **背景图** | 不影响文字阅读 |
| **流程图** | 文字标签最好独立 |
| **LOGO** | 注意分辨率和比例 |
| **帮助图片** | 多语言项目需额外处理 |

## ③ 计算/选型示例

- 已知：  
  一个按钮图片上直接写了“启动”两个字。
- 公式：  
  **文字图片化 → 多语言维护困难**
- 计算：  
  切换英文后，图片仍然显示中文。
- 结论：  
  按钮文字应使用文本资源，不建议嵌入图片。

> [!tip] 记忆口诀
> 图片管图形，文字管语言，别把文字画死在图片里。

# 13 翻译缺失检查

> [!info] 核心结论
> 多语言工程必须检查目标语言是否存在空白、未翻译、翻译过期或文本超出控件范围。

## ① 分类/公式/规则

> [!warning] 检查公式
> **多语言完整 = 所有文本ID在所有目标语言中都有有效文本**

## ② 参考表

| 问题 | 影响 |
|---|---|
| **未翻译** | 运行时显示空白或原文 |
| **翻译过长** | 按钮文字被截断 |
| **翻译错误** | 误导操作人员 |
| **单位漏翻** | 参数含义不清 |
| **报警漏翻** | 影响故障处理 |
| **新增文本未同步** | 某些语言缺失 |
| **编码问题** | 特殊字符显示异常 |

## ③ 计算/选型示例

- 已知：  
  中文“参数设置”翻译成英文后为“Parameter Configuration Settings”。
- 公式：  
  **文本长度增加 → 控件可能放不下**
- 计算：  
  英文通常比中文更长，按钮宽度可能需要调整。
- 结论：  
  翻译完成后必须检查画面显示效果。

> [!tip] 记忆口诀
> 翻译不是填完就结束，还要看画面放不放得下。

# 14 编辑语言与运行语言

> [!info] 核心结论
> 编辑语言是在工程环境中修改文本用的语言，运行语言是HMI/SCADA实际运行时显示给操作员看的语言。

## ① 分类/公式/规则

> [!warning] 区分规则
> **编辑语言用于工程编辑，运行语言用于现场显示**

## ② 参考表

| 类型 | 说明 |
|---|---|
| **编辑语言** | 工程师当前编辑文本的语言 |
| **参考语言** | 翻译对照语言 |
| **运行语言** | 现场HMI当前显示语言 |
| **默认语言** | 系统启动后默认显示语言 |
| **语言切换按钮** | 运行时切换语言 |
| **语言变量** | 控制当前显示语言 |
| **语言资源库** | 存放多语言文本 |

## ③ 计算/选型示例

- 已知：  
  工程编辑时选择中文，现场运行时需要显示英文。
- 公式：  
  **现场显示语言 = 运行语言设置**
- 计算：  
  仅改变编辑语言不一定改变现场默认运行语言。
- 结论：  
  交付前要检查运行语言和默认语言设置。

> [!tip] 记忆口诀
> 编辑语言给工程师看，运行语言给操作员看。

> **可导入资源 = 文本ID不变 + 列名正确 + 编码正确 + 格式不破坏**

# 18 常见故障排查

> [!info] 核心结论
> 任务面板相关问题排查按 **查找范围、替换内容、编辑语言、参考语言、文本资源、字体编码、运行语言** 的顺序进行。

## ① 分类/公式/规则

> [!warning] 排错公式
> **语言资源异常 = 编辑语言错误 + 翻译缺失 + 文本未资源化 + 字体不支持 + 运行语言未切换**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **查找不到文本** | 查找范围不对、文本在资源库或图片中 |
| **替换后功能异常** | 误改变量名或脚本内容 |
| **切语言后仍显示中文** | 文本未加入语言资源 |
| **翻译显示空白** | 目标语言缺失 |
| **文字被截断** | 翻译过长，控件太小 |
| **显示乱码** | 编码或字体问题 |
| **参考语言不对** | 参考语言选择错误 |
| **运行语言不变** | 只改了编辑语言，未改运行语言 |

## ③ 计算/选型示例

- 已知：  
  英文运行时有些按钮显示空白。
- 公式：  
  **按钮空白 = 目标语言文本缺失**
- 计算：  
  检查英文资源表对应文本ID是否为空。
- 结论：  
  应补齐英文翻译并重新下载运行工程。

> [!tip] 记忆口诀
> 多语言显示异常，先查资源表，再查运行语言。

# 19 任务面板快速检查表

> [!info] 核心结论
> 使用任务面板维护工程时，要先确认查找范围和语言设置，再进行替换、翻译、导入导出和运行测试。

## ① 分类/公式/规则

> [!note] 十步检查法
> ① 查找前确认查找范围  
> ② 替换前备份工程  
> ③ 替换前预览结果  
> ④ 替换后检查变量、脚本和画面  
> ⑤ 确认编辑语言是否为目标语言  
> ⑥ 确认参考语言是否为原始语言  
> ⑦ 检查文本资源是否完整  
> ⑧ 检查报警、按钮、参数和单位翻译  
> ⑨ 检查字体、编码和文本长度  
> ⑩ 下载运行后测试语言切换和画面显示

> [!warning] 最终检查公式
> **合格多语言工程 = 文本完整 + 翻译准确 + 显示正常 + 运行语言可切换 + 资源可维护**

## ② 参考表

| 检查项 | 目标 |
|---|---|
| **查找范围** | 避免漏查或误替换 |
| **替换确认** | 防止误改变量和脚本 |
| **编辑语言** | 确认正在修改目标语言 |
| **参考语言** | 翻译有原文对照 |
| **文本资源** | 所有语言内容完整 |
| **报警文本** | 故障说明准确 |
| **单位符号** | 参数含义清楚 |
| **字体编码** | 不乱码、不缺字 |
| **运行测试** | 现场切换正常 |

## ③ 计算/选型示例

- 已知：  
  工程需要从中文单语言升级为中英文双语言。
- 公式：  
  **中英文工程 = 中文参考语言 + 英文编辑语言 + 文本资源补全 + 运行切换测试**
- 计算：  
  先整理中文文本，再补英文翻译，最后检查画面长度和运行切换。
- 结论：  
  多语言工程应统一通过语言资源维护，不建议复制两套画面。

> [!tip] 选型口诀
> 查找先定范围，替换先做备份；翻译看参考语言，编辑看目标语言；文本进资源，运行要测试。
