# 01 HMI 总览

> [!info] 核心结论
> SIMATIC HMI 选型核心看 **项目规模、画面复杂度、操作方式、安装环境、软件平台**。

## ① 分类/公式/规则

> [!note] 🟦 HMI 分类主线
> 普通小设备看 **精简系列面板**；  
> 新平台小设备看 **Unified Basic Panel**；  
> 中高端设备看 **精智面板 / Unified Comfort Panel**；  
> 人员移动操作看 **移动式面板**；  
> 恶劣环境看 **SIPLUS / SIPLUS RAIL**。

> [!warning] 🟧 通用选型公式
> **HMI等级 = 画面数量 + 变量数量 + 通讯对象 + 操作复杂度 + 环境要求**

## ② 参考表

| HMI 类型 | 主要定位 |
|---|---|
| **SIMATIC 精简系列面板** | 基础操作 / 小型设备 |
| **SIMATIC Unified Basic Panel** | 新一代基础面板 |
| **SIMATIC 精智面板** | 中高端操作面板 |
| **SIMATIC Unified Comfort Panel** | 新一代高性能面板 |
| **SIMATIC 移动式面板** | 手持操作 / 移动调试 |
| **HMI SIPLUS** | 恶劣工业环境 |
| **HMI SIPLUS RAIL** | 轨道交通 / 高要求环境 |

## ③ 计算/选型示例

- 已知：  
  一台小型设备，只需要启动、停止、报警查看和少量参数设置。
- 公式：  
  **HMI需求 = 操作画面 + 参数画面 + 报警画面 + 通讯对象**
- 计算：  
  画面少、变量少、通讯对象少。
- 结论：  
  可优先考虑 **SIMATIC 精简系列面板** 或 **Unified Basic Panel**，具体型号需按屏幕尺寸、接口和软件版本校核。

> [!tip] 记忆口诀
> 小设备用Basic，大设备用Comfort；新平台看Unified，恶劣环境选SIPLUS。

# 02 SIMATIC 精简系列面板

> [!info] 核心结论
> SIMATIC 精简系列面板适合小型设备和基础人机交互，重点是经济、够用、简单。

## ① 分类/公式/规则

> [!note] 🟦 定位
> 精简系列面板主要用于基础操作场景，例如按钮操作、状态显示、报警查看、简单参数设定。

> [!warning] 🟧 基础面板判断公式
> **基础HMI需求 = 少量画面 + 少量变量 + 简单报警 + 单PLC通讯**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 控制规模 | **小型 / 单机设备** |
| 画面数量 | 少量基础画面 |
| 操作功能 | 启停、状态、报警、参数 |
| 通讯对象 | 通常连接单台PLC |
| 成本 | 成本敏感项目优先 |

## ③ 计算/选型示例

- 已知：  
  设备需要 5个操作画面、1个报警画面、少量参数设定。
- 公式：  
  **面板等级 = 画面数量 + 变量数量 + 通讯对象**
- 计算：  
  画面数量少，通讯对象简单。
- 结论：  
  可优先选择 **SIMATIC 精简系列面板**，屏幕尺寸和通讯接口需按厂家手册校核。

> [!tip] 记忆口诀
> 画面少、功能轻、预算紧，精简面板最合适。

# 03 SIMATIC Unified Basic Panel

> [!info] 核心结论
> Unified Basic Panel 适合新平台项目中的基础HMI应用，重点关注 WinCC Unified 兼容性。

## ① 分类/公式/规则

> [!success] 🟩 定位
> Unified Basic Panel 是面向新一代工程平台的基础面板，适合新设备开发和基础操作场景。

> [!warning] 🟧 平台校核公式
> **Unified可用性 = TIA版本支持 + WinCC Unified支持 + PLC通讯兼容 + 项目库兼容**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 项目类型 | **新项目 / 新平台** |
| 软件平台 | WinCC Unified |
| 功能定位 | 基础操作、基础显示 |
| 通讯 | PLC、驱动、远程设备 |
| 注意 | 需校核TIA版本和授权 |

## ③ 计算/选型示例

- 已知：  
  新设备项目准备统一使用 WinCC Unified 平台。
- 公式：  
  **平台适配 = 软件版本 + 面板型号 + PLC型号 + 通讯协议**
- 计算：  
  若项目整体采用 Unified 平台，后期维护和扩展更统一。
- 结论：  
  基础HMI可优先考虑 **SIMATIC Unified Basic Panel**，具体型号需按软件版本和厂家手册校核。

> [!tip] 记忆口诀
> 新平台、轻应用，Unified Basic先看清。

# 04 SIMATIC 精智面板

> [!info] 核心结论
> SIMATIC 精智面板适合中高端设备，适用于画面更多、变量更多、操作更复杂的场景。

## ① 分类/公式/规则

> [!note] 🟦 定位
> 精智面板适合设备级和产线级HMI，常用于复杂画面、报警、趋势、配方和多对象通讯。

> [!warning] 🟧 中高端HMI判断公式
> **复杂度 = 画面数量 + 变量数量 + 报警数量 + 趋势记录 + 配方管理**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 控制规模 | **中型 / 复杂设备** |
| 画面 | 多画面、多菜单 |
| 数据 | 报警、趋势、配方 |
| 通讯 | 多PLC、多设备 |
| 性能 | 响应速度、存储能力 |

## ③ 计算/选型示例

- 已知：  
  设备需要 30个画面、报警记录、趋势显示、配方管理。
- 公式：  
  **HMI负载 = 画面 + 报警 + 趋势 + 配方 + 通讯**
- 计算：  
  功能较多，普通基础面板可能不够。
- 结论：  
  可优先考虑 **SIMATIC 精智面板**，具体屏幕尺寸、存储能力、通讯能力需按厂家手册校核。

> [!tip] 记忆口诀
> 画面多、数据多、功能多，精智面板更稳妥。

# 05 SIMATIC Unified Comfort Panel

> [!info] 核心结论
> Unified Comfort Panel 适合新一代中高端HMI项目，重点是高性能和 Unified 平台集成。

## ① 分类/公式/规则

> [!success] 🟩 定位
> Unified Comfort Panel 面向新一代高性能HMI应用，适合复杂设备、产线和数字化项目。

> [!warning] 🟧 高端Unified判断公式
> **选Unified Comfort = 高画面复杂度 + 高变量量 + 新平台 + 后续数字化扩展**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 项目类型 | **新一代中高端项目** |
| 软件平台 | WinCC Unified |
| 画面需求 | 复杂画面、高交互 |
| 数据需求 | 报警、趋势、配方、记录 |
| 扩展 | 数字化、联网、后期维护 |

## ③ 计算/选型示例

- 已知：  
  一条产线需要多个复杂操作画面、数据记录、趋势显示和后续联网扩展。
- 公式：  
  **面板等级 = 画面复杂度 + 数据功能 + 平台统一性 + 扩展需求**
- 计算：  
  项目复杂度高，并且要求新平台统一维护。
- 结论：  
  优先考虑 **SIMATIC Unified Comfort Panel**，具体型号需按屏幕尺寸、性能、授权和软件版本校核。

> [!tip] 记忆口诀
> 新平台做高端，Unified Comfort更合适。

# 06 SIMATIC 移动式面板

> [!info] 核心结论
> SIMATIC 移动式面板适合操作人员需要在设备周围移动调试、点动、维护的场景。

## ① 分类/公式/规则

> [!abstract] 🟪 定位
> 移动式面板用于手持操作，常见于大型设备、机器人区域、输送线、需要就地调试的场景。

> [!warning] 🟧 移动操作判断公式
> **移动需求 = 设备范围大 + 就地操作 + 调试频繁 + 安全确认**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 操作方式 | **手持 / 移动操作** |
| 使用场景 | 调试、维护、点动 |
| 设备范围 | 大型设备、长产线 |
| 安全需求 | 急停、使能、权限 |
| 注意 | 需校核安全功能和接口 |

## ③ 计算/选型示例

- 已知：  
  一条输送线较长，操作员需要在不同位置点动电机、查看状态。
- 公式：  
  **移动价值 = 固定屏不足 + 就地操作需求 + 调试便利性**
- 计算：  
  固定HMI距离远，现场调试不方便。
- 结论：  
  可考虑 **SIMATIC 移动式面板**，安全功能、连接方式、防护等级需按实际规范/厂家手册校核。

> [!tip] 记忆口诀
> 人要跟着设备走，就选移动式面板。

# 07 HMI SIPLUS

> [!info] 核心结论
> HMI SIPLUS 适合温度、湿度、粉尘、腐蚀等更严苛的工业环境。

## ① 分类/公式/规则

> [!success] 🟩 定位
> SIPLUS 可理解为面向恶劣环境的增强型HMI，适合普通面板难以长期稳定工作的场合。

> [!warning] 🟧 环境选型公式
> **环境风险 = 温度 + 湿度 + 粉尘 + 振动 + 腐蚀 + 安装位置**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 环境 | **恶劣工业环境** |
| 风险 | 高温、低温、湿度、粉尘 |
| 使用场景 | 户外、重工、特殊车间 |
| 重点 | 防护、耐受、可靠性 |
| 注意 | 需按环境等级校核 |

## ③ 计算/选型示例

- 已知：  
  设备安装在潮湿、高粉尘环境中，普通面板故障率较高。
- 公式：  
  **可靠性需求 = 环境恶劣程度 + 连续运行时间 + 停机损失**
- 计算：  
  环境风险高，停机成本高。
- 结论：  
  可考虑 **HMI SIPLUS**，具体耐温、耐湿、防护等级需按厂家手册校核。

> [!tip] 记忆口诀
> 环境越恶劣，越要看SIPLUS。

# 08 HMI SIPLUS RAIL

> [!info] 核心结论
> HMI SIPLUS RAIL 适合轨道交通或对振动、环境、认证要求更高的应用场景。

## ① 分类/公式/规则

> [!abstract] 🟪 定位
> SIPLUS RAIL 面向轨道交通及类似高可靠应用，重点关注认证、振动、环境适应性和长期稳定性。

> [!warning] 🟧 轨道应用判断公式
> **RAIL需求 = 行业认证 + 振动冲击 + 温湿度 + 长周期运行 + 高可靠性**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 行业 | **轨道交通 / 特殊行业** |
| 环境 | 振动、冲击、温湿度 |
| 可靠性 | 长时间稳定运行 |
| 认证 | 行业标准、项目规范 |
| 注意 | 必须按项目规范校核 |

## ③ 计算/选型示例

- 已知：  
  HMI需要安装在轨道交通相关设备中，项目对认证和抗振动有要求。
- 公式：  
  **适用性 = 认证满足 + 环境满足 + 通讯满足 + 维护满足**
- 计算：  
  普通HMI可能无法满足行业规范和环境要求。
- 结论：  
  应评估 **HMI SIPLUS RAIL**，具体认证、接口、安装方式需按实际规范/厂家手册校核。

> [!tip] 记忆口诀
> 普通工业看SIPLUS，轨道交通看RAIL。

# 09 HMI 快速选型表

> [!info] 核心结论
> HMI选型不要只看屏幕大小，要同时看功能、平台、环境和维护周期。

## ① 分类/公式/规则

> [!note] 🟦 四步选型法
> ① 看设备规模  
> ② 看画面复杂度  
> ③ 看软件平台  
> ④ 看现场环境

> [!warning] 🟧 最终选型公式
> **推荐HMI = 屏幕尺寸 + 画面复杂度 + 通讯需求 + 环境等级 + 软件平台**

## ② 参考表

| 需求 | 推荐方向 |
|---|---|
| 小型基础设备 | **精简系列面板** |
| 新平台基础设备 | **Unified Basic Panel** |
| 中高端设备 | **精智面板** |
| 新平台高端设备 | **Unified Comfort Panel** |
| 就地移动操作 | **移动式面板** |
| 恶劣工业环境 | **HMI SIPLUS** |
| 轨道交通场景 | **HMI SIPLUS RAIL** |

## ③ 计算/选型示例

- 已知：  
  一台设备需要 10寸屏、20个画面、报警记录、趋势显示，并安装在普通室内控制柜。
- 公式：  
  **HMI等级 = 屏幕尺寸 + 画面数量 + 数据功能 + 环境要求**
- 计算：  
  画面和数据功能较多，但环境不恶劣。
- 结论：  
  可优先考虑 **精智面板** 或 **Unified Comfort Panel**；若项目采用 Unified 平台，则优先考虑 **Unified Comfort Panel**。

> [!tip] 选型口诀
> 基础Basic，中高Comfort；新平台Unified，恶劣SIPLUS，移动操作选Mobile。
