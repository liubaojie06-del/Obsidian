# 01 控制器总览

> [!info] 核心结论
> SIMATIC 控制器选型，核心是看 **项目规模、控制对象、通讯需求、生命周期**。

## ① 分类/公式/规则

> [!note] 🟦 分类主线
> 小型设备看 **S7-1200**；  
> 中大型产线看 **S7-1500**；  
> 多轴运动看 **Drive Controller**；  
> 分布式现场控制看 **ET200 CPU**。

> [!warning] 🟧 通用选型公式
> **控制器等级 = I/O规模 + 通讯数量 + 程序复杂度 + 节拍要求 + 后期扩展**

## ② 参考表

| 控制器 | 主要定位 |
|---|---|
| **S7-1200** | 小型设备 / 单机控制 |
| **S7-1200 G2** | 新一代小型PLC |
| **S7-1500** | 中大型设备 / 产线控制 |
| **Drive Controller** | 运动控制 / 驱动集成 |
| **S7-300** | 老项目维护 / 改造 |
| **S7-400** | 大型老系统 / 过程控制 |
| **ET200 CPU** | 分布式现场控制 |
| **Device proxy** | 设备代理 / 工程映射 |

## ③ 计算/选型示例

- 已知：  
  一台设备需要普通逻辑控制，I/O点数不多，通讯对象较少。
- 公式：  
  **控制规模 = I/O点数 + 通讯对象 + 程序复杂度**
- 计算：  
  小规模 + 少量通讯 + 简单逻辑。
- 结论：  
  优先考虑 **S7-1200 / S7-1200 G2**，具体型号需按厂家手册校核。

> [!tip] 记忆口诀
> 小机1200，大线1500，轴多Drive，分布ET200，老线300/400。

# 02 SIMATIC S7-1200

> [!info] 核心结论
> S7-1200 适合小型设备、单机设备和成本敏感型自动化项目。

## ① 分类/公式/规则

> [!note] 🟦 定位
> S7-1200 是小型PLC，适合基础逻辑控制、少量模拟量、简单通讯和小规模扩展。

> [!warning] 🟧 I/O预留公式
> **推荐点数 = 实际点数 × 1.2～1.3**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 控制规模 | **小型 / 单机** |
| I/O点数 | DI、DO、AI、AO |
| 通讯 | PROFINET、串口等 |
| 扩展 | 信号模块、通讯模块 |
| 应用 | 包装机、小输送线、小泵站 |

## ③ 计算/选型示例

- 已知：  
  设备需要 **16DI、12DO、2AI**。
- 公式：  
  **推荐点数 = 实际点数 × 1.25**
- 计算：  
  - DI：16 × 1.25 ≈ **20点**
  - DO：12 × 1.25 ≈ **15点**
  - AI：2 × 1.25 ≈ **3点**
- 结论：  
  CPU本体和扩展模块需覆盖 **20DI、15DO、3AI**，具体型号需按厂家手册校核。

> [!tip] 记忆口诀
> 1200管小机，点数先统计，余量留三成，扩展别忘记。

# 03 SIMATIC S7-1200 G2

> [!info] 核心结论
> S7-1200 G2 更适合新项目、小型设备升级和后续长期维护项目。

## ① 分类/公式/规则

> [!success] 🟩 定位
> S7-1200 G2 可理解为新一代小型PLC，适合新设备开发、小型控制系统升级。

> [!warning] 🟧 升级判断公式
> **升级可行性 = 程序兼容 + 模块兼容 + 通讯兼容 + 软件版本支持**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 项目类型 | **新项目 / 升级项目** |
| 软件 | TIA Portal版本 |
| 硬件 | CPU、扩展模块兼容 |
| 通讯 | HMI、变频器、远程I/O |
| 注意 | 需按厂家兼容清单校核 |

## ③ 计算/选型示例

- 已知：  
  原设备使用S7-1200，新项目准备升级。
- 公式：  
  **升级风险 = 软件不兼容 + 硬件不兼容 + 通讯不兼容**
- 计算：  
  检查TIA版本、CPU型号、模块订货号、HMI和变频器通讯。
- 结论：  
  新项目可优先评估 **S7-1200 G2**；老项目替换需按实际规范/厂家手册校核。

> [!tip] 记忆口诀
> 新机看G2，旧机看兼容，程序能迁移，升级才轻松。

# 04 SIMATIC S7-1500

> [!info] 核心结论
> S7-1500 是中大型自动化项目的主力平台，适合复杂逻辑、高性能通讯和产线控制。

## ① 分类/公式/规则

> [!note] 🟦 定位
> S7-1500 是中大型PLC，适合复杂设备、自动化产线、多个站点联动控制。

> [!warning] 🟧 负载判断公式
> **CPU负载 = 程序复杂度 + 通讯负载 + I/O规模 + 工艺节拍**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 控制规模 | **中大型 / 产线级** |
| 程序 | 联锁、报警、配方、数据处理 |
| 通讯 | HMI、远程I/O、变频器、上位机 |
| 性能 | 扫描周期、存储容量 |
| 扩展 | 后续工位、模块余量 |

## ③ 计算/选型示例

- 已知：  
  产线有多个远程I/O站、3台HMI、若干变频器。
- 公式：  
  **系统复杂度 = 通讯对象 + I/O站点 + 程序功能**
- 计算：  
  通讯对象多，程序复杂，后续扩展需求明显。
- 结论：  
  优先选择 **S7-1500**，并按CPU性能、通讯连接数和模块数量校核。

> [!tip] 记忆口诀
> 产线复杂上1500，通讯多、点数多、节拍快都找它。

# 05 SIMATIC Drive Controller

> [!info] 核心结论
> Drive Controller 适合伺服轴多、同步要求高、运动控制复杂的设备。

## ① 分类/公式/规则

> [!abstract] 🟪 定位
> Drive Controller 面向运动控制和驱动集成，适合多轴控制、同步控制、复杂运动工艺。

> [!warning] 🟧 运动复杂度公式
> **运动复杂度 = 轴数 + 同步关系 + 节拍要求 + 驱动通讯周期**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 控制对象 | **伺服轴 / 驱动器** |
| 运动功能 | 定位、同步、电子齿轮、凸轮 |
| 应用 | 包装、印刷、搬运、卷绕 |
| 校核 | 轴数、周期、驱动兼容 |
| 注意 | 需按厂家手册确认支持范围 |

## ③ 计算/选型示例

- 已知：  
  设备有多根伺服轴，需要同步运行。
- 公式：  
  **运动复杂度 = 轴数 + 同步关系 + 节拍要求**
- 计算：  
  多轴 + 同步 + 高节拍，普通逻辑PLC负担较大。
- 结论：  
  优先评估 **SIMATIC Drive Controller**，具体型号按轴数和驱动系统校核。

> [!tip] 记忆口诀
> 逻辑多选PLC，轴多选Drive；同步越复杂，越要一体化。

# 06 SIMATIC S7-300

> [!info] 核心结论
> S7-300 多见于老设备和存量项目，主要用于维护、备件替换和改造评估。

## ① 分类/公式/规则

> [!note] 🟦 定位
> S7-300 是经典PLC平台，常见于老产线、旧设备和历史项目。

> [!warning] 🟧 替换原则
> **替换成功 = 型号一致 + 电气参数一致 + 地址一致 + 程序兼容**

## ② 参考表

| 场景 | 建议 |
|---|---|
| 老设备维修 | 优先按原型号替换 |
| 模块损坏 | 查订货号、点数、电压 |
| 程序维护 | 注意软件版本 |
| 新项目 | 不建议盲目沿用 |
| 升级方向 | 可评估迁移到S7-1500 |

## ③ 计算/选型示例

- 已知：  
  现场S7-300设备某个DI模块损坏。
- 公式：  
  **替换检查 = 订货号 + 点数 + 电压 + 地址**
- 计算：  
  核对原模块型号、输入电压、硬件组态地址。
- 结论：  
  优先同型号替换；若备件困难，再评估系统升级。

> [!tip] 记忆口诀
> S7-300多是老朋友，维修先照旧，升级再评估。

# 07 SIMATIC S7-400

> [!info] 核心结论
> S7-400 常用于大型老系统，维护和改造时要关注整体架构，而不是只看CPU。

## ① 分类/公式/规则

> [!abstract] 🟪 定位
> S7-400 是大型经典PLC平台，常用于大型产线、过程控制和高可靠性系统。

> [!warning] 🟧 改造风险公式
> **改造风险 = 停机时间 + 备件可得性 + 程序复杂度 + 通讯兼容性**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 系统规模 | **大型 / 老系统** |
| 架构 | 机架、电源、CPU、通讯模块 |
| 通讯 | 远程I/O、上位机、其他控制器 |
| 风险 | 停产、备件、版本兼容 |
| 升级 | 通常评估S7-1500方案 |

## ③ 计算/选型示例

- 已知：  
  大型老产线使用S7-400，计划局部改造。
- 公式：  
  **改造工作量 = 硬件迁移 + 程序迁移 + 通讯迁移**
- 计算：  
  若只换模块，尽量保持原结构；若整线升级，需完整评估停机和调试时间。
- 结论：  
  S7-400改造必须做系统级评估，关键参数需按实际规范/厂家手册校核。

> [!tip] 记忆口诀
> S7-400系统大，改造别只看CPU，要看整套架构图。

# 08 SIMATIC ET200 CPU

> [!info] 核心结论
> ET200 CPU 适合分布式控制，可把部分控制逻辑放到现场I/O站附近执行。

## ① 分类/公式/规则

> [!success] 🟩 定位
> ET200 CPU 是带CPU能力的分布式控制节点，不只是普通远程I/O。

> [!warning] 🟧 分布式判断公式
> **分布价值 = 减少布线 + 本地控制 + 维护便利 + 故障隔离**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 控制位置 | **现场分布式控制** |
| I/O位置 | 靠近传感器和执行器 |
| 优点 | 减少长距离布线 |
| 通讯 | 与主站、HMI、驱动联网 |
| 校核 | CPU能力、模块组合、网络拓扑 |

## ③ 计算/选型示例

- 已知：  
  输送线很长，多个控制柜分散布置。
- 公式：  
  **布线收益 = 原长距离电缆 - 分布式现场接线**
- 计算：  
  信号若全部拉回主柜，布线复杂、维护困难；现场控制更清晰。
- 结论：  
  可考虑 **ET200 CPU** 做分布式控制，网络结构需按现场拓扑校核。

> [!tip] 记忆口诀
> 远距离别硬拉线，ET200放现场，控制更清爽。

# 09 Device Proxy

> [!info] 核心结论
> Device proxy 不是实际PLC CPU，而是工程中的设备代理对象，用于表示、映射或集成外部设备。

## ① 分类/公式/规则

> [!abstract] 🟪 定位
> Device proxy 是工程里的“设备替身”，用于网络规划、接口映射和外部设备集成。

> [!warning] 🟧 代理判断公式
> **需要代理 = 外部设备存在 + 工程需表达 + 通讯需映射**

## ② 参考表

| 项目 | 说明 |
|---|---|
| 类型 | **设备代理** |
| 作用 | 代表外部设备 |
| 用途 | 工程集成、接口映射 |
| 注意 | 不等同于真实CPU |
| 校核 | 需按TIA项目结构确认 |

## ③ 计算/选型示例

- 已知：  
  项目中有第三方设备，需要在TIA工程中体现通讯关系。
- 公式：  
  **代理需求 = 第三方设备 + 网络关系 + 工程映射**
- 计算：  
  设备不是标准SIMATIC CPU，但工程中需要表示它的接口关系。
- 结论：  
  可使用 **Device proxy** 辅助工程集成，具体配置需按项目和设备手册校核。

> [!tip] 记忆口诀
> Device proxy不是CPU，是工程里的设备替身。

# 10 快速选型口诀

> [!info] 核心结论
> 选控制器不要先看型号，要先看项目需求。

## ① 分类/公式/规则

> [!note] 🟦 四步选型法
> ① 看规模 → ② 看对象 → ③ 看通讯 → ④ 看生命周期

> [!warning] 🟧 最终选型公式
> **推荐控制器 = 控制规模 + 运动需求 + 通讯复杂度 + 维护周期**

## ② 参考表

| 需求 | 推荐方向 |
|---|---|
| 小型单机 | **S7-1200** |
| 新小型设备 | **S7-1200 G2** |
| 中大型产线 | **S7-1500** |
| 多轴运动 | **Drive Controller** |
| 老线维护 | **S7-300 / S7-400** |
| 分布式控制 | **ET200 CPU** |
| 外部设备映射 | **Device proxy** |

## ③ 计算/选型示例

- 已知：  
  项目包含多站I/O、多台HMI、变频器通讯，后期还要扩展。
- 公式：  
  **控制器等级 = I/O规模 + 通讯数量 + 扩展余量**
- 计算：  
  I/O站多、通讯对象多、后续扩展明显。
- 结论：  
  优先考虑 **S7-1500**；若同时存在大量伺服轴，再评估 **Drive Controller**。

> [!tip] 记忆口诀
> 小型1200，新机G2；大线1500，轴多Drive；远程ET200，老线300/400；代理用Proxy。
