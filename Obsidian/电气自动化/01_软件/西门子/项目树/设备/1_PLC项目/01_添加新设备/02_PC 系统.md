# 01 PC 系统总览

> [!info] 核心结论
> PC 系统主要用于 **上位机、HMI运行时、工业数据处理、PC-Based控制、用户自定义应用** 等场景。

## ① 分类/公式/规则

> [!note] 🟦 分类主线
> 普通办公环境看 **常规 PC**；  
> 工业现场长期运行看 **工业 PC**；  
> 需要 PC + PLC 一体化控制看 **SIMATIC S7 开放式控制器 / 嵌入式控制器**；  
> 需要软件运行环境看 **HMI应用软件 / 用户应用程序**。

> [!warning] 🟧 通用选型公式
> **PC系统等级 = 运行软件 + 通讯对象 + 环境等级 + 实时性要求 + 数据处理量**

## ② 参考表

| 类型 | 主要定位 |
|---|---|
| **常规 PC** | 普通PC / 工程调试 / 非严苛环境 |
| **工业 PC** | 工业现场长期运行 |
| **S7 开放式控制器** | PC + PLC 融合控制 |
| **S7 嵌入式控制器** | 嵌入式自动化控制 |
| **控制器应用程序** | 软件控制器 / 控制类应用 |
| **HMI 应用软件** | HMI Runtime / 操作画面 |
| **用户应用程序** | 自定义软件 / 第三方程序 |

## ③ 计算/选型示例

- 已知：  
  项目需要在电脑上运行HMI画面，同时采集PLC数据。
- 公式：  
  **PC需求 = HMI运行 + PLC通讯 + 数据记录 + 环境要求**
- 计算：  
  如果安装在办公室，可考虑常规PC；如果安装在车间控制柜，应优先考虑工业PC。
- 结论：  
  工业现场长期运行建议选 **工业 PC**，具体CPU、内存、硬盘、系统版本、通讯接口需按实际规范/厂家手册校核。

> [!tip] 记忆口诀
> 办公调试用常规，现场长期用工业；要控制选S7，跑画面选HMI。

# 02 常规 PC

> [!info] 核心结论
> 常规 PC 适合工程调试、办公环境、临时运行和非严苛场景，不建议直接用于恶劣工业现场长期运行。

## ① 分类/公式/规则

> [!note] 🟦 定位
> 常规 PC 通常指普通台式机、笔记本或商用电脑，可用于工程软件、调试、监控或轻量级运行任务。

> [!warning] 🟧 适用判断公式
> **常规PC适用性 = 环境稳定 + 实时性低 + 停机影响小 + 成本敏感**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 使用环境 | **办公室 / 调试现场** |
| 运行任务 | 工程软件、简单监控 |
| 优点 | 成本低、获取方便 |
| 风险 | 抗干扰、散热、稳定性较弱 |
| 注意 | 不适合恶劣现场长期运行 |

## ③ 计算/选型示例

- 已知：  
  工程师需要一台电脑安装 TIA Portal，用于程序下载和调试。
- 公式：  
  **电脑配置 = 软件要求 + 项目规模 + 调试需求**
- 计算：  
  只用于工程调试，不承担现场连续生产任务。
- 结论：  
  可使用 **常规 PC / 笔记本**；但CPU、内存、硬盘容量和系统版本需满足软件要求。

> [!tip] 记忆口诀
> 调试办公用常规，上线生产要慎重。

---

# 03 工业 PC

> [!info] 核心结论
> 工业 PC 适合车间、控制柜、产线等工业现场长期运行，重点看稳定性、环境适应性和接口能力。

## ① 分类/公式/规则

> [!note] 🟦 定位
> 工业 PC 是面向工业环境设计的计算机，常用于HMI运行、数据采集、边缘计算、上位机监控和工业通讯。

> [!warning] 🟧 工业PC选型公式
> **工业PC配置 = CPU性能 + 内存容量 + 存储方式 + 通讯接口 + 环境等级**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 使用环境 | **车间 / 控制柜 / 产线** |
| 核心优势 | 稳定、耐用、适合长期运行 |
| 常见任务 | HMI、SCADA、数据采集 |
| 重点接口 | 以太网、USB、串口、扩展槽 |
| 校核 | 温度、防护、安装方式、电源 |

## ③ 计算/选型示例

- 已知：  
  产线需要一台PC连续运行HMI Runtime，并记录生产数据。
- 公式：  
  **可靠性需求 = 连续运行时间 + 环境恶劣程度 + 停机损失**
- 计算：  
  设备在车间长期运行，停机会影响生产。
- 结论：  
  优先选择 **工业 PC**，并按CPU、内存、硬盘、散热、防护等级和通讯接口校核。

> [!tip] 记忆口诀
> 能不能跑看配置，能不能稳看工业等级。

# 04 SIMATIC S7 开放式控制器

> [!info] 核心结论
> S7 开放式控制器适合需要 **PLC控制能力 + PC开放环境** 的自动化项目。

## ① 分类/公式/规则

> [!note] 🟦 定位
> 开放式控制器可理解为把PLC控制与PC系统结合，用于同时处理自动化控制、数据处理、HMI或用户软件。

> [!warning] 🟧 开放式控制判断公式
> **开放式控制需求 = PLC控制 + PC应用 + 数据处理 + 高集成度**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 控制方式 | **PC-Based控制** |
| 适合场景 | 控制 + 数据 + 应用集成 |
| 优点 | 集成度高、开放性强 |
| 风险 | 软件版本、实时性、系统维护 |
| 校核 | 控制性能、OS、授权、兼容性 |

## ③ 计算/选型示例

- 已知：  
  项目既要运行控制逻辑，又要运行数据分析或第三方软件。
- 公式：  
  **系统集成度 = 控制任务 + PC任务 + 通讯任务**
- 计算：  
  如果用传统PLC + 独立PC，系统会增加通讯和维护复杂度。
- 结论：  
  可评估 **SIMATIC S7 开放式控制器**，但实时性、软件兼容性和授权需按厂家手册校核。

> [!tip] 记忆口诀
> 既要PLC控制，又要PC开放，就看S7开放式控制器。

# 05 SIMATIC S7 嵌入式控制器

> [!info] 核心结论
> S7 嵌入式控制器适合空间紧凑、功能集成、需要稳定自动化控制的应用场景。

## ① 分类/公式/规则

> [!note] 🟦 定位
> 嵌入式控制器强调紧凑、集成和稳定，适合设备内部自动化控制或专用控制系统。

> [!warning] 🟧 嵌入式控制判断公式
> **嵌入式需求 = 空间限制 + 稳定运行 + 控制任务 + 集成安装**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 控制类型 | **嵌入式自动化控制** |
| 适合场景 | 设备内部、专用系统 |
| 优点 | 集成、紧凑、稳定 |
| 关注 | 接口数量、扩展能力 |
| 校核 | 型号、软件版本、通讯能力 |

## ③ 计算/选型示例

- 已知：  
  一台设备空间有限，但需要稳定控制和通讯功能。
- 公式：  
  **嵌入价值 = 节省空间 + 提高集成度 + 降低布线复杂度**
- 计算：  
  如果传统PLC和PC分开安装，占用空间和接线复杂度更高。
- 结论：  
  可考虑 **SIMATIC S7 嵌入式控制器**，具体型号和接口能力需按实际规范/厂家手册校核。

> [!tip] 记忆口诀
> 空间小、要集成，嵌入式控制更省心。

# 06 SIMATIC 控制器应用程序

> [!info] 核心结论
> SIMATIC 控制器应用程序用于PC系统中的控制类软件组件，重点关注实时性、授权和与PLC/设备的通讯关系。

## ① 分类/公式/规则

> [!note] 🟦 定位
> 控制器应用程序通常用于在PC系统中承载控制相关功能，例如软件控制器、控制任务或自动化应用组件。

> [!warning] 🟧 控制应用判断公式
> **控制应用需求 = 控制逻辑 + 实时要求 + 通讯接口 + 软件授权**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 应用类型 | **控制类软件** |
| 运行平台 | PC / 工业PC / 控制器平台 |
| 关键能力 | 实时性、通讯、稳定性 |
| 风险点 | 系统版本、授权、兼容性 |
| 校核 | 软件说明和厂家手册 |

## ③ 计算/选型示例

- 已知：  
  项目希望在工业PC中运行控制任务，同时与现场I/O和HMI通讯。
- 公式：  
  **控制软件负载 = 控制周期 + 通讯数量 + 数据处理量**
- 计算：  
  控制周期越短、通讯对象越多，对系统要求越高。
- 结论：  
  需要评估 **SIMATIC 控制器应用程序** 的实时能力、授权和兼容性，关键参数需按厂家手册校核。

> [!tip] 记忆口诀
> 软件能不能控，看实时；系统能不能稳，看兼容。

# 07 SIMATIC HMI 应用软件

> [!info] 核心结论
> SIMATIC HMI 应用软件用于在PC或工业PC上运行操作画面，是PC系统做人机界面的核心组件。

## ① 分类/公式/规则

> [!note] 🟦 定位
> HMI 应用软件通常用于运行画面、报警、趋势、配方、数据记录等人机交互功能。

> [!warning] 🟧 HMI软件选型公式
> **HMI软件需求 = 画面数量 + 变量数量 + 报警数量 + 数据记录 + 客户端数量**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 应用类型 | **HMI Runtime** |
| 主要功能 | 画面、报警、趋势、配方 |
| 运行平台 | 常规PC / 工业PC |
| 通讯对象 | PLC、驱动、数据库等 |
| 校核 | 授权点数、版本、系统要求 |

## ③ 计算/选型示例

- 已知：  
  一台工业PC需要显示设备画面、记录报警、显示趋势曲线。
- 公式：  
  **HMI负载 = 画面 + 变量 + 报警 + 趋势 + 数据记录**
- 计算：  
  功能越多，对授权、性能和存储要求越高。
- 结论：  
  需要选择合适的 **SIMATIC HMI 应用软件**，并按变量点数、版本兼容性和运行平台校核。

> [!tip] 记忆口诀
> 只看画面不够，还要看变量、报警、趋势和授权。

# 08 用户应用程序

> [!info] 核心结论
> 用户应用程序是项目自定义软件，适合数据处理、报表、算法、接口转换和第三方系统集成。

## ① 分类/公式/规则

> [!note] 🟦 定位
> 用户应用程序通常由用户或集成商开发，用于补充标准PLC/HMI无法直接完成的功能。

> [!warning] 🟧 用户程序判断公式
> **用户程序需求 = 标准功能不足 + 数据处理 + 外部接口 + 特殊算法**

## ② 参考表

| 项目 | 选型关注点 |
|---|---|
| 应用类型 | **自定义软件** |
| 常见功能 | 报表、算法、接口、数据库 |
| 通讯对象 | PLC、HMI、MES、数据库 |
| 风险点 | 稳定性、维护性、安全性 |
| 校核 | 通讯协议、运行权限、系统资源 |

## ③ 计算/选型示例

- 已知：  
  产线需要把PLC数据上传到数据库，并自动生成生产报表。
- 公式：  
  **用户程序价值 = 数据采集 + 数据处理 + 外部系统对接**
- 计算：  
  标准HMI功能可能无法完全满足报表和数据库逻辑。
- 结论：  
  可开发 **用户应用程序**，但需重点校核通讯协议、异常处理、数据安全和长期维护。

> [!tip] 记忆口诀
> 标准软件做不到，就用用户程序补一刀。

# 09 PC 系统快速选型表

> [!info] 核心结论
> PC系统选型不要只看电脑配置，更要看它承担的是调试、显示、控制，还是数据处理。

## ① 分类/公式/规则

> [!note] 🟦 四步选型法
> ① 看运行环境  
> ② 看运行软件  
> ③ 看通讯对象  
> ④ 看实时性和稳定性

> [!warning] 🟧 最终选型公式
> **推荐PC系统 = 环境等级 + 软件负载 + 通讯复杂度 + 实时性要求 + 维护周期**

## ② 参考表

| 需求 | 推荐方向 |
|---|---|
| 工程调试 / 办公 | **常规 PC** |
| 车间长期运行 | **工业 PC** |
| PC + PLC融合控制 | **S7 开放式控制器** |
| 紧凑集成控制 | **S7 嵌入式控制器** |
| 软件控制任务 | **控制器应用程序** |
| PC上运行HMI | **HMI 应用软件** |
| 特殊数据/算法 | **用户应用程序** |

## ③ 计算/选型示例

- 已知：  
  项目需要工业现场连续运行HMI，并把PLC数据上传数据库。
- 公式：  
  **系统需求 = 工业环境 + HMI运行 + PLC通讯 + 数据库接口**
- 计算：  
  现场长期运行不建议使用普通办公PC；还需要支持HMI和用户程序。
- 结论：  
  可采用 **工业 PC + SIMATIC HMI 应用软件 + 用户应用程序** 的组合，具体配置需按实际规范/厂家手册校核。

> [!tip] 选型口诀
> 调试用常规，现场用工业；控制看S7，画面看HMI，特殊功能用户写。
