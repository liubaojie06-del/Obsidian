# 01 设备和网络总览

> [!info] 核心结论
> **设备和网络** 是 TIA Portal 中进行 PLC、HMI、驱动、远程I/O 等硬件组态和通讯连接的核心入口。

## ① 分类/公式/规则

> [!note] 🟦 核心作用
> 设备和网络主要完成三件事：  
> **添加设备、配置硬件、建立通讯网络**。

> [!warning] 🟧 通用配置公式
> **完整组态 = 设备型号 + 模块配置 + 网络连接 + 地址参数 + 编译下载**

## ② 参考表

| 项目 | 作用 |
|---|---|
| **设备视图** | 配置CPU、模块、机架 |
| **网络视图** | 连接PLC、HMI、驱动、I/O |
| **拓扑视图** | 查看实际网络端口连接 |
| **地址配置** | 设置IP、设备名、站地址 |
| **通讯连接** | 建立PROFINET、PROFIBUS等通讯 |

## ③ 计算/选型示例

- 已知：  
  项目中有一台PLC、一台HMI、一个远程I/O站。
- 公式：  
  **网络组态 = PLC + HMI + 远程I/O + 通讯协议 + 地址规划**
- 计算：  
  PLC、HMI、远程I/O都需要加入项目，并设置通讯网络和IP地址。
- 结论：  
  应在 **设备和网络** 中完成设备添加、网络连接、地址分配和编译检查。

> [!tip] 记忆口诀
> 先加设备，再连网络；地址设对，通讯才稳。

# 02 设备视图

> [!info] 核心结论
> 设备视图主要用于配置硬件结构，包括CPU、信号模块、通讯模块、电源模块等。

## ① 分类/公式/规则

> [!note] 🟦 设备视图用途
> 设备视图像一张“电气硬件装配图”，用于确认PLC或站点里装了哪些模块。

> [!warning] 🟧 硬件组态公式
> **硬件组态 = CPU型号 + 模块型号 + 插槽位置 + 地址范围 + 参数设置**

## ② 参考表

| 配置对象 | 重点检查 |
|---|---|
| **CPU** | 型号、固件版本、IP地址 |
| **DI模块** | 点数、电压、输入地址 |
| **DO模块** | 点数、电流、输出地址 |
| **AI/AO模块** | 量程、分辨率、地址 |
| **通讯模块** | 协议、端口、站地址 |
| **电源模块** | 容量、供电方式 |

## ③ 计算/选型示例

- 已知：  
  项目需要 16DI、16DO、4AI。
- 公式：  
  **模块需求 = DI点数 + DO点数 + AI点数 + 预留余量**
- 计算：  
  16DI、16DO、4AI 可通过CPU本体加扩展模块实现，建议预留20%～30%余量。
- 结论：  
  在设备视图中添加对应模块，并确认地址范围不冲突；具体模块能力需按厂家手册校核。

> [!tip] 记忆口诀
> 设备视图看硬件，型号插槽别配错。

# 03 网络视图

> [!info] 核心结论
> 网络视图主要用于建立设备之间的通讯关系，比如PLC连接HMI、远程I/O、驱动器或上位机。

## ① 分类/公式/规则

> [!note] 🟦 网络视图用途
> 网络视图像一张“通讯关系图”，重点看设备之间有没有正确连接到同一网络。

> [!warning] 🟧 网络连接公式
> **通讯成功 = 同一网络 + 地址正确 + 设备名正确 + 协议匹配 + 编译无错误**

## ② 参考表

| 通讯对象 | 常见连接方式 |
|---|---|
| **PLC ↔ HMI** | PROFINET / 以太网 |
| **PLC ↔ 远程I/O** | PROFINET / PROFIBUS |
| **PLC ↔ 驱动器** | PROFINET / PROFIBUS |
| **PLC ↔ 上位机** | 以太网 / OPC / TCP |
| **PLC ↔ PLC** | S7通讯 / PROFINET |

## ③ 计算/选型示例

- 已知：  
  PLC IP为 192.168.0.1，HMI IP为 192.168.0.10。
- 公式：  
  **同网段判断 = IP地址前三段一致 + 子网掩码一致**
- 计算：  
  两者都在 192.168.0.x 网段，若子网掩码一致，则可处于同一网络。
- 结论：  
  PLC和HMI可在网络视图中建立连接，但仍需确认设备名、通讯驱动和下载对象正确。

> [!tip] 记忆口诀
> 网络视图看连接，同网同名才能通。

# 04 拓扑视图

> [!info] 核心结论
> 拓扑视图用于查看或规划设备端口之间的实际物理连接关系，适合PROFINET网络排查。

## ① 分类/公式/规则

> [!note] 🟦 拓扑视图用途
> 拓扑视图关注“网线从哪个端口接到哪个端口”，更接近现场真实接线。

> [!warning] 🟧 拓扑校核公式
> **拓扑正确 = 设备端口正确 + 交换机连接正确 + 现场接线一致 + 在线诊断一致**

## ② 参考表

| 项目 | 检查重点 |
|---|---|
| **端口连接** | P1、P2是否接对 |
| **交换机** | 端口分配、网络层级 |
| **环网** | 冗余协议、环网方向 |
| **现场网线** | 线缆、接头、屏蔽 |
| **在线诊断** | 实际拓扑与项目一致 |

## ③ 计算/选型示例

- 已知：  
  PLC能识别HMI，但某个远程I/O站通讯异常。
- 公式：  
  **故障定位 = 网络连接 + IP地址 + 设备名 + 端口拓扑**
- 计算：  
  若IP和设备名正确，但仍通讯异常，需要检查实际网线端口和拓扑是否一致。
- 结论：  
  可进入拓扑视图或在线诊断，检查设备端口连接是否与项目组态一致。

> [!tip] 记忆口诀
> 网络能不能通，看地址；现场接没接对，看拓扑。

# 05 地址规划

> [!info] 核心结论
> 地址规划是设备通讯稳定的基础，IP地址、设备名、I/O地址都不能混乱。

## ① 分类/公式/规则

> [!note] 🟦 地址类型
> TIA Portal 中常见地址包括：  
> **IP地址、PROFINET设备名、PROFIBUS站地址、I/O地址、变量地址**。

> [!warning] 🟧 地址规划公式
> **地址稳定 = 编号统一 + 不重复 + 易识别 + 与电气图一致**

## ② 参考表

| 地址类型 | 作用 |
|---|---|
| **IP地址** | 以太网通讯定位 |
| **设备名** | PROFINET设备识别 |
| **站地址** | PROFIBUS设备识别 |
| **I/O地址** | 程序读取输入输出 |
| **变量地址** | PLC与HMI数据交互 |

## ③ 计算/选型示例

- 已知：  
  项目有1台PLC、1台HMI、3个远程I/O站。
- 公式：  
  **IP规划 = 网段统一 + 设备编号 + 功能分类**
- 计算：  
  - PLC：192.168.0.1  
  - HMI：192.168.0.10  
  - I/O站1：192.168.0.21  
  - I/O站2：192.168.0.22  
  - I/O站3：192.168.0.23
- 结论：  
  地址要统一规划，避免重复；实际IP段需按现场网络规范校核。

> [!tip] 记忆口诀
> IP不重复，设备名要对；地址规划好，通讯少烦恼。

# 06 通讯排错

> [!info] 核心结论
> 通讯故障排查要按顺序进行：先看物理连接，再看地址，再看组态，最后看程序。

## ① 分类/公式/规则

> [!note] 🟦 排错顺序
> 通讯不通时，不要一上来就改程序，应先确认网络和设备状态。

> [!warning] 🟧 通讯排错公式
> **通讯故障 = 网线问题 + 地址错误 + 设备名错误 + 组态不一致 + 程序访问错误**

## ② 参考表

| 排查步骤 | 检查内容 |
|---|---|
| **1 物理连接** | 网线、电源、端口灯 |
| **2 地址参数** | IP、子网掩码、站地址 |
| **3 设备名** | PROFINET设备名是否一致 |
| **4 硬件组态** | 型号、版本、模块是否匹配 |
| **5 编译下载** | 是否编译通过并下载 |
| **6 在线诊断** | 查看报警和故障信息 |

## ③ 计算/选型示例

- 已知：  
  HMI无法连接PLC。
- 公式：  
  **HMI通讯检查 = PLC在线 + IP同网段 + 连接对象正确 + HMI下载正确**
- 计算：  
  先Ping PLC，再检查HMI连接设置，最后检查项目是否下载到正确HMI。
- 结论：  
  若PLC在线、IP正确、连接对象正确，仍无法通讯，则继续检查防火墙、驱动、访问权限和软件版本。

> [!tip] 记忆口诀
> 先灯后线，再IP；设备名对，通讯才对。

# 07 设备和网络快速检查表

> [!info] 核心结论
> 每次下载项目前，先检查设备、网络、地址、编译，能减少大部分现场通讯问题。

## ① 分类/公式/规则

> [!note] 🟦 下载前四查
> ① 查设备型号  
> ② 查模块插槽  
> ③ 查网络地址  
> ④ 查编译结果

> [!warning] 🟧 最终检查公式
> **可下载项目 = 硬件一致 + 网络正确 + 地址无冲突 + 编译无错误**

## ② 参考表

| 检查项 | 是否关键 |
|---|---|
| **CPU型号一致** | 必查 |
| **模块型号一致** | 必查 |
| **IP地址不冲突** | 必查 |
| **设备名正确** | 必查 |
| **通讯连接存在** | 必查 |
| **编译无错误** | 必查 |
| **在线诊断正常** | 调试必查 |

## ③ 计算/选型示例

- 已知：  
  新增一台远程I/O站后，项目需要重新下载。
- 公式：  
  **新增设备流程 = 添加设备 + 连接网络 + 分配地址 + 编译 + 下载 + 在线诊断**
- 计算：  
  新设备必须完成硬件配置、网络连接和地址分配，否则PLC无法正常识别。
- 结论：  
  新增设备后，一定要重新编译并下载硬件组态；具体设备型号和参数需按厂家手册校核。

> [!tip] 选型口诀
> 设备先组态，网络再连接；地址不冲突，编译再下载。
