# 01 未分组的设备总览

> [!info] 核心结论
> **未分组的设备** 是项目中已经添加、但还没有归入设备组或网络结构中的设备集合。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> 未分组的设备不是错误，而是 TIA Portal 中临时放置设备的区域，方便后续整理、组态和联网。

> [!warning] 🟧 判断公式
> **未分组设备 = 已添加设备 + 未归类 + 未整理到设备组**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **位置** | 项目树中的设备区域 |
| **状态** | 已添加但未分组 |
| **对象** | PLC、HMI、驱动、远程I/O等 |
| **作用** | 临时存放、等待组态 |
| **后续操作** | 分组、联网、配置参数、下载 |

## ③ 计算/选型示例

- 已知：  
  项目中新添加了一台 PLC 和一台 HMI，但还没有整理结构。
- 公式：  
  **项目整理 = 添加设备 + 分组命名 + 网络连接 + 参数配置**
- 计算：  
  新增设备先出现在未分组区域，后续需要按功能或区域整理。
- 结论：  
  应将设备归入合适的设备组，并在设备和网络中完成通讯配置。

> [!tip] 记忆口诀
> 未分组不是错，只是设备还没“归队”。

# 02 未分组设备的作用

> [!info] 核心结论
> 未分组设备的主要作用，是作为新设备的临时存放区，方便后续统一整理项目结构。

## ① 分类/公式/规则

> [!note] 🟦 作用分类
> 未分组设备通常用于：  
> ① 新添加设备  
> ② 临时测试设备  
> ③ 尚未归类的PLC/HMI/驱动  
> ④ 从其他项目复制来的设备

> [!warning] 🟧 项目管理公式
> **清晰项目 = 设备命名清楚 + 分组清楚 + 网络关系清楚**

## ② 参考表

| 使用场景 | 说明 |
|---|---|
| **新建项目** | 设备先临时出现 |
| **添加新设备** | 还未归入设备组 |
| **复制设备** | 从其他项目导入 |
| **测试设备** | 临时调试使用 |
| **项目整理前** | 结构还未规划完成 |

## ③ 计算/选型示例

- 已知：  
  一个项目中有 PLC、HMI、3个远程I/O站。
- 公式：  
  **设备结构 = 主控制器 + 操作界面 + 现场I/O + 驱动设备**
- 计算：  
  如果全部放在未分组设备中，后期查找和维护会变困难。
- 结论：  
  建议按工位、产线、控制柜或功能区进行分组管理。

> [!tip] 记忆口诀
> 设备越多越要分组，不然后期找设备会迷路。

# 03 什么时候会出现未分组设备

> [!info] 核心结论
> 当设备已经加入项目，但还没有被整理到具体设备组时，就会出现在未分组设备中。

## ① 分类/公式/规则

> [!note] 🟦 常见触发情况
> 新增PLC、新增HMI、新增驱动、导入硬件、复制设备时，都可能出现未分组设备。

> [!warning] 🟧 出现原因公式
> **出现未分组 = 新增设备 + 未移动 + 未建立项目结构**

## ② 参考表

| 情况 | 是否常见 |
|---|---|
| **添加新PLC** | 常见 |
| **添加新HMI** | 常见 |
| **添加驱动器** | 常见 |
| **添加远程I/O** | 常见 |
| **导入旧项目设备** | 常见 |
| **复制粘贴设备** | 常见 |

## ③ 计算/选型示例

- 已知：  
  在项目中新增一台 S7-1200 PLC。
- 公式：  
  **新增设备流程 = 添加设备 + 设置名称 + 配置网络 + 下载组态**
- 计算：  
  设备刚添加时，可能暂时显示在未分组设备下。
- 结论：  
  需要进一步设置设备名称、IP地址、硬件模块和网络连接。

> [!tip] 记忆口诀
> 新设备刚进项目，先住“未分组宿舍”。

# 04 如何整理未分组设备

> [!info] 核心结论
> 整理未分组设备的关键，是按项目逻辑进行命名、分组和网络规划。

## ① 分类/公式/规则

> [!note] 🟦 整理步骤
> ① 修改设备名称  
> ② 按功能或区域分组  
> ③ 检查硬件组态  
> ④ 建立网络连接  
> ⑤ 编译并保存项目

> [!warning] 🟧 整理公式
> **设备整理 = 命名规范 + 分组清晰 + 网络明确 + 地址唯一**

## ② 参考表

| 整理方式 | 适合场景 |
|---|---|
| **按工位分组** | 多工位设备 |
| **按产线分组** | 整线项目 |
| **按控制柜分组** | 多控制柜系统 |
| **按设备类型分组** | PLC、HMI、驱动分类 |
| **按功能分组** | 输送、包装、检测等 |

## ③ 计算/选型示例

- 已知：  
  项目中有 Main_PLC、HMI_01、Drive_01、IO_Station_01。
- 公式：  
  **分组逻辑 = 控制区域 + 设备类型 + 设备编号**
- 计算：  
  可建立“主控柜”“操作站”“驱动系统”“远程I/O”等分组。
- 结论：  
  设备分组后，项目树更清晰，后期调试和维护更方便。

> [!tip] 记忆口诀
> 名字先规范，分组再清楚，项目维护不迷路。

# 05 未分组设备与网络配置

> [!info] 核心结论
> 未分组设备仍然可以进行硬件和网络配置，但正式项目中建议先整理再联网。

## ① 分类/公式/规则

> [!note] 🟦 网络配置重点
> 设备是否分组，不等于设备是否能通讯；通讯是否正常，关键看网络连接、IP地址、设备名和协议设置。

> [!warning] 🟧 通讯成功公式
> **通讯成功 = 网络连接正确 + IP不冲突 + 设备名正确 + 编译下载正确**

## ② 参考表

| 配置项 | 重点检查 |
|---|---|
| **IP地址** | 不重复、同网段 |
| **设备名** | PROFINET名称正确 |
| **网络接口** | PN/IE接口连接正确 |
| **通讯协议** | 与PLC/HMI/驱动匹配 |
| **编译结果** | 无硬件组态错误 |

## ③ 计算/选型示例

- 已知：  
  一台 HMI 在未分组设备中，需要连接 PLC。
- 公式：  
  **HMI通讯 = HMI连接对象 + PLC地址 + 网络接口 + 下载到HMI**
- 计算：  
  即使设备未分组，只要网络连接和通讯参数正确，仍可建立通讯。
- 结论：  
  建议整理分组后再配置网络，减少项目混乱和误连风险。

> [!tip] 记忆口诀
> 分组管管理，通讯看地址；设备能不能通，关键看网络。

# 06 常见问题排查

> [!info] 核心结论
> 未分组设备本身不是故障，真正需要排查的是设备是否配置完整、地址是否正确、网络是否连通。

## ① 分类/公式/规则

> [!note] 🟦 排查顺序
> 先看设备是否添加正确，再看硬件组态，最后看网络和下载状态。

> [!warning] 🟧 排错公式
> **项目异常 = 设备型号错误 + 模块不匹配 + 地址冲突 + 网络未连接 + 未下载组态**

## ② 参考表

| 问题 | 可能原因 |
|---|---|
| **设备找不到** | 未添加或名称不清晰 |
| **通讯失败** | IP、设备名、网络连接错误 |
| **编译报错** | 硬件型号或模块不匹配 |
| **下载失败** | 接口选择错误或设备不在线 |
| **项目混乱** | 设备长期未分组整理 |

## ③ 计算/选型示例

- 已知：  
  PLC和HMI都在未分组设备下，但HMI无法连接PLC。
- 公式：  
  **通讯排查 = 在线状态 + IP同网段 + HMI连接对象 + 项目下载**
- 计算：  
  先确认PLC在线，再检查HMI连接对象是否选择正确PLC。
- 结论：  
  未分组不是主要原因，应重点检查网络配置、设备名和下载对象。

> [!tip] 记忆口诀
> 看到未分组别慌，先查组态再查网。

# 07 快速整理检查表

> [!info] 核心结论
> 项目交付前，应尽量清理未分组设备，让设备结构清晰、命名规范、网络明确。

## ① 分类/公式/规则

> [!note] 🟦 交付前五查
> ① 设备名称是否清楚  
> ② 是否按区域分组  
> ③ IP地址是否唯一  
> ④ 网络连接是否正确  
> ⑤ 是否编译无错误

> [!warning] 🟧 最终检查公式
> **清晰项目 = 无多余设备 + 分组明确 + 地址规范 + 编译通过**

## ② 参考表

| 检查项 | 建议 |
|---|---|
| **未使用设备** | 删除或备注 |
| **临时测试设备** | 删除或单独分组 |
| **正式设备** | 按区域归类 |
| **设备名称** | 使用统一命名规则 |
| **网络地址** | 按项目规范规划 |
| **硬件组态** | 与现场实物一致 |

## ③ 计算/选型示例

- 已知：  
  项目调试完成后，未分组设备中仍有多个临时设备。
- 公式：  
  **项目清理 = 删除无用设备 + 整理正式设备 + 编译检查**
- 计算：  
  临时设备可能导致误下载、误连或后期维护混乱。
- 结论：  
  交付前建议清理未分组设备，只保留正式使用的设备，并完成最终编译校核。

> [!tip] 选型口诀
> 未分组先整理，临时设备要清理；命名规范化，交付更专业。
