# 01 在线访问总览

> [!info] 核心结论
> **在线访问** 是 TIA Portal 连接真实PLC、仿真PLC、HMI、驱动和现场设备的入口，关键是选对通讯接口。

## ① 分类/公式/规则

> [!note] 🟦 功能主线
> 在线访问主要用于：  
> **搜索设备、下载程序、在线监控、分配设备名、修改IP、诊断通讯状态**。

> [!warning] 🟧 在线连接公式
> **在线成功 = 正确接口 + 正确网段 + 设备在线 + 防火墙允许 + 通讯协议匹配**

## ② 参考表

| 接口类型 | 常见用途 |
|---|---|
| **Intel Ethernet** | 有线网口连接PLC |
| **Wi-Fi** | 无线网络访问设备 |
| **VMware Virtual Adapter** | 虚拟机网络通讯 |
| **Siemens PLCSIM Virtual Ethernet Adapter** | PLCSIM仿真通讯 |
| **PC internal [本地]** | 本机内部通讯 |
| **PLCSIM [PN/IE]** | PLC仿真接口 |
| **USB [S7USB]** | USB方式连接PLC或适配器 |
| **TAP-Windows Adapter** | VPN/虚拟网卡相关 |

## ③ 计算/选型示例

- 已知：  
  电脑通过网线连接真实PLC，需要下载程序。
- 公式：  
  **真实PLC连接 = 有线网卡 + PLC同网段 + PN/IE接口 + 在线搜索**
- 计算：  
  如果PLC IP为 192.168.0.1，电脑网卡应设置在同一网段，例如 192.168.0.100。
- 结论：  
  应优先选择真实有线网卡，例如 **Intel(R) Ethernet Connection**，不要误选VMware或PLCSIM虚拟网卡。

> [!tip] 记忆口诀
> 真PLC选真网卡，仿真PLC选PLCSIM。

# 02 显示/隐藏接口

> [!info] 核心结论
> **显示/隐藏接口** 用于管理在线访问列表中显示哪些网卡，避免接口太多导致误选。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> 电脑安装虚拟机、VPN、PLCSIM后，TIA Portal 会显示很多接口；不常用的接口可以隐藏，常用接口保留。

> [!warning] 🟧 接口管理公式
> **清晰接口列表 = 保留常用接口 + 隐藏无关接口 + 命名识别清楚**

## ② 参考表

| 操作 | 作用 |
|---|---|
| **显示接口** | 让接口出现在在线访问列表 |
| **隐藏接口** | 减少干扰，防止误选 |
| **展开接口** | 查看该接口下的可访问设备 |
| **刷新搜索** | 重新查找在线设备 |
| **检查状态** | 判断接口是否可用 |

## ③ 计算/选型示例

- 已知：  
  在线访问中有10多个网卡，但现场只用有线网口连接PLC。
- 公式：  
  **接口选择效率 = 常用接口清晰度 - 无关接口干扰**
- 计算：  
  隐藏VMware、TAP、未使用Wi-Fi等接口，可减少误选。
- 结论：  
  建议只显示常用的 **有线网卡、PLCSIM接口、USB接口**，其他接口按需要显示。

> [!tip] 记忆口诀
> 接口太多先隐藏，现场调试不慌张。

# 03 Intel 有线网卡

> [!info] 核心结论
> **Intel Ethernet** 通常是真实有线网卡，是连接现场PLC、HMI、驱动、远程I/O最常用的接口。

## ① 分类/公式/规则

> [!note] 🟦 使用场景
> 当电脑通过网线直接连接PLC，或接入工业交换机时，通常选择有线网卡。

> [!warning] 🟧 有线连接公式
> **有线通讯 = 网线连接 + 网卡启用 + IP同网段 + PN/IE接口正确**

## ② 参考表

| 项目 | 检查重点 |
|---|---|
| **网线** | 是否插好，网口灯是否亮 |
| **网卡状态** | 是否启用 |
| **IP地址** | 是否与PLC同网段 |
| **子网掩码** | 是否匹配 |
| **防火墙** | 是否阻挡TIA通讯 |
| **接口类型** | 选择PN/IE相关接口 |

## ③ 计算/选型示例

- 已知：  
  PLC IP = 192.168.1.10，子网掩码 = 255.255.255.0。
- 公式：  
  **同网段 = IP前三段一致 + 子网掩码一致**
- 计算：  
  电脑可设置为 192.168.1.100，子网掩码 255.255.255.0。
- 结论：  
  选择 **Intel Ethernet** 后在线搜索，通常可找到PLC；若找不到，检查设备名、防火墙和网络连接。

> [!tip] 记忆口诀
> 网线插真口，IP同网段，PLC才好找。

# 04 VMware 虚拟网卡

> [!info] 核心结论
> **VMware Virtual Ethernet Adapter** 是虚拟机网卡，主要用于虚拟机和主机之间通讯，不是现场PLC的默认选择。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> VMware VMnet1、VMnet8 等接口通常来自虚拟机软件，用于Host-only、NAT等虚拟网络。

> [!warning] 🟧 虚拟机通讯公式
> **虚拟机通讯 = 虚拟网卡模式 + 主机IP + 虚拟机IP + 网络桥接设置**

## ② 参考表

| 网卡 | 常见含义 |
|---|---|
| **VMnet1** | Host-only网络 |
| **VMnet8** | NAT网络 |
| **桥接模式** | 虚拟机接入真实网络 |
| **NAT模式** | 虚拟机通过主机上网 |
| **Host-only** | 仅主机与虚拟机通讯 |

## ③ 计算/选型示例

- 已知：  
  TIA Portal 安装在虚拟机中，需要连接真实PLC。
- 公式：  
  **虚拟机连PLC = 桥接真实网卡 + 虚拟机IP同网段 + TIA选正确接口**
- 计算：  
  如果虚拟机使用NAT，可能无法直接搜索现场PLC；桥接到真实有线网卡更常用。
- 结论：  
  真实PLC调试时，不要随意选择VMnet接口，需按虚拟机网络模式校核。

> [!tip] 记忆口诀
> VMnet是虚拟网，连真PLC先看桥接。

# 05 Siemens PLCSIM 虚拟以太网适配器

> [!info] 核心结论
> **Siemens PLCSIM Virtual Ethernet Adapter** 用于TIA与PLCSIM仿真PLC通讯，适合离线调试和程序仿真。

## ① 分类/公式/规则

> [!note] 🟦 使用场景
> 当没有真实PLC，只需要在电脑中仿真程序运行时，可使用PLCSIM相关接口。

> [!warning] 🟧 仿真通讯公式
> **PLCSIM通讯 = 启动仿真PLC + 选择PLCSIM接口 + 下载到仿真设备 + 在线监控**

## ② 参考表

| 项目 | 检查重点 |
|---|---|
| **PLCSIM是否启动** | 必查 |
| **仿真CPU型号** | 与项目CPU匹配 |
| **接口选择** | PLCSIM相关接口 |
| **下载目标** | 下载到仿真PLC |
| **在线监控** | 观察程序运行状态 |

## ③ 计算/选型示例

- 已知：  
  没有真实PLC，但想测试S7-1200程序逻辑。
- 公式：  
  **仿真流程 = 编译项目 + 启动PLCSIM + 下载程序 + 在线监控**
- 计算：  
  如果选择真实网卡，可能找不到仿真PLC；应选择PLCSIM虚拟接口。
- 结论：  
  离线仿真优先选择 **Siemens PLCSIM Virtual Ethernet Adapter** 或 **PLCSIM [PN/IE]**。

> [!tip] 记忆口诀
> 没有真PLC，PLCSIM来模拟。

# 06 TAP-Windows Adapter

> [!info] 核心结论
> **TAP-Windows Adapter** 多与VPN或虚拟网络相关，现场PLC调试一般不优先选择。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> TAP适配器通常由VPN、虚拟网络或远程连接软件创建，用于虚拟网络通道。

> [!warning] 🟧 TAP使用判断公式
> **是否选TAP = 是否通过VPN访问 + 是否有远程网络路由 + 是否能访问目标设备**

## ② 参考表

| 场景 | 建议 |
|---|---|
| **本地网线连接PLC** | 不优先选TAP |
| **VPN远程调试** | 可能需要选TAP |
| **本机仿真** | 优先选PLCSIM接口 |
| **虚拟机网络** | 优先检查VMware网卡 |
| **接口不确定** | 先Ping目标设备 |

## ③ 计算/选型示例

- 已知：  
  工程师通过VPN远程访问客户现场PLC。
- 公式：  
  **远程访问 = VPN连接 + 路由正确 + 目标IP可达 + TIA接口正确**
- 计算：  
  如果PLC只能通过VPN网段访问，TAP适配器可能参与通讯。
- 结论：  
  是否选择 **TAP-Windows Adapter** 需根据VPN软件和网络路由确认，不能盲选。

> [!tip] 记忆口诀
> TAP多半是VPN，本地调试少碰它。

# 07 Wi-Fi 无线网卡

> [!info] 核心结论
> **Wi-Fi网卡** 可用于无线访问设备，但工业现场调试通常优先使用有线网口，更稳定可靠。

## ① 分类/公式/规则

> [!note] 🟦 使用场景
> Wi-Fi适合临时访问、无线调试或通过无线网络连接到PLC所在网段的场合。

> [!warning] 🟧 无线通讯公式
> **Wi-Fi通讯 = 接入正确无线网络 + IP同网段 + 信号稳定 + 防火墙允许**

## ② 参考表

| 项目 | 检查重点 |
|---|---|
| **无线网络** | 是否接入PLC所在网络 |
| **IP地址** | 是否同网段 |
| **信号强度** | 是否稳定 |
| **延迟** | 是否影响下载和监控 |
| **安全策略** | 是否允许工业设备通讯 |

## ③ 计算/选型示例

- 已知：  
  电脑连接了工厂Wi-Fi，但搜索不到PLC。
- 公式：  
  **Wi-Fi访问PLC = 同一网络 + 路由允许 + 无隔离策略**
- 计算：  
  很多企业Wi-Fi会隔离终端，导致电脑无法直接访问PLC。
- 结论：  
  若无线搜索不到PLC，建议改用有线网口，或联系网络管理员确认路由和隔离策略。

> [!tip] 记忆口诀
> Wi-Fi方便但不稳，现场下载用有线更放心。

# 08 PC internal [本地]

> [!info] 核心结论
> **PC internal [本地]** 通常用于本机内部通讯或软件组件之间访问，不是连接现场PLC的常规外部网口。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> PC internal 更偏向本机内部对象或本地运行环境，与真实网线接口不同。

> [!warning] 🟧 本地接口判断公式
> **本地通讯 = 本机运行组件 + 本地接口 + 软件内部连接**

## ② 参考表

| 场景 | 是否常用 |
|---|---|
| **真实PLC下载** | 通常不用 |
| **本机软件通讯** | 可能使用 |
| **HMI Runtime本地测试** | 可能使用 |
| **PLCSIM仿真** | 视项目设置而定 |
| **现场网络诊断** | 优先选真实网卡 |

## ③ 计算/选型示例

- 已知：  
  需要下载程序到现场真实PLC。
- 公式：  
  **真实设备下载 = 外部网卡 + PLC在线 + IP同网段**
- 计算：  
  PC internal 不代表外部网口，选择它可能找不到现场PLC。
- 结论：  
  真实PLC下载时，应优先选择有线网卡或对应USB接口，不优先选择PC internal。

> [!tip] 记忆口诀
> PC internal看本机，现场PLC看网卡。

# 09 PLCSIM [PN/IE]

> [!info] 核心结论
> **PLCSIM [PN/IE]** 是用于PROFINET/工业以太网仿真通讯的接口，适合程序离线测试。

## ① 分类/公式/规则

> [!note] 🟦 核心理解
> PN/IE代表PROFINET/Industrial Ethernet，PLCSIM [PN/IE]通常用于仿真PLC的以太网通讯。

> [!warning] 🟧 PLCSIM接口公式
> **仿真在线 = PLCSIM运行 + PN/IE接口选择正确 + 下载目标正确**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **接口类型** | PN/IE仿真接口 |
| **适用对象** | PLCSIM仿真PLC |
| **常用操作** | 下载、监控、测试逻辑 |
| **优势** | 无需真实PLC |
| **限制** | 硬件行为需按仿真支持范围校核 |

## ③ 计算/选型示例

- 已知：  
  想在电脑上测试PLC程序的启停逻辑。
- 公式：  
  **离线测试 = PLCSIM + 下载程序 + 在线监控 + 强制/修改变量**
- 计算：  
  启动PLCSIM后，选择PLCSIM [PN/IE]进行下载和在线监控。
- 结论：  
  程序逻辑测试可使用 **PLCSIM [PN/IE]**，但真实硬件、通讯和现场响应仍需实机校核。

> [!tip] 记忆口诀
> PN/IE连以太，PLCSIM管仿真。

# 10 USB [S7USB]

> [!info] 核心结论
> **USB [S7USB]** 用于通过USB方式连接PLC或通讯适配器，适合特定设备或维护场景。

## ① 分类/公式/规则

> [!note] 🟦 使用场景
> 当PLC或通讯适配器支持USB连接时，可以选择USB [S7USB]进行访问。

> [!warning] 🟧 USB连接公式
> **USB通讯 = 驱动正常 + USB线连接 + 设备支持 + 接口选择正确**

## ② 参考表

| 项目 | 检查重点 |
|---|---|
| **USB线** | 是否可靠连接 |
| **驱动程序** | 是否正确安装 |
| **设备支持** | 是否支持S7USB |
| **接口选择** | TIA中选USB接口 |
| **稳定性** | 避免下载中断开 |

## ③ 计算/选型示例

- 已知：  
  现场无法通过以太网访问PLC，但PLC支持USB通讯。
- 公式：  
  **USB访问 = USB连接 + 驱动识别 + TIA选择S7USB**
- 计算：  
  电脑设备管理器能识别USB通讯设备后，再在TIA中选择USB接口。
- 结论：  
  可尝试使用 **USB [S7USB]** 在线访问，具体支持范围需按设备型号和厂家手册校核。

> [!tip] 记忆口诀
> 网口不通别慌，USB也能帮忙。

# 11 在线接口快速选择表

> [!info] 核心结论
> 接口选择的核心原则是：真实设备选真实网卡，仿真设备选PLCSIM，特殊连接选USB或VPN接口。

## ① 分类/公式/规则

> [!note] 🟦 四步选择法
> ① 判断是真实PLC还是仿真PLC  
> ② 判断是网线、Wi-Fi、USB还是VPN  
> ③ 设置电脑IP或连接方式  
> ④ 在线搜索并下载/诊断

> [!warning] 🟧 最终选择公式
> **正确接口 = 设备类型 + 连接方式 + 网络路径 + TIA访问接口**

## ② 参考表

| 需求 | 推荐接口 |
|---|---|
| 真实PLC网线连接 | **Intel Ethernet / 有线网卡** |
| 真实PLC无线访问 | **Wi-Fi网卡** |
| PLCSIM仿真 | **PLCSIM [PN/IE]** |
| PLCSIM虚拟网卡 | **Siemens PLCSIM Virtual Ethernet Adapter** |
| 虚拟机中调试 | **桥接后的真实网卡** |
| VPN远程访问 | **TAP或VPN对应接口** |
| USB连接PLC | **USB [S7USB]** |
| 本机内部通讯 | **PC internal [本地]** |

## ③ 计算/选型示例

- 已知：  
  TIA在线访问列表中有 Intel网卡、VMware网卡、PLCSIM网卡、Wi-Fi和USB。
- 公式：  
  **接口选择 = 目标设备 + 实际连接方式**
- 计算：  
  - 连接真实PLC：选Intel有线网卡  
  - 做仿真：选PLCSIM接口  
  - 用USB维护：选USB [S7USB]  
  - 通过VPN：检查TAP或VPN接口
- 结论：  
  不要看到接口就随便选，先确认目标设备和连接路径。

> [!tip] 选型口诀
> 真机选真网，仿真选PLCSIM；VPN看TAP，USB看S7USB。
