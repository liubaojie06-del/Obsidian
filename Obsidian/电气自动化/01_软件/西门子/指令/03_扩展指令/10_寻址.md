## 05 寻址 | 选型速记

> [!info] 核心结论
> 寻址指令的核心是：**位置查 ID 用 GEO2LOG，ID 查位置用 LOG2GEO，地址查模块用 IO2MOD，模块查地址用 RD_ADDR。**

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">总口诀</strong><br>
位置找 ID 用 GEO2LOG，ID 找位置用 LOG2GEO；地址找模块用 IO2MOD，模块找地址用 RD_ADDR。
</div>

### ① 分类/公式/规则

| 类型 | 常用指令 |
|---|---|
| 几何位置转硬件 ID | **GEO2LOG** |
| 硬件 ID 转几何位置 | **LOG2GEO** |
| I/O 地址转模块 ID | **IO2MOD** |
| 模块 ID 读取地址 | **RD_ADDR** |

### ② 参考表

| 需求 | 推荐指令 |
|---|---|
| 已知插槽，查硬件标识符 | **GEO2LOG** |
| 已知硬件标识符，查插槽 | **LOG2GEO** |
| 已知 IO 地址，查所属模块 | **IO2MOD** |
| 已知模块硬件 ID，查地址范围 | **RD_ADDR** |
| 做诊断定位 | 常组合使用 **IO2MOD + RD_ADDR** |
| 做模块识别 | 常组合使用 **GEO2LOG + LOG2GEO** |

### ③ 计算/选型示例

- 已知：HMI 报警显示某输入地址异常，需要定位对应硬件模块
- 公式/规则：地址反查模块用 **IO2MOD**
- 操作：输入异常 IO 地址，得到模块硬件标识符；再配合 **LOG2GEO** 查插槽位置
- 结论：可快速定位故障点在硬件组态中的实际位置

> [!tip] 记忆口诀
> **位置、地址、模块 ID，三者互查靠寻址。**
