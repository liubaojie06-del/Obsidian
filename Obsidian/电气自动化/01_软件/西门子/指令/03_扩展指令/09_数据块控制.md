- 结论：生成新的数据块，用于保存批次数据

> [!tip] 记忆口诀
> **要新建存储区，CREATE_DB 来开地。**

## 06 数据块控制 | 选型速记

> [!info] 核心结论
> 数据块控制的核心是：**创建用 CREATE_DB，读装载区用 READ_DBL，写装载区用 WRIT_DBL，查属性用 ATTR_DB，删除用 DELETE_DB。**

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">总口诀</strong><br>
建库 CREATE，读库 READ，写库 WRIT，查库 ATTR，删库 DELETE。
</div>

### ① 分类/公式/规则

| 类型 | 常用指令 |
|---|---|
| 创建数据块 | **CREATE_DB** |
| 读取装载存储器 DB | **READ_DBL** |
| 写入装载存储器 DB | **WRIT_DBL** |
| 查询数据块属性 | **ATTR_DB** |
| 删除数据块 | **DELETE_DB** |

### ② 参考表

| 需求 | 推荐指令 |
|---|---|
| 运行中新建 DB | **CREATE_DB** |
| 从装载存储器读取 DB 数据 | **READ_DBL** |
| 将数据保存到装载存储器 DB | **WRIT_DBL** |
| 判断 DB 是否存在或读取属性 | **ATTR_DB** |
| 删除不再使用的数据块 | **DELETE_DB** |

### ③ 计算/选型示例

- 已知：批次开始时创建 DB，批次结束后保存数据并清理临时 DB
- 公式/规则：创建、写入、删除分别使用对应 DB 控制指令
- 操作：用 **CREATE_DB** 创建数据块；用 **WRIT_DBL** 保存数据；确认无用后用 **DELETE_DB** 删除
- 结论：这是典型的动态数据块管理流程

> [!tip] 记忆口诀
> **DB 生命周期：先 CREATE，再 READ/WRIT，查 ATTR，最后 DELETE。**
