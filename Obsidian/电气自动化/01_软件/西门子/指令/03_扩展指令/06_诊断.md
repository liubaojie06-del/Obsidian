## 11 诊断 | 选型速记

> [!info] 核心结论
> 诊断指令的核心是：**OB 原因看 RD_SINFO，灯态看 LED，设备身份看 Get_IM_Data，站点模块看 DeviceStates/ModuleStates，详细诊断看 GET_DIAG。**

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">总口诀</strong><br>
查 OB 用 RD_SINFO，查灯用 LED；查身份用 IM，查设备模块看 States，查诊断用 GET_DIAG。
</div>

### ① 分类/公式/规则

| 类型 | 常用指令 |
|---|---|
| OB 启动信息 | **RD_SINFO** |
| LED 状态 | **LED** |
| 标识维护数据 | **Get_IM_Data** |
| IO 设备信息 | **GetStationInfo / Get_Name** |
| 校验与存储卡 | **GetChecksum / GetSMCinfo** |
| 设备/模块状态 | **DeviceStates / ModuleStates** |
| 诊断信息 | **GET_DIAG** |

### ② 参考表

| 需求 | 推荐指令 |
|---|---|
| 判断 OB 为什么启动 | **RD_SINFO** |
| 读取 CPU/模块 LED 状态 | **LED** |
| 读取设备 I&M 信息 | **Get_IM_Data** |
| 读取 IO 设备信息 | **GetStationInfo** |
| 读取校验和 | **GetChecksum** |
| 读取存储卡信息 | **GetSMCinfo** |
| 读取 IO 设备状态 | **DeviceStates** |
| 读取模块状态 | **ModuleStates** |
| 读取详细诊断信息 | **GET_DIAG** |

### ③ 计算/选型示例

- 已知：远程 IO 站报警，HMI 需要显示设备状态和详细诊断
- 公式/规则：先查设备/模块状态，再读详细诊断
- 操作：使用 **DeviceStates / ModuleStates** 定位范围，再用 **GET_DIAG** 读取详细信息
- 结论：这是典型的 IO 设备诊断流程

> [!tip] 记忆口诀
> **先看状态定范围，再读诊断找原因。**
