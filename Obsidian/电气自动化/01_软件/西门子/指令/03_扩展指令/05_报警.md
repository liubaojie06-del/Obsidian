## 01 `Gen_UsrMsg` | 生成用户诊断报警

> [!info] 核心结论
> **Gen_UsrMsg** 用于生成用户自定义诊断报警，适合把程序中的故障、状态或提示信息上报给诊断系统/HMI。

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">版本</strong><br>
报警分类版本：<strong>V1.9</strong><br>
指令版本：<strong>V1.1</strong>
</div>

### ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| 所属分类 | **报警** |
| 指令名称 | **Gen_UsrMsg** |
| 功能描述 | **生成用户诊断报警** |
| 版本 | **V1.1** |
| 核心作用 | 生成自定义诊断报警 |
| 使用重点 | 报警文本、报警编号、触发条件需按项目规范设置 |

### ② 参考表

| 使用场景 | 说明 |
|---|---|
| 设备故障提示 | 如电机过载、传感器异常 |
| 工艺状态提示 | 如压力低、温度高、料位不足 |
| HMI 报警显示 | 把 PLC 内部诊断信息显示给操作员 |
| 维护诊断 | 方便维修人员快速定位问题 |
| 参数细节 | 需按实际软件/厂家手册校核 |

<div style="background:#F7F9FC;border-left:6px solid #FF6B35;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#FF6B35;">重点规则</strong><br>
报警不是只“点亮一个灯”，而是要让操作员知道：<strong>哪里错、为什么错、怎么处理</strong>。
</div>

### ③ 计算/选型示例

- 已知：电机运行时，热继电器反馈故障信号为 1
- 公式/规则：故障条件成立时生成用户诊断报警
- 操作：使用 **Gen_UsrMsg**，配置报警编号、报警文本和触发条件
- 结论：HMI 或诊断系统可显示对应用户报警，方便现场排查

> [!tip] 记忆口诀
> **程序报警别只亮灯，Gen_UsrMsg 把故障说清楚。**


## 02 报警 | 选型速记

> [!info] 核心结论
> 报警指令的核心是：**条件触发报警，文本说明原因，编号方便追踪，诊断帮助维修。**

<div style="background:#F7F9FC;border-left:6px solid #27AE60;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#27AE60;">总口诀</strong><br>
有故障，给编号；写清楚，能诊断；报警不是吓人，是帮人排故。
</div>

### ① 分类/公式/规则

| 类型 | 常用指令 |
|---|---|
| 用户诊断报警 | **Gen_UsrMsg** |
| 报警触发条件 | 故障位 / 状态位 / 比较结果 |
| 报警显示对象 | HMI / 诊断缓冲区 / 上位系统 |
| 报警维护重点 | 编号、文本、等级、复位方式 |

### ② 参考表

| 需求 | 推荐做法 |
|---|---|
| 生成用户报警 | 使用 **Gen_UsrMsg** |
| 显示故障原因 | 配置清晰报警文本 |
| 区分不同故障 | 使用不同报警编号 |
| 方便维修定位 | 文本包含设备名、故障点、处理方向 |
| 避免误报警 | 触发条件需加滤波/延时，按现场需求校核 |

### ③ 计算/选型示例

- 已知：压力低于下限 0.2 MPa 时需要报警
- 公式/规则：`压力 <= 下限` 时触发报警
- 操作：先用比较指令判断压力是否低，再用 **Gen_UsrMsg** 生成报警
- 结论：操作员可看到“压力过低”类诊断报警

> [!tip] 记忆口诀
> **报警三件套：条件准、文本清、编号明。**
