## 01 `ACK_FCT_WARN` | 确认超出 F 循环时间

> [!info] 核心结论
> **ACK_FCT_WARN** 用于确认与 **F 循环时间超限**相关的警告，属于安全扩展类指令，使用前必须按实际安全规范和厂家手册校核。

<div style="background:#F7F9FC;border-left:6px solid #FF6B35;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#FF6B35;">安全提醒</strong><br>
安全相关指令不能只看“能否复位”，还要确认：<strong>为什么超时、是否排除风险、是否允许重新确认</strong>。
</div>

### ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| 所属分类 | **安全扩展** |
| 指令名称 | **ACK_FCT_WARN** |
| 功能描述 | **确认超出 F 循环时间...** |
| 版本 | **V1.0** |
| 核心作用 | 确认 F 循环时间相关警告 |
| 使用重点 | 需按实际安全程序、F-CPU、厂家手册和安全规范校核 |

### ② 参考表

| 关键点 | 说明 |
|---|---|
| F 循环时间 | 安全程序允许的循环执行时间范围 |
| 超时警告 | 可能表示安全程序执行周期异常 |
| 确认动作 | 用于确认相关警告状态 |
| 使用条件 | 必须确认故障原因已排除 |
| 风险提示 | 不可用作“强行复位”手段 |

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">重点规则</strong><br>
安全警告确认前，先查原因；原因未排除，不建议直接确认。
</div>

### ③ 计算/选型示例

- 已知：F 程序运行中出现 F 循环时间超限警告
- 公式/规则：安全类警告必须先诊断原因，再执行确认
- 操作：检查 F 程序周期、CPU 负载、通信状态和安全任务配置后，再评估是否使用 **ACK_FCT_WARN**
- 结论：适合用于确认 F 循环时间相关警告，但必须按安全规范执行

> [!tip] 记忆口诀
> **F 超时先查因，确认之前别心急。**


## 02 安全扩展 | 选型速记

> [!info] 核心结论
> 安全扩展指令的核心是：**先诊断风险，再确认警告；安全逻辑不能靠“复位”掩盖问题。**

<div style="background:#F7F9FC;border-left:6px solid #27AE60;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#27AE60;">总口诀</strong><br>
安全先排故，确认再执行；F 时间超限，先查周期和负载。
</div>

### ① 分类/公式/规则

| 类型 | 常用指令 |
|---|---|
| F 循环时间警告确认 | **ACK_FCT_WARN** |
| 安全程序诊断 | 需结合 F-CPU 诊断信息 |
| 超时原因排查 | CPU 负载、程序周期、通信、任务配置 |
| 安全确认原则 | 原因排除后再确认 |

### ② 参考表

| 需求 | 推荐做法 |
|---|---|
| 确认 F 循环时间警告 | 使用 **ACK_FCT_WARN** |
| 判断是否能确认 | 先查安全诊断信息 |
| 排查循环时间异常 | 检查 F 程序周期和 CPU 负载 |
| 处理安全相关故障 | 按项目安全规范 / 厂家手册执行 |
| 防止误复位 | 加权限、条件和操作确认 |

### ③ 计算/选型示例

- 已知：设备运行中出现 F 循环时间超限警告
- 公式/规则：安全警告确认前必须完成原因排查
- 操作：先检查安全程序周期与诊断信息，确认风险排除后，再执行 **ACK_FCT_WARN**
- 结论：这是典型的**安全警告确认**场景

> [!tip] 记忆口诀
> **安全不是清报警，先排风险再确认。**
