# 01｜Compact PID 总览

> [!info] 核心结论
> **Compact PID 是 TIA Portal 中用于闭环调节的一组 PID 控制指令，核心成员包括 `PID_Compact`、`PID_3Step`、`PID_Temp`。**

<div class="card blue">
<strong>图片识别</strong><br>
截图中 Compact PID 目录下有 3 个指令：<br>
<strong>PID_Compact</strong>：集成调节功能<br>
<strong>PID_3Step</strong>：集成阀门调节功能<br>
<strong>PID_Temp</strong>：温度 PID 控制器
</div>

## ① 分类/公式/规则

| 指令 | 中文含义 | 核心作用 | 典型对象 |
|---|---|---|---|
| **PID_Compact** | 紧凑型 PID 控制器 | 通用连续量调节 | 压力、流量、液位、速度 |
| **PID_3Step** | 三步 PID 控制器 | 三点式执行器调节 | 电动阀、电动风门 |
| **PID_Temp** | 温度 PID 控制器 | 温度闭环控制 | 加热炉、模温、热封、烘箱 |

<div class="card orange">
<strong>PID 本质</strong><br>
PID = 设定值 SP 与实际值 PV 比较，根据偏差自动计算输出。
</div>

$$
e = SP - PV
$$

## ② 参考表

| 控制需求 | 推荐指令 | 输出形式 |
|---|---|---|
| **压力恒定** | `PID_Compact` | 模拟量 / 百分比 |
| **流量调节** | `PID_Compact` | 阀门开度 / 泵频率 |
| **液位控制** | `PID_Compact` | 变频泵 / 调节阀 |
| **电动阀开关调节** | `PID_3Step` | 开阀 / 关阀 |
| **电动风门调节** | `PID_3Step` | 开方向 / 关方向 |
| **加热控温** | `PID_Temp` | PWM / SSR / 调功器 |
| **加热+冷却控温** | `PID_Temp` | 加热输出 + 冷却输出 |

## ③ 计算/选型示例

- **已知：** 水箱液位需要稳定在 `500mm`，执行器是变频泵。
- **公式：** 连续调节对象 → 选择 `PID_Compact`
- **计算：**
  - 设定值：`500mm`
  - 实际值：液位传感器反馈
  - 输出：`0~100%`
  - 变频器频率：`PID输出% × 最大频率`
- **结论：** 液位 + 变频泵属于连续调节，适合 `PID_Compact`。

> [!tip] 记忆口诀
> **连续调节 Compact，三步阀门 3Step，温度控制用 Temp。**

# 02｜PID_Compact

> [!info] 核心结论
> **PID_Compact 是通用型 PID 控制器，适合压力、流量、液位、速度等连续量闭环控制。**

<div class="card blue">
<strong>一句话理解</strong><br>
PID_Compact 就是 PLC 里的“自动调节器”：目标是多少，反馈是多少，输出自动算多少。
</div>

## ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| **指令名称** | `PID_Compact` |
| **中文含义** | 紧凑型 PID 控制器 |
| **作用** | 根据设定值和实际值偏差，计算连续控制输出 |
| **输出特点** | 连续量输出，常见为 `0~100%` |
| **适合场景** | 压力、流量、液位、速度、张力 |
| **一句话理解** | 有设定、有反馈、有连续输出，就可以考虑它 |

<div class="card orange">
<strong>重点公式</strong><br>
偏差 = 设定值 - 实际值
</div>

$$
e = SP - PV
$$

$$
u(t)=K_p\left[e(t)+\frac{1}{T_i}\int e(t)dt+T_d\frac{de(t)}{dt}\right]
$$

## ② 参考表

| 参数 | 中文含义 | 作用 | 注意 |
|---|---|---|---|
| **SP / Setpoint** | 设定值 | 目标值 | 单位要和 PV 一致 |
| **PV / Input** | 实际值 | 传感器反馈 | 需先做工程量换算 |
| **Output** | 控制输出 | 给执行器的调节量 | 常需限幅 |
| **Kp** | 比例增益 | 提高响应速度 | 太大易振荡 |
| **Ti** | 积分时间 | 消除静差 | 太强易超调 |
| **Td** | 微分时间 | 抑制变化趋势 | 对噪声敏感 |
| **Manual** | 手动模式 | 调试输出 | 切自动前先验证方向 |
| **Error / Status** | 错误与状态 | 诊断故障 | 需结合手册校核 |

## ③ 计算/选型示例

- **已知：**
  - 压力设定值：`6bar`
  - 当前压力：`4bar`
  - PID 输出：`60%`
  - 变频器最大频率：`50Hz`
- **公式：**
  - 偏差 = `SP - PV`
  - 频率 = `PID输出% × 最大频率`
- **计算：**
  - 偏差：`6 - 4 = 2bar`
  - 频率：`60% × 50Hz = 30Hz`
- **结论：** 当前压力偏低，PID 输出可换算为 `30Hz` 给变频器。

> [!tip] 记忆口诀
> **P 管现在，I 管过去，D 管趋势；反馈要准，方向要对。**

# 03｜PID_3Step

> [!info] 核心结论
> **PID_3Step 用于三步式执行器控制，适合只有“开方向”和“关方向”的电动阀、电动风门。**

<div class="card green">
<strong>一句话理解</strong><br>
PID_3Step 不输出 0-10V，而是输出“开一点、停一下、关一点”。
</div>

## ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| **指令名称** | `PID_3Step` |
| **中文含义** | 三步 PID 控制器 |
| **作用** | 控制三点式执行器做开/停/关调节 |
| **输出特点** | 两路方向输出：`Open`、`Close` |
| **适合场景** | 电动阀、电动风门、三位式执行器 |
| **一句话理解** | 执行器只有开关方向，没有模拟量输入，就考虑它 |

<div class="card orange">
<strong>三步控制规则</strong><br>
偏低就开，偏高就关，接近目标就停。
</div>

| 偏差状态 | 控制动作 | 输出状态 |
|---|---|---|
| **PV < SP** | 增大过程量 | `Open = TRUE` |
| **PV > SP** | 减小过程量 | `Close = TRUE` |
| **PV ≈ SP** | 保持当前位置 | `Open = FALSE`，`Close = FALSE` |

## ② 参考表

| 参数 | 中文含义 | 作用 | 注意 |
|---|---|---|---|
| **SP / Setpoint** | 设定值 | 目标值 | 与反馈同单位 |
| **PV / Input** | 实际值 | 过程反馈 | 先换算工程量 |
| **Open** | 开方向输出 | 阀门/风门开大 | 不可与 Close 同时动作 |
| **Close** | 关方向输出 | 阀门/风门关小 | 需互锁保护 |
| **Deadband** | 死区 | 小偏差不动作 | 太小会频繁动作 |
| **Min Pulse** | 最小脉冲时间 | 防止输出太短无效 | 按执行器响应校核 |
| **Pause Time** | 最小暂停时间 | 防止频繁点动 | 保护机械寿命 |
| **Actuator Runtime** | 全行程时间 | 估算阀门开度变化 | 需按铭牌或实测校核 |

## ③ 计算/选型示例

- **已知：**
  - 电动阀从全关到全开需要 `60s`
  - 本次开阀输出 `3s`
- **公式：** 开度变化 ≈ `动作时间 ÷ 全行程时间 × 100%`
- **计算：**
  - `3 ÷ 60 × 100% = 5%`
- **结论：** 该开阀脉冲大约让阀门增加 `5%` 开度，实际需按执行器特性校核。

> [!tip] 记忆口诀
> **模拟量阀用 Compact，开关电动阀用 3Step。**

# 04｜PID_Temp

> [!info] 核心结论
> **PID_Temp 是温度控制专用 PID，适合加热、冷却、恒温、热封、模温、烘箱等温控场景。**

<div class="card red">
<strong>温控特点</strong><br>
温度系统通常有热惯性和滞后，参数不能调得太猛，否则容易超温和振荡。
</div>

## ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| **指令名称** | `PID_Temp` |
| **中文含义** | 温度 PID 控制器 |
| **作用** | 控制温度稳定在设定值附近 |
| **控制对象** | 加热器、冷却阀、模温机、烘箱 |
| **输入信号** | 热电偶、PT100、温度变送器 |
| **输出方式** | PWM、SSR、可控硅调功、模拟量 |
| **一句话理解** | 专门解决温度升得稳、停得准、不乱冲的问题 |

<div class="card orange">
<strong>重点公式</strong><br>
温度偏差 = 设定温度 - 实际温度
</div>

$$
e_T = T_{set} - T_{pv}
$$

<div class="card blue">
<strong>温控逻辑</strong><br>
温度低 → 增加加热；温度高 → 减少加热或开启冷却。
</div>

## ② 参考表

| 控制对象 | 执行器 | 推荐输出方式 | 注意 |
|---|---|---|---|
| **加热棒** | SSR | PWM | 周期不要太短 |
| **加热炉** | 可控硅调功器 | 模拟量 / 调功 | 注意功率裕量 |
| **模温机** | 加热 + 冷却阀 | 双向控制 | 方向必须确认 |
| **烘箱** | 加热器 + 风机 | PWM / 开关 | 热惯性大 |
| **热封刀** | 加热器 | PWM / 调功 | 防止超温 |

| 参数 | 作用 | 使用注意 |
|---|---|---|
| **Setpoint** | 温度设定值 | 单位通常为 ℃ |
| **Input / PV** | 实际温度 | 传感器必须准确 |
| **Heat Output** | 加热输出 | 需要限幅 |
| **Cool Output** | 冷却输出 | 防止方向反 |
| **Cycle Time** | PWM 周期 | 接触器不宜频繁动作 |
| **Auto Tuning** | 自动整定 | 使用前确认现场安全 |

## ③ 计算/选型示例

- **已知：**
  - 温度设定：`180℃`
  - 当前温度：`150℃`
  - PID 输出：`40%`
  - PWM 周期：`10s`
- **公式：**
  - 温度偏差 = `设定温度 - 实际温度`
  - 导通时间 = `输出百分比 × PWM周期`
- **计算：**
  - 温度偏差：`180 - 150 = 30℃`
  - 导通时间：`40% × 10s = 4s`
- **结论：** 当前温度偏低，PWM 周期内加热器导通约 `4s`。

> [!tip] 记忆口诀
> **控温先测准，再看方向；先手动，再自动；先稳定，再调快。**

# 05｜Compact PID 选型卡

> [!info] 核心结论
> **Compact PID 选型的关键不是看名字，而是看执行器输出形式：连续量、三步量、温度专用。**

<div class="card blue">
<strong>选型口诀</strong><br>
连续调节用 Compact，三步执行用 3Step，温度控制用 Temp。
</div>

## ① 分类/公式/规则

| 判断问题 | 选择结果 |
|---|---|
| **执行器是 0-10V / 4-20mA / 频率给定？** | `PID_Compact` |
| **执行器只有开阀、关阀两个方向？** | `PID_3Step` |
| **控制对象是温度？** | `PID_Temp` |
| **只是普通开关启停？** | 不建议用 PID |
| **是伺服定位？** | 用 Motion Control，不用 PID |
| **是气缸动作？** | 用顺序控制 / FB，不用 PID |

## ② 参考表

| 场景 | 推荐指令 | 判断依据 |
|---|---|---|
| **水泵恒压** | `PID_Compact` | 连续频率给定 |
| **流量调节阀** | `PID_Compact` | 模拟量阀门 |
| **电动三通阀** | `PID_3Step` | 开/关方向控制 |
| **电动风门** | `PID_3Step` | 三步式执行器 |
| **热封刀温控** | `PID_Temp` | 温度闭环 |
| **烘箱恒温** | `PID_Temp` | 热惯性系统 |

## ③ 计算/选型示例

- **已知：** 一个阀门只有“开阀”和“关阀”两个数字量输入，没有 `0-10V` 或 `4-20mA` 输入。
- **公式：** 三步执行器 = `Open + Close + Stop`
- **计算：**
  - 开阀信号：数字量输出 1
  - 关阀信号：数字量输出 2
  - 两路都断开：阀门停止
- **结论：** 该阀门适合 `PID_3Step`，不适合直接用 `PID_Compact`。

> [!tip] 记忆口诀
> **看执行器选 PID：模拟量选 Compact，三点式选 3Step，加热控温选 Temp。**

# 06｜Compact PID 调试流程

> [!info] 核心结论
> **PID 调试不要一上来就改参数，应先确认反馈、输出、方向、限幅和安全条件。**

<div class="card orange">
<strong>标准调试流程</strong><br>
确认反馈 → 手动输出 → 确认方向 → 设置限幅 → 切自动 → 调 PID 参数。
</div>

## ① 分类/公式/规则

| 步骤 | 动作 | 重点 |
|---|---|---|
| **1** | 检查传感器 | PV 是否真实 |
| **2** | 工程量换算 | 单位是否正确 |
| **3** | 手动输出 | 执行器是否动作 |
| **4** | 确认方向 | 输出增大，PV 是否按预期变化 |
| **5** | 设置限幅 | 防止超温、超压、超速 |
| **6** | 切自动 | 进入闭环控制 |
| **7** | 参数整定 | 调 Kp、Ti、Td |
| **8** | 趋势观察 | 用 Trace 看 SP、PV、Output |

## ② 参考表

| 现象 | 可能原因 | 调整方向 |
|---|---|---|
| **响应很慢** | Kp 小 / 输出受限 | 适当增大 Kp 或查执行器 |
| **超调明显** | 参数太猛 / 惯性大 | 降 Kp，减弱积分 |
| **长期有偏差** | 积分不足 | 检查 Ti 和输出限幅 |
| **来回振荡** | Kp 太大 / 积分太强 | 降 Kp 或增大 Ti |
| **输出一直满值** | 执行器能力不足 / 反馈异常 | 先查硬件 |
| **越调越远** | 控制方向反 | 先修正方向 |
| **PV 跳动大** | 传感器干扰 / 滤波不足 | 查屏蔽接地和滤波 |

## ③ 计算/选型示例

- **已知：**
  - 设定压力：`5bar`
  - 实际压力：`0bar`
  - PID 输出已经 `100%`
- **公式：** 满输出无变化 → 先查反馈或执行器
- **计算：**
  - 输出满值说明 PID 已经尽力调节
  - PV 不变化可能是传感器、阀门、泵、量程或接线异常
- **结论：** 不要先调 PID 参数，应先检查传感器和执行器链路。

> [!tip] 记忆口诀
> **PID 不稳先别调，先看反馈准不准。**

# 07｜Compact PID 速记总卡

> [!info] 核心结论
> **Compact PID 的核心就是闭环控制：设定值给目标，实际值给反馈，PID 输出管执行器。**

<div class="card blue">
<strong>总口诀</strong><br>
先看对象，再看输出；反馈要准，方向要对；参数最后再调。
</div>

## ① 分类/公式/规则

| 主轴 | 关键词 |
|---|---|
| **输入** | 设定值 `SP`、实际值 `PV` |
| **计算** | 偏差 `e = SP - PV` |
| **输出** | 控制量 `Output` |
| **对象** | 压力、流量、液位、温度、速度 |
| **核心参数** | `Kp`、`Ti`、`Td` |
| **核心风险** | 方向反、量程错、输出无限幅 |
| **诊断重点** | `Error`、`Status`、趋势曲线 |

## ② 参考表

| 指令 | 最适合场景 | 一句话 |
|---|---|---|
| **PID_Compact** | 通用连续量 | 什么都能连续调一点 |
| **PID_3Step** | 三步阀门 | 开一点、关一点、停一下 |
| **PID_Temp** | 温度控制 | 控温专用，考虑热惯性 |

<div class="card red">
<strong>避坑提醒</strong><br>
PID 参数不是万能药；反馈错误、方向错误、执行器故障时，调参数只会越调越乱。
</div>

## ③ 计算/选型示例

- **已知：** 要给项目建立 PID 控制方案。
- **公式：** PID 选型 = `控制对象 + 执行器类型 + 输出形式`
- **计算：**
  - 压力 + 变频器 → `PID_Compact`
  - 风门 + 开关方向控制 → `PID_3Step`
  - 加热炉 + SSR → `PID_Temp`
- **结论：** 先看对象，再看执行器，最后选 PID 块。

> [!tip] 记忆口诀
> **Compact 管连续，3Step 管阀门，Temp 管温度；反馈准，方向对，PID 才能稳。**
