# 01｜MC_Power：启动/禁用轴

> [!info] 核心结论
> **MC_Power 用来给运动轴“上使能”或“取消使能”，是所有运动指令前的第一步。**

<div class="card blue">
<strong>一句话理解</strong><br>
轴想运动，先 Power；轴没使能，Move 指令都白给。
</div>

## ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| **指令名称** | `MC_Power` |
| **中文含义** | 启动/禁用轴 |
| **所属类别** | Motion Control 轴控制 |
| **核心作用** | 使能或禁止工艺轴 |
| **使用前提** | 轴对象已组态，驱动器无严重故障 |

<div class="card orange">
<strong>控制规则</strong><br>
Enable = TRUE → 轴使能；Enable = FALSE → 轴取消使能。
</div>

## ② 参考表

| 常用接口 | 作用 | 使用注意 |
|---|---|---|
| **Axis** | 指定轴对象 | 连接 `TO_PositioningAxis` 等工艺对象 |
| **Enable** | 使能命令 | 需要保持为 TRUE |
| **Status** | 轴已使能状态 | TRUE 表示轴已准备好 |
| **Busy** | 正在处理 | 等待状态变化 |
| **Error** | 指令错误 | 需结合 ErrorID |
| **ErrorID** | 错误代码 | 按帮助文档诊断 |

## ③ 计算/选型示例

- **已知：** 伺服轴需要执行绝对定位。
- **公式：** 运动前条件 = `轴使能 + 无故障 + 已回零`
- **计算：**
  - 先执行 `MC_Power.Enable = TRUE`
  - 等待 `Status = TRUE`
  - 再执行 `MC_Home` 或定位指令
- **结论：** `MC_Power` 是运动控制的入口指令。

> [!tip] 记忆口诀
> **先 Power，再 Home；先使能，再运动。**

# 02｜MC_Reset：确认错误 / 复位轴

> [!info] 核心结论
> **MC_Reset 用于清除轴或运动控制指令的错误状态，让轴重新回到可操作状态。**

<div class="card red">
<strong>避坑提醒</strong><br>
Reset 不是万能清故障，必须先处理真实故障原因，例如急停、限位、驱动报警。
</div>

## ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| **指令名称** | `MC_Reset` |
| **中文含义** | 确认错误 / 复位轴 |
| **核心作用** | 清除运动控制错误 |
| **触发方式** | 通常使用 `Execute` 上升沿 |
| **使用场景** | 轴报警、指令报错、驱动器故障恢复后 |

<div class="card orange">
<strong>复位规则</strong><br>
先排除故障原因，再给 Reset 脉冲。
</div>

## ② 参考表

| 常用接口 | 作用 | 使用注意 |
|---|---|---|
| **Axis** | 指定轴对象 | 必须连接对应工艺轴 |
| **Execute** | 复位触发 | 建议用上升沿 |
| **Done** | 复位完成 | TRUE 表示复位完成 |
| **Busy** | 正在复位 | TRUE 时不要重复触发 |
| **Error** | 复位失败 | 需查 ErrorID |
| **ErrorID** | 错误代码 | 结合轴诊断查看 |

## ③ 计算/选型示例

- **已知：** 轴因正限位报警停止。
- **公式：** 复位流程 = `解除限位原因 + MC_Reset + 重新使能`
- **计算：**
  - 移除限位触发原因
  - 给 `MC_Reset.Execute` 一个上升沿
  - 等待 `Done = TRUE`
  - 再执行 `MC_Power`
- **结论：** Reset 只负责确认错误，不负责解决硬件问题。

> [!tip] 记忆口诀
> **故障先处理，Reset 再复位。**

# 03｜MC_Home：归位轴 / 建立零点

> [!info] 核心结论
> **MC_Home 用于给轴建立参考点，让 PLC 知道当前位置和机械坐标之间的关系。**

<div class="card blue">
<strong>一句话理解</strong><br>
没回零的轴，只知道自己在动；回零后的轴，才知道自己在哪。
</div>

## ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| **指令名称** | `MC_Home` |
| **中文含义** | 归位轴 / 回零 / 设置参考点 |
| **核心作用** | 建立轴坐标基准 |
| **常见对象** | 增量编码器轴、伺服滑台、步进轴 |
| **使用前提** | 轴已使能，回零参数已配置 |

<div class="card orange">
<strong>核心规则</strong><br>
增量轴通常必须先回零，才能做可靠的绝对定位。
</div>

## ② 参考表

| 常用接口 | 作用 | 使用注意 |
|---|---|---|
| **Axis** | 指定轴对象 | 连接定位轴 |
| **Execute** | 启动回零 | 使用上升沿 |
| **Position** | 回零后设定位置 | 常设为 `0.0` |
| **Done** | 回零完成 | 完成后轴已参考 |
| **Busy** | 正在回零 | 不要重复触发 |
| **Error** | 回零错误 | 查限位、方向、原点信号 |
| **ErrorID** | 错误代码 | 按实际诊断校核 |

## ③ 计算/选型示例

- **已知：** 丝杆轴上电后当前位置未知。
- **公式：** 增量轴定位前 = `MC_Power + MC_Home`
- **计算：**
  - 执行 `MC_Power`
  - 执行 `MC_Home`
  - 回零完成后当前位置设为 `0.0mm`
- **结论：** 回零完成后，才能安全执行绝对定位。

> [!tip] 记忆口诀
> **增量先回零，绝对才定位。**

# 04｜MC_Halt：暂停轴

> [!info] 核心结论
> **MC_Halt 用于按设定减速度让轴受控停止，适合暂停当前运动。**

<div class="card orange">
<strong>重点区别</strong><br>
MC_Halt 是受控暂停；急停和安全停机需要专门安全回路，不能只靠 MC_Halt。
</div>

## ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| **指令名称** | `MC_Halt` |
| **中文含义** | 暂停轴 / 受控停止 |
| **核心作用** | 按动态参数停止当前轴运动 |
| **适用场景** | 暂停定位、工艺中断、普通停止 |
| **不适合场景** | 安全急停，需安全回路校核 |

## ② 参考表

| 常用接口 | 作用 | 使用注意 |
|---|---|---|
| **Axis** | 指定轴对象 | 要停止的轴 |
| **Execute** | 停止触发 | 上升沿触发 |
| **Deceleration** | 减速度 | 太大可能冲击或报警 |
| **Jerk** | 加加速度 | 用于平滑停止 |
| **Done** | 停止完成 | 轴速度降为 0 |
| **Busy** | 停止中 | 正在减速 |
| **Error** | 停止失败 | 查 ErrorID |

## ③ 计算/选型示例

- **已知：** 轴速度 `200 mm/s`，希望 `0.5 s` 内停下。
- **公式：** 减速度 = `速度 ÷ 时间`
- **计算：**
  - `200 ÷ 0.5 = 400 mm/s²`
- **结论：** 可初步设置减速度约 `400 mm/s²`，实际需按机械负载和驱动器能力校核。

> [!tip] 记忆口诀
> **正常停用 Halt，安全停靠安全回路。**

# 05｜MC_MoveAbsolute：绝对定位

> [!info] 核心结论
> **MC_MoveAbsolute 用于让轴运动到一个明确的绝对坐标位置。**

<div class="card blue">
<strong>一句话理解</strong><br>
绝对定位就是告诉轴：去坐标 100mm，不管你现在在哪里。
</div>

## ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| **指令名称** | `MC_MoveAbsolute` |
| **中文含义** | 绝对方式定位 |
| **核心作用** | 移动到指定坐标 |
| **前提条件** | 轴已使能，通常需要已回零 |
| **典型场景** | 去工位、回原点、定点搬运 |

<div class="card orange">
<strong>重点公式</strong><br>
绝对定位目标 = 坐标系中的固定位置
</div>

## ② 参考表

| 常用接口 | 作用 | 使用注意 |
|---|---|---|
| **Axis** | 指定轴对象 | 定位轴 |
| **Execute** | 启动定位 | 使用上升沿 |
| **Position** | 目标位置 | 单位取决于轴组态 |
| **Velocity** | 目标速度 | 不能超过轴限制 |
| **Acceleration** | 加速度 | 影响启动冲击 |
| **Deceleration** | 减速度 | 影响停止平稳性 |
| **Done** | 定位完成 | 到达目标位置 |
| **Busy** | 正在运动 | 不要重复启动 |
| **Error** | 定位错误 | 查超限、未回零、轴故障 |

## ③ 计算/选型示例

- **已知：** 当前轴在 `20mm`，目标是 `100mm`。
- **公式：** 实际移动距离 = `目标位置 - 当前位置`
- **计算：**
  - `100 - 20 = 80mm`
- **结论：** 轴会向目标坐标 `100mm` 运动，实际移动 `80mm`。

> [!tip] 记忆口诀
> **目标是坐标，用 Absolute。**

# 06｜MC_MoveRelative：相对定位

> [!info] 核心结论
> **MC_MoveRelative 用于让轴在当前位置基础上移动一段距离。**

<div class="card green">
<strong>一句话理解</strong><br>
相对定位不是去某个坐标，而是“从现在开始再走多少”。
</div>

## ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| **指令名称** | `MC_MoveRelative` |
| **中文含义** | 相对方式定位 |
| **核心作用** | 基于当前位置移动指定距离 |
| **前提条件** | 轴已使能 |
| **典型场景** | 步进送料、微调、相对补偿 |

<div class="card orange">
<strong>重点公式</strong><br>
最终位置 = 当前实际位置 + 相对移动距离
</div>

## ② 参考表

| 常用接口 | 作用 | 使用注意 |
|---|---|---|
| **Axis** | 指定轴对象 | 定位轴 |
| **Execute** | 启动移动 | 上升沿触发 |
| **Distance** | 相对距离 | 正负号决定方向 |
| **Velocity** | 速度 | 按轴单位设置 |
| **Acceleration** | 加速度 | 需按机械能力校核 |
| **Deceleration** | 减速度 | 避免冲击 |
| **Done** | 移动完成 | 达到相对位移 |
| **Busy** | 正在移动 | 避免重复触发 |
| **Error** | 移动错误 | 查限位和参数 |

## ③ 计算/选型示例

- **已知：** 当前轴位置 `50mm`，需要再前进 `30mm`。
- **公式：** 最终位置 = `当前位置 + Distance`
- **计算：**
  - `50 + 30 = 80mm`
- **结论：** 执行后轴目标位置约为 `80mm`。

> [!tip] 记忆口诀
> **目标是距离，用 Relative。**

# 07｜MC_MoveVelocity：速度运行

> [!info] 核心结论
> **MC_MoveVelocity 用于让轴按指定速度连续运行，直到被停止或新命令中断。**

<div class="card blue">
<strong>一句话理解</strong><br>
速度运行不是去某个位置，而是让轴一直按某个速度跑。
</div>

## ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| **指令名称** | `MC_MoveVelocity` |
| **中文含义** | 以预定义速度运行 |
| **核心作用** | 按速度连续运动 |
| **典型场景** | 输送、卷绕、连续送料、测试运行 |
| **停止方式** | 通常用 `MC_Halt` 或其它停止逻辑 |

<div class="card orange">
<strong>重点规则</strong><br>
速度运行一般不会因为到达某个位置自动停止，必须设计停止条件。
</div>

## ② 参考表

| 常用接口 | 作用 | 使用注意 |
|---|---|---|
| **Axis** | 指定轴对象 | 运动轴 |
| **Execute** | 启动速度运行 | 上升沿触发 |
| **Velocity** | 运行速度 | 正负号可能决定方向 |
| **Acceleration** | 加速度 | 启动斜率 |
| **Deceleration** | 减速度 | 停止斜率 |
| **InVelocity** | 达到速度 | 表示进入稳定速度 |
| **Busy** | 正在执行 | 运动持续中 |
| **Error** | 运行错误 | 查轴状态 |

## ③ 计算/选型示例

- **已知：** 输送带需要以 `300 mm/s` 连续运行。
- **公式：** 连续输送 = `速度指令 + 停止条件`
- **计算：**
  - `MC_MoveVelocity.Velocity = 300`
  - 到达检测位置后执行 `MC_Halt`
- **结论：** 连续运动场景适合用 `MC_MoveVelocity`。

> [!tip] 记忆口诀
> **位置控制看 Move，连续运行看 Velocity。**

# 08｜MC_MoveJog：点动运行

> [!info] 核心结论
> **MC_MoveJog 用于手动调试时按正向或反向点动轴，常用于找位置、调方向、低速试机。**

<div class="card green">
<strong>工程建议</strong><br>
新轴调试时，先用 Jog 低速点动确认方向，再执行回零和定位。
</div>

## ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| **指令名称** | `MC_MoveJog` |
| **中文含义** | 点动模式运行 |
| **核心作用** | 手动正/反方向移动轴 |
| **典型场景** | 调试、找零、维修、手动操作 |
| **安全重点** | 低速、小范围、限位有效 |

## ② 参考表

| 常用接口 | 作用 | 使用注意 |
|---|---|---|
| **Axis** | 指定轴对象 | 运动轴 |
| **JogForward** | 正向点动 | 按住或保持时正向运行 |
| **JogBackward** | 反向点动 | 按住或保持时反向运行 |
| **Velocity** | 点动速度 | 初次调试要设低 |
| **Acceleration** | 加速度 | 防止冲击 |
| **Deceleration** | 减速度 | 松开后平稳停止 |
| **Busy** | 正在点动 | 运动状态 |
| **Error** | 点动错误 | 查限位、轴状态 |

## ③ 计算/选型示例

- **已知：** 新调试滑台，机械行程未知。
- **公式：** 新轴调试 = `低速 Jog + 观察方向 + 校验限位`
- **计算：**
  - 点动速度先设 `10 mm/s`
  - 正向点动观察坐标是否增加
  - 反向点动检查负限位方向
- **结论：** `MC_MoveJog` 是轴调试阶段最常用的安全测试指令。

> [!tip] 记忆口诀
> **新轴先 Jog，别上来就 Move。**

# 09｜MC_CommandTable：按移动顺序运行

> [!info] 核心结论
> **MC_CommandTable 用于执行预先配置好的运动命令表，让轴按步骤自动运行。**

<div class="card purple">
<strong>一句话理解</strong><br>
CommandTable 就是运动流程表：第 1 步去哪，第 2 步等多久，第 3 步再去哪。
</div>

## ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| **指令名称** | `MC_CommandTable` |
| **中文含义** | 按移动顺序运行 |
| **核心作用** | 执行命令表中的多步运动 |
| **常见搭配** | `TO_CommandTable` |
| **典型场景** | 固定多点位、往复运动、测试循环 |

<div class="card orange">
<strong>选型规则</strong><br>
单点动作直接用 Move，多步固定流程用 CommandTable。
</div>

## ② 参考表

| 常用接口 | 作用 | 使用注意 |
|---|---|---|
| **Axis** | 指定轴对象 | 执行命令的轴 |
| **CommandTable** | 命令表对象 | 连接 `TO_CommandTable` |
| **Execute** | 启动命令表 | 上升沿触发 |
| **Done** | 表执行完成 | 全部步骤完成 |
| **Busy** | 正在执行 | 不要重复启动 |
| **Error** | 表执行错误 | 查当前步骤和 ErrorID |
| **ErrorID** | 错误代码 | 结合轴状态诊断 |

## ③ 计算/选型示例

- **已知：** 滑台要执行 `0mm → 100mm → 200mm → 0mm`。
- **公式：** 固定多步点位 = `TO_CommandTable + MC_CommandTable`
- **计算：**
  - 在命令表写入 4 个动作步骤
  - 轴使能并回零
  - 执行 `MC_CommandTable`
- **结论：** 固定多步动作适合用命令表统一管理。

> [!tip] 记忆口诀
> **单步用 Move，多步用 Table。**

# 10｜MC_ChangeDynamic：更改轴的动态参数

> [!info] 核心结论
> **MC_ChangeDynamic 用于在线修改轴运动的速度、加速度、减速度、加加速度等动态参数。**

<div class="card orange">
<strong>一句话理解</strong><br>
ChangeDynamic 就是改变轴“跑得多快、起得多猛、停得多柔”。
</div>

## ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| **指令名称** | `MC_ChangeDynamic` |
| **中文含义** | 更改轴的动态参数 |
| **核心作用** | 修改轴运行动态特性 |
| **常见参数** | 速度、加速度、减速度、加加速度 |
| **典型场景** | 工艺切换、低速/高速模式、柔性启停 |

<div class="card blue">
<strong>动态口诀</strong><br>
速度管快慢，加速度管起步，减速度管停车，Jerk 管柔顺。
</div>

## ② 参考表

| 动态参数 | 作用 | 设置过大风险 |
|---|---|---|
| **Velocity** | 运行速度 | 冲击、跟随误差 |
| **Acceleration** | 启动加速度 | 抖动、过流 |
| **Deceleration** | 停止减速度 | 过压、冲击 |
| **Jerk** | 加加速度 | 太大不柔顺，太小响应慢 |
| **Dynamic limit** | 轴限制 | 需按轴组态和驱动器校核 |

## ③ 计算/选型示例

- **已知：** 低速模式速度 `50 mm/s`，高速模式速度 `300 mm/s`。
- **公式：** 工艺模式不同 → 动态参数不同
- **计算：**
  - 调试模式：低速度、低加速度
  - 生产模式：高速度、合理加速度
  - 切换时调用 `MC_ChangeDynamic`
- **结论：** 不同工艺节拍可用不同动态参数，但需按机械能力校核。

> [!tip] 记忆口诀
> **慢速调试保安全，高速生产看节拍。**

# 11｜MC_WriteParam：写入工艺对象参数

> [!info] 核心结论
> **MC_WriteParam 用于向运动控制工艺对象写入指定参数，适合调试、参数切换和运行中配置。**

<div class="card red">
<strong>避坑提醒</strong><br>
写参数会改变工艺对象行为，写入前必须确认参数编号、数据类型和允许写入条件。
</div>

## ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| **指令名称** | `MC_WriteParam` |
| **中文含义** | 写入工艺对象参数 |
| **核心作用** | 修改轴或工艺对象参数 |
| **典型场景** | 修改限位、动态参数、配置参数 |
| **风险点** | 参数号错误、类型错误、运行中不允许修改 |

<div class="card orange">
<strong>使用规则</strong><br>

</div>

## ② 参考表

## ③ 计算/选型示例

- **已知：** 工艺需要根据产品型号切换某个轴参数。
- **公式：** 产品参数切换 = `读取产品型号 + 写入对应参数`
- **计算：**
  - 产品 A 使用参数组 A
  - 产品 B 使用参数组 B
  - 切换时调用 `MC_WriteParam`
- **结论：** `MC_WriteParam` 适合做工艺参数动态切换，但必须按厂家手册校核可写参数。

# 13｜Motion Control 指令总表

> [!info] 核心结论
> **Motion Control 指令的主线是：使能、复位、回零、运动、停止、参数读写。**

## ① 分类/公式/规则

| 分类 | 指令 | 中文含义 |
|---|---|---|
| **轴准备** | `MC_Power` | 启动/禁用轴 |
| **故障处理** | `MC_Reset` | 确认错误/复位 |
| **坐标建立** | `MC_Home` | 归位轴/设置参考点 |
| **停止控制** | `MC_Halt` | 暂停轴 |
| **位置运动** | `MC_MoveAbsolute` | 绝对定位 |
| **位置运动** | `MC_MoveRelative` | 相对定位 |
| **速度运动** | `MC_MoveVelocity` | 速度运行 |
| **手动调试** | `MC_MoveJog` | 点动运行 |
| **流程运动** | `MC_CommandTable` | 按命令表运行 |
| **动态调整** | `MC_ChangeDynamic` | 改变动态参数 |
| **参数写入** | `MC_WriteParam` | 写工艺对象参数 |
| **参数读取** | `MC_ReadParam` | 读工艺对象参数 |

## ② 参考表

| 任务 | 优先选择 |
|---|---|
| **轴上电使能** | `MC_Power` |
| **轴报警后复位** | `MC_Reset` |
| **增量轴找零点** | `MC_Home` |
| **去固定坐标** | `MC_MoveAbsolute` |
| **再走一段距离** | `MC_MoveRelative` |
| **连续按速度跑** | `MC_MoveVelocity` |
| **手动低速调试** | `MC_MoveJog` |
| **固定多步动作** | `MC_CommandTable` |
| **改速度/加速度** | `MC_ChangeDynamic` |
| **读写轴参数** | `MC_ReadParam` / `MC_WriteParam` |

## ③ 计算/选型示例

- **已知：** 一根伺服轴要自动去 `100mm` 工位。
- **公式：** 标准定位流程 = `Power → Home → MoveAbsolute → Monitor`
- **计算：**
  - `MC_Power` 使能轴
  - `MC_Home` 建立零点
  - `MC_MoveAbsolute.Position = 100.0`
  - 监控 `Done / Busy / Error`
- **结论：** 标准定位轴程序通常围绕这几个 MC 指令组合完成。

> [!tip] 记忆口诀
> **Power 使能，Home 回零；Absolute 到点，Relative 走距；Velocity 连跑，Jog 调试。**
