## 01 `--(JMP)` | 若 RLO = "1" 则跳转

> [!info] 核心结论
> **--(JMP)** 是条件跳转指令，当当前逻辑结果 **RLO = 1** 时，程序跳转到指定标签。

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">重点规则</strong><br>
RLO 为 <strong>1</strong> 才跳转；RLO 为 0 时继续顺序执行。
</div>

### ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| 所属分类 | **程序控制指令** |
| 指令名称 | **--(JMP)** |
| 功能描述 | **若 RLO = "1" 则跳转** |
| 核心作用 | 条件成立时跳转 |
| 配套指令 | 需要配合 **LABEL** 使用 |

### ② 参考表

| RLO 状态 | JMP 动作 |
|---|---|
| RLO = 0 | 不跳转 |
| RLO = 1 | **跳转到指定标签** |

### ③ 计算/选型示例

- 已知：故障条件成立时，跳过后续普通运行逻辑
- 公式：`RLO = 1 → JMP`
- 计算：故障触点导通，RLO = 1
- 结论：程序跳转到故障处理标签

> [!tip] 记忆口诀
> **JMP 看 1，条件成立才跳去。**


## 02 `--(JMPN)` | 若 RLO = "0" 则跳转

> [!info] 核心结论
> **--(JMPN)** 是反条件跳转指令，当当前逻辑结果 **RLO = 0** 时跳转。

<div style="background:#F7F9FC;border-left:6px solid #FF6B35;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#FF6B35;">重点规则</strong><br>
JMPN 的 N 可以理解为 <strong>NOT</strong>，条件不成立时跳转。
</div>

### ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| 所属分类 | **程序控制指令** |
| 指令名称 | **--(JMPN)** |
| 功能描述 | **若 RLO = "0" 则跳转** |
| 核心作用 | 条件不成立时跳转 |
| 配套指令 | 需要配合 **LABEL** 使用 |

### ② 参考表

| RLO 状态 | JMPN 动作 |
|---|---|
| RLO = 0 | **跳转到指定标签** |
| RLO = 1 | 不跳转 |

### ③ 计算/选型示例

- 已知：允许条件不满足时，跳过执行段
- 公式：`RLO = 0 → JMPN`
- 计算：允许信号为 0，RLO = 0
- 结论：程序跳转到指定标签，跳过当前功能段

> [!tip] 记忆口诀
> **JMPN 看 0，条件不成就跳走。**


## 03 `LABEL` | 跳转标签

> [!info] 核心结论
> **LABEL** 是跳转目标位置，用于标记 JMP、JMPN 等跳转指令要跳到哪里。

<div style="background:#F7F9FC;border-left:6px solid #27AE60;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#27AE60;">重点规则</strong><br>
有跳转，就要有标签；标签名称要清楚，避免跳错位置。
</div>

### ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| 所属分类 | **程序控制指令** |
| 指令名称 | **LABEL** |
| 功能描述 | **跳转标签** |
| 核心作用 | 定义跳转目标 |
| 常见配合 | JMP / JMPN / SWITCH |

### ② 参考表

| 场景 | 建议 |
|---|---|
| 单条件跳转 | JMP + LABEL |
| 反条件跳转 | JMPN + LABEL |
| 多分支跳转 | SWITCH / JMP_LIST + LABEL |
| 标签命名 | 建议体现功能含义 |

### ③ 计算/选型示例

- 已知：故障时需要跳转到故障处理段
- 公式/规则：跳转目标必须有标签
- 操作：在故障处理段前放置 **LABEL**
- 结论：JMP 才能跳到正确位置

> [!tip] 记忆口诀
> **JMP 是箭头，LABEL 是靶心。**

- 结论：适合多个模式互斥执行

> [!tip] 记忆口诀
> **SWITCH 像分岔口，数值不同走不同路。**


## 06 `--(RET)` | 返回

> [!info] 核心结论
> **--(RET)** 用于从当前程序块或调用逻辑中返回，常用于提前结束当前块执行。

<div style="background:#F7F9FC;border-left:6px solid #FF6B35;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#FF6B35;">重点提醒</strong><br>
RET 会改变程序执行流程，使用前要确认后续逻辑是否允许被跳过。
</div>

### ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| 所属分类 | **程序控制指令** |
| 指令名称 | **--(RET)** |
| 功能描述 | **返回** |
| 核心作用 | 从当前执行段返回 |
| 注意事项 | 是否影响后续逻辑需按程序结构校核 |

### ② 参考表

| 场景 | 是否适合 |
|---|---|
| 条件不满足，提前退出 | 适合 |
| 故障时跳过普通逻辑 | 可用 |
| 必须执行的安全逻辑 | 不建议轻易跳过 |
| 调试临时跳过 | 需谨慎 |

### ③ 计算/选型示例

- 已知：某功能未启用时，不需要执行后续复杂计算
- 公式/规则：未启用则提前返回
- 操作：条件满足时执行 **--(RET)**
- 结论：减少无效逻辑执行

> [!tip] 记忆口诀
> **RET 是返回键，不该执行就提前退。**

| 周期监视 | **RE_TRIGR** |
| 错误诊断 | **GET_ERROR / GET_ERR_ID** |
| 性能测量 | **RUNTIME** |

### ② 参考表

| 需求 | 推荐指令 |

- 公式/规则：不同模式进入不同程序段
- 操作：使用 **SWITCH** 根据模式号分配跳转
- 结论：这是典型的多分支程序控制场景

> [!tip] 记忆口诀
> **单路跳转用 JMP，多路分配用 SWITCH；查错看 ERROR，测速看 RUNTIME。**
