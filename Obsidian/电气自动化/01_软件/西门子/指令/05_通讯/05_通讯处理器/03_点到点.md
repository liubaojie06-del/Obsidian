## 01 `PORT_CFG` | 动态组态通信参数

> [!info] 核心结论
> **PORT_CFG** 用于动态配置点到点通信端口参数，是串行通信前的基础设置。

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">版本</strong><br>
分类版本：<strong>V1.0</strong><br>
指令版本：<strong>V1.0</strong>
</div>

### ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| 所属分类 | **点到点** |
| 指令名称 | **PORT_CFG** |
| 功能描述 | **动态组态通信参数** |
| 版本 | **V1.0** |
| 核心作用 | 配置通信端口参数 |
| 使用重点 | 波特率、校验位、数据位、停止位需按设备手册校核 |

### ② 参考表

| 参数 | 说明 |
|---|---|
| 波特率 | 双方通信速度必须一致 |
| 数据位 | 常见为 7 位或 8 位 |
| 校验位 | None / Even / Odd 等 |
| 停止位 | 常见 1 位或 2 位 |
| 通信端口 | 对应实际串口或通信模块 |

### ③ 计算/选型示例

- 已知：PLC 需要通过 RS-232 与扫码枪通信
- 公式/规则：通信前先配置端口参数
- 操作：使用 **PORT_CFG** 设置波特率、校验位、数据位和停止位
- 结论：端口参数一致后，才能稳定收发数据

> [!tip] 记忆口诀
> **点到点先配口，波特校验别搞错。**

## 09 点到点 | 选型速记

> [!info] 核心结论
> 点到点通信的核心是：**先配端口，再配收发；发送用 SEND_PTP，接收用 RCV_PTP，缓冲异常用 RCV_RST。**

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">总口诀</strong><br>
端口 PORT_CFG，发送 SEND_CFG，接收 RCV_CFG；发用 SEND_PTP，收用 RCV_PTP，清缓存用 RCV_RST。
</div>

### ① 分类/公式/规则

| 类型 | 常用指令 |
|---|---|
| 通信端口组态 | **PORT_CFG** |
| 发送参数组态 | **SEND_CFG** |
| 接收参数组态 | **RCV_CFG** |
| 发送数据 | **SEND_PTP** |
| 接收消息 | **RCV_PTP** |
| 清接收缓存 | **RCV_RST** |
| 查询 RS-232 信号 | **SGN_GET** |
| 设置 RS-232 信号 | **SGN_SET** |

### ② 参考表

| 需求 | 推荐指令 |
|---|---|
| 动态配置通信端口 | **PORT_CFG** |
| 配置发送规则 | **SEND_CFG** |
| 配置接收规则 | **RCV_CFG** |
| 发送缓冲区数据 | **SEND_PTP** |
| 启用接收消息 | **RCV_PTP** |
| 删除接收缓冲区 | **RCV_RST** |
| 查询 RS-232 信号 | **SGN_GET** |
| 设置 RS-232 信号 | **SGN_SET** |

### ③ 计算/选型示例

- 已知：PLC 通过 RS-232 读取扫码枪条码
- 公式/规则：先配置端口和接收参数，再启用接收
- 操作：使用 **PORT_CFG** 配置端口；用 **RCV_CFG** 设置接收规则；用 **RCV_PTP** 接收条码；异常时用 **RCV_RST** 清缓存
- 结论：这是典型的点到点串行接收流程

> [!tip] 记忆口诀
> **配置在前，收发在后；信号先查，缓存会清。**
