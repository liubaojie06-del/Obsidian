## 01 `TSEND_C` | 正在建立连接和发送数据

> [!info] 核心结论
> **TSEND_C** 是带连接管理的发送指令，可自动建立通信连接并发送数据，适合 TCP 类通信发送场景。

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">版本</strong><br>
分类版本：<strong>V8.1</strong><br>
指令版本：<strong>V3.2</strong>
</div>

### ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| 所属分类 | **开放式用户通信** |
| 指令名称 | **TSEND_C** |
| 功能描述 | **正在建立连接和发送数据** |
| 版本 | **V3.2** |
| 核心作用 | 建立连接 + 发送数据 |
| 使用重点 | 连接参数、发送缓冲区、数据长度需按实际 CPU/通信对象校核 |

### ② 参考表

| 适用场景 | 说明 |
|---|---|
| TCP 数据发送 | 建立连接后发送数据 |
| PLC 到上位机发送 | 上传状态、报警、生产数据 |
| PLC 到 PLC 通信 | 发送控制字、设定值 |
| 自动连接管理 | 比 TSEND 更集成 |

### ③ 计算/选型示例

- 已知：PLC 需要主动连接上位机并发送生产数据
- 公式/规则：需要“连接 + 发送”一体化
- 操作：选择 **TSEND_C**
- 结论：适合 TCP 发送类开放式通信

> [!tip] 记忆口诀
> **发送带建连，优先 TSEND_C。**

## 14 开放式用户通信 | 选型速记

> [!info] 核心结论
> 开放式用户通信的核心是：**带连接管理用 TSEND_C/TRCV_C，手动连接用 TCON + TSEND/TRCV，UDP 用 TUSEND/TURCV，诊断用 T_DIAG。**

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">总口诀</strong><br>
带 C 自动管连接，不带 C 手动建连接；TCP 看 SEND/RCV，UDP 看 USEND/URCV，异常先 DIAG 再 RESET。
</div>

### ① 分类/公式/规则

| 类型 | 常用指令 |
|---|---|
| 集成连接发送 | **TSEND_C** |
| 集成连接接收 | **TRCV_C** |
| 邮件发送 | **TMAIL_C** |
| 建立/断开连接 | **TCON / TDISCON** |
| 连接收发 | **TSEND / TRCV** |
| UDP 收发 | **TUSEND / TURCV** |
| 连接复位 | **T_RESET** |
| 连接诊断 | **T_DIAG** |
| 接口配置 | **T_CONFIG** |

### ② 参考表

| 需求 | 推荐指令 |
|---|---|
| 自动建立连接并发送 | **TSEND_C** |
| 自动建立连接并接收 | **TRCV_C** |
| 手动建立连接 | **TCON** |
| 手动断开连接 | **TDISCON** |
| 已有连接发送数据 | **TSEND** |
| 已有连接接收数据 | **TRCV** |
| UDP 发送 | **TUSEND** |
| UDP 接收 | **TURCV** |
| 发送邮件 | **TMAIL_C** |
| 检查连接状态 | **T_DIAG** |
| 复位异常连接 | **T_RESET** |

### ③ 计算/选型示例

- 已知：PLC 要和上位机进行 TCP 通信，同时要求程序简单、自动管理连接
- 公式/规则：自动连接收发优先选带 `_C` 的指令
- 操作：发送选 **TSEND_C**，接收选 **TRCV_C**
- 结论：比手动使用 TCON + TSEND/TRCV 更简洁

> [!tip] 记忆口诀
> **简单通信用带 C，复杂控制手动 TCON；UDP 用 U，查错用 DIAG。**
