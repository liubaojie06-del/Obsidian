## 01 `TM_MAIL` | 发送电子邮件

> [!info] 核心结论
> **TM_MAIL** 用于通过 PLC 远程服务功能发送电子邮件，适合报警通知、维护提醒和运行状态上报。

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">版本</strong><br>
分类版本：<strong>V1.9</strong><br>
指令版本：<strong>V1.4</strong>
</div>

### ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| 所属分类 | **远程服务** |
| 指令名称 | **TM_MAIL** |
| 功能描述 | **发送电子邮件** |
| 版本 | **V1.4** |
| 核心作用 | PLC 发送邮件通知 |
| 使用重点 | SMTP 服务器、收件人、主题、正文、网络权限需按现场网络校核 |

<div style="background:#F7F9FC;border-left:6px solid #FF6B35;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#FF6B35;">重点规则</strong><br>
邮件发送不是只写收件人，还要确认：<strong>网络通不通、服务器对不对、账号权限够不够</strong>。
</div>

### ② 参考表

| 使用场景 | 说明 |
|---|---|
| 故障报警邮件 | 设备严重故障时通知维护人员 |
| 保养提醒 | 运行时间到达后发送维护提示 |
| 状态上报 | 定时发送设备运行状态 |
| 生产通知 | 发送批次完成、产量统计等信息 |
| 远程维护 | 让维护人员提前获得故障信息 |
| 参数细节 | 需按实际 CPU、通信模块和邮件服务器校核 |

### ③ 计算/选型示例

- 已知：设备发生急停报警，需要通知维修人员
- 公式/规则：报警触发后发送邮件
- 操作：使用 **TM_MAIL**，配置收件人、邮件主题、正文内容和 SMTP 参数
- 结论：维修人员可收到设备报警邮件，便于快速响应

> [!tip] 记忆口诀
> **远程报警不靠喊，TM_MAIL 邮件传。**


## 02 远程服务 | 选型速记

> [!info] 核心结论
> 远程服务的核心是：**重要事件自动通知，邮件内容要写清设备、故障和处理方向。**

<div style="background:#F7F9FC;border-left:6px solid #27AE60;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#27AE60;">总口诀</strong><br>
有报警，发邮件；写清楚，谁故障；能追溯，才好修。
</div>

### ① 分类/公式/规则

| 类型 | 常用指令 |
|---|---|
| 邮件发送 | **TM_MAIL** |
| 触发条件 | 报警位 / 故障位 / 保养到期 / 批次完成 |
| 邮件内容 | 设备名、故障点、时间、处理建议 |
| 网络条件 | SMTP、账号权限、网络连通性 |

### ② 参考表

| 需求 | 推荐做法 |
|---|---|
| 设备故障自动通知 | 使用 **TM_MAIL** |
| 发送维护提醒 | 按运行时间或保养周期触发 |
| 发送生产完成通知 | 批次结束后触发 |
| 邮件无法发送 | 检查 SMTP、网络、账号、端口和权限 |
| 防止频繁轰炸 | 报警触发需加沿触发、延时或发送间隔 |

### ③ 计算/选型示例

- 已知：电机运行累计达到 `500 h` 后需要保养提醒
- 公式/规则：累计时间 ≥ 保养阈值时触发邮件
- 操作：运行时间到达后，用 **TM_MAIL** 发送保养提醒邮件
- 结论：维护人员可提前收到提醒，减少漏保养风险

> [!tip] 记忆口诀
> **邮件三要素：何时发、发给谁、说清啥。**
