## 01 `GET` | 从远程 CPU 读取数据

> [!info] 核心结论
> **GET** 用于从远程 CPU 主动读取数据，适合 PLC 之间的数据采集、状态读取和联机信息获取。

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">版本</strong><br>
分类版本：<strong>V1.3</strong><br>
指令版本：<strong>V1.3</strong>
</div>

### ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| 所属分类 | **S7 通信** |
| 指令名称 | **GET** |
| 功能描述 | **从远程 CPU 读取数据** |
| 版本 | **V1.3** |
| 核心作用 | 本地 CPU 主动读取远程 CPU 数据 |
| 使用重点 | 通信连接、远程地址、数据长度、访问权限需按实际项目校核 |

### ② 参考表

| 使用场景 | 说明 |
|---|---|
| 读取远程状态 | 读取另一台 CPU 的运行状态、报警状态 |
| 采集生产数据 | 从远程设备读取计数、温度、压力等数据 |
| 多 PLC 联动 | 一台 PLC 获取另一台 PLC 的工艺状态 |
| 数据监控 | 中央 PLC 汇总多个远程 CPU 数据 |
| 风险提醒 | 需确认远程 CPU 允许被访问 |

<div style="background:#F7F9FC;border-left:6px solid #27AE60;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#27AE60;">重点规则</strong><br>
GET 是<strong>主动读取</strong>：本地 CPU 去远程 CPU “拿数据”。
</div>

### ③ 计算/选型示例

- 已知：本地 CPU 需要读取远程 CPU 中的产量计数 `Remote_Count`
- 公式/规则：远程数据读取用 **GET**
- 操作：建立 S7 通信连接，配置远程 CPU 地址、读取区域和本地接收变量
- 结论：本地 CPU 可获得远程 CPU 的计数值，用于集中监控

> [!tip] 记忆口诀
> **GET 是去拿，本地 PLC 读远程。**


## 02 `PUT` | 向远程 CPU 写入数据

> [!info] 核心结论
> **PUT** 用于向远程 CPU 主动写入数据，适合 PLC 之间发送设定值、控制命令和状态同步。

<div style="background:#F7F9FC;border-left:6px solid #FF6B35;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#FF6B35;">版本</strong><br>
分类版本：<strong>V1.3</strong><br>
指令版本：<strong>V1.3</strong>
</div>

### ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| 所属分类 | **S7 通信** |
| 指令名称 | **PUT** |
| 功能描述 | **向远程 CPU 写入数据** |
| 版本 | **V1.3** |
| 核心作用 | 本地 CPU 主动写数据到远程 CPU |
| 使用重点 | 写入地址、数据类型、权限、安全联锁需按实际项目校核 |

### ② 参考表

| 使用场景 | 说明 |
|---|---|
| 写入设定值 | 向远程 CPU 下发速度、温度、压力设定 |
| 发送控制命令 | 远程启动、停止、复位等命令 |
| 状态同步 | 把本地状态写给另一台 PLC |
| 分布式控制 | 多台 CPU 之间交换控制数据 |
| 风险提醒 | 写入远程数据前必须确认安全条件 |

<div style="background:#F7F9FC;border-left:6px solid #9B59B6;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#9B59B6;">重点规则</strong><br>
PUT 是<strong>主动写入</strong>：本地 CPU 把数据“送到远程 CPU”。
</div>

### ③ 计算/选型示例

- 已知：本地 CPU 需要把目标速度 `Speed_Set = 1200 rpm` 写入远程 CPU
- 公式/规则：远程数据写入用 **PUT**
- 操作：建立 S7 通信连接，配置远程写入地址和本地发送变量
- 结论：远程 CPU 收到新的速度设定值，可用于后续控制

> [!tip] 记忆口诀
> **PUT 是去放，本地 PLC 写远程。**


## 03 S7 通信 | 选型速记

> [!info] 核心结论
> S7 通信的核心是：**读远程用 GET，写远程用 PUT；读写前先确认连接、地址、权限和安全条件。**

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">总口诀</strong><br>
GET 读远程，PUT 写远程；读写先建连接，地址权限要核清。
</div>

### ① 分类/公式/规则

| 类型 | 常用指令 |
|---|---|
| 从远程 CPU 读取数据 | **GET** |
| 向远程 CPU 写入数据 | **PUT** |

### ② 参考表

| 需求 | 推荐指令 |
|---|---|
| 读取远程 PLC 的状态位 | **GET** |
| 读取远程 PLC 的生产数据 | **GET** |
| 向远程 PLC 下发设定值 | **PUT** |
| 向远程 PLC 写入控制命令 | **PUT** |
| 多台 PLC 数据汇总 | 常用 **GET** |
| 多台 PLC 指令下发 | 常用 **PUT** |

### ③ 计算/选型示例

- 已知：中央 PLC 需要读取远程设备产量，并向远程设备下发目标速度
- 公式/规则：读取用 **GET**，写入用 **PUT**
- 操作：用 **GET** 读取远程产量数据；用 **PUT** 写入目标速度设定值
- 结论：这是典型的 S7 通信双向数据交换场景

> [!tip] 记忆口诀
> **读用 GET，写用 PUT；远程通信，先查连接再传输。**
