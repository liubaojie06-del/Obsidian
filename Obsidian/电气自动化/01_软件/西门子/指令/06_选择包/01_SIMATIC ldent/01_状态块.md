## 01 `Reader_Status` | 读出阅读器的状态信息

> [!info] 核心结论
> **Reader_Status** 用于读取 SIMATIC Ident 阅读器的状态信息，适合判断阅读器是否正常、是否忙碌、是否存在错误。

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">版本</strong><br>
指令版本：<strong>V4.5</strong>
</div>

### ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| 所属分类 | **状态块** |
| 指令名称 | **Reader_Status** |
| 功能描述 | **读出阅读器的状态信息** |
| 版本 | **V4.5** |
| 核心作用 | 获取阅读器运行/通信/错误状态 |
| 使用重点 | 状态码、错误码、忙信号需按实际阅读器手册校核 |

<div style="background:#F7F9FC;border-left:6px solid #FF6B35;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#FF6B35;">重点规则</strong><br>
读写失败别先怀疑标签，先看阅读器状态：<strong>是否在线、是否忙、是否报错</strong>。
</div>

### ② 参考表

| 状态方向 | 说明 |
|---|---|
| 在线状态 | 判断阅读器是否可通信 |
| 忙状态 | 判断当前是否正在执行读写任务 |
| 错误状态 | 判断是否存在设备或通信故障 |
| 诊断状态 | 辅助定位读写异常原因 |
| 参数细节 | 需按实际型号和厂家手册校核 |

### ③ 计算/选型示例

- 已知：PLC 执行 RFID 读取指令后没有得到数据
- 公式/规则：先检查阅读器当前状态
- 操作：使用 **Reader_Status** 读取阅读器状态信息
- 结论：可判断问题是阅读器未就绪、通信异常，还是当前任务忙碌

> [!tip] 记忆口诀
> **读写不成功，先查 Reader_Status。**


## 02 `Tag_Status` | 读出应答器的状态信息

> [!info] 核心结论
> **Tag_Status** 用于读取 RFID 应答器/标签的状态信息，适合判断标签是否存在、是否可读写、是否有异常。

<div style="background:#F7F9FC;border-left:6px solid #27AE60;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#27AE60;">版本</strong><br>
指令版本：<strong>V4.4</strong>
</div>

### ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| 所属分类 | **状态块** |
| 指令名称 | **Tag_Status** |
| 功能描述 | **读出应答器的状态信息** |
| 版本 | **V4.4** |
| 核心作用 | 获取标签/应答器状态 |
| 使用重点 | 标签类型、存储区、读写权限、状态码需按实际标签手册校核 |

<div style="background:#F7F9FC;border-left:6px solid #9B59B6;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#9B59B6;">重点规则</strong><br>
标签状态重点看：<strong>有没有标签、能不能读、能不能写、是否超出范围</strong>。
</div>

### ② 参考表

| 状态方向 | 说明 |
|---|---|
| 标签存在 | 判断识别区域内是否有应答器 |
| 标签类型 | 判断是否匹配当前读写器/项目 |
| 读写状态 | 判断是否允许读取或写入 |
| 存储区状态 | 判断地址、长度是否有效 |
| 异常状态 | 如读写失败、权限不允许、标签不匹配等 |

### ③ 计算/选型示例

- 已知：写入 RFID 标签时提示失败
- 公式/规则：先确认标签状态和写入条件
- 操作：使用 **Tag_Status** 读取应答器状态信息
- 结论：可判断是否标签未到位、标签不可写、地址错误或类型不匹配

> [!tip] 记忆口诀
> **标签能不能用，Tag_Status 先确认。**


## 03 状态块 | 选型速记

> [!info] 核心结论
> 状态块的核心是：**查阅读器用 Reader_Status，查标签/应答器用 Tag_Status。**

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">总口诀</strong><br>
设备状态看 Reader，标签状态看 Tag；读写异常先查状态，再做复位或重试。
</div>

### ① 分类/公式/规则

| 类型 | 常用指令 |
|---|---|
| 阅读器状态 | **Reader_Status** |
| 应答器/标签状态 | **Tag_Status** |

### ② 参考表

| 需求 | 推荐指令 |
|---|---|
| 判断阅读器是否在线 | **Reader_Status** |
| 判断阅读器是否忙碌 | **Reader_Status** |
| 判断阅读器是否故障 | **Reader_Status** |
| 判断是否有标签 | **Tag_Status** |
| 判断标签是否可读写 | **Tag_Status** |
| 判断标签类型是否匹配 | **Tag_Status** |
| 排查 RFID 读写异常 | **Reader_Status + Tag_Status** |

### ③ 计算/选型示例

- 已知：RFID 读写任务失败，需要判断原因
- 公式/规则：先查设备，再查标签
- 操作：先用 **Reader_Status** 查看阅读器是否正常；再用 **Tag_Status** 查看标签是否存在、可读写
- 结论：可快速区分是阅读器问题、标签问题，还是读写参数问题

> [!tip] 记忆口诀
> **先查 Reader 是否能干活，再查 Tag 是否能读写。**
