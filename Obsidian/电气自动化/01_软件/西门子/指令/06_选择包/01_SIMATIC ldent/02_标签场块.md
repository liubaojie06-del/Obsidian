## 01 `Write_Tagfield` | 写入标签场

> [!info] 核心结论
> **Write_Tagfield** 用于向标签场写入数据，适合批量写入 RFID 标签相关区域或指定标签场数据。

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">版本</strong><br>
指令版本：<strong>V1.3</strong>
</div>

### ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| 所属分类 | **标签场块** |
| 指令名称 | **Write_Tagfield** |
| 功能描述 | **写入标签场** |
| 版本 | **V1.3** |
| 核心作用 | 向标签场写入数据 |
| 使用重点 | 写入地址、长度、标签类型、写保护需按实际阅读器/标签手册校核 |

<div style="background:#F7F9FC;border-left:6px solid #FF6B35;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#FF6B35;">重点规则</strong><br>
写入标签场前，先确认：<strong>标签在位、区域可写、地址正确、长度不越界</strong>。
</div>

### ② 参考表

| 使用场景 | 说明 |
|---|---|
| 写入产品信息 | 产品编号、批次号、工序号 |
| 写入工艺状态 | 当前工序完成状态 |
| 写入检测结果 | 合格/不合格、测试值 |
| 写入追溯数据 | 生产时间、线体编号、设备编号 |
| 风险提醒 | 写错地址可能覆盖关键数据 |

### ③ 计算/选型示例

- 已知：产品检测完成后，需要把结果写入 RFID 标签场
- 公式/规则：向标签场写入数据用 **Write_Tagfield**
- 操作：确认标签在位后，设置写入区域、长度和写入内容
- 结论：检测结果被写入标签场，后续工位可读取追溯

> [!tip] 记忆口诀
> **写标签场先核对，地址长度别写错。**


## 02 `Read_Tagfield` | 读取标签场

> [!info] 核心结论
> **Read_Tagfield** 用于从标签场读取数据，适合读取 RFID 标签中的产品、工艺和追溯信息。

<div style="background:#F7F9FC;border-left:6px solid #27AE60;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#27AE60;">版本</strong><br>
指令版本：<strong>V1.3</strong>
</div>

### ① 分类/公式/规则

| 项目 | 内容 |
|---|---|
| 所属分类 | **标签场块** |
| 指令名称 | **Read_Tagfield** |
| 功能描述 | **读取标签场** |
| 版本 | **V1.3** |
| 核心作用 | 从标签场读取数据 |
| 使用重点 | 读取地址、长度、数据缓冲区、标签状态需按实际设备手册校核 |

<div style="background:#F7F9FC;border-left:6px solid #9B59B6;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#9B59B6;">重点规则</strong><br>
读取标签场前，先确认：<strong>标签存在、阅读器正常、读取范围正确</strong>。
</div>

### ② 参考表

| 使用场景 | 说明 |
|---|---|
| 读取产品编号 | 判断当前产品身份 |
| 读取批次信息 | 用于生产追溯 |
| 读取工艺状态 | 判断上一工序是否完成 |
| 读取检测结果 | 后续工位决定放行或剔除 |
| 异常排查 | 读取失败时配合 Reader_Status / Tag_Status |

### ③ 计算/选型示例

- 已知：产品进入工位后，需要读取标签场中的产品编号
- 公式/规则：从标签场读取数据用 **Read_Tagfield**
- 操作：设置读取区域、长度和接收缓冲区
- 结论：PLC 获得产品编号，可用于工艺选择和追溯判断

> [!tip] 记忆口诀
> **读标签场看身份，数据进来再判断。**


## 03 标签场块 | 选型速记

> [!info] 核心结论
> 标签场块的核心是：**读标签场用 Read_Tagfield，写标签场用 Write_Tagfield；读写前先确认标签和阅读器状态。**

<div style="background:#F7F9FC;border-left:6px solid #173B63;padding:12px;border-radius:12px;margin:10px 0;">
<strong style="color:#173B63;">总口诀</strong><br>
读用 Read_Tagfield，写用 Write_Tagfield；先查状态，再做读写。
</div>

### ① 分类/公式/规则

| 类型 | 常用指令 |
|---|---|
| 写入标签场 | **Write_Tagfield** |
| 读取标签场 | **Read_Tagfield** |
| 阅读器状态检查 | Reader_Status |
| 标签状态检查 | Tag_Status |

### ② 参考表

| 需求 | 推荐指令 |
|---|---|
| 向标签场写入数据 | **Write_Tagfield** |
| 从标签场读取数据 | **Read_Tagfield** |
| 判断阅读器是否正常 | Reader_Status |
| 判断标签是否可读写 | Tag_Status |
| 做产品追溯 | **Read_Tagfield + Write_Tagfield** |
| 读写失败排查 | Reader_Status + Tag_Status |

### ③ 计算/选型示例

- 已知：产品进站读取编号，出站写入检测结果
- 公式/规则：进站读取用 **Read_Tagfield**，出站写入用 **Write_Tagfield**
- 操作：进站读取产品信息；检测完成后写入结果数据
- 结论：这是典型的标签场追溯读写流程

> [!tip] 记忆口诀
> **进站读身份，出站写结果；读写标签场，状态先确认。**
