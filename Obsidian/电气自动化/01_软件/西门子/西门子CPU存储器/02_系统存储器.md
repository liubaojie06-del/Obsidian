# 01 系统存储器总览

> [!info] 核心结论
> **系统存储器** 是PLC CPU在运行过程中用于管理输入、输出、位存储、定时器、计数器、临时数据和系统状态的内部存储区域。  
> 简单理解：它是PLC运行时用于“存状态、存过程、存中间结果”的工作区域。

## ① 分类/公式/规则

> [!warning] 基本公式
> **系统存储器 = 输入映像区 + 输出映像区 + 位存储区 + 定时器 + 计数器 + 临时数据 + 系统状态数据**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **英文名称** | System Memory |
| **主要作用** | 保存PLC运行过程中的状态和中间数据 |
| **典型内容** | I区、Q区、M区、T、C、L区、系统状态 |
| **与工作存储器区别** | 工作存储器用于执行程序，系统存储器用于运行过程数据 |
| **与保持存储器关系** | 部分系统存储器区域可设置保持 |
| **访问方式** | 通过地址或变量访问 |
| **风险点** | 地址复用、误写输出、临时变量误用、保持配置错误 |

## ③ 计算/选型示例

- 已知：  
  PLC程序中使用M10.0作为电机运行中间状态。
- 公式：  
  **M10.0属于位存储区，是系统存储器的一部分**
- 计算：  
  程序可以读取和写入M10.0，用于逻辑判断。
- 结论：  
  系统存储器为PLC程序提供了大量运行状态和中间变量存储空间。

> [!tip] 记忆口诀
> 程序执行靠工作存储器，过程状态靠系统存储器。

# 02 系统存储器的作用

> [!info] 核心结论
> 系统存储器的作用是支持PLC扫描周期运行，保存输入状态、输出状态、中间标志、定时计数和临时运算结果。

## ① 分类/公式/规则

> [!warning] 运行公式
> **PLC扫描 = 读取输入 → 更新输入映像 → 执行程序 → 更新输出映像 → 刷新输出**

## ② 参考表

| 作用 | 说明 |
|---|---|
| **保存输入状态** | 记录现场输入信号 |
| **保存输出状态** | 记录程序计算后的输出结果 |
| **保存中间标志** | M区用于中间逻辑 |
| **支持定时器** | 保存定时状态和当前值 |
| **支持计数器** | 保存计数状态和当前值 |
| **保存临时变量** | 块执行时使用 |
| **保存系统状态** | CPU、通讯、诊断等运行状态 |

## ③ 计算/选型示例

- 已知：  
  输入I0.0接启动按钮，输出Q0.0接电机接触器。
- 公式：  
  **I0.0状态 → 程序逻辑 → Q0.0状态**
- 计算：  
  CPU先读取I0.0，再根据程序决定是否写Q0.0。
- 结论：  
  系统存储器贯穿PLC输入、逻辑处理和输出刷新全过程。

> [!tip] 记忆口诀
> 输入进系统存储器，程序算逻辑，输出再送现场。

# 03 输入映像区 I

> [!info] 核心结论
> **输入映像区** 用于保存PLC在扫描周期开始时采集到的现场输入信号状态。

## ① 分类/公式/规则

> [!warning] 输入映像规则
> **输入映像区 = 扫描开始时现场输入的快照**

## ② 参考表

| 地址 | 含义 |
|---|---|
| **I0.0** | 第0字节第0位输入 |
| **I0.1** | 第0字节第1位输入 |
| **IB0** | 输入字节0 |
| **IW0** | 输入字0 |
| **ID0** | 输入双字0 |
| **数字量输入** | 按位读取ON/OFF |
| **模拟量输入** | 常通过字或双字读取数值 |

## ③ 计算/选型示例

- 已知：  
  启动按钮接入I0.0，按钮按下时I0.0=1。
- 公式：  
  **按钮按下 → 输入映像I0.0=1**
- 计算：  
  程序读取I0.0为1后，可以执行启动逻辑。
- 结论：  
  输入映像区是PLC程序判断现场输入状态的基础。

> [!tip] 记忆口诀
> I区看输入，现场信号先进I区。

# 04 输出映像区 Q

> [!info] 核心结论
> **输出映像区** 用于保存PLC程序计算后的输出状态，扫描周期结束时再刷新到实际输出端子。

## ① 分类/公式/规则

> [!warning] 输出映像规则
> **程序写Q区 → 扫描结束刷新到现场输出**

## ② 参考表

| 地址 | 含义 |
|---|---|
| **Q0.0** | 第0字节第0位输出 |
| **Q0.1** | 第0字节第1位输出 |
| **QB0** | 输出字节0 |
| **QW0** | 输出字0 |
| **QD0** | 输出双字0 |
| **数字量输出** | 控制继电器、电磁阀、指示灯 |
| **模拟量输出** | 控制变频器、阀门开度、设定信号 |

## ③ 计算/选型示例

- 已知：  
  电磁阀接在Q0.2。
- 公式：  
  **程序条件满足 → Q0.2=1 → 电磁阀得电**
- 计算：  
  CPU扫描结束时把Q0.2状态刷新到实际输出模块。
- 结论：  
  输出映像区决定PLC最终给现场执行元件的输出状态。

> [!tip] 记忆口诀
> Q区管输出，写Q就是准备控制现场。

# 05 位存储区 M

> [!info] 核心结论
> **M区** 是PLC常用的中间标志存储区，适合保存逻辑状态、中间条件、步骤状态和辅助位。

## ① 分类/公式/规则

> [!warning] M区规则
> **M区适合中间逻辑，不建议随意当成所有数据的仓库**

## ② 参考表

| 地址 | 用途 |
|---|---|
| **M0.0** | 位标志 |
| **MB0** | 存储字节 |
| **MW0** | 存储字 |
| **MD0** | 存储双字 |
| **中间状态** | 保存逻辑判断结果 |
| **步骤标志** | 顺序控制步骤 |
| **保持标志** | 部分M区可配置保持 |

## ③ 计算/选型示例

- 已知：  
  M10.0用于表示“自动运行允许”。
- 公式：  
  **启动条件满足 + 无报警 + 自动模式 → M10.0=1**
- 计算：  
  程序后续逻辑可以直接判断M10.0。
- 结论：  
  M区适合保存多个条件综合后的中间结果。

> [!tip] 记忆口诀
> M区是中间标志区，用来帮程序记状态。

# 06 定时器 T

> [!info] 核心结论
> **定时器** 是系统存储器中用于时间控制的对象，常用于延时启动、延时停止、动作超时和周期脉冲。

## ① 分类/公式/规则

> [!warning] 定时公式
> **定时完成 = 当前累计时间 ≥ 设定时间**

## ② 参考表

| 定时应用 | 示例 |
|---|---|
| **延时启动** | 按下启动后延时3秒启动电机 |
| **延时停止** | 停止后风机延时运行 |
| **动作超时** | 气缸5秒未到位报警 |
| **脉冲定时** | 产生固定宽度脉冲 |
| **周期定时** | 周期触发采样或刷新 |
| **防抖延时** | 输入稳定后才确认 |
| **工艺时间** | 保压、冷却、加热时间 |

## ③ 计算/选型示例

- 已知：  
  气缸伸出命令发出后，要求3秒内必须到位。
- 公式：  
  **伸出命令ON且3秒未到位 → 超时报警**
- 计算：  
  定时器达到3秒时，如果伸出到位信号仍为0，则报警。
- 结论：  
  定时器常用于动作超时保护。

> [!tip] 记忆口诀
> 要等多久，用定时器；超过多久没完成，用超时报警。

# 07 计数器 C

> [!info] 核心结论
> **计数器** 用于记录事件发生次数，例如产品计数、动作次数、报警次数和循环次数。

## ① 分类/公式/规则

> [!warning] 计数公式
> **当前计数值 = 原计数值 + 有效脉冲次数**

## ② 参考表

| 计数应用 | 示例 |
|---|---|
| **产品计数** | 每检测到一个产品加1 |
| **动作次数** | 气缸动作次数 |
| **报警次数** | 某故障发生次数 |
| **循环次数** | 自动循环次数 |
| **批次数量** | 当前批次生产数量 |
| **维护计数** | 到次数提示保养 |
| **分拣计数** | OK/NG分别计数 |

## ③ 计算/选型示例

- 已知：  
  传感器每检测到一个产品产生一次脉冲，当前计数为120。
- 公式：  
  **新计数 = 120 + 1**
- 计算：  
  新计数 = **121**
- 结论：  
  计数器适合处理离散事件的次数统计。

> [!tip] 记忆口诀
> 时间用定时器，次数用计数器。

# 08 局部数据区 L

> [!info] 核心结论
> **局部数据区** 用于保存程序块执行过程中的临时变量和局部数据，块执行结束后通常不适合用来保存长期状态。

## ① 分类/公式/规则

> [!warning] L区规则
> **局部数据只在块调用过程中有效，不适合保存设备长期状态**

## ② 参考表

| 局部数据 | 说明 |
|---|---|
| **临时变量** | 中间计算结果 |
| **局部布尔值** | 块内判断条件 |
| **局部整数** | 块内计算变量 |
| **局部Real** | 工程量换算中间值 |
| **循环变量** | For循环索引 |
| **临时结构** | 块内短期使用 |
| **不保持状态** | 不应用于长期记忆 |

## ③ 计算/选型示例

- 已知：  
  某FC中使用临时变量TempValue计算压力工程值。
- 公式：  
  **TempValue = 原始值 × 比例系数**
- 计算：  
  计算完成后把结果写入DB或输出变量。
- 结论：  
  临时变量适合计算过程，不适合保存最终状态。

> [!tip] 记忆口诀
> L区只临时用，状态别放L区。

# 09 I0.0:P 输入外设直接访问

> [!info] 核心结论
> **I0.0:P** 表示直接读取实际输入模块上的 I0.0 信号，而不是读取普通输入映像区 I0.0。  
> 简单理解：**I0.0 是扫描周期开始时的输入快照，I0.0:P 是直接看当前模块输入状态。**

## ① 分类/公式/规则

> [!warning] 输入直接访问公式
> **I0.0:P = 直接读取外设输入点 I0.0 的当前状态**

## ② 参考表

| 写法 | 含义 |
|---|---|
| **I0.0** | 读取输入映像区中的 I0.0 |
| **I0.0:P** | 直接读取实际输入模块上的 I0.0 |
| **:P** | Peripheral，表示外设直接访问 |
| **适合场景** | 特殊诊断、需要读取当前输入状态 |
| **不适合场景** | 普通按钮、常规联锁逻辑大量使用 |
| **注意点** | 直接访问不等于高速计数功能 |
| **风险点** | 与普通扫描映像混用时，逻辑分析更复杂 |

## ③ 计算/选型示例

- 已知：  
  启动按钮接在 I0.0。
- 公式：  
  **I0.0 = 本周期输入映像值**  
  **I0.0:P = 当前实际输入模块值**
- 计算：  
  如果输入信号在扫描周期中途变化，I0.0 可能还保持本周期开始时的值，而 I0.0:P 可能读到当前变化后的值。
- 结论：  
  常规逻辑优先用 I0.0；特殊场景才考虑 I0.0:P。

> [!tip] 记忆口诀
> I区看映像，I:P看外设当前值。

# 10 Q0.0:P 输出外设直接访问

> [!info] 核心结论
> **Q0.0:P** 表示直接写入实际输出模块上的 Q0.0 输出点，而不是只写普通输出映像区 Q0.0。  
> 简单理解：**Q0.0 通常等扫描结束刷新输出，Q0.0:P 是直接对输出外设写入。**

## ① 分类/公式/规则

> [!warning] 输出直接访问公式
> **Q0.0:P = 直接写入外设输出点 Q0.0**

## ② 参考表

| 写法 | 含义 |
|---|---|
| **Q0.0** | 写入输出映像区，通常扫描结束刷新到输出模块 |
| **Q0.0:P** | 直接写入实际输出模块 Q0.0 |
| **:P** | Peripheral，表示外设直接访问 |
| **适合场景** | 特殊输出测试、特殊模块访问、特殊控制需求 |
| **不适合场景** | 普通设备控制、HMI直接控制输出 |
| **注意点** | 可能被后续普通 Q 区逻辑覆盖 |
| **风险点** | 直接写输出容易绕过正常联锁逻辑 |

## ③ 计算/选型示例

- 已知：  
  电磁阀接在 Q0.0。
- 公式：  
  **Q0.0 = 程序写输出映像，周期末刷新**  
  **Q0.0:P = 直接写实际输出模块**
- 计算：  
  如果程序某处写 Q0.0:P=1，但后续逻辑又把 Q0.0=0，最终输出状态可能被普通输出逻辑覆盖。
- 结论：  
  常规控制建议写命令变量，由PLC联锁后控制 Q0.0；不要随意用 Q0.0:P 直接控制现场输出。

> [!tip] 记忆口诀
> Q区按周期输出，Q:P直接打到外设；能不用就少用，安全联锁不能绕。
# 11 系统存储器与扫描周期

> [!info] 核心结论
> PLC扫描周期中，系统存储器不断被读取和更新，输入映像、输出映像和中间标志共同参与程序执行。

## ① 分类/公式/规则

> [!warning] 扫描流程
> **读取输入 → 执行程序 → 更新输出 → 系统维护**

## ② 参考表

| 扫描阶段 | 系统存储器作用 |
|---|---|
| **读取输入** | 更新I区输入映像 |
| **执行程序** | 读取I/M/DB，写M/Q/DB |
| **定时计数** | 更新T/C相关状态 |
| **输出处理** | 根据Q区刷新物理输出 |
| **通讯处理** | 更新通讯相关状态 |
| **诊断处理** | 记录系统异常信息 |
| **进入下一周期** | 继续重复扫描 |

## ③ 计算/选型示例

- 已知：  
  输入信号在一个扫描周期中途变化。
- 公式：  
  **程序通常读取的是本周期输入映像**
- 计算：  
  信号变化可能在下一次输入刷新时才被程序看到。
- 结论：  
  理解输入映像区有助于分析快速信号漏检问题。

> [!tip] 记忆口诀
> PLC按周期看世界，不是每一瞬间都直接看现场。

# 12 系统存储器与直接外设访问

> [!info] 核心结论
> 一些PLC支持直接访问外设输入输出，用于绕过过程映像区读取或写入实际模块状态，但使用时要谨慎。

## ① 分类/公式/规则

> [!warning] 访问区别
> **过程映像访问 = 访问I/Q映像区**  
> **外设直接访问 = 直接访问实际I/O模块**

## ② 参考表

| 访问方式 | 特点 |
|---|---|
| **I区访问** | 读取输入映像，稳定常用 |
| **Q区访问** | 写输出映像，扫描结束刷新 |
| **外设输入访问** | 直接读模块当前值 |
| **外设输出访问** | 直接写模块输出 |
| **适用场景** | 高速、特殊诊断、特殊模块 |
| **风险** | 与普通扫描逻辑混用可能难排查 |
| **建议** | 常规逻辑优先使用过程映像 |

## ③ 计算/选型示例

- 已知：  
  模拟量模块需要读取最新值，而不是等待普通映像刷新。
- 公式：  
  **需要最新外设数据 → 可考虑直接外设访问**
- 计算：  
  程序直接读取模块当前值。
- 结论：  
  直接外设访问适合特殊场景，普通控制逻辑不建议滥用。

> [!tip] 记忆口诀
> 常规用映像，特殊才直读外设。

# 13 系统存储器与保持

> [!info] 核心结论
> 系统存储器中的部分区域可以配置保持，例如部分M区、计数值或DB变量，但并不是所有系统存储器数据都会自动保持。

## ① 分类/公式/规则

> [!warning] 保持规则
> **需要断电保留的数据，必须明确配置保持**

## ② 参考表

| 区域 | 是否适合保持 |
|---|---|
| **累计产量M区** | 可保持，但建议结构化管理 |
| **当前配方号** | 建议保持 |
| **运行小时** | 建议保持 |
| **启动命令位** | 不建议保持 |
| **输出控制位** | 不建议保持 |
| **临时变量L区** | 不适合保持 |
| **动作中间标志** | 视逻辑决定 |

## ③ 计算/选型示例

- 已知：  
  M20.0是启动命令，断电前为1。
- 公式：  
  **启动命令保持 → 上电误启动风险**
- 计算：  
  上电后若仍为1，可能导致设备误动作。
- 结论：  
  命令位一般不应设置保持，累计值和参数才适合保持。

> [!tip] 记忆口诀
> 该记住的保持，不该记住的上电清零。

# 14 系统存储器地址规划

> [!info] 核心结论
> 系统存储器地址必须规划清楚，避免M区、定时器、计数器、输入输出地址被重复使用或含义混乱。

## ① 分类/公式/规则

> [!warning] 地址规划公式

# 18 系统存储器常见错误

> [!info] 核心结论
> 系统存储器常见错误包括地址重复、数据类型错误、命令位保持、HMI直接写输出、临时变量保存状态和输入输出映像理解错误。

## ① 分类/公式/规则

> [!warning] 错误公式
> **系统存储器异常 = 地址冲突 + 类型错误 + 保持错误 + 访问方式错误**

## ② 参考表

| 错误 | 后果 |
|---|---|
| **M区重复使用** | 逻辑互相覆盖 |
| **数据类型不匹配** | 数值显示异常 |
| **命令位设置保持** | 上电误动作 |
| **HMI直接写Q点** | 绕过PLC安全联锁 |
| **临时变量存状态** | 状态丢失 |
| **未规划定时器** | 定时逻辑混乱 |
| **输入映像误解** | 快速信号漏判 |

- 公式：  
  **MW10包含MB10和MB11，M11.0属于MW10的一部分**
- 计算：  
  写MW10可能会改变M11.0。
- 结论：  
  位、字节、字、双字地址重叠时必须特别小心。

> [!tip] 记忆口诀
> M位、MB、MW、MD可能重叠，地址规划不能乱。

# 20 系统存储器快速检查表

> [!info] 核心结论
> 系统存储器检查的目标是确保输入输出正确、中间标志清晰、定时计数可靠、地址不冲突、HMI访问安全。

> ① 检查I区输入地址是否与现场点表一致

> ⑧ 检查HMI是否直接写输出或危险地址

| **T定时器** | 时间逻辑可靠 |
| **C计数器** | 次数统计准确 |
| **L区** | 只作临时使用 |
| **保持设置** | 该保留才保留 |

## ③ 计算/选型示例

- 已知：  
  一个项目中HMI写M区命令，PLC再根据安全条件控制Q区输出。

# 21 I0.0:P 输入外设直接访问

> [!info] 核心结论
> **I0.0:P** 表示直接读取实际输入模块上的 I0.0 信号，而不是读取普通输入映像区 I0.0。  
> 简单理解：**I0.0 是扫描周期开始时的输入快照，I0.0:P 是直接看当前模块输入状态。**

## ① 分类/公式/规则

> [!warning] 输入直接访问公式
> **I0.0:P = 直接读取外设输入点 I0.0 的当前状态**

## ② 参考表

| 写法 | 含义 |
|---|---|
| **I0.0** | 读取输入映像区中的 I0.0 |
| **I0.0:P** | 直接读取实际输入模块上的 I0.0 |
| **:P** | Peripheral，表示外设直接访问 |
| **适合场景** | 特殊诊断、需要读取当前输入状态 |
| **不适合场景** | 普通按钮、常规联锁逻辑大量使用 |
| **注意点** | 直接访问不等于高速计数功能 |
| **风险点** | 与普通扫描映像混用时，逻辑分析更复杂 |

## ③ 计算/选型示例

- 已知：  
  启动按钮接在 I0.0。
- 公式：  
  **I0.0 = 本周期输入映像值**  
  **I0.0:P = 当前实际输入模块值**
- 计算：  
  如果输入信号在扫描周期中途变化，I0.0 可能还保持本周期开始时的值，而 I0.0:P 可能读到当前变化后的值。
- 结论：  
  常规逻辑优先用 I0.0；特殊场景才考虑 I0.0:P。

> [!tip] 记忆口诀
> I区看映像，I:P看外设当前值。

# 22 Q0.0:P 输出外设直接访问

> [!info] 核心结论
> **Q0.0:P** 表示直接写入实际输出模块上的 Q0.0 输出点，而不是只写普通输出映像区 Q0.0。  
> 简单理解：**Q0.0 通常等扫描结束刷新输出，Q0.0:P 是直接对输出外设写入。**

## ① 分类/公式/规则

> [!warning] 输出直接访问公式
> **Q0.0:P = 直接写入外设输出点 Q0.0**

## ② 参考表

| 写法 | 含义 |
|---|---|
| **Q0.0** | 写入输出映像区，通常扫描结束刷新到输出模块 |
| **Q0.0:P** | 直接写入实际输出模块 Q0.0 |
| **:P** | Peripheral，表示外设直接访问 |
| **适合场景** | 特殊输出测试、特殊模块访问、特殊控制需求 |
| **不适合场景** | 普通设备控制、HMI直接控制输出 |
| **注意点** | 可能被后续普通 Q 区逻辑覆盖 |
| **风险点** | 直接写输出容易绕过正常联锁逻辑 |

## ③ 计算/选型示例

- 已知：  
  电磁阀接在 Q0.0。
- 公式：  
  **Q0.0 = 程序写输出映像，周期末刷新**  
  **Q0.0:P = 直接写实际输出模块**
- 计算：  
  如果程序某处写 Q0.0:P=1，但后续逻辑又把 Q0.0=0，最终输出状态可能被普通输出逻辑覆盖。
- 结论：  
  常规控制建议写命令变量，由PLC联锁后控制 Q0.0；不要随意用 Q0.0:P 直接控制现场输出。

> [!tip] 记忆口诀
> Q区按周期输出，Q:P直接打到外设；能不用就少用，安全联锁不能绕。
