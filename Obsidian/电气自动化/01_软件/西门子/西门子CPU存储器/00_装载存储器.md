# 01 装载存储器总览

> [!info] 核心结论
> **装载存储器** 是PLC中用于保存用户程序、数据块、硬件组态和系统相关数据的存储区域。  
> 简单理解：它像PLC里的“程序仓库”，程序下载到PLC后，首先保存在装载存储器中。

## ① 分类/公式/规则

> [!warning] 基本公式
> **装载存储器 = 保存用户程序 + 保存数据块 + 保存组态信息 + 供CPU加载运行**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **英文名称** | Load Memory |
| **主要作用** | 保存下载到PLC的工程数据 |
| **保存内容** | 程序块、数据块、硬件组态、系统数据 |
| **掉电保持** | 通常具备非易失保存能力 |
| **运行关系** | 程序运行时会加载到工作存储器执行 |
| **常见载体** | CPU内部存储区、存储卡、工程存储介质 |
| **典型对象** | OB、FC、FB、DB、系统数据等 |

## ③ 计算/选型示例

- 已知：  
  PLC程序、数据块和硬件组态下载到CPU。
- 公式：  
  **下载工程 → 装载存储器保存 → 工作存储器执行**
- 计算：  
  如果装载存储器容量不足，工程可能无法完整下载。
- 结论：  
  装载存储器容量必须大于工程实际占用容量。

> [!tip] 记忆口诀
> 装载存储器负责“存程序”，工作存储器负责“跑程序”。

# 02 装载存储器的作用

> [!info] 核心结论
> 装载存储器的作用是长期保存PLC工程内容，让CPU在启动或运行时能够从中加载程序和数据。

## ① 分类/公式/规则

> [!warning] 作用公式
> **PLC能运行 = 装载存储器中有有效程序 + CPU能加载执行**

## ② 参考表

| 作用 | 说明 |
|---|---|
| **保存用户程序** | 保存OB、FC、FB等逻辑块 |
| **保存数据块** | 保存DB初始值、结构和相关数据 |
| **保存硬件组态** | 保存模块、网络、设备配置 |
| **支持掉电恢复** | 重新上电后程序仍存在 |
| **支持程序下载** | 工程师下载程序的目标区域 |
| **支持程序启动** | CPU启动时从中读取需要的数据 |
| **支持工程维护** | 在线修改、下载、更新程序 |

## ③ 计算/选型示例

- 已知：  
  PLC断电后重新上电，程序仍然存在。
- 公式：  
  **程序不丢失 = 装载存储器保存有效程序**
- 计算：  
  CPU从装载存储器读取程序并恢复运行。
- 结论：  
  装载存储器是PLC掉电后程序不丢失的关键。

> [!tip] 记忆口诀
> PLC断电不忘程序，靠的就是装载存储器。

# 03 装载存储器与工作存储器

> [!info] 核心结论
> 装载存储器负责保存程序，工作存储器负责CPU实际运行程序。两者不能混淆。

## ① 分类/公式/规则

> [!warning] 关系公式
> **装载存储器保存程序，工作存储器执行程序**

## ② 参考表

| 对比项 | 装载存储器 | 工作存储器 |
|---|---|---|
| 作用 | 保存工程和程序 | 执行程序 |
| 英文 | Load Memory | Work Memory |
| 类比 | 硬盘/仓库 | 内存/运行区 |
| 掉电后 | 通常保留 | 通常不作为长期保存区 |
| 容量不足影响 | 无法下载或保存工程 | 程序无法运行或运行受限 |
| 存放内容 | 程序块、数据块、组态 | 正在执行的代码和数据 |
| 关注点 | 下载容量 | 运行容量 |

## ③ 计算/选型示例

- 已知：  
  工程可以下载到PLC，但CPU提示工作存储器不足。
- 公式：  
  **装载够用 ≠ 工作够用**
- 计算：  
  工程能存进去，但运行时需要的工作空间不够。
- 结论：  
  选型时要同时看装载存储器和工作存储器。

> [!tip] 记忆口诀
> 装载存储器像硬盘，工作存储器像运行内存。

# 04 装载存储器保存什么

> [!info] 核心结论
> 装载存储器主要保存PLC运行需要的工程对象，包括程序块、数据块、系统数据和硬件配置。

## ① 分类/公式/规则

> [!warning] 保存内容公式
> **装载内容 = 程序块 + 数据块 + 系统数据 + 硬件组态**

## ② 参考表

| 内容 | 示例 |
|---|---|
| **组织块** | OB1、循环中断OB、启动OB |
| **功能块** | FB电机控制、FB阀门控制 |
| **函数** | FC计算、FC转换、FC报警处理 |
| **数据块** | 全局DB、实例DB |
| **系统数据** | CPU和模块相关系统配置 |
| **硬件组态** | I/O模块、通讯模块、设备参数 |
| **通信配置** | HMI连接、PLC网络相关配置 |

## ③ 计算/选型示例

- 已知：  
  项目包含大量FB、DB和硬件组态。
- 公式：  
  **工程对象越多 → 装载存储器占用越大**
- 计算：  
  程序块、数据块、组态越复杂，占用空间越高。
- 结论：  
  大型项目应提前核算装载存储器容量。

> [!tip] 记忆口诀
> 程序、数据、组态，都是装载存储器的重要内容。

# 05 装载存储器容量

> [!info] 核心结论
> 装载存储器容量决定PLC能保存多大的工程。容量不足时，程序可能无法下载或下载不完整。

## ① 分类/公式/规则

> [!warning] 容量判断公式
> **装载存储器剩余容量 = 总容量 - 已占用容量**

## ② 参考表

| 影响容量的内容 | 说明 |
|---|---|
| **程序块数量** | OB、FC、FB越多，占用越大 |
| **数据块数量** | DB越多，占用越大 |
| **数据块初始值** | 大数组、大结构会增加占用 |
| **硬件组态** | 模块、通讯配置越多，占用越大 |
| **诊断信息** | 部分系统数据也占空间 |
| **库对象** | 复杂标准块可能占用较大 |
| **冗余无用块** | 不清理会浪费空间 |

## ③ 计算/选型示例

- 已知：  
  装载存储器总容量为4MB，工程已占用3.2MB。
- 公式：  
  **剩余容量 = 4 - 3.2**
- 计算：  
  剩余容量 = **0.8MB**
- 结论：  
  后续扩展程序时要注意剩余空间是否足够。

> [!tip] 记忆口诀
> 程序下载前，先看装载空间够不够。

# 06 装载存储器不足

> [!info] 核心结论
> 装载存储器不足会导致程序无法下载、块无法更新、工程无法完整保存或CPU提示存储空间不够。

## ① 分类/公式/规则

> [!warning] 不足判断公式
> **工程占用容量 > 装载存储器可用容量 → 下载失败**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **下载失败** | 装载存储器空间不足 |
| **块无法装载** | 单个块或整体工程过大 |
| **提示存储器不足** | CPU装载容量不够 |
| **更新程序失败** | 新版本占用更大 |
| **大型DB下载失败** | 数据块数组过大 |
| **运行异常** | 下载不完整或块不一致 |
| **无法扩展功能** | 选型时容量预留不足 |

## ③ 计算/选型示例

- 已知：  
  CPU装载存储器剩余200KB，新增加的程序块需要350KB。
- 公式：  
  **350KB > 200KB → 空间不足**
- 计算：  
  新程序无法正常下载。
- 结论：  
  需要优化程序、删除无用块，或更换更大容量CPU/存储卡。

> [!tip] 记忆口诀
> 装载空间不够，程序再对也下不进去。

# 07 装载存储器与存储卡

> [!info] 核心结论
> 在很多PLC系统中，装载存储器可能由CPU内部存储器或专用存储卡承担。存储卡损坏、容量不足或不匹配都会影响程序下载和运行。

## ① 分类/公式/规则

> [!warning] 存储卡规则
> **存储卡 = 装载存储器的重要载体之一**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **内部装载存储器** | CPU内部自带的装载空间 |
| **外部存储卡** | 用于保存程序和系统数据 |
| **容量选择** | 要大于工程占用容量 |
| **型号匹配** | 要符合PLC型号要求 |
| **损坏风险** | 存储卡损坏会影响程序保存 |
| **更换CPU** | 可能需要转移存储卡或重新下载 |
| **备份作用** | 存储卡可作为工程运行载体 |

## ③ 计算/选型示例

- 已知：  
  工程占用8MB，存储卡容量4MB。
- 公式：  
  **工程容量 > 存储卡容量**
- 计算：  
  8MB > 4MB，存储卡容量不足。
- 结论：  
  应选择容量更大的存储卡。

> [!tip] 记忆口诀
> 存储卡太小，PLC装不下完整工程。

# 08 装载存储器与下载

> [!info] 核心结论
> 工程师把程序下载到PLC时，程序会写入装载存储器，再由CPU根据需要加载到运行区域。

## ① 分类/公式/规则

> [!warning] 下载链路
> **工程软件 → PLC连接 → 装载存储器 → 工作存储器执行**

## ② 参考表

| 下载内容 | 说明 |
|---|---|
| **完整下载** | 下载整个工程或所有块 |
| **增量下载** | 只下载修改过的块 |
| **硬件组态下载** | 更新模块和通讯配置 |
| **程序块下载** | 更新OB、FC、FB |
| **数据块下载** | 更新DB结构或初始值 |
| **在线修改** | 局部修改并下载 |
| **重新编译下载** | 大改后重新生成并下载 |

## ③ 计算/选型示例

- 已知：  
  修改了一个FB并下载。
- 公式：  
  **修改块 → 写入装载存储器 → CPU更新执行内容**
- 计算：  
  新FB保存到装载存储器，CPU运行新逻辑。
- 结论：  
  下载操作本质上是在更新PLC的装载内容。

> [!tip] 记忆口诀
> 下载不是直接“跑程序”，而是先把程序装进PLC。

# 09 装载存储器与启动

> [!info] 核心结论
> PLC启动时，需要从装载存储器中读取有效程序和配置，才能进入正常运行状态。

## ① 分类/公式/规则

> [!warning] 启动公式
> **CPU启动 = 检查装载内容 + 加载必要程序 + 初始化数据 + 进入运行**

## ② 参考表

| 启动相关项 | 说明 |
|---|---|
| **程序有效性** | 检查装载的程序是否完整 |
| **硬件组态** | 检查模块配置是否匹配 |
| **启动OB** | 执行启动相关逻辑 |
| **数据初始化** | 根据DB设置初始化 |
| **保持数据恢复** | 恢复需要保持的数据 |
| **故障诊断** | 程序或组态错误会报警 |
| **进入RUN** | 正常加载后进入运行 |

## ③ 计算/选型示例

- 已知：  
  CPU上电后无法进入RUN。
- 公式：  
  **无法启动 = 程序无效 / 组态错误 / 装载数据异常**
- 计算：  
  需要检查装载存储器中的程序和硬件配置是否正确。
- 结论：  
  启动失败不一定是硬件坏，也可能是装载内容有问题。

> [!tip] 记忆口诀
> PLC启动要先读程序，程序不对就跑不起来。

# 10 装载存储器与数据块

> [!info] 核心结论
> 数据块会占用装载存储器，尤其是大数组、大结构、配方数据和复杂实例DB会明显增加占用空间。

## ① 分类/公式/规则

> [!warning] DB容量公式
> **DB占用 ≈ 数据结构大小 + 初始值 + 系统管理开销**

## ② 参考表

| DB类型 | 说明 |
|---|---|
| **全局DB** | 保存全局参数、状态、配方 |
| **实例DB** | 保存FB实例数据 |
| **大数组DB** | 容量占用较明显 |
| **配方DB** | 保存产品工艺参数 |
| **报警DB** | 保存报警位、故障码 |
| **历史缓冲DB** | 可能占用较大 |
| **临时数据** | 通常不建议长期放大容量DB |

## ③ 计算/选型示例

- 已知：  
  一个Real数组有1000个元素，每个Real占4字节。
- 公式：  
  **容量 = 1000 × 4**
- 计算：  
  容量 = **4000字节 ≈ 4KB**
- 结论：  
  大数组会增加DB占用，也会增加装载存储器压力。

> [!tip] 记忆口诀
> DB越大，装载占用越大；数组越多，空间越要算。

# 11 装载存储器与保持数据

> [!info] 核心结论
> 装载存储器与保持数据不是同一个概念。装载存储器保存程序和初始数据，保持数据用于断电后保留运行中变化的关键值。

## ① 分类/公式/规则

> [!warning] 区分公式
> **装载存储器保存工程，保持存储器保存运行中需要保留的变量值**

## ② 参考表

| 对比项 | 装载存储器 | 保持存储器 |
|---|---|---|
| 主要作用 | 保存程序和工程 | 保存断电保持变量 |
| 保存对象 | 程序块、DB初始值、组态 | 计数、参数、状态 |
| 是否运行变化 | 通常保存工程内容 | 运行中会变化 |
| 掉电后 | 程序不丢失 | 保持变量不丢失 |
| 典型数据 | FB、FC、DB、系统数据 | 产量计数、配方号、累计值 |
| 风险 | 容量不足无法下载 | 未设置保持导致数据丢失 |

## ③ 计算/选型示例

- 已知：  
  设备断电后，累计产量丢失，但程序还在。
- 公式：  
  **程序还在 = 装载存储器正常**  
  **产量丢失 = 保持数据未配置或失效**
- 计算：  
  问题不在装载存储器，而在保持变量设置。
- 结论：  
  不要把装载存储器和保持存储器混为一谈。

> [!tip] 记忆口诀
> 程序不丢靠装载，数值不丢靠保持。

# 12 装载存储器与程序优化

> [!info] 核心结论
> 当装载存储器占用过高时，可以通过清理无用块、优化DB结构、减少重复程序、封装公共逻辑来降低占用。

## ① 分类/公式/规则

> [!warning] 优化公式
> **占用降低 = 删除无用内容 + 合理封装 + 减少重复 + 优化数据结构**

## ② 参考表

| 优化方式 | 说明 |
|---|---|
| **删除无用块** | 清理旧版本FC、FB、DB |
| **减少重复逻辑** | 共用函数或功能块 |
| **优化DB结构** | 避免过大无用数组 |
| **清理测试变量** | 删除调试遗留数据 |
| **检查库对象** | 删除未使用的大型库块 |
| **分模块管理** | 避免重复下载无关模块 |
| **压缩工程资源** | 减少不必要资源占用 |

## ③ 计算/选型示例

- 已知：  
  项目中保留了多个旧版本FB：FB_Motor_V1、V2、V3。
- 公式：  
  **未使用旧块 → 可清理空间**
- 计算：  
  若只使用V3，则V1、V2可删除或归档到工程备份中。
- 结论：  
  清理无用块可以释放装载存储器。

> [!tip] 记忆口诀
> PLC里不要长期堆旧块，备份放电脑，运行放必要内容。

# 13 装载存储器与CPU选型

> [!info] 核心结论
> CPU选型时不能只看I/O点数和扫描速度，也要看装载存储器和工作存储器是否满足项目规模。

## ① 分类/公式/规则

> [!warning] 选型公式
> **CPU容量需求 = 当前工程占用 + 未来扩展余量**

## ② 参考表

| 选型考虑 | 说明 |
|---|---|
| **程序规模** | 程序块数量和复杂度 |
| **数据规模** | DB数量、数组、配方 |
| **通讯功能** | HMI、SCADA、网关、MES |
| **报警数量** | 报警位、故障码、诊断数据 |
| **配方数量** | 产品越多，数据越大 |
| **未来扩展** | 预留20%~50%空间更稳妥 |
| **存储卡需求** | 是否需要更大容量存储卡 |

## ③ 计算/选型示例

- 已知：  
  当前工程预计占用装载存储器3MB，未来可能增加50%。
- 公式：  
  **预计最终容量 = 3 × 1.5**
- 计算：  
  最终约为 **4.5MB**
- 结论：  
  选型时装载存储器容量应至少大于4.5MB，并留安全余量。

> [!tip] 记忆口诀
> CPU选型别只看点数，程序空间也要看。

# 14 装载存储器与在线修改

> [!info] 核心结论
> 在线修改程序时，新块或修改后的块需要写入装载存储器。如果空间不足，在线修改可能失败。

## ① 分类/公式/规则

> [!warning] 在线修改规则
> **在线修改成功 = 通讯正常 + CPU允许 + 装载空间足够 + 块一致性正确**

## ② 参考表

| 在线修改项 | 影响 |
|---|---|
| **修改FC/FB** | 更新程序块 |
| **修改DB结构** | 可能影响实例和数据 |
| **修改硬件组态** | 可能需要停机或重启 |
| **新增块** | 增加装载占用 |
| **删除块** | 释放部分空间 |
| **下载块不一致** | 可能导致运行异常 |
| **在线备份** | 修改前建议上传或保存版本 |

## ③ 计算/选型示例

- 已知：  
  在线新增一个功能块需要100KB，装载存储器剩余60KB。
- 公式：  
  **100KB > 60KB → 在线下载失败**
- 计算：  
  新功能块无法写入PLC。
- 结论：  
  在线扩展程序前也要检查装载存储器剩余容量。

> [!tip] 记忆口诀
> 在线修改也要占空间，不是随便加块。

# 15 装载存储器与备份

> [!info] 核心结论
> 装载存储器中保存的是PLC运行所需内容，但工程备份仍应保存在电脑、服务器或版本库中，不能只依赖PLC里的程序。

## ① 分类/公式/规则

> [!warning] 备份规则
> **可靠备份 = 离线工程文件 + PLC在线程序 + 版本说明 + 参数/配方备份**

## ② 参考表

| 备份对象 | 说明 |
|---|---|
| **PLC工程文件** | 完整源工程 |
| **在线程序** | CPU中的实际程序 |
| **硬件组态** | 模块和通讯配置 |
| **HMI工程** | 画面和变量 |
| **配方数据** | 工艺参数 |
| **版本说明** | 修改记录 |
| **存储卡镜像** | 特殊项目可备份存储卡内容 |

## ③ 计算/选型示例

- 已知：  
  现场PLC还能运行，但电脑工程文件丢失。
- 公式：  
  **PLC有程序 ≠ 有完整工程管理资料**
- 计算：  
  有些注释、符号、版本说明、HMI配套文件可能不完整。
- 结论：  
  不能只把PLC装载存储器当作唯一备份。

> [!tip] 记忆口诀
> PLC里有程序，不代表工程资料完整。

# 16 装载存储器常见故障

> [!info] 核心结论
> 装载存储器相关故障常见于容量不足、存储卡异常、程序损坏、下载中断、版本不一致和硬件组态不匹配。

## ① 分类/公式/规则

> [!warning] 故障公式
> **装载异常 = 容量不足 + 存储介质异常 + 程序不一致 + 下载失败**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **无法下载程序** | 装载存储器不足 |
| **CPU无法启动** | 装载程序无效或组态错误 |
| **存储卡报警** | 卡损坏、未插好、型号不匹配 |
| **下载后程序异常** | 块版本不一致 |
| **断电后程序丢失** | 存储介质异常或下载未保存 |
| **新增DB失败** | DB过大或空间不足 |
| **在线修改失败** | 剩余装载空间不足 |

## ③ 计算/选型示例

- 已知：  
  程序下载到一半中断，CPU后续启动异常。
- 公式：  
  **下载中断 → 装载内容可能不完整**
- 计算：  
  需要重新完整下载工程，确认程序和组态一致。
- 结论：  
  下载过程中应保证通讯和电源稳定。

> [!tip] 记忆口诀
> 下载时断电断线，最容易造成程序不完整。

# 17 装载存储器排查步骤

> [!info] 核心结论
> 排查装载存储器问题时，应按容量、存储介质、程序一致性、下载状态、CPU诊断信息逐项检查。

## ① 分类/公式/规则

> [!warning] 排查公式
> **装载问题排查 = 看容量 + 看存储卡 + 看诊断 + 看下载记录 + 看程序一致性**

## ② 参考表

| 步骤 | 检查内容 |
|---|---|
| **1** | 查看CPU诊断缓冲区 |
| **2** | 查看装载存储器已用/剩余容量 |
| **3** | 检查存储卡是否正常 |
| **4** | 检查工程是否完整下载 |
| **5** | 检查程序块是否一致 |
| **6** | 检查DB是否过大 |
| **7** | 检查硬件组态是否匹配 |
| **8** | 必要时重新编译并完整下载 |

## ③ 计算/选型示例

- 已知：  
  CPU提示装载存储器不足。
- 公式：  
  **处理顺序 = 清理无用块 → 优化程序 → 更换更大容量硬件**
- 计算：  
  先删除未使用块和测试DB，仍不足再考虑更换CPU或存储卡。
- 结论：  
  不要一上来就换硬件，先确认是否有无用占用。

> [!tip] 记忆口诀
> 先查容量，再查介质；先清无用，再换硬件。

# 18 装载存储器快速检查表

> [!info] 核心结论
> 装载存储器检查的目标是确保PLC能完整保存工程、正常启动、可靠运行，并为后续扩展留有空间。

## ① 分类/公式/规则

> [!note] 十步检查法
> ① 检查装载存储器总容量和剩余容量  
> ② 检查工作存储器是否也足够  
> ③ 检查程序块、数据块、组态是否完整下载  
> ④ 检查大型DB、数组、配方数据是否占用过大  
> ⑤ 检查存储卡型号和容量是否匹配  
> ⑥ 检查CPU诊断缓冲区是否有存储器报警  
> ⑦ 检查在线修改后程序块是否一致  
> ⑧ 检查断电重启后程序是否正常保留  
> ⑨ 检查工程文件、参数和配方是否独立备份  
> ⑩ 选型时预留未来扩展容量

> [!warning] 最终检查公式
> **合格装载存储器 = 容量足够 + 程序完整 + 存储可靠 + 启动正常 + 备份完整**

## ② 参考表

| 检查项 | 目标 |
|---|---|
| **容量** | 工程能完整下载 |
| **存储卡** | 介质可靠、型号匹配 |
| **程序块** | 程序一致、不缺块 |
| **数据块** | DB结构和容量合理 |
| **启动** | 上电后能正常运行 |
| **诊断** | 无存储器相关报警 |
| **备份** | 工程不只存在PLC中 |
| **扩展** | 后续功能有余量 |

## ③ 计算/选型示例

- 已知：  
  当前工程占用装载存储器70%，未来还要增加视觉通讯、配方和报警功能。
- 公式：  
  **当前占用70% + 后续扩展 → 容量风险**
- 计算：  
  剩余30%可能不足以支撑后续扩展。
- 结论：  
  建议提前评估程序增长，必要时选择更大容量CPU或存储卡。

> [!tip] 选型口诀
> 装载看下载，工作看运行；程序看容量，DB看大小；存储卡要匹配，工程备份不能少。
