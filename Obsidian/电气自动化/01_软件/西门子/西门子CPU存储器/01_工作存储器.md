# 01 工作存储器总览

> [!info] 核心结论
> **工作存储器** 是PLC CPU实际运行用户程序时使用的存储区域。  
> 简单理解：它像PLC的“运行内存”，程序真正执行时，需要占用工作存储器。

## ① 分类/公式/规则

> [!warning] 基本公式
> **工作存储器 = CPU运行程序所需的代码空间 + 数据空间**

## ② 参考表

| 项目 | 说明 |
|---|---|
| **英文名称** | Work Memory |
| **主要作用** | 支持CPU执行用户程序 |
| **运行对象** | OB、FC、FB、DB运行时数据 |
| **与装载存储器关系** | 程序从装载存储器加载后在工作存储器中运行 |
| **容量不足影响** | 程序无法运行、下载失败或CPU报错 |
| **关注重点** | 程序运行容量，而不是单纯下载容量 |
| **典型场景** | 大程序、大DB、多FB实例、大量通讯数据 |

## ③ 计算/选型示例

- 已知：  
  PLC工程可以下载，但CPU提示工作存储器不足。
- 公式：  
  **能下载 ≠ 能运行**
- 计算：  
  装载存储器能保存工程，但工作存储器不足以执行程序。
- 结论：  
  PLC选型时必须同时看装载存储器和工作存储器。

> [!tip] 记忆口诀
> 装载存储器负责“存”，工作存储器负责“跑”。

# 02 工作存储器的作用

> [!info] 核心结论
> 工作存储器用于保存CPU当前运行所需的程序代码和运行数据，是PLC循环扫描和逻辑执行的基础。

## ① 分类/公式/规则

> [!warning] 运行公式
> **CPU运行 = 从工作存储器读取程序 + 处理数据 + 输出结果**

## ② 参考表

| 作用 | 说明 |
|---|---|
| **执行程序** | CPU执行OB、FC、FB逻辑 |
| **处理运行数据** | 保存运行过程中需要的数据 |
| **支持数据块运行** | DB、实例DB运行时占用空间 |
| **支持中断程序** | 中断OB执行也需要工作存储器 |
| **支持通讯处理** | 通讯数据和缓冲区可能占用资源 |
| **支持复杂算法** | 计算、数组、结构体会增加占用 |
| **决定运行能力** | 程序越复杂，对工作存储器要求越高 |

## ③ 计算/选型示例

- 已知：  
  程序中增加大量运动控制和通讯功能后，CPU内存报警。
- 公式：  
  **程序复杂度增加 → 工作存储器占用增加**
- 计算：  
  新增FB、实例DB、通讯缓存都会占用工作存储器。
- 结论：  
  功能扩展前要检查工作存储器余量。

> [!tip] 记忆口诀
> 程序越复杂，运行内存越要够。

# 03 工作存储器与装载存储器

> [!info] 核心结论
> 装载存储器保存工程，工作存储器运行工程。装载够用不代表工作够用。

## ① 分类/公式/规则

> [!warning] 对比公式
> **装载存储器 = 程序仓库**  
> **工作存储器 = 程序运行区**

## ② 参考表

| 对比项 | 装载存储器 | 工作存储器 |
|---|---|---|
| 英文 | Load Memory | Work Memory |
| 类比 | 硬盘、仓库 | 内存、运行区 |
| 作用 | 保存工程内容 | 执行程序逻辑 |
| 掉电影响 | 通常保留程序 | 不是主要长期保存区 |
| 容量不足 | 无法保存或下载工程 | 无法运行或运行受限 |
| 关注阶段 | 下载、保存、启动 | 运行、扫描、执行 |
| 选型重点 | 工程能否装下 | 程序能否跑得动 |

## ③ 计算/选型示例

- 已知：  
  装载存储器剩余很多，但工作存储器不足。
- 公式：  
  **存得下 ≠ 跑得动**
- 计算：  
  程序文件可以保存，但运行时占用空间超过CPU能力。
- 结论：  
  不能只看装载存储器容量。

> [!tip] 记忆口诀
> 硬盘大不代表内存够，PLC也是一样。

# 04 工作存储器保存什么

> [!info] 核心结论
> 工作存储器保存CPU当前运行需要的程序代码、数据块运行数据、实例数据和部分系统运行数据。

## ① 分类/公式/规则

> [!warning] 占用公式
> **工作存储器占用 = 程序代码占用 + 数据块占用 + 实例数据占用 + 系统开销**

## ② 参考表

| 内容 | 示例 |
|---|---|
| **程序代码** | OB、FC、FB运行代码 |
| **全局数据块** | 全局DB运行数据 |
| **实例数据块** | FB对应的实例DB |
| **临时数据** | 块调用时的局部临时变量 |
| **系统数据** | CPU运行管理相关数据 |
| **通讯数据** | 通讯缓冲、连接相关数据 |
| **复杂结构数据** | 数组、结构体、配方数据等 |

## ③ 计算/选型示例

- 已知：  
  程序中有大量FB，每个FB都有实例DB。
- 公式：  
  **FB数量多 + 实例DB多 → 工作存储器占用增加**
- 计算：  
  每增加一个FB实例，都可能增加实例数据占用。
- 结论：  
  大量设备标准块复用时，要关注实例DB容量。

> [!tip] 记忆口诀
> FB越多，实例越多，工作存储器压力越大。

# 05 工作存储器容量

> [!info] 核心结论
> 工作存储器容量决定CPU能运行多大规模的程序和数据。容量不足时，即使工程能下载，也可能无法正常运行。

## ① 分类/公式/规则

> [!warning] 容量公式
> **工作存储器剩余容量 = 工作存储器总容量 - 当前运行占用容量**

## ② 参考表

| 影响容量的因素 | 说明 |
|---|---|
| **程序块数量** | OB、FC、FB越多，占用越高 |
| **数据块大小** | 大DB、大数组占用明显 |
| **实例DB数量** | 多FB实例会增加占用 |
| **通讯连接数量** | HMI、SCADA、网关、MES连接会增加资源需求 |
| **报警和诊断数据** | 故障码、状态字、诊断缓冲相关数据 |
| **配方数据** | 产品越多、参数越多，占用越大 |
| **复杂计算** | 大量结构体、数组、临时变量会增加运行占用 |

## ③ 计算/选型示例

- 已知：  
  工作存储器总容量为1MB，当前程序运行占用750KB。
- 公式：  
  **剩余容量 = 1024KB - 750KB**
- 计算：  
  剩余容量 = **274KB**
- 结论：  
  后续扩展功能时，应确认274KB是否足够。

> [!tip] 记忆口诀
> 工作存储器要留余量，不能刚好用满。

# 06 工作存储器不足

> [!info] 核心结论
> 工作存储器不足会导致程序无法下载、CPU无法进入RUN、块无法执行、在线修改失败或诊断缓冲区报存储器错误。

## ① 分类/公式/规则

> [!warning] 不足判断公式
> **运行所需工作存储器 > CPU可用工作存储器 → 运行失败或下载失败**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **CPU无法进入RUN** | 工作存储器不足或程序不一致 |
| **下载失败** | 新程序运行占用超过CPU能力 |
| **在线修改失败** | 修改后的块需要更多运行空间 |
| **程序运行异常** | 数据块或实例数据超出容量 |
| **新增功能失败** | 扩展功能占用过大 |
| **诊断报警** | CPU提示工作存储器不足 |
| **扫描周期变长** | 程序复杂度和数据处理量增加 |

## ③ 计算/选型示例

- 已知：  
  剩余工作存储器100KB，新功能运行需要180KB。
- 公式：  
  **180KB > 100KB → 工作存储器不足**
- 计算：  
  新功能无法可靠运行。
- 结论：  
  需要优化程序、减少数据占用，或更换更大容量CPU。

> [!tip] 记忆口诀
> 工作空间不够，程序就跑不动。

# 07 工作存储器与扫描周期

> [!info] 核心结论
> 工作存储器容量本身不等于扫描速度，但程序规模、数据量和运行逻辑会同时影响工作存储器占用和扫描周期。

## ① 分类/公式/规则

> [!warning] 扫描负载公式
> **扫描负载 = 程序指令量 + 数据处理量 + 通讯处理量 + 中断任务量**

## ② 参考表

| 影响项 | 说明 |
|---|---|
| **程序逻辑复杂** | 扫描时间增加 |
| **大数组循环** | 数据处理时间增加 |
| **多FB调用** | 代码和实例数据增加 |
| **通讯数据多** | 通讯负载增加 |
| **报警判断多** | 状态扫描和判断增加 |
| **配方处理多** | 参数搬运和比较增加 |
| **中断频繁** | 主循环可能被打断 |

## ③ 计算/选型示例

- 已知：  
  程序中每个扫描周期都循环处理5000个数组元素。
- 公式：  
  **循环次数越多 → 扫描时间越长**
- 计算：  
  如果每次扫描都处理全部数据，CPU负载会明显增加。
- 结论：  
  大数组处理应分周期、分批或按需执行。

> [!tip] 记忆口诀
> 工作存储器管能不能跑，扫描周期管跑得快不快。

# 08 工作存储器与数据块

> [!info] 核心结论
> 数据块是工作存储器占用的重要来源，尤其是大数组、配方DB、实例DB和历史缓冲DB。

## ① 分类/公式/规则

> [!warning] DB占用公式
> **DB占用 ≈ 变量数量 × 数据类型大小 + 结构开销**

## ② 参考表

| DB类型 | 对工作存储器的影响 |
|---|---|
| **全局DB** | 保存全局运行数据 |
| **实例DB** | 每个FB实例都可能占用空间 |
| **配方DB** | 产品参数越多，占用越大 |
| **报警DB** | 报警位、故障码、状态字 |
| **缓存DB** | 批量数据、通讯缓冲 |
| **数组DB** | 元素越多，占用越大 |
| **结构体DB** | 结构复杂时占用增加 |

## ③ 计算/选型示例

- 已知：  
  一个DInt数组有2000个元素，每个DInt占4字节。
- 公式：  
  **容量 = 2000 × 4**
- 计算：  
  容量 = **8000字节 ≈ 8KB**
- 结论：  
  单个数组看似不大，但多个大数组叠加会明显占用工作存储器。

> [!tip] 记忆口诀
> 大DB、大数组，是工作存储器占用大户。

# 09 工作存储器与实例DB

> [!info] 核心结论
> FB每创建一个实例，通常会产生对应实例数据。大量设备使用标准FB时，实例DB占用会快速增加。

## ① 分类/公式/规则

> [!warning] 实例占用公式
> **实例DB总占用 = 单个实例DB大小 × 实例数量**

## ② 参考表

| 场景 | 说明 |
|---|---|
| **电机FB** | 每台电机一个实例 |
| **阀门FB** | 每个阀门一个实例 |
| **气缸FB** | 每个气缸一个实例 |
| **伺服轴FB** | 每个轴一个实例 |
| **报警FB** | 每个设备或报警组一个实例 |
| **通讯FB** | 每个通讯对象一个实例 |
| **工位FB** | 每个工位一个实例 |

## ③ 计算/选型示例

- 已知：  
  一个电机FB实例DB大小为300字节，项目有80台电机。
- 公式：  
  **总占用 = 300 × 80**
- 计算：  
  总占用 = **24000字节 ≈ 24KB**
- 结论：  
  标准块实例多时，需要核算总实例数据占用。

> [!tip] 记忆口诀
> 一个实例不大，几十上百个实例就要算容量。

# 10 工作存储器与临时变量

> [!info] 核心结论
> 临时变量在块执行时使用，虽然通常不长期保存，但大量临时数据、复杂结构和深层调用会增加运行资源压力。

## ① 分类/公式/规则

> [!warning] 临时数据规则
> **临时变量只为执行服务，不适合保存长期状态**

## ② 参考表

| 临时变量类型 | 注意 |
|---|---|
| **局部计算变量** | 适合临时计算 |
| **循环变量** | 用于循环索引 |
| **中间结果** | 用后即丢 |
| **临时数组** | 大数组会增加负担 |
| **结构体临时变量** | 复杂结构占用较高 |
| **多层调用临时数据** | 调用层级深时占用增加 |
| **状态保持变量** | 不应放临时变量，应放DB或保持变量 |

## ③ 计算/选型示例

- 已知：  
  某FC中定义了一个1000个Real的临时数组。
- 公式：  
  **容量 = 1000 × 4**
- 计算：  
  临时数组约占 **4KB**
- 结论：  
  大型临时数组应谨慎使用，必要时改为合理的数据块或分批处理。

> [!tip] 记忆口诀
> 临时变量别存状态，大临时数组要谨慎。

# 11 工作存储器与保持存储器

> [!info] 核心结论
> 工作存储器用于程序运行，保持存储器用于断电后保留关键变量值。两者作用不同。

## ① 分类/公式/规则

> [!warning] 区分公式
> **工作存储器负责运行，保持存储器负责断电保留**

## ② 参考表

| 对比项 | 工作存储器 | 保持存储器 |
|---|---|---|
| 作用 | 执行程序和处理数据 | 断电保持关键变量 |
| 英文 | Work Memory | Retentive Memory |
| 典型数据 | 运行代码、DB运行数据 | 累计值、配方号、参数 |
| 掉电后 | 不作为主要保存依据 | 需要保持的数据不丢 |
| 关注点 | 程序能否运行 | 数据断电是否保留 |
| 容量不足影响 | 程序无法运行 | 保持数据不完整 |
| 配置重点 | 程序规模和运行数据 | 哪些变量需要保持 |

## ③ 计算/选型示例

- 已知：  
  PLC断电后程序正常，但累计产量归零。
- 公式：  
  **程序正常 = 工作/装载正常**  
  **产量归零 = 保持配置问题**
- 计算：  
  应检查累计产量变量是否设置为保持。
- 结论：  
  运行内存正常不代表保持数据正常。

> [!tip] 记忆口诀
> 程序能跑看工作，数据不丢看保持。

# 12 工作存储器与CPU选型

> [!info] 核心结论
> CPU选型时，工作存储器必须满足当前程序运行需求，并预留未来扩展空间。

## ① 分类/公式/规则

> [!warning] 选型公式
> **工作存储器需求 = 当前程序运行占用 + 未来扩展余量**

## ② 参考表

| 选型考虑 | 说明 |
|---|---|
| **程序规模** | 控制逻辑越多，占用越大 |
| **设备数量** | 电机、阀门、气缸越多，实例越多 |
| **通讯数量** | HMI、SCADA、网关、MES越多，通讯资源越多 |
| **配方数量** | 产品和参数越多，占用越大 |
| **报警数量** | 报警位和诊断信息越多，占用越高 |
| **未来扩展** | 建议预留20%~50% |
| **扫描周期要求** | 高速设备要同时考虑CPU性能 |

## ③ 计算/选型示例

- 已知：  
  当前程序预计占用工作存储器500KB，未来扩展预计增加40%。
- 公式：  
  **预计需求 = 500 × 1.4**
- 计算：  
  预计需求 = **700KB**
- 结论：  
  CPU工作存储器应至少大于700KB，并留安全余量。

> [!tip] 记忆口诀
> CPU选型不只看I/O点，也要看运行内存够不够。

# 13 工作存储器与程序优化

> [!info] 核心结论
> 当工作存储器占用过高时，可以通过减少重复代码、优化数据结构、清理无用DB、合理封装和分批处理来降低占用。

## ① 分类/公式/规则

> [!warning] 优化公式
> **占用降低 = 少重复 + 少大数组 + 少无用块 + 合理结构化**

## ② 参考表

| 优化方式 | 说明 |
|---|---|
| **删除无用块** | 清理旧版本FC、FB、DB |
| **减少重复代码** | 共用FB/FC |
| **优化DB结构** | 删除不用变量，压缩结构 |
| **避免超大数组** | 必要时分区、分批处理 |
| **减少实例数量** | 合理合并相同逻辑 |
| **按需执行算法** | 不要每个扫描周期执行重计算 |
| **减少无用通讯变量** | 只采集需要的数据 |

## ③ 计算/选型示例

- 已知：  
  程序中有5个几乎相同的FC分别处理相同逻辑。
- 公式：  
  **重复逻辑 → 封装复用**
- 计算：  
  可合并成一个通用FC，减少重复代码。
- 结论：  
  复用通用块有助于降低程序占用并提高维护性。

> [!tip] 记忆口诀
> 重复代码越多，内存浪费越多，维护也越难。

# 14 工作存储器与在线修改

> [!info] 核心结论
> 在线修改程序时，修改后的块和运行数据需要占用工作存储器。剩余空间不足时，在线修改可能失败。

## ① 分类/公式/规则

> [!warning] 在线修改公式
> **在线修改成功 = 工作存储器够 + 装载存储器够 + 块一致性正确**

## ② 参考表

| 在线修改内容 | 影响 |
|---|---|
| **新增FB** | 增加代码和实例占用 |
| **新增DB** | 增加数据占用 |
| **扩大DB结构** | 增加运行数据空间 |
| **新增通讯变量** | 增加通讯和变量处理负担 |
| **新增报警逻辑** | 增加判断和数据占用 |
| **修改结构体** | 可能影响相关DB |
| **大改程序** | 可能需要完整下载和停机 |

## ③ 计算/选型示例

- 已知：  
  在线新增一个配方DB需要120KB，工作存储器剩余80KB。
- 公式：  
  **120KB > 80KB → 在线修改失败**
- 计算：  
  剩余工作存储器不足以支持新增DB。
- 结论：  
  在线修改前应检查剩余工作存储器。

> [!tip] 记忆口诀
> 在线加功能，也要有运行空间。

# 15 工作存储器与通讯

> [!info] 核心结论
> HMI、SCADA、网关、MES、驱动器等通讯功能会增加CPU运行负担，也可能占用一定工作资源。

## ① 分类/公式/规则

> [!warning] 通讯负载公式
> **通讯负载 = 连接数量 × 变量数量 × 刷新频率**

## ② 参考表

| 通讯对象 | 影响 |
|---|---|
| **HMI** | 读取状态、写入命令 |
| **SCADA** | 大量变量采集 |
| **工业网关** | 协议转换和数据采集 |
| **MES系统** | 生产数据上传下载 |
| **变频器** | 状态、频率、故障码 |
| **伺服驱动器** | 位置、速度、报警 |
| **远程I/O** | 输入输出刷新 |

## ③ 计算/选型示例

- 已知：  
  SCADA每秒读取1000个变量。
- 公式：  
  **读取频率高 + 变量多 → 通讯负载高**
- 计算：  
  CPU需要处理大量通讯请求和变量更新。
- 结论：  
  大量通讯变量应分组、分周期、按需采集。

> [!tip] 记忆口诀
> 变量读得越多越快，PLC通讯压力越大。

# 16 工作存储器常见故障

> [!info] 核心结论
> 工作存储器相关故障常见于容量不足、DB过大、实例过多、在线修改失败、程序无法进入RUN和扫描周期异常增加。

## ① 分类/公式/规则

> [!warning] 故障公式
> **工作存储器异常 = 程序过大 + 数据过大 + 实例过多 + 运行资源不足**

## ② 参考表

| 现象 | 可能原因 |
|---|---|
| **CPU无法RUN** | 工作存储器不足 |
| **下载新程序失败** | 新程序运行占用过大 |
| **在线修改失败** | 剩余工作空间不足 |
| **扫描周期变长** | 程序和数据处理负载增加 |
| **DB下载异常** | DB结构过大 |
| **新增设备失败** | 实例DB过多 |
| **诊断提示内存不足** | 工作存储器容量不足 |

## ③ 计算/选型示例

- 已知：  
  增加50个气缸控制实例后，CPU提示内存不足。
- 公式：  
  **新增实例占用 = 单实例占用 × 新增数量**
- 计算：  
  若单实例占用500字节，50个实例约占25KB。
- 结论：  
  大量新增设备对象前要计算实例占用。

> [!tip] 记忆口诀
> 新增设备不只是加I/O，也是在加程序和内存占用。

# 17 工作存储器排查步骤

> [!info] 核心结论
> 排查工作存储器问题时，应先看CPU诊断和容量占用，再检查大DB、实例DB、重复块和新增功能。

## ① 分类/公式/规则

> [!warning] 排查公式
> **工作存储器排查 = 看诊断 + 看占用 + 查大DB + 查实例 + 查新增功能**

## ② 参考表

| 步骤 | 检查内容 |
|---|---|
| **1** | 查看CPU诊断缓冲区 |
| **2** | 查看工作存储器总容量和剩余容量 |
| **3** | 检查最近新增的程序块 |
| **4** | 检查大型DB和数组 |
| **5** | 检查FB实例数量 |
| **6** | 删除无用块和测试DB |
| **7** | 优化重复逻辑和数据结构 |
| **8** | 必要时更换更大容量CPU |

## ③ 计算/选型示例

- 已知：  
  旧程序正常，新增配方功能后内存报警。
- 公式：  
  **故障出现在新增后 → 优先查新增功能占用**
- 计算：  
  配方DB、配方处理FB、HMI通讯变量都可能增加工作存储器占用。
- 结论：  
  内存问题要重点关注最近新增内容。

> [!tip] 记忆口诀
> 以前正常，新增后异常，先查新增部分。

# 18 工作存储器快速检查表

> [!info] 核心结论
> 工作存储器检查的目标是确保CPU有足够空间执行程序、处理数据、支持通讯，并为后续扩展留有余量。

## ① 分类/公式/规则

> [!note] 十步检查法
> ① 检查工作存储器总容量和剩余容量  
> ② 同时检查装载存储器是否足够  
> ③ 检查程序块数量和大小  
> ④ 检查全局DB、实例DB和大数组  
> ⑤ 检查通讯连接和变量数量  
> ⑥ 检查报警、配方、趋势相关数据占用  
> ⑦ 检查在线修改后块是否一致  
> ⑧ 检查CPU诊断缓冲区是否有内存报警  
> ⑨ 优化无用块、重复逻辑和过大数据结构  
> ⑩ CPU选型时预留未来扩展容量

> [!warning] 最终检查公式
> **合格工作存储器 = 容量足够 + 数据合理 + 程序可运行 + 扩展有余量 + 诊断无报警**

## ② 参考表

| 检查项 | 目标 |
|---|---|
| **容量** | 程序能运行 |
| **DB大小** | 数据不过大 |
| **实例数量** | 标准块复用不过量 |
| **通讯变量** | 通讯负载合理 |
| **扫描周期** | 运行不拖慢 |
| **诊断信息** | 无内存相关报警 |
| **优化空间** | 能清理无用内容 |
| **扩展余量** | 后续功能可增加 |

## ③ 计算/选型示例

- 已知：  
  当前工作存储器已占用85%，后续还要增加配方、MES通讯和更多报警。
- 公式：  
  **占用85% + 后续扩展 → 高风险**
- 计算：  
  剩余15%可能不足以支撑新增功能。
- 结论：  
  建议先优化程序，若仍不足，应考虑更大容量CPU。

> [!tip] 选型口诀
> 装载看能不能下载，工作看能不能运行；DB看大小，FB看实例；通讯看变量，扩展看余量。
