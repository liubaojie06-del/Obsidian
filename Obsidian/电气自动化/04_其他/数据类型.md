# 01 数据类型总览

> [!info] 核心结论
> **数据类型** 决定PLC如何存储、解释和运算变量。  
> 简单理解：同样一串二进制数据，如果数据类型不同，PLC解释出来的含义就不同。

## ① 分类/公式/规则

> [!warning] 基本公式
> **变量 = 变量名 + 数据类型 + 地址/存储位置 + 初始值 + 注释**

## ② 参考表

| 数据类型类别 | 常见类型 |
|---|---|
| **位类型** | Bool |
| **整数类型** | SInt、Int、DInt、LInt |
| **无符号整数** | USInt、UInt、UDInt、ULInt |
| **浮点数** | Real、LReal |
| **位串类型** | Byte、Word、DWord、LWord |
| **字符/字符串** | Char、WChar、String、WString |
| **时间日期** | Time、Date、Time_Of_Day、Date_And_Time、DTL |
| **复合类型** | Array、Struct、UDT |

## ③ 计算/选型示例

- 已知：  
  传感器只有“有信号/无信号”两种状态。
- 公式：  
  **两种状态 → Bool**
- 计算：  
  0表示无信号，1表示有信号。
- 结论：  
  开关量、到位信号、报警位、命令位通常用Bool。

> [!tip] 记忆口诀
> 开关用Bool，计数用Int/DInt，小数用Real，状态字用Word，文本用String。

# 02 Bool 布尔型

> [!info] 核心结论
> **Bool** 只有两个状态：0和1，常用于开关量、命令、反馈、报警、模式位和条件判断。

## ① 分类/公式/规则

> [!warning] Bool规则
> **Bool = 0 / 1 = False / True**

## ② 参考表

| Bool变量 | 用途 |
|---|---|
| **StartCmd** | 启动命令 |
| **StopCmd** | 停止命令 |
| **RunFB** | 运行反馈 |
| **Alarm** | 报警状态 |
| **AutoMode** | 自动模式 |
| **DoorClosed** | 安全门关闭 |
| **SensorOK** | 传感器有效 |

## ③ 计算/选型示例

- 已知：  
  安全门关闭信号只有开和关。
- 公式：  
  **两态信号 → Bool**
- 计算：  
  DoorClosed=1表示安全门已关闭。
- 结论：  
  安全门、光电、按钮、到位开关都适合Bool。

> [!tip] 记忆口诀
> 只有是和否，就用Bool。

# 03 Byte / Word / DWord / LWord 位串类型

> [!info] 核心结论
> **Byte、Word、DWord、LWord** 常用于状态字、报警字、位组合、通讯数据和掩码操作。

## ① 分类/公式/规则

> [!warning] 位数公式
> **Byte=8位，Word=16位，DWord=32位，LWord=64位**

## ② 参考表

| 类型 | 位数 | 常见用途 |
|---|---:|---|
| **Byte** | 8位 | 小状态组、通讯字节 |
| **Word** | 16位 | 报警字、状态字 |
| **DWord** | 32位 | 大状态组、通讯数据 |
| **LWord** | 64位 | 大型位组合 |
| **Bit操作** | 按位判断 | AlarmWord.0、StatusWord.3 |
| **掩码操作** | AND/OR/XOR | 提取或设置某些位 |
| **通讯映射** | Modbus寄存器 | 常用Word或DWord |

## ③ 计算/选型示例

- 已知：  
  一个报警字有16个报警位。
- 公式：  
  **1个Word = 16个Bool位**
- 计算：  
  AlarmWord.0~AlarmWord.15 可表示16条报警。
- 结论：  
  报警位很多时，可以用Word集中管理。

> [!tip] 记忆口诀
> 单个位用Bool，一组位用Word。

# 04 SInt / Int / DInt / LInt 有符号整数

> [!info] 核心结论
> 有符号整数可以表示正数、负数和0，常用于计数、位置偏差、补偿量、状态码和普通整数参数。

## ① 分类/公式/规则

> [!warning] 有符号整数范围
> **有符号整数 = 可正可负**

## ② 参考表

| 类型 | 位数 | 典型范围 |
|---|---:|---|
| **SInt** | 8位 | -128 ~ 127 |
| **Int** | 16位 | -32768 ~ 32767 |
| **DInt** | 32位 | -2147483648 ~ 2147483647 |
| **LInt** | 64位 | 超大整数范围 |
| **常用选择** | Int / DInt | 普通PLC项目最常见 |
| **可负参数** | 位置补偿、误差 | 适合有符号整数 |
| **大计数** | 累计产量 | 优先DInt或LInt |

## ③ 计算/选型示例

- 已知：  
  位置补偿范围为 -1000 ~ +1000。
- 公式：  
  **存在负数 → 选择有符号整数**
- 计算：  
  Int范围足够覆盖 -1000 ~ +1000。
- 结论：  
  该补偿量可用Int。

> [!tip] 记忆口诀
> 可能有负数，用有符号；数值很大，用DInt或LInt。

# 05 USInt / UInt / UDInt / ULInt 无符号整数

> [!info] 核心结论
> 无符号整数只能表示0和正数，适合计数、编号、长度、数量、地址、编码等不可能为负的数据。

## ① 分类/公式/规则

> [!warning] 无符号整数范围
> **无符号整数 = 只表示非负数**

## ② 参考表

| 类型 | 位数 | 典型范围 |
|---|---:|---|
| **USInt** | 8位 | 0 ~ 255 |
| **UInt** | 16位 | 0 ~ 65535 |
| **UDInt** | 32位 | 0 ~ 4294967295 |
| **ULInt** | 64位 | 超大非负整数 |
| **常见用途** | 数量、编号、长度 | 不需要负数 |
| **注意点** | 不能存负数 | 写入负数会异常 |
| **通讯场景** | 寄存器值 | 常见UInt/UDInt |

## ③ 计算/选型示例

- 已知：  
  产品编号范围为0~9999。
- 公式：  
  **非负编号 → UInt**
- 计算：  
  UInt最大65535，能覆盖0~9999。
- 结论：  
  产品编号可用UInt。

> [!tip] 记忆口诀
> 数量编号不为负，无符号整数很合适。

# 06 Real / LReal 浮点数

> [!info] 核心结论
> **Real和LReal** 用于带小数的工程量，例如温度、压力、速度、流量、位置、比例系数等。

## ① 分类/公式/规则

> [!warning] 浮点数规则
> **小数值、比例换算、工程量 → 用Real或LReal**

## ② 参考表

| 类型 | 位数 | 用途 |
|---|---:|---|
| **Real** | 32位 | 常规小数工程量 |
| **LReal** | 64位 | 高精度小数计算 |
| **温度** | 23.5℃ | Real |
| **压力** | 0.65MPa | Real |
| **速度** | 125.8mm/s | Real |
| **比例系数** | 0.001 | Real/LReal |
| **高精度计算** | 复杂算法 | LReal |

## ③ 计算/选型示例

- 已知：  
  压力传感器显示0.62MPa。
- 公式：  
  **带小数工程量 → Real**
- 计算：  
  Pressure_PV = 0.62。
- 结论：  
  压力、温度、速度等过程值常用Real。

> [!tip] 记忆口诀
> 有小数，用Real；要求高精度，用LReal。

# 07 Char / WChar 字符类型

> [!info] 核心结论
> **Char和WChar** 用于保存单个字符，常见于条码、通信字符、文本处理和字符串拆分。

## ① 分类/公式/规则

> [!warning] 字符规则
> **单个字符 → Char/WChar，多个字符 → String/WString**

## ② 参考表

| 类型 | 说明 |
|---|---|
| **Char** | 单字节字符 |
| **WChar** | 宽字符，适合更多字符集 |
| **单个字母** | A、B、C |
| **单个数字字符** | '1'、'2' |
| **通信字符** | 起始符、结束符 |
| **条码拆分** | 逐字符处理 |
| **注意点** | 字符'1'不等于数值1 |

## ③ 计算/选型示例

- 已知：  
  通讯协议中结束符为字符“#”。
- 公式：  
  **单个符号 → Char**
- 计算：  
  EndChar = '#'
- 结论：  
  协议控制符适合用Char表示。

> [!tip] 记忆口诀
> 一个字符用Char，一串字符用String。

# 08 String / WString 字符串类型

> [!info] 核心结论
> **String和WString** 用于保存文本，例如产品名称、批次号、条码、用户名称、配方名称和报警文本。

## ① 分类/公式/规则

> [!warning] 字符串公式
> **字符串 = 最大长度 + 当前长度 + 字符内容**

## ② 参考表

| 字符串用途 | 示例 |
|---|---|
| **产品名称** | Product_A |
| **批次号** | B20260507 |
| **条码** | 693xxxxxxxx |
| **配方名称** | Recipe_01 |
| **用户名** | Engineer01 |
| **工单号** | WO-2026-001 |
| **报警文本** | Servo Fault |

## ③ 计算/选型示例

- 已知：  
  条码最大长度为32位。
- 公式：  
  **字符串长度 ≥ 最大条码长度**
- 计算：  
  Barcode可设置为String[32]或更大。
- 结论：  
  字符串长度不足会导致文本被截断。

> [!tip] 记忆口诀
> 字符串先定最大长度，长度不够就会被截断。

# 09 Time 时间间隔类型

> [!info] 核心结论
> **Time** 表示时间间隔，常用于定时器设定值、延时、超时、保压时间、冷却时间等。

## ① 分类/公式/规则

> [!warning] Time规则
> **Time表示持续多久，不表示几点几分**

## ② 参考表

| Time用途 | 示例 |
|---|---|
| **延时启动** | T#3s |
| **动作超时** | T#5s |
| **保压时间** | T#2.5s |
| **冷却时间** | T#30s |
| **报警延时** | T#500ms |
| **周期触发** | T#1min |
| **注意点** | 与日期时间不同 |

## ③ 计算/选型示例

- 已知：  
  气缸伸出超过3秒未到位就报警。
- 公式：  
  **超时时间 = T#3s**
- 计算：  
  定时器累计到3秒仍未到位，则触发报警。
- 结论：  
  动作超时适合使用Time类型。

> [!tip] 记忆口诀
> 多久用Time，几点用Time_Of_Day或日期时间。

# 10 Date / Time_Of_Day / Date_And_Time / DTL

> [!info] 核心结论
> 日期时间类型用于保存日期、一天中的时间、完整时间戳，常用于报警记录、生产记录、报表和追溯。

## ① 分类/公式/规则

> [!warning] 日期时间选择
> **日期用Date，时刻用Time_Of_Day，完整时间戳用Date_And_Time或DTL**

## ② 参考表

| 类型 | 含义 |
|---|---|
| **Date** | 日期，例如2026-05-07 |
| **Time_Of_Day** | 一天中的时间，例如14:30:00 |
| **Date_And_Time** | 日期+时间 |
| **DTL** | 结构化日期时间 |
| **报警时间** | 适合完整时间戳 |
| **批次开始时间** | 适合完整时间戳 |
| **定时任务** | 可用时间相关类型 |

## ③ 计算/选型示例

- 已知：  
  需要记录报警发生的年月日时分秒。
- 公式：  
  **完整追溯时间 → Date_And_Time / DTL**
- 计算：  
  保存报警发生时间戳。
- 结论：  
  报警、生产、操作记录应使用完整时间戳。

> [!tip] 记忆口诀
> 只看日期用Date，只看时刻用TOD，追溯事件用完整时间戳。

# 11 Array 数组类型

> [!info] 核心结论
> **Array** 用于保存一组相同类型的数据，适合多工位、多产品、多报警、多参数的批量管理。

## ① 分类/公式/规则

> [!warning] 数组公式
> **数组容量 = 元素数量 × 单个元素大小**

## ② 参考表

| 数组用途 | 示例 |
|---|---|
| **报警位数组** | Alarm[1..100] |
| **产品参数数组** | RecipeSpeed[1..20] |
| **工位状态数组** | StationState[1..10] |
| **温度采样数组** | TempLog[1..60] |
| **电机状态数组** | MotorRun[1..50] |
| **计数数组** | Count[1..8] |
| **注意点** | 下标范围要清楚 |

## ③ 计算/选型示例

- 已知：  
  有20个Real参数，每个Real占4字节。
- 公式：  
  **容量 = 20 × 4**
- 计算：  
  容量 = **80字节**
- 结论：  
  数组适合批量管理同类参数，但要注意容量和下标。

> [!tip] 记忆口诀
> 同类型成一排，用数组最方便。

# 12 Struct 结构体类型

> [!info] 核心结论
> **Struct** 用于把不同类型的数据组合成一个整体，适合设备对象、工位对象、配方对象和报警对象。

## ① 分类/公式/规则

> [!warning] 结构体公式
> **结构体 = 多个不同类型变量组成的一个数据对象**

## ② 参考表

| 结构体对象 | 字段示例 |
|---|---|
| **电机结构体** | StartCmd、RunFB、Alarm、Current |
| **阀门结构体** | OpenCmd、CloseCmd、OpenFB、CloseFB |
| **气缸结构体** | ExtendCmd、RetractCmd、ExtendFB、Alarm |
| **配方结构体** | Speed、Pressure、Temp、Time |
| **报警结构体** | Active、Ack、Code、Level |
| **工位结构体** | Mode、State、Count、Alarm |
| **用户结构体** | Name、Level、LoginState |

## ③ 计算/选型示例

- 已知：  
  每台电机都需要命令、反馈、报警、电流。
- 公式：  
  **电机对象 = 命令 + 反馈 + 报警 + 电流**
- 计算：  
  可建立MotorStruct统一管理。
- 结论：  
  结构体让程序更清晰、更适合标准化。

> [!tip] 记忆口诀
> 一个设备一组数据，结构体最清楚。

# 13 UDT 用户自定义数据类型

> [!info] 核心结论
> **UDT** 是用户自定义的数据类型，可用于标准化设备、工位、配方、报警等复杂数据结构。

## ① 分类/公式/规则

> [!warning] UDT公式
> **UDT = 可重复使用的结构体模板**

## ② 参考表

| UDT类型 | 用途 |
|---|---|
| **UDT_Motor** | 电机标准数据 |
| **UDT_Valve** | 阀门标准数据 |
| **UDT_Cylinder** | 气缸标准数据 |
| **UDT_Recipe** | 配方参数 |
| **UDT_Alarm** | 报警信息 |
| **UDT_Station** | 工位数据 |
| **UDT_Comm** | 通讯状态 |

## ③ 计算/选型示例

- 已知：  
  项目有30台电机，数据结构完全相同。
- 公式：  
  **相同结构重复使用 → UDT**
- 计算：  
  建立UDT_Motor，再创建Motor[1..30]。
- 结论：  
  UDT适合标准化大型项目的数据结构。

> [!tip] 记忆口诀
> 结构要复用，就做UDT。

# 14 数据类型与地址大小

> [!info] 核心结论
> 不同数据类型占用不同字节数，地址分配时必须避免重叠和错位。

## ① 分类/公式/规则

> [!warning] 地址容量规则
> **变量占用空间 = 数据类型字节数**

## ② 参考表

| 类型 | 常见占用 |
|---|---:|
| **Bool** | 1位 |
| **Byte / SInt / USInt** | 1字节 |
| **Word / Int / UInt** | 2字节 |
| **DWord / DInt / UDInt / Real** | 4字节 |
| **LWord / LInt / ULInt / LReal** | 8字节 |
| **String** | 与最大长度有关 |
| **Array/Struct** | 各成员累加 |

## ③ 计算/选型示例

- 已知：  
  DB10.DBD0为Real，占用4字节。
- 公式：  
  **Real占用4字节**
- 计算：  
  DB10.DBD0占用DBB0~DBB3。
- 结论：  
  下一个变量不能再从DBB2开始，否则地址重叠。

> [!tip] 记忆口诀
> 类型决定大小，大小决定地址范围。

# 15 数据类型不匹配

> [!info] 核心结论
> 数据类型不匹配会导致数值异常、通讯错误、HMI显示乱码、计算错误和程序报警。

## ① 分类/公式/规则

> [!warning] 匹配公式
> **数据正确 = PLC类型 = HMI类型 = 通讯解析类型**

## ② 参考表

| 错误类型 | 后果 |
|---|---|
| **Real按Int读取** | 数值异常 |
| **DInt按Word读取** | 数据截断 |
| **有符号当无符号** | 负数变大数 |
| **String长度不够** | 文本截断 |
| **Word位映射错** | 报警文本错位 |
| **字节序错误** | 高低字节反了 |
| **地址错位** | 读到别的变量 |

## ③ 计算/选型示例

- 已知：  
  PLC压力值是Real，HMI变量设成Int。
- 公式：  
  **类型不匹配 → 解析错误**
- 计算：  
  HMI可能显示异常大数或乱码。
- 结论：  
  HMI变量类型必须与PLC变量类型一致。

> [!tip] 记忆口诀
> 地址对了还不够，数据类型也必须对。

# 16 数据类型选型规则

> [!info] 核心结论
> 数据类型选择要根据数据含义、范围、是否有小数、是否有负数、是否要文本、是否批量管理来决定。

## ① 分类/公式/规则

> [!warning] 选型公式
> **数据类型 = 数据范围 + 精度要求 + 工程含义**

## ② 参考表

| 数据需求 | 推荐类型 |
|---|---|
| **开关量** | Bool |
| **小整数** | Int |
| **大计数** | DInt / LInt |
| **非负编号** | UInt / UDInt |
| **带小数工程量** | Real |
| **高精度小数** | LReal |
| **状态字/报警字** | Word / DWord |
| **文本** | String |
| **一组同类数据** | Array |
| **设备对象数据** | Struct / UDT |

## ③ 计算/选型示例

- 已知：  
  累计产量最大可能达到2,000,000件。
- 公式：  
  **2,000,000 > Int最大32767**
- 计算：  
  Int不够，应选择DInt或UDInt。
- 结论：  
  大范围累计值不要用Int。

> [!tip] 记忆口诀
> 先看范围，再看小数，再看正负，最后看结构。

# 17 数据类型与HMI/通讯

> [!info] 核心结论
> HMI、SCADA、网关、Modbus、OPC UA等通讯系统读取PLC变量时，必须正确匹配数据类型、地址和字节顺序。

## ① 分类/公式/规则

> [!warning] 通讯解析公式
> **通讯数据正确 = 地址正确 + 类型正确 + 字节序正确**

## ② 参考表

| 通讯变量 | 注意 |
|---|---|
| **Bool** | 位地址要对应 |
| **Word** | 常用于Modbus寄存器 |
| **DInt** | 通常占2个Word |
| **Real** | 通常占2个Word，注意字节序 |
| **String** | 长度和编码要一致 |
| **Array** | 起始地址和长度要准确 |
| **Struct** | 成员偏移要清楚 |

## ③ 计算/选型示例

- 已知：  
  一个Real通过Modbus读取，占用2个寄存器。
- 公式：  
  **Real = 32位 = 2个16位寄存器**
- 计算：  
  如果高低字顺序反了，数值会异常。
- 结论：  
  通讯中Real和DInt特别要注意字节序/字顺序。

> [!tip] 记忆口诀
> 通讯读错数，常查类型、地址、字节序。

# 18 数据类型快速检查表

> [!info] 核心结论
> 数据类型检查的目标是确保变量含义清楚、范围足够、精度合适、通讯一致、HMI显示正确。

## ① 分类/公式/规则

> [!note] 十步检查法
> ① 确认变量是开关量、整数、小数、文本还是结构数据  
> ② 检查是否需要负数  
> ③ 检查最大值和最小值范围  
> ④ 检查是否需要小数和精度  
> ⑤ 检查PLC、HMI、通讯网关的数据类型是否一致  
> ⑥ 检查地址占用字节数是否正确  
> ⑦ 检查Word、DWord是否有位定义表  
> ⑧ 检查String最大长度是否足够  
> ⑨ 检查Array下标范围是否正确  
> ⑩ 上线前测试最小值、最大值、异常值和通讯显示值

> [!warning] 最终检查公式
> **合格数据类型 = 范围够 + 精度够 + 类型一致 + 地址不重叠 + 注释清楚**

## ② 参考表

| 检查项 | 目标 |
|---|---|
| **Bool** | 表示开关状态 |
| **Int/DInt** | 范围足够 |
| **Real** | 小数显示正确 |
| **Word** | 位定义清楚 |
| **String** | 长度足够 |
| **Array** | 下标正确 |
| **Struct/UDT** | 结构清晰 |
| **通讯类型** | 与外部系统一致 |

## ③ 计算/选型示例

- 已知：  
  一个速度设定范围为0~1200.0mm/s，需要1位小数。
- 公式：  
  **带小数 + 工程量 → Real**
- 计算：  
  Speed_SP = 1200.0可用Real保存。
- 结论：  
  速度、压力、温度等带小数参数通常优先选择Real。

> [!tip] 选型口诀
> 开关Bool，整数Int；大数DInt，小数Real；状态Word，文本String；一组Array，一对象Struct。
